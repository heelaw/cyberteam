# Skill - 用户运营专家

## 基本信息
- **名称**: user-operations
- **版本**: v1.0.0
- **类型**: 运营Skill
- **归属**: 增长BG/运营部

## 功能概述

用户全生命周期运营Skill，覆盖拉新、激活、留存、变现、推荐五大阶段。

## 触发条件

- 用户留存率下降
- 用户活跃度下降
- 用户流失率上升
- 新用户转化率低
- 需要设计用户激励体系

## 工作流程

### Phase 1: 用户分层
```
1. 数据提取 → 用户基础数据
2. RFM分析 → 用户价值分层
3. 行为分析 → 用户类型划分
4. 分层输出 → 分层运营策略
```

### Phase 2: 生命周期诊断
```
1. 漏斗分析 → 各阶段转化率
2. 流失点识别 → 关键流失节点
3. 原因归因 → 产品/运营/外部
4. 诊断报告 → 优化建议
```

### Phase 3: 激励体系设计
```
1. 目标行为 → 期望用户行为
2. 激励方式 → 物质/精神/社交
3. 成本测算 → ROI计算
4. 实施方案 → 小范围测试
```

## 核心方法

### 用户生命周期管理 (AARRR)
| 阶段 | 核心问题 | 运营策略 |
|------|----------|----------|
| Acquisition | 如何获取用户？ | 渠道优化、裂变活动 |
| Activation | 如何激活用户？ | 新手引导、关键动作 |
| Retention | 如何留存用户？ | 价值输出、分层运营 |
| Revenue | 如何变现用户？ | 会员体系、付费场景 |
| Referral | 如何推荐用户？ | 分享激励、口碑运营 |

### RFM用户分层
- **R (Recency)**: 最近一次消费时间
- **F (Frequency)**: 消费频率
- **M (Monetary)**: 消费金额

### 留存诊断四步法
1. 定义流失：连续N天无活跃
2. 归因分析：为什么流失？
3. 策略设计：如何召回？
4. ROI验证：召回是否值得？

## 输出

| 输出 | 格式 | 接收方 |
|------|------|--------|
| 用户分层报告 | Markdown | 运营总监 |
| 留存诊断报告 | Markdown | 运营总监 |
| 激励体系方案 | Markdown | 运营总监 |

## 使用示例

```
输入: 用户留存率下降，3日留存从35%降到28%
处理: 
  1. RFM分层分析
  2. 漏斗诊断
  3. 流失归因
  4. 激励方案
输出: 留存优化方案 + ROI预估
```

## 质量标准

- 分层数据必须来自真实用户行为
- 激励成本必须有ROI计算支撑
- 方案必须包含AB测试设计
