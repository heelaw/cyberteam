```yaml

  社交货币: 利用社交心理 → 传播扩散型
```

## 三、文案结构

### 3.1 标题公式

```yaml
痛点型:
  "最招人烦的5句话"
  "为什么你的内容没人看"

数字型:
  "10万+人的选择"
  "3个技巧让你的效率翻倍"

疑问型:
  "35岁还没升管理层失败吗"
  "AI写作真的能替代人工吗"

反差型:
  "月薪5千到3万他做对了3件事"
  "从每天3小时到30分钟"

对比型:
  "用AI写作后，我每天多出2小时"
```

### 3.2 开头结构

```yaml
黄金3秒原则:
  - 必须抓住注意力
  - 必须建立共鸣
  - 必须承诺价值

开头类型:
  场景代入:
    "深夜2点，你还在对着空白文档发呆..."

  痛点共鸣:
    "每天不知道写什么，这种感觉你一定不陌生..."

  数据钩子:
    "10万+创作者都在用的AI写作工具..."

  金句开头:
    "效率不是做更多，而是用更好的工具..."
```

### 3.3 正文结构

```yaml
总分总结构:
  观点 → 论证 → 总结

问题解决结构:
  问题 → 原因 → 方案

对比论证结构:
  错误做法 → 正确做法 → 效果对比

故事化结构:
  背景 → 冲突 → 解决 → 启发
```

### 3.4 结尾设计

```yaml
行动号召:
  - 明确告诉用户下一步做什么
  - 降低行动门槛
  - 制造紧迫感

金句提炼:
  - 用一句话总结核心观点
  - 便于记忆和传播

互动引导:
  - 设计评论区话题
  - 引导用户分享
  - 鼓励用户关注
```

## 四、写作技巧

### 4.1 情感触发词

```yaml
焦虑词:
  "深夜"、"发呆"、"不知道"、"赶不上"

 释然词:
  "原来如此"、"原来可以"、"轻松搞定"

  激励词:
  "每天进步"、"持续成长"、"找到方法"

  紧迫词:
  "现在就试"、"限时免费"、"错过不再"
```

### 4.2 数据使用

```yaml
数据增强说服力:
  "10万+用户的选择"
  "效率提升300%"
  "每天节省2.5小时"

数据来源:
  - 官方数据
  - 第三方报告
  - 用户案例
  - A/B测试结果
```

### 4.3 对比技巧

```yaml
前后对比:
  "以前：3小时 → 现在：30分钟"

正误对比:
  "错误做法：XXX → 正确做法：XXX"

 优劣对比:
  "工具A：贵而慢 → 工具B：便宜快"
```

## 五、质量标准

### 5.1 文案评分

| 维度 | 权重 | 评分标准 |
|------|------|----------|
| **吸引力** | 25% | 标题是否吸引点击 |
| **共鸣度** | 25% | 是否产生情感共鸣 |
| **逻辑性** | 20% | 论证是否严密 |
| **可读性** | 15% | 是否易读易懂 |
| **传播性** | 15% | 是否易于分享 |

### 5.2 通过标准

```yaml
必须满足:
  - 总分 ≥ 80 分
  - 吸引力 ≥ 8 分
  - 共鸣度 ≥ 8 分
  - 无明显逻辑漏洞

优秀标准:
  - 总分 ≥ 90 分
  - 所有维度 ≥ 8 分
  - 有独特创意点
  - 有传播金句
```

## 七、创作流程

```yaml
标准创作流程:

Step 1: 理解需求 (2分钟)
  - 阅读用户画像
  - 理解创意方向
  - 明确平台特性

Step 2: 标题创作 (3分钟)
  - 生成多个标题备选
  - 选择最佳标题
  - 优化标题吸引力

Step 3: 开头设计 (3分钟)
  - 选择开头类型
  - 设计黄金3秒
  - 建立情感连接

Step 4: 正文撰写 (7分钟)
  - 搭建文章结构
  - 填充具体内容
  - 优化表达方式

Step 5: 结尾设计 (2分钟)
  - 设计行动号召
  - 提炼金句
  - 引导互动

Step 6: 整体优化 (3分钟)
  - 检查逻辑性
  - 优化可读性
  - 增强传播性

总计: 20分钟/篇
```

## 八、常见错误

### 8.1 标题错误

```yaml
❌ 错误示例:
  - "AI写作工具介绍" (太无聊)
  - "一个好用的工具" (太模糊)
  - "试试这个吧" (太随意)

✅ 正确示例:
  - "用AI写作后，我每天多出2小时"
  - "3个技巧让你的写作效率翻倍"
  - "从每天3小时到30分钟，我是怎么做到的"
```

### 8.2 内容错误

```yaml
❌ 常见问题:
  - 自说自话，不关心用户
  - 罗列功能，没有情感连接
  - 逻辑混乱，不知所云
  - 没有数据支撑，缺乏可信度

✅ 改进方向:
  - 从用户视角出发
  - 建立情感连接
  - 逻辑清晰严密
  - 数据支撑观点
```
