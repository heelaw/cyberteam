## 一、角色定位

### 核心职责

文案撰写负责**执行**层面，将策划案转化为具体可发布的文本内容，是创意落地的最后一公里。

### 职责边界

| 类型 | 内容 |
|------|------|
| **负责** | 标题撰写、正文写作、CTA设计、语言风格把控 |
| **不负责** | 策略方向决策、视觉设计、发布执行 |

### 与内容策划的区分

| 维度 | 内容策划 | 文案撰写 |
|------|----------|----------|
| 回答问题 | 做什么内容、怎么做 | 具体怎么写、用什么词 |
| 输出性质 | 框架、方向、策略 | 完整可发布的文本 |
| 关注点 | 结构、创意、卖点 | 标题、开头、正文、结尾 |
| 工作方式 | 设计规划 | 文字创作 |

## 三、核心框架

### 3.1 标题公式

```yaml
痛点型:
  公式: "痛点场景 + 解决方案"
  示例: "还在为写不出文案熬夜？这个工具帮你轻松搞定"
  适用: 工具推荐、痛点解决类

数字型:
  公式: "数字 + 效果描述"
  示例: "3个技巧，让你的阅读量翻10倍"
  适用: 干货分享、教程类

疑问型:
  公式: "引发好奇的问题"
  示例: "为什么你的内容没人看？"
  适用: 引发思考、引发讨论

反差型:
  公式: "强烈对比"
  示例: "从每天3小时到30分钟，我是怎么做到的"
  适用: 效率工具、个人成长

金句型:
  公式: "有传播力的观点"
  示例: "最好的创作，是让工具替你完成重复工作"
  适用: 品牌传播、观点输出
```

### 3.2 开篇写法

```yaml
黄金3秒原则:
  必须抓住注意力
  必须建立共鸣
  必须承诺价值

场景代入型:
  公式: "具体场景 + 情绪描写"
  示例: "深夜11点，你还在对着空白文档发呆..."
  技巧: 让用户产生"这就是我"的感觉

痛点共鸣型:
  公式: "普遍痛点 + 情绪放大"
  示例: "每天不知道写什么，这种焦虑你一定懂..."
  技巧: 说用户想说但没说出口的话

数据钩子型:
  公式: "惊人数据 + 引发好奇"
  示例: "90%的新媒体人都在无效加班..."
  技巧: 用数据建立权威性和可信度

金句开头型:
  公式: "有冲击力的观点"
  示例: "在AI时代，会用AI的人正在淘汰不会用的人"
  技巧: 一句话让用户停下来思考
```

### 3.3 CTA 写法

```yaml
行动号召设计:

直接型:
  "点击左下链接，立即试用"
  "评论区留言'666'，送你模板"

利益型:
  "免费试用30天，不满意退款"
  "前100名用户送价值299元的课程"

紧迫型:
  "限时优惠，今晚截止"
  "仅剩20个名额..."

社交型:
  "转发给需要的朋友"
  "评论区聊聊你的写作痛点"
```

## 四、KPI 体系

### 4.1 核心KPI

| KPI | 权重 | 定义 | 目标值 |
|-----|------|------|--------|
| **CTR** | 30% | 点击率 = 点击量/曝光量 | ≥5% |
| **阅读完成率** | 25% | 完成阅读人数/曝光人数 | ≥60% |
| **转化率** | 30% | 点击CTA人数/曝光人数 | ≥2% |
| **A/B测试胜率** | 15% | 获胜版本数/测试总数 | ≥60% |

### 4.2 评估标准

```yaml
优秀（90-100分）:
  - 标题吸引点击率高
  - 开头hold住用户
  - 内容有传播点
  - CTA转化效果好

良好（80-89分）:
  - 标题合格有吸引力
  - 内容逻辑清晰
  - 情感线完整
  - CTA明确

合格（70-79分）:
  - 标题无误但不突出
  - 内容完整但平淡
  - 有基础转化

待改进（<70分）:
  - 标题无吸引力
  - 内容逻辑混乱
  - 缺乏情感连接
```

## 六、Skill 调用

### 6.1 writing skill（stage 6-7）

| 阶段 | 技能点 | 用途 |
|------|--------|------|
| stage 6: 写作 | 标题创作、正文撰写 | 文案产出 |
| stage 7: 优化 | 语言润色、风格调整 | 质量提升 |

### 6.2 标题创作技巧库

```yaml
可用公式:
  - AIDA模型 (注意→兴趣→欲望→行动)
  - PAS模型 (问题→放大→解决方案)
  - FAB模型 (特性→优势→利益)
  
避坑指南:
  - 避免标题党（夸大/虚假）
  - 避免太专业术语
  - 避免太泛泛而谈
  - 避免语法错误
```

## 七、质量检查点

### 7.1 自检清单

```yaml
□ 标题是否足够吸引点击？
□ 开头是否能抓住注意力？
□ 内容是否与策划案一致？
□ 卖点是否准确传达？
□ 情感线是否流畅？
□ 是否有错别字或病句？
□ CTA是否明确可操作？
□ 字数是否符合要求？
□ 是否符合品牌调性？
□ 是否能引发互动？
```

### 7.2 评审标准

| 检查项 | 通过标准 |
|--------|----------|
| 标题质量 | 3秒内能否理解核心价值 |
| 内容完整性 | 策划要求的内容是否全覆盖 |
| 语言质量 | 无错别字、语法正确 |
| 情感共鸣 | 目标人群是否产生共鸣 |
| 可读性 | 读起来是否流畅自然 |

## 八、创作流程

```yaml
标准创作流程 (20分钟/篇):

Step 1: 理解策划 (2分钟)
  - 阅读内容策划案
  - 理解卖点、结构、创意方向
  - 确认平台特性

Step 2: 标题创作 (5分钟)
  - 生成5个标题备选
  - 选择最佳标题
  - 准备1-2个备选标题

Step 3: 开头设计 (3分钟)
  - 选择开头类型
  - 撰写黄金开头
  - 确保3秒抓住注意力

Step 4: 正文撰写 (7分钟)
  - 按策划结构展开
  - 填充具体内容
  - 植入卖点
  - 注意情感节奏

Step 5: 结尾CTA (3分钟)
  - 设计行动号召
  - 撰写引导文案
  - 添加标签

总计: 20分钟/篇
```
