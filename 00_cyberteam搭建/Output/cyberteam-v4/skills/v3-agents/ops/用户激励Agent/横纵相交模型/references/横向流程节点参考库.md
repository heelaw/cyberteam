# 常见转化流程横向节点库

## 一、电商购物类

### 典型流程（6个节点）

```
访问 → 浏览 → 加购 → 结算 → 支付 → 完成
```

### 各节点标准行为

| 节点 | 关键行为 | 常用优化方向 |
|------|----------|--------------|
| 访问 | 入口选择/首屏停留 | 入口优化/首屏内容 |
| 浏览 | 商品点击/搜索/分类 | 商品推荐/搜索优化 |
| 加购 | 加入购物车/立即购买 | 加购激励/购物车提醒 |
| 结算 | 地址选择/优惠使用 | 流程简化/优惠展示 |
| 支付 | 支付方式/支付确认 | 支付便捷/信任建设 |
| 完成 | 订单确认/分享 | 订单激励/裂变引导 |

## 八、自定义流程设计指南

### 节点设计原则

1. **完整性**: 覆盖用户完成目标的完整路径
2. **独立性**: 每个节点有明确的开始和结束
3. **可测量**: 每个节点有可追踪的转化数据
4. **适度性**: 节点数量控制在5-8个最佳

### 节点命名规范

- 使用动词: 访问/浏览/选择/购买
- 明确具体: "加入购物车"而非"转化"
- 统一格式: [动词]+[名词]

### 节点边界定义

| 原则 | 说明 |
|------|------|
| 进入标准 | 满足什么条件算进入这个节点 |
| 成功标准 | 完成什么算这个节点转化成功 |
| 流失标准 | 什么情况算从这个节点流失 |
