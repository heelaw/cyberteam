# 复盘报告模板

## 复盘框架

### 标准复盘模型 (4R)

| 阶段 | 问题 | 输出 |
|------|------|------|
| **Result (结果)** | 实际发生了什么？ | 结果对比表 |
| **Review (回顾)** | 目标是什么？为什么？ | 目标回顾 |
| **Reason (原因)** | 为什么成功/失败？ | 原因分析 |
| **Result (规律)** | 我们学到了什么？ | 经验沉淀 |

## 复盘会议流程

### 会议准备

```markdown
## 复盘会议准备

### 会前 (提前2天)
- [ ] 数据汇总完成
- [ ] 初稿提前发出
- [ ] 参会人预览

### 会中 (1-2小时)

| 环节 | 时间 | 内容 | 主持人 |
|------|------|------|--------|
| 开场 | 5分钟 | 会议目标 | 负责人 |
| 结果回顾 | 15分钟 | 目标完成情况 | 数据负责人 |
| 原因分析 | 30分钟 | 深度讨论 | 全体 |
| 经验沉淀 | 20分钟 | 提炼经验 | 全体 |
| 下期规划 | 20分钟 | 目标&行动 | 负责人 |
| 总结 | 10分钟 | 关键决策 | 负责人 |

### 会后 (1天内)
- [ ] 复盘报告定稿
- [ ] 行动项发出
- [ ] 下期规划确认
```

## 复盘检查清单

### 复盘前

```
[ ] 1. 数据准备
    - [ ] 所有指标数据完整
    - [ ] 数据来源可靠
    - [ ] 计算口径一致

[ ] 2. 材料准备
    - [ ] 初稿提前发出
    - [ ] 异常数据标注
    - [ ] 问题清单准备

[ ] 3. 人员准备
    - [ ] 关键人确认参加
    - [ ] 主持人确定
    - [ ] 记录人指定
```

### 复盘中

```
[ ] 1. 结果透明
    - [ ] 数据如实呈现
    - [ ] 好坏都讲
    - [ ] 避免归因主观

[ ] 2. 分析深入
    - [ ] 追问5个为什么
    - [ ] 区分直接/根本原因
    - [ ] 外部/内部因素

[ ] 3. 经验可靠
    - [ ] 经验有数据支撑
    - [ ] 可复制性评估
    - [ ] 明确适用场景
```

### 复盘后

```
[ ] 1. 文档完整
    - [ ] 报告及时发出
    - [ ] 行动项明确
    - [ ] 责任人确认

[ ] 2. 跟踪到位
    - [ ] 行动项入系统
    - [ ] 下次复盘追踪
    - [ ] 经验入库
```

## 复盘案例

### 案例: Q1复盘摘要

```markdown
# Q1复盘摘要

## 核心结论
Q1北极星指标完成度95%，基本达标，但在用户留存指标上存在明显短板。

## 关键数据

| 指标 | Q4 | Q1目标 | Q1实际 | 环比 |
|------|-----|--------|--------|------|
| 作业提交率 | 62% | 68% | 65% | +3% |
| 完播率 | 70% | 75% | 78% | +8% |
| 续费率 | 65% | 70% | 68% | +3% |

## 主要亮点
1. 完播率超额完成，课程质量提升措施有效
2. 催交机制优化，提交率稳步提升

## 主要不足
1. 续费率未达目标，老用户活跃度下降
2. 跨部门协作效率需提升

## Q2重点
1. 提升续费率至72% (优先)
2. 优化老用户活跃体系
3. 建立跨部门协作SOP
```
