# 流程五符号使用规范

## 符号定义

| 符号 | 名称 | 含义 | 使用场景 |
|------|------|------|----------|
| ○ | 开始/结束 | 流程的起点或终点 | 每个流程必须有1个开始和1个结束 |
| □ | 操作步骤 | 一个具体的可执行动作 | 流程中的每个步骤 |
| ◇ | 判断节点 | 需要做出是/否或多个选择的点 | 条件分支处 |
| → | 箭头 | 指示流程的流向方向 | 连接各步骤 |
| ▽ | 数据/文档 | 输入或输出的数据/文档 | 信息传递处 |

## 符号组合规范

### 正确组合

```
○ → □ → □ → ◇是 → □ → ○
          ↓否
         □ → ○
```

### 常见错误

1. **判断节点后无否路径**
   - 错误：◇ → □（只有一种后续）
   - 正确：◇ → □（是） / □（否）

2. **步骤描述模糊**
   - 错误：□ 优化体验
   - 正确：□ 缩短页面加载时间至2秒内

3. **缺少数据标注**
   - 错误：□ 用户注册
   - 正确：□ 用户注册 ▽注册信息→CRM系统

## 流程图绘制步骤

1. **明确起点**：确定流程的触发条件
2. **列出步骤**：按时间顺序列出所有步骤
3. **识别判断**：标出需要判断的节点
4. **连接流向**：用箭头连接各步骤
5. **标注数据**：标注输入输出的数据
6. **标注责任**：每个步骤标注责任人

## 示例：用户下单流程

```
    ○ 开始
    ↓
□ 查看商品详情
    ↓
◇ 加入购物车?
    ↓是        ↓否
□ 去购物车   □ 继续浏览
    ↓            ↓
□ 确认订单   →→→→→↓
    ↓
□ 选择地址
    ↓
□ 选择支付方式
    ↓
◇ 支付成功?
    ↓是        ↓否
○ 订单完成   □ 重新支付
```

## 符号使用检查清单

- [ ] 流程有明确的开始○和结束○
- [ ] 每个□都是一个可执行的动作
- [ ] 每个◇都有是/否两种后续路径
- [ ] 箭头流向清晰，无交叉
- [ ] 关键的▽数据/文档已标注
- [ ] 每个步骤有明确责任人
