# LTV与CAC计算和优化

## 一、核心概念

### LTV (Life Time Value)

**定义**：用户生命周期价值，即一个用户从开始使用产品到最终流失，为企业带来的总收入或利润。

**计算公式**：
```
LTV = (月购买频次 × 客单价 × 毛利率) / 月流失率
```

**简化公式**：
```
LTV = ARPU × 用户生命周期
```

### CAC (Customer Acquisition Cost)

**定义**：获客成本，即获取一个新用户所花费的成本。

**计算公式**：
```
CAC = 总营销费用 / 新增用户数
```

**营销费用包括**：
- 广告投放费用
- 渠道合作费用
- 营销人员成本
- 营销工具费用

### LTV/CAC 比值

**含义**：衡量用户价值和获客成本的健康度

**评估标准**：
- **LTV/CAC > 3**：健康状态，用户价值远大于获客成本
- **LTV/CAC = 2-3**：可接受，但需要优化
- **LTV/CAC < 2**：不健康，获客成本过高
- **LTV/CAC < 1**：危险状态，亏损运营

**回收期**：
```
回收期 = CAC / (ARPU × 毛利率)
```

理想回收期：12个月内

## 二、LTV计算方法

### 方法1：基础计算法

**公式**：
```
LTV = ARPU × 用户生命周期（月）
```

**示例**：
- ARPU：50元/月
- 平均用户生命周期：24个月
- LTV = 50 × 24 = 1200元

### 方法2： cohort法

**步骤**：
1. 按获取时间将用户分组（cohort）
2. 计算每个cohort的累积收入
3. 观察收入曲线趋于稳定的点
4. 该点的累积收入即为LTV

**优点**：
- 考虑了用户留存变化
- 更加准确
- 可以观察不同时期用户差异

**示例**：
```
某月份获取的用户cohort：
Month 1: 10,000元
Month 2: 18,000元
Month 3: 24,000元
Month 4: 28,000元
Month 5: 30,000元
Month 6: 31,000元（趋于稳定）
LTV = 31,000元 / 用户数 = 310元/人
```

### 方法3：预测模型法

**使用公式**：
```
LTV = (月购买频次 × 客单价 × 毛利率) / 月流失率
```

**示例**：
- 月购买频次：2次
- 客单价：100元
- 毛利率：40%
- 月流失率：5%

```
LTV = (2 × 100 × 0.4) / 0.05
    = 80 / 0.05
    = 1600元
```

## 三、CAC计算方法

### 方法1：简单平均法

**公式**：
```
CAC = 总营销费用 / 新增用户数
```

**示例**：
- 月度营销费用：100,000元
- 月度新增用户：5,000人
- CAC = 100,000 / 5,000 = 20元/人

### 方法2：渠道分层计算

**为每个渠道单独计算CAC**：

| 渠道 | 营销费用 | 新增用户 | CAC |
|------|----------|----------|-----|
| 信息流广告 | 50,000 | 1,000 | 50元 |
| 搜索广告 | 30,000 | 2,000 | 15元 |
| 社交媒体 | 10,000 | 500 | 20元 |
| SEO | 5,000 | 1,500 | 3.3元 |
| 总计 | 95,000 | 5,000 | 19元 |

**用途**：对比不同渠道效率，优化投放策略

### 方法3：全成本计算法

**包含所有相关成本**：
- 直接营销费用
- 营销人员薪资
- 营销工具费用
- 创意制作费用
- 渠道合作费用

**公式**：
```
CAC = 全部获客成本 / 新增付费用户数
```

**注意**：对于付费用户，分母用新增付费用户数

## 四、LTV与CAC优化

### LTV优化策略

**1. 延长用户生命周期**
- 提升留存率
- 防止用户流失
- 用户召回

**2. 提升ARPU**
- 提高客单价
- 增加购买频次
- 交叉销售和向上销售
- 付费转化

**3. 提升毛利率**
- 优化成本结构
- 提升高毛利产品占比
- 降低运营成本

### CAC优化策略

**1. 优化渠道投放**
- 停止高CAC低ROI的渠道
- 增加低CAC高ROI的渠道投放
- A/B测试优化广告素材

**2. 提升转化率**
- 优化落地页
- 优化注册流程
- 优化首次体验

**3. 自然获客**
- 内容营销
- 社交媒体营销
- SEO优化
- 用户推荐裂变

**4. 精准投放**
- 用户画像精准定位
- lookalike人群
- 重投放

## 五、不同阶段的LTV/CAC策略

### 引入期（初创期）

**特点**：
- LTV不稳定
- CAC较高
- 需要快速获取用户

**策略**：
- LTV/CAC可以接受1-2
- 重点在快速获取用户
- 验证商业模式

### 成长期

**特点**：
- LTT逐步稳定
- CAC开始优化
- 需要平衡增长和盈利

**策略**：
- LTV/CAC目标2-3
- 优化渠道投放
- 提升LTV

### 成熟期

**特点**：
- LTV稳定
- CAC优化到位
- 追求盈利

**策略**：
- LTV/CAC目标>3
- 精细化运营
- 最大化利润

## 六、数据分析方法

### Cohort分析

**目的**：分析不同时期用户的LTV变化

**方法**：
1. 按月份或季度分组
2. 计算每个cohort的累积收入
3. 绘制累积收入曲线
4. 对比不同cohort的差异

**用途**：
- 观察用户质量变化趋势
- 评估运营效果
- 预测未来LTV

### 渠道分析

**目的**：分析不同渠道的用户价值

**方法**：
1. 为每个渠道计算LTV和CAC
2. 计算LTV/CAC比值
3. 计算回收期

**示例**：

| 渠道 | CAC | LTV | LTV/CAC | 回收期 | 评估 |
|------|-----|-----|---------|--------|------|
| 搜索广告 | 20 | 100 | 5.0 | 2.4月 | 优秀 |
| 信息流 | 50 | 150 | 3.0 | 4个月 | 良好 |
| 社交媒体 | 30 | 60 | 2.0 | 6个月 | 一般 |
| SEO | 5 | 80 | 16.0 | 0.75月 | 极好 |

### 留存分析

**留存对LTV的影响**：

| 留存率 | 生命周期(月) | ARPU=50元时LTV |
|--------|--------------|----------------|
| 10% | 10 | 500元 |
| 8% | 12.5 | 625元 |
| 5% | 20 | 1000元 |
| 3% | 33.3 | 1665元 |

**启示**：提升留存率对LTV影响巨大

## 七、常见问题

### Q1: LTV计算不准确怎么办？

**原因**：
- 用户行为不稳定
- 留存率变化大
- 产品快速迭代

**解决方案**：
1. 使用cohort法计算，更准确
2. 定期更新LTV预测模型
3. 用多个方法交叉验证
4. 设置LTV的置信区间

### Q2: LTV/CAC太低怎么办？

**短期策略**：
1. 停止高CAC低ROI渠道投放
2. 集中资源在高ROI渠道
3. 优化广告投放效率

**长期策略**：
1. 提升产品价值
2. 提升留存率延长生命周期
3. 提升ARPU

### Q3: 如何平衡增长和盈利？

**阶段策略**：
- **融资期**：可以接受LTV/CAC 1-2，追求增长
- **盈亏平衡期**：LTV/CAC 2-3，平衡增长和盈利
- **盈利期**：LTV/CAC > 3，追求利润

**判断依据**：
- 公司资金状况
- 市场竞争情况
- 融资环境

### Q4: 回收期太长怎么办？

**解决方案**：
1. 提升ARPU（推荐付费、提升客单价）
2. 提升前期收入（引导早期付费）
3. 降低CAC（优化获客效率）
4. 提升留存（延长生命周期）

### SQL查询示例

```sql
-- 计算LTV（cohort法）
WITH cohort_users AS (
    SELECT
        user_id,
        DATE_TRUNC('month', first_order_date) AS cohort_month
    FROM users
),
cohort_revenue AS (
    SELECT
        c.cohort_month,
        DATEDIFF(MONTH, c.cohort_month, DATE_TRUNC('month', o.order_date)) AS month_number,
        SUM(o.amount) AS revenue
    FROM cohort_users c
    JOIN orders o ON c.user_id = o.user_id
    GROUP BY c.cohort_month, month_number
)
SELECT
    cohort_month,
    SUM(revenue) OVER (PARTITION BY cohort_month ORDER BY month_number) AS cumulative_revenue,
    COUNT(DISTINCT user_id) OVER (PARTITION BY cohort_month) AS cohort_size,
    SUM(revenue) OVER (PARTITION BY cohort_month ORDER BY month_number) / COUNT(DISTINCT user_id) OVER (PARTITION BY cohort_month) AS LTV
FROM cohort_revenue;
```
