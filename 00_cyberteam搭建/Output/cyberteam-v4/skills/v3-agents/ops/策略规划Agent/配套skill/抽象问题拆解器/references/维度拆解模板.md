# 维度拆解模板

## 模板说明
用于将抽象概念拆解为具体的可分析维度。

## 模板内容

```
# [概念名称] - 维度拆解

## 概念定义回顾
- **操作性定义**：[1-2句话]
- **核心关键词**：[列出3-5个关键词]

## 维度发散（第一阶段）

### 方法1：同行维度
[列出与概念同级别的其他维度]
- 维度A
- 维度B
- 维度C

### 方法2：反向维度
[列出概念的反面或对立面]
- 反向维度1
- 反向维度2

### 方法3：经典框架套用
[尝试套用PEST、SWOT等框架]
- P：[政治]维度
- E：[经济]维度
- S：[社会]维度
- T：[技术]维度

### 方法4：用户旅程维度
[从用户旅程的各个环节思考]
- 认知阶段：[相关维度]
- 考虑阶段：[相关维度]
- 购买阶段：[相关维度]
- 使用阶段：[相关维度]
- 推荐阶段：[相关维度]

### 方法5：系统构成维度
[从系统组成部分思考]
- 输入：[相关维度]
- 过程：[相关维度]
- 输出：[相关维度]
```

## 使用说明
1. 先回顾概念定义
2. 用多种方法发散候选维度
3. 按MECE原则收敛
4. 为每个维度给出精确定义
5. 验证互斥性和全面性
6. 输出维度矩阵
