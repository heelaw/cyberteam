# 维度验证清单模板

## 模板说明
用于验证维度拆解的完整性和正确性。

## 模板内容

```
# [概念名称] - 维度验证清单

## 基础信息
- **概念操作性定义**：[定义]
- **维度数量**：[N个]
- **验证日期**：[日期]
- **验证人**：[姓名]

## 验证检查表

### 1. 定义验证

- [ ] **定义清晰性**
  - [ ] 操作性定义能被完全不理解概念的人理解
  - [ ] 定义不包含概念本身的词汇（无循环定义）
  - [ ] 定义不包含比喻或模糊比喻

- [ ] **定义稳定性**
  - [ ] 同一概念在不同场景下定义保持一致
  - [ ] 定义不随时间轻易变化

### 2. 维度独立性验证

- [ ] **维度两两互斥**
  - [ ] 维度1与维度2之间无重叠
  - [ ] 维度1与维度3之间无重叠
  - [ ] 维度2与维度3之间无重叠
  - [ ] [继续检查所有维度对...]

- [ ] **维度分类标准一致**
  - [ ] 所有维度使用相同的分类标准
  - [ ] 分类标准不模糊

### 3. 维度全面性验证

- [ ] **概念覆盖完整**
  - [ ] 主要维度已被识别（覆盖90%以上概念含义）
  - [ ] 无重大维度遗漏
  - [ ] 边缘情况有处理方式

- [ ] **维度数量合理**
  - [ ] 维度数量在3-7个之间（符合7±2原则）
  - [ ] 不过多（避免过度拆解）
  - [ ] 不过少（避免拆解不充分）

### 4. 可测量性验证

| 维度 | 是否有度量方法 | 度量方法是否可行 | 是否有判断标准 |
|------|----------------|-----------------|----------------|
| 维度1 | [是/否] | [是/否] | [是/否] |
| 维度2 | [是/否] | [是/否] | [是/否] |
| 维度3 | [是/否] | [是/否] | [是/否] |

- [ ] **逐项检查**
  - [ ] 维度1的度量方法：[方法]，可行性：[评估]
  - [ ] 维度2的度量方法：[方法]，可行性：[评估]
  - [ ] 维度3的度量方法：[方法]，可行性：[评估]

### 5. 维度层次验证

- [ ] **拆解深度适中**
  - [ ] 拆解到可操作的层级（通常2-3层）
  - [ ] 不在某一维度上过度深挖
  - [ ] 各维度拆解深度基本一致

- [ ] **层次逻辑清晰**
  - [ ] 维度与概念之间的层次关系明确
  - [ ] 下一层级对上一层级的支撑关系清晰

### 6. 实用性验证

- [ ] **实际可应用**
  - [ ] 维度在实际分析中可操作
  - [ ] 数据/信息在实践中可获取
  - [ ] 不存在理论存在但实践无法获取的维度

- [ ] **判断可执行**
  - [ ] 判断标准明确可执行
  - [ ] 不同评估者对同一判断标准结果一致
```

## 使用说明
1. 逐项对照检查清单验证
2. 特别关注互斥性和全面性
3. 用还原测试验证拆分是否有效
4. 记录发现的问题
5. 根据结论决定是否需要修改
