# UE模型拆解法

## Skill概述

UE模型（Unit Economics）拆解法帮助用户理解并计算单个用户/单元的经济价值，是评估业务可持续性的核心方法。

## 输入格式

```json
{
  "type": "object",
  "required": ["basic_metrics"],
  "properties": {
    "basic_metrics": {
      "type": "object",
      "properties": {
        "arpu": {"type": "number", "description": "用户平均收入（元/月）"},
        "gross_margin": {"type": "number", "description": "毛利率（%）"},
        "retention_rate": {"type": "number", "description": "留存率（%）"},
        "lifespan": {"type": "number", "description": "用户生命周期（月）"},
        "marketing_spend": {"type": "number", "description": "营销总费用（元）"},
        "new_users": {"type": "number", "description": "新增用户数"}
      }
    }
  }
}
```

## 输出格式

```json
{
  "type": "object",
  "properties": {
    "ltv": {"type": "number", "description": "用户生命周期价值（元）"},
    "cac": {"type": "number", "description": "用户获取成本（元）"},
    "roi": {"type": "number", "description": "投入产出比"},
    "payback_period": {"type": "number", "description": "回本周期（月）"},

  }
}
```

## 使用示例

### 示例1：在线教育公司UE分析

**输入**:
```
ARPU: 500元/月
毛利率: 70%
年留存率: 80%
用户生命周期: 24个月
营销费用: 100万元
新增用户: 100人
```

**输出**:
```
UE模型分析结果：

LTV = 500 × 0.7 × 24 = 8,400元
CAC = 1,000,000 ÷ 100 = 10,000元
ROI = 8,400 ÷ 10,000 = 0.84

健康度：⚠️ 亏损（ROI < 1）
回本周期：约28个月

问题诊断：
- LTV偏低，需提升ARPU或延长生命周期
- CAC过高，需优化获客渠道

优化建议：
1. 提升客单价：推出高价课程包
2. 增加复购：引入续费优惠机制
3. 降低CAC：优化投放渠道组合
4. 提高留存：增加用户粘性功能
```

## 错误处理

| 错误场景 | 处理方式 |
|----------|----------|
| 数据缺失 | 说明需要哪些数据，给出行业参考值供估算 |
| 数据异常（如ROI<0.1） | 标注数据可能有问题，建议核实后重新计算 |
| 行业差异 | 说明不同行业的LTV/CAC基准不同，参考同行数据 |
| 用户群体差异 | 指出不同用户群体的UE可能差异很大，建议分群计算 |

## 独特个性

本Skill采用**"计算-判断-建议"三步闭环法**，不仅计算指标，更给出健康度诊断和可执行的优化建议，特别适合业务负责人快速判断UE模型是否健康并找到优化方向。

## 解决的问题

- 不知道单个用户能为公司创造多少价值
- 不清楚获取一个用户需要花多少钱
- 无法判断业务是否可持续

## 核心概念

### LTV（用户生命周期价值）

用户在整个生命周期内为公司创造的总价值。

**计算公式**：
```
LTV = 客单价 × 复购次数 × 毛利率
```

或更精确的公式：
```
LTV = (ARPU × 毛利率) × 用户生命周期(月)
```

### CAC（用户获取成本）

获取一个用户需要花费的成本。

**计算公式**：
```
CAC = 总营销费用 ÷ 新增用户数
```

### ROI（投资回报率）

投入产出比。

**计算公式**：
```
ROI = LTV ÷ CAC
```

- ROI > 3：业务健康，可持续扩张
- ROI 1-3：业务可行，需优化
- ROI < 1：亏损，需立即调整

### 盈亏平衡点

多长时间可以收回获取用户的成本。

**计算公式**：
```
回本周期 = CAC ÷ (ARPU × 毛利率 - 运营成本)
```

## 工作流程

### Step 1：收集数据

需要收集以下数据：
- 客单价 / ARPU（用户平均收入）
- 复购率 / 复购次数
- 用户生命周期（月）
- 毛利率
- 营销总费用
- 新增用户数

### Step 2：计算LTV

根据数据计算用户生命周期价值。

### Step 3：计算CAC

计算用户获取成本。

### Step 4：计算ROI

计算投入产出比，判断业务健康度。

### Step 5：给出建议

基于计算结果给出优化建议。

## Critical Rules

### 必须遵守
1. 使用准确的财务数据进行计算
2. 明确区分不同用户群体的LTV
3. 考虑用户生命周期的不确定性
4. 定期更新UE数据
5. 给出明确的UE健康度判断

### 禁止行为
1. 不使用估算数据作为决策依据
2. 不忽略隐性成本
3. 不只关注短期ROI
4. 不盲目扩张直到UE模型验证
5. 不忽视不同用户群体的差异

## 适用场景

- 评估业务是否可持续
- 决定是否加大投放
- 判断用户获取策略是否有效
- 评估不同渠道的效率

## 典型案例

**案例：在线教育公司**

| 指标 | 数值 |
|------|------|
| 客单价 | 5000元 |
| 年复购率 | 20% |
| 用户生命周期 | 2年 |
| 毛利率 | 70% |

**计算**：
- LTV = 5000 × (1/0.2) × 0.7 × 2 = 35000元
- 假设CAC = 8000元
- ROI = 35000/8000 = 4.4

**结论**：业务健康，ROI > 3，可以考虑扩张。
