# 并列式拆解公式模板

## 公式结构

```
总指标 = 维度1子指标 + 维度2子指标 + 维度3子指标 + ... + 维度N子指标
```

## 验证方法

| 验证步骤 | 操作 | 判定标准 |
|---------|------|----------|
| 1 | 计算各子指标之和 | Σ子指标 = 总指标 |
| 2 | 检查子指标数量 | 通常3-7个为宜 |
| 3 | 检查维度一致性 | 同一层级必须在同一维度 |
| 4 | 检查无重叠 | 子指标之间无从属关系 |

## 标准输出格式

```markdown
# [总指标名称]并列式拆解

## 核心公式
[总指标] = [子指标1]([占比1]) + [子指标2]([占比2]) + [子指标3]([占比3])

## 各子指标详情

| 指标名称 | 衡量公式 | 数据来源 | 当前值 | 占比 |
|---------|---------|---------|-------|------|
| 子指标1 | 公式定义 | 系统A | 数值 | XX% |
| 子指标2 | 公式定义 | 系统B | 数值 | XX% |
| 子指标3 | 公式定义 | 系统C | 数值 | XX% |

## 验证结果
- 子指标之和: XX%
- 与总指标一致: 是/否
- 无重叠: 是/否
- 无遗漏: 是/否

## 执行建议
1. [优先级1]: [具体建议]
2. [优先级2]: [具体建议]
3. [优先级3]: [具体建议]
```

## 常见维度模板

### 按生命周期阶段
```
用户运营 = 用户获取 + 用户激活 + 用户留存 + 用户付费 + 用户传播
```

### 按渠道来源
```
总流量 = 渠道A + 渠道B + 渠道C + 渠道D + 渠道E
```

### 按产品类型
```
总收入 = 产品A收入 + 产品B收入 + 产品C收入 + 服务收入
```

### 按地域分布
```
总销售额 = 华东区 + 华南区 + 华北区 + 西部区 + 海外
```

### 按客户分层
```
总客户数 = 高价值客户 + 中价值客户 + 低价值客户 + 潜在客户
```
