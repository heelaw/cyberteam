### 使用示例

#### 场景1：电商平台年度规划

**用户输入**：
- 规划周期：年度规划
- 产品阶段：成长期
- 当前指标：月GMV 500万，用户30万
- 核心目标：提升付费转化率、提升用户留存
- 资源约束：团队15人，年度预算300万

**预期输出**：
- 分为4个季度的里程碑规划
- 每个季度有明确的策略组合（防守型40%+进攻型50%+探索型10%）
- 详细的资源分配和监测指标

#### 场景2：SaaS产品季度复盘

**用户输入**：
- 规划周期：季度规划
- 产品阶段：探索期向成长期过渡
- 当前指标：付费用户2000，月流失率15%
- 核心目标：验证PMF、找到增长引擎

**预期输出**：
- 以PMF验证为核心的策略设计
- 明确的验证里程碑和判断标准
- 灵活的资源调配方案

### 错误处理

| 错误类型 | 触发条件 | 处理方式 |
|----------|----------|----------|
| 目标模糊 | 目标描述使用"提升品牌影响力"等模糊词汇 | 要求用户明确具体指标和目标值 |
| 阶段错配 | 探索期产品使用成熟期策略 | 指出问题，提供该阶段的典型策略 |
| 资源超限 | 策略所需资源超出承受能力 | 重新评估，砍掉非核心策略 |
| 目标冲突 | 多个目标相互矛盾 | 使用ICE评分确定优先级 |
| 缺乏验证 | 阶段目标无法验证 | 要求设定可衡量的里程碑 |

## 核心定义

中长期运营规划制定是一套用于制定3-12个月运营规划的方法论。该方法强调从战略到执行的完整闭环，通过"明确目标-拆分阶段-制定策略-执行监控"四步法，确保规划既有战略高度又能落地执行。

## 适用场景

- 年度运营规划制定
- 季度运营计划制定
- 半年运营复盘与调整
- 新业务线运营规划
- 团队年度OKR制定

## 专家指导原则

### 必须遵守（5条）

1. **目标先行原则** - 规划必须从清晰的目标出发，无目标不规划
2. **阶段拆分原则** - 长期目标必须拆分为可验证的阶段性里程碑
3. **策略匹配原则** - 每个阶段目标必须有对应的策略支撑
4. **资源明确原则** - 每个策略必须明确所需资源和预期产出
5. **监控闭环原则** - 规划必须包含监控指标和调整机制

### 禁止行为（5条）

1. **禁止目标模糊** - 不得使用"提升品牌影响力"这类模糊目标
2. **禁止缺乏验证** - 不得设定无法验证的阶段性目标
3. **禁止策略空洞** - 不得只有策略方向而无具体执行计划
4. **禁止资源空想** - 不得制定超出实际资源承受能力的规划
5. **禁止静态规划** - 不得将规划视为固定不变，拒绝动态调整

## 规划框架概览

### 四步规划法

```
第一步：明确目标
    ↓
┌───────────────────────┐
│ 第二步：拆分阶段        │
│ - 划分关键时间节点      │
│ - 设定阶段性里程碑      │
│ - 确定阶段验收标准      │
└───────────────────────┘
    ↓
┌───────────────────────┐
│ 第三步：制定策略        │
│ - 设计核心运营策略      │
│ - 配置资源投入         │
│ - 制定关键动作         │
└───────────────────────┘
    ↓
┌───────────────────────┐
│ 第四步：执行监控        │
│ - 建立数据监测体系      │
│ - 设置预警机制         │
│ - 制定复盘节奏         │
└───────────────────────┘
```

## 第一步：明确目标

### 目标设定标准（SMART+）

| 标准 | 说明 | 示例 |
|------|------|------|
| **S-Specific** | 具体可描述 | "提升付费转化率" 而非 "提升收入" |
| **M-Measurable** | 可量化衡量 | 具体数值目标，如 "从3%提升至5%" |
| **A-Achievable** | 现实可达成 | 基于现有资源和发展趋势合理设定 |
| **R-Relevant** | 与战略相关 | 与公司/产品整体战略方向一致 |
| **T-Time-bound** | 有时间边界 | 明确在哪个时间点达成 |
| **+-Positive** | 正向描述 | 描述期望结果而非规避问题 |

### 目标分解结构

```
公司战略目标
    ↓
部门/业务线目标
    ↓
运营核心指标（最多3个）
    ↓
拆解指标（每个核心指标拆解2-3个）
```

### 目标优先级判断

当存在多个目标时，使用ICE评分确定优先级：

| 维度 | 说明 | 评分 |
|------|------|------|
| **Impact** | 对核心指标的影响程度 | 1-5分 |
| **Confidence** | 对目标达成的信心程度 | 1-5分 |
| **Ease** | 达成难度 | 1-5分 |

**ICE = Impact × Confidence × Ease**

## 第二步：拆分阶段

### 时间节点划分原则

| 规划周期 | 建议阶段划分 |
|----------|--------------|
| 季度规划（3个月） | 4周+4周+4周 或 月度×3 |
| 半年规划（6个月） | 月度×6 或 季度×2 |
| 年度规划（12个月） | 季度×4 或 月度×12 |

### 阶段里程碑设定

每个阶段需设定明确的里程碑：

| 里程碑类型 | 说明 | 示例 |
|------------|------|------|
| **结果里程碑** | 核心指标的阶段性目标 | "Q1末付费用户达10万" |
| **产出里程碑** | 关键动作的交付物 | "上线用户分层体系2.0" |
| **验证里程碑** | 假设验证的关键节点 | "完成A/B测试验证转化率假设" |

### 阶段验收标准

每个阶段结束时需验证：

1. 里程碑达成情况
2. 策略执行情况
3. 资源使用效率
4. 外部环境变化

## 第三步：制定策略

### 策略设计框架

```
核心策略1
├── 策略目标：[该策略要达成的子目标]
├── 目标用户：[策略覆盖的目标用户群]
├── 核心动作：[3-5个关键执行动作]
├── 资源需求：[人力/预算/渠道]
├── 关键指标：[衡量该策略效果的指标]
└── 风险预案：[可能的风险及应对]
```

### 策略组合设计

| 策略类型 | 占比建议 | 作用 |
|----------|----------|------|
| **防守型策略** | 30-40% | 保障存量业务稳定 |
| **进攻型策略** | 40-50% | 驱动核心指标增长 |
| **探索型策略** | 10-20% | 探索新增长点 |

### 资源规划

| 资源类型 | 规划要素 |
|----------|----------|
| **人力资源** | 关键岗位需求、时间投入 |
| **预算资源** | 各策略分配比例、ROI预期 |
| **渠道资源** | 渠道组合、优先级排序 |
| **数据资源** | 需要建设的数据能力 |

## 第四步：执行监控

### 监控体系设计

| 监控层级 | 频率 | 内容 |
|----------|------|------|
| **日常监控** | 每日 | 核心指标波动、异常预警 |
| **周度复盘** | 每周 | 策略执行进度、问题识别 |
| **月度复盘** | 每月 | 里程碑达成评估、策略调整 |
| **阶段复盘** | 每阶段 | 全面复盘、规划修订 |

### 预警机制

| 预警级别 | 触发条件 | 处理方式 |
|----------|----------|----------|
| **黄色预警** | 指标偏离计划10-20% | 增加监控频率，分析原因 |
| **橙色预警** | 指标偏离计划20-30% | 启动专项分析，准备预案 |
| **红色预警** | 指标偏离计划30%+ | 触发策略调整流程 |

### 复盘框架

| 复盘维度 | 核心问题 |
|----------|----------|
| **目标回顾** | 阶段目标是什么？完成度如何？ |
| **过程分析** | 关键动作执行情况？遇到什么障碍？ |
| **结果评估** | 产出符合预期吗？学到了什么？ |
| **规律总结** | 有什么可复用的经验？有什么坑要避免？ |
| **调整优化** | 下阶段规划需要如何调整？ |

## 工作流程（5步）

### 步骤1: 输入与背景分析

**输入**: 业务背景信息和规划需求

**处理**:
- 了解产品当前状态和所处阶段
- 明确规划的时间范围和约束条件
- 确认核心利益相关方
- 识别关键成功因素

**输出**: 规划背景说明书

**成功标准**: 背景信息完整，能支撑后续目标设定

---

### 步骤2: 目标设定与分解

**输入**: 规划背景说明书

**处理**:
- 设定规划周期的核心目标（SMART+）
- 分解目标到各子维度
- 使用ICE评分确定目标优先级
- 与利益相关方确认目标共识

**输出**: 目标设定文档（含分解结构）

**成功标准**: 目标具体可衡量，优先级清晰

---

### 步骤3: 阶段拆分与里程碑

**输入**: 目标设定文档

**处理**:
- 划分关键时间节点
- 为每个阶段设定里程碑
- 确定阶段验收标准
- 预留应急调整窗口

**输出**: 阶段拆分方案

**成功标准**: 阶段划分合理，里程碑可验证

---

### 步骤4: 策略制定与资源匹配

**输入**: 阶段拆分方案

**处理**:
- 为每个阶段设计核心策略
- 明确策略执行的关键动作
- 匹配资源需求和预算
- 制定风险预案

**输出**: 策略规划文档

**成功标准**: 策略具体可执行，资源匹配合理

---

### 步骤5: 监控体系与规划输出

**输入**: 策略规划文档

**处理**:
- 设计监控指标体系
- 设置预警阈值
- 制定复盘节奏
- 整合形成完整规划文档

**输出**: 中长期运营规划文档

**成功标准**: 监控机制完整，规划可直接执行
