# 五维度评估详解

## 概述

产品状态评估的五维度框架是一套全面的产品健康度评估体系，通过渠道质量、功能使用、复购率、活跃度、数据趋势五个维度，系统性地评估一个产品的运营状态。

---

## 维度一：渠道质量

### 1.1 核心指标

| 指标 | 定义 | 计算方式 | 评估标准 |
|-----|------|---------|---------|
| 渠道流量占比 | 各渠道带来的流量比例 | 某渠道UV/总UV×100% | 占比稳定为优 |
| 渠道转化率 | 渠道流量的最终转化 | 转化用户/渠道UV×100% | >行业平均为优 |
| CAC | 渠道获客成本 | 渠道成本/获客数 | <目标值为优 |
| 渠道ROI | 渠道投入产出比 | 渠道营收/渠道成本 | >1为正 |

### 1.2 渠道评估方法

**Step 1：渠道拆分**
- 按平台：App/小程序/Web/H5
- 按来源：搜索/社交/付费/直连
- 按活动：活动流量/自然流量

**Step 2：效率计算**
```python
各渠道效率得分 = 转化率×40% + (1/CAC标准化)×30% + 流量占比×30%
```

**Step 3：渠道分类**
| 类型 | 特征 | 策略 |
|-----|-----|-----|
| 明星渠道 | 高流量、高转化、低CAC | 加大投入 |
| 问题渠道 | 低转化或高CAC | 优化或削减 |
| 潜力渠道 | 高流量但效率待提升 | 优化提升 |
| 长尾渠道 | 流量小但稳定 | 维持现状 |

### 1.3 渠道质量维度权重

| 评估项 | 权重 | 说明 |
|-------|-----|-----|
| 流量质量 | 30% | 流量规模和稳定性 |
| 转化效率 | 30% | 转化率水平 |
| 获客成本 | 20% | CAC合理性 |
| 增长性 | 20% | 增长趋势 |

## 维度二：功能使用

### 2.1 核心指标

| 指标 | 定义 | 计算方式 | 评估标准 |
|-----|------|---------|---------|
| 功能渗透率 | 使用该功能的用户占比 | 使用UV/总UV×100% | >50%为核心功能 |
| 功能使用深度 | 用户使用的程度 | 人均使用次数/时长 | 越高越好 |
| 功能转化率 | 使用后转化的比例 | 转化数/功能使用UV | 因功能而异 |
| 功能留存率 | 下次访问继续使用的比例 | 回访时仍使用的用户/上次使用用户 | >40%为粘性功能 |

### 2.2 功能分类矩阵

```
                  使用渗透率
                    高    低
              ┌─────────┬─────────┐
        高    │ 核心功能 │ 辅助功能 │
              │  重点维护 │  引导曝光 │
   使用深度   ├─────────┼─────────┤
        低    │ 待优化   │ 低价值   │
              │ 功能问题  │ 可删除   │
              └─────────┴─────────┘
```

### 2.3 功能评估方法

**Step 1：识别核心功能**
- 渗透率Top3的功能为核心功能
- 或对业务目标贡献最大的功能

**Step 2：分析使用深度**
- 人均使用次数
- 人均使用时长
- 单次使用时长

**Step 3：追踪功能漏斗**
```
功能曝光 → 功能点击 → 功能使用 → 达成目标
```

**Step 4：功能问题识别**
- 渗透率持续下降
- 使用深度下降
- 漏斗某环节流失大

### 2.4 功能健康度权重

| 评估项 | 权重 | 说明 |
|-------|-----|-----|
| 核心功能渗透 | 40% | 核心功能覆盖度 |
| 功能使用深度 | 30% | 用户使用程度 |
| 功能增长性 | 30% | 趋势好坏 |

## 维度三：复购率

### 3.1 核心指标

| 指标 | 定义 | 计算方式 | 评估标准 |
|-----|------|---------|---------|
| N日复购率 | N日内有复购的用户占比 | N日内复购用户/总购买用户 | 越高越好 |
| 复购周期 | 用户平均多久复购一次 | 平均复购间隔天数 | 越短越好 |
| 复购频次 | 用户平均复购次数 | 总复购订单/复购用户数 | 越高越好 |
| LTV | 用户生命周期价值 | 用户总营收-总成本 | 越高越好 |

### 3.2 复购率分析

**复购率时间维度**：
| 复购率 | 含义 | 评估 |
|-------|-----|-----|
| 7日复购率 | 周活跃度 | 电商>15%为优 |
| 30日复购率 | 月活跃度 | 电商>30%为优 |
| 90日复购率 | 季度活跃度 | 电商>50%为优 |

**复购周期分布**：
| 周期 | 用户占比 | 策略 |
|-----|---------|-----|
| 7天内 | 核心复购用户 | 重点维护 |
| 7-30天 | 稳定复购用户 | 持续运营 |
| 30-90天 | 唤醒用户 | 召回策略 |
| 90天以上 | 流失风险用户 | 强力召回 |

### 3.3 LTV分析

```python
LTV = ARPU × 留存月数 × 毛利率

或者

LTV = Σ (月收入 × 该月留存率) × 毛利率
```

**LTV vs CAC**：
| 比率 | 含义 | 决策 |
|-----|-----|-----|
| LTV/CAC > 3 | 优秀 | 可加大投入 |
| LTV/CAC 1-3 | 良好 | 维持规模 |
| LTV/CAC < 1 | 亏损 | 需优化 |

### 3.4 复购健康度权重

| 评估项 | 权重 | 说明 |
|-------|-----|-----|
| 复购率 | 40% | 核心复购指标 |
| 复购周期 | 30% | 复购质量 |
| LTV | 30% | 用户价值 |

## 维度四：活跃度

### 4.1 核心指标

| 指标 | 定义 | 计算方式 | 评估标准 |
|-----|------|---------|---------|
| DAU | 日活跃用户数 | 日活用户数 | 增长为优 |
| MAU | 月活跃用户数 | 月活用户数 | 增长为优 |
| DAU/MAU | 粘性指标 | DAU/MAU×100% | >20%为优 |
| 日活率 | 某日活跃占MAU比 | 日活/MAU | 稳定为优 |

### 4.2 活跃度分布

**用户分层**：
| 类型 | 定义 | 占比 | 策略 |
|-----|-----|-----|-----|
| 高活跃 | 每日使用 | 核心用户 | 重点维护 |
| 中活跃 | 每周使用 | 活跃用户 | 持续运营 |
| 低活跃 | 每月使用 | 沉默用户 | 唤醒召回 |
| 流失用户 | 超过30天未使用 | 流失用户 | 召回或放弃 |

### 4.3 活跃度趋势

**移动平均分析**：
- 7日移动平均：看短期趋势
- 30日移动平均：看中期趋势
- 趋势向上：增长态势良好
- 趋势向下：需要干预

**DAU/MAU趋势解读**：
| 趋势 | 含义 | 应对 |
|-----|-----|-----|
| DAU涨+DAU/MAU涨 | 良性增长 | 保持 |
| DAU涨+DAU/MAU跌 | 增量不增粘性 | 关注留存 |
| DAU跌+DAU/MAU涨 | 老用户粘性高 | 加大拉新 |
| DAU跌+DAU/MAU跌 | 全面下滑 | 紧急干预 |

### 4.4 活跃度健康度权重

| 评估项 | 权重 | 说明 |
|-------|-----|-----|
| 绝对值 | 30% | DAU/MAU水平 |
| 趋势 | 40% | 增长还是下跌 |
| 结构 | 30% | 用户分层是否健康 |

## 维度五：数据趋势

### 5.1 核心指标

| 指标 | 定义 | 评估 |
|-----|-----|-----|
| 增长率 | 指标的增长速度 | 正向为优 |
| 增长质量 | 增长是否健康 | 留存率是否同步增长 |
| 行业位置 | 在行业中的位置 | 领先/跟随/落后 |

### 5.2 趋势分析方法

**同比增长**：
```python
同比增长率 = (本期-去年同期)/去年同期×100%
```

**环比增长**：
```python
环比增长率 = (本期-上期)/上期×100%
```

**移动平均趋势**：
- MA7：7日移动平均
- MA30：30日移动平均
- MA90：90日移动平均

### 5.3 增长动能分析

**增长来源拆解**：
```
用户增长 = 新用户 × 留存率 + 已有用户活跃 + 流失用户回流
```

**动能判断**：
| 动能 | 表现 | 判断 |
|-----|-----|-----|
| 正向动能 | 各因素同向增长 | 健康增长 |
| 负向动能 | 各因素同向下跌 | 需干预 |
| 结构性动能 | 部分增长部分下跌 | 需优化结构 |

### 5.4 行业对标

| 对标维度 | 内容 | 价值 |
|---------|-----|-----|
| 规模对标 | 用户量级、GMV体量 | 判断市场位置 |
| 效率对标 | 转化率、复购率、ARPU | 判断竞争力 |
| 增长对标 | 增长率、增速 | 判断发展势头 |

### 5.5 趋势健康度权重

| 评估项 | 权重 | 说明 |
|-------|-----|-----|
| 增长速度 | 40% | 增长率高低 |
| 增长质量 | 30% | 增长是否健康 |
| 行业位置 | 30% | 相对竞争力 |

## 综合评级计算

### 维度权重分配

| 维度 | 权重 | 说明 |
|-----|-----|-----|
| 渠道质量 | 20% | 获客能力 |
| 功能使用 | 20% | 产品力 |
| 复购率 | 25% | 用户价值（最重要） |
| 活跃度 | 20% | 用户粘性 |
| 数据趋势 | 15% | 发展动能 |

### 评级标准

| 综合得分 | 评级 | 行动建议 |
|---------|-----|---------|
| 90-100 | 优秀 | 行业标杆，保持领先 |
| 75-89 | 良好 | 有优势，持续优化 |
| 60-74 | 合格 | 有短板，重点改进 |
| <60 | 不合格 | 多方面问题，急需变革 |

### 评级输出模板

```
综合评级报告：

渠道质量：X/100 (权重20%) = XX分
功能使用：X/100 (权重20%) = XX分
复购率：X/100 (权重25%) = XX分
活跃度：X/100 (权重20%) = XX分
数据趋势：X/100 (权重15%) = XX分

综合得分：XX分
综合评级：[优秀/良好/合格/不合格]
```
