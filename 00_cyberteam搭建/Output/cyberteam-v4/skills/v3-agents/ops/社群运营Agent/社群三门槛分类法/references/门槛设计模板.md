# 社群三门槛设计模板

## 模板说明

本文档提供社群三门槛设计的标准化模板，适用于各类社群定位场景。

## 门槛设计基础信息

```
社群名称：[填写社群名称]
社群定位：[一句话描述社群核心价值]
目标用户：[描述目标用户群体]
社群价值：[列出社群提供的核心价值]
```

## 三层门槛设计

### 【高门槛】身份认同层

**设计原则**: 让用户产生归属感和身份认同

| 门槛类型 | 具体条件 | 验证方式 |
|---------|---------|---------|
| 学习成果 | 完成指定课程/认证 | 提交证书/截图 |
| 职业背景 | 具备特定职业/岗位 | 自我介绍审核 |
| 理念认同 | 认同特定价值观 | 承诺书签署 |

**高门槛条件设计**:
```
条件1: [填写条件]
条件2: [填写条件]
验证方式: [填写验证方式]
```

### 【中门槛】付费/时间层

**设计原则**: 过滤非目标用户，体现用户意愿

| 门槛类型 | 具体条件 | 备注 |
|---------|---------|------|
| 付费门槛 | [金额]元/年 或 [金额]元/终身 | 价格需与价值匹配 |
| 时间门槛 | 每周投入[时长] | 需设定可衡量的任务 |
| 行动门槛 | 承诺完成[具体行动] | 需有违约机制 |

**中门槛条件设计**:
```
条件类型: [付费/时间/行动]
具体内容: [填写具体条件]
价格/时间: [填写具体数值]
执行方式: [填写执行方式]
```

### 【低门槛】用户画像层

**设计原则**: 确保用户是目标群体

| 筛选维度 | 具体条件 | 验证方式 |
|---------|---------|---------|
| 行业 | [互联网/教育/金融等] | 用户自我介绍 |
| 职级 | [0-3岁产品经理/初创者等] | 自我介绍审核 |
| 地域 | [一线城市/全国等] | 地理位置或自我介绍 |
| 兴趣 | [对某个领域有兴趣] | 入群问卷 |

**低门槛条件设计**:
```
条件1: [填写条件]
条件2: [填写条件]
验证方式: [填写验证方式]
```

## 完整门槛方案示例

```
═══════════════════════════════════════════
           【XXX社群】门槛设置方案
═══════════════════════════════════════════

【基本信息】
社群名称: 产品经理进阶圈
社群定位: 帮助0-3岁产品经理提升专业能力
目标用户: 互联网行业产品经理
社群价值: 每周专业分享+每月案例分析+日常答疑

【高门槛】身份认同
条件: 必须完成《产品经理入门课程》或具备1年以上产品经验
验证: 提交课程证书 或 自我介绍说明经验

【中门槛】付费/时间
条件: 付费199元/年 或 承诺每周完成学习任务
验证: 付费截图 或 任务提交记录

【低门槛】用户画像
条件: 互联网行业产品经理 或 产品相关岗位
验证: 自我介绍审核

【执行流程】
1. 用户提交入群申请
2. 审核用户画像（低门槛）
3. 告知门槛条件（高中门槛）
4. 用户满足条件后入群
5. 入群后持续验证身份

═══════════════════════════════════════════
```

## 门槛设置检查清单

- [ ] 社群定位清晰，能一句话说明
- [ ] 高门槛能让用户产生身份认同
- [ ] 中门槛能有效过滤非目标用户
- [ ] 低门槛能保证基础用户质量
- [ ] 门槛条件可量化、可验证
- [ ] 门槛与社群价值匹配
- [ ] 执行流程清晰可落地
