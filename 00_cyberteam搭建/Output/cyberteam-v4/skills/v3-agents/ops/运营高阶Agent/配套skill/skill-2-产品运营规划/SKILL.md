# 产品运营规划 SKILL

## 基本信息

| 属性 | 值 |
|------|-----|
| **Skill名称** | product-ops-planning |
| **版本** | v1.0 |
| **触发关键词** | 产品运营、生命周期、运营规划、中长期、破局点 |
| **适用场景** | 新产品上线规划、现有产品运营策略制定、运营规划方案输出 |
| **输入** | 产品信息、用户数据、市场环境、资源条件 |
| **输出** | 完整的产品运营规划方案（含阶段目标、策略、路径） |

---

## 核心定义

产品运营规划是指基于产品生命周期理论，结合影响运营策略的5种核心因素，为产品制定科学的中长期运营策略和执行路径。

**5种核心因素**：
1. 产品阶段（探索期/成长期/成熟期/衰退期）
2. 用户特征（用户画像、需求特点）
3. 竞争环境（市场格局、竞品动态）
4. 资源条件（人、钱、资源）
5. 团队能力（组织能力、执行力）

---

## Critical Rules（必须遵守）

1. **阶段优先判断**：先确定产品所处生命周期阶段，不同阶段策略重心不同
2. **因素综合分析**：5种因素必须综合考量，不能只看单一因素
3. **破局点聚焦**：每个阶段只选1-2个核心破局点，集中资源突破

---

## 禁止行为

1. **禁止套用固定模板**：每个产品情况不同，不能生搬他人成功经验
2. **禁止忽略产品阶段**：探索期做增长和成熟期做增长策略完全不同
3. **禁止面面俱到**：贪多求全导致资源分散，最终什么都做不好
4. **禁止忽视竞争环境**：闭门造车不考虑竞品动态是致命错误
5. **禁止缺乏阶段性目标**：没有里程碑的规划是空谈
6. **禁止忽视团队能力**：再好的策略执行不了等于零

---

## 工作流程

### Step 1: 产品阶段判断

**输入**：产品关键指标数据

**处理**：
根据以下指标判断产品阶段：

| 阶段 | 用户增长 | 市场认知 | 产品迭代 | 商业化 |
|------|----------|----------|----------|--------|
| 探索期 | 缓慢 | 未知 | 频繁 | 未开始 |
| 成长期 | 快速 | 逐步建立 | 稳定 | 探索中 |
| 成熟期 | 放缓 | 稳定 | 减少 | 主力 |
| 衰退期 | 下降 | 下降 | 停滞 | 维持 |

**输出**：产品当前所处阶段及依据

**成功标准**：能给出明确的阶段判断和3个以上支撑依据

---

### Step 2: 5因素分析

**输入**：产品阶段判断结果

**处理**：
逐项分析5种核心因素：

**因素1-产品阶段**：
- 探索期：找Product-Market Fit
- 成长期：快速放大用户规模
- 成熟期：深挖用户价值和商业化
- 衰退期：转型或战略收缩

**因素2-用户特征**：
- 核心用户是谁
- 用户核心需求是什么
- 用户决策路径是什么

**因素3-竞争环境**：
- 直接竞品有哪些
- 差异化优势在哪里
- 市场空白点在哪里

**因素4-资源条件**：
- 有多少人
- 有多少钱
- 有多少资源支持

**因素5-团队能力**：
- 团队擅长什么
- 组织效率如何
- 执行力评估

**输出**：
```
5因素分析矩阵：
┌─────────────┬──────────────┬────────────┐
│   因素      │    现状      │   启示     │
├─────────────┼──────────────┼────────────┤
│ 产品阶段    │ [阶段判断]   │ [策略重点] │
│ 用户特征    │ [用户画像]   │ [切入点]   │
│ 竞争环境    │ [竞品分析]   │ [差异化]   │
│ 资源条件    │ [资源盘点]   │ [优先级]   │
│ 团队能力    │ [能力评估]   │ [执行保障] │
└─────────────┴──────────────┴────────────┘
```

**成功标准**：每个因素都有清晰的现状描述和策略启示

---

### Step 3: 找到核心运营要点

**输入**：5因素分析结果

**处理**：
- 综合5个因素的启示
- 找出交集点（多重因素指向的方向）
- 确定1-2个核心运营破局点

**输出**：
```
核心运营要点确定：
1. 破局点1：[方向]
   - 依据：[因素1]+[因素2]的组合启示
2. 破局点2：[方向]
   - 依据：[因素3]+[因素4]的组合启示
```

**成功标准**：破局点不超过2个，且有充分依据支撑

---

### Step 4: 制定阶段性目标

**输入**：核心运营要点

**处理**：
- 将破局点转化为可量化的阶段目标
- 设置里程碑和时间节点
- 定义每个阶段的成功标准

**输出**：
```
阶段性目标规划：
┌────────────┬────────────────┬──────────────┬──────────────┐
│   阶段     │    时间节点    │    目标指标  │   成功标准   │
├────────────┼────────────────┼──────────────┼──────────────┤
│ 第一阶段   │ [日期]         │ [量化指标]   │ [验收条件]   │
│ 第二阶段   │ [日期]         │ [量化指标]   │ [验收条件]   │
│ 第三阶段   │ [日期]         │ [量化指标]   │ [验收条件]   │
└────────────┴────────────────┴──────────────┴──────────────┘
```

**成功标准**：目标可量化、时间节点明确、成功标准可验证

---

### Step 5: 制定执行路径

**输入**：阶段性目标

**处理**：
- 为每个阶段制定具体的行动项
- 明确资源投入和负责人
- 设置复盘和调整机制

**输出**：
```
执行路径：
第一阶段：[阶段名称]
├── 行动项1：[具体动作]
├── 行动项2：[具体动作]
├── 资源投入：[人/钱/资源]
└── 风险预案：[可能风险及应对]

[重复第二、三阶段...]
```

**成功标准**：每项行动都有明确的执行人和验收条件

---

### Step 6: 效果评估机制

**输入**：执行路径和阶段性目标

**处理**：
- 建立数据监控体系
- 设置预警机制
- 制定迭代优化机制

**输出**：完整的产品运营规划方案

**成功标准**：能回答"做什么、怎么做、做到什么程度、如何复盘"

---

## 评估维度

| 维度 | 权重 | 说明 |
|------|------|------|
| 阶段判断准确性 | 20% | 产品阶段判断是否有依据 |
| 因素分析完整性 | 20% | 5种因素是否都分析到位 |
| 破局点聚焦度 | 20% | 是否集中资源突破核心点 |
| 目标可量化性 | 15% | 目标是否可衡量 |
| 执行路径清晰度 | 15% | 行动项是否具体可执行 |
| 评估机制完善度 | 10% | 是否有复盘和调整机制 |

## 应用场景示例

### 场景：新品上市运营规划

**输入**：一款B端SaaS产品，处于探索期向成长期过渡阶段

**分析**：
1. 阶段判断：成长期初期
2. 5因素分析完成
3. 破局点：口碑传播+核心客户打造
4. 阶段目标：3个月获取50家标杆客户
5. 执行路径：确定具体获客方式和推广计划

**输出**：完整的新品上市运营规划方案
