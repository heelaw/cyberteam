# 生命周期判断法

## Skill概述

生命周期判断法帮助用户判断企业当前处于什么发展阶段，是初创期、成长期、成熟期还是转型期。

## 使用示例

### 示例1：判断一家A轮SaaS公司

**输入**:
```
公司：某CRM SaaS公司
融资：A轮
团队：50人
收入：月收入200万，增长中
```

**输出**:
```
当前阶段：成长期
置信度：85%

各维度匹配度：
- 业务维度：成长期特征（收入增长中）
- 团队维度：成长期特征（50人规模）
- 财务维度：成长期特征（A轮）
- 竞争维度：成熟期特征（市场稳定）

判断依据：符合成长期的核心特征（A轮、快速增长），团队规模适中

下阶段预测：预计12-18个月进入成熟期
关键指标：ARR、Churn Rate、CAC/LTV
```

## 错误处理

| 错误场景 | 处理方式 |
|----------|----------|
| 企业信息不足 | 主动询问关键指标：融资、团队、收入、增长 |
| 特殊行业难判断 | 基于行业标准调整判断维度，说明行业特殊性 |
| 处于过渡期 | 标注"过渡期"，说明从X期向Y期过渡 |
| 信息矛盾 | 指出矛盾点，要求澄清或标注"待确认" |

## 独特个性

本Skill采用**多维度交叉验证法**（业务×团队×财务×竞争），而非单一维度判断，能准确识别企业真实发展阶段，特别适合判断处于过渡期或特征模糊的企业。

## 企业生命周期四阶段

### 1. 初创期

- **融资阶段**：未融资/天使轮
- **团队规模**：10人以下
- **业务状态**：探索商业模式
- **核心任务**：找到PMF

### 2. 成长期

- **融资阶段**：A轮-B轮
- **团队规模**：10-100人
- **业务状态**：快速增长
- **核心任务**：验证商业模式

### 3. 成熟期

- **融资阶段**：C轮后/已盈利
- **团队规模**：100人以上
- **业务状态**：稳定增长
- **核心任务**：优化放大

### 4. 转型期

- **融资阶段**：可能需要融资
- **团队规模**：视情况而定
- **业务状态**：增长放缓
- **核心任务**：寻找新增长点

## 判断维度

### 业务维度

- 收入规模
- 增长速度
- 商业模式成熟度

### 团队维度

- 团队规模
- 组织架构完善度
- 核心岗位到位情况

### 财务维度

- 融资阶段
- 盈利情况
- 资金储备

### 竞争维度

- 市场地位
- 竞争壁垒
- 市场份额

## 工作流程

### Step 1：收集信息

收集企业基本信息。

### Step 2：多维度评估

从业务、团队、财务、竞争维度评估。

### Step 3：综合判断

基于多维度信息判断阶段。

### Step 4：验证确认

与典型阶段特征对比验证。

## Critical Rules

### 必须遵守

1. 基于多维度信息综合判断
2. 考虑行业特点和业务特殊性
3. 动态看待阶段演变
4. 给出明确的阶段判断
5. 考虑企业的特殊情况

### 禁止行为

1. 不单纯以融资阶段判断
2. 不忽略业务实际状态
3. 不将阶段绝对化
4. 不忽视企业可能处于过渡期
5. 不为迎合用户而刻意美化判断
