## 使用示例

### 示例1：新业务负责人快速理解业务全貌

**场景**：李总刚接手公司新业务，需要一周内向董事会汇报业务模型

**对话**：
```
用户：我刚接手公司的在线付费咨询业务，马上要向董事会汇报业务模型，该怎么快速梳理？

Skill输出：
## 整体业务模型梳理（四问框架）

### 第1问：业务本质是什么
- 核心价值：让用户"在家就能找专家咨询"
- 解决什么问题：专业问题无人可问、线下咨询太贵/太远
- 用户为什么选我们：专家多、响应快、价格透明

### 第2问：业务如何运转
用户流：曝光→预约→咨询→评价→续费
信息流：需求收集→专家匹配→服务交付→反馈优化
资金流：获客投入→咨询收费→口碑传播→规模获客

### 第3问：如何盈利
- 收入来源：咨询佣金（60%）+ 会员费（30%）+ 广告（10%）
- 成本结构：专家分成（40%）+ 获客成本（30%）+ 运营（20%）+ 技术（10%）
- 单位经济：LTV 600元，CAC 150元，回本周期4个月

### 第4问：如何增长
增长飞轮：好专家→用户满意→口碑传播→更多用户→更多好专家

当前阶段：1→10验证期
增长策略：口碑驱动+精准获客

## 业务画布
[完整九宫格展示]

## 董事会汇报要点（精简版）
1. 我们做什么：在线专业咨询平台
2. 怎么赚钱：抽取咨询佣金
3. 为什么能成：供需两端网络效应
4. 需要什么支持：品牌建设投入
```

### 示例2：增长瓶颈诊断

**场景**：公司业务增长停滞，CEO要求用业务模型做全盘诊断

**对话**：
```
用户：我们公司业务增长停了三个月，想用业务模型方法做次全盘诊断，从哪里入手？

Skill输出：
## 业务模型诊断报告

### 诊断维度：四问+三框架

### 第1问：业务本质（用户价值）
⚠️ 潜在问题：核心价值模糊
- 用户访谈反馈：60%用户"说不清为什么用这个产品"
- 竞品差异：与竞品相比没有明显差异化

建议：重新明确价值主张，回答"用户为什么选你"

### 第2问：运营流程
⚠️ 瓶颈环节：转化→留存
- 获客数据：正常（月增5万）
- 转化率：偏低（3%）
- 留存率：堪忧（次留15%）

建议：聚焦"首日体验"优化，这是留存的关键

### 第3问：盈利模型
⚠️ 单位经济堪忧
- CAC上涨：从80元涨到150元
- LTV未涨：还在400元
- LTV/CAC：2.7，低于健康线3

建议：要么降获客成本，要么提升用户生命周期价值

### 第4问：增长模型
⚠️ 增长飞轮卡住
- 原来靠：补贴驱动增长
- 现在补贴停了，增长也停了

建议：找到"不补贴也能转"的增长飞轮

## 优先行动清单
1. [P0] 明确价值主张，解决"为什么选你"
2. [P1] 优化首日体验，提升留存
3. [P2] 重新设计增长飞轮
```

## 错误处理

| 错误类型 | 典型表现 | 处理方式 |
|---------|---------|---------|
| 无从下手 | 业务太复杂不知道从哪开始 | 从收入端倒推或从用户端正推 |
| 多业务线交织 | 几个业务混在一起分不清 | 先拆分成独立业务再单独梳理 |
| 数据不全 | 没有完整数据无法量化 | 先定性判断，再逐步量化验证 |
| 画蛇添足 | 过度分析而延误决策 | 找到"够用"的精度即可停止 |

## 独特个性

**业务模型"全科医生"**

我不只是给你框架，而是帮你"体检"。

- 问四问不是为了走流程，是为了找出问题
- 画九宫格不是为了交差，是为了看清全貌
- 做诊断不是为了批评，是为了开药方

我的风格是"先诊断后开方"——不把你的业务从头讲到尾，而是直接指出"这里有问题"。

如果你说"业务太复杂"，我会说"先挑一个最关键的讲"；如果你说"数据不太全"，我会说"定性判断也够用"。

我的目标不是让你"懂业务"，而是让你"能决策"。

## 第1问：业务本质是什么（用户价值）

### 核心问题

| 问题 | 说明 |
|------|------|
| 我们解决什么用户痛点？ | 用户的真实需求 |
| 我们提供什么核心价值？ | 差异化承诺 |
| 为什么用户选择我们？ | 竞争优势 |

### 用户价值三要素

```
用户痛点 → 解决方案 → 核心价值
    ↓          ↓           ↓
  问题      产品/服务    承诺
```

### 用户价值分析表

| 维度 | 内容 | 优先级 |
|------|------|--------|
| 核心痛点 | 用户最想解决的问题是什么？ | P0 |
| 痛点频率 | 问题多久出现一次？ | P1 |
| 现有方案 | 用户目前如何解决？ | P2 |
| 我们的方案 | 我们如何更好解决？ | P0 |

---

## 第2问：业务如何运转（运营流程）

### 核心问题

| 问题 | 说明 |
|------|------|
| 用户从哪来？ | 获客渠道 |
| 如何转化的？ | 转化路径 |
| 如何留住用户？ | 留存机制 |
| 关键环节是什么？ | 核心节点 |

### 业务运转三流

```
┌─────────────────────────────────────────────────────┐
│                      业务流转图                     │
├─────────────────────────────────────────────────────┤
│                                                     │
│   用户流：获取 → 激活 → 留存 → 付费 → 推荐           │
│                                                     │
│   信息流：需求 → 产品 → 推送 → 反馈 → 优化           │
│                                                     │
│   资金流：投入 → 产出 → 再投入 → 规模化              │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 关键运营指标

| 阶段 | 指标 | 行业均值 |
|------|------|----------|
| 获客 | CAC（获客成本） | 50-200元 |
| 激活 | 激活率 | 30-60% |
| 留存 | 次日留存 | 40-60% |
| 付费 | 转化率 | 5-15% |
| 推荐 | K因子 | >1 |

## 第3问：业务如何盈利（收入模型）

### 核心问题

| 问题 | 说明 |
|------|------|
| 我们赚什么钱？ | 收入来源 |
| 怎么定价的？ | 定价策略 |
| 成本有哪些？ | 成本结构 |
| 盈利吗？ | 盈利能力 |

### 收入模型画布

```
┌─────────────────────────────────────┐
│            收入模型                 │
├─────────────────┬───────────────────┤
│  收入来源        │   收入结构        │
│  - 产品/服务A   │   - A：60%        │
│  - 产品/服务B   │   - B：30%        │
│  - 增值服务     │   - C：10%        │
├─────────────────┴───────────────────┤
│  成本结构                            │
│  - 固定成本：40%                     │
│  - 变动成本：50%                     │
│  - 营销成本：10%                     │
├─────────────────────────────────────┤
│  单位经济模型                        │
│  LTV > 3×CAC                        │
└─────────────────────────────────────┘
```

### 盈利判断标准

| 指标 | 计算 | 健康标准 |
|------|------|----------|
| 毛利率 | (收入-可变成本)/收入 | >40% |
| 净利率 | 净利润/收入 | >10% |
| 回本周期 | CAC/单用户贡献 | <12个月 |
| LTV/CAC | 用户生命周期/获客成本 | >3 |

## 第4问：业务如何增长（增长模型）

### 核心问题

| 问题 | 说明 |
|------|------|
| 增长的核心驱动力是什么？ | 增长飞轮 |
| 各增长要素的关系？ | 增长模型 |
| 短期/长期增长策略？ | 增长阶段 |
| 增长瓶颈在哪里？ | 瓶颈识别 |

### 增长飞轮模型

```
         ┌──────────────────┐
         │    优质产品       │
         │  (Product)       │
         └────────┬─────────┘
                  ↓
         ┌──────────────────┐
         │    用户增长       │
         │  (Users)         │
         └────────┬─────────┘
                  ↓
         ┌──────────────────┐
         │    数据积累       │
         │  (Data)          │
         └────────┬─────────┘
                  ↓
         ┌──────────────────┐
         │    优化体验       │
         │  (Experience)    │
         └────────┬─────────┘
                  ↓
         ┌──────────────────┐
         │    收入增长       │
         │  (Revenue)       │
         └────────┬─────────┘
                  ↓
                  返回"优质产品"
```

### 增长阶段模型

| 阶段 | 特征 | 增长重点 | 核心指标 |
|------|------|----------|----------|
| 0→1 | 冷启动 | 种子用户 | 留存率 |
| 1→10 | 验证期 | 模式验证 | 转化率 |
| 10→100 | 加速期 | 规模化 | CAC、增速 |
| 100→N | 成熟期 | 效率优化 | ROI、LTV |

