# 创业阶段检查卡

## 概述

创业和业务发展可分为不同阶段，每个阶段有不同的重点任务和成功标准。

---

## 阶段模型

```
阶段0：想法期
    ↓
阶段1：0→1（冷启动）
    ↓
阶段2：1→10（验证）
    ↓
阶段3：10→100（加速）
    ↓
阶段4：100→N（成熟）
```

---

## 阶段0：想法期

### 核心任务

```
1. 验证需求是否真实
2. 定义核心用户群体
3. 形成初步解决方案
4. 设计商业模式
```

### 成功标准

| 检查项 | 标准 | 状态 |
|--------|------|------|
| 用户访谈 | 20+访谈，80%有需求 | [ ] |
| 愿意付费 | >50%表示愿意付费 | [ ] |
| 市场规模 | TAM>1亿 | [ ] |
| 竞品分析 | 存在差异化机会 | [ ] |

### 常见陷阱

```
陷阱1：自我验证
- 只找认同你的人验证
- 解决：找中立用户访谈

陷阱2：需求伪真实
- 用户说想要但不付费
- 解决：要求用户付出真实行动

陷阱3：市场想太大
- 目标市场定义太宽泛
- 解决：聚焦细分市场
```

### 通过标准

```
[ ] 需求真实存在
[ ] 愿意付费验证通过
[ ] 差异化机会明确
[ ] 团队能力匹配
```

## 阶段1：0→1（冷启动）

### 核心任务

```
1. 完成MVP开发
2. 找到100个种子用户
3. 验证核心价值
4. 找到PMF（产品市场契合点）
```

### 成功标准

| 检查项 | 标准 | 状态 |
|--------|------|------|
| MVP完成 | 核心功能上线 | [ ] |
| 用户留存 | 7日留存>40% | [ ] |
| 用户满意度 | NPS>40 | [ ] |
| 留存曲线 | 曲线趋缓 | [ ] |

### PMF判断标准

```
strong PMF：
- 用户主动推荐
- 需求增长超过团队能力
- 不用大力推广用户自然增长

weak PMF：
- 需要持续推广才能维持增长
- 用户留存不稳定
- 用户反馈一般
```

### 常见陷阱

```
陷阱1：MVP太大
- 做了1年才上线
- 解决：只做核心功能，2-4周上线

陷阱2：用户太杂
- 什么用户都接
- 解决：聚焦核心用户群体

陷阱3：过早商业化
- 产品没打磨好就开始收费
- 解决：先验证价值，再考虑收费
```

### 通过标准

```
[ ] 找到PMF（Strong PMF）
[ ] 留存曲线稳定
[ ] 用户主动推荐
[ ] NPS>40
```

## 阶段2：1→10（验证）

### 核心任务

```
1. 验证增长模式
2. 找到可复制的获客方式
3. 优化单位经济模型
4. 扩大团队规模
```

### 成功标准

| 检查项 | 标准 | 状态 |
|--------|------|------|
| 增长验证 | 月增速>20% | [ ] |
| CAC验证 | CAC可接受 | [ ] |
| LTV/CAC | >3 | [ ] |
| 续费率 | >60% | [ ] |

### 增长模式验证

```
验证的增长方式：
1. [ ] 口碑增长：K因子>0.3
2. [ ] 投放增长：ROI>1
3. [ ] 裂变增长：邀请率>10%
4. [ ] 渠道增长：某渠道CAC<均值

可复制标准：
- 单一渠道月获客>1000
- CAC稳定或下降
- 用户质量不下降
```

### 常见陷阱

```
陷阱1：增长假象
- 靠补贴/活动带来的虚假增长
- 解决：去掉补贴看自然增长

陷阱2：规模不经济
- 规模越大CAC越高
- 解决：找到规模经济的临界点

陷阱3：团队跟不上
- 业务增长快，团队能力跟不上
- 解决：提前储备人才
```

### 通过标准

```
[ ] 找到可复制的增长模式
[ ] 单位经济模型健康
[ ] 团队能力匹配
[ ] 能融到足够的钱
```

## 阶段3：10→100（加速）

### 核心任务

```
1. 规模化获客
2. 扩大团队
3. 优化运营效率
4. 准备竞争壁垒
```

### 成功标准

| 检查项 | 标准 | 状态 |
|--------|------|------|
| 规模 | 月新增>10000 | [ ] |
| 市场份额 | 进入细分市场前三 | [ ] |
| 品牌认知 | 品牌搜索量提升 | [ ] |
| 运营效率 | 单位成本下降 | [ ] |

### 竞争壁垒建设

```
壁垒类型：
1. [ ] 网络效应：用户越多价值越大
2. [ ] 规模经济：规模越大成本越低
3. [ ] 品牌壁垒：用户认知度高
4. [ ] 技术壁垒：技术难以复制
5. [ ] 独家资源：独家供应链/渠道
```

### 常见陷阱

```
陷阱1：，增长失速
- 增长速度突然放缓
- 解决：开拓新渠道/新市场

陷阱2：竞争恶化
- 竞品价格战
- 解决：差异化竞争，回归价值

陷阱3：组织膨胀病
- 人数增加效率下降
- 解决：保持精简，强化文化
```

### 通过标准

```
[ ] 成为细分市场头部
[ ] 建立竞争壁垒
[ ] 组织效率保持
[ ] 财务可持续
```

---

## 阶段4：100→N（成熟）

### 核心任务

```
1. 效率优化
2. 新业务探索
3. 国际化/多元化
4. 组织进化
```

### 成功标准

| 检查项 | 标准 | 状态 |
|--------|------|------|
| 市场份额 | >30% | [ ] |
| 盈利能力 | 稳定盈利 | [ ] |
| 新业务 | 第二增长曲线出现 | [ ] |
| 组织健康 | 管理成熟度提升 | [ ] |

### 常见陷阱

```
陷阱1：创新者窘境
- 被新进入者颠覆
- 解决：内部孵化新业务

陷阱2：组织僵化
- 决策变慢，执行力下降
- 解决：保持小团队运作模式

陷阱3：战略分散
- 新业务太多，资源分散
- 解决：聚焦核心，砍掉边缘
```
