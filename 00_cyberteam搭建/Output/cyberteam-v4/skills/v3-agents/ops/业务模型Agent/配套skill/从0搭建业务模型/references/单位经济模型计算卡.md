# 单位经济模型计算卡

## 什么是单位经济模型

单位经济模型（Unit Economics）分析的是：**单个用户/订单为单位时的成本和收入关系**。

```
单位经济模型核心问题：
- 获取一个用户，我能赚多少钱？
- 钱多久能收回来？
- 这个生意值得做吗？
```

---

## 核心指标定义

### LTV（用户生命周期价值）

```
定义：一个用户在整个生命周期内为企业贡献的总价值

计算公式：

基础版：
LTV = 用户平均月收入 × 平均生命周期（月）

完整版：
LTV = Σ (月收入 × 该月留存率) × 毛利率

简化版（适合订阅模式）：
LTV = 月订阅费 × 平均订阅月数 × 毛利率
```

### CAC（客户获取成本）

```
定义：获取一个新客户所需的平均成本

计算公式：
CAC = 营销总费用 ÷ 新增客户数

细分CAC：
- 有机CAC = 自然流量获客成本
- 付费CAC = 付费投放获客成本
- 混合CAC = (营销费用-自然流量价值) ÷ 付费获客数
```

### LTV/CAC比值

```
定义：用户价值与获客成本的比值

判断标准：

LTV/CAC > 3    → 优秀，可大规模获客
LTV/CAC = 1~3  → 良好，稳健增长
LTV/CAC < 1    → 危险，亏损状态

理想情况：
- LTV > 3×CAC
- 回本周期 < 12个月
```

### 回本周期

```
定义：从获客到收回成本的时间

计算公式：
回本周期 = CAC ÷ (月用户贡献 - 月服务成本)

判断标准：
回本周期 < 6个月  → 优秀
回本周期 < 12个月 → 良好
回本周期 > 18个月 → 风险
回本周期 > 24个月 → 危险
```

### ARPU（用户平均收入）

```
定义：每个用户的平均收入

计算公式：
ARPU = 总收入 ÷ 总用户数

按月计算：
月ARPU = 月收入 ÷ 月活跃用户数
```

## 计算示例

### 示例1：订阅制SaaS

```
背景：
- 月订阅费：99元
- 平均订阅时长：8个月
- 毛利率：70%
- CAC：500元

计算：

LTV = 99 × 8 × 70% = 554.4元
LTV/CAC = 554.4 ÷ 500 = 1.11
回本周期 = 500 ÷ (99×70% - 服务成本) ≈ 8个月

结论：LTV/CAC偏低，需要提升LTV或降低CAC
```

### 示例2：电商平台

```
背景：
- 平均客单价：200元
- 毛利率：30%
- 平均订单数/月：2次
- 平均生命周期：24个月
- CAC：80元

计算：

月ARPU = 200 × 2 × 30% = 120元
LTV = 120 × 24 = 2880元
LTV/CAC = 2880 ÷ 80 = 36

结论：LTV/CAC优秀，可大规模投放
```

### 示例3：免费+增值游戏

```
背景：
- 免费用户：1000人
- 付费转化率：5%
- 付费用户月ARPU：150元
- 付费用户平均生命周期：6个月
- 毛利率：60%
- CAC：30元（面向所有用户）

计算：

整体LTV = 1000×5%×150×6×60% ÷ 1000 = 27元
整体CAC = 30元
LTV/CAC = 27 ÷ 30 = 0.9 < 1

结论：单纯买量模式不可行，需要提升付费转化或LTV
```

## 优化方向

### 提升LTV

| 方法 | 说明 | 适用场景 |
|------|------|----------|
| 提升月ARPU | 增加付费点、提升客单价 | 所有业务 |
| 延长生命周期 | 会员体系、续费激励 | 订阅型 |
| 提升毛利率 | 优化供应链、降低服务成本 | 实物电商 |
| 提升复购率 | 积分体系、专属优惠 | 复购型 |

### 降低CAC

| 方法 | 说明 | 适用场景 |
|------|------|----------|
| 优化渠道 | 找到更低CAC渠道 | 所有业务 |
| 优化素材 | 提升CTR，降低CPM | 投放业务 |
| 口碑裂变 | K因子>1时使用 | 有忠诚用户 |
| SEO/内容 | 降低有机流量成本 | 内容型 |

### 加速回本

| 方法 | 说明 | 适用场景 |
|------|------|----------|
| 首次优惠 | 降低首次付费门槛 | 付费转化难 |
| 预付优惠 | 鼓励年付/季付 | 订阅型 |
| 提升首单价值 | 首单高价值产品 | 高客单业务 |

## 常见问题诊断

### 问题1：LTV/CAC<1

```
表现：获客越多，亏得越多

原因分析：
- CAC太高
- LTV太低
- 用户生命周期太短

解决方向：
1. 先止血：暂停/减少付费获客
2. 找低CAC渠道：口碑、裂变、内容
3. 提升LTV：提升服务质量、增加付费点
```

### 问题2：回本周期>18个月

```
表现：钱压在里面太久

原因分析：
- CAC太高
- 月用户贡献太低
- 服务成本太高

解决方向：
1. 降低CAC
2. 提升月贡献
3. 优化服务成本
```

### 问题3：LTV不稳定

```
表现：不同用户LTV差异大

原因分析：
- 用户分层没做好
- 服务质量不一致
- 用户画像不清晰

解决方向：
1. 用户分群分析
2. 针对性运营
3. 提升核心用户LTV
```
