# 从0起步验证路径模板

## 模板说明

此模板用于类型一项目（从0起步独立项目）的破局规划。核心方法是以终为始，倒推验证路径。

## 核心原则

1. **以终为始** - 先想清楚做成是什么样，再倒推验证路径
2. **假设驱动** - 每个验证步骤都是验证一个具体假设
3. **最小成本** - 用最少资源获取最大验证信息
4. **逐步加码** - 验证成功后再增加资源投入

## 模板内容

### 1. 终极目标定义

**项目**：[项目名称]
**终极目标**：[这件事做成是什么样？]

| 维度 | 目标描述 |
|------|---------|
| 用户价值 | 用户获得什么？ |
| 商业价值 | 商业模式是什么？ |
| 竞争壁垒 | 凭什么能赢？ |
| 规模目标 |要做到多大？ |

### 2. 关键里程碑倒推

从终极目标倒推到第一个可验证的假设：

```
终极目标
    ↓ 需要什么？（Milestone 3）
        ↓ 需要什么？（Milestone 2）
            ↓ 需要什么？（Milestone 1）
                ↓ 第一个可验证的假设是什么？（Hypothesis 0）
```

| 里程碑 | 描述 | 验证假设 | 假设是否成立 |
|-------|------|---------|-------------|
| M3 (终极) | [描述] | - | - |
| M2 | [描述] | [假设] | □ 是 □ 否 |
| M1 | [描述] | [假设] | □ 是 □ 否 |
| H0 | [描述] | [假设] | □ 是 □ 否 |

### 3. 第一个假设验证方案

**第一个假设**（H0）：[描述]
**为什么这是第一个]： [解释为什么应该先验证这个]

#### 验证方案

| 验证活动 | 具体做法 | 资源投入 | 验证成功标准 |
|---------|---------|---------|-------------|
| [活动1] | [步骤] | [资源] | [标准] |
| [活动2] | [步骤] | [资源] | [标准] |

#### 验证时间线

```
第1-2周：[活动1执行]
第3-4周：[活动2执行 + 数据收集]

```

#### 验证结果

| 结果 | 结论 | 下一步 |
|------|------|-------|
| 成功 | H0成立 | 进入H1验证 |
| 失败 | H0不成立 | 调整假设或终止 |

### 4. 资源配置计划

根据验证阶段逐步加码：

| 阶段 | 阶段名称 | 资源投入 | 加码条件 |
|------|---------|---------|---------|
| Stage 0 | 假设验证期 | 最少资源 | 持续验证 |
| Stage 1 | 模型验证期 | 中等资源 | H0验证成功 |
| Stage 2 | 规模扩张期 | 充足资源 | 核心模型验证成功 |

### 5. 心态管理

从0起步项目最怕的是：**过于急躁**

| 常见心态问题 | 应对方法 |
|-------------|---------|
| "为什么这么久没进展" | 验证本身就需时间，这是正常的 |
| "要不要换个方向" | 先完成当前假设验证再决定 |
| "别人怎么做" | 每个人情况不同，专注自己的验证 |
| "老板会不会失去耐心" | 定期汇报进展，管理期望 |

### 6. 里程碑示例

**终极目标**：做一个年营收1000万的在线职业规划平台

```
M3: 年营收1000万
    ↓ 需要什么？
M2: 稳定的付费用户群体（≥1万付费用户）
    ↓ 需要什么？
M1: 验证跑通的服务产品（≥1000付费用户）
    ↓ 需要什么？
H0: 用户对付费职业规划服务有真实需求（≥100用户愿意付费尝试）
```

## 从0起步项目检查清单

在每个阶段开始前，检查以下事项：

- [ ] 终极目标是否清晰？
- [ ] 关键里程碑是否倒推清楚？
- [ ] 第一个假设是否明确？
- [ ] 验证方案是否具体可执行？
- [ ] 成功/失败标准是否清晰？
- [ ] 资源配置是否与阶段匹配？
