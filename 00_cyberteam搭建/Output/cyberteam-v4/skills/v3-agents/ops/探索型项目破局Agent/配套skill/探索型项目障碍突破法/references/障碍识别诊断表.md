# 障碍识别诊断表

## 模板说明

此模板用于快速识别探索型项目遇到的障碍属于哪种类型，是匹配正确突破方法的前提。

## 诊断问题清单

根据你当前的困境，回答以下问题：

### Q1: 最让你困扰的是什么？

| 选项 | 描述 | 对应障碍类型 |
|------|------|-------------|
| A | 不确定方向是不是对的，想换个方向试试 | 方向迷雾 |
| B | 预算/人手/时间不够，不知道怎么推进 | 资源困境 |
| C | 验证方法试了很多，假设还是无法验证 | 验证僵局 |
| D | 团队士气低落，有人想离职/放弃 | 团队动摇 |
| E | 老板/上级给的压力太大，要求看到结果 | 外部压力 |

### Q2: 这个问题的触发点是什么？

| 选项 | 描述 | 对应障碍类型 |
|------|------|-------------|
| A | 尝试了某个方向但效果不好 | 方向迷雾 |
| B | 项目预算被砍/人员被调走 | 资源困境 |
| C | 验证了3次以上还没结论 | 验证僵局 |
| D | 有人私下表达了不满/想离职 | 团队动摇 |
| E | 老板/上级最近追问进展 | 外部压力 |

### Q3: 之前尝试过什么解决方法？

| 选项 | 描述 | 对应障碍类型 |
|------|------|-------------|
| A | 换了个方向/调整了策略 | 方向迷雾 |
| B | 砍掉了部分功能/降低了标准 | 资源困境 |
| C | 换了验证方法/增加了样本量 | 验证僵局 |
| D | 开会鼓舞士气/做了团队建设 | 团队动摇 |
| E | 做了更多汇报材料/加强了沟通 | 外部压力 |

### Q4: 如果这个问题不解决，项目会怎样？

| 选项 | 描述 | 对应障碍类型 |
|------|------|-------------|
| A | 会继续在错误方向上浪费时间 | 方向迷雾 |
| B | 会因为资源不足而停滞 | 资源困境 |
| C | 会一直卡在这里无法推进 | 验证僵局 |
| D | 团队会进一步流失 | 团队动摇 |
| E | 会被叫停或失去支持 | 外部压力 |

## 诊断规则

- **A选项居多** → 方向迷雾
- **B选项居多** → 资源困境
- **C选项居多** → 验证僵局
- **D选项居多** → 团队动摇
- **E选项居多** → 外部压力

**注意**：如果选项分散，可能是多种障碍同时存在，需要优先级排序

## 多障碍并存时的优先级

当多种障碍并存时，按以下优先级处理：

| 优先级 | 障碍类型 | 处理原则 |
|-------|---------|---------|
| P0 | 团队动摇 | 人心散了什么都推不动，必须先解决 |
| P1 | 方向迷雾 | 方向不对越努力越错，必须先确认 |
| P2 | 外部压力 | 如果上级不认可，项目没有资源支持 |
| P3 | 资源困境 | 在确认方向后优化资源配置 |
| P4 | 验证僵局 | 最后解决，用正确的方法突破 |

## 障碍自我检测清单

如果你不确定自己是否遇到了障碍，可以用这个清单自检：

| 检测项 | 是 | 否 |
|-------|----|----|
| 过去两周没有实质进展 | □ | □ |
| 团队成员私下抱怨增加 | □ | □ |
| 核心假设无法验证 | □ | □ |
| 预算或时间被压缩 | □ | □ |
| 关键干系人态度变负面 | □ | □ |

**如果有3个以上"是"**，说明项目遇到了障碍，需要诊断和处理。
