# 漏斗分析方法

## 漏斗构建步骤

### Step 1: 定义转化路径

列出用户从首次接触到完成转化的所有步骤。

### Step 2: 收集各步骤数据

获取每个步骤的用户数量或事件数。

### Step 3: 计算各环节转化率

```
环节转化率 = 下一环节用户数 / 本环节用户数 × 100%
```

### Step 4: 识别异常环节

与行业基准或历史数据对比，找出异常低的环节。

### Step 5: 分析流失原因

针对异常环节，分析用户流失的可能原因。

---

## 流失分析框架

| 流失原因类别 | 具体原因 | 分析方法 |
|-------------|---------|---------|
| 流量质量 | 引来的用户不精准 | 检查定向设置 |
| 物料问题 | Banner/文案不吸引 | A/B测试 |
| 路径设计 | 步骤太多或逻辑不清 | 用户行为录制 |
| 信任问题 | 用户对产品有顾虑 | 用户访谈 |
| 决策成本 | 价格/复杂度太高 | 竞品对比分析 |

---

## 优化优先级

### 优先级判断矩阵

| 影响度 | 实施难度 | 优先级 |
|--------|----------|--------|
| 高 | 低 | P0 |
| 高 | 高 | P1 |
| 低 | 低 | P2 |
| 低 | 高 | P3 |

### 决策原则

1. **先优化高影响、低难度的环节**
2. **再优化高影响、高难度的环节**
3. **低影响的不优化或延后**
