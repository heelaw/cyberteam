# 单体经济模型模块

## 模块概述

本模块提供单体经济模型(Unit Economics)分析的标准方法和工具，用于评估单个客户的盈利能力和商业模式的健康度。

## 核心指标

### 1. CAC (Customer Acquisition Cost)

**客户获取成本**

```
CAC = 总获客成本 / 新增客户数

总获客成本包括：
├── 营销广告费
├── 销售团队佣金
├── 营销工具费用
└── 其他相关费用
```

**计算示例**：
```yaml
营销花费: $10,000
销售佣金: $5,000
工具费用: $1,000
────────────────────
总成本: $16,000

新增客户: 200人
────────────────────
CAC = $16,000 / 200 = $80
```

**注意事项**：
- 排除现有客户维护成本
- 按渠道分别计算更准确
- 包含所有直接和间接成本
- 定期更新确保数据准确

### 2. LTV (Lifetime Value)

**客户生命周期价值**

**计算方法1：简单法**
```
LTV = ARPU × 客户生命周期

ARPU = 月/年收入
客户生命周期 = 1 / 流失率
```

**计算方法2：精细法**
```
LTV = (每月收入 - 每月服务成本) × 毛利率 × 客户生命周期

其中：
- 每月服务成本不包括获客成本
- 毛利率 = (收入 - 服务成本) / 收入
- 客户生命周期 = 1 / 月流失率
```

**计算示例**：
```yaml
方法1:
  ARPU: $100/月
  月流失率: 5%
  生命周期: 1 / 0.05 = 20月
  LTV = $100 × 20 = $2,000

方法2:
  月收入: $100
  服务成本: $30/月
  毛利率: 70%
  月流失率: 5%
  LTV = ($100 - $30) × 0.70 × 20 = $980
```

**行业基准**：

| 行业 | LTV:CAC比率 | 回本周期 |
|------|------------|----------|
| SaaS | > 3:1 | < 12个月 |
| 电商 | > 3:1 | < 6个月 |
| 移动应用 | > 2:1 | < 3个月 |
| O2O服务 | > 3:1 | < 6个月 |

### 3. 回本周期 (Payback Period)

**收回获客成本所需时间**

```
回本周期 = CAC / (ARPU × 毛利率)
```

**示例**：
```yaml
CAC: $150
ARPU: $50/月
毛利率: 60%
────────────────────────────
回本周期 = $150 / ($50 × 0.60) = 5个月
```

**健康标准**：
- ✅ 优秀：< 6个月
- 🟡 良好：6-12个月
- ⚠️ 需关注：12-18个月
- ❌ 不健康：> 18个月

### 4. 边际贡献

**每个客户的毛利贡献**

```
边际贡献 = 收入 - 服务成本 - 支付成本 - 销售成本
边际贡献率 = 边际贡献 / 收入
```

**用途**：
- 评估定价策略
- 计算盈亏平衡点
- 决定营销预算

## 分析框架

### 1. 健康度评估

**LTV:CAC 比率解读**：

```
比率 < 1: ❌ 严重亏损
    每获得一个客户都在亏钱，商业模式不可持续

比率 = 1-2: ⚠️ 亏损边缘
    收回成本需要很长时间，风险较高

比率 = 2-3: 🟡 基本健康
    可以盈利，但增长受限

比率 = 3-5: ✅ 健康
    良好的盈利能力，可持续增长

比率 > 5: 🚀 优秀
    非常健康的商业模式，可以激进增长
```

### 3. 敏感性分析

**关键变量影响**：

```
LTV:CAC = f(ARPU, 流失率, CAC, 毛利率)

变量变化对 LTV:CAC 的影响：
├── ARPU +10% → LTV:CAC +15%
├── 流失率 -10% → LTV:CAC +12%
├── CAC -10% → LTV:CAC +11%
└── 毛利率 +5% → LTV:CAC +8%
```

**优化杠杆排序**：

| 优化方向 | 影响程度 | 实施难度 | 预期效果 | 优先级 |
|---------|----------|----------|----------|--------|
| 提升ARPU | 高 | 中 | +15-20% | P1 |
| 降低流失率 | 高 | 高 | +12-18% | P2 |
| 降低CAC | 中 | 低 | +10-15% | P1 |
| 提升毛利率 | 中 | 中 | +8-12% | P3 |

## 应用场景

### 场景1：评估获客渠道

**决策矩阵**：

```
         LTV:CAC < 3    LTV:CAC 3-5    LTV:CAC > 5
CAC低    维持现状       加大投入       激进扩张
CAC高    优化或放弃     谨慎测试       小规模尝试
```

### 场景2：定价决策

**分析框架**：

```python
# 当前定价
current_price = $100
current_arpu = $100
current_ltv = $2000
current_ltv_cac = 2.5  # 不健康

# 提价10%
new_price = $110
new_arpu = $110
# 假设流失率从5%增加到6%
new_churn = 0.06
new_ltv = $110 * (1/0.06) = $1833
# 假设CAC不变
new_ltv_cac = $1833 / $80 = 22.9  # 大幅改善

结论：提价虽然会轻微影响流失率，
但整体LTV:CAC比率大幅提升
```

### 场景3：融资准备

**投资人关注的问题**：

```
Q1: 你们的单体经济模型如何？
A: LTV是$2000，CAC是$150，
   LTV:CAC比率是13.3:1，回本周期4个月

Q2: 增长的同时还能保持吗？
A: 是的，我们的边际成本很低，
   规模化后CAC会进一步下降

Q3: 毛利率如何？
A: 70%毛利率，边际贡献率40%
```

## 常见误区

### 误区1：忽略时间维度

**问题**：用短期数据计算LTV
**正确**：使用足够长的观察期（通常6-12个月）

### 误区2：混淆相关性

**问题**：认为降低CAC就能改善整体
**正确**：LTV和CAC需要综合考虑，有时高CAC是值得的

### 误区3：忽视细分

**问题**：用平均值掩盖问题
**正确**：细分客户和渠道，找出隐藏的机会和风险

### 误区4：静态思维

**问题**：假设指标不变
**正确**：考虑规模效应和学习曲线的影响

## 最佳实践

### 1. 建立监控体系

```yaml
实时监控:
  - 每日新增客户数
  - 每日获客成本
  - 每日活跃客户数

周度分析:
  - 各渠道CAC和LTV
  - 客户留存变化
  - ARPU趋势

月度复盘:
  - 整体LTV:CAC比率
  - 回本周期变化
  - 优化措施效果
```

### 2. 设定预警阈值

```yaml
红色警报:
  - LTV:CAC < 2:1
  - 回本周期 > 12个月
  - 流失率 > 10%/月

黄色预警:
  - LTV:CAC 2-3:1
  - 回本周期 9-12个月
  - 流失率 5-10%/月

绿色健康:
  - LTV:CAC > 3:1
  - 回本周期 < 9个月
  - 流失率 < 5%/月
```

### 3. 持续优化循环

```
1. 测量 → 收集准确数据
2. 分析 → 找出问题和机会
3. 优化 → 实施改进措施
4. 验证 → 评估效果
5. 迭代 → 持续改进
```
