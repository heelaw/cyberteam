# user-stratifier

## 简介

对用户进行分层，差别化制定运营策略。区分核心用户和普通用户，制定不同运营方式。

## 触发关键词

```
用户分层、用户分类、用户划分、核心用户、RFM模型
```

## 触发场景描述

```
当需要对用户进行精细化运营时触发：
- 需要区分高价值用户
- 需要制定差异化策略
- 用户体量大无法统一运营
- 需要提升运营效率
```

## 输入

### 必需输入

| 字段 | 类型 | 说明 |
|------|------|------|
| user_data | object | 用户数据 |
| stratification_purpose | string | 分层目的 |

### 可选输入

| 字段 | 类型 | 说明 |
|------|------|------|
| existing_segments | array | 已有分层 |

## 方法论：用户分层方法

### 常见分层维度

| 维度 | 说明 |
|------|------|
| 价值分层 | 高/中/低价值 |
| 行为分层 | 活跃/沉默/流失 |
| 生命周期 | 新用户/成长/成熟/衰退 |
| RFM | 最近一次消费/消费频率/消费金额 |

### 策略匹配原则

- 高价值用户：重点维护
- 潜力用户：重点培育
- 普通用户：标准运营
- 流失风险用户：重点挽回

### 禁止行为

1. 禁止分层标准过于复杂难以执行
2. 禁止只分层不制定差异化策略
3. 禁止忽视沉默用户群体
4. 禁止分层体系长期不更新
5. 禁止用单一维度进行用户分层
