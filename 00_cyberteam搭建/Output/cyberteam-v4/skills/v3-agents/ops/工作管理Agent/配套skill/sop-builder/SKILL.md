# sop-builder

## 简介

建立和使用标准化操作流程（SOP）。通过标准化流程提升工作效率，确保工作质量一致性。

## 触发关键词

```
SOP建立、SOP编写、标准化流程、操作手册、工作流程、流程规范
```

## 触发场景描述

```
当需要建立或优化工作流程时触发：
- 需要将重复性工作标准化
- 需要培训新人快速上手
- 需要确保工作质量一致性
- 需要提升团队协作效率
```

## 输入

### 必需输入

| 字段 | 类型 | 说明 |
|------|------|------|
| work_description | string | 工作内容描述 |

## 方法论：SOP建立步骤

### 1. 明确目的

- 确定SOP要解决什么问题
- 设定标准化目标

### 2. 梳理流程

- 记录当前流程
- 识别关键步骤
- 找出问题点

### 3. 优化流程

- 删除不必要步骤
- 合并相似步骤
- 简化复杂步骤

### 4. 编写SOP

- 使用统一模板
- 步骤清晰可执行
- 明确责任人和标准

### 5. 验证更新

- 试运行验证
- 收集反馈优化
- 定期更新维护

## Critical Rules

### 必须遵守

1. 步骤必须有明确的动作和产出
2. 必须有明确的责任人
3. 必须有可衡量的完成标准
4. 必须包含异常处理

### 禁止行为

1. 禁止步骤描述模糊
2. 禁止没有检查点
3. 禁止SOP一成不变
4. 禁止复杂流程不简化
5. 禁止SOP与实际执行脱节

## 评估清单

### 评估维度

| 维度 | 标准 | 权重 |
|------|------|------|
| 完整性 | 包含所有必要步骤 | 25% |
| 可执行性 | 步骤清晰可执行 | 30% |
| 标准化 | 有明确的完成标准 | 25% |
| 可维护性 | 易于更新维护 | 20% |
