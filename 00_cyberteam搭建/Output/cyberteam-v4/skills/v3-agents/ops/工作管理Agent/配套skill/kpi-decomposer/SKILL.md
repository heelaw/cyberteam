# kpi-decomposer

## 简介

将业务KPI合理拆分为可执行的团队任务。通过指标拆解确保团队按时高效完成目标。

## 触发关键词

```
指标拆解、KPI拆分、目标分解、业务指标、团队KPI、任务分配
```

## 触发场景描述

```
当需要将业务目标分解到团队任务时触发：
- 需要将公司目标分解到团队
- 需要将季度目标拆解到月度/周
- 需要将指标分配到个人
- 需要设定可执行的里程碑
```

## 输入

### 必需输入

| 字段 | 类型 | 说明 |
|------|------|------|
| overall_kpi | string | 总体KPI目标 |
| current_baseline | object | 当前基础数据 |
| team_structure | object | 团队结构 |

### 可选输入

| 字段 | 类型 | 说明 |

## 方法论：KPI拆解方法

### 1. 理解总体目标

- 明确KPI的定义和计算方式
- 了解当前基础和目标差距
- 确定时间周期

### 2. 拆解维度

- 按时间：年→季度→月→周
- 按团队：总体→部门→个人
- 按渠道：总→各渠道
- 按用户：总→各分层

### 3. 设定里程碑

- 关键时间节点
- 阶段目标
- 缓冲时间

### 4. 任务映射

- 指标→关键任务
- 任务→责任人
- 设定可交付标准

## Critical Rules

### 必须遵守

1. 拆解后的目标之和等于总体目标
2. 每个任务必须有明确的指标
3. 必须有时间节点
4. 必须有责任人

### 禁止行为

1. 禁止拆解后目标大于总体
2. 禁止任务没有对应指标
3. 禁止没有里程碑
4. 禁止指标无法衡量
5. 禁止忽视团队实际能力

## 评估清单

### 评估维度

| 维度 | 标准 | 权重 |
|------|------|------|
| 目标一致性 | 分解后等于总体目标 | 30% |
| 可执行性 | 任务有明确指标 | 30% |
| 责任清晰 | 任务到人 | 20% |
| 时间合理 | 有里程碑和缓冲 | 20% |
