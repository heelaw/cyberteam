# KPI拆解方法论

## 核心概念定义

KPI拆解是将公司战略目标逐级分解为可执行、可衡量指标的过程。好的KPI体系需要实现"上下对齐、左右协同"，确保每个团队和个人都对公司目标做出贡献。

## 方法论步骤

### 第一阶段：目标澄清（Clarification）
1. 明确公司级战略目标（BSC四个维度）
2. 识别关键成功因素（KSF）
3. 确定目标值和挑战值
4. 设定评估周期

### 第二阶段：指标分解（Decomposition）
1. 采用瀑布式分解：从公司→部门→团队→个人
2. 遵循"MECE"原则：相互独立，完全穷尽
3. 确定指标权重：聚焦核心，避免平均分配
4. 定义计算口径：确保数据可获取、可验证

### 第三阶段：落地对接（Alignment）
1. 与执行者沟通可行性
2. 识别指标间的因果关系
3. 建立数据追踪机制
4. 制定改进行动计划

## 关键成功因素

| 因素 | 说明 |
|------|------|
| 战略对齐 | 指标必须能追溯到战略目标 |
| 可控性 | 执行者对指标有一定控制力 |
| 可衡量 | 数据来源明确，计算逻辑清晰 |
| 时效性 | 至少每月追踪一次 |

## 常见误区

1. **唯数字论** - 追求数字漂亮而忽视实际业务健康度
2. **指标过载** - 设置过多KPI导致注意力分散
3. **分解一刀切** - 用同一逻辑分解不同类型指标
4. **只下不下** - 分解后缺少资源和支持
