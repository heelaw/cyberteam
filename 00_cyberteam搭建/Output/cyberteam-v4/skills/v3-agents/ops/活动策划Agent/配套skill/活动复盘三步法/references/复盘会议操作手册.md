## 二、会议流程

### 2.1 会议时间分配

| 环节 | 时长 | 主持 | 内容 |
|------|------|------|------|
| 开场 | 5分钟 | 活动负责人 | 会议目的和议程 |
| 目标回顾 | 10分钟 | 活动负责人 | 原始目标确认 |
| 数据分析 | 15分钟 | 数据负责人 | 结果数据分析 |
| 归因讨论 | 25分钟 | 全体 | 成功/失败因素分析 |
| 结论提炼 | 15分钟 | 全体 | 核心结论和建议 |
| 任务分配 | 10分钟 | 活动负责人 | 后续行动项 |
| 总结 | 5分钟 | 活动负责人 | 会议总结 |

**总时长建议**：1.5小时

### 2.2 各环节详细流程

#### 开场（5分钟）

```
主持人开场：
1. 说明复盘目的
2. 介绍议程安排
3. 强调复盘原则（开放、客观、不追责）
4. 预计结束时间
```

#### 目标回顾（10分钟）

```
内容清单：
1. 活动原始目标是什么？
2. 目标拆解到哪些渠道/阶段？
3. 当时的假设和前提条件？
4. 如何定义活动成功？

确认：
□ 全体对目标达成共识
□ 没有争议后进入下一环节
```

#### 数据分析（15分钟）

```
汇报内容：
1. 各指标目标vs实际对比
2. 完成率分析
3. 关键数据发现（超额/未完成）
4. 异常数据点标注

讨论：
□ 数据解读是否有争议？
□ 需要补充的数据？
□ 数据背后的原因？
```

#### 归因讨论（25分钟）

```
讨论框架：
1. 成功因素（目标超额完成时）
   - 哪些因素贡献最大？
   - 这些因素可控吗？
   - 下次如何复制？

2. 失败因素（目标未完成时）
   - 根本原因是什么？
   - 是策略问题还是执行问题？
   - 是内部问题还是外部问题？

讨论规则：
- 用"5个为什么"深挖根因
- 不追责、不甩锅
- 聚焦"如何改进"而非"谁的责任"
```

#### 结论提炼（15分钟）

```
产出：
1. 核心结论（2-3条）
   - 成功规律
   - 失败教训
   - 方法论更新

2. 可执行建议（3-5条）
   - 按优先级排序
   - 明确负责人
   - 设置执行时间
```

#### 任务分配（10分钟）

```
每个行动项需要明确：
□ 行动内容（做什么）
□ 负责人（谁来做）
□ 参与人（谁配合）
□ 完成时间（什么时候）
□ 预期效果（做到什么程度）
```

## 三、会议主持技巧

### 3.1 常见问题应对

| 问题 | 应对策略 |
|------|----------|
| 讨论偏离主题 | 拉回主题："我们先记录这个，会后单独讨论" |
| 争论激烈 | 暂停争论："我们先记录两种观点，会后验证" |
| 沉默不语 | 点名提问："XX，你对这个怎么看？" |
| 流于形式 | 追问深挖："这个结论能指导什么具体行动？" |
| 互相推诿 | 聚焦改进："过去的不追究，我们看未来怎么做" |

### 3.2 引导技巧

```
追问技巧：
- "能举个例子吗？"
- "这个结论的证据是什么？"
- "如果要做，下一步是什么？"
- "还有其他的吗？"

总结技巧：
- "我理解你的意思是...，对吗？"
- "我们来确认一下结论..."
- "如果用一句话总结今天的收获是..."
```
