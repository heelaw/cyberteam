# 目标分解结构图与操作手册

## 一、目标分解基本概念

### 什么是目标分解
将一个大的、模糊的目标，通过一定的逻辑，拆解为多个小的、具体的、可执行的目标或行动。

### 目标分解的目的
1. **可管理**：大目标无法直接管理，分解后才能跟踪
2. **可执行**：一线员工需要具体动作，分解后才能执行
3. **可衡量**：分解后的目标有明确的衡量标准
4. **可分配**：分解后才知道谁该负责什么

### 目标分解的原则
1. **支撑原则**：每个分解后的目标都必须支撑上级目标
2. **独立原则**：分解后的目标相互独立，不重复
3. **可管理原则**：分解到可管理的颗粒度
4. **责任明确原则**：每个分解后的目标都有明确责任人

## 二、三维度分解法

### 维度1: 时间维度
```
年度目标
    ↓
季度目标 → Q1 → Q2 → Q3 → Q4
    ↓
月度目标
    ↓
周度目标
    ↓
日/日常行动
```

**适用场景**：
- 有明确时间周期的目标
- 需要阶段性交付物的目标
- 里程碑明确的项目

### 维度2: 空间维度（组织维度）
```
公司目标
    ↓
部门目标
    ↓
团队目标
    ↓
个人目标
```

**适用场景**：
- 需要多层级分担的目标
- 有明确组织架构的组织
- 强调责任归属的目标

### 维度3: 逻辑维度（要素维度）
```
最终目标
    ↓
中间里程碑
    ↓
关键结果
    ↓
起始行动
```

**适用场景**：
- 复杂的业务目标
- 需要逐步验证的目标
- 有内在逻辑顺序的目标

## 五、操作手册

### Step 1: 确定分解维度
根据目标特点选择分解维度：
- 有明确时间节点 → 从时间维度分解
- 需要多层级分担 → 从空间维度分解
- 复杂的业务目标 → 从逻辑维度分解
- 复杂情况 → 多维度结合

### Step 2: 执行分解
1. 写出上级目标
2. 识别支撑上级目标的关键要素
3. 将关键要素转化为下级目标
4. 重复直到颗粒度合适

### Step 3: 检查分解质量
检查清单：
- [ ] 每个下级目标都能支撑上级目标吗？
- [ ] 下级目标之间相互独立吗？
- [ ] 每个下级目标都可衡量吗？
- [ ] 每个下级目标都有责任人吗？
- [ ] 分解的颗粒度合适吗？（不太粗也不太细）

### Step 4: 补充行动和资源
- 为每个目标补充具体行动
- 明确行动的时间节点
- 评估资源需求

## 六、分解常见问题

### Q1: 目标分解太细怎么办？
**判断标准**：如果分解后的目标不能再拆解出有意义的子目标，就说明够了。

**调整方法**：合并相似的下级目标

### Q2: 目标分解太粗怎么办？
**判断标准**：如果一个下级责任人无法独立完成目标，需要进一步分解。

**调整方法**：按照"责任到人"原则继续分解

### Q3: 目标之间有重叠怎么办？
**调整方法**：重新梳理逻辑，确保MECE（相互独立，完全穷尽）

### Q4: 有些目标无法分解怎么办？
**可能原因**：上级目标本身有问题

**调整方法**：回到上级，重新审视目标本身是否合理
