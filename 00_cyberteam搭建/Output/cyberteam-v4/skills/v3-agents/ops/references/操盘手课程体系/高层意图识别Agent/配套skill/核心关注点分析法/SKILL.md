# 核心关注点分析法

## Skill概述

核心关注点分析法帮助老板/高层识别当前阶段最重要的问题，理解资源应该投入哪里。

```json
{
  "type": "object",
  "required": ["company_context", "current_question"],
  "properties": {
    "company_context": {
      "type": "object",
      "properties": {
        "industry": {"type": "string"},
        "stage": {"type": "string"},
```

## 使用示例

### 示例1：成长期公司老板困惑

**输入**:
```
公司：B2B SaaS公司，A轮，60人
困惑：要不要招更多销售扩大规模，还是先打磨产品？
```

**输出**:
```
当前阶段：成长期
问题层级：战略层
优先级：P0

核心关注点：验证商业模式可行性

理由：
1. A轮核心任务是验证商业模式，非规模扩张
2. 60人规模需关注组织效率，非单纯加人
3. 产品打磨是规模化的前提，否则CAC/LTV不健康

行动建议：
1. 先优化产品到NPS>40
2. 验证销售SOP可复制性
3. 确认CAC/LTV健康后再扩招销售
```

## 错误处理

| 错误场景 | 处理方式 |
|----------|----------|
| 信息不足 | 主动询问：公司阶段、团队规模、核心困惑 |
| 多个问题并列 | 使用优先级矩阵排序，给出P0/P1/P2 |
| 问题模糊 | 帮助澄清问题本质，指出真正的问题所在 |
| 资源约束未知 | 询问资源约束情况，给出可行性建议 |

## 独特个性

本Skill采用**"阶段×层级×优先级"三维定位法**，将老板的模糊困惑转化为清晰的优先级排序，避免"什么都重要就是什么都不重要"的困境，特别适合资源有限的成长期企业。

## 分析框架

### 阶段视角

不同阶段的核心关注点不同：

| 阶段 | 核心关注点 |
|------|------------|
| 初创期 | 找到PMF，活下来 |
| 成长期 | 规模化增长，占市场 |
| 成熟期 | 效率提升，稳增长 |
| 转型期 | 找到新增长点 |

### 问题层级

- **战略层**：方向、模式、竞争
- **业务层**：产品、市场、渠道
- **组织层**：团队、能力、文化
- **执行层**：效率、流程、结果

### 优先级矩阵

| 维度 | 重要 | 不重要 |
|------|------|--------|
| 紧急 | 立即做 | 授权做 |
| 不紧急 | 计划做 | 不做 |

## 工作流程

### Step 1：了解背景

了解公司当前的基本情况。

### Step 2：识别阶段

判断公司当前的生命周期阶段。

### Step 3：分析问题层级

识别问题属于哪个层级。

### Step 4：确定优先级

使用优先级矩阵确定关注点。

### Step 5：明确核心命题

给出当前最核心的关注点。

## Critical Rules

### 必须遵守

1. 基于阶段判断核心关注点
2. 区分问题层级
3. 给出明确优先级
4. 考虑企业的实际情况
5. 给出具体的行动建议

### 禁止行为

1. 不凭感觉给建议
2. 不脱离阶段背景
3. 不混淆优先级
4. 不给模糊的建议
5. 不忽视资源约束
