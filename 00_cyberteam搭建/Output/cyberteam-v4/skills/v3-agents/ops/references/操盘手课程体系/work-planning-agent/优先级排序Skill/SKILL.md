## 优先级排序方法论

### 1. 艾森豪威尔矩阵

```
                    紧急                不紧急
            ┌─────────────────┬─────────────────┐
     重要    │   立即做         │   计划做         │
            │  (危机、deadline) │  (规划、成长)    │
高层+--------┼─────────────────┼─────────────────┤
            │   委托做         │   不做           │
   不重要   │  (琐事、干扰)    │  (浪费时间)      │
            └─────────────────┴─────────────────┘
```

### 2. ICE评分法

| 维度 | 说明 | 打分(1-10) |
|------|------|------------|
| **I**mpact 影响 | 完成后带来的价值 | 1-10 |
| **C**onfidence 信心 | 对完成目标的信心程度 | 1-10 |
| **E**ase 容易 | 完成的难易程度 | 1-10 |

**总分 = Impact + Confidence + Ease**
**排序：高分优先**

### 3. 二八法则

```
80%的价值来自20%的任务
识别那20%的关键任务，优先完成
```

## 分析流程

### Step 1: 收集任务清单

**处理**：
- 列出所有待办任务
- 标注每个任务的截止时间
- 标注每个任务的预期价值

**输出**: 完整任务清单

### Step 2: 应用排序方法

**处理**：
- 用艾森豪威尔矩阵分类
- 用ICE法打分
- 用二八法则识别关键任务

**输出**: 优先级分类结果

### Step 3: 确认排序结果

**处理**：
- 验证排序逻辑
- 调整冲突任务
- 确认最终排序

**输出**: 优先级排序清单

## 优先级排序示例

### 示例1：日常工作任务

```
任务清单：
- 写Q2季度报告（周五截止）
- 回复客户邮件（紧急）
- 参加团队例会（周三）
- 优化工作流程（重要不紧急）
- 处理报销单据（琐事）

排序结果：
1. 写季度报告（紧急+重要，ICE: 9+8+7=24）
2. 回复客户邮件（紧急，ICE: 8+9+9=26）
3. 团队例会（固定时间，ICE: 7+8+10=25）
4. 优化流程（重要不紧急，ICE: 9+6+5=20）
5. 处理报销（琐事，ICE: 3+10+10=23，但价值低）
```

### 示例2：项目任务

```
项目任务：
- 完成核心功能开发（影响大）
- 修复严重bug（紧急）
- 写技术文档（重要不紧急）
- 优化代码性能（可延后）

排序结果：
1. 核心功能开发（ICE: 10+8+5=23）
2. 修复bug（ICE: 9+9+8=26）
3. 技术文档（ICE: 7+7+8=22）
4. 代码优化（ICE: 5+6+7=18）
```

## Critical Rules

1. **重要且紧急优先**: 这是铁律，不能违背

2. **价值优先**: 优先做价值高的任务

3. **考虑截止时间**: 紧急任务有deadline压力

4. **避免陷入琐事**: 不要被不重要的事占满时间

5. **定期重排**: 优先级会随时间变化，需要重新评估

---

## 禁止行为

1. **禁止按喜好排序**: 不能想做哪个做哪个

2. **禁止忽视紧急**: 紧急任务即使不重要也要处理

3. **禁止满负荷**: 不要排到100%时间，留有余量

4. **禁止一成不变**: 优先级不是固定的，要动态调整

5. **禁止只排不做**: 排序只是开始，执行才是关键

## 输入输出

### 必要输入
- 待排序的任务清单

- 优先级分类（艾森豪威尔矩阵）
- ICE评分排序表
- 最终执行顺序
- 时间分配建议
