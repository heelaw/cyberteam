# process-optimizer

## 简介

分析和优化工作流程，通过流程改进提升效率和质量。

## 触发关键词

```
流程优化、流程改进、效率提升、质量改进、工作改进
```

## 触发场景描述

```
当需要优化现有流程时触发：
- 流程效率低下需要改进
- 质量不稳定需要控制
- 成本过高需要降低
- 需要消除浪费
```

## 输入

### 必需输入

| 字段 | 类型 | 说明 |
|------|------|------|
| current_process | string | 当前流程 |
| pain_points | array | 问题点 |

### 可选输入

| 字段 | 类型 | 说明 |
|------|------|------|
| benchmark_process | string | 标杆流程 |

## 方法论：流程优化

### 1. 识别浪费

- 不必要的步骤
- 等待时间
- 重复工作
- 信息流转延迟

### 2. 分析瓶颈

- 找出流程中最慢的环节
- 分析阻塞原因

### 3. 设计优化

- 删除不必要步骤
- 简化复杂步骤
- 并行化处理
- 自动化替代

### 4. 验证效果

- 对比优化前后
- 量化改进效果

## Critical Rules

### 必须遵守

1. 识别最关键的瓶颈
2. 量化优化效果
3. 确保可执行性
4. 平衡效率和质量

### 禁止行为

1. 禁止只优化表面问题
2. 禁止破坏性变更
3. 禁止忽视员工感受
4. 禁止优化后不验证效果
5. 禁止照搬其他公司流程
