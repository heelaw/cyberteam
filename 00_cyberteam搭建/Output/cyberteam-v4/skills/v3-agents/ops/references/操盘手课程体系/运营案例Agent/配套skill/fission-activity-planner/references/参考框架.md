# fission-activity-planner 参考资料

## 1. 裂变活动类型分类

```
类型1: 抽奖型
代表: 支付宝中国锦鲤
特点: 单个大奖，高诱惑，低成本
适用: 品牌传播、话题造势

类型2: 红包型
代表: 拼多多天天领现金
特点: 人人有份，阶梯奖励，损失厌恶
适用: 快速拉新、用户增长

类型3: 任务型
代表: 滴滴追风竞速赛
特点: 完成任务获取奖励，门槛清晰
适用: 精准拉新、提升活跃

类型4: 互惠型
代表: 微信读书互赠书币
特点: 双向获利，社交关系驱动
适用: 社交产品、有好友关系的产品

类型5: 竞赛型
代表: 早起打卡挑战赛
特点: 竞争驱动，排行榜机制
适用: 提升留存、活跃
```

## 2. 激励设计原则

```
1. 即时反馈: 每一步操作都要有反馈
2. 损失厌恶: 接近目标时的激励最强
3. 社交货币: 奖励要有展示价值和话题性
4. 稀缺感: 限量/限时/名额限制
```

## 3. 传播节奏三阶段

```
预热期(1-3天): 制造悬念，引发期待
爆发期(1-3天): 最大化传播和参与
持续期(活动进行中): 维持热度，形成长尾
```

## 4. 风险清单

```
P0级风险:
1. 羊毛党风险 → 账号/设备/行为限制
2. 兑现风险 → 提前备货/设置上限/明确规则
3. 负面舆情 → 客服话术/舆情监控/应急响应

P1级风险:
4. 传播效果不及预期 → KOL储备/备选渠道
5. 技术故障 → 技术预案/灰度发布/实时监控
```
