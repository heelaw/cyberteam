### 3. 命名规范

#### 文件命名
- **小写字母** + **下划线**
- 使用**英文命名**（便于搜索）
- 描述性名称，见名知意

| 类型 | 命名示例 | 说明 |
|-----|---------|-----|
| 案例文档 | `xiaohongshu_koc_campaign.md` | 平台_类型_案例 |
| 工具文档 | `douyin_ads_guide.md` | 平台/工具_指南 |
| 行业文档 | `ecommerce_metrics_2025.md` | 行业_主题_年份 |
| 模板文档 | `promotion_plan_template.md` | 类型_template |

#### 标题规范
- 标题使用**中文**（便于阅读）
- 一级标题：# 文档主标题
- 二级标题：## 主要章节
- 三级标题：### 子章节

---

### 4. 内容质量标准

#### ✅ 必须包含
- **概述/背景**：让读者快速理解内容
- **结构清晰**：使用标题层级和列表
- **实用性强**：可操作的具体内容
- **数据支撑**：有数据或案例支撑观点

#### ⚠️ 避免内容
- 过于理论化缺乏实操
- 信息过时未标注时间
- 没有明确结构的堆砌
- 缺少来源的不实信息

## 🎯 内容优先级建议

### 高优先级（建议优先添加）
1. **成功案例库**：不同行业、不同渠道的真实案例
2. **广告平台指南**：抖音、小红书、百度等主流平台
3. **数据模板**：方案模板、报告模板、检查清单
4. **行业基准数据**：转化率、客单价、ROI等行业标准

### 中优先级
1. **工具使用指南**：分析工具、投放工具
2. **失败案例分析**：从反面学习经验
3. **术语词典**：统一概念理解

### 低优先级
1. 理论研究文章
2. 过时的案例分析
3. 过于泛泛的行业资讯
