## 填写示例

### 基本信息

| 项目 | 内容 |
|------|------|
| 复盘主题 | Q1用户调研项目 |
| 复盘时间 | 2026-03-15 |
| 参与人员 | 张三、李四、王五 |
| 项目周期 | 2026-01-01 - 2026-02-28 |

### 目标回顾

**原始目标**: 完成20份深度用户访谈，形成用户洞察报告
**调整情况**: 无调整
**目标达成度**: 110%（完成22份）

### 结果评估

| 指标 | 目标值 | 实际值 | 达成率 |
|------|--------|--------|--------|
| 访谈数量 | 20份 | 22份 | 110% |
| 报告质量评分 | ≥85分 | 92分 | 108% |
| 建议采纳率 | ≥50% | 65% | 130% |

### 过程分析

**做得好的地方（Keep）**:
| 成功因素 | 具体表现 | 可复用性 |
|----------|----------|----------|
| 预约话术优化 | 提前设计并测试预约话术，转化率提升30% | 高 |
| 访谈技巧培训 | 统一培训访谈技巧，保证访谈质量一致性 | 高 |

**需要改进的地方（Problem）**:
| 问题点 | 根本原因 | 严重程度 |
|--------|----------|----------|
| 访谈周期过长 | 预约流程分散，没有集中执行 | 中 |
| 用户池偏窄 | 仅使用现有用户，新用户触达不足 | 中 |

### 经验总结

**方法论层面**: 用户洞察需要多渠道交叉验证
**流程层面**: 访谈执行应集中时间段，提高效率
**工具层面**: 使用在线表单收集用户基础信息，节省访谈时间

### 改进行动

**短期改进（本月内）**:
| 行动项 | 责任人 | 完成时间 | 验证方式 |
|--------|--------|----------|----------|
| 整理访谈话术模板 | 张三 | 3月20日 | 模板审核通过 |
| 优化预约流程 | 李四 | 3月25日 | 单次预约转化≥40% |

**长期改进（下个周期）**:
| 行动项 | 责任人 | 完成时间 | 验证方式 |
|--------|--------|----------|----------|
| 拓展社区招募渠道 | 王五 | 4月底 | 新渠道贡献≥10%用户 |

## 复盘话术

引导团队复盘时可以使用以下话术：

**开场**:
```
今天我们来做结构化复盘，目的是总结经验而不是追究责任。
请大家真实地分享做得好的和需要改进的地方。
```

**引导深度思考**:
- "这个结果背后的主要原因是什么？"
- "如果重来一次，有什么是我们一定会改变的？"
- "这些经验可以复用到其他项目中吗？"

**收尾**:
```
感谢大家的分享。我们今天的改进行动是...，责任人分别是...。
下次复盘时我们会回顾这些行动的执行情况。
```
