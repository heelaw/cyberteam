# 7个问题找方向法 - 检查清单

## 自检清单

### 问题1: 用户是谁
- [ ] 有明确的主要用户群体描述
- [ ] 用户特征具体可感知
- [ ] 有使用场景描述
- [ ] 避免了"所有人"的泛化

### 问题2: 用户的问题
- [ ] 列出了用户面临的主要问题
- [ ] 区分了表面问题和深层问题
- [ ] 问题按重要程度排序
- [ ] 问题有具体描述而非抽象概念

### 问题3: 产品功能
- [ ] 列出了核心功能和辅助功能
- [ ] 每个功能对应解决的问题
- [ ] 功能描述具体明确
- [ ] 避免功能堆砌，有主次之分

### 问题4: 如何解决
- [ ] 描述了完整的解决路径
- [ ] 突出了关键节点
- [ ] 说明了用户的感受
- [ ] 强调了方法的独特性

### 问题5: 与竞品差异
- [ ] 列出了主要竞品
- [ ] 各维度对比清晰
- [ ] 差异化优势明确
- [ ] 避免"全面超越"等空洞表述

### 问题6: 用户收益
- [ ] 功能转化为用户收益
- [ ] 收益具体可感知
- [ ] 有量化数据（尽量）
- [ ] 区分短期和长期收益

### 问题7: 证明依据
- [ ] 有数据支撑
- [ ] 有案例支撑
- [ ] 有用户证言支撑
- [ ] 证明与卖点强相关

## 卖点方向选择决策清单

### 方向1: 用户需求匹配度
- [ ] 用户真的在乎这个点
- [ ] 用户会为此付费
- [ ] 用户能感知到这个价值

### 方向2: 差异化程度
- [ ] 竞品没有强调这个点
- [ ] 我们能做得更好
- [ ] 这个差异是可感知的

### 方向3: 可执行性
- [ ] 有证明依据
- [ ] 能通过文案传达
- [ ] 符合产品实际情况

## 最终输出检查

- [ ] 输出3个核心卖点方向
- [ ] 每个方向有差异化说明
- [ ] 每个方向有证明依据
- [ ] 有具体的使用建议
