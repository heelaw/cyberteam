# 4P模型 - 方法论

## 核心框架

4P是一种故事型的文案框架，包括Picture（画面）、Promise（承诺）、Prove（证明）、Push（推进）。通过创造画面感、建立情感连接、提供证据支撑、引导最终行动来完成深度说服。

## 原理说明

### Picture - 创造画面
**核心**: 让用户"看到"一个具体场景

**方法**:
- 选择有代表性的场景
- 使用具体细节
- 加入感官描述
- 让人物有代入感

**衡量标准**: 用户能在脑中形成画面

### Promise - 做出承诺
**核心**: 给出有吸引力的价值承诺

**方法**:
- 承诺要与画面情境对应
- 承诺要具体可感知
- 承诺要有吸引力
- 承诺要可实现

**衡量标准**: 用户被承诺吸引

### Prove - 提供证明
**核心**: 用证据支撑承诺

**方法**:
- 数据证明
- 案例证明
- 证言证明
- 专家背书

**衡量标准**: 证据充分支撑承诺

### Push - 给出推进
**核心**: 引导用户行动

**方法**:
- 清晰的行动指引
- 符合品牌的语气
- 适当的紧迫感
- 情感的最终升华

**衡量标准**: 推进自然不生硬

## 与AIDA的区别

| 维度 | AIDA | 4P |
|------|------|-----|
| 重点 | 说服力 | 情感连接 |
| 结构 | 递进式 | 故事型 |
| 适用 | 销售/促销 | 品牌/信任 |
| 语气 | 直接 | 柔和 |

## 使用技巧

### Picture常见错误
- 场景太抽象
- 细节太少
- 代入感不强
- 场景与产品无关

### Promise常见错误
- 承诺过大
- 与画面不符
- 承诺不具体
- 承诺无法验证

### Prove常见错误
- 证据不充分
- 证据不相关
- 证明过于生硬
- 数据来源不明

### Push常见错误
- 推进太激进
- 语气突变
- 与前面调性不符
- 紧迫感过于压迫
