# 用户分层法 - 方法论

## 核心框架

用户分层是将用户按照某种维度或多种维度进行分类的过程，帮助运营者找到核心目标用户，并针对不同层级用户制定不同策略。

## 原理说明

### 为什么需要分层
- 不同用户需求不同
- 不同用户转化难度不同
- 不同用户商业价值不同
- 资源有限必须聚焦

### 分层维度
1. **基础属性**：年龄、职业、地域、收入
2. **行为特征**：使用频率、付费意愿、参与度
3. **需求动机**：核心需求、痛点强度、决策动机
4. **生命周期**：潜在用户、新用户、活跃用户、流失用户

## 实施步骤

### 第一步：收集数据
- 用户数据库
- 调研数据
- 行为数据

### 第二步：多维分层
- 单一维度分层
- 多维度交叉分析

### 第三步：价值评估
- 商业价值
- 转化难度
- 战略意义

### 第四步：聚焦核心
- 确定核心用户
- 确定次级用户
- 制定差异化策略

## 使用技巧

### 分层要精准
- 用具体描述，不用模糊标签
- "25-35岁互联网从业者" > "年轻人"

### 聚焦要坚决
- 核心用户只选1-2个群体
- 资源向核心用户倾斜
