# 需求类型判断法 - 检查清单

## 信号收集检查

### 渠道来源
- [ ] 明确渠道来源
- [ ] 渠道能说明需求程度
- [ ] 有其他信号交叉验证

### 搜索关键词
- [ ] 关键词已分析
- [ ] 能反映用户意图
- [ ] 类型指示清晰

### 用户行为
- [ ] 主要行为已记录
- [ ] 行为模式已识别
- [ ] 与类型判断一致

## 判断准确性检查

### 明确需求
- [ ] 用户知道要什么
- [ ] 搜索词具体
- [ ] 行为带有目的性

### 模糊需求
- [ ] 用户有痛点
- [ ] 不知道如何解决
- [ ] 行为带有探索性

### 潜在需求
- [ ] 用户无明确意识
- [ ] 需要触发
- [ ] 需要唤醒

## 策略匹配检查

### 明确需求策略
- [ ] 突出比较优势
- [ ] 提供决策证据
- [ ] 简化购买流程

### 模糊需求策略
- [ ] 教育引导为主
- [ ] 痛点共鸣
- [ ] 提供解决方案

### 潜在需求策略
- [ ] 场景唤醒
- [ ] 痛点激发
- [ ] 价值教育

## 阶段引导检查

- [ ] 有清晰的阶段划分
- [ ] 各阶段文案策略不同
- [ ] 有明确的转化引导
