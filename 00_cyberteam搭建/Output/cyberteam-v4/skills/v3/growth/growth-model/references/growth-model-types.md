# 增长模型类型详解

## 两种增长模型类型对比

### 全链漏斗型

**特点**: 将北极星指标分解为漏斗各步骤的转化率，适用于线性流程

**公式结构**:
```
北极星指标 = 流量 × 转化率A × 转化率B × 转化率C × ...
```

**适用场景**:
- 清晰的线性用户路径
- 多步骤转化流程
- 漏斗各步骤有明确的转化率

**优点**:
- 直观易懂，便于向团队传达
- 容易找到转化率最低的"破口"
- 快速搭建，快速见效

**缺点**:
- 不是可直接计算的公式
- 无法精确计算改善某个指标对北极星指标的影响

**案例：网易云音乐**
```
总听歌时长 = 下载量 × 首日打开率 × 首次听歌率 × 持续听歌率 × 人均听歌时长
```

---

### 因子分解型

**特点**: 将北极星指标按维度分解后乘法综合，适用于复杂业务

**公式结构**:
```
北极星指标 = 维度A × 维度B
           = (子维度1 + 子维度2) × 维度B
           = (A1 + A2) × B × C
```

**适用场景**:
- 业务维度复杂
- 需要精确计算改善影响
- 多维度并行结构

**优点**:
- 可精确计算每个变量的影响
- 支持多维度交叉分析
- 可直接计算改善某指标对北极星指标的影响

**缺点**:
- 搭建复杂，需要更多数据
- 不够直观

**案例：Airbnb**
```
总预定天数 = 新用户预定天数 + 老用户预定天数
          = (新用户数 × 新用户首次预定率 × 人均预定天数) 
          + (老用户数 × 老用户复购率 × 人均预定天数)
```

## 模型构建三步法

### 第一步：定义北极星指标

**问题清单**:
1. 你的北极星指标是什么？
2. 它是数量型指标还是比率型指标？
3. 计算方式是什么？

### 第二步：绘制核心转化路径

**关键问题**:
1. 用户从0到体验价值经历哪些步骤？
2. 每一步的转化节点是什么？
3. 哪些步骤是可测量的？

### 第三步：组装增长模型

**组装原则**:
1. 先用加法分解（按用户群组）
2. 再用乘法分解（按转化路径）
3. 确保拆分满足MECE原则

## 增长模型应用场景

### 应用1：找杠杆（四两拨千斤）

**方法**: 对比各变量的当前值与行业均值，找出差距最大的作为发力点

**案例**:
```
总销售额 = 新用户销售额 + 老用户复购销售额
新用户销售额 = 流量 × 注册率 × 购买率 × 人均购买额

发现：注册率15%，行业均值20%
→ 发力点：提升注册率（差距最大）
```

### 应用2：捋线头（复杂问题抽象化）

**方法**: 将复杂业务拆解为可理解的变量组合

**案例**:
```
电商销售额 = 品类A销售额 + 品类B销售额 + ...
          = (品类A流量 × 品类A转化率 × 品类A客单价)
          + (品类B流量 × 品类B转化率 × 品类B客单价)
```

### 应用3：搭乐高（团队协作框架）

**方法**: 按增长模型分配指标给团队，确保MECE原则

**案例**:
```
总DAU = 新用户DAU + 老用户DAU
新用户DAU → 获客团队负责
老用户DAU → 留存团队负责
```

## 增长模型模板

### 模板1：电商类
```
销售额 = 流量 × 转化率 × 客单价 × 复购率
       = (自然流量 + 付费流量) × 转化率 × 客单价 × 复购率
```

### 模板2：SaaS类
```
MRR = 活跃客户数 × 人均订阅费 × 续费率
    = 新增客户 × 激活率 × 人均订阅费 
    + 现有客户 × 续费率 × 人均订阅费
```

### 模板3：内容平台类
```
总观看时长 = 用户数 × 人均观看时长
           = (新增用户 + 留存用户) × 人均观看时长
           = (新增用户 × 激活率 + 留存用户) × 人均观看时长
```

### 模板4：交易市场类
```
平台GMV = 买家数 × 人均购买额
        = 卖家数 × 人均销售额 × 撮合率
```
