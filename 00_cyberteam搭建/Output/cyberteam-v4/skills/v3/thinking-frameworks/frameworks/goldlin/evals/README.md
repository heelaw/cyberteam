# 吉德林法则专家测试用例

## 正面测试

### 测试1: 职业困惑
- 输入:
```
【吉德林法则分析请求】
初步困扰：工作没意思，看不到发展
主观感受：焦虑、迷茫
背景：工作3年
```
- 期望输出:
  - 原始问题倾倒
  - 问题解构（事实/情绪/假设/期望）
  - 影响评估
  - 核心问题重述
  - 解决方向
- 验证项:
  - [ ] 充分倾倒
  - [ ] 四类分离正确
  - [ ] 核心问题具体

### 测试2: 创业纠结
- 输入:
```
【吉德林法则分析请求】
初步困扰：要不要离职创业
主观感受：害怕失败
背景：有大公司offer
```
- 期望输出:
  - 问题分离
  - 影响评估
  - 核心问题重述
  - 启发自用户
- 验证项:
  - [ ] 事实和情绪分离
  - [ ] 核心问题清晰
  - [ ] 解决思路由用户产出

## 负面测试

### 测试1: 跳过倾倒
- 输入:
```
【吉德林法则分析请求】
初步困扰：团队问题
主观感受：烦躁
```
- 期望行为: 不能跳过倾倒步骤直接给建议
- 验证项:
  - [ ] 充分表达所有困扰
  - [ ] 不打断
  - [ ] 不筛选

### 测试2: 替用户判断
- 输入:
```
【吉德林法则分析请求】
初步困扰：选择困难
主观感受：纠结
```
- 期望行为: 不能替用户判断问题重不重要
- 验证项:
  - [ ] 不评判
  - [ ] 不说"这不重要"
  - [ ] 方案由用户自生
