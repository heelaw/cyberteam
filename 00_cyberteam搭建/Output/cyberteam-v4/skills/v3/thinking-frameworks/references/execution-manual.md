> **版本**: v3.1
> **更新日期**: 2026-03-24
> **说明**: 本手册定义了思考天团框架的选择、执行、质量标准

## 一、框架选择指南

### 1.1 按任务类型选择

| 任务类型 | 推荐框架 | 备选框架 |
|----------|----------|----------|
| **需求分析** | 5W1H1Y | 5W2H, 鱼骨图 |
| **竞争分析** | SWOT | 波特五力, BCG矩阵 |
| **任务拆解** | WBS | 甘特图, 关键路径 |
| **目标设定** | OKR | SMART, 平衡计分卡 |
| **问题诊断** | 第一性原理 | 5Why, 根本原因分析 |
| **创新设计** | 设计思维 | SCAMPER, TRIZ |
| **决策分析** | 决策树 | 成本收益, 贝叶斯 |
| **流程优化** | PDCA | 精益, 六西格玛 |
| **市场分析** | PEST | 波特五力, 价值链 |
| **团队协作** | 六顶思考帽 | RACI, RACI矩阵 |

---

### 1.2 按行业选择

| 行业 | 推荐框架 |
|------|----------|
| **互联网/科技** | 设计思维, MVP, 增长黑客 |
| **制造业** | 精益生产, 六西格玛 |
| **金融** | 波特五力, DCF, 风险矩阵 |
| **医疗** | FMEA, 根本原因分析 |
| **教育** | 布鲁姆分类法, ADDIE |
| **营销** | 4P, AIDA, 用户旅程地图 |

---

### 1.3 按复杂度选择

| 复杂度 | 框架数量 | 推荐组合 |
|--------|----------|----------|
| **简单** (1-2个) | 1-2个框架 | SWOT + 5W1H |
| **中等** (3-5个) | 3-5个框架 | SWOT + 波特五力 + WBS |
| **复杂** (5+个) | 5+个框架 | PEST + SWOT + 波特五力 + 价值链 + WBS |

## 二、框架执行标准

### 2.1 通用执行流程

```
1. 明确目标
   └── 确定分析目的和预期产出

2. 选择框架
   └── 根据任务类型和复杂度选择

3. 收集信息
   └── 必要时调用 MCP 工具收集数据

4. 执行框架
   └── 按框架结构逐步分析

5. 整合结果
   └── 将各框架结果整合

6. 验证结论
   └── 检查逻辑完整性和可行性

7. 输出报告
   └── 按标准格式输出
```

---

### 2.2 SWOT 分析执行标准

**适用场景**: 竞争态势分析、战略制定

**执行步骤**:

```
1. 收集信息 (MCP web_search)
   └── 搜索: 竞品信息, 行业趋势

2. 优势 (Strengths)
   ├── 内部能力: 技术、品牌、资源
   └── 差异化: 独特卖点

3. 劣势 (Weaknesses)
   ├── 内部不足: 资金、人才、渠道
   └── 竞争劣势: 价格、质量、服务

4. 机会 (Opportunities)
   ├── 市场机会: 新需求、新市场
   └── 政策机会: 法规变化

5. 威胁 (Threats)
   ├── 竞争威胁: 新进入者、替代品
   └── 市场威胁: 经济下行

6. 策略制定
   ├── SO: 利用优势抓住机会
   ├── WO: 克服劣势抓住机会
   ├── ST: 利用优势避免威胁
   └── WT: 克服劣势避免威胁
```

**输出标准**:

```markdown
## SWOT 分析报告

### 优势 (Strengths)
1. [优势1]: [说明]
2. [优势2]: [说明]

### 劣势 (Weaknesses)
1. [劣势1]: [说明]
2. [劣势2]: [说明]

### 机会 (Opportunities)
1. [机会1]: [说明]
2. [机会2]: [说明]

### 威胁 (Threats)
1. [威胁1]: [说明]
2. [威胁2]: [说明]

### 策略矩阵
| 策略 | 组合 | 具体行动 |
|------|------|----------|
| SO | S×O | [行动1] |
| WO | W×O | [行动2] |
| ST | S×T | [行动3] |
| WT | W×T | [行动4] |

### 优先级排序
1. [优先级1]: [理由]
2. [优先级2]: [理由]
```

### 2.3 WBS 任务拆解执行标准

**适用场景**: 项目规划、任务分解

**执行步骤**:

```
1. 定义项目目标
   └── 明确最终交付物

2. 识别主要交付物
   └── 列出所有主要产出

3. 分解任务
   └── 每个交付物分解为 3-7 个任务

4. 子任务细化
   └── 每个任务分解为 3-7 个子任务

5. 分配资源
   └── 确定责任人/团队

6. 估算工时
   └── 评估每个任务的时间

7. 确定依赖
   └── 识别任务间的依赖关系
```

**输出标准**:

```markdown
## WBS 任务分解

### 项目: [项目名称]

#### 1.0 [主要交付物1]
  1.1 [子任务1.1]
    1.1.1 [具体任务] - [责任人] - [工时]
    1.1.2 [具体任务] - [责任人] - [工时]
  1.2 [子任务1.2]
    1.2.1 [具体任务] - [责任人] - [工时]

#### 2.0 [主要交付物2]
  2.1 [子任务2.1]
    2.1.1 [具体任务] - [责任人] - [工时]

### 依赖关系
| 任务 | 前置任务 | 负责人 |
|------|----------|--------|
| 1.1.1 | - | 张三 |
| 1.1.2 | 1.1.1 | 李四 |

### 总工时: [X] 小时
### 预计周期: [Y] 天
```

### 2.4 OKR 执行标准

**适用场景**: 目标设定、绩效管理

**执行步骤**:

```
1. 定义使命愿景
   └── 公司/团队的长期目标

2. 确定本周期 O (Objective)
   └── 3-5 个有挑战性的目标

3. 设定 KR (Key Results)
   └── 每个 O 有 2-4 个 KR
   └── KR 必须可量化

4. 对齐 OKR
   └── 确保与上级 OKR 一致

5. 制定行动方案
   └── 识别达成 KR 所需行动

6. 周期性复盘
   └── 每周检查进度
   └── 每月评分复盘
```

**输出标准**:

```markdown
## OKR

### 本周期: 2026年Q1

### O1: [有挑战性的目标]
  KR1.1: [可量化结果] - 目标值: X, 当前: Y
  KR1.2: [可量化结果] - 目标值: X, 当前: Y
  KR1.3: [可量化结果] - 目标值: X, 当前: Y

### O2: [有挑战性的目标]
  KR2.1: [可量化结果] - 目标值: X, 当前: Y
  KR2.2: [可量化结果] - 目标值: X, 当前: Y

### O3: [有挑战性的目标]
  KR3.1: [可量化结果] - 目标值: X, 当前: Y
  KR3.2: [可量化结果] - 目标值: X, 当前: Y

### 行动方案
| 行动 | 负责人 | 截止日期 | 状态 |
|------|--------|----------|------|
| [行动1] | [人] | [日期] | [状态] |
```

## 三、框架组合策略

### 3.1 战略规划组合

```
PEST (宏观分析)
  ↓
SWOT (态势分析)
  ↓
波特五力 (行业分析)
  ↓
价值链 (竞争优势)
  ↓
BCG 矩阵 (业务组合)
  ↓
战略选择
```

### 3.2 产品开发组合

```
设计思维 (用户洞察)
  ↓
WBS (任务拆解)
  ↓
MVP (快速验证)
  ↓
PDCA (持续迭代)
```

### 3.3 营销策划组合

```
5W1H1Y (需求分析)
  ↓
用户画像 (目标用户)
  ↓
AIDA (文案策略)
  ↓
用户旅程地图 (触点优化)
```

## 四、质量标准

### 4.1 分析深度标准

| 框架 | 最低深度 | 推荐深度 |
|------|----------|----------|
| SWOT | 4项各2条 | 4项各5条 |
| 波特五力 | 5力各2条 | 5力各5条 |
| WBS | 3级 | 4级 |
| OKR | O:3个, KR:各2个 | O:3个, KR:各3个 |

---

### 4.2 结论标准

1. **明确性**: 每个分析必须有明确结论
2. **可操作性**: 结论必须可落地执行
3. **数据支撑**: 结论必须有数据/事实支撑
4. **优先级**: 结论必须有优先级排序

---

### 4.3 格式标准

- 使用 Markdown 格式
- 表格使用标准格式
- 列表不超过 3 层
- 关键数据突出显示
- 适当使用图表

## 六、常见问题

### 6.1 框架选择困惑

**问题**: 不知道该用哪个框架

**解决**:
1. 明确分析目的
2. 参考"框架选择指南"
3. 咨询 Strategy Agent
4. 组合使用多个框架

---

### 6.2 分析深度不够

**问题**: SWOT 只有 1-2 条

**解决**:
1. 调用 MCP web_search 补充信息
2. 调用 MCP webReader 读取报告
3. 使用"5Why"深入追问
4. 参考竞品分析案例

---

### 6.3 结论不明确

**问题**: 分析完了不知道该做什么

**解决**:
1. 使用策略矩阵 (SO/WO/ST/WT)
2. 确定优先级 (Impact × Effort)
3. 制定行动计划
4. 设定 OKR
