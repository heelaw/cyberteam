> **版本**: v3.1

> **技能体系**: baoyu-skills (宝禹技能库)

## 一、技能体系概述

### 1.1 定位

baoyu-skills 是 CyberTeam 内容运营部的执行工具集，提供从内容创作到多平台发布的一站式技能支持。

### 1.2 核心能力

| 能力 | 描述 | 技能数量 |
|------|------|----------|
| **图像生成** | 封面图、系列图、海报 | 5 个 |
| **多平台发布** | 小红书、公众号、微博、抖音 | 5 个 |
| **格式转换** | Markdown、HTML、图片处理 | 4 个 |
| **内容优化** | 标题优化、标签生成 | 3 个 |

### 1.3 技能分类

```
baoyu-skills/
├── 图像生成类
│   ├── cover-image      (封面图生成)
│   ├── xhs-images       (小红书系列图)
│   ├── poster-design    (海报设计)
│   ├── product-shot     (产品图)
│   └── infographic      (信息图)
├── 多平台发布类
│   ├── post-to-wechat   (公众号发布)
│   ├── post-to-xhs      (小红书发布)
│   ├── post-to-weibo    (微博发布)
│   ├── post-to-douyin   (抖音发布)
│   └── post-to-linkedin (LinkedIn发布)
├── 格式转换类
│   ├── format-markdown  (Markdown格式化)
│   ├── format-html      (HTML转换)
│   ├── image-resize     (图片调整)
│   └── image-compress   (图片压缩)
└── 内容优化类
    ├── title-generator  (标题生成)
    ├── tag-generator    (标签生成)
    └── content-optimizer(内容优化)
```

## 二、核心技能详解

### 2.1 图像生成类

#### cover-image - 封面图生成

| 属性 | 说明 |
|------|------|
| **文件** | `skills/cover-image.md` |
| **功能** | 生成多平台封面图 |
| **支持平台** | 微信公众号 (2.35:1)、小红书 (3:4)、微博 (16:9) |
| **参数** | `--platform`、`--size`、`--text`、`--theme` |
| **输出** | JPG/PNG 图片 |

#### xhs-images - 小红书系列图

| 属性 | 说明 |
|------|------|
| **文件** | `skills/xhs-images.md` |
| **功能** | 生成小红书系列图 (3/5/9张) |
| **叙事结构** | 问题-方案-效果、引入-分析-总结 |
| **参数** | `--count`、`--theme`、`--style` |
| **输出** | 3/5/9 张 3:4 竖版图片 |

### 2.2 多平台发布类

#### post-to-wechat - 微信公众号发布

| 属性 | 说明 |

| **参数** | `--title`、`--content`、`--cover`、`--summary`、`--tags`、`--original` |
| **格式** | 支持 Markdown，转换为微信格式 |

#### post-to-xhs - 小红书发布

| 属性 | 说明 |

## 四、执行流程

### 4.1 完整内容创作流程

```
1. 内容策划
   └── 确定选题和核心卖点

2. 标题生成
   └── 使用高点击率标题公式
       痛点型/数字型/悬念型/情绪型

3. 封面图生成
   └── cover-image (根据平台选择比例)
       baoyu-skills → cover-image --platform wechat --text "标题"

4. 系列图生成 (小红书)
   └── xhs-images (3/5/9张叙事结构)
       baoyu-skills → xhs-images --count 5 --theme "护肤教程"

5. 文案撰写
   └── 按平台规范撰写正文
       参考: platforms/xiaohongshu.md 或 wechat-official-account.md

6. 标签/摘要设置
   └── 选择平台热门标签
       参考: platforms/ 下的标签规范

7. 多平台发布
   └── post-to-wechat (公众号)
   └── post-to-xhs (小红书)
```

### 4.2 快速发布流程 (5分钟)

```
1. 确定内容主题 (1分钟)
2. 生成标题 (1分钟)
3. 生成封面图 (1分钟)
4. 撰写简短正文 (1分钟)
5. 设置标签并发布 (1分钟)
```

## 五、Persona 专家

### 5.1 小红书内容专家

| 属性 | 说明 |
|------|------|
| **人设文件** | `persona/xiaohongshu-expert.md` |
| **核心能力** | 爆款标题、系列图设计、内容策划、标签策略 |
| **平台算法** | 完播率、互动率、关注转化 |
| **输出标准** | 标题 + 系列图 + 文案 + 标签 |

### 5.2 使用方式

```
用户: "帮我写一篇小红书种草笔记"

内容运营部:
  → 小红书专家 (xiaohongshu-expert.md) 接管
  → 确认产品/卖点/目标人群
  → 设计标题公式: 痛点型/数字型
  → 生成系列图: baoyu-xhs-images
  → 撰写种草文案
  → 选择标签组合
  → 输出: 完整笔记方案
```

## 六、内容复用策略

### 6.1 同一内容的平台适配

```
核心内容
  ├── → 公众号 (深度版本)
  │     baoyu-post-to-wechat (1000-3000字)
  │
  ├── → 小红书 (种草版本)
  │     baoyu-xhs-images + baoyu-post-to-xhs (500-1000字)
  │
  ├── → 微博 (短版本)
  │     baoyu-post-to-weibo (140字内)
  │
  └── → 抖音 (视频版本)
        baoyu-post-to-douyin (15秒-5分钟)
```

### 6.2 内容矩阵示例

```
产品发布内容矩阵:

1篇公众号深度文章
  └── 拆解 → 3篇小红书种草笔记
              └── 拆解 → 5条微博短推
                          └── 拆解 → 10条抖音短视频
```

## 七、质量标准

### 7.1 图片质量

| 项目 | 要求 |
|------|------|
| **分辨率** | 1242x1660px (小红书) / 1080x460px (公众号) |
| **文件大小** | ≤5MB |
| **格式** | JPG/PNG |
| **文字** | 清晰可读，无错别字 |

### 7.2 文案质量

| 项目 | 要求 |
|------|------|
| **标题** | 符合平台规范，有吸引力 |
| **正文** | 结构清晰，干货有价值 |
| **标签** | 相关性强，热度适中 |
| **合规** | 无敏感词，无虚假宣传 |

### 7.3 发布质量

| 项目 | 要求 |
|------|------|
| **格式** | Markdown 正确转换 |
| **图片** | 全部正常加载 |
| **链接** | 有效可访问 |
| **标签** | 设置正确 |

## 八、与思考天团集成

### 8.1 内容策略协作

```
思考天团 (战略分析师)
  └── 输出: SWOT/波特五力分析报告
        → 内容运营部:
            - 根据竞品劣势 → 差异化卖点
            - 根据市场机会 → 内容选题方向
            - 根据目标人群 → 内容风格定位

示例:
  战略分析师: "竞品在 X 方面表现弱，我们是 Y 优势"
    → 小红书专家: 基于 Y 优势 → 种草内容策划
    → baoyu-xhs-images: 生成差异化系列图
```
