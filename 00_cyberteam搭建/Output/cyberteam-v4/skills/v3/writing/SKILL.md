## 1. 工作流概述

转化型文案创作工作流是一个系统化的9阶段流程，旨在确保每篇文案都经过完整的创作、审核、打磨流程，最终产出高转化率的营销文案。

### 1.0 触发场景判断

本流程不是万能的——日常推文、简单回复、即时消息不在此列。**当文案工作从"写一段文字"升级为"解决一个商业问题"时**，本流程才被触发。

#### 必须触发完整流程的场景

| 场景 | 特征 | 触发原因 |
|------|------|----------|
| **核心营销战役与新品上市** | 项目制、高投入、跨团队协作、结果可衡量 | 资源投入大，不允许失误；需要Stage 0-1作为协作基准；必须通过Stage 8验证ROI |
| **高价值转化型内容** | 详情页/Landing Page/SEM/高价服务推广 | 决策成本高，需深度说服；付费广告文案质量直接决定CPC和CVR，必须经过策略分析 |
| **重要品牌或公关沟通** | 官方通稿/危机公关声明/重要公开信 | 风险极高，一言不慎引发舆论危机；面向多方受众，需要清晰的用户分层（Stage 1）和场景分析 |
| **体系化内容运营建设** | 搭建AI工作流/运营专栏/制定内容矩阵 | 需要SOP标准化确保质量稳定；人机协作中人类核心工作在策略定义（Stage 0-4）和审核（Stage 7） |

#### 核心触发特征

```
具备以下一个或多个特征 → 触发完整9阶段流程：
│
├── 项目制：有明确起止时间、预算、目标和团队分工
├── 结果导向：对曝光/互动/转化率/销售额有明确KPI要求
├── 高风险/高投入：试错成本高，不容随意发挥
├── 涉及复杂判断：目标用户多元、需求不明、场景特殊或竞争激烈
└── 需要团队/人机协同：文案是协作链条中的一环
```

#### 无需触发完整流程的场景

日常推文、简单回复、即时内容创作 → 直接进入简化流程，不走Stage 0-8。

### 1.1 工作流目标

- 确保文案基于真实用户反馈和数据分析
- 严格遵循投放平台的特点和规则
- 每一句话都服务于最终转化
- 深刻理解用户需求和心理
- 避免冗余信息，突出核心价值
- 采用口语化表达，自然流畅
- 包含明确的行动召唤
- 确保合规合法，无虚假宣传

## 2. 策略三角决策流程

### 2.1 什么是策略三角

策略三角是 Stage 1 和 Stage 4 的核心决策框架，用于确定文案的沟通策略方向。

```
                    ┌─────────────────┐
                    │    目标用户      │
                    │  (WHO)          │
                    └────────┬────────┘
                             │
         ┌───────────────────┼───────────────────┐
         │                   │                   │
         ↓                   ↓                   ↓
┌─────────────────┐ ┌─────────────────┐ ┌─────────────────┐
│   说什么        │ │   怎么说        │ │   在哪说        │
│   (WHAT)        │ │   (HOW)         │ │   (WHERE)       │
└─────────────────┘ └─────────────────┘ └─────────────────┘
```

### 2.2 策略三角决策维度

| 维度 | 决策问题 | 产出 |
|------|----------|------|
| **WHO - 目标用户** | 我们的内容给谁看？ | 用户画像卡 |
| **WHAT - 说什么** | 用户最关心什么？我们要传达什么？ | 核心卖点+痛点匹配 |
| **HOW - 怎么说** | 用什么语气、逻辑、风格？ | 说服模式+沟通基调 |
| **WHERE - 在哪说** | 用户在什么场景接触内容？ | 渠道策略+内容适配 |

## 3. 需求强度三等级判断标准

### 3.1 需求强度定义

| 等级 | 名称 | 用户心理 | 行为特征 | 文案策略 |
|------|------|----------|----------|----------|
| **L1** | 强需求 | "我必须解决这个问题" | 主动搜索、比较、立即购买 | 强调方案优势+行动便利性 |
| **L2** | 中需求 | "这个问题可以考虑解决" | 浏览信息、被动种草、犹豫中 | 制造紧迫感+差异化价值 |
| **L3** | 弱需求 | "我好像不需要这个" | 随意浏览、兴趣缺缺 | 重构认知+制造认知失衡 |

### 3.2 需求强度判断流程

```
Step 1: 用户是否主动表达过相关需求？
  ├─ 是 → 继续 Step 2
  └─ 否 → 可能为 L3 弱需求

Step 2: 用户是否有明确的问题或痛点？
  ├─ 是 → 继续 Step 3
  └─ 否 → 可能为 L2 中需求

Step 3: 用户是否在主动寻找解决方案？
  ├─ 是 → L1 强需求
  └─ 否 → L2 中需求
```

### 3.3 各需求强度的核心文案策略

| 需求等级 | 开篇策略 | 核心卖点表达 | CTA设计 |
|----------|----------|--------------|----------|
| **L1 强需求** | 直接切入方案 | 优势对比+效率证明 | 立即行动/限时优惠 |
| **L2 中需求** | 制造共鸣/场景代入 | 价值凸显+差异化 | 引导尝试/关注账号 |
| **L3 弱需求** | 颠覆认知/好奇心 | 新视角/新概念 | 引发思考/引导讨论 |

## 4. 场景分类与跨场景引导

### 4.1 四大场景类型

| 场景类型 | 用户状态 | 核心诉求 | 内容策略 |
|----------|----------|----------|----------|
| **主动搜索** | 有明确目的，主动寻找 | 快速找到答案 | 直接提供解决方案+核心卖点 |
| **社交浏览** | 无明确目的，碎片时间 | 消磨时间/发现有趣内容 | 制造吸引力+快速传递价值 |
| **内容种草** | 被KOL/内容触发兴趣 | 了解更多/被说服 | 情感共鸣+真实体验分享 |
| **决策辅助** | 犹豫期，比较选择中 | 消除疑虑/坚定信心 | 消除顾虑+社会证明+紧迫感 |

### 4.2 场景判断标准

| 判断维度 | 问题 | 场景判断 |
|----------|------|----------|
| 用户意图 | 用户是在找什么？ | 有目的→主动搜索；无目的→社交浏览 |
| 触发动机 | 用户为什么看到这个内容？ | 主动找→搜索场景；被动看到→浏览/种草 |
| 决策阶段 | 用户在购买漏斗哪个阶段？ | 需求唤醒→种草；方案比较→决策辅助 |

### 4.3 跨场景引导方法

| 起点场景 | 目标场景 | 引导策略 | 操作要点 |
|----------|----------|----------|----------|
| 社交浏览 | 电商购买 | 制造好奇心+跳转入口 | 悬念式结尾+优惠信息 |
| 内容种草 | 公众号深度 | 提供更多价值+关注引导 | 深度内容预告+关注利益点 |
| 短视频 | 图文详情 | 补充细节+完整信息 | 视频引导图文，图文引导购买 |
| 朋友圈 | 小程序 | 即时体验+便捷购买 | 卡片式内容+小程序入口 |

### 4.4 场景适配检查清单

```
□ 内容是否匹配当前场景的用户心态？
□ 内容的阅读时长是否符合场景？
□ 内容是否设置了合理的跨场景引导？
□ 不同渠道的内容是否做了差异化调整？
```

## 5. 阅读时间四档判断

### 5.1 时间档位划分

| 档位 | 时间范围 | 适用场景 | 内容策略 |
|------|----------|----------|----------|
| **极短档** | <10秒 | 开屏广告/信息流首位 | 单点突破，1个核心卖点 |
| **短档** | 10-30秒 | 小红书图文/微博 | 场景代入，2-3个要点 |
| **中档** | 30秒-2分钟 | 公众号短文/知乎回答 | 完整故事，5-6个要点 |
| **长档** | >2分钟 | 深度长文/品牌故事 | 完整AIDA，多层次论证 |

### 5.2 时间档位判断流程

```
Step 1: 确定渠道基础阅读时长
  ├─ 抖音/快手视频 → 15-60秒
  ├─ 小红书图文 → 10-30秒
  ├─ 公众号 → 30秒-5分钟
  └─ 知乎/小红书长文 → 2-10分钟

Step 2: 根据内容复杂度调整
  ├─ 新产品/新概念 → +30秒理解时间
  ├─ 成熟品类 → 可缩短50%
  └─ 需要教育用户 → +1-2分钟

Step 3: 确定最终档位
  根据 Step 1 + Step 2 得出最终阅读时间档位
```

### 5.3 各档位内容结构模板

| 档位 | 结构模板 |
|------|----------|
| **极短档** | 标题 + 1句核心价值 + CTA |
| **短档** | 吸睛开头 + 场景/痛点 + 解决方案 + CTA |
| **中档** | 标题 + 钩子 + 痛点共鸣 + 方案展示 + 证据支撑 + CTA |
| **长档** | 完整AIDA结构 + 深度论证 + 信任背书 + CTA |

## 6. 阶段执行详解

### 6.1 Stage 0：工作流初始化

**目标**: 确认用户需求，初始化工作流配置

**执行步骤**:

1. 向用户确认基本需求
2. 初始化工作流配置
3. **输出**: `workflow/00-init/{product}-{YYYYMMDD}-init.md`

**使用模板**: `workflow/00-init/任务简报模板.md`

---

### 6.2 Stage 1：用户需求挖掘

**目标**: 通过产品信息和真实用户反馈，深度分析目标人群

**执行步骤**:

1. **基础信息分析**:
   - 人口统计特征：年龄、性别、收入、教育背景
   - 心理特征：兴趣爱好、价值观、生活方式
   - 行为特征：购买习惯、消费频率、品牌忠诚度

2. **真实用户反馈收集**:
   ```bash
   site:reddit.com "[product] sucks OR frustrating OR annoying"
   site:x.com "[product] problem OR issue OR bug"
   site:zhihu.com "[产品] 难用 OR 吐槽 OR 问题"
   site:xiaohongshu.com "[产品] 真实测评 OR 吐槽"
   ```

3. **输出**: `workflow/01-user-research/{product}-{YYYYMMDD}-user-research.md`

**策略三角应用**:
- 在此阶段完成 WHO（目标用户）的初步判断
- 结合用户调研，明确用户的场景类型

---

### 6.3 Stage 2：产品卖点挖掘

**目标**: 基于用户反馈和产品信息，深度挖掘产品核心卖点

**7个核心问题框架**:

```
Q1: 产品有哪些值得人关注的细节？
Q2: 产品能解决什么问题？为何能解决？
Q3: 对比其他同类产品有何显著特点与不同？
Q4: 竞争对手存在哪些弱点是我们能做得更好的？
Q5: 有哪些设计生产中的细节可以体现产品的好？
Q6: 有哪些实际发生的结果和用户行为可以体现产品的好？
Q7: 有哪些人、事物、品牌的背书可以体现产品的好？
```

**三层分析**:

| 层级 | 分析内容 | 示例 |
|------|----------|------|
| **物理层** | 材料、工艺、设计、参数 | "航空级铝合金材质" |
| **体验层** | 使用感受、便捷性、效果 | "拿起手机就知道是什么材质" |
| **情感层** | 安全感、自信感、归属感 | "拿着有面子" |

**策略三角应用**:
- 在此阶段完成 WHAT（说什么）的初步判断
- 核心卖点是策略三角的核心输出之一

---

### 6.4 Stage 3：痛点定位

**目标**: 将用户痛点与产品卖点精准匹配

**痛点优先级排序**:

| 维度 | 评估标准 | 权重 |
|------|----------|------|
| **频次** | 多少人提到这个问题 | ⭐⭐⭐ |
| **情绪强度** | 轻微抱怨 vs 强烈不满 | ⭐⭐⭐ |
| **影响范围** | 影响多少用户 | ⭐⭐ |

**卖点-痛点匹配矩阵**:

| 痛点 | 匹配卖点 | 解决效果 | 说服逻辑 |
|------|----------|----------|----------|
| [痛点1] | [卖点1] | [效果描述] | [说服逻辑] |
| [痛点2] | [卖点2] | [效果描述] | [说服逻辑] |

---

### 6.5 Stage 4：需求类型判断

**目标**: 判断用户对产品的需求类型，确定文案的说服逻辑

**三种需求类型**:

| 类型 | 特征 | 文案策略 |
|------|------|----------|
| **类型A: 刚性需求** | 生活中离不开，需要持续使用 | 用事实和数据说服用户 |
| **类型B: 可有可无需求** | 用了更好，不用不影响 | 通过场景植入激发用户欲望 |
| **类型C: 趋于没有需求** | 用户认为不需要 | 制造认知失衡，重构用户认知 |

**策略三角应用**:
- 在此阶段完成 HOW（怎么说）的判断
- 结合需求强度等级（见第3章），确定说服模式
- 结合场景类型（见第4章），确定沟通基调

---

### 6.6 Stage 5：场景与渠道分析

**目标**: 结合投放渠道特点，确定文案形式、长度和结构

**渠道特征**:

| 渠道 | 转化能力 | 阅读时长 | 内容形式 |
|------|----------|----------|----------|
| 小红书 | 种草转化 | 10-30秒 | 图文 |
| 公众号 | 深度转化 | 3-5分钟 | 图文 |
| 微博 | 话题转化 | 10-20秒 | 图文/短视频 |
| 抖音/快手 | 即时转化 | 15-60秒 | 短视频 |

**策略三角应用**:
- 在此阶段完成 WHERE（在哪说）的判断
- 结合阅读时间档位（见第5章），确定内容长度
- 输出渠道适配策略文档

---

### 6.7 Stage 6：文案撰写

**目标**: 撰写符合策略、具有转化力的初稿文案

**文案撰写原则**:

1. **讲"人话"**: 口语化、接地气
2. **有数据**: 用具体数据支撑
3. **有细节**: 用具体描述替代抽象形容
4. **有场景**: 增强代入感
5. **有情感**: 调动用户情绪

**文案结构**:

```
标题: 吸引注意力
  ↓
钩子: 激发兴趣
  ↓
痛点: 戳中用户痛点
  ↓
解决方案: 展示产品价值
  ↓
证据: 用数据或案例证明
  ↓
CTA: 明确的行动召唤
```

---

### 6.8 Stage 7：文案优化

**目标**: 对初稿文案进行检查、删改和优化

**质量检查清单**:

```
- [ ] 核心卖点突出
- [ ] 痛点匹配准确
- [ ] 符合目标人群
- [ ] 符合平台特性
- [ ] 标题吸引眼球
- [ ] 内容简洁有力
- [ ] 有明确的行动召唤
- [ ] 语言口语化
- [ ] 无虚假宣传
- [ ] 无敏感内容
```

**优化方向**:

| 方向 | 说明 |
|------|------|
| **内容精简** | 删除冗余信息 |
| **通俗易懂** | 简化复杂概念 |
| **吸引力增强** | 优化标题和开头 |
| **场景适配** | 调整内容结构 |
| **感知引导** | 增强用户感受 |

---

### 6.9 Stage 8：效果追踪与迭代

**目标**: 提供文案效果追踪建议，支持后续迭代优化

**效果追踪指标**:

| 指标 | 说明 | 目标值 |
|------|------|--------|
| **CTR** | 点击率 | 1-3% |
| **CVR** | 转化率 | 0.5-2% |
| **CPC** | 单次点击成本 | 越低越好 |
| **CPA** | 单次转化成本 | 越低越好 |
| **停留时长** | 用户停留时间 | >30秒 |
| **分享率** | 分享比例 | >1% |

| 文案撰写 | 结构完整，语言生动 |
| 文案优化 | 精简有力，转化导向 |
| 效果追踪与迭代 | 指标明确，建议可操作 |
