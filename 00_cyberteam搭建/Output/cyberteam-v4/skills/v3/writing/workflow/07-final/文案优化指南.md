## 1. 三轮删减法

### 1.1 第一轮：结构性删减

**目标**: 删除与核心目标无关的内容

**检查项**:

| 删除类型 | 判断标准 | 处理方式 |
|----------|----------|----------|
| 无关内容 | 不能直接服务于核心卖点 | 删除 |
| 重复表达 | 同一观点已在前文表达 | 删除 |
| 冗余过渡 | 过渡句不增加任何信息 | 删除 |
| 可有可无 | 删掉不影响理解的句子 | 删除 |

**执行流程**:
```
Step 1: 逐段审视，每段问自己"删掉这这段有影响吗？"
Step 2: 如果回答是"有"→保留；回答是"没有"→删除
Step 3: 检查删除后逻辑是否连贯
Step 4: 补充过渡（如需要）
```

**删减比例目标**: 10-20%

---

### 1.2 第二轮：表达性删减

**目标**: 精简表达，提升阅读效率

**检查项**:

| 删除类型 | 判断标准 | 处理方式 |
|----------|----------|----------|
| 修饰词 | 删掉不影响意思的形容词/副词 | 删除或简化 |
| 重复表达 | 同一个意思重复说 | 保留最有力的一个 |
| 空洞形容词 | "非常好""特别棒"等无实质内容 | 替换为具体描述 |
| 过度解释 | 解释到过于细致的程度 | 适度删减 |

**常见需删除的词**:

| 类型 | 示例词 | 替换方式 |
|------|--------|----------|
| 程度副词 | 非常、特别、极其、相当 | 删除或替换为具体数值 |
| 情感形容词 | 非常好、特别棒、太赞了 | 替换为具体效果描述 |
| 模糊表述 | 很大、很小、很快 | 替换为具体数据 |
| 口头禅 | 真的、确实、不得不承认 | 删除 |

**执行流程**:
```
Step 1: 通读全文，标记冗余表达
Step 2: 逐个判断：删掉是否影响意思？
Step 3: 保留最有力、最具体的表达
Step 4: 读出声，检查是否流畅
```

**删减比例目标**: 15-25%

---

### 1.3 第三轮：感知性优化

**目标**: 增强可读性和感染力

**检查项**:

| 优化类型 | 判断标准 | 处理方式 |
|----------|----------|----------|
| 长句拆分 | 超过35字的句子 | 必须拆分 |
| 专业术语 | 普通用户难以理解 | 改写为类比 |
| 抽象表达 | 无法形成画面感 | 改为具体/场景化 |
| 句式单一 | 全篇陈述句/疑问句 | 丰富句式变化 |

**优化前后对比**:

| 问题类型 | 优化前 | 优化后 |
|----------|--------|--------|
| 长句 | "这个产品的效果非常好，能够帮助用户在短时间内实现皮肤状态的显著改善" | "用3天，皮肤就能看到变化" |
| 术语 | "富含透明质酸钠成分" | "含有玻尿酸，补水力提升3倍" |
| 抽象 | "让你变得更美" | "照镜子时，忍不住多看自己两眼" |
| 句式单一 | 全部是陈述句 | 适当加入反问、感叹 |

**执行流程**:
```
Step 1: 检查每句是否超过35字
Step 2: 检查是否有生涩专业术语
Step 3: 检查是否抽象、无法形成画面
Step 4: 通读检查节奏感
```

## 2. 改写标准

### 2.1 长句改短句

**标准**: >35字必须拆分，目标15-20字

**判断方法**:
- 默读一遍，超过一口气读完的长度 → 拆分
- 主语超过2个 → 拆分
- 从句超过1层 → 拆分

**改写模板**:

| 原文 | 问题 | 改写 |
|------|------|------|
| "这个产品采用了最新的科技，能够有效解决用户在日常生活中遇到的各种皮肤问题" | 主语过多 | "这款产品用最新科技，专门解决各种皮肤问题" |

### 2.2 专业术语改类比

**标准**: 通过"向父母解释"测试

**测试方法**:
```
能否用一句话向非专业的父母解释清楚？
├─ 能 → 保留或简化
└─ 不能 → 改写为类比
```

**改写方法**:

| 术语 | 类比改写 |
|------|----------|
| "抗氧化" | "防止皮肤生锈" |
| "补水保湿" | "给皮肤喝水，喝完还要锁住" |
| "屏障修复" | "给皮肤建保护层" |

### 2.3 抽象改具体

**标准**: 数据化 > 场景化 > 动作化

| 抽象表达 | 优化方向 | 改写示例 |
|----------|----------|----------|
| "效果很好" | 数据化 | "7天淡化细纹" |
| "使用方便" | 场景化 | "早上洗漱完顺手一抹就行" |
| "质量不错" | 动作化 | "摔了2次都没坏" |

## 3. 自检清单（11项/4层）

### 3.1 A层：策略层

| 序号 | 检查项 | 通过标准 | 不通过处理 |
|------|--------|----------|------------|
| A1 | 用户匹配 | 目标用户看到是否有共鸣？ | 返回Stage 1重新分析用户 |
| A2 | 卖点突出 | 核心卖点是否在最显眼位置？ | 调整内容结构，强化卖点 |
| A3 | 目标一致 | 文案目标是否与策略目标一致？ | 重新明确目标，调整内容 |

### 3.2 B层：内容层

| 序号 | 检查项 | 通过标准 | 不通过处理 |
|------|--------|----------|------------|
| B1 | 概念清晰 | 用户能否理解核心概念？ | 简化或增加解释 |
| B2 | 逻辑严密 | 论证是否有漏洞？ | 补充论据或删除有漏洞的部分 |
| B3 | 证据充分 | 关键主张是否有证据支撑？ | 补充数据/案例/背书 |

### 3.3 C层：表达层

| 序号 | 检查项 | 通过标准 | 不通过处理 |
|------|--------|----------|------------|
| C1 | 开头有力 | 开头能否抓住注意力？ | 重新撰写开头 |
| C2 | 内容精简 | 每句是否必要？有无冗余？ | 执行三轮删减法 |
| C3 | 画面感 | 能否形成清晰画面？ | 改写为具体/场景化表达 |

### 3.4 D层：人性化层

| 序号 | 检查项 | 通过标准 | 不通过处理 |
|------|--------|----------|------------|
| D1 | 人情味 | 是否像真人说话？ | 增加口语化表达 |
| D2 | 行动引导 | CTA是否清晰可操作？ | 优化CTA |
| D3 | 整体节奏 | 读起来是否流畅有节奏？ | 调整句式和段落 |

### 3.5 自检执行表

```
□ A层：策略层
  □ A1: 用户匹配检查
  □ A2: 卖点突出检查
  □ A3: 目标一致检查

□ B层：内容层
  □ B1: 概念清晰检查
  □ B2: 逻辑严密检查
  □ B3: 证据充分检查

□ C层：表达层
  □ C1: 开头有力检查
  □ C2: 内容精简检查
  □ C3: 画面感检查

□ D层：人性化层
  □ D1: 人情味检查
  □ D2: 行动引导检查
  □ D3: 整体节奏检查
```

## 4. 问题归因与阶段返回

### 4.1 问题类型与返回阶段对照

| 问题类型 | 问题表现 | 返回阶段 | 处理方式 |
|----------|----------|----------|----------|
| 卖点问题 | 卖点不突出/卖点选错/卖点与痛点不匹配 | Stage 2/1 | 重新分析卖点或痛点 |
| 策略问题 | 说服逻辑不对/用户定位错/需求强度判断错 | Stage 4/1 | 重新做策略三角决策 |
| 结构问题 | 内容结构混乱/逻辑不连贯/比例失调 | Stage 5 | 重新选择内容结构 |
| 表达问题 | 语言不流畅/冗余拖沓/不够吸引人 | Stage 7 | 执行三轮删减法或重写 |

### 4.2 阶段返回决策树

```
发现问题
    │
    ▼
┌─────────────────────────────────────┐
│ 判断问题属于哪个层面？               │
└─────────────────────────────────────┘
    │
    ├─ 策略层问题 ──────────────┐
    │                            │
    ▼                            ▼
┌───────────┐            ┌───────────┐
│ Stage 1   │            │ Stage 4   │
│ 用户/痛点 │            │ 策略决策  │
└───────────┘            └───────────┘
    │
    ├─ 内容层问题 ──────────────┐
    │                            │
    ▼                            ▼
┌───────────┐            ┌───────────┐
│ Stage 2   │            │ Stage 5   │
│ 卖点分析  │            │ 结构选择  │
└───────────┘            └───────────┘
    │
    └─ 表达层问题 ──────────────→ Stage 7 表达优化
```

### 4.3 返回阶段执行要求

| 返回阶段 | 执行要求 | 验收标准 |
|----------|----------|----------|
| Stage 1 | 补充用户画像或痛点分析 | 新增分析有数据支撑 |
| Stage 2 | 重新评估卖点优先级 | 卖点与痛点匹配度≥80% |
| Stage 4 | 重新做策略三角决策 | 策略陈述清晰合理 |
| Stage 5 | 重新选择内容结构 | 结构适配目标/渠道 |
| Stage 7 | 执行三轮删减法 | 11项自检全部通过 |

## 5. 文案质量评级

### 5.1 评级标准

| 等级 | 综合评分 | 定义 | 标准 |
|------|----------|------|------|
| **优秀** | 90-100分 | 超出预期 | 11项全过，且有亮点 |
| **良好** | 75-89分 | 达到预期 | 11项全过 |
| **合格** | 60-74分 | 基本可用 | 策略层+内容层全过，表达层允许1项未过 |
| **不合格** | <60分 | 需要修改 | 策略层或内容层有未通过项 |

### 5.2 评分明细表

| 维度 | 检查项 | 评分(1-10) | 备注 |
|------|--------|-------------|------|
| **策略层** | A1 用户匹配 | | |
| | A2 卖点突出 | | |
| | A3 目标一致 | | |
| **内容层** | B1 概念清晰 | | |
| | B2 逻辑严密 | | |
| | B3 证据充分 | | |
| **表达层** | C1 开头有力 | | |
| | C2 内容精简 | | |
| | C3 画面感 | | |
| **人性化层** | D1 人情味 | | |
| | D2 行动引导 | | |
| | D3 整体节奏 | | |
| **总分** | - | /120 | 转换为百分制 |

### 5.3 各等级处理方式

| 等级 | 处理方式 | 说明 |
|------|----------|------|
| **优秀** | 直接发布 | 可进入Stage 8发布环节 |
| **良好** | 轻微优化后发布 | 根据反馈微调后发布 |
| **合格** | 需要优化后发布 | 执行三轮删减法或局部重写 |
| **不合格** | 返回对应阶段修改 | 根据问题归因返回Stage 1-7 |
