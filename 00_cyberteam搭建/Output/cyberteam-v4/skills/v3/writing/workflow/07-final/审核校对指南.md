## 1. 审核驳回标准

### 1.1 必须修改项（硬性标准）

| 错误类型 | 判断标准 | 处理方式 |
|----------|----------|----------|
| **事实错误** | 内容与产品/服务实际情况不符 | 必须修改 |
| **逻辑硬伤** | 论证有明显漏洞或自相矛盾 | 必须修改 |
| **合规风险** | 违反广告法/平台规定/法律法规 | 必须修改 |
| **品牌严重偏离** | 与品牌调性/价值观严重不符 | 必须修改 |
| **虚假宣传** | 无法兑现的承诺或夸大表述 | 必须修改 |
| **敏感内容** | 涉及政治/宗教/色情/暴力等 | 必须修改 |
| **竞品攻击** | 直接贬低竞品或恶意比较 | 必须修改 |

### 1.2 建议优化项（软性标准）

| 问题类型 | 判断标准 | 处理方式 |
|----------|----------|----------|
| **表达冗余** | 有更简洁的表达方式 | 建议优化 |
| **语气欠妥** | 语气与目标人群/品牌不符 | 建议调整 |
| **有更优方案** | 存在效果更好的表达方式 | 建议优化 |
| **排版问题** | 分段/标点/格式不规范 | 建议调整 |
| **用词不当** | 个别词语不够精准 | 建议微调 |

### 1.3 审核检查清单

```
【硬性检查 - 必须通过】
□ 无事实错误
□ 逻辑自洽
□ 合规合法（广告法/平台规范）
□ 品牌调性一致
□ 无虚假宣传
□ 无敏感内容
□ 无竞品攻击

【软性检查 - 建议优化】
□ 表达精炼
□ 语气匹配
□ 整体流畅
□ 排版规范
□ 用词精准
```

## 2. 审核流程

### 2.1 三级审核机制

| 审核级别 | 审核人 | 审核内容 | 审核时长 |
|----------|--------|----------|----------|
| **一级审核** | 操盘手 | 策略合规+内容质量 | 30分钟内 |
| **二级审核** | 内容总监 | 品牌调性+整体质量 | 2小时内 |
| **三级审核** | 客户/产品方 | 事实准确+品牌确认 | 按客户要求 |

### 2.2 分级审核标准

| 文案类型 | 审核级别 | 说明 |
|----------|----------|------|
| 日常推广文案 | 一级审核 | 操盘手自审或互审 |
| 重要活动文案 | 二级审核 | 必须内容总监审核 |
| 品牌官方文案 | 三级审核 | 必须客户确认 |
| 危机公关文案 | 升级审核 | 法务+公关+高层联合 |

### 2.3 审核执行流程

```
文案提交
    │
    ▼
┌─────────────────────┐
│ 一级审核（操盘手）   │
│ ├─ 硬性检查         │
│ ├─ 软性检查         │
│ └─ 填写审核意见     │
└─────────────────────┘
    │
    ├── 通过 ──────────────────┐
    │                          │
    ▼                          ▼
┌─────────────────────┐    ┌─────────────────────┐
│ 二级审核（内容总监） │    │ 返回修改           │
│ ├─ 品牌调性         │    │ ├─ 操盘手修改      │
│ ├─ 整体质量         │    │ └─ 重新提交审核    │
│ └─ 填写审核意见     │    └─────────────────────┘
└─────────────────────┘
    │
    ├── 通过 ──────────────────┐
    │                          │
    ▼                          ▼
┌─────────────────────┐    ┌─────────────────────┐
│ 三级审核（客户）     │    │ 返回修改           │
│ ├─ 事实准确         │    │ ├─ 按意见修改      │
│ └─ 品牌确认         │    │ └─ 重新提交审核    │
└─────────────────────┘    └─────────────────────┘
    │
    ├── 通过 ──→ 进入发布环节
    │
    └── 不通过 ──→ 按意见修改
```

### 2.4 审核意见格式

| 意见类型 | 格式要求 | 示例 |
|----------|----------|------|
| 必须修改 | [位置] + [问题] + [建议] | "[第3段] "非常好"过于空洞，建议改为具体描述" |
| 建议优化 | [位置] + [问题] + [建议] | "[标题] 可尝试加入数字，增强可信度" |
| 确认通过 | 无需修改 | "审核通过，可发布" |

## 3. 异议仲裁机制

### 3.1 异议产生场景

| 场景 | 说明 |
|------|------|
| 撰写者与审核者意见不一致 | 操盘手认为OK，审核者认为需修改 |
| 多级审核意见冲突 | 一级审核通过，二级审核不通过 |
| 客户与团队意见冲突 | 客户要求与策略方向冲突 |

### 3.2 异议处理流程

```
异议产生
    │
    ▼
┌─────────────────────┐
│ Step 1: 撰写者书面陈述 │
│ ├─ 说明坚持理由     │
│ ├─ 提供支撑依据     │
│ └─ 24小时内提交     │
└─────────────────────┘
    │
    ▼
┌─────────────────────┐
│ Step 2: 操盘手调解   │
│ ├─ 评估双方意见     │
│ ├─ 提出折中方案     │
│ └─ 协商解决         │
└─────────────────────┘
    │
    ├── 解决 ──→ 执行修改
    │
    └── 未解决 ──→ 升级
    │
    ▼
┌─────────────────────┐
│ Step 3: 内容总监裁决 │
│ ├─ 最终决策         │
│ ├─ 说明理由         │
│ └─ 必须执行         │
└─────────────────────┘
    │
    └── 特殊情况 ──→ 升级至内容总监或客户高层
```

### 3.3 升级条件

| 升级条件 | 升级对象 | 处理要求 |
|----------|----------|----------|
| 涉及品牌重大决策 | 内容总监 | 必须书面决策 |
| 涉及法律合规风险 | 法务部门 | 必须法务审核 |
| 涉及客户核心诉求 | 客户高层 | 需双方高层确认 |
| 涉及竞品敏感问题 | 公关部门 | 必须公关审核 |

## 4. A/B测试操作标准

### 4.1 测试元素优先级

| 优先级 | 测试元素 | 影响程度 | 测试价值 |
|--------|----------|----------|----------|
| **P0** | 标题 | 点击率 | 极高 |
| **P1** | 封面/配图 | 点击率 | 高 |
| **P2** | 开头钩子 | 完读率 | 高 |
| **P3** | 核心卖点 | 转化率 | 中 |
| **P4** | CTA | 转化率 | 中 |
| **P5** | 其他细节 | 转化率 | 低 |

### 4.2 判断标准

| 指标 | 说明 | 判断标准 |
|------|------|----------|
| **CTR** | 点击率 | 提升≥10%为显著 |
| **CVR** | 转化率 | 提升≥15%为显著 |
| **统计显著性** | p值 | p<0.05才可判定有效 |

### 4.3 样本量估算

| 基准转化率 | 最小样本量/组 | 说明 |
|------------|---------------|------|
| 1% | 10,000 | 需要较大流量 |
| 2% | 5,000 | 需要一定流量 |
| 5% | 2,000 | 中等流量可测试 |
| 10% | 1,000 | 较小样本即可 |
| 20% | 500 | 小样本快速测试 |

**公式**:
```
最小样本量 = 16 × p × (1-p) / (MDE)²

其中:
p = 基准转化率
MDE = 最小可检测差异（通常设为p的10%）
```

### 4.4 A/B测试执行流程

```
测试准备
    │
    ├─ 确定测试元素（按优先级）
    ├─ 设计测试方案（A vs B）
    ├─ 准备测试素材
    └─ 确定流量分配比例
    │
    ▼
测试执行
    │
    ├─ 同时上线，避免时间因素
    ├─ 监控数据波动
    ├─ 记录异常情况
    └─ 保持变量唯一
    │
    ▼
结果判定
    │
    ├─ 达到最小样本量
    ├─ 计算统计显著性（p值）
    ├─ 评估效果提升
    └─ 得出结论
    │
    ▼
后续处理
    │
    ├─ 选择胜出方案
    ├─ 沉淀测试经验
    └─ 规划下一轮测试
```

## 5. 审核责任矩阵

### 5.1 RACI矩阵

| 任务 | 撰写者 | 操盘手 | 内容总监 | 客户/产品方 | 法务 | 公关 |
|------|--------|--------|----------|-------------|------|------|
| 初稿撰写 | R | - | - | - | - | - |
| 一级审核 | I | R/A | - | - | - | - |
| 二级审核 | I | C | R/A | - | - | - |
| 三级审核 | I | C | C | R/A | - | - |
| 合规审核 | I | C | C | C | R/A | - |
| 品牌审核 | I | C | C | R | - | R/A |
| 最终发布 | I | C | A | C | - | - |

**R** = 负责执行 | **A** = 最终审批 | **C** = 咨询 | **I** = 知会

### 5.2 各角色审核要点

| 角色 | 核心职责 | 审核重点 |
|------|----------|----------|
| 撰写者 | 提供高质量初稿 | 自审自查 |
| 操盘手 | 策略合规+效率把控 | 策略层+合规层 |
| 内容总监 | 品牌调性+整体质量 | 品牌层+质量层 |
| 客户/产品方 | 事实准确+品牌确认 | 事实层+品牌层 |
| 法务 | 合规风险控制 | 法律层 |
| 公关 | 舆情风险控制 | 舆论层 |
