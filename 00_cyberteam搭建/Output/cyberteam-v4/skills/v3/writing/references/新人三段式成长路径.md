# Content Ops 新人三段式成长路径

> **版本**: v1.0
> **创建日期**: 2026-03-24
> **适用范围**: Content Ops 部门全员（含 AI Agent）
> **负责人**: 资深操盘手（指定为导师）

---

## 一、成长路径总览

```
第一阶段：观摩与复刻
  时间：第1个月
  核心：不是创造，是理解
  ↓
第二阶段：在护栏内执行
  时间：第2-3个月
  核心：用模板跑通小闭环
  ↓
第三阶段：参与创造与沉淀
  时间：第3个月及以后
  核心：产出 + 反哺知识库
```

---

## 二、第一阶段：观摩与复刻（第1个月）

### 2.1 核心目标

> 不着急上手，先把"好的标准"看清楚。

### 2.2 学习任务

**Task 1：完成 S 级 Demo 全案学习包**

导师提供 1-2 个完整的、成功的 S 级项目全案包，新人必须逐项阅读并理解：

```
全案包内容清单：
├── 原始需求邮件/工单
├── 任务简报（Stage 0）
├── 用户调研报告（Stage 1）
├── 卖点分析报告（Stage 2）
├── 痛点定位文档（Stage 3）
├── 核心文案策略决策卡（Stage 4）
├── 内容策划案（Stage 5）
├── 文案初稿 → 修改稿 → 定稿（Stage 6）
├── 审核意见书（Stage 7）
├── 发布报告（Stage 8）
└── T+30 数据复盘报告
```

**Task 2：复刻产出对比**

新人根据原始需求，**独立产出**自己的版本：

```
Step 1: 阅读原始需求（不看历史输出）
Step 2: 独立完成策略简报
Step 3: 独立完成内容策划案
Step 4: 独立完成文案撰写
Step 5: 将自己的版本与历史版本逐项对比
Step 6: 在导师指导下，分析每一个差异点的原因
```

**Task 3：填写《复刻分析报告》**

```
复刻分析报告模板：
1. 我的策略简报 vs 历史版本 → 差异点及原因
2. 我的策划案 vs 历史版本 → 差异点及原因
3. 我的文案 vs 历史版本 → 差异点及原因
4. 我认为历史版本做得好的是...
5. 我认为可以改进的是...
6. 我从这次复刻中学到的最重要的1件事是...
```

### 2.3 验收标准

- 能完整复述 S 级项目的完整流程
- 能说出策略分析师在关键节点做了哪些决策
- 能用自己的语言解释"为什么要这样写开头"

### 2.4 导师职责

- 提供 Demo 全案包
- 主持每周 1 次复刻复盘会
- 回答"为什么这样做"而非"做什么"

## 三、第二阶段：在护栏内执行（第2-3个月）

### 3.1 核心目标

> 在明确的边界内，独立完成一个完整的 B 级项目。

### 3.2 学习任务

**Task 1：独立负责一个 B 级需求**

- 分配一个定义清晰的 B 级项目
- 由新人独立完成全部流程（使用模板）
- 导师仅在关键节点复核，不全程陪同

**Task 2：使用《优秀案例自查对照表》**

每个检查项旁边都附有 3 个历史优秀范例，新人对着"抄感觉"：

| 检查项 | 历史优秀范例 |
|--------|-------------|
| 标题是否抓人 | [优秀标题1] / [优秀标题2] / [优秀标题3] |
| 开头是否有钩子 | [优秀开头1] / [优秀开头2] / [优秀开头3] |
| 痛点是否共鸣 | [优秀痛点1] / [优秀痛点2] / [优秀痛点3] |
| FAB 是否完成 | [优秀FAB1] / [优秀FAB2] / [优秀FAB3] |
| CTA 是否有力 | [优秀CTA1] / [优秀CTA2] / [优秀CTA3] |

**Task 3：完成 B 级项目交付**

- 按时交付
- 使用标准模板
- 附上《自查对照表》填写记录

### 3.3 验收标准

- B 级项目能在 3 个工作日内完成交付
- 自查对照表填写完整，无跳项
- 交付质量通过审核，无必须修改项

---

## 四、第三阶段：参与创造与沉淀（第3个月及以后）

### 4.1 核心目标

> 参与 S/A 级项目，同时开始反哺知识库。

### 4.2 学习任务

**Task 1：参与 A 级项目**

- 在资深文案/操盘手带领下，参与 A 级项目的部分模块
- 逐步承担更复杂的任务

**Task 2：强制知识沉淀**

每个项目后，必须向知识库提交至少 1 条：

```
知识沉淀条目模板：
类型：□ 模板  □ 清单  □ 避坑指南  □ 范例  □ 方法论
来源项目：[项目名称]
内容：
标题：[简洁描述这个知识点]
正文：
- 适用场景：[什么时候用]
- 具体内容：[核心做法]
- 注意事项：[坑在哪里]
- 范例：[如果有]
```

### 4.3 验收标准

- A 级项目参与质量达到"良好"评级
- 每月知识沉淀 ≥ 1 条
- 能独立完成一个 B 级到 A 级之间难度的完整项目

## 五、导师制规范

### 5.1 导师职责

| 职责 | 说明 | 频次 |
|------|------|------|
| 提供 Demo 包 | 准备并解释完整全案 | 第1周 |
| 主持复盘会 | 逐项对比复刻产出 | 每周1次 |
| 分配 B 级任务 | 选择适合新人的项目 | 第2周起 |
| 复核关键节点 | 审核自查表和最终交付 | 按需 |
| 知识库维护 | 帮助新人将经验格式化 | 持续 |

### 5.2 导师考核

- 新人在第一阶段结束后能通过验收 → 导师绩效 +1
- 知识库沉淀条目来自新人 → 导师绩效 +0.5
- 新人在 3 个月内达到独立期标准 → 导师绩效 +2
