## 二、核心技能详解

### 2.1 /codex - 代码实现

**用途**: 根据需求实现代码

**使用场景**:

| 场景 | 说明 | 示例 |
|------|------|------|
| 新功能开发 | 实现新功能代码 | 实现用户登录功能 |
| 代码重构 | 重写现有代码 | 重构为 TypeScript |
| Bug 修复 | 修复已知 Bug | 修复登录超时问题 |
| 脚本编写 | 编写辅助脚本 | 数据迁移脚本 |

**执行流程**:

```
1. 明确需求
   └── 功能描述清晰

2. 分析现有代码
   ├── 读取相关文件
   ├── 理解代码结构
   └── 确认接口定义

3. 实现代码
   ├── 编写新代码
   ├── 保持代码风格一致
   └── 添加必要注释

4. 自测验证
   ├── 语法检查
   ├── 逻辑验证
   └── 边界测试

5. 输出报告
   ├── 实现文件列表
   ├── 修改内容摘要
   └── 注意事项
```

**输出标准**:

```markdown
## /codex 代码实现报告

### 需求
[功能描述]

### 实现文件
| 文件 | 操作 | 行数 |
|------|------|------|
| src/auth.js | 新增 | 150 |
| src/login.js | 修改 | 80 |

### 核心实现
```javascript
[关键代码片段]
```

### 接口定义
| 接口 | 方法 | 说明 |
|------|------|------|
| /api/login | POST | 用户登录 |

### 测试建议
- [ ] 测试登录成功
- [ ] 测试密码错误
- [ ] 测试账户锁定

### 注意事项
[需要关注的点]
```

### 2.2 /review - 代码审查

**用途**: 审查代码质量

**审查维度**:

| 维度 | 检查项 | 权重 |
|------|--------|------|
| **正确性** | 功能是否正确实现 | ⭐⭐⭐ |
| **可读性** | 代码是否清晰易懂 | ⭐⭐ |
| **性能** | 是否有性能问题 | ⭐⭐⭐ |
| **安全** | 是否有安全漏洞 | ⭐⭐⭐ |
| **测试** | 是否有测试覆盖 | ⭐⭐ |

---

### 2.3 /qa - QA 测试

**用途**: 系统化测试并修复问题

**测试流程**:

```
1. 探索测试
   └── 了解系统功能

2. 执行测试
   ├── 功能测试
   ├── 界面测试
   └── 流程测试

3. 发现问题
   ├── 记录 Bug
   ├── 分类优先级
   └── 复现步骤

4. 修复验证
   └── 修复后重新测试

5. 输出报告
   └── 测试结果汇总
```

---

### 2.4 /ship - 部署发布

**用途**: 代码合并、部署、发布

**发布流程**:

```
1. 代码合并
   └── 检测并合并主分支

2. 运行测试
   └── 执行 CI 测试

3. 代码审查
   └── /review 最终审查

4. 版本更新
   └── 更新 CHANGELOG

5. 提交推送
   └── 提交并推送到远程

6. 创建 PR
   └── 创建 Pull Request

7. 部署验证
   └── 验证部署成功
```

## 四、质量标准

### 4.1 代码质量标准

| 检查项 | 标准 | 权重 |
|--------|------|------|
| **语法正确** | 无语法错误 | 必须 |
| **功能正确** | 满足需求 | 必须 |
| **代码风格** | 符合规范 | 高 |
| **性能** | 无明显性能问题 | 高 |
| **安全** | 无安全漏洞 | 必须 |
| **测试覆盖** | 核心功能有测试 | 中 |

---

### 4.2 审查标准

```yaml
通过条件:
  - 无阻塞性问题 (P0/P1)
  - 无安全漏洞
  - 代码风格符合规范
  - 功能正确实现

需要修改:
  - 有 P2/P3 问题
  - 代码可读性差
  - 缺少必要注释

拒绝:
  - 有 P0/P1 问题
  - 存在安全漏洞
  - 功能实现错误
```

---

### 4.3 测试标准

| 测试类型 | 覆盖率要求 | 说明 |
|----------|------------|------|
| **单元测试** | 核心功能 80%+ | 每个函数 |
| **集成测试** | 主要流程 100% | API 流程 |
| **端到端测试** | 核心场景 100% | 用户流程 |

## 七、故障排查

### 7.1 常见问题

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| **代码实现失败** | 需求不清晰 | 明确需求后再实现 |
| **审查发现大量问题** | 代码质量差 | 按优先级修复 |
| **测试失败** | 实现有 Bug | /codex 修复 Bug |
| **部署失败** | 环境问题 | 检查配置和环境 |

---

### 7.2 错误处理流程

```
1. 识别问题类型
   └── 实现 / 审查 / 测试 / 部署

2. 分析根因
   └── 找到问题的根本原因

3. 制定解决方案
   └── 确定修复方案

4. 执行修复
   └── 按方案修复

5. 验证修复
   └── 确保问题解决

6. 预防措施
   └── 避免同类问题
```
