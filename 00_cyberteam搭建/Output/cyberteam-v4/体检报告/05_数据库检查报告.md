### 2.2 发现的问题

#### [CRITICAL-1] TodoItem 表未创建

**位置**: `backend/app/models/__init__.py:191-229`

**问题**: 模型定义存在，但 SQLite 数据库中没有创建 `todo_items` 表。

```python
# models/__init__.py:191
class TodoItem(Base):
    """TODO 项表。"""
    __tablename__ = "todo_items"
    # ... 模型定义完整
```

**实际数据库表**:
```
$ sqlite3 cyberteam.db ".tables"
tasks  expert_outputs  debate_rounds  department_outputs  final_reports
# 注意: todo_items 不存在!
```

**影响**:
- TODO API (`backend/app/api/todos.py`) 所有 CRUD 操作会失败
- `/todos` 端点返回 500 错误
- 用户无法创建任何 TODO 项

**修复建议**:
1. 执行 `Base.metadata.create_all()` 时确保 `TodoItem` 被注册
2. 手动运行 SQL 创建表:
   ```sql
   CREATE TABLE todo_items (
       id VARCHAR(36) PRIMARY KEY,
       title VARCHAR(500) NOT NULL,
       description TEXT,
       state VARCHAR(11) NOT NULL DEFAULT 'todo',
       priority VARCHAR(6) NOT NULL DEFAULT 'medium',
       category VARCHAR(100),
       tags JSON NOT NULL DEFAULT '[]',
       due_date DATETIME,
       completed_at DATETIME,
       created_at DATETIME NOT NULL,
       updated_at DATETIME NOT NULL
   );
   ```

#### [HIGH-1] 缺失外键约束

**问题**: 所有子表 (`expert_outputs`, `debate_rounds`, `department_outputs`) 的 `task_id` 列都没有外键约束。

```sql
-- expert_outputs.task_id 没有 FK 约束
CREATE TABLE expert_outputs (
    id INTEGER NOT NULL,
    task_id VARCHAR(36) NOT NULL,  -- 应该添加: REFERENCES tasks(task_id)
    ...
);
```

**影响**:
- 允许创建指向不存在任务的记录
- 删除任务时不会自动清理子记录 (孤立数据)
- 无法保证引用完整性

**修复建议**:
```sql
-- 添加外键约束 (需要重建表)
ALTER TABLE expert_outputs
ADD CONSTRAINT fk_expert_outputs_task
FOREIGN KEY (task_id) REFERENCES tasks(task_id) ON DELETE CASCADE;
```

---

#### [HIGH-2] 缺失唯一约束

**位置**: `backend/app/models/__init__.py:95-122`

**问题**: `ExpertOutput` 缺少 `(task_id, expert_id, round)` 唯一约束，允许重复记录。

```python
class ExpertOutput(Base):
    """专家输出表。"""
    __tablename__ = "expert_outputs"
    id: Mapped[int] = mapped_column(Integer, primary_key=True, autoincrement=True)
    task_id: Mapped[str] = mapped_column(String(36), index=True)
    expert_id: Mapped[str] = mapped_column(String(50))
    round: Mapped[int] = mapped_column(Integer, default=1)
    # 缺少: __table_args__ = (UniqueConstraint('task_id', 'expert_id', 'round'),)
```

**影响**:
- 同一专家对同一任务可以创建多条同一轮次的输出
- 评分聚合可能计算错误

**修复建议**:
```python
from sqlalchemy import UniqueConstraint

class ExpertOutput(Base):
    __tablename__ = "expert_outputs"
    __table_args__ = (
        UniqueConstraint('task_id', 'expert_id', 'round', name='uq_expert_output'),
    )
```

## 三、数据库连接检查

### 3.1 连接配置

**文件**: `backend/app/db.py`

```python
_engine = create_async_engine(

    echo=settings.debug,
    future=True,
)
```

### 3.2 发现的问题

#### [MEDIUM-1] 缺少连接池配置

**问题**: SQLite 使用 `aiosqlite`，没有配置连接池大小。

```python
# 当前配置
create_async_engine(settings.database_url, echo=settings.debug, future=True)

# 建议配置
create_async_engine(
    settings.database_url,
    echo=settings.debug,
    future=True,
    pool_size=5,        # 不适用于 SQLite
    max_overflow=10,    # 不适用于 SQLite
    pool_pre_ping=True,
)
```

**说明**: SQLite 不支持真正的连接池，但可以配置 `connect_args`:
```python
create_async_engine(
    settings.database_url,
    connect_args={"check_same_thread": False},
)
```

#### [MEDIUM-2] 缺少数据库健康检查

**位置**: `backend/app/api/health.py:66-79`

**问题**: `readiness_check` 硬编码返回 `"database": "ok"`，没有真正检查。

```python
@router.get("/ready")
async def readiness_check():
    return {
        "status": "ready" if ok_count == len(modules) else "degraded",
        "checks": {
            "database": "ok",  # 硬编码!
            # ...
        },
    }
```

**修复建议**:
```python
@router.get("/ready")
async def readiness_check(db: AsyncSession = Depends(get_db)):
    try:
        await db.execute(select(1))
        db_status = "ok"
    except Exception as e:
        db_status = f"error: {e}"
    return {"checks": {"database": db_status, ...}}
```

## 四、持久化逻辑检查

### 4.1 Projects API 使用内存存储

#### [CRITICAL-2] Projects 数据不持久化

**位置**: `backend/app/api/projects.py:23-26`

**问题**: 项目数据存储在内存字典中，服务重启后丢失。

```python
# 项目数据存储（临时使用内存，后续迁移到数据库）
_projects: Dict[str, Dict[str, Any]] = {}  # 服务重启后清空!
```

**影响**:
- 所有项目数据在服务重启后丢失
- 无法跨会话保持项目状态
- 违反数据持久化原则

**修复建议**:
1. 创建 `Project` 模型迁移到数据库
2. 使用 `Projects` 表替代内存字典

```python
# 建议的 Project 模型
class Project(Base):
    __tablename__ = "projects"
    project_id: Mapped[str] = mapped_column(String(36), primary_key=True)
    name: Mapped[str] = mapped_column(String(200))
    description: Mapped[Optional[str]] = mapped_column(Text)
    goal: Mapped[str] = mapped_column(Text)
    status: Mapped[str] = mapped_column(String(20), default="active")
    tags: Mapped[str] = mapped_column(JSON, default=list)
    metadata: Mapped[Optional[str]] = mapped_column(JSON)
    created_at: Mapped[datetime] = mapped_column(DateTime, default=datetime.utcnow)
    updated_at: Mapped[datetime] = mapped_column(DateTime, default=datetime.utcnow)
```

## 八、其他问题

### [LOW-1] 使用已废弃的 datetime.utcnow

**位置**: `backend/app/models/__init__.py:64`

```python
created_at: Mapped[datetime] = mapped_column(DateTime, default=datetime.utcnow)
```

**说明**: Python 3.12+ 已废弃 `datetime.utcnow()`，应使用 `datetime.now(timezone.utc)`。

**修复建议**:
```python
from datetime import datetime, timezone

created_at: Mapped[datetime] = mapped_column(
    DateTime(timezone=True),
    default=lambda: datetime.now(timezone.utc)
)
```

---

### [LOW-2] 状态枚举使用 VARCHAR

**问题**: `TaskState` 枚举存储为 VARCHAR(9)，可以使用 INTEGER 提高效率。

```sql
-- 当前
state VARCHAR(9) NOT NULL  -- 'pending', 'thinking', ...

-- 建议
state INTEGER NOT NULL     -- 0, 1, 2, ...
```

**说明**: 考虑到可读性和 SQLite 特性，当前方案可接受。
