## P0-1: CEO模块 TeamMessage 未定义错误

### 问题描述
- **文件**: `engine/ceo/launcher.py`
- **问题**: 当 `MAILBOX_AVAILABLE=False` 时，`TeamMessage` 类型未定义，但类型注解仍使用它
- **错误**: `NameError: name 'TeamMessage' is not defined` 在第130行

### 修复方案
在 `try-except ImportError` 块中添加 `TeamMessage = Any` 回退定义：

```python
try:
    from CYBERTEAM.team.models import MessageType, TeamMessage
    from CYBERTEAM.mcp.mailbox import MailboxManager
    MAILBOX_AVAILABLE = True
except ImportError:
    MAILBOX_AVAILABLE = False
    TeamMessage = Any  # 回退类型，避免类型注解报错
    logging.warning("MailboxManager 不可用，使用本地模拟模式")
```

### 验证结果
```bash
$ python3 -c "from engine.ceo.ceo import CEORouter; print('CEO OK')"
CEO OK
```

## P0-3: 内存存储缺少警告提示

### 问题描述
- **文件**: 
  - `backend/app/api/skills.py`
  - `backend/app/api/teams.py`
  - `backend/app/api/projects.py`
- **问题**: 使用内存字典存储数据，重启后丢失，但没有警告提示

### 修复方案
在每个文件的内存存储定义后添加警告日志：

```python
# ── 内存存储（生产环境应使用数据库）──
# ⚠️ 警告：此存储在进程重启后丢失。生产环境应使用数据库。
_skills_db: Dict[str, dict] = {}
logging.warning("Skills API 使用内存存储，进程重启后数据将丢失（仅用于开发/测试）")
```

### 验证结果
所有三个文件都已添加警告日志。

## 根因分析

### 问题1 & 2: 模块导入链断裂

**根本原因**: 代码重构不完整，某些模块导出与使用不匹配

**教训**: 
- 每次重构后应运行完整的导入测试
- 导出列表 (`__all__`) 应与实际使用的符号保持同步
- 缺少的符号应创建而不是移除引用

### 问题3: 开发环境与生产环境混淆

**根本原因**: 开发阶段使用内存存储是合理的，但缺少明确警告

**教训**:
- 内存存储应有明确的警告标识
- 注释应说明这是开发用途
