# downstream - 运营总监

## 下游去向

| 下游 | 角色 | 调度方式 | 产出要求 |
|------|------|----------|----------|
| 用户运营专家 | 直接下级 | 任务dispatch → 会议讨论 → 方案交付 | 方案存入 `{project}/decisions/` |
| 内容运营专家 | 直接下级 | 任务dispatch → 会议讨论 → 方案交付 | 方案存入 `{project}/decisions/` |
| 活动运营专家 | 直接下级 | 任务dispatch → 会议讨论 → 方案交付 | 方案存入 `{project}/decisions/` |
| 增长总监 | 上级汇报 | 方案呈报 → 决策获取 | 决策记录存入 `{project}/decisions/` |

---

## 调度流程

```
┌─────────────────────────────────────────────────────────────────┐
│  1. 创建项目（运营总监）                                         │
│     → projects/{project_id}/metadata.yaml                      │
│     → projects/{project_id}/context/business_context.md          │
└─────────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────────┐
│  2. 向下调度（运营总监 → 专家Agent）                             │
│     → 发送 task_dispatch JSON                                   │
│     → 指定 input_files（必须读取的上下文）                        │
│     → 指定 expected_output（输出文件路径）                        │
└─────────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────────┐
│  3. 专家执行并回报                                               │
│     → 专家读取 input_files                                      │
│     → 专家生成方案 → 存入 expected_output                        │
│     → 专家发送 task_completion JSON                             │
└─────────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────────┐
│  4. 会议讨论（可选）                                             │
│     → 运营总监召集专家会议                                       │
│     → 记录存入 projects/{project_id}/meetings/meeting_*.md      │
└─────────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────────┐
│  5. 方案汇总上报（运营总监 → 增长总监）                            │
│     → 生成 options.md（A/B/C方案）                              │
│     → 增长总监决策                                               │
│     → 决策记录存入 decisions/decision.md                        │
└─────────────────────────────────────────────────────────────────┘
```

## 专家Agent任务模板

### 任务dispatch格式

```json
{
  "type": "task_dispatch",
  "project_id": "proj_20260325_001",
  "task_id": "task_001",
  "task_name": "用户分层方案设计",
  "expert": "用户运营专家",
  "input_files": [
    "projects/proj_20260325_001/context/business_context.md",
    "projects/proj_20260325_001/context/user_persona.md"
  ],
  "expected_output": "projects/proj_20260325_001/decisions/user_stratification.md",
  "template": "projects/_template/decisions/options.md",
  "deadline": "2026-03-26T18:00:00Z",
  "callback_meeting": "mtg_20260326_001"
}
```

### 专家产出验收标准

| 产出类型 | 验收标准 |
|----------|----------|
| 诊断报告 | 包含问题/根因/影响/量化指标 |
| 方案选项 | 包含A/B/C三方案，每个方案有资源/收益/风险 |
| 执行计划 | 包含负责人/时间节点/成功指标 |
| 用户画像 | 包含人群定义/核心行为/需求痛点 |

## 升级机制

| 情况 | 升级条件 | 升级内容 | 接收方 |
|------|----------|----------|--------|
| 资源不足 | 人力/预算无法支撑任务 | 资源申请+替代方案 | 增长总监 |
| 跨部门阻塞 | 产品/技术无法配合 | 协调需求+时间影响 | 增长总监 |
| 战略分歧 | 与增长总监判断不一致 | 数据分析+推荐方案 | CEO |
