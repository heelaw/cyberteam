# downstream - 增长BG Agent

## 直接下游

| 下游Agent | 关系 | 说明 |
|-----------|------|------|
| 技术BG | 技术依赖 | 需要技术实现时 |
| 资产中台 | 资产调用 | 调用baoyu技能（图片生成、文案创作等） |
| 通信中台 | 任务交接 | 需要多Agent协作时 |
| 监控中台 | 进度汇报 | 任务进度和状态更新 |

## 输出物

| 输出物 | 下游使用方 | 用途 |
|--------|------------|------|
| 内容文案 | 发布渠道 | 公众号/小红书/抖音 |
| 活动方案 | 运营执行 | SOP文档 |
| 增长策略 | 技术BG | 系统功能需求 |
| 数据分析 | 决策层 | 战略决策支持 |
| 用户画像 | 产品BG | 产品设计参考 |

## 产出分类

### 内容运营产出

| 产出物 | 格式 | 下游使用 |
|--------|------|----------|
| 小红书文案 | markdown | 发布 |
| 抖音脚本 | markdown | 拍摄 |
| 公众号文章 | markdown | 发布 |
| 配图建议 | JSON | baoyu生成 |

### 活动运营产出

| 产出物 | 格式 | 下游使用 |
|--------|------|----------|
| 活动方案 | markdown | 执行 |
| 传播路径图 | markdown | 设计 |
| 物料清单 | JSON | 采购 |
| SOP文档 | markdown | 执行 |

### 数据分析产出

| 产出物 | 格式 | 下游使用 |
|--------|------|----------|
| 数据报告 | markdown | 决策 |
| 趋势图表 | JSON | 可视化 |
| 优化建议 | markdown | 执行 |

## 下游数据流

```
增长BG执行
    ↓
    ├── 内容产出 → 资产中台(baoyu) → 发布渠道
    ├── 活动方案 → 运营执行团队
    ├── 技术需求 → 技术BG → 开发实现
    └── 数据报告 → CEO/PM → 决策支持
```
