# upstream - 增长BG Agent

## 直接上游

| 上游Agent | 关系 | 说明 |
|-----------|------|------|
| CEO | 任务路由 | 增长BG的唯一上游，接收CEO的L3A路由任务 |
| PM协调器 | 任务协同 | 高复杂度任务时的PM协调 |
| 记忆中台 | 数据依赖 | 获取历史运营数据和案例 |

## 数据依赖

| 数据源 | 格式 | 刷新频率 | 说明 |
|--------|------|----------|------|
| 用户任务描述 | text | 每次任务 | CEO路由传递 |
| 历史内容数据 | JSON | 按需 | 记忆中台召回 |
| 竞品数据 | JSON | 每周更新 | 市场研究数据 |
| 用户画像 | JSON | 每月更新 | 目标用户特征 |

## 运营专业上游

| 上游资源 | 接口类型 | 说明 |
|----------|----------|------|
| 运营操盘手Agent | internal | 业务认知、商业模型 |
| 内容运营Agent | internal | 内容策划、矩阵搭建 |
| 用户运营Agent | internal | 用户生命周期、会员体系 |
| 增长Agent | internal | AARRR、裂变策略 |
| 数据驱动Agent | internal | 数据分析、指标体系 |

## 上游数据流

```
用户输入 → CEO路由 → 增长BG
                      ↑
                  记忆中台（历史数据）
                  运营专家Agent团（专业能力）
```
