# upstream - CEO 总指挥

## 直接上游

| 上游Agent | 关系 | 说明 |
|-----------|------|------|
| 用户 | 直接输入 | CEO的唯一上游，接收用户的所有任务输入 |
| 记忆中台 | 数据依赖 | 获取历史任务上下文和记忆 |
| 通信中台 | 消息依赖 | 接收用户反馈和外部消息 |

## 数据依赖

| 数据源 | 格式 | 刷新频率 | 说明 |
|--------|------|----------|------|
| 用户输入 | text | 实时 | 每次用户交互 |
| 历史上下文 | JSON | 每次任务 | 从记忆中台获取 |
| 会话状态 | dict | 实时 | 当前会话上下文 |
| 专家知识库 | markdown | 按需 | 思维注入引擎调用 |

## 外部依赖

| 外部系统 | 接口类型 | 说明 |
|----------|----------|------|
| gstack Skills | /plan-eng-review, /review, /qa | 工程计划审查、代码审查、测试 |
| CyberTeam Adapter | API | Swarm团队创建 |
| 思维注入引擎 | SDK | 100+专家思维调用 |

## 特殊上游说明

1. **用户输入**: CEO的所有决策都基于用户输入，不需要其他Agent的直接上游
2. **质量门禁**: DevQA循环的结果会影响路由决策（L1计划审批、L3结果评审）
3. **思维专家**: 通过CEOThinkingEngine按需调用100+思维专家

## 上游数据流

```
用户输入 → CEO路由 → [质量门禁] → 路由决策
                ↑           ↑
            记忆中台      DevQA循环
            获取上下文    质量保障
```
