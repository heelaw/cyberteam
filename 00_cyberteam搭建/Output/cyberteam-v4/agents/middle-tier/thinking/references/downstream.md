# downstream - 思维注入中台

## 直接下游

| 下游Agent | 关系 | 说明 |
|-----------|------|------|
| CEO | 决策支持 | CEO路由决策时的思维支撑 |
| PM协调器 | 规划支持 | 任务拆解和方案设计 |
| 部门BG | 执行支持 | 各BG执行的思维注入 |
| 辩论引擎 | 观点注入 | 多专家观点辩论 |

## 输出物

| 输出物 | 下游使用方 | 用途 |
|--------|------------|------|
| 专家列表 | 调用方 | 了解被注入的专家 |
| 分析框架 | CEO/PM | 决策参考 |
| 多角度观点 | 辩论引擎 | 辩论输入 |
| 思维模式 | 执行Agent | 执行指导 |

## 注入报告格式

```yaml
思维注入报告:
  场景识别: {场景类型}
  注入专家:
    - {专家ID} ({专家名称}): {核心思维}
  专家组合: {数量}/100
  注入强度: {百分比}%
  协同度评分: {分数}
  综合置信度: {置信度}
  分析框架:
    - 框架1: {说明}
    - 框架2: {说明}
```

## 输出分类

### 战略类输出

| 输出物 | 格式 | 下游使用 |
|--------|------|----------|
| 战略分析框架 | markdown | CEO决策 |
| 竞争分析 | markdown | 战略规划 |
| 市场进入方案 | markdown | 战略执行 |

### 产品类输出

| 输出物 | 格式 | 下游使用 |
|--------|------|----------|
| 需求分析 | markdown | 产品BG |
| 用户体验框架 | markdown | 产品BG |
| 增长策略 | markdown | 增长BG |

### 技术类输出

| 输出物 | 格式 | 下游使用 |
|--------|------|----------|
| 架构评审 | markdown | 技术BG |
| 安全性分析 | markdown | 安全审查 |
| 性能优化建议 | markdown | 技术BG |

## 下游数据流

```
思维注入
    ↓
    ├── 专家列表 + 分析框架 → CEO → 路由决策
    ├── 专家列表 + 任务拆解 → PM协调器 → 任务分解
    ├── 专家思维 + 执行指导 → 部门BG → 任务执行
    └── 多专家观点 → 辩论引擎 → 观点收敛
```
