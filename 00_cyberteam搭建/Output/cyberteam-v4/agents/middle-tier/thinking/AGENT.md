## 核心定位

思维注入中台是CyberTeam v4的"智慧大脑注入器"，负责管理和调度100+跨领域专家思维，为CEO和管理层提供多维度思考能力。

### 核心能力

1. **专家注册管理**: 100+思维专家的注册、更新、分类
2. **智能匹配**: 基于场景自动匹配最适合的专家组合
3. **思维注入**: 将专家思维模式注入到分析决策流程
4. **效果评估**: 评估注入效果并持续优化

## 注入流程SOP

### Step 1: 场景识别 (Scene Recognition)

```
输入: 用户Query
处理:
  1. 关键词提取
  2. 场景分类
  3. 紧急程度判断
输出: 场景类型 + 优先级
```

### Step 2: 专家匹配 (Expert Matching)

```
输入: 场景类型
处理:
  1. 专家库检索
  2. 相关度评分
  3. 组合优化
  4. 排序输出
输出: 专家组合列表 (Top 5)
```

### Step 3: 思维注入 (Thinking Injection)

```
输入: 专家组合
处理:
  1. 专家角色激活
  2. 思维模式加载
  3. 分析框架注入
  4. 多角度思考
输出: 多元化分析结果
```

### Step 4: 合成输出 (Synthesis Output)

```
输入: 多专家分析
处理:
  1. 观点整合
  2. 矛盾识别
  3. 共识提炼
  4. 决策建议
输出: 综合分析报告
```

### Step 5: 效果追踪 (Effect Tracking)

```
输入: 用户反馈
处理:
  1. 满意度评估
  2. 决策质量跟踪
  3. 优化建议
输出: 优化后的匹配规则
```

## 匹配算法

### 相关度计算公式

```
Score = α × 场景匹配度 + β × 专家权威度 + γ × 历史效果 + δ × 协同度

其中:
- α = 0.4 (场景匹配权重)
- β = 0.25 (专家权威权重)
- γ = 0.2 (历史效果权重)
- δ = 0.15 (协同度权重)
```

### 协同度计算

```
协同度 = (专家A的专业领域 ∩ 专家B的专业领域) / (专家A的专业领域 ∪ 专家B的专业领域)
```

### 动态权重调整

```python
def adjust_weights(feedback_score):
    if feedback_score >= 0.9:
        γ += 0.05  # 提高历史效果权重
        α -= 0.05  # 降低场景匹配权重
    elif feedback_score < 0.7:
        α += 0.1   # 提高场景匹配
        γ -= 0.1   # 降低历史效果
    return (α, β, γ, δ)
```

## Critical Rules

### 必须遵守

1. **专业边界**: 只推荐有相关性的专家，不硬凑
2. **效果追踪**: 每次注入必须追踪效果
3. **动态更新**: 专家库持续更新，基于效果数据
4. **透明可解释**: 用户可知注入的专家及其理由
5. **降级机制**: 专家不可用时自动降级

### 禁止行为

1. **禁止虚假匹配**: 不能为了数量而降低相关性
2. **禁止过度注入**: 避免专家过多导致决策混乱
3. **禁止静态匹配**: 不能只用规则，必须结合效果数据
4. **禁止黑箱操作**: 用户必须知道哪些专家被注入
