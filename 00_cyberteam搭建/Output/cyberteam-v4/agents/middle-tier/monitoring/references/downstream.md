# downstream - 监控中台

## 直接下游

| 下游系统 | 关系 | 说明 |
|----------|------|------|
| CEO | 告警上报 | P0/P1告警上报CEO |
| PM协调器 | 进度汇报 | 阻塞和风险通知 |
| 执行Agent | 干预指令 | 循环突破指令 |
| 通信中台 | 告警通知 | 消息路由 |
| 外部告警系统 | 告警转发 | Alertmanager/钉钉/邮件 |

## 输出物

| 输出物 | 下游使用方 | 用途 |
|--------|------------|------|
| 循环告警 | CEO/PM | 异常干预决策 |
| 进度报告 | CEO/PM | 项目状态了解 |
| 风险预警 | PM协调器 | 风险管理 |
| 干预指令 | 执行Agent | 循环突破指导 |
| 告警通知 | 外部系统 | 告警通知 |

## 告警报告格式

```yaml
告警报告:
  告警ID: ALT-{日期}-{序号}
  时间: {触发时间}
  持续: {持续时长}
  Agent: {Agent名称}
  循环类型: {类型}
  循环深度: {深度值}
  影响任务: {任务ID} ({任务描述})
  建议操作: [操作1] [操作2]
  响应状态: {状态}
  SLA剩余: {剩余时间}
```

## 进度报告格式

```yaml
进度报告:
  总体进度: {已完成}/{总任务} ({百分比})
  各状态统计:
    运行中: {数量}
    阻塞: {数量}
    待启动: {数量}
    完成: {数量}
  风险提示:
    高风险: {数量}个任务循环
    中风险: {数量}个任务超时
    阻塞: {数量}个任务等待依赖
  预计完成: {时间}
  信心度: {百分比}
```

## 输出分类

### 实时告警输出

| 输出物 | 格式 | 下游使用 |
|--------|------|----------|
| 循环检测告警 | JSON | CEO/PM干预 |
| 超时告警 | JSON | PM协调 |
| 死锁告警 | JSON | CEO/PM立即处理 |
| 资源告警 | JSON | DevOps处理 |

### 定期报告输出

| 输出物 | 格式 | 下游使用 |
|--------|------|----------|
| 每日进度报告 | markdown | CEO/PM日报 |
| 周报 | markdown | 管理层汇报 |
| 风险评估 | JSON | PM风险管理 |
| 系统健康报告 | JSON | DevOps |

## 下游数据流

```
监控中台
    ↓
    ├── P0/P1告警 → CEO → 立即干预
    ├── 进度报告 → PM协调器 → 任务管理
    ├── 风险预警 → PM协调器 → 风险管理
    ├── 循环干预 → 执行Agent → 策略调整
    ├── 告警通知 → 通信中台 → 消息推送
    └── 告警转发 → 外部系统 → 钉钉/邮件/短信
```
