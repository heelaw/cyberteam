## 一、设计目标

V4 核心引擎实现从"方案制定"到"部门执行"的完整流程，连接 L1/L2/L3 三层架构：

```
用户输入 → CEO(L1) 路由 → Strategy+PM(L2) 协调 → 部门执行(L3) → 结果聚合 → 用户交付
```

## 三、执行流程

```
                    ┌─────────────┐
                    │   用户输入   │
                    └──────┬──────┘
                           ↓
                    ┌─────────────┐
                    │  CEO 路由   │  ← L1: 意图识别 + 复杂度评估
                    │  (ceo.py)  │
                    └──────┬──────┘
                           ↓
                    ┌─────────────┐
                    │  Strategy   │  ← L2: 5W1H1Y + MECE + 方案设计
                    │(strategy.py)│
                    └──────┬──────┘
                           ↓
                    ┌─────────────┐
                    │  PM 协调    │  ← L2: 任务调度 + 进度监控
                    │  (pm.py)   │
                    └──────┬──────┘
                           ↓
         ┌─────────────────┼─────────────────┐
         ↓                 ↓                 ↓
    ┌─────────┐      ┌─────────┐      ┌─────────┐
    │ 部门执行 │      │ Gstack  │      │ 独立    │
    │(dept.py)│      │ Skills  │      │ Agents  │
    └────┬────┘      └─────────┘      └─────────┘
         │
         └──────────────┬───────────────────┘
                        ↓
                 ┌─────────────┐
                 │ 结果聚合    │  ← 整合 L3 结果
                 └──────┬──────┘
                        ↓
                 ┌─────────────┐
                 │  最终交付   │  ← 返回用户
                 └─────────────┘
```

### 5.2 核心类

```python
@dataclass
class RoutingResult:
    decision: str           # "路由决策"
    target: str             # "L2"|"L3A"|"L3B"|"L3C"
    target_name: str        # "PM"|"数据分析部"|"/review"|"gsd-executor"
    intent: str             # "识别的意图"
    complexity: str         # "高"|"中"|"低"
    reason: str             # "路由理由"

@dataclass
class ExecutionPlan:
    task_id: str
    decomposition: dict     # WBS 任务树
    framework: str          # 选用的思维框架
    schedule: list          # 执行顺序
    resources: dict         # 需要的资源

@dataclass
class ExecutionResult:
    status: str             # "success"|"failure"
    outputs: dict           # 各部门产出
    metrics: dict           # 性能指标
    errors: list            # 错误列表
```

## 七、质量保障

### 7.1 五级质量门禁

| 级别 | 检查点 | 通过条件 |
|------|--------|----------|
| L0 | 输入校验 | 无错误 |
| L1 | 计划审批 | ≥70分 |
| L2 | 过程检查 | 正常/警告 |
| L3 | 结果评审 | ≥70分 |
| L4 | 交付终审 | ≥75分 |

### 7.2 PUA 触发机制

```python
# 自动触发 PUA
if task.failed_count >= 2:
    pua.activate(task, level=task.failed_count - 1)
```
