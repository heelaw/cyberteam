## 2. 7个专家Agent能力清单

### 2.1 增长BG总监 (Growth-BG-Leader)

#### 核心能力来源
- **策略规划Agent**: 战略规划框架、目标拆解方法
- **团队管理Agent**: 团队建设方法论、管理工具箱
- **策略执行Agent**: 执行监控体系、结果追踪方法

#### 集成Skills列表
**通用型（可授权给全公司）**：
- `growth-reporting`: 汇报框架（结构化向上沟通）
- `growth-time-mgmt`: 时间管理（优先级四象限）
- `growth-progress-track`: 进度管理（甘特图+里程碑）
- `growth-retrospective`: 复盘模板（目标达成+数据+问题+行动）
- `growth-1v1-communication`: 1v1沟通（一对一SOP）
- `growth-team-incentive`: 团队激励（激励设计四步法）
- `growth-intent-analysis`: 高层意图识别（5W1H1Y问题拆解）
- `growth-talent-review`: 人力盘点（能力评估矩阵）
- `growth-idp-plan`: IDP计划（个人发展计划）

**专用型（仅总监使用）**：
- `growth-strategy-planning`: 策略规划（战略框架+目标拆解）
- `growth-execution-monitor`: 执行监控（结果追踪+预警机制）

#### 子模块
无子模块（总监为管理角色）

#### 决策链概述
1. 接收CEO任务 → 使用`growth-intent-analysis`拆解问题
2. 判断任务类型 → 匹配对应专家或组建专家组
3. 分配任务 + 授权Skill → 监控执行进度
4. 汇总结果 → 使用`growth-reporting`向CEO汇报

### 3.2 专家之间的协作模式

#### 协作模式1：战役模式（全员协同）
- **触发场景**：新品上市、618/双11大促、品牌战役
- **参与专家**：全部6个专家
- **协作Skill**：全部协作型Skill
- **决策链**：
  1. 总监使用`growth-strategy-planning`制定战役战略
  2. 各专家使用专用Skill执行各自任务
  3. 通过协作型Skill共享数据和进度
  4. 总监使用`growth-execution-monitor`监控进度
  5. 战役结束后使用`growth-retrospective`复盘

#### 协作模式2：专家组模式（2-3个专家协同）
- **触发场景**：用户增长、内容营销、品牌建设
- **参与专家**：相关专家2-3个
- **协作Skill**：相关协作型Skill
- **示例**：
  - **用户增长**：增长营销 + 用户运营 + 效果营销
  - **内容营销**：内容运营 + 品牌营销 + 效果营销
  - **品牌建设**：品牌营销 + 内容运营 + 活动运营

#### 协作模式3：单兵模式（单个专家独立处理）
- **触发场景**：日常运营、独立任务
- **参与专家**：单个专家
- **协作Skill**：仅使用专家专用Skill
- **示例**：
  - 日常内容生产 → 内容运营专家独立处理
  - 日常投放优化 → 效果营销专家独立处理

### 3.3 Skills调用权限矩阵

| Skill类型 | 权限范围 | 调用规则 | 示例 |
|----------|---------|---------|------|
| **通用型Skill** | 全公司可用 | 需总监授权 | growth-reporting, growth-time-mgmt |
| **专家型Skill** | 仅对应专家 | 自动调用 | user-retention-diagnosis（仅用户运营） |

## 4. 测试场景（5个典型任务）

### 场景1：新品上市战役（完整流程）

#### 任务描述
某SaaS产品发布新版本，需要制定完整的上市营销战役。

#### 执行流程

**阶段1：战略规划（增长BG总监）**
```
调用Skill：
- growth-intent-analysis: 拆解CEO需求
- growth-strategy-planning: 制定战役战略
- growth-progress-track: 建立战役进度表

产出：
- 战役目标：30天内获取10000个新用户
```
