## 🔧 融合实施方案

### 步骤1：为每个 v4 Agent 添加 `references` 字段

在每个 `AGENT.md` 中添加：

```yaml
## references

### 核心参考 Agent
- 用户运营Agent: ../../skills/v3-agents/ops/用户运营Agent/
- 用户激励Agent: ../../skills/v3-agents/ops/用户激励Agent/
- 用户洞察Agent: ../../skills/v3-agents/ops/用户洞察Agent/

### 配套 Skills
- 用户留存诊断: ../../skills/v3-agents/ops/用户激励Agent/配套skill/用户留存诊断/
- 横纵相交模型: ../../skills/v3-agents/ops/用户激励Agent/配套skill/横纵相交模型/
- 价值洼地诊断: ../../skills/v3-agents/ops/用户激励Agent/配套skill/价值洼地诊断/
- 激励体系设计: ../../skills/v3-agents/ops/用户激励Agent/配套skill/激励体系设计/
```

### 步骤2：创建 Skill 符号链接

在 `cyberteam-v4` 中创建 `skills/` 目录，为每个 v3 Skill 创建符号链接：

```bash
cd Output/cyberteam-v4/
mkdir -p skills/user-operations/
cd skills/user-operations/
ln -s ../../skills/v3-agents/ops/用户激励Agent/配套skill/用户留存诊断/ user-operations-retention-diagnosis
```

### 步骤3：更新 Agent 配置

在每个 `AGENT.md` 的 `skills` 字段中添加引用：

```yaml
## skills

### 核心 Skills
- user-operations-retention-diagnosis: 用户留存诊断
- user-operations-funnel-analysis: 横纵相交模型
- user-operations-value-gap: 价值洼地诊断
- user-operations-incentive-design: 激励体系设计
```
