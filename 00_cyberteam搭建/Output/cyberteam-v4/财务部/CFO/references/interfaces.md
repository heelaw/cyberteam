# CFO 接口协议

## 输入接口

### 来自CEO
| 消息类型 | 内容 | 触发条件 |
|---------|------|----------|
| 战略咨询 | 请求财务视角的战略建议 | CEO决策前 |
| 预算审批 | 年度/季度预算审批 | 例行审批 |
| 投资审批 | 重大投资决策意见 | 项目评审 |
| 危机响应 | 财务风险事件响应 | 突发事件 |

### 来自各BG
| 消息类型 | 内容 | 触发条件 |
|---------|------|----------|
| 预算申请 | 各部门预算请求 | 预算周期 |
| 成本咨询 | 成本核算问题 | 日常咨询 |
| 投资提案 | 项目投资申请 | 项目发起 |

## 输出接口

### 流向CEO
| 消息类型 | 内容 | 频率 |
|---------|------|------|
| 财务月报 | 财务报表+分析 | 每月 |
| 预算审批 | 预算审批意见 | 按需 |
| 投资意见 | 投资决策建议 | 按需 |
| 风险预警 | 财务风险提示 | 实时 |

### 流向各BG
| 消息类型 | 内容 | 频率 |
|---------|------|------|
| 预算批复 | 预算分配结果 | 年度 |
| 成本反馈 | 成本分析报告 | 按需 |
| 投资评估 | 项目评估报告 | 按需 |

## 内部接口

### 流向预算管理
```
输入: CEO战略目标、业务BG预算申请
输出: 预算编制指导、预算批复
```

### 流向成本控制
```
输入: 各部门费用数据、投资项目
输出: 成本分析要求、降本建议
```

### 流向投资分析
```
输入: 投资项目提案、战略优先级
输出: 投资评估任务、评估标准
```

## 数据格式

### 财务报告标准格式
```json
{
  "report_type": "月度/季度/年度报告",
  "period": "2026-Q1",
  "highlights": ["核心指标摘要"],
  "data": {
    "revenue": "收入数据",
    "cost": "成本数据",
    "profit": "利润数据",
    "cash_flow": "现金流数据"
  },
  "analysis": "财务分析",
  "risk_alerts": ["风险提示"],
  "recommendations": ["建议"]
}
```

### 预算申请标准格式
```json
{
  "department": "申请部门",
  "budget_type": "资本支出/运营费用",
  "amount": "申请金额",
  "purpose": "用途说明",
  "expected_roi": "预期回报",
  "risk_assessment": "风险评估",
  "alternatives": "替代方案"
}
```
