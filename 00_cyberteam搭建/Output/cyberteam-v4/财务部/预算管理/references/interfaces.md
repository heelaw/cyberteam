# 预算管理 接口协议

## 输入接口

### 来自CFO
| 消息类型 | 内容 | 触发条件 |
|---------|------|----------|
| 编制指令 | 年度预算编制启动 | 每年1月 |
| 调整审批 | 重大预算调整审批 | 超阈值申请 |
| 分析需求 | 预算执行分析需求 | 例行/按需 |

### 来自各BG
| 消息类型 | 内容 | 触发条件 |
|---------|------|----------|
| 预算申请 | 部门年度/季度预算 | 编制周期 |
| 调整申请 | 预算调整请求 | 业务变化 |
| 咨询请求 | 预算政策咨询 | 日常 |

## 输出接口

### 流向CFO
| 消息类型 | 内容 | 频率 |
|---------|------|------|
| 预算汇总 | 年度预算汇总方案 | 2月 |
| 执行报告 | 月度预算执行报告 | 每月 |
| 调整建议 | 调整审批建议 | 按需 |
| 预警通知 | 重大偏差预警 | 实时 |

### 流向各BG
| 消息类型 | 内容 | 频率 |
|---------|------|------|
| 编制指南 | 年度预算编制指引 | 每年1月 |
| 执行反馈 | 月度执行情况通知 | 每月 |
| 调整批复 | 调整申请批复 | 按需 |
| 预警提醒 | 预算预警通知 | 实时 |

### 流向成本控制
```json
{
  "data_type": "预算费用数据",
  "period": "2026-Q1",
  "department": "增长BG",
  "amount": 5000000,
  "category": "运营费用",
  "breakdown": {
    "人力成本": 3000000,
    "市场费用": 1500000,
    "运营费用": 500000
  }
}
```

## 数据格式

### 预算申请表
```json
{
  "department": "部门名称",
  "fiscal_year": 2026,
  "budget_type": "年度预算/季度调整",
  "total_amount": "预算总额",
  "items": [
    {
      "item_name": "费用项目",
      "amount": "金额",
      "yoy_change": "同比变化",
      "justification": "申请理由",
      "expected_output": "预期产出"
    }
  ],
  "approval_status": "待审批"
}
```

### 预算执行报告
```json
{
  "period": "2026-Q1",
  "department": "增长BG",
  "budget_total": 5000000,
  "budget_used": 1200000,
  "execution_rate": 24.0,
  "forecast": 4800000,
  "variance": -200000,
  "variance_rate": -4.0,
  "alerts": ["需要关注：市场费用超支"],
  "recommendations": ["建议增加市场费用弹性预算"]
}
```

### 预警通知
```json
{
  "alert_type": "budget_variance",
  "severity": "warning",
  "department": "产品BG",
  "budget_item": "研发人力",
  "threshold": 10,
  "actual_variance": 15.5,
  "amount_exceeded": 155000,
  "recommended_action": "启动预算调整审批"
}
```
