## 使用示例

### 示例1: 资源有限的创业公司

**用户问题**："我们是做SaaS的创业公司，产品不错但用户增长很慢，应该怎么突破？"

**分析过程**：

**业务诊断**
- 阶段：早期创业
- 瓶颈：用户获取困难
- 资源：有限（100万预算，5人团队）

**破局点识别**
- 推荐：头牌用户杠杆
- 理由：资源有限时，用标杆客户背书最有效

**破局机会**
1. 找到行业标杆客户免费使用
2. 打造可复制的成功案例
3. 利用案例进行传播

**杠杆路径**
- 关键行动：筛选10家潜在标杆客户，提供免费服务
- 资源需求：1人负责客户成功
- 预期效果：获取3-5家标杆，形成传播势能

---

### 示例2: 成熟业务的增长瓶颈

**用户问题**："我们是做电商的，增长遇到瓶颈，有什么新的增长点？"

**分析过程**：

**业务诊断**
- 阶段：成熟期
- 瓶颈：增长乏力
- 资源：中等

**破局点识别**
- 推荐：细分头部杠杆
- 理由：在成熟市场寻找细分机会

**破局机会**
1. 母婴品类细分（辅食、童装）
2. 跨境电商
3. 私域流量运营

**杠杆路径**
- 关键行动：选择跨境电商作为第二增长曲线
- 资源需求：组建3人小团队试水
- 预期效果：6个月GMV达到500万

## 错误处理

| 错误类型 | 典型表现 | 处理方式 |
|---------|---------|---------|
| 破局点模糊 | "我想全面发展" | 强制聚焦，要求选择一个破局点 |
| 资源误判 | "我们资源还可以" | 追问具体数字，客观评估 |
| 复制竞品 | "竞品这么做我们也做" | 分析自身条件是否匹配 |
| 忽视竞争 | "竞争对手不会反应" | 要求分析竞品可能的应对 |
| 短视行为 | "先收割一波再说" | 强调长期价值 |
| 假破局 | "搞个假的成功案例" | 禁止这种做法，要求真实可行 |
| 脱离用户 | "我觉得用户需要" | 要求验证用户需求 |

## 核心概念

### 破局点类型

| 类型 | 定义 | 特征 | 适用场景 |
|------|------|------|----------|
| **头牌用户** | 能带来示范效应的标杆用户 | 有影响力、可传播、代表性强 | 需要建立信任的场景 |
| **细分头部** | 在细分市场中做到头部位置 | 垂直、专注、壁垒高 | 资源有限时的聚焦策略 |
| **稀缺资源** | 获取或控制稀缺资源 | 独占性、壁垒高、可持续 | 有资源获取能力时 |

### 三种杠杆详解

#### 1. 头牌用户杠杆

**定义**：找到并拿下行业内有影响力的标杆用户，让他们成为你的"活招牌"。

**操作步骤**：
1. 识别行业内的意见领袖（KOL/KOC）
2. 分析他们的痛点和需求
3. 提供定制化解决方案
4. 获取可传播的成功案例
5. 利用案例吸引更多用户

**适用条件**：
- 目标用户中有明显的影响力节点
- 产品/服务能真正解决他们的问题
- 有能力提供优质服务

#### 2. 细分头部杠杆

**定义**：在资源有限时，选择一个细分市场做到头部，建立根据地。

**操作步骤**：
1. 市场细分，找到机会最大的垂直领域
2. 集中资源在该细分市场建立优势
3. 建立品牌认知和用户口碑
4. 以点带面，向相邻市场扩展

**适用条件**：
- 资源有限，无法正面竞争
- 细分市场有足够的空间和成长性
- 能够建立差异化优势

#### 3. 稀缺资源杠杆

**定义**：获取或控制某种稀缺资源，形成竞争壁垒。

**操作步骤**：
1. 识别行业内的关键稀缺资源
2. 建立获取渠道或替代方案
3. 通过独占或优先使用权建立优势
4. 围绕稀缺资源构建商业模式

**适用条件**：
- 有资源获取能力或独特渠道
- 稀缺资源能持续保持稀缺性
- 能将稀缺性转化为商业价值

### 破局点选择矩阵

| 资源状况 | 破局点类型 | 杠杆选择 |
|----------|------------|----------|
| 资源充足 | 稀缺资源 | 并购、独家合作、战略投资 |
| 资源中等 | 细分头部 | 聚焦垂直市场、建立壁垒 |
| 资源有限 | 头牌用户 | 找到KOL/KOC、打造标杆案例 |

## Critical Rules

### 必须遵守（≥5条）

1. **先识别再行动** - 必须先明确破局点类型，再选择杠杆
2. **聚焦优先** - 资源有限时，必须集中火力在一个破局点上
3. **验证假设** - 破局点的可行性必须经过验证，不能凭空假设
4. **考虑竞争反应** - 选择杠杆时必须考虑竞争对手的可能反应
5. **评估复制性** - 成功案例必须考虑能否规模化复制
6. **尊重资源约束** - 必须在实际资源约束下选择可行的杠杆
7. **关注长期价值** - 优先选择有长期价值的破局点，而非短期利益

### 禁止行为（≥5条）

1. **禁止分散资源** - 不能同时追求多个破局点
2. **禁止盲目复制** - 不能复制别人的成功破局点而不考虑自身条件
3. **禁止忽视竞争** - 不能假设竞争对手不会反应
4. **禁止透支资源** - 不能为了破局而超出自身承受能力
5. **禁止短视行为** - 不能只顾短期效果而破坏长期价值
6. **禁止虚假破局** - 不能伪造成功案例来"制造"破局点
7. **禁止脱离用户** - 破局点必须真正解决用户问题，而非自嗨

## Workflow Steps

### Step 1: 业务诊断

| 项目 | 内容 |
|------|------|
| **输入** | 用户描述的业务场景、当前瓶颈、资源状况 |
| **处理** | 分析当前业务的主要矛盾和限制条件 |
| **输出** | 业务诊断报告：业务阶段、核心瓶颈、资源状况 |
| **成功标准** | 明确当前业务的主要限制因素 |

### Step 2: 识别破局点类型

| 项目 | 内容 |
|------|------|
| **输入** | 业务诊断报告 |
| **处理** | 根据业务特点和资源状况，识别最合适的破局点类型（头牌用户/细分头部/稀缺资源） |
| **输出** | 破局点类型建议及理由 |
| **成功标准** | 选择一个明确的破局点类型 |

### Step 3: 寻找破局机会

| 项目 | 内容 |
|------|------|
| **输入** | 破局点类型、业务诊断 |
| **处理** | 列出该类型下的具体机会点，筛选最有潜力的1-3个 |
| **输出** | 破局机会清单，每个机会包含：机会描述、可行性评估、潜在风险 |
| **成功标准** | 识别出具体可行的破局机会 |

### Step 4: 设计杠杆路径

| 项目 | 内容 |
|------|------|
| **输入** | 选定的破局机会 |
| **处理** | 设计具体的杠杆实施路径，包括：关键行动、资源需求、时间节点、预期效果 |
| **输出** | 杠杆路径设计书 |
| **成功标准** | 路径清晰、可执行、有时间节点 |

### Step 5: 评估风险与收益

| 项目 | 内容 |
|------|------|
| **输入** | 杠杆路径设计书 |
| **处理** | 评估：成功概率、潜在风险、资源投入、预期收益、竞争反应 |
| **输出** | 风险收益评估报告 |
| **成功标准** | 风险可控、收益大于成本 |

### Step 6: 确定实施方案

| 项目 | 内容 |
|------|------|
| **输入** | 风险收益评估 |
| **处理** | 确定最终实施方案，包括：启动条件、关键里程碑、止损点、负责人 |
| **输出** | 实施方案文档 |
| **成功标准** | 方案具体、可落地、有明确的验证标准 |
