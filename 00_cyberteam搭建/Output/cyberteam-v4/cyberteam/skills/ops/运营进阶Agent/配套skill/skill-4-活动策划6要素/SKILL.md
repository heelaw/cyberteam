## 核心定义

### 什么是活动策划6要素？

策划一个完整活动需要考虑的6个基本要素：

```
┌─────────────────────────────────────────────────────┐
│                  活动策划6要素                       │
├─────────────────────────────────────────────────────┤
│                                                     │
│   1. 活动目标（Goal）                               │
│      - 为什么要做这个活动？                         │
│      - 衡量指标是什么？                             │
│                                                     │
│   2. 目标用户（User）                               │
│      - 活动面向谁？                                 │
│      - 用户的核心需求是什么？                       │
│                                                     │
│   3. 活动内容（Content）                            │
│      - 具体做什么？                                 │
│      - 玩法设计                                     │
│                                                     │
│   4. 激励设计（Incentive）                          │
│      - 用户能得到什么？                             │
│      - 如何让用户愿意参与？                         │
│                                                     │
│   5. 传播机制（Spread）                             │
│      - 如何让更多人知道？                           │
│      - 如何让用户主动传播？                         │
│                                                     │
│   6. 风险评估（Risk）                               │
│      - 可能出现什么问题？                           │
│      - 如何预防和应对？                             │
│                                                     │
└─────────────────────────────────────────────────────┘
```

---

## Critical Rules（必须遵守）

1. **目标必须可衡量** - 不能说"提升品牌影响力"，要说"新增1000付费用户"
2. **用户必须精准** - 不能说"面向所有用户"，要有具体的用户画像
3. **玩法必须有差异化** - 不能照抄竞品，要有微创新
4. **激励必须有吸引力** - 要真正打动用户，不是自嗨
5. **传播必须有设计** - 不能只靠自然流量，要有裂变机制

---

## 禁止行为

1. **禁止无目标策划** - 不知道为什么要做活动就动手
2. **禁止目标贪多** - 一个活动聚焦一个核心目标
3. **禁止用户模糊** - 不清楚用户是谁就设计活动
4. **禁止玩法复杂** - 用户门槛太高，参与率会很低
5. **禁止激励模糊** - 不能说"丰富奖品"，要具体
6. **禁止无风险预案** - 必须提前考虑可能的风险
7. **禁止只看执行** - 活动后必须复盘总结

## Workflow Steps

### 步骤1：明确活动目标（输入→处理→输出）

**输入**：
- 业务背景
- 当前业务痛点

**处理**（目标设定）：
```
SMART原则：
- Specific（具体）：具体要达成的指标
- Measurable（可衡量）：有明确的数字
- Achievable（可实现）：基于现有资源
- Relevant（相关）：与业务目标相关
- Time-bound（有时限）：有明确的截止日期

常见活动目标类型：
| 目标类型 | 具体指标 | 适用场景 |
|----------|---------|---------|
| 拉新 | 新增用户数、新增付费 | 增长阶段 |
| 促活 | DAU增长、活跃率 | 成熟阶段 |
| 转化 | 付费率、GMV | 变现阶段 |
| 留存 | 留存率、复购率 | 流失阶段 |
```

**输出**：
- 活动核心目标（1个）
- 衡量指标（3个以内）
- 目标数值

**成功标准**：目标具体可衡量，符合SMART原则

---

### 步骤2：确定目标用户（输入→处理→输出）

**输入**：
- 活动目标
- 用户画像数据

**处理**（用户圈定）：
```
圈定维度：
1. 基础属性
   - 年龄、性别、职业
   - 地域分布

2. 行为特征
   - 活跃度（高/中/低）
   - 付费情况（付费/非付费）
   - 产品使用深度

3. 需求特征
   - 核心痛点
   - 潜在需求

用户分层策略：
| 用户类型 | 特征 | 活动策略 |
|----------|------|---------|
| 新用户 | 未付费/低活跃 | 首单优惠 |
| 活跃用户 | 高活跃/未付费 | 限时免费 |
| 付费用户 | 已付费 | 专属权益 |
| 沉默用户 | 低活跃有付费 | 召回激励 |
```

**输出**：
- 目标用户画像（3个以内）
- 用户痛点清单
- 用户参与动机

**成功标准**：有清晰的用户画像和需求分析

---

### 步骤3：设计活动内容（输入→处理→输出）

**输入**：
- 活动目标
- 目标用户
- 资源预算

**处理**（内容设计）：
```
常见活动形式：
| 形式 | 特点 | 适用目标 |
|------|------|---------|
| 抽奖 | 简单直接 | 拉新、促活 |
| 优惠券 | 促进转化 | 转化、复购 |
| 任务挑战 | 增强参与感 | 促活、留存 |
| 投票评选 | 社交传播 | 拉新、传播 |
| 限时秒杀 | 紧迫感 | 转化、GMV |
| 拼团 | 裂变传播 | 拉新、转化 |
| 内容征集 | UGC生产 | 促活、内容 |

玩法微创新方法：
1. 组合创新：把两种形式组合（抽奖+优惠券）
2. 门槛创新：改变参与门槛（完成任务才能抽奖）
3. 奖励创新：改变奖励形式（实物+虚拟+荣誉）
4. 场景创新：改变使用场景（618提前购）
```

**输出**：
- 活动主题
- 活动形式
- 核心玩法描述
- 活动流程图

**成功标准**：玩法清晰、门槛合理、有差异化

---

### 步骤4：设计激励机制（输入→处理→输出）

**输入**：
- 活动玩法
- 预算
- 用户需求

**处理**（激励设计）：
```
激励类型：
| 类型 | 示例 | 适用场景 |
|------|------|---------|
| 物质激励 | 现金、优惠券、实物 | 价格敏感用户 |
| 精神激励 | 排名、徽章、荣誉 | 成就感驱动用户 |
| 社交激励 | 分享、邀请、组队 | 社交驱动用户 |
| 功能激励 | 会员特权、功能使用权 | 深度用户 |

激励设计原则：
1. 即时反馈：参与就能得到反馈
2. 阶梯激励：完成更多获得更多
3. 稀缺感：限量、限时、限身份
4. 可感知：用户能感受到价值

激励成本核算：
- 拉新成本：CAC = 预算 / 新增用户
- 转化成本：CPC = 预算 / 转化人数
- 留存成本：CRC = 预算 / 留存用户
```

**输出**：
- 激励清单
- 激励规则
- 成本预算
- ROI预估

**成功标准**：激励有吸引力，成本可接受

---

### 步骤5：设计传播机制（输入→处理→输出）

**输入**：
- 活动内容
- 激励设计
- 传播预算

**处理**（传播设计）：
```
传播路径设计：
1. 主动传播（用户自愿分享）
   - 利益驱动：分享得优惠券
   - 内容驱动：有趣想分享
   - 荣誉驱动：晒成绩得奖励

2. 被动传播（触达更多用户）
   - 站内push/弹窗
   - 站外广告投放
   - 社媒渠道覆盖

裂变机制设计：
| 机制 | 玩法 | 适用场景 |
|------|------|---------|
| 邀请裂变 | 邀请好友得奖励 | 拉新 |
| 拼团裂变 | 多人拼团享优惠 | 转化 |
| 分销裂变 | 成为分销员拿佣金 | GMV |
| 助力裂变 | 好友助力解锁 | 传播 |

传播节奏规划：
- 预热期：活动前1-3天
- 引爆期：活动第一天
- 持续期：活动中间阶段
- 收尾期：活动最后1-2天
```

**输出**：
- 传播渠道清单
- 裂变机制设计
- 传播节奏表
- 传播素材需求

**成功标准**：有清晰的传播路径和裂变机制

---

### 步骤6：风险评估与预案（输入→处理→输出）

**输入**：
- 活动方案
- 历史活动数据

**处理**（风险识别）：
```
常见风险类型：
| 风险类型 | 可能问题 | 预防措施 | 应对方案 |
|----------|---------|---------|---------|
| 技术风险 | 服务器崩溃 | 压测扩容 | 降级方案 |
| 运营风险 | 刷单作弊 | 规则限制 | 风控拦截 |
| 物流风险 | 奖品发放延迟 | 提前备货 | 替代方案 |
| 舆论风险 | 负面评价 | 舆情监控 | 快速响应 |
| 财务风险 | 预算超支 | 实时监控 | 动态调整 |

风险评估矩阵：
| 发生概率 | 影响程度 | 风险等级 |
|----------|---------|---------|
| 高 | 高 | P0（必须预防）|
| 高 | 低 | P1（重点关注）|
| 低 | 高 | P2（需要有预案）|
| 低 | 低 | P3（可接受）|
```

**输出**：
- 风险清单
- 预防措施
- 应对预案
- 监控指标

**成功标准**：识别主要风险，有应对预案

---

## 评估维度

| 维度 | 定义 | 权重 |
|------|------|------|
| 目标明确性 | 目标是否具体可衡量 | 20% |
| 用户精准性 | 用户画像是否清晰 | 15% |
| 玩法差异化 | 是否有创新点 | 20% |
| 激励吸引力 | 激励是否有吸引力 | 20% |
| 传播有效性 | 传播机制是否可行 | 15% |
| 风险可控性 | 风险是否有预案 | 10% |
