# upstream - 汇报框架Agent

## 上游依赖

| 上游 | 角色 | 接口 | 输入内容 |
|------|------|------|----------|
| 运营总监 | 直接上级 | 汇报任务下达 | 汇报主题、对象、要求 |
| 增长总监 | 间接上级 | 战略目标同步 | KPI、里程碑要求 |
| 产品部 | 协作方 | 产品进展同步 | 项目进度、功能上线 |
| 数据团队 | 协作方 | 指标数据提供 | KPI完成情况、关键指标 |
| 营销部 | 协作方 | 营销效果同步 | 渠道数据、活动效果 |

---

## 输入文件标准格式

### 运营总监 → 汇报框架Agent

```yaml
# 汇报任务
task: 周报/项目汇报/述职
subject: 汇报主题
audience:
  - 领导职级
  - 关注重点
  - 偏好风格
deadline: 汇报时间
requirements:
  - 时长要求
  - 结构要求
  - 数据要求
```

### 数据团队 → 汇报框架Agent

```yaml
# 数据支持
metrics:
  - 指标1: 数值 + 同比/环比
  - 指标2: 数值 + 同比/环比
achievements:
  - 成果1: 量化描述
  - 成果2: 量化描述
challenges:
  - 问题1: 影响分析
  - 问题2: 影响分析
```

---

## 决策请求格式

### 汇报框架Agent → 运营总监

```json
{
  "type": "report_review",
  "agent": "汇报框架Agent",
  "task": "周报汇报",
  "report_draft": "已生成汇报稿",
  "summary": "核心结论：项目进度正常，预计提前3天完成",
  "confirmation_needed": ["数据准确性确认", "结论是否需要调整"]
}
```
