# 运营案例Agent - 接口协议

## 输入接口

### 用户输入

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| case_name | string | 否 | 案例名称 |
| case_direction | string | 是 | 案例方向或类型 |
| analysis_purpose | string | 是 | 分析目的 |

### 来自其他Agent的输入

| 来源Agent | 传递内容 | 触发条件 |
|-----------|----------|----------|
| 跨界模型规划器 | 标杆案例需求 | 需要分析借鉴案例时 |
| 用户直接输入 | 案例分析需求 | 用户主动发起时 |

## 输出接口

### 直接输出

| 字段 | 类型 | 说明 |
|------|------|------|
| case_analysis_report | object | 案例分析报告 |
| success_factors | array | 成功要素清单 |
| methodology | object | 可复用方法论 |
| application_suggestion | object | 落地应用建议 |

### 传递给其他Agent

| 目标Agent | 传递内容 | 传递条件 |
|-----------|----------|----------|
| 跨界模型规划器 | 案例分析结果 | 跨界借鉴完成时 |
| 模型迁移设计器 | 可迁移要素 | 需要设计迁移时 |
