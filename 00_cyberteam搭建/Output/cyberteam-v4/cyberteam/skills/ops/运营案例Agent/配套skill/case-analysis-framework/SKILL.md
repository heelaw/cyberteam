# SKILL: case-analysis-framework

> 技能名称：运营案例拆解方法论
> 版本: v1.0
> 难度等级: medium
> 关联Agent: 运营案例Agent

---

## 1. 技能概述

本技能提供标准化的运营案例拆解框架，帮助运营从业者高效分析成功案例，建立个人运营弹药库。

**适用场景**:
- 遇到从未做过的运营任务，需要学习参考
- 定期积累案例，建立个人弹药库
- 分析竞品或行业标杆案例

**核心价值**:
- 10分钟拆解一个案例的核心逻辑
- 结构化封装案例亮点到弹药库
- 形成可复用的运营方法论

## 3. 工作流 Steps

### Step 1: 确定拆解类型

**输入**: 用户提出的案例分析需求

**处理**:
```
判断拆解类型：
├── 单一案例深度拆解
│   └── 适用于：遇到具体问题，找一个案例来学习
└── 批量案例结构化拆解
    └── 适用于：建立弹药库，需要系统积累某一类案例
```

**输出**: 确定拆解类型

**成功标准**: 明确是单一拆解还是批量拆解

---

### Step 2: 单一案例深度拆解（流程A）

**输入**: 待拆解的案例名称或背景

**处理**:

#### 2.1 还原案例成果

```
┌─────────────────────────────────────────────────────────────┐
│ 还原案例成果                                                 │
├─────────────────────────────────────────────────────────────┤
│ 1. 案例基本信息                                              │
│    - 品牌/产品名称                                           │
│    - 运营时间段                                              │
│    - 所属领域/行业                                           │
│                                                             │
│ 2. 核心数据指标                                              │
│    - 曝光量/用户增长/转化率等                               │
│    - 行业对比（如果有）                                      │
│                                                             │
│ 3. 成功程度评估                                              │
│    - 超出预期/符合预期/未达预期                             │
└─────────────────────────────────────────────────────────────┘
```

#### 2.2 归纳业务目的

```
┌─────────────────────────────────────────────────────────────┐
│ 业务目的分类                                                 │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  [A] 流量与传播                                              │
│      - 拉新获客                                             │
│      - 品牌曝光                                             │
│      - 话题造势                                             │
│                                                             │
│  [B] 营销与转化                                              │
│      - 付费转化                                             │
│      - 会员转化                                             │
│      - 订单转化                                             │
│                                                             │
│  [C] 留存与活跃                                              │
│      - 用户留存                                             │
│      - 提升活跃度                                           │
│      - 提升使用时长                                         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 2.3 提炼执行细节

```
┌─────────────────────────────────────────────────────────────┐
│ 执行细节四维度                                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  [1] 流程设计亮点                                            │
│      - 用户路径是否顺畅                                      │
│      - 入口/出口设计是否合理                                 │
│      - 路径是否短且高效                                      │
│                                                             │
│  [2] 互动传播机制                                            │
│      - 分享激励机制是什么                                   │
│      - 社交关系链如何应用                                    │
│      - 传播触点有哪些                                       │
│                                                             │
│  [3] 营销转化策略                                            │
│      - 转化路径如何设计                                      │
│      - 转化钩子是什么                                       │
│      - 转化障碍如何消除                                      │
│                                                             │
│  [4] 留存促活机制                                            │
│      - 如何让用户持续回来                                    │
│      - 有什么签到/任务/激励机制                              │
│      - 社区/内容如何留住用户                                 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 2.4 输出案例分析报告

**输出格式**:
```
## [案例名称] 深度拆解报告

### 一、基本信息
- 品牌/产品：
- 运营时间：
- 所属领域：

### 二、核心成果
- 数据指标：
- 成功程度：

### 三、业务目的
- 目的类型：[A/B/C]
- 核心目标：

### 四、执行亮点
| 维度 | 亮点 | 可复用程度 |
|------|------|-----------|
| 流程设计 |     | 高/中/低   |
| 互动传播 |     | 高/中/低   |
| 营销转化 |     | 高/中/低   |
| 留存促活 |     | 高/中/低   |

### 五、可复用经验
1. ...
2. ...
3. ...

### 六、可借鉴教训
1. ...
```

**成功标准**: 报告完整覆盖四个维度，有具体可复用经验

### Step 3: 批量案例结构化拆解（流程B）

**输入**: 用户选择要批量拆解的案例类型

**处理**:

#### 3.1 选择拆解结构

```
┌─────────────────────────────────────────────────────────────┐
│ 案例类型对应的拆解结构                                       │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  [活动类案例]                                                │
│  ├── 主题：活动名称的吸引力和识别度                         │
│  ├── 形式：活动玩法的机制设计                               │
│  ├── 流程：用户参与路径与体验                               │
│  └── 类型：裂变型/传播型/留存型                              │
│                                                             │
│  [内容类案例]                                                │
│  ├── 标题：能否快速抓住注意力                               │
│  ├── 结构：总分总/先观点后案例等                             │
│  ├── 表现形式：图文/短视频/H5/口播等                        │
│  ├── 开篇结语：留存技巧                                     │
│  └── 素材金句：可积累的素材                                 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**处理**: 对每个案例按选定结构逐项填写

**输出格式**:
```
## [案例类型] 批量拆解记录表

| 案例名称 | 主题亮点 | 形式/玩法 | 流程设计 | 类型/分类 | 封装到弹药库 |
|---------|---------|----------|---------|----------|-------------|
|         |         |          |         |          |             |
```

#### 3.3 封装到弹药库

```
弹药库结构：
├── 活动主题弹药库/
│   ├── 裂变类主题
│   ├── 传播类主题
│   └── 留存类主题
├── 活动形式弹药库/
│   ├── 红包类玩法
│   ├── 任务类玩法
│   └── 游戏类玩法
├── 活动流程弹药库/

    └── 金句素材
```

## 5. Critical Rules

### 5.1 必须遵守

1. **先框架后细节** - 始终先提供或确认拆解框架，再深入分析
2. **围绕业务目的** - 所有分析必须指向明确的业务目的
3. **数据要有来源** - 引用的数据必须说明来源或标注"待核实"
4. **经验要可复用** - 总结的经验必须提炼出可复用的规律
5. **弹药库思维** - 每次拆解都要思考如何封装到弹药库

### 5.2 禁止行为

1. **不编造数据** - 不确定的数据标注"待核实"，不编造
2. **不脱离框架** - 始终按标准框架输出，不用自由发挥
3. **不只给结论** - 每个结论都要有分析过程支撑
4. **不忽视失败点** - 同样要分析案例的不足和教训
5. **不跳过封装** - 每次拆解必须输出弹药库条目

---

## 6. 评估维度

| 维度 | 说明 | 最低要求 |
|------|------|----------|
| 框架完整性 | 是否覆盖标准拆解框架的所有要素 | 100% |
| 目的明确性 | 业务目的是否清晰且合理 | 清晰 |
| 亮点识别度 | 执行亮点是否能被识别和复用 | 3个以上 |
| 经验可复用性 | 总结的经验是否可直接指导实践 | 2条以上 |
| 弹药库价值 | 封装条目是否有长期积累价值 | 1条以上 |

## 8. 使用示例

### 示例1：单一案例深度拆解

**用户输入**:
```
分析一下瑞幸咖啡9.9元策略为什么成功
```

**Agent响应**:
```
[按照Step 2流程A执行]

## 瑞幸咖啡9.9元策略深度拆解报告

### 一、基本信息
- 品牌/产品：瑞幸咖啡
- 运营时间：2023年6月至今
- 所属领域：餐饮/咖啡新零售

### 二、核心成果
- 数据指标：月活用户大幅提升，门店加速扩张
- 成功程度：超出预期，成为行业现象级营销

### 三、业务目的
- 目的类型：[A]流量与传播 + [B]营销与转化
- 核心目标：抢占市场份额，打破星巴克高端定位

### 四、执行亮点
[按四维度详细分析...]

### 五、可复用经验
1. 低价是手段不是目的，LTV > CAC才能持续
2. 要有明确的退出策略，如何从低价回归正常价格
3. 精选SKU参与活动，选择高毛利高周转产品

### 六、可借鉴教训
1. 过度低价可能损害品牌调性
2. 需要算好账，LTV必须覆盖补贴成本
```
