# 群内分享六步流程

## Skill概述

通过六步流程组织高质量的群内分享，从设计到反馈形成完整闭环。本技能确保分享有准备、有预告、有互动、有反馈。

## 错误处理

| 错误类型 | 原因 | 处理方式 |
|---------|------|---------|
| 嘉宾迟到 | 时间管理问题 | 提示：提前10分钟联系，准备暖场 |
| 问题太少 | 话题不吸引人 | 提示：助理准备备用问题 |
| 分享超时 | 没有控制节奏 | 提示：主持人要主动控场 |
| 冷场 | 互动设计不足 | 提示：培养活跃用户带头提问 |

## Critical Rules（必须遵守）

1. **禁止无准备的分享**: 分享前必须与嘉宾充分沟通，确保内容质量
2. **禁止没有预告就分享**: 用户没有预期就不会有参与
3. **禁止分享中无人跟进**: 分享过程中要有人实时跟进答疑
4. **禁止分享后无反馈**: 没有反馈的分享无法改进
5. **禁止分享时间失控**: 预设时间要严格执行，超时会影响用户体验
6. **禁止分享频率过高**: 过多分享会让用户疲劳，过少会让用户遗忘

## Workflow Steps（标准流程）

### Step 1: 设计流程

**输入**: 社群定位、成员需求
**处理**: 设计分享的整体流程和规则
**输出**: 完整的分享流程方案
**成功标准**: 流程清晰、时间可控、参与度高

**流程设计要点**:
```
【时间设计】
- 分享时长：建议30-60分钟
- 互动时间：分享后留10-15分钟
- 缓冲时间：每个环节预留5分钟

【形式设计】
- 纯分享型：嘉宾分享为主
- 互动型：分享+讨论
- 问答型：以Q&A为主

【分工设计】
- 主讲嘉宾：负责内容分享
- 主持群主：负责流程把控
- 助理：负责答疑、整理
```

**标准流程模板**:
```
【分享流程】

19:55 - 19:58  群主暖场，介绍嘉宾
19:58 - 20:00  嘉宾入场准备
20:00 - 20:40  嘉宾正式分享（约40分钟）
20:40 - 20:50  互动答疑时间
20:50 - 21:00  总结、预告下次

【规则提醒】
- 分享中禁言，答疑时开放
- 文字优先，语音需转文字
- 违规发言助理清理
```

### Step 2: 确定人选

**输入**: 分享主题
**处理**: 选择合适的分享嘉宾
**输出**: 确定分享嘉宾和分享主题
**成功标准**: 嘉宾有经验、内容有干货、用户有兴趣

**嘉宾选择标准**:
```
【优先选择】
✓ 有实操经验的从业者
✓ 近期有成功案例的人
✓ 能讲真话、有独特见解的人
✓ 有良好表达能力的人

【慎重选择】
△ 纯理论研究者（可能太枯燥）
△ 过度营销倾向的人
△ 表达能力欠佳的人
```

**嘉宾沟通要点**:
```
【沟通内容】
1. 分享主题和大纲
2. 分享时间和时长
3. 分享形式（语音/文字/PPT）
4. 预期互动内容
5. 注意事项（禁言规则等）

【提前确认】
1. 嘉宾资料和照片
2. 分享PPT或大纲
3. 互动问题准备
```

### Step 3: 发布预告

**输入**: 分享主题和嘉宾信息
**处理**: 撰写并发布预告
**输出**: 吸引人的分享预告
**成功标准**: 预告发出后有足够报名/关注

**预告模板**:
```
═══════════════════════════════════════════
🎙️ [社群名称] 第[X]期分享

【分享主题】
[吸引人的主题]

【分享嘉宾】
[嘉宾昵称]
[嘉宾身份/Title]
[一句话个人简介]

【分享大纲】
1. [要点1]
2. [要点2]
3. [要点3]

【分享时间】
[日期] [时间] - [时间]
预计时长：[X]分钟

【报名方式】
留言"参加"或"感兴趣"

【预告福利】
提前提问有机会被回答哦～

═══════════════════════════════════════════
```

**预告发布节奏**:
```
【提前3-5天】发布正式预告
【提前1天】再次提醒
【提前1小时】最后一次提醒
【分享前30分钟】暖场准备
```

### Step 4: 异常处理

**输入**: 可能出现的异常情况
**处理**: 准备异常处理方案
**输出**: 异常处理预案
**成功标准**: 出现问题能快速应对

**常见异常及处理**:

| 异常情况 | 处理方式 |
|---------|---------|
| 嘉宾迟到 | 提前10分钟联系，准备备用话题 |
| 嘉宾临时有事 | 延期或换嘉宾，提前通知 |
| 网络问题 | 切换文字/语音，耐心等待 |
| 有人捣乱 | 助理立刻处理，严重的移出 |
| 问题太少 | 助理准备几个备用问题 |
| 问题太水 | 主持人筛选有价值问题 |
| 冷场 | 助理引导、活跃用户带动 |

**应急话术**:
```
【嘉宾迟到】
各位稍等，嘉宾正在准备中，预计X分钟后开始～

【嘉宾临时有事】
非常抱歉，今天的分享因[原因]延期
具体时间另行通知，敬请谅解～

【问题太少】
大家有问题吗？我先列几个常见问题：
Q1: ...
Q2: ...
```

### Step 5: 组织分享

**输入**: 准备好的分享内容
**处理**: 按流程执行分享
**输出**: 高质量的分享完成
**成功标准**: 分享顺畅、互动良好、时间把控到位

**分享执行流程**:

```
【开始前30分钟】
- 助理到位准备
- 发送预告提醒
- 确认嘉宾在线

【开始前10分钟】
- 群主/主持人暖场
- 介绍嘉宾
- 预告分享流程

【分享进行中】
- 嘉宾按计划分享
- 助理实时跟进
- 用户禁言或文字跟发

【答疑环节】
- 开放互动
- 主持人梳理问题
- 嘉宾逐一回答

【结束环节】
- 感谢嘉宾
- 预告下次
- 收集反馈
```

**分享话术**:

```
【暖场】
各位好～ 欢迎来到[社群名称]分享！

今天我们邀请到[嘉宾]分享[主题]
在开始之前，我先介绍一下流程：

[流程说明]

【开始】
好，我们正式开始～
@嘉宾 请开始你的分享

【转场】
感谢[嘉宾]的分享，讲得非常精彩！

现在进入答疑环节，大家有什么问题可以提问～
助理@[助理昵称] 会整理问题

【总结】
今天的分享到此结束！
感谢@嘉宾 的精彩分享，干货满满～

预告一下：[下次分享预告]

大家晚安！
```

### Step 6: 反馈总结

**输入**: 分享过程中的数据和反馈
**处理**: 收集反馈、总结复盘
**输出**: 分享总结文档和下次改进点
**成功标准**: 有总结、有改进、参与者满意

**反馈收集**:

```
【分享后24小时内】
1. 感谢嘉宾，发送总结确认
2. 收集参与者反馈
3. 整理分享内容文字版

【反馈收集话术】
大家觉得今天的分享怎么样？
欢迎留言反馈～ 你的意见很重要！

【总结模板】
═══════════════════════════════════════════

【分享主题】
[主题]

2. [要点2]
3. [要点3]

【精彩观点】
"[嘉宾观点1]"
"[嘉宾观点2]"

【互动问题TOP3】
Q1: [问题] - [回答摘要]
Q2: [问题] - [回答摘要]
Q3: [问题] - [回答摘要]

【参与者反馈】
- [反馈1]
- [反馈2]

【下期预告】
[预告内容]

═══════════════════════════════════════════
```

## 注意事项

1. **提前沟通**: 与嘉宾充分沟通是成功分享的基础
2. **节奏把控**: 严格按照预设时间执行
3. **互动设计**: 分享不是单向输出，要设计互动环节
4. **氛围维护**: 分享中要有人实时维护氛围
5. **内容存档**: 分享内容要整理存档，供后人学习
