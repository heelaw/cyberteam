# 用户进群引导三步法

## Skill概述

通过"告知-说明-确认"三步流程，科学引导新用户加入社群，降低流失率，提升转化率。本技能确保新用户在入群前充分了解社群定位和规则。

## 输入格式

```json
{
  "user_info": {
    "nickname": "string",
    "source": "string",
    "background": "string"
  },
  "community_info": {
    "name": "string",
    "type": "付费|免费|高端",
    "positioning": "string",
    "core_value": "string[]"
  },
  "rules": {
    "encouraged": "string[]",
    "forbidden": "string[]"
  }
}
```

## 输出格式

```json
{
  "guidance_process": {
    "step1_inform": {
      "message": "string",
      "key_points": "string[]"
    },
    "step2_explain": {
      "message": "string",
      "rules_summary": "string"
    },
    "step3_confirm": {
      "message": "string",
      "confirmation_required": true
    }
  },
  "welcome_message": "string",
  "followup_plan": "string"
}
```

## 错误处理

| 错误类型 | 原因 | 处理方式 |
|---------|------|---------|
| 用户不愿确认 | 觉得流程太繁琐 | 提示：可简化流程，但必须有确认环节 |
| 用户被拉群后退出 | 不了解社群定位 | 提示：加强Step 1的告知环节 |
| 用户违规后狡辩 | 不知道群规 | 提示：保存确认记录，有据可查 |
| 引导话术太长 | 信息过载 | 提示：分步发送，避免一次性发送 |

## 核心方法论

用户进群引导是社群运营的关键环节，通过"告知-说明-确认"三步法，帮助新用户快速了解社群并融入其中：

```
┌─────────────────────────────────────────┐
│         用户进群引导三步法                │
├─────────────────────────────────────────┤
│  Step 1: 告知 → 告知群定位（这是什么样的群） │
├─────────────────────────────────────────┤
│  Step 2: 说明 → 说明规则（能做什么/不能做什么）│
├─────────────────────────────────────────┤
│  Step 3: 确认 → 获得确认后再拉群           │
└─────────────────────────────────────────┘
```

## Critical Rules（必须遵守）

1. **禁止无引导直接拉群**: 用户不知情被拉入群，会产生排斥感和迷失感
2. **禁止一次性发送所有信息**: 信息过载会导致用户忽略重要内容
3. **禁止缺少用户确认环节**: 没有确认的引导，用户不会真正接受规则
4. **禁止使用模糊话术**: 如"大家都很随意"等，让用户不知道边界在哪里
5. **禁止入群后长时间沉默**: 新用户进群后没有欢迎，会感到被忽视
6. **禁止强制要求自我介绍**: 有些用户不愿公开身份，应提供替代方案

## Workflow Steps（标准流程）

### Step 1: 告知群定位

**输入**: 用户申请入群
**处理**: 在拉用户进群前，先发送群定位介绍
**输出**: 让用户知道这是一个什么样的社群
**成功标准**: 用户能清晰了解社群的核心价值和能获得的帮助

**标准话术模板**:
```
[用户昵称]，欢迎申请加入【XXX社群】！

先给你介绍一下我们社群：
🌟 社群定位：帮助[目标用户]实现[核心价值]
📚 社群内容：每周[具体内容]、每月[具体活动]
👥 成员构成：已有[人数]位[目标用户]加入
⏰ 交流方式：日常在群里[交流形式]

如果你觉得适合你，我再拉你入群~
```

### Step 2: 说明规则

**输入**: 用户了解了群定位
**处理**: 告知社群的支持行为和禁止行为
**输出**: 用户清楚知道在群里能做什么、不能做什么
**成功标准**: 用户明确社群边界，不会因为不了解规则而踩雷

**标准话术模板**:
```
在入群前，先说一下我们的群规：

✅ 欢迎行为：
- 欢迎提问，我会尽力解答
- 鼓励分享你的经验和案例
- 鼓励参与每周[具体活动]

❌ 禁止行为：
- 禁止发广告/拉人/私加好友
- 禁止发无关链接
- 禁止讨论敏感话题

基本上就是一个[轻松/专业/互助]的学习交流群~
```

### Step 3: 获得确认

**输入**: 用户已知晓群定位和规则
**处理**: 明确询问用户是否接受，确认后再拉群
**输出**: 用户明确表示接受，此时拉群
**成功标准**: 用户明确回复"好的"或"可以"，再执行拉群操作

**标准话术模板**:
```
以上就是我们社群的基本情况~

确认一下：
1. 你是否认可我们的社群定位？
2. 你是否愿意遵守我们的群规？

如果没问题，我就拉你入群了哦~
（回复"确认"即可）
```

### Step 4: 执行拉群

**输入**: 用户确认接受
**处理**: 执行拉群操作
**输出**: 用户成功加入社群
**成功标准**: 用户顺利进入社群，无技术问题

### Step 5: 入群后欢迎

**输入**: 用户已入群
**处理**: 发送欢迎消息，介绍老成员，帮助新用户融入
**输出**: 新用户感受到被欢迎，有归属感
**成功标准**: 新用户在群内有互动意愿

**标准话术模板**:
```
🎉 欢迎[用户昵称]加入【XXX社群】！

我是群主[群主昵称]，有问题可以随时@我

给你介绍一下：
- 我们的老成员[简介]，有什么问题可以找他
- 社群的日常安排是[简介]
- 建议你可以先看看群公告，了解[具体内容]

有任何问题随时问，群里见~
```

### Step 6: 跟进观察

**输入**: 新用户入群后24-48小时
**处理**: 观察新用户是否有互动，给予必要的引导
**输出**: 新用户开始融入社群
**成功标准**: 新用户有至少一次互动（提问/回答/参与活动）

## 话术模板汇总

### 场景A：付费社群引导

```
[用户昵称]，感谢你的支持！🎉

在拉你入群前，先了解一下：

【社群简介】
这是一个面向[目标用户]的[付费社群名称]
已帮助[人数]位成员实现[成果]

【社群服务】
- 每周[服务内容]
- 每月[服务内容]
- 日常答疑[服务形式]

【群规要点】
✅ 欢迎：[行为]
❌ 禁止：[行为]

确认以上内容无误，请回复"已了解"，我即刻拉你入群~
```

### 场景B：免费社群引导

```
[用户昵称]，欢迎申请加入！🙌

先简单了解一下我们：

【社群简介】
帮助[目标用户]实现[价值]的学习型社群

【入群条件】
需要你完成以下任一任务：
1. 分享你最近在学习的[领域]内容
2. 回答新人的入群问题：[具体问题]

完成后回复"已完成"，我拉你入群~
```

### 场景C：高端社群引导

```
[用户昵称]，欢迎！

我们社群是[描述社群高端定位]

【入群审核】
需要你提供以下信息：
1. 你的职业背景/公司职位
2. 你希望通过社群获得什么
3. 你能可以为社群贡献什么

信息审核通过后，我会在24小时内拉你入群~
```

## 适用场景

| 场景 | 侧重点 | 特殊要求 |
|------|--------|---------|
| 付费社群 | 强调价值、明确规则 | 确认付费信息 |
| 免费社群 | 筛选用户、降低流失 | 任务式引导 |
| 高端社群 | 审核严格、身份认同 | 背景审核 |
| 地域社群 | 线下活动组织 | 地域确认 |

## 注意事项

1. **节奏把控**: 不要急于拉群，让用户有消化信息的时间
2. **个性化调整**: 根据社群调性调整话术风格
3. **留出选择权**: 用户有权选择不入群，不要强拉
4. **记录存档**: 保存用户的确认记录，有争议时有据可查
