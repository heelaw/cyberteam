# 群规执行策略

## Skill概述

通过"合适时机告知+严格执法"策略，确保群规有效执行，维护社群秩序。本技能帮助社群运营者处理各种违规情况，保持社群健康发展。

## 错误处理

| 错误类型 | 原因 | 处理方式 |
|---------|------|---------|
| 执法犹豫 | 怕得罪人 | 提示：犹豫会让更多人效仿违规 |
| 双重标准 | 是熟人就放松 | 提示：执法必须一视同仁 |
| 公开羞辱 | 情绪上头 | 提示：维护用户尊严，私下处理 |
| 情绪化执法 | 被挑衅 | 提示：保持冷静，按规则处理 |

## 核心方法论

群规执行是社群运营的关键环节，通过"合适时机+严格执法"确保群规有效性：

```
┌─────────────────────────────────────────┐
│         群规执行两大原则                  │
├─────────────────────────────────────────┤
│  原则1: 合适时机告知                      │
│  → 在用户需要知道的时候告知，不是灌输了事  │
├─────────────────────────────────────────┤
│  原则2: 严格执法                         │
│  → 警告→移出，按流程执行，不打折扣        │
└─────────────────────────────────────────┘
```

## Critical Rules（必须遵守）

1. **禁止执法双重标准**: 对所有人一视同仁，不能因亲疏远近而放松执法
2. **禁止执法犹豫不决**: 发现违规要立即处理，不能拖延或视而不见
3. **禁止公开羞辱违规者**: 执法要维护用户尊严，不要长篇大论批评
4. **禁止情绪化执法**: 即使被挑衅也要保持冷静，按规则处理
5. **禁止只执法不告知**: 处理后要让当事人知道原因和改进方式
6. **禁止执法后不做记录**: 违规记录是后续处理的依据

## Workflow Steps（标准流程）

### Step 1: 建立违规判定标准

**输入**: 群规文档
**处理**: 明确区分轻度违规、重度违规、严重违规
**输出**: 违规分级标准和对应处理方式
**成功标准**: 每条规则都能对应到具体违规级别

**违规分级参考**:
```
【轻度违规】→ 提醒/警告
- 首次发无关链接
- 偶尔刷屏
- 不当言论但非恶意

【重度违规】→ 警告+惩罚
- 重复轻度违规
- 发广告/推广
- 私加好友骚扰

【严重违规】→ 直接移出
- 诈骗/虚假信息
- 人身攻击/网暴
- 泄露社群隐私
```

### Step 2: 设计告知时机

**输入**: 违规判定标准
**处理**: 确定在不同场景下的告知策略
**输出**: 告知时机方案
**成功标准**: 用户在合适的时机了解群规

**告知时机策略**:
```
场景1: 新用户入群
→ 入群前：配合用户进群引导三步法告知
→ 入群后：欢迎语+群规摘要

场景2: 日常运营
→ 每周定期：群规小提醒
→ 有人违规：处理时再次强调规则

场景3: 特殊时期
→ 大型活动前：活动专属规则
→ 问题高发期：主动预防提醒
```

### Step 3: 执行轻度违规处理

**输入**: 发现轻度违规行为
**处理**: 私下提醒或群内轻提示
**输出**: 用户意识到问题并改正
**成功标准**: 用户停止违规行为

**处理话术**:
```
私下提醒：
[用户昵称]，不好意思，群规禁止发投票/问卷哦
下次注意～谢谢配合

群内轻提示：
@一下 建议大家发投票/问卷之前先问问群主～

处理后：
用户已撤回/删除相关内容
如果用户配合，可以简单说一句"谢谢配合"
```

### Step 4: 执行重度违规处理

**输入**: 发现重度违规行为或重复违规
**处理**: 正式警告+说明后果
**输出**: 用户认识错误并承诺改正
**成功标准**: 用户明确表示改正意愿

**处理话术**:
```
@用户 提醒：群规禁止发广告，违规会直接移出
这是第一次警告，下次违规将直接移出社群
请删除相关内容，谢谢配合～

处理后记录：
[日期] [用户昵称] 第一次警告 发广告
```

### Step 5: 执行严重违规处理

**输入**: 发现严重违规行为
**处理**: 直接移出，无需多言
**输出**: 维护社群安全和秩序
**成功标准**: 快速处理，不影响社群氛围

**处理话术**:
```
直接移出，无需提前通知
移出后私下说明（如有必要）：
[用户昵称]，抱歉，因违反群规（[具体原因]）
你已被移出社群，如有疑问可以联系我～
[具体原因说明]

公开简要说明：
@所有人 违规内容已被处理，感谢理解～
```

### Step 6: 违规记录与复盘

**输入**: 所有违规处理记录
**处理**: 定期复盘，优化群规和执行策略
**输出**: 执行策略持续改进
**成功标准**: 违规率下降，社群秩序改善

**记录表单**:

```

请大家共同维护社群秩序～

（如有必要补充）
提醒：群规禁止[行为]，违规将直接移出
```

## 特殊场景处理

### 场景1：用户狡辩

**用户反应**: "我不知道这个不能做"

**处理策略**: 
- 私下说明：群规在入群前已告知，可提供记录
- 不要争论，直接执行规则
- 态度坚定但友好

### 场景2：用户求情

**用户反应**: "能不能给一次机会"

**处理策略**:
- 明确说明规则面前没有例外
- 如果是首次轻度违规，可以给一次机会
- 重度/严重违规不给机会

### 场景3：群友起哄

**用户反应**: 群友在违规者被处理时起哄

**处理策略**:
- 不理会，继续正常运营
- 必要时@所有人转移话题
- 避免二次冲突

### 场景4：管理员违规

**用户反应**: 发现管理员也违规了

**处理策略**:
- 管理员更要以身作则
- 管理员违规应更严格处理
- 不能有特权思维

## 注意事项

1. **快速响应**: 发现违规要立即处理，拖延会让问题扩大
2. **保留证据**: 处理前截图保存证据
3. **统一标准**: 所有管理员执行标准要一致
4. **安全第一**: 发现诈骗等违法行为要立即处理
5. **持续优化**: 根据实际情况调整处理策略
