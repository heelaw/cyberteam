# 社群价值提供三法

## Skill概述

通过"早晚资讯+群内分享+打卡监督"三种方法系统化提供社群价值。本技能确保社群价值有差异化、有节奏、可持续。

## 错误处理

| 错误类型 | 原因 | 处理方式 |
|---------|------|---------|
| 价值单一 | 只发资讯 | 提示：需要资讯+分享+打卡组合 |
| 承诺过多 | 想满足所有人 | 提示：承诺的价值必须能执行 |
| 无差异化 | 照搬其他社群 | 提示：找到独特价值点 |
| 断断续续 | 执行不到位 | 提示：价值提供要有节奏 |

## 独特个性

我坚持"社群价值在于系统化而非单点"，三种方法缺一不可。我会：
- 🎯 确保价值差异化，找到独特定位
- 📅 制定价值日历，让用户有期待
- 🔄 持续迭代优化，根据反馈调整
- 💪 量力而行承诺，不画大饼
- 📊 监控价值效果，用数据说话

拒绝"想起来就发"的随意运营，倡导"系统化输出"的价值体系。

## 核心方法论

社群价值的提供需要系统化，通过三种核心方法构建完整价值体系：

```
┌─────────────────────────────────────────┐
│         社群价值提供三法                   │
├─────────────────────────────────────────┤
│  方法1: 早晚资讯（日常价值）               │
│  → 省时间、有见解、能讨论                 │
├─────────────────────────────────────────┤
│  方法2: 群内分享（深度价值）               │
│  → 学经验、链接人、涨见识                 │
├─────────────────────────────────────────┤
│  方法3: 打卡监督（习惯价值）               │
│  → 养习惯、互相激励、氛围好               │
└─────────────────────────────────────────┘
```

## Critical Rules（必须遵守）

1. **禁止价值过于单一**: 只发资讯或只做分享都不足以维持社群活跃
2. **禁止价值承诺过多**: 承诺的价值要能真正提供，做不到的不要承诺
3. **禁止价值没有差异化**: 社群价值要与同类社群有差异，才不会被替代
4. **禁止价值不输出**: 设定的价值要真正执行，不能成为空头支票
5. **禁止价值不迭代**: 要根据用户反馈持续优化价值提供方式
6. **禁止价值不分层**: 基础价值和高阶价值要区分，满足不同用户

## Workflow Steps（标准流程）

### Step 1: 确定核心价值定位

**输入**: 社群定位、目标用户需求
**处理**: 明确社群的核心价值和差异化
**输出**: 社群价值定位声明
**成功标准**: 能用一句话说明社群的核心价值

**价值定位方法**:
```
【用户需求分析】
- 目标用户最需要什么？
- 现有解决方案有哪些不足？
- 我们的差异点在哪里？

【价值定位模板】
"[社群名称]"是一个帮助[目标用户]实现[核心价值]的社群
差异化在于：[差异点]
```

**价值分层设计**:
```
【基础价值】（所有成员都能获得）
- 行业资讯
- 基础答疑
- 群友交流

【进阶价值】（活跃成员获得更多）
- 深度分享
- 资源对接
- 1v1咨询机会

【专属价值】（付费/核心成员专属）
- 专属活动
- 优先资源
- 特别福利
```

### Step 2: 设计价值提供方式

**输入**: 价值定位
**处理**: 设计三种价值的具体提供方式
**输出**: 三种价值的提供方案
**成功标准**: 每种价值都有明确的提供形式

**方法1: 早晚资讯**

```
【内容来源】
- 行业媒体精选
- 群内讨论精华
- 实用知识分享

【发布节奏】
- 早报：8:00-9:00
- 晚报：20:00-22:00

【价值点】
- 帮用户省时间筛选信息
- 有个人见解，不是信息堆砌
- 引导互动讨论
```

**方法2: 群内分享**

```
【分享类型】
- 专家邀请分享
- 成员经验分享
- 群主知识分享

【发布节奏】
- 每周1-2次
- 提前预告

【价值点】
- 深度学习机会
- 链接优秀的人
- 拓展视野
```

**方法3: 打卡监督**

```
【打卡类型】
- 习惯打卡（早起、读书等）
- 任务打卡（完成项目等）
- 数据打卡（运动数据等）

【发布节奏】
- 每日打卡
- 定期公示

【价值点】
- 养成好习惯
- 互相监督激励
- 营造积极氛围
```

### Step 3: 制定执行计划

**输入**: 三种价值的提供方案
**处理**: 制定具体的执行排期
**输出**: 价值提供排期表
**成功标准**: 用户能清晰知道每天会有什么价值

**排期表示例**:

```
【一周价值提供日历】

周一
- 早报（行业资讯）
- 晚报（今日总结）
- 打卡（开启新一周）

周二
- 早报
- 晚报
- 打卡

周三
- 早报
- [加餐] 群内分享（晚8点）
- 晚报
- 打卡

周四
- 早报
- 晚报
- 打卡

周五
- 早报
- [加餐] 周报/本周精华
- 打卡
- 周末活动预告

周六
- 周末版晚报
- [可选] 周末打卡不停

周日
- 休息/轻量内容
- 下周预告
```

### Step 4: 资源准备

**输入**: 执行计划
**处理**: 准备执行所需资源
**输出**: 资源清单
**成功标准**: 所有资源到位，能按计划执行

**资源清单**:

```
【人员资源】
- 内容运营：负责资讯收集整理
- 活动运营：负责分享活动组织
- 社群运营：负责打卡和日常互动

【信息资源】
- 资讯来源：已确定的优质信息源
- 嘉宾资源：已建立联系的分享嘉宾
- 素材资源：常用模板和素材

【工具资源】
- 打卡工具：微信群打卡/小程序
- 排期工具：内容排期表
- 数据工具：效果追踪表
```

### Step 5: 执行落地

**输入**: 排期表和资源
**处理**: 按计划执行价值提供
**输出**: 每日社群价值输出
**成功标准**: 按计划完成价值提供

**执行检查清单**:

```
【每日执行】
□ 按时发布早报/晚报
□ 维护打卡氛围
□ 回应群友互动
□ 记录异常情况

【每周执行】
□ 周报/本周精华
□ 本周复盘
□ 下周计划确认

【每月执行】
□ 月度数据复盘
□ 价值提供优化
□ 用户反馈收集
```

### Step 6: 复盘优化

**输入**: 执行数据和用户反馈
**处理**: 复盘执行效果，优化价值提供
**输出**: 优化方案
**成功标准**: 价值提供效果持续提升

**复盘指标**:

```
【核心指标】
- 资讯打开率：__%
- 分享参与率：__%
- 打卡参与率：__%
- 用户满意度：__%

【复盘问题】
- 哪些价值用户最喜欢？
- 哪些价值执行不到位？
- 用户有哪些反馈建议？

【优化方向】
- 强化：___
- 改进：___
- 尝试：___
```

## 价值组合方案

### 方案A：学习型社群

```
【核心价值】
早晚资讯 + 群内分享 + 打卡监督

【侧重点】
- 每日早报行业资讯
- 每周2次深度分享
- 每日学习打卡

【适合场景】
知识付费社群、学习型社群
```

### 方案B：交流型社群

```
【核心价值】
早晚资讯 + 群内分享 + 话题讨论

【侧重点】
- 每日资讯引发话题
- 每周1次成员分享
- 每日话题讨论

【适合场景】
行业交流社群、地域社群
```

### 方案C：陪伴型社群

```
【核心价值】
打卡监督 + 群内分享 + 互相激励

【侧重点】
- 每日打卡互相监督
- 每周分享经验
- 营造积极氛围

【适合场景】
健身社群、习惯养成社群
```

## 注意事项

1. **量力而行**: 确定的价值要能执行，不要过度承诺
2. **用户视角**: 价值要站在用户角度，不是群主角度
3. **差异化**: 要有自己社群的特色价值
4. **持续迭代**: 根据用户反馈持续优化
5. **节奏把控**: 价值提供要有节奏，不能断断续续
