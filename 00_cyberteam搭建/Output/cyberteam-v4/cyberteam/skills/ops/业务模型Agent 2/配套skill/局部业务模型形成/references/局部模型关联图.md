# 局部模型关联图

## 概述

局部模型不是孤立的，它们之间存在各种关联关系。理解这些关联是避免"局部最优≠整体最优"的关键。

---

## 局部模型之间的关联类型

### 1. 串联关系

一个局部模型的输出是另一个局部模型的输入

```
局部模型A → 局部模型B → 局部模型C
  (获客)    →   (转化)   →   (留存)
```

**特点**：
- 上下游依赖
- 前端影响后端
- 需要标准化的接口（用户数据）

**案例**：
```
获客模型（新用户数据）→ 激活模型（新用户激活）→ 留存模型（用户留存）
```

---

### 2. 并联关系

多个局部模型服务同一目标

```
              → 局部模型A ─→
整体目标 ───→               ──→ 共同产出
              → 局部模型B ─→
```

**特点**：
- 多路径达成目标
- 需要竞合分析
- 资源分配优化

**案例**：
```
增长目标 ──→ 付费投放模型 ──→ 新用户
         ──→ 裂变模型 ──→ 新用户
         ──→ 内容获客模型 ──→ 新用户
```

---

### 3. 反馈关系

下游局部模型的效果反向影响上游

```
局部模型A → 局部模型B → 局部模型C
   ↑                        │
   └────────────────────────┘
        反馈信号
```

**特点**：
- 长期影响短期
- 口碑效应
- 需要全局优化

**案例**：
```
获客 → 转化 → 留存 → 变现 → 用户满意度 → 推荐 → 获客
```

---

## 局部关联图绘制方法

### Step 1：识别所有局部模型

```
1. 列出所有局部模型
2. 确定每个模型的核心输入和输出
```

### Step 2：确定关联类型

```
对每对局部模型判断：
- 是串联关系吗？
- 是并联关系吗？
- 是反馈关系吗？
- 还是独立关系？
```

### Step 3：绘制关联图

```
使用以下符号：

→ 串联关系（输入→输出）
⇄ 并联关系（共同服务）
↺ 反馈关系（下游→上游）
✕ 冲突关系（互相削弱）
```

## 局部联动设计原则

### 原则1：接口标准化

```
为局部模型之间的"数据传递"建立标准：

新用户数据标准：
{
  "user_id": "唯一标识",
  "source": "来源渠道",
  "quality_score": "质量评分",
  "first_action_time": "首次行为时间",
  "..."
}

标准化好处：
- 数据传递无损耗
- 便于追踪和归因
- 支持自动化
```

### 原则2：联动指标设计

```
在局部模型KPI中加入联动指标：

获客团队KPI：
- 新用户数量（70%）
- 新用户7日留存率（30%）← 联动指标

转化团队KPI：
- 首单转化率（50%）
- 新用户质量评分（30%）← 联动指标
- 获客团队满意度（20%）← 联动指标
```

### 原则3：定期联合会诊

```
月度局部模型联合会诊：

议题1：各局部模型的输出是否满足下游输入需求？
议题2：资源如何在局部模型间分配最优？
议题3：有没有局部模型成为整体瓶颈？
议题4：局部模型目标是否需要调整？
```

## 联动问题诊断

### 问题类型1：上游灌水

```
表现：
- 获客量很大，但激活率很低
- 进来的用户质量差

原因：
- 获客团队只考核数量，不考核质量
- 数据传递不完整

解决方案：
- 增加获客质量指标
- 建立用户质量评分体系
- 上游质量纳入上游考核
```

### 问题类型2：下游阻塞

```
表现：
- 各渠道获客正常，但整体增长慢
- 留存数据异常

原因：
- 某个环节成为瓶颈
- 上下游不匹配

解决方案：
- 绘制全链路关联图
- 找到阻塞点
- 优先解决瓶颈
```

### 问题类型3：各自为战

```
表现：
- 各局部模型都完成了自己的指标
- 但整体目标没有达成

原因：
- 局部目标与整体目标脱节
- 缺乏联动机制

解决方案：
- 重审局部目标与整体目标的对齐
- 建立联动KPI
- 定期会诊机制
```

## 关联图绘制练习

### 练习题目

绘制一个在线教育平台的局部模型关联图，包含：获客、试听体验、付费转化、课程学习、续费、传播六个局部模型。

### 关联图要求

```
1. 识别串联关系
2. 识别反馈关系
3. 标注关键指标
4. 识别潜在的联动问题
```
