# 目标设定标准表

## SMART目标原则

### 标准定义

| 字母 | 含义 | 说明 | 验证问题 |
|------|------|------|---------|
| **S** | Specific (具体) | 目标清晰明确 | "具体要达成什么？" |
| **M** | Measurable (可衡量) | 有明确的衡量标准 | "怎么判断完成了？" |
| **A** | Achievable (可达成) | 目标可实现 | "努力能达到吗？" |
| **R** | Relevant (相关) | 与业务目标相关 | "为什么重要？" |
| **T** | Time-bound (有时限) | 有明确的截止日期 | "什么时候完成？" |

## 目标值设定方法

### 历史数据法

适用于: 业务稳定、有历史数据积累

```markdown
## 历史数据法

基准值 = 历史最佳值
增长惯性 = 近3期平均增长率

目标值 = 基准值 × (1 + 增长惯性 × 调整系数)

调整系数:
- 业务上升期: 1.0-1.2
- 业务平稳期: 0.8-1.0
- 业务调整期: 0.5-0.8
```

### 行业对标法

适用于: 有可比的行业数据

```markdown
## 行业对标法

行业领先值 = 行业标杆企业指标
追赶系数 = (当前值 / 行业领先值)

目标值 = 当前值 + (行业领先值 - 当前值) × 追赶努力系数

追赶努力系数:
- 快速追赶: 0.5 (一半差距)
- 稳健追赶: 0.3 (三成差距)
- 保守追赶: 0.2 (两成差距)
```

### 资源投入法

适用于: 新业务或重大投入项目

```markdown
## 资源投入法

投入资源 = 人力 + 资金 + 时间
资源转化率 = 历史资源投入产出比

预期产出 = 投入资源 × 资源转化率 × 效率系数

效率系数:
- 高效团队: 1.2
- 标准团队: 1.0
- 新团队: 0.8
```

### 综合设定法

适用于: 重要目标，多维度考量

```markdown
## 综合设定法

目标值 = min(历史法, 行业法, 投入法) × 信心系数 + max(...) × (1-信心系数)

信心系数: 根据团队能力和资源投入确定的信心度 (0-1)
- 信心满满: 0.8
- 较有信心: 0.6
- 一般信心: 0.4
- 信心不足: 0.2
```

## 目标值评估标准

### 目标合理性评估

| 评估维度 | 评估问题 | 评分(1-5) |
|---------|---------|-----------|
| 挑战性 | 目标是否需要努力才能达到？ | |
| 可达成性 | 在现有资源下能实现吗？ | |
| 可衡量性 | 能否用明确的数据衡量？ | |
| 相关性 | 与业务大目标是否一致？ | |
| 时限性 | 是否有明确的截止日期？ | |

### 目标校准会议

```markdown
## 目标校准会议

### 参会人员
- 上级主管
- 同级协作
- 下属代表

### 议程
1. 各方陈述目标设定依据 (10分钟/人)
2. 质疑和讨论 (20分钟)
3. 校准和调整 (15分钟)
4. 达成共识 (5分钟)

### 输出
- 校准后的目标值
- 资源承诺
- 支持事项
```

## 目标分解方法

### 纵向分解（时间维度）

```
年度目标 → 季度目标 → 月度目标 → 周目标

示例:
年度目标: 作业提交率 85%
├─ Q1: 70% (+2%)
├─ Q2: 75% (+5%)
├─ Q3: 80% (+5%)
└─ Q4: 85% (+5%)
    ├─ 1月: 68%
    ├─ 2月: 70%
    └─ ...
```

### 横向分解（组织维度）

```
公司目标 → 部门目标 → 团队目标 → 个人目标

示例:
公司目标: GMV 增长 30%
├─ 市场部: 获客 10000+
├─ 运营部: 转化率提升至 5%
├─ 产品部: 客单价提升至 200元
└─ 客服部: 复购率提升至 40%
```

### 维度分解（业务维度）

```
北极星指标 → 一级子指标 → 二级子指标

示例:
作业提交率
├─ 完播率 → 人均完播课程数
├─ 完成率 → 按时完成率
└─ 主动提交率 → 催交转化率
```

## 目标设定检查清单

```
[ ] 1. 具体性检查
    - 目标描述是否清晰？
    - 是否明确了要做什么？

[ ] 2. 可衡量性检查
    - 有明确的数值目标吗？
    - 如何判断目标达成？

[ ] 3. 可达成性检查
    - 参考了历史数据吗？
    - 考虑了资源约束吗？

[ ] 4. 相关性检查
    - 与业务大目标一致吗？
    - 为什么这个目标重要？

[ ] 5. 时限性检查
    - 有明确的截止日期吗？
    - 阶段里程碑清晰吗？
```

## 目标案例

### 案例1: 电商GMV目标

| 维度 | 内容 |
|------|------|
| **S (具体)** | 月度GMV从50万提升到65万 |
| **M (可衡量)** | GMV增长30%，订单量增长25%，客单价提升5% |
| **A (可达成)** | 基于历史增长率15%/季，当前资源可支撑30%增长 |
| **R (相关)** | GMV增长直接支撑公司收入目标 |
| **T (时限)** | Q2内完成（4-6月） |

### 案例2: 教育完课率目标

| 维度 | 内容 |
|------|------|
| **S (具体)** | 核心课程完课率从45%提升到55% |
| **M (可衡量)** | 完课率=完成课程人数/报名人数，按周统计 |
| **A (可达成)** | 完播率提升10%，作业完成率提升8% |
| **R (相关)** | 完课率是学习效果的核心指标，影响续费率 |
| **T (时限)** | Q2末达成，每月底评估进度 |

### 案例3: SaaS续费率目标

| 维度 | 内容 |
|------|------|
| **S (具体)** | 年度续费率从70%提升到80% |
| **M (可衡量)** | 续费率=到期续费团队数/到期总团队数 |
| **A (可达成)** | 需提升活跃团队占比，加强客户成功团队 |
| **R (相关)** | 续费率直接影响ARR，是SaaS核心指标 |
| **T (时限)** | 年度目标，分解到季度追踪 |
