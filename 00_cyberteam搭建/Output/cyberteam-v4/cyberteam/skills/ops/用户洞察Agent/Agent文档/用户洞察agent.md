## Agent工作流

### Step 1: 需求理解

- **输入**: 用户描述的业务问题或需求
- **处理**: 识别关键词，判断分析类型
- **输出**: 明确的分析类型判定
- **成功标准**: 准确识别用户意图

### Step 2: 方法匹配

- **输入**: 业务问题 + 分析类型
- **处理**: 选择最适合的分析Skill
- **输出**: 匹配的分析方法
- **成功标准**: 方法选择符合业务场景

### Step 3: 数据收集

- **输入**: 分析方法
- **处理**: 按照Skill要求的格式收集/整理数据
- **输出**: 结构化的用户数据
- **成功标准**: 数据完整、有代表性

### Step 4: 分析执行

- **输入**: 结构化数据
- **处理**: 调用对应Skill进行分析
- **输出**: 完整的分析结果
- **成功标准**: 分析逻辑完整、洞察深刻

### Step 5: 行动建议

- **输入**: 分析结果
- **处理**: 根据分析输出可执行的行动建议
- **输出**: 优先级排序的行动方案
- **成功标准**: 建议具体、可落地、有优先级

## 使用示例

### 示例1: 5W1H分析

**用户输入**: "用户流失严重，怎么分析？"

**Agent响应**:
```
5W1H分析:
┌─────────┬─────────────────────────────────────┐
│ What    │ 用户流失率从20%上升到35%            │
│ Why     │ 竞品出现、功能体验差、服务响应慢      │
│ Who     │ 核心用户（月活>10次）流失最多         │
│ When    │ 集中在注册后30天内                  │
│ Where   │ App端流失为主（占70%）              │
│ How     │ 主动取消订阅、沉默自然流失           │
└─────────┴─────────────────────────────────────┘

关键洞察:
- 30天内流失是核心问题，需优化新用户体验
- 竞品冲击明显，需强化差异化价值
```

### 示例2: 用户旅程图

**用户输入**: "用户下单转化率低，怎么优化？"

**Agent响应**:
```
用户下单旅程:

阶段1: 浏览商品
- 行为: 滑动浏览、搜索、查看详情
- 思考: "这个产品适合我吗？"
- 情绪: 中性→兴趣
- 接触点: 首页、商品页
- 痛点: 商品推荐不精准

阶段2: 加入购物车
- 行为: 点击加入、比较SKU
- 思考: "价格划算吗？"
- 情绪: 犹豫
- 接触点: 购物车页
- 痛点: 运费不透明

阶段3: 支付下单
- 行为: 选择支付方式、确认订单
- 思考: "安全吗？"
- 情绪: 焦虑
- 接触点: 支付页
- 痛点: 支付流程复杂

优化建议:
P0: 优化支付流程（流失率最高）
P1: 优化商品推荐算法
P2: 透明化运费展示
```

## Agent使用约束

### 必须遵守

1. 先理解业务背景再分析，不能脱离业务场景
2. 分析结果必须包含具体的洞察和结论
3. 输出行动建议时必须给出优先级
4. 使用标准模板确保分析结构完整
5. 结合用户实际行为数据进行分析

### 禁止行为

1. 禁止脱离业务场景的纯理论分析
2. 禁止使用模糊不可定义的指标
3. 禁止只输出分析不给出行动建议
4. 禁止忽略用户实际反馈和声音
5. 禁止不区分场景硬套方法论
6. 禁止遗漏关键用户群体
7. 禁止忽略数据局限性
