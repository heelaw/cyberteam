# 根因验证方法指南

## Why分析的重要性

5W1H分析中，Why（为什么）是核心。一个分析的质量取决于根因假设的质量和验证方法的严谨性。

---

## 根因验证方法

### 方法一：数据验证法

**适用场景**: 有足够的定量数据

**验证步骤**:
1. 提出根因假设
2. 找到与该假设相关的指标数据
3. 计算相关性或因果性
4. 评估置信度

**示例**:
```
假设: 页面加载速度慢导致转化下降

验证:
1. 收集页面加载时间数据
2. 对比加载快vs慢的用户的转化率
3. 计算相关系数
4. 结论: 加载时间>3秒的用户转化率低40%
```

---

### 方法二：用户反馈验证法

**适用场景**: 有用户访谈或反馈数据

**验证步骤**:
1. 收集相关用户反馈
2. 归类整理反馈内容
3. 统计各原因的提及频率
4. 交叉验证

**示例**:
```
假设: 价格是用户流失的主要原因

验证:
1. 收集近30天投诉/反馈
2. 归类：价格相关/体验相关/产品相关
3. 统计：价格相关占45%
4. 结合流失数据交叉验证
```

---

### 方法三：AB测试验证法

**适用场景**: 能够设计对照实验

**验证步骤**:
1. 设计实验组和对照组
2. 控制单一变量
3. 收集实验数据
4. 统计显著性分析

**示例**:
```
假设: 运费不透明导致加购后放弃

验证:
1. 实验组: 展示运费明细
2. 对照组: 不展示
3. 对比加购到下单转化率
4. 结果: 实验组转化提升25%，显著
```

---

### 方法四：对比分析法

**适用场景**: 有可比较的参照组

**验证步骤**:
1. 找到问题用户和非问题用户
2. 对比两组的行为特征差异
3. 识别关键差异点
4. 推断可能原因

**示例**:
```
假设: 新用户流失是因为不了解产品

验证:
1. 对比流失用户vs留存用户
2. 留存用户首日使用功能更多
3. 流失用户首日体验步骤更少
4. 推断: 新手引导缺失是原因
```

## 置信度评级标准

| 置信度 | 标准 | 说明 |
|--------|------|------|
| 高 | 多维度数据一致指向 | 有2种以上验证方法支撑 |
| 中 | 单维度数据支撑 | 仅有定性或定量单一证据 |
| 低 | 推测性结论 | 无直接证据，仅理论推断 |

## 常见错误

### 错误1: 相关性当因果性
```
错误: 用户流失前都访问过某页面，所以该页面导致流失

正确: 控制其他变量，验证因果关系
```

### 错误2: 样本偏差
```
错误: 基于投诉用户分析得出"价格太贵"是主因

正确: 考虑投诉率基数和代表性
```

### 错误3: 事后归因
```
错误: 上线新功能后指标下降，所以新功能导致下降

正确: 对比同期其他变量，排除外部因素
```

### 错误4: 忽视混杂因素
```
错误: 移动端转化低，所以移动体验差

正确: 控制用户类型、设备、网络等混杂因素
```
