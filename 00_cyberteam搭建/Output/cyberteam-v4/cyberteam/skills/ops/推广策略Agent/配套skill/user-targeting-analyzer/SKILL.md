# user-targeting-analyzer

## 简介

精准定义潜在用户是找到合适推广入口的前提条件。本Skill通过三种方法帮助分析并精准定义目标用户画像。

## 触发关键词

```
用户画像、精准定义用户、目标用户分析、用户调研、用户数据、用户假设验证、小范围测试
```

## 触发场景描述

```
当需要明确以下问题时触发本Skill：
- 产品/服务的目标用户是谁
- 如何精准定义潜在用户画像
- 如何找到匹配的推广入口
- 用户数据不足时如何验证假设
```

## 输入

### 必需输入

| 字段 | 类型 | 说明 |
|------|------|------|
| product_info | string | 产品/服务基本信息 |
| current_users | string (optional) | 现有用户数据（如果有） |

### 可选输入

| 字段 | 类型 | 说明 |
|------|------|------|
| marketing_budget | string | 营销预算范围 |
| product_stage | string | 产品阶段（初创/成长/成熟） |

## 方法论：三步法精准定义用户

### 方法一：分析现有产品数据

**数据来源：**
- 百度统计、Google Analytics
- 站内后台数据
- 第三方数据分析工具

**分析维度：**
- 用户地域分布
- 年龄性别比例
- 访问来源渠道
- 用户行为路径

### 方法二：进行用户调研和用户访谈

**用户访谈：**
- 电话访谈
- 深度访谈录音
- 一对一交流

**用户调研：**
- 在线问卷调查
- 邮件调研
- 委托第三方专业调研机构（适用于预算充足的情况）

### 方法三：小范围测试验证假设

**适用场景：** 用户画像不清晰的情况下

**操作方法：**
1. 基于现有认知做出用户假设
2. 小范围投放推广测试
3. 收集反馈数据
4. 调整用户画像定义
5. 迭代优化

## 工作流步骤

### Step 1: 收集现有数据

- **输入:** product_info, current_users
- **处理:** 梳理现有用户数据结构，确定可用数据源
- **输出:** 数据源清单和数据完整性报告
- **成功标准:** 明确有哪些可用数据，识别数据缺口

### Step 2: 选择分析方法

- **输入:** 数据完整性报告、marketing_budget
- **处理:** 根据数据完整性和预算选择合适的方法组合
- **输出:** 分析方法组合建议
- **成功标准:** 方法组合覆盖定性和定量验证

### Step 3: 执行用户画像分析

- **输入:** 产品信息、选定的分析方法
- **处理:**
  - 方法一：数据分析 → 用户画像草稿
  - 方法二：调研访谈 → 定性用户特征
  - 方法三：测试验证 → 画像校正
- **输出:** 三种方法交叉验证后的用户画像
- **成功标准:** 画像包含人口统计、行为特征、目标动机三个层面

### Step 4: 输出用户画像报告

- **输入:** 分析结果
- **处理:** 格式化输出用户画像文档
- **输出:** 完整的用户画像报告
- **成功标准:** 报告可直接用于推广渠道选择

## Critical Rules

### 必须遵守

1. 先精准定义用户，再选择推广渠道
2. 定量和定性方法交叉验证
3. 用户画像必须包含可行动的描述
4. 数据不足时，通过小范围测试补足

### 禁止行为

1. 禁止凭直觉假设用户，不做数据验证
2. 禁止只依赖单一数据源
3. 禁止模糊的用户描述（如"年轻人"需明确年龄范围）
4. 禁止跳过用户假设验证直接投放
5. 禁止忽视用户反馈数据

## 评估清单

### 评估维度

| 维度 | 标准 | 权重 |
|------|------|------|
| 人口统计完整性 | 包含年龄/性别/地域/职业 | 20% |
| 行为特征清晰度 | 明确在线习惯/购买行为 | 25% |
| 目标动机明确性 | 清晰描述用户目标和动机 | 25% |
| 数据验证可靠性 | 三种方法交叉验证 | 20% |
| 可行动性 | 画像可直接指导渠道选择 | 10% |

### 评估标准

- **90-100分：** 五个维度全部达标，画像精准可操作
- **75-89分：** 四个维度达标，有轻微模糊之处
- **60-74分：** 三个维度达标，需要补充验证
- **<60分：** 画像不完整，需要重新分析
