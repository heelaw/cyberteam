**对话**：
```
用户：我们公司要做会员运营，但会员业务是全新模块，该怎么搭建局部模型？

## 会员运营局部模型设计

### 定位
- 在整体中的角色：提升用户生命周期价值
- 与其他模块关系：获客→激活→会员→续费

### 目标
- 会员付费率：15%
- 会员续费率：70%
- 会员ARPU：300元/年

### 策略
1. 获客期：识别高潜用户，推会员价值
2. 转化期：限时优惠+专属权益
3. 权益期：差异化服务+积分体系
4. 续费期：到期前30天启动续费引导

- 会员转化率 = 付费会员数 / 活跃用户数
- 会员续费率 = 续费会员数 / 到期会员数
- 会员活跃率 = 每月使用权益会员数 / 总会员数

### 资源需求

- 预算：权益成本+营销费用
- 系统：会员管理后台

## 关键洞察
会员业务的本质是"锁定用户"：
- 权益设计要"有感知"——用户真能用上
- 价格锚定要"有意思"——年卡比月卡划算太多
- 续费提醒要"有温度"——提前而不是过期后
```

## 错误处理

| 错误类型 | 典型表现 | 处理方式 |
|---------|---------|---------|
| 边界不清 | 局部模型和整体模型混淆 | 先画大圈确定边界，再聚焦小圈 |
| 要素不全 | 只谈策略不谈资源和目标 | 五要素缺一不可，尤其是"资源" |
| 局部最优 | 某环节优化损害整体 | 每次决策前问"对整体有什么影响" |
| 关联断裂 | 各局部模型老死不相往来 | 建立局部之间的指标联动机制 |

当用户出现以下情况时触发本Skill：

- 团队分工需要，各负责局部模型

## 核心概念：什么是局部业务模型

### 定义

局部业务模型是指在整体业务模型中，聚焦于某个特定环节、渠道或用户群体的子模型。

```
整体业务模型
├── 局部模型A（获客）
├── 局部模型B（转化）
├── 局部模型C（留存）
├── 局部模型D（变现）
└── ...
```

### 局部模型 vs 整体模型

| 维度 | 整体业务模型 | 局部业务模型 |
|------|-------------|-------------|
| 范围 | 业务全局 | 单个环节/渠道 |
| 视角 | 宏观视角 | 微观视角 |
| 目的 | 理解全貌 | 深度优化 |
| 使用场景 | 战略规划 | 执行落地 |
| 分析深度 | 广而浅 | 深而精 |

## 局部业务模型的形成方法

### 方法一：按业务环节拆分

```
整体业务 → 获客 → 激活 → 留存 → 变现 → 传播
              ↓       ↓      ↓      ↓      ↓
           获客模型  激活模型 留存模型 变现模型 传播模型
```

| 环节 | 局部模型核心问题 |
|------|-----------------|
| 获客 | 如何低成本获取精准用户？ |
| 激活 | 如何让用户快速体验到价值？ |
| 留存 | 如何让用户持续使用？ |
| 变现 | 如何提升付费转化和ARPU？ |
| 传播 | 如何激发用户口碑传播？ |

### 方法二：按渠道拆分

```
整体业务 → 渠道A → 渠道B → 渠道C → 渠道D
              ↓       ↓       ↓       ↓
           渠道A模型 渠道B模型 渠道C模型 渠道D模型
```

| 渠道类型 | 局部模型关注点 |
|----------|---------------|
| 自然流量 | SEO、内容质量 |
| 付费流量 | ROI、精准度 |
| 社交裂变 | K因子、传播系数 |
| 合作渠道 | 分成比例、质控 |

### 方法三：按用户群体拆分

```
整体业务 → 用户群A → 用户群B → 用户群C
              ↓        ↓        ↓
           A群模型   B群模型   C群模型
```

| 用户分层 | 局部模型策略 |
|----------|-------------|
| 核心用户 | VIP服务、深度运营 |
| 活跃用户 | 激励体系、会员权益 |
| 普通用户 | 培养转化、价值教育 |
| 沉默用户 | 唤醒策略、流失分析 |

## 局部模型的五要素

每个局部模型也需要完整的五要素：

### 局部模型五要素

```
┌─────────────────────────────────────────┐
│              局部模型画布                │
├─────────────────┬─────────────────────────┤
│  1. 定位        │  在整体中的位置和作用    │
├─────────────────┼─────────────────────────┤
│  2. 目标        │  本模块要达成的目标      │
├─────────────────┼─────────────────────────┤
│  3. 策略        │  实现目标的方法          │
├─────────────────┼─────────────────────────┤
│  4. 指标        │  衡量效果的指标          │
├─────────────────┼─────────────────────────┤
│  5. 资源        │  需要投入的资源          │
└─────────────────┴─────────────────────────┘
```

### 五要素详解

#### 要素1：定位

| 问题 | 回答 |
|------|------|
| 这个模块在整体中是什么角色？ | |
| 与其他模块的关系是什么？ | |
| 它的边界在哪里？ | |

#### 要素2：目标

| 问题 | 回答 |
|------|------|
| 本模块的核心目标是什么？ | |

## 三个常见误区

### 误区一：局部最优≠整体最优

**错误做法**：
```
为了提升A渠道的转化率，大幅增加A渠道的预算，
结果：A渠道转化提升，但整体ROI下降
```

**正确做法**：
```
评估A渠道对整体的贡献：
- A渠道用户质量如何？
- A渠道与其他渠道的协同效应？
- 预算重新分配是否更优？
```

**决策原则**：
```
局部优化必须服务于整体目标
如果局部优化损害整体，即使局部指标好看也要叫停
```

### 误区二：忽视局部之间的关联

**错误做法**：
```
获客团队拼命获客，不考虑后续转化承接
→ 用户进来后发现体验差，大量流失
```

**正确做法**：
```
建立局部联动机制：
- 获客指标 + 转化指标 联动考核
- 新用户质量纳入获客团队KPI
- 建立局部模型之间的反馈机制
```

**关联类型**：
| 局部关系 | 关联方式 |
|----------|----------|
| 获客→转化 | 新用户质量评估 |
| 转化→留存 | 首日体验追踪 |
| 留存→变现 | 付费意愿分析 |
| 变现→获客 | 用户推荐率追踪 |

### 误区三：局部模型僵化不变

**错误做法**：
```
业务已经转型，但局部模型还是一年前的
→ 局部模型与实际业务脱节
```

**正确做法**：
```
定期检视局部模型：
- 季度审视：局部模型是否还匹配整体？
- 月度复盘：局部目标是否需要调整？
- 周度追踪：执行策略是否有效？
```

**动态调整原则**：
```
局部模型 = 业务理解 + 实践验证 + 持续迭代
没有一成不变的局部模型，只有不断进化的局部模型
```

## 局部模型优化案例

### 案例：电商平台获客局部模型优化

#### 背景
某电商平台整体增长放缓，聚焦获客局部模型分析

#### 局部模型五要素

| 要素 | 内容 |
|------|------|
| 定位 | 整体获客入口，负责用户增长 |
| 目标 | 月新增有效用户50万，CAC<100元 |
| 策略 | 抖音为主+快手为辅+信息流补充 |
| 指标 | 曝光→点击→注册→首单 各环节转化率 |
| 资源 | 月预算500万，团队10人 |

#### 问题诊断

| 问题 | 表现 | 根因 |
|------|------|------|
| 获客成本上升 | CAC从80元涨到150元 | 流量红利消退 |
| 用户质量下降 | 首单转化率从30%降到15% | 渠道刷量 |
| 渠道效果波动 | 各渠道效果不稳定 | 策略未及时调整 |

#### 优化方案

| 策略 | 具体动作 | 预期效果 |
|------|---------|----------|
| 渠道结构优化 | 降低低效渠道预算30% | CAC下降15% |
| 素材迭代加速 | 每周新产3套素材 | CTR提升20% |
| 落地页优化 | 提升首单转化率 | ROI提升25% |
| 数据监控强化 | 建立实时监控系统 | 问题及时发现 |

## 实战练习

**练习题目**：为一个在线教育平台设计"会员续费"局部模型

**参考答案框架**：
- **定位**：在整体模型中负责用户价值最大化，连接首次付费和长期变现

- **策略**：到期前7天触达→价值提醒→专属优惠→续费成功
- **指标**：续费率、续费提醒打开率、优惠转化率
- **资源**：运营团队2人、短信/Push通道、优惠预算
