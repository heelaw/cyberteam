# 概念定义澄清模板

## 模板说明
用于将模糊的抽象概念转化为清晰的操作性定义。

## 模板内容

```
# [抽象概念] - 概念定义澄清

## 原始表述
- **用户描述**：[原始表述原文]
- **使用场景**：[在什么情况下提到这个概念]

## 追问澄清

### Q1：这个概念具体指什么？
- **用户回答**：[记录回答]
- **追问要点**：能否举例说明？
  - 正面例子：[例子]
  - 反面例子：[例子]

### Q2：这个概念的反义词是什么？
- **用户回答**：[记录回答]
- **分析**：通过反义词理解概念的边界

### Q3：XX好/不好时，你怎么看出来？
- **用户回答**：[记录回答]
- **提炼指标**：[从中提取可能的衡量指标]

### Q4：如果向一个小学生解释，你会怎么说？
- **用户回答**：[记录回答]
- **简化版本**：[尝试用最简单的语言描述]

### Q5：这个概念与其他概念的区分是什么？
- **类似概念**：[类似的词有哪些]
- **区分点**：[与类似概念的核心区别]

## 操作性定义

### 定义表述
> [用1-2句话给出清晰、可理解的操作性定义]
> 定义必须回答：这个概念说的到底是什么？

### 关键词提取
| 关键词 | 含义 | 为什么重要 |
|--------|------|------------|
| [词1] | [含义] | [原因] |
| [词2] | [含义] | [原因] |

### 定义边界
- **包含范围**：这个概念包含什么？
  1. [包含1]
  2. [包含2]

- **不包含范围**：这个概念不包含什么？
  1. [不包含1]
  2. [不包含2]

## 概念可视化

### 类比说明
[用一个生活化的类比来解释这个概念]
- 类比对象：[什么]
- 类比点：[如何相似]
- 差异点：[但仍有什么不同]

### 概念层次
```
[更高层次的概念 - 如"质量"的上层是"价值"]
    ↑
[本概念]
    ↓
[具体表现/指标 - 如"质量"的表现是"耐用性"、"一致性"等]
```

## 验证定义

### 可理解性验证
- [ ] 第一次听到这个概念的人能否理解？
- [ ] 理解后能否向第三人转述？
- [ ] 举例时是否不会偏离定义？

### 完整性验证
- [ ] 定义能否覆盖用户提到的所有场景？
- [ ] 是否存在定义无法覆盖的边缘情况？
  - 边缘情况1：[描述] → [如何处理]

### 区分性验证
- [ ] 能否清晰区分本概念与类似概念？
- [ ] 是否会与其他概念混淆？

## 操作性定义的最终表述

> [将以上分析整合成最终的操作性定义]

## 相关子概念（如果需要进一步拆解）
| 子概念 | 定义 | 与父概念关系 |
|--------|------|--------------|
| [子概念1] | [定义] | [关系] |
| [子概念2] | [定义] | [关系] |
```

## 使用说明
1. 记录原始表述和使用场景
2. 通过追问逐步澄清概念含义
3. 提取关键词，形成操作性定义
4. 用类比和可视化帮助理解
5. 验证定义的正确性和完整性
6. 确定是否需要进一步拆解子概念
