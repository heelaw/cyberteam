## 技能描述

**一句话定义**：从目标逆向推导实现路径的框架构建方法，遵循MECE原则，确保每个层级的策略都指向最终目标。

**详细描述**：

自上而下框架构建法是战略规划的核心方法，适用于目标明确但实现路径不清晰的问题。它从最终目标出发，逆向分解出层层支撑的子目标，直到可执行的行动项。

这种方法确保所有工作都聚焦于目标，避免"做事但不做对的事"的陷阱。

## 核心原理

### 目标树法则

```
        [最终目标]
            │
    ┌───────┼───────┐
    ▼       ▼       ▼
 [子目标1] [子目标2] [子目标3]
    │       │       │
    ▼       ▼       ▼
 [行动项] [行动项] [行动项]
```

### MECE原则

每个层级必须满足：
- **Mutually Exclusive（互斥）**：子项之间无重叠
- **Collectively Exhaustive（全面）**：子项之和等于母项

---

## Critical Rules（必须遵守）

1. **目标必须SMART化** - 不明确的目标无法拆解，必须先转化为具体、可衡量、可实现、相关性、有时限的目标
2. **拆解深度不超过5层** - 超过5层的人类难以理解和执行，需要进一步抽象或分组
3. **每层必须有判断标准** - 每个分支都需要明确的优劣判断标准，否则无法做决策
4. **禁止跳过中间层级** - 不能从最高目标直接跳到最细行动项，必须逐级分解
5. **保持因果链条** - 每个子目标必须是父目标的可信前提，因果关系必须可验证
6. **行动项必须可执行** - 最底层的行动项必须是可以直接执行的动作，不能是模糊的方向
7. **识别关键假设** - 每个拆解分支都必须标注关键假设，假设不成立时需要备用方案

## Workflow Steps（标准化流程）

### 步骤1：目标澄清

| 项目 | 内容 |
|------|------|
| 输入 | 用户描述的目标（可能是模糊的） |
| 处理 | 通过追问将目标SMART化 |
| 输出 | 澄清后的SMART目标 |
| 成功标准 | 目标包含具体的衡量指标、截止日期、可验收的结果 |

**追问清单**：
- "这个目标实现后，具体是什么样子？"（可视化）
- "如何衡量目标是否达成？"（KPI）
- "实现这个目标的资源限制是什么？"（约束）
- "这个目标与其他目标的优先级如何？"（关联）

---

### 步骤2：逆向拆解

| 项目 | 内容 |
|------|------|
| 输入 | SMART目标 |
| 处理 | 从目标逆向列出所有可能的实现路径 |
| 输出 | 第一层子目标列表 |
| 成功标准 | 子目标满足MECE，且都是目标的可信前提 |

**拆解原则**：
- 按实现路径分组，而非按职能分组
- 确保每个子目标都是"必要非充分"的
- 检查是否有遗漏的关键路径

---

### 步骤3：可行性验证

| 项目 | 内容 |
|------|------|
| 输入 | 子目标列表 |
| 处理 | 评估每个子目标的可行性 |
| 输出 | 可行性评估报告，标注风险点 |
| 成功标准 | 每个子目标都有明确的可行性判断和风险标注 |

**评估维度**：
- 技术可行性
- 资源可行性
- 时间可行性
- 市场可行性

---

### 步骤4：行动展开

| 项目 | 内容 |
|------|------|
| 输入 | 可行的子目标列表 |
| 处理 | 将每个子目标继续拆解为可执行行动 |
| 输出 | 完整的目标树结构 |
| 成功标准 | 最底层行动项都是可直接执行的 |

**行动项标准**：
- Who（谁执行）
- What（做什么）
- When（何时完成）
- How（如何做）
- Check（如何验收）

---

### 步骤5：框架整合

| 项目 | 内容 |
|------|------|
| 输入 | 完整的目标树结构 |
| 处理 | 整合为结构化框架文档 |
| 输出 | 自上而下框架文档，包含：目标树、假设清单、风险点、优先级 |
| 成功标准 | 文档清晰展示从目标到行动的完整逻辑链条 |

## 使用限制

1. **不适用于**：目标完全模糊、约束条件完全未知、需要在探索中定义目标的情况
2. **前提条件**：用户必须有明确的目标（或愿意花时间澄清目标）
3. **局限性**：自上而下容易忽略执行层面的细节问题，需要与自下而上结合使用
