# 审核评分标准

## 一、评分体系

### 1.1 总体评分

| 维度 | 权重 | 最高分 |
|-----|------|-------|
| 目标匹配度 | 30% | 30 |
| 差异化 | 25% | 25 |
| 用户体验 | 25% | 25 |
| 可执行性 | 15% | 15 |
| 合规安全性 | 10% | 10 |
| **总计** | 100% | 100 |

### 1.2 评级标准

| 得分 | 评级 | 行动 |
|-----|------|------|
| 90-100 | 优秀 | 直接通过 |
| 75-89 | 良好 | 小幅修改后通过 |
| 60-74 | 及格 | 重大修改后通过 |
| <60 | 不及格 | 否决，重新设计 |

---

## 二、各维度评分细则

### 2.1 目标匹配度（30分）

| 子项 | 最高分 | 评分标准 |
|-----|-------|---------|
| KPI贡献路径 | 15分 | 清晰可量化: 15分 / 基本清晰: 10分 / 模糊: 5分 / 无关: 0分 |
| 目标受众匹配 | 10分 | 精准: 10分 / 基本精准: 7分 / 有偏差: 4分 / 不匹配: 0分 |
| 目标类型一致 | 5分 | 完全一致: 5分 / 基本一致: 3分 / 不一致: 0分 |

### 2.2 差异化（25分）

| 子项 | 最高分 | 评分标准 |
|-----|-------|---------|
| 独特性 | 10分 | 极具差异化: 10分 / 有差异: 7分 / 相似: 4分 / 雷同: 0分 |
| 用户价值 | 8分 | 价值突出: 8分 / 价值明确: 5分 / 价值一般: 3分 / 无价值: 0分 |
| 记忆点 | 7分 | 记忆点突出: 7分 / 有记忆点: 5分 / 一般: 3分 / 无: 0分 |

### 2.3 用户体验（25分）

| 子项 | 最高分 | 评分标准 |
|-----|-------|---------|
| 参与门槛 | 7分 | 1步: 7分 / 2步: 5分 / 3步: 3分 / 4步+: 0分 |
| 认知成本 | 6分 | 10秒理解: 6分 / 30秒: 4分 / 需仔细看: 2分 / 难理解: 0分 |
| 激励机制 | 7分 | 极有吸引力: 7分 / 有吸引力: 5分 / 一般: 3分 / 无吸引力: 0分 |
| 反馈机制 | 5分 | 即时清晰: 5分 / 及时: 3分 / 一般: 2分 / 无反馈: 0分 |

### 2.4 可执行性（15分）

| 子项 | 最高分 | 评分标准 |
|-----|-------|---------|
| 技术可行性 | 6分 | 完全可行: 6分 / 小幅改造: 4分 / 需新增: 2分 / 不可行: 0分 |
| 资源充足性 | 5分 | 完全充足: 5分 / 基本够: 3分 / 勉强: 2分 / 不足: 0分 |
| 时间可行性 | 4分 | 时间充裕: 4分 / 基本够: 3分 / 紧张: 2分 / 不够: 0分 |

### 2.5 合规安全性（10分）

| 子项 | 最高分 | 评分标准 |
|-----|-------|---------|
| 广告法合规 | 6分 | 完全合规: 6分 / 基本合规: 4分 / 有瑕疵: 2分 / 违规: 0分 |
| 隐私/版权合规 | 4分 | 完全合规: 4分 / 基本合规: 3分 / 有风险: 1分 / 明显违规: 0分 |

---

## 三、否决条件

### 3.1 绝对否决

以下任一条件触发，直接否决（不计算总分）：

| 维度 | 否决条件 |
|-----|---------|
| 目标匹配度 | KPI贡献<5分 |
| 用户体验 | 参与门槛>4步 |
| 用户体验 | 认知成本=0分 |
| 可执行性 | 技术可行性=0分 |
| 合规安全性 | 广告法违规 |

### 3.2 否决后的处理

```
触发否决条件
    ↓
确定否决原因
    ↓
修改方案
    ↓
重新提交审核
```

## 四、评分案例

### 案例1: 优秀评分（92分）

| 维度 | 得分 | 说明 |
|-----|-----|------|
| 目标匹配度 | 28/30 | KPI贡献清晰(14) + 精准匹配(10) + 类型一致(4) |
| 差异化 | 23/25 | 极具差异化(10) + 价值突出(7) + 记忆点突出(6) |
| 用户体验 | 23/25 | 1步参与(7) + 10秒理解(6) + 有吸引力(6) + 即时反馈(4) |
| 可执行性 | 14/15 | 完全可行(6) + 资源充足(5) + 时间充裕(3) |
| 合规安全性 | 9/10 | 完全合规(6) + 隐私合规(3) |
| **总计** | 92/100 | **评级: 优秀** |

### 案例2: 及格评分（68分）

| 维度 | 得分 | 说明 |
|-----|-----|------|
| 目标匹配度 | 20/30 | 基本清晰(8) + 基本精准(8) + 基本一致(4) |
| 差异化 | 16/25 | 有差异(6) + 价值一般(5) + 一般(5) |
| 用户体验 | 15/25 | 3步参与(3) + 需仔细看(3) + 吸引力一般(5) + 基本有反馈(4) |
| 可执行性 | 12/15 | 小幅改造(4) + 基本够(4) + 时间充裕(4) |
| 合规安全性 | 5/10 | 基本合规(4) + 基本合规(1) |
| **总计** | 68/100 | **评级: 及格** |

### 案例3: 否决（触发否决条件）

| 维度 | 得分 | 说明 |
|-----|-----|------|
| 用户体验 | 0/7 | 参与门槛5步，触发否决条件 |
| **审核结果** | **否决** | 需要重新设计 |

## 五、评分检查清单

- [ ] 已完整阅读创意方案

- [ ] 出具评分明细
- [ ] 标注主要问题
- [ ] 给出修改建议
