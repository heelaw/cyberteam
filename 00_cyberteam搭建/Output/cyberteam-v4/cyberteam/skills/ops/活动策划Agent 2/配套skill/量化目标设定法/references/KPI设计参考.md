# KPI设计参考

## 一、KPI设计原则

### 1.1 SMART原则

| 字母 | 含义 | 要求 |
|-----|------|------|
| S | Specific（具体） | 目标具体清晰 |
| M | Measurable（可衡量） | 有明确的衡量标准 |
| A | Achievable（可达成） | 在努力下可实现 |
| R | Relevant（相关） | 与业务目标相关 |
| T | Time-bound（有时限） | 有明确的截止日期 |

### 1.2 KPI设计原则

| 原则 | 说明 | 示例 |
|-----|------|------|
| 结果导向 | 关注最终结果 | 付费人数 > 领券人数 |
| 可量化 | 用数字说话 | DAU提升20% |
| 可追踪 | 有数据支撑 | 有埋点和监测方案 |
| 可调整 | 根据业务调整 | 设置预警和调整机制 |

## 二、各目标KPI参考

### 2.1 拉新KPI

**核心KPI**:

| KPI | 定义 | 基准参考 | 目标设定 |
|-----|------|---------|---------|
| 新用户数 | 活动期间新增注册用户 | 根据历史数据 | 增长率20-50% |

| KPI | 定义 | 用途 |

| 新用户活跃率 | 新用户活跃占比 | 衡量激活效果 |
| 渠道流量占比 | 各渠道流量占比 | 优化渠道配比 |

## 三、KPI选择指南

### 3.1 选择原则

1. **与目标直接相关**
   - 选择能直接衡量目标是否达成的指标
   - 避免选择间接或滞后指标

2. **数据可采集**
   - 必须有明确的数据来源
   - 必须能实时或定期监测

3. **行业可比**
   - 选择行业通用的指标
   - 便于对标和竞品分析

4. **团队可影响**
   - 团队能通过努力改善该指标
   - 避免选择完全不可控的指标

### 3.2 KPI数量控制

| 活动周期 | 核心KPI数量 | 辅助KPI数量 |
|---------|-----------|------------|
| <7天 | 1-2个 | 2-3个 |
| 7-30天 | 2-3个 | 3-5个 |
| >30天 | 3-5个 | 5-8个 |

## 四、KPI基准值设定

### 4.1 基准值来源

| 来源 | 说明 | 优先级 |
|-----|------|-------|
| 历史数据 | 上一期或同期数据 | 最高 |
| 行业数据 | 行业平均水平 | 高 |
| 竞品数据 | 竞品公开数据 | 中 |
| 经验估算 | 专家判断 | 低 |

### 4.2 基准值计算

```
同比增长率 = (本期值 - 去年同期) / 去年同期 × 100%
环比增长率 = (本期值 - 上期值) / 上期值 × 100%
行业对标值 = 行业TOP10平均值 × 调整系数
```

## 五、目标值设定方法

### 5.1 目标值设定方法

| 方法 | 适用场景 | 优点 | 缺点 |
|-----|---------|-----|------|
| 基础目标法 | 保守估计 | 保底可达成 | 可能过于保守 |
| 进阶目标法 | 正常预期 | 合理有挑战 | 需要努力达成 |
| 冲刺目标法 | 激励团队 | 最大潜力挖掘 | 可能打击信心 |
| 对标目标法 | 行业比较 | 有参考依据 | 数据可能不准确 |

### 5.2 目标值公式

```
目标值 = 基准值 × (1 + 增长率)

示例:
基准值: 1000新用户
增长率: 30%（基于历史数据和资源投入）
目标值: 1000 × 1.3 = 1300新用户
```

## 六、KPI监测与预警

### 6.1 监测频率

| KPI类型 | 监测频率 | 预警阈值 |
|---------|---------|---------|
| 结果指标 | 每日 | ±20% |
| 过程指标 | 每4小时 | ±30% |
| 实时指标 | 实时 | ±50% |

### 6.2 预警机制

```markdown
## KPI预警机制

### 预警级别
- 黄色预警: 低于预期10-20%
- 橙色预警: 低于预期20-30%
- 红色预警: 低于预期30%以上

### 预警响应
- 黄色: 关注，48小时内分析原因
- 橙色: 重点关注，24小时内制定对策
- 红色: 立即响应，4小时内启动应急
```
