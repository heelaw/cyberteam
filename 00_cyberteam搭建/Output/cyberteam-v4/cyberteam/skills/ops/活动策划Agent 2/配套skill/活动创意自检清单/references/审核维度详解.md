# 审核维度详解

## 一、目标匹配度审核

### 1.1 KPI贡献路径

**检查问题**:
- 创意能直接贡献哪个KPI？
- 贡献路径是否清晰？
- 贡献量级是否与目标匹配？

**评估标准**:
| 等级 | 描述 |
|-----|------|
| 优秀 | KPI贡献路径清晰，可量化 |
| 良好 | KPI贡献路径基本清晰 |
| 及格 | 能说明贡献但不够具体 |
| 不及格 | 无法说明对KPI的贡献 |

### 1.2 目标受众匹配

**检查问题**:
- 目标人群会参与这个活动吗？
- 活动形式符合受众偏好吗？
- 触达渠道覆盖目标人群吗？

**评估标准**:
| 等级 | 描述 |
|-----|------|
| 优秀 | 受众精准，匹配度高 |
| 良好 | 受众基本精准 |
| 及格 | 受众有一定关联 |
| 不及格 | 受众完全不匹配 |

---

## 二、差异化审核

### 2.1 独特性评估

**检查问题**:
- 与竞品有什么明显差异？
- 差异化点用户能感知吗？
- 差异化能否持续？

**评估标准**:
| 等级 | 描述 |
|-----|------|
| 优秀 | 与竞品明显不同，有独特价值 |
| 良好 | 有一定差异性 |
| 及格 | 与竞品略有不同 |
| 不及格 | 与竞品高度相似 |

### 2.2 用户价值评估

**检查问题**:
- 用户能得到什么？
- 这个价值对用户重要吗？
- 用户能感知到这个价值吗？

**评估标准**:
| 等级 | 描述 |
|-----|------|
| 优秀 | 用户价值明确且感知强 |
| 良好 | 用户价值比较明确 |
| 及格 | 用户价值一般 |
| 不及格 | 用户价值模糊或无价值 |

### 2.3 记忆点评估

**检查问题**:
- 活动有什么让人记住的点？
- 这个点能传播吗？
- 能否用一句话描述？

**评估标准**:
| 等级 | 描述 |
|-----|------|
| 优秀 | 有强记忆点，易传播 |
| 良好 | 有记忆点 |
| 及格 | 记忆点一般 |
| 不及格 | 没有任何记忆点 |

---

## 三、用户体验审核

### 3.1 参与门槛评估

**检查问题**:
- 用户需要几步完成参与？
- 是否有复杂的操作或输入？
- 门槛是否与奖励成正比？

**评估标准**:
| 参与步骤 | 评级 |
|---------|------|
| 1步 | 优秀 |
| 2步 | 良好 |
| 3步 | 及格 |
| 4步以上 | 不及格 |

### 3.2 认知成本评估

**检查问题**:
- 用户能否快速理解活动规则？
- 规则描述是否清晰无歧义？
- 是否需要客服解释？

**评估标准**:
| 等级 | 描述 |
|-----|------|
| 优秀 | 10秒内完全理解 |
| 良好 | 30秒内理解 |
| 及格 | 需要仔细阅读理解 |
| 不及格 | 难以理解或容易误解 |

### 3.3 激励机制评估

**检查问题**:
- 奖励对用户有吸引力吗？
- 奖励感知价值高吗？
- 激励是否及时兑现？

**评估标准**:
| 等级 | 描述 |
|-----|------|
| 优秀 | 激励极有吸引力 |
| 良好 | 激励有吸引力 |
| 及格 | 激励一般 |
| 不及格 | 激励无吸引力 |

### 3.4 反馈机制评估

**检查问题**:
- 用户操作后是否有反馈？
- 反馈是否及时？
- 反馈是否清晰？

**评估标准**:
| 等级 | 描述 |
|-----|------|
| 优秀 | 即时、清晰、有成就感 |
| 良好 | 及时、清晰 |
| 及格 | 有反馈但不够及时 |
| 不及格 | 完全没有反馈 |

---

## 四、可执行性审核

### 4.1 技术可行性评估

**检查问题**:
- 现有技术能否实现？
- 需要新增开发吗？
- 技术风险大吗？

**评估标准**:
| 等级 | 描述 |
|-----|------|
| 优秀 | 现有技术完全可支持 |
| 良好 | 小幅改造即可支持 |
| 及格 | 需要新增开发 |
| 不及格 | 技术实现困难或无法实现 |

### 4.2 资源充足性评估

**检查问题**:
- 人力是否充足？
- 物料是否准备到位？
- 预算是否足够？

**评估标准**:
| 等级 | 描述 |
|-----|------|
| 优秀 | 资源完全充足 |
| 良好 | 资源基本充足 |
| 及格 | 资源勉强够用 |
| 不及格 | 资源严重不足 |

---

## 五、合规安全性审核

### 5.1 广告法合规评估

**检查问题**:
- 是否使用禁用词汇（最高级、国家级等）？
- 价格宣传是否有依据？
- 是否有虚假宣传？

**评估标准**:
| 等级 | 描述 |
|-----|------|
| 优秀 | 完全合规，无风险点 |
| 良好 | 完全合规 |
| 及格 | 有小瑕疵但无违规 |
| 不及格 | 明显违规或高风险 |

### 5.2 隐私合规评估

**检查问题**:
- 是否过度收集用户数据？
- 隐私政策是否合规？
- 是否有授权？

**评估标准**:
| 等级 | 描述 |
|-----|------|
| 优秀 | 完全合规，有保护措施 |
| 良好 | 完全合规 |
| 及格 | 基本合规 |
| 不及格 | 有隐私风险 |

---

## 六、综合评级计算

### 6.1 评分权重

| 维度 | 权重 | 说明 |
|-----|------|------|
| 目标匹配度 | 30% | 核心中的核心 |
| 差异化 | 25% | 竞争力来源 |
| 用户体验 | 25% | 决定参与率 |
| 可执行性 | 15% | 落地保障 |
| 合规安全性 | 10% | 底线要求 |

### 6.2 评级标准

| 综合得分 | 评级 | 说明 |
|---------|------|------|
| 90-100 | 优秀 | 直接通过，可执行 |
| 75-89 | 良好 | 小幅修改后通过 |
| 60-74 | 及格 | 重大修改后通过 |
| <60 | 不及格 | 否决，重新设计 |

### 6.3 否决条件

以下任一条件触发，直接否决：

- 目标匹配度 < 60%
- 用户体验 < 60%
- 可执行性 < 60%
- 合规安全性不通过
