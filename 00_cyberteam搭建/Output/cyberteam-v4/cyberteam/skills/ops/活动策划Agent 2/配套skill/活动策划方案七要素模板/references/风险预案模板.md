# 风险预案模板

## 一、风险识别框架

### 1.1 风险来源分类

| 类别 | 描述 | 示例 |
|-----|------|------|
| 技术风险 | 系统、技术相关问题 | 服务器崩溃、接口故障 |
| 运营风险 | 活动执行、用户行为问题 | 刷单、薅羊毛 |
| 外部风险 | 外部环境、竞争、舆情 | 竞品反击、负面舆情 |
| 政策风险 | 法规、监管变化 | 广告法违规、平台规则 |
| 资源风险 | 人力、物力、财力不足 | 人员变动、预算削减 |

### 1.2 风险评估矩阵

| | 低概率 | 中概率 | 高概率 |
|---|-------|-------|-------|
| **高影响** | 关注 | 重点防控 | 极高优先级 |
| **中影响** | 监控 | 常规防控 | 重点防控 |
| **低影响** | 可接受 | 关注 | 监控 |

## 四、应急响应机制

### 4.1 响应等级

| 等级 | 触发条件 | 响应时间 | 参与人员 |
|-----|---------|---------|---------|
| P0 | 重大事故，影响业务 | 立即 | 总监+所有相关 |
| P1 | 严重问题，影响部分功能 | 30分钟内 | 经理+相关 |
| P2 | 一般问题，影响用户体验 | 2小时内 | 负责人 |

### 4.2 响应流程

```
风险/问题发生
    ↓
判断等级
    ↓
P0: 立即电话通知 + 15分钟内启动会议
P1: 30分钟内响应 + 制定方案
P2: 2小时内处理
    ↓
执行应急预案
    ↓
问题解决
    ↓
复盘总结
```

### 4.3 沟通机制

**内部沟通**:
- P0: 实时群播报，每30分钟更新
- P1: 每1小时更新进度
- P2: 处理完成后同步

**外部沟通**:
- 用户: 官方渠道公告
- 媒体: PR团队统一口径
- 监管: 法务对接
