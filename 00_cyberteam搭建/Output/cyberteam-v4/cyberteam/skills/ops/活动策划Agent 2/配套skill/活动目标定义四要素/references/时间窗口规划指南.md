# 时间窗口规划指南

## 一、活动周期规划

### 1.1 周期类型

| 周期类型 | 时长 | 适用场景 | 特点 |
|---------|------|---------|------|
| 短周期 | 1-7天 | 热点营销、节日促销 | 高爆发、快节奏 |
| 中周期 | 7-30天 | 大促活动、品牌营销 | 预热+爆发+长尾 |
| 长周期 | 30天以上 | 品牌建设、用户养成 | 持续运营、习惯培养 |

### 1.2 周期阶段划分

#### 短周期活动（1-7天）
```
[无预热期] → [爆发期] → [收尾期]
```

#### 中周期活动（7-30天）
```
[预热期] → [爆发期] → [长尾期] → [收尾期]
```

#### 长周期活动（30天+）
```
[启动期] → [稳定期] → [高潮期] → [收尾期]
```

## 二、关键节点规划

### 2.1 预热期

**时长**: 通常为正式活动时间的1/3至1/2

**目标**:
- 制造悬念和期待
- 积累目标用户
- 锁定资源位

**关键任务**:
| 时间 | 任务 | 交付物 |
|-----|------|-------|
| -7天 | 活动素材准备 | 海报、文案、页面 |
| -5天 | 渠道资源锁定 | 排期确认 |
| -3天 | 内部测试 | 测试报告 |
| -1天 | 最终确认 | 上线checklist |

### 2.2 爆发期

**时长**: 通常为1-3天（大型活动可延长）

**目标**:
- 最大化流量获取
- 达成核心KPI
- 制造话题声量

**关键任务**:
| 时间 | 任务 | 交付物 |
|-----|------|-------|
| D-day | 0点正式开启 | 监控大屏 |
| D-day | 全天实时监控 | 数据战报 |
| D+1 | 实时调整策略 | 优化方案 |

### 2.3 长尾期

**时长**: 通常为爆发期的2-3倍

**目标**:
- 持续转化
- 积累活动资产
- 为下次活动铺垫

**关键任务**:
- 持续运营
- 数据收集
- 用户维护

## 四、时间窗口评估清单

### 4.1 评估维度

- [ ] 活动周期是否与业务节奏匹配
- [ ] 预热期是否足够积累势能
- [ ] 爆发期是否在用户活跃时段
- [ ] 长尾期是否避免与竞品冲突
- [ ] 关键节点是否有预留buffer

### 4.2 风险检查

- [ ] 是否与重要节假日冲突
- [ ] 是否在竞品活跃期
- [ ] 是否有政策或监管风险
- [ ] 技术资源是否充足
