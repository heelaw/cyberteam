# 场景清单

## 用户状态与文案风格

### 用户状态分类

| 状态 | 描述 | 特征 | 适用文案类型 |
|------|------|------|--------------|
| 不知道 | 不知道有这需求 | 痛点未感知 | 教育型、痛点型 |
| 知道 | 知道但不了解 | 认知阶段 | 介绍型、科普型 |
| 考虑中 | 在比较选择 | 决策阶段 | 对比型、证据型 |
| 决定买 | 犹豫不决 | 临门一脚 | 促销型、紧迫感型 |

## 场景分类

### 1. 认知阶段（不知道）

**用户特征**：
- 不知道自己有这个问题
- 不知道有这个产品
- 问题存在但不痛苦

**文案目标**：
- 唤醒痛点
- 教育用户
- 建立认知

**文案风格**：
- 痛点唤醒：描述痛苦场景
- 科普教育：解释问题原因
- 解决方案：介绍产品价值

**案例**：
```
原文案：我们的数据分析工具很好用
痛点型：还在用Excel手工整理数据？每天加班到深夜？
科普型：为什么你的Excel总是卡死？数据量超过10万必须用专业工具
```

---

### 2. 了解阶段（知道）

**用户特征**：
- 知道有这产品
- 不了解具体能做什么
- 愿意花时间了解

**文案目标**：
- 介绍产品价值
- 建立品牌形象
- 区别于竞品

**文案风格**：
- 产品介绍：核心功能说明
- 场景展示：使用场景描述
- 价值主张：能解决什么问题

**案例**：
```
产品：在线设计工具
文案：3分钟出海报，不会PS也能用
场景：做海报、发朋友圈、店铺装修
```

---

### 3. 决策阶段（考虑中）

**用户特征**：
- 在多个产品中比较
- 想知道有什么区别
- 寻求购买理由

**文案目标**：
- 建立信任
- 消除顾虑
- 给出购买理由

**文案风格**：
- 对比型：我们vs竞品
- 证据型：案例、数据、证明
- 信任型：口碑、背书、保障

**案例**：
```
对比型：比XX多100个功能，价格只有一半
证据型：10万企业都在用，日均处理100万订单
信任型：7天无理由退款，不满意随时退
```

---

### 4. 购买阶段（决定买）

**用户特征**：
- 已经决定要买
- 在犹豫什么时候买
- 需要临门一脚

**文案目标**：
- 促成行动
- 减少顾虑
- 制造紧迫感

**文案风格**：
- 促销型：优惠、折扣、赠品
- 紧迫感：限时、限量、限价
- 保障型：售后、承诺、托底

**案例**：
```
促销型：今日下单立减200，赠价值300元礼品
紧迫感：仅剩50个名额，24小时后恢复原价
保障型：终身免费升级，7天无理由退款
```

## 常见场景清单

### 电商场景

| 阶段 | 用户状态 | 文案重点 | 示例 |
|------|----------|----------|------|
| 曝光 | 不知道 | 痛点唤醒 | "还在为选礼物发愁？" |
| 点击 | 知道 | 卖点吸引 | "送这个，再也不怕送错礼" |
| 详情 | 考虑中 | 信任建立 | "10万+好评，98%复购" |
| 成交 | 决定买 | 促销紧迫 | "限时满减，最后3小时" |

### 教育场景

| 阶段 | 用户状态 | 文案重点 | 示例 |
|------|----------|----------|------|
| 认知 | 不知道 | 痛点+方案 | "工资低？可能是技能不够" |
| 兴趣 | 知道 | 效果承诺 | "3个月学会运营，月薪翻倍" |
| 欲望 | 考虑中 | 案例+证据 | "学员案例，平均涨薪50%" |
| 行动 | 决定买 | 促销+保障 | "今日报名送价值500课程" |

### SaaS场景

| 阶段 | 用户状态 | 文案重点 | 示例 |
|------|----------|----------|------|
| 认知 | 不知道 | 问题+方案 | "还在为团队协作烦恼？" |
| 了解 | 知道 | 功能+场景 | "一个工具搞定团队所有事" |
| 评估 | 考虑中 | 对比+案例 | "XX企业都在用" |
| 成交 | 决定买 | 试用+优惠 | "免费试用30天" |
