## 1. 工作流概述

转化型文案创作工作流是一个系统化的9阶段流程，旨在确保每篇文案都经过完整的创作、审核、打磨流程，最终产出高转化率的营销文案。

### 1.0 触发场景判断

本流程不是万能的——日常推文、简单回复、即时消息不在此列。**当文案工作从"写一段文字"升级为"解决一个商业问题"时**，本流程才被触发。

#### 必须触发完整流程的场景

| 场景 | 特征 | 触发原因 |
|------|------|----------|
| **核心营销战役与新品上市** | 项目制、高投入、跨团队协作、结果可衡量 | 资源投入大，不允许失误 |
| **高价值转化型内容** | 详情页/Landing Page/SEM/高价服务推广 | 决策成本高，需深度说服 |
| **重要品牌或公关沟通** | 官方通稿/危机公关声明/重要公开信 | 风险极高，一言不慎引发舆论危机 |
| **体系化内容运营建设** | 搭建AI工作流/运营专栏/制定内容矩阵 | 需要SOP标准化确保质量稳定 |

### 1.1 工作流目标

- 确保文案基于真实用户反馈和数据分析
- 严格遵循投放平台的特点和规则
- 每一句话都服务于最终转化
- 深刻理解用户需求和心理

## 3. 九阶段流程

| Stage | 名称 | 核心产出 |
|-------|------|----------|
| 0 | 工作流初始化 | 任务简报 |
| 1 | 用户需求挖掘 | 用户画像卡 |
| 2 | 产品卖点挖掘 | FAB分析 |
| 3 | 痛点定位 | 痛点-卖点匹配矩阵 |
| 4 | 需求类型判断 | 说服策略 |
| 5 | 场景与渠道分析 | 渠道适配方案 |
| 6 | 文案撰写 | 初稿文案 |
| 7 | 文案优化 | 定稿文案 |
| 8 | 效果追踪 | 复盘报告 |

## 5. 质量控制

### 5.1 各阶段质量要求

| 阶段 | 质量要求 |
|------|----------|
| 用户需求挖掘 | 基于真实用户反馈，无主观臆断 |
| 产品卖点挖掘 | 突出差异化优势，与痛点精准匹配 |
| 痛点定位 | 优先级清晰，说服逻辑严密 |
| 文案撰写 | 结构完整，语言生动 |

### 5.2 质量检查清单

```markdown
- [ ] 核心卖点突出
- [ ] 痛点匹配准确
- [ ] 符合目标人群
- [ ] 符合平台特性
- [ ] 有明确的行动召唤
- [ ] 语言口语化
- [ ] 无虚假宣传
```
