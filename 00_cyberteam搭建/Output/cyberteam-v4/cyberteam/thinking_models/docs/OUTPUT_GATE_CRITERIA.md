# 思维天团输出门禁标准

> **版本**: v1.0
> **目的**: 定义"什么算过Gate"，只有过Gate的输出才能被标记为"完成"
> **Gate定义**: 门禁检查点，只有通过检查的输出才能流向下一环节或被交付

---

## 一、门禁核心理念

### 1.1 为什么需要门禁？

当前问题：
- 输出是否"完成"没有客观标准
- "我写完了"不等于"写好了"
- 缺乏对输出质量的自动检查机制

### 1.2 Gate的三个层级

```
┌──────────────────────────────────────────────┐
│  Gate 1: 完整性 Gate（能否交差）              │
│  → 所有必要元素是否存在？                      │
├──────────────────────────────────────────────┤
│  Gate 2: 质量 Gate（能否发布）                 │
│  → 内容是否达到"短文风格"标准？                │
├──────────────────────────────────────────────┤
│  Gate 3: 价值 Gate（能否交付）                │
│  → 是否有真正的洞察，还是正确的废话？           │
└──────────────────────────────────────────────┘
```

---

## 二、Gate 1：完整性检查

### 2.1 检查清单（每项必须通过）

```
□ 输入问题被明确识别
□ 专家使用了正确的分析框架
□ 至少覆盖了3个分析维度（以六顶思考帽为例）
□ 给出了明确的结论/建议
□ 结论与分析过程有逻辑关联
□ Critical Rules被遵守（每条专家的禁止行为清单）
```

### 2.2 完整性评分

| 问题 | 扣分 |
|------|------|
| 输入问题不明确 | -20 |
| 分析框架错误或不完整 | -20 |
| 分析维度不足（<3个） | -15 |
| 结论模糊（如"因人而异"） | -20 |
| 结论与分析无逻辑关联 | -20 |
| 违反Critical Rules | -25（直接否决） |

**Gate 1 通过标准**: >= 70分

### 2.3 完整性示例

```
【输入】"我应该选A还是B？"

□ 完整性合格：
  "选B。从机会成本看，A的真正成本是你在B能获得的成长。
   两个选择的差距不是"稳定vs风险"，而是"有限游戏vs无限游戏""

□ 完整性不合格（结论模糊）：
  "这个问题因人而异，需要根据你的具体情况来判断"

□ 完整性不合格（框架缺失）：
  "选B。感觉B更好"
  （没有用任何分析框架支撑判断）
```

---

## 三、Gate 2：风格质量检查

### 3.1 风格检查清单

```
□ 长度在对应层级范围内（L1-L4）
□ 无禁止的ASCII元素（═══、┌┬┐、---）
□ 无多级标题（###）
□ 无冗长的表格（能用自然语言描述）
□ 无"会议纪要腔"（首先、其次、最后、综上所述）
□ 每段不超过3句话
□ 有emoji作为视觉锚点（每段不超过1个）
□ 核心判断在第一句
□ 用 → 表达逻辑流向
□ 人设声音清晰（不是 impersonal 报告腔）
```

### 3.2 风格评分

| 问题 | 扣分 |
|------|------|
| 长度超出层级上限 20%以上 | -15 |
| 有任何禁止的ASCII元素 | -20 |
| 有多级标题 | -10 |
| 有冗长表格（>4行） | -15 |
| 有"会议纪要腔" | -15 |
| 单段超过5句话 | -10 |
| 没有emoji | -5 |
| 核心判断不在第一句 | -15 |
| 没有用 → 表达逻辑 | -10 |
| 人设声音不清晰 | -20 |

**Gate 2 通过标准**: >= 80分（零容忍项除外）

### 3.3 零容忍项（有任何一项直接否决）

```
✗ 零容忍：
- 使用了 ═══ 或 ┌┬┐ 等装饰性ASCII
- 超过200字的纯表格描述
- 使用了"综上所述"、"总而言之"等报告腔
- 整篇没有任何人称代词（你/我/他）
```

---

## 四、Gate 3：价值质量检查

### 4.1 价值检查清单

```
□ 是否有"反直觉"的洞察？（不是显而易见的废话）
□ 是否有具体的数据或案例支撑？（不是"一般来说"）
□ 是否有个性化的判断？（不是"要看情况"）
□ 是否能指导具体行动？（不是"你需要仔细考虑"）
□ 是否有"啊哈时刻"？（读者会不会觉得"我没想到这个"）
```

### 4.2 价值评分

| 维度 | 权重 | 评分标准 |
|------|------|---------|
| 反直觉洞察 | 30% | 至少1个"多数人没想到"的观点 |
| 具体支撑 | 25% | 有数据/案例，不只是抽象论述 |
| 个性化判断 | 20% | 给出了明确的立场，不是骑墙 |
| 可执行性 | 15% | 建议具体可操作，不是正确的废话 |
| 洞察深度 | 10% | 有"啊哈时刻" |

**Gate 3 通过标准**: >= 70分

### 4.3 价值示例

```
【输入】"我应该创业还是继续打工？"

□ 价值合格：
"如果你35岁前还没创过业，出去创一次。
统计上，35岁后创业的成功率开始下降，不是因为能力，
而是因为『该死的锚定效应』——你已经开始用『打工仔』的框架
套自己，而创业需要的是『玩家』框架。
真正的风险不是创业失败，而是『一辈子觉得自己没试过』。
✅ 行动：给自己设一个2年的窗口期，期间用『我能学到什么』
而不是『我能赚多少』来评估进展。"

□ 价值不合格（正确的废话）：
"创业还是打工取决于你的风险偏好和职业规划。
建议仔细考虑自己的长期目标，做一个理性的决策。
如果你喜欢稳定，打工更适合；如果你愿意冒险，创业更好。"
（全是正确的废话，没有洞察，没有立场，没有行动）
```

## 五、Gate流程

### 5.1 输出标准流程

```
输入 → Gate 1(完整性) → Gate 2(风格) → Gate 3(价值) → 输出
          ↓失败               ↓失败            ↓失败
       打回补充           打回修改风格        打回重写洞察
```

### 5.2 门禁状态

每个输出有三个状态：

| 状态 | 含义 | 标记 |
|------|------|------|
| **草稿** | 刚输出，未过Gate | `[草稿]` |
| **Gate通过** | 通过所有Gate，可以交付 | `[✅Gate]` |
| **需要重写** | 未通过Gate，需要修改 | `[❌Gate]` |

### 5.3 门禁失败处理

```
Gate 1 失败（完整性）：
→ 返回专家补充缺失元素
→ 触发词："请补充[缺失元素]，当前报告缺少[具体内容]"

Gate 2 失败（风格）：
→ 返回修改风格
→ 触发词："请用短文风格重写，当前输出有[风格问题]，参考格式规范第X条"

Gate 3 失败（洞察）：
→ 返回重写洞察
→ 触发词："当前输出缺乏反直觉洞察，请提供1个『多数人没想到』的观点"
```

## 六、专家专属门禁规则

### 6.1 卡尼曼决策专家

```
Gate 1 特定检查：
□ 是否识别了至少3个认知偏差
□ 是否量化了期望值（如果可以）
□ 是否给出了明确的决策建议

Gate 3 特定检查：
□ 是否有"系统1在骗你"类型的洞察
□ 偏差识别是否具体（不是泛泛的"要理性"）
□ 决策建议是否可执行（不是"谨慎决策"）
```

### 6.2 六顶思考帽专家

```
Gate 1 特定检查：
□ 是否六帽齐全（不能跳过任何一帽）
□ 每帽是否有实质性内容（不是1句话敷衍）
□ 蓝色帽子是否有整合结论

Gate 3 特定检查：
□ 是否有"某顶帽子揭示了反直觉发现"
□ 整合结论是否超越了各帽之和（不是简单汇总）
```

### 6.3 第一性原理专家

```
Gate 1 特定检查：
□ 是否真正"刨到根"（至少追问3层why）
□ 是否拆解到物理层/本质层
□ 结论是否回到原问题

Gate 3 特定检查：
□ 是否有"打破常识假设"的发现
□ 拆解过程是否有"啊，原来是这样"的递进
```
