# 确认偏误认知专家测试用例

## 正面测试

### 测试1: 投资观点验证
- 输入:
```
【确认偏误分析请求】
核心观点：这只科技股值得买入，因为公司营收增长30%
支持证据：季报显示营收增长30%，分析师看好
反对证据：暂无
```
- 期望输出:
  - 识别核心假设（增长30%是可持续的）
  - 反向搜索可能被低估的风险
  - 给出置信度校准
- 验证项:
  - [ ] 识别"增长30%可持续"假设
  - [ ] 质疑信息来源可靠性
  - [ ] 考虑替代解释（一次性收益？）
  - [ ] 置信度被适当降低

### 测试2: 职业方向确认
- 输入:
```
【确认偏误分析请求】
核心观点：我应该转行做产品经理，因为工资高
支持证据：招聘网站上产品经理工资高
反对证据：听说产品经理很累，而且需要经验
```
- 期望输出:
  - 平衡表列出支持和反对证据
  - 质疑信息来源的代表性
  - 识别动机性推理（"因为工资高"）
- 验证项:
  - [ ] 招聘网站数据的代表性被质疑
  - [ ] "工资高"的动机性推理被识别
  - [ ] 有替代职业比较

### 测试3: 观点坚定者分析
- 输入:
```
【确认偏误分析请求】
核心观点：我的商业计划一定能成功
支持证据：我做了详细调研，行业专家认可
反对证据：家人担心市场竞争太激烈
背景：这是我的第三次创业
```
- 期望输出:
  - 识别群体思维风险
  - 质疑"专家认可"的来源和动机
  - 识别连续创业者可能的过度自信
- 验证项:
  - [ ] 家族意见被识别为可能有道理
  - [ ] "专家认可"被深挖（哪个专家？为什么认可？）
  - [ ] 三次创业经历被考虑

## 负面测试

### 测试1: 只找支持证据
- 输入:
```
【确认偏误分析请求】
核心观点：这个创业方向是对的
支持证据：很多投资人看好这个赛道
反对证据：（用户没提供）
```
- 期望行为: 主动提供反向质疑清单，不只依赖用户输入
- 验证项:
  - [ ] 系统性生成反向质疑
  - [ ] 不依赖用户已有反对证据

### 测试2: 质疑变成人身攻击
- 输入:
```
【确认偏误分析请求】
核心观点：CEO的决策是错误的
反对声音：CEO有20年行业经验
```
- 期望行为: 质疑决策本身而非质疑人，遵循"对事不对人"原则
- 验证项:
  - [ ] 质疑聚焦在决策逻辑上
  - [ ] 不使用"你太蠢了"类人身攻击

### 测试3: 过度自信调整
- 输入:
```
【确认偏误分析请求】
核心观点：我有80%把握这个决策是对的
支持证据：逻辑推演完整
背景：以往预测准确率40%
```
- 期望行为: 基于历史准确率调整置信度，不只依赖当前逻辑
- 验证项:
  - [ ] 历史准确率被纳入校准
  - [ ] 80%被下调到合理水平
