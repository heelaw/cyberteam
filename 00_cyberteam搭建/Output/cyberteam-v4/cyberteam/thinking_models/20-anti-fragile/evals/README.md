# 反脆弱决策专家测试用例

## 正面测试

### 测试1: 业务连续性
- 输入:
```
【反脆弱请求】
系统：电商平台
脆弱点：单点服务器故障
暴露程度：高
背景：曾经历多次宕机
```
- 期望输出:
  - 脆弱性分析
  - 杠铃策略配置
  - 黑天鹅预案
- 验证项:
  - [ ] 识别核心脆弱点
  - [ ] 杠铃策略具体
  - [ ] 有冗余设计

### 测试2: 投资组合
- 输入:
```
【反脆弱请求】
系统：投资组合
脆弱点：市场大幅下跌
暴露程度：极高
背景：All-in单一资产
```
- 期望输出:
  - 杠铃配置方案
  - 风险对冲
  - 极端情景预案
- 验证项:
  - [ ] 配置包含高风险/低风险
  - [ ] 有可选性
  - [ ] 不依赖预测

## 负面测试

### 测试1: 伪反脆弱
- 输入:
```
【反脆弱请求】
系统：银行存款
脆弱点：收益低
背景：追求稳定
```
- 期望行为: 识别表面稳定实为脆弱，不将保守当反脆弱
- 验证项:
  - [ ] 识别通胀脆弱性
  - [ ] 指出隐性风险
  - [ ] 建议适度暴露

### 测试2: 过度工程
- 输入:
```
【反脆弱请求】
系统：个人时间管理
脆弱点：计划赶不上变化
背景：追求完美
```
- 期望行为: 避免过度复杂化，保持简单有效的反脆弱设计
- 验证项:
  - [ ] 不过度复杂
  - [ ] 保持可选择性
  - [ ] 成本可控
