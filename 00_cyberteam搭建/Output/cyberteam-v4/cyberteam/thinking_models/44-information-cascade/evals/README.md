# 信息级联专家测试用例

## 正面测试

### 测试1: 投资跟风
- 输入:
```
【信息级联分析请求】
现象：大家都在买某只股票
群体行为：朋友圈都在讨论，说涨到100没问题
你的判断：我不太确定，但大家都买应该没错
专家意见：各大券商都给出买入评级
```
- 期望输出:
  - 级联三要素诊断
  - 私有信息分析
  - 独立判断检验
- 验证项:
  - [ ] 能识别这是信息级联的典型场景
  - [ ] 能明确用户的私有信息
  - [ ] 有"如果不知道他人选择"的测试

### 测试2: 专家共识
- 输入:
```
【信息级联分析请求】
现象：所有专家都说应该这样做
群体行为：行业会议上大家都同意这个方案
你的判断：我有自己的疑虑
专家意见：顶级顾问都背书了
```
- 期望输出:
  - 级联风险评估
  - 独立思考清单
  - 决策建议
- 验证项:
  - [ ] 能质疑专家共识可能是级联结果
  - [ ] 能评估用户的私有信息
  - [ ] 能给出"跟随/坚持/等待"的选项

## 负面测试

### 测试1: 盲目跟随
- 输入:
```
【信息级联分析请求】
现象：电商都在做双11促销
群体行为：不参加双11就会被淘汰
你的判断：确实应该跟随
专家意见：（没有独立专家意见）
```
- 期望行为: 不能因为"大家都在做"就跟随
- 验证项:
  - [ ] 必须有独立判断步骤
  - [ ] 必须明确用户的私有信息
  - [ ] 必须评估群体选择是否有合理依据

### 测试2: 从众安全
- 输入:
```
【信息级联分析请求】
现象：所有人都选择A方案
群体行为：没人提出B方案
你的判断：既然所有人都在选A，我也选A
专家意见：（没有）
```
- 期望行为: 不能认为跟随是"安全"的选择
- 验证项:
  - [ ] 必须质疑所有人选A不代表A正确
  - [ ] 必须鼓励用户表达异见
  - [ ] 必须说明"异见"可能有价值
