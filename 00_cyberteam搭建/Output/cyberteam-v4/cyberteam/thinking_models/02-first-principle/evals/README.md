# 第一性原理专家测试用例

## 正面测试

### 测试1: 商业创新
- 输入:
```
【第一性原理分析请求】
目标：颠覆某个传统行业
现状：大家都用传统方法
最终愿景：开创全新的解决方案
约束：资源有限
```
- 期望输出:
  - 假设挑战（质疑"一直这么做"）
  - 第一性原理（物理事实）
  - 重构方案（新思路）
  - 多米诺骨牌（24小时可启动的行动）
- 验证项:
  - [ ] 挑战了类比思维
  - [ ] 基于物理事实重构
  - [ ] 有具体行动

### 测试2: 个人发展
- 输入:
```
【第一性原理分析请求】
目标：快速提升能力
现状：参加各种培训
最终愿景：成为专家
约束：时间有限
```
- 期望输出:
  - 假设挑战
  - 第一性原理
  - 重构方案
  - 多米诺骨牌
- 验证项:
  - [ ] 质疑"培训=学习"的假设
  - [ ] 找到能力提升的本质
  - [ ] 行动具体可执行

## 负面测试

### 测试1: 使用类比推理
- 输入:
```
【第一性原理分析请求】
目标：改进产品
现状：参考竞品
```
- 期望行为: 不能用"就像...或类似于..."，必须基于物理事实
- 验证项:
  - [ ] 挑战了类比思维
  - [ ] 不是简单的借鉴
  - [ ] 重构基于事实

### 测试2: 替用户思考
- 输入:
```
【第一性原理分析请求】
目标：解决某个复杂问题
现状：用户自己也没有想清楚
```
- 期望行为: 不能替用户给答案，要用助产士模式引导
- 验证项:
  - [ ] 用提问引导
  - [ ] 洞见由用户自己产出
  - [ ] 不是直接给方案
