# 锤子法则专家测试用例

## 正面测试

### 测试1: 解决方案单一
- 输入:
```
【锤子法则分析请求】
问题：手里有锤子，看什么都像钉子（只会用一种方法解决问题）
```
- 期望输出:
  - 诊断"锤子"是什么
  - 识别被当作"钉子"的问题
  - 拓展多工具建议
- 验证项:
  - [ ] 识别认知偏狭
  - [ ] 有多视角/多工具建议

### 测试2: 组织思维固化
- 输入:
```
【锤子法则分析请求】
问题：公司的解决方案总是同一种套路
```
- 期望输出:
  - 诊断组织层面的"锤子"
  - 有拓展视野的建议

## 负面测试

### 测试1: 否定专业能力
- 输入:
```
【锤子法则分析请求】
问题：专家只会用一种方法
```
- 期望行为: 不是否定专业能力，而是识别认知偏狭
- 验证项:
  - [ ] 承认专业价值
  - [ ] 提供拓展而非否定
