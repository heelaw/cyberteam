# 5Why分析专家评估标准

## 评估维度

| 维度 | 权重 | 评分标准 |
|------|------|----------|
| 完整性 | 20% | 章节是否齐全（触发词、输入格式、分析框架、输出格式、Critical Rules） |
| 准确性 | 30% | 5Why是否正确（因果链追溯、根本原因验证、所以测试） |
| 可执行性 | 30% | 输出是否可操作（因果链分析、根本对策、预防机制是否明确） |
| 独特性 | 20% | 是否有差异化（与一般问题分析的区分、5Why溯源独特价值） |

## 及格线

- 标准模式: >= 85分
- 严格模式: >= 95分

## 快速检验清单

- [ ] 有触发词（5个场景以上）
- [ ] 有输入格式（初始问题、背景、已知原因）
- [ ] 有输出格式（因果链分析、根本原因验证、根本对策）
- [ ] 有分析框架（四阶段：问题界定→追问→验证→对策）
- [ ] 有Critical Rules（必须遵守5条、禁止行为5条）
- [ ] 有所以测试验证

## 核心检验点

### 理论准确性
- [ ] 追问严守逻辑链
- [ ] 所以测试验证
- [ ] 对事不对人

### 框架完整性
- [ ] 因果链完整
- [ ] 追问到系统层面
- [ ] 根本对策有层级

### 输出有效性
- [ ] 根本原因可验证
- [ ] 对策按优先级
- [ ] 预防机制具体
