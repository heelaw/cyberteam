# 5Why分析专家测试用例

## 正面测试

### 测试1: 质量问题分析
- 输入:
```
【5Why分析请求】
初始问题：产品次品率从1%上升到5%
背景：上周更换了原材料供应商
已知原因：无
```
- 期望输出:
  - 因果链（问题→原因→原因→...→根本原因）
  - 根本原因验证（所以测试）
  - 根本对策（应急/改进/预防）
- 验证项:
  - [ ] 因果链完整
  - [ ] 所以测试验证
  - [ ] 对策有层级

### 测试2: 服务问题分析
- 输入:
```
【5Why分析请求】
初始问题：客户投诉增加30%
背景：本季度推出新产品
已知原因：可能是产品问题
```
- 期望输出:
  - 因果链分析
  - 根本原因
  - 对策建议
- 验证项:
  - [ ] 追问到系统层面
  - [ ] 不是归咎于人
  - [ ] 验证逻辑正确

## 负面测试

### 测试1: 归咎于人
- 输入:
```
【5Why分析请求】
初始问题：项目延期
背景：团队成员问题
```
- 期望行为: 不能归咎于人，要从流程和系统角度分析
- 验证项:
  - [ ] 对事不对人
  - [ ] 追问到系统层面
  - [ ] 不是"员工不认真"

### 测试2: 停在表面
- 输入:
```
【5Why分析请求】
初始问题：销售额下降
背景：市场竞争激烈
```
- 期望行为: 不能接受第一个原因就结束，要追问到根本
- 验证项:
  - [ ] 至少追问5层
  - [ ] 达到系统层面
  - [ ] 表面原因不可接受
