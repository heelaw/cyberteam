# 二八定律专家测试用例

## 正面测试

### 测试1: 客户价值分析

- **输入**:
```
【二八定律分析请求】

分析对象：电商平台的客户群体
数据范围：平台有10万注册用户，年GMV 1亿元
关注重点：如何优化营销资源分配
```
- **期望输出**:
  - 识别头部客户（可能2万用户贡献8000万GMV）
  - 贡献分布表
  - 杠杆指数分析
  - 长尾客户评估（是否值得投入）
- **验证项**:
  - [ ] 识别出关键20%客户
  - [ ] 给出具体的贡献分布（头部/颈部/尾部）
  - [ ] 计算杠杆指数
  - [ ] 长尾评估有明确建议
  - [ ] 有聚焦策略和动态策略

---

### 测试2: 产品组合优化

- **输入**:
```
【二八定律分析请求】

分析对象：SaaS产品功能模块
数据范围：产品有50个功能模块
约束条件：研发资源有限，只能优先开发部分功能
```
- **期望输出**:
  - 识别贡献80%用户价值的20%功能
  - 功能优先级排序
  - 效率分析（杠杆指数）
  - 长尾功能评估
- **验证项**:
  - [ ] 功能有优先级分类
  - [ ] 识别关键功能
  - [ ] 有资源分配建议
  - [ ] 动态策略：什么条件下调整

---

### 测试3: 供应链关键节点

- **输入**:
```
【二八定律分析请求】

分析对象：供应链风险管理
数据范围：100个供应商，80%采购额来自15个供应商
核心目标：识别需要重点管理的供应商
```
- **期望输出**:
  - 识别关键供应商（头部）
  - 贡献分布分析
  - 风险杠杆点定位
  - 供应商管理策略
- **验证项**:
  - [ ] 供应商分级清晰
  - [ ] 关键供应商明确
  - [ ] 有风险管理建议
  - [ ] 避免将所有供应商同等管理

## 负面测试

### 测试1: 机械80/20套用

- **输入**:
```
【二八定律分析请求】

分析对象：企业营销预算分配
当前状态：营销预算1000万，按80/20分配给头部渠道
约束条件：有人建议必须严格按照80/20执行
```
- **期望行为**: 输出应指出80/20只是经验法则，实际比例可能是70/30或90/10，需要根据实际情况调整
- **验证项**:
  - [ ] 指出80/20是参考而非固定比例
  - [ ] 讨论实际比例可能偏离的原因
  - [ ] 建议根据数据动态确定比例

---

### 测试2: 忽视长尾战略价值

- **输入**:
```
【二八定律分析请求】

分析对象：电商平台产品品类
分析结论：20%品类贡献80%销售额，建议砍掉长尾品类
```
- **期望行为**: 输出应指出长尾可能有战略价值（品牌完整性、用户粘性等），不应简单砍掉
- **验证项**:
  - [ ] 评估长尾品类的战略价值
  - [ ] 区分绝对价值和战略价值
  - [ ] 给出"条件投入"而非简单二分

---

### 测试3: 相关性当因果

- **输入**:
```
【二八定律分析请求】

分析对象：产品销量与某个功能的关系
发现：购买该产品的用户80%都使用了某个功能
结论：该功能是销售的"头部原因"
```
- **期望行为**: 输出应指出这可能是相关性而非因果性，该功能可能是结果而非原因
- **验证项**:
  - [ ] 质疑因果推断的合理性
  - [ ] 区分相关性和因果性
  - [ ] 建议进一步验证因果关系

## 边界测试

### 测试1: 均匀分布情况

- **输入**:
```
【二八定律分析请求】

分析对象：客服问题类型分布
数据：100种问题类型，没有明显集中，每种类型贡献1%左右
```
- **期望行为**: 识别分布可能不是二八分布，需要调整策略而非强行套用二八定律
- **验证项**:
  - [ ] 识别这不是典型的二八分布
  - [ ] 不强行套用二八定律
  - [ ] 给出均匀分布下的资源分配建议

---

### 测试2: 动态边界识别

- **输入**:
```
【二八定律分析请求】

分析对象：社交媒体内容分发
当前状态：头部账号贡献80%互动
问题：头部效应越来越强，是否应该调整策略
```
- **期望行为**: 分析头部和长尾边界可能随时间变化，提供动态调整机制
- **验证项**:
  - [ ] 讨论边界动态变化
  - [ ] 设计触发调整的条件
  - [ ] 提供动态策略

---

### 测试3: 多维度分析

- **输入**:
```
【二八定律分析请求】

分析对象：企业客户
维度A：按收入，20%客户贡献80%收入
维度B：按利润，30%客户贡献80%利润
维度C：按增长潜力，15%客户贡献80%未来增长
```
- **期望行为**: 需要综合多维度分析，不能只看单一维度
- **验证项**:
  - [ ] 展示多维度分析
  - [ ] 讨论维度冲突时的权衡
  - [ ] 提供加权评估方法
