# 推论阶梯专家测试用例

## 正面测试

### 测试1: 商业决策分析
- 输入:
```
【推论阶梯分析请求】
结论/判断：这个供应商不可靠，需要更换
原始数据：上季度交付有2次延迟
背景：年度供应商评估
```
- 期望输出:
  - 七层推理还原
  - 假设检验
  - 数据缺口
- 验证项:
  - [ ] 有完整的推理层次追溯
  - [ ] 有假设识别和检验
  - [ ] 有数据缺口分析

### 测试2: 团队分歧分析
- 输入:
```
【推论阶梯分析请求】
结论/判断：市场策略应该以线上为主
原始数据：线上渠道增长30%，线下持平
背景：战略讨论会议
```
- 期望输出:
  - 选择性观察分析
  - 假设识别
  - 替代解释
- 验证项:
  - [ ] 能追溯从数据到结论的跳跃
  - [ ] 能识别隐藏假设
  - [ ] 能给出修正建议

## 负面测试

### 测试1: 跳过层次
- 输入:
```
【推论阶梯分析请求】
结论/判断：他不适合做领导
原始数据：他在会议上发言不多
背景：晋升讨论
```
- 期望行为: 不能跳过推理层次
- 验证项:
  - [ ] 必须追溯每一步推理
  - [ ] 不能只看结论
  - [ ] 必须识别选择性观察

### 测试2: 接受未经检验的假设
- 输入:
```
【推论阶梯分析请求】
结论/判断：用户不会为此付费
原始数据：竞品也没做这个功能
背景：产品规划
```
- 期望行为: 不能接受未经检验的假设
- 验证项:
  - [ ] 必须检验假设"竞品没做=用户不需要"
  - [ ] 必须识别数据缺口
  - [ ] 必须质疑结论的推导过程
