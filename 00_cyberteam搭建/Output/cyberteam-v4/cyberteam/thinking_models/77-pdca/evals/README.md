# PDCA循环专家测试用例

## 正面测试

### 测试1: 流程优化
- 输入:
```
【PDCA循环分析请求】
问题/目标：优化客服响应时间，从平均4小时降至1小时
当前状态：客服团队10人，使用旧系统
改进意愿：中等改进
```
- 期望输出:
  - P阶段：目标SMART分析、原因分析、行动计划
  - D阶段：执行记录、数据收集计划
  - C阶段：效果评估、偏差分析
  - A阶段：标准化、持续改进
- 验证项:
  - [ ] 四阶段完整闭环
  - [ ] 有量化的目标值
  - [ ] 有根因分析

### 测试2: 质量问题改进
- 输入:
```
【PDCA循环分析请求】
问题/目标：产品不良率从5%降至2%
当前状态：生产线3条，质量检测靠人工
改进意愿：大改进
```
- 期望输出:
  - 有SMART目标
  - 有根因追溯（5Why或其他）
  - 有数据驱动的效果评估

## 负面测试

### 测试1: 跳步
- 输入:
```
【PDCA循环分析请求】
问题/目标：立即执行一个新方案
```
- 期望行为: 不能跳过Plan阶段
- 验证项:
  - [ ] 必须有计划阶段
  - [ ] 不能直接执行

### 测试2: 无数据
- 输入:
```
【PDCA循环分析请求】
问题/目标：感觉流程有改进
```
- 期望行为: 不能凭感觉说"有改进"
- 验证项:
  - [ ] 必须有量化指标
  - [ ] 必须有对比数据
