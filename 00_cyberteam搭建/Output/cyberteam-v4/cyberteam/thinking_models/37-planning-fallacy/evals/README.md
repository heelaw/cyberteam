# 规划谬误专家测试用例

## 正面测试

### 测试1: 项目时间估算
- 输入:
```
【规划谬误分析请求】
计划/项目：开发一个电商网站
初步估算：2个月完成
时间/预算：20万预算
类似经验：没有做过完整电商项目
```
- 期望输出:
  - 内部视角估算详细分解
  - 外部基准数据对比
  - 带缓冲的最终估算
- 验证项:
  - [ ] 有内部视角的任务分解
  - [ ] 有外部基准数据查询
  - [ ] 有1.5-2.0倍的缓冲建议
  - [ ] 有乐观/最可能/悲观三档估算

### 测试2: 成本超支分析
- 输入:
```
【规划谬误分析请求】
计划/项目：办公室装修
初步估算：50万
时间/预算：3个月
类似经验：以前做过家庭装修
```
- 期望输出:
  - 识别乐观偏差来源
  - 风险缓冲建议
  - 里程碑设置
- 验证项:
  - [ ] 能识别"家庭装修经验"的偏差影响
  - [ ] 有风险金分配建议
  - [ ] 有分阶段里程碑

## 负面测试

### 测试1: 只看内部视角
- 输入:
```
【规划谬误分析请求】
计划/项目：开发APP
初步估算：按任务分解需要3个月
时间/预算：30万
类似经验：我很熟悉开发流程
```
- 期望行为: 不能只基于内部视角做估算
- 验证项:
  - [ ] 必须有外部基准数据查询
  - [ ] 必须有偏差识别步骤

### 测试2: 零缓冲承诺
- 输入:
```
【规划谬误分析请求】
计划/项目：完成年度报告
初步估算：1周
时间/预算：1万
类似经验：做过类似报告
```
- 期望行为: 不能不做缓冲就承诺
- 验证项:
  - [ ] 必须加入缓冲系数
  - [ ] 必须区分对内估算和对外承诺
