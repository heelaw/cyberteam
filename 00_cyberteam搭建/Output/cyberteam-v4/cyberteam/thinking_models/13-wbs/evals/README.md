# WBS任务分解专家测试用例

## 正面测试

### 测试1: 项目启动
- 输入:
```
【WBS任务分解请求】
目标：发布新产品V2.0
时间限制：3个月
资源限制：团队5人
参与者：产品、技术、测试
```
- 期望输出:
  - WBS分解结构
  - 交付物清单
  - 任务清单（估算、责任人）
  - 甘特图
  - 风险与依赖
- 验证项:
  - [ ] 100%分解
  - [ ] 任务≤80小时
  - [ ] 责任到人
  - [ ] 有时间估算

### 测试2: 年度计划分解
- 输入:
```
【WBS任务分解请求】
目标：实现年营收翻倍
时间限制：12个月
资源限制：预算有限
参与者：全公司
```
- 期望输出:
  - 目标分解
  - 关键交付物
  - 季度任务
  - 里程碑
- 验证项:
  - [ ] 分解到可执行
  - [ ] 识别关键里程碑
  - [ ] 有风险识别

## 负面测试

### 测试1: 模糊分解
- 输入:
```
【WBS任务分解请求】
目标：做好市场推广
```
- 期望行为: 不能分解成"做XXX的一部分"
- 验证项:
  - [ ] 每个任务有交付物
  - [ ] 不能模糊
  - [ ] 分解到可执行级别

### 测试2: 无估算
- 输入:
```
【WBS任务分解请求】
目标：完成某项目
```
- 期望行为: 必须有时间估算
- 验证项:
  - [ ] 每个任务有估算
  - [ ] 有乐观/悲观估计
  - [ ] 识别依赖关系
