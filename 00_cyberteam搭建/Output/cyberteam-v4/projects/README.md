# projects - 项目工作目录

> **核心原则**: 每个项目 = 一个独立文件夹。所有Agent产出必须集中存放，禁止散落在各Agent目录下。

---

## 目录结构

```
projects/
├── _template/                   # 项目模板（新项目从此复制）
│   ├── 00_项目资料/              # 项目元数据
│   │   └── metadata.yaml         # 项目信息模板
│   ├── 01_Agent会议纪要/          # Agent讨论与决策
│   │   ├── 总监决策/             # 总监的判断和决策
│   │   │   └── 决策模板.md
│   │   ├── 专家讨论/             # 各专家的分析和讨论
│   │   │   └── 专家分析模板.md
│   │   └── 对话记录/             # Agent间的对话日志
│   ├── 02_计划方案/              # 计划与方案
│   │   ├── 任务分解/             # 工作任务分解
│   │   ├── 执行计划/             # 执行步骤
│   │   └── 方案选项/             # 方案A/B/C
│   └── 03_最终输出/              # 最终产出
│       ├── 文案产出/             # 所有文案物料
│       ├── 分析报告/             # 市场/用户分析
│       └── 汇总方案/             # 完整方案文档
│
└── {项目名称中文}/              # 具体项目（每次任务创建）
    ├── 00_项目资料/              # 项目元数据
    ├── 01_Agent会议纪要/          # Agent讨论与决策
    ├── 02_计划方案/              # 计划与方案
    └── 03_最终输出/              # 最终产出
```

## 使用流程

### 1. 创建新项目

```bash
cp -r projects/_template projects/{项目名称}
```

### 2. 初始化项目元数据

编辑 `projects/{项目名称}/00_项目资料/metadata.yaml`：
- project_id: proj_YYYYMMDD_XXX
- project_name: 项目名称
- project_type: 类型
- participants: 参与Agent

### 3. 引导用户提供业务数据

使用模板引导用户提供：
- 产品信息
- 当前业务数据
- 痛点问题
- 约束条件

### 4. Agent读取后开始工作

Agent必须读取以下文件后才开始决策：
- `00_项目资料/metadata.yaml`
- `00_项目资料/business_context.md`

### 5. 产出存入规范位置

| 产出类型 | 存入位置 |
|----------|----------|
| 总监决策 | `01_Agent会议纪要/总监决策/` |
| 专家分析 | `01_Agent会议纪要/专家讨论/` |
| Agent对话 | `01_Agent会议纪要/对话记录/` |
| 任务分解 | `02_计划方案/任务分解/` |
| 执行计划 | `02_计划方案/执行计划/` |
| 方案选项 | `02_计划方案/方案选项/` |
| 文案产出 | `03_最终输出/文案产出/` |
| 分析报告 | `03_最终输出/分析报告/` |
| 汇总方案 | `03_最终输出/汇总方案/` |

### 6. 项目归档

重要决策和方案 → 复制到 `shared/knowledge_base/`

---

## ⚠️ 强制规则

1. **禁止散落**: 所有项目产出必须存入项目文件夹，禁止存入各Agent目录
2. **禁止重复**: 同一产出不重复存放多个副本
3. **版本控制**: 方案迭代需标注版本号(v1.0/v2.0)
4. **命名规范**: 所有文件使用中文命名
