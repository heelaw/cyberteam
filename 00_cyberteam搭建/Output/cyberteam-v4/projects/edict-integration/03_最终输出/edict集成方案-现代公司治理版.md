## 二、状态机设计

### 2.1 核心状态定义

| 状态 | 代码 | 职责 | 负责角色 |
|------|------|------|----------|
| **Intake** | `INTAKE` | 任务接收、分拣、优先级初判 | Intake Officer |
| **Draft** | `DRAFT` | 方案起草、设计、文档编写 | Solution Architect |
| **Review** | `REVIEW` | 方案评审、风险评估、审批 | Review Committee |
| **Dispatch** | `DISPATCH` | 任务派发、部门分配 | Dispatch Manager |
| **Execute** | `EXECUTE` | 部门执行、落地实施 | Department Agent |
| **QA** | `QA` | 质量审查、验收测试 | QA Engineer |
| **Done** | `DONE` | 完成归档 | System |
| **Blocked** | `BLOCKED` | 阻塞、等待资源 | 各角色上报 |
| **Cancelled** | `CANCELLED` | 取消 | CEO/COO |

### 2.2 状态流转规则

```
INTAKE → DRAFT → REVIEW → DISPATCH → EXECUTE → QA → DONE
   ↓         ↓         ↓          ↓          ↓
CANCELLED  BLOCKED   BLOCKED   BLOCKED    BLOCKED
                                  ↓
                               CANCELLED
```

### 2.3 流转合法性矩阵

| 当前状态 | 合法下一状态 |
|----------|--------------|
| INTAKE | DRAFT, CANCELLED |
| DRAFT | REVIEW, CANCELLED, BLOCKED |
| REVIEW | DISPATCH, DRAFT, CANCELLED |
| DISPATCH | EXECUTE, NEXT, CANCELLED, BLOCKED |
| NEXT | EXECUTE, CANCELLED |
| EXECUTE | QA, DONE, BLOCKED, CANCELLED |
| QA | DONE, EXECUTE, CANCELLED |
| BLOCKED | INTAKE, DRAFT, REVIEW, DISPATCH, EXECUTE |

## 四、与 CyberTeam v4 的集成

### 4.1 集成点

```
edict 状态机 (task_service.py)
         ↓
    CyberTeam v4 执行引擎
         ↓
┌────────────────────────────────────────────────────┐
│  Intake Agent   →  任务接收 (CEO派发)              │
│  Draft Agent    →  方案起草 (COO组织)             │
│  Review Agent   →  评审审议 (专家团队)             │
│  Dispatch Agent  →  调度派发 (COO执行)              │
│  [部门] Agent   →  六部执行 (执行层)              │
│  QA Agent       →  质量审查 (评审委员会)           │
└────────────────────────────────────────────────────┘
```

### 4.2 数据模型映射

| edict 字段 | CyberTeam v4 映射 | 说明 |
|------------|-------------------|------|
| `state` | `phase` | 执行阶段 |
| `org` | `department` | 执行部门 |
| `official` | `owner` | 责任人 |
| `now` | `progress` | 当前进展 |
| `block` | `blocker` | 阻塞原因 |
| `output` | `deliverable` | 最终产出 |
| `ac` | `acceptance_criteria` | 验收标准 |
| `target_dept` | `assigned_dept` | 目标部门 |
