## Skill描述（PUSHY）

邀请嘉宾分享，但现场冷场、没人互动、分享后没转化？

问题根源：缺乏系统的分享流程！

群内分享六步流程是一套从"预热→分享→互动→转化"的完整流程，让每一场分享都有价值、有互动、有结果。

## Critical Rules（必须遵守）

1. **提前预热**: 分享前3天开始预热，让用户知道要来
2. **明确价值**: 分享前要让用户知道能得到什么
3. **互动设计**: 分享中要有互动环节，不能只是单向输出
4. **即时答疑**: 分享后要有答疑环节
5. **内容归档**: 分享后要有文字/录音沉淀
6. **跟进转化**: 分享后要有后续跟进

---

## Workflow Steps（标准化步骤）

### Step 1: 分享前预热（提前3天）

**输入**: 分享主题、嘉宾信息、目标受众

**处理**:
1. 撰写预热文案
2. 发布预告通知
3. 收集用户问题
4. 确认嘉宾准备

**输出**: 预热文案 + 报名名单

**成功标准**:
- 预热文案包含分享主题、嘉宾、价值、时间
- 报名人数符合预期（>20人）
- 用户问题收集到位

---

### Step 2: 分享前提醒（分享当天）

**输入**: 预热文案、报名名单

**处理**:
1. 上午发送提醒
2. 分享前1小时再次提醒
3. 确认嘉宾到位
4. 准备互动问题

**输出**: 提醒发送记录

**成功标准**:
- 至少2次提醒
- 嘉宾确认到位
- 互动问题准备完毕

---

### Step 3: 分享进行中（执行阶段）

**输入**: 嘉宾、分享主题、互动问题

**处理**:
1. 开场介绍嘉宾
2. 嘉宾按大纲分享
3. 穿插互动环节
4. 答疑环节

**输出**: 分享执行记录

**成功标准**:
- 分享时长控制在计划内
- 至少有2次互动
- 用户反馈积极

---

### Step 4: 分享后沉淀（分享结束后）

**输入**: 分享录音/内容、用户反馈

**处理**:
1. 整理录音/文字内容
2. 发布精华摘要
3. 收集用户反馈
4. 表扬参与者和嘉宾

**输出**: 沉淀内容 + 反馈收集

**成功标准**:
- 24小时内发布精华摘要
- 用户反馈收集 >50%
- 沉淀内容完整

---

### Step 5: 转化跟进（分享后3天内）

**输入**: 参与者名单、沉淀内容

**处理**:
1. 发送完整资料给参与者
2. 跟进有需求的用户
3. 引导下一步行动
4. 邀请反馈

**输出**: 转化跟进记录

**成功标准**:
- 资料发送给所有参与者
- 有需求的用户得到跟进
- 有下一步行动指引

---

### Step 6: 效果复盘（分享后7天内）

**输入**: 全流程数据、反馈

**处理**:
1. 统计分享数据
2. 分析反馈意见
3. 提炼改进点
4. 更新流程模板

**输出**: 复盘报告

**成功标准**:
- 有完整的数据统计
- 有具体的改进建议
- 流程得到优化

## 六步执行模板

### Step 1: 预热文案模板

```
🎙️ 分享预告

【分享主题】
[主题]

【分享嘉宾】
[姓名] | [职位/Title]
[一句简介]

【你将获得】
• [价值点1]
• [价值点2]
• [价值点3]

【分享时间】
[日期] [时间]
预计时长：[X]分钟

【报名方式】
回复"报名"+你最想问的一个问题

👥 已报名：[X]人

⚠️ 名额有限，先到先得
```

## 嘉宾沟通指南

### 邀请嘉宾话术

```
[嘉宾名字]，您好！

我是[社群名称]的管理员[名字]。

我们社群是一个[社群定位]的社群，
目前有[X]位成员，主要是[目标用户]。

看到您在[领域]有很多实战经验，
想邀请您来群里做一次分享。

【分享主题】
[初步主题]

【分享形式】
• 时长：[X]分钟分享 + [X]分钟答疑
• 形式：语音/文字/录屏
• 时间：您方便的时间

【社群支持】
• 我们会提前预热宣传
• 整理分享内容沉淀
• 给予嘉宾适当曝光

有意向的话，我们可以详细聊聊分享主题~

期待您的回复！
```

### 嘉宾注意事项

| 注意事项 | 说明 |
|---------|------|
| 时间把控 | 提前告知分享时长，建议留出答疑时间 |
| 内容形式 | 语音方便录音沉淀，文字方便转发 |
| 互动设计 | 建议准备2-3个互动点 |
| 资料提供 | 提前确认能提供的资料 |
| 后续合作 | 确认是否有后续合作意向 |
