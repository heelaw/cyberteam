# LTV基础概念

## 1. 什么是LTV？

### 定义
**LTV（Life Time Value，用户生命周期价值）** 是指一个用户在整个生命周期内为产品贡献的总价值。

### 核心含义
- **全生命周期**: 从用户首次使用到最后一次使用的整个时间段
- **价值总和**: 用户带来的所有收入（可能是多次交易、订阅、广告等）
- **净价值**: 通常指净利润，也可指收入（需明确区分）

### 为什么LTV重要？
1. **获客决策**: LTV/CAC > 3:1 才值得大规模获客
2. **产品优先级**: LTV高的功能应优先开发
3. **定价策略**: 会员定价通常参考LTV
4. **投资估值**: LTV是估算公司估值的重要指标

## 2. LTV计算公式

### 2.1 简单LTV公式（Simple LTV）

**适用场景**: 流失率稳定、产品成熟的SaaS或订阅产品

```
LTV = ARPU × (1 / 月流失率)
    = ARPU × 用户平均生命周期（月）
```

**示例**:
- ARPU = ¥50/月
- 月流失率 = 5%
- LTV = 50 × (1/0.05) = 50 × 20 = ¥1000

### 2.2 完整LTV公式（考虑时间价值）

**适用场景**: 需要考虑货币时间价值的产品

```
LTV = Σ [ARPU_t × (1+discount)^-t × 存活率_t] - 用户获取成本
```

其中:
- ARPU_t: 第t个月的平均用户收入
- discount: 折现率（通常用5-10%）
- 存活率_t: 第t个月用户仍然活跃的概率

### 2.3 Cohort LTV公式

**适用场景**: 电商、零售等有明确复购周期的产品

```
LTV = Σ (Cohort各月收入 × 该月存活用户数) / 初始用户数
```

**示例（电商Cohort分析）**:
| 月份 | 收入 | 存活用户 | 存活率 |
|------|------|----------|--------|
| 第1月 | ¥100 | 1000 | 100% |
| 第2月 | ¥80 | 700 | 70% |
| 第3月 | ¥60 | 500 | 50% |
| 第4月 | ¥50 | 400 | 40% |

LTV = (100+80+60+50) / 1 = ¥290

### 2.4 Marginal LTV公式

**适用场景**: 游戏、增值服务等付费深度差异大的产品

```
LTV = Σ (每层用户付费额 × 该层用户占比)
```

**示例（游戏付费分层）**:
| 付费层 | 付费金额 | 用户占比 |
|--------|----------|----------|
| 免费用户 | ¥0 | 70% |
| 小R | ¥50 | 20% |
| 中R | ¥500 | 8% |
| 大R | ¥5000 | 2% |

LTV = 0×0.7 + 50×0.2 + 500×0.08 + 5000×0.02 = ¥10 + ¥40 + ¥100 = ¥150

## 3. LTV的类型

### 3.1 按价值类型分类

| 类型 | 定义 | 使用场景 |
|------|------|----------|
| **收入LTV** | 用户贡献的总收入 | 评估市场规模 |
| **利润LTV** | 用户贡献的净利润 | 评估真实收益 |
| **贡献LTV** | 收入 - 变动成本 | 评估边际贡献 |

### 3.2 按时间维度分类

| 类型 | 定义 | 使用场景 |
|------|------|----------|
| **历史LTV** | 基于历史数据计算 | 验证过去决策 |
| **预测LTV** | 基于模型预测未来 | 指导未来决策 |

### 3.3 按计算精度分类

| 类型 | 定义 | 精度 |
|------|------|------|
| **简单LTV** | ARPU/流失率 | 粗略估计 |
| **Cohort LTV** | 按Cohort分析 | 较准确 |
| **预测LTV** | 机器学习模型 | 精确预测 |

## 4. 核心数据指标

### 4.1 ARPU（Average Revenue Per User）
- **定义**: 平均每用户收入
- **计算**: 总收入 / 用户数
- **注意**: 需要明确时间周期（月/季/年）

### 4.2 流失率（Churn Rate）
- **定义**: 在特定时间段内流失的用户比例
- **计算**: 流失用户数 / 期初用户数
- **分类**:
  - 收入流失率（Revenue Churn）
  - 用户流失率（Customer Churn）

### 4.3 复购率（Repurchase Rate）
- **定义**: 发生重复购买的用户比例
- **计算**: 复购用户数 / 总购买用户数
- **电商常用**: 30天复购率、90天复购率

### 4.4 客单价（AOV, Average Order Value）
- **定义**: 平均订单金额
- **计算**: 总销售额 / 订单数

### 4.5 用户生命周期
- **定义**: 用户从首次使用到流失的平均时间
- **计算**: 1 / 月流失率（简单估算）
- **注意**: 不同产品差异巨大（游戏1-3月，SaaS 2-5年）

### 4.6 CAC（Customer Acquisition Cost）
- **定义**: 单用户获取成本
- **计算**: 总获客费用 / 新增用户数

## 5. LTV与CAC的关系

### 5.1 核心比值：LTV/CAC

| LTV/CAC比值 | 含义 | 建议 |
|-------------|------|------|
| > 5:1 | 优秀 | 可大幅增加获客投入 |
| 3:1 ~ 5:1 | 健康 | 可稳步获客 |
| 1:1 ~ 3:1 | 警告 | 需优化获客效率 |
| < 1:1 | 危险 | 商业模式不成立 |

### 5.2 回本周期
- **定义**: CAC通过用户收入回收的时间
- **计算**: CAC / 月ARPU
- **健康标准**: 回本周期 < 12个月

### 5.3 常见误区

| 误区 | 正确做法 |
|------|----------|
| 只看收入不看成本 | 使用利润LTV |
| 忽略流失率 | 准确计算生命周期 |
| 不考虑时间价值 | 使用折现率 |
| 静态计算 | 定期更新，进行敏感性分析 |

## 6. 不同产品类型的LTV特点

### 6.1 SaaS产品
- **特征**: 订阅制、月/年会费
- **关键指标**: 流失率、续费率、扩展收入
- **典型LTV**: 12-36个月订阅费

### 6.2 电商产品
- **特征**: 多次购买、复购重要
- **关键指标**: 复购率、客单价、购买频次
- **典型LTV**: 3-12个月购买额

### 6.3 游戏产品
- **特征**: 少数用户贡献大部分收入（2/8法则）
- **关键指标**: 付费率、ARPPU、付费深度
- **典型LTV**: 首充+持续付费总和

### 6.4 订阅产品
- **特征**: 连续订阅、自动续费
- **关键指标**: 续费率、订阅时长、取消率
- **典型LTV**: 平均订阅周期 × 月费

## 7. 数据来源与验证

### 7.1 常用数据来源

| 数据类型 | 数据来源 | 可靠性 |
|----------|----------|--------|
| 财务数据 | 财务系统 | 高 |
| 交易数据 | 交易数据库 | 高 |
| 用户行为 | BI系统 | 中 |
| 投放数据 | 广告平台 | 中 |

### 7.2 数据验证要点
- **时间范围一致**: 所有数据使用相同时间段
- **口径一致**: 用户定义、收入定义保持一致
- **交叉验证**: 用多个数据源相互验证
- **异常值处理**: 识别并处理异常数据
