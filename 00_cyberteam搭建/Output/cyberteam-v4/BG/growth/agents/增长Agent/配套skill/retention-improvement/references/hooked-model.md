# Hooked模型详解

## 模型概述

Hooked模型由Nir Eyal提出，核心是帮助用户形成习惯的闭环：

```
触发 → 行动 → 多变奖励 → 投入 → 触发（循环）
```

**核心目标**: 让用户不需要外部激励就能自发回来使用产品

---

## 四个阶段详解

### 1. 触发（Trigger）

**定义**: 促使你采取行动的 cue

**类型**:

#### 外部触发

| 类型 | 例子 | 适用场景 |
|------|------|---------|
| 推送通知 | "您有3条新消息" | 召回流失用户 |
| 邮件 | 周报、优惠信息 | 定期召回 |
| 短信 | 订单通知、验证码 | 交易相关 |
| 广告 | 投放素材 | 获取新用户 |

#### 内部触发

| 类型 | 触发情绪 | 例子 |
|------|---------|------|
| 无聊 | 时间空闲 | 刷抖音 |
| 焦虑 | 需要信息 | 刷新闻 |
| 孤独 | 需要社交 | 刷朋友圈 |
| 习惯 | 固定场景 | 早起看天气 |

**设计要点**:
- 外部触发：找到用户需要的理由
- 内部触发：与用户情绪/场景关联
- 最终目标是让用户不需要外部触发就能回来

---

### 2. 行动（Action）

**定义**: 在期待奖励时采取的行动

**公式**:
```
B = MAT
B = Behavior（行为）
M = Motivation（动机）
A = Ability（能力）
T = Trigger（触发）
```

**提高行动的方法**:

#### 增强动机
- 让用户感觉好
- 让用户看到收益
- 减少负面感受

#### 降低摩擦
- 简化步骤
- 减少输入
- 提供默认值

**案例**:
```
微信：发送消息
- 动机：社交需求
- 摩擦：需要打字
- 降低摩擦：语音转文字、表情包
```

---

### 3. 多变奖励（Variable Reward）

**定义**: 超出预期的奖励，激发渴望

**类型**:

#### 社交奖励
- 他人的关注、认可
- 点赞、评论、转发
- 排名、竞争

#### 成就奖励
- 徽章、等级、积分
- 完成任务
- 解锁成就

#### 实质奖励
- 优惠、红包
- 内容、信息
- 功能解锁

**关键：多变**

单一奖励会让人习惯，多变奖励让人上瘾：
```
固定奖励：每天看天气预报（知道就是那个温度）
多变奖励：刷抖音（永远不知道下一个视频是什么）
```

---

### 4. 投入（Investment）

**定义**: 用户为提升产品而贡献的个人数据、内容、社会资本

**投入类型**:

| 类型 | 例子 | 效果 |
|------|------|------|
| 数据投入 | 个人信息、历史记录 | 更懂你，更精准 |
| 内容投入 | 发布的文章/视频 | 不想放弃已有粉丝 |
| 声誉投入 | 等级、徽章、粉丝 | 不舍得放弃积累 |
| 技能投入 | 学习成本、使用习惯 | 换产品成本高 |

**投入→触发循环**:
```
用户在产品投入越多
→ 产品变得越好用（个性化）
→ 用户越依赖产品
→ 用户越愿意继续投入
→ 产品变得更好用（正向循环）
```

## Hooked模型应用案例

### 案例1：微信

```
触发：消息通知（外部）、社交需求（内部）
行动：打开微信
奖励：社交奖励（消息、朋友圈）、实质奖励（支付、红包）
投入：社交关系、内容发布、使用数据

结果：每天必用的习惯产品
```

### 案例2：抖音

```
触发：推送通知（外部）、无聊/碎片时间（内部）
行动：刷视频
奖励：多变的内容奖励（永远不知道下一个视频）
投入：关注账号、发布作品、花时间

结果：刷到停不下来的习惯产品
```

### 案例3：Keep

```
触发：推送通知提醒健身（外部）、健康焦虑（内部）
行动：开始健身
奖励：成就奖励（完成打卡、徽章）、实质奖励（课程解锁）
投入：健身数据、记录、课程进度

结果：帮助养成健身习惯
```

## Hooked模型设计检查清单

### 触发设计
- [ ] 是否有外部触发（推送/邮件）？
- [ ] 是否关联内部触发（情绪/场景）？
- [ ] 外部触发是否足够频繁？

### 行动设计
- [ ] 目标行为是否清晰？
- [ ] 动机是否足够强？
- [ ] 摩擦是否足够低？

### 奖励设计
- [ ] 是否有社交奖励？
- [ ] 是否有成就奖励？
- [ ] 是否有实质奖励？
- [ ] 奖励是否足够多变？

### 投入设计
- [ ] 用户能否投入数据？
- [ ] 用户能否投入内容？
- [ ] 用户能否投入声誉/社会资本？
- [ ] 投入是否能触发下一次使用？
