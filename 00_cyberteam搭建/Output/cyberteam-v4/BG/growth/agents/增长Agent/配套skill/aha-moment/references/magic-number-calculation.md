# 魔法数字计算方法详解

## 魔法数字定义

**格式**: 用户在激活时间窗口内完成X次某行为

**例子**:
- Facebook: 7天内添加7个好友
- YouTube: 首日观看30分钟
- 美颜相机: 首日使用1次滤镜

---

## 方法A：边际效应最大法

### 核心原理

找到留存率提升幅度最大的行为次数拐点

### 步骤

**Step 1**: 收集首日激活行为次数分布数据

| 行为次数 | 用户数 |
|---------|--------|
| 0次 | 4000 |
| 1次 | 3200 |
| 2次 | 1800 |
| 3次 | 600 |
| 4次 | 300 |
| 5次 | 100 |

**Step 2**: 计算各次数组的次日留存率

| 行为次数 | 用户数 | 次日留存用户 | 次日留存率 |
|---------|--------|------------|-----------|
| 0次 | 4000 | 832 | 20.8% |
| 1次 | 3200 | 1062 | 33.2% |
| 2次 | 1800 | 682 | 37.9% |
| 3次 | 600 | 234 | 39.0% |
| 4次 | 300 | 120 | 40.0% |
| 5次 | 100 | 40 | 40.0% |

**Step 3**: 绘制留存率曲线，找到拐点

**观察曲线形状**:
```
0→1次: 20.8% → 33.2% (+12.4%)  ← 边际效益最大
1→2次: 33.2% → 37.9% (+4.7%)
2→3次: 37.9% → 39.0% (+1.1%)
3→4次: 39.0% → 40.0% (+1.0%)
4→5次: 40.0% → 40.0% (0%)
```

**拐点判断**:
- 0→1次的边际效益最大
- 1次之后，留存率提升趋于平缓
- 结论：魔法数字是1次

---

### 边际效益计算公式

```
边际效益 = (N次留存率 - N-1次留存率) / (N次 - (N-1次))
```

**案例**:
```
0→1次边际效益 = (33.2% - 20.8%) / (1 - 0) = 12.4%
1→2次边际效益 = (37.9% - 33.2%) / (2 - 1) = 4.7%
2→3次边际效益 = (39.0% - 37.9%) / (3 - 2) = 1.1%
```

## 方法B：韦恩图法

### 核心原理

找到"有该行为且留存"的用户占总用户比例最高的点

### 概念定义

**群体A**: 有该行为且第二天留存
**群体B**: 有该行为但第二天流失
**群体C**: 无该行为但第二天留存

### 步骤

**Step 1**: 收集数据

| 首次行为次数 | A有行为+留存 | B有行为+流失 | C无行为+留存 |
|------------|-------------|-------------|-------------|
| ≥0次 | 6980 | 0 | 0 |
| ≥1次 | 3221 | X | 832 |
| ≥2次 | ... | ... | ... |

**Step 2**: 计算各类占比

**总用户** = A + B + C

**占比** = A / 总用户

**Step 3**: 找到占比最高的点

**案例数据**:
| 首次行为次数 | A有行为+留存 | 总用户 | A占比 |
|------------|-------------|--------|------|
| ≥0次 | 6980 | 6980 | 100% |
| ≥1次 | 3221 | 4053 | 79.5% |
| ≥2次 | ... | ... | ... |

**Step 4**: 找最大占比点

如果"≥1次"的A占比最高（32%），则魔法数字是1次

## 两种方法对比

| 方法 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| 边际效应法 | 直观易懂 | 需要足够样本 | 适合快速分析 |
| 韦恩图法 | 更精确 | 计算复杂 | 适合精确分析 |

## 魔法数字应用

### 应用1：阶段性激活

**思路**: 先推动用户达到基础次数，再提升到最优次数

**案例**:
```
美颜相机：
- 基础魔法数字：1次（首日1次滤镜）
- 进阶魔法数字：8次（8次后留存达理想水平）

执行策略：
1. 新用户开机 → 引导完成1次滤镜（基础激活）
2. 用户完成1次后 → 继续引导至8次（深度激活）
```

---

### 应用2：分层干预

**思路**: 不同次数的用户采取不同激活策略

| 次数 | 状态 | 策略 |
|------|------|------|
| 0次 | 未激活 | 强引导触发第一次 |
| 1-7次 | 部分激活 | 持续引导增加次数 |
| 8次+ | 完全激活 | 恭喜成功激活 |

---

## 注意事项

### 1. 魔法数字不是绝对的

**原因**:
- 是统计最优解，代表多数用户的情况
- 不同用户群可能有不同的最优次数
- 可以根据用户画像细分

### 2. 魔法数字可能变化

**原因**:
- 产品迭代可能改变用户行为
- 竞品情况变化可能影响用户预期
- 需要定期重新验证

### 3. 不要机械执行

**错误做法**:
- 用户完成了8次但没有真正体验价值 → 仍然没有真正激活
- 强制推送让用户完成任务次数 → 可能导致流失

**正确做法**:
- 引导用户自然达到次数
- 确保每次完成都有价值体验
- 关注用户实际感知而非数字
