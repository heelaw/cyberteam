# 产品核心定位模板

## Identity & Memory

### 背景

产品核心定位（价值主张）是产品策略的基石，它决定了产品为谁服务、解决什么问题、为什么值得用户选择。本 Skill 来自「增长黑客入门」课程，提供了一套系统化的价值主张定义方法。

**核心方法论**：
- 价值主张3要素：用户需求 + 产品解决方案 + 差异化价值
- 3个应用好处：明确产品方向、指导营销策略、便于用户沟通
- 句型模板：为[目标用户]提供[解决方案]，解决[用户痛点]
- 四象限验证：需求频次 × 变现难易

### 适用人群

- 产品经理：定义新产品或迭代现有产品定位
- 创业者：明确产品价值和市场竞争位置
- 运营人员：理解产品核心价值以指导运营策略
- 增长团队：制定以价值主张为基础的增长策略

### 常见误区

- 只强调功能特点，忽略用户实际收益
- 价值主张过于宽泛，缺乏差异化
- 没有验证价值主张的商业可行性
- 将产品定位与品牌定位混淆

---

## Core Mission

### 解决什么问题

帮助产品团队明确产品的核心价值主张，回答「产品为谁提供什么价值，为什么选择我们」这个核心问题。

### 使用场景

- 新产品立项，需要定义产品定位
- 现有产品增长遇到瓶颈，需要重新梳理价值主张
- 竞品分析，需要明确差异化竞争策略
- 融资演示，需要清晰表达产品价值
- 营销推广，需要统一价值主张口径

---

## Critical Rules

### 必须遵守

1. **三要素完整**: 价值主张必须包含「用户需求 + 产品解决方案 + 差异化价值」三个要素，缺一不可
2. **用户视角**: 用用户收益（Benefit）而非产品功能（Feature）来描述价值
3. **可验证性**: 价值主张必须能够通过4个问题验证：用户是否真正需要？能否持续满足？是否有差异化？能否支撑商业化？
4. **四象限优先**: 优先选择「需求频次高」象限的产品定位，无论变现难易

### 禁止行为（5条）⭐

- ❌ 只描述产品功能，不说明用户收益
- ❌ 使用模糊词汇如「优质」「高效」「领先」
- ❌ 选择需求频次低且变现难的价值主张
- ❌ 价值主张与实际产品功能不符
- ❌ 忽视技术可行性（团队无法实现的技术方案）

---

## Success Metrics

### 输出质量指标

- **三要素覆盖率**: 100%（用户需求、解决方案、差异化价值都必须有）
- **验证问题通过率**: 4/4（全部通过为合格）
- **四象限定位**: 必须在「需求频次高」的两个象限内

### 业务结果指标

- **团队对齐度**: 相关方对价值主张的认同度 ≥80%
- **文案一致性**: 营销文案与核心价值主张的一致性 ≥90%

## Workflow Process

### Step 1: 明确目标用户

**输入**: 产品初步构想、目标市场描述

**处理**:
1. 确定产品服务的核心用户群体
2. 描述目标用户的特征（年龄、职业、收入等）
3. 明确用户使用场景
4. 识别用户核心痛点

**输出**:
```json
{
  "target_user": {
    "core_group": "核心用户群体",
    "characteristics": {"age": "范围", "occupation": "职业", "income": "收入"},
    "scenarios": ["使用场景1", "使用场景2"],
    "pain_points": ["痛点1", "痛点2"]
  }
}
```

**成功标准**: 能清晰描述目标用户画像和使用场景

---

### Step 2: 定义用户需求

**输入**: 目标用户信息

**处理**:
1. 识别目标用户的核心需求（列举3-5个）
2. 评估每个需求的频次（高频/低频）
3. 评估每个需求的强度（刚需/非刚需）
4. 选择1-3个核心需求

**输出**:
```json
{
  "user_needs": [
    {"need": "需求1", "frequency": "高频", "intensity": "刚需"},
    {"need": "需求2", "frequency": "低频", "intensity": "非刚需"}
  ],
  "core_needs": ["核心需求1", "核心需求2"]
}
```

**成功标准**: 明确用户最核心的1-3个需求

---

### Step 3: 设计解决方案

**输入**: 用户核心需求

**处理**:
1. 提出满足用户需求的产品方案
2. 定义产品的核心功能
3. 描述产品如何解决用户痛点
4. 验证方案与需求的对应关系

**输出**:
```json
{
  "solution": {
    "description": "产品方案描述",
    "core_features": ["功能1", "功能2"],
    "pain_point_mapping": {"痛点1": "解决方案1", "痛点2": "解决方案2"}
  }
}
```

**成功标准**: 方案能直接对应用户需求

---

### Step 4: 提炼差异化价值

**输入**: 产品方案、竞品信息

**处理**:
1. 分析主要竞品或替代方案的价值主张
2. 对比找出产品的独特价值点
3. 明确"为什么选我们而不是竞品"的理由

**输出**:
```json
{
  "differentiation": {
    "competitors": {"竞品A": "其价值主张", "竞品B": "其价值主张"},
    "unique_value": ["差异化1", "差异化2"],
    "why_us": "选择我们的核心理由"
  }
}
```

**成功标准**: 能清晰说明"为什么选我们而不是竞品"

---

### Step 5: 应用句型模板

**输入**: 目标用户、解决方案、痛点

**处理**:
1. 使用价值主张公式：为[目标用户]提供[解决方案]，解决[用户痛点]
2. 填入前4步确定的内容
3. 优化表述，使其简洁有力

**输出**:
```json
{
  "value_proposition": "为[目标用户]提供[解决方案]，解决[用户痛点]"
}
```

**成功标准**: 产出完整的价值主张句子

---

### Step 6: 四象限验证

**输入**: 产品需求频次、变现方式

**处理**:
1. 评估需求频次（高/低）
2. 评估变现难易（易/难）
3. 在四象限中确定位置
4. 判断是否为优先选择的象限

**输出**:
```json
{
  "quadrant": {
    "demand_frequency": "高/低",
    "monetization_difficulty": "易/难",
    "position": "优先选择/可尝试/直接放弃"
  }
}
```

**成功标准**: 定位在"需求频次高"的优先选择象限

---

### Step 7: 四问题验证

**输入**: 价值主张初稿

**处理**:
1. 用户是否真正需要？（真实需求）
2. 能否持续满足？（可持续性）
3. 是否有差异化？（竞争优势）
4. 能否支撑商业化？（变现能力）

**输出**:
```json
{
  "validation": {
    "q1_real_need": "是/否",
    "q2_sustainable": "是/否",
    "q3_differentiated": "是/否",
    "q4_monetizable": "是/否",
    "pass_count": 4,
    "result": "通过/需要调整"
  }
}
```

**成功标准**: 至少3个问题得到肯定回答

## Deliverable Template

### 价值主张定义模板

```markdown
# [产品名称] 价值主张定义

## 1. 目标用户

**核心用户群体**：[描述]

**用户画像**：
- 特征：[年龄、职业、收入等]
- 场景：[使用场景]
- 痛点：[核心痛点]

## 2. 用户需求

| 需求 | 频次 | 强度 |
|------|------|------|
| 需求1 | 高频/低频 | 刚需/非刚需 |
| 需求2 | 高频/低频 | 刚需/非刚需 |

**核心需求**：[选择1-3个]

## 3. 产品解决方案

**解决方案**：[描述产品如何满足用户需求]

**核心功能**：
- 功能1：[描述]
- 功能2：[描述]

## 4. 差异化价值

**竞品分析**：
- 竞品A：[价值主张]
- 竞品B：[价值主张]

**我们的差异化**：
- 差异化1：[描述]
- 差异化2：[描述]

## 5. 价值主张

**句型模板输出**：

为[目标用户]提供[解决方案]，解决[用户痛点]

**完整价值主张**：[填写]

## 6. 四象限定位

| | 变现容易 | 变现难 |
|---|---|---|
| **需求频次高** | ✅ 优先选择 | ✅ 优先选择 |
| **需求频次低** | 可尝试 | ❌ 直接放弃 |

**当前定位**：[象限名称]

## 7. 四问题验证

| 问题 | 回答 | 说明 |
|------|------|------|
| 用户是否真正需要？ | 是/否 | |
| 能否持续满足？ | 是/否 | |
| 是否有差异化？ | 是/否 | |
| 能否支撑商业化？ | 是/否 | |

**验证结果**：通过/需要调整
```

### 使用示例

**输入**：
- 目标用户：中小企业主
- 用户需求：快速建立线上销售渠道
- 解决方案：SaaS 电商小程序
- 差异化价值：免费开通、5分钟快速上线

**输出价值主张**：
> 为中小企业主提供快速开通的 SaaS 电商小程序，解决缺乏技术团队无法快速建立线上销售渠道的问题

## Communication Style

1. **结构化输出**: 按照模板格式清晰呈现
2. **数据驱动**: 尽量用量化的方式描述
3. **用户视角**: 始终从用户收益角度出发
4. **验证优先**: 在确定价值主张前必须完成验证步骤
