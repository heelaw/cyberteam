# 门槛设计案例集

## 案例一：知识付费社群

```
═══════════════════════════════════════════
【高门槛】身份认同
条件: 必须完成《产品经理入门课程》（线上课程免费）
验证: 提交课程完成截图

【中门槛】付费/时间
条件: 付费99元/年（包含专属资料包+答疑服务）
验证: 付费截图

【低门槛】用户画像
条件: 互联网行业产品经理或想转行做产品的人群
验证: 自我介绍审核

【执行亮点】
1. 高门槛免费，降低入群意愿门槛
2. 用课程完成作为身份认同标志
3. 中门槛99元定价与价值匹配
```

## 案例二：行业交流社群

```
═══════════════════════════════════════════
【高门槛】身份认同
条件: 在知名互联网公司担任产品/运营岗位
验证: 脉脉个人主页或工牌验证

【中门槛】付费/时间
条件: 付费299元/年 或 邀请1位同行入群
验证: 付费截图 或 邀请记录

【低门槛】用户画像
条件: 5年以上互联网从业经验
验证: 简历审核或自我介绍

【执行亮点】
1. 高门槛用公司背景作为身份标识
2. 中门槛提供付费或邀请两个选项
3. 形成用户自发传播机制
```

## 案例三：地域性社群

```
═══════════════════════════════════════════
【高门槛】身份认同
条件: 参加过3次以上线下活动
验证: 活动签到记录

【中门槛】付费/时间
条件: 免费（但需承诺每月至少参加1次活动）
验证: 活动参与记录

【低门槛】用户画像
条件: 生活在[城市名]且从事[行业]
验证: 地理位置+自我介绍

【执行亮点】
1. 高门槛鼓励线下参与
2. 免费+承诺机制降低进入门槛
3. 地理+行业双重筛选
```

## 案例四：高端私密社群

```
═══════════════════════════════════════════
【高门槛】身份认同
条件: 必须是公司创始人/CEO/高管
验证: 营业执照/名片/LinkedIn

【中门槛】付费/时间
条件: 付费1999元/年 或 老会员推荐
验证: 付费截图 或 推荐人确认

【低门槛】用户画像
条件: 所创办公司至少完成A轮融资
验证: 融资新闻或公开信息

【执行亮点】
1. 高门槛强调身份稀缺性
2. 融资背景作为硬性门槛
3. 老推荐机制保证社群纯净度
```

## 案例五：兴趣型社群

```
═══════════════════════════════════════════
【高门槛】身份认同
条件: 在社群专属平台发布过至少3篇原创内容
验证: 内容发布记录

【中门槛】付费/时间
条件: 免费（但需完成新人任务包）
验证: 任务完成截图

【低门槛】用户画像
条件: 对[具体兴趣领域]有浓厚兴趣
验证: 入群问卷调查

【执行亮点】
1. 用内容贡献作为身份标识
2. 新人任务代替付费，降低门槛
3. 兴趣问卷确保用户相关性
```
