# 进度追踪机制设计模板

## 一、追踪机制框架

### 追踪层级设计
| 层级 | 追踪对象 | 追踪频率 | 追踪方式 | 负责人 |
|------|----------|----------|----------|--------|
| L1 | 战略目标/OKR | 季度/月 | 战略回顾会 | 高层管理 |
| L2 | 部门目标/KPI | 月度 | 部门复盘会 | 部门负责人 |
| L3 | 团队任务/行动 | 周度 | 周报+周会 | 团队负责人 |
| L4 | 个人行动 | 每日 | 站会+日报 | 个人 |

### 追踪原则
1. **透明**：进度信息对相关方透明
2. **及时**：问题及时暴露，不要掩盖
3. **可追溯**：有历史记录可查
4. **聚焦**：追踪关键里程碑，不过度细枝末节

## 二、追踪工具选择

### 工具对比
| 工具 | 适用场景 | 优点 | 缺点 |
|------|----------|------|------|
| 物理看板 | 同地团队，<20人 | 直观可见，便于沟通 | 不适合远程 |
| 数字看板 | 远程团队，任何规模 | 随时访问，自动统计 | 需要工具支持 |
| Excel/表格 | 简单追踪，临时使用 | 简单易用 | 难以协作 |
| 专业项目管理 | 复杂项目，大团队 | 功能强大 | 学习成本高 |

### 推荐方案
| 团队规模 | 推荐工具 |
|----------|----------|
| <10人 | 物理看板 + 简单周报 |
| 10-50人 | 数字看板（飞书/Notion/Jira）|
| >50人 | 专业项目管理工具 |

## 五、追踪会议模板

### 周进度会议（60分钟）

```
【周进度会议议程】
时间：每周___  ___:___ - ___:___
时长：60分钟
参加人：____________

议程：
1. 进度同步（20分钟）
   - 各团队/个人汇报进度
   - 识别阻碍和问题

2. 问题升级（15分钟）
   - 讨论升级的问题
   - 快速决策或指定跟进

3. 资源协调（10分钟）
   - 需要跨团队协调的事项
   - 资源冲突解决

4. 下周计划确认（10分钟）
   - 确认下周重点任务
   - 明确责任人

5. 临时议题（5分钟）
   - 其他需要讨论的事项
```

### 月度复盘会议

```
【月度复盘会议议程】
时间：每月最后一周  ___:___ - ___:___
时长：2小时
参加人：____________

议程：
1. 整体进度回顾（30分钟）
   - 里程碑完成情况
   - 目标达成率分析

2. 偏差深度分析（30分钟）
   - 重大偏差原因分析
   - 经验教训总结

3. 风险重新评估（20分钟）
   - 现有风险状态更新
   - 新增/消除风险

4. 下月计划调整（20分钟）
   - 根据当前情况调整计划
   - 资源重新分配

5. 团队反馈（20分钟）
   - 一线执行反馈
   - 改进建议收集
```

## 六、追踪规范

### 状态定义
| 状态 | 定义 | 响应要求 |
|------|------|----------|
| 绿色（Green）| 正常推进，按计划完成 | 正常追踪 |
| 黄色（Yellow）| 有风险，但可控 | 本周内分析原因 |
| 红色（Red）| 严重偏差，需要干预 | 立即处理或升级 |

### 偏差阈值
| 偏差类型 | 绿色 | 黄色 | 红色 |
|----------|------|------|------|
| 进度偏差 | <5% | 5-15% | >15% |
| 范围偏差 | <5% | 5-10% | >10% |
| 资源偏差 | <5% | 5-15% | >15% |

### 升级规则
| 条件 | 升级对象 | 升级时限 |
|------|----------|----------|
| 红灯状态持续3天以上 | 直接上级 | 立即 |
| 需要跨部门资源协调 | 项目 sponsor | 24小时内 |
| 可能影响上级目标 | 上级管理层 | 48小时内 |
