# 产品运营规划 - 测试案例

## 测试案例1

**难度**：medium
**场景**：新产品进入市场

**输入**：
```
背景：一家教育科技公司推出了首款AI辅导产品，定位K12数学辅导市场。产品上线3个月，目前有注册用户5万，但日活只有800人，付费用户仅120人。产品团队正在讨论下一步运营策略。
市场环境：K12教育竞争激烈，头部玩家（学而思、作业帮）占据大部分市场，但AI辅导仍是蓝海。
资源情况：团队15人，市场预算50万/月。
```

**预期输出**：
```
1. 阶段判断：探索期向成长期过渡
   依据：用户增长缓慢、产品价值尚未验证、竞争格局未定

2. 5因素分析：
   - 产品阶段：探索期-成长期过渡，需验证产品-市场匹配
   - 用户特征：K12家长为主，关注提分效果，愿意为效果付费
   - 竞争环境：传统巨头转型慢，AI教育有差异化空间
   - 资源条件：团队精简，预算有限，需聚焦单点突破
   - 团队能力：AI技术能力强，教育内容积累弱

3. 核心破局点（聚焦1点）：
   破局点：打造标杆案例，用效果说话
   依据：有限资源应集中于验证产品价值，而非铺量

4. 阶段目标：
   - 第一阶段（1-2月）：获取30个付费用户完成服务，验证续费率
   - 第二阶段（3-4月）：基于续费数据优化，迭代产品
   - 第三阶段（5-6月）：复制成功模式，扩大用户规模

5. 执行路径：[具体行动项、责任人、预算]

6. 评估机制：周度复盘核心指标（付费转化率、续费率）
```

**验收标准**：
- [ ] 阶段判断有数据支撑
- [ ] 破局点聚焦而非分散
- [ ] 目标可量化可验证
- [ ] 充分考虑了资源约束

## 测试案例2

**难度**：simple
**场景**：成熟产品寻求增长

**输入**：
```
背景：一款有3年历史的工具类产品，目前DAU稳定在50万，但增长停滞。产品已完成商业化探索，年营收1亿。用户主要是25-35岁职场人群，使用场景是办公文档处理。
竞争态势：市场份额稳定，没有强力挑战者，但也没有明显增长空间。
团队情况：运营团队20人，年预算500万。
```

**预期输出**：
```
1. 阶段判断：成熟期
   依据：DAU增长停滞、市场份额稳定、商业化成熟

2. 5因素分析：
   - 产品阶段：成熟期，核心任务是深挖价值+探索新增长点
   - 用户特征：职场人群，工具依赖度高，付费意愿强
   - 竞争环境：竞争缓和，但增长空间有限
   - 资源条件：资源充足，可支撑新方向探索
   - 团队能力：运营能力强，有能力做创新尝试

3. 核心破局点（建议2个）：
   破局点1：深耕付费用户价值（提升ARPU）
   破局点2：探索B端企业用户市场

4. 阶段目标：
   - 第一阶段（1-3月）：用户分层，识别高价值用户特征
   - 第二阶段（4-6月）：针对高价值用户推出增值服务
   - 第三阶段（7-12月）：企业版产品研发和市场验证

5. 执行路径：[具体行动项]

6. 评估机制：月度复盘ARPU和付费转化率变化
```

**验收标准**：
- [ ] 判断成熟期依据充分
- [ ] 破局点符合成熟期特点（深耕+探索）
- [ ] 考虑了与竞品的差异化
