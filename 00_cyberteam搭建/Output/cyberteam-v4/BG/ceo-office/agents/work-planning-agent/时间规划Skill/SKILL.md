## 时间规划方法论

### 1. 时间块规划法

```
把一天分成若干时间块
每个时间块专注做一件事
```

**时间块类型**：
- 深度工作块（2-3小时）：处理重要复杂任务
- 会议块（1-2小时）：团队协作
- 碎片块（30分钟）：处理邮件、消息
- 缓冲块（15分钟）：休息、调整

### 2. 能量管理

```
跟随能量节奏安排任务
高能量 → 重要任务
低能量 → 琐碎任务
```

**能量周期**：
- 上午9-11点：黄金时间 → 核心工作
- 下午2-4点：次黄金时间 → 协作沟通
- 下午4-5点：低谷时间 → 整理复盘

### 3. GTD（Getting Things Done）

```
收集 → 处理 → 组织 → 回顾 → 执行
```

---

## 分析流程

### Step 1: 盘点可用时间

**处理**：
- 计算每天可用工作时间
- 扣除会议、沟通等固定时间
- 计算净可用时间

**输出**: 可用时间清单

### Step 2: 分配任务到时间

**处理**：
- 按优先级分配任务
- 匹配任务类型和时间块
- 考虑能量周期

**输出**: 初始时间表

### Step 3: 优化调整

**处理**：
- 检查是否有冲突
- 确保重要任务有足够时间
- 预留缓冲和应急时间

**输出**: 最终时间计划

## Critical Rules

1. **要事第一**: 重要任务必须安排在黄金时间

2. **匹配能量**: 高价值任务配高能量时段

3. **留有缓冲**: 计划不要排满，留10-20%缓冲

4. **固定节奏**: 建立稳定的工作节奏

5. **定期回顾**: 每天/每周复盘时间使用

---

## 禁止行为

1. **禁止排满**: 不能把时间排到100%

2. **禁止随意打断**: 深度工作块不要被随意打断

3. **禁止忽视休息**: 长时间工作必须安排休息

4. **禁止不会调整**: 计划要有弹性，能调整

5. **禁止只规划不执行**: 规划是开始，执行是关键

## 输入输出

### 必要输入
- 任务清单（含优先级）
- 会议/固定时间
- 工作时长

### 输出格式
- 时间块规划表
- 每日时间安排
- 黄金时间使用建议
