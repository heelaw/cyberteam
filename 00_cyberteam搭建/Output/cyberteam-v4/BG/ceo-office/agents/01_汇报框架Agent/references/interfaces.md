# interfaces - 汇报框架Agent

## 核心原则

**所有Agent决策前必须读取项目资料库。所有Agent产出必须存入项目目录。**

---

## 一、输入规范 (Input)

### 1.1 触发条件

| 场景 | 触发词 | 说明 |
|------|--------|------|
| 周报月报 | 周报、月报、工作总结 | 周期性汇报 |
| 项目汇报 | 项目汇报、进展汇报 | 阶段性汇报 |
| 述职报告 | 述职、转正答辩 | 评估性汇报 |
| 即兴汇报 | 电梯汇报、走廊汇报 | 30秒-2分钟 |
| 向上管理 | 领导关注、协调请求 | 管理场景 |

### 1.2 必读信息

| 信息类型 | 内容 | 说明 |
|----------|------|------|
| 汇报对象 | 领导职级、关注点、风格 | 影响汇报方式 |
| 项目背景 | 目标、进度、成果 | 汇报内容基础 |
| 数据支撑 | KPI完成、关键指标 | 结论支撑 |

---

## 二、输出规范 (Output)

### 2.1 输出格式

```markdown
## 汇报标题

### 结论（电梯法则）
[一句话核心结论]

### 背景
[1-2句背景说明]

### 问题/挑战
- [问题1] + 数据支撑
- [问题2] + 影响分析

### 方案选项
| 方案 | 做法 | 资源 | 效果 | 风险 |
|------|------|------|------|------|
| A    | ...  | ...  | ...  | ...  |
| B    | ...  | ...  | ...  | ...  |

### 请求
- [ ] 资源请求：...
- [ ] 决策请求：...
- [ ] 授权请求：...
```

### 2.2 输出存放

```
{project}/decisions/          # 决策类汇报产出
{project}/meetings/           # 会议记录
```

---

## 三、协作接口

### 3.1 向上汇报

| 接收方 | 内容 | 格式 |
|--------|------|------|
| 运营总监 | 运营进展汇报 | 运营周报/专项汇报 |
| 增长总监 | 战略目标进展 | 里程碑汇报 |
| CEO | 重大决策请求 | 方案+请求 |

### 3.2 横向协作

| 协作方 | 场景 | 产出 |
|--------|------|------|
| 产品部 | 产品进展汇报 | PRD评审记录 |
| 数据团队 | 指标数据获取 | 数据分析报告 |
| 营销部 | 营销效果汇报 | 渠道效果报告 |

---

## 四、汇报类型适配

| 类型 | 时长 | 结构 | 重点 |
|------|------|------|------|
| 即兴汇报 | 30秒 | 结论+原因 | 电梯法则 |
| 周报/月报 | 5分钟 | 成果+问题+计划 | 数据支撑 |
| 项目汇报 | 10分钟 | 背景+进展+风险+请求 | 方案导向 |
| 述职报告 | 15分钟 | 角色+贡献+复盘+规划 | 价值体现 |
