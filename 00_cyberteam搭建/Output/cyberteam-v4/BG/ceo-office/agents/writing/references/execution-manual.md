# 转化型文案创作执行手册

## 1. 工作流概述

转化型文案创作工作流是一个系统化的9阶段流程，旨在确保每篇文案都经过完整的创作、审核、打磨流程，最终产出高转化率的营销文案。

### 1.1 工作流目标

- 确保文案基于真实用户反馈和数据分析
- 严格遵循投放平台的特点和规则
- 每一句话都服务于最终转化
- 深刻理解用户需求和心理
- 避免冗余信息，突出核心价值
- 采用口语化表达，自然流畅
- 包含明确的行动召唤
- 确保合规合法，无虚假宣传

### 1.2 核心原则

- **数据驱动**: 所有决策基于真实数据和用户反馈
- **用户中心**: 从用户需求和痛点出发
- **平台适配**: 针对不同渠道定制文案
- **转化导向**: 每一步都服务于最终转化目标
- **质量优先**: 严格的审核和优化流程

## 2. 阶段执行指南

### 2.1 Stage 0：工作流初始化

**目标**: 确认用户需求，初始化工作流配置

**执行步骤**:

1. 向用户确认基本需求：
   ```
   我是转化型文案创作专家，为了打造高转化率文案，我需要确认：
   
   1. 产品/服务名称：
   2. 核心功能或卖点：
   3. 目标用户群体：
   4. 投放平台/渠道：
   5. 是否需要收集真实用户反馈？(y/n)
   6. 参考文案或竞品信息（可选）：
   7. 已有用户评价或反馈（可选）：
   ```

2. 根据用户回复，初始化工作流配置

3. **输出文件**：`workflow/00-init/{product}-{YYYYMMDD}-init.md`

4. **完成后**：
   ```
   ✅ Stage 0 完成：工作流初始化
   - 配置已保存到：[路径]
   - 产品：[产品名称]
   - 目标平台：[平台名称]
   - 是否收集用户反馈：[是/否]
   
   ⏸️ 是否进入 Stage 1（用户需求挖掘）？
   ```

### 2.2 Stage 1：用户需求挖掘

**目标**: 通过产品信息和真实用户反馈，深度分析目标人群，建立精准用户画像

**执行步骤**:

1. **基础信息分析**：
   - 人口统计特征：年龄、性别、收入、教育背景
   - 心理特征：兴趣爱好、价值观、生活方式
   - 行为特征：购买习惯、消费频率、品牌忠诚度

2. **真实用户反馈收集**（如果用户选择是）：
   ```bash
   # 搜索示例
   site:reddit.com "[product] sucks OR frustrating OR annoying"
   site:x.com "[product] problem OR issue OR bug"
   site:zhihu.com "[产品] 难用 OR 吐槽 OR 问题"
   site:xiaohongshu.com "[产品] 真实测评 OR 吐槽"
   ```

3. **用户反馈分类**：
   - 功能缺失
   - 体验问题
   - 性能问题
   - 定价问题
   - 竞品对比

4. **输出文件**：`workflow/01-user-research/{product}-{YYYYMMDD}-user-research.md`
   - 使用模板：`assets/user-research-template.md`

5. **完成后**：
   ```
   ✅ Stage 1 完成：用户需求挖掘
   - 报告已保存到：[路径]
   - 分析了 [N] 条用户反馈
   - 识别了 [M] 个核心痛点
   
   ⏸️ 是否进入 Stage 2（产品卖点挖掘）？
   ```

### 2.3 Stage 2：产品卖点挖掘

**目标**: 基于用户反馈和产品信息，深度挖掘产品核心卖点，并将功能转化为用户价值

**执行步骤**:

1. **结构化卖点挖掘**：
   - 物理层：材料、工艺、设计、参数
   - 体验层：使用场景中的独特感受
   - 情感层：细节触发的心理反应

2. **基于用户反馈优化卖点**：
   - 突出解决用户真实痛点的卖点
   - 弱化用户不关心的功能
   - 强调与竞品的差异化优势

3. **输出文件**：`workflow/02-selling-points/{product}-{YYYYMMDD}-selling-points.md`
   - 使用模板：`assets/selling-points-template.md`

4. **完成后**：
   ```
   ✅ Stage 2 完成：产品卖点挖掘
   - 报告已保存到：[路径]
   - 提炼了 [N] 个核心卖点
   - 匹配了 [M] 个用户痛点
   
   ⏸️ 是否进入 Stage 3（痛点定位）？
   ```

### 2.4 Stage 3：痛点定位

**目标**: 将用户痛点与产品卖点精准匹配，形成强有力的说服逻辑

**执行步骤**:

1. **痛点优先级排序**：
   - 频次：多少人提到这个问题
   - 情绪强度：轻微抱怨 vs 强烈不满
   - 影响范围：影响多少用户

2. **卖点-痛点匹配矩阵**：
   | 痛点 | 匹配卖点 | 解决效果 | 说服逻辑 |
   |------|----------|----------|----------|
   | [痛点1] | [卖点1] | [效果描述] | [说服逻辑] |
   | [痛点2] | [卖点2] | [效果描述] | [说服逻辑] |

3. **输出文件**：`workflow/03-pain-points/{product}-{YYYYMMDD}-pain-points.md`
   - 使用模板：`assets/pain-points-template.md`

4. **完成后**：
   ```
   ✅ Stage 3 完成：痛点定位
   - 报告已保存到：[路径]
   - 匹配了 [N] 对卖点-痛点
   - 形成了 [M] 个说服逻辑
   
   ⏸️ 是否进入 Stage 4（需求类型判断）？
   ```

### 2.5 Stage 4：需求类型判断

**目标**: 判断用户对产品的需求类型，确定文案的说服逻辑

**执行步骤**:

1. **需求类型分析**：
   - **类型A：刚性需求**：生活中离不开，需要持续使用
     - 文案策略：用事实和数据说服用户
   - **类型B：可有可无需求**：用了更好，不用不影响
     - 文案策略：通过场景植入激发用户欲望
   - **类型C：趋于没有需求**：用户认为不需要
     - 文案策略：制造认知失衡，重构用户认知

2. **确定文案策略**：
   - 核心诉求点
   - 说服逻辑
   - 情感诉求
   - 行动召唤（CTA）

3. **输出文件**：`workflow/04-strategy/{product}-{YYYYMMDD}-strategy.md`
   - 使用模板：`assets/strategy-template.md`

4. **完成后**：
   ```
   ✅ Stage 4 完成：需求类型判断
   - 报告已保存到：[路径]
   - 需求类型：[类型A/B/C]
   - 核心策略：[策略描述]
   
   ⏸️ 是否进入 Stage 5（场景与渠道分析）？
   ```

### 2.6 Stage 5：场景与渠道分析

**目标**: 结合投放渠道特点，确定文案形式、长度和结构

**执行步骤**:

1. **渠道特征分析**：
   - 转化能力：可直接转化 vs 需跨场景转化
   - 阅读时长：10秒以内 vs 1-3分钟 vs 5分钟以上
   - 内容形式：图文 vs 视频

2. **渠道策略制定**：
   - 标题策略
   - 内容结构
   - 语气风格
   - 视觉元素建议

3. **输出文件**：`workflow/05-channel-strategy/{product}-{YYYYMMDD}-channel-strategy.md`
   - 使用模板：`assets/channel-strategy-template.md`

4. **完成后**：
   ```
   ✅ Stage 5 完成：场景与渠道分析
   - 报告已保存到：[路径]
   - 渠道：[渠道名称]
   - 文案形式：[图文/视频]
   - 阅读时长：[时长范围]
   
   ⏸️ 是否进入 Stage 6（文案撰写）？
   ```

### 2.7 Stage 6：文案撰写

**目标**: 基于前面的分析结果，撰写符合策略、具有转化力的初稿文案

**执行步骤**:

1. **文案撰写原则**：
   - 讲"人话"：口语化、接地气
   - 有数据：用具体数据支撑
   - 有细节：用具体描述替代抽象形容
   - 有场景：增强代入感
   - 有情感：调动用户情绪

2. **文案结构**：
   - 标题：吸引注意力
   - 钩子：激发兴趣
   - 痛点：戳中用户痛点
   - 解决方案：展示产品价值
   - 证据：用数据或案例证明
   - CTA：明确的行动召唤

3. **输出文件**：`workflow/06-draft/{platform}-{product}-{YYYYMMDD}-draft.md`
   - 使用模板：`assets/copywriting-template.md`

4. **完成后**：
   ```
   ✅ Stage 6 完成：文案撰写
   - 初稿已保存到：[路径]
   - 撰写了 [N] 版不同风格的文案
   - 包含标题、正文和 CTA
   
   ⏸️ 是否进入 Stage 7（文案优化）？
   ```

### 2.8 Stage 7：文案优化

**目标**: 对初稿文案进行检查、删改和优化，确保文案符合策略、精炼有力

**执行步骤**:

1. **质量检查清单**：
   - [ ] 核心卖点突出
   - [ ] 痛点匹配准确
   - [ ] 符合目标人群
   - [ ] 符合平台特性
   - [ ] 标题吸引眼球
   - [ ] 内容简洁有力
   - [ ] 有明确的行动召唤
   - [ ] 语言口语化
   - [ ] 无虚假宣传
   - [ ] 无敏感内容

2. **优化方向**：
   - 内容精简：删除冗余信息
   - 通俗易懂：简化复杂概念
   - 吸引力增强：优化标题和开头
   - 场景适配：调整内容结构
   - 感知引导：增强用户感受

3. **A/B 测试建议**：
   - 建议测试的版本
   - 测试指标
   - 测试方法

4. **输出文件**：`workflow/07-final/{platform}-{product}-{YYYYMMDD}-final.md`

5. **完成后**：
   ```
   ✅ Stage 7 完成：文案优化
   - 最终文案已保存到：[路径]
   - 优化了 [N] 个关键问题
   - 提供了 [M] 版 A/B 测试建议
   
   ⏸️ 是否进入 Stage 8（效果追踪与迭代）？
   ```

### 2.9 Stage 8：效果追踪与迭代

**目标**: 提供文案效果追踪建议，支持后续迭代优化

**执行步骤**:

1. **效果追踪指标**：
   - 点击率（CTR）
   - 转化率（CVR）
   - 成本转化比（CPC/CPA）
   - 停留时长
   - 分享率

2. **数据分析方法**：
   - 对比不同版本的表现
   - 分析高转化文案的共同特点
   - 识别需要优化的环节

3. **迭代建议**：
   - 基于数据的优化方向
   - 后续测试计划
   - 长期优化策略

4. **输出文件**：`workflow/08-tracking/{product}-{YYYYMMDD}-tracking.md`

5. **完成后**：
   ```
   ✅ Stage 8 完成：效果追踪与迭代
   - 建议已保存到：[路径]
   - 提供了 [N] 个关键指标
   - 给出了 [M] 条迭代建议
   
   🎉 转化型文案创作工作流已全部完成！
   - 最终文案：[文件路径]
   - A/B 测试建议：[文件路径]
   - 效果追踪方案：[文件路径]

   ```

## 3. 质量控制

### 3.1 各阶段质量要求

| 阶段 | 质量要求 |
|------|----------|
| 用户需求挖掘 | 基于真实用户反馈，无主观臆断 |
| 产品卖点挖掘 | 突出差异化优势，与痛点精准匹配 |
| 痛点定位 | 优先级清晰，说服逻辑严密 |
| 需求类型判断 | 基于数据分析，策略明确 |
| 场景与渠道分析 | 严格适配渠道特性 |
| 文案撰写 | 结构完整，语言生动 |
| 文案优化 | 精简有力，转化导向 |
| 效果追踪与迭代 | 指标明确，建议可操作 |

### 3.2 审核流程

1. **自我审核**：各阶段完成后，执行人员需进行自我审核
2. **交叉审核**：重要文案需经过至少1人交叉审核
3. **最终审核**：最终文案需经过策略制定者审核

### 3.3 常见问题与解决方案

| 问题 | 解决方案 |
|------|----------|
| 卖点与痛点不匹配 | 重新分析用户反馈，调整卖点挖掘方向 |
| 文案缺乏吸引力 | 优化标题和钩子，增强场景代入感 |
| 转化率低 | 重新分析用户需求，调整说服逻辑 |
| 渠道表现差异大 | 针对不同渠道重新优化文案 |

## 8. 附录

### 8.1 术语表

| 术语 | 解释 |
|------|------|
| CTR | 点击率，点击次数/展示次数 |
| CVR | 转化率，转化次数/点击次数 |
| CPC | 单次点击成本，总花费/点击次数 |
| CPA | 单次转化成本，总花费/转化次数 |
| CTA | 行动召唤，引导用户采取行动的文案 |
| UGC | 用户生成内容 |
| ROI | 投资回报率 |

### 8.2 法律合规

- 严格遵守《广告法》等相关法律法规

- 尊重知识产权，避免侵权
- 保护用户隐私，不泄露个人信息
