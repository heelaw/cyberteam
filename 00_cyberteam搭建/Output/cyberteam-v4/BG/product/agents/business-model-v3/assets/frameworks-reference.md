# 商业模式框架参考

本文档提供常见商业模式的分析框架、核心指标和最佳实践。

## 框架使用指南

### 如何选择合适的框架？

根据业务特征选择：

```
1. 确定核心价值交付方式
   ├── 产品销售 → 电商DTC框架
   ├── 服务提供 → O2O预定框架
   └── 软件服务 → SaaS订阅框架

2. 确定收入模式
   ├── 一次性交易 → 电商DTC框架
   ├── 订阅制 → SaaS订阅框架
   └── 预定制服务 → O2O预定框架

3. 确定用户交互方式
   ├── 线上完成 → 电商DTC或SaaS
   └── 线上线下结合 → O2O预定框架
```

## 预置框架详解

### 1. O2O预定模式

**适用场景**:
- 本地服务（美容、健身、教育、医疗）
- 高客单价定制服务
- 需要预约的服务业
- 体验导向型服务

**核心特征**:
- 线上获客，线下履约
- 预收款模式（现金流优势）
- 产能约束（服务能力有限）
- 爽约风险

**关键指标**:
- CAC: $50-500
- LTV: $500-5000
- 爽约率: < 10%
- 预收款比例: 80-100%
- LTV:CAC > 3:1

**标准转化路径**:
```
触达 → 完播 → 兴趣 → 预约 → 到店 → 交付 → 复购
100%   30%    8%     6%    90%   100%   45%/年
```

**关键风险**:
- 高爽约率 → 缓解: 预收款、预约提醒、弹性改期
- 履约能力不足 → 缓解: 产能规划、供应商多元化
- 服务质量不稳定 → 缓解: 标准化培训、质检体系

**详细配置**: `frameworks/o2o-preorder.yaml`

### 2. SaaS订阅模式

**适用场景**:
- 企业级软件服务
- 云平台服务
- 生产力工具
- 数据分析工具

**核心特征**:
- 订阅制收入
- 持续交付价值
- 高毛利
- 规模效应

**关键指标**:
- MRR/ARR增长率
- Churn Rate: < 5%（月度）
- LTV:CAC > 3:1
- 回本周期: < 12个月

**标准转化路径**:
```
网站访问 → 注册 → 激活 → 试用 → 付费转化
100%      5%    40%   30%    20%
```

**关键风险**:
- 高流失率 → 缓解: 增加转换成本、持续价值交付
- 增长放缓 → 缓解: 扩展市场、产品创新
- 竞争加剧 → 缓解: 差异化、护城河建设

**详细配置**: `frameworks/saas-subscription.yaml`

### 3. 电商DTC模式

**适用场景**:
- 垂直品类电商
- 自有品牌电商
- 订阅制电商
- 跨境电商

**核心特征**:
- 直接面向消费者
- 一次性交易
- 复购驱动
- 库存管理

**关键指标**:
- AOV: $50-200
- 复购率: 30-50%
- 购买频率: 2-6次/年
- 毛利率: 20-50%

**标准转化路径**:
```
曝光 → 点击 → 访问 → 加购 → 结账 → 购买
100%  3%    90%   10%   30%   70%
```

**关键风险**:
- 库存积压 → 缓解: 需求预测、小批量测试
- 低复购率 → 缓解: 会员体系、推荐算法
- 高CAC → 缓解: 提高复购、口碑营销

**详细配置**: `frameworks/ecommerce-dtc.yaml`

## 自定义框架创建

### 步骤指南

**1. 定义框架基本信息**:
```yaml
name: "框架名称"
version: "1.0"
description: "框架描述"
applicable_scenarios:
  - "场景1"
  - "场景2"
```

**2. 设定核心指标**:
```yaml
key_metrics:
  acquisition:
    - metric: CAC
      benchmark: "$X-Y"
  retention:
    - metric: 留存率
      benchmark: "X-Y%"
```

**3. 设计转化路径**:
```yaml
standard_funnel_steps:
  - step: 1
    name: "步骤名"
    typical_rate: X-Y
```

**4. 定义成本结构**:
```yaml
cost_structure:
  variable_costs:
    - item: "成本项"
      range: "X-Y% of revenue"
  profit_margins:
    - metric: "毛利率"
      range: "X-Y%"
```

**5. 识别风险与机会**:
```yaml
common_risks:
  - risk: "风险描述"
    mitigation: "缓解措施"
growth_strategies:
  - strategy: "策略描述"
    kpis: ["KPI1", "KPI2"]
```

## 最佳实践

### 框架选择

1. **匹配核心业务**: 框架必须与核心业务模式匹配
2. **考虑发展阶段**: 早期阶段可简化，成熟阶段需精细化
3. **保持灵活性**: 框架是指南，不是规则，可根据情况调整
4. **持续迭代**: 随业务发展不断优化框架

### 框架应用

1. **完整使用**: 首次分析建议完整使用框架
2. **重点突出**: 根据业务特点关注重点部分
3. **数据驱动**: 基于真实数据填写框架
4. **对标分析**: 与行业基准对比找出差距
