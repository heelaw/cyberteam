## 核心框架

### 框架一：战略-目标-指标三层映射模型

```
全局战略 ←→ 业务目标 ←→ 执行指标
   ↑            ↑            ↑
 愿景层      战术层       行动层
```

**三层映射原则**：
- 全局战略决定局部目标的战略方向
- 业务目标必须承接战略的核心诉求
- 执行指标必须能度量目标达成

### 框架二：局部目标四要素模型

| 要素 | 说明 | 问句 |
|------|------|------|
| **边界** | 明确局部业务单元的范围和职责 | 这个目标归谁管？ |
| **贡献** | 明确对全局目标的贡献方式 | 如何支撑整体战略？ |
| **时窗** | 明确目标达成的时间周期 | 多久要看到结果？ |
| **标准** | 明确目标达成的衡量标准 | 怎样算完成了？ |

### 框架三：目标优先级矩阵

```
                    影响度
              高           低
          ┌─────────┬─────────┐
      高  │  核心    │  支撑    │
          │  目标    │  目标    │
          ├─────────┼─────────┤
 Urgency  │  突击    │  削减    │
      低  │  目标    │  目标    │
          └─────────┴─────────┘
```

## 执行步骤

### Step 1: 理解全局战略上下文

**目的**：确保局部目标与全局战略保持一致

**操作流程**：

1. **获取战略文档**
   - 阅读公司/团队的战略规划文件
   - 理解战略的愿景、使命、核心目标
   - 识别战略的关键成功因素

2. **提取战略关键词**
   - 列出3-5个战略核心词
   - 识别战略重点突破方向
   - 理解战略约束条件

3. **问答检验理解**

   | 问题 | 回答要点 |
   |------|----------|
   | 战略要解决的核心问题是什么？ | 痛点 + 机会 |
   | 战略成功的标志是什么？ | 核心指标变化 |
   | 战略的时间窗口是什么？ | 短/中/长期 |
   | 战略的资源边界是什么？ | 人/财/物约束 |

**输出**：战略理解备忘录（300字）

---

### Step 2: 识别局部业务单元

**目的**：明确目标制定的责任主体和范围边界

**操作流程**：

1. **绘制业务单元地图**

   ```
   ┌─────────────────────────────────────────────────────┐
   │                    公司/团队整体                      │
   ├─────────────┬─────────────┬─────────────┬──────────┤
   │  业务单元A  │  业务单元B  │  业务单元C  │  业务单元D │
   ├──────┬──────┼──────┬──────┼──────┬──────┤          │
   │ 子单元 │子单元│ 子单元 │子单元│ 子单元 │子单元│          │
   │  A1   │  A2  │  B1   │  B2  │  C1   │  C2  │          │
   └──────┴──────┴──────┴──────┴──────┴──────┴──────────┘
   ```

2. **定义局部单元特征**

   | 特征维度 | 填写内容 |
   |----------|----------|
   | 单元名称 | [具体名称] |
   | 负责人员 | [责任人] |
   | 核心职能 | [3个核心职责] |
   | 服务对象 | [内/外部客户] |
   | 资源权限 | [可调配资源] |

3. **识别单元间的依赖关系**

   - 上游依赖：谁给我输入？
   - 下游输出：谁依赖我的产出？
   - 平行协作：谁和我并行工作？

---

### Step 3: 制定SMART局部目标

**目的**：产出具体、可衡量、可实现、相关、有时限的业务目标

**SMART目标检查表**：

| 字母 | 要求 | 不符合示例 | 符合示例 |
|------|------|------------|----------|
| **S** Specific 具体 | 目标清晰明确 | "提升业绩" | "新增100家合作渠道" |
| **M** Measurable 可衡量 | 有量化指标 | "客户满意度提升" | "NPS评分从40提升至55" |
| **A** Achievable 可实现 | 有资源支持 | "业绩翻10倍" | "业绩增长30%（行业平均15%）" |
| **R** Relevant 相关 | 与战略相关 | "开发内部聊天工具" | "搭建客户反馈系统" |
| **T** Time-bound 有时限 | 有明确截止 | "尽快完成" | "Q3末完成上线" |

**目标编写模板**：

```
在 [时间周期] 内，[责任单元] 需要 [具体行动]，
达到 [量化指标]，以支撑 [战略贡献点]。

示例：
在 Q2（4-6月）内，用户增长团队需要 通过内容营销+裂变活动，
新增注册用户 5万（环比Q1增长25%），以支撑公司年度
用户规模翻倍的核心战略。
```

**多目标组合模板**：

| 序号 | 目标描述 | 指标 | 权重 | 责任人 | 截止日期 |
|------|----------|------|------|--------|----------|
| 1 | [目标1] | [KPI] | 30% | @ | YYYY-MM-DD |
| 2 | [目标2] | [KPI] | 40% | @ | YYYY-MM-DD |
| 3 | [目标3] | [KPI] | 30% | @ | YYYY-MM-DD |

---

### Step 4: 目标评审与对齐

**目的**：确保目标既具挑战性又可实现

**评审维度**：

1. **战略对齐度评审**
   - 问：每个目标如何支撑战略？
   - 答：（填写支撑逻辑）

2. **资源充足度评审**
   - 问：现有资源能否支撑目标？
   - 问：需要争取哪些额外资源？

3. **风险可控度评审**
   - 问：目标达成的核心风险是什么？
   - 问：风险应对预案是什么？

4. **团队共识度评审**
   - 问：目标责任人是否认可？
   - 问：相关协作方是否清楚？

**评审输出表**：

| 目标 | 战略对齐 | 资源充足 | 风险可控 | 共识达成 | 综合评级 |
|------|----------|----------|----------|----------|----------|
| 目标1 | ☑️ 强 | ☑️ 充足 | ⚠️ 中等 | ☑️ 共识 | 通过 |
| 目标2 | ☑️ 强 | ❌ 不足 | ☑️ 可控 | ⚠️ 待确认 | 需修改 |
| 目标3 | ⚠️ 弱 | ☑️ 充足 | ☑️ 可控 | ☑️ 共识 | 需加强 |

## 使用示例

### 完整场景：电商公司季度局部目标制定

**背景**：
- 全局战略：2026年实现GMV 100亿，年增长率50%
- 当前状态：Q1 GMV 20亿，Q2目标需达到28亿
- 局部单元：用户增长部门

**Step 1: 理解全局战略**
```
战略核心词：GMV增长、用户规模、市场份额
战略成功标志：GMV年增50%，新用户增长100万
战略时间窗口：2026年底完成100亿GMV
```

**Step 2: 识别局部单元**
```
单元名称：用户增长部
核心职能：用户获取、激活、留存
服务对象：潜在用户、存量用户
```

**Step 3: 制定SMART目标**
```
目标1：在Q2（4-6月）内，通过内容营销+付费投放，
       新增注册用户15万，环比Q1增长50%

目标2：在Q2内，优化新用户首购转化路径，
       将首购率从12%提升至18%

目标3：在Q2内，策划2场大型裂变活动，
       活动期间日均新增用户突破1万
```

**Step 4: 评审与对齐**
```
战略对齐：3个目标均直接支撑GMV增长战略
资源评估：需增加30%营销预算
风险识别：付费投放ROI可能不达预期
共识达成：部门全员认可，已与产品协作对齐
```

## 检查清单

### 目标制定前
- [ ] 已理解全局战略方向
- [ ] 已识别所有相关局部单元
- [ ] 已收集历史数据作为基准
- [ ] 已确认资源边界条件

### 目标制定中
- [ ] 每个目标符合SMART原则
- [ ] 目标之间相互协调不冲突
- [ ] 目标权重分配合理
- [ ] 已设定阶段性里程碑

### 目标制定后
- [ ] 已与责任人确认目标
- [ ] 已与协作方对齐依赖
- [ ] 已建立复盘机制
- [ ] 已制定目标调整预案
