# 成功三要素分析

## 身份定位

```
┌─────────────────────────────────────────────────────────────┐
│  🎯 成功三要素分析                                          │
│  用途: 诊断方向/路径/执行，识别成败关键                    │
│  配套Agent: 策略执行Agent                                    │
└─────────────────────────────────────────────────────────────┘
```

---

## 核心框架

```
成功 = f(方向 × 路径 × 执行)

方向: 做什么是对的
路径: 怎么做是对的
执行: 做得够不够好

三要素全备 → 成功概率高 (>70%)
缺一 → 有风险 (40-70%)
缺二 → 需要重新规划 (<40%)
缺三 → 基本不可行 (<20%)
```

---

## 执行步骤

### Step 1: 方向诊断

```
检查维度:
  □ 市场时机
    - 市场是否成熟
    - 时机是否正确
    - 外部环境是否支持

  □ 战略选择
    - 是否选择了正确的战场
    - 是否选择了正确的打法
    - 是否在正确的赛道上

  □ 资源配置
    - 投入的资源是否足够
    - 资源配置是否合理
    - 资源与目标是否匹配

方向诊断结论:
  ✅ 方向正确 → 可继续
  ⚠️ 方向存疑 → 需要重新审视
  ❌ 方向错误 → 需重新规划
```

### Step 2: 路径诊断

```
检查维度:
  □ 路径清晰度
    - 从起点到终点的路径是否清晰
    - 里程碑是否明确
    - 每个阶段的目标是否清晰

  □ 依赖关系
    - 路径上的依赖是否理清
    - 哪些是前置条件
    - 哪些可以并行

  □ 关键节点
    - 哪些是关键节点
    - 节点失败的影响
    - 是否有备选路径

路径诊断结论:
  ✅ 路径清晰 → 可执行
  ⚠️ 路径模糊 → 需要细化
  ❌ 路径不通 → 需要重新设计
```

### Step 3: 执行诊断

```
检查维度:
  □ 团队能力
    - 团队能力是否匹配
    - 是否有能力缺口
    - 是否需要引入外部能力

  □ 执行机制
    - 是否有明确的执行计划
    - 是否有足够的资源支持
    - 是否有监控和纠偏机制

  □ 激励机制
    - 团队动力是否充足
    - 是否有足够的激励
    - 是否能持续执行

执行诊断结论:
  ✅ 执行到位 → 成功概率高
  ⚠️ 执行一般 → 需要加强
  ❌ 执行不足 → 需要先解决执行问题
```

## 三要素评估矩阵

```
┌──────────┬──────────────────────────────────────────────────┐
│ 三要素    │ 诊断结论                                        │
├──────────┼──────────────────────────────────────────────────┤
│ 方向      │ 正确 / 存疑 / 错误                              │
│ 路径      │ 清晰 / 模糊 / 不通                              │
│ 执行      │ 到位 / 一般 / 不足                              │
├──────────┼──────────────────────────────────────────────────┤
│ 综合判断  │                                                  │
│ 成功概率  │ ≥70% / 40-70% / <40% / <20%                   │
│ 主要短板  │ [识别出的主要问题]                              │
│ 改进建议  │ [建议的改进方向]                               │
└──────────┴──────────────────────────────────────────────────┘
```

---

## 典型问题诊断

```
问题: 方向对了，但做不成
→ 路径问题: 重新设计路径，找到更清晰的路径

问题: 方向路径都对，但没进展
→ 执行问题: 检查团队、能力、资源、激励机制

问题: 做了很多，但没效果
→ 方向问题: 重新审视市场、时机、战略选择

问题: 什么都做了，但就是差一口气
→ 多要素问题: 找出最弱的一环，重点突破
```

## 使用示例

```
输入: 公司数字化转型项目推进困难

诊断:
  方向: 存疑
    - 战略目标清晰但优先级不明
    - 资源投入与目标不匹配

  路径: 模糊
    - 缺少清晰的里程碑
    - 各部门各自为战
    - 依赖关系不清晰

  执行: 一般
    - 团队能力基本具备
    - 缺少一把手强力推动
    - 激励机制不明确

综合判断:
  成功概率: 40-70% (有风险)
  主要短板: 路径不清晰
  改进建议:
    1. 重新设计清晰的路径
    2. 明确里程碑和依赖
    3. 强化一把手工程
```
