## 核心框架

### 框架一：目标漏斗拆解模型

```
                    ┌───────────────┐
                    │   最终目标     │  Level 4: 战略目标
                    │   (北极星指标) │
                    └───────┬───────┘
                            │
                    ┌───────┴───────┐
                    │   阶段目标     │  Level 3: 里程碑目标
                    │   (里程碑)    │
                    └───────┬───────┘
                            │
                    ┌───────┴───────┐
                    │   子目标       │  Level 2: 战术目标
                    │   (战术分解)  │
                    └───────┬───────┘
                            │
                    ┌───────┴───────┐
                    │   行动计划     │  Level 1: 行动目标
                    │   (可执行任务) │
                    └───────────────┘
```

**拆解原则**：
- 每层漏斗的子项之和必须100%支撑上一层
- 层级之间必须有明确的逻辑推导关系
- 底层行动必须可直接执行和度量

### 框架二：行动要素五问模型

| 要素 | 问题 | 回答模板 |
|------|------|----------|
| **做什么** | 行动的具体内容是什么？ | [动词]+[对象]+[标准] |
| **谁来做** | 行动的责任人是谁？ | [角色/人员] |
| **怎么做** | 行动的执行方式是什么？ | [方法/工具/流程] |
| **何时做** | 行动的时间窗口是什么？ | [开始]-[结束] |
| **做到什么程度** | 行动的质量标准是什么？ | [量化指标] |

### 框架三：任务依赖矩阵

| 任务 | 前置任务 | 所需资源 | 输出物 | 风险 |
|------|----------|----------|--------|------|
| 任务A | 无 | 人力×2 | 报告A | 低 |
| 任务B | 任务A | 人力×1, 预算×1万 | 报告B | 中 |
| 任务C | 任务A, 任务B | 人力×3 | 产品C | 低 |
| 任务D | 任务B | 人力×1 | 方案D | 高 |

## 执行步骤

### Step 1: 明确目标定义与成功标准

**目的**：统一目标认知，建立拆解基准

**操作流程**：

1. **目标定义标准化**
   
   使用目标定义模板：
   ```
   [责任主体] 在 [时间范围] 内，通过 [核心路径]，
   达成 [量化指标] 的 [目标类型]。
   
   示例：
   市场部 在 Q2（4-6月）内，通过 内容营销+渠道投放，
   达成 新增获客 5万 的 用户增长目标。
   ```

2. **设定成功标准**
   
   | 标准维度 | 具体指标 | 基准值 | 目标值 |
   |----------|----------|--------|--------|
   | 数量标准 | [如：新增用户数] | [基准] | [目标] |
   | 质量标准 | [如：转化率] | [基准] | [目标] |
   | 时效标准 | [如：完成时间] | [基准] | [目标] |
   | 成本标准 | [如：获客成本] | [基准] | [目标] |

3. **识别约束边界**
   
   - 时间边界：必须在何时完成？
   - 资源边界：可调配多少人/财/物？
   - 质量边界：最低要达到什么标准？
   - 范围边界：哪些是做的，哪些是不做的？

---

### Step 2: 设计目标-行动映射关系

**目的**：建立从目标到行动的系统性推导

**操作流程**：

1. **逆向推导法（从结果倒推行动）**
   
   ```
   最终目标：达成5万新增用户
   
   问题拆解：
   如何获得5万新用户？
   ├─ 渠道投放：贡献3万 → 需要多少预算/投放量？
   ├─ 内容营销：贡献1.5万 → 需要多少内容/曝光？
   └─ 活动裂变：贡献0.5万 → 需要什么活动形式？
   ```

2. **正向构建法（从行动推导结果）**
   
   ```
   可执行行动：
   ├─ 行动A：投放10个信息流渠道
   │   预计产出：2.5万用户（单渠道2500×10）
   ├─ 行动B：产出100篇优质内容
   │   预计产出：1.5万用户（单篇150×100）
   └─ 行动C：策划2场裂变活动
       预计产出：1万用户（单场5000×2）
   合计：5万用户 ✓
   ```

3. **交叉验证**
   
   两种方法推导的结果应该一致，若不一致需调整。

---

### Step 3: 构建任务分解结构（WBS）

**目的**：将行动分解为可管理的最小任务单元

**WBS编写规范**：

```
格式：[层级编码] [任务名称] [负责人] [工时] [截止日期]

示例：
1.0    Q2用户增长项目                    @张三    90天
1.1    渠道投放执行                     @李四    60天
1.1.1  渠道筛选与洽谈                   @李四    5天    04-05
1.1.2  投放素材制作                     @王五    10天   04-15
1.1.3  投放计划制定与上线              @李四    3天    04-18
1.1.4  投放数据监控与优化               @李四    42天   04-19~06-01
1.2    内容营销执行                     @赵六    80天
...（继续分解）
```

**任务粒度标准**：
- 最小任务：2小时 ~ 3天工作量
- 单任务不超过1周，否则需继续分解
- 每个任务有明确产出物

**任务依赖标记**：
- FS（Finish-Start）：A完成后开始B
- SS（Start-Start）：A开始后开始B
- FF（Finish-Finish）：A完成后B完成

### Step 5: 建立追踪与复盘机制

**目的**：确保计划可执行、可监控、可调整

**追踪机制**：

| 周期 | 追踪内容 | 责任人 | 输出 |
|------|----------|--------|------|
| 日 | 关键指标数据 | 数据分析师 | 日报 |
| 周 | 任务完成率/里程碑达成 | 项目经理 | 周报 |
| 月 | 目标达成进度/偏差分析 | 部门负责人 | 月报 |

**复盘框架**：

```
本次复盘：[目标名称]
复盘时间：[日期]

1. 目标达成情况
   - 计划值：X
   - 实际值：Y
   - 偏差率：(Y-X)/X

2. 成功因素（做得好的）
   - [因素1]
   - [因素2]

3. 失败因素（需改进的）
   - [因素1]
   - [因素2]

4. 经验教训
   - [可复用的方法]
   - [需避免的坑]

5. 下一步行动
   - [调整措施]
   - [资源追加]
```

## 使用示例

### 完整场景：月度销售目标拆解

**背景**：
- 目标：销售部6月达成销售额300万
- 团队：5名销售
- 产品线：3条

**Step 1: 明确目标定义**
```
销售部 在 2026年6月内，通过 团队协作+客户开发+跟进转化，
达成 销售额300万 的 业绩目标。

成功标准：
- 数量：300万销售额
- 客户：新签客户≥30家
- 转化：商机转化率≥20%
```

**Step 2: 目标-行动映射**
```
300万销售额 = 客单价×客户数

路径A（多客户小单）：
  客单价5万 × 60家新客户 = 300万
  需开发60家新客户

路径B（少客户大单）：
  客单价15万 × 20家新客户 = 300万
  需开发20家新客户

采用路径C（组合）：
  大客户5家（75万）+ 中客户10家（100万）+ 小客户25家（125万）= 300万
```

**Step 3: WBS任务分解**
```
1.0    6月销售目标达成                @销售总监   30天
1.1    目标分解与分配                 @销售总监   2天
1.2    重点客户开发                   @5名销售    28天
1.2.1  大客户开发（≥15万）           @张三、李四  15家
1.2.2  中客户开发（8-15万）           @王五、赵六  10家
1.2.3  小客户开发（<8万）             @孙七       35家
1.3    销售跟进与成交                 @5名销售    28天
1.4    合同签订与回款                 @5名销售    30天
1.5    周度销售例会                   @销售总监   4次
```

**Step 4: 资源与时间**
```
人力分配：
- 张三：大客户开发，2家/周
- 李四：大客户开发，2家/周
- 王五：中客户开发，4家/周
- 赵六：中客户开发，3家/周
- 孙七：小客户开发，9家/周

时间节奏：
W1：完成目标分配，启动大客户开发
W2-W3：密集开发，目标完成50%
W4：查漏补缺，冲刺目标
```
