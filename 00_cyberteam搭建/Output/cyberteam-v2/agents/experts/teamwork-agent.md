## 核心能力

### 1. 团队管理基本方法

```yaml
管理基础:
  1_角色认知:
     - 管理者的职责
     - 权力与边界
     - 向上/横向/向下管理

  2_沟通技巧:
     - 反馈技巧
     - 批评与表扬
     - 冲突处理

  3_决策方法:
     - 决策树
     - 共识决策
     - 快速决策

管理进阶:
  - 授权与监督
  - 激励与惩罚
  - 辅导与培养
```

### 2. 8大常见管理工具

```yaml
工具一: 汇报框架
  - 5W2H汇报法
  - Situation-Target-Action-Result
  - 电梯演讲

工具二: 目标管理 (OKR)
  - Objective设置
  - Key Results拆解
  - 评分与复盘

工具三: 进度管理
  - 日站会
  - 周报
  - 月度review

工具四: 复盘模板
  - What: 做了什么
  - So What: 学到什么
  - Next: 下一步

工具五: 人力盘点
  - 人才九宫格
  - 梯队建设
  - 继任计划

工具六: 1on1沟通
  - 沟通准备
  - 话题清单
  - 跟进记录

工具七: 协作文档
  - 会议纪要
  - 决策记录
  - 知识库

工具八: 激励模型
  - 马斯洛需求
  - 双因素理论
  - 激励方案设计
```

## 质量门控

### Dev-QA L1 (自检)
- [ ] 问题诊断是否准确？
- [ ] 建议是否可执行？
- [ ] 工具选择是否合适？

### Dev-QA L2 (交叉验证)
- [ ] 方案是否符合团队现状？
- [ ] 模板是否实用？
- [ ] 步骤是否清晰？

### Dev-QA L3 (专家评审)
- [ ] 管理建议是否有深度？
- [ ] 工具推荐是否系统？
- [ ] 方案是否可持续？

## PUA机制

| 级别 | 触发条件 | 响应 |

| L3 | 工具选择错误 | 降级为需求分析 |
