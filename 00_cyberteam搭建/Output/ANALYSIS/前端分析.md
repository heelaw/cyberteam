## 二、现有页面和功能

### 2.1 Chat 页面 (`/pages/chat/index.tsx`)
**功能:**
- 多会话管理 (以 working_directory 分组)
- 实时消息展示 (user/assistant/system 三种角色)
- Claude Code CLI 集成 (`/opt/homebrew/bin/claude`)
- 目录浏览器 (内置 + 系统文件夹选择器)
- Provider 选择器
- 消息持久化 (JSON 文件存储)

**核心 Tauri 调用:**
- `invoke('send_to_claude')` - 发送消息到 Claude CLI
- `invoke('list_sessions')` / `invoke('create_session')` / `invoke('delete_session')`
- `invoke('get_session_messages')`
- `invoke('list_providers')`
- `invoke('browse_directory')` - 浏览目录
- `invoke('validate_directory')` - 验证目录
- `invoke('get_home_directory')`
- `listen('claude-output')` - 监听 Claude 输出事件

### 2.2 Settings 页面 (`/pages/settings/index.tsx`)
**功能:**
- API Provider CRUD
- Provider 预设模板 (Anthropic, MiniMax, OpenRouter, Moonshot, Kimi)
- 连接测试
- 模型映射 (default/reasoning/small)

**核心 Tauri 调用:**
- `invoke('list_providers')` / `invoke('create_provider')` / `invoke('update_provider')` / `invoke('delete_provider')`
- `invoke('test_provider')`

### 2.3 Agents 页面 (`/pages/agents/index.tsx`)
**功能:**
- Agent 列表展示 (CEO, COO, CFO, CTO, 信息部, 质疑者)
- 状态显示 (idle/running/error)
- 刷新 / 启动 / 配置 / 日志按钮

**注意:** 当前 `list_agents` 是硬编码返回默认值，**未连接真实 Agent 系统**

### 2.4 Skills 页面 (`/pages/skills/index.tsx`)
**功能:**
- 技能列表展示
- 按部门分类筛选 (信息部/策略部/内容部/设计部/工程部/运营部)
- 技能启用/禁用切换

**注意:** 当前 `toggle_skill` 仅打印日志，**未连接真实 Skill 系统**

## 三、与 v4 后端的连接方式

### 当前架构 (独立桌面应用)

```
┌─────────────────────────────────────────────────────────────┐
│                    CyberTeam Desktop                         │
├─────────────────────────────────────────────────────────────┤
│  React Frontend (TypeScript + Tailwind)                      │
│    │                                                         │
│    │ invoke() / listen()                                     │
│    ▼                                                         │
│  Tauri IPC Bridge                                            │
│    │                                                         │
│    │ Rust Commands                                           │
│    ▼                                                         │
│  Rust Backend (main.rs)                                     │
│    ├── send_to_claude() → Claude Code CLI                    │
│    ├── JSON File Database (providers/sessions/messages)      │
│    └── File System Operations                                │
└─────────────────────────────────────────────────────────────┘
```

### 关键发现

**1. NOT 连接 CyberTeam v4 后端**
- 桌面端通过 `invoke('send_to_claude')` 直接调用 Claude Code CLI
- 不经过任何 HTTP API
- 不连接 CyberTeam-v4 的任何后端服务

**2. 数据存储**
- 使用 JSON 文件存储在 `~/.local/share/cyberteam-desktop/`
  - `providers.json` - API Provider 配置
  - `sessions.json` - 会话列表
  - `messages.json` - 聊天消息
  - `settings.json` - 设置

**3. Claude CLI 集成**
```rust
cmd.envs(&env_vars);
cmd.args(["--dangerously-skip-permissions", "--print"]);
cmd.stdin(Stdio::piped());
cmd.stdout(Stdio::piped());
```
- 使用 `--print` 模式单次交互
- 使用 `--dangerously-skip-permissions` 跳过权限
- 环境变量传递 API Key 和 Base URL

## 四、需要补充什么才能实现完整功能

### 4.1 缺失的核心功能

| 功能 | 当前状态 | 需要补充 |
|------|----------|----------|
| CyberTeam v4 后端连接 | ❌ 无 | 需要新增 HTTP 客户端，连接 v4 API |
| Agent 真实管理 | ❌ 硬编码 | 需要连接 Agent 服务 |
| Skill 真实管理 | ❌ 硬编码 | 需要连接 Skill 服务 |
| 多轮对话上下文 | ⚠️ 部分 | 仅支持单次 ` --print` 调用 |
| 流式输出 | ❌ 无 | Claude CLI `--print` 不支持流式 |
| 会话历史加载 | ⚠️ 仅存储 | 未实现从后端加载历史 |

### 4.2 架构改造建议

**方案 A: 混合架构 (推荐)**
```
┌──────────────────────────────────────────────────────────────┐
│                    CyberTeam Desktop                          │
├──────────────────────────────────────────────────────────────┤
│  React Frontend                                               │
│    │                              │                           │
│    │ invoke()                     │ fetch() / WebSocket        │
│    ▼                              ▼                           │
│  Tauri IPC                          HTTP Client               │
│    │ (本地系统操作)                 │ (连接 v4 后端)             │
│    ▼                              ▼                           │
│  Rust Backend                       CyberTeam v4 Backend      │
│    ├── 文件系统                      ├── /api/agents           │
│    ├── Claude CLI 调用              ├── /api/skills           │
│    └── JSON 存储                    ├── /api/sessions         │
│                                        └── SSE 流式           │
└──────────────────────────────────────────────────────────────┘
```

**需要补充的模块:**

1. **HTTP 客户端层** (前端或 Tauri 层)
   - 添加 `fetch` 或 `reqwest` (Rust) 客户端
   - 连接 CyberTeam v4 API (假设运行在 `http://localhost:8080` 或类似地址)

2. **Agent 管理**
   - 替换硬编码 `list_agents` 为真实 API 调用
   - 实现 Agent 启动/停止/日志查看

3. **Skill 管理**
   - 替换 `toggle_skill` 为真实 API 调用
   - Skill 同步到后端

4. **会话同步**
   - 将本地会话同步到 v4 后端
   - 支持会话历史在多设备间同步

5. **流式输出** (可选)
   - Claude CLI `--print` 不支持流式
   - 如需流式，考虑使用 Claude API 原生 SDK

### 4.3 具体改造文件

| 文件 | 改造内容 |
|------|----------|
| `src/pages/agents/index.tsx` | 连接 v4 `/api/agents` |
| `src/pages/skills/index.tsx` | 连接 v4 `/api/skills` |
| `src/pages/chat/index.tsx` | 可选：SSE 流式输出 |
| `src-tauri/src/main.rs` | 新增 HTTP 客户端命令 |
| `src-tauri/src/lib.rs` | 新增 Agent/Skill 服务调用 |

## 五、总结

### 当前状态
- ✅ 完整的 Tauri 桌面应用框架
- ✅ Chat 功能可用 (基于 Claude CLI)
- ✅ Provider 配置管理
- ❌ **未连接 CyberTeam v4 后端**
- ❌ Agent/Skill 页面是占位符

### 与 v4 的关系
CyberTeam Desktop 是一个**独立运行的桌面应用**，通过 Claude Code CLI 与 AI 交互。要连接 CyberTeam v4 后端服务，需要新增 HTTP 客户端层来实现完整的多 Agent 协作功能。
