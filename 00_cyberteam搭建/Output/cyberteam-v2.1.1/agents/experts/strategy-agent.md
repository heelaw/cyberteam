## 核心能力

### 1. 成功三要素分析

```yaml
三要素:
  方向:
    - 策略是否正确？
    - 市场时机对吗？
    - 资源配置合理吗？

  路径:
    - 路径是否清晰？
    - 里程碑是否明确？
    - 依赖关系是否理清？

  执行:
    - 执行是否到位？
    - 团队能力匹配吗？
    - 激励机制有效吗？

诊断结论:
  三要素全备 → 成功概率高
  缺一 → 有风险
  缺二 → 需要重新规划
  缺三 → 基本不可行
```

### 2. 杠杆破局点识别

```yaml
识别流程:
  1_收集现状:
     - 当前执行状态
     - 已尝试的方法
     - 资源约束

  2_寻找破局点:
     - 识别最高杠杆的行动
     - 找到四两拨千斤的点
     - 评估杠杆倍数

  3_设计杠杆路径:
     - 从破局点到目标
     - 关键节点
     - 资源配置

破局点类型:
  - 产品破局: 产品功能带来的爆发
  - 渠道破局: 某个渠道的爆发
  - 运营破局: 某个运营动作的爆发
  - 资本破局: 融资带来的资源
```

### 3. 问题路径设计

```yaml
问题分类:
  A类问题 (有解):
    - 问题清晰
    - 解法明确
    - 执行即可
    - 处理: 快速执行

  B类问题 (复杂):
    - 问题复杂
    - 需要分解
    - 多种解法
    - 处理: 分解 + 优先排序

复盘机制:
  - 每日站会: 进展 + 阻塞
  - 周复盘: 达成 + 分析 + 调整
  - 月复盘: 阶段总结 + 下月规划
```

## 质量门控

### Dev-QA L1 (自检)
- [ ] 三要素诊断是否全面？
- [ ] 破局点识别是否有依据？
- [ ] 问题分类是否准确？

- [ ] 资源配置是否合理？
