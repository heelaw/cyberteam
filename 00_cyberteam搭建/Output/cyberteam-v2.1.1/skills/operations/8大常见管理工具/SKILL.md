## 工具详解

### 工具 #1: OKR目标管理

**定义**：Objectives and Key Results，目标与关键结果管理法

**核心结构**：
```
O（Objective）- 目标：定性描述，激励人心
   └─ KR1（Key Result）：关键结果1（量化）
   └─ KR2（Key Result）：关键结果2（量化）
   └─ KR3（Key Result）：关键结果3（量化）
```

**OKR模板**：

```
【季度OKR模板】

O1: [定性目标，鼓舞人心]
KR1: [可量化的关键结果1]
KR2: [可量化的关键结果2]
KR3: [可量化的关键结果3]

O2: [第二个目标]
KR1: [关键结果]
KR2: [关键结果]

执行检查（每周期）：
├─ 进度：[0-100%]
├─ 信心指数：[1-10]
└─ 状态：[正常/有风险/已偏离]
```

**使用场景**：
- 公司/团队战略对齐
- 个人目标设定
- 项目目标追踪

**填写示例**：

```
O1: 成为行业用户增长标杆
KR1: 季度末DAU达到500万（当前380万）
KR2: 用户留存率提升至45%（当前38%）
KR3: 付费转化率达8%（当前6.5%）
```

**使用要点**：
- [ ] O有挑战性（80%完成=成功）
- [ ] KR可量化、有时限
- [ ] 周度check-in回顾
- [ ] 透明公开全员可见

### 工具 #3: 看板/看板管理

**定义**：可视化工作流程的任务管理工具（Kanban）

**看板结构**：

```
┌────────────┬────────────┬────────────┬────────────┐
│   To Do    │   Doing    │   Review   │   Done    │
│  (待办)    │  (进行中)  │  (审核中)  │  (已完成)  │
├────────────┼────────────┼────────────┼────────────┤
│ ┌────────┐│ ┌────────┐│ ┌────────┐│ ┌────────┐│
│ │ 任务A  ││ │ 任务C  ││ │ 任务E  ││ │ 任务B  ││
│ │ @张三  ││ │ @李四  ││ │ @王五  ││ │ @张三  ││
│ │ P1     ││ │ P2     ││ │ P1     ││ │ P2     ││
│ └────────┘│ └────────┘│ └────────┘│ └────────┘│
│ ┌────────┐│            │            │ ┌────────┐│
│ │ 任务D  ││            │            │ │ 任务F  ││
│ │ @赵六  ││            │            │ │ @李四  ││
│ │ P2     ││            │            │ │ P1     ││
│ └────────┘│            │            │ └────────┘│
└────────────┴────────────┴────────────┴────────────┘
```

**卡片模板**：

```
┌─────────────────────────────────┐
│ [任务名称]                      │
├─────────────────────────────────┤
│ 描述：[具体要做什么]             │
│ 负责人：@XXX                    │
│ 优先级：P0/P1/P2/P3             │
│ 截止：YYYY-MM-DD                │
│ 预计工时：X小时                 │
│ 标签：#标签1 #标签2              │
├─────────────────────────────────┤
│ 备注：[补充说明]                │
└─────────────────────────────────┘
```

**使用场景**：
- 敏捷团队协作
- 日常任务追踪
- 工作流程可视化

**看板规则（WIP限制）**：
- To Do：无限制
- Doing：最多3项（限制在制品数量）
- Review：最多2项
- Done：无限制

**使用要点**：
- [ ] 限制WIP数量
- [ ] 任务卡片信息完整
- [ ] 每日更新状态
- [ ] 定期回顾优化

### 工具 #5: RACI矩阵

**定义**：明确项目/任务中各角色责任的矩阵工具

**RACI定义**：

| 字母 | 含义 | 说明 |
|------|------|------|
| **R** Responsible | 负责执行 | 实际完成任务的人 |
| **A** Accountable | 最终责任 | 对结果负总责的人 |
| **C** Consulted | 需咨询 | 提供信息/建议的人 |
| **I** Informed | 需通知 | 需要知道进展的人 |

**RACI矩阵模板**：

```
┌─────────────────────────────────────────────────────────────────┐
│                     项目RACI矩阵                                 │
├─────────────┬──────┬──────┬──────┬──────┬──────┬──────┬──────┤
│   任务      │ PM   │ 设计 │ 开发 │ 测试 │ 市场 │ 客户 │ 销售 │
│ ────────────┼──────┼──────┼──────┼──────┼──────┼──────┼──────┤
│ 需求收集    │  A   │  C   │  I   │  I   │  C   │  R   │  I   │
│ 方案设计    │  A   │  R   │  C   │  I   │  I   │  I   │  I   │
│ 开发实现    │  I   │  I   │  R/A │  C   │  I   │  I   │  I   │
│ 测试验收    │  A   │  I   │  I   │  R   │  I   │  I   │  I   │
│ 上线发布    │  A   │  I   │  R   │  I   │  R   │  I   │  C   │
│ 市场推广    │  A   │  C   │  I   │  I   │  R   │  I   │  C   │
│ 客户培训    │  I   │  I   │  I   │  I   │  C   │  R   │  A   │
└─────────────┴──────┴──────┴──────┴──────┴──────┴──────┴──────┘
```

**使用场景**：
- 跨部门协作项目
- 职责边界不清时
- 新项目启动规划

**使用要点**：
- [ ] 每个任务必须有A（唯一）
- [ ] 每个任务至少有R
- [ ] C和I适度，不过多
- [ ] 提前沟通确认
