## 核心框架

```
构建流程:
  碎片收集 → 归类整理 → 模式识别 → 框架提炼 → 验证完善

应用场景:
  - 新领域认知
  - 专家访谈整理
  - 信息碎片整合
  - 经验归纳总结
```

---

## 执行步骤

### Step 1: 碎片收集

```
收集原则:
  □ 不分类，先收集
    - 先穷举所有碎片
    - 不做预判
    - 广度优先

  □ 多渠道来源
    - 访谈记录
    - 文档资料
    - 数据分析
    - 观察体验

  □ 记录格式
    - 原始信息
    - 来源
    - 可信度
```

### Step 2: 归类整理

```
归类方法:
  □ 相似性归类
    - 找共同主题
    - 按主题分组
    - 命名组别

  □ 时间顺序
    - 按发展阶段
    - 按时间线
    - 识别阶段特征

  □ 重要程度
    - 高频出现
    - 核心影响
    - 关键驱动

归类检查:
  - 每个碎片有归属
  - 组别之间边界清晰
  - 没有碎片被遗漏
```

### Step 3: 模式识别

```
识别模式:
  □ 结构模式
    - 层级关系
    - 并列关系
    - 递进关系

  □ 因果模式
    - 原因→结果
    - 条件→结论
    - 投入→产出

  □ 演化模式
    - 阶段演进
    - 升级路径
    - 周期规律

输出:
  - 核心模式: [2-3个核心模式]
  - 关系图谱: [碎片间关系]
```

### Step 4: 框架提炼

```
提炼步骤:
  □ 命名框架
    - 用1-2个词命名
    - 体现核心逻辑
    - 便于记忆

  □ 确定维度
    - 核心维度 2-5个
    - 维度之间关系
    - 维度覆盖范围

  □ 定义边界
    - 适用场景
    - 不适用场景
    - 使用限制
```

## 使用示例

```
输入: 整理20位SaaS创始人访谈

Step 1: 碎片收集
  - 100+条语录/观点
  - 20个公司案例
  - 10个行业数据

Step 2: 归类整理
  主题组1: 产品战略 (20条)
  主题组2: 销售模式 (18条)
  主题组3: 团队建设 (15条)
  主题组4: 融资节奏 (12条)

Step 3: 模式识别
  核心模式:
    - PMF是生死线
    - 销售驱动增长
    - 团队文化是壁垒
  关系: PMF → 销售模式 → 规模化

Step 4: 框架提炼
  框架: SaaS增长飞轮
  维度:
    - 产品力 (PMF)
    - 销售力 (GTM)
    - 组织力 (文化)
  边界: 适用于B2B SaaS
```
