## 执行步骤

### Step 1: 团队评估

```
检查项:
  □ 创始人背景
    - 教育经历
    - 过往履历
    - 连续创业经验

  □ 团队完整性
    - 技术团队
    - 产品团队
    - 运营团队
    - 商业团队

  □ 股权结构
    - 创始人持股
    - 团队期权池
    - 历史投资人

评估结论: 优秀 / 良好 / 一般 / 存疑
```

### Step 2: 市场评估

```
检查项:
  □ 市场规模 (TAM/SAM/SOM)
    - TAM: 总可寻址市场
    - SAM: 可服务市场
    - SOM: 可获得市场

  □ 市场增速
    - 历史增速
    - 未来预测
    - 驱动因素

  □ 竞争格局
    - 直接竞品
    - 间接竞品
    - 替代品

评估结论: 大市场 / 中市场 / 小市场
```

### Step 3: 产品评估

```
检查项:
  □ 产品成熟度
    - MVP / 正式版 / 规模化
    - 产品迭代速度
    - 技术债务

  □ 技术壁垒
    - 核心技术
    - 专利布局
    - 难以复制点

  □ 差异化
    - 差异化程度
    - 用户认知
    - 品牌定位

评估结论: 强壁垒 / 中壁垒 / 弱壁垒
```

### Step 4: 商业模式评估

```
检查项:
  □ 收入模型
    - SaaS / 交易抽成 / 订阅 / 广告
    - 收入可预测性
    - 收入增长曲线

  □ LTV (生命周期价值)
    - 平均客单价
    - 平均生命周期
    - 复购率

  □ CAC (获客成本)
    - 获客渠道
    - 获客效率
    - LTV/CAC比

评估结论: 健康 (>3) / 一般 (1-3) / 危险 (<1)
```

### Step 5: 财务评估

```
检查项:
  □ 收入核实
    - ARR/MRR
    - 收入增长率
    - 收入质量

  □ 成本结构
    - 固定成本
    - 变动成本
    - 边际成本

  □ 现金流
    - 经营现金流
    - 资金消耗率
    - runway

评估结论: 优质 / 可接受 / 风险
```

### Step 6: 估值与投资条款

```
检查项:
  □ 估值方法选择
    - DCF现金流折现
    - 市盈率/市销率
    - 用户价值法
    - 对比法

  □ 估值区间
    - 乐观情景
    - 基本情景
    - 保守情景

  □ 投资条款
    - 股份比例
    - 董事会席位
    - 清算优先权
    - 反稀释条款

评估结论: 有吸引力 / 可接受 / 偏贵
```

## 综合评估矩阵

```
┌──────────┬─────────┬─────────┬─────────┬─────────┬─────────┐
│  维度     │ 评估    │ 权重    │ 得分    │ 加权得分 │ 备注     │
├──────────┼─────────┼─────────┼─────────┼─────────┼─────────┤
│ 团队     │         │ 25%     │ /10     │         │         │
├──────────┼─────────┼─────────┼─────────┼─────────┼─────────┤
│ 市场     │         │ 20%     │ /10     │         │         │
├──────────┼─────────┼─────────┼─────────┼─────────┼─────────┤
│ 产品     │         │ 15%     │ /10     │         │         │
├──────────┼─────────┼─────────┼─────────┼─────────┼─────────┤
│ 商业模式  │         │ 15%     │ /10     │         │         │
├──────────┼─────────┼─────────┼─────────┼─────────┼─────────┤
│ 财务     │         │ 15%     │ /10     │         │         │
├──────────┼─────────┼─────────┼─────────┼─────────┼─────────┤
│ 估值     │         │ 10%     │ /10     │         │         │
└──────────┴─────────┴─────────┴─────────┴─────────┴─────────┘

加权得分 >= 7.0 → 强烈推荐
加权得分 5.0-7.0 → 推荐
加权得分 3.0-5.0 → 观望
加权得分 < 3.0 → 不推荐
```

## 使用示例

### 示例1: A轮融资评估

```
输入:
  - 项目: B2B SaaS平台
  - 阶段: A轮
  - 月收入: 100万
  - 团队: 20人

评估结果:
  - 团队: 8分 (创始人有大厂背景)
  - 市场: 7分 (市场规模500亿)
  - 产品: 7分 (产品已规模化)
  - 商业模式: 8分 (LTV/CAC=5)
  - 财务: 6分 (现金流为负)
  - 估值: 6分 (估值偏贵)

  综合得分: 7.15 → 强烈推荐
```

### 示例2: 天使轮评估

```
输入:
  - 项目: 消费品牌
  - 阶段: 天使轮
  - 月收入: 10万
  - 团队: 5人

评估结果:
  - 团队: 6分 (首次创业)
  - 市场: 8分 (消费市场万亿)
  - 产品: 5分 (产品早期)
  - 商业模式: 5分 (模型待验证)
  - 财务: 4分 (财务不规范)
  - 估值: 7分 (估值合理)

  综合得分: 5.85 → 推荐
```

## 注意事项

```
⚠️ 评估陷阱:
  - 过度依赖创始人PPT
  - 忽视市场验证
  - LTV/CAC计算口径不一致
  - runway计算不保守

✅ 最佳实践:
  - 交叉验证数据
  - 实地访谈团队
  - 核算保守假设
  - 评估退出路径
```
