# 抽象问题具体化

## 身份定位

```
┌─────────────────────────────────────────────────────────────┐
│  🔮 抽象问题具体化                                         │
│  用途: 把抽象概念转化为可执行的具体问题                     │
│  配套Agent: 框架思维Agent                                    │
└─────────────────────────────────────────────────────────────┘
```

---

## 核心框架

```
抽象 → 具象转化:
  抽象概念 = 什么？
  具体问题 = 做什么？怎么做？做到什么程度？

转化检查:
  - 能否定义 → 是/否
  - 能否衡量 → 是/否
  - 能否执行 → 是/否
  - 能否验证 → 是/否
```

---

## 执行步骤

### Step 1: 概念拆解

```
拆解方法:
  □ 定义法
    - 这个概念的本质是什么
    - 和相近概念的区别是什么
    - 包含哪些子概念

  □ 追问法
    - 为什么是这个
    - 具体指什么
    - 有哪些表现

  □ 类比法
    - 像什么
    - 和什么类似
    - 不同于什么
```

### Step 2: 要素提取

```
核心要素:
  □ 变量要素
    - 有哪些变量
    - 变量的关系
    - 变量的取值范围

  □ 评估维度
    - 从哪些维度评估
    - 维度之间权重
    - 如何打分

  □ 执行主体
    - 谁来做
    - 什么时间做
    - 需要什么资源
```

### Step 3: 具体化设计

```
具体化输出:
  □ 问题具体化
    - 原问题: [抽象描述]
    - 具体问题: [具体描述]
    - 子问题: [拆解的子问题]

  □ 行动具体化
    - 行动1: [做什么]
    - 行动2: [做什么]
    - 验收标准: [做到什么程度]

  □ 验证具体化
    - 指标: [如何衡量]
    - 目标值: [多少算成功]
    - 数据来源: [从哪获取]
```

---

## 使用示例

```
原问题: "如何提升用户体验"

Step 1: 概念拆解
  本质: 用户使用产品时的感受
  子概念: 易用性 / 效率 / 美观 / 情感

Step 2: 要素提取
  维度: 功能体验 / 交互体验 / 视觉体验
  变量: 响应速度 / 操作步骤 / 界面美观度

Step 3: 具体化
  具体问题:
    - 如何缩短核心路径的操作步骤 (目标: 从5步→3步)
    - 如何提升页面加载速度 (目标: 从3s→1s)
    - 如何统一视觉风格 (目标: NPS提升10分)

  可执行: 是
  可验证: NPS + 行为数据
```
