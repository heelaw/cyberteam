# 横纵相交模型

## 身份定位

```
┌─────────────────────────────────────────────────────────────┐
│  ➕ 横纵相交模型                                            │
│  用途: 交叉分析找出高价值优化点                             │
│  配套Agent: 用户激励Agent                                   │
└─────────────────────────────────────────────────────────────┘
```

---

## 核心框架

```
纵向分析: 行为视角
  用户做了什么？
  关键行为 → 行为频率 → 行为漏斗

横向分析: 节点视角
  用户在哪里？
  触点 → 流程 → 节点

交叉分析: 优化视角
  用户在哪里流失？
  行为 × 节点 → 优化点
```

---

## 执行步骤

### Step 1: 纵向分析 (行为)

```
分析维度:
  □ 关键行为识别
    - 用户必须做的行为 (核心行为)
    - 高价值用户的行为特征
    - 低价值用户的缺失行为

  □ 行为频率分析
    - 核心行为使用频率
    - 频率与留存相关性
    - 魔法数字识别

  □ 行为漏斗
    - 行为A → 行为B → 行为C
    - 每步转化率
    - 流失点识别
```

### Step 2: 横向分析 (节点)

```
分析维度:
  □ 触点梳理
    - 用户接触产品的触点
    - 触点的功能/目的
    - 触点的重要程度

  □ 流程节点
    - 用户旅程中的关键节点
    - 入职/激活
    - 留存/活跃
    - 转化/变现

  □ 节点价值
    - 每个节点的业务价值
    - 节点的效率
    - 节点的阻塞点
```

### Step 3: 交叉分析矩阵

```
构建矩阵:
           │ 触点A   │ 触点B   │ 触点C   │
  ─────────┼─────────┼─────────┼─────────┤
  行为A    │ 频繁/高 │ 频繁/低 │ 偶尔/中 │
  行为B    │ 偶尔/中 │ 频繁/高 │ 频繁/低 │
  行为C    │ 频繁/低 │ 偶尔/高 │ 频繁/中 │

识别机会:
  □ 高频+低效 → 优化节点
    - 用户频繁使用，但效率低
    - 优化后体验大幅提升

  □ 偶尔+高效 → 引导更多
    - 少量用户高效
    - 扩大覆盖可带来大增长

  □ 频繁+高效 → 保持优势
    - 这是护城河
    - 不要乱改
```

---

## 交叉分析应用

```
增长应用:
  - AARRR × 触点 → 漏斗优化
  - 行为 × 节点 → 激励设计
  - 频率 × 价值 → 优先级排序

激励设计:
  - 找高频低价值行为 → 引导提升
  - 找低频高价值行为 → 设计激励
  - 找流失关键节点 → 设计干预
```

## 使用示例

```
输入: 电商App用户活跃下降

纵向分析:
  高价值行为: 收藏、评价、分享
  频率: 收藏5次/月 → LTV高
  漏斗: 浏览→加购→购买→复购(流失严重)

横向分析:
  触点: 首页、商品页、购物车、个人中心
  节点: 发现→选择→购买→复购

交叉分析:
  ┌──────────┬────────────┬────────────┐
  │          │ 加购率    │ 收藏率    │
  ├──────────┼────────────┼────────────┤
  │ 首页     │ 5%/浏览   │ 3%/浏览   │
  ├──────────┼────────────┼────────────┤
  │ 搜索结果 │ 15%/浏览   │ 8%/浏览   │ ← 高机会
  ├──────────┼────────────┼────────────┤
  │ 商品详情 │ 25%/浏览   │ 12%/浏览  │
  └──────────┴────────────┴────────────┘

结论:
  - 搜索结果页加购/收藏效率最高
  - 应加大搜索流量
  - 优化搜索结果排序
```
