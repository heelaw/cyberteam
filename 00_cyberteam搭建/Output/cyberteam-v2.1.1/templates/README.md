# CyberTeam v2 Skill 模板使用指南

## 概述

本目录包含 CyberTeam v2 标准化 Skill 模板，用于升级 60 个粗糙的 v2.1 Skills。

## 模板文件

| 文件 | 用途 |
|------|------|
| `SKILL_TEMPLATE_V2.md` | 通用标准模板（推荐） |
| `SKILL_TEMPLATE.md` | 原有模板（中国运营专家专用） |

## 填写指南

### Front Matter (YAML 头部)

| 占位符 | 填写说明 | 示例 |
|--------|----------|------|
| `{{SKILL_NAME}}` | 小写英文，连字符分隔 | `financial-due-diligence` |
| `{{SKILL_DESCRIPTION}}` | 简洁描述技能用途 | `财务尽职调查法 - 对业务进行财务层面的深度核查` |
| `{{AGENT_NAME}}` | 关联的 Agent ID | `investor-agent` |
| `{{TRIGGER_CONDITIONS}}` | 触发关键词列表 | `用户输入涉及: 财务尽调/成本分析/现金流/投资核查` |

### 身份定位

| 占位符 | 填写说明 | 示例 |
|--------|----------|------|
| `{{ICON}}` | Emoji 图标 | `🔍` |
| `{{SKILL_DISPLAY_NAME}}` | 中文显示名称 | `财务尽职调查法` |
| `{{PURPOSE}}` | 一句话说明用途 | `财务层面深度核查，识别风险和价值` |

### 三条红线

必须填写 3 条不可触碰的安全底线：

| 占位符 | 填写说明 | 示例 |
|--------|----------|------|
| `{{REDLINE_N_TITLE}}` | 红线主题（简短） | `闭环意识` |
| `{{REDLINE_N_CONTENT}}` | 具体说明 | `声称"已完成"前，必须跑验证命令、贴出输出证据` |

### 核心行为协议

| 占位符 | 填写说明 |
|--------|----------|
| `{{OWNER_MINDSET}}` | 描述 Owner 意识要求，主动承担责任 |
| `{{EXECUTION_DISCIPLINE}}` | 描述执行纪律和约束 |
| `{{OUTPUT_FORMAT_SPEC}}` | 指定输出格式要求（Markdown/JSON/表格等） |

### Success Metrics

至少填写 3 个成功指标：

| 占位符 | 填写说明 | 示例 |
|--------|----------|------|
| `{{METRIC_N}}` | 指标名称 | `收入核实完整度` |
| `{{TARGET_N}}` | 目标值 | `100%` |
| `{{MEASUREMENT_N}}` | 测量方式 | `检查项全部完成` |
| `{{ALTERNATIVE_PASS_CONDITION}}` | 替代通过条件 | `核心风险已识别` |

### Examples

至少填写 2 个示例：

| 占位符 | 填写说明 |
|--------|----------|
| `{{EXAMPLE_N_TITLE}}` | 示例标题 |
| `{{EXAMPLE_N_INPUT}}` | 输入场景描述 |
| `{{EXAMPLE_N_OUTPUT}}` | 预期输出结果 |
| `{{EXAMPLE_N_KEY_POINTS}}` | 关键要点 |

### Error Handling

| 占位符 | 填写说明 | 示例 |
|--------|----------|------|
| `{{ERROR_TYPE_N}}` | 错误类型 | `数据缺失` |
| `{{HANDLING_N}}` | 处理方式 | `要求补充必要数据` |
| `{{FALLBACK_N}}` | 降级方案 | `基于已有数据做部分分析` |

### Tools

| 占位符 | 填写说明 | 示例 |
|--------|----------|------|
| `{{REQUIRED_TOOL_N}}` | 必需工具名称 | `Read` |
| `{{TOOL_N_PURPOSE}}` | 工具用途 | `读取财务数据文件` |
| `{{OPTIONAL_TOOL_N}}` | 可选工具名称 | `WebSearch` |

### References

至少填写 3 条参考资料：

| 占位符 | 填写说明 | 示例 |
|--------|----------|------|
| `{{REFERENCE_N}}` | 资料描述 | `《财务尽职调查指南》` |

### 协作关系

| 占位符 | 填写说明 | 示例 |
|--------|----------|------|
| `{{PREREQUISITE_SKILL_N}}` | 前置 Skill 名称 | `财务报表分析` |
| `{{PREREQUISITE_REASON}}` | 前置原因 | `需要先掌握基础财务分析` |
| `{{NEXT_SKILL_N}}` | 后续 Skill 名称 | `投资决策评估` |
| `{{NEXT_REASON}}` | 后续原因 | `尽调后进入决策阶段` |
| `{{PARALLEL_SKILL_N}}` | 并行 Skill 名称 | `法律尽职调查` |
| `{{PARALLEL_REASON}}` | 并行原因 | `同时进行以提高效率` |

### Sub-agent 协议

保持默认模板内容，只需替换：

| 占位符 | 填写说明 |
|--------|----------|
| `{{CATEGORY}}` | Skill 分类目录名 |
| `{{SKILL_NAME}}` | Skill 目录名 |

### 执行步骤

至少填写 5 个步骤，每个步骤包含：

| 占位符 | 填写说明 | 示例 |
|--------|----------|------|
| `{{STEP_N_TITLE}}` | 步骤标题 | `收入核实` |
| `{{STEP_N_CONTENT}}` | 步骤详细内容 | `检查收入确认原则、收入质量、收入增长...` |

## 使用流程

### 1. 复制模板

```bash
cp templates/SKILL_TEMPLATE_V2.md skills/[分类]/[Skill名称]/SKILL.md
```

### 2. 填写占位符

按照上述指南，依次替换所有 `{{PLACEHOLDER}}`。

### 3. 删除未使用章节

如果某些章节不适用（如可选工具），可以删除该章节。

### 4. 自定义扩展

根据 Skill 特性，可以添加自定义章节。

## 检查清单

升级完成后，确保：

- [ ] 所有占位符已替换
- [ ] 至少 3 条红线
- [ ] 至少 3 个成功指标
- [ ] 至少 2 个示例
- [ ] 至少 5 个执行步骤
- [ ] 已关联配套 Agent
- [ ] 已填写触发条件
- [ ] 已标注协作关系
