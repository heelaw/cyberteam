## 三、Agent 协作流程

### 3.1 典型协作场景

#### 场景1：增长方案

```
用户输入："帮我做一个增长方案"

触发Agent：
1. 增长Agent（主导）
2. 北极星指标Agent（配合）
3. 渠道推广Agent（配合）
4. 文案Agent（配合）

协作流程：
增长Agent → 北极星指标Agent（确定指标）
         → 渠道推广Agent（制定渠道策略）
         → 文案Agent（配合素材）
         → 汇总输出增长方案
```

#### 场景2：转化优化

```
用户输入："帮我优化转化流程"

触发Agent：
1. 营销转化Agent（主导）
2. 文案Agent（配合）
3. 数据驱动Agent（配合）

协作流程：
营销转化Agent → 数据驱动Agent（分析数据）
            → 文案Agent（优化文案）
            → 汇总输出优化方案
```

#### 场景3：用户留存

```
用户输入："提升用户留存"

触发Agent：
1. 用户运营Agent（主导）
2. 用户洞察Agent（配合）
3. 活动策划Agent（配合）

协作流程：
用户运营Agent → 用户洞察Agent（分析用户）
             → 活动策划Agent（设计留存活动）
             → 汇总输出留存方案
```

### 3.2 Agent 任务依赖配置

```yaml
# 任务依赖配置示例
tasks:
  - name: "增长分析"
    agent: "增长Agent"
    output: "增长分析报告"

  - name: "指标制定"
    agent: "北极星指标Agent"
    blocked_by: ["增长分析"]
    output: "北极星指标文档"

  - name: "渠道策略"
    agent: "渠道推广Agent"
    blocked_by: ["指标制定"]
    output: "渠道策略文档"

  - name: "文案配合"
    agent: "文案Agent"
    blocked_by: ["渠道策略"]
    output: "转化文案"

  - name: "汇总方案"
    agent: "CEO"
    blocked_by: ["渠道策略", "文案配合"]
    output: "完整增长方案"
```

## 四、与思维框架的联动

### 4.1 运营问题 + 思维框架

| 运营问题 | 触发思维框架 | 配合运营Agent |
|----------|--------------|---------------|
| 增长方向不确定 | swot_tows, first_principle | 增长Agent |
| 转化率低 | five_why, kahneman | 营销转化Agent |
| 用户流失 | five_why, systems_thinking | 用户运营Agent |
| 活动效果差 | pre_mortem, reverse_thinking | 活动策划Agent |
| 策略不清晰 | mckinsey, six_hats | 策略规划Agent |
| 商业模式不清晰 | business_model_canvas | 商业模型Agent |

### 4.2 协作流程

```
用户输入 → CEO分析 → 思维框架选择 → 运营Agent选择 → 并行/串行执行 → 结果聚合
```
