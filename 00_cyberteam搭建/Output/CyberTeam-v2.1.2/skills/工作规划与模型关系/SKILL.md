## 模型对比矩阵

```
┌──────────┬──────────┬──────────┬──────────┬──────────────┐
│ 维度     │ OKR      │ KPI      │ BSC      │ 战略解码    │
├──────────┼──────────┼──────────┼──────────┼──────────────┤
│ 适用场景 │ 创新驱动 │ 执行驱动 │ 综合评估 │ 战略落地    │
│ 目标性质 │ 挑战性   │ 可达成   │ 多维度   │ 分解性      │
│ 周期     │ 季度     │ 月/季度  │ 年度     │ 半年度      │
│ 沟通重点 │ 对齐透明 │ 考核挂钩 │ 均衡发展 │ 上下对齐    │
│ 优势     │ 激发创新 │ 简单直接 │ 全面均衡 │ 战略落地    │
│ 劣势     │ 不易量化 │ 短期行为 │ 复杂    │ 执行复杂    │
└──────────┴──────────┴──────────┴──────────┴──────────────┘
```

---

## 选择决策树

```
问题: 选哪个规划模型？

第一步: 这个工作的核心驱动是什么？
  → 创新驱动 → OKR
  → 执行驱动 → KPI
  → 全面评估 → BSC

第二步: 是战略落地还是日常管理？
  → 战略落地 → 战略解码
  → 日常管理 → OKR/KPI

第三步: 需要考核还是不需要？
  → 需要考核 → KPI
  → 不需要考核 → OKR
```

---

## 执行步骤

### Step 1: 场景识别

```
评估问题:
  □ 创新 vs 执行
    - 需要探索新方向 → OKR
    - 路径明确执行 → KPI

  □ 战略 vs 日常
    - 公司级战略分解 → 战略解码+OKR
    - 部门日常工作 → OKR/KPI

  □ 考核 vs 激励
    - 与绩效挂钩 → KPI
    - 与成长挂钩 → OKR
```

### Step 2: 模型选择

```
选择结论:
  建议模型: ________
  选择理由: ________
  配合使用: ________

示例:
  创新部门 → OKR为主
  执行部门 → KPI为主
  高管层 → BSC+战略解码
```

### Step 3: 模型落地

```
OKR模板:
  Objective: ________
  Key Results:
    - KR1: ________
    - KR2: ________
    - KR3: ________

KPI模板:
  指标: ________
  目标值: ________
  权重: ________
  数据来源: ________
```

## 使用示例

```
输入: 电商公司运营部年度规划

场景分析:
  - 运营部: 执行为主，但需要创新
  - 日常管理: KPI
  - 新业务探索: OKR

选择方案:
  - 基础运营: KPI (80%权重)
    - 销售额/KPI
    - 转化率/KPI
    - 库存周转/KPI

  - 创新探索: OKR (20%权重)
    - O: 探索新增长模式
    - KR1: 测试3个新渠道
    - KR2: 找到1个ROI>3的渠道
```
