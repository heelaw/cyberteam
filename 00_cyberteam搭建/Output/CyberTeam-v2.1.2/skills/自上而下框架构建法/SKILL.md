## 核心框架

```
构建流程:
  全局视角 → 维度分解 → 层次展开 → 体系化

MECE原则:
  相互独立 (Mutually Exclusive)
  完全穷尽 (Collectively Exhaustive)

验证: 子项相加 = 整体
```

---

## 执行步骤

### Step 1: 全局视角

```
定义边界:
  □ 问题边界
    - 这个问题包含什么
    - 不包含什么
    - 与其他问题的关系

  □ 分析维度
    - 从哪些维度分析
    - 维度是否互斥
    - 是否覆盖全面

  □ 顶层结构
    - 核心问题是什么
    - 分几个部分
    - 部分之间的关系

输出:
  顶层: [核心问题]
  维度: [维度1, 维度2, 维度3]
  边界: [明确的边界定义]
```

### Step 2: 维度分解

```
分解方法:
  □ 按要素分解
    - 构成要素
    - 时间要素
    - 空间要素

  □ 按流程分解
    - 阶段划分
    - 先后顺序
    - 依赖关系

  □ 按类别分解
    - 二分法
    - 矩阵法
    - 多重分类

检查MECE:
  - 是否相互独立？
  - 是否完全穷尽？
  - 有没有遗漏？
  - 有没有重复？
```

### Step 3: 逐层展开

```
展开层次:
  第一层: 核心要素
  第二层: 子要素 (每个主要素2-5个)
  第三层: 具体项 (每个子要素1-3个)

展开原则:
  - 不超过7个 (人类记忆限制)
  - 同层同标准
  - 下一层支撑上一层
```

### Step 4: 填充与验证

```
填充内容:
  □ 每个叶子节点
    - 定义清晰
    - 有数据支撑
    - 可执行

  □ 关系梳理
    - 要素间关系
    - 优先级关系
    - 依赖关系

验证框架:
  - 逻辑完整性
  - 数据可获取
  - 结论可推导
```

## 使用示例

```
问题: 如何提升电商平台用户满意度

Step 1: 全局视角
  核心: 用户满意度
  维度: 产品 / 服务 / 价格 / 体验
  边界: B2C电商, 不含物流

Step 2: 维度分解 (MECE)
  产品: 品质 / 品类 / 丰富度
  服务: 客服 / 售后 / 响应速度
  价格: 竞争力 / 促销活动
  体验: 页面 / 支付 / 物流跟踪

Step 3: 展开
  产品-品质:
    - 供应商管理
    - 质检标准
    - 退换政策
  服务-客服:
    - 响应时间 < 1min
    - 解决率 > 90%

Step 4: 填充
  每个叶子节点都有:
    - 当前状态
    - 目标值
    - 改进措施
```
