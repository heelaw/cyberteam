# 合取谬误专家测试用例

## 正面测试

### 测试1: Linda问题
- 输入:
```
【合取谬误分析请求】
陈述/预测：Linda更可能是银行出纳且是女权运动参与者
组成部分：1.银行出纳 2.女权运动参与者
你认为的可能性：比"银行出纳"更具体，所以更可能
依据：描述很详细，听起来很真实
```
- 期望输出:
  - 分解为联合陈述
  - P(A且B) ≤ P(A)检验
  - 逻辑谬误识别
- 验证项:
  - [ ] 能识别"银行出纳且女权主义者"的概率不可能比"银行出纳"更高
  - [ ] 有P(A)和P(B)的分别估计
  - [ ] 能解释为什么"详细"不等于"更可能"

### 测试2: 预测过细
- 输入:
```
【合取谬误分析请求】
陈述/预测：明年经济会复苏，然后房价会涨，然后我就能买房了
组成部分：1.经济复苏 2.房价涨 3.我能买房
你认为的可能性：感觉这个链条很合理
依据：政府出台了很多政策
```
- 期望输出:
  - 三步联合分解
  - 每步概率分别评估
  - 链条整体概率上限
- 验证项:
  - [ ] 能分解为P1×P2×P3
  - [ ] 能说明联合概率的上限
  - [ ] 能指出"感觉合理"是认知偏差

## 负面测试

### 测试1: 详细=可能
- 输入:
```
【合取谬误分析请求】
陈述/预测：这个创业者会成功
组成部分：1.他很聪明 2.他很努力 3.他有好的想法 4.市场环境好
你认为的可能性：具备这么多条件，成功的可能性很大
依据：每个条件听起来都很正面
```
- 期望行为: 不能接受"多条件=高概率"
- 验证项:
  - [ ] 必须强调P(A且B且C且D) ≤ P(A)
  - [ ] 必须有概率上限检验

### 测试2: 忽视基础概率
- 输入:
```
【合取谬误分析请求】
陈述/预测：这个产品会爆火
组成部分：1.产品质量好 2.营销强 3.团队优秀
你认为的可能性：具备了成功要素
依据：竞品都没这么强
```
- 期望行为: 不能忽视基础比率
- 验证项:
  - [ ] 必须先确定"产品爆火"的基础比率
  - [ ] 必须评估联合条件的综合影响
