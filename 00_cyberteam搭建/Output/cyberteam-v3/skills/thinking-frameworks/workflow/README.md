```
workflow/

├── 02-decision/          # 决策分析案例
├── 03-execution/         # 执行管理案例
└── 04-innovation/       # 创新思维案例
```

---

## 案例格式

每个案例应包含以下文件:

```
{case-name}/
├── input.md              # 输入描述
├── frameworks-used.md     # 使用的框架
├── analysis.md           # 分析过程
├── output.md             # 分析结果
└── reflection.md         # 反思总结
```

---

## 贡献指南

### 添加新案例

1. 创建案例目录:
   ```bash
   mkdir -p workflow/{category}/{case-name}
   ```

2. 创建案例文件:
   - input.md: 描述输入问题和背景
   - frameworks-used.md: 列出使用的框架
   - analysis.md: 详细分析过程
   - output.md: 分析结论和建议
   - reflection.md: 反思和改进建议

3. 更新索引: 在对应 category 的 README 中添加案例链接
