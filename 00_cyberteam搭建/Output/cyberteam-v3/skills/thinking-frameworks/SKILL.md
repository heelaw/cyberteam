# 思考天团 - 思维框架体系

> **版本**: v3.1
> **分类**: 战略决策支持
> **技能体系**: 思考天团 (Thinking Frameworks)
> **触发词**: 思考, 分析, 框架, 战略, SWOT, 第一性原理, WBS, OKR, 波特五力

---

## 一、技能体系概述

### 1.1 定位

思考天团是 CyberTeam 的战略决策支持系统，为 CEO/L2 Agent 提供多维度思维框架，用于复杂问题的分析、决策和规划。

### 1.2 核心能力

| 能力 | 描述 | 适用场景 |
|------|------|----------|
| **战略分析** | SWOT、波特五力、PEST 等框架 | 市场进入、竞争分析 |
| **问题分解** | WBS、第一性原理、5Why | 复杂任务拆解、根因分析 |
| **目标管理** | OKR、KPI、SMART | 目标设定、绩效管理 |
| **决策支持** | 决策树、利弊分析、场景模拟 | 关键决策点 |
| **创新思考** | 第一性原理、类比推理、设计思维 | 产品创新、突破性方案 |

## 三、框架选择指南

### 3.1 按任务类型

| 任务类型 | 推荐框架组合 | 优先级 |
|----------|-------------|--------|
| **市场进入分析** | 波特五力 + SWOT + PEST | P0 |
| **竞争态势分析** | SWOT + 波特五力 | P0 |
| **产品规划** | 第一性原理 + OKR + WBS | P0 |
| **项目启动** | WBS + OKR | P0 |
| **战略规划** | SWOT + 波特五力 + BCG | P1 |
| **问题诊断** | 5Why + 第一性原理 | P0 |
| **目标设定** | OKR + SMART | P0 |
| **关键决策** | 决策树 + 利弊分析 | P1 |

### 3.2 按复杂度

| 复杂度 | 推荐框架 | 执行时间 |
|--------|----------|----------|
| **简单** (< 1小时) | SWOT 精简版 | 5-10 分钟 |
| **中等** (1-4小时) | SWOT + 1个补充框架 | 30-60 分钟 |
| **复杂** (半天+) | 全套框架组合 | 2-4 小时 |

### 3.3 框架组合示例

```
示例 1: 新产品市场进入
  波特五力 (行业结构)
    ↓
  SWOT (竞争态势)
    ↓
  第一性原理 (产品差异化)
    ↓
  OKR (目标设定)
    ↓
  WBS (任务分解)

示例 2: 竞争分析报告
  PEST (宏观环境)
    ↓
  波特五力 (行业结构)
    ↓
  SWOT (竞争态势)
    ↓
  价值链分析 (竞争优势)
```

## 四、Persona 专家

### 4.1 战略分析师

| 属性 | 说明 |
|------|------|
| **人设文件** | `persona/strategist.md` |
| **核心能力** | SWOT、波特五力、PEST、BCG、价值链 |
| **输出标准** | 执行摘要 + 框架分析 + 战略建议 |
| **沟通风格** | 专业但通俗，结论先行 |

### 4.2 使用方式

```
用户: "分析一下某市场的竞争态势"

CEO/L2 Agent:
  → 调用 thinking-frameworks
  → 战略分析师 (strategist.md) 接管
  → 选择框架组合: 波特五力 + SWOT
  → 执行分析
  → 输出: 战略分析报告
```

## 五、执行流程

### 5.1 标准流程

```
1. 需求理解
   ├── 明确分析目标
   ├── 确定分析范围
   └── 选择框架组合

2. 信息收集
   ├── 内部数据
   ├── 外部市场数据
   └── 竞品信息

3. 框架分析
   ├── 按框架结构展开
   ├── 填充分析内容
   └── 交叉验证

4. 综合输出
   ├── 执行摘要
   ├── 详细分析
   ├── 战略建议
   └── 风险提示
```

### 5.2 快速分析 (5分钟)

```
1. 确认目标 (1分钟)
2. SWOT 快速扫描 (2分钟)
3. 给出结论 (1分钟)
4. 指出深化方向 (1分钟)
```

### 5.3 详细分析 (30分钟)

```
1. 确认范围和目标 (5分钟)
2. 全面收集信息 (10分钟)
3. 系统性框架分析 (10分钟)
4. 战略建议输出 (5分钟)
```

## 七、质量标准

### 7.1 输出质量检查

| 维度 | 要求 |
|------|------|
| **完整性** | 所有框架维度都分析到位 |
| **深度** | 至少 3 层思考 |
| **数据** | 有具体数据支撑 |
| **可操作性** | 建议必须可执行 |
| **逻辑性** | 推理过程清晰 |

### 7.2 禁止事项

- 空泛结论 (如"需要提升能力")
- 数据造假或夸大
- 立场先行 (预设结论)
- 只说机会不说风险
