## 1. 数据分析流程

### 1.1 T+0 实时监控（发布后2小时内）

**监控任务**:

| 任务 | 内容 | 频率 |
|------|------|------|
| 异常检测 | 检查数据是否正常 | 每30分钟 |
| 初步反馈 | 收集评论/私信初步反馈 | 持续 |
| 竞品动态 | 监控竞品反应 | 持续 |

**异常判断标准**:
| 指标 | 异常阈值 | 处理方式 |
|------|----------|----------|
| CTR | <预期50% | 立即排查 |
| 投诉/差评 | >5条/小时 | 立即处理 |
| 平台限流 | 曝光突降 | 排查原因 |

**T+0 输出模板**:
```
【T+0 实时监控报告】
发布时间：
当前数据：
├─ 曝光量：
├─ 点击率(CTR)：
├─ 互动率：
└─ 负面评论数：

异常情况：
处理措施：
下一步：
```

## 2. 核心指标解读

### 2.1 指标定义

| 指标 | 定义 | 计算公式 | 行业参考 |
|------|------|----------|----------|
| **CTR** | 点击率 | 点击/曝光×100% | 1-5% |
| **CVR** | 转化率 | 转化/点击×100% | 0.5-3% |
| **CPC** | 点击成本 | 花费/点击 | 越低越好 |
| **CPA** | 转化成本 | 花费/转化 | 越低越好 |
| **互动率** | 互动占比 | (赞+评+转+藏)/曝光×100% | 1-10% |
| **完播率** | 看完比例 | 看完/播放×100% | 30-60% |
| **负面率** | 负面占比 | 负面/总评论×100% | <5% |

### 2.2 指标影响因素

| 指标 | 主要影响因素 | 次要因素 |
|------|--------------|----------|
| **CTR** | 标题、封面、前3秒 | 发布时间、账号权重 |
| **CVR** | 卖点表达、信任背书、CTA | 落地页体验、价格 |
| **CPC** | 平台竞价、出价策略 | 素材质量 |
| **互动率** | 内容共鸣、话题性 | 发布时间 |
| **完播率** | 内容节奏、信息密度 | 视频质量 |

### 2.3 指标优化方向

| 指标问题 | 优化方向 | 返回阶段 |
|----------|----------|----------|
| CTR低 | 优化标题/封面 | Stage 6/7 |
| CVR低 | 优化卖点表达/CTA | Stage 6/7 |
| 互动率低 | 增强情感共鸣 | Stage 4 |
| 完播率低 | 优化内容节奏 | Stage 6 |
| 负面率高 | 检查内容合规性 | Stage 7 |

## 4. 迭代决策规则

### 4.1 数据问题与返回阶段对照

| 数据问题 | 问题表现 | 返回阶段 | 优先级 |
|----------|----------|----------|--------|
| 卖点问题 | 卖点不匹配用户需求 | Stage 2 | P0 |
| 痛点问题 | 痛点共鸣不足 | Stage 3 | P0 |
| 策略问题 | 说服逻辑偏差 | Stage 4 | P1 |
| 结构问题 | 内容结构不匹配场景 | Stage 5 | P1 |
| 表达问题 | 语言表达不吸引人 | Stage 6/7 | P2 |

### 4.2 迭代决策流程

```
数据分析结果
    │
    ▼
┌─────────────────────────────────────┐
│ 判断问题类型                         │
└─────────────────────────────────────┘
    │
    ├─ 卖点/痛点问题 ──→ Stage 2/3
    ├─ 策略问题 ────────→ Stage 4
    ├─ 结构问题 ────────→ Stage 5
    └─ 表达问题 ────────→ Stage 6/7
    │
    ▼
重新制定方案
    │
    ▼
小范围测试（10%流量）
    │
    ├─ 效果提升 ──→ 全量上线
    └─ 效果下降 ──→ 调整后重新测试
```

### 4.3 迭代优先级

| 优先级 | 问题类型 | 迭代策略 | 预期调整幅度 |
|--------|----------|----------|--------------|
| **P0** | CTR低 | 标题/封面优化 | 小幅调整 |
| **P0** | CVR低 | 卖点/CTA优化 | 中幅调整 |
| **P1** | 互动率低 | 情感共鸣优化 | 中幅调整 |
| **P2** | 完播率低 | 内容节奏优化 | 小幅调整 |

### 4.4 知识沉淀要求

| 沉淀类型 | 内容要求 | 沉淀位置 |
|----------|----------|----------|
| 成功经验 | 标题公式/结构模型/表达技巧 | 模板库 |
| 失败教训 | 问题归因/优化方向/注意事项 | 避坑指南 |
| 数据规律 | 渠道特征/时段规律/人群偏好 | 数据分析库 |

```

第一章：项目概述
├── 1.1 项目背景
├── 1.2 项目目标
├── 1.3 执行时间
└── 1.4 核心团队

第二章：执行过程
├── 2.1 策略制定
├── 2.2 内容生产
├── 2.3 审核发布
└── 2.4 投放执行

第三章：数据表现
├── 3.1 核心指标汇总
├── 3.2 趋势分析
├── 3.3 竞品对比
└── 3.4 ROI评估

第四章：亮点总结
├── 4.1 执行亮点
├── 4.2 策略亮点
└── 4.3 创新亮点

第五章：问题分析
├── 5.1 策略问题
├── 5.2 执行问题
└── 5.3 协同问题

第六章：归因分析
├── 6.1 成功因素归因
├── 6.2 失败因素归因
└── 6.3 外部因素分析

第七章：经验沉淀
├── 7.1 可复用方法
├── 7.2 优化方向建议
└── 7.3 知识入库

第八章：后续计划
├── 8.1 迭代计划
├── 8.2 延伸方向
└── 8.3 下期规划
```

### 5.2 复盘报告填写说明

| 章节 | 填写要求 | 参考维度 |
|------|----------|----------|
| 第一章 | 简明扼要 | 背景/目标/时间/团队 |
| 第二章 | 过程记录 | 关键节点/决策点/执行难点 |
| 第三章 | 数据说话 | 指标对比/趋势/归因 |
| 第四章 | 具体可复制 | 有数据支撑的方法论 |
| 第五章 | 深入分析 | 根因分析而非表面描述 |
| 第六章 | 客观全面 | 内外部因素兼顾 |
| 第七章 | 落地可执行 | 具体可操作的建议 |
| 第八章 | 清晰可追踪 | 有明确时间和责任人 |

### 5.3 复盘时间要求

| 复盘类型 | 时间节点 | 参与人 | 输出 |
|----------|----------|--------|------|
| 日常复盘 | 单条内容结束后 | 操盘手 | 复盘小结 |
| 周度复盘 | 每周五 | 操盘手+内容总监 | 周报 |
| 项目复盘 | 项目结束后3天内 | 全团队 | 完整报告 |
| 月度复盘 | 每月最后一天 | 全团队 | 月度报告 |
