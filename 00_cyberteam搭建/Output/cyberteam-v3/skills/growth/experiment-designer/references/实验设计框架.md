# 实验设计框架

## 实验设计流程

### 1. 实验假设

**定义**: 实验假设是对因果关系的预测，是实验设计的核心。

**好的假设应该**:
- 明确：清晰描述变量关系
- 可测：可以通过数据验证
- 具体：针对特定用户群体和场景

**假设模板**:
```
如果 [对某用户群体] 做 [某改动]，那么 [某指标] 将会 [提升/下降]，因为 [原因]。
```

**示例**:
- 如果对新用户展示 APP 引导教程，那么 7 日留存率将会提升 10%，因为减少新用户的上手难度。

### 2. 样本量计算

#### 统计显著性基础

- **p值**: 结果偶然发生的概率，p < 0.05 表示结果具有统计显著性
- **置信区间**: 指标真实值的范围，通常使用 95% 置信区间
- **统计功效**: 正确检测到真实效果的概率，通常要求 ≥ 80%

#### 样本量计算公式

```
n = (Zα/2 + Zβ)² × (σ² + σ²) / δ²
```

其中：
- n = 每组所需样本量
- Zα/2 = 显著性水平对应的Z值（α=0.05时，Z=1.96）
- Zβ = 统计功效对应的Z值（80%功效时，Z=0.84）
- σ = 标准差
- δ = 期望的最小效果量

#### 简化估算

| 基础转化率 | 最小可检测效果 | 每组样本量 |
|------------|----------------|------------|
| 5% | 5%相对提升 | 30,000 |
| 10% | 5%相对提升 | 15,000 |
| 20% | 5%相对提升 | 7,000 |
| 50% | 5%相对提升 | 2,500 |

#### 确定实验周期

```
实验周期 = 样本量 / 日均流量
```

考虑因素：
- 最小周期：通常至少 1 周（覆盖工作日/周末差异）
- 完整周期：覆盖完整的用户行为周期（如 7 天、30 天）

### 3. 对照组设计

#### 基本原则

- **随机分配**: 确保实验组和对照组特征相似
- **同时进行**: 避免时间因素影响
- **唯一变量**: 只改变一个变量

#### 对照组类型

| 类型 | 描述 | 适用场景 |
|------|------|----------|
| 空白对照 | 不做任何改动 | 验证自然波动 |
| 标准对照 | 使用当前方案 | 验证新方案是否更好 |
| 阳性对照 | 使用已知有效方案 | 新方案 vs 最佳方案 |

#### 流量分配

- **标准分配**: 50% / 50%
- **保守分配**: 10% / 90%（新功能风险高）
- **加速分配**: 90% / 10%（新功能确信度高）

### 4. 指标设计

#### 指标类型

| 类型 | 定义 | 示例 |
|------|------|------|
| 核心指标 | 直接反映实验目标 | 转化率、留存率 |
| 辅助指标 | 反映次要目标 | 页面浏览量、点击率 |
| 护栏指标 | 确保不损害其他方面 | 客服投诉率、系统性能 |

#### 指标选择原则

1. **与北极星指标相关**
2. **可归因**: 可以归因到实验变量
3. **可测量**: 有可靠的数据采集
4. **及时**: 可以快速看到变化

#### 指标定义模板

```
指标名称: [名称]
计算公式: [公式]
数据来源: [埋点/系统]
统计周期: [天/周/月]
```

### 5. 数据埋点

#### 埋点设计

| 事件 | 描述 | 参数 |
|------|------|------|
| 曝光事件 | 用户看到实验内容 | 实验版本、用户ID、时间 |
| 行为事件 | 用户完成操作 | 行为类型、路径、转化 |
| 业务事件 | 业务指标变化 | 订单、金额、关键行为 |

#### 埋点检查清单

- [ ] 埋点是否覆盖所有实验组和对照组
- [ ] 埋点数据是否包含必要的维度信息
- [ ] 埋点数据是否与业务系统数据一致
- [ ] 埋点是否有足够的样本量

| 步骤 | 任务 | 负责人 |
|------|------|--------|
| 1 | 实验方案评审 | PM |
| 2 | 技术方案评审 | 研发 |

|------|------|------|

## 常见问题

### Q1: 样本量不够怎么办？

解决方案：
1. 延长实验周期
2. 扩大流量分配
3. 降低统计显著性要求（谨慎）
4. 选择更大效果量的实验

### Q2: 实验组和对照组差异太大？

可能原因：
1. 随机分组不均匀
2. 流量分配问题
3. 时间因素影响

解决方案：
1. 检查随机分组逻辑
2. 检查流量分配配置
3. 延长实验周期观察

### Q3: 结果不显著怎么办？

可能原因：
1. 样本量不够
2. 效果量太小
3. 实验设计问题

解决方案：
1. 增加样本量
2. 检查是否真的有效果
3. 分析负向影响
