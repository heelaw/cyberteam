## 案例格式

每个案例应包含以下文件:

```
{case-name}/
├── input.md              # 需求描述
├── content/              # 创作内容
│   ├── images/          # 生成的图片
│   └── copy.md          # 文案内容
├── published.md         # 发布结果
└── metrics.md           # 效果数据
```

---

## 贡献指南

### 添加新案例

1. 创建案例目录:
   ```bash
   mkdir -p workflow/{category}/{case-name}
   mkdir -p workflow/{category}/{case-name}/content/images
   ```

2. 创建案例文件:
   - input.md: 描述需求和目标
   - content/copy.md: 创作的内容
   - content/images/: 生成的图片
   - published.md: 发布状态和链接
   - metrics.md: 效果数据

3. 更新索引: 在对应 category 的 README 中添加案例链接
