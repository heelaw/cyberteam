# baoyu-skills - 微信公众号发布

> **版本**: v3.1
> **分类**: 多平台发布
> **技能**: baoyu-post-to-wechat
> **触发词**: 发布公众号, post-to-wechat, 公众号发布, 微信发布

---

## ⚠️ 铁律

1. **内容合规**: 必须符合微信公众平台规范
2. **格式正确**: 必须使用正确的 Markdown 格式
3. **版权合规**: 图片必须无版权问题
4. **封面规范**: 必须使用符合规范的封面图

---

## 一、技能说明

### 1.1 功能概述

将 Markdown 格式的内容发布到微信公众号，支持封面图、摘要、标签等。

### 1.2 支持的功能

| 功能 | 支持 | 说明 |
|------|------|------|
| Markdown 格式 | ✅ | 自动转换 |
| 封面图 | ✅ | 支持自定义封面 |
| 摘要 | ✅ | 自动生成或自定义 |
| 标签 | ✅ | 最多 3 个 |
| 原创声明 | ✅ | 可选开启 |
| 定时发布 | ⏳ | 计划中 |
| 多图文 | ⏳ | 计划中 |

---

## 二、调用参数

### 2.1 参数说明

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `title` | string | 是 | 文章标题 |
| `content` | string | 是 | 文章内容 (Markdown) |
| `--cover` | string | 否 | 封面图路径 |
| `--summary` | string | 否 | 文章摘要 |
| `--tags` | string | 否 | 标签 (逗号分隔) |
| `--author` | string | 否 | 作者名称 |
| `--original` | boolean | 否 | 开启原创声明 |
| `--draft` | boolean | 否 | 仅保存为草稿 |
| `--publish` | boolean | 否 | 直接发布 |

### 2.2 参数示例

```bash
# 基础发布
baoyu-post-to-wechat "文章标题" "## 正文内容..."

# 带封面图发布
baoyu-post-to-wechat "文章标题" "## 正文" --cover /path/to/cover.jpg

# 完整参数
baoyu-post-to-wechat "深度解析 AI 写作行业趋势" "## 行业现状..." \
  --cover /path/to/cover.jpg \
  --summary "本文深入分析 AI 写作行业的现状和未来趋势" \
  --tags "AI写作,行业分析,科技" \
  --author "CyberTeam" \
  --original

# 仅保存草稿
baoyu-post-to-wechat "草稿文章" "## 内容" --draft
```

## 三、执行流程

```
1. 准备内容
   └── 确保 Markdown 格式正确
   └── 检查内容合规性

2. 准备封面图
   └── 使用 baoyu-cover-image 生成
   └── 确保 2.35:1 比例

3. 编写摘要
   └── 自动生成或自定义
   └── 控制在 50-120 字

4. 设置标签
   └── 选择 1-3 个相关标签
   └── 优先选择热门标签

5. 格式转换
   └── Markdown → HTML
   └── 处理图片链接

6. 发布
   └── 草稿保存或直接发布

7. 验证
   └── 检查发布状态
   └── 记录链接
```

## 四、内容规范

### 4.1 标题规范

| 规范 | 说明 | 示例 |
|------|------|------|
| 长度 | 14字以内最佳 | "AI写作的现状与未来" |
| 风格 | 悬念/数字/情绪 | "5分钟学会 AI 写作" |
| 避免 | 过于标题党 | ❌ "震惊！AI 竟然..." |

### 4.2 正文规范

| 项目 | 规范 |
|------|------|
| **图片** | ≤5MB，建议≤2MB |
| **图片数量** | 无限制，建议≤10张 |
| **代码块** | 支持高亮 |
| **表格** | 支持 |
| **引用** | 支持 |
| **emoji** | 支持但克制 |

### 4.3 封面图规范

| 项目 | 规范 |
|------|------|
| **比例** | 2.35:1 |
| **尺寸** | 900x383 px (最小) |
| **格式** | JPG/PNG |
| **大小** | ≤5MB |
| **文字** | 可选，建议有 |

### 4.4 摘要规范

| 项目 | 规范 |
|------|------|
| **长度** | 50-120 字 |
| **内容** | 文章核心亮点 |
| **风格** | 吸引点击，有价值感 |
| **来源** | 自动生成或手动编写 |

### 4.5 标签规范

| 项目 | 规范 |
|------|------|
| **数量** | 1-3 个 |
| **选择** | 相关性 + 热度 |
| **格式** | 话题标签 |

## 五、Markdown 格式指南

### 5.1 支持的格式

```markdown
# 一级标题
## 二级标题
### 三级标题

**粗体** *斜体*

- 无序列表
1. 有序列表

> 引用

```代码块```

| 表格 | 表头 |
|------|------|
| 内容 | 数据 |

![图片描述](图片链接)

[链接文字](链接地址)
```

### 5.2 不支持的格式

```markdown
❌ 脚注
❌ 任务列表 (- [ ] )
❌ 目录 (TOC)
```

### 5.3 图片处理

```markdown
# 本地图片
![描述](./images/example.jpg)

# 网络图片
![描述](https://example.com/image.jpg)

# 建议
- 使用图床托管图片
- 优先使用网络图片
- 确保图片链接长期有效
```
