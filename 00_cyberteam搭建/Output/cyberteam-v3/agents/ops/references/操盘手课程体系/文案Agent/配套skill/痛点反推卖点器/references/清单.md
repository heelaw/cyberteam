# 痛点反推卖点法 - 检查清单

## 痛点挖掘检查

### 真实性
- [ ] 有用户原话/数据支撑
- [ ] 不是凭空想象
- [ ] 高频出现（非个例）

### 具体性
- [ ] 有具体场景描述
- [ ] 有具体时间/地点/人物
- [ ] 避免了"不好用"等模糊词

### 深度
- [ ] 区分了表面/深层/本质痛点
- [ ] 挖掘到了本质问题
- [ ] 用户原话被充分引用

## 情绪放大检查

### 共鸣度
- [ ] 情绪是真实的人类情感
- [ ] 不是刻意夸大
- [ ] 大多数目标用户能感同身受

### 强度
- [ ] 情绪强度 ≥ 7/10
- [ ] 足以驱动决策
- [ ] 有明确的情绪词汇

## 解决方案对应检查

### 对应性
- [ ] 每个痛点有对应功能
- [ ] 功能描述具体
- [ ] 用户能理解对应关系

### 可信度
- [ ] 有量化效果
- [ ] 数据合理可信
- [ ] 不是夸大宣传

## 卖点输出检查

### 穿透力
- [ ] 卖点文案直击痛点
- [ ] 有情绪共鸣
- [ ] 用户能快速理解

### 可执行
- [ ] 可以转化为文案
- [ ] 可以落地执行
- [ ] 符合产品实际
