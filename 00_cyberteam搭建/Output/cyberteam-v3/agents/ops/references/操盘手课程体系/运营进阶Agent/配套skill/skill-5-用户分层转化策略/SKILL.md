## 核心定义

### 什么是用户分层？

根据用户的特征、行为、价值等维度，将用户划分为不同的群体，针对每个群体制定差异化的运营策略。

### 用户分层的价值

```
┌─────────────────────────────────────────────────────┐
│              用户分层运营价值                       │
├─────────────────────────────────────────────────────┤
│                                                     │
│   1. 精准触达                                       │
│      - 了解不同用户的核心需求                       │
│      - 制定针对性的内容和激励                       │
│                                                     │
│   2. 效率提升                                       │
│      - 资源投入到最需要的用户身上                   │
│      - 避免资源浪费在低价值用户                    │
│                                                     │
│   3. 效果优化                                       │
│      - 针对性的策略效果更好                         │
│      - 提升整体转化率和ROI                          │
│                                                     │
└─────────────────────────────────────────────────────┘
```

## Critical Rules（必须遵守）

1. **分层维度要可量化** - 分层标准必须能通过数据衡量
2. **分层必须可行动** - 每个层级必须有对应的运营策略
3. **动态更新分层** - 用户会流动，分层要定期更新
4. **资源聚焦高价值** - 高价值用户要有更精细的运营
5. **沉默用户要警惕** - 沉默不代表流失，要及时干预

---

## 禁止行为

1. **禁止只看单一维度** - 不能只用付费与否分层
2. **禁止分层后不运营** - 分层是为了制定策略，不是为了好看
3. **禁止忽视沉默用户** - 沉默用户有流失风险
4. **禁止策略无差异化** - 不同层级用同样策略等于没分层
5. **禁止分层太细** - 层级太多难以运营，建议不超过5层
6. **禁止只看当前价值** - 要考虑用户潜在价值
7. **禁止脱离业务场景** - 分层要结合业务目标

## Workflow Steps

### 步骤1：确定分层维度（输入→处理→输出）

**输入**：
- 业务类型
- 运营目标
- 可用的用户数据

**处理**（选择分层维度）：

**维度A：用户生命周期**
| 阶段 | 定义 | 核心策略 |
|------|------|---------|
| 新用户 | 首次使用/注册 | 引导体验 |
| 成长用户 | 持续使用 | 促进深度使用 |
| 成熟用户 | 频繁使用/付费 | 提升忠诚度 |
| 沉默用户 | 长时间未使用 | 召回激活 |
| 流失用户 | 彻底不再使用 | 尝试挽回 |

**维度B：用户价值**
| 层级 | 定义 | 核心策略 |
|------|------|---------|
| 高价值用户 | LTV高/活跃度高 | VIP服务 |
| 中价值用户 | LTV中等/活跃度中等 | 培养转化 |
| 低价值用户 | LTV低/活跃度低 | 激活或放弃 |

**维度C：用户行为**
| 行为类型 | 定义 | 核心策略 |
|----------|------|---------|
| 核心行为用户 | 完成核心功能 | 引导深度使用 |
| 边缘行为用户 | 只用边缘功能 | 引导核心功能 |
| 无行为用户 | 注册但未使用 | 激活引导 |

**输出**：
- 分层维度选择（1-2个维度）
- 各层级定义标准
- 数据来源说明

**成功标准**：维度可量化，各层有明确定义

---

### 步骤2：用户数据提取（输入→处理→输出）

**输入**：
- 分层维度
- 用户行为数据
- 用户属性数据

**处理**（数据提取）：
```
提取数据清单：
1. 用户基础属性
   - 注册时间
   - 地域分布
   - 设备类型

2. 用户行为数据
   - 首次使用时间
   - 最近使用时间
   - 累计使用次数
   - 核心功能使用情况

3. 付费数据（如适用）
   - 首次付费时间
   - 累计付费金额
   - 付费频次

4. 渠道数据
   - 来源渠道
   - 获客成本

分层阈值设定：
- 使用数据分布确定阈值
- 参考业务经验调整
- 验证阈值合理性
```

**输出**：
- 用户分层数据表
- 各层级用户数量
- 各层级用户占比

**成功标准**：数据完整，阈值合理

---

### 步骤3：制定分层策略（输入→处理→输出）

**输入**：
- 用户分层数据
- 各层级用户特征
- 业务目标

**处理**（策略制定）：

**新用户策略框架**：
| 策略维度 | 内容 |
|----------|------|
| 核心目标 | 建立认知，引导体验 |
| 内容策略 | 产品价值介绍、快速上手指南 |
| 激励策略 | 新手礼包、首次优惠 |
| 触达渠道 | 弹窗、PUSH、邮件 |
| 成功指标 | 7日留存率、核心功能使用率 |

**活跃用户策略框架**：
| 策略维度 | 内容 |
|----------|------|
| 核心目标 | 深度使用，提升黏性 |
| 内容策略 | 新功能介绍、使用技巧 |
| 激励策略 | 等级权益、专属福利 |
| 触达渠道 | PUSH、消息中心 |
| 成功指标 | DAU、核心功能使用频次 |

**付费用户策略框架**：
| 策略维度 | 内容 |
|----------|------|
| 核心目标 | 续费/升级，推荐传播 |
| 内容策略 | 专属权益、升级价值 |
| 激励策略 | 邀请奖励、专属优惠 |
| 触达渠道 | 短信、邮件、专属客服 |
| 成功指标 | 续费率、LTV |

**沉默用户策略框架**：
| 策略维度 | 内容 |
|----------|------|
| 核心目标 | 召回激活，了解原因 |
| 内容策略 | 专属优惠、最新功能 |
| 激励策略 | 召回礼包、限时优惠 |
| 触达渠道 | 短信、邮件、外呼 |
| 成功指标 | 召回率、召回后留存率 |

**流失用户策略框架**：
| 策略维度 | 内容 |
|----------|------|
| 核心目标 | 挽回或收集反馈 |
| 内容策略 | 调研问卷、挽回优惠 |
| 激励策略 | 终极挽回礼包 |
| 触达渠道 | 邮件、短信 |
| 成功指标 | 挽回率、反馈收集量 |

**输出**：
- 各层级用户画像
- 各层级运营策略
- 各层级触达计划

**成功标准**：每个层级都有明确的策略

---

### 步骤4：设计转化路径（输入→处理→输出）

**输入**：
- 分层策略
- 用户痛点
- 可用资源

**处理**（路径设计）：

**用户需求类型与转化策略**：
| 需求类型 | 用户特征 | 转化策略 |
|----------|---------|---------|
| 明确需求 | 目标明确，主动搜索 | 直接展示优势，提供决策依据 |
| 模糊需求 | 有困扰但不清晰 | 教育引导，帮助明确需求 |
| 潜在需求 | 不知道自己有需求 | 创造场景，激发需求 |

**转化场景与文案策略**：
| 场景 | 用户状态 | 文案策略 |
|------|---------|---------|
| 地铁/公交 | 短时（3-5秒） | 短平快，一眼看懂 |
| 办公室 | 中等时间 | 详细，有深度 |
| 睡前 | 长时间 | 故事性强，情感丰富 |

**输出**：
- 用户转化路径图
- 各节点文案要点
- 触达时机设计

**成功标准**：路径清晰，时机合理

---

## 评估维度

| 维度 | 定义 | 权重 |
|------|------|------|
| 分层合理性 | 维度选择和阈值设定是否合理 | 25% |
| 策略差异化 | 各层级策略是否有明显差异 | 25% |
| 策略可执行性 | 策略是否能落地执行 | 20% |
| 转化路径清晰度 | 转化路径是否明确 | 15% |
| 监测完善度 | 是否有监控指标和预警 | 15% |
