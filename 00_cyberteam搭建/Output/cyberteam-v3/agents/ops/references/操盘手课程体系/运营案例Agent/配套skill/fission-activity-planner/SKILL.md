# SKILL: fission-activity-planner

> 技能名称：裂变型活动策划
> 版本: v1.0
> 难度等级: medium
> 关联Agent: 运营案例Agent
> 来源案例: 支付宝中国锦鲤、拼多多天天领现金、瑞幸咖啡等

---

## 1. 技能概述

本技能提供裂变型活动的系统策划框架，帮助运营从业者从0到1策划一场低成本高传播的裂变活动。

**核心能力**:
- 策划裂变活动的完整框架
- 设计有效的裂变机制和激励方案
- 规划传播节奏和执行路径
- 评估活动效果和风险

**适用场景**:
- 策划新用户获取活动
- 设计老带新活动机制
- 制定裂变传播方案
- 优化现有裂变活动

## 3. 工作流 Steps

### Step 1: 明确活动目标

**输入**: 业务背景和活动目标

**处理**:
```
┌─────────────────────────────────────────────────────────────┐
│ 裂变活动目标分类                                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  [A] 拉新为主                                                │
│      - 新用户数                                              │
│      - 新用户占比                                            │
│      - 获客成本（CAC）                                       │
│                                                             │
│  [B] 传播为主                                                │
│      - 曝光量                                                │
│      - 转发数                                                │
│      - 话题热度                                              │
│                                                             │
│  [C] 转化为主                                                │
│      - 订单数                                                │
│      - 销售额                                                │
│      - 付费率                                                │
│                                                             │
│  [D] 综合型                                                  │
│      - 兼顾拉新、传播、转化                                   │

│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**输出**: 确定活动核心目标和优先级

**成功标准**: 目标SMART化（具体/可衡量/可达成/相关/有时限）

### Step 2: 设计裂变机制

**输入**: 活动目标

**处理**:

#### 2.1 选择裂变类型

```
┌─────────────────────────────────────────────────────────────┐
│ 裂变类型选择                                                 │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  [1] 抽奖型                                                  │
│      - 代表：支付宝中国锦鲤                                  │
│      - 特点：单个大奖，高诱惑，低成本                        │
│      - 适用：品牌传播、话题造势                              │
│      - 关键：大奖要有话题性，奖品总价值感强                  │
│                                                             │
│  [2] 红包型                                                  │
│      - 代表：拼多多天天领现金                                │
│      - 特点：人人有份，阶梯奖励，损失厌恶                    │
│      - 适用：快速拉新、用户增长                              │
│      - 关键：奖励要即时可见，越接近目标越有动力              │
│                                                             │
│  [3] 任务型                                                  │
│      - 代表：滴滴追风竞速赛                                  │
│      - 特点：完成任务获取奖励，门槛清晰                     │
│      - 适用：精准拉新、提升活跃                              │
│      - 关键：任务难度要适中，有超级奖励降低初始门槛          │
│                                                             │
│  [4] 互惠型                                                  │
│      - 代表：微信读书互赠书币                                │
│      - 特点：双向获利，社交关系驱动                          │
│      - 适用：社交产品、有好友关系的产品                      │
│      - 关键：礼物要对双方都有价值                            │
│                                                             │
│  [5] 竞赛型                                                  │
│      - 代表：早起打卡挑战赛                                  │
│      - 特点：竞争驱动，排行榜机制                            │
│      - 适用：提升留存、活跃                                  │
│      - 关键：竞赛要公平，奖励要有成就感                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 2.2 设计激励方案

**核心激励要素**:
```
┌─────────────────────────────────────────────────────────────┐
│ 激励方案设计                                                 │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  【奖励设置】                                                │
│  - 发起人奖励：完成任务后获得什么                           │
│  - 助力人奖励：帮助好友后获得什么                           │
│  - 阶梯奖励：完成多少任务获得额外奖励                       │
│                                                             │
│  【激励原则】                                                │
│  - 即时反馈：每一步操作都要有反馈                          │
│  - 损失厌恶：接近目标时的激励最强                          │
│  - 社交货币：奖励要有展示价值和话题性                      │
│  - 稀缺感：限量/限时/名额限制                              │
│                                                             │
│  【成本控制】                                                │
│  - 设定单人获客成本上限                                    │
│  - 阶梯式奖励控制总成本                                    │
│  - 识别羊毛党，设置防薅机制                                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**输出**: 确定裂变类型和激励方案

**成功标准**: 激励方案有话题性、成本可控、有防薅机制

│                                                             │

│                                                             │
│  【主题类型参考】                                            │

│                                                             │
│  【文案技巧】                                                │
│  - 直白务实：用户一眼看懂能获得什么                        │

### Step 5: 设计用户路径

**输入**: 裂变机制和传播节奏

**处理**:

```
┌─────────────────────────────────────────────────────────────┐
│ 用户路径设计                                                 │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  【新用户路径】                                              │
│  触达 → 感兴趣 → 参与活动 → 注册/下载 → 体验产品 → 转化    │
│    ↓         ↓           ↓            ↓          ↓         │
│  广告/推送  分享内容    领取奖励     完成首单    付费/会员  │
│                                                             │
│  【老用户路径】                                              │
│  触达 → 想起需求 → 参与活动 → 分享邀请 → 获得奖励 → 复购   │
│    ↓         ↓           ↓            ↓          ↓         │
│  推送/推送  优惠信息    完成任务    好友参与    再次购买   │
│                                                             │
│  【路径设计原则】                                            │
│  1. 入口要浅：用户能快速找到参与入口                       │
│  2. 步骤要少：每多一步流失30%以上的用户                    │
│  3. 反馈要快：每个操作都要有即时的正向反馈                 │
│  4. 障碍要清：移除一切可能阻碍用户的因素                   │
│                                                             │
│  【新老用户差异化】                                          │
│  - 新用户：侧重降低首次体验门槛                            │
│  - 老用户：侧重激励分享和持续参与                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**输出**: 完整的用户路径地图

**成功标准**: 路径清晰、步骤少、有差异化设计

**成功标准**: P0风险有预案，评估指标明确

## 5. Critical Rules

### 5.1 必须遵守

1. **目标先行** - 先明确活动目标再设计活动
2. **激励要有话题性** - 奖励本身要能引发传播
3. **路径要短** - 从触达到参与不超过3步
4. **成本要可控** - 必须设定单人获客成本上限
5. **防薅要前置** - 在方案设计阶段就考虑防薅

### 5.2 禁止行为

1. **不设计单向获益** - 发起人和助力人都要获利
2. **不忽视新用户体验** - 新用户路径同样重要
3. **不跳过风险评估** - 每个活动必须评估羊毛党风险
4. **不承诺无法兑现的奖励** - 确保有能力兑现所有奖励
5. **不依赖单一渠道** - 传播要私域+公域结合

## 6. 评估维度

| 维度 | 说明 | 最低要求 |
|------|------|----------|
| 目标明确性 | 活动目标是否SMART | 3个指标以上 |
| 机制有效性 | 裂变机制是否有吸引力 | 有话题性激励 |
| 路径合理性 | 用户路径是否顺畅 | 不超过5步 |
| 成本可控性 | 成本是否在预算内 | CAC有上限 |
| 风险完备性 | 是否有P0风险预案 | P0级有预案 |
