# 框架式思维与问题拆解 - 测试案例

## 测试案例1

**难度**：medium
**场景**：电商平台转化率下降

**输入**：
```
问题描述：某电商App近一个月转化率从3.5%下降到2.8%，客单价也有下滑，需要分析原因并提出改善方案。
```

**预期输出**：
```
1. 问题定性：指标异常问题（太具体，缺乏全局视角）

2. 元素识别：
   - 概念性名词：转化率、客单价
   - 非具体动词：下降、改善
   - 具体动词：分析

3. 拆解策略（相关式）：
   转化率 = UV × 点击率 × 加购率 × 支付转化率
   GMV = UV × 点击率 × 加购率 × 支付转化率 × 客单价

4. 构建框架：
   [GMV下降]
      │
   ┌──┴──┐
   │     │
[流量] [转化] [客单]
   │     │     │
  UV  点击 加购 支付
           率   率  转化率

5. 验证：需补充历史数据对比和竞品分析
```

**验收标准**：
- [ ] 拆解公式逻辑正确
- [ ] 能指导下一步分析方向
- [ ] 结合了电商业务场景

## 测试案例2

**难度**：simple
**场景**：社群运营问题

**输入**：
```
问题描述：运营了一个500人的微信群，但每天发言人数只有20-30人，如何提升社群活跃度？
```

**预期输出**：
```
1. 问题定性：活跃度不足（抽象问题）

2. 元素识别：
   - 概念性名词：活跃度、社群
   - 非具体动词：提升

3. 拆解策略（并列式+相关式）：
   活跃度 = 发言人数 × 人均发言次数
   发言人数 = 核心用户 + 普通用户

4. 构建框架：
   [社群活跃度]
        │
   ┌────┴────┐
   │         │
[发言人数] [人均次数]
   │         │
 核心用户  互动内容
  普通用户  激励机制

5. 优先级：先提升发言人数，再提升人均次数
```

**验收标准**：
- [ ] 拆解清晰易懂
- [ ] 有明确的行动方向
- [ ] 考虑社群特点

## 测试案例3

**难度**：medium
**场景**：内容运营策略制定

**输入**：
```
问题描述：负责一款资讯App的内容运营，希望提升用户的阅读时长和完读率，但不知道从何入手。
```

**预期输出**：
```
1. 问题定性：策略规划问题（抽象）

2. 元素识别：
   - 概念性名词：阅读时长、完读率、内容运营
   - 非具体动词：提升

3. 拆解策略：
   阅读时长 = DAU × 人均阅读篇数 × 篇均时长
   完读率 = 完整阅读人数 ÷ 开始阅读人数

4. 框架构建：
   [阅读时长提升]
        │
   ┌────┴─────┐
   │          │
[DAU]    [篇均时长]
   │          │
 拉新    内容质量
 留存    推荐算法
        │
     人均篇数
        │
    推送时机
    内容匹配

5. 行动优先级：
   P0：提升内容质量（完读率基础）
   P1：优化推荐算法（篇均时长）
   P2：改进推送策略（DAU）
```

**验收标准**：
- [ ] 拆解全面覆盖影响因素
- [ ] 有清晰的优先级排序
- [ ] 结合了内容产品特点
