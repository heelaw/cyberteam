# 需求层次判断标准

## 五层次需求模型

### 第一层：生理需求（功能性需求）

**定义**: 满足基本生存和生理舒适的需求

**特征**:
- 用户明确知道要什么
- 可以量化衡量
- 解决"能用"的问题

**常见示例**:
| 行业 | 生理需求 |
|------|----------|
| 餐饮 | 吃饱、味道好、卫生 |
| 服装 | 保暖、舒适、合身 |
| 出行 | 准时、到达目的地 |
| 医疗 | 治好病、缓解症状 |

**判断题**:
- [ ] 用户能清晰描述这个需求？
- [ ] 这个需求可以被具体测量？
- [ ] 竞品也能满足这个需求？

---

### 第二层：安全需求（可靠性需求）

**定义**: 保障安全、稳定、可靠的需求

**特征**:
- 关注风险规避
- 重视稳定性和可靠性
- 解决"用得放心"的问题

**常见示例**:
| 行业 | 安全需求 |
|------|----------|
| 金融 | 资金安全、本金保障 |
| 医疗 | 无副作用、治疗安全 |
| 食品 | 食品安全、无添加 |
| 保险 | 保障全面、理赔便捷 |

**判断题**:
- [ ] 用户关注风险和安全感？
- [ ] 用户需要保证和承诺？
- [ ] 信任建立很重要？

---

### 第三层：社交需求（归属感需求）

**定义**: 融入群体、获得归属感的需求

**特征**:
- 重视群体认同
- 有社交展示属性
- 解决"用得有面子"的问题

**常见示例**:
| 行业 | 社交需求 |
|------|----------|
| 服装 | 品牌认同、潮流感 |
| 电子产品 | 身份象征、科技感 |
| 健身 | 社群归属、一起打卡 |
| 旅游 | 朋友圈分享、晒图 |

**判断题**:
- [ ] 用户会考虑别人怎么看？
- [ ] 产品有社交展示价值？
- [ ] 用户愿意分享和推荐？

---

### 第四层：尊重需求（成就感需求）

**定义**: 获得认可、尊重的需求

**特征**:
- 重视自我实现和成就感
- 愿意为"增值体验"付费
- 解决"用得有成就感"的问题

**常见示例**:
| 行业 | 尊重需求 |
|------|----------|
| 教育 | 获得证书、证明能力 |
| 奢侈品 | 限量版、专属感 |
| 高端服务 | VIP待遇、一对一服务 |
| 职业发展 | 升职、加薪、认可 |

**判断题**:
- [ ] 用户希望被认可？
- [ ] 用户愿意为"特殊待遇"付费？
- [ ] 成就感很重要？

---

### 第五层：自我实现（价值观需求）

**定义**: 实现自我理想、符合价值观的需求

**特征**:
- 与用户价值观高度契合
- 具有使命感和意义感
- 解决"用得认同"的问题

**常见示例**:
| 行业 | 自我实现需求 |
|------|--------------|
| 环保产品 | 绿色消费、保护环境 |
| 公益 | 帮助他人、回馈社会 |
| 创作 | 自我表达、创作自由 |
| 健康 | 健康生活方式的认同 |

**判断题**:
- [ ] 用户有明确的价值观？
- [ ] 产品与用户价值观契合？
- [ ] 用户愿意为此代言？

---

## 需求层次判断流程

```
问题1: 用户能清晰描述这个需求？
  是 → 可能是生理需求
  否 → 继续问题2

问题2: 用户关注风险和安全感？
  是 → 安全需求
  否 → 继续问题3

问题3: 用户会在乎别人怎么看？
  是 → 社交需求
  否 → 继续问题4

问题4: 用户希望获得成就感？
  是 → 尊重需求
  否 → 自我实现需求
```

## 注意事项

1. **需求可以叠加**: 同一产品可能同时满足多个层次需求
2. **层次可以升级**: 随着市场发展，需求层次可能提升
3. **判断要证据**: 不能猜测，要基于用户调研数据
