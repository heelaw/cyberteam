## 二、L2 协调层映射

### 2.1 Strategy Agent

**职责**: 战略规划、方案设计、思维框架注入

**调用 Skills**:

| 场景 | 调用 Skill | 调用方式 | 触发条件 |
|------|-----------|----------|----------|
| 需求分析 | 思考天团 - 5W1H1Y | 自动注入 | 收到用户需求时 |
| 任务拆解 | 思考天团 - WBS | 自动注入 | 需要拆解任务时 |
| 深度分析 | 思考天团 - SWOT、波特五力 | 自动注入 | 需要深度分析时 |
| 市场调研 | MCP - web_search | 主动调用 | 需要市场信息时 |
| 资料收集 | MCP - webReader | 主动调用 | 需要读取网页时 |
| 目标设定 | 思考天团 - SMART、OKR | 自动注入 | 设定目标时 |
| 创新思考 | 思考天团 - 设计思维、六顶思考帽 | 自动注入 | 需要创新时 |

**默认注入框架**:
- 5W1H1Y (需求分析)
- WBS (任务拆解)
- SWOT (态势分析)
- 第一性原理 (深度思考)
- OKR (目标管理)

### 2.2 PM Agent

**职责**: 任务调度、进度监控、结果汇总

**调用 Skills**:

| 场景 | 调用 Skill | 调用方式 | 触发条件 |
|------|-----------|----------|----------|

### 2.3 Ops Agent (运营总监)

**职责**: 全链路运营调度、部门协调、数据追踪

**调用 Skills**:

| 场景 | 调用 Skill | 调用方式 | 触发条件 |
|------|-----------|----------|----------|
| 任务启动 | **pua skill** | 自动注入 | 任何任务开始时 |
| 失败激励 | **pua skill** (L1-L4) | 自动触发 | 连续失败2+次 |
| 运营分析 | 思考天团 - 五因素诊断 | 自动注入 | 运营规划时 |
| 复盘沉淀 | **pua skill** - 5步方法论 | 自动注入 | 任务完成复盘 |
| 主动鞭策 | **pua skill** - Owner意识 | 自动注入 | 被动等待时 |
| 压力升级 | **pua skill** - 味道切换 | 决策调用 | 需要调整语气时 |

**压力升级映射**:

| 失败次数 | PUA Level | 旁白风格 | 强制动作 |
|----------|-----------|----------|----------|
| 2次 | L1 温和失望 | 阿里味 | 切换本质不同方案 |
| 3次 | L2 灵魂拷问 | 阿里味 | 搜索+读源码+3假设 |
| 4次 | L3 361考核 | 阿里味 | 7项检查清单 |
| 5次+ | L4 毕业警告 | 升级味道 | 拼命模式 |

**PUA Skill 引用**: `skills/pua/SKILL.md`

## 九、特殊情况处理

### 9.1 Skill 调用失败

**场景**: 调用 Skill 失败或无响应

**处理流程**:
```
1. 重试 1 次 (不同参数)
2. 切换备用 Skill
3. 降级到基础实现
4. 记录错误并报告
```

**示例**:
```yaml
baoyu-xhs-images 失败:
  1. 重试: baoyu-xhs-images --count 3 (减少数量)
  2. 备用: baoyu-cover-image (单张)
  3. 降级: MCP understand_image + 手动生成
  4. 报告: 记录到错误日志
```

---

### 9.2 多 Skill 协作

**场景**: 需要多个 Skill 配合完成

**协作模式**:
```yaml
串行协作:
  - Skill A 输出 → Skill B 输入
  - 示例: 思考天团 (用户画像) → baoyu-skills (生成内容)

并行协作:
  - 多个 Skill 同时执行
  - 示例: expert-user-persona + expert-scenario-role 并行分析

反馈协作:
  - Skill A → Skill B → 反馈到 Skill A
  - 示例: 撰写 → 审核 → 修改
```

## 十、最佳实践

### 10.1 Skill 调用原则

1. **优先级明确**: MiniMax MCP > 其他 MCP > 内置工具
2. **最小依赖**: 优先使用简单 Skill，避免过度复杂
3. **失败降级**: 始终准备备用方案
4. **结果验证**: 调用后验证输出质量
5. **记录日志**: 记录所有 Skill 调用

---

### 10.2 性能优化

1. **并行调用**: 独立 Skills 并行执行
2. **缓存结果**: 相同输入复用输出
3. **批量操作**: 合并相似请求
4. **超时控制**: 设置合理超时时间

---

### 10.3 质量控制

1. **输入验证**: 调用前验证参数
2. **输出检查**: 调用后检查结果
3. **异常处理**: 优雅处理失败
4. **反馈循环**: 持续优化调用策略
