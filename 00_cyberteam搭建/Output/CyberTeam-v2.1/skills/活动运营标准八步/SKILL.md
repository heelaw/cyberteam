| 编号 | 驱动力 | 适用场景 | 组合公式 |

| #1 | 利益驱动 | 新人注册、首单转化 | #1利益 + #2稀缺 |

| #3 | 竞争驱动 | 促活、GMV提升 | #3竞争 + #6荣誉 |

| #5 | 成就驱动 | 留存、任务 | #5成就 + #8游戏 |
| #6 | 荣誉驱动 | 品牌传播 | #6荣誉 + #4社交 |
| #7 | 恐惧驱动 | 召回、转化 | #7恐惧 + #1利益 |
| #8 | 游戏驱动 | 留存、互动 | #8游戏 + #5成就 |
| #9 | 好奇心驱动 | 预热、话题 | #9好奇心 + #2稀缺 |

### 二、目标-漏斗-发力点模型（策划核心工具）

```
活动目标（北极星）
       ↓ 公式拆解
   GMV = UV × 点击率 × 注册率 × 首付率 × 客单价
       ↓ 漏斗分析
   各环节转化率
       ↓ 瓶颈识别
   最薄弱环节 = 核心发力点
       ↓ 资源配置
   P1优先投入 → P2次之 → P3视情况
```

## 执行步骤（融合版）

### Step 1: 战略理解与目标设定

**目的**：明确活动在整体战略中的定位，确定目标和发力点

**操作流程**：

1. **理解战略背景**

   | 问题 | 回答 |
   |------|------|
   | 活动服务于什么战略目标？ | 用户增长/GMV提升/品牌曝光/留存 |
   | 活动在战略中的优先级？ | 高/中/低 |
   | 核心差异化是什么？ | 独特卖点 |
   | 预算和资源边界是什么？ | 预算范围/团队规模 |

2. **设定活动目标（公式化）**

   ```
   活动目标 = 业务指标 + 用户指标 + 传播指标

   公式拆解：
   GMV = UV × 点击率 × 首付率 × 客单价
   新用户 = 曝光 × 点击率 × 注册率 × 首付率

   示例：
   ├─ 业务指标：GMV 500万，新付费用户 1万
   ├─ 用户指标：活动参与 10万，UV 50万
   └─ 传播指标：分享率 5%，话题阅读量 1000万
   ```

3. **确定核心发力点**

   ```
   漏斗分析 → 识别瓶颈 → 计算发力价值

   发力优先级 = 提升空间 / 优化难度

   示例：
   - 首付率15%→25%（提升67%，难度中）→ P1优先
   - 点击率12%→18%（提升50%，难度低）→ P1次之
   - UV 80万→100万（提升25%，难度高）→ P2
   ```

### Step 2: 活动方案策划

**目的**：设计活动核心机制和驱动力组合

**操作流程**：

1. **确定活动类型和驱动力组合**

   | 活动类型 | 核心驱动 | 辅助驱动 | 激励重点 |
   |----------|----------|----------|----------|
   | 促销类 | #1利益 + #2稀缺 | #7恐惧 | 立即获得感 |
   | 获客类 | #4社交 + #1利益 | #3竞争 | 邀请排行榜 |
   | 留存类 | #5成就 + #8游戏 | #3竞争 | 任务+徽章 |
   | 品牌类 | #9好奇心 | #6荣誉 | 悬念+专属 |
   | 召回类 | #7恐惧 + #1利益 | #2稀缺 | 限时召回 |

2. **设计核心机制**

   ```
   活动机制 = 参与方式 + 驱动力组合 + 传播路径

   模板：
   [用户角色] 通过 [参与动作] 获得 [驱动力激励]，
   并通过 [传播机制] 触发 [裂变效果]

   示例：
   老用户通过分享邀请新用户，获得 [利益激励#1 + 竞争激励#3]，
   新用户完成首单后，老用户再得 [社交激励#4]，形成裂变闭环。
   ```

3. **撰写活动策划案**

   ```
   【活动策划案模板 v3.0】

   一、活动概述
   ├─ 活动名称：[名称]
   ├─ 活动主题：[主题]
   ├─ 活动时间：[起止日期]
   ├─ 活动目标：[量化目标]
   └─ 核心发力点：[P1优先优化环节]

   二、驱动力设计
   ├─ 主驱动力：[#X + #Y]
   ├─ 辅助驱动力：[#X]
   └─ 驱动力组合公式：[如 #2+#7=1.5倍紧迫]

   三、活动机制
   ├─ 参与条件：[谁可以参与]
   ├─ 参与路径：[如何参与]
   ├─ 激励机制：[九大驱动力对应设计]
   └─ 传播机制：[裂变路径]

   四、目标拆解
   ├─ 公式：[GMV = ...]
   ├─ 各环节目标值：[漏斗各层目标]
   └─ 发力点配置：[P1/P2/P3]

   五、资源需求
   ├─ 运营：[人员分工]
   ├─ 技术：[开发需求]
   ├─ 设计：[物料需求]
   └─ 预算：[费用明细+ROI预估]
   ```

### Step 3: 活动设计与落地

**目的**：将方案转化为可执行的设计稿和功能

**操作流程**：

1. **驱动力可视化设计**

   ```
   各驱动力的视觉表达：
   ├─ #1利益：红包/优惠券/礼品图标
   ├─ #2稀缺："仅剩X份"红色标签
   ├─ #3竞争：排行榜/排名徽章
   ├─ #5成就：进度条/徽章墙
   ├─ #7恐惧：倒计时/即将失效标签
   └─ #8游戏：积分/等级/收集元素
   ```

2. **文案撰写清单**

   | 驱动力 | 文案模板 | 示例 |
   |--------|----------|------|
   | #1利益 | [利益描述]+[获取方式] | "注册即送50元新人礼包" |
   | #2稀缺 | [稀缺描述]+[紧迫感] | "仅剩100份，错过再等一年" |
   | #7恐惧 | [损失描述]+[行动触发] | "您的专属优惠将于24h后失效" |

3. **功能开发需求**

   ```
   技术需求清单：
   ├─ 驱动力功能：激励机制配置、奖励发放
   ├─ 监控功能：实时数据看板、漏斗分析
   ├─ 测试功能：A/B测试框架
   └─ 裂变功能：分享追踪、邀请关系链
   ```

**目的**：确保活动就绪上线

   |----------|--------|------|

   | 商品/SKU | 备货/价格设置 | □ |
   | 库存 | 活动库存锁定 | □ |
   | 物料 | Banner/文案/客服 | □ |
   | 人员 | 排班/培训/话术 | □ |

2. **A/B测试准备**

   ```
   测试场景：
   ├─ 驱动力A/B测试：利益驱动 vs 稀缺驱动
   ├─ 文案A/B测试：不同驱动力文案效果
   └─ 路径A/B测试：参与流程优化
   ```

3. **预案演练**

   - 客服话术演练（针对各驱动力问题）
   - 紧急问题响应演练
   - 系统故障切换演练

### Step 5: 预热与传播

**目的**：蓄水造势，激活驱动力

**操作流程**：

1. **预热节奏安排**

   ```
   预热周期：活动前7天
   ├─ D-7：悬念释放（#9好奇心）→ "神秘大礼即将揭晓"
   ├─ D-5：价值传递（#1利益）→ 核心利益点曝光
   ├─ D-3：规则说明 + 驱动力组合发布
   ├─ D-1：最后召集（#2稀缺+#7恐惧）→ "明日开启，限量抢"
   └─ D-Day：正式启动
   ```

2. **传播渠道矩阵**

   | 渠道 | 内容类型 | 驱动力 | 目标 |
   |------|----------|--------|------|
   | 公众号 | 悬念预热 | #9好奇心 | 触达100万 |
   | 社群 | 实时播报 | #3竞争 | 活跃20万 |
   | APP弹窗 | 紧迫Push | #7恐惧 | 打开率30% |
   | 外部投放 | 利益诱导 | #1利益 | 引流50万 |

### Step 6: 正式执行与上线

**目的**：正式启动并确保稳定运行

**操作流程**：

1. **上线检查清单**

   ```
   【活动上线Checklist v3.0】
   □ 驱动力激励配置确认
   □ 0:00 活动准时开启
   □ 0:30 首小时数据+发力点监控
   □ 2:00 凌晨场确认
   □ 9:00 早高峰启动确认
   □ 12:00 午间数据+调优判断
   □ 18:00 晚高峰监控
   □ 24:00 当日数据汇总+发力效果评估
   ```

2. **实时监控看板（融合发力点）**

   | 环节 | 警戒值 | 当前值 | 状态 | 发力优先级 |
   |------|--------|--------|------|------------|
   | 点击率 | >10% | 12% | 🟢 | P2 |
   | 首付率 | >20% | 15% | 🔴 | **P1** |
   | 客单价 | >200 | 180 | 🟡 | P3 |

3. **调优策略（基于发力点）**

   | 问题 | 表现 | 调优手段（驱动力） |
   |------|------|---------------------|
   | 首付率低 | 转化漏斗断 | #1利益加码 + #7恐惧强化 |
   | 流量不足 | UV低于预期 | #2稀缺预热 + #9好奇心释放 |
   | 客单价低 | 转化OK但GMV低 | #2稀缺满减门槛 + #3竞争排行 |

### Step 7: 执行监控与调整

**目的**：实时优化，确保目标达成

**操作流程**：

1. **小时级数据监控**

   ```
   每小时输出：
   ├─ 各环节转化率 + 发力点效果评估
   ├─ 驱动力效果数据（各驱动力参与率）
   ├─ GMV/客单价实时
   └─ 异常预警
   ```

2. **实时调优决策**

   ```
   调优触发条件：
   ├─ 发力点指标持续低于目标 → 强化该环节驱动力
   ├─ 某驱动力参与率低 → 调整激励强度
   └─ 实时数据异常 → 启动预案

   调优工具箱：
   ├─ #1利益：临时加码优惠券
   ├─ #2稀缺：实时剩余库存播报
   ├─ #7恐惧：倒计时强化
   └─ #3竞争：实时排行榜更新
   ```
