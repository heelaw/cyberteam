## 核心框架

### 框架一：团队管理四力模型

```
                    ┌─────────────┐
                    │   凝聚力    │  团队基础
                    └──────┬──────┘
                           │
         ┌─────────────────┼─────────────────┐
         │                 │                 │
    ┌────┴────┐       ┌────┴────┐       ┌────┴────┐
    │  执行力  │       │  沟通力  │       │  学习力  │
    │ (做事)  │       │ (协作)  │       │ (成长)  │
    └─────────┘       └─────────┘       └─────────┘

四力平衡：
凝聚力是基础 + 执行力是核心 + 沟通力是保障 + 学习力是动力
```

### 框架二：管理者角色矩阵

| 角色 | 职责 | 日常行为 |
|------|------|----------|
| **战略家** | 指明方向 | 定目标、做决策 |
| **组织者** | 调配资源 | 排兵布阵、分工授权 |
| **教练** | 培养下属 | 指导、反馈、激励 |
| **连接者** | 跨部门协调 | 沟通、对齐、推动 |

### 框架三：团队发展阶段

```
阶段1：形成期 → 阶段2：风暴期 → 阶段3：规范期 → 阶段4：成熟期
  ↓                ↓                ↓                ↓
 建立信任        解决冲突         形成默契         高效协作

管理重点：
├─ 形成期：明确目标、清晰分工
├─ 风暴期：协调冲突、坚定方向
├─ 规范期：放权授权、流程优化
└─ 成熟期：持续创新、自我驱动
```

## 执行步骤

### Step 1: 团队目标设定

**目的**：让团队有明确的方向和共同的奋斗目标

**操作流程**：

1. **目标传递与分解**
   
   ```
   公司战略 → 部门目标 → 团队目标 → 个人目标
   
   目标分解原则：
   ├─ 纵向对齐：个人目标支撑团队目标
   ├─ 横向协同：团队成员目标相互配合
   └─ SMART原则：具体、可衡量、可实现、相关、有时限
   ```

2. **目标沟通与共识**
   
   | 步骤 | 内容 | 方式 |
   |------|------|------|
   | 传达 | 传递目标和背景 | 团队会议 |
   | 讨论 | 收集意见和问题 | 开放式讨论 |
   | 调整 | 根据反馈优化目标 | 协商确定 |
   | 确认 | 达成共识并承诺 | 书面确认 |

3. **目标责任到人**
   
   ```
   目标责任表：
   ┌─────────────────────────────────────────┐
   │ 目标：[具体目标描述]                      │
   │ 负责人：@张三                            │
   │ 协作人：@李四                            │
   │ 资源支持：[需要的资源]                    │
   │ 里程碑：[关键节点]                       │
   │ 完成标准：[如何衡量]                      │
   └─────────────────────────────────────────┘
   ```

---

### Step 2: 任务分配与授权

**目的**：合理分配工作，有效授权

**操作流程**：

1. **任务分析**
   
   | 维度 | 问题 | 判断 |
   |------|------|------|
   | 复杂度 | 任务有多复杂？ | 高/中/低 |
   | 紧急度 | 时间要求？ | 紧急/常规 |
   | 重要性 | 影响多大？ | 关键/一般 |
   | 成熟度 | 下属能力？ | 成熟/成长/新手 |

2. **分配策略选择**

   | 情境 | 策略 | 说明 |
   |------|------|------|
   | 高复杂性 + 低成熟度 | 指令式 | 具体指导，详细监督 |
   | 高复杂性 + 高成熟度 | 授权式 | 给予目标，自主完成 |
   | 低复杂性 + 低成熟度 | 教练式 | 引导思考，逐步培养 |
   | 低复杂性 + 高成熟度 | 支持式 | 提供资源，自行决策 |

3. **授权检查清单**
   
   ```
   有效授权要素：
   ├─ 任务清晰：明白要做什么
   ├─ 权责明确：有哪些权限
   ├─ 资源到位：需要的支持
   ├─ 标准明确：做到什么程度
   ├─ 反馈机制：如何汇报进展
   └─ 纠错机会：允许适度试错
   ```

---

### Step 3: 沟通与协调

**目的**：确保信息畅通，及时解决问题

**操作流程**：

1. **日常沟通机制**
   
   | 类型 | 频率 | 内容 | 时长 |
   |------|------|------|------|
   | 早会 | 每日 | 今日计划/阻塞问题 | 15分钟 |
   | 周会 | 每周 | 周报/进度/问题 | 1小时 |
   | 月会 | 每月 | 月报/复盘/规划 | 2小时 |
   | 一对一 | 每周/双周 | 个人发展/深度沟通 | 30分钟 |

2. **有效沟通技巧**
   
   ```
   沟通四步法：
   1. 开场：说明沟通目的
   2. 倾听：充分了解对方观点
   3. 表达：清晰阐述自己的想法
   4. 确认：达成共识，明确行动
   
   反馈技巧：
   ├─ BID反馈：Behavior行为 + Impact影响 + Desired outcome期望
   ├─ SBI反馈：Situation情境 + Behavior行为 + Impact影响
   └─ 负面反馈：先肯定再建议，私下进行
   ```

3. **冲突处理流程**
   
   ```
   冲突处理五步法：
   1. 识别：察觉冲突存在
   2. 分析：了解冲突原因和双方诉求
   3. 介入：组织双方沟通
   4. 调解：引导双方找到共同点
   5. 解决：达成共识并跟进执行
   ```

---

### Step 4: 绩效评估与反馈

**目的**：客观评价工作表现，促进持续改进

**操作流程**：

1. **绩效指标设定**
   
   ```
   KPI设定模板：
   ┌─────────────────────────────────────────┐
   │ 指标名称：[指标]                         │
   │ 目标值：[要达成的数值]                   │
   │ 衡量标准：[如何计算]                      │
   │ 数据来源：[从哪里取数]                   │
   │ 权重：[在整体中的重要性]                 │
   └─────────────────────────────────────────┘
   ```

2. **绩效评估流程**
   
   ```
   评估周期：月度跟踪 + 季度评估 + 年度考核
   
   评估维度：
   ├─ 结果指标：完成了什么（40%）
   ├─ 过程指标：怎么完成的（30%）
   ├─ 能力指标：能力有无提升（20%）
   └─ 态度指标：投入程度如何（10%）
   ```

3. **绩效面谈技巧**
   
   ```
   绩效面谈结构：
   ├─ 开场（10%）：轻松氛围，说明目的
   ├─ 员工自评（20%）：让员工先说
   ├─ 管理者反馈（40%）：肯定优点，指出不足
   ├─ 共同讨论（20%）：分析原因，制定改进
   └─ 总结承诺（10%）：明确下期目标
   
   面谈原则：
   ├─ 具体而非抽象：用事实说话
   ├─ 双向而非单向：倾听员工想法
   ├─ 建设性：关注未来改进
   └─ 书面记录：双方确认签字
   ```

   │  └─ 成长：培训机会 + 晋升通道            │
   ├─────────────────────────────────────────┤
   │              精神激励                    │
   │  ├─ 认可：公开表扬 + 荣誉奖励           │
   │  ├─ 成就：授权挑战 + 发展空间           │
   │  └─ 关系：团队氛围 + 归属感             │
   └─────────────────────────────────────────┘
   ```

2. **人才发展支持**
   
   ```
   培养方式矩阵：
   ┌─────────┬─────────────────┬─────────────────┐
   │         │    单个人        │     团队        │
   ├─────────┼─────────────────┼─────────────────┤
   │ 在岗    │ 1对1辅导        │ 轮岗历练        │

   | 骨干 | 发展空间 | 授权更大任务，晋升机会 |

   | 老员工 | 持续动力 | 新鲜感，适度轮岗 |

## 常见问题处理

### 问题1：团队士气低落

```
诊断流程：
├─ 访谈了解：倾听员工声音
├─ 根因分析：钱/发展/关系/工作本身
└─ 针对性解决

常见原因与对策：
├─ 薪酬不具竞争力 → 争取调薪/优化激励
├─ 看不到发展 → 明确晋升通道
├─ 工作氛围差 → 优化管理方式
└─ 缺乏成就感 → 调整任务设置
```

### 问题2：执行力不足

```
诊断流程：
├─ 目标清晰吗？ → 明确化
├─ 能力匹配吗？ → 培训/支援
├─ 激励到位吗？ → 强化激励
└─ 流程顺畅吗？ → 优化流程

提升执行力方法：
├─ 目标分解到周/日
├─ 过程检查与反馈
├─ 及时认可与奖惩
└─ 解决阻塞问题
```

### 问题3：团队协作差

```
诊断流程：
├─ 目标一致吗？ → 对齐目标
├─ 职责清晰吗？ → 明确分工
├─ 沟通顺畅吗？ → 改善机制
└─ 利益绑定吗？ → 协作激励

协作改善方法：
├─ 建立共同目标
├─ 明确职责边界
├─ 定期协作复盘
└─ 设置协作奖励
```

## 使用示例

### 场景：运营团队月度管理

**Step 1: 月度目标设定**

```
团队月度目标：
├─ 核心KPI：GMV增长30%
├─ 重点项目：618活动筹备
└─ 能力建设：数据分析培训

目标分解：
├─ @小王（活动运营）：活动GMV 200万
├─ @小李（用户运营）：新增用户5万
└─ @小张（内容运营）：发布30篇内容
```

**Step 2: 周例会执行**

```
周三团队周会（1小时）：

议程：
├─ 15分钟：各成员周报汇报
├─ 30分钟：618活动进展讨论
├─ 10分钟：问题升级与资源协调
└─ 5分钟：下周工作预告

会后行动：
├─ 整理会议纪要
├─ 跟进待办事项
└─ 更新项目看板
```

**Step 3: 一对一沟通**

```
与成员A的一对一（30分钟）：

开场："今天想听听你的想法，有什么想聊的？"

讨论话题：
├─ 工作中的挑战
├─ 需要的支持
├─ 职业发展想法
└─ 对团队的建议

总结承诺：
"下周我们试试……，1个月后我们review效果"
```
