└────────────────────────┬────────────────────────────────┘
                         ▼

│  Step 2: 事实核查 (fact-check)                              │
│  - 数据溯源验证                                             │

└────────────────────────┬────────────────────────────────┘
                         ▼

│  - 竞品描述检查                                             │

                         ▼

## 2. 三种审核类型

### 2.1 快速审核（轻量级）

适用场景：日常发布前快速检查

审核项目：
- 合规风险扫描（P0/P1 必检）
- 基础事实核查（数据声明）

输出：问题列表 + 通过/不通过结论

### 2.2 标准审核（推荐）

适用场景：正式内容发布

审核项目：
- 完整事实核查
- 品牌调性审核
- 合规风险扫描（全项）

输出：完整审核报告 + 修改建议

### 2.3 深度审核（严格）

适用场景：重要品牌内容、合作推广

审核项目：
- 标准审核全部项目
- 多平台规则适配检查
- 知识产权深度审查
- 竞品内容专项分析

输出：完整报告 + 修改方案 + 风险评估

## 3. 输出标准

### 3.1 问题位置标注

使用「段落 + 句子」定位：

```
第1段：「原文内容」
     ↑  ↑
     │  └ 句子编号
     └ 段落编号
```

### 3.2 问题分类标准

| 类别 | 编码 | 说明 |
|------|------|------|
| 事实错误 | F-XX | Fact 相关问题 |
| 逻辑问题 | L-XX | Logic 相关问题 |
| 引用问题 | Q-XX | Quote 相关问题 |
| 语气问题 | T-XX | Tone 相关问题 |
| 用词问题 | W-XX | Word 相关问题 |
| 广告法 | A-XX | Advertising 法规模问题 |
| 平台规则 | P-XX | Platform 平台规则问题 |
| 知识产权 | I-XX | IP 知识产权问题 |

### 3.3 风险等级说明

| 等级 | 标记 | 含义 | 操作要求 |
|------|------|------|----------|
| 严重 | P0 | 致命风险 | 必须修改，否则不可发布 |
| 较高 | P1 | 高风险 | 建议修改，否则可能受限 |
| 一般 | P2 | 优化项 | 可选修改，提升质量 |

### 3.4 审核结论

| 结论 | 条件 | 说明 |
|------|------|------|
| **通过** | P0=0, P1=0 | 可直接发布 |
| **建议修改后发布** | P0=0, P1>0 | 修改 P1 项后发布 |
| **必须修改** | P0>0 | 修改所有 P0 项后重新审核 |
| **人工复核** | 特殊情况 | 涉及重大风险，需人工判断 |

## 4. 与人工审核的协作

### 4.1 AI 审核边界

**AI 擅长**：
- 文本模式识别（违禁词、敏感词）
- 规则化检查（广告法、平台规则）
- 结构化输出（问题列表、格式报告）

**AI 不擅长**：
- 主观质量判断（文采、创意）
- 语境理解（讽刺、反语）
- 最新政策解读
- 品牌意图揣摩

### 4.2 协作流程

```
AI 审核（初筛）
    ↓
    ├─ P0 问题 → 强制修改 → AI 复审
    │
    ├─ P1 问题 → 人工确认 → 修改/保留 → 备注理由
    │
    └─ P2 问题 → 人工参考 → 可选修改
         ↓
    人工终审（品牌负责人）
         ↓
    发布
```

### 4.3 人工复核触发条件

以下情况必须人工复核：

1. 涉及政治、宗教、性别等敏感话题
2. 涉及竞品比较且可能被认定为贬低
3. 医疗、健康、食品功效相关声明
4. 投资、金融相关承诺
5. 涉及未成年人内容
6. AI 判断不确定（置信度 < 80%）

### 4.4 复核记录

人工复核时需记录：

```markdown
## 人工复核记录

复核人：[姓名]
复核时间：[时间]
复核结论：[通过/修改后通过/不通过]

P1 问题处理：
- 问题 A-01：[保留/修改] - 理由：...

P2 问题处理：
- 问题 B-02：[保留/忽略] - 理由：...

特殊说明：
[其他需要记录的事项]
```

## 5. 常见场景处理

### 5.1 数据来源标注缺失

处理方式：
1. 标记为 P1 问题
2. 建议补充数据来源
3. 如数据为估算，需明确标注

### 5.2 竞品对比边界

处理方式：
1. 事实对比 → 可接受
2. 数据对比 → 需数据来源支撑
3. 主观贬低 → P1 问题
4. 贬损性描述 → P0 问题

### 5.3 引号引用争议

处理方式：
1. 直接引用 → 必须原文一致
2. 间接引用 → 需标注「据...」
3. 疑似编造 → 标记为 P0 风险

### 5.4 平台规则冲突

处理方式：
1. 识别多平台发布需求
2. 针对各平台单独检查
3. 冲突项列出，供选择

## 6. 效率优化建议

1. **批量审核**：同类内容可批量提交
2. **模板复用**：常见问题建立修改模板
3. **增量审核**：修改后仅审核变更部分
4. **历史参考**：类似问题参考历史处理记录
