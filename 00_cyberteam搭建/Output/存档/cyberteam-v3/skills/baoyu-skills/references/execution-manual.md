## 二、图像生成技能详解

### 2.1 baoyu-cover-image

**用途**: 生成单张封面图

**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| title | string | 是 | 封面标题 |
| --type | string | 否 | 尺寸类型: portrait/landscape/square |
| --style | string | 否 | 风格: modern/minimal/vibrant |
| --color | string | 否 | 主色调 |

**执行流程**:

```
1. 确定封面主题和标题
   └── 标题要有吸引力

2. 选择尺寸规格
   ├── 小红书: portrait (3:4)
   ├── 公众号: landscape (2.35:1)
   └── 微博: square (1:1)

3. 选择风格
   ├── modern: 简洁现代
   ├── minimal: 极简风格
   └── vibrant: 活泼鲜艳

4. 生成图像
   └── baoyu-cover-image "标题" --type portrait

5. 审核优化
   └── 检查图像质量，必要时重新生成
```

**输出标准**:

```markdown
## 封面图生成报告

### 封面信息
- 标题: [标题]
- 尺寸: [宽]x[高] ([比例])
- 风格: [风格]
- 主色调: [颜色]

### 生成结果
- 文件: [文件名].png
- 路径: [存储路径]
- 分辨率: 1242x1660 px

### 审核结果
- [x] 标题清晰可见
- [x] 色彩协调
- [x] 无版权问题
- [ ] 需要优化: [问题描述]

### 优化建议
[如有优化需要，提供具体建议]
```

### 2.2 baoyu-xhs-images

**用途**: 生成小红书系列图 (1-9张)

**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| title | string | 是 | 内容主题 |
| --count | number | 否 | 数量 (1-9, 默认3) |
| --aspect | string | 否 | 比例: 3:4/16:9/1:1 |
| --publish | boolean | 否 | 是否直接发布 |

**执行流程**:

```
1. 确定内容主题
   └── 明确系列图的叙事逻辑

2. 规划图片数量
   ├── 1张: 单一主题强调
   ├── 3张: 问题-方案-结果
   ├── 5张: 完整故事线
   └── 9张: 详细教程/攻略

3. 确定比例
   └── 小红书推荐 3:4 (竖版)

4. 生成系列图
   └── baoyu-xhs-images "主题" --count 5

5. 审核发布
   ├── 检查图片连贯性
   ├── 验证文字清晰度
   └── 发布 (可选)
```

**输出标准**:

```markdown
## 小红书系列图报告

### 系列信息
- 主题: [主题]
- 数量: [N] 张
- 尺寸: 1242x1660 px (3:4)

### 系列结构
1. [图1描述]
2. [图2描述]
3. [图3描述]
...

### 生成结果
| 图片 | 文件 | 状态 |
|------|------|------|
| 封面 | xhs-cover-001.png | ✅ |
| 内容1 | xhs-content-001.png | ✅ |
| ... | ... | ... |

### 发布状态
- 平台: 小红书
- 状态: [已发布/待发布]
- 链接: [链接]
```

## 三、多平台发布技能详解

### 3.1 baoyu-post-to-wechat

**用途**: 发布到微信公众号

**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| title | string | 是 | 文章标题 |
| content | string | 是 | 文章内容 (Markdown) |
| --cover | string | 否 | 封面图路径 |
| --tags | string | 否 | 标签 |
| --draft | boolean | 否 | 是否仅保存为草稿 |

**执行流程**:

```
1. 准备内容
   ├── 标题: 吸引人、有关键词
   ├── 正文: Markdown 格式
   └── 封面: 2.35:1 比例

2. 格式检查
   ├── Markdown 语法正确
   ├── 图片链接有效
   └── 排版美观

3. 发布
   └── baoyu-post-to-wechat [标题] [内容] --cover [图]

4. 验证
   ├── 检查发布状态
   └── 记录链接
```

**公众号规范**:

| 项目 | 规范 |
|------|------|
| **封面** | 2.35:1, 900x383 px, JPG/PNG |
| **标题** | 14字以内，避免过长 |
| **正文** | Markdown, 图片≤5MB |
| **标签** | 最多3个 |
| **阅读时长** | 3-5分钟最佳 |

## 四、格式转换技能详解

### 4.1 baoyu-format-markdown

**用途**: 格式化 Markdown 内容

**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| content | string | 是 | 原始内容 |
| --style | string | 否 | 风格: simple/rich/minimal |
| --platform | string | 否 | 平台: wechat/xhs/weibo |

**执行流程**:

```
1. 分析原始内容
   └── 识别标题、列表、代码块等

2. 应用格式
   ├── 标题层级规范化
   ├── 列表符号统一
   ├── 代码块优化
   └── 图片处理

3. 平台适配
   └── 根据平台调整格式

4. 输出
   └── 生成标准化 Markdown
```

## 六、内容创作完整流程

### 6.1 小红书笔记创作流程

```
1. 需求确认
   └── 主题、目标用户、内容方向

2. 图像生成 (baoyu-xhs-images)
   ├── baoyu-xhs-images "主题" --count 5
   └── 输出: 5张系列图

3. 文案撰写 (7专家协作)
   ├── expert-user-persona: 用户画像
   ├── expert-copywriter: 文案撰写
   └── expert-content-reviewer: 内容审核

4. 格式优化 (baoyu-format-markdown)
   └── baoyu-format-markdown [内容] --platform xhs

5. 发布 (baoyu-xhs-images --publish)
   └── 直接发布到小红书

6. 效果追踪
   └── 记录曝光、点赞、收藏数据
```

---

### 6.2 公众号文章创作流程

```
1. 需求确认
   └── 主题、字数、图片需求

2. 内容创作 (writing skill)
   ├── Stage 1-4: 用户研究、卖点、痛点、策略
   ├── Stage 5-6: 渠道策略、文案撰写
   └── Stage 7-8: 优化、追踪

3. 封面生成 (baoyu-cover-image)
   └── baoyu-cover-image "标题" --type landscape

4. 格式转换 (baoyu-format-markdown)
   └── baoyu-format-markdown [内容] --platform wechat

5. 发布 (baoyu-post-to-wechat)
   └── baoyu-post-to-wechat [标题] [内容] --cover [图]

6. 效果追踪
   └── 记录阅读量、在看、分享数据
```

## 七、质量标准

### 7.1 图像质量标准

| 检查项 | 标准 |
|--------|------|
| **清晰度** | ≥72 DPI, 无模糊 |
| **文字** | 可识别, 无错别字 |
| **色彩** | 协调, 无色差 |
| **版权** | 无侵权, 有授权 |
| **比例** | 符合平台要求 |
| **大小** | ≤5MB |

---

### 7.2 文案质量标准

| 检查项 | 标准 |
|--------|------|
| **标题** | 吸引人, 有关键词 |
| **结构** | 清晰, 分段合理 |
| **内容** | 有价值, 无错误 |
| **CTA** | 明确, 有行动引导 |
| **合规** | 无敏感词, 符合广告法 |

---

### 7.3 发布质量标准

| 检查项 | 标准 |
|--------|------|
| **格式** | 符合平台规范 |
| **图片** | 加载正常, 显示正确 |
| **链接** | 有效, 可访问 |
| **标签** | 准确, 无堆砌 |
| **时间** | 最佳发布时间 |
