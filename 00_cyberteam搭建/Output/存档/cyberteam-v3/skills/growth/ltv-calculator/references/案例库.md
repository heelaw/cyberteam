# 案例库

## 案例1: Netflix（订阅流媒体）

### 背景信息
- **产品类型**: 订阅制流媒体
- **商业模式**: 会员订阅费（月费/年费）
- **市场**: 全球190+国家
- **数据周期**: 2019-2023

### 核心数据

| 指标 | 数值 | 说明 |
|------|------|------|
| 月费 | $15.49 | 标准套餐 |
| 年费 | $139 | 约$11.58/月 |
| 月流失率 | 3.5% | 行业领先水平 |
| 用户生命周期 | 28.5月 | 1/0.035 |
| ARPU | $11.58 | 年费折算月均 |
| CAC | ~$40 | 主流市场均值 |

### LTV计算

**使用模型**: Simple LTV（流失稳定）

```
LTV = ARPU × (1 / 月流失率)
    = $11.58 × (1 / 0.035)
    = $11.58 × 28.57
    = $330.76
```

### LTV/CAC分析

```
LTV/CAC = $330.76 / $40 = 8.27:1
```

### 决策与结果

| 决策 | 理由 | 结果 |
|------|------|------|
| 大规模全球扩张 | LTV/CAC > 5:1 | 用户从2亿增至2.6亿 |
| 高内容投入 | 用户价值支撑内容成本 | 内容成本$170亿/年 |
| 提价策略 | 验证用户价值承受力 | 多次提价用户仍增长 |

### 关键洞察

1. **极低流失率是核心竞争力**: 3.5%月流失率意味着用户生命周期近30个月
2. **年费提升LTV**: 年费用户ARPU更低但生命周期更长
3. **全球差异**: 不同市场CAC差异大，需分市场计算LTV

## 案例2: 京东（电商平台）

### 背景信息
- **产品类型**: B2C电商平台
- **商业模式**: 商品销售+平台服务+物流服务
- **市场**: 中国大陆
- **数据周期**: 2022-2024

### 核心数据

| 指标 | 数值 | 说明 |
|------|------|------|
| 年活跃用户 | 5.9亿 | 2023年 |
| 年订单量 | 6.3亿 | 2023年 |
| 人均年订单 | 10.7单 | 6.3亿/5.9亿 |
| 客单价 | ¥182 | 年GMV/年订单 |
| 综合毛利率 | 14.8% | 2023年 |
| 获客成本CAC | ~¥80 | 主流渠道均值 |

### LTV计算

**使用模型**: Cohort LTV（复购周期明确）

**年度Cohort分析**:

| 用户年份 | 当年消费 | 次年复购率 | 次年消费 | 第3年消费 |
|----------|----------|------------|----------|------------|
| 第1年 | ¥1820 | 65% | ¥1183 | - |
| 第2年 | - | 45% | ¥819 | ¥368 |

**简化计算**:
```
首年LTV = ¥1820
次年LTV = ¥1183 × 65% = ¥769
第3年LTV = ¥819 × 45% = ¥369

总LTV = 1820 + 769 + 369 = ¥2958
```

**考虑毛利后**:
```
利润LTV = ¥2958 × 14.8% = ¥438
```

### LTV/CAC分析

```
LTV/CAC = ¥2958 / ¥80 = 37:1 (收入)
利润LTV/CAC = ¥438 / ¥80 = 5.5:1
```

### 决策与结果

| 决策 | 理由 | 结果 |
|------|------|------|
| 会员PLUS体系 | 提升复购和客单价 | PLUS会员年消费是非会员8倍 |
| 京东物流开放 | 提升用户体验 | 物流收入成新增长点 |
| 品类扩张 | 提升用户钱包份额 | 品类从3C扩展到全品类 |

### 关键洞察

1. **复购率是关键**: 电商LTV核心来自复购，不是首单
2. **毛利差异大**: 不同品类毛利差异显著，需分品类计算
3. **PLUS会员**: 付费会员ARPU是非会员的5-8倍

## 案例3: 网易云音乐（音乐订阅）

### 背景信息
- **产品类型**: 音乐流媒体+社交
- **商业模式**: 会员订阅+广告+增值服务
- **市场**: 中国
- **数据周期**: 2022-2024

### 核心数据

| 指标 | 数值 | 说明 |
|------|------|------|
| 付费会员数 | 4000万+ | 2024年 |
| 会员价格 | ¥8/月 | 首月¥8，后续¥15 |
| 整体付费率 | 约20% | 付费会员/活跃用户 |
| ARPPU | ¥9.2 | 会员收入/付费用户 |
| 月流失率 | 8% | 行业中等水平 |
| CAC | ~¥15 | 渠道投放均值 |

### LTV计算

**使用模型**: Marginal LTV（用户分层明显）

**用户分层**:

| 用户类型 | 占比 | 贡献 |
|----------|------|------|
| 免费用户 | 80% | ¥0.5/月（广告） |
| 付费会员 | 20% | ¥9.2/月 |

**Simple LTV（付费会员）**:
```
LTV = ¥9.2 × (1/0.08) = ¥115
```

**综合LTV（含免费用户广告价值）**:
```
免费用户LTV = ¥0.5 × (1/0.15) = ¥3.3
付费用户LTV = ¥9.2 × (1/0.08) = ¥115
综合LTV = ¥3.3×0.8 + ¥115×0.2 = ¥2.64 + ¥23 = ¥25.64
```

### LTV/CAC分析

```
LTV/CAC（整体）= ¥25.64 / ¥15 = 1.7:1 (警告)
LTV/CAC（付费会员）= ¥115 / ¥15 = 7.7:1 (优秀)
```

### 决策与结果

| 决策 | 理由 | 结果 |
|------|------|------|
| 聚焦付费转化 | 付费会员LTV高 | 付费率从10%提到20% |
| 社交功能强化 | 提升免费用户粘性 | 用户停留时长提升 |
| 音乐+社交差异化 | 社区价值提升LTV | 用户留存率改善 |

### 关键洞察

1. **免费用户也有价值**: 广告收入可以覆盖部分CAC
2. **分层计算必要**: 整体LTV掩盖了付费用户的真实价值
3. **社交提升LTV**: 网易云音乐通过社交功能提升留存，间接提升LTV

## 案例4: 某SaaS产品（B2B）

### 背景信息
- **产品类型**: B2B SaaS协作工具
- **商业模式**: 订阅制年费
- **客户**: 中小企业
- **数据周期**: 2023年

### 核心数据

| 指标 | 数值 | 说明 |
|------|------|------|
| 年费定价 | ¥12,000 | 基础版 |
| 扩展模块 | +¥5,000 | 可选增值 |
| 月流失率 | 6% | 中小企业SaaS平均 |
| 扩展率 | 30% | 30%客户购买扩展 |
| CAC | ¥8,000 | 销售+市场成本 |
| 毛利 | 75% | SaaS高毛利 |

### LTV计算

**基础用户LTV**:
```
LTV = ¥12,000 × (1/0.06) × 12月
    = ¥12,000 × 16.67 × 12
    = ¥2,400,000 / 100用户
    = ¥24,000/用户
```

**考虑扩展收入**:
```
扩展收入 = ¥5,000 × 30% = ¥1,500/年
扩展LTV = ¥1,500 × 16.67 = ¥25,000
总LTV = ¥24,000 + ¥25,000 = ¥49,000
```

**考虑毛利后**:
```
利润LTV = ¥49,000 × 75% = ¥36,750
```

### LTV/CAC分析

```
LTV/CAC = ¥49,000 / ¥8,000 = 6.1:1
利润LTV/CAC = ¥36,750 / ¥8,000 = 4.6:1
```

### 决策与结果

| 决策 | 理由 | 结果 |
|------|------|------|
| 销售驱动增长 | LTV/CAC > 5:1 | 组建销售团队 |
| 扩展模块推广 | 扩展收入显著 | 扩展率提升到40% |
| 降低流失 | 流失降低10%可提升LTV 15% | 客户成功团队 |

### 关键洞察

1. **扩展收入重要**: B2B SaaS的LTV不能只看基础订阅
2. **客户成功降低流失**: 主动服务显著降低流失率
3. **年费预付**: 年费模式现金流好，LTV计算需考虑时间价值

## 案例5: 某游戏产品（手游）

### 背景信息
- **产品类型**: RPG手游
- **商业模式**: 虚拟物品+抽卡
- **市场**: 中国
- **数据周期**: 2023年

### 核心数据

| 指标 | 数值 | 说明 |
|------|------|------|
| 付费率 | 3.5% | 100个用户3.5个付费 |
| ARPPU | ¥280 | 月付费用户平均 |
| 首充转化 | 15% | 首充用户占比 |
| 月流失率 | 35% | 早期游戏高流失 |
| 大R占比 | 2% | 贡献50%收入 |
| CAC | ¥25 | 买量成本 |

### LTV计算

**使用模型**: Marginal LTV（付费分层明显）

**用户分层**:

| 分层 | 月付费 | 占比 | 说明 |
|------|--------|------|------|
| 免费 | ¥0 | 96.5% | 不付费 |
| 首充 | ¥6 | 2% | 仅首充 |
| 小R | ¥30 | 1% | 月卡/小礼包 |
| 中R | ¥150 | 0.4% | 中等付费 |
| 大R | ¥2000+ | 0.1% | 头部玩家 |

**分层计算**:
```
免费用户LTV = ¥0 × (1/0.35) = ¥0
首充用户LTV = ¥6 × 3月 × 20%复充 = ¥3.6
小R用户LTV = ¥30 × (1/0.2) = ¥150
中R用户LTV = ¥150 × (1/0.15) = ¥1000
大R用户LTV = ¥2000 × (1/0.1) = ¥20,000
```

**加权综合LTV**:
```
综合LTV = 0×0.965 + 3.6×0.02 + 150×0.01 + 1000×0.004 + 20000×0.001
        = 0 + 0.072 + 1.5 + 4 + 20
        = ¥25.57
```

### LTV/CAC分析

```
LTV/CAC = ¥25.57 / ¥25 = 1.02:1 (危险边缘)
```

### 问题诊断与优化

| 问题 | 分析 | 优化建议 |
|------|------|----------|
| 整体LTV/CAC接近1 | 买量成本高，回收困难 | 优化投放ROI |
| 大R贡献过高 | 收入集中风险 | 提升中小R付费 |
| 流失率高 | 35%月流失过高 | 优化留存活动 |

### 决策与结果

| 决策 | 理由 | 结果 |
|------|------|------|
| 精细化投放 | LTV/CAC接近1 | 分渠道核算 |
| 提升中小R付费 | 分散收入风险 | 推出月卡/成长基金 |
| 留存优化 | 降低流失提升LTV | 签到/登录奖励 |
