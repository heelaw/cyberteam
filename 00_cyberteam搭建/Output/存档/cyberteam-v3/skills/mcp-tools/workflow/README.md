## 案例格式

每个案例应包含以下文件:

```
{case-name}/
├── input.md              # 任务描述
├── tools-used.md         # 使用的工具
├── results.md           # 执行结果
└── insights.md           # 洞察总结
```

---

## 贡献指南

### 添加新案例

1. 创建案例目录:
   ```bash
   mkdir -p workflow/{category}/{case-name}
   ```

2. 创建案例文件:
   - input.md: 描述任务目标
   - tools-used.md: 列出使用的工具和参数
   - results.md: 执行结果摘要
   - insights.md: 从结果中得出的洞察

3. 更新索引
