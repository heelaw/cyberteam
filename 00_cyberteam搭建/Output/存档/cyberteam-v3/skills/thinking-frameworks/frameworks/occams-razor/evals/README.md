# 奥卡姆剃刀专家测试用例

## 正面测试

### 测试1: 原因分析
- 输入:
```
【奥卡姆剃刀分析请求】
现象/问题：销售团队士气低落
已有解释：1)薪酬不公平 2)管理方式有问题 3)工作量太大
复杂度要求：中等
```
- 期望输出:
  - 穷举了所有候选解释
  - 每个解释的复杂度评估
  - 解释力对比
  - 推荐最简洁且解释力足够的
- 验证项:
  - [ ] 不因某个解释简单就否定它
  - [ ] 评估了每个解释的证据支持度

### 测试2: 方案选择
- 输入:
```
【奥卡姆剃刀分析请求】
现象/问题：需要提高团队协作效率
候选方案：A)引入复杂的企业协作软件 B)先从简单的晨会制度开始 C)外聘咨询顾问做全套诊断
复杂度要求：高（需要考虑实施复杂度）
```
- 期望输出:
  - 各方案的复杂度评估
  - 各方案的实施难度
  - 基于平衡艺术的推荐
- 验证项:
  - [ ] 识别了过度工程化的问题
  - [ ] 接受了"简单方案显著更好"的情况

## 负面测试

### 测试1: 幼儿化错误
- 输入:
```
【奥卡姆剃刀分析请求】
现象/问题：为什么苹果落地？
候选解释：1)因为万有引力 2)因为地球在吸引它 3)因为时空弯曲
```
- 期望行为: 不能选择过度简化的解释
- 验证项:
  - [ ] 识别万有引力是正确但更深层的解释
  - [ ] 知道何时需要更深层的解释

### 测试2: 忽视证据
- 输入:
```
【奥卡姆剃刀分析请求】
现象/问题：为什么这个项目失败了？
候选解释：1)团队不行（复杂，贬低团队）2)计划过于激进（简单，但有证据）
```
- 期望行为: 不能因为某个解释"伤害感情"就拒绝证据
- 验证项:
  - [ ] 基于证据而非动机做选择
  - [ ] 不因简单解释而拒绝正确但复杂的解释

### 测试3: 预设偏好
- 输入:
```
【奥卡姆剃刀分析请求】
现象/问题：为什么用户不喜欢新产品？
候选解释：1)产品确实有问题（证据充分）2)竞争对手抹黑（假设多）
```
- 期望行为: 不能预设"简单的就是好的"
- 验证项:
  - [ ] 评估了证据支持度
  - [ ] 推荐了有证据支持但可能更复杂的解释
