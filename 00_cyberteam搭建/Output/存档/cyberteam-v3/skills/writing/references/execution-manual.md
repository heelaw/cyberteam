## 1. 工作流概述

转化型文案创作工作流是一个系统化的9阶段流程，旨在确保每篇文案都经过完整的创作、审核、打磨流程，最终产出高转化率的营销文案。

### 1.1 9阶段工作流

| Stage | 名称 | 输入 | 输出 | 目录 |
|-------|------|------|------|------|
| **Stage 0** | 工作流初始化 | 用户需求 | 工作流配置 | `workflow/00-init/` |
| **Stage 1** | 用户需求挖掘 | 产品信息 | 用户研究报告 | `workflow/01-user-research/` |
| **Stage 2** | 产品卖点挖掘 | 用户研究报告 | 卖点分析报告 | `workflow/02-selling-points/` |
| **Stage 3** | 痛点定位 | 卖点分析报告 | 痛点匹配报告 | `workflow/03-pain-points/` |
| **Stage 4** | 需求类型判断 | 痛点匹配报告 | 文案策略报告 | `workflow/04-strategy/` |
| **Stage 5** | 场景与渠道分析 | 文案策略报告 | 渠道策略报告 | `workflow/05-channel-strategy/` |
| **Stage 6** | 文案撰写 | 渠道策略报告 | 文案初稿 | `workflow/06-draft/` |
| **Stage 7** | 文案优化 | 文案初稿 | 最终文案 | `workflow/07-final/` |
| **Stage 8** | 效果追踪与迭代 | 最终文案 | 效果追踪建议 | `workflow/08-tracking/` |

### 1.2 工作流目标

- 确保文案基于真实用户反馈和数据分析
- 严格遵循投放平台的特点和规则
- 每一句话都服务于最终转化
- 深刻理解用户需求和心理
- 避免冗余信息，突出核心价值
- 采用口语化表达，自然流畅
- 包含明确的行动召唤
- 确保合规合法，无虚假宣传

## 2. 策略三角决策流程

### 2.1 什么是策略三角

策略三角是文案创作的核心决策框架，用于确定文案的沟通策略方向。它由四个核心维度组成：

```
                    ┌─────────────────┐
                    │    目标用户      │
                    │  (WHO)          │
                    └────────┬────────┘
                             │
         ┌───────────────────┼───────────────────┐
         │                   │                   │
         ↓                   ↓                   ↓
┌─────────────────┐ ┌─────────────────┐ ┌─────────────────┐
│   说什么        │ │   怎么说        │ │   在哪说        │
│   (WHAT)        │ │   (HOW)         │ │   (WHERE)       │
└─────────────────┘ └─────────────────┘ └─────────────────┘
```

### 2.2 策略三角四维度详解

| 维度 | 核心问题 | 决策内容 | 产出文档 |
|------|----------|----------|----------|
| **WHO** | 我们的内容给谁看？ | 目标用户画像 | 用户画像卡 |
| **WHAT** | 用户最关心什么？我们要传达什么？ | 核心卖点+痛点匹配 | 卖点分析报告 |
| **HOW** | 用什么语气、逻辑、风格？ | 说服模式+沟通基调 | 策略决策卡 |
| **WHERE** | 用户在什么场景接触内容？ | 渠道策略+内容适配 | 渠道策略报告 |

### 2.3 策略三角执行流程

```
┌─────────────────────────────────────────────────────────┐
│                    策略三角决策流程                       │
└─────────────────────────────────────────────────────────┘

Step 1: 确定目标用户（WHO）
   │
   ├── 分析用户基础属性（年龄/性别/地域/职业）
   ├── 分析用户数字生活（App使用/KOL关注/内容偏好）
   ├── 分析用户场景与行为（关键场景/行为路径）
   ├── 分析用户心理与情感（痛点/渴望/焦虑/决策障碍）
   └── 输出：深度用户画像卡

Step 2: 确定说什么（WHAT）
   │
   ├── 挖掘产品三层卖点（物理层/体验层/情感层）
   ├── 回答7个核心问题
   ├── 匹配用户痛点
   └── 输出：卖点分析报告 + 痛点匹配报告

Step 3: 确定怎么说（HOW）
   │
   ├── 判断需求强度（L1/L2/L3）
   ├── 确定场景类型（主动搜索/社交浏览/内容种草/决策辅助）
   ├── 选择说服模式（理性/情感/社会证明/稀缺）
   └── 输出：核心文案策略决策卡

Step 4: 确定在哪说（WHERE）
   │
   ├── 分析渠道特征（转化能力/阅读时长/内容形式）
   ├── 判断阅读时间档位
   ├── 制定渠道差异化策略
   └── 输出：渠道策略报告

Step 5: 整合策略陈述
   │
   └── 一句话总结策略方向
```

### 2.4 策略三角检查清单

```
□ WHO - 目标用户是否清晰？
  ├─ 人口统计特征明确
  ├─ 数字生活特征明确
  ├─ 场景与行为路径清晰
  └─ 心理与情感洞察到位

□ WHAT - 说什么是否精准？
  ├─ 核心卖点明确
  ├─ 卖点与痛点匹配
  └─ 信息层级清晰

□ HOW - 怎么说是否合适？
  ├─ 需求强度判断正确
  ├─ 场景类型判断正确
  └─ 说服模式匹配

□ WHERE - 在哪说是否适配？
  ├─ 渠道选择正确
  ├─ 内容长度合适
  └─ 跨场景引导设置
```

## 3. 需求强度三等级判断标准

### 3.1 需求强度定义

| 等级 | 名称 | 用户心理 | 行为特征 | 文案策略 |
|------|------|----------|----------|----------|
| **L1** | 强需求 | "我必须解决这个问题" | 主动搜索、比较、立即购买 | 强调方案优势+行动便利性 |
| **L2** | 中需求 | "这个问题可以考虑解决" | 浏览信息、被动种草、犹豫中 | 制造紧迫感+差异化价值 |
| **L3** | 弱需求 | "我好像不需要这个" | 随意浏览、兴趣缺缺 | 重构认知+制造认知失衡 |

### 3.2 需求强度详细判断标准

#### L1 强需求特征

| 判断维度 | 具体表现 | 示例 |
|----------|----------|------|
| 主动表达 | 用户主动提及需求 | "我想找一款能祛痘的精华" |
| 行为证据 | 用户在主动搜索解决方案 | 搜索相关关键词、浏览对比内容 |
| 时间敏感 | 有明确的截止时间 | "下周要参加婚礼" |
| 付费意愿 | 有明确的预算范围 | "我愿意花500以内" |

#### L2 中需求特征

| 判断维度 | 具体表现 | 示例 |
|----------|----------|------|
| 模糊表达 | 用户有困扰但不紧迫 | "最近皮肤好像有点干" |
| 被动接收 | 用户在看相关内容但未主动搜索 | 浏览种草内容、点赞收藏 |
| 可延迟 | 没有明确的时间压力 | "有空的话可以试试" |
| 比较心态 | 用户在多个方案中犹豫 | 对比多个品牌/产品 |

#### L3 弱需求特征

| 判断维度 | 具体表现 | 示例 |
|----------|----------|------|
| 无感表达 | 用户认为自己不需要 | "我不需要这个" |
| 无意识 | 用户还没意识到问题存在 | 潜在需求待挖掘 |
| 优先级低 | 用户有更重要的事情 | 其他需求排在前面 |
| 认知偏差 | 用户对产品/品类有误解 | 认为产品不适合自己 |

### 3.3 需求强度判断流程

```
Step 1: 用户是否主动表达过相关需求？
  │
  ├─ 是 → 继续 Step 2
  └─ 否 → 继续 Step 4

Step 2: 用户是否有明确的问题或痛点？
  │
  ├─ 是 → 继续 Step 3
  └─ 否 → L2 中需求

Step 3: 用户是否在主动寻找解决方案？
  │
  ├─ 是 → L1 强需求
  └─ 否 → L2 中需求

Step 4: 用户是否意识到自己有这个需求？
  │
  ├─ 是（但没表达）→ L2 中需求
  └─ 否 → L3 弱需求
```

### 3.4 各需求强度的核心文案策略

| 需求等级 | 开篇策略 | 核心卖点表达 | CTA设计 |
|----------|----------|--------------|----------|
| **L1 强需求** | 直接切入方案 | 优势对比+效率证明 | 立即行动/限时优惠 |
| **L2 中需求** | 制造共鸣/场景代入 | 价值凸显+差异化 | 引导尝试/关注账号 |
| **L3 弱需求** | 颠覆认知/好奇心 | 新视角/新概念 | 引发思考/引导讨论 |

### 3.5 需求强度升级策略

> 如何将低需求升级为高需求

| 起点 | 目标 | 升级策略 |
|------|------|----------|
| L3 → L2 | 唤醒需求 | 制造认知冲突、场景触发 |
| L2 → L1 | 强化需求 | 制造紧迫感、限时优惠 |
| L3 → L1 | 重构认知 | 教育用户、重建需求 |

## 4. 场景分类与跨场景引导

### 4.1 四大场景类型

| 场景类型 | 用户状态 | 核心诉求 | 内容策略 |
|----------|----------|----------|----------|
| **主动搜索** | 有明确目的，主动寻找 | 快速找到答案 | 直接提供解决方案+核心卖点 |
| **社交浏览** | 无明确目的，碎片时间 | 消磨时间/发现有趣内容 | 制造吸引力+快速传递价值 |
| **内容种草** | 被KOL/内容触发兴趣 | 了解更多/被说服 | 情感共鸣+真实体验分享 |
| **决策辅助** | 犹豫期，比较选择中 | 消除疑虑/坚定信心 | 消除顾虑+社会证明+紧迫感 |

### 4.2 场景特征详解

#### 主动搜索场景

| 特征 | 描述 |
|------|------|
| 用户意图 | 有明确的搜索目的 |
| 触发动机 | 用户主动发起搜索行为 |
| 决策阶段 | 需求唤醒后期/方案比较期 |
| 心理状态 | 有耐心，愿意花时间了解 |
| 内容期望 | 想要快速找到答案和解决方案 |

**内容策略**：
- 标题直接点明解决方案
- 内容结构清晰，便于快速扫描
- 核心卖点前置
- 提供完整的产品信息

#### 社交浏览场景

| 特征 | 描述 |
|------|------|
| 用户意图 | 无明确目的，消磨时间 |
| 触发动机 | 算法推荐/朋友分享 |
| 决策阶段 | 需求唤醒前期 |
| 心理状态 | 注意力分散，耐心有限 |
| 内容期望 | 想要看到有趣/有用/有共鸣的内容 |

**内容策略**：
- 标题足够吸睛，引发点击
- 开头3秒必须抓住注意力
- 内容轻松易读
- 提供情绪价值或社交货币

#### 内容种草场景

| 特征 | 描述 |
|------|------|
| 用户意图 | 被内容触发兴趣 |
| 触发动机 | KOL推荐/朋友分享/算法推荐 |
| 决策阶段 | 需求唤醒中期 |
| 心理状态 | 好奇、信任、想了解更多 |
| 内容期望 | 想要看到真实的体验分享 |

**内容策略**：
- 使用真实用户视角
- 包含详细的体验过程
- 提供情感共鸣点
- 自然植入产品信息

#### 决策辅助场景

| 特征 | 描述 |
|------|------|
| 用户意图 | 在多个方案中做选择 |
| 触发动机 | 主动搜索/被动看到对比 |
| 决策阶段 | 购买决策前期 |
| 心理状态 | 犹豫、比较、寻找确定性 |
| 内容期望 | 想要找到选择理由 |

**内容策略**：
- 提供对比优势
- 展示社会证明（用户评价）
- 消除购买顾虑
- 制造适当的紧迫感

### 4.3 场景判断标准

| 判断维度 | 问题 | 场景判断 |
|----------|------|----------|
| 用户意图 | 用户是在找什么？ | 有目的→主动搜索；无目的→社交浏览 |
| 触发动机 | 用户为什么看到这个内容？ | 主动找→搜索场景；被动看到→浏览/种草 |
| 决策阶段 | 用户在购买漏斗哪个阶段？ | 需求唤醒→种草；方案比较→决策辅助 |
| 心理状态 | 用户当前是什么心态？ | 急迫→搜索；放松→浏览；犹豫→决策辅助 |

### 4.4 跨场景引导方法

| 起点场景 | 目标场景 | 引导策略 | 操作要点 |
|----------|----------|----------|----------|
| 社交浏览 | 电商购买 | 制造好奇心+跳转入口 | 悬念式结尾+优惠信息 |
| 内容种草 | 公众号深度 | 提供更多价值+关注引导 | 深度内容预告+关注利益点 |
| 短视频 | 图文详情 | 补充细节+完整信息 | 视频引导图文，图文引导购买 |
| 朋友圈 | 小程序 | 即时体验+便捷购买 | 卡片式内容+小程序入口 |
| 知乎/小红书 | 私域社群 | 提供独家内容+社群价值 | 引导加群/关注公众号 |
| 信息流广告 | 落地页 | 强化卖点+降低门槛 | 简化信息+突出行动 |

### 4.5 场景适配检查清单

```
□ 内容是否匹配当前场景的用户心态？
□ 内容的阅读时长是否符合场景？
□ 内容是否设置了合理的跨场景引导？
□ 不同渠道的内容是否做了差异化调整？
□ 是否在适当时机植入了行动召唤？
□ 场景切换的逻辑是否顺畅自然？
```

## 5. 阅读时间四档判断

### 5.1 时间档位划分

| 档位 | 时间范围 | 适用渠道 | 典型内容 |
|------|----------|----------|----------|
| **极短档** | <10秒 | 开屏广告/信息流首位/搜索结果首位 | 单点突破，1个核心卖点 |
| **短档** | 10-30秒 | 小红书图文/微博/信息流 | 场景代入，2-3个要点 |
| **中档** | 30秒-2分钟 | 公众号短文/知乎回答/小红书长图文 | 完整故事，5-6个要点 |
| **长档** | >2分钟 | 深度长文/品牌故事/测评报告 | 完整AIDA，多层次论证 |

### 5.2 各渠道基础阅读时间

| 渠道 | 基础阅读时长 | 内容形式 | 档位判断 |
|------|--------------|----------|----------|
| **抖音/快手** | 15-60秒 | 短视频 | 短档-中档 |
| **小红书图文** | 10-30秒 | 图文 | 短档 |
| **小红书长图文** | 1-5分钟 | 图文 | 中档-长档 |
| **微博** | 10-20秒 | 短图文 | 短档 |
| **公众号** | 30秒-5分钟 | 图文 | 中档-长档 |
| **知乎** | 1-10分钟 | 图文 | 中档-长档 |
| **B站** | 3-30分钟 | 视频 | 中档-长档 |
| **信息流广告** | 5-30秒 | 图文/短视频 | 极短档-短档 |

### 5.3 时间档位判断流程

```
Step 1: 确定渠道基础阅读时长
  │
  ├─ 抖音/快手视频 → 15-60秒（短档-中档）
  ├─ 小红书图文 → 10-30秒（短档）
  ├─ 小红书长图文 → 1-5分钟（中档-长档）
  ├─ 公众号 → 30秒-5分钟（中档-长档）
  ├─ 知乎/小红书长文 → 2-10分钟（长档）
  └─ B站视频 → 3-30分钟（长档）

Step 2: 根据内容复杂度调整
  │
  ├─ 新产品/新概念 → +30秒理解时间
  ├─ 成熟品类 → 可缩短50%
  ├─ 需要教育用户 → +1-2分钟
  └─ 高信任品牌 → 可缩短30%

Step 3: 根据用户需求强度调整
  │
  ├─ L1 强需求 → 用户愿意花更多时间
  └─ L3 弱需求 → 用户耐心有限

Step 4: 确定最终档位
  │
  └── 根据 Step 1 + Step 2 + Step 3 得出最终阅读时间档位
```

### 5.4 各档位内容结构模板

#### 极短档（<10秒）模板

```
[强力标题/画面]
    ↓
[1句核心价值主张]
    ↓
[CTA]
```

**示例**：
```
"用了3个月，皮肤真的变好了！"
↓
"现在下单，立减200"
```

#### 短档（10-30秒）模板

```
[吸睛开头 - 制造好奇心/共鸣]
    ↓
[场景/痛点 - 快速代入]
    ↓
[解决方案 - 1-2个核心卖点]
    ↓
[CTA - 明确的行动指引]
```

**示例**：
```
"姐妹们，我终于找到了不闷痘的防晒！"
↓
"之前用的防晒都让我长痘，夏天真的太痛苦了"
↓
"这个防晒质地超轻薄，完全不闷，而且SPF50+够用了"
↓
"点击链接看看，还有小样可以先试"
```

#### 中档（30秒-2分钟）模板

```
[标题 - 明确主题/价值承诺]
    ↓
[钩子 - 引发兴趣/好奇心]
    ↓
[痛点共鸣 - 用户痛点描述]
    ↓
[解决方案 - 产品/服务介绍]
    ↓
[证据支撑 - 数据/案例/证言]
    ↓
[差异化说明 - 与竞品对比]
    ↓
[CTA - 明确的行动召唤]
```

#### 长档（>2分钟）模板

```
[完整AIDA结构]
    ↓
[Attention - 吸引注意]
  - 震撼数据/惊人事实
  - 引发共鸣的场景
  - 有趣的故事开头
    ↓
[Interest - 激发兴趣]
  - 与用户相关的问题
  - 有价值的信息
  - 悬念设置
    ↓
[Desire - 唤起欲望]
  - 产品/服务展示
  - 解决方案呈现
  - 情感共鸣
  - 社会证明
    ↓
[Action - 促使行动]
  - 明确的CTA
  - 紧迫感（可选）
  - 便利性（可选）
```

### 5.5 时间档位与内容精简技巧

| 档位 | 精简技巧 |
|------|----------|
| 长档→中档 | 删除次要论点、缩短案例、简化论证 |
| 中档→短档 | 删除过渡段落、保留核心卖点、简化开头 |
| 短档→极短档 | 保留1个核心卖点、1句话总结 |

## 8. 内容结构模型库

> 详见: `workflow/05-channel-strategy/内容结构模型库.md`

### 8.1 10种内容结构模型

| 模型 | 适用目标 | 核心特征 |
|------|----------|----------|
| AIDA | 促使立即购买 | 注意→兴趣→欲望→行动 |
| 4P | 建立品牌信任 | 问题→过程→证明→承诺 |
| 情景-问题-方案 | 引发情感共鸣 | 场景→问题→解决方案 |
| 结构化Q&A | 解答用户疑问 | 问题→回答→收尾CTA |
| 故事-启发结论 | 叙事启发 | 故事→启示→结论 |
| 悬念-道出真相 | 好奇心驱动 | 悬念→展开→真相 |
| 拼图式留白 | 碎片化阅读 | 多点→留白 |
| 归纳型 | 系统性论证 | 结论先行→论点支撑 |
| 演绎型 | 线性叙事 | 背景→冲突→高潮→结局 |
| 对比型 | 优劣凸显 | 使用前→使用后→差异→原因 |

### 8.2 内容结构选择决策树

```
Step 1: 核心目标是什么？
  └─ 立即转化 → 建立信任 → 引发思考

Step 2: 用户决策阶段？
  └─ 不知道需求 → 在比较 → 在犹豫 → 想行动

Step 3: 产品复杂度？
  └─ 需要解释 → 容易理解

Step 4: 内容长度？
  └─ 短(<500字) → 中等(500-1500字) → 长(>1500字)
```

## 10. 文案优化指南

> 详见: `workflow/07-final/文案优化指南.md`

### 10.1 三轮删减法

| 轮次 | 目标 | 检查项 |
|------|------|--------|
| 第一轮 | 结构性删减 | 删除无关/重复/冗余内容 |
| 第二轮 | 表达性删减 | 删除修饰词/重复表达/空洞形容词 |
| 第三轮 | 感知性优化 | 拆分长句/改写术语/具体化抽象 |

### 10.2 改写标准

| 标准 | 要求 |
|------|------|
| 长句改短 | >35字必须拆分，目标15-20字 |
| 术语改类比 | 通过"向父母解释"测试 |
| 抽象改具体 | 数据化 > 场景化 > 动作化 |

### 10.3 自检清单（11项/4层）

| 层级 | 检查项 |
|------|--------|
| A策略层 | 用户匹配/卖点突出/目标一致 |
| B内容层 | 概念清晰/逻辑严密/证据充分 |
| C表达层 | 开头有力/内容精简/画面感 |
| D人性化层 | 人情味/行动引导/整体节奏 |

### 10.4 问题归因与阶段返回

| 问题类型 | 返回阶段 |
|----------|----------|
| 卖点问题 | Stage 2 |
| 痛点问题 | Stage 3 |
| 策略问题 | Stage 4 |
| 结构问题 | Stage 5 |
| 表达问题 | Stage 7 |
