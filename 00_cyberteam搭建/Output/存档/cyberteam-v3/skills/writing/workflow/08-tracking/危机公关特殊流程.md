## 1. 与普通流程的核心区别

### 1.1 可跳过的环节

| 普通环节 | 危机场景处理 | 原因 |
|----------|--------------|------|
| Stage 1 用户调研 | 可跳过 | 危机情况下用户调研无意义 |
| 多轮打磨 | 可跳过 | 时效性优先 |
| A/B测试 | 禁止 | 无时间进行测试 |
| 多级审核 | 简化 | 速度优先 |

### 1.2 需强化的环节

| 普通环节 | 危机场景强化要求 |
|----------|------------------|
| 合规审核 | 法务必须参与 |
| 品牌确认 | 高层必须参与 |
| 公关审核 | 公关必须参与 |
| 舆情监控 | 24小时不间断 |

### 1.3 角色介入要求

| 角色 | 普通流程 | 危机流程 |
|------|----------|----------|
| 操盘手 | 主导撰写 | 提供支持 |
| 内容总监 | 审核把关 | 策略制定 |
| 法务 | 可选参与 | **必须参与** |
| 公关 | 可选参与 | **必须参与** |
| 高层 | 重大决策参与 | **全程参与** |

### 1.4 流程对比图

```
【普通流程】
Stage 0 → Stage 1-7 → 审核 → 发布

【危机流程】
┌─────────────────────────────────────────────────────┐
│ 策略制定（1h内）→ 极简版策略要点                      │
│      ↓                                              │
│ 快速撰写（30min内）→ 操盘手主导                      │
│      ↓                                              │
│ 联合终审（30min内）→ 法务+公关+高层                  │
│      ↓                                              │
│ 发布执行 → 同时启动舆情监控                          │
└─────────────────────────────────────────────────────┘
```

## 2. 内容结构固定模板

### 2.1 危机公关文案三段论

| 段落 | 内容 | 要求 |
|------|------|------|
| **第一段：事实陈述** | 发生了什么（客观描述） | 准确、简洁、不隐瞒 |
| **第二段：态度表达** | 我们怎么看/怎么想 | 诚恳、负责、不推卸 |
| **第三段：改进承诺** | 我们会怎么做 | 具体、可落地、有时间 |

### 2.2 三段论模板

```
【第一段：事实陈述】
关于[事件名称]的情况说明：
[时间]，[地点/渠道]，[发生了什么事]。

【第二段：态度表达】
我们[品牌方]对[此事/此问题]高度重视/深表遗憾。
[对此事的根本态度，如：这是我们不愿看到的/我们对此深感抱歉]

【第三段：改进承诺】
针对[具体问题]，我们将采取以下措施：
1. [具体措施1，预计完成时间]
2. [具体措施2，预计完成时间]
3. [具体措施3，预计完成时间]
如有进一步问题，请联系：[联系方式]
```

### 2.3 不同场景的内容适配

| 危机类型 | 第一段调整 | 第二段调整 | 第三段调整 |
|----------|------------|------------|------------|
| 产品问题 | 描述具体问题 | 表达歉意+责任感 | 提供解决方案+补偿 |
| 服务问题 | 描述服务缺陷 | 承认不足 | 改进服务承诺 |
| 舆情误解 | 澄清事实真相 | 表达理解 | 主动沟通 |
| 合作问题 | 说明合作状态 | 表达遗憾 | 后续安排 |

### 2.4 内容禁忌清单

| 禁忌类型 | 错误示例 | 正确表达 |
|----------|----------|----------|
| 推卸责任 | "是用户使用不当" | "我们会进一步优化使用说明" |
| 过度解释 | 冗长的技术解释 | 简洁的事实说明 |
| 转移话题 | "其他品牌也这样" | 聚焦自身问题 |
| 空洞承诺 | "我们会努力改进" | "具体措施+时间节点" |
| 情绪化表达 | "我们很委屈" | 客观理性的态度 |

## 3. 时间要求

### 3.1 各环节时间要求

| 环节 | 普通要求 | 危机要求 | 责任人 |
|------|----------|----------|--------|
| 策略制定 | 不限 | **1小时内输出极简版策略要点** | 内容总监 |
| 初稿撰写 | 1-2天 | **30分钟内完成** | 操盘手 |
| 审核校对 | 2-4小时 | **30分钟内完成联合终审** | 法务+公关+高层 |
| 发布执行 | 不限 | **审核通过后立即发布** | 运营 |
| 舆情监控 | 可选 | **24小时高强度监控** | 公关 |

### 3.2 黄金响应时间

| 响应时间 | 效果评估 |
|----------|----------|
| **1小时内** | 最佳，主动可控 |
| **1-4小时** | 良好，反应迅速 |
| **4-12小时** | 一般，略显被动 |
| **12-24小时** | 较差，舆论失控风险 |
| **24小时以上** | 很差，危机扩大 |

### 3.3 舆情监控时间表

| 时段 | 监控频率 | 汇报要求 |
|------|----------|----------|
| 发布后0-2小时 | 每15分钟汇报 | 即时发现问题 |
| 发布后2-6小时 | 每30分钟汇报 | 持续跟踪 |
| 发布后6-24小时 | 每1小时汇报 | 趋于稳定 |
| 发布后24-72小时 | 每4小时汇报 | 持续关注 |

### 3.4 升级机制

| 预警等级 | 判断标准 | 升级动作 |
|----------|----------|----------|
| **红色预警** | 热搜/大量媒体转载/用户激增 | 立即召开紧急会议 |
| **橙色预警** | 负面评论>100/扩散趋势明显 | 内容总监+公关立即介入 |
| **黄色预警** | 负面评论增加/有扩散趋势 | 加强监控，准备预案 |

## 4. 核心原则

### 4.1 准确性 > 创造性

**普通文案**: 可以适度艺术化表达
**危机文案**: 必须100%准确，零错误

| 要求 | 说明 |
|------|------|
| 事实准确 | 每一个事实陈述必须有据可查 |
| 数据准确 | 所有数字必须与实际一致 |
| 措辞准确 | 避免歧义，避免过度承诺 |
| 来源明确 | 引用必须有明确来源 |

### 4.2 一致性 > 创意性

**普通文案**: 鼓励差异化创意
**危机文案**: 必须统一口径，统一形象

| 要求 | 说明 |
|------|------|
| 口径统一 | 所有对外内容保持一致 |
| 立场统一 | 所有渠道表达相同态度 |
| 行动统一 | 各部门按统一策略执行 |
| 形象统一 | 保持品牌负责任的形象 |

### 4.3 速度优先原则

| 原则 | 说明 | 执行要求 |
|------|------|----------|
| **先发制人** | 主动说明比被动回应好 | 第一时间发声 |
| **简洁明了** | 废话少说，直击要点 | 控制文案长度 |
| **持续更新** | 保持信息更新 | 定期发布进展 |

### 4.4 诚恳负责原则

| 原则 | 说明 | 表达方式 |
|------|------|----------|
| **承认问题** | 不否认，不推卸 | "这是我们的责任" |
| **表达歉意** | 真诚道歉，不走过场 | 具体道歉，非套路 |
| **积极解决** | 行动比言语重要 | 公布具体措施 |
| **持续沟通** | 保持信息公开 | 定期更新进展 |

## 5. 危机类型与应对策略

### 5.1 危机类型分类

| 危机类型 | 特征 | 典型案例 | 应对优先级 |
|----------|------|----------|------------|
| **产品质量** | 产品存在缺陷/安全隐患 | 召回/质量门 | P0 |
| **虚假宣传** | 宣传内容与实际不符 | 功效夸大 | P0 |
| **服务问题** | 服务态度/质量问题 | 客服冲突 | P1 |
| **舆情误解** | 舆论误读/断章取义 | 媒体曲解 | P1 |
| **合作纠纷** | 与合作方产生矛盾 | 供应商/主播纠纷 | P2 |

### 5.2 各类型应对策略

#### 产品质量类（P0）

```
核心策略：主动承认 + 快速解决 + 全面召回

文案要点：
├─ 承认问题（不推卸）
├─ 说明影响范围
├─ 公布解决方案
├─ 承诺补偿措施
└─ 说明预防措施

审核重点：
└─ 法务必须确认法律风险
```

#### 虚假宣传类（P0）

```
核心策略：澄清事实 + 纠正错误 + 承担责任

文案要点：
├─ 说明实际情况
├─ 解释宣传初衷
├─ 承认表述不当
├─ 承诺更正内容
└─ 承担相应责任

审核重点：
├─ 法务确认合规性
└─ 公关评估舆论影响
```

#### 服务问题类（P1）

```
核心策略：诚恳道歉 + 调查原因 + 改进服务

文案要点：
├─ 对服务问题表示歉意
├─ 说明调查进展
├─ 公布改进措施
└─ 承诺提升服务

审核重点：
└─ 公关确认舆论走向
```

#### 舆情误解类（P1）

```
核心策略：澄清事实 + 还原真相 + 理性沟通

文案要点：
├─ 说明实际情况
├─ 提供证据支撑
├─ 表达理解公众关切
└─ 呼吁理性看待

审核重点：
├─ 确认事实准确性
└─ 公关评估传播风险
```

### 5.3 危机应对检查清单

```
【危机响应启动】
□ 确认危机类型和等级
□ 启动危机响应机制
□ 通知相关人员
□ 组建危机小组

【策略制定】
□ 收集完整信息
□ 评估影响范围
□ 制定应对策略
□ 准备文案模板

【文案撰写】
□ 事实陈述准确
□ 态度表达诚恳
□ 承诺具体可落地
□ 无法律风险

【审核发布】
□ 法务审核通过
□ 公关审核通过
□ 高层审批通过
□ 统一口径发布

【舆情监控】
□ 启动24小时监控
□ 设置预警机制
□ 定期汇报进展
□ 准备后续预案

【收尾工作】
□ 危机解除确认
□ 后续跟进计划
□ 复盘总结
□ 知识沉淀
```
