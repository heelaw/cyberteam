# Writing Skill Workflow

> **版本**: v3.1
> **更新日期**: 2026-03-24

---

## 目录结构

```
workflow/
├── 00-init/              # 初始化配置
├── 01-user-research/     # 用户需求挖掘
├── 02-selling-points/    # 产品卖点挖掘
├── 03-pain-points/       # 痛点定位
├── 04-strategy/         # 需求类型判断
├── 05-channel-strategy/  # 场景与渠道分析
├── 06-draft/            # 文案撰写
├── 07-final/            # 文案优化
└── 08-tracking/          # 效果追踪
```

---

## 案例格式

每个案例应包含以下文件:

```
{product}-{YYYYMMDD}-{stage}/
├── input.md              # 输入描述
├── analysis.md           # 分析过程
├── output.md             # 输出结果
└── review.md             # 审核意见
```

---

## 贡献指南

### 添加新案例

1. 创建案例目录:
   ```bash
   mkdir -p workflow/{stage}/{product}-{YYYYMMDD}
   ```

2. 创建案例文件:
   - input.md: 输入描述和需求
   - analysis.md: 分析过程
   - output.md: 输出结果
   - review.md: 审核意见

3. 添加到案例索引
