```
│  │                  │                  │                  │    │
│  │  业务逻辑        │  技术实现        │  通用功能        │    │
│  └──────────────────┴──────────────────┴──────────────────┘    │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 3.3 路由决策规则

```yaml
L1 CEO 路由决策:

规则1: 任务类型判断
  if 任务包含"业务分析/增长/内容/数据":
    → 路由到 CyberTeam 部门
  elif 任务包含"代码/测试/部署":
    → 路由到 Gstack skills
  elif 任务包含"通用功能开发":
    → 路由到 gsd-* agents
  else:
    → 询问用户

规则2: 复杂度判断
  if 任务复杂度 == "高" AND 涉及多领域:
    → 触发专家协作 (TeamCreate)
  elif 任务复杂度 == "中":
    → 串行执行
  else:
    → 直接回复

规则3: 资源可用性判断
  if 目标部门/Agent 不可用:
    → 降级到备用方案
    → 或询问用户
```

### 3.4 层间调用协议

```yaml
L1 → L2 调用协议:
  消息格式:
    from: CEO
    to: PM/Strategy
    action: coordinate_task
    params:
      task: "任务描述"
      complexity: "高/中/低"
      experts: ["专家1", "专家2"]
    context:
      user_request: "原始用户请求"
      intent: "识别的意图"

L2 → L3 调用协议:
  消息格式:
    from: PM/Strategy
    to: 部门/Gstack/Agent
    action: execute_subtask
    params:
      subtask: "子任务描述"
      skills: ["/skill1", "/skill2"]
      dependencies: ["依赖1", "依赖2"]
    context:
      parent_task: "父任务ID"
      expert_opinions: ["专家意见1", "专家意见2"]

L3 → L2 反馈协议:
  消息格式:
    from: 部门/Gstack/Agent
    to: PM
    action: report_result
    result:
      status: "success/failure"
      output: "输出内容"
      metrics:
        duration: 120
        token_usage: 5000
    context:
      subtask: "子任务ID"
      errors: ["错误1"] (如果有)
```
