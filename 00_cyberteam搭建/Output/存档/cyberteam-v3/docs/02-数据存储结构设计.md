## 一、存储架构概述

### 1.1 存储位置

```yaml
根目录: ~/.cyberteam/

目录结构:
  ~/.cyberteam/
    ├── tasks/          # 任务数据
    ├── decisions/      # 决策日志
    ├── messages/       # 通信历史
    ├── metrics/        # 性能指标
    ├── logs/           # 日志文件
    ├── cache/          # 缓存数据
    └── config/         # 配置文件
```

### 1.2 存储原则

| 原则 | 说明 |
|------|------|
| **人类可读** | 优先使用 JSON 格式 |
| **易于调试** | 文件命名清晰，内容可理解 |
| **高性能** | 热数据缓存，冷数据归档 |
| **可扩展** | 支持分布式存储扩展 |

## 六、日志存储

### 6.1 目录结构

```yaml
~/.cyberteam/logs/
  application.log    # 应用日志
  messages.log       # 消息日志
  errors.log         # 错误日志
  performance.log    # 性能日志
  archive/           # 归档日志 (按日期)
```

### 6.2 日志格式

```yaml
格式: JSON Lines

示例:
  {"timestamp":"2026-03-24T15:10:00Z","level":"INFO","service":"PM","message":"任务创建","task_id":"TASK-001"}
  {"timestamp":"2026-03-24T15:10:05Z","level":"DEBUG","service":"PM","message":"消息发送","message_id":"MSG-001"}
  {"timestamp":"2026-03-24T15:10:10Z","level":"ERROR","service":"data-analytics","message":"处理失败","error":"..."}
```

### 6.3 日志级别

| 级别 | 用途 | 示例 |
|------|------|------|
| **DEBUG** | 调试信息 | 消息发送详情 |
| **INFO** | 一般信息 | 任务创建 |
| **WARN** | 警告信息 | 超时警告 |
| **ERROR** | 错误信息 | 处理失败 |
| **FATAL** | 致命错误 | 系统崩溃 |

### 6.4 日志轮转

```yaml
轮转策略:
  大小: 10 MB
  日期: 每天
  保留: 30 天

轮转方式:
  - 达到大小限制 → 创建新文件
  - 达到日期限制 → 创建新文件
  - 超过保留期限 → 删除或归档
```

## 七、缓存存储

### 7.1 目录结构

```yaml
~/.cyberteam/cache/
  skills/            # Skill 缓存
  responses/         # 响应缓存
  temp/              # 临时文件
```

### 7.2 缓存策略

```yaml
Skill 缓存:
  内容: Skill 执行结果
  时长: 1 小时
  触发: 相同输入复用输出

响应缓存:
  内容: API 响应
  时长: 30 分钟
  触发: 相同请求复用响应

临时文件:
  内容: 中间处理结果
  时长: 1 天
  触发: 定期清理
```

### 7.3 缓存键设计

```yaml
格式: {type}:{hash(params)}

示例:
  skill:leverage-analysis:abc123
  response:mcp__MiniMax__web_search:def456
  temp:task-processing:ghi789
```

## 十二、数据迁移

### 12.1 迁移策略

```yaml
v2.1 → v3 迁移:
  1. 分析 v2.1 数据结构
  2. 设计迁移脚本
  3. 测试迁移
  4. 执行迁移
  5. 验证结果
```

### 12.2 迁移脚本

```python
#!/usr/bin/env python3
"""
CyberTeam v2.1 → v3 数据迁移脚本
"""

import json
import shutil
from pathlib import Path

def migrate_tasks():
    """迁移任务数据"""
    v2_tasks = Path("~/.cyberteam-v2/tasks/").expanduser()
    v3_tasks = Path("~/.cyberteam/tasks/").expanduser()

    for task_file in v2_tasks.glob("*.json"):
        with open(task_file) as f:
            v2_task = json.load(f)

        # 转换为 v3 格式
        v3_task = convert_task(v2_task)

        # 保存到 v3 位置
        output_file = v3_tasks / v3_task["status"] / task_file.name
        output_file.write_text(json.dumps(v3_task, indent=2))

def convert_task(v2_task):
    """转换任务格式"""
    return {
        "task_id": v2_task["id"],
        "title": v2_task["title"],
        "status": v2_task["status"],
        # ... 其他字段转换
    }

if __name__ == "__main__":
    migrate_tasks()
    print("迁移完成")
```
