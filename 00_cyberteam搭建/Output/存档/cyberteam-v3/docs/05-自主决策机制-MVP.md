```yaml
自主决策目标:
  - 减少 80% 的用户询问
  - 提升 Agent 自主性
  - 保持决策质量
```

## 二、自主决策范围

### 2.1 可自主决策的事项

```yaml
L1 (CEO) 可自主决策:
  简单咨询:
    - 问候、感谢、状态查询
    → 直接回复，不记录

  路由决策:
    - 明确的任务类型识别
    → 按规则路由，不记录

L2 (PM/Strategy) 可自主决策:
  任务拆解:
    - 标准任务拆解 (WBS)
    - 专家协作模式选择
    - → 记录决策日志

  资源分配:
    - 标准任务时长估算
    - 标准 Skill 调用
    - → 记录决策日志

L3 (部门) 可自主决策:
  执行细节:
    - 具体实现方式
    - 工具选择
    - → 记录决策日志

  质量标准:
    - 基础质量检查
    - 格式规范检查
    - → 记录决策日志
```

### 2.2 需要升级决策的事项

```yaml
必须升级决策:
  - 影响范围大的决策
  - 成本高的决策
  - 风险高的决策
  - 有争议的决策

升级路径:
  L3 → L2 → L1 → 用户
```

## 三、决策边界

### 3.1 边界定义

```yaml
L1 边界:
  可以:
    - 需求分拣
    - 意图识别
    - 路由决策

  不可以:
    - 自行修改用户需求
    - 自行改变任务目标

L2 边界:
  可以:
    - 任务拆解
    - 模式选择
    - 资源分配

  不可以:
    - 自行改变任务优先级
    - 自行增加任务成本

L3 边界:
  可以:
    - 执行细节决策
    - 工具选择
    - 实现方式

  不可以:
    - 自行改变交付标准
    - 自行降低质量要求
```

### 3.2 边界检查机制

```python
def check_decision_boundary(agent, decision):
    """检查决策是否在边界内"""

    # 获取 Agent 配置的边界
    boundaries = get_agent_boundaries(agent)

    # 检查决策类型
    if decision["type"] not in boundaries["allowed_types"]:
        return False, "决策类型不在允许范围"

    # 检查决策影响
    if decision["impact"] > boundaries["max_impact"]:
        return False, "决策影响超出边界"

    # 检查决策成本
    if decision["cost"] > boundaries["max_cost"]:
        return False, "决策成本超出边界"

    # 检查决策风险
    if decision["risk"] > boundaries["max_risk"]:
        return False, "决策风险超出边界"

    return True, "在边界内"
```

## 四、决策规则库

### 4.1 决策规则示例

```yaml
L1 CEO 决策规则:
  规则1: 需求分拣
    if 用户输入 in ["你好", "谢谢", "在吗"]:
      → 简单咨询，直接回复
    else:
      → 正式任务，继续处理

  规则2: 意图识别
    if 关键词 in ["增长", "数据", "分析"]:
      → 意图 = "数据分析"
    elif 关键词 in ["内容", "文案", "发布"]:
      → 意图 = "内容运营"
    else:
      → 意图 = "未知"，询问用户

  规则3: 路由决策
    if 意图 == "数据分析" AND 复杂度 == "高":
      → 路由到 L2 PM + Strategy
    elif 意图 == "内容运营" AND 复杂度 == "高":
      → 路由到 L2 PM + Strategy
    else:
      → 直接路由到 L3

L2 PM 决策规则:
  规则1: 协作模式选择
    if 任务复杂度 == "高":
      → 协作模式 = "并行创意风暴"
    elif 任务类型 == "常规内容":
      → 协作模式 = "串行评审"
    else:
      → 协作模式 = "无协作"

  规则2: 专家选择
    if 任务类型 == "内容创作":
      → 专家 = ["用户画像", "文案撰写", "内容审核"]
    elif 任务类型 == "数据分析":
      → 专家 = ["增长黑客", "数据分析"]
    else:
      → 专家 = []

L3 部门决策规则:
  规则1: 工具选择
    if 需要图像生成:
      → 使用 baoyu-image-gen
    if 需要发布:
      → 根据平台选择 baoyu-post-to-*

  规则2: 质量标准
    if 平台 == "小红书":
      → 质量标准 = "小红书标准"
    if 平台 == "公众号":
      → 质量标准 = "公众号标准"
```

### 4.2 决策冲突处理

```yaml
冲突类型1: 多个规则匹配
  解决: 优先级高的规则优先
  示例: 用户输入同时匹配"增长"和"内容"
    → 优先级: 内容 > 增长

冲突类型2: 规则之间矛盾
  解决: 使用默认规则
  示例: 协作模式选择冲突
    → 默认: 无协作

冲突类型3: 边界不清晰
  解决: 升级决策
  示例: 决策影响边界不清晰
    → 升级到上一级 Agent
```
