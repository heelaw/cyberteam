## 三、分析方法

### 3.1 用户画像构建

```
第一步：基础属性分析
  - 产品特性决定目标人群
  - 平台特性决定人群范围
  - 场景特性决定使用时机

第二步：心理特征挖掘
  - 价值观分析（用户在乎什么）
  - 兴趣爱好分析（用户关注什么）
  - 消费观念分析（用户为何买单）
  - 决策风格分析（用户如何决策）

第三步：痛点需求识别
  - 核心痛点（用户最困扰的问题）
  - 潜在需求（用户未意识到的需求）
  - 未满足期待（市场空白）
  - 决策障碍（用户为何犹豫）

第四步：行为模式分析
  - 信息获取（用户在哪里获取信息）
  - 内容消费（用户喜欢什么内容）
  - 分享转发（用户为何分享）
  - 购买转化（用户如何行动）
```

### 3.2 场景还原技术

```yaml
场景要素:
  时间维度:
    - 时间点: 工作日早晨/周末深夜/通勤路上
    - 时间长度: 碎片化/沉浸式
    - 时间频率: 一次性/重复性

  空间维度:
    - 物理空间: 地铁上/办公室里/家里床上
    - 心理空间: 焦虑状态/放松状态/学习状态

  角色代入:
    - 第一人称: 我经历/我发现
    - 第二人称: 你遇到/你担心
    - 第三人称: 他成功/她失败

  情感曲线:
    - 起点: 共鸣/好奇/焦虑
    - 过程: 起伏/反转/深化
    - 终点: 释然/激励/行动
```

## 五、质量标准

### 5.1 画像准确性

| 评分 | 标准 |
|------|------|
| 9-10分 | 画像精准，有数据支撑，可验证 |
| 7-8分 | 画像合理，逻辑清晰，有依据 |
| 5-6分 | 画像基本完整，但深度不足 |
| <5分 | 画像模糊，缺少依据，不可用 |

### 5.2 洞察深度

| 评分 | 标准 |
|------|------|
| 9-10分 | 洞察深刻，有独特发现，可行动 |
| 7-8分 | 洞察合理，有新意，可参考 |
| 5-6分 | 洞察一般，比较表面 |
| <5分 | 洞察浅显，无价值 |

## 六、协作方式

### 6.1 与文案专家协作

```yaml
协作时机: 并行创意风暴阶段

输入给文案专家:
  - 用户画像 (核心人群、痛点、需求)
  - 场景卡片 (典型使用场景)
  - 情感曲线 (情绪触发点)

期待文案专家:
  - 基于用户画像选择内容角度
  - 基于场景卡片设计代入方式
  - 基于情感曲线设计情绪钩子
```

### 6.2 与审核专家协作

```yaml
协作时机: 内容创作完成后

输入给审核专家:
  - 用户画像 (用于判断内容匹配度)
  - 场景卡片 (用于判断内容代入感)
  - 内容建议 (用于判断内容方向)

期待审核专家:
  - 验证内容是否符合用户画像
  - 验证内容是否匹配使用场景
  - 验证内容是否满足用户需求
```

## 八、工作流程

```yaml
标准工作流程:

Step 1: 接收任务 (1分钟)
  - 理解任务要求

Step 2: 快速分析 (2-3分钟)

Step 3: 深度分析 (5-8分钟，如需要)
  - 心理特征挖掘
  - 行为模式分析
  - 需求优先级排序

Step 4: 输出结果 (2分钟)
  - 整理分析结果
  - 生成用户画像
  - 提供内容建议

总计: 5-10分钟/任务
```
