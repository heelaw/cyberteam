# 测试驱动开发 (TDD)

## 身份定位

```
+----------------------------------------------------------+
|  测试驱动开发                                            |
|  用途: 红→绿→重构，确保100%测试覆盖                     |
|  周期: Red → Green → Refactor                              |
+----------------------------------------------------------+
```

## TDD循环

```
+----------------------------------------------------------+
|                                                          |
|   Red (红)                                             |
|   - 先写一个失败的测试                                   |
|   - 只写调用代码和期望的输出                             |
|   - 运行测试，确认失败                                   |
|                                                          |
|   Green (绿)                                           |
|   - 写最简单的代码让测试通过                             |
|   - 不管代码质量，只求通过                               |
|   - 运行测试，确认通过                                   |
|                                                          |
|   Refactor (重构)                                      |
|   - 优化代码质量                                         |
|   - 消除重复                                             |
|   - 保持测试通过                                         |
|   - 运行完整测试套件                                     |
|                                                          |
+----------------------------------------------------------+
```

## TDD原则

1. **先写测试** — 不写代码直到有失败的测试
2. **最小实现** — 只写让测试通过的最少代码
3. **快速迭代** — 测试→实现→重构，小步前进
4. **100%覆盖** — 每个函数都有测试

## Jest集成示例

```javascript
// 1. 先写测试 (Red)
describe('calculateDiscount', () => {
  it('should apply 10% discount for orders over $100', () => {
    expect(calculateDiscount(150)).toBe(135);
  });

  it('should not apply discount for orders under $100', () => {
    expect(calculateDiscount(50)).toBe(50);
  });
});

// 2. 再写实现 (Green)
function calculateDiscount(amount) {
  if (amount > 100) {
    return amount * 0.9;
  }
  return amount;
}

// 3. 重构
// 提取常量、优化命名、提升可读性
```

## 测试覆盖率要求

| 类型 | 覆盖率要求 |
|------|------------|
| 单元测试 | 100% |
| 集成测试 | 80% |
| E2E测试 | 关键路径 |
