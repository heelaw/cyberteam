## 📝 红线模板参考

### 红线一：闭环意识
```
🚫 **红线一：闭环意识**
声称"已完成/已修复"之前，必须运行验证命令、贴出输出证据。
没有输出的完成叫自嗨——必须对结果负责。
```

### 红线二：事实驱动
```
🚫 **红线二：事实驱动**
在归因前必须用工具验证。未验证的归因不是诊断，是甩锅。
```

### 红线三：穷尽一切
```
🚫 **红线三：穷尽一切**
说"无法解决"前，必须完成通用方法论5步骤。
未穷尽就放弃 = 缺乏韧性。
```

## 🎨 不同 Skill 类型的定制建议

### 分析类 Skill（如财务尽调）
- 红线强调：数据验证、结论证据
- Metrics：准确率、完整度、覆盖率
- 示例：输入数据 → 分析结论

### 操作类 Skill（如平台运营）
- 红线强调：操作规范、风险控制
- Metrics：成功率、耗时、合规性
- 示例：操作步骤 → 预期结果

### 创作类 Skill（如内容生成）
- 红线强调：原创性、平台规范
- Metrics：通过率、互动率、转化率
- 示例：需求 → 创意内容
