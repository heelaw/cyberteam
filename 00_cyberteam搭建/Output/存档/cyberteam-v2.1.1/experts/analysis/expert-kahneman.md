## 思维框架

### 系统1 vs 系统2

| 特征 | 系统1 (快思考) | 系统2 (慢思考) |
|------|--------------|--------------|
| 速度 | 快速、自动 | 缓慢、费力 |
| 努力 | 无需意志力 | 需要意志力 |
| 典型 | 识别物体、情绪反应 | 复杂计算、逻辑推理 |
| 常见错误 | 偏见、启发式判断 | 懒惰、确认偏误 |

### 常见认知偏见

```
1. 锚定效应 (Anchoring)
   - 第一个数字会影响后续判断
   - 应对: 考虑多个锚点

2. 可得性启发 (Availability)
   - 最近/印象深刻的事件权重过高
   - 应对: 查看统计数据

3. 代表性启发 (Representativeness)
   - 以偏概全，忽视基础概率
   - 应对: 参考基准率

4. 确认偏误 (Confirmation Bias)
   - 只看到支持自己观点的信息
   - 应对: 主动寻找反对证据

5. 损失厌恶 (Loss Aversion)
   - 损失的影响是同等收益的2倍
   - 应对: 框架效应重构

6. 现状偏误 (Status Quo)
   - 过度偏好当前状态
   - 应对: 把现状作为一个选项评估
```

## 注入模板

```
## 卡尼曼决策框架

**问题识别**：
这是系统1还是系统2决策？

**系统1警示**（需警惕）：
- 第一反应是否太快速？
- 是否受到锚定效应影响？
- 是否忽略了基础概率？

**系统2分析**：
1. 列出所有可能选项
2. 评估每个选项的概率和影响
3. 考虑基准率
4. 主动寻找反对证据
5. 考虑损失厌恶程度

**最终建议**：
- 使用框架效应重构选项
- 设定决策截止时间避免无限分析
- 记录决策以供复盘
```

## 使用示例

### 示例1: 投资决策

```
输入: "要不要投资这个区块链项目？"

卡尼曼分析:
- 系统1反应: "很热！很多人都在投！" (可得性启发)
- 系统1反应: "创始团队很牛" (代表性启发)

系统2分析:
1. 查看项目白皮书的技术可行性
2. 评估市场规模和竞争格局
3. 参考历史同类项目成功率（基础概率）
4. 评估个人风险承受能力
5. 损失厌恶：如果是亏50%能接受吗？
```

### 示例2: 招聘决策

```
输入: "要不要录用这个候选人？"

卡尼曼分析:
- 系统1: "第一印象很好" (可能只是候选人会说话)
- 系统1: "名校毕业" (光环效应)

系统2:
1. 列出岗位核心能力要求
2. 逐项评估候选人匹配度
3. 参考该岗位历史录用成功率
4. 多轮面试减少偏见
```
