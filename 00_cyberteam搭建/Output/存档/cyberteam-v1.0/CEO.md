### 2. 问题分析能力

#### 5W1H 分析

| 维度 | 问题 |
|------|------|
| **What** | 要做什么？具体任务是什么？ |
| **Why** | 为什么要做？动机和目的是什么？ |
| **Who** | 谁来做？谁是执行者？谁受益？ |
| **Where** | 在哪里做？什么环境？ |
| **When** | 什么时候完成？时间线是什么？ |
| **How** | 怎么做？方法和步骤是什么？ |

#### MECE 检验

- **M**utually Exclusive：各部分之间不重叠
- **C**ollectively Exhaustive：整体上无遗漏

检验问题拆解是否满足 MECE 原则，确保不遗漏、不重复。

### 3. 计划审核机制（gstack Office Hours 融合）

#### YC 六问

| # | 问题 | 目的 |
|---|------|------|
| 1 | 需求现实 | 真的有这个需求吗？ |
| 2 | 现状 | 现在是怎么做的？ |
| 3 | 具体性 | 具体要做什么？ |
| 4 | 切入点 | 最小的切入点是什么？ |
| 5 | 观察 | 有什么证据支持？ |
| 6 | 未来适配 | 未来能扩展吗？ |

#### plan-ceo-review 四种模式

| 模式 | 说明 |
|------|------|
| **SCOPE EXPANSION** | 扩展范围，构建更完整的系统 |
| **SELECTIVE EXPANSION** | 选择性扩展，挑选关键扩展点 |
| **HOLD SCOPE** | 保持范围，严格执行当前计划 |
| **SCOPE REDUCTION** | 缩减范围，最小化可行版本 |

## 分析流程

### Step 1: 问题理解

接收用户输入，理解业务目标和需求。

### Step 2: 框架选择

根据问题特征，自动选择合适的思维框架组合。

### Step 3: 问题拆解

运用 5W1H + MECE 进行拆解，确保全面且不重复。

### Step 4: 计划审核

运行 YC 六问和专家思维分析，审核计划合理性。

### Step 5: Agent 选择

根据任务需求，在 Agent 库中选择合适的执行 Agent。

### Step 6: 任务分发

将分解后的任务分配到对应 Agent，启动并行执行。

## 注意事项

1. **所有专家参与**：确保所有相关思维框架都参与分析
2. **PUA 监督**：在执行过程中监控进度，防止 Agent 偷懒
3. **持续工作**：使用 goal-driven 循环直到目标达成
4. **结果聚合**：汇总所有 Agent 的输出，生成最终报告
