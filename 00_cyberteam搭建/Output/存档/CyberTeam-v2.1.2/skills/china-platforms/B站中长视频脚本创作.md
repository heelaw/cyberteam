# B站中长视频脚本创作

## 概述

B站用户以高学历年轻人为主，对内容质量要求高，愿意花时间看完优质内容。中长视频是B站的核心内容形式。本skill帮助创作者创作有深度、高质量的B站视频脚本。

## B站内容特点

### 与短视频的差异

```
短视频（抖音/快手）：
- 时长：15-60秒
- 核心：前3秒决定成败
- 目标：快速抓住注意力
- 深度：浅、碎片化

B站中长视频：
- 时长：8-30分钟
- 核心：价值承诺+内容兑现
- 目标：让用户觉得"值"
- 深度：深、系统、有信息增量
```

### B站用户特征

```
- 高学历、高收入
- 愿意为优质内容花时间
- 重视内容深度和真实性
- 反感硬广和标题党
- 弹幕文化认同感强
- 三连（点赞+投币+收藏）是重要认可
```

## 脚本结构模板

### 知识类视频（10-15分钟）

```markdown
## 开头（0:00-1:00）黄金30秒
- 抛出核心问题或悬念
- 价值预告："看完这期，你会XXX"
- 直奔主题，避免冗长自我介绍

## 第一部分（1:00-4:00）背景铺垫
- 交代必要背景信息
- 用具体案例或数据引入
- 设置弹幕互动点："到这里扣1"

## 第二部分（4:00-10:00）核心内容
- 分2-3个小论点展开
- 每个论点配具体案例/数据/对比
- 每2-3分钟设置节奏变化点

## 第三部分（10:00-13:00）深度延伸
- 给出独到见解或预测
- 呼应开头的悬念/问题
- 设置"高能"片段引发弹幕高潮

## 结尾（13:00-15:00）收束与互动
- 一句话总结核心观点
- 引导互动："你怎么看？评论区聊聊"
- 预告下期内容（系列内容）
- "一键三连"引导要自然
```

### 测评类视频（8-12分钟）

```markdown
## 开头（0:00-0:30）
- 直接展示产品
- 给出初步印象："今天测的这款XXX"

## 基本信息（0:30-2:00）
- 品牌/型号/价格
- 适用人群/场景
- 同类产品对比定位

## 核心测评（2:00-8:00）
- 测评维度1（2分钟）
- 测评维度2（2分钟）
- 测评维度3（2分钟）
- 对比同类产品

## 总结（8:00-10:00）
- 优缺点总结
- 适合人群
- 购买建议
- 评分（可选）
```

### 教程类视频（15-20分钟）

```markdown
## 开头（0:00-0:45）
- 展示最终效果
- 说明教程难度和适合人群
- 预告需要的时间和工具

## 工具准备（0:45-2:00）
- 所需材料清单
- 推荐品牌（可选）
- 准备工作说明

## 分步骤演示（2:00-15:00）
- Step 1：（3-4分钟）
- Step 2：（3-4分钟）
- Step 3：（3-4分钟）
- Step N...

## 常见问题（15:00-17:00）
- 新手常犯的错误
- 如何避免
- 遇到问题怎么办

## 结尾（17:00-18:00）
- 成果展示
- 进阶方向建议
- 三连引导
```

## 脚本撰写技巧

### 价值承诺设计

```markdown
## 开头的价值预告示例

知识类：
"这期视频，我会把XXX的底层逻辑讲清楚，看完之后你也能XXX"

测评类：
"这期我花了XX时间深度体验这款XXX，告诉你它到底值不值得买"

教程类：
"跟着我做，保证你也能学会XXX，这个方法连新手都能轻松掌握"
```

### 信息密度控制

```markdown
## 信息密度原则

1. 每分钟至少1个"钩子"
   - 新观点
   - 新数据
   - 新案例
   - 金句/梗

2. 避免无效画面
   - 不要有等待思考的停顿
   - 不要有重复啰嗦的表达

3. 节奏变化
   - 知识输出 → 案例展示 → 总结
   - 保持节奏变化避免单调
```

### 弹幕互动点设计

```markdown
## 弹幕触发点示例

1. "到这里扣1" - 进度确认
2. "前方高能" - 预告精彩内容
3. "弹幕最多的版本" - 多版本对比
4. "有人发现了吗" - 细节彩蛋
5. "反对/支持" - 引发讨论
```

## 三连引导技巧

```markdown
## 自然的三连引导

❌ 不要：
"觉得有用的话请一键三连支持一下"
（太刻意，用户反感）

✓✓✓ 要：
"这期内容准备了很久，如果对你有帮助的话，
 一枚硬币或者一个收藏都是对我最大的支持"

或者：
"你们的三连是我继续做下去的最大动力"
（真诚表达，弱化功利感）
```
