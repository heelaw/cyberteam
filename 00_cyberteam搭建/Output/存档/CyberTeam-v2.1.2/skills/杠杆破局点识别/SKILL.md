# 杠杆破局点识别

## 身份定位

```
┌─────────────────────────────────────────────────────────────┐
│  ⚡ 杠杆破局点识别                                          │
│  用途: 找到四两拨千斤的突破口                              │
│  配套Agent: 策略执行Agent                                   │
└─────────────────────────────────────────────────────────────┘
```

---

## 核心框架

```
杠杆效应 = 投入 × 杠杆倍数

高杠杆: 投入小 → 效果大
中杠杆: 投入中 → 效果中
低杠杆: 投入大 → 效果小

破局点 = 高杠杆 × 高确定性 × 可执行
```

---

## 执行步骤

### Step 1: 现状扫描

```
收集信息:
  □ 当前状态
    - 业务数据
    - 用户反馈
    - 团队状态

  □ 已尝试的方法
    - 做过什么
    - 结果如何
    - 效果持续性

  □ 资源约束
    - 人力限制
    - 预算限制
    - 时间限制
```

### Step 2: 寻找高杠杆点

```
识别方法:
  □ 瓶颈分析
    - 哪里堵了？
    - 堵的原因是什么？
    - 疏通哪里最有效？

  □ 杠杆评估矩阵
    ┌──────────┬─────────────┬─────────────┐
    │ 杠杆点   │ 杠杆倍数    │ 确定性      │
    ├──────────┼─────────────┼─────────────┤
    │          │ 高/中/低   │ 高/中/低    │
    └──────────┴─────────────┴─────────────┘

  高杠杆点特征:
    - 解决一个问题，同时疏通多个环节
    - 投入小，但影响面大
    - 效果可持续，不是一次性的
```

### Step 3: 设计杠杆路径

```
路径设计:
  □ 从破局点到目标
    - 破局点是什么
    - 破局后到达哪里
    - 最终目标是什么

  □ 资源配置
    - 核心资源投入
    - 辅助资源配合
    - 优先级排序

  □ 验证机制
    - 如何验证破局成功
    - 快速验证方法
    - 失败回退方案
```

### Step 4: 优先级排序

```
排序维度:
  □ 杠杆倍数 (效果)
  □ 确定性 (能否成功)
  □ 可执行性 (能否落地)

优先级公式:
  优先级 = (杠杆倍数 × 确定性 × 可执行性)^0.5

破局点选择:
  Top 1: 优先级最高，立即执行
  Top 2: 次优先，准备执行
  Top 3: 观察项，待时机
```

## 使用示例

```
输入: 电商平台增长陷入瓶颈

Step 1: 现状扫描
  - 用户增长停滞
  - 获客成本持续上升
  - 留存率中等

Step 2: 寻找杠杆点
  - 瓶颈: 留存不高导致获客效率低
  - 杠杆评估:
    - 提升留存: 杠杆高 + 确定性中
    - 优化获客: 杠杆中 + 确定性高
    - 增加SKU: 杠杆中 + 确定性低

Step 3: 设计路径
  核心破局点: 提升留存
    - 路径: 用户分层 → 精准激励 → 留存提升
    - 资源: 产品+运营+数据
    - 验证: 留存率+30天提升

Step 4: 优先级
  Top 1: 留存提升 (立即执行)
  Top 2: 获客优化 (下月执行)
```
