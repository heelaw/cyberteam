# 具体问题全局化

## 身份定位

```
┌─────────────────────────────────────────────────────────────┐
│  🔭 具体问题全局化                                         │
│  用途: 从局部看全局，识别系统性影响                         │
│  配套Agent: 框架思维Agent                                    │
└─────────────────────────────────────────────────────────────┘
```

---

## 核心框架

```
局部 → 全局扩展:
  追问目的 → 识别系统性
  角色转换 → 高维视角
  影响分析 → 关联扩展

全局化检查:
  - 同类问题还有哪些？
  - 上下游影响是什么？
  - 长期影响是什么？
```

---

## 执行步骤

### Step 1: 追问目的

```
追问链:
  问题 → 为什么这个问题重要 → 得到什么 → 为什么重要 → ...

  示例:
    Q: 如何提升转化率
    A: 提升GMV
    Q: 为什么GMV重要
    A: 提升收入
    Q: 为什么收入重要
    A: 公司发展
    Q: 为什么公司发展重要
    A: 实现愿景

  追问结论:
    核心目的: 实现愿景
    直接目标: 提升收入
    手段: 提升GMV
```

### Step 2: 角色转换

```
视角转换:
  □ 从执行者 → 决策者
    - 决策者关心什么
    - 优先级是什么
    - 资源如何分配

  □ 从局部 → 全局
    - 在整体中什么位置
    - 与其他部分关系
    - 整体优先什么

  □ 从当前 → 长期
    - 3年后看现在
    - 长期影响是什么
    - 格局是什么
```

### Step 3: 影响扩展

```
影响分析:
  □ 横向扩展 (同类)
    - 这个问题在其他场景有吗
    - 解决这个能解决多少类似问题
    - 系统性解决方案是什么

  □ 纵向扩展 (上下游)
    - 上游什么会影响这个问题
    - 下游这个问题会影响什么
    - 上下游联动解决方案是什么

  □ 时间扩展
    - 短期影响 (1-3月)
    - 中期影响 (6-12月)
    - 长期影响 (3年+)
```

---

## 全局化检查清单

```
□ 追问到核心目的了吗
□ 从决策者视角看了吗
□ 横向: 同类问题识别了吗
□ 纵向: 上下游影响分析了吗
□ 时间: 长期影响评估了吗
□ 行动: 全局最优解是什么
```

---

## 使用示例

```
原问题: 如何提升客服满意度

Step 1: 追问目的
  客服满意度 → 用户留存 → 口碑传播 → 增长
  核心目的: 增长

Step 2: 角色转换
  决策者视角: 客服是成本中心还是增长引擎
  发现: 客服可以从成本中心变成增长引擎

Step 3: 影响扩展
  横向: 客服→物流→产品→运营 (全链路体验)
  纵向: 客服差评 → 产品问题暴露 → 推动产品优化
  时间: 短期满意度↑ → 长期口碑↑ → 品牌资产↑

结论: 不是"提升客服满意度"，而是"建立全链路体验优化体系"
```
