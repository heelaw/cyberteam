## 四类留存问题模型

```
┌─────────────────────────────────────────────────────────────┐
│                    四类留存问题                               │
│                                                             │
│   用户流失 = max(匹配问题, 体验问题, 认知问题, 竞争问题)     │
│                                                             │
│   1_产品匹配问题: 用户不是目标用户                          │
│      → 解决: 精准获客                                       │
│                                                             │
│   2_产品体验问题: 产品体验差                                │
│      → 解决: 产品优化                                       │
│                                                             │
│   3_价值认知问题: 用户感知不到价值                          │
│      → 解决: 教育引导                                       │
│                                                             │
│   4_竞争流失问题: 被竞品吸引                                │
│      → 解决: 差异化激励                                     │
└─────────────────────────────────────────────────────────────┘
```

---

## 执行步骤

### Step 1: 留存曲线分析

```
分析内容:
  □ 留存曲线形状
    - 理想曲线: 平滑下降后稳定
    - 悬崖曲线: 早期大量流失后稳定
    - 微笑曲线: 先降后升
    - 持续下降: 产品根本性问题

  □ 关键节点
    - 次日留存
    - 7日留存
    - 30日留存
    - 90日留存

  □ 留存基准对比
    - vs 行业基准
    - vs 竞品数据
    - vs 历史数据

输出:
  留存曲线: [类型]
  关键问题: [流失最严重的节点]
```

### Step 2: 流失原因归类

```
诊断方法:
  □ 用户分层对比
    - 留存高用户特征
    - 留存低用户特征
    - 特征差异分析

  □ 行为对比
    - 留存用户做了什么
    - 流失用户没做什么
    - 关键行为识别

  □ 流失时机
    - 在哪个阶段流失
    - 触发流失的事件
    - 流失前的最后行为

流失原因归类:
  □ 产品匹配 → 获客质量问题
  □ 产品体验 → 产品体验问题
  □ 价值认知 → 用户教育问题
  □ 竞争流失 → 留存激励问题
```

### Step 3: 优先级排序

```
排序方法:
  □ 影响量: 影响多少用户
  □ 可解决性: 能否通过产品/运营解决
  □ 投入产出: ROI评估

优先级矩阵:
  ┌──────────────┬──────────────┬──────────────┐
  │ 影响大+可解决  │ 影响中+可解决  │ 影响小        │
  ├──────────────┼──────────────┼──────────────┤
  │ P0: 立即做    │ P1: 尽快做    │ P3: 观察     │
  └──────────────┴──────────────┴──────────────┘
```

## 使用示例

```
输入: 某社交App留存异常

诊断结果:
  留存曲线: 悬崖曲线
    - 次日留存: 35% (行业40%)

    - 30日留存: 8% (行业12%)

  流失节点:
    - 第3-5天流失最多

  原因归类:
    1_匹配问题: 30% (获客不精准)
    2_体验问题: 40% (新用户体验差)
    3_认知问题: 20% (不知道价值在哪)
    4_竞争问题: 10%

  优先级:
    P0: 优化新用户体验 (影响大+可解决)
    P1: 精准获客 (影响中+可解决)
```
