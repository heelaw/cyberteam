## 核心原理

### YC 六问

| # | 问题 | 目的 |
|---|------|------|
| 1 | 需求现实 | 真的有这个需求吗？验证需求的真实性 |
| 2 | 现状 | 现在是怎么做的？了解当前解决方案 |
| 3 | 具体性 | 具体要做什么？明确具体任务 |
| 4 | 切入点 | 最小的切入点是什么？找到最小可行切入点 |
| 5 | 观察 | 有什么证据支持？验证假设 |
| 6 | 未来适配 | 未来能扩展吗？评估长期可行性 |

## 使用流程

### Step 1: 需求现实（Demand Reality）

```
问题：真的有这个需求吗？

分析维度：
- 目标用户真的需要这个吗？
- 需求是真实还是伪需求？
- 市场规模有多大？
- 痛点够不够痛？

输出：需求真实性评估
```

### Step 2: 现状分析（Status Quo）

```
问题：现在是怎么做的？

分析维度：
- 当前解决方案是什么？
- 用户现在如何解决这个问题？
- 现有方案的缺点是什么？
- 为什么需要新方案？

输出：现状分析报告
```

### Step 3: 具体性确认（Specificity）

```
问题：具体要做什么？

分析维度：
- 具体要构建什么？
- 核心功能是什么？
- MVP 包含哪些功能？
- 优先级是什么？

输出：具体任务清单
```

### Step 4: 切入点识别（Wedge）

```
问题：最小的切入点是什么？

分析维度：
- 哪个点最容易切入？
- 最小可行产品是什么？
- 如何验证核心假设？
- 切入点有多窄/宽？

输出：切入点策略
```

### Step 5: 观察验证（Observation）

```
问题：有什么证据支持？

分析维度：
- 用户访谈结果？
- 数据支持？
- 竞品分析？
- 专家意见？

输出：证据清单
```

### Step 6: 未来适配（Future Fit）

```
问题：未来能扩展吗？

分析维度：
- 长期愿景是什么？
- 可扩展性如何？
- 护城河是什么？
- 竞争壁垒？

输出：扩展性评估
```

## 在 CyberTeam 中的应用

### 集成到 CEO

CEO 在分析问题后，运行 Office Hours 进行计划审核：

```python
class CEO:
    def analyze(self, user_input):
        # 1. 基础分析
        analysis = self.basic_analysis(user_input)

        # 2. Office Hours 审核
        review = self.run_office_hours(analysis)

        # 3. 综合输出
        return self.combine(analysis, review)
```

## 与 plan-ceo-review 联动

### plan-ceo-review 四种模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **SCOPE EXPANSION** | 扩展范围，构建更完整系统 | 资源充足，目标远大 |
| **SELECTIVE EXPANSION** | 选择性扩展 | 资源有限，选择性增强 |
| **HOLD SCOPE** | 保持范围 | 资源紧张，严格执行 |
| **SCOPE REDUCTION** | 缩减范围 | 时间紧迫，最小可行 |

### 联动流程

```
Office Hours 审核
    │
    ├──▶ 可行性高 ──▶ 推荐 SCOPE EXPANSION
    │
    ├──▶ 可行性中 ──▶ 推荐 SELECTIVE EXPANSION
    │
    └──▶ 可行性低 ──▶ 推荐 SCOPE REDUCTION
```

## 注意事项

1. **六问必须全回答**：不能跳过任何一个问题
2. **基于证据**：每个结论都需要有证据支持
3. **具体可执行**：建议必须具体可执行
4. **与后续联动**：审核结果指导后续执行
