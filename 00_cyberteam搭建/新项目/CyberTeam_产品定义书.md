# CyberTeam 产品定义书

## 版本
v0.2

## 1. 产品一句话定义

CyberTeam 是一个基于用户本地 Claude Code 的 macOS 桌面端 AI 军团操作系统。用户可以在里面创建公司、部门、Agent 和 Skill，通过类微信聊天界面与 AI 军团协作，并将协作结果沉淀到 Playground 进行审核和交付。

## 2. 产品愿景

让一个人拥有一整支 AI 公司。

用户不是“和 AI 聊天”，而是：
- 组织一家公司
- 招募 AI 员工
- 创建部门和岗位
- 让 Agent 之间协作讨论
- 用群聊推动任务
- 用 CEO 角色做最后审核
- 用 Playground 输出成果

## 3. 产品目标

### 3.1 短期目标
- 做出能跑的 macOS 桌面端
- 能连接用户本地 Claude Code
- 能创建 Agent
- 能聊天
- 能建群聊
- 能做基础协作

### 3.2 中期目标
- 支持组织架构
- 支持部门管理
- 支持 Skill 管理
- 支持 @Agent 任务
- 支持 Agent 自动讨论
- 支持 Playground 输出

### 3.3 长期目标
- 成为个人和小团队的 AI 公司系统
- 支持模板市场
- 支持高度可扩展的 Agent 生态
- 支持更复杂的协作和审批流

## 4. 目标用户

### 4.1 主要用户
个人创作者 / 创业者 / 小团队：需要一个能帮自己干活的 AI 团队，一个能管理 Agent 的桌面软件，一个能产生正式交付物的系统。

### 4.2 次级用户
高度使用 Claude Code 的开发者：需要更强的 GUI、可视化地管理 Agent、二次开发能力。

### 4.3 后续用户
小型组织 / 创新团队：需要共享部门、共享知识、AI 协同、过程留痕、高质量输出。

### 4.4 明确不优先的用户
大型企业 IT 中台、传统 SaaS 企业客户、要求云端多租户和复杂权限的组织。第一版不优先服务这些场景。

## 5. 产品核心特征

### 5.1 基于用户本地 Claude Code
- 软件不替代 Claude Code
- 软件调用用户本地 Claude Code
- 保留 Claude Code 的最新能力
- 支持用户已有配置和命令

### 5.2 AI 军团
- Agent 是“员工”
- 部门是“组织单元”
- 公司是“整体结构”
- Skill 是“能力插件”

### 5.3 类微信聊天
- 真实聊天体验
- 头像、昵称、岗位清晰
- 群聊自然
- @ 提及自然
- 对话有实时感
- 支持私聊、群聊、部门群聊三种基本形态

### 5.4 Playground 产出
- 讨论不止停留在聊天
- 最终沉淀成文档/摘要/记录/审核结果
- 便于复用和归档

### 5.5 CEO 审核
- 重大结果需要 CEO 过目
- 风险任务要审核
- 输出结果可定稿

## 6. 产品组成

### 6.1 公司模块
用于定义整体组织：公司名称、头像、简介、默认风格、版本。

### 6.2 部门模块
用于定义组织层级：讨论层、执行层、专家层、审核层、市场层、运营层。

### 6.2.1 组织分层建议
第一版建议只保留：CEO、讨论层、执行层。其他层级作为后续扩展。
### 6.3 Agent 模块
用于定义“员工”：名称、岗位、头像、性格、技能、权限、部门归属。

### 6.4 Skill 模块
用于定义能力：研究、设计、运营、内容、法务、财务、产品、审核。

### 6.5 Chat 模块
用于日常协作：私聊、群聊、部门群聊、@任务、讨论、汇总。

### 6.6 Playground 模块
用于输出交付：会议记录、摘要、结论、任务清单、审核文档。

## 7. 关键使用场景

### 场景 1：用户创建公司
用户第一次进入软件：创建公司、设置 CEO、选择模板、初始化几个基础部门。

### 场景 2：用户创建部门
用户添加：产品部、运营部、设计部、技术部、法务部、财务部。

### 场景 3：用户招募 Agent
用户从市场安装：产品总监、运营总监、设计师、数据分析师、法务审查员。

### 场景 4：用户发起任务
用户在群聊里 @产品总监、@设计师、@运营总监，让他们一起讨论一个问题。

### 场景 5：CEO 汇总
讨论结束后：CEO 总结，给出决策，产出 Playground 文档。

## 9. 产品差异化

CyberTeam 的差异化不是“我也能聊天”，而是：
1. 我能组织 AI
2. 我能把 AI 变成公司
3. 我能让 AI 像员工一样协作
4. 我能把讨论变成交付物
5. 我能基于用户本地 Claude Code 做二次开发

- 可交付

## 12. 产品边界

### 12.1 你做什么
组织、协作、聊天、输出、审核、市场、模板。

### 12.2 你不做什么
不自己训练模型，不替代 Claude Code，不做纯云端平台起步，不做大而全企业中台。

## 13. 产品成功标准

1. 能在 macOS 上安装
2. 能连接 Claude Code
3. 能创建公司/部门/Agent
4. 能和 Agent 聊天
5. 能群聊并 @Agent
6. 能让 Agent 互相讨论
7. 能生成 Playground 产物
8. 能让 CEO 最终审核

## 14. 结论

CyberTeam 不是“聊天 + Agent”。它是一个让用户在本地 macOS 上拥有 AI 公司、AI 团队、AI 员工和 AI 会议系统的桌面产品。
