## 一、愿景与定位

### 公司愿景
成为全球领先的 AI Agent 协作操作系统，构建能够自我进化、自动协作的智能组织。

### 核心能力
- **思维注入引擎**：100+ 思维专家按需调用
- **能力中台**：gstack / ClawTeam / agency-agents / baoyu-skills
- **DevQA 质量体系**：全流程质量门禁

### 商业模式
```
┌─────────────────────────────────────────────────────────┐
│                      用户/客户                           │
│            （需求输入 → 方案交付 → 价值确认）              │
└─────────────────────────────────────────────────────────┘
```

## 三、层级职责矩阵

### L1: CEO 决策层

| 职责 | 说明 | 工具/能力 |
|------|------|----------|
| 意图分析 | 5W1H 拆解用户需求 | CEO Thinking Engine |
| 团队组建 | 根据任务匹配专家 | 专家推荐算法 |
| 战略决策 | 资源配置、优先级排序 | 100思维专家注入 |
| 质量门禁 | 最终方案审批 | DevQA体系 |

### L2: 总经办

| 部门 | 职责 | 关键产出 |
|------|------|----------|
| **战略部** | 长期规划、投资分析、商业模式 | 年度规划、战略报告 |
| **法务部** | 合规审查、合同审核、风险控制 | 合规清单、合同模板 |
| **品牌PR** | 品牌建设、公共关系、媒体合作 | 品牌手册、PR计划 |
| **质量中台** | DevQA审核、质量标准制定 | QA报告、质量评级 |

### L3: 事业群

#### 产品事业群 (Product BG)

| 部门 | 职责 | 关键产出 |
|------|------|----------|
| **产品部** | 需求分析、产品设计、PRD编写 | 产品路线图、PRD |
| **设计部** | UI/UX设计、视觉规范、品牌设计 | 设计稿、设计系统 |

#### 技术事业群 (Technology BG)

| 部门 | 职责 | 关键产出 |
|------|------|----------|
| **研发部** | 工程开发、代码实现、技术攻关 | 可交付代码 |
| **架构部** | 技术架构、技术选型、性能优化 | 架构文档、技术方案 |

#### 增长事业群 (Growth BG)

| 部门 | 职责 | 关键产出 |
|------|------|----------|
| **运营部** | 用户运营、内容运营、活动运营 | 运营方案、数据报告 |
| **营销部** | 增长营销、渠道运营、品牌推广 | 营销方案、增长数据 |

### L4: 执行层（Agent小组）

| Agent类型 | 职责 | 协作方式 |
|-----------|------|----------|
| 产品经理Agent | 需求细化、PRD撰写、进度跟进 | 跨部门协作 |
| 工程师Agent | 代码实现、单元测试、代码审查 | 敏捷开发 |
| 设计师Agent | 设计稿输出、设计标注、设计验收 | 协作评审 |
| 运营Agent | 运营执行、数据分析、用户反馈 | 实时协作 |
| 营销Agent | 营销内容创作、渠道管理、效果追踪 | 多渠道协同 |

### L5: 中台层

| 中台 | 核心能力 | 服务对象 |
|------|----------|----------|
| **思维注入中台** | 100+思维专家按需调用、专家匹配 | 全公司 |
| **资产库中台** | gstack skills、agency-agents、baoyu-skills | 全公司 |
| **通信协作中台** | ClawTeam消息传递、任务分发、结果汇聚 | 全公司 |
| **记忆存储中台** | 向量数据库、历史记录、上下文管理 | 全公司 |
| **监控中台** | 循环检测、进度追踪、异常告警 | 全公司 |
| **知识沉淀中台** | 最佳实践、经验复用、知识库管理 | 全公司 |

## 四、协作流程

### 4.1 任务处理流程

```
┌─────────────────────────────────────────────────────────────────┐
│                        1. 需求输入                              │
│                    用户描述需求/问题/目标                          │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                     2. CEO 意图分析                             │
│            5W1H拆解 → 任务拆解 → 专家匹配                        │

                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                     3. 战略部分析                                │
│              战略部评估 → 资源配置方案 → CEO审批                  │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                     4. BG任务分派                               │
│    Product BG ←→ Technology BG ←→ Growth BG                    │
│              各BG并行执行 │ 自组织协作                           │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐

                              ↓
┌─────────────────────────────────────────────────────────────────┐

## 六、Agent 小组配置

### 6.1 标准小组配置

每个执行小组（3-5人）:

```
┌─────────────────────────────────────────┐
│           Agent 执行小组                  │
├─────────────────────────────────────────┤
│  👑 小组组长 (Agent Lead)               │
│     - 任务分配                          │
│     - 进度管理                          │
│     - 结果汇总                          │
├─────────────────────────────────────────┤
│  👤 核心成员 (Core Agent) × 2-3         │
│     - 具体执行                          │
│     - 质量自检                          │
├─────────────────────────────────────────┤
│  👤 协作成员 (Support Agent) × 1-2     │
│     - 跨组协作                          │
│     - 资源协调                          │

| 任务类型 | 小组配置 | 预计时长 |
|----------|----------|----------|

## 九、关键成功因素

1. **CEO 思维注入能力**：100专家的高效调用是核心差异化
2. **质量中台独立性**：不受业务压力影响，确保质量标准
3. **中台复用率**：资产和能力的复用率是效率的关键
4. **Agent 自组织能力**：执行层的自驱动是敏捷的基础
5. **知识沉淀机制**：经验的复用是持续进化的保障
