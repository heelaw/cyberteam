# 代码审查

对未提交的变更进行全面的安全和质量审查：

1. 获取更改的文件： git diff --name-only HEAD

2. 对于每个更改的文件，检查：

**安全问题（严重）：**
- 硬编码凭证、API 密钥、令牌
- SQL注入漏洞
- XSS漏洞  
- 缺少输入验证
- 不安全的依赖关系
- 路径遍历风险

**代码质量（高）：**
- 功能 > 50 行
- 文件 > 800 行
- 嵌套深度 > 4 层
- 缺少错误处理
- console.log 语句
- TODO/FIXME 评论
- 缺少公共 API 的 JSDoc

**最佳实践（中）：**
- 突变模式（使用不可变代替）
- 代码/评论中表情符号的使用
- 缺少新代码的测试
- 辅助功能问题 (a11y)

3. 生成报告：
   - 严重性：严重、高、中、低
   - 文件位置和行号
   - 问题描述
   - 建议修复

4. 如果发现严重或严重问题，则阻止提交

永远不要批准有安全漏洞的代码！
