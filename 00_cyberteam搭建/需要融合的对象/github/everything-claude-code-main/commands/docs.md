# /文档

## 目的

查找库、框架或 API 的最新文档，并返回包含相关代码片段的汇总答案。使用 Context7 MCP（resolve-library-id 和 query-docs），因此答案反映当前文档，而不是训练数据。

＃＃ 用法```
/docs [library name] [question]
```对多单词参数使用引号，以便将它们解析为单个标记。示例：`/docs“Next.js”“如何配置中间件？”`

如果省略库或问题，则提示用户：
1. 库或产品名称（例如 Next.js、Prisma、Supabase）。
2. 具体问题或任务（例如“如何设置中间件？”、“身份验证方法”）。

## 工作流程

1. **解析库 ID** — 使用库名称和用户的问题调用 Context7 工具 `resolve-library-id` 以获取 Context7 兼容的库 ID（例如 `/vercel/next.js`）。
2. **查询文档** — 使用该库 ID 和用户的问题调用“query-docs”。
3. **总结** — 返回简洁的答案并包含所获取文档中的相关代码示例。提及库（以及相关版本）。

## 输出

用户会收到由当前文档支持的简短、准确的答案，以及任何有帮助的代码片段。如果 Context7 不可用，请说明并根据训练数据进行回答，并注明文档可能已过时。
