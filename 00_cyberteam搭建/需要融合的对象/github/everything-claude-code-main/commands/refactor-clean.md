# 重构清理

通过每一步的测试验证来安全地识别和删除死代码。

## 步骤 1：检测死代码

根据项目类型运行分析工具：

|工具|它发现了什么 |命令 |
|------|--------------|---------|
|克尼普|未使用的导出、文件、依赖项 | `npx knip` |
|部门检查|未使用的 npm 依赖项 | `npx depcheck` |
| ts 剪枝 |未使用的 TypeScript 导出 | `npx ts-prune` |
|秃鹫|未使用的 Python 代码 | `秃鹫 src/` |
|死代码 |未使用的 Go 代码 | `死代码./...` |
|货运-udeps |未使用的 Rust 依赖项 | `货物+夜间 udeps` |

如果没有可用的工具，请使用 Grep 查找零导入的导出：```
# Find exports, then check if they're imported anywhere
```## 步骤 2：对结果进行分类

将发现结果分类为安全级别：

|等级 |示例 |行动|
|------|----------|--------|
| **安全** |未使用的实用程序、测试助手、内部函数 |放心删除 |
| **注意** |组件、API 路由、中间件 |验证没有动态导入或外部消费者 |
| **危险** |配置文件、入口点、类型定义 |触摸前先进行调查|

## 步骤 3：安全删除循环

对于每个 SAFE 项目：

1. **运行完整的测试套件** — 建立基线（全绿色）
2. **删除死代码** — 使用编辑工具进行手术切除
3. **重新运行测试套件** — 验证没有损坏
4. **如果测试失败** — 立即使用 `git checkout -- <file>` 恢复并跳过此项
5. **如果测试通过** — 移至下一项

## 步骤 4：处理注意事项

删除 CAUTION 项目之前：
- 搜索动态导入：`import()`、`require()`、`__import__`
- 搜索字符串引用：路由名称、配置中的组件名称
- 检查是否从公共包API导出
- 验证没有外部消费者（检查依赖项是否已发布）

## 步骤 5：合并重复项

删除死代码后，查找：
- 近乎重复的函数（>80%相似）——合并为一个
- 冗余类型定义——合并
- 不增加任何价值的包装函数——内联它们
- 无目的的再出口——消除间接性

## 步骤 6：总结

报告结果：```
Dead Code Cleanup
──────────────────────────────
Deleted:   12 unused functions
           3 unused files
           5 unused dependencies
Skipped:   2 items (tests failed)
Saved:     ~450 lines removed
──────────────────────────────
All tests passing ✅
```## 规则

- **在未先运行测试的情况下切勿删除**
- **一次删除一个** — 原子更改使回滚变得容易
- **如果不确定则跳过** — 保留无效代码比破坏生产更好
- **清理时不要重构** - 单独关注（先清理，稍后重构）
