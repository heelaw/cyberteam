分析并优化以下提示以获得最大 ECC 杠杆。

## 你的任务

将 **prompt-optimizer** 技能应用于下面的用户输入。遵循 6 阶段分析流程：

0. **项目检测** — 读取 CLAUDE.md，从项目文件（package.json、go.mod、pyproject.toml 等）中检测技术堆栈
1. **意图检测** - 对任务类型进行分类（新功能、错误修复、重构、研究、测试、审查、文档、基础设施、设计）
2. **范围评估** - 评估复杂性（TRIVIAL / LOW / MEDIUM / HIGH / EPIC），使用代码库大小作为检测到的信号
3. **ECC 组件匹配** — 映射到特定技能、命令、代理和模型层
4. **缺失上下文检测** — 识别差距。如果缺少 3 个以上关键项目，请在生成之前要求用户澄清
5. **工作流程和模型** — 确定生命周期位置，推荐模型层，并在 HIGH/EPIC 时分为多个提示

## 输出要求

- 当前诊断、推荐的 ECC 组件以及使用提示优化器技能中的输出格式的优化提示
- 提供**完整版本**（详细）和**快速版本**（紧凑，因意图类型而异）
- 以与用户输入相同的语言进行响应
- 优化的提示必须完整并准备好复制粘贴到新会话中
- 以页脚结束，提供调整或明确的下一步以启动单独的执行请求
