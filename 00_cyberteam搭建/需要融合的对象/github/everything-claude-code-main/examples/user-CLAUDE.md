# 用户级 CLAUDE.md 示例

这是一个示例用户级 CLAUDE.md 文件。放置在`~/.claude/CLAUDE.md`。

用户级配置全局适用于所有项目。用于：
- 个人编码偏好
- 您始终希望执行的通用规则
- 链接到您的模块化规则

---

## 核心理念

你是克劳德·科德。我使用专门的代理和技能来完成复杂的任务。

**关键原则：**
1. **代理优先**：将复杂的工作委托给专门的代理
2. **并行执行**：尽可能使用具有多个代理的任务工具
3. **执行前计划**：使用计划模式进行复杂操作
4. **测试驱动**：在实施之前编写测试
5. **安全第一**：绝不在安全性上妥协

## 个人喜好

### 隐私
- 始终编辑日志；切勿粘贴机密（API 密钥/令牌/密码/JWT）
- 共享前检查输出 - 删除任何敏感数据

### 代码风格
- 代码、注释或文档中没有表情符号
- 更喜欢不变性 - 从不改变对象或数组
- 许多小文件覆盖少数大文件
- 典型 200-400 行，每个文件最多 800 行

### git
- 常规提交：`feat:`、`fix:`、`refactor:`、`docs:`、`test:`
- 在提交之前始终在本地进行测试
- 小而集中的承诺

### 测试
- TDD：首先编写测试
- 最低覆盖率 80%
- 关键流程的单元+集成+E2E

### 知识捕获
- 个人调试笔记、偏好和临时上下文→自动记忆
- 团队/项目知识（架构决策、API 变更、实施操作手册）→ 遵循项目现有的文档结构
- 如果当前任务已经产生相关文档、评论或示例，请勿在其他地方重复相同的知识
- 如果没有明显的项目文档位置，请在创建新的顶级文档之前询问
