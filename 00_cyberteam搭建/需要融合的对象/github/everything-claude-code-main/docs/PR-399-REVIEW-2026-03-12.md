## 调查结果

### 中等

1.`技能/持续学习-v2/hooks/observe.sh`

新的“CLAUDE_CODE_ENTRYPOINT”防护使用已知的有限允许列表
非“cli”值（“sdk-ts”、“sdk-py”、“sdk-cli”、“mcp”、“remote”）。

这留下了一个前向兼容性漏洞：任何未来的非“cli”入口点值
将会失败并被视为交互式。这重新引入了确切的
PR 试图阻止的一类自动会话观察。

更安全的规则是：

- 只允许`cli`
- 将所有其他显式入口点视为自动化
- 当变量未设置时，将默认后备保留为“cli”

建议形状：```bash
case "${CLAUDE_CODE_ENTRYPOINT:-cli}" in
  cli) ;;
  *) exit 0 ;;
esac
```## 合并推荐

- 它关闭`observer-loop.sh`中的ECC自我观察循环

非交互式入口点。
