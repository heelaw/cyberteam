# 공통 패턴

## 스켈레톤 프로젝트

새 기능을 구현할 때:
1. 검증된 스켈레톤 프로젝트를 검색
2. 重要信息：
   - 보안 평і
   - 확장성 분석
   - 관련성 점수
   - 구현 계획
3. 장 적합한 것을 기반으로 클론
4. 검증된 구조 내에서 반복 개선

## 디자인 패턴

### 리포지토리 패턴

重要信息：
- 功能：findAll、findById、创建、更新、删除
- 구체적 구현 Been 저장소 세부사항 처리 (데ք베스, API, 파일 등)
- 비즈니스 로직은 저장소 메커니즘 Been 아닌 추상 인터페 Been 스에 의존
- 소스의 쉬운 교체 및 모킹을 통한 테스트 단순화 і능

### API 응답 형식

모든 API 应用程序：
- 성공/상태 표시자 포함
- 데로드 포함 (에러 시 null)
- 에러 메시지 필드 포함 (성공 시 null)
- 페다답션 응답에 메타데간터 포함 (总数、页数、限制)
