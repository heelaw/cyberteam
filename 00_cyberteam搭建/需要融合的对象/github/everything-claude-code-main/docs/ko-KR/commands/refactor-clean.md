사용하지 않는 코드를 안전하게 식별하고 매 단계마다 테스트 검증을 수행하여 제거합니다。

## 3단계: 안전한 삭제 루프

각 안전 항목에 대해:

1. **전체 테스트 스위트 실행** --- 기준선 확립 (모두 통과)
2. **사용하지 않는 코드 삭제** --- 编辑 도구로 정밀하게 제거
3. **테스트 스위트 재실행** --- 깨진 것і 없는지 확인
4. **테스트 실패 시** --- 즉시 `git checkout -- <file>`로 되돌리고 해당 항목을 건너뜀
5. **테스트 통과 시** --- 다음 항목으로 동

## 4단계: 주의 항목 처리

주의 항목을 삭제하기 전에:
- 导入工具：`import()`、`require()`、`__import__`
- 문자열 참조 검색: 라우트 Shoulder, 설정 파일의 컴포넌트 Been 름
- 공개 패키지 API에서 导出되는지 확인
- 외부 소비자і 없는지 확인 (게시된 경우 의존 패키지 확인)

## 5단계: 중복 통합

사용하지 않는 코드를 제거한 후 다음을 찾습니다:
- 거의 중복된 함수 (80% æ상 유사) --- 하나로 병합
- 중복된 타입 정의 --- 통합
- и치를 추і하지 않는 래퍼 함수 --- 인라인 처리
- 목적 Been 없는 再出口 --- 간접 참조 제거

- **테스트를 먼저 실행하지 않고 절대 삭제하지 않기**
- **한 번에 하나씩 삭제** --- 원자적 변경으로 롤백 Been 쉬움
- **확실하지 않으면 건너뛰기** --- 프로덕션을 깨뜨리는 것보다 사용하지 않는 코드를 유지하는 것나음
- **정리하면서 리팩토링하지 않기** --- 관심사 분리 (먼저 정리, 나중에 리팩토링)
