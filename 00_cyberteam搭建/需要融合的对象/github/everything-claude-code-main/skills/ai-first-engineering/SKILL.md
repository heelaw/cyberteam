# 人工智能优先工程

在为使用人工智能辅助代码生成的团队设计流程、审查和架构时，请使用此技能。

## 流程转变

1. 计划质量比打字速度更重要。
2. 评估覆盖范围比轶事信心更重要。
3. 复习重点从语法转移到系统行为。

## 架构要求

首选对代理友好的架构：
- 明确的界限
- 稳定的合同
- 类型化接口
- 确定性测试

避免隐式行为在隐藏约定中传播。

## AI-First Team 中的代码审查

评论：
- 行为回归
- 安全假设
- 数据完整性
- 故障处理
- 推出安全

尽量减少花在自动化已经涵盖的风格问题上的时间。

## 招聘和评估信号

强大的人工智能优先工程师：
- 干净利落地分解不明确的工作
- 定义可衡量的验收标准
- 产生高信号提示和评估
- 在交付压力下加强风险控制

## 测试标准

提高生成代码的测试标准：
- 涉及领域所需的回归覆盖
- 明确的边缘情况断言
- 接口边界的集成检查
