# 代理工程

将此技能用于工程工作流程，其中人工智能代理执行大部分实施工作，而人类则执行质量和风险控制。

## 操作原理

1. 执行前定义完成标准。
2. 将工作分解为代理大小的单元。
3. 按任务复杂性路由模型层。
4. 通过评估和回归检查进行测量。

## Eval-First 循环

1. 定义能力评估和回归评估。
2. 运行基线并捕获故障签名。
3.执行实施。
4. 重新运行评估并比较增量。

## 任务分解

应用 15 分钟单位规则：
- 每个单元应可独立验证
- 每个单位应该有一个主要风险
- 每个单元应公开明确的完成条件

## 模型路由

- 俳句：分类、样板转换、缩小编辑
- Sonnet：实现和重构
- Opus：架构、根本原因分析、多文件不变量

## 会话策略

- 继续为紧密耦合的单元举行会议。
- 在主要阶段转变后开始新的会话。
- 里程碑完成后进行压缩，而不是在主动调试期间进行压缩。

## AI 生成代码的审查重点

优先考虑：
- 不变量和边缘情况
- 错误边界
- 安全和授权假设
- 隐藏的耦合和推出风险

当自动格式/lint 已经强制执行样式时，不要将审核周期浪费在仅样式的分歧上。

## 成本纪律

每个任务的跟踪：
- 型号
- 代币估计
- 重试
- 挂钟时间
- 成功/失败

仅当较低层因明显的推理差距而失败时才升级模型层。
