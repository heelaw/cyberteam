# 特工线束构造

当您改进代理的计划、调用工具、从错误中恢复以及完成时收敛的方式时，请使用此技能。

## 核心模型

代理输出质量受到以下因素的限制：
1. 行动空间品质
2. 观察质量
3. 恢复质量
4. 背景预算质量

## 动作空间设计

1. 使用稳定、明确的工具名称。
2. 保持输入模式优先且范围狭窄。
3. 返回确定性的输出形状。
4. 除非不可能隔离，否则避免使用包罗万象的工具。

## 粒度规则

- 使用微型工具进行高风险操作（部署、迁移、权限）。
- 使用中等工具进行常见的编辑/读取/搜索循环。
- 仅当往返开销是主要成本时才使用宏工具。

## 观察设计

每个工具响应应包括：
- `状态`：成功|警告|错误
- `summary`：一行结果
- `next_actions`：可操作的后续行动
- `artifacts`：文件路径/ID

## 错误恢复合约

对于每个错误路径，包括：
- 根本原因提示
- 安全重试指令
- 明确的停止条件

## 情境预算

1. 保持系统提示最小化且不变。
2. 将大量指导转移到按需加载的技能中。
3. 更喜欢引用文件而不是内联长文档。
4. 在阶段边界处进行压缩，而不是任意的令牌阈值。

## 架构模式指导

- ReAct：最适合路径不确定的探索性任务。
- 函数调用：最适合结构化确定性流程。
- 混合（推荐）：ReAct 规划 + 类型化工具执行。

## 基准测试

曲目：
- 完成率
- 每个任务的重试次数
- 通过@1 和通过@3
- 每个成功任务的成本

## 反模式

- 语义重叠的工具太多。
- 不透明的工具输出，没有恢复提示。
- 仅错误输出，没有后续步骤。
- 上下文超载，包含不相关的引用。
