# 内容引擎

将一个想法转化为强大的平台原生内容，而不是到处发布相同的内容。

## 何时激活

- 撰写 X 个帖子或话题
- 起草 LinkedIn 帖子或发布更新
- 编写短片或 YouTube 解说脚本
- 将文章、播客、演示或文档重新调整为社交内容
- 围绕发布、里程碑或主题制定轻量级内容计划

## 第一个问题

澄清：
- 来源资产：我们正在改编什么
- 受众：建设者、投资者、客户、运营商或普通受众
- 平台：X、LinkedIn、TikTok、YouTube、时事通讯或多平台
- 目标：意识、转化、招募、权威、启动支持或参与

## 核心规则

1、适配平台。不要交叉发布相同的副本。
2. 钩子比摘要更重要。
3.每一个帖子都应该有一个清晰的想法。
4. 使用具体细节而不是口号。
5. 保持要求小而明确。

## 平台指导

### X
- 打开速度快
- 每个帖子或线程中的每条推文有一个想法
- 除非必要，否则将链接保留在主体之外
- 避免主题标签垃圾邮件

### 领英
- 强大的第一线
- 短段落
- 围绕教训、结果和要点进行更明确的框架

### TikTok/短视频
- 前3秒必须打断注意力
- 剧本围绕视觉效果而不仅仅是旁白
- 一项演示、一项索赔、一项 CTA

### YouTube
- 尽早显示结果
- 按章节的结构
- 每 20-30 秒刷新一次视觉效果

### 时事通讯
- 提供一个透明镜片，而不是一堆不相关的物品
- 使章节标题可略读
- 让开头段落发挥实际作用

## 重新利用流程

默认级联：
1. 锚定资产：文章、视频、演示、备忘录或启动文档
2. 提取3-7个原子想法
3. 编写平台原生变体
4. 修剪输出之间的重复
5.使 CTA 与平台意图保持一致

## 可交付成果

当询问活动时，返回：
- 核心角
- 特定于平台的草案
- 可选的过帐顺序
- 可选的 CTA 变体
- 发布前需要任何缺失的输入

## 质量门

发货前：
- 每个草稿都适合其平台
- 挂钩坚固且具体
- 没有通用的炒作语言
- 除非有要求，否则不得跨平台复制副本
- CTA 与内容和受众相匹配
