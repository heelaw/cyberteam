# 战略紧凑技能

建议在工作流程中的战略点手动“/compact”，而不是依赖任意自动压缩。

## 何时激活

- 运行接近上下文限制的长会话（200K+ 令牌）
- 处理多阶段任务（研究→计划→实施→测试）
- 在同一会话中的不相关任务之间切换
- 完成一个重要的里程碑并开始新的工作后
- 当反应减慢或变得不太连贯时（上下文压力）

## 为什么要进行战略压缩？

自动压缩在任意点触发：
- 经常在任务进行到一半时失去重要的背景
- 没有逻辑任务边界的意识
- 可以中断复杂的多步操作

逻辑边界的战略压缩：
- **探索之后，执行之前** —— 紧凑研究背景，保留实施计划
- **完成里程碑后** — 下一阶段的全新开始
- **在重大背景变化之前** - 在不同任务之前清除探索背景

## 它是如何工作的

`suggest-compact.js` 脚本在 PreToolUse（编辑/写入）上运行并且：

1. **跟踪工具调用** — 计算会话中的工具调用次数
2. **阈值检测** — 建议配置阈值（默认值：50 次调用）
3. **定期提醒** — 达到阈值后每 25 次呼叫提醒一次

## 挂钩设置

添加到您的“~/.claude/settings.json”：```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Edit",
        "hooks": [{ "type": "command", "command": "node ~/.claude/skills/strategic-compact/suggest-compact.js" }]
      },
      {
        "matcher": "Write",
        "hooks": [{ "type": "command", "command": "node ~/.claude/skills/strategic-compact/suggest-compact.js" }]
      }
    ]
  }
}
```## 配置

环境变量：
- `COMPACT_THRESHOLD` — 在第一个建议之前调用工具（默认值：50）

## 压实决策指南

使用此表来决定何时进行压缩：

|相变|袖珍的？ |为什么 |
|-----------------|----------|-----|
|研究→规划|是的 |研究背景庞大；计划是蒸馏输出 |
|规划→实施|是的 |计划位于 TodoWrite 或文件中；释放代码上下文 |
|实施→测试|也许|如果测试引用最近的代码，则保留；切换焦点时紧凑 |
|调试 → 下一个功能 |是的 |调试跟踪会污染不相关工作的上下文 |
|实施中期 |没有 |丢失变量名称、文件路径和部分状态的代价高昂 |
|尝试失败后 |是的 |在尝试新方法之前清除死胡同的推理 |

## 什么能经受住压实

了解持续存在的内容可以帮助您充满信心地压缩：

|坚持 |迷失|
|----------|------|
| CLAUDE.md 说明 |中级推理与分析 |
| TodoWrite 任务列表 |您之前阅读过的文件内容 |
|内存文件(`~/.claude/memory/`) |多步骤对话上下文 |
| Git 状态（提交、分支）|工具调用历史记录和计数 |
|磁盘上的文件 |口头表述细致入微的用户偏好 |

## 最佳实践

1. **规划后压缩** — 一旦在 TodoWrite 中最终确定计划，压缩以重新开始
2. **调试后紧凑** — 在继续之前清除错误解决上下文
3. **不要在实现过程中进行压缩** - 保留相关更改的上下文
4. **阅读建议** — 钩子告诉您*何时*，您决定*如果*
5. **压缩前写入** — 压缩前将重要上下文保存到文件或内存中
6. **使用带有摘要的“/compact”** - 添加自定义消息：“/compact 接下来重点关注实现身份验证中间件”

## 代币优化模式

### 触发器表延迟加载
不要在会话开始时加载完整的技能内容，而是使用将关键字映射到技能路径的触发器表。技能仅在触发时加载，将基线上下文减少 50% 以上：

|触发|技能|加载时间 |
|--------|---------|------------|
| “测试”、“tdd”、“覆盖范围”| TDD 工作流程 |用户提及测试 |
| “安全”、“身份验证”、“xss” |安全审查|安全相关工作|
| “部署”、“ci/cd”|部署模式 |部署环境 |

### 上下文构成意识
监控上下文窗口的消耗情况：
- **CLAUDE.md 文件** — 始终加载，保持精简
- **加载技能** — 每个技能增加 1-5K 代币
- **对话历史记录** — 随着每次交流而增长
- **工具结果** — 文件读取、搜索结果批量添加

### 重复指令检测
重复上下文的常见来源：
- `~/.claude/rules/` 和项目 `.claude/rules/` 中的规则相同
- 重复 CLAUDE.md 指令的技能
- 涵盖重叠领域的多种技能

### 上下文优化工具
-“令牌优化器”MCP — 通过内容重复数据删除自动减少 95% 以上的令牌
- `context-mode` — 上下文虚拟化（演示了 315KB 到 5.4KB）
