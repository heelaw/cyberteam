# Next.js 和 Turbopack

Next.js 16+ 默认使用 Turbopack 进行本地开发：用 Rust 编写的增量捆绑器，可显着加快开发启动和热更新速度。

## 何时使用

- **Turbopack（默认开发）**：用于日常开发。更快的冷启动和 HMR，尤其是在大型应用程序中。
- **Webpack（旧版开发）**：仅当您遇到 Turbopack 错误或依赖于开发中的仅 webpack 插件时才使用。使用“--webpack”（或“--no-turbopack”，具体取决于您的 Next.js 版本；检查您的版本的文档）禁用。
- **生产**：生产构建行为（“下一个构建”）可能会使用 Turbopack 或 webpack，具体取决于 Next.js 版本；检查官方 Next.js 文档以了解您的版本。

使用场合：开发或调试 Next.js 16+ 应用程序、诊断开发启动缓慢或 HMR，或优化生产包。

## 它是如何工作的

- **Turbopack**：Next.js 开发的增量捆绑器。使用文件系统缓存，因此重新启动速度更快（例如，大型项目为 5-14 倍）。
- **开发中的默认**：从 Next.js 16 开始，除非禁用，否则“next dev”将与 Turbopack 一起运行。
- **文件系统缓存**：重新启动重用以前的工作；缓存通常位于“.next”下；基本使用不需要额外的配置。
- **捆绑分析器（Next.js 16.1+）**：实验性捆绑分析器，用于检查输出并查找严重依赖项；通过配置或实验标志启用（请参阅适用于您的版本的 Next.js 文档）。

## 示例

### 命令```bash
next dev
next build
next start
```### 用法

使用 Turbopack 运行“next dev”进行本地开发。使用捆绑分析器（请参阅 Next.js 文档）来优化代码分割并修剪大型依赖项。尽可能首选应用路由器和服务器组件。

## 最佳实践

- 继续使用最新的 Next.js 16.x，以获得稳定的 Turbopack 和缓存行为。
- 如果开发速度很慢，请确保您使用 Turbopack（默认）并且缓存不会被不必要地清除。
- 对于生产包大小问题，请使用适合您版本的官方 Next.js 包分析工具。
