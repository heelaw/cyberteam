# 检查点命令

保存当前验证状态并创建进度检查点：$ARGUMENTS

## 你的任务

创建当前进度的快照，包括：

1. **测试状态** - 哪些测试通过/失败
2. **覆盖率** - 当前覆盖率指标
3. **构建状态** - 构建成功或错误
4. **代码更改** - 修改摘要
5. **后续步骤** - 尚待完成的工作

## 检查点格式

### 检查点：[时间戳]

**测试**
- 总计：X
- 传球：是
- 失败：Z
- 覆盖率：XX%

**构建**
- 状态： ✅ 通过 / ❌ 失败
- 错误：[如果有]

**自上次检查点以来的变化**```
git diff --stat [last-checkpoint-commit]
```**已完成的任务**
- [x] 任务 1
- [x] 任务 2
- [ ] 任务 3（进行中）

**阻塞问题**
- [问题描述]

**后续步骤**
1. 步骤1
2. 步骤2

## 与验证循环一起使用

检查点与验证循环集成：```
/plan → implement → /checkpoint → /verify → /checkpoint → implement → ...
```使用检查点来：
- 在有风险的更改之前保存状态
- 跟踪各个阶段的进展
- 如果需要启用回滚
- 文件验证点

---

**提示**：在自然断点处创建检查点：每个阶段之后、主要重构之前、修复关键错误之后。
