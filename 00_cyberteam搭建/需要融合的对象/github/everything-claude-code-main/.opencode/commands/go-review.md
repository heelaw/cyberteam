# 去查看命令

查看 Go 代码以了解惯用模式和最佳实践：$ARGUMENTS

## 你的任务

1. **分析Go代码**的习惯用法和模式
2. **检查并发性** - goroutine、通道、互斥体
3. **查看错误处理** - 正确的错误包装
4. **验证性能** - 分配、瓶颈

## 审核清单

### 惯用的围棋
- [ ] 包命名（小写，无下划线）
- [ ] 变量命名（驼峰式，短）
- [ ] 接口命名（以-er结尾）
- [ ] 命名错误（以 Err 开头）

### 错误处理
- [ ] 检查错误，而不是忽略错误
- [ ] 用上下文包裹的错误 (`fmt.Errorf("...: %w", err)`)
- [ ] 正确使用哨兵错误
- [ ] 需要时自定义错误类型

### 并发
- [ ] Goroutine 得到妥善管理
- [ ] 通道适当缓冲
- [ ] 无数据竞争（使用 `-race` 标志）
- [ ] 上下文已传递用于取消
- [ ] WaitGroups 正确使用

### 性能
- [ ] 避免不必要的分配
- [ ] 使用 `sync.Pool` 进行频繁分配
- [ ] 更喜欢小型结构的值接收器
- [ ] 缓冲区 I/O 操作

### 代码组织
- [ ] 小型、专注的软件包
- [ ] 明确依赖方向
- [ ] 私有代码的内部包
- [ ] Godoc 对出口的评论
