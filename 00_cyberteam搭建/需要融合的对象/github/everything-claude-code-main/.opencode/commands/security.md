# 安全审查命令

进行全面的安全审查：$ARGUMENTS

## 你的任务

按照 OWASP 指南和安全最佳实践分析指定代码的安全漏洞。

## 安全检查表

### OWASP 前 10 名

1. **注入**（SQL、NoSQL、操作系统命令、LDAP）
   - 检查参数化查询
   - 验证输入净化
   - 审查动态查询构造

2. **身份验证失效**
   - 密码存储（bcrypt、argon2）
   - 会话管理
   - 多重身份验证
   - 密码重置流程

3. **敏感数据暴露**
   - 静态和传输中加密
   - 正确的密钥管理
   - PII 处理

4. **XML 外部实体 (XXE)**
   - 禁用 DTD 处理
   - XML 输入验证

5. **访问控制损坏**
   - 每个端点的授权检查
   - 基于角色的访问控制
   - 资源所有权验证

6. **安全配置错误**
   - 删除了默认凭据
   - 错误处理不会泄漏信息
   - 配置安全标头

7. **跨站脚本（XSS）**
   - 输出编码
   - 内容安全政策
   - 输入净化

8. **不安全的反序列化**
   - 验证序列化数据
   - 实施完整性检查

9. **使用具有已知漏洞的组件**
   - 运行“npm 审核”
   - 检查是否有过时的依赖项

10. **日志记录和监控不足**
    - 记录安全事件
    - 日志中没有敏感数据
    - 警报已配置

### 额外检查

- [ ] 代码中的秘密（API 密钥、密码）
- [ ] 环境变量处理
- [ ] CORS配置
- [ ] 速率限制
- [ ] CSRF 保护
- [ ] 安全 cookie 标志
