# 构建修复命令

通过最小的更改修复构建和 TypeScript 错误：$ARGUMENTS

## 你的任务

1. **运行类型检查**：`npx tsc --noEmit`
2. **收集所有错误**
3. **用最少的改动一一修复错误**
4. **验证每个修复**不会引入新的错误
5. **运行最终检查**以确认所有错误均已解决

## 方法

### 要做：
- ✅ 使用正确的类型修复类型错误
- ✅ 添加缺失的导入
- ✅修复语法错误
- ✅ 进行最小的改变
- ✅ 保留现有行为
- ✅ 每次更改后运行 `tsc --noEmit`

### 不要：
- ❌重构代码
- ❌添加新功能
- ❌ 改变架构
- ❌ 使用“any”类型（除非绝对必要）
- ❌添加`@ts-ignore`评论
- ❌改变业务逻辑

## 常见错误修复

|错误|修复 |
|--------|-----|
|类型“X”不可分配给类型“Y” |添加正确的类型注释 |
|属性“X”不存在 |向界面添加属性或修复属性名称 |
|找不到模块“X”|安装包或修复导入路径 |
| 'X' 类型的参数不可赋值 |强制转换或修复函数签名 |
|对象可能是“未定义” |添加空检查或可选链接 |

## 验证步骤

修复后：
1. `npx tsc --noEmit` - 应显示 0 个错误
2. `npm run build` - 应该成功
3. `npm test` - 测试仍应通过

---

**重要**：仅专注于修复错误。没有重构，没有改进，没有架构变化。以最小的差异实现绿色构建。
