# Rust 审查命令

查看 Rust 代码以了解惯用模式和最佳实践：$ARGUMENTS

## 你的任务

1. **分析 Rust 代码** 的习语和模式
2. **检查所有权** - 借用、生命周期、不必要的克隆
3. **检查错误处理** - 正确的“？”传播，在生产中没有解包
4. **验证安全性** - 不安全的使用、注入、秘密

## 审核清单

### 安全（关键）
- [ ] 生产路径中没有未选中的 `unwrap()`/`expect()`
- [ ] `unsafe` 块有 `// SAFETY:` 注释
- [ ] 无 SQL/命令注入
- [ ] 没有硬编码的秘密

### 所有权（高）
- [ ] 没有不必要的 `.clone()` 来满足借用检查器
- [ ] 在函数参数中`&str`优先于`String`
- [ ] `&[T]` 在函数参数中优先于 `Vec<T>`
- [ ] 在省略有效的地方没有过多的生命周期注释

### 错误处理（高）
- [ ] 使用“?”传播错误；在 `anyhow`/`eyre` 应用程序代码中使用 `.context()`
- [ ] 没有静默错误（`let _ = result;`）
- [ ] `thiserror` 表示库错误，`anyhow` 表示应用程序错误

### 并发（高）
- [ ] 异步上下文中无阻塞
- [ ] 首选有界通道
- [ ]`Mutex`中毒已处理
- [ ] `Send`/`Sync` 边界正确

### 代码质量（中）
- [ ] 50行以下的函数
- [ ] 无深层嵌套（>4 层）
- [ ] 业务枚举的详尽匹配
- [ ] Clippy 警告得到解决
