# 计划命令

为以下内容创建详细的实施计划：$ARGUMENTS

## 你的任务

1. **重申要求** - 明确需要构建什么
2. **识别风险** - 暴露潜在问题、阻碍因素和依赖性
3. **创建步骤计划** - 将实施分解为阶段
4. **等待确认** - 在继续之前必须获得用户批准

## 输出格式

### 要求重述
[清晰、简洁地重述将要建设的内容]

### 实施阶段
[第一阶段：描述]
- 步骤 1.1
- 步骤 1.2
...

[第二阶段：描述]
- 步骤 2.1
- 步骤 2.2
...

### 依赖关系
[列出所需的外部依赖项、API、服务]

### 风险
- 高：[可能阻碍实施的关键风险]
- 中：[需要解决的中等风险]
- 低：[小问题]

### 估计复杂度
[高/中/低以及时间估计]

**等待确认**：继续执行此计划吗？ （是/否/修改）

---

**关键**：在用户明确使用“是”、“继续”或类似的肯定响应进行确认之前，请勿编写任何代码。
