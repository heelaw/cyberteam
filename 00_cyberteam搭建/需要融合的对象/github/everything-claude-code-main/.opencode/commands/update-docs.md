# 更新文档命令

更新文档以反映最近的更改：$ARGUMENTS

## 你的任务

1. **识别更改的代码** - `git diff --name-only`
2. **查找相关文档** - 自述文件、API 文档、指南
3. **更新文档** - 与代码保持同步
4. **验证准确性** - 文档匹配实施

## 文档类型

### 自述文件.md
- 安装说明
- 快速入门指南
- 功能概述
- 配置选项

### API 文档
- 端点描述
- 请求/响应格式
- 身份验证详细信息
- 错误代码

### 代码注释
- 公共 API 的 JSDoc
- 复杂的逻辑解释
- TODO/FIXME 清理

### 指南
- 操作方法教程
- 架构决策 (ADR)
- 故障排除指南

## 更新清单

- [ ] 自述文件反映了当前的功能
- [ ] API 文档匹配端点
- [ ] JSDoc 更新了更改的功能
- [ ] 示例正在运行
- [ ] 链接有效
- [ ] 版本号已更新

## 文档质量

### 良好的文档
- 准确且最新
- 清晰简洁
- 有工作示例
- 涵盖边缘情况

### 避免
- 过时的信息
- 缺少参数
- 损坏的例子
- 含糊的语言

---

**重要**：文档应与代码更改一起更新，而不是事后才想到。
