# 编排命令

为这项复杂的任务协调多个专业代理：$ARGUMENTS

## 你的任务

1. **分析任务复杂性**并分解为子任务
2. **为每个子任务确定最佳代理**
3. **创建具有依赖关系的执行计划**
4. **协调执行** - 尽可能并行
5. **将结果综合**成统一输出

## 编排模式

### 顺序执行```
planner → tdd-guide → code-reviewer → security-reviewer
```使用时间：后面的任务取决于前面的结果

### 并行执行```
┌→ security-reviewer
planner →├→ code-reviewer
└→ architect
```使用时间：任务是独立的

### 扇出/扇入```
         ┌→ agent-1 ─┐
planner →├→ agent-2 ─┼→ synthesizer
         └→ agent-3 ─┘
```使用时机：需要多个视角

## 执行计划格式

### 第一阶段：[姓名]
- 代理：[代理名称]
- 任务：[具体任务]
- 取决于：[无或前一阶段]

### 第 2 阶段：[名称]（并行）
- 特工 A：[特工姓名]
  - 任务：[具体任务]
- 特工 B：[特工姓名]
  - 任务：[具体任务]
- 取决于：第一阶段

### 第三阶段：综合
- 合并第二阶段的结果
- 生成统一的输出

## 协调规则

1. **执行前计划** - 首先创建完整的执行计划
2. **最小化切换** - 减少上下文切换
3. **尽可能并行化** - 并行独立任务
4. **清晰的界限** - 每个代理都有特定的范围
5. **单一事实来源** - 一名特工拥有每件文物

---

**注意**：复杂的任务受益于多代理编排。简单的任务应该直接使用单个代理。
