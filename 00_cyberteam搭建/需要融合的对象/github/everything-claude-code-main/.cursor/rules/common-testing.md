# 测试要求

## 最低测试覆盖率：80%

测试类型（全部必需）：
1. **单元测试** - 单独的功能、实用程序、组件
2. **集成测试** - API端点、数据库操作
3. **E2E 测试** - 关键用户流程（每种语言选择的框架）

## 测试驱动开发

强制性工作流程：
1.先写测试（红色）
2. 运行测试 - 它应该失败
3. 编写最小实现（绿色）
4. 运行测试 - 它应该通过
5.重构（改进）
6. 验证覆盖率（80%+）

## 测试失败故障排除

1.使用**tdd-guide**代理
2. 检查测试隔离
3.验证mock是否正确
4.修复实施，而不是测试（除非测试是错误的）

## 代理支持

- **tdd-guide** - 主动使用新功能，强制先写测试
