# 常见模式

## 骨架项目

实现新功能时：
1.寻找久经考验的骨架项目
2. 使用并行代理来评估选项：
   - 安全评估
   - 可扩展性分析
   - 相关性评分
   - 实施规划
3.克隆最佳匹配作为基础
4. 在经过验证的结构内进行迭代

## 设计模式

### 存储库模式

将数据访问封装在一致的接口后面：
- 定义标准操作：findAll、findById、创建、更新、删除
- 具体实现处理存储细节（数据库、API、文件等）
- 业务逻辑依赖于抽象接口，而不是存储机制
- 可以轻松交换数据源并简化模拟测试

### API 响应格式

对所有 API 响应使用一致的信封：
- 包括成功/状态指示器
- 包括数据有效负载（错误时可为空）
- 包含错误消息字段（成功时可为空）
- 包括分页响应的元数据（总数、页数、限制）
