# 文章写作

编写听起来像真人或品牌的长篇内容，而不是通用的人工智能输出。

## 何时激活

- 起草博客文章、论文、发布文章、指南、教程或时事通讯问题
- 将笔记、成绩单或研究转化为精美的文章
- 从示例中匹配现有的创始人、运营商或品牌声音
- 在已经写好的长文中收紧结构、节奏和证据

## 核心规则

1. 以具体事物开头：示例、输出、轶事、数字、屏幕截图描述或代码块。
2. 在示例之后而不是之前进行解释。
3. 喜欢简短、直接的句子，而不是填充的句子。
4. 使用可用且有来源的具体数字。
5. 切勿发明传记事实、公司指标或客户证据。

## 语音捕捉工作流程

如果用户想要特定的声音，请收集以下一项或多项：
- 发表的文章
- 时事通讯
- X / LinkedIn 帖子
- 文档或备忘录
- 简短的风格指南

然后提取：
- 句子长度和节奏
- 声音是否正式、对话或尖锐
- 喜欢的修辞手段，如括号、列表、片段或问题
- 对幽默、观点和逆向框架的容忍
- 格式化习惯，例如标题、项目符号、代码块和引述

如果没有给出语音参考，则默认为直接的、操作员风格的语音：具体、实用且低调。

## 禁止的模式

删除并重写其中任何一个：
- 通用的开头，例如“在当今快速发展的环境中”
- 填充过渡，例如“Moreover”和“Furthermore”
- 诸如“游戏规则改变者”、“尖端”或“革命性”之类的炒作短语
- 没有证据的模糊说法
- 传记或可信度主张没有提供的背景支持

## 写作过程

1. 明确受众和目的。
2. 构建一个骨架轮廓，每个部分都有一个目的。
3. 每个部分以证据、示例或场景开始。
4. 仅在下一个句子出现的地方进行扩展。
5. 删除任何听起来模板化或自我吹嘘的内容。

## 结构指导

### 技术指南
- 以读者得到的内容开头
- 在每个主要部分使用代码或终端示例
- 以具体的要点结束，而不是软性的总结

### 散文/观点文章
- 从紧张、矛盾或敏锐的观察开始
- 每节保留一个论点
- 使用赢得意见的例子

### 时事通讯
- 保持第一屏的强大
- 将洞察力与更新结合起来，而不是日记填充物
- 使用清晰的部分标签和易于浏览的结构

## 质量门

发货前：
- 根据所提供的来源验证事实主张
- 删除填充物和公司语言
- 确认声音与提供的示例匹配
- 确保每个部分都添加新信息
- 检查目标平台的格式
