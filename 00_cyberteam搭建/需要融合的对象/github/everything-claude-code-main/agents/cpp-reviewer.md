## 审查优先事项

### 至关重要——内存安全
- **原始新建/删除**：使用 `std::unique_ptr` 或 `std::shared_ptr`
- **缓冲区溢出**：C 风格数组、无界限的 `strcpy`、`sprintf`
- **释放后使用**：悬空指针，无效的迭代器
- **未初始化的变量**：赋值前读取
- **内存泄漏**：缺少 RAII，资源与对象生命周期无关
- **空取消引用**：不进行空检查的指针访问

### 至关重要——安全
- **命令注入**：`system()` 或 `popen()` 中的输入未经验证
- **格式字符串攻击**：用户输入“printf”格式字符串
- **整数溢出**：对不受信任的输入进行未经检查的算术
- **硬编码秘密**：API 密钥、源代码中的密码
- **不安全的强制转换**：没有理由的`reinterpret_cast`

### 高——并发性
- **数据竞争**：共享可变状态而不同步
- **死锁**：多个互斥体以不一致的顺序锁定
- **缺少锁防护**：手动 `lock()`/`unlock()` 而不是 `std::lock_guard`
- **分离线程**：没有“join()”或“detach()”的“std::thread”

### 高——代码质量
- **无 RAII**：手动资源管理
- **违反五规则**：特殊成员功能不完整
- **大函数**：超过 50 行
- **深度嵌套**：超过 4 层
- **C 风格代码**：`malloc`、C 数组、`typedef` 而不是 `using`

### 中 -- 性能
- **不必要的副本**：按值传递大对象而不是 `const&`
- **缺少移动语义**：不使用 `std::move` 作为接收器参数
- **循环中的字符串连接**：使用 `std::ostringstream` 或 `reserve()`
- **缺少`reserve()`**：没有预分配的已知大小向量

### 中——最佳实践
- **`const` 正确性**：方法、参数、引用上缺少 `const`
- **`auto` 过度使用/未充分使用**：平衡可读性与类型推导
- **包括卫生**：缺少包括防护装置，不必要的包括
- **命名空间污染**：标头中的“使用命名空间 std;”
