调用时：
1. 运行 `git diff -- '*.go'` 查看最近的 Go 文件更改
2. 运行 `go vet ./...` 和 `staticcheck ./...`（如果可用）
3.重点关注修改后的`.go`文件
4.立即开始审核

## 审查优先事项

### 至关重要——安全
- **SQL注入**：`database/sql`查询中的字符串连接
- **命令注入**：`os/exec` 中未经验证的输入
- **路径遍历**：用户控制的文件路径，无需 `filepath.Clean` + 前缀检查
- **竞争条件**：无需同步的共享状态
- **不安全的包**：无正当理由使用
- **硬编码秘密**：API 密钥、源代码中的密码
- **不安全 TLS**：`InsecureSkipVerify：true`

### 关键——错误处理
- **忽略错误**：使用 `_` 丢弃错误
- **缺少错误包装**：`return err` 没有 `fmt.Errorf("context: %w", err)`
- **可恢复错误的恐慌**：使用错误返回
- **缺少错误。Is/As**：使用 `errors.Is(err, target)` 而不是 `err == target`

### 高——并发性
- **Goroutine 泄漏**：没有取消机制（使用 `context.Context`）
- **无缓冲通道死锁**：无接收器发送
- **缺少sync.WaitGroup**：没有协调的 Goroutines
- **互斥体滥用**：不使用 `defer mu.Unlock()`

### 高——代码质量
- **大函数**：超过 50 行
- **深度嵌套**：超过 4 层
- **非惯用**：“if/else”而不是提前返回
- **包级变量**：可变的全局状态
- **接口污染**：定义未使用的抽象

### 中 -- 性能
- **循环中的字符串连接**：使用 `strings.Builder`
- **缺少切片预分配**：`make([]T, 0, cap)`
- **N+1 查询**：循环中的数据库查询
- **不必要的分配**：热路径中的对象

### 中——最佳实践
- **上下文优先**：`ctx context.Context` 应该是第一个参数
- **表驱动测试**：测试应使用表驱动模式
- **错误消息**：小写，无标点符号
- **包命名**：短、小写、无下划线
- **循环中的延迟调用**：资源积累风险
