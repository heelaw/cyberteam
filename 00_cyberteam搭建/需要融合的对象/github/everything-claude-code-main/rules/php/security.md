# PHP 安全

> 此文件使用 PHP 特定内容扩展了 [common/security.md](../common/security.md)。

## 输入和输出

- 在框架边界验证请求输入（“FormRequest”、Symfony Validator 或显式 DTO 验证）。
- 默认情况下转义模板中的输出；将原始 HTML 渲染视为必须合理的例外。
- 切勿在未经验证的情况下信任查询参数、cookie、标头或上传的文件元数据。

## 数据库安全

- 对所有动态查询使用准备好的语句（`PDO`、Doctrine、Eloquent 查询生成器）。
- 避免在控制器/视图中构建字符串 SQL。
- 仔细范围 ORM 批量分配并将可写字段列入白名单。

## 秘密和依赖

- 从环境变量或秘密管理器加载秘密，而不是从提交的配置文件中加载秘密。
- 在 CI 中运行“编写者审核”，并在添加依赖项之前检查新的包维护者信任。
- 故意固定主要版本并快速删除废弃的软件包。

## 身份验证和会话安全

- 使用 `password_hash()` / `password_verify()` 进行密码存储。
- 身份验证和权限更改后重新生成会话标识符。
- 对状态更改的 Web 请求实施 CSRF 保护。
