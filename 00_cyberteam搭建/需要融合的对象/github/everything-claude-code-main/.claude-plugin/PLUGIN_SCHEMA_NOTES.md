# 插件清单架构注释

本文档捕获了 Claude Code 插件清单验证器的**未记录但强制的约束**。

这些规则基于真实的安装失败、验证器行为以及与已知工作插件的比较。
它们的存在是为了防止无声的破坏和重复的回归。

如果您编辑 `.claude-plugin/plugin.json`，请先阅读此内容。

---

## 摘要（首先阅读本文）

Claude 插件清单验证器**严格且固执己见**。
它强制执行公共模式引用中未完整记录的规则。

最常见的故障模式是：

> 清单看起来合理，但验证器拒绝它并出现模糊错误，例如
> `代理：输入无效`

本文档解释了原因。

---

## 必填字段

### `版本`（强制）

即使在某些示例中省略了“version”字段，验证器仍需要该字段。

如果缺少，安装可能会在市场安装或 CLI 验证期间失败。

例子：```json
{
  "version": "1.1.0"
}
```---

## 字段形状规则

以下字段**必须始终是数组**：

*`代理`
* `命令`
*`技能`
*“钩子”（如果存在）

即使只有一项，**也不接受字符串**。

＃＃＃ 无效的```json
{
  "agents": "./agents"
}
```＃＃＃ 有效的```json
{
  "agents": ["./agents/planner.md"]
}
```这在所有组件路径字段中一致适用。

---

## 路径解析规则（关键）

### 代理必须使用显式文件路径

验证器**不接受“代理”的目录路径**。

即使以下操作也会失败：```json
{
  "agents": ["./agents/"]
}
```相反，您必须显式枚举代理文件：```json
{
  "agents": [
    "./agents/planner.md",
    "./agents/architect.md",
    "./agents/code-reviewer.md"
  ]
}
```这是验证错误的最常见来源。

### 命令和技能

* `commands` 和 `skills` 接受目录路径 **仅当包装在数组中时**
* 显式文件路径是最安全且最面向未来的

---

## 验证器行为注释

* `claude 插件验证`比一些市场预览更严格
* 如果路径不明确，验证可能在本地通过，但在安装过程中失败
* 错误通常是通用的（“无效输入”）并且不表明根本原因
* 跨平台安装（尤其是 Windows）对路径假设的宽容度较低

假设验证者是敌对的并且是字面意思的。

---

## `hooks` 字段：请勿添加

> ⚠️ **关键：** 不要在 `plugin.json` 中添加 `"hooks"` 字段。这是通过回归测试强制执行的。

### 为什么这很重要

Claude Code v2.1+ **按照约定从任何已安装的插件自动加载** `hooks/hooks.json`。如果您还在“plugin.json”中声明它，您将得到：```
Duplicate hooks file detected: ./hooks/hooks.json resolves to already-loaded file.
The standard hooks/hooks.json is loaded automatically, so manifest.hooks should
only reference additional hook files.
```### 触发器的历史

这导致此存储库中出现重复的修复/恢复周期：

|提交 |行动|触发|
|--------|--------|---------|
| `22ad036` |添加挂钩 |用户报告“hooks 未加载” |
| `a7bc5f2` |移除挂钩 |用户报告“重复钩子错误”(#52) |
| `779085e` |添加挂钩 |用户报告“代理未加载”(#88) |
| `e3a1306` |移除挂钩 |用户报告“重复钩子错误”(#103) |

**根本原因：** Claude Code CLI 更改了版本之间的行为：
- v2.1 之前：需要明确的 `hooks` 声明
- v2.1+：按惯例自动加载，重复时出错

### 当前规则（通过测试强制执行）

“tests/hooks/hooks.test.js”中的测试“plugin.json 没有显式的钩子声明”可防止重新引入此内容。

**如果您要添加其他挂钩文件**（不是 `hooks/hooks.json`），则可以声明这些文件。但不能声明标准的 `hooks/hooks.json`。

---

## 已知的反模式

这些看起来正确但被拒绝：

* 字符串值而不是数组
* `agents` 的目录数组
* 缺少“版本”
* 依赖推断路径
* 假设市场行为符合本地验证
* **添加 `"hooks": "./hooks/hooks.json"`** - 按约定自动加载，导致重复错误

避免聪明。明确一点。

---

## 最小已知良好示例```json
{
  "version": "1.1.0",
  "agents": [
    "./agents/planner.md",
    "./agents/code-reviewer.md"
  ],
  "commands": ["./commands/"],
  "skills": ["./skills/"]
}
```该结构已针对 Claude 插件验证器进行了验证。

**重要提示：**注意没有“hooks”字段。按照惯例，`hooks/hooks.json` 文件会自动加载。显式添加它会导致重复错误。

---

## 对贡献者的推荐

在提交涉及“plugin.json”的更改之前：

1.为代理使用显式文件路径
2. 确保所有组件字段都是数组
3. 包含一个“版本”
4. 运行：```bash
claude plugin validate .claude-plugin/plugin.json
```如果有疑问，请选择冗长而不是方便。

---

## 为什么这个文件存在

该存储库被广泛分叉并用作参考实现。

在这里记录验证器的怪癖：

* 防止重复出现问题
* 减少贡献者的挫败感
* 随着生态系统的发展保持插件的稳定性

如果验证器发生变化，请先更新此文档。
