# Video Player Component 重构文档

## 📁 文件结构

```
Video/
├── index.tsx                 # 主组件入口
├── style.ts                  # 样式定义
├── README.md                 # 文档说明
├── components/               # 子组件
│   ├── index.ts             # 组件统一导出
│   └── PlayerStates.tsx     # 播放器状态展示组件
├── hooks/                   # 自定义 Hooks
│   └── useXgPlayer.ts       # xgplayer 播放器管理 Hook
└── utils/                   # 工具函数
    ├── index.ts             # 工具函数统一导出
    ├── i18n.ts              # 国际化配置和文本管理
    └── fullscreenStyles.ts  # 全屏样式注入工具
```

## 🔧 架构设计

### 主组件 (`index.tsx`)
- **职责**: UI 布局和数据流管理
- **特点**: 简洁清晰，专注于组件组合和属性传递
- **行数**: 从 375 行精简到 138 行

### 自定义 Hook (`hooks/useXgPlayer.ts`)
- **职责**: 播放器初始化、事件监听、状态管理
- **特点**: 复用性强，逻辑内聚
- **功能**:
  - 播放器生命周期管理
  - 容器尺寸检测和重试机制
  - 全屏事件处理
  - 错误处理和状态更新

### 状态组件 (`components/PlayerStates.tsx`)
- **职责**: 不同播放器状态的 UI 展示
- **特点**: 纯展示组件，易于测试
- **状态类型**:
  - 加载中状态
  - 错误状态
  - 文件加载状态
  - 无效文件状态

### 国际化工具 (`utils/i18n.ts`)
- **职责**: 多语言支持和文本管理
- **特点**: 类型安全，自动语言检测
- **功能**:
  - 中英文语言包加载
  - 浏览器语言自动检测
  - 类型安全的文本键管理

### 样式工具 (`utils/fullscreenStyles.ts`)
- **职责**: 全屏样式注入和管理
- **特点**: 避免重复注入，全局样式统一管理

## 🚀 重构优势

### 1. 代码可读性提升
- **分离关注点**: 每个模块职责单一，易于理解
- **减少嵌套**: 主组件逻辑扁平化
- **命名清晰**: 文件和函数名称语义化

### 2. 可维护性增强
- **模块化**: 功能拆分为独立模块，便于维护
- **松耦合**: 模块间依赖清晰，修改影响范围可控
- **易扩展**: 新功能可以独立添加而不影响现有代码

### 3. 可测试性改进
- **单元测试友好**: 每个模块可以独立测试
- **Mock 容易**: Hook 和工具函数易于模拟
- **覆盖率提高**: 逻辑分离后测试覆盖更全面

### 4. 开发体验优化
- **类型安全**: TypeScript 类型定义完善
- **智能提示**: 统一导出提供更好的 IDE 支持
- **调试方便**: 模块化后问题定位更精确

## 🎯 使用方式

### 基本用法保持不变
```tsx
<VideoPreview
  data={{ file_id: "xxx", file_name: "video.mp4" }}
  onDownload={() => {}}
  // ... 其他属性
/>
```

### 内部模块可独立使用
```tsx
// 使用播放器 Hook
const { isLoading, playerReady } = useXgPlayer({
  fileUrl,
  fileId,
  containerRef,
})

// 使用国际化文本
const text = getLocalizedText("loading")

// 注入全屏样式
injectFullscreenStyles()
```
