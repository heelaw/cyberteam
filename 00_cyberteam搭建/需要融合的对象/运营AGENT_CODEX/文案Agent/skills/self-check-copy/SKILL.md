# 文案自检器

## 作用

文案交付前的最后一道质量门禁。

不是评好坏，而是逐项核对七问，强制识别风险，输出通过或回退决策。

## 启动前

- 必须有一版可检查的文案。
- 必须知道主写对象、卖点、场景是什么。
- 如果不知道，回对应 Skill 补齐后再来自检。

## 七问顺序

1. 写给谁？
2. 说了什么卖点？
3. 场景对不对？
4. 结构对不对？
5. 有没有明确动作？
6. 证据够不够？
7. 有没有夸大或硬承诺？

## SOP

1. 确认主写对象是否清楚（不是"所有人"，不是"目标用户"这种空标签）。
2. 确认主卖点是否是用户能感知的结果（不是功能名词，不是口号）。
3. 确认长度、介质、阅读状态是否和场景匹配。
4. 确认结构是否符合当前说服目标（不是把需要证据的写成纯情绪）。
5. 确认 CTA 是否具体、可执行、和场景匹配（不是同时催三件事）。
6. 确认是否有事实、案例、数据、过程或边界说明（不是空话代替证据）。
7. 确认是否写了无法保证的结果或绝对化表达。

## 七问怎么查

### 1. 写给谁

- 是否已经选出主写对象。
- 是否混了多个互相冲突的人群。
- 是否把辅助对象误写成主对象。

### 2. 说了什么卖点

- 是否说的是用户能感知的结果，而不是功能名词。
- 是否把主卖点和辅助卖点混在一起。
- 是否把口号当卖点。

### 3. 场景对不对

- 长度、介质、阅读状态是否匹配。
- 是否在短场景里塞了长解释。
- 是否忽略平台预设。

### 4. 结构对不对

- 是否符合当前说服目标。
- 是否把需要证据的内容写成了纯情绪。
- 是否把结构顺序写反。

### 5. 有没有明确动作

- 是否知道用户下一步要干什么。
- CTA 是否具体、可执行、和场景匹配。
- 是否同时催了多个动作。

### 6. 证据够不够

- 是否有事实、案例、数据、过程或边界说明。
- 是否用空话代替证据。
- 是否把推断包装成事实。

### 7. 有没有夸大或硬承诺

- 是否写了无法保证的结果。
- 是否把条件、限制和适用范围说清楚。
- 是否出现绝对化表达。

## 通过规则（7/7 Gate）

- 主写对象清楚。
- 主卖点清楚。
- 场景清楚。
- 结构完整。
- 动作明确。
- 证据足够。
- 风险被标出。
- **7 项全部满足才可通过。**

## 评分法

- 每问 1 分，共 7 分。
- **7 分：直接通过。**
- 6 分及以下：回退补齐。

## 回退路径

| 问题 | 回退到 |
|------|--------|
| 对象不清 | `segment-users` |
| 卖点不清 | `research-sellpoint-forward` 或 `research-sellpoint-backward` |
| 场景不清 | `judge-scenario` |
| 结构不清 | `select-structure` |
| 文字太虚 | `edit-copy` |
| 证据不够 | `edit-copy` 补证据 |
| 还不闭环 | 继续补证据，不直接交付 |

## 回退优先级

1. 对象不清先修对象。
2. 卖点不清先修卖点。
3. 场景不清先修场景。
4. 结构不清先修结构。
5. 动作不清先修动作。
6. 证据不清先补证据。
7. 风险不清先补风险。

## 交付骨架

- 先给判断（通过/回退）。
- 再给风险点（逐项列出）。
- 再给修改建议（可执行）。
- 最后给是否通过（明确）。

## 量化门禁

- 必须逐项检查 7 问，不能只给总评。
- 必须指出至少 1 个具体问题或风险。
- 必须给出可执行修改建议。
- 必须明确是否通过。
- 如果有任何硬承诺或边界缺失，默认不通过。
- **必须 7 项全部为真才可通过。**

## 通过判定

- 7 项全部为真且无硬承诺 → **pass**。
- 4-6 项为真 → **fail but repairable**。
- 0-3 项为真 → **fail**。

## 一票否决

- 只给"整体不错"之类的空评语。
- 不写风险点。
- 不给回退路径。
- 把硬承诺藏在措辞里。
- 把辅助对象当主对象。

## 新人校验

- 新人能不能逐项核对七问，而不是只给总评。
- 新人能不能指出具体问题而不是说"还行"。
- 新人能不能给出可执行修改建议。
- 新人能不能明确说出通过还是不通过。
- 新人能不能识别硬承诺和夸大表达。

## 常见漏项

- 只检查总感觉，没有逐项核对。
- 只看文案好不好看，没有看证据和风险。
- 忘记检查 CTA 是否和场景匹配。
- 忘记检查是否夸大承诺。
- 忘记检查是否把辅助对象写成主对象。

## 绝对不能

- 不能只给总评不逐项核对。
- 不能把风险藏起来。
- 不能把不能证明的东西判成通过。
- 不能忽略 CTA 和场景匹配。
- 不能把"感觉不错"当成通过标准。
- 不能把辅助对象误判为主对象。
- **不能把 6 分当成通过。**
