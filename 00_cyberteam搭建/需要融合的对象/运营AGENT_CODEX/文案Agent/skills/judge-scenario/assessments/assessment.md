# 考核标准

- 是否识别渠道与介质。
- 是否识别阅读时长和用户状态。
- 是否能指导长短文案选择。
- 是否明确平台预设、CTA 强弱和证据深度。

## 量化门禁

- 必须明确平台或触点类型。
- 必须明确阅读时长或注意力窗口。
- 必须给出 CTA 强度判断。
- 必须指出证据深度要求。
- 必须给出平台预设或混合渠道拆分建议。
- 不能只说“短/中/长”，必须落到平台或触点。

## 一票否决补充

- 在短触点里写长解释。
- 在长决策里只写口号。
- 不看用户接收状态就定长度。
- 混合渠道时不拆分主渠道和辅渠道。

# QA测试库

## 测试用例1：抖音短视频（<60秒）

**输入**：
- 渠道：抖音短视频
- 用户接收状态：被动刷到（首页推荐）
- 内容目的：引流到直播间

**预期输出**：
- 场景标签：短触点-抖音-引流型
- 平台预设：抖音短视频引流直播间
- 长度建议：≤60秒
- CTA强度：强（点击头像进直播间，前3秒必须有钩子）
- 表达节奏：冲突/结果→钩子→行动指引

**验收标准**：
- PASS：判断为短触点，CTA时机标注"前3秒"
- FAIL：判为中触点或长触点
- FAIL：没有标注钩子出现时机

## 测试用例2：小红书图文种草

**输入**：
- 渠道：小红书图文
- 用户接收状态：主动浏览（发现页）
- 内容目的：引发收藏和评论

**预期输出**：
- 场景标签：中触点-小红书-种草型
- 平台预设：小红书图文，默认中触点，场景共鸣优先
- 长度建议：20-90秒阅读
- CTA强度：中强（收藏/评论/关注，不直接硬推购买）
- 表达节奏：场景共鸣→个人体验→结论→行动入口

**验收标准**：
- PASS：判断为中触点，CTA标注"中强"而非"强"
- FAIL：判为短触点（字数和互动量需要中触点）
- FAIL：CTA标注"强"（不适合小红书生态）

## 测试用例3：微信公众号服务号（深度文章）

**输入**：
- 渠道：微信公众号服务号
- 用户接收状态：推送通知打开
- 内容目的：让读者预约演示

**预期输出**：
- 场景标签：中长触点-公众号-服务号-预约型
- 平台预设：公众号服务号，允许先解释再推进
- 长度建议：2-6分钟
- CTA强度：中等（预约演示/领取资料，不是直接购买）
- 表达节奏：入口建立→解释→案例→证明→行动

**验收标准**：
- PASS：判断为中长触点而非短触点
- FAIL：判为短触点（服务号推送打开的用户有深度阅读意愿）
- FAIL：CTA直接给"立即购买"

## 测试用例4：B2B官网落地页

**输入**：
- 渠道：B2B官网Hero区
- 用户接收状态：自然搜索落地（主动搜索词带入）
- 内容目的：获取MQL（市场认可线索）

**预期输出**：
- 场景标签：短触点-B2B官网-Hero
- 平台预设：B2B官网，决策触点
- 长度建议：3-8秒（Hero区注意力）
- CTA强度：强（获取方案/预约演示）
- 表达节奏：Headline（谁的什么问题+什么结果）→证据→CTA

**验收标准**：
- PASS：判断为短触点（Hero区注意力极短）
- FAIL：Headline写成产品功能描述（而不是受众+结果）
- FAIL：没有处理信任和风险（决策触点必须处理）

## 测试用例5：混合渠道（朋友圈+详情页同发）

**输入**：
- 主渠道：微信朋友圈
- 辅渠道：品牌详情页
- 内容目的：同一素材同时投放

**预期输出**：
- 主渠道判断：短触点（10秒内阅读）
- 辅渠道判断：中触点（详情页用户有深度意图）
- 混合处理：首屏按主渠道（最短窗口），正文按辅渠道（最深需求）
- CTA处理：主渠道CTA优先，辅渠道CTA作为辅助动作

**验收标准**：
- PASS：明确区分主辅渠道，并给出不同的长度和CTA建议
- FAIL：只给一个场景判断，没有区分主辅
- FAIL：首屏按辅渠道的中段处理（超出主渠道阅读窗口）

## 测试用例6：详情页首屏 vs 中段

**输入**：
- 渠道：产品详情页
- 用户接收状态：刚打开详情页（入口）
- 内容目的：让用户判断值不值得继续看

**预期输出**：
- 首屏判断：短触点（5-15秒注意力）
- 中段判断：中触点（30-120秒注意力）
- 首屏节奏：核心主张+1-2个关键证据+强CTA
- 中段节奏：卖点展开+证据+FAQ+反对意见

**验收标准**：
- PASS：拆分首屏和中段，分别给出不同的触点判断
- FAIL：把整个详情页当成单一触点处理
- FAIL：首屏开始铺长背景（超短注意力窗口）

## 测试用例7：用户状态=被动刷到（抖音信息流）

**输入**：
- 渠道：抖音信息流广告
- 用户接收状态：被动刷到（首页推荐）
- 内容目的：让用户点击购买

**预期输出**：
- 场景标签：短触点-抖音信息流-被动型
- 钩子要求：前3秒必须有冲突/利益/好奇点
- CTA时机：不能放最后一屏，在视频中段（5-10秒）就要出现
- 视频-落地页一致性：视频卖点和落地页核心动作必须一致

**验收标准**：
- PASS：判断为短触点，钩子时机标注"前3秒"，CTA不放在结尾
- FAIL：开头3秒没有强制要求感知点
- FAIL：CTA放在最后一屏才出现

## 测试用例8：强意愿用户（私信对话）

**输入**：
- 渠道：微信私信
- 用户接收状态：用户主动问价并问规格（强意愿）
- 内容目的：直接给方案并约成单

**预期输出**：
- 场景标签：决策触点-私信-强意愿型
- 话术节奏：直接给方案+结果+最低门槛动作
- CTA强度：直接（发购买链接/约时间/发地址）
- 不需要：建立认知、长背景解释

**验收标准**：
- PASS：判断为决策触点，直接推方案和动作
- FAIL：对强意愿用户还在做认知教育（浪费时机）
- FAIL：给过多选项造成选择瘫痪（强意愿用户要减少决策摩擦）

## 测试用例9：弱意愿用户（私信对话）

**输入**：
- 渠道：微信私信
- 用户接收状态：用户3天前加好友，没有说过话（弱意愿）
- 内容目的：激活对话

**预期输出**：
- 场景标签：中触点-私信-弱意愿激活型
- 话术节奏：先钩子（对方感兴趣的内容），再了解需求，不能硬推
- CTA强度：弱（引发回复的问题，不直接要结果）
- 禁忌：不能直接推产品、不能催问"考虑得怎么样了"

**验收标准**：
- PASS：判断为中触点，给出发起对话的钩子问题
- FAIL：对弱意愿用户直接推产品（引发反感）
- FAIL：问"考虑得怎么样了"（给压力但没有价值）

## 测试用例10：场景与结构冲突

**输入**：
- 渠道：微信公众号（深度文章）
- 用户接收状态：主动搜索关键词进来的（已有一定认知）
- 文案结构（误用）：选了AIDA

**场景-结构冲突分析**：
- 公众号深度文章：本质是"解释+证明"结构
- AIDA：注意→兴趣→欲望→行动，节奏快，不适合深度解释

**正确输出**：
- 场景判断：中长触点
- 结构建议：优先解释→证明结构（或4P），不适合AIDA
- 理由：公众号用户有认知基础但需要理解深度，不需要AIDA的层层推进加速感

**验收标准**：
- PASS：指出AIDA不适合此场景，给出替代结构
- FAIL：不管场景直接套AIDA
- FAIL：没有说明为什么选这个结构
