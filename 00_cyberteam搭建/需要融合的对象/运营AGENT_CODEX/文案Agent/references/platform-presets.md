# 平台预设

## 目的

把抽象的“短 / 中 / 长”映射成可直接执行的平台预设，避免场景判断停留在空层。

## 快速映射

| 平台 / 触点 | 阅读窗口 | 表达重心 | CTA 强度 |
|---|---|---|---|
| 海报 / 广告位 | 3-8 秒 | 一个主张 + 一个利益点 | 很强，但只能一个动作 |
| 抖音短视频脚本 | 15-60 秒 | 钩子、冲突、结果、证据 | 强，前 3 秒必须有入口 |
| 小红书图文 | 20-90 秒 | 场景、体验、细节、对比 | 中强，先共鸣后动作 |
| 公众号文章 | 2-6 分钟 | 逻辑、解释、案例、边界 | 中等，先理解后转化 |
| 详情页首屏 | 5-15 秒 | 主卖点、核心证据、下一步 | 强，首屏要完成判断 |
| 详情页中段 | 30-120 秒 | 证据展开、FAQ、反对意见 | 中等，服务理解和信任 |
| 私信 / 销售话术 | 对话式 | 问题、判断、定制回应 | 按对方状态动态调整 |
| 长文 / 教育内容 | 3-10 分钟 | 认知重构、结构化解释 | 弱到中，先建立信任 |

## 混合触点

- 详情页、长文和公众号正文常常是混合触点：标题 / 首屏按短触点处理，正文按中长触点处理。
- 短视频标题、封面和首句按短触点处理，口播后半段再按中触点处理。
- 如果同一载体同时服务唤起和促成，先做最短入口，再做中段证据。

## 极端场景

- 高风险：先写边界、风险和适用条件，再写主张。
- 高时效：先写结论和动作，压缩解释链。
- 高争议：先写证据和对比，再写行动。
- 高情绪：先共鸣，再解释，再推进。

## 用法

1. 先定平台，再定阅读窗口。
2. 再决定是短触点、中触点，还是长触点。
3. 如果平台本身限制很强，优先服从平台，而不是服从内容完整性。
4. 如果一个平台有两个阶段，标题 / 首屏 / 开头按短触点处理，正文按中长触点处理。
5. 极端场景下，CTA 强度和证据深度要重新校准，不能沿用普通模板。

## 典型落地

- 小红书：用生活场景、对比和细节拉近感知，少讲抽象定义。
- 抖音：先钩子，再冲突，再结果，最后给动作。
- 公众号：允许先解释，再证明，再推进。
- 详情页：首屏先给判断，中段再给证据，最后给风险说明。
- 海报：只讲一个最强判断，不塞第二个卖点。
