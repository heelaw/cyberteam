# 反脆弱决策专家测试用例

## 正面测试

### 测试1: 业务连续性方案脆弱性分析

- **输入**:
```
【反脆弱分析请求】

分析对象：电商平台服务器架构
当前状态：单服务器部署，无备份
核心目标：确保业务连续性
约束条件：预算有限，需要在10万以内
```
- **期望输出**:
  - 识别出单点故障风险
  - 给出冗余备份方案
  - 提供杠铃策略配置（保守层：基础冗余；激进层：云原生弹性架构）
  - 包含压力测试场景
  - 给出反脆弱指数评分
- **验证项**:
  - [ ] 识别了结构脆弱（单点故障）
  - [ ] 给出了冗余机制建议
  - [ ] 包含杠铃策略配置（保守+激进）
  - [ ] 有底线思维（明确最坏情况损失边界）
  - [ ] 有反脆弱指数（1-10）

---

### 测试2: 投资组合反脆弱设计

- **输入**:
```
【反脆弱分析请求】

分析对象：个人投资组合
当前状态：100%配置在单一科技股
核心目标：在市场波动中保值增值
可选：年轻投资者，30年投资周期
```
- **期望输出**:
  - 识别预测脆弱和优化脆弱
  - 给出杠铃式资产配置
  - 可选择性分析（多种资产类别）
  - 黑天鹅防御方案
- **验证项**:
  - [ ] 识别了预测脆弱（依赖单一股票预测）
  - [ ] 杠铃配置包含极度保守层（如国债、货币基金）
  - [ ] 杠铃配置包含极度激进层（如高增长资产）
  - [ ] 明确禁止中等风险区域
  - [ ] 可选择性评估完整

---

### 测试3: 产品发布反脆弱策略

- **输入**:
```
【反脆弱分析请求】

分析对象：新软件产品发布计划
当前状态：计划3个月后全面上市
核心目标：降低发布风险
约束条件：团队资源有限
```
- **期望输出**:
  - 识别脆弱性维度
  - 设计分阶段发布机制
  - 杠铃策略配置（小规模测试vs全面推广）
  - 快速修复机制
  - 黑天鹅应对
- **验证项**:
  - [ ] 识别了传导脆弱（失败影响扩大）
  - [ ] 设计了渐进式发布策略
  - [ ] 有压力测试场景
  - [ ] 有逆向获益方案
  - [ ] 输出包含完整的分析报告格式

## 负面测试

### 测试1: 过度保守陷阱

- **输入**:
```
【反脆弱分析请求】

分析对象：初创公司技术架构
当前状态：所有服务本地部署
核心目标：稳定可靠
约束条件：为了绝对稳定不考虑云服务
```
- **期望行为**: 输出应指出这种"过度保护"反而会削弱反脆弱能力，因为消除了波动和学习机会
- **验证项**:
  - [ ] 指出过度保守的问题
  - [ ] 解释为什么消除波动会削弱能力
  - [ ] 提供平衡建议（适度冗余+适度风险暴露）

---

### 测试2: 虚假预测依赖

- **输入**:
```
【反脆弱分析请求】

分析对象：明年市场策略
当前状态：依赖第三方市场预测报告做决策
核心目标：降低不确定性
```
- **期望行为**: 输出应指出依赖精确预测的脆弱性，建议建立抗预测结构
- **验证项**:
  - [ ] 识别预测脆弱
  - [ ] 批判依赖预测的思维
  - [ ] 提供可选择性设计作为替代

---

### 测试3: 线性外推陷阱

- **输入**:
```
【反脆弱分析请求】

分析对象：用户增长策略
当前状态：基于过去3年线性增长做预算规划
核心目标：规划明年资源
```
- **期望行为**: 输出应指出线性思维的脆弱性，识别极端情景的可能性
- **验证项**:
  - [ ] 识别优化脆弱（过度依赖历史线性外推）
  - [ ] 指出非线性风险
  - [ ] 提供压力测试场景

---

## 边界测试

### 测试1: 极小样本分析

- **输入**: 数据极其有限（仅2-3个数据点）
- **期望行为**: 识别数据不足带来的预测脆弱，建议用杠铃策略应对
- **验证项**:
  - [ ] 识别数据不足的脆弱性
  - [ ] 不强行做精确预测
  - [ ] 提供在小数据下可行的反脆弱策略
