# 邓巴数字专家测试用例

## 正面测试

### 测试1: 团队管理规模
- 输入:
```
【邓巴数字分析请求】
场景：团队管理
当前规模：30人
目标：提升管理效率
资源限制：管理者1人
```
- 期望输出:
  - 社交层级诊断
  - 邓巴压力评估
  - 优化策略
  - 行动清单
- 验证项:
  - [ ] 规模与邓巴数字对比
  - [ ] 分层管理建议具体
  - [ ] 有预警信号

### 测试2: 社群运营规模
- 输入:
```
【邓巴数字分析请求】
场景：社群运营
当前规模：200人
目标：保持活跃度
资源限制：运营人员2人
```
- 期望输出:
  - 层级分布诊断
  - 邓巴压力评估
  - 分层运营策略
- 验证项:
  - [ ] 识别过载信号
  - [ ] 分层策略具体
  - [ ] KOL结构建议有效

## 负面测试

### 测试1: 贪多心态
- 输入:
```
【邓巴数字分析请求】
场景：人脉管理
当前规模：50人
目标：扩展到300人
资源限制：时间有限
```
- 期望行为: 不能鼓励盲目扩展，要尊重认知极限
- 验证项:
  - [ ] 警告数量增长的代价
  - [ ] 强调质量>数量
  - [ ] 分层维护策略

### 测试2: 平均分配精力
- 输入:
```
【邓巴数字分析请求】
场景：个人关系维护
当前规模：各层级人数平均
目标：提升关系质量
资源限制：精力有限
```
- 期望行为: 不能建议平均分配精力，要分层维护
- 验证项:
  - [ ] 区分层级重要性
  - [ ] 精力分配不平均
  - [ ] 维护频率因层级而异
