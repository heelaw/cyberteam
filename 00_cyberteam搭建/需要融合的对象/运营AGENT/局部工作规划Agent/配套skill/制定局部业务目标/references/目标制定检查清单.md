# 目标制定检查清单

## 一、目标来源检查

### □ 战略承接检查
- [ ] 理解公司战略目标和核心诉求
- [ ] 明确上级部门/领导的具体期望
- [ ] 了解公司当前的核心KPI和关注点
- [ ] 知道目标与公司战略的关联关系

### □ 业务模型关联检查
- [ ] 明确业务所处阶段（探索/成长/成熟/转型）
- [ ] 理解核心业务模型（AARRR、增长飞轮等）
- [ ] 知道局部业务在整体模型中的位置
- [ ] 能说明目标与业务模型的关联

### □ 环境分析检查
- [ ] 分析了市场趋势和竞争态势
- [ ] 了解了用户需求变化
- [ ] 评估了技术/政策等外部因素影响
- [ ] 识别了机会和威胁

---

## 二、目标质量检查（SMART原则）

### □ Specific（具体）
- [ ] 目标描述清晰，明确要达成什么
- [ ] 不使用"提升"、"加强"等模糊词汇
- [ ] 有明确的业务含义

### □ Measurable（可衡量）
- [ ] 有明确的衡量指标
- [ ] 指标有计算公式或明确口径
- [ ] 有基线数据作为参考

### □ Achievable（可实现）
- [ ] 评估了历史增长趋势
- [ ] 考虑了现有资源和能力
- [ ] 与执行团队确认过可行性
- [ ] 区分了"挑战目标"和"保底目标"

### □ Relevant（相关）
- [ ] 目标与公司战略方向一致
- [ ] 目标与上级目标有支撑关系
- [ ] 目标与业务本质需求相关
- [ ] 不是为做而做

### □ Time-bound（有时限）
- [ ] 有明确的开始和结束时间
- [ ] 有里程碑节点
- [ ] 时间设置合理（不过于紧张也不过于宽松）

---

## 三、衡量指标检查

### □ 指标完整性
- [ ] 每个目标有至少2个衡量指标
- [ ] 指标覆盖不同维度（量级、效率、质量）
- [ ] 能区分领先指标和滞后指标

### □ 指标有效性
- [ ] 指标能真实反映目标达成情况
- [ ] 指标数据可获取
- [ ] 指标口径明确，不会产生歧义
- [ ] 指标之间不重复、不矛盾

### □ 指标值合理性
- [ ] 目标值有依据（历史增长、行业对比、竞品分析）
- [ ] 增长率/改善幅度合理
- [ ] 考虑了资源投入的可能

---

## 四、对齐确认检查

### □ 垂直对齐（上下对齐）
- [ ] 与上级/老板确认过目标
- [ ] 与下属团队沟通过目标
- [ ] 各级目标之间有清晰的支撑关系

### □ 水平对齐（跨部门对齐）
- [ ] 识别了相关的上下游部门
- [ ] 与相关方沟通过协作需求
- [ ] 识别并处理了潜在冲突
- [ ] 有书面的对齐记录

### □ 对齐闭环
- [ ] 目标获得相关方认可
- [ ] 有签字确认或邮件确认
- [ ] 对齐过程中有反馈和调整

---

## 五、资源与约束检查

### □ 资源评估
- [ ] 评估了人力需求与现有人员能力
- [ ] 评估了预算需求与现有预算
- [ ] 评估了技术/工具需求
- [ ] 识别了资源缺口

### □ 约束识别
- [ ] 明确了时间约束
- [ ] 明确了政策/合规约束
- [ ] 明确了组织架构/流程约束
- [ ] 有应对约束的预案

---

## 六、追踪机制检查

### □ 追踪计划
- [ ] 确定了追踪频率（周/月/季）
- [ ] 确定了数据来源和获取方式
- [ ] 确定了追踪责任人和汇报机制

### □ 复盘安排
- [ ] 设定了阶段性复盘节点
- [ ] 明确了复盘参与者和议程
- [ ] 建立了调整触发条件和流程

---

## 七、常见问题自查

### 问题1: 目标太多太杂
**检查**：
- [ ] 目标数量是否超过5个？
- [ ] 是否有轻重缓急排序？
- [ ] 核心目标是否突出？

**调整建议**：聚焦最重要的3个目标

### 问题2: 目标依赖外部因素
**检查**：
- [ ] 目标是否依赖无法控制的外部因素？
- [ ] 是否设定了可控的行动目标？

**调整建议**：将外部依赖目标转化为内部行动目标

### 问题3: 目标表述模糊
**检查**：
- [ ] 能否用一句话说清目标？
- [ ] 别人能否理解你的目标？
- [ ] 能否说清如何衡量达成？

**调整建议**：用具体、可衡量的语言重新表述

---

## 八、最终确认

在提交/发布目标前，确认以下事项：

- [ ] 目标经过深思熟虑，不是拍脑袋
- [ ] 目标与战略保持一致
- [ ] 目标符合SMART原则
- [ ] 有足够的指标来衡量
- [ ] 与相关方进行了对齐
- [ ] 资源条件能够支撑
- [ ] 有明确的追踪机制
