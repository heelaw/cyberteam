# 规划模型对比表

## 一、OKR模型

### 核心概念
- **O（Objective）**：目标，回答"我们要去哪里"
- **KR（Key Results）**：关键结果，回答"如何衡量是否到达"

### 适用场景
- 探索期业务，需要聚焦方向
- 转型期业务，需要突破创新
- 扁平化组织，强调自驱动

### 优势
1. 聚焦重点，鼓励挑战
2. 保持灵活性，适应变化
3. 促进上下对齐，透明公开
4. 激发创新，突破常规

### 局限
1. 难以精确衡量，依赖主观判断
2. 需要高度自驱，文化要求高
3. 落地执行需要大量沟通
4. 不适合强KPI导向的团队

### 实施要点
- 每个周期（季度）设定3-5个O
- 每个O对应3-5个KR
- KR必须可衡量，有明确目标值
- 周期结束时进行评分（0-1分）

---

## 二、KPI模型

### 核心概念
- 关键绩效指标
- 结果导向，指标明确
- 与绩效挂钩

### 适用场景
- 成熟期业务，追求效率稳定
- 流程化运作，指标明确
- 需要精确评估绩效的团队

### 优势
1. 指标清晰，易于追踪
2. 与绩效挂钩，激励明确
3. 便于横向比较
4. 执行方向明确

### 局限
1. 可能导致短期主义
2. 指标僵化，难以适应变化
3. 忽略过程和突破
4. 创新空间受限

### 实施要点
- 设定3-7个核心KPI
- 指标必须有明确定义和计算方式
- 目标值要经过可行性验证
- 定期检视指标有效性

---

## 三、SMART原则

### 核心概念
目标必须满足五要素：
- **S（Specific）**：具体
- **M（Measurable）**：可衡量
- **A（Achievable）**：可实现
- **R（Relevant）**：相关
- **T（Time-bound）**：有时限

### 适用场景
- 任何目标设定
- 个人/团队/部门目标
- 短期/中期/长期目标

### 实施检查清单
- [ ] 目标描述是否具体？
- [ ] 如何衡量目标是否达成？
- [ ] 目标是否可实现？
- [ ] 目标与战略是否相关？
- [ ] 目标是否有明确时限？

---

## 四、PDCA循环

### 核心概念
- **Plan**：计划
- **Do**：执行
- **Check**：检查
- **Act**：改进

### 适用场景
- 持续改进的工作
- 运营优化类工作
- 需要循环迭代的项目

### 优势
1. 强调持续改进
2. 形成闭环管理
3. 便于经验沉淀
4. 适应动态调整

### 局限
1. 迭代周期难把控
2. 可能过于形式化
3. 不适合创新探索
4. 改进效果可能递减

---

## 五、模型选择决策树

```
业务阶段是什么？
    │
    ├─ 探索期 → 选择OKR
    │
    ├─ 成长期 → 选择KPI + OKR组合
    │           （运营稳定用KPI，突破目标用OKR）
    │
    ├─ 成熟期 → 选择KPI
    │           （追求效率，指标明确）
    │
    └─ 转型期 → 选择OKR
                （需要聚焦新方向）
```

## 六、模型混用注意事项

### 禁止行为
1. 同一层级同一时间只用一种核心模型
2. 不要同时推行OKR和KPI导致混乱
3. 不要用KPI考核OKR目标

### 推荐做法
1. 主模型+辅助工具结合
2. 不同层级可用不同模型
3. 明确区分考核和发展目标
4. 保持模型与文化的匹配

---

## 七、实践建议

### 小团队（<10人）
- 推荐模型：OKR为主
- 优势：灵活、快速对齐
- 注意：沟通成本可控

### 中型团队（10-50人）
- 推荐模型：OKR + KPI组合
- 优势：平衡创新与执行
- 注意：明确各自适用范围

### 大型团队（>50人）
- 推荐模型：KPI为主，OKR用于特定突破项目
- 优势：统一方向，易于管理
- 注意：防止形式化

---

*本文档用于帮助选择适合的规划模型，实际选择需要结合业务特点和组织文化*
