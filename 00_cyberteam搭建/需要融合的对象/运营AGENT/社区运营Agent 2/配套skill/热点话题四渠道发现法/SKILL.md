## 核心定位
通过监控四大核心渠道（微博热搜、抖音热榜、知乎热榜、贴吧论坛）发现热点话题，结合产品特性策划话题角度，通过马甲号引爆话题讨论，引导用户产出优质UGC内容。

## Critical Rules

### 必须遵守
1. **热点必须实时监控** - 不同渠道的热搜更新时间不同，必须分时段监控
2. **筛选必须结合产品** - 不是所有热点都要蹭，要选择与产品/社区相关的热点
3. **角度必须有独特性** - 避免泛泛而谈，要有独特的切入角度
4. **执行必须分优先级** - 多个热点同时发生时，必须分清主次，集中资源引爆核心话题
5. **预案必须有应变方案** - 话题发展不可预测，必须准备多种应对方案

### 禁止行为
1. **禁止盲目蹭热点** - 不能什么热点都跟，要与社区定位相符
2. **禁止政治敏感话题** - 不碰政治敏感话题，红线不能踩
3. **禁止未经核实传播** - 不传播未经证实的信息
4. **禁止强行关联** - 不能硬把热点和社区关联，用户会反感
5. **禁止单一渠道依赖** - 不能只看一个渠道，要综合四大渠道判断

## workflow_steps

### Step 1：四大渠道热点监控
**输入**：
- 监控时间段（建议早/中/晚三次）
- 社区主题关键词

**处理**：
1. 微博热搜监控
   - 热搜榜单（每2小时刷新）
   - 超级热搜榜
   - 话题排行榜
   - 筛选：娱乐/社会/民生/行业相关
2. 抖音热榜监控
   - 抖音热榜（实时更新）
   - 挑战榜
   - 搜索热词
   - 筛选：视频类/可延展话题
3. 知乎热榜监控
   - 知乎热榜（实时）
   - 行业垂直热榜
   - 筛选：深度讨论类/观点类话题
4. 贴吧论坛监控
   - 贴吧热帖
   - 豆瓣小组
   - 虎扑/天涯等
   - 筛选：社区类/垂直兴趣类

**输出**：
- 各渠道热点清单（含热度/来源/更新时间）
- 热点话题汇总表
- 初步相关度评估

**成功标准**：
- 覆盖全部四大渠道
- 每个渠道至少采集TOP20热点
- 标注每个热点的来源和热度

### Step 2：热点筛选与关联分析
**输入**：
- 各渠道热点清单
- 社区主题和用户画像
- 当前运营目标

**处理**：
1. 相关度评估
   - 与社区主题的相关程度
   - 与目标用户的匹配程度
   - 与当前运营目标的契合度
2. 热度评估
   - 话题所处生命周期（爆发期/上升期/平稳期）
   - 话题持续性判断（昙花一现/持续发酵）
   - 话题延展性（能否多角度切入）
3. 风险评估
   - 政治敏感风险
   - 舆论反转风险
   - 平台规则风险
4. 优先级排序
   - P0：必须跟（高相关+高热度+低风险）
   - P1：可以跟（中高相关+中热度）
   - P2：观察（相关度一般但有潜力）

**输出**：
- 筛选后的重点话题清单（含优先级）
- 各话题的详细分析报告
- 话题风险评估

**成功标准**：
- 筛选标准明确可解释
- P0话题不超过3个
- 每个话题有清晰的相关度说明

### Step 3：话题角度策划
**输入**：
- 重点话题清单
- 话题详细分析报告
- 社区调性和用户特征

**处理**：
1. 角度挖掘
   - 从话题中找与社区的连接点
   - 设计独特的切入角度
   - 避免泛泛而谈
2. 内容方向设计
   - 确定UGC引导方向
   - 设计参与门槛（不能太高）
   - 准备话题素材（如背景信息/讨论框架）
3. 话术设计
   - 话题发布文案
   - 引导用户参与的话术
   - 互动回复模板
4. 风险预案
   - 话题反转应对方案
   - 负面评论处理方案
   - 话题失控应急预案

**输出**：
- 话题策划方案（每个P0话题一份）
- 话题素材包
- 话题话术文案
- 风险预案

**成功标准**：
- 每个重点话题有独特角度
- 内容方向具体可执行
- 话术自然不生硬
- 风险预案完备

### Step 4：话题引爆执行
**输入**：
- 话题策划方案
- 可用资源（马甲号/推荐位/预算）
- 发布时间计划

**处理**：
1. 资源分配
   - 马甲号分工（谁发布/谁回复/谁互动）
   - 推荐位安排
   - 预算分配
2. 时间节奏
   - 预热期（发布话题前）
   - 引爆期（话题发布后2小时）
   - 持续期（话题延续）
3. 数据监控
   - 实时监控话题数据
   - 根据数据调整执行
   - 记录关键数据节点
4. 复盘总结
   - 数据复盘（参与人数/UGC产出/转化）
   - 执行复盘（好的/需改进的）
   - 方法论沉淀

**输出**：
- 执行排期表（含时间/人员/资源）
- 话题引爆执行方案
- 数据监控报表
- 复盘报告模板

**成功标准**：
- 执行方案责任到人
- 时间节点精确到分钟
- 数据监控实时准确
- 复盘报告完整
