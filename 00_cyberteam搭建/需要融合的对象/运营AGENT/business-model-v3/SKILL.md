# cyberwiz-businessmodel-v3.0

## 概述

业务模型是企业经营的核心框架，它定义了企业如何创造价值、传递价值和获取价值。本Skill基于三大商业模型（商业模式、商业经营模型、业务模型）和业务操盘手知识体系，帮助用户系统化地梳理业务全局视图，分析收入公式，映射转化链路，定义运营指标，设定北极星指标，管理探索型项目，并提供针对性的破局点建议。

## 🆕 v3.0 新特性 - 业务操盘手能力体系

### Stage 1：业务模型映射

**目标**：构建基于三大模型的全局业务视图

**执行步骤**：

1. **商业模式分析**：
   - 市场定位：目标市场、用户群体、竞争格局
   - 价值主张：核心价值、差异化优势
   - 收入模式：收入来源、定价策略
   - 渠道策略：获客渠道、分销网络

2. **商业经营模型分析**：
   - 成本结构：固定成本、可变成本、边际成本
   - 利润模型：毛利率、净利率、ROI
   - 现金流：收入周期、支出周期、资金周转率

3. **业务模型分析**：
   - 核心业务流程：价值链、关键环节
   - 资源配置：人力、技术、资金
   - 核心能力：技术优势、运营能力、品牌影响力

4. **MCP插件使用**：
   - 使用WebSearch插件参考相似产品/服务的商业模式
   - 分析行业最佳实践和成功案例

5. **输出**：全局业务模型

6. **完成后**：
   ```
   ✅ Stage 1 完成：业务模型映射
   - 商业模式：[商业模式概述]
   - 商业经营模型：[商业经营模型概述]
   - 业务模型：[业务模型概述]
   
   {执行模式提示}
   ```
   
   其中 {执行模式提示} 根据执行模式显示：
   - 自动执行模式：`🚀 自动执行中，正在进入 Stage 2（收入公式分析）...`
   - 手动执行模式：`⏸️ 是否进入 Stage 2（收入公式分析）？`

### Stage 2：收入公式分析

**目标**：拆解并定义详细的收入计算公式

**执行步骤**：

1. **收入构成拆解**：
   - 直接收入：产品销售、服务收费等
   - 间接收入：广告、佣金、数据服务等
   - 增值收入：会员费、定制化服务等

2. **收入公式构建**：
   - 核心收入公式：如收入=流量×转化率×客单价
   - 各收入来源的详细公式
   - 变量关系分析

3. **敏感度分析**：
   - 识别影响收入的关键变量
   - 分析变量变化对收入的影响程度
   - 确定核心优化方向

4. **输出**：详细收入公式

5. **完成后**：
   ```
   ✅ Stage 2 完成：收入公式分析
   - 核心收入公式：[核心收入公式]
   - 收入来源数量：[N]个
   - 关键变量：[关键变量列表]
   
   {执行模式提示}
   ```

### Stage 5：破局点建议

**目标**：基于数据分析和行业对标，提供针对性的优化建议

**执行步骤**：

1. **瓶颈识别**：
   - 基于转化链路和运营指标，识别当前业务的核心瓶颈
   - 分析瓶颈产生的根本原因
   - 评估瓶颈对整体业务的影响程度

2. **破局策略设计**：
   - 短期策略：快速提升关键指标的方法

   - 长期策略：业务模型升级和创新

3. **行业对标借鉴**：
   - 分析行业领先企业的解决方案
   - 参考相似业务的成功案例
   - 结合自身业务特点进行调整

4. **风险评估**：
   - 分析破局策略的潜在风险
   - 制定风险应对预案
   - 评估投入产出比

5. **输出**：破局点分析与建议

### v2.0 工具和框架

#### core/ - 核心引擎
- `parameter_extractor.py` - 智能参数提取器

- `validation_engine.py` - 业务模型验证引擎
  - 数学关系验证（LTV > CAC等）
  - 行业基准对比
  - 一致性检查
  - 完整性验证

#### tools/ - 增量更新工具
- `incremental_updater.py` - 增量更新器
  - 单个指标更新
  - 转化漏斗更新
  - 影响分析
  - 自动备份和变更记录

#### frameworks/ - 商业模式框架
- `o2o-preorder.yaml` - O2O预定模式框架
  - 适用场景：本地服务、高客单价定制服务
  - 核心指标：CAC $50-500、爽约率 <10%
  - 标准转化路径：7步漏斗

- `saas-subscription.yaml` - SaaS订阅模式框架
  - 适用场景：企业软件、云平台、生产力工具
  - 核心指标：MRR/ARR、Churn Rate <5%
  - 标准转化路径：5步漏斗

- `ecommerce-dtc.yaml` - 电商DTC模式框架
  - 适用场景：垂直品类电商、自有品牌电商
  - 核心指标：AOV $50-200、复购率 30-50%
  - 标准转化路径：6步漏斗

#### config/ - 配置文件
- `validator_rules.yaml` - 验证规则配置
  - 可自定义的验证规则
  - 行业基准数据

## 输入规范

### 必需输入
- **公司/产品**: 需要分析业务模型的公司或产品
- **业务目标**: 业务分析要达成的目标

### 可选输入
- **业务数据**: 现有的业务指标和运营数据
- **产品阶段**: 产品所处的发展阶段
- **参考竞品**: 相似的竞品或案例

## 输出规范

### 标准输出
- **业务模型报告**: 包含商业模式、商业经营模型、业务模型的全局视图
- **收入公式**: 详细的收入计算公式和变量分析

### 可选输出
- **转化链路图**: 完整的用户转化路径
- **运营指标体系**: 各环节的关键指标定义
- **破局点建议**: 基于分析的优化建议

## 使用流程

1. **初始化工作流**: 确认业务基本信息（产品、用户、业务阶段等）
2. **映射业务模型**: 分析商业模式、经营模型和业务模型
3. **拆解收入公式**: 构建收入计算公式，识别关键变量
4. **分析转化链路**: 绘制用户转化路径，识别关键节点
5. **定义运营指标**: 为各环节设定可衡量的指标
6. **提出破局点**: 基于分析识别业务瓶颈并给出优化建议
