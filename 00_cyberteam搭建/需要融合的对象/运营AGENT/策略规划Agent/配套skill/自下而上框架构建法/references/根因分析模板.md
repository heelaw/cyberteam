# 5Why根因分析模板

## 模板说明
用于通过连续追问"为什么"来追溯问题的根本原因。

## 模板内容

```

## 问题陈述
- **问题描述**：[用一句话准确描述问题]
- **影响范围**：[谁/什么受影响]
- **发现时间**：[何时发现]

## 5Why分析链

### Why 1（第一层追问）
- **问题**：为什么[问题现象]？
- **回答**：[直接原因]
- **追问依据**：[基于什么事实或数据]

### Why 2（第二层追问）
- **问题**：为什么[第一层回答]？
- **回答**：[第二层原因]
- **追问依据**：[基于什么事实或数据]

### Why 3（第三层追问）
- **问题**：为什么[第二层回答]？
- **回答**：[第三层原因]
- **追问依据**：[基于什么事实或数据]

### Why 4（第四层追问）
- **问题**：为什么[第三层回答]？
- **回答**：[第四层原因]
- **追问依据**：[基于什么事实或数据]

### Why 5（第五层追问）
- **问题**：为什么[第四层回答]？
- **回答**：[根本原因]
- **追问依据**：[基于什么事实或数据]

## 分析链可视化

```

    ↓ Why 1

[第二层原因]

### 验证问题
- [ ] 根因能解释第一层原因吗？

- [ ] 根因本身是事实还是假设？
- [ ] 如果根因成立，从根本上解决能防止问题再次发生吗？

## 使用说明
1. 准确描述要分析的问题
2. 每次追问基于事实，不基于假设
3. 连续追问至少5次，直到触及系统层面
4. 验证根因能解释整个链条
5. 用反例检验根因的稳健性
6. 从根因倒推解决方案
