## 二、快速验证清单

### 用户需求验证

- [ ] 用户访谈完成（至少5人）
- [ ] 目标用户画像清晰
- [ ] 需求场景描述具体
- [ ] 需求强度有分级
- [ ] 付费意愿有调研

### 功能价值验证

- [ ] 原型/流程图已确认
- [ ] 核心用户路径可用
- [ ] 数据埋点已设计
- [ ] 灰度发布策略已定
- [ ] 回滚方案已准备

### 定价验证

- [ ] 价格带已确定
- [ ] 对比产品已选
- [ ] 付费转化路径已设计
- [ ] 优惠策略已准备

### 渠道验证

- [ ] 渠道目标已明确
- [ ] 素材/内容已准备
- [ ] 数据追踪已设置
- [ ] 预算已分配
- [ ] 承接方案已准备

## 四、验证方法选择指南

### 不同场景的验证方法

| 场景 | 推荐方法 | 成本 | 周期 | 可靠性 |
|------|----------|------|------|--------|
| 用户需求 | 用户访谈 | 低 | 1-3天 | 中 |
| 用户需求 | 问卷调研 | 低 | 3-7天 | 中 |
| 付费意愿 | 价格测试 | 中 | 7-14天 | 高 |
| 功能效果 | 灰度发布 | 中 | 7-14天 | 高 |
| 转化效果 | A/B测试 | 中高 | 7-21天 | 高 |
| 渠道效果 | 小规模投放 | 中高 | 7-14天 | 高 |
| 内容效果 | 数据监测 | 低 | 3-7天 | 中 |

### 验证方法对比

| 方法 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| 访谈 | 深入、灵活 | 样本少，主观强 | 需求探索 |
| 问卷 | 样本大，可量化 | 设计要求高 | 需求验证 |
| 投票 | 快速，简单 | 不够深入 | 快速筛选 |
| 原型测试 | 真实体验 | 需原型 | 功能验证 |
| 灰度发布 | 真实环境 | 有风险 | 功能验证 |
| A/B测试 | 数据驱动 | 需流量 | 效果验证 |
