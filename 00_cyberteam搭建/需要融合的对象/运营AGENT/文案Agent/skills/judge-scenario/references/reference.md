# 场景判断

## 适用场景

- 需要决定文案长度、语气和承载方式。
- 需要区分短文案、中长文案和长文案。
- 需要判断当前介质适不适合强说服。

## 先判断四件事

1. 渠道是什么。
2. 用户在什么状态下接收。
3. 阅读时长有多长。
4. 当前更适合唤起、解释还是促成。

## 场景分层

| 场景 | 特征 | 长度 | 表达重心 |
|---|---|---|---|
| 短触点 | 注意力短、切换快 | 短 | 结论、利益点、动作 |
| 中触点 | 有一定阅读时间 | 中 | 问题、方案、证据 |
| 长触点 | 需要理解和比较 | 长 | 逻辑、证据、反对意见 |
| 决策触点 | 用户已进入比较或下单 | 视情况 | 信任、风险、行动 |

## 输出骨架

- 场景标签
- 文案长度建议
- 表达节奏建议
- 需要的证据深度
- CTA 强度
