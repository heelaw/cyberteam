# 正向卖点提炼

## 适用场景

- 已有产品资料、功能说明或竞品信息。
- 需要先找出“为什么值得买/看/点”。
- 需要在进入文案前先建立卖点池。

## 先做三件事

1. 先列事实，不急着写话术。
2. 先补证据，不急着做承诺。
3. 先分候选，不急着选唯一答案。

## 七个方向

| 方向 | 核心问题 | 怎么收集 | 常见来源 |
|---|---|---|---|
| 功能独特性 | 这个产品到底有什么独特功能 | 产品说明、版本记录、竞品对比、演示 | 产品文档、PRD、官网、demo |
| 效果承诺 | 用了以后能带来什么结果 | 案例、数据、前后对比、用户证言 | 成功案例、复盘、数据报表 |
| 解决问题 | 它具体解决了什么麻烦 | 用户痛点、客服记录、评论、访谈 | 售后、问答、社媒、调研 |
| 用户身份 | 谁会用，代表什么身份 | 用户画像、社群语言、购买场景 | 社群、私聊、问卷、成交记录 |
| 价格优势 | 价格上为什么更划算 | 比价表、套餐、优惠、总拥有成本 | 竞品页、报价单、活动页 |
| 服务保障 | 还有什么售后和服务承诺 | SLA、退换、陪跑、培训、客服 | 服务协议、FAQ、承诺页 |
| 情感价值 | 还能满足什么情绪和认同 | 口碑、故事、社群话术、品牌表达 | 用户故事、品牌资料、评论 |

## 每个方向怎么收集

- 功能独特性：把“别人没有的点”列出来，再问“用户为什么在意”。
- 效果承诺：优先找可量化结果，找不到再找阶段性结果。
- 解决问题：直接收集用户说过的痛点词，不自己改写。
- 用户身份：看用户在什么身份场景里提起产品。
- 价格优势：不要只看标价，要看对比对象和使用周期。
- 服务保障：把隐性保障显性化，写清边界和条件。
- 情感价值：找用户愿意转述、愿意分享的那部分话。

## 筛选规则

1. 差异化是否明显。
2. 用户是否真的在意。
3. 是否有证据支撑。
4. 是否能被文案自然表达。
5. 是否会被竞品轻易复制。

## 输出骨架

- 产品画像
- 七方向候选清单
- 主卖点
- 辅助卖点
- 证据来源
- 证据缺口
