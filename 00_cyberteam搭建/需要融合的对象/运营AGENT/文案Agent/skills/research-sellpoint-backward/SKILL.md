# 痛点反推卖点器

## 作用

从痛点出发，反推出可表达的卖点。

## 不做的事

- 不从功能口号出发。
- 不把空洞安慰当卖点。

## SOP

1. 先收集真实痛点和原话。
2. 还原痛点发生的场景。
3. 找出阻碍、代价和反复出现的失败点。
4. 反推用户真正想要的结果。
5. 把结果翻译成卖点表达并标出证据缺口。

## 输出

- 痛点清单
- 场景清单
- 阻碍清单
- 卖点表达
- 证据缺口
