# 反向卖点提炼

## 适用场景

- 只有痛点、评论、抱怨或需求描述。
- 没有完整产品资料。
- 需要从用户语言反推表达材料。

## 收集渠道

- 电商/应用商店差评
- 客服工单和售后记录
- 社媒评论和问答平台
- 访谈录音和聊天记录
- 竞品评论和替代方案反馈
- 搜索词和问题词

## 四步流程

1. 提炼用户痛点。
2. 还原痛点发生场景。
3. 找出阻碍和代价。
4. 反推真正想要的结果，再翻译成卖点。

## 反推逻辑

- 痛点不是卖点，痛点背后的结果才是。
- 阻碍不是卖点，用户想跨过去的那道坎才是。
- 结果不是抽象愿望，要翻译成用户能感知的表达。

## 输出骨架

- 痛点
- 场景
- 阻碍
- 代价
- 期望结果
- 卖点表达
- 证据缺口
