# 用户分层

## 适用场景

- 同一产品面对多类用户，需要先选主写对象。
- 不知道该从哪个人群切入。
- 需要让文案“对人说话”，而不是对抽象人群说话。

## 七个分层维度

| 维度 | 看什么 | 怎么收集 | 输出什么 |
|---|---|---|---|
| 身份角色 | 他是谁 | 表单、资料、访谈、组织关系 | 购买者、使用者、影响者 |
| 阶段 | 他处在什么阶段 | 新客/老客、认知/考虑/决策、生命周期 | 现阶段最该说什么 |
| 行为强度 | 他用得多不多 | 使用频率、停留时长、转化路径 | 高频/低频、重度/轻度 |
| 目标强度 | 他有多想解决 | 主动询问、主动比较、主动试用 | 强意愿/弱意愿 |
| 价格敏感度 | 他对价格有多敏感 | 比价、优惠响应、退款原因 | 低价型/价值型 |
| 认知水平 | 他懂多少 | 专业词使用、问题深度、历史购买 | 新手/熟手/专家 |
| 决策权 | 他能不能拍板 | 采购链路、审批链路、群体影响力 | 拍板者/参与者/使用者 |

## 分层顺序

1. 先按决策权和角色分主次。
2. 再按阶段和意愿找切入口。
3. 再看行为、价格敏感和认知水平。
4. 最后确定主写对象和辅助对象。

## 输出骨架

- 主写对象
- 辅助对象
- 各层特征
- 各层诉求
- 各层敏感点
- 应该避开的泛化描述
