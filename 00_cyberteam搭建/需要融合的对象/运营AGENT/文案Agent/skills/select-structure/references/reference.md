# 结构选择

## 适用场景

- 卖点和用户已经清楚，但结构未定。
- 需要在多种结构中做选择，而不是机械套模板。

## 结构库

| 结构 | 适合什么 | 不适合什么 |
|---|---|---|
| AIDA | 先抓注意，再层层推进 | 用户已经很热、只差一句话 |
| 4P | 先问题，再放大，再承诺，再证明 | 没有清楚痛点时 |
| PAS | 痛点明确、需要快速共鸣 | 需要大量证据的长决策 |
| 直述式 | 短场景、强明确诉求 | 复杂价值、需要解释时 |
| 对比式 | 用户在比较方案 | 没有竞品或替代方案时 |
| 故事式 | 需要建立理解和认同 | 极短触点 |
| 清单式 | 需要快速扫描和执行 | 需要强情绪推动时 |

## 选择规则

1. 先看用户是否知道问题。
2. 再看用户是否需要解释或证据。
3. 再看阅读时长。
4. 再看是短触点还是长决策。
5. 最后确定主结构和备用结构。

## 输出骨架

- 结构名称
- 选择理由
- 写作顺序
- 备用结构
- 为什么不选其他结构
