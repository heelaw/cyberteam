# 需求类型判断器

## 作用

先把需求证据采全，再判断需求层级和下一步动作。

## 不做的事

- 不跳过证据采集直接下结论。
- 不把明确、模糊、潜在三类需求混写成一段话。

## SOP

1. 按七个方向收集需求证据。
2. 判断需求是否已经被用户清晰说出。
3. 判断问题是否存在，但目标还不完整。
4. 判断用户是否还没有形成需求意识。
5. 输出需求类型、依据、缺口和下一步动作。

## 输出

- 需求类型
- 证据地图
- 判断依据
- 缺口清单
- 下一步动作
