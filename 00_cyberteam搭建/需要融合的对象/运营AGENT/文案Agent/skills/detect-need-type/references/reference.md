# 需求识别

## 适用场景

- 用户只给了零散信息，不能直接开写。
- 需要判断该先表达、先解释，还是先唤醒。
- 需要先把需求证据采齐，再进入写作或研究。

## 七向采集

| 方向 | 看什么 | 怎么收集 | 典型信号 |
|---|---|---|---|
| 原话 | 用户自己怎么说 | 访谈、聊天记录、咨询记录、评论原文 | “我想要”“我需要”“我卡在” |
| 行为 | 用户实际做了什么 | 埋点、漏斗、留存、流失、转化路径 | 跳出、反复点击、卡点、放弃 |
| 搜索 | 用户主动找什么 | 搜索词、话题词、问答平台、站内搜索 | 问题词、比较词、解决词 |
| 反馈 | 用户不满什么 | 差评、投诉、售后、客服工单 | “太慢了”“不值”“不会用” |
| 对话 | 用户和销售/客服怎么聊 | 录音、纪要、私聊、群聊 | 反复追问、反复确认、犹豫 |
| 对比 | 用户拿什么方案比较 | 竞品评论、替代方案、比价记录 | “和XX比”“有没有更便宜的” |
| 数据 | 哪些指标在提示需求 | 转化率、复购率、退款率、使用频率 | 高流失、低转化、低使用 |

## 采集顺序

1. 先看原话，确认用户的字面表达。
2. 再看行为，确认真实卡点。
3. 再看搜索和反馈，确认用户自己在找什么、抱怨什么。
4. 再看对话和对比，确认用户在权衡什么。
5. 最后用数据校验，避免只凭感觉判断。

## 判定规则

1. 明确需求：目标、对象、结果都说清了。
2. 模糊需求：问题说清了，但目标、约束或结果不完整。
3. 潜在需求：只有现象，没有需求意识。

## 输出骨架

- 需求类型
- 采集到的证据
- 判断依据
- 还缺什么
- 下一步先问什么
