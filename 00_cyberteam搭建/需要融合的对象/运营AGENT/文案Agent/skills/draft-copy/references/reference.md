# 初稿生成

## 适用场景

- 上游已给出卖点、用户和结构。
- 需要输出第一版可用草稿。
- 不要求一次写满，只要求先能用。

## 写作顺序

1. 结论 / 主张
2. 理由 / 解释
3. 证据 / 案例
4. 行动 / CTA

## 不同长度怎么写

| 长度 | 重点 | 常见模块 |
|---|---|---|
| 短文案 | 先打中，再推动 | 标题、利益点、动作 |
| 中文案 | 先解释，再证明 | 问题、方案、证据、CTA |
| 长文案 | 先建立理解，再推进 | 场景、痛点、方案、证据、反对意见、CTA |

## 生成原则

- 先能用，再优化。
- 一段只解决一个问题。
- 证据优先于形容词。
- 缺口要显式标注，不要偷补。

## 输出骨架

- 初稿正文
- 模块说明
- 可删改段
- 待确认信息
