# 需求类型判断法 - 成功案例库

## 案例1: 明确需求用户

**用户画像**: 张先生，企业IT负责人

**需求信号**:
- 渠道：百度搜索"XX系统 vs YY系统"
- 关键词：具体产品对比词
- 行为：浏览对比文章、查看价格页
- 询问：详细询问功能差异和部署方式

**判断结果**: 明确需求（置信度90%）

**文案策略**:
- 标题：《XX系统 vs YY系统：5维度深度对比》
- 内容：功能对比表、案例证明、价格透明
- CTA：立即预约演示

## 案例2: 模糊需求用户

**用户画像**: 李女士，电商标杆运营

**需求信号**:
- 渠道：从公众号文章跳转
- 关键词："团队协作效率低怎么办"
- 行为：浏览文章、查看解决方案页
- 询问：如何提升团队效率

**判断结果**: 模糊需求（置信度85%）

**文案策略**:
- 标题：《为什么团队总是忙但效率低？》
- 内容：痛点共鸣、原因分析、解决方案介绍
- CTA：领取解决方案指南

## 案例3: 潜在需求用户

**用户画像**: 王老板，中小企业主

**需求信号**:
- 渠道：朋友圈广告
- 关键词：无明确搜索
- 行为：刷朋友圈看到广告
- 询问：没有明确询问

**判断结果**: 潜在需求（置信度80%）

**文案策略**:
- 标题：《当你的员工每天多花2小时在...》
- 内容：场景描述、痛点激发、价值展示
- CTA：看看别人怎么做

---

## 需求阶段引导案例

**用户旅程**:
1. 潜在需求：不知道有问题 → 场景唤醒内容
2. 模糊需求：知道有问题 → 痛点共鸣+教育内容
3. 明确需求：知道要什么 → 对比+证明内容
4. 决策：准备购买 → 促销+简化流程

**文案配置**:
- 阶段1-2：内容营销为主
- 阶段3：对比内容+案例
- 阶段4：促销+便捷入口
