# 7个问题找方向法 - 方法论

## 核心框架

7个问题找方向法是一种系统化思考产品卖点的思维框架，通过7个相互关联的问题，帮助运营者从用户视角找到产品的核心卖点方向。

## 原理说明

### 用户中心思维
所有问题都围绕用户展开：
- 用户是谁 → 用户的问题 → 如何解决问题 → 得到什么收益

### 差异化思维
通过与竞品对比，找到独特优势：
- 和竞品有什么不同 → 差异化优势 → 证明依据

### 系统完整性
7个问题形成完整闭环：
- 用户理解（问题1-2）
- 产品理解（问题3-4）
- 差异理解（问题5-7）

## 实施步骤

### 第一步：用户定义
明确回答"用户是谁"，避免"所有人都是用户"的泛化

### 第二步：问题挖掘
深入挖掘用户真实问题，而非表面需求

### 第三步：功能对应
建立"功能-问题-收益"的清晰对应关系

### 第四步：差异聚焦
通过竞品对比找到独特价值点

### 第五步：证据支撑
用数据、案例证明卖点的可信度

## 使用技巧

### 问题回答原则
- 具体 > 泛化：避免"高品质"等空洞词
- 用户视角 > 产品视角：说用户收益，不说产品功能
- 可量化 > 不可量化：收益尽量量化

### 常见错误
- 问题1回答过于宽泛
- 问题2跳过深层问题
- 问题7证明不足
