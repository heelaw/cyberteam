# AIDA文案 - 检查清单

## Attention检查

### 标题
- [ ] 承诺用户收益而非产品功能
- [ ] 制造好奇心或引发共鸣
- [ ] 足够具体有说服力
- [ ] 长度适中（10-20字）

### 开头
- [ ] 3秒内抓住注意力
- [ ] 与目标用户相关
- [ ] 建立情感连接
- [ ] 引导继续阅读

## Interest检查

### 痛点展开
- [ ] 列出主要痛点
- [ ] 痛点具体可感知
- [ ] 与目标用户相关
- [ ] 有数据或案例支撑

### 内容质量
- [ ] 保持对话感
- [ ] 层层递进
- [ ] 不堆砌功能
- [ ] 围绕用户收益

## Desire检查

### 解决方案
- [ ] 清晰展示解决方案
- [ ] 对应前面痛点
- [ ] 用户能理解
- [ ] 有差异化

### 收益量化
- [ ] 收益具体可量化
- [ ] 数据可信
- [ ] 符合用户预期
- [ ] 有对比参照

### 情感触发
- [ ] 有情感共鸣点
- [ ] 触发购买动机
- [ ] 不过于夸张
- [ ] 与品牌调性一致

### 社会证明
- [ ] 有案例支撑
- [ ] 有数据支撑
- [ ] 证言真实
- [ ] 与目标用户相关

## Action检查

### CTA
- [ ] 清晰明确
- [ ] 具体可操作
- [ ] 有吸引力
- [ ] 位置合适

### 紧迫感
- [ ] 有时间限制
- [ ] 有数量限制
- [ ] 合情合理
- [ ] 不过于压迫

### 障碍移除
- [ ] 解除担忧
- [ ] 降低风险
- [ ] 简化流程
- [ ] 增强信任

## 整体检查

- [ ] 四阶段结构完整
- [ ] 各阶段比例合理
- [ ] 逻辑连贯流畅
- [ ] 用户视角贯穿始终
