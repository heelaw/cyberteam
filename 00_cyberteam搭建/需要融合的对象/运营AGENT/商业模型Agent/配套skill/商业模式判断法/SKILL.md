# 商业模式判断法

## Skill概述

商业模式判断法帮助用户识别和判断其业务属于哪种商业模式类型，是理解业务本质的第一步。

## 使用示例

### 示例1：判断外卖平台商业模式

**输入**:
```
业务：连接用户和餐厅，用户下单后餐厅配送，平台收取佣金
```

**输出**:
```
商业模式画像：平台型 + B2C + 交易抽成型 + 撮合端

交易模式：平台型（撮合用户和餐厅）
客户类型：B2C（餐厅对消费者）
付费模式：交易抽成（按订单金额收取佣金）
价值链位置：平台端（提供撮合服务）

类似案例：美团外卖、饿了么
```

## 错误处理

| 错误场景 | 处理方式 |
|----------|----------|
| 业务描述模糊 | 主动询问关键信息：盈利方式、交易双方是谁 |
| 混合模式难判断 | 按收入来源主次判断，说明主模式和辅模式 |
| 跨多个类型 | 指出多类型特征，说明核心类型 |
| 信息不足 | 基于可获得信息判断，标注"待确认" |

## 独特个性

本Skill采用**四维度交叉定位法**（交易×客户×付费×价值链），而非简单套用分类标签，能精准识别复杂商业模式的本质，特别适合判断创新型业务和混合模式。

## 商业模式核心维度

### 1. 交易模式

- **平台型**：撮合买卖双方，收取佣金/服务费
- **自营型**：自建产品/服务，自主销售
- **混合型**：平台+自营共存

### 2. 客户类型

- **B2B**：企业对企业
- **B2C**：企业对消费者
- **C2C**：消费者对消费者
- **G2B/G2C**：政府对企/消费者

### 3. 付费模式

- **订阅制**：定期付费，持续付费
- **一次性付费**：单次购买
- **交易抽成型**：按交易额收费
- **免费增值**：基础免费，增值付费
- **广告收入**：靠广告变现

### 4. 价值链位置

- **生产端**：制造产品
- **流通端**：分销/零售
- **服务端**：提供服务
- **平台端**：撮合交易

## 工作流程

### Step 1：了解业务

收集业务基本描述。

### Step 2：判断交易模式

判断是平台型、自营型还是混合型。

### Step 3：判断客户类型

判断主要客户群体。

### Step 4：判断付费模式

判断主要收入来源。

### Step 5：形成商业模式画像

综合判断，给出完整的商业模式定位。

## Critical Rules

### 必须遵守

1. 客观判断，基于业务实际特征
2. 区分主次，明确商业模式的核心类型
3. 考虑混合情况，不简单二分
4. 基于多维度信息综合判断
5. 给出明确的商业模式定位

### 禁止行为

1. 不随意套用商业模式标签
2. 不忽略业务细节差异
3. 不将模式简单等同于行业
4. 不忽视商业模式的演变可能
5. 不为迎合用户而刻意美化
