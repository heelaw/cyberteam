# 指标拆解树状图

## 北极星指标拆解方法

### 拆解原则

1. **因果驱动** - 子指标能驱动北极星指标变化
2. **独立可追** - 每个子指标可独立追踪和分析
3. **责任到岗** - 每个子指标有明确负责团队
4. **数据可得** - 每个子指标数据可准确获取

---

## 拆解逻辑

### 公式拆解法

适用于北极星指标可以表示为数学公式的场景：

```
北极星指标 = A × B × C

例如:
收入 = 用户数 × 转化率 × 客单价
    = (新用户 + 老用户) × 转化率 × 客单价
```

### 流程拆解法

适用于北极星指标可以表示为业务流程各阶段转化的场景：

```
北极星指标: 核心转化
    ↓
阶段1: 引流 → 阶段2: 激活 → 阶段3: 留存 → 阶段4: 变现
  ↓           ↓             ↓              ↓
 指标A      指标B          指标C           指标D
```

### 维度拆解法

适用于需要从多个维度分析北极星指标的场景：

```
北极星指标
    ↓
┌───────┼───────┐
↓       ↓       ↓
渠道维度  用户维度  产品维度
```

## 在线教育类指标拆解示例

### 案例: 作业提交率拆解

```markdown
# 北极星指标: 核心课程作业提交率

## 一级拆解

作业提交率
    │
    ├── 完播端
    │   ├── 核心课程完播率
    │   │   ├── 人均完播课程数
    │   │   └── 完播课程占比
    │   │
    │   └── 视频完播率
    │       ├── 完播率分布
    │       └── 跳出点分析
    │
    ├── 完成端
    │   ├── 课程-作业转化率
    │   │   ├── 学完课程后提交作业比例
    │   │   └── 跨科目提交分析
    │   │
    │   └── 作业完成率
    │       ├── 按时完成率
    │       └── 补交率
    │
    └── 提交端
        ├── 主动提交率
        │   ├── 首次催交前提交率
        │   └── 无需催交率
        │
        └── 催交转化率
            ├── 一次催交转化率
            └── 二次催交转化率


## 公式拆解

作业提交率 = 完播率 × 作业完成率 × 主动提交率

= (完播课程数/应完播课程数)
  × (完成作业数/应完成作业数)
  × (主动提交数/应提交数)


## 二级拆解详细

### 完播率相关
- 核心课程完播率 = 完播课程数 / 核心课程总数
- 视频完播率 = 实际观看时长 / 视频总时长

### 作业完成率相关
- 课程-作业转化率 = 学完某课并提交作业数 / 学完该课学员数
- 作业完成率 = 实际完成作业数 / 应完成作业数

### 主动提交率相关
- 主动提交率 = 首次催交前提交数 / 应提交总数
- 催交转化率 = 催交后提交数 / 催交总数
```

## 拆解后指标管理

### 指标责任矩阵 (RACI)

| 指标 | 数据分析 | 产品 | 运营 | 技术 | 客服 |
|------|---------|------|------|------|------|
| 北极星指标 | R | A | R | S | I |
| 一级子指标1 | R | A | R | S | I |
| 一级子指标2 | R | A | R | S | I |
| 二级子指标 | R | I | A | S | I |

> R=负责执行 A=最终责任 S=支持 I=知情

### 异常预警机制

| 指标波动 | 预警级别 | 响应时限 | 处理流程 |
|---------|---------|---------|---------|
| ±5%以内 | 正常 | - | 常规追踪 |
| ±5%-10% | 关注 | 24小时 | 原因分析 |
| ±10%-20% | 警告 | 4小时 | 专项分析 |
| ±20%以上 | 紧急 | 1小时 | 立即复盘 |

## 拆解检查清单

```
[ ] 1. 拆解完整性
    - 北极星指标能否被所有子指标解释？
    - 子指标之和/积是否等于北极星指标？

[ ] 2. 驱动有效性
    - 每个子指标变化是否影响北极星指标？
    - 影响方向是否正确？

[ ] 3. 数据可得性
    - 每个子指标是否有数据来源？
    - 数据获取成本是否可接受？

[ ] 4. 责任明确性
    - 每个子指标是否有明确负责人？
    - 团队是否理解自己的指标？

[ ] 5. 可操作 性
    - 团队能否通过行动影响子指标？
    - 影响路径是否清晰？
```

## 常见拆解模式

### 电商/交易类
```
北极星指标: GMV
    ├── 流量 × 转化 × 客单
    ├── 新客 × 转化 × 客单 + 老客 × 转化 × 客单
    └── 引流 → 激活 → 转化 → 复购
```

### 工具/平台类
```
北极星指标: 活跃用户
    ├── 新增活跃 + 存量活跃
    ├── 主动活跃 + 被动活跃
    └── 功能A活跃 + 功能B活跃 + 功能C活跃
```

### 教育/内容类
```
北极星指标: 完课率/完成率
    ├── 完播 × 完成
    ├── 主动完成 + 被动完成
    └── 内容消耗 → 作业完成 → 效果验证
```

### 社区/社交类
```
北极星指标: 互动量
    ├── 发帖量 × 回复率
    ├── 活跃用户 × 人均互动
    └── 新增互动 + 存量互动
```
