**Step 1: 回顾目标**
- 原始目标：GMV 100万
- 实际结果：GMV 70万（达成率70%）
- 偏差原因：流量获取不足，转化率未达预期

**Step 2: 分析过程**
- 关键节点：预热期→正式期→返场期
- 问题节点：预热期流量只有预期的50%

**Step 3: 提炼规律**
- 成功因素：活动选品准确、用户反馈好
- 失败教训：流量预测过于乐观、投放启动太晚
- 可复用：活动节奏把控经验

**Step 4: 下次行动**
1. 活动前2周开始投放测试
2. 建立流量预警机制

## 错误处理

| 错误类型 | 典型表现 | 处理方式 |
|---------|---------|---------|
| 缺乏数据 | "我不知道具体数据" | 引导获取基本数据，或基于关键事件复盘 |
| 目标模糊 | "我也没明确目标是什么" | 协助还原目标，或从结果反推预期目标 |
| 只谈感受 | "感觉这次运气不好" | 追问具体事件，用事实替代感受 |
| 归咎外因 | "主要是竞品太低价" | 引导先从内部可控因素分析 |
| 只批评不肯定 | "这里不行，那里也不行" | 提醒必须包含成功因素 |
| 无行动计划 | "下次注意" | 必须输出具体的下次行动 |
| 复盘拖延 | "过段时间再说" | 强调48小时内复盘的重要性 |

## Critical Rules（必须遵守）

1. **禁止只谈感受不谈事实** - 复盘必须基于数据和事实
2. **禁止只归咎外因** - 先从自身找原因再分析外部
3. **禁止只总结不复盘** - 必须提炼可复用的规律
4. **禁止只批评不肯定** - 优点和问题同样重要
5. **禁止复盘后不行动** - 没有行动改变的复盘无意义
6. **禁止复盘拖延** - 执行后48小时内必须复盘
7. **禁止形式主义复盘** - 复盘要深入不要流于形式

## Workflow Steps

### Step 1: 回顾目标

**输入**:
- 原始目标设定
- 目标衡量标准
- 资源条件约束

**处理**:
1. 明确当初设定的目标
2. 确认目标的衡量指标
3. 梳理可用的资源条件
4. 对照目标与实际结果

**输出**:
- 目标对照表
- 目标达成度评估
- 目标偏差说明

**成功标准**: 目标清晰可量化，偏差有明确定义

---

### Step 2: 分析过程

**输入**:
- 执行过程记录
- 时间节点数据
- 关键决策点

**处理**:
1. 梳理执行的关键节点
2. 分析每个节点的得失
3. 识别关键转折点
4. 还原决策过程和依据

**输出**:
- 执行时间线
- 关键节点分析表
- 决策点回顾

**成功标准**: 过程还原真实，节点分析到位

---

### Step 3: 提炼规律

**输入**:
- 目标对照表
- 过程分析结果

**处理**:
1. 识别成功的关键因素
2. 识别失败的根本原因
3. 提炼可复用的方法
4. 区分偶然和必然因素

**输出**:
- 成功因素清单
- 失败原因清单
- 可复用规律清单
- 假设校验清单

**成功标准**: 规律提炼具体，可指导后续行动

---

### Step 4: 指导行动

**输入**:
- 提炼的规律
- 识别的问题

**处理**:
1. 制定改进行动计划
2. 明确责任人和时间节点
3. 确定下次执行的验证标准
4. 建立复盘成果追踪机制

**输出**:
- 下次行动计划
- 改进措施清单
- 验证标准
- 追踪机制

**成功标准**: 行动具体可执行，有明确时间节点
