# 用户旅程图 Reference

## 适用场景

还原用户完整体验路径，分析行为、思考、情绪、接触点和痛点，适用于产品设计、体验优化

## 方法核心

- 只处理本步骤，不替代上游采集或下游落地。
- 把结论压缩为：判断 + 依据 + 下一步。
- 信息不足时先标缺口，不要硬编结论。

## 渐进式阅读顺序

1. 先读本文件，确认方法边界。
2. 再按需查看课程映射和案例。
3. 最后只补充缺口信息，不把全部背景塞进上下文。

## 输出骨架

- 输入边界
- 关键判断
- 结果
- 风险/缺口
- 下一步

## 常见错误

- 把多个 SOP 混成一个大流程。
- 用空泛词代替具体判断。
- 不标注事实和推断。
- 明明信息不足，却强行给结论。
