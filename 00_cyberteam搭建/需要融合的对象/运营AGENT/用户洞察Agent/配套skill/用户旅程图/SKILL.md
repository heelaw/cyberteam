# 用户旅程图

## 核心定义

用户旅程图（Customer Journey Map）是还原用户从认知到完成目标全过程中，每个阶段的行为、思考、情绪、接触点和痛点的可视化工具。

## 核心要素

| 要素 | 定义 | 分析价值 |
|------|------|---------|
| 阶段 | 用户旅程中的主要步骤 | 理解用户进程 |
| 行为 | 用户实际做了什么 | 还原真实路径 |
| 思考 | 用户当时在想什么 | 理解用户动机 |
| 情绪 | 用户感受如何变化 | 识别情感高点/低点 |
| 接触点 | 用户与产品的触点 | 定位优化机会 |
| 痛点 | 用户遇到的问题 | 确定改进重点 |

## 错误处理

| 错误类型 | 处理方式 |
|---------|---------|
| 阶段划分困难 | 引导用户按目标节点划分 |
| 情绪数据缺失 | 基于行为推断情绪，标注推断依据 |
| 接触点不明确 | 聚焦核心触点，次要触点可简化 |
| 痛点证据不足 | 标注需补充数据，基于经验判断

## Critical Rules

### 必须遵守（5条）

1. **阶段完整性** - 必须覆盖用户从认知到完成的完整旅程
2. **视角一致性** - 全程保持用户视角，不能以运营者视角替代
3. **情绪真实性** - 情绪判断必须有数据或反馈支撑
4. **痛点可验证** - 每个痛点必须有具体证据，不能主观臆断
5. **接触点准确** - 每个接触点必须对应真实的产品界面/功能

### 禁止行为（5条）

1. **禁止阶段跳跃** - 不能跳过用户必经的中间阶段
2. **禁止假设情绪** - 不能主观臆断用户情绪，必须有依据
3. **禁止遗漏触点** - 核心触点不能遗漏
4. **禁止脱离场景** - 不能脱离具体使用场景描述行为
5. **禁止过于细节** - 阶段划分不能过细导致失去分析价值

## Workflow Steps

### Step 1: 明确业务目标

**输入**: 用户描述的业务问题或优化目标

**处理**:
- 确认需要分析的用户的核心目标
- 确定旅程的起点和终点
- 与用户确认分析范围

**输出**:
```json
{
  "用户目标": "完成在线课程学习",
  "旅程起点": "发现课程",
  "旅程终点": "完成全部课程学习",
  "分析范围": "App端学习流程"
}
```

**成功标准**: 目标明确、边界清晰

---

### Step 2: 划分旅程阶段

**输入**: 用户目标和分析范围

**处理**:
- 基于用户目标划分主要阶段
- 每个阶段应是用户有明显目标差异的节点
- 通常4-8个阶段为宜

**输出**:
```json
{
  "阶段划分": [
    {"序号": 1, "阶段": "课程发现", "目标": "找到感兴趣的课程"},
    {"序号": 2, "阶段": "课程评估", "目标": "判断课程是否值得购买"},
    {"序号": 3, "阶段": "购买决策", "目标": "完成购买支付"},
    {"序号": 4, "阶段": "开始学习", "目标": "启动第一次学习"},
    {"序号": 5, "阶段": "持续学习", "目标": "完成阶段性学习"},
    {"序号": 6, "阶段": "完成学习", "目标": "完成全部课程"}
  ]
}
```

**成功标准**: 阶段划分合理、用户目标清晰

---

### Step 3: 分析各阶段要素

**输入**: 阶段划分结果

**处理**:
- 逐阶段分析行为、思考、情绪、接触点
- 收集行为数据支撑分析
- 识别每个阶段的痛点

**输出**:
```json
{
  "阶段1_课程发现": {
    "行为": ["浏览推荐", "搜索关键词", "查看课程详情"],
    "思考": ["这个课程适合我吗", "内容质量怎么样", "价格合理吗"],
    "情绪": "中性→好奇",
    "接触点": ["首页推荐", "搜索结果页", "课程卡片"],
    "痛点": ["推荐不精准", "课程信息难以判断质量"]
  },
  "阶段2_课程评估": {
    "行为": ["查看目录", "试看视频", "阅读评价"],
    "思考": ["学完能得到什么", "和其他课程比怎么样"],
    "情绪": "犹豫",
    "接触点": ["课程详情页", "试看视频", "评价区"],
    "痛点": ["试看内容有限", "评价真实性存疑"]
  }
}
```

**成功标准**: 每个阶段有完整的行为→思考→情绪→接触点→痛点分析

---

### Step 4: 量化情绪曲线

**输入**: 各阶段情绪分析

**处理**:
- 将各阶段情绪量化为1-5分
- 绘制情绪曲线图
- 识别情绪高点（惊喜时刻）和低点（挫败时刻）

**输出**:
```json
{
  "情绪曲线": [
    {"阶段": "课程发现", "分数": 3, "描述": "无感"},
    {"阶段": "课程评估", "分数": 2.5, "描述": "犹豫"},
    {"阶段": "购买决策", "分数": 2, "描述": "焦虑"},
    {"阶段": "开始学习", "分数": 3.5, "描述": "期待"},
    {"阶段": "持续学习", "分数": 3, "描述": "平稳"},
    {"阶段": "完成学习", "分数": 4, "描述": "满足"}
  ],
  "情绪高点": {"阶段": "完成学习", "原因": "达成目标成就感"},
  "情绪低点": {"阶段": "购买决策", "原因": "支付安全和价格顾虑"}
}
```

**成功标准**: 情绪量化有数据支撑，高低点定位准确

---

### Step 5: 识别关键痛点

**输入**: 完整旅程分析

**处理**:
- 统计各痛点的出现频率和影响程度
- 区分核心痛点和边缘痛点
- 评估痛点与情绪低点的关联

**输出**:
```json
{
  "痛点排序": [
    {
      "痛点": "支付时担心安全",
      "阶段": "购买决策",
      "影响程度": "高",
      "情绪影响": "焦虑",
      "证据": "支付页跳出率40%"
    },
    {
      "痛点": "推荐不精准",
      "阶段": "课程发现",
      "影响程度": "中",
      "情绪影响": "失望",
      "证据": "点击率仅5%"
    }
  ]
}
```

**成功标准**: 痛点有数据证据，影响程度评估合理

---

### Step 6: 输出优化建议

**输入**: 完整用户旅程分析

**处理**:
- 针对核心痛点提出解决方案
- 按影响程度排序优化优先级
- 设计惊喜时刻提升用户体验

**输出**:
```markdown
# 用户旅程优化建议

## 核心痛点优化

| 优先级 | 痛点 | 阶段 | 解决方案 | 预期效果 |
|--------|------|------|---------|---------|
| P0 | 支付安全焦虑 | 购买决策 | 展示安全认证、增强保障说明 | 转化率+15% |
| P1 | 推荐不精准 | 课程发现 | 引入个性化推荐算法 | 点击率+20% |
| P2 | 试看内容有限 | 课程评估 | 开放更多试看内容 | 评估转化+10% |

## 惊喜时刻设计

| 阶段 | 当前体验 | 优化方案 |
|------|---------|---------|
| 完成学习 | 无反馈 | 学习完成勋章+分享激励 |
| 开始学习 | 直接播放 | 学习计划引导+同伴进度 |

## 旅程优化地图

阶段1(发现)→阶段2(评估)→阶段3(购买)→阶段4(开始)→阶段5(持续)→阶段6(完成)
   ↓            ↓            ↓           ↓           ↓           ↓
 精准推荐     更多试看     消除焦虑    计划引导    进度激励    成就反馈
```

**成功标准**: 建议具体可落地，与痛点对应明确
