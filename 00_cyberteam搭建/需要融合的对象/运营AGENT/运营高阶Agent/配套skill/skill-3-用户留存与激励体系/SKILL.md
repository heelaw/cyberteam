# 用户留存与激励体系 SKILL

## 基本信息

| 属性 | 值 |
|------|-----|
| **Skill名称** | user-retention-incentive |
| **版本** | v1.0 |
| **触发关键词** | 用户留存、用户激励、活跃提升、价值洼地、用户激励体系 |
| **适用场景** | 用户流失严重、活跃度低、产品价值无法传达给用户 |
| **输入** | 用户行为数据、产品功能列表、用户反馈 |
| **输出** | 用户激励体系设计方案（含价值洼地诊断+激励策略） |

---

## 核心定义

用户激励体系是通过物质激励、精神激励、社交激励的组合，引导用户完成目标行为，实现产品价值传递的系统性方法。

**底层逻辑**：
- 产品提供长期价值
- 运营创造短期价值
- 用户激励填平价值洼地

**核心公式**：
```
价值洼地 = 用户需求 - 产品供给
激励体系 = 物质激励 + 精神激励 + 社交激励
```

---

## Critical Rules（必须遵守）

1. **先诊断再设计**：必须先找到价值洼地，不能凭感觉设计激励
2. **激励与目标行为挂钩**：激励必须指向期望的用户行为，而非泛化
3. **短期价值引导长期价值**：通过即时激励让用户体验产品长期价值

---

## 禁止行为

1. **禁止纯物质激励**：只靠发红包、优惠券留住用户是饮鸩止渴
2. **禁止激励与产品价值脱钩**：激励要能体现产品核心价值
3. **禁止忽视用户路径**：不了解用户典型路径就无法设计有效激励
4. **禁止一次性激励**：必须设计持续激励避免用户流失
5. **禁止忽略精神激励**：高级用户更需要精神层面的满足感
6. **禁止激励过于复杂**：用户看不懂或操作成本高会导致激励失效

## 工作流程

### Step 1: 诊断用户流失原因

**输入**：用户行为数据、用户反馈、产品数据

**处理**：
分析用户留存的4种障碍：

| 障碍类型 | 表现 | 诊断方法 |
|----------|------|----------|
| 产品价值不明确 | 用户不知道产品能带来什么 | 用户访谈、数据分析 |
| 价值获取成本高 | 用户需要太多努力才能体验价值 | 路径分析 |
| 激励不足 | 用户缺乏行动动力 | 行为漏斗分析 |
| 替代品吸引 | 用户被竞品抢走 | 流失去向分析 |

**输出**：用户流失原因诊断报告

**成功标准**：能明确指出核心流失原因（非表面现象）

---

### Step 2: 诊断产品价值洼地

**输入**：用户需求分析、产品功能列表

**处理**：
```
价值洼地诊断：

用户需求（想要）vs 产品供给（已有）

┌─────────────┬──────────────┬──────────────┐
│   需求类型   │    产品供给   │   缺口分析   │
├─────────────┼──────────────┼──────────────┤
│ [用户想要1] │ [产品能提供]  │ [缺口：XX]   │
│ [用户想要2] │ [产品能提供]  │ [缺口：XX]   │
│ [用户想要3] │ [产品能提供]  │ [缺口：XX]   │
└─────────────┴──────────────┴──────────────┘

核心价值洼地：[最需填补的1-2个缺口]
```

**成功标准**：找出1-2个核心价值洼地作为激励切入点

---

### Step 3: 设计用户引导路径

**输入**：价值洼地分析结果

**处理**：
- 明确用户典型使用路径
- 识别路径中的关键节点
- 设计引导用户走完路径的机制

**输出**：
```
用户引导路径设计：

[用户进入]
    ↓
[触发点]：用户被激活的刺激
    ↓
[核心行为1]：需要引导的第一步行为
    ↓
[激励点1]：完成行为后的即时奖励
    ↓
[核心行为2]：需要引导的第二步行为
    ↓
[激励点2]：完成行为后的即时奖励
    ↓
...
    ↓
[价值体验]：用户真正体验到产品价值
    ↓
[留存]：成为长期活跃用户
```

**成功标准**：路径清晰、节点合理、激励到位

---

### Step 4: 设计激励体系

**输入**：引导路径和激励点

**处理**：
设计三层激励组合：

**物质激励**（基础保障）：
- 新手礼包
- 完成任务奖励
- 连续活跃奖励
- 等级权益

**精神激励**（价值认同）：
- 成就徽章
- 排行榜
- 称号认证
- 成长可视化

**社交激励**（关系绑定）：
- 邀请好友
- 社区互动
- 团队挑战
- 粉丝关注

**输出**：
```
激励体系设计：

┌─────────────┬──────────────┬──────────────┐
│   激励类型   │   具体措施    │   触发条件   │
├─────────────┼──────────────┼──────────────┤
│ 物质激励     │ [具体奖励]    │ [触发条件]   │
│ 精神激励     │ [具体措施]    │ [触发条件]   │
│ 社交激励     │ [具体措施]    │ [触发条件]   │
└─────────────┴──────────────┴──────────────┘
```

**成功标准**：三类激励都有，且相互配合形成体系

---

### Step 5: 验证激励有效性

**输入**：激励体系设计方案

**处理**：
- 评估激励成本与预期效果
- 检查是否存在漏洞（薅羊毛风险）
- 验证激励与产品价值的关联度

**输出**：激励体系可行性评估

**成功标准**：激励有效、成本可控、风险可管

---

## 评估维度

| 维度 | 权重 | 说明 |
|------|------|------|
| 诊断准确性 | 25% | 价值洼地识别是否准确 |
| 路径合理性 | 20% | 用户引导路径是否顺畅 |
| 激励有效性 | 20% | 激励是否能驱动目标行为 |
| 激励平衡性 | 15% | 三层激励是否平衡配合 |
| 成本可控性 | 10% | 激励成本是否可持续 |
| 风险可控性 | 10% | 是否有防薅羊毛机制 |

## 应用场景示例

### 场景：提升社区产品用户活跃度

**输入**：某知识社区产品，DAU下降，用户发帖量减少

**分析**：
1. 诊断流失原因：主要是发帖后缺乏反馈
2. 价值洼地：用户渴望被认可，但产品缺乏反馈机制
3. 引导路径：发帖→即时反馈→更多互动→持续发帖
4. 激励设计：
   - 物质：发帖奖励积分、优质内容奖金
   - 精神：精选内容置顶、优质创作者认证
   - 社交：建立创作者社群、粉丝互动功能

**输出**：完整的用户激励体系设计方案
