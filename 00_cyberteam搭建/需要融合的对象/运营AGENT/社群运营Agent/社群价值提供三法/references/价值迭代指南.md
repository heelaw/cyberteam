# 社群价值迭代指南

## 迭代时机

### 应该迭代的情况

```
【信号1】核心指标持续下降
- 资讯打开率连续2周下降
- 分享参与率低于50%
- 打卡参与率低于40%

【信号2】用户反馈集中
- 多次收到类似建议
- 负面评价增加
- 用户流失提及原因

【信号3】外部环境变化
- 竞争对手做出调整
- 行业趋势变化
- 用户需求变化

【信号4】固定周期
- 每季度复盘
- 每年大复盘
```

## 迭代方法

### 步骤1：问题诊断

```
【数据分析】
1. 收集近一个月的数据
2. 找出下降指标
3. 对比历史数据

【用户调研】
1. 设计调研问卷
2. 收集用户反馈
3. 整理反馈类型

【问题归类】
□ 内容问题（质量/形式/时间）
□ 活动问题（频率/内容/嘉宾）
□ 氛围问题（互动/管理/规则）
□ 价值问题（预期/差异/匹配）
```

### 步骤2：方案设计

```
【确定迭代方向】
优先级1（紧急）：_____
优先级2（重要）：_____
优先级3（优化）：_____

【设计优化方案】
针对问题1：
- 现状：_____
- 原因：_____
- 优化：_____
- 预期效果：_____

针对问题2：（同上）
```

### 步骤3：小范围测试

```
【测试计划】
测试范围：_____
测试时间：_____
验证指标：_____

【测试执行】
□ 向部分用户征求意见
□ 小范围试运行
□ 收集测试反馈

【测试评估】
达到预期：□ 是 □ 否
继续测试：□ 是 □ 否
全量推广：□ 是 □ 否
```

### 步骤4：全量推广

```
【推广准备】
□ 新方案内容准备完成
□ 人员培训/通知完成
□ 应急预案准备完成

【正式推广】
□ 发布推广公告
□ 执行新方案
□ 密切监控数据

【推广后跟进】
□ 第一时间收集反馈
□ 关注核心指标变化
□ 及时处理异常
```

## 迭代案例

### 案例：早报打开率下降

```
【问题诊断】
- 数据：早报打开率从60%下降到40%
- 调研：用户反馈"内容太长了""没时间看"
- 原因：内容从5条减少到3条精华，但形式没变

【迭代方案】
方案1：缩短内容+划重点
方案2：增加简短版+详情链接

【小范围测试】
测试组100人，测试1周
结果：方案1打开率提升到55%

【全量推广】
采用方案1，正式执行
效果：整体打开率提升到52%
```
