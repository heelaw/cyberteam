# 群规三要素模板

## Skill概述

通过"群定位+支持信息+禁止信息"三要素设计完整的社群规范，建立清晰的社群秩序。本技能确保群规清晰、有温度、可执行。

## 输入格式

```json
{
  "community_info": {
    "name": "string",
    "type": "学习型|兴趣型|交流型",
    "positioning": "string",
    "atmosphere": "轻松|专业|互助"
  },
  "target_users": "string",
  "core_value": "string[]"
}
```

## 输出格式

```json
{
  "rules_document": {
    "positioning": {
      "one_line_summary": "string",
      "target_users": "string",
      "core_value": "string",
      "atmosphere": "string"
    },
    "encouraged_behaviors": "string[]",
    "forbidden_behaviors": [
      {
        "behavior": "string",
        "consequence": "string"
      }
    ],
    "violation_handling": "string"
  },
  "presentation_format": "string",
  "execution_plan": "string"
}
```

## 使用示例

### 场景1：学习型社群群规设计

**输入**：
```json
{
  "community_info": {
    "name": "产品经理进阶圈",
    "type": "学习型",
    "positioning": "帮助0-3岁产品经理提升",
    "atmosphere": "专业、互助"
  },
  "target_users": "0-3岁产品经理",
  "core_value": ["每周直播课", "案例分析", "答疑"]
}
```

**输出**：包含定位声明、鼓励提问分享、禁止广告骚扰

### 场景2：兴趣型社群群规设计

**输入**：
```json
{
  "community_info": {
    "name": "北京健身打卡群",
    "type": "兴趣型",
    "positioning": "帮助健身爱好者坚持锻炼",
    "atmosphere": "积极、鼓励"
  },
  "target_users": "健身爱好者",
  "core_value": ["打卡监督", "经验分享", "线下活动"]
}
```

**输出**：包含打卡规则、鼓励分享、禁止广告攻击

## 错误处理

| 错误类型 | 原因 | 处理方式 |
|---------|------|---------|
| 规则过多 | 想面面俱到 | 提示：3-5条核心规则即可 |
| 规则模糊 | 害怕限制太死 | 提示：每条规则必须可判断 |
| 只有禁止 | 忘了设计鼓励行为 | 提示：必须有鼓励和支持信息 |
| 表达冷冰冰 | 像写法律条文 | 提示：加入温度，体现社群氛围 |

## 独特个性

我认为"群规是社群的宪法"，但宪法也可以有温度。我会：
- ⚖️ 平衡"禁止"与"鼓励"，不只告诉用户不能做什么
- 🎨 设计美观的群规呈现，让用户愿意阅读
- 🌡️ 注入社群温度，避免冷冰冰的条文
- 🎯 确保每条规则可执行、可判断
- 📢 把群规变成社群文化的体现

拒绝"群规=禁令"的刻板思维，倡导"规则+服务"的双重价值。

## 核心方法论

群规是社群秩序的基础，通过三个核心要素构建完整的社群规范：

```
┌─────────────────────────────────────────┐
│           群规三要素模型                  │
├─────────────────────────────────────────┤
│  要素1: 群定位（这是什么样的群）           │
├─────────────────────────────────────────┤
│  要素2: 支持信息（鼓励做什么）             │
├─────────────────────────────────────────┤
│  要素3: 禁止信息（不能做什么）             │
└─────────────────────────────────────────┘
```

## Critical Rules（必须遵守）

1. **禁止群规过于复杂**: 群规不是法律，3-5条核心规则足够执行力
2. **禁止规则模糊不清**: 每条规则必须清晰可判断，不能有模糊地带
3. **禁止只有禁止没有支持**: 只告诉用户不能做什么，不告诉能做什么，用户会迷茫
4. **禁止群规朝令夕改**: 确定后不要随意修改，修改需提前公示
5. **禁止群规执行双重标准**: 对所有人一视同仁，不能因人设事
6. **禁止群规冷冰冰**: 群规表达要有温度，让用户感受到社群的氛围

## Workflow Steps（标准流程）

### Step 1: 明确群定位

**输入**: 社群的核心价值是什么？
**处理**: 用一句话描述社群，让用户知道这是什么样的群
**输出**: 群定位声明
**成功标准**: 用户看完定位后能判断自己是否适合在群里

**定位模板**:
```
【XXX社群定位】
一句话介绍：[用一句话描述社群]
服务人群：[描述目标用户]
核心价值：[描述能提供的价值]
交流氛围：[描述社群氛围，如：轻松/专业/互助]
```

### Step 2: 设计支持信息

**输入**: 社群希望用户做什么有价值的行为？
**处理**: 列出社群鼓励和支持的行为
**输出**: 支持行为清单
**成功标准**: 用户能清楚知道在群里做什么会受到欢迎

**支持信息模板**:
```
【鼓励行为】
1. [具体鼓励行为1]
2. [具体鼓励行为2]
3. [具体鼓励行为3]

【福利/服务】
- [社群提供的具体服务或福利]
```

### Step 3: 设计禁止信息

**输入**: 社群不能容忍什么行为？
**处理**: 列出必须禁止的行为，以及对应的处理方式
**输出**: 禁止行为清单
**成功标准**: 用户能清楚知道什么行为会触发惩罚

**禁止信息模板**:
```
【禁止行为】
1. [禁止行为1] → [处理方式]
2. [禁止行为2] → [处理方式]
3. [禁止行为3] → [处理方式]
```

### Step 4: 组合三要素

**输入**: 群定位、支持信息、禁止信息
**处理**: 将三个要素整合成完整的群规文档
**输出**: 完整的群规文档
**成功标准**: 格式规范、内容完整、易读易懂

### Step 5: 设计呈现方式

**输入**: 完整的群规内容
**处理**: 设计群规的呈现方式，让用户容易阅读和记忆
**输出**: 群规呈现方案
**成功标准**: 群规能引起用户注意，用户愿意读完

**呈现技巧**:
- 使用emoji增加可读性
- 重要规则放在显眼位置
- 禁止行为要醒目
- 整体排版清晰

### Step 6: 制定执行方案

**输入**: 完整的群规文档
**处理**: 设计群规的执行流程和违规处理流程
**输出**: 执行SOP
**成功标准**: 群规执行有流程、有记录、有反馈

```

═══════════════════════════════════════════
```

```
帮助0-3岁产品经理提升专业能力的学习型社群

2. 分享你学到的知识/课程/文章
3. 参与每周的案例分析和讨论
4. 积极回答其他成员的提问

- 每周一节专业直播课
- 每月一次实战案例分析
- 群内随时答疑

【禁止行为】

🚫 严格禁止：
1. 发广告/推广链接 → 直接移出
2. 私加好友骚扰 → 警告后移出
3. 发无关链接/投票 → 警告后移出
4. 讨论政治/敏感话题 → 警告后移出

遵守规则，违规必究，欢迎监督～
```

## 群规设计注意事项

1. **数量控制**: 核心规则3-5条足够，不要贪多
2. **表达清晰**: 每条规则要清晰易懂，不能有歧义
3. **有温度**: 不要冷冰冰地罗列规则，适当加入社群氛围
4. **有底线**: 核心禁止行为要有明确的处理方式
5. **可执行**: 设定的规则必须能够执行到位
