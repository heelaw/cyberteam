# 用户洞察画布

## 适用场景

- 用户是谁还不清楚。
- 场景和需求没有被说透。
- 需要先完成营销前置判断，再进入渠道或转化设计。

## SOP

1. 收集产品、用户、竞品、社区或历史数据。
2. 区分已知事实和推断。
3. 提炼用户、场景、需求三要素。
4. 输出用户故事、核心痛点和关键触发点。
5. 标记可验证假设。

## 规则

- 用户必须具体到人群，不要只写“年轻人”。
- 场景必须可还原，不要写空泛标签。
- 需求必须分层，至少区分表层和深层。
- 核心需求不超过 3 个。

## 输出骨架

- 用户画像
- 场景地图
- 需求分层
- 用户故事
- 核心转化点
- 待验证项
