## 一、MVP是什么

**思考天团**是一个多专家协作分析系统，通过整合不同思维框架的专家观点，为复杂业务问题提供结构化的分析和可执行的建议。

### 核心价值

1. **多视角分析**: 一次调用卡尼曼、第一性原理、六顶思考帽等多位专家
2. **结构化整合**: 将分歧转化为决策树，而非简单的投票
3. **可执行导向**: 每条建议附带KPI、时间线、风险信号和资源需求
4. **决策支持**: 提供明确的执行依赖和阶段性检查点

---

## 二、MVP解决的问题

**典型业务问题**: 某在线教育平台注册转化率从15%跌至8%，促销失效，如何重新激活增长？

### 解决路径

```
问题诊断 → 专家分析 → 共识提取 → 分歧决策树 → 执行方案
    │                                                    │
    └────────────────────────────────────────────────────┘
                           ↓
                    3条可执行建议
```

## 五、三条核心建议

### 建议1: 立即暂停促销（置信度95%）

- **行动**: 停止所有促销活动至少2周
- **KPI**: 转化率维持8%±0.5%
- **风险**: 3大撤退条件触发则立即恢复
- **资源**: 2人 × 2周 = 4人周，$0成本

### 建议2: 启动用户诊断（置信度90%）

- **行动**: 20+用户深度访谈 + 行为数据分析
- **KPI**: 识别Top3价值感知断点
- **风险**: 样本偏差、访谈失联则调整方案
- **资源**: 4人 × 6周 = 24人周，$8K-$12K

### 建议3: 重构产品体验（置信度85%）

- **行动**: 学习路径 + 社群督学机制
- **KPI**: 完课率12%→20%，留存35%→45%
- **风险**: 内测参与率<20%则重新设计
- **资源**: 6人 × 12周 = 72人周，$30K-$50K

## 八、适用场景

### 适合使用思考天团的情况

- ✅ 复杂业务问题（涉及多个未知变量）
- ✅ 团队意见分歧严重，需要结构化决策
- ✅ 需要快速产出可执行方案
- ✅ 问题已停滞一段时间，需要新视角

### 不适合使用思考天团的情况

- ❌ 问题边界非常清晰，直接执行即可
- ❌ 需要深入行业专业知识（思考天团是通用框架）
- ❌ 时间极其紧迫（完整流程需要3天+）

## 九、如何使用

### 1. 发起分析

```markdown
@思考天团 [您的业务问题]
```

### 2. 等待分析

思考天团自动：
1. 调用3位专家独立分析
2. 整合共识和分歧
3. 输出3条建议 + 决策树

### 3. 查看报告

- **mvp-final.md**: 完整分析报告
- **mvp-final-eval.md**: 质量评估
- **integrator.md**: 整合器模板

### 4. 执行建议

按照执行顺序：
1. 建议1（立即执行）
2. 建议2（第1-6周）
3. 建议3（第7-12周）
