# 思考天团 协作流程

## 三种协作模式

---

## 模式一：并行独立分析

### 适用场景

- 用户要求"多角度分析"
- 问题需要2-5个不同视角
- 快速获取多方观点

### 流程图

```
用户问题
    ↓
意图识别（选择N个专家）
    ↓
    ├──────────────────────────────┐
    ↓                              ↓
专家A分析 专家B分析 专家C分析 ... 专家N分析
    ↓          ↓          ↓             ↓
    └──────────────────────────────┘
                ↓
            结果整合
                ↓
            最终报告
```

### 详细步骤

| 步骤 | 操作 | 输出 |
|------|------|------|
| 1 | 接收问题，解析关键词 | 问题理解备忘录 |
| 2 | 匹配专家（2-5个） | 专家列表 + 理由 |
| 3 | 并行发送分析请求 | 各专家独立分析中 |
| 4 | 收集各专家结果 | 观点汇编 |
| 5 | 提炼共识与分歧 | 整合备忘录 |
| 6 | 生成最终报告 | 结构化报告 |

### 时间预估

- 简单问题（2个专家）：2-3分钟
- 中等难度（3-4个专家）：4-5分钟
- 复杂问题（5个专家）：6-8分钟

## 模式二：交叉辩论模式

### 适用场景

- 重大决策，需要权衡利弊
- 两个以上方案需要选择
- 需要挑战假设、发现漏洞

### 流程图

```
用户问题 + 辩论议题
        ↓
    确定辩手
        ↓
    ┌────┴────┐
    ↓         ↓
正方A      反方B
    ↓         ↓
正方论证 ←──质疑──→ 反方论证
    ↓         ↓
    └────┬────┘
         ↓
    第三方补充（可选）
         ↓
    主持人综合裁决
         ↓
     最终结论
```

### 详细步骤

| 步骤 | 操作 | 输出 |
|------|------|------|
| 1 | 确定辩论议题和辩手 | 辩论框架 |
| 2 | 正方陈述核心观点 | 正方立场书 |
| 3 | 反方质疑正方观点 | 反方质疑清单 |
| 4 | 正方回应反方质疑 | 正方辩护 |
| 5 | 反方回应 + 补充 | 反方立场书 |
| 6 | 第三方补充视角 | 补充观点 |
| 7 | 主持人综合裁决 | 裁决报告 |
| 8 | 生成最终结论 | 结构化结论 |

### 辩论规则

1. 每方发言不超过3轮
2. 质疑必须基于事实和逻辑
3. 主持人保持中立
4. 最终由主持人综合裁决

### 时间预估

- 标准辩论（2v2）：8-10分钟
- 复杂辩论（3v3）：12-15分钟

## 模式三：链式分析模式

### 适用场景

- 问题解决型任务
- 需要多步骤推导
- 目标明确，需要找到路径

### 预设分析链

| 链编号 | 适用问题 | 分析链顺序 |
|--------|----------|------------|
| 链A | 问题诊断 | Goldlin → FiveWhy → FirstPrinciple |
| 链B | 目标实现 | Grow → WBS → Kiss |
| 链C | 战略规划 | SwotTows → ReverseThinking → FiveDimension |
| 链D | 决策分析 | Kahneman → SixHats → AiBoard |
| 链E | 创新突破 | FirstPrinciple → ReverseThinking → SixHats |

### 流程图

```
定义终局（逆向思维）
        ↓
分解问题（5Why/Goldlin）
        ↓
重构方案（第一性原理）
        ↓
验证方案（多视角）
        ↓
行动计划（WBS）
        ↓
整合输出
```

### 详细步骤

| 步骤 | 操作 | 输出 |
|------|------|------|
| 1 | 确定问题类型 | 选择分析链 |
| 2 | 第一步分析 | 中间结果A |
| 3 | 第二步分析（使用A） | 中间结果B |
| 4 | 第三步分析（使用B） | 中间结果C |
| 5 | 方案验证（可选） | 验证报告 |
| 6 | 行动计划生成 | 行动清单 |
| 7 | 整合输出 | 完整报告 |

### 时间预估

- 短链（3步）：5-7分钟
- 长链（4-5步）：8-12分钟

---

## 模式选择决策树

```
用户问题
    ↓
问题复杂度高吗？
    ├─ 否 → 问题类型明确吗？
    │       ├─ 否 → 链式（链A：诊断链）
    │       └─ 是 → 单专家 或 2-3专家并行
    │
    └─ 是 → 需要权衡利弊吗？
            ├─ 否 → 并行独立分析
            └─ 是 → 交叉辩论模式
```

---

## 结果整合规范

### 整合原则

1. **不堆砌** - 每专家提炼核心洞察≤3点
2. **有取舍** - 识别共识，标注分歧
3. **有结论** - 必须给出清晰的综合判断
4. **可执行** - 建议必须是可落地的行动

### 整合模板

```
【共识点】
- 共识1：...
- 共识2：...

【分歧点】
- 分歧1：A方认为...，B方认为...
- 分歧2：...

【综合结论】
[基于共识，化解分歧后的最终判断]

【行动优先级】
1. [最高优先级行动]
2. [次要行动]
3. [可选行动]
```

---

## 质量控制

### 每个节点的检查项

| 阶段 | 检查项 |
|------|--------|
| 问题理解 | 是否抓住了核心问题？ |
| 专家选择 | 是否选择了最相关的专家？ |
| 分析过程 | 各专家是否发挥了专长？ |
| 整合输出 | 是否提炼了核心观点？ |
| 最终结论 | 是否清晰、可执行？ |

### 失败处理

| 情况 | 处理方式 |
|------|----------|
| 专家分析不完整 | 要求重新分析或换专家 |
| 观点冲突无法调和 | 保留分歧，给出主持人判断 |
| 问题超出范围 | 明确说明，返回用户确认 |
