# 系列位置效应专家测试用例

## 正面测试

### 测试1: 演讲设计
- 输入:
```
【系列位置效应分析请求】
场景类型：30分钟产品发布会
信息内容：产品功能、公司优势、合作案例、技术架构
时间限制：30分钟
目标受众：投资人
```
- 期望输出:
  - 信息结构分析
  - 序列优化设计
  - 呈现技巧
- 验证项:
  - [ ] 有信息重要性排序
  - [ ] 核心信息放两端
  - [ ] 有时间分配

### 测试2: 教学设计
- 输入:
```
【系列位置效应分析请求】
场景类型：2小时培训课程
信息内容：理论知识、实操演示、案例分析、总结要点
时间限制：2小时
目标受众：新员工
```
- 期望输出:
  - 序列优化
  - 节奏设计
  - 中间内容处理
- 验证项:
  - [ ] 有30%开场+40%内容+30%结尾分配
  - [ ] 有打破中间遗忘的技巧
  - [ ] 有结尾强化建议

## 负面测试

### 测试1: 核心放中间
- 输入:
```
【系列位置效应分析请求】
场景类型：5分钟汇报
信息内容：背景、核心结论、支持数据、行动计划
时间限制：5分钟
目标受众：领导
```
- 期望行为: 核心结论不能放中间
- 验证项:
  - [ ] 核心结论必须在开头或结尾
  - [ ] 不能接受"核心放中间"的设计

### 测试2: 信息堆砌
- 输入:
```
【系列位置效应分析请求】
场景类型：演讲
信息内容：（20个要点全部要讲）
时间限制：10分钟
目标受众：听众
```
- 期望行为: 不能试图记住所有信息
- 验证项:
  - [ ] 必须精选核心信息
  - [ ] 必须有信息筛选
  - [ ] 必须避免信息堆砌
