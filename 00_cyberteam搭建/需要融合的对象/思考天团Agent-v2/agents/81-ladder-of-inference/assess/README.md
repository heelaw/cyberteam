# 推论阶梯专家评估标准

## 评估维度

| 维度 | 权重 | 评分标准 |
|------|------|----------|
| 完整性 | 20% | 章节是否齐全（触发词、输入格式，分析框架、输出格式、Critical Rules） |
| 准确性 | 30% | 推论阶梯理论是否正确（七层推理、假设检验、数据缺口分析） |
| 可执行性 | 30% | 输出是否可操作（逆向还原推理、假设检验、数据缺口是否明确） |
| 独特性 | 20% | 是否有差异化（与一般逻辑检验的区分、推理层次追溯） |

## 及格线

- 标准模式: >= 85分
- 严格模式: >= 95分

## 快速检验清单

- [ ] 有触发词（5个场景以上）
- [ ] 有输入格式（结论/判断、原始数据、背景）
- [ ] 有输出格式（推理还原七层、假设检验、数据缺口、修正建议）
- [ ] 有分析框架（四步法：逆向还原推理→假设检验→数据缺口分析）
- [ ] 有Critical Rules（必须遵守4条、禁止行为4条）
- [ ] 有协作配置（批判性思维、5Why、确认偏误）

## 核心检验点

### 理论准确性
- [ ] 七层推理层次正确
- [ ] 逆向追溯方法正确
- [ ] 假设类型识别正确

### 框架完整性
- [ ] 每层追问有方法
- [ ] 假设检验有指导
- [ ] 数据缺口清单完整

### 输出有效性
- [ ] 推理还原完整
- [ ] 修正建议可操作
- [ ] 行动建议具体

## 相关专家

- 推论阶梯与批判性思维：整体逻辑检验
- 推论阶梯与5Why：深层假设追问
