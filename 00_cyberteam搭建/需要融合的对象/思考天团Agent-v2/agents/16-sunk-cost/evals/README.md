# 沉没成本决策专家测试用例

## 正面测试

### 测试1: 项目困境止损
- 输入:
```
【沉没成本分析请求】
困境描述：软件项目开发两年，预算已超支150%，进度延期18个月
已投入：500万研发费用，20人团队工作24个月
坚持理由：团队付出很多，放弃对不起大家
```
- 期望输出:
  - 清晰区分沉没成本（500万和24个月）
  - 量化继续坚持的额外损失
  - 给出止损触发条件
  - 提供心理建设建议
- 验证项:
  - [ ] 500万被标记为沉没成本（不应影响决策）
  - [ ] 有继续坚持vs止损的损失对比
  - [ ] 给出了具体的止损触发条件

### 测试2: 投资亏损决策
- 输入:
```
【沉没成本分析请求】
困境描述：股票投资亏损40%，当前持仓100万
已投入：买入成本166万，当前市值100万
坚持理由：已经亏了66万，卖出就是真的亏了
```
- 期望输出:
  - 指出"亏损已发生"而非"卖出才亏损"
  - 分析基本面是否恶化
  - 给出加仓/持有/止损的决策条件
- 验证项:
  - [ ] 66万是沉没成本已明确
  - [ ] 区分"账面损失"和"真实损失"
  - [ ] 有止损参考指标（跌幅/基本面）

### 测试3: 关系维持分析
- 输入:
```
【沉没成本分析请求】
困境描述：恋爱3年，双方频繁争吵，是否该分手？
已投入：3年时间、双方家人已认识、共同朋友多
坚持理由：已经这么久，放弃太可惜
```
- 期望输出:
  - 时间成本正确归类（沉没成本）
  - 评估继续坚持的边际收益
  - 量化止损后的机会收益
- 验证项:
  - [ ] 3年是沉没成本
  - [ ] 评估的是"未来收益"而非"过去投入"
  - [ ] 考虑了止损后的新机会

## 负面测试

### 测试1: 沉没成本辩护
- 输入:
```
【沉没成本分析请求】
困境描述：已经投入很多，要不要继续？
已投入：100万
坚持理由：已经投了这么多，不继续就浪费了
```
- 期望行为: 拒绝"浪费"作为决策依据，明确100万已发生无法挽回
- 验证项:
  - [ ] 明确指出"浪费"逻辑谬误
  - [ ] 决策基于未来收益

### 测试2: 一刀切止损
- 输入:
```
【沉没成本分析请求】
困境描述：项目遇到困难，要不要放弃？
已投入：50万，进度完成60%
背景：市场环境变化，原计划需要调整但核心价值仍存在
```
- 期望行为: 不建议一刀切，分析是否值得调整而非直接放弃
- 验证项:
  - [ ] 区分"项目失败"和"需要调整"
  - [ ] 给出继续坚持的具体条件

### 测试3: 忽视机会成本
- 输入:
```
【沉没成本分析请求】
困境描述：要不要关掉实体店专注线上？
已投入：实体店装修费200万
坚持理由：装修才2年，关了太浪费
```
- 期望行为: 200万是沉没成本，分析止损后的资源能做什么
- 验证项:
  - [ ] 识别机会成本（线上业务可能带来的收益）
  - [ ] 止损建议考虑资源再分配
