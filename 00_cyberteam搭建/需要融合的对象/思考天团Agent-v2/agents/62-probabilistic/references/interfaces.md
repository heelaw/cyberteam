# 接口协议

## 输入接口

### 用户输入格式
```
【概率思维分析请求】

决策/判断：[需要做概率判断的内容]
选项：[可能的选项或情景]
已有数据：[你已经掌握的概率信息]
新证据：[如果有新信息]
置信度：[你对当前判断的自信程度]
```

### 系统输入Schema
```json
{
  "task": "分析任务",
  "context": {
    "decision": "string - 需要做概率判断的内容",
    "options": "array - 可能的选项",
    "existing_data": "object - 已有概率信息",
    "new_evidence": "object - 新证据(可选)",
    "confidence": "number - 置信度(可选)"
  },
  "previous_experts": ["上一个专家"],
  "user_id": "用户标识"
}
```

## 输出接口

### 用户输出格式
```
═══════════════════════════════════════════
      『概率思维』分析报告
═══════════════════════════════════════════

【判断/决策】
[需要做概率判断的内容]

【不确定性识别】
类型：[哪种不确定性]
范围：[最坏→最好的可能]

【概率分解】
因素分解：
├── 因素A：P=[X%]，[依据]
├── 因素B：P=[Y%]，[依据]
└── 综合概率：P=[Z%]

【期望值分析】（如适用）
选项A：EV=[值]
选项B：EV=[值]
推荐：基于EV的选择

【贝叶斯更新】（如适用）
先验概率：[初始估计]
新证据：[观测到的数据]
更新后概率：[贝叶斯计算结果]

【置信区间】
概率范围：X% ~ Y%
置信度：Z%

【风险提示】
⚠️ 尾部风险：[低概率高冲击事件]
⚠️ 基础率忽视风险：[可能被忽视的基础概率]
```

### 系统输出Schema
```json
{
  "expert_id": "62-probabilistic",
  "insights": ["概率估计", "期望值分析", "贝叶斯更新"],
  "confidence": 0.85,
  "handoff": {
    "next_expert": "建议下一个专家",
    "reason": "交接原因"
  }
}
```

## CLI接口

### 触发命令
| 命令 | 功能 | 用法 |
|------|------|------|
| `/思考天团-62-probabilistic` | 启动专家分析 | `/思考天团-62-probabilistic` |

### 调用方式
```bash
# 直接调用
/思考天团-62-probabilistic

# 带参数调用
/思考天团-62-probabilistic [问题描述]
```
