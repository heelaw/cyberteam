# 麦肯锡框架专家测试用例

## 正面测试

### 测试1: 工作方法论提炼
- 输入:
```
【麦肯锡框架提炼请求】
核心思路/步骤：问题分析→原因查找→方案制定→执行跟踪
使用场景：顾问项目方法论
目标受众：咨询顾问
```
- 期望输出:
  - MECE分析
  - 框架方案（2-3个）
  - 使用说明
  - 记忆口诀
- 验证项:
  - [ ] MECE检查完整
  - [ ] 2-3个方案
  - [ ] 有案例

### 测试2: 品牌方法论
- 输入:
```
【麦肯锡框架提炼请求】
核心思路/步骤：定位→内容→渠道→转化→复购
使用场景：品牌运营
目标受众：运营团队
```
- 期望输出:
  - 框架方案
  - MECE检查
  - 使用说明
- 验证项:
  - [ ] 简洁有力
  - [ ] 朗朗上口
  - [ ] 有维度解释

## 负面测试

### 测试1: 照搬原版
- 输入:
```
【麦肯锡框架提炼请求】
核心思路：波特五力
使用场景：战略分析
```
- 期望行为: 不能直接复制麦肯锡原版框架
- 验证项:
  - [ ] 是用户自己的框架
  - [ ] 不是照搬
  - [ ] 有用户特色的提炼

### 测试2: 维度过多
- 输入:
```
【麦肯锡框架提炼请求】
核心思路：有10个步骤
使用场景：复杂流程
```
- 期望行为: 维度不能超过7个，要精简
- 验证项:
  - [ ] 合并到7个以内
  - [ ] 不冗余
  - [ ] 保持核心
