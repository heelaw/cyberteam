# 事前尸检专家测试用例

## 正面测试

### 测试1: 新项目启动风险识别
- 输入:
```
【事前尸检请求】
项目/决策描述：开发一款新的社交APP，目标是在6个月内获取100万用户
预期目标：日活10万，月留存40%
时间线：6个月开发+上线
团队成员：产品经理2人、开发8人、设计2人、运营3人
已知风险：暂无
```
- 期望输出:
  - 有完整的失败场景假设（时间点、严重程度）
  - 识别了团队会听到的抱怨
  - 有内外部风险矩阵
  - 有预防措施、应急预案、早期预警、恢复计划
- 验证项:
  - [ ] 失败时间点合理（不会是第一周）
  - [ ] 风险涵盖规划、执行、团队、技术、外部
  - [ ] 每项风险有可能性(1-5)和影响度(1-5)
  - [ ] 防御措施具体可执行

### 测试2: 决策前隐患识别
- 输入:
```
【事前尸检请求】
项目/决策描述：建议公司进入东南亚市场
预期目标：一年内在2个国家建立本地团队
时间线：3个月准备+9个月执行
团队成员：国际业务负责人1人、本地合伙人招募中
已知风险：竞争对手已提前布局
```
- 期望输出:
  - 假设失败的严重程度要匹配决策重要性
  - 外部风险要特别关注（政策法规、竞争对手）
  - 有市场适应性和合规性风险
- 验证项:
  - [ ] 识别了政策法规风险
  - [ ] 识别了竞争对手先发制人的可能
  - [ ] 有本地化挑战的具体分析

### 测试3: 计划评审风险识别
- 输入:
```
【事前尸检请求】
项目/决策描述：季度销售目标定在1000万，同比增长50%
预期目标：完成1000万销售额
时间线：1个季度（90天）
团队成员：销售团队20人
已知风险：市场竞争加剧
```
- 期望输出:
  - 失败假设要与激进目标匹配
  - 风险要有优先级排序
  - 有具体的监控指标
- 验证项:
  - [ ] 风险分计算正确（可能性×影响度）
  - [ ] 有早期预警指标（如周销售额低于X万）
  - [ ] 有撤退条件（如连续2周未达50%进度）

## 负面测试

### 测试1: 表面风险识别
- 输入:
```
【事前尸检请求】
项目/决策描述：上线新功能
预期目标：功能顺利上线
时间线：2周
团队成员：开发5人、测试2人
已知风险：无
```
- 期望行为: 不能只给出模糊风险（如"可能沟通不畅"），必须具体化
- 验证项:
  - [ ] 风险不能是泛泛的"团队协作"问题
  - [ ] 必须识别具体的失败模式和根因

### 测试2: 无防御措施
- 输入:
```
【事前尸检请求】
项目/决策描述：重要系统重构
预期目标：重构成功，系统稳定
时间线：1个月
团队成员：架构师1人、高级开发3人
已知风险：历史代码复杂
```
- 期望行为: 不能只列出风险而不给缓释方案
- 验证项:
  - [ ] 每个高优先级风险都有对应的防御措施
  - [ ] 防御措施包含预防、应急、预警、恢复四种

### 测试3: 乐观主义倾向
- 输入:
```
【事前尸检请求】
项目/决策描述：并购一家创业公司
预期目标：顺利完成并购，整合两家业务
时间线：6个月并购+6个月整合
团队成员：BD 2人、法务1人、HR 1人
已知风险：文化差异
```
- 期望行为: 不能低估风险可能性和影响度
- 验证项:
  - [ ] 文化冲突的风险分要高于3
  - [ ] 要考虑整合失败的完整场景
