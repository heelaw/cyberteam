# 事前尸检专家评估标准

## 评估维度

| 维度 | 权重 | 评分标准 |
|------|------|----------|
| 完整性 | 20% | 章节是否齐全（触发词、输入格式、分析框架、输出格式、Critical Rules） |
| 准确性 | 30% | 事前尸检理论是否正确（假设失败、回推原因、前置防御） |
| 可执行性 | 30% | 输出是否可操作（防御措施、预警指标、恢复计划是否明确） |
| 独特性 | 20% | 是否有差异化（与复盘专家的区分、风险识别框架的实用性） |

## 及格线

- 标准模式: >= 85分
- 严格模式: >= 95分

## 快速检验清单

- [ ] 有触发词（4个场景以上）
- [ ] 有输入格式（项目描述、预期目标、时间线）
- [ ] 有输出格式（失败假设、致败因素、防御措施）
- [ ] 有分析框架（五步法：假设失败→症状识别→发散→聚类→防御）
- [ ] 有Critical Rules（必须遵守5条、禁止行为5条）
- [ ] 有CLI命令（如有）
- [ ] 有handoff协议

## 核心检验点

### 理论准确性
- [ ] 假设失败方法论正确（先假设失败再回推）
- [ ] 失败症状识别包含三个层次（彻底/严重/部分）
- [ ] 致败因素涵盖内部和外部

### 框架完整性
- [ ] 风险优先级评估包含可能性和影响度
- [ ] 防御措施包含四种类型（预防/应急/预警/恢复）
- [ ] 输出格式包含行动建议

### 输出有效性
- [ ] 失败场景假设格式正确
- [ ] 致败因素分析有归类
- [ ] 有具体的早期预警指标
- [ ] 有量化的阈值
