# 迭代效应专家测试用例

## 正面测试

### 测试1: 产品迭代规划
- 输入:
```
【迭代效应分析请求】
问题/目标：改进电商App用户体验
当前状态：现有App功能齐全但用户体验差
资源约束：3人团队，3个月时间
风险偏好：稳健
```
- 期望输出:
  - 迭代策略设计
  - MVP定义
  - 迭代路线图
- 验证项:
  - [ ] 有MVP功能定义
  - [ ] 有版本迭代规划
  - [ ] 有反馈机制

### 测试2: 创业验证
- 输入:
```
【迭代效应分析请求】
问题/目标：验证订阅制商业模式是否可行
当前状态：已有原型产品
资源约束：有限资金，6个月
风险偏好：激进
```
- 期望输出:
  - 迭代路径
  - 关键里程碑
  - Pivot判断标准
- 验证项:
  - [ ] 有明确的迭代目标
  - [ ] 有验证指标
  - [ ] 有Pivot条件

## 负面测试

### 测试1: 大而全
- 输入:
```
【迭代效应分析请求】
问题/目标：一次性开发完整企业系统
当前状态：刚开始规划
资源约束：（充足）
风险偏好：（无偏好）
```
- 期望行为: 不能接受大而全的方案
- 验证项:
  - [ ] 必须拆分为小迭代
  - [ ] 必须有MVP定义
  - [ ] 必须聚焦最小可用改进

### 测试2: 闭门迭代
- 输入:
```
【迭代效应分析请求】
问题/目标：完善产品功能
当前状态：开发团队在内部开发
资源约束：（充足）
风险偏好：保守
```
- 期望行为: 不能闭门迭代，必须有用户反馈
- 验证项:
  - [ ] 必须有用户反馈机制
  - [ ] 必须有数据收集计划
  - [ ] 不能无验收标准迭代
