# 迭代效应专家评估标准

## 评估维度

| 维度 | 权重 | 评分标准 |
|------|------|----------|
| 完整性 | 20% | 章节是否齐全（触发词、输入格式，分析框架、输出格式、Critical Rules） |
| 准确性 | 30% | 迭代效应理论是否正确（小步快跑、反馈驱动、累积优势、突破临界点） |
| 可执行性 | 30% | 输出是否可操作（迭代策略、MVP设计、迭代路径、反馈机制是否明确） |
| 独特性 | 20% | 是否有差异化（与激进改革的区分、渐进优化机制） |

## 及格线

- 标准模式: >= 85分
- 严格模式: >= 95分

## 快速检验清单

- [ ] 有触发词（5个场景以上）
- [ ] 有输入格式（问题/目标、当前状态、资源约束、风险偏好）
- [ ] 有输出格式（迭代策略、MVP设计、迭代路线图、反馈机制）
- [ ] 有分析框架（四步法：迭代策略设计→迭代路径规划→反馈机制设计→迭代效应评估）
- [ ] 有Critical Rules（必须遵守4条、禁止行为4条）
- [ ] 有协作配置（PDCA、WBS、二阶思维）

## 核心检验点

### 理论准确性
- [ ] 小步快跑概念正确
- [ ] 反馈驱动机制正确
- [ ] 临界点概念正确

### 框架完整性
- [ ] MVP定义有方法
- [ ] 迭代路径有规划
- [ ] Pivot vs Persevere判断标准明确

### 输出有效性
- [ ] 迭代路线图可执行
- [ ] 反馈指标可量化
- [ ] 临界预估有依据

## 相关专家

- 迭代效应与PDCA：迭代是PDCA循环的具体实现
- 迭代效应与MVP：MVP是迭代的起点
