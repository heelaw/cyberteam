# 长尾理论测试用例

## 正面测试

### 测试1: 音乐流媒体平台分析
- 输入:
```
【长尾分析请求】
分析对象：在线音乐平台
当前状况：头部歌曲（头部1%）占据60%播放量
目标用户特征：年轻用户，追求个性化
```
- 期望输出:
  - 头部分析（主流唱片公司版权）
  - 长尾评估（独立音乐人作品）
  - 聚合效应（个性化推荐算法降低发现成本）
  - 成本结构（数字存储成本低，推荐算法成本可控）
  - 策略建议（头部+长尾，算法驱动发现）
- 验证项:
  - [ ] 头尾划分有数据支撑
  - [ ] 聚合临界点计算合理
  - [ ] 推荐成本被重点分析

### 测试2: 图书电商平台分析
- 输入:
```
【长尾分析请求】
分析对象：在线图书零售
当前状况：当当/京东/亚马逊主导，实体书库存有限
已有产品线：畅销书+部分长尾书籍
```
- 期望输出:
  - 头部（畅销书/教辅）
  - 长尾（绝版书/小众书籍/专业著作）
  - 聚合效应（绝版书拍卖/专业书专题）
  - 成本结构（实体仓储 vs 数字版）
- 验证项:
  - [ ] 区分了实体和数字长尾
  - [ ] 聚合成本分析到位
  - [ ] 提及Kindle等数字出版

### 测试3: SaaS工具市场分析
- 输入:
```
【长尾分析请求】
分析对象：企业SaaS工具
当前状况：头部（CRM/ERP）占据大部分市场
目标用户特征：中小企业，预算有限
```
- 期望输出:
  - 头部（通用型SaaS）
  - 长尾（垂直行业工具/细分功能）
  - 聚合效应（工具整合平台/应用市场）
  - 策略建议（聚焦垂直行业/专注细分功能）
- 验证项:
  - [ ] 识别出长尾工具的开发成本问题
  - [ ] 聚合效应考虑平台模式

## 负面测试

### 测试1: 伪需求放大场景
- 输入: 用户提出"所有小众需求都值得满足"
- 期望行为: 指出伪需求识别的重要性，聚合临界点的限制
- 验证项:
  - [ ] 不盲目认同所有长尾需求
  - [ ] 强调需求真实性验证

### 测试2: 成本忽视场景
- 输入: 某业务提出无限扩充SKU
- 期望行为: 分析存储和分销成本限制，给出合理边界
- 验证项:
  - [ ] 明确成本结构约束
  - [ ] 给出最小可行市场规模

### 测试3: 头部逻辑做长尾
- 输入: 用爆款思维运营长尾产品
- 期望行为: 指出头尾策略差异，避免用头部逻辑做长尾
- 验证项:
  - [ ] 区分头尾运营逻辑
  - [ ] 提供差异化的长尾运营方法
