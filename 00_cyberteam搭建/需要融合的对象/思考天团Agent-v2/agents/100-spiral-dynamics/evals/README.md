# 螺旋动力学专家测试用例

## 正面测试

### 测试1: 组织变革分析
- 输入:
```
【螺旋动力学分析请求】
分析对象：某传统企业
当前阶段：Blue阶段（秩序权威）
观察到的现象：年轻员工频繁离职
分析目的：理解变革阻力
```
- 期望输出:
  - 当前阶段诊断
  - 价值特征详析
  - 过渡分析
  - 变革引导
- 验证项:
  - [ ] 阶段诊断准确
  - [ ] 过渡挑战识别
  - [ ] 变革路径具体

### 测试2: 个人成长阶段
- 输入:
```
【螺旋动力学分析请求】
分析对象：个人职业发展
当前阶段：Orange阶段（成就导向）
观察到的现象：追求晋升和收入但感到空虚
分析目的：如何提升价值层次
```
- 期望输出:
  - 当前阶段分析
  - 过渡到Green的可能性
  - 成长路径
- 验证项:
  - [ ] 阶段判断合理
  - [ ] 过渡分析有洞察
  - [ ] 成长建议具体

## 负面测试

### 测试1: 轻视低级阶段
- 输入:
```
【螺旋动力学分析请求】
分析对象：某组织
当前阶段：Red阶段（力量崇拜）
观察到的现象：权力斗争激烈
```
- 期望行为: 不能轻视Red阶段，每个阶段都有其功能
- 验证项:
  - [ ] 承认Red阶段的价值
  - [ ] 不做道德评判
  - [ ] 理解阶段的必要性

### 测试2: 强制跳跃
- 输入:
```
【螺旋动力学分析请求】
分析对象：刚进入职场的年轻人
当前阶段：刚到Orange
观察到的现象：想直接进入Yellow
分析目的：加速成长
```
- 期望行为: 不能跳过过渡阶段，要先整合当前阶段
- 验证项:
  - [ ] 强调阶段不能跳跃
  - [ ] 过渡需要时间
  - [ ] 先整合再提升
