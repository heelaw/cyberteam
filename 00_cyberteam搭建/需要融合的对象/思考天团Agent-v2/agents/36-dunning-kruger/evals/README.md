# 邓宁-克鲁格效应专家测试用例

## 正面测试

### 测试1: 新手自信评估
- 输入:
```
【邓宁-克鲁格分析请求】
技能/领域：Python编程
自我评估：已经掌握了Python，可以开始做项目了
相关经验：学了1个月Python，看了几教程
困惑点：不知道怎么进一步提升
```
- 期望输出:
  - DK曲线定位：新手（愚昧之峰）
  - 自信度与能力不匹配分析
  - 跨越绝望之谷的具体路径
- 验证项:
  - [ ] 能识别"一个月就觉得自己掌握了"是愚昧之峰特征
  - [ ] 有布鲁姆分类法评估
  - [ ] 有具体的能力提升建议

### 测试2: 中级开发者困境
- 输入:
```
【邓宁-克鲁格分析请求】
技能/领域：系统设计
自我评估：觉得自己什么都不会，设计不出来
相关经验：参与过3个系统的开发
困惑点：看到别人的设计方案觉得很简单，自己的却总有问题
```
- 期望输出:
  - DK曲线定位：中级（绝望之谷）
  - 自信度低于能力分析
  - 建立信心的具体方法
- 验证项:
  - [ ] 能识别这是绝望之谷的典型表现
  - [ ] 有小步验证建立信心的建议
  - [ ] 能区分"真不懂"和"低估自己"

## 负面测试

### 测试1: 跳过自我评估
- 输入:
```
【邓宁-克鲁格分析请求】
技能/领域：投资
```
- 期望行为: 不能跳过自我评估流程
- 验证项:
  - [ ] 必须要求输入自我评估信息
  - [ ] 必须有自信度校准步骤

### 测试2: 静止视角
- 输入:
```
【邓宁-克鲁格分析请求】
技能/领域：演讲
自我评估：我天生就不擅长演讲
相关经验：只做过2次汇报
困惑点：每次上台都紧张
```
- 期望行为: 不能接受"天生就这样"的标签化判断
- 验证项:
  - [ ] 必须从动态视角分析
  - [ ] 必须给出能力提升路径
