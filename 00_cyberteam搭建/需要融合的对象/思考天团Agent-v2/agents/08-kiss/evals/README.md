# KISS复盘专家测试用例

## 正面测试

### 测试1: 项目复盘
- 输入:
```
【KISS复盘请求】
复盘主题：某产品发布项目
时间范围：过去3个月
参与者：项目团队10人
背景：项目成功上线但有延期
```
- 期望输出:
  - K保持（≥3条）
  - I改进（≥3条）
  - S开始（≥2条）
  - S停止（≥2条）
  - 行动计划（人有时间）
- 验证项:
  - [ ] 四维都有内容
  - [ ] 具体非模糊
  - [ ] 行动计划可执行

### 测试2: 个人季度复盘
- 输入:
```
【KISS复盘请求】
复盘主题：个人季度工作
时间范围：Q1
参与者：个人
背景：年初设定目标
```
- 期望输出:
  - K/I/S/S四维度
  - 具体内容
  - 下季度计划
- 验证项:
  - [ ] 面向未来
  - [ ] 对事不对人
  - [ ] 有衡量标准

## 负面测试

### 测试1: 只说"不错"
- 输入:
```
【KISS复盘请求】
复盘主题：某团队活动
时间范围：上周
参与者：团队
```
- 期望行为: 不能模糊评价，要有具体描述
- 验证项:
  - [ ] 具体说明好在哪
  - [ ] 不是简单的"不错"
  - [ ] 有原因分析

### 测试2: 四维失衡
- 输入:
```
【KISS复盘请求】
复盘主题：某工作
只有做得好的地方
```
- 期望行为: 必须四维平衡，不能只有K没有I/S/S
- 验证项:
  - [ ] I改进必须有
  - [ ] S开始必须有
  - [ ] S停止必须有
