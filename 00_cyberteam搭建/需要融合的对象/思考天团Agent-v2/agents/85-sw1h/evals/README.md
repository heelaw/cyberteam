# SW1H分析法专家测试用例

## 正面测试

### 测试1: 新项目启动
- 输入:
```
【SW1H分析请求】
分析对象：启动新电商项目
分析目的：全面评估可行性
背景：有创业想法
```
- 期望输出:
  - 七维度完整分析
  - 利益相关者识别
  - 综合评估
- 验证项:
  - [ ] 七维度全部覆盖
  - [ ] 每个维度有具体分析
  - [ ] 有综合评估

### 测试2: 问题全面诊断
- 输入:
```
【SW1H分析请求】
分析对象：销售额下滑问题
分析目的：问题诊断
背景：季度业绩下滑
```
- 期望输出:
  - 七维度诊断
  - 原因分析
  - 改进建议
- 验证项:
  - [ ] 七维度全部覆盖
  - [ ] 有具体原因分析
  - [ ] 有可执行建议

## 负面测试

### 测试1: 遗漏维度
- 输入:
```
【SW1H分析请求】
分析对象：市场扩张策略
分析目的：方案评估
背景：考虑进入新市场
```
- 期望行为: 不能遗漏任何维度
- 验证项:
  - [ ] 七维度必须全部覆盖
  - [ ] 不能跳过任何一个维度
  - [ ] 每个维度必须有实质内容

### 测试2: 蜻蜓点水
- 输入:
```
【SW1H分析请求】
分析对象：团队管理问题
分析目的：全面了解
背景：管理遇到挑战
```
- 期望行为: 不能对每个维度都浅尝辄止
- 验证项:
  - [ ] 每个维度必须有深入分析
  - [ ] 不能只填框架没有洞察
  - [ ] 必须有具体分析和结论
