# 护城河理论专家测试用例

## 正面测试

### 测试1: 企业护城河评估
- 输入:
```
【护城河分析请求】
分析对象：某互联网平台
竞争对手：主要竞争对手2家
行业特点：平台经济
时间范围：5年
```
- 期望输出:
  - 护城河来源识别（四大来源）
  - 宽度评估（综合评分、等级）
  - 侵蚀风险分析
  - 战略建议
- 验证项:
  - [ ] 四种护城河都分析
  - [ ] 宽度评分合理
  - [ ] 侵蚀信号具体

### 测试2: 个人职业护城河
- 输入:
```
【护城河分析请求】
分析对象：个人职业发展
竞争对手：同岗位竞争者
行业特点：技术行业
时间范围：3年
```
- 期望输出:
  - 职业护城河识别
  - 护城河宽度评估
  - 构建策略建议
- 验证项:
  - [ ] 护城河概念应用于个人
  - [ ] 构建路径具体
  - [ ] 有时间规划

## 负面测试

### 测试1: 把风口当护城河
- 输入:
```
【护城河分析请求】
分析对象：某风口上的创业公司
竞争对手：多家
行业特点：新兴市场
时间范围：2年
```
- 期望行为: 不能把短期优势当护城河，要有持久性评估
- 验证项:
  - [ ] 区分短期优势和护城河
  - [ ] 持久性是核心标准
  - [ ] 不能仅看当前增长

### 测试2: 无护城河企业
- 输入:
```
【护城河分析请求】
分析对象：某纯竞争性行业企业
竞争对手：很多
行业特点：产品同质化
时间范围：1年
```
- 期望行为: 不能美化无护城河的企业
- 验证项:
  - [ ] 诚实评估护城河缺失
  - [ ] 给出转型或效率建议
  - [ ] 不强行使其"有护城河"
