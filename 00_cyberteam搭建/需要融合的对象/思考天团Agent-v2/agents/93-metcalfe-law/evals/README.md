# 梅特卡夫法则专家测试用例

## 正面测试

### 测试1: 平台投资分析
- 输入:
```
【梅特卡夫法则分析请求】
产品/平台：某社交电商平台
当前用户数：100万
用户增长：月增长20%
竞争格局：三家竞争
```
- 期望输出:
  - 网络效应识别（直接/间接/双边）
  - 临界用户规模计算
  - 护城河评估
  - 增长策略建议
- 验证项:
  - [ ] 网络效应类型判断正确
  - [ ] 临界点分析具体
  - [ ] 阶段判断准确

### 测试2: 产品网络效应评估
- 输入:
```
【梅特卡夫法则分析请求】
产品/平台：企业SaaS工具
当前用户数：5000
用户增长：年增长50%
竞争格局：分散市场
```
- 期望输出:
  - 网络效应类型识别
  - 梅特卡夫适用度评估
  - 当前阶段判断
  - 增长建议
- 验证项:
  - [ ] 区分真正的网络效应和伪网络效应
  - [ ] 临界点计算合理
  - [ ] 策略因阶段不同

## 负面测试

### 测试1: 伪网络效应误判
- 输入:
```
【梅特卡夫法则分析请求】
产品/平台：某矿泉水品牌
当前用户数：100万
用户增长：稳定
竞争格局：激烈竞争
```
- 期望行为: 不能将无网络效应的产品误判为有梅特卡夫效应
- 验证项:
  - [ ] 识别真正的网络效应
  - [ ] 不能混淆用户多和有效网络
  - [ ] 梅特卡夫适用度评估合理

### 测试2: 线性外推错误
- 输入:
```
【梅特卡夫法则分析请求】
产品/平台：某短视频平台
当前用户数：1000万
用户增长：100%年增长
竞争格局：头部竞争
```
- 期望行为: 不能用线性思维外推网络价值
- 验证项:
  - [ ] 强调指数增长特性
  - [ ] 临界点前后策略区分
  - [ ] 负网络效应警告
