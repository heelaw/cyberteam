# 接口协议

## 输入接口

### 用户输入格式
```
【系统整合分析请求】

问题描述：[需要整合分析的问题]
已识别的子系统：[你发现的子要素]
层次困惑：[你不确定如何划分的部分]
整合目标：[你希望达到的理解程度]
```

### 系统输入Schema
```json
{
  "task": "分析任务",
  "context": {
    "problem_description": "string - 需要整合分析的问题",
    "identified_subsystems": "array - 已识别的子要素",
    "layer_confusion": "string - 层次困惑描述",
    "integration_goal": "string - 整合目标"
  },
  "previous_experts": ["上一个专家"],
  "user_id": "用户标识"
}
```

## 输出接口

### 用户输出格式
```
═══════════════════════════════════════════
      『系统整合』分析报告
═══════════════════════════════════════════

【问题重构】
[从系统整合视角重新表述问题]

【层次结构】
🏛️ 宏观层：[描述]
🏢 中观层：[描述]
⚙️ 微观层：[描述]
🔮 元层：[描述]

【边界分析】
📍 系统边界：[划定依据]
↔️ 接口点：[与外部的交互]
🔗 边界特性：[刚性/渗透性]

【整合地图】
关键整合点：
├── 层次间：[整合方式]
├── 子系统间：[协调机制]
└── 时间维度：[演化路径]

【涌现分析】
已识别的涌现行为：
1. [涌现特性1]
2. [涌现特性2]

⚠️ 潜在意外涌现：[可能但未预期的效应]

【整合建议】
最优整合路径：...
关键杠杆点：...
```

### 系统输出Schema
```json
{
  "expert_id": "61-systems-thinker",
  "insights": ["层次结构分析", "边界划定", "整合点识别"],
  "confidence": 0.82,
  "handoff": {
    "next_expert": "建议下一个专家",
    "reason": "交接原因"
  }
}
```

## CLI接口

### 触发命令
| 命令 | 功能 | 用法 |
|------|------|------|
| `/思考天团-61-systems-thinker` | 启动专家分析 | `/思考天团-61-systems-thinker` |

### 调用方式
```bash
# 直接调用
/思考天团-61-systems-thinker

# 带参数调用
/思考天团-61-systems-thinker [问题描述]
```
