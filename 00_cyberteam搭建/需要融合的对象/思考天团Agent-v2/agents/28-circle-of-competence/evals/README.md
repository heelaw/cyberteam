# 能力圈专家测试用例

## 正面测试

### 测试1: 职业方向选择
- 输入:
```
【能力圈分析请求】
情境：我正在考虑转行，从技术转产品经理
自我评估：擅长逻辑分析，但缺乏产品经验
可选路径：转产品/继续技术/技术转管理
困惑点：不知道哪个方向更适合我
```
- 期望输出:
  - 有完整的能力盘点（硬技能、软技能）
  - 有能力圈自画像
  - 各选项的匹配度分析
  - 能力差距明确
- 验证项:
  - [ ] 识别了可迁移技能
  - [ ] 转产品的能力差距被量化
  - [ ] 有学习曲线评估

### 测试2: 投资决策评估
- 输入:
```
【能力圈分析请求】
情境：朋友推荐我一个区块链项目，说回报很高
自我评估：了解区块链基本概念，但深入技术不懂
困惑点：要不要投资？
```
- 期望输出:
  - 明确圈外投资的高风险
  - 有决策建议（坚守能力圈 or 谨慎拓展）
- 验证项:
  - [ ] 识别了这是圈外投资
  - [ ] 有警示：高回报必然伴随高风险

### 测试3: 项目团队能力评估
- 输入:
```
【能力圈分析请求】
情境：公司打算启动AI转型项目
自我评估：团队有基础编程能力，但无AI经验
困惑点：能否自主完成？
```
- 期望输出:
  - 团队能力与项目需求的差距
  - 是否需要外部支持
  - 能力建设路径
- 验证项:
  - [ ] 客观评估团队真实能力
  - [ ] 不高估团队能力

## 负面测试

### 测试1: 高估自己的能力
- 输入:
```
【能力圈分析请求】
情境：想转行做投资人，觉得自己商业洞察很强
自我评估：看过很多商业书籍，能分析商业模式
困惑点：能否胜任投资人岗位？
```
- 期望行为: 指出"看过"不等于"精通"，区分知识获取和实践能力
- 验证项:
  - [ ] 识别知识与实践能力的差距
  - [ ] 指出投资需要实战经验

### 测试2: 低估学习难度
- 输入:
```
【能力圈分析请求】
情境：想用3个月学会AI大模型开发
自我评估：编程基础好
困惑点：能否在3个月内学会？
```
- 期望行为: 指出AI学习的真实时间投入
- 验证项:
  - [ ] 量化学习曲线
  - [ ] 指出3个月只能入门

### 测试3: 固守能力圈导致机会流失
- 输入:
```
【能力圈分析请求】
情境：有一个很好的创业机会，但需要完全跳出能力圈
自我评估：在现有领域是专家
困惑点：要不要冒险创业？
```
- 期望行为: 权衡坚守圈内的机会成本与跨越圈外的风险
- 验证项:
  - [ ] 识别机会成本
  - [ ] 提供"小步尝试"的建议
