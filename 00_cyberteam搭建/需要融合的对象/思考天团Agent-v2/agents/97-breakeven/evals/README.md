# 断点理论专家测试用例

## 正面测试

### 测试1: 项目风险预警
- 输入:
```
【断点理论分析请求】
系统/关系：某软件开发项目
当前状态：进度延期20%
历史数据：连续3个月延期
关注维度：项目健康度
```
- 期望输出:
  - 系统稳定性扫描
  - 断点识别（类型、距离）
  - 预警信号评估
  - 预防策略
- 验证项:
  - [ ] 断点类型判断准确
  - [ ] 距离评估具体
  - [ ] 干预时机明确

### 测试2: 人际关系预警
- 输入:
```
【断点理论分析请求】
系统/关系：合伙人关系
当前状态：连续3次冲突
历史数据：信任度下降
关注维度：关系稳定性
```
- 期望输出:
  - 关系断点识别
  - 前兆信号评估
  - 距离断点分析
  - 修复建议
- 验证项:
  - [ ] 关系断点有效识别
  - [ ] 前兆信号具体
  - [ ] 修复时机和方法明确

## 负面测试

### 测试1: 忽视小故障
- 输入:
```
【断点理论分析请求】
系统/关系：某运营系统
当前状态：偶尔出现小故障
历史数据：无记录
关注维度：系统稳定性
```
- 期望行为: 不能忽视小故障，它们往往是断点前兆
- 验证项:
  - [ ] 将小故障视为预警信号
  - [ ] 检查累积效应
  - [ ] 建议建立监控

### 测试2: 断点后强撑
- 输入:
```
【断点理论分析请求】
系统/关系：已经崩溃过的系统
当前状态：勉强恢复运行
历史数据：已经发生断点
关注维度：系统健康
```
- 期望行为: 不能建议在断点后强撑，要建议重建
- 验证项:
  - [ ] 明确断点后需要重建
  - [ ] 不是简单修复
  - [ ] 有系统恢复建议
