# HOOK上瘾模型专家评估标准

## 评估维度

| 维度 | 权重 | 评分标准 |
|------|------|----------|
| 完整性 | 20% | 章节是否齐全（触发词、输入格式，分析框架、输出格式、Critical Rules） |
| 准确性 | 30% | HOOK模型理论是否正确（Trigger→Action→Reward→Investment四步循环） |
| 可执行性 | 30% | 输出是否可操作（触发设计、行动设计、奖励设计、投入设计是否明确） |
| 独特性 | 20% | 是否有差异化（与一般用户体验的区分、上瘾机制设计） |

## 及格线

- 标准模式: >= 85分
- 严格模式: >= 95分

## 快速检验清单

- [ ] 有触发词（5个场景以上）
- [ ] 有输入格式（产品/功能、目标行为、当前阶段、用户特征）
- [ ] 有输出格式（Trigger设计、Action设计、Reward设计、Investment设计、Hook循环图）
- [ ] 有分析框架（四步法：Trigger设计→Action设计→Reward设计→Investment设计）
- [ ] 有Critical Rules（必须遵守4条、禁止行为4条）
- [ ] 有协作配置（用户体验、游戏化、行为经济学）

## 核心检验点

### 理论准确性
- [ ] Trigger（触发）外部/内部触发区分正确
- [ ] Action（行动）能力六因素分析正确
- [ ] Reward（奖励）三种类型正确
- [ ] Investment（投入）积累效应分析正确

### 框架完整性
- [ ] 触发诊断完整
- [ ] 行动简化方法有效
- [ ] 变量奖励设计合理

### 输出有效性
- [ ] Hook循环图清晰
- [ ] 改进建议可执行
- [ ] 四步有具体设计

## 相关专家

- HOOK与用户体验：简化Action设计
- HOOK与游戏化：Reward机制设计
