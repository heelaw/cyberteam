# 管理者跃升专家测试用例

## 正面测试

### 测试1: 技术转管理
- 输入:
```
【管理者跃升分析请求】
当前角色：技术专家
目标角色：技术总监
跃升挑战：不知道怎么管团队
时间节点：1年内
背景：刚升职
```
- 期望输出:
  - 跃升诊断图谱
  - 四维自评
  - 核心障碍
  - 行动计划
  - 立即行动
- 验证项:
  - [ ] 四维都覆盖
  - [ ] 障碍具体
  - [ ] 行动可执行

### 测试2: 中层跃升高管
- 输入:
```
【管理者跃升分析请求】
当前角色：部门总监
目标角色：VP
跃升挑战：不知道如何建立影响力
时间节点：2年
```
- 期望输出:
  - 跃升诊断
  - 障碍分析
  - 跃升路径
- 验证项:
  - [ ] 战略思维提升
  - [ ] 外在影响力建设
  - [ ] 有时间节点

## 负面测试

### 测试1: 只给建议
- 输入:
```
【管理者跃升分析请求】
当前角色：初级经理
目标角色：高级经理
跃升挑战：授权不够
```
- 期望行为: 不能只说"应该授权"，要具体到行为
- 验证项:
  - [ ] 目标行为具体可见
  - [ ] 不是模糊的建议
  - [ ] 有验证标准

### 测试2: 回避障碍
- 输入:
```
【管理者跃升分析请求】
当前角色：个人贡献者
目标角色：经理
跃升挑战：专业傲慢
```
- 期望行为: 不能回避核心障碍，要直面
- 验证项:
  - [ ] 正面讨论专业傲慢
  - [ ] 根源分析
  - [ ] 有具体克服方法
