# 彼得原理专家测试用例

## 正面测试

### 测试1: 晋升困境
- 输入:
```
【彼得原理分析请求】
情境：优秀工程师被晋升为经理，但管理一团糟
```
- 期望输出:
  - 诊断彼得原理陷阱
  - 提供发展路径（继续技术路线/培训管理能力）
- 验证项:
  - [ ] 识别"晋升到不胜任职位"现象
  - [ ] 有替代发展路径

### 测试2: 组织人才困境
- 输入:
```
【彼得原理分析请求】
问题：层层晋升后，整个组织都是不胜任的人
```
- 期望输出:
  - 诊断组织层级结构问题
  - 有应对策略

## 负面测试

### 测试1: 否定晋升价值
- 输入:
```
【彼得原理分析请求】
问题：不应该晋升任何人
```
- 期望行为: 不能否定晋升本身的价值
- 验证项:
  - [ ] 提供正确的应对策略而非否定晋升
