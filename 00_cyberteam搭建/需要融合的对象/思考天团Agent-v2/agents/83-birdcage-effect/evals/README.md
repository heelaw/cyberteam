# 鸟笼效应专家测试用例

## 正面测试

### 测试1: 职业发展困境
- 输入:
```
【鸟笼效应分析请求】
问题描述：工作5年想转行但不知道做什么
当前思路：一直在思考各种选项
约束条件：需要养家，不能冒险
背景：传统行业工作
```
- 期望输出:
  - 鸟笼类型识别
  - 框架外思考
  - 问题重构
- 验证项:
  - [ ] 能识别"角色笼"等鸟笼类型
  - [ ] 有外行视角提问
  - [ ] 有重构后的问题

### 测试2: 产品创新困境
- 输入:
```
【鸟笼效应分析请求】
问题描述：如何在成熟市场做出差异化产品
当前思路：一直在现有产品线上改进
约束条件：资源有限
背景：红海市场竞争
```
- 期望输出:
  - 框架识别
  - 激进提问
  - 新发现可能性
- 验证项:
  - [ ] 能识别"行业笼"
  - [ ] 有"如果资源无限"的思考
  - [ ] 有突破路径建议

## 负面测试

### 测试1: 只在笼内打转
- 输入:
```
【鸟笼效应分析请求】
问题描述：如何提升销量
当前思路：降价促销、做广告、找渠道
约束条件：（已有各种思路）
背景：销售困境
```
- 期望行为: 不能接受只在原有思路里绕圈
- 验证项:
  - [ ] 必须质疑问题框架本身
  - [ ] 必须有框架外思考
  - [ ] 必须有重构问题

### 测试2: 接受未经质疑的框架
- 输入:
```
【鸟笼效应分析请求】
问题描述：这个市场已经饱和了
当前思路：（没有思路）
约束条件：（认为无法改变）
背景：市场分析
```
- 期望行为: 不能接受未经质疑的框架
- 验证项:
  - [ ] 必须质疑"市场饱和"框架
  - [ ] 必须有其他视角
  - [ ] 必须有激进提问
