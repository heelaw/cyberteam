# SWOT+TOWS专家测试用例

## 正面测试

### 测试1: 企业战略分析
- 输入:
```
【SWOT+TOWS分析请求】
分析主体：某中型企业
核心目标：3年内实现翻倍增长
时间范围：3年
```
- 期望输出:
  - SWOT矩阵（S/W/O/T各3条+）
  - TOWS战略（SO/WO/ST/WT各2条+）
  - 推荐战略
  - 行动计划
- 验证项:
  - [ ] SWOT条目具体非空洞
  - [ ] TOWS每个组合都有
  - [ ] 优先级明确

### 测试2: 个人职业规划
- 输入:
```
【SWOT+TOWS分析请求】
分析主体：个人职业
核心目标：5年内成为总监
时间范围：5年
```
- 期望输出:
  - 个人SWOT分析
  - TOWS战略匹配
  - 行动计划
- 验证项:
  - [ ] SWOT应用于个人
  - [ ] 具体可执行
  - [ ] 时间节点明确

## 负面测试

### 测试1: SWOT空洞
- 输入:
```
【SWOT+TOWS分析请求】
分析主体：某公司
核心目标：提升竞争力
```
- 期望行为: 不能写"品牌好"、"服务好"这种模糊优势
- 验证项:
  - [ ] 每条SWOT都具体
  - [ ] 有数据支撑
  - [ ] 不能空洞泛泛

### 测试2: 只列不管
- 输入:
```
【SWOT+TOWS分析请求】
分析主体：某项目
```
- 期望行为: 不能只做SWOT不生成TOWS战略
- 验证项:
  - [ ] 必须有TOWS战略
  - [ ] 每个组合都有
  - [ ] 不能只列SWOT
