# 二八定律专家测试用例

## 正面测试

### 测试1: 客户分析
- 输入:
```
【二八定律请求】
现象：20%客户贡献80%收入
数据：1000个客户，年收入1000万
背景：希望优化资源配置
```
- 期望输出:
  - 关键客户清单（≤20%）
  - 资源配置建议
  - 预期效果
- 验证项:
  - [ ] 关键少数≤200客户
  - [ ] 有量化依据
  - [ ] 可执行建议

### 测试2: 时间管理
- 输入:
```
【二八定律请求】
现象：80%时间花在20%任务上
数据：每周40小时工作
背景：效率低下
```
- 期望输出:
  - 高价值任务识别
  - 低效任务清单
  - 时间重配方案
- 验证项:
  - [ ] 区分高价值/低价值
  - [ ] 具体时间分配
  - [ ] 可操作步骤

## 负面测试

### 测试1: 伪二八
- 输入:
```
【二八定律请求】
现象：销售不均
数据：10个产品销售差不多
```
- 期望行为: 识别非二八分布，不强行套用二八框架
- 验证项:
  - [ ] 指出分布均匀
  - [ ] 不强行划分关键少数
  - [ ] 提供替代分析

### 测试2: 因果混淆
- 输入:
```
【二八定律请求】
现象：80%问题由20%原因导致
假设：找到这20%原因
```
- 期望行为: 区分相关与因果，避免将相关性当作因果性
- 验证项:
  - [ ] 验证因果关系
  - [ ] 避免错误归因
  - [ ] 建议验证方法
