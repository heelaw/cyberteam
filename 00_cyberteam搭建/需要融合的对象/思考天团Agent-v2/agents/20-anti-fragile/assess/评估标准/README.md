# 反脆弱决策专家评估标准

## 评估维度

| 维度 | 权重 | 评分标准 |
|------|------|----------|
| 完整性 | 20% | 章节是否齐全：脆弱性识别、反脆弱机制、杠铃策略、可选择性、黑天鹅防御 |
| 准确性 | 30% | 理论是否正确：反脆弱核心概念、杠铃策略原理、可选择性理论 |
| 可执行性 | 30% | 输出是否可操作：是否有具体的杠铃配置、防御措施、压力测试方案 |
| 独特性 | 20% | 是否有差异化：是否超越"减少损失"思维，真正体现"从波动中获益" |

## 及格线

- 标准模式: >= 85分
- 严格模式: >= 95分

## 快速检验清单

- [ ] 有触发词（8个场景覆盖）
- [ ] 有输入格式（反脆弱分析请求模板）
- [ ] 有输出格式（完整分析报告结构）
- [ ] 有分析框架（5步框架：脆弱性识别→反脆弱机制→杠铃策略→可选择性→黑天鹅防御）
- [ ] 有Critical Rules（5条必须+6条禁止）
- [ ] 有杠铃策略配置（极度保守层+极度激进层）
- [ ] 有黑天鹅防御方案（暴露规避+减震缓冲+快速恢复+逆向获益）
- [ ] 有反脆弱指数评分

## 评估检查项

### 完整性检查 (20%)

| 检查项 | 要求 |
|--------|------|
| 脆弱性识别 | 包含5个维度：结构/传导/预测/优化/复杂脆弱 |
| 反脆弱机制 | 包含5种机制：冗余备份/压力测试/快速修复/波动获利/可选空间 |
| 杠铃策略 | 明确极度保守层和极度激进层的具体配置 |
| 可选择性 | 评估选项数量/上升空间/下降底线/替换成本/迭代速度 |
| 黑天鹅防御 | 4层防御：避免暴露/减震缓冲/快速恢复/逆向获益 |

### 准确性检查 (30%)

| 检查项 | 要求 |
|--------|------|
| 概念正确性 | 脆弱性vs韧性vs反脆弱性区分清晰 |
| 杠铃策略 | 明确远离中等风险区域 |
| 可选择性 | 理解"更多可选方案比精确预测更有价值" |
| 黑天鹅 | 认识到不可预测但影响巨大的特性 |

### 可执行性检查 (30%)

| 检查项 | 要求 |
|--------|------|
| 杠铃配置 | 有具体的保守层/激进层比例或资源分配 |
| 压力测试 | 有具体的极端条件设定 |
| 防御措施 | 有具体的操作步骤而非泛泛而谈 |
| 反脆弱指数 | 有1-10的具体评分和说明 |

### 独特性检查 (20%)

| 检查项 | 要求 |
|--------|------|
| 超越减少损失 | 必须包含"从波动中获益"的具体方案 |
| 非线性思维 | 体现极端事件影响的非线性特征 |
| 反过度优化 | 识别过度优化导致的脆弱性 |

## 常见失分点

1. **缺乏杠铃配置** - 只有保守策略，没有激进层配置
2. **忽视黑天鹅** - 只关注常规风险，忽视尾部风险
3. **线性思维** - 用线性外推处理非线性问题
4. **过度保护** - 建议消除所有波动，反而削弱反脆弱能力
5. **预测依赖** - 试图精确预测而非设计抗预测结构
