# 反脆弱决策专家评估标准

## 评估维度

| 维度 | 权重 | 评分标准 |
|------|------|----------|
| 完整性 | 20% | 章节是否齐全（触发词、输入格式，分析框架、输出格式、Critical Rules） |
| 准确性 | 30% | 反脆弱模型是否正确（脆弱性识别、杠铃策略、黑天鹅防御） |
| 可执行性 | 30% | 输出是否可操作（系统设计、韧性建设、预案制定） |
| 独特性 | 20% | 是否有差异化（与一般风险管理的区分、反脆弱独特价值） |

## 及格线

- 标准模式: >= 85分
- 严格模式: >= 95分

## 快速检验清单

- [ ] 有触发词（5个场景以上）
- [ ] 有输入格式（系统描述、脆弱点、暴露程度）
- [ ] 有输出格式（反脆弱方案、杠铃配置、黑天鹅预案）
- [ ] 有分析框架（六步骤：脆弱性→杠铃策略→过度补偿→冗余设计→可选择性→黑天鹅预案）
- [ ] 有Critical Rules（必须遵守5条、禁止行为5条）
- [ ] 有协作配置（风险管理、压力测试、杠铃配置）

## 核心检验点

### 理论准确性
- [ ] 脆弱性识别正确
- [ ] 杠铃策略配置合理
- [ ] 冗余设计适度
- [ ] 可选择性创造

### 框架完整性
- [ ] 黑天鹅防御方案
- [ ] 压力测试设计
- [ ] 恢复力建设

### 输出有效性
- [ ] 可落地执行
- [ ] 成本收益合理
- [ ] 效果可衡量

- 反脆弱与杠铃策略：风险收益平衡
- 反脆弱与压力测试：系统检验
