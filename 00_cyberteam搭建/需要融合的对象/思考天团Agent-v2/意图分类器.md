## 路由决策算法

### 第一步：提取关键词

```
输入：用户问题
处理：分词 → 去停用词 → 提取关键词
输出：关键词列表 + 出现位置
```

### 第二步：匹配类型

```
对每个类型T：
  score[T] = Σ keyword_match_score × position_weight × context_weight

输出：得分最高的类型 + 置信度
```

### 第三步：选择专家

```
IF 置信度 > 0.8:
  选择该类型的主要专家（1-2个）
ELIF 置信度 > 0.5:
  选择该类型专家 + 一个相关类型专家
ELSE:
  选择2-3个相关类型专家进行并行分析
```

## 默认行为

### 问句类型推断

| 问句模式 | 默认类型 | 默认专家 |
|----------|----------|----------|
| "我应该..." | T1 决策选择 | kahneman |
| "为什么..." | T2 原因诊断 | fivewhy |
| "如何实现..." | T3 目标实现 | grow |
| " ... 好吗" | T1 决策选择 | kahneman, ai-board |
| " ... 怎么样" | T6 多角度 | six-hats |
| "有没有办法..." | T5 创新突破 | first-principle |

### 无匹配时的默认处理

```
未匹配任何关键词时：
→ 使用T6（多角度分析）
→ 调用six-hats + kahneman（2个专家）
```

## 复合问题处理

### 识别复合问题

- 问题中包含多个"和"、"或者"、"还是"
- 问题包含多个不同类型的关键词

### 复合问题路由规则

```
IF 复合问题包含 "选择" + "原因":
  → 主要类型T1，次要参考T2
  → 主要专家kahneman，次要fivewhy

IF 复合问题包含 "目标" + "风险":
  → 主要类型T3，次要参考T1
  → 主要专家grow，次要kahneman

IF 复合问题包含 "创新" + "战略":
  → 主要类型T5，次要参考T4
  → 主要专家first-principle，次要swot-tows
```
