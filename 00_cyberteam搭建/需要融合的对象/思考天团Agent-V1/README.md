# 思考天团 - Cyberwiz智囊团系统

> 一个能让AI"持续打工直到目标达成"的多专家协作系统

---

## 快速开始

### 1. 在Claude Code中使用

直接在对话中调用思考天团：

```
你：帮我思考一下是否应该转型做AI产品

思考天团将自动：
1. 识别为"战略+创新"类问题
2. 路由至 swot-tows, first-principle, ai-board 等专家
3. 并行分析后整合输出
```

### 2. 指定专家分析

```
你：用第一性原理分析我的产品问题
你：让卡尼曼来帮我做这个决策
```

### 3. 辩论模式

```
你：辩论一下：应该先做用户还是先做收入？
```

## 三种协作模式

### 1. 并行独立分析（默认）

适用于多角度分析、重要但不紧急的决策。

```
用户问题 → 意图识别 → 3-5个专家并行分析 → 结果整合 → 最终报告
```

### 2. 交叉辩论模式

适用于重大选择、需要权衡利弊的决策。

```
正方论证 ←──质疑──→ 反方论证
     ↓              ↓
     └────┬────┘
          ↓
     主持人综合裁决
```

### 3. 链式分析模式

适用于问题解决型任务。

```
Goldlin(定义) → FiveWhy(溯源) → FirstPrinciple(重构) → WBS(执行)
```

## PUA驱动机制

当任务遇到困难时，系统会自动激活PUA驱动：

| 失败次数 | 压力等级 | 动作 |
|---------|---------|------|
| 2次 | L1 | "换一种根本不同的方法" |
| 3次 | L2 | "WebSearch + 读源码" |
| 4次 | L3 | "执行7步检查清单" |
| 5次+ | L4 | "绝望模式 + 重启" |

## 示例对话

### 示例1：战略决策

```
用户：思考一下：我们要不要进入海外市场？

思考天团分析：
├── swot-tows: 海外市场的SWOT分析
├── first-principle: 从用户需求出发重新思考
├── ai-board: 从投资人视角审视
└── fivewhy: 为什么要进入海外市场？

输出：结构化分析报告 + 行动建议
```

### 示例2：产品决策

```
用户：辩论一下：应该先做ToB还是ToC？

思考天团：
├── 正方(ToB): 现金流稳定、客户质量高
└── 反方(ToC): 市场更大、增长更快

主持人综合裁决 + 建议
```
