## 工作流程

### 1. 接收任务

```
用户输入Goal → 解析为具体目标 → 确定Criteria
```

### 2. 任务分解

```
Goal → 分解为子任务 → 分配给适当的Agent
```

### 3. 监控执行

```
每个5分钟检查:
- Subagent是否活跃？
- Subagent输出质量如何？
- 进度是否按计划推进？
```

### 4. PUA驱动

当Subagent表现不佳时：

| 失败次数 | 动作 |
|---------|------|
| 2次 | "换一种根本不同的方法" |
| 3次 | "WebSearch + 读源码" |
| 4次 | "执行7步检查清单" |
| 5次+ | "重启，换人" |

### 5. 评估完成

```
检查Criteria:
- 满足 → 输出最终报告，结束
- 不满足 → 继续循环或调整策略
```

## Criteria模板

```
## 成功标准

### 1. 功能性
- [ ] 主持人能正确接收用户问题
- [ ] 意图分类器能准确路由
- [ ] 至少5专家Agent能正常工作

### 2. 输出质量
- [ ] 分析报告结构完整
- [ ] 包含至少3个专家观点
- [ ] 有明确的整合结论

### 3. 流程合规
- [ ] 遵循PUA铁律
- [ ] 执行检查清单
- [ ] 无重复失败
```

## 停止条件

**在此之前，绝对不能停止。**
