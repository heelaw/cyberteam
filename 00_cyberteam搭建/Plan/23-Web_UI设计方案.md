# CyberTeam Web UI 设计方案

**版本**: v1.0
**日期**: 2026-03-24
**设计风格**: 军机处风格 + Claude HUD 融合
**参考系统**: edict 三省六部 / clawteam-v2 HUD

---

## 一、技术栈选择

### 核心方案：单文件HTML + Python标准库后端

| 层级 | 技术选型 | 理由 |
|------|----------|------|
| **前端** | 原生 HTML/CSS/JS (无框架) | 零依赖、易部署、单文件维护 |
| **后端** | Python stdlib only | 参考 edict server.py，零第三方依赖 |
| **通信** | Fetch API + SSE | 实时状态推送 |
| **图标** | Emoji + CSS | 军机处风格，无需外部字体 |
| **构建** | 无需构建 | 直接运行 |

### 备选方案（可选升级路径）

| 层级 | 备选技术 | 何时升级 |
|------|----------|----------|
| 前端 | Vue 3 / React | UI复杂度超过3000行时 |
| 后端 | FastAPI | 需要WebSocket时 |
| 样式 | Tailwind CSS | 需多人协作时 |

### 设计语言

```
颜色变量:
  --bg:       #07090f    (深空背景)
  --panel:    #0f1219    (面板背景)
```

## 五、数据流设计

### 整体架构

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   前端UI    │ ←→  │  Python后端  │ ←→  │  数据存储    │
│  dashboard  │     │   server    │     │  JSON文件    │
│   .html     │     │   .py        │     │             │
└─────────────┘     └─────────────┘     └─────────────┘
      │                   │                   │
      │  Fetch/SSE        │                   │
      └───────────────────┴───────────────────┘
                     REST API
```

### API 设计

| 方法 | 路径 | 功能 |
|------|------|------|
| `GET` | `/` | 返回 dashboard.html |
| `GET` | `/api/status` | 全局状态 |
| `GET` | `/api/tasks` | 任务列表 |
| `POST` | `/api/tasks` | 创建任务 |
| `PUT` | `/api/tasks/:id` | 更新任务 |
| `GET` | `/api/agents` | Agent状态 |
| `GET` | `/api/skills` | 技能库 |
| `GET` | `/api/archive` | 归档列表 |
| `GET` | `/api/news` | 天下要闻 |
| `SSE` | `/api/stream` | 实时推送 |

### 数据文件

```
cyberteam_data/
├── tasks.json          # 任务数据
├── agents.json         # Agent状态
├── skills.json         # 技能库
├── archive.json        # 归档记录
├── news.json           # 资讯缓存
├── config.json         # 模型配置
└── logs/
    └── events.json     # 事件日志
```

### 实时推送 (SSE)

```python
# 后端推送示例
def sse_stream():
    yield "data: {\"type\": \"heartbeat\", \"agent\": \"gsd-planner\", \"status\": \"working\"}\n\n# 前端接收
const eventSource = new EventSource('/api/stream');
eventSource.onmessage = (e) => {
    const data = JSON.parse(e.data);
    updateUI(data);
};
```
