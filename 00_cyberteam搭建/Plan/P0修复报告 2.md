### C1: Q64/Q65任务状态模型不兼容

**问题描述**:
- Q64定义的TaskStatus: `CREATED, QUEUED, RUNNING, WAITING_DEPS, PAUSED, COMPLETED, FAILED, CANCELLED`
- Q65的QueuedTask status: `queued, running, completed, failed, cancelled`
- 两者不兼容，导致状态无法同步

**修复方案**:

统一状态模型，以Q64的TaskStatus为准:

```python
# Q65 task_queue.py - 修改 TaskStatus 导入
from Q64.expert_agent_protocol.task import TaskStatus, ExpertTask

# Q65 QueuedTask 改用 Q64 的状态
@dataclass
class QueuedTask:
    task_id: str = field(default_factory=lambda: str(uuid.uuid4()))
    expert_id: str = ""
    question_id: str = ""
    prompt: str = ""
    priority: TaskPriority = TaskPriority.NORMAL
    dependencies: List[str] = field(default_factory=list)
    created_at: float = field(default_factory=time.time)
    # 改用 Q64 的 TaskStatus
    status: TaskStatus = TaskStatus.CREATED

    def __lt__(self, other):
        if self.priority != other.priority:
            return self.priority.value > other.priority.value
        return self.created_at < other.created_at
```

**状态映射表**:

| Q65旧状态 | Q64新状态 | 说明 |
|------------|-----------|------|
| queued | QUEUED | 已入队 |
| running | RUNNING | 执行中 |
| completed | COMPLETED | 已完成 |
| failed | FAILED | 失败 |
| cancelled | CANCELLED | 已取消 |
| (无) | WAITING_DEPS | 等待依赖 |
| (无) | PAUSED | 已暂停 |
| (无) | CREATED | 已创建 |

### C2: TaskStore定义但从未被Q65使用

**问题描述**:
- Q64精确定义了TaskStore持久化接口
- Q65的ParallelExecutor使用内存中的TaskQueue
- TaskStore从未被实际调用

**修复方案**:

在Q65的ParallelExecutor中集成TaskStore:

```python
# Q65 parallel_executor.py
from Q64.expert_agent_protocol.task import TaskStore, ExpertTask, TaskStatus

class ParallelExecutor:
    def __init__(
        self,
        max_concurrent: int = 4,
        timeout_config: TimeoutConfig = None,
        output_dir: str = "thinking-team/workspace/tasks",
        storage_path: str = "thinking-team/workspace/tasks"  # 新增参数
    ):
        self.max_concurrent = max_concurrent
        self.timeout_config = timeout_config or TimeoutConfig()

        # 使用 Q64 的 TaskStore
        self.task_store = TaskStore(storage_path)
        self.task_queue = TaskQueue()
        self.scheduler = TaskScheduler(max_concurrent=max_concurrent)
        self.timeout_controller = TimeoutController(self.timeout_config)

        self._running_tasks: Dict[str, str] = {}

    def _sync_to_task_store(self, task_id: str, expert_id: str, status: TaskStatus):
        """同步任务状态到TaskStore"""
        task = self.task_store.get_task(task_id)
        if task:
            task.status = status
            self.task_store.update_task(task)
```

### C5: Q66 L3未调用Q70 UnifiedQualityScorer

**问题描述**:
- Q66 L3的ResultReviewer有独立的_score_completeness等方法
- Q70提供了统一的UnifiedQualityScorer
- 两者维度权重不一致，"统一"名存实亡

**修复方案**:

Q66 L3改用Q70的UnifiedQualityScorer:

```python
# Q66 result_reviewer.py - 修改 L3 实现
from Q70.unified_quality_scorer import UnifiedQualityScorer, QualityScore

class ResultReviewer:
    """L3: 结果评审 - 改用Q70统一评分器"""

    def __init__(self):
        # 使用 Q70 的统一评分器
        self.unified_scorer = UnifiedQualityScorer()

    def review(self, outputs: Dict[str, Dict]) -> ReviewResult:
        """评审专家输出"""

        # 调用 Q70 统一评分
        combined_output = {
            "output": "\n".join(o.get("output", "") for o in outputs.values()),
            "expert_outputs": outputs
        }

        quality_score: QualityScore = self.unified_scorer.score(
            combined_output,
            stage="result_review"
        )

        # 转换格式
        dimension_scores = {
            "completeness": quality_score.breakdown.get("completeness", 0),
            "professionalism": quality_score.breakdown.get("professionalism", 0),
            "practicality": quality_score.breakdown.get("practicality", 0),
            "logic": quality_score.breakdown.get("logic", 0),
            "innovation": quality_score.breakdown.get("innovation", 0),
            "safety": quality_score.breakdown.get("safety", 0)
        }

        total_score = quality_score.total

        # 判断状态
        if total_score >= self.PASS_SCORE:
            status = ReviewStatus.PASS
        elif total_score >= self.REVISION_THRESHOLD:
            status = ReviewStatus.NEEDS_REVISION
        else:
            status = ReviewStatus.FAIL

        return ReviewResult(
            status=status,
            score=total_score,
            dimensions=dimension_scores,
            feedback=self._generate_feedback(dimension_scores, outputs),
            issues=quality_score.issues
        )
```

### C6: TimeoutController.remaining()方法缺失

**问题描述**:
- Q65的TimeoutController只有get_elapsed()方法
- 没有remaining()方法获取剩余时间
- C6提到is_alive()返回bool的问题（但实际代码没有is_alive()）

**修复方案**:

在Q65 TimeoutController中添加remaining()方法:

```python
# Q65 timeout_controller.py
class TimeoutController:
    def __init__(self, config: TimeoutConfig = None):
        self.config = config or TimeoutConfig()
        self._timers: Dict[str, threading.Timer] = {}
        self._callbacks: Dict[str, Callable] = {}
        self._task_start_times: Dict[str, float] = {}
        self._original_timeouts: Dict[str, float] = {}

    def remaining(self, task_id: str) -> float:
        """
        获取任务剩余时间（秒）

        Returns:
            剩余时间，如果任务不存在返回0
        """
        if task_id not in self._task_start_times:
            return 0

        if task_id not in self._original_timeouts:
            return 0

        elapsed = self.get_elapsed(task_id)
        original_timeout = self._original_timeouts[task_id]

        return max(0, original_timeout - elapsed)

    def get_elapsed(self, task_id: str) -> float:
        """获取已用时间"""
        start = self._task_start_times.get(task_id)
        if start is None:
            return 0
        return time.time() - start

    def extend(self, task_id: str, extra_time: float):
        """延长超时时间"""
        if task_id in self._timers:
            self._timers[task_id].cancel()

            elapsed = self.get_elapsed(task_id)
            original_timeout = self._original_timeouts.get(task_id, 300)
            remaining = original_timeout - elapsed

            new_timeout = max(remaining + extra_time, 60)
            timer = threading.Timer(new_timeout, self._on_timeout, args=(task_id,))
            timer.daemon = True
            self._timers[task_id] = timer
            self._original_timeouts[task_id] = new_timeout
            timer.start()
```
