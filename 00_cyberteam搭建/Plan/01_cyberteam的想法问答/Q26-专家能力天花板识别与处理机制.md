## 一、问题背景

### 1.1 核心挑战

14个思维专家各有能力边界（Q23已分析），当问题超出专家能力范围时：
- 直接拒绝用户体验差
- 强制回答质量差
- 需要智能识别和优雅处理

### 1.2 具体场景示例

| 场景 | 问题 | 超出类型 |
|------|------|----------|
| Kahneman + OAuth2 | "如何实现OAuth2？" | 领域跨界（心理学→技术） |

| GROW + 无目标 | "我不知道该做什么" | 前置条件缺失 |

## 二、能力天花板量化模型

### 2.1 天花板定义

**能力天花板** = 问题所需能力与专家实际能力的差距

```
天花板高度 = 问题复杂度 - 专家能力
天花板高度 > 0 → 超出能力范围
```

### 2.2 量化维度

每个问题可以用5个维度衡量：

| 维度 | 英文 | 说明 | 取值范围 |
|------|------|------|----------|
| 领域跨度 | Domain Span | 问题涉及的专业领域数量 | 1-N |
| 复杂度 | Complexity | 问题的逻辑复杂度 | 1-5 |
| 前置条件 | Prerequisites | 问题是否满足专家工作前置条件 | 0/1 |
| 意义性 | Meaningfulness | 问题是否有明确答案 | 0/1 |
| 时效性 | Timeliness | 问题是否有时效要求 | 0/1 |

### 2.3 专家能力画像

每个专家的能力用以下指标描述：

```json
{
  "expert_id": "kahneman",
  "primary_domains": ["psychology", "decision-making", "cognitive-bias"],
  "secondary_domains": ["behavioral-economics"],
  "can_handle_technical": false,
  "can_handle_creative": false,
  "requires_prerequisite": true,
  "required_prerequisite": "has_decision_point",
  "min_complexity": 2,
  "max_complexity": 5,
  "meaning_threshold": 0.5,
  "boundary_rules": [
    {"type": "domain_forbidden", "value": "technical_implementation"},
    {"type": "complexity_too_low", "threshold": 2}
  ]
}
```

### 2.4 天花板等级

| 天花板等级 | 天花板高度 | 识别特征 | 推荐处理方式 |
|------------|------------|----------|--------------|
| **L0 无天花板** | 0 | 问题完全在能力内 | 正常处理 |
| **L1 轻微超限** | 1 | 领域轻微跨界 | 扩展处理（降级） |
| **L2 中度超限** | 2 | 能力完全不匹配 | 路由/转介 |
| **L3 重度超限** | 3+ | 完全无意义/前置条件缺失 | 拒绝+引导 |

## 三、实时识别算法

### 3.1 识别流程总览

```
用户问题输入
    │
    ├── 第一层：基础过滤
    │   ├── 是否有效问题？(Meaningfulness)
    │   ├── 是否简单问题？(30秒内有答案)
    │   └── 是否闲聊？
    │
    ├── 第二层：前置条件检查
    │   └── 专家工作所需前提是否存在？
    │
    ├── 第三层：领域匹配
    │   └── 问题领域 ∈ 专家能力领域？
    │
    ├── 第四层：复杂度匹配
    │   └── 问题复杂度 ∈ 专家能力范围？
    │
    └── 输出：天花板等级 + 处理建议
```

### 3.2 第一层：基础过滤

```python
def basic_filter(question: str) -> FilterResult:
    """
    第一层：过滤无意义、简单、闲聊问题
    """

    # 1.1 检查问题长度
    if len(question.strip()) < 5:
        return FilterResult(
            passed=False,
            reason="问题过短",
            suggestion="请提供更具体的问题"
        )

    # 1.2 检查是否闲聊
    greetings = ["你好", "hello", "hi", "在吗", "忙吗"]
    if any(g in question.lower() for g in greetings):
        return FilterResult(
            passed=False,
            reason="闲聊",
            suggestion="请直接提出你的问题"
        )

    # 1.3 检查是否简单问题（30秒内有答案）
    if is_simple_question(question):
        return FilterResult(
            passed=True,
            skip_expert=True,
            suggestion="简单问题可直接回答"
        )

    # 1.4 检查是否有意义
    if not has_meaning(question):
        return FilterResult(
            passed=False,
            reason="无意义问题",
            suggestion="请提出有明确目标的问题"
        )

    return FilterResult(passed=True)
```

### 3.4 第三层：领域匹配

```python
def check_domain_match(question: str, expert: str) -> DomainCheckResult:
    """
    第三层：检查问题领域是否在专家能力内
    """

    # 定义各专家的能力领域
    EXPERT_DOMAINS = {
        "kahneman": {
            "primary": ["psychology", "decision", "cognitive-bias", "behavioral"],
            "forbidden": ["technical", "code", "programming", "implementation"]
        },
        "first_principle": {
            "primary": ["innovation", "restructuring", "breakthrough"],
            "forbidden": ["routine", "execution", "daily-operation"]
        },
        "manager_leap": {
            "primary": ["management", "leadership", "organization", "team"],
            "forbidden": ["technical", "code", "sql", "database", "programming"]
        },
        "ai_board": {
            "primary": ["ai", "technology", "machine-learning", "algorithm"],
            "forbidden": ["traditional-industry", "manufacturing", "operations"]
        },
        # ... 其他专家
    }

    # 提取问题中的领域关键词
    question_domains = extract_domains(question)

    expert_domains = EXPERT_DOMAINS[expert]

    # 检查是否在禁止领域
    forbidden_match = question_domains & set(expert_domains.get("forbidden", []))
    if forbidden_match:
        return DomainCheckResult(
            passed=False,
            ceiling_level=2,
            reason=f"问题涉及禁止领域: {forbidden_match}",
            suggestion=f"此问题属于技术领域，不适合{expert}专家"
        )

    # 检查是否在主领域
    primary_match = question_domains & set(expert_domains.get("primary", []))
    if primary_match:
        return DomainCheckResult(
            passed=True,
            ceiling_level=0,
            matched_domains=primary_match
        )

    # 不在任何领域
    return DomainCheckResult(
        passed=False,
        ceiling_level=1,
        reason="问题不在专家能力领域内",
        suggestion=f"此问题领域与{expert}专长不匹配"
    )
```

### 3.5 第四层：复杂度匹配

```python
def check_complexity_match(question: str, expert: str) -> ComplexityCheckResult:
    """
    第四层：检查问题复杂度是否在专家能力范围内
    """

    COMPLEXITY_RANGES = {
        "kahneman": {"min": 2, "max": 5},      # 需要有选择困境
        "first_principle": {"min": 3, "max": 5},  # 需要创新性
        "six_hats": {"min": 3, "max": 5},      # 需要多视角
        "grow": {"min": 1, "max": 5},
        "five_why": {"min": 2, "max": 5},
        "goldlin": {"min": 1, "max": 5},
        "swot_tows": {"min": 2, "max": 5},
        "mckinsey": {"min": 2, "max": 5},
        "wbs": {"min": 2, "max": 5},
        "kiss": {"min": 3, "max": 5},          # 需要有复杂度才能简化
        "reverse_thinking": {"min": 2, "max": 5},
        "five_dimension": {"min": 3, "max": 5},  # 需要多维度
        "ai_board": {"min": 2, "max": 5},
        "manager_leap": {"min": 2, "max": 5}
    }

    # 计算问题复杂度
    question_complexity = calculate_complexity(question)
    allowed_range = COMPLEXITY_RANGES.get(expert, {"min": 1, "max": 5})

    if question_complexity < allowed_range["min"]:
        return ComplexityCheckResult(
            passed=False,
            ceiling_level=1,
            reason=f"问题过于简单（复杂度{question_complexity} < 最低要求{allowed_range['min']}）",
            suggestion=f"此问题过于简单，不需要{expert}专家处理"
        )

    if question_complexity > allowed_range["max"]:
        return ComplexityCheckResult(
            passed=False,
            ceiling_level=2,
            reason=f"问题过于复杂（复杂度{question_complexity} > 最高能力{allowed_range['max']}）",
            suggestion="此问题超出专家处理能力，建议升级处理"
        )

    return ComplexityCheckResult(
        passed=True,
        ceiling_level=0,
        complexity=question_complexity
    )
```

### 3.6 综合识别引擎

```python
def detect_ceiling(question: str, expert: str) -> CeilingDetectionResult:
    """
    综合天花板检测引擎
    执行4层检测，返回天花板等级和处理建议
    """

    # 第一层：基础过滤
    basic_result = basic_filter(question)
    if not basic_result.passed:
        return CeilingDetectionResult(
            ceiling_level=3,
            detection_layers_passed=0,
            reason=basic_result.reason,
            suggestion=basic_result.suggestion,
            action="reject"
        )

    # 第二层：前置条件检查
    prereq_result = check_prerequisite(question, expert)
    if not prereq_result.passed:
        return CeilingDetectionResult(
            ceiling_level=2,
            detection_layers_passed=1,
            reason=f"前置条件缺失 - {prereq_result.required}",
            suggestion=prereq_result.suggestion,
            action="guide_prerequisite"
        )

    # 第三层：领域匹配
    domain_result = check_domain_match(question, expert)
    if not domain_result.passed:
        return CeilingDetectionResult(
            ceiling_level=domain_result.ceiling_level,
            detection_layers_passed=2,
            reason=domain_result.reason,
            suggestion=domain_result.suggestion,
            action="route_to_other",
            recommended_route=domain_result.get("recommended_expert")
        )

    # 第四层：复杂度匹配
    complexity_result = check_complexity_match(question, expert)
    if not complexity_result.passed:
        return CeilingDetectionResult(
            ceiling_level=complexity_result.ceiling_level,
            detection_layers_passed=3,
            reason=complexity_result.reason,
            suggestion=complexity_result.suggestion,
            action="degrade"
        )

    # 全部通过
    return CeilingDetectionResult(
        ceiling_level=0,
        detection_layers_passed=4,
        reason="问题完全在能力范围内",
        action="process_normally"
    )
```

## 四、处理策略

### 4.1 处理策略总览

| 天花板等级 | 行为 | 处理方式 | 响应模板 |
|------------|------|----------|----------|
| **L0 无天花板** | 正常处理 | 直接调用专家 | 专家标准响应 |
| **L1 轻微超限** | 降级处理 | 简化版专家能力 | 缩小范围处理 |
| **L2 中度超限** | 路由转介 | 引导至其他专家 | 推荐专家+原因 |
| **L3 重度超限** | 拒绝引导 | 拒绝+补充信息引导 | 拒绝+补充问题引导 |

### 4.2 L0: 正常处理

**触发条件**: ceiling_level = 0

**处理流程**:
```
直接调用对应专家
    ↓
返回标准响应
```

**响应示例**（Kahneman处理决策问题）:
```
用户: "我该选择创业还是继续打工？"
→ 调用Kahneman
→ 返回决策分析
```

### 4.3 L1: 降级处理（缩小范围）

**触发条件**: ceiling_level = 1

**适用场景**:
- 问题复杂度略低于专家要求
- 问题领域略有跨界但核心能力可覆盖

**处理流程**:
```
识别问题核心能力
    ↓
使用专家核心能力处理
    ↓
明确告知处理范围
```

**响应模板**:
```
[开场说明] 这个问题部分超出我的专业范围，我将[缩小范围]处理。
[核心处理] [使用专家核心能力进行分析]
[范围说明] 注意：本分析仅从[专家领域]角度，不涉及[跨界领域]
[建议] 如需[跨界领域]建议，建议咨询[推荐专家]
```

**响应示例**（Kahneman处理略跨界问题）:
```
用户: "我该选择A公司还是B公司？工资差不多，但A公司文化好，B公司成长快"

[开场说明] 这个问题涉及职业选择，我将只从决策心理学角度分析。

[核心处理] 从认知偏差角度看：
1. 锚定效应：你可能过度关注最初接触的信息
2. 损失厌恶：描述方式会影响你的选择
3. 框架效应：同样的选择用不同方式描述，感受不同

[范围说明] 本分析仅从心理角度，不涉及具体行业前景分析

[建议] 如需行业前景分析，建议咨询AIboard
```

### 4.4 L2: 路由转介

**触发条件**: ceiling_level = 2

**适用场景**:
- 问题完全在禁止领域
- 问题领域与专家能力完全不匹配

**处理流程**:
```
识别问题真实领域
    ↓
查找匹配专家
    ↓
提供路由建议
```

**响应模板**:
```
[现状说明] 您的问题涉及[问题领域]，这超出了我的专业范围。
[原因解释] 我专精于[专家领域]，对[问题领域]不擅长。
[推荐建议] 建议您咨询[推荐专家]：
  - [推荐专家1]：[推荐理由]
  - [推荐专家2]：[推荐理由]
[替代方案] 如果您想从[共同角度]探讨，我也可以协助
```

**响应示例**（Kahneman处理技术问题）:
```
用户: "如何实现OAuth2？"

[现状说明] 您的问题涉及技术实现，这超出了我的专业范围。

[原因解释] 我专精于决策心理学和认知偏差，对编程实现不擅长。

[推荐建议] 建议您咨询以下专家：
  - AIboard：适合AI/技术相关问题
  - 技术实现专家：适合代码开发问题

[替代方案] 如果您在选择认证方案时有困惑，我可以从用户心理角度提供建议
```

### 4.5 L3: 拒绝引导

- 缺少关键前置条件

```

    ↓
提供补充问题引导
    ↓

```

**响应模板**:
```

[原因说明] 因为：[具体原因]
[引导提问] 为了更好地帮助您，请补充以下信息：
  1. [缺失信息1]
  2. [缺失信息2]
  3. [缺失信息3]
[示例] 例如："我想[目标]，目前有[约束]，该如何[行动]？"
```

### 8.1 与Q23路由系统的关系

| Q23 | Q25 |
|-----|-----|
| 定义14个专家的能力边界 | 检测具体问题是否超出边界 |

**集成方式**: Q25作为Q23路由后的二次保障

```
用户问题
    ↓
Q23路由系统 → 选择推荐专家
    ↓
Q25检测引擎 → 验证问题是否在专家能力内
    ↓
    ├── 在能力内 → 专家处理
    ├── 轻微超限 → 降级处理+说明
    ├── 中度超限 → 路由到其他专家
    └── 重度超限 → 拒绝+引导
```

### 8.2 与Q24评级系统的关系

Q24定义了专家的能力等级，Q25可以利用这些信息调整处理策略：

- **L3资深专家**: 边界更灵活，L1超限可尝试扩展处理
- **L2高级专家**: 边界清晰，L1超限建议降级处理
- **L1初级专家**: 边界严格，L1超限也建议转介
