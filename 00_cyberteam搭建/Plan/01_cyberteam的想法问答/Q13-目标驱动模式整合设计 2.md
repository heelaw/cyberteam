## 一、Goal-Driven 核心机制分析

### 1.1 四大核心概念

来自 goal-driven-main 仓库，它的核心概念：

| 概念 | 说明 | 对应你的架构 |
|------|------|-------------|
| **Goal** | 最终目标 | 用户的需求 |
| **Criteria for success** | 成功标准 | 方案审核通过的标准 |
| **Subagent** | 执行Agent | 执行层的Agent |
| **Master Agent** | 主Agent | CEO/董事会 |

### 1.2 核心循环逻辑

```python
# 伪代码
while (criteria not met):
    # 让子Agent持续工作
    subagent.work_on_goal()

    # Master Agent定期检查
    if subagent.is_inactive():
        # 检查目标是否达成
        if not check_criteria():
            # 未达成，重启子Agent继续
            subagent.restart()
        else:
            # 达成，停止
            subagent.stop()

# 循环直到成功标准满足
```

### 3.2 实现代码

```python
class GoalDrivenSystem:
    """目标驱动系统"""

    def __init__(self):
        self.master = MasterAgent()
        self.execution_layer = ExecutionLayer()
        self.pua_supervisor = PUASupervisor()  # 可选
        self.goal = None
        self.criteria = None

    async def start(self, user_question):
        """
        阶段1: 用户提出问题
        """
        # 1. 理解问题，转化为Goal
        self.goal = await self.define_goal(user_question)

        # 2. 生成解决方案
        solution = await self.board.discuss(self.goal)

        # 返回给用户审核

    async def approve(self, solution, approved_criteria):
        """
        阶段2+3: 用户批准目标和成功标准
        """
        self.goal = solution
        self.criteria = approved_criteria

        # 启动持续执行循环
        await self.execution_loop()

    async def execution_loop(self):
        """核心循环"""
        iteration = 0
        max_iterations = 1000  # 防止无限循环

        while not self.check_criteria() and iteration < max_iterations:
            iteration += 1

            # 1. 执行Agent工作
            result = await self.execution_layer.execute(self.goal)

            # 2. PUA监督（可选）
            if self.pua_supervisor:
                feedback = await self.pua_supervisor.supervise(result)
                if feedback["needs_retry"]:
                    continue  # 重试

            # 3. Master Agent评估
            if not self.check_criteria():
                # 未达成，继续
                await self.master.continue_work()
            else:
                # 达成，停止
                await self.master.stop()
                break

    def check_criteria(self) -> bool:
        """检查成功标准是否满足"""
        # 评估当前结果是否满足criteria
        return self.master.evaluate(self.criteria)
```

## 四、关键设计点

### 4.1 成功标准定义

成功标准必须**可量化、可验证**：

```yaml
# 好的成功标准示例
criteria: |
  1. DAU提升10%（数据验证）
  2. 留存率提升5%（数据验证）
  3. 方案执行完成（人工确认）
  4. ROI为正（财务验证）

# 不好的成功标准
criteria: |
  1. 业务变好
  2. 用户满意度提高
```

### 4.2 Master Agent的职责

```python
class MasterAgent:
    """Master Agent - 目标驱动系统的核心"""

    async def evaluate(self, criteria):
        """评估是否满足成功标准"""
        # 检查每个标准
        for criterion in criteria:
            if not self.check(criterion):
                return False
        return True

    async def monitor(self, subagent):
        """定期检查子Agent状态"""
        # 每5分钟检查一次
        if not subagent.is_active():
            if not self.evaluate(self.criteria):
                # 未达成，重启
                subagent.restart()
            else:
                # 达成，停止
                subagent.stop()

    async def decide_continue(self, result):
        """决策是否继续"""
        if self.evaluate(result):
            return "stop"
        else:
            return "continue_with_feedback"
```

### 4.3 防止无限循环

```python
# 安全机制
class SafetyMechanism:
    def __init__(self):
        self.max_iterations = 1000   # 最大迭代次数
        self.max_time = 24 * 3600   # 最大时间（24小时）
        self.max_cost = 1000         # 最大花费（美元）

    def should_stop(self, iteration, elapsed_time, cost):
        """判断是否应该停止"""
        if iteration >= self.max_iterations:
            return True, "达到最大迭代次数"

        if elapsed_time >= self.max_time:
            return True, "达到最大时间"

        if cost >= self.max_cost:
            return True, "达到最大花费"

        return False, None
```

## 五、总结

### Goal-Driven 模式与你的架构完美整合

| 你的三阶段 | Goal-Driven对应 | 状态 |
|-----------|----------------|------|
| 提出问题 | 定义 Goal | ✅ 可实现 |
| 审核方案 | 定义 Criteria | ✅ 可实现 |
| 批准执行 | while 循环 | ✅ 可实现 |

### 整合后的完整架构

```
用户 → [问题] → 董事会（审核方案）
                ↓
         [批准目标和成功标准]
                ↓
         Master Agent（CEO）→ 持续循环执行
                ↓
         执行层（Subagents）+ PUA Supervisor
                ↓
         定期检查标准是否满足
                ↓
         满足 → 输出结果
         不满足 → 继续执行
```

### 关键价值

1. **目标明确**：用户批准后才会启动
2. **持续执行**：直到成功标准满足
3. **可量化**：成功标准必须可验证
4. **防无限循环**：设置最大迭代/时间/花费
