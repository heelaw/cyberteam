# Q20: 思考天团文件输出与存储体系设计

**问题**: 搭建思考天团系统时，整个文件输出体系和文件系统应该如何设计？

---

## 一、设计目标

### 1.1 核心需求

| 需求 | 说明 |
|------|------|
| **多Agent协作** | 14个思维专家 + 180个运营Agent并行/交叉工作 |
| **状态追踪** | 每个任务的进度、输出、状态可查询 |
| **版本管理** | 保留每个版本的输出，支持回溯 |
| **实时监控** | Web看板需要读取实时状态 |
| **知识积累** | 跨任务的模式识别和经验沉淀 |

### 1.2 约束条件

- 使用文件系统（轻量，无需额外基础设施）
- 支持多Agent并发写入
- 实时可读（看板监控）

### 4.3 读写锁机制

```python
# lock.py - 防止并发写入冲突
import fcntl
import json

class TaskLock:
    @staticmethod
    def read(task_file):
        with open(task_file, 'r') as f:
            fcntl.flock(f.fileno(), fcntl.LOCK_SH)  # 共享锁
            data = json.load(f)
            fcntl.flock(f.fileno(), fcntl.LOCK_UN)
            return data

    @staticmethod
    def write(task_file, data):
        with open(task_file, 'r+') as f:
            fcntl.flock(f.fileno(), fcntl.LOCK_EX)  # 排他锁
            f.seek(0)
            json.dump(data, f, indent=2)
            f.truncate()
            fcntl.flock(f.fileno(), fcntl.LOCK_UN)
```

## 五、版本管理

### 5.1 自动版本化

```python
# version_control.py
import shutil
from datetime import datetime

def save_version(question_id: str, report_content: str):
    """保存报告版本"""
    version_dir = f"output/reports/{question_id}/versions"
    timestamp = datetime.now().strftime("%Y%m%d_%H%M%S")

    # 保存版本
    with open(f"{version_dir}/v_{timestamp}.md", "w") as f:
        f.write(report_content)

    # 更新latest链接
    latest_link = f"{version_dir}/latest.md"
    if os.path.exists(latest_link):
        os.remove(latest_link)
    os.symlink(f"v_{timestamp}.md", latest_link)

    # 保留策略：保留30天/50个版本
    cleanup_old_versions(version_dir, keep=50)
```

### 5.2 版本对比

```python
# diff.py
def diff_versions(version_a: str, version_b: str):
    """对比两个版本差异"""
    import difflib
    diff = difflib.unified_diff(
        version_a.splitlines(),
        version_b.splitlines(),
        fromfile='version_a',
        tofile='version_b'
    )
    return '\n'.join(diff)
```

## 七、存储策略

### 7.1 分层存储

| 数据类型 | 存储位置 | 保留策略 |
|----------|----------|----------|
| **热数据** | workspace/tasks/ | 处理中（实时） |
| **温数据** | output/reports/ | 90天 |
| **冷数据** | output/archives/ | 1年 |
| **知识数据** | knowledge/ | 永久 |

### 7.2 清理策略

```python
# cleanup.py
import os
import time
from datetime import datetime, timedelta

def cleanup():
    # 清理超过90天的处理中任务（异常中断）
    cutoff = time.time() - (90 * 24 * 3600)
    for f in glob("workspace/tasks/*/metadata.json"):
        if os.path.getmtime(f) < cutoff:
            # 标记为异常，移到归档
            move_to_archive(f, status="abandoned")

    # 清理超过1年的报告
    cutoff = datetime.now() - timedelta(days=365)
    for f in glob("output/reports/*/report.md"):
        if datetime.fromtimestamp(os.path.getmtime(f)) < cutoff:
            compress_and_move(f, "output/archives/")

    # 清理超过30天的日志
    cutoff = time.time() - (30 * 24 * 3600)
    for f in glob("logs/execution/*.log"):
        if os.path.getmtime(f) < cutoff:
            os.remove(f)
```
