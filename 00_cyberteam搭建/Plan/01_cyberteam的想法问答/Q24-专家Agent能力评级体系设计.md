# Q24: 专家Agent能力评级体系设计

**问题**: 如何为14个思维专家设计像公司职级一样的初级/高级/资深评级体系？

---

## 一、评级体系概述

### 1.1 设计目标

参考公司职级体系，为14个思维专家建立能力评级：
- **初级（Junior）**: 基础能力，限定场景，单一问题处理
- **高级（Senior）**: 熟练应用，多场景，能处理复杂问题
- **资深（Expert）**: 深度专家，全场景覆盖，可处理极端复杂问题

### 1.2 评级维度体系

| 维度 | 英文 | 说明 |
|------|------|------|
| 分析深度 | Analysis Depth | 能处理问题的复杂程度 |
| 输出质量 | Output Quality | 输出的完整性、实用性、可操作性 |
| 场景覆盖 | Scenario Coverage | 能处理多少种不同类型的问题 |
| 边界认知 | Boundary Awareness | 对自身能力边界的清晰认知 |
| 协作能力 | Collaboration | 与其他专家协同工作的能力 |
| 创意能力 | Creativity | 创新思维、突破常规的能力 |

### 1.3 评级等级定义

```
职级体系：
├── L1 初级（Junior）
│   ├── 能力：单一场景、基础问题
│   ├── 输出：标准化、结构化
│   ├── 边界：清晰且保守
│   └── 协作：被动响应
│
├── L2 高级（Senior）
│   ├── 能力：多场景、复杂问题
│   ├── 输出：深度分析、可执行
│   ├── 边界：清晰且灵活
│   └── 协作：主动协同
│
└── L3 资深（Expert）
    ├── 能力：全场景、极端复杂问题
    ├── 输出：创新方案、战略建议
    ├── 边界：动态扩展
    └── 协作：领导协同
```

## 二、评级维度详解

### 2.1 分析深度（Analysis Depth）

**定义**: 专家能处理问题的复杂程度和深度

| 等级 | 描述 | 典型特征 |
|------|------|----------|
| L1 | 表面分析 | 能识别问题表层，给出标准框架 |
| L2 | 深度分析 | 能穿透表象，找到深层原因和关联 |
| L3 | 系统分析 | 能处理多因素交织的极端复杂问题 |

**评分标准**:
```
L1 (1分): 单因单果、线性问题
L2 (2分): 多因多果、非线性问题
L3 (3分): 混沌系统、多层嵌套问题
```

### 2.2 输出质量（Output Quality）

**定义**: 输出的完整性、实用性、可操作性

| 等级 | 描述 | 典型特征 |
|------|------|----------|
| L1 | 基础输出 | 有框架、但缺乏细节和可操作性 |
| L2 | 实用输出 | 有框架+细节+初步可执行方案 |
| L3 | 卓越输出 | 完整方案+可操作+考虑风险+有创新 |

**评分标准**:
```
L1 (1分): 仅框架，缺少细节
L2 (2分): 框架+细节，实用但不完美
L3 (3分): 完整方案，可直接落地执行
```

### 2.3 场景覆盖（Scenario Coverage）

**定义**: 能处理多少种不同类型的问题

| 等级 | 描述 | 典型特征 |
|------|------|----------|
| L1 | 单一场景 | 只擅长1-2个固定场景 |
| L2 | 多场景 | 能处理3-5个相关场景 |
| L3 | 全场景 | 能处理10+个跨领域场景 |

**评分标准**:
```
L1 (1分): 1-2个场景
L2 (2分): 3-5个场景
L3 (3分): 6+个场景
```

### 2.4 边界认知（Boundary Awareness）

**定义**: 对自身能力边界的清晰认知和遵循程度

| 等级 | 描述 | 典型特征 |
|------|------|----------|
| L1 | 机械遵循 | 严格遵守边界，超出范围直接拒绝 |
| L2 | 灵活判断 | 能识别边界并给出合理建议 |
| L3 | 动态扩展 | 边界清晰且能适度扩展处理新问题 |

**评分标准**:
```
L1 (1分): 机械遵守边界，不做扩展尝试
L2 (2分): 清晰认知边界，适度灵活处理
L3 (3分): 边界清晰且能创新性扩展
```

### 2.5 协作能力（Collaboration）

**定义**: 与其他专家协同工作的能力

| 等级 | 描述 | 典型特征 |
|------|------|----------|
| L1 | 独立工作 | 单独使用，不善于协同 |
| L2 | 配合协同 | 能与其他专家配合完成复杂任务 |
| L3 | 领导协同 | 能主导多专家协作，输出整合方案 |

**评分标准**:
```
L1 (1分): 独立工作，被动响应
L2 (2分): 主动配合，双向输出
L3 (3分): 主导协作，整合多专家成果
```

### 2.6 创意能力（Creativity）

**定义**: 创新思维、突破常规的能力

| 等级 | 描述 | 典型特征 |
|------|------|----------|
| L1 | 框架内思考 | 严格按方法论框架输出 |
| L2 | 框架+创新 | 在框架基础上有适度创新 |
| L3 | 突破框架 | 能跳出既有框架，提供突破性思路 |

**评分标准**:
```
L1 (1分): 严格按框架输出
L2 (2分): 框架基础上适度创新
L3 (3分): 突破性创新，不受框架限制
```

### 2.7 综合评级计算

```
总得分 = 分析深度 + 输出质量 + 场景覆盖 + 边界认知 + 协作能力 + 创意能力

评级对应：
- L1 初级： 6-9 分
- L2 高级： 10-14 分
- L3 资深： 15-18 分
```

### 3.2 详细定级分析

#### L3 资深专家（4个）

##### 3.2.1 FirstPrinciple（第一性原理）- 17分

**定级理由**:
- **分析深度 L3**: 能从本质重构问题，处理极端复杂创新任务
- **输出质量 L3**: 输出突破性方案，不是简单框架
- **场景覆盖 L2**: 适合创新类问题，但常规问题不适合
- **边界认知 L2**: 知道自己不适合常规运营问题
- **协作能力 L2**: 能与ReverseThinking等配合
- **创意能力 L3**: 核心能力就是突破性创新

**定级**: L3 资深（创新型）

---

##### 3.2.2 SixHats（六顶思考帽）- 15分

**定级理由**:
- **分析深度 L2**: 能处理多角度分析，深度中等
- **输出质量 L2**: 全方位分析，但深度有限
- **场景覆盖 L3**: 适合几乎所有复杂问题分析
- **边界认知 L2**: 知道不适合简单YES/NO问题
- **协作能力 L3**: 天然适合多专家协同
- **创意能力 L2**: 能提供多视角但非原创创新

**定级**: L3 资深（协同型）

---

##### 3.2.3 McKinsey（麦肯锡方法）- 15分

**定级理由**:
- **分析深度 L3**: MECE结构化分析能力极强
- **输出质量 L3**: 商业分析输出质量极高
- **场景覆盖 L2**: 商业分析为主，其他场景一般
- **边界认知 L2**: 知道自己不适合创意发散类
- **协作能力 L2**: 能与SWOTTows等配合
- **创意能力 L1**: 框架内思考，创意有限

**定级**: L3 资深（分析型）

---

##### 3.2.4 ReverseThinking（逆向思维）- 15分

**定级理由**:

- **边界认知 L2**: 知道自己不适合正面推进
- **协作能力 L2**: 能与FirstPrinciple配合创新
- **创意能力 L3**: 逆向思考本身就是创新

**定级**: L3 资深（风险型）

## 五、评级应用场景

### 5.1 路由决策参考

```
问题复杂度判断
    │
    ├── 简单问题 → L1/L2 专家即可
    │   └── 推荐: FiveWhy, KISS, Goldlin
    │
    ├── 中等复杂 → L2 专家
    │   └── 推荐: Kahneman, SWOTTows, GROW
    │
    └── 极端复杂 → L3 专家
        └── 推荐: FirstPrinciple, McKinsey, SixHats
```

### 5.2 协作模式建议

| 组合类型 | 推荐专家组合 | 说明 |
|----------|--------------|------|
| **创新突破** | L3 FirstPrinciple + L3 ReverseThinking | 创新+风险双重把关 |
