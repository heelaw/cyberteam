# 思考天团审计日志体系设计

**问题**: 设计思考天团系统的审计日志体系，记录关键操作以满足合规和追溯需求

**参考**: Q20-思考天团文件输出与存储体系设计

---

## 一、设计目标

### 1.1 核心需求

| 需求 | 说明 |
|------|------|
| **合规要求** | 记录关键操作，满足审计追溯要求 |
| **问题排查** | 通过日志定位系统异常和用户问题 |
| **性能分析** | 分析专家调用耗时，优化系统性能 |
| **用户追溯** | 用户可查看自己问题的处理历史 |
| **安全审计** | 检测异常操作，保障系统安全 |

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| **完整性** | 记录所有关键操作不断连 |
| **可追溯性** | 每条操作可追溯到具体用户、任务、时间 |
| **高性能** | 异步写入，不阻塞主流程 |
| **可查询** | 支持多维度日志检索 |
| **与Q20一致** | 复用现有目录结构和文件格式 |

## 二、需要记录的关键操作

### 2.1 操作分类

根据思考天团系统特性，将审计日志分为四大类：

| 类别 | 标识 | 说明 |
|------|------|------|
| **用户交互** | USER | 用户提问、查看报告、反馈评价 |
| **任务调度** | TASK | 任务创建、分发、执行、完成、失败 |
| **专家调用** | EXPERT | 专家启动、分析、输出、辩论消息 |
| **系统事件** | SYSTEM | 启动、停止、异常、配置变更 |

### 2.2 详细操作清单

#### 2.2.1 用户交互类（USER）

| 操作ID | 操作名称 | 说明 |
|--------|----------|------|
| USER_ASK | 用户提问 | 用户提交问题到系统 |
| USER_VIEW_STATUS | 查看状态 | 用户查看任务进度 |
| USER_VIEW_REPORT | 查看报告 | 用户查看最终分析报告 |

| USER_CANCEL | 取消任务 | 用户主动取消处理中的任务 |
| USER_LOGIN | 用户登录 | 用户身份认证（如果需要） |

#### 2.2.2 任务调度类（TASK）

| 操作ID | 操作名称 | 说明 |
|--------|----------|------|
| TASK_CREATE | 任务创建 | 主持人创建新任务 |
| TASK_ROUTE | 任务路由 | 主持人确定专家组合和协作模式 |

#### 2.2.4 系统事件类（SYSTEM）

| 操作ID | 操作名称 | 说明 |

## 四、存储策略

### 4.1 目录结构（基于Q20）

根据Q20设计，审计日志存放在 `logs/audit/` 目录下：

```
logs/
├── execution/          # 执行日志（业务日志）
│   └── {question_id}/
│       └── {timestamp}.log
├── errors/             # 错误日志
│   └── {date}.error.log
└── audit/              # 审计日志（本次设计）
    ├── {year}/
    │   └── {month}/
    │       ├── {date}.audit.log        # 按天存储
    │       └── {date}.audit.index      # 索引文件
    ├── latest/                         # 软链接指向最新文件
    └── archive/                        # 归档目录
```

### 4.2 文件格式

| 格式 | 用途 | 说明 |
|------|------|------|
| **JSONL** | 审计日志主格式 | 每行一条JSON，便于流式处理和grep |
| **.index** | 索引文件 | 记录每条日志的偏移量，支持快速定位 |
| **.meta** | 元信息 | 记录文件创建时间、记录数等 |

### 4.3 索引文件格式

```json
{
  "date": "2026-03-20",
  "file": "2026-03-20.audit.log",
  "created_at": "2026-03-20T23:59:59.000Z",
  "record_count": 15420,
  "size_bytes": 15728640,
  "indexes": [
    {
      "log_id": "log_20260320_001_0001",
      "timestamp": "2026-03-20T10:00:00.000Z",
      "event_type": "USER_ASK",
      "question_id": "q_20260320_001",
      "offset": 0,
      "length": 512
    }
  ]
}
```

### 4.4 日志轮转策略

| 策略 | 参数 | 说明 |
|------|------|------|
| **时间轮转** | 每天一个文件 | 0点切换新文件 |
| **大小轮转** | 100MB/文件 | 单文件过大时拆分 |
| **压缩归档** | 7天后压缩 | gzip压缩节省空间 |
| **定期清理** | 90天 | 超出保留期删除 |

### 4.5 保留期限

| 数据类型 | 保留期 | 存储位置 |
|----------|--------|----------|
| **热数据** | 7天内 | `logs/audit/{year}/{month}/` |
| **温数据** | 8-30天 | `logs/audit/{year}/{month}/` 压缩 |
| **冷数据** | 31-90天 | `logs/audit/archive/` 压缩 |
| **归档** | 90天后 | 可选择导出到对象存储 |

### 4.6 清理脚本

```python
# cleanup_audit.py
import os
import gzip
import shutil
from datetime import datetime, timedelta

AUDIT_DIR = "logs/audit"
RETENTION_DAYS = 90

def cleanup_audit_logs():
    today = datetime.now()

    for root, dirs, files in os.walk(AUDIT_DIR):
        for file in files:
            if not file.endswith(".audit.log"):
                continue

            file_path = os.path.join(root, file)
            file_date = datetime.strptime(file[:10], "%Y-%m-%d")
            age = (today - file_date).days

            if age > RETENTION_DAYS:
                # 删除超过90天的日志
                os.remove(file_path)
            elif age > 7 and not file.endswith(".gz"):
                # 7-30天：压缩
                compress_log(file_path)
            elif age > 30 and not file.endswith(".gz"):
                # 30天后移动到归档目录
                move_to_archive(file_path)

def compress_log(file_path):
    """压缩日志文件"""
    with open(file_path, 'rb') as f_in:
        with gzip.open(file_path + '.gz', 'wb') as f_out:
            shutil.copyfileobj(f_in, f_out)
    os.remove(file_path)
```

## 六、日志记录实现

### 6.1 日志写入器

```python
# audit_logger.py
import json
import uuid
from datetime import datetime
from pathlib import Path
from typing import Dict, Any, Optional
import threading

class AuditLogger:
    def __init__(self, base_dir: str = "logs/audit"):
        self.base_dir = Path(base_dir)
        self._lock = threading.Lock()

    def _get_log_file(self, timestamp: datetime = None) -> Path:
        """获取当日日志文件路径"""
        if timestamp is None:
            timestamp = datetime.now()
        year = timestamp.strftime("%Y")
        month = timestamp.strftime("%m")
        date = timestamp.strftime("%Y-%m-%d")

        log_dir = self.base_dir / year / month
        log_dir.mkdir(parents=True, exist_ok=True)
        return log_dir / f"{date}.audit.log"

    def log(self, event_type: str, event_category: str,
            question_id: str = None, task_id: str = None,
            extra: Dict[str, Any] = None):
        """记录审计日志"""
        timestamp = datetime.utcnow()

        log_entry = {
            "log_id": f"log_{timestamp.strftime('%Y%m%d_%H%M%S')}_{uuid.uuid4().hex[:4]}",
            "timestamp": timestamp.isoformat() + "Z",
            "event_type": event_type,
            "event_category": event_category,
            "question_id": question_id,
            "task_id": task_id
        }

        if extra:
            log_entry.update(extra)

        # 异步写入
        self._write_async(log_entry)

    def _write_async(self, log_entry: Dict):
        """异步写入日志"""
        def _write():
            with self._lock:
                log_file = self._get_log_file()
                with open(log_file, 'a', encoding='utf-8') as f:
                    f.write(json.dumps(log_entry, ensure_ascii=False) + '\n')

        # 简化的异步实现（实际可使用asyncio）
        thread = threading.Thread(target=_write)
        thread.daemon = True
        thread.start()

# 全局单例
audit_logger = AuditLogger()
```

from functools import wraps

def audit_log(event_type: str, event_category: str):
    """审计日志装饰器"""
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            # 记录开始
            question_id = kwargs.get('question_id')
            task_id = kwargs.get('task_id')

            audit_logger.log(
                event_type=event_type + "_START",
                event_category=event_category,
                question_id=question_id,
                task_id=task_id
            )

            result = func(*args, **kwargs)

            # 记录完成
            audit_logger.log(
                event_type=event_type + "_COMPLETE",
                event_category=event_category,
                question_id=question_id,
                task_id=task_id,
                extra={"result": "success"}
            )

            return result
        return wrapper
    return decorator

# context.py
from contextlib import contextmanager

@contextmanager
def audit_context(event_type: str, event_category: str,
                  question_id: str = None, task_id: str = None,
                  extra: Dict = None):
    """审计上下文管理器"""
    start_time = datetime.utcnow()

    audit_logger.log(
        event_type=event_type + "_START",
        event_category=event_category,
        question_id=question_id,
        task_id=task_id,
        extra=extra
    )

    try:
        yield
        # 成功
        audit_logger.log(
            event_type=event_type + "_COMPLETE",
            event_category=event_category,
            question_id=question_id,
            task_id=task_id,
            extra={"duration_ms": (datetime.utcnow() - start_time).total_seconds() * 1000}
        )
    except Exception as e:
        # 失败
        audit_logger.log(
            event_type=event_type + "_ERROR",
            event_category=event_category,
            question_id=question_id,
            task_id=task_id,
            extra={"error": str(e)}
        )
        raise

# 使用示例
with audit_context("TASK", "TASK", question_id="q_001", task_id="task_001"):
    # 任务执行逻辑
    pass
