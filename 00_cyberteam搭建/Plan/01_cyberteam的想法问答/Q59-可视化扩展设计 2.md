# Q59: 思考天团可视化扩展设计

**问题**: 除了Web看板，还能提供什么可视化？思维导图、决策树等可视化如何集成？

---

## 一、可视化类型总览

思考天团的核心产出是「多专家协作过程」，可视化应围绕这个过程展开。

| 可视化类型 | 核心价值 | 生成方式 | 展示位置 |
|------------|----------|----------|----------|
| **思维导图** | 展示观点演变与分支 | 动态实时 | Web看板 |
| **决策树** | 展示方案选择分支 | 动态实时 | Web看板 |
| **关系图** | 展示专家协作网络 | 动态生成 | Web看板 |
| **时间线** | 展示任务进度与时序 | 动态实时 | Web看板 |
| **对比表格** | 展示多方案差异 | 静态生成 | 报告页面 |
| **甘特图** | 展示任务排期 | 静态生成 | 报告页面 |

## 二、每种可视化的场景

### 2.1 思维导图（Mind Map）

**场景**: 专家辩论过程中的观点演变

```
思维导图结构:
                    [核心问题]
                         │
          ┌──────────────┼──────────────┐
          │              │              │
    [Kahneman视角]  [FirstPrinciple]  [SixHats]
          │              │              │
    认知偏差分析    本质重构        六维分析
          │              │              │
    锚定效应 ←→ 成本重构 ←→ 风险评估
          │              │              │
          └──────────────┼──────────────┘
                         │
                   [综合结论]
```

**触发条件**:
- 专家之间产生观点交叉
- 同一问题有3个以上不同视角
- 需要展示论点的层级关系

**数据来源**:
```python
mind_map_data = {
    "root": "如何提升DAU 10%",
    "branches": [
        {"expert": "Kahneman", "insight": "认知偏差", "children": ["锚定效应", "确认偏差"]},
        {"expert": "FirstPrinciple", "insight": "本质重构", "children": ["成本结构", "价值主张"]},
        {"expert": "SixHats", "insight": "风险评估", "children": ["红色风险", "黑色风险"]}
    ]
}
```

### 2.2 决策树（Decision Tree）

**场景**: 方案选择与分支路径展示

```
决策树结构:
                      [起点:DAU提升方案]
                             │
        ┌────────────────────┼────────────────────┐
        │                    │                    │
   [渠道获客]           [用户留存]           [变现优化]
        │                    │                    │
   ┌────┴────┐         ┌────┴────┐         ┌────┴────┐
   │         │         │         │         │         │
[SEO/SEA] [投放]   [签到]   [Push]   [会员]   [广告]
   │         │         │         │         │         │
  收益高   收益中   成本低   风险高   收益高   收益低
```

**触发条件**:
- 存在多个可选方案
- 每个方案有明确的评估维度
- 需要展示决策逻辑路径

**数据来源**:
```python
decision_tree = {
    "root": {"label": "DAU提升", "expert": "Strategy"},
    "options": [
        {"label": "渠道获客", "expert": "Growth", "children": ["SEO/SEA", "信息流投放"]},
        {"label": "用户留存", "expert": "Retention", "children": ["签到机制", "Push唤醒"]},
        {"label": "变现优化", "expert": "Monetization", "children": ["会员体系", "广告变现"]}
    ]
}
```

### 2.3 关系图（Relationship Graph）

**场景**: 专家之间的协作网络与信息流

```
关系图结构:

    [Kahneman] ──────→ [FirstPrinciple]
        │                     │
        │ 观点引用              │ 逻辑验证
        ↓                     ↓
    [SixHats] ←──────── [Triz]
        │
        │ 风险补充
        ↓
    [Growth] ───────────────→ [最终报告]
```

**节点类型**:
| 节点 | 类型 | 说明 |
|------|------|------|
| 专家 | Expert | 思考天团成员 |
| 中间结论 | Insight | 专家产出 |
| 最终报告 | Report | 整合输出 |

**边类型**:
| 边 | 含义 |
|----|------|
| 观点引用 | A引用了B的观点 |
| 逻辑验证 | A验证了B的推理 |
| 风险补充 | A补充了B的风险 |
| 协作完成 | A协助B完成 |

**触发条件**:
- 任意两个专家产生交互
- 信息在专家之间传递
- 需要可视化协作网络

**场景**: 任务执行进度与时序展示

```
时间线结构:
═══════════════════════════════════════════════════════════════════

[问题提交]  10:00
    │
    ├─[Kahneman] ████████████░░░░░░░░░░░░ 10:00-10:05 (5min)
    │
    ├─[FirstPrinciple] ░░░░█████████░░░░░░ 10:03-10:08 (5min)
    │
    ├─[SixHats] ░░░░░░░░░████████████░░░ 10:06-10:12 (6min)
    │
    └─[整合输出] ░░░░░░░░░░░░░░░██████████ 10:12-10:15 (3min)

═══════════════════════════════════════════════════════════════════
总耗时: 15分钟
```

**触发条件**:
- 任意任务开始或完成
- 需要展示并行/串行关系
- 展示等待与阻塞

```
│   实施成本    │   高         │    低        │    中        │
├──────────────┼──────────────┼──────────────┼──────────────┤
│   风险等级    │   高         │    低        │    中        │
├──────────────┼──────────────┼──────────────┼──────────────┤
```

- 存在3个以上可选方案

**触发条件**:
- 任务存在依赖关系

- 展示关键路径

## 三、可视化如何生成

### 3.1 静态生成 vs 动态生成

| 可视化类型 | 生成方式 | 说明 |
|------------|----------|------|
| **思维导图** | 动态实时 | 边推理边生成，实时更新 |
| **决策树** | 动态实时 | 方案出现时即时生成 |
| **关系图** | 动态实时 | 专家交互时即时更新 |
| **时间线** | 动态实时 | 任务状态变化时更新 |
| **对比表格** | 静态生成 | 整合阶段一次性生成 |
| **甘特图** | 静态生成 | 完成后生成报告 |

### 3.2 生成技术栈


### 3.3 生成流程

```python
# 可视化生成流程

class VisualizationGenerator:
    """可视化生成器"""

    def __init__(self):
        self.mind_map = MindMapData()
        self.decision_tree = DecisionTreeData()
        self.relationship = RelationshipGraph()
        self.timeline = TimelineData()

    def on_expert_message(self, msg: ExpertMessage):
        """专家消息触发"""
        # 1. 提取观点，更新思维导图
        if msg.has_insight():
            self.mind_map.add_branch(
                expert=msg.from,
                insight=msg.insight,
                parent=msg.references
            )

        # 2. 更新关系图
        self.relationship.add_edge(
            from_node=msg.from,
            to_node=msg.to,
            edge_type="观点引用" if msg.references else "独立观点"
        )

    def on_task_state_change(self, task: Task):
        """任务状态变更触发"""
        # 更新时间线
        self.timeline.update_task(
            task_id=task.id,
            status=task.status,
            start_time=task.started_at,
            end_time=task.completed_at
        )

    def on_solution_proposed(self, solutions: List[Solution]):
        """方案提出触发"""
        # 生成决策树
        self.decision_tree.build_tree(
            root="问题",
            solutions=solutions
        )

    def generate_mermaid(self, viz_type: str) -> str:
        """生成Mermaid语法"""
        if viz_type == "mindmap":
            return self.mind_map.to_mermaid()
        elif viz_type == "flowchart":
            return self.decision_tree.to_mermaid()
        elif viz_type == "gantt":
            return self.timeline.to_mermaid()
```

## 七、总结

### 可视化类型与应用场景

| 可视化 | 核心场景 | 生成方式 | 优先级 |
|--------|----------|----------|--------|
| **时间线** | 任务进度 | 动态实时 | P0 |
| **关系图** | 专家协作网络 | 动态实时 | P0 |
| **思维导图** | 观点演变 | 动态实时 | P1 |
| **决策树** | 方案分支 | 动态实时 | P1 |
| **对比表格** | 多方案对比 | 静态生成 | P2 |
| **甘特图** | 任务排期 | 静态生成 | P2 |

### 实现顺序建议

1. **第一阶段（P0）**: 时间线 + 关系图
   - 最容易实现，复用现有任务状态
   - 用户感知最强

2. **第二阶段（P1）**: 思维导图 + 决策树
   - 需要提取专家观点结构
   - 需要方案数据

3. **第三阶段（P2）**: 对比表格 + 甘特图
   - 整合阶段产出
   - 报告导出用

### 与Web看板的关系

- **复用**: WebSocket、任务状态、前端框架
- **扩展**: 新增可视化Tab、新增组件
- **一致**: 保持相同的视觉风格和交互模式
