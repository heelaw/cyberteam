# Q21: 问题处理全流程产出文件清单

**问题**: 用户抛出问题到最终产出，中间所有产出文件有多少？分别是什么？存到哪里？

---

## 一、完整流程概览

### 1.1 流程阶段

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                          问题处理全流程（7个阶段）                             │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  阶段1: 提交问题       input/questions/pending/{id}.json                  │
│       ↓                                                                    │
│  阶段2: 意图分析       workspace/tasks/{id}/metadata.json                  │
│       ↓                                                                    │
│  阶段3: 任务创建       workspace/tasks/{id}/board.json                    │
│       ↓                                                                    │
│  阶段4: Agent执行      workspace/tasks/{id}/tasks/*.json                   │
│       ↓                                                                    │
│  阶段5: 交叉辩论       workspace/messages/{id}/                            │
│       ↓                                                                    │
│  阶段6: 整合输出       output/reports/{id}/                                │
│       ↓                                                                    │
│  阶段7: 知识沉淀       knowledge/                                          │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 1.2 文件数量统计

| 阶段 | 文件数量 | 文件类型 |
|------|----------|----------|
| 阶段1 | 1-2 | JSON |
| 阶段2 | 1 | JSON |
| 阶段3 | 1 | JSON |
| 阶段4 | N×2 | JSON + MD |
| 阶段5 | N×2 | JSON |
| 阶段6 | 3-5 | MD + JSON |
| 阶段7 | 1-3 | MD + JSON |
| **总计** | **15-30+** | |

> 注：N = 匹配的专家数量（通常3-8个）

---

## 二、阶段详解

### 阶段1: 提交问题

**用户操作**: `thinking-team ask "如何提升DAU 10%？"`

**产出文件**:

| 文件路径 | 数量 | 说明 |
|----------|------|------|
| `input/questions/pending/q_001.json` | 1 | 问题原始记录 |
| `input/context/q_001/user背景.md` | 0-1 | 用户提供的背景（可选） |

**文件示例**:
```json
// input/questions/pending/q_001.json
{
  "id": "q_001",
  "question": "如何提升DAU 10%？",
  "status": "pending",
  "created_at": "2026-03-20T10:00:00Z",
  "user": {
    "id": "user_001",
    "name": "张三",
    "role": "产品负责人"
  }
}
```

---

### 阶段2: 意图分析

**系统操作**: 主持人Agent进行意图识别

**产出文件**:

| 文件路径 | 数量 | 说明 |
|----------|------|------|
| `workspace/tasks/q_001/metadata.json` | 1 | 任务元信息 |

**文件示例**:
```json
// workspace/tasks/q_001/metadata.json
{
  "question_id": "q_001",
  "intent": {
    "primary": "增长策略",
    "secondary": ["用户激活", "留存提升"],
    "keywords": ["DAU", "增长", "指标"]
  },
  "routing": {
    "matched_experts": ["kahneman", "first_principle", "grow", "reverse_thinking"],
    "mode": "debate",
    "chain": null
  },
  "priority": "high",
  "estimated_duration_minutes": 15
}
```

---

### 阶段3: 任务创建

**系统操作**: 创建任务看板

**产出文件**:

| 文件路径 | 数量 | 说明 |
|----------|------|------|
| `workspace/tasks/q_001/board.json` | 1 | 看板主状态 |
| `workspace/tasks/q_001/messages/` | 1 | 消息目录 |

**文件示例**:
```json
// workspace/tasks/q_001/board.json
{
  "question_id": "q_001",
  "question": "如何提升DAU 10%？",
  "status": "running",
  "phase": "analysis",
  "progress": 0,
  "experts": [
    {"id": "kahneman", "status": "pending", "progress": 0},
    {"id": "first_principle", "status": "pending", "progress": 0},
    {"id": "grow", "status": "pending", "progress": 0},
    {"id": "reverse_thinking", "status": "pending", "progress": 0}
  ],
  "created_at": "2026-03-20T10:00:05Z"
}
```

---

### 阶段4: Agent执行

**系统操作**: 每个专家Agent并行/链式执行

**产出文件**（以Kahneman为例）:

| 文件路径 | 数量 | 说明 |
|----------|------|------|
| `workspace/tasks/q_001/tasks/kahneman.json` | 1 | 任务状态 |
| `workspace/tasks/q_001/tasks/kahneman/output.md` | 1 | 专家分析结果 |

**文件示例**:
```json
// workspace/tasks/q_001/tasks/kahneman.json
{
  "task_id": "task_kahneman_q_001",
  "expert_id": "kahneman",
  "question_id": "q_001",
  "status": "completed",
  "progress": 100,
  "started_at": "2026-03-20T10:00:10Z",
  "completed_at": "2026-03-20T10:02:30Z",
  "duration_seconds": 140,
  "model": "claude-sonnet-4-6",
  "tokens_used": 8500
}
```

```markdown
<!-- workspace/tasks/q_001/tasks/kahneman/output.md -->
# Kahneman 分析报告

## 核心观点

从认知偏差角度分析，DAU提升10%面临以下挑战：

### 1. 锚定效应
当前团队可能过度锚定现有用户群，忽视潜在用户需求。

### 2. 确认偏差
团队可能只收集支持增长假设的数据，忽视负面信号。

### 3. 损失厌恶
用户对产品价值的感知可能低于实际，需要强化价值锚点。

## 建议

1. 重新定义成功指标...
2. 建立反向验证机制...
```

**4个专家的产出**:

```
workspace/tasks/q_001/tasks/
├── kahneman/
│   ├── kahneman.json
│   └── output.md
├── first_principle/
│   ├── first_principle.json
│   └── output.md
├── grow/
│   ├── grow.json
│   └── output.md
└── reverse_thinking/
    ├── reverse_thinking.json
    └── output.md
```

> **本阶段文件总数**: 专家数 × 2 = 4 × 2 = 8个

---

### 阶段5: 交叉辩论

**系统操作**: 专家间消息传递

**产出文件**:

| 文件路径 | 数量 | 说明 |
|----------|------|------|
| `workspace/messages/q_001/inbox/{expert_id}/` | N×M | 接收的消息 |
| `workspace/messages/q_001/sent/{expert_id}/` | N×M | 发送的消息 |

**文件示例**:
```json
// workspace/messages/q_001/inbox/kahneman/msg_001.json
{
  "message_id": "msg_001",
  "from": "first_principle",
  "to": "kahneman",
  "content": "同意你的锚定效应分析。我从成本结构重构的角度补充...",
  "timestamp": "2026-03-20T10:03:00Z",
  "reply_to": null
}
```

```json
// workspace/messages/q_001/sent/first_principle/msg_002.json
{
  "message_id": "msg_002",
  "from": "first_principle",
  "to": "kahneman",
  "content": "关于你提到的损失厌恶，我同意。用户的心理账户...",
  "timestamp": "2026-03-20T10:03:30Z",
  "reply_to": "msg_001"
}
```

**辩论消息流**:
```
kahneman ────msg_001───▶ first_principle
    ▲                        │
    │                        ▼
    │                   reverse_thinking
    │                        │
    │                        ▼
    └──────msg_003──────────┘ (达成共识)
```

> **本阶段文件总数**: 8-16个消息文件

---

### 阶段6: 整合输出

**系统操作**: 主持人整合所有输出生成最终报告

**产出文件**:

| 文件路径 | 数量 | 说明 |
|----------|------|------|
| `output/reports/q_001/报告.md` | 1 | 完整分析报告 |
| `output/reports/q_001/摘要.md` | 1 | 执行摘要 |
| `output/reports/q_001/决策建议.json` | 1 | 结构化决策建议 |
| `output/reports/q_001/versions/v_001.md` | 1 | 初始版本 |

**文件示例**:
```markdown
<!-- output/reports/q_001/报告.md -->
# 思考天团分析报告

**问题**: 如何提升DAU 10%？
**生成时间**: 2026-03-20 10:30:00
**分析模式**: 交叉辩论

---

## 一、问题定义

[用户原始问题]

## 二、专家分析

### 2.1 Kahneman（卡尼曼决策）✓

- 锚定效应
- 确认偏差
- 损失厌恶

### 2.2 FirstPrinciple（第一性原理）✓

**核心观点**: 重构成本结构

### 2.3 GROW（目标模型）✓

**核心观点**: 设定SMART目标

### 2.4 ReverseThinking（逆向思维）✓

**核心观点**: 识别潜在风险

| FirstPrinciple | 成本重构 | ReverseThinking提出风险点 |

## 四、决策建议

### 4.1 短期行动（1-2周）

1. **重新定义成功指标**
   - 责任: 产品团队
   - 期限: 1周

2. **建立A/B测试机制**
   - 责任: 数据团队
   - 期限: 2周

### 4.2 中期策略（1-3月）

1. **用户分层运营**
2. **激活策略优化**

### 4.3 长期战略（3-6月）

1. **产品矩阵扩展**

| 用户疲劳 | 中 | 高 | 节奏控制 |
```

## 四、文件访问频率

### 4.1 热数据（频繁读写）

| 文件 | 读写频率 | 用途 |
|------|----------|------|
| `board.json` | 每秒 | Web看板实时显示 |
| `{expert_id}.json` | 每Agent | 任务状态更新 |
| `messages/` | 实时 | 辩论过程 |

### 4.2 温数据（定期读写）

| 文件 | 读写频率 | 用途 |
|------|----------|------|
| `报告.md` | 生成后只读 | 用户查看 |
| `决策建议.json` | 生成后只读 | API调用 |

### 4.3 冷数据（很少读写）

| 文件 | 读写频率 | 用途 |
|------|----------|------|
| `knowledge/` | 增量更新 | 跨任务学习 |
| `versions/` | 查询时读 | 版本对比 |

## 五、CLI操作与文件对应

### 5.1 用户命令

```bash
# 1. 提交问题
$ thinking-team ask "如何提升DAU 10%？"
# → 创建 input/questions/pending/q_001.json

# 2. 查看进度（读取board.json）
$ thinking-team status q_001
# ← 读取 workspace/tasks/q_001/board.json

# 3. 查看专家分析（读取expert output）
$ thinking-team view kahneman q_001
# ← 读取 workspace/tasks/q_001/tasks/kahneman/output.md

# 4. 查看报告（读取最终报告）
$ thinking-team report q_001
# ← 读取 output/reports/q_001/报告.md

# 5. 查看知识（读取知识库）
$ thinking-team patterns
# ← 读取 knowledge/patterns/decision-patterns.md
```

### 5.2 看板访问

```
浏览器访问: http://localhost:8080/board/q_001

读取的文件:
├── workspace/tasks/q_001/board.json (实时)
├── workspace/tasks/q_001/tasks/*.json (实时)
└── output/reports/q_001/摘要.md (完成后)
```

## 六、总结

### 6.1 文件数量

| 类型 | 数量 |
|------|------|
| **核心文件** | 约 20-35 个 |
| **单专家** | 2 个 (状态 + 输出) |
| **消息文件** | 8-16 个 |

### 6.2 文件分类

| 分类 | 位置 | 生命周期 |
|------|------|----------|
| **输入** | input/ | 问题生命周期 |
| **运行时** | workspace/ | 处理期间 |
| **输出** | output/ | 长期保存 |
| **知识** | knowledge/ | 永久积累 |

### 6.3 关键文件

| 文件 | 重要性 | 说明 |
|------|--------|------|
| `board.json` | ⭐⭐⭐⭐⭐ | 看板主状态，实时更新 |
| `{expert}/output.md` | ⭐⭐⭐⭐⭐ | 专家分析产出 |
| `报告.md` | ⭐⭐⭐⭐⭐ | 最终交付物 |
| `knowledge/` | ⭐⭐⭐ | 持续迭代优化 |
