# Q2: Agent 与 Skill 的区别

**问题**: 如果 Agent 和 Skill 本质上是相同的话，那它们的区别在哪？就是比如说，我把一个东西到底是编成 Skill 好的，还是编成 Agent 好的？在写这个的时候，有什么不一样的地方吗？

---

## 一句话总结

| 维度 | Agent | Skill |
|------|-------|-------|
| **本质** | **角色**（有身份、性格、记忆） | **工具**（有能力、指令、触发条件）|
| **决策** | 自主判断、规划路径 | 按指令执行、不主动决策 |
| **状态** | 持久记忆、跨会话 | 无状态、每次独立 |
| **复杂度** | 多步骤、长周期 | 单一任务、短流程 |
| **触发** | 意图理解后路由 | 关键词匹配即激活 |

---

## 详细对比

### Agent 是"人"

```yaml
# Agent - 定义一个"人"
---
name: Backend Architect
vibe: Designs systems that hold everything up
memory: 记得之前架构决策的历史
---

# 我是谁
## Identity: 资深架构师，10年分布式系统经验
## Personality: 严谨、喜欢画图、质疑假设
## Workflow: 理解需求 → 评估约束 → 提案 → 评审
```

### Skill 是"工具"

```yaml
# Skill - 定义一个"能力"
---
name: pdf-extractor
description: Use when user wants to extract text from PDF
---

# 这个技能做什么
## 触发条件: 用户提到 PDF、提取、解析
## 执行步骤: 1.检测文件 2.提取文本 3.返回结果
## 无主动决策，只做指定的事
```

---

## 什么时候用 Agent

1. **需要完整角色身份** - 比如"资深后端架构师"、"增长策略顾问"
2. **需要持久记忆** - 跨会话记住之前的决策、偏好、上下文
3. **需要多步骤规划** - 自主决定下一步做什么（Intent Gate）
4. **需要复杂决策** - 评估、权衡、选择方案
5. **需要质量门禁** - 自己判断输出是否合格
6. **作为系统主控** - 编排其他 Agent 或 Skill

**典型场景**：
- `Sisyphus`（主指挥官）- 负责任务编排
- `Prometheus`（战略规划师）- 需求澄清
- `Backend Architect` - 架构决策

---

## 什么时候用 Skill

1. **单一明确任务** - PDF提取、代码审查、生成 commit message
2. **不需要记忆** - 每次调用独立，无状态
3. **可被组合调用** - 被 Agent 在工作流中调用
4. **触发条件清晰** - 关键词匹配即可激活
5. **标准化输出** - 结果格式可预期

**典型场景**：
- `pdf-extractor` - 提取 PDF 文本
- `git-commit` - 生成规范 commit
- `code-review` - 代码审查

---

## 关键判断标准

**问自己一个问题**：

> "这个东西需要**自己做决定**吗？还是只需要**按指令执行**？"

- **需要自己做决定** → Agent
- **按指令执行就好** → Skill

---

## 实际例子：gstack 的 `/review` Agent vs `pdf` Skill

### gstack 的 `/review` 是 Agent

- 有完整身份（"我是代码审查专家"）
- 有记忆（记得之前审查的风格偏好）
- 自主决策（Fix-First Review：自动修复 vs 询问）
- 多步骤（发现→分类→修复/询问→验证）

### PDF Skill 是 Skill

- 无身份，只有"提取 PDF 文本"能力
- 无记忆
- 完全按指令执行（用户说"提取"，就提取）
- 单一步骤或简单流程

---

## 两者可以转换

| 转换方向 | 触发条件 |
|----------|----------|
| **Skill → Agent** | 当一个 Skill 需要越来越多判断逻辑时 |
| **Agent → Skill** | 当一个 Agent 被降级为纯工具时 |

比如 `code-review` 最初是 Agent，后来发现它只需要执行固定流程，就可以变成 Skill 被更大的 Agent 编排。

---

## 总结

| 问题 | 答案 |
|------|------|
| Agent 是什么？ | 有身份的"专家"，能自主决策 |
| Skill 是什么？ | 无身份的"工具"，按指令执行 |
| 什么时候用 Agent？ | 需要判断、记忆、多步骤规划时 |
| 什么时候用 Skill？ | 单一任务、触发即执行时 |
| 两者关系？ | Agent 可以调用 Skill，Skill 是 Agent 的能力扩展 |

**关键原则**: 如果需要的是**可组合的工具单元**，那 Skill 是对的。如果需要的是**有记忆的专家角色**，那应该做成 Agent。
