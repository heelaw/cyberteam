# Q7: 董事会层讨论机制详解

**问题**: 关于"层级一"的董事会层如何进行讨论决策：

1. 任务拆解与背景补充：
   用户给出问题后，系统首先根据知识库中已有的实际业务背景进行拆解。用户需要将业务背景等信息存放在知识库里，Agent Team 会自动检索并补充背景，随后进行问题的拆解、讨论与分析。

2. 全自动自主讨论：
   我希望这些 Agent 能够并行且自主地主持讨论并汇总方案。这个过程应当是全自动的，不需要用户参与。请问这是否能够实现？

3. 会议记录与文件机制：
   如果可以实现，系统是否会有类似人类开会的"会议记录"？
   (a) 是否有一个清晰的文件机制，让用户直观地看到每个 Agent 输出了什么方案、哪些方案被讨论后认为不合适、新方案迭代了哪些内容？
   (b) 整个过程的方案分发和讨论路径是怎样的？
   (c) 它能否像"群聊"一样，让 Agent 之间不断地进行交叉验证、讨论和辩论？

## 一、任务拆解与背景补充

### 1.1 知识库检索机制

**参考模式**：OpenViking 的双层记忆系统

```python
# OpenViking 的 MemoryStore
class MemoryStore:
    # L1: MEMORY.md - 长期事实存储（知识库）
    # L2: HISTORY.md - grep 可搜索的历史记录

    async def retrieve_background(self, user_question):
        # 1. 从知识库检索相关内容
        relevant_docs = await self.search_knowledge_base(
            query=user_question,
            top_k=5
        )

        # 2. 提取业务背景
        background = self.extract_background(relevant_docs)

        # 3. 返回给 Agent
        return background
```

class BackgroundEnricher:

        # Step 1: 检索知识库

        # Step 4: 返回给董事会

## 二、全自动自主讨论机制

### 2.1 并行讨论模式

**参考模式**：gstack 的 Nexus Spatial Discovery（8 Agent 并行）

```python
class BoardDiscussion:
    async def autonomous_discuss(self, question, background):
        # Step 1: 分发任务给各位总监（并行）
        discussion_tasks = [
            self.operation_director.analyze(
                question=question,
                background=background,
                perspective="运营视角"
            ),
            self.finance_director.analyze(
                question=question,
                background=background,
                perspective="财务视角"
            ),
            self.product_director.analyze(
                question=question,
                background=background,
                perspective="产品视角"
            ),
            self.marketing_director.analyze(
                question=question,
                background=background,
                perspective="市场视角"
            )
        ]

        # 并行执行所有总监的分析
        initial_reports = await gather(discussion_tasks)

        # Step 2: 交叉验证（第二轮）
        validated_reports = await self.cross_validate(initial_reports)

        # Step 3: CEO 汇总
        final_plan = await self.ceo.synthesize(
            question=question,
            reports=validated_reports
        )

        return final_plan
```

**关键点**：整个过程全自动，不需要用户参与。

### 2.2 讨论深度控制

```python
# 讨论轮次控制
class DiscussionConfig:
    # 可配置参数
    max_rounds = 3           # 最多讨论3轮
    min_consensus = 0.7      # 70%共识即通过
    auto_terminate = True    # 达成共识后自动停止

    # 讨论停止条件
    async def should_stop(self, reports):
        # 条件1: 达到最大轮次
        if self.current_round >= self.max_rounds:
            return True

        # 条件2: 达成共识
        consensus_score = await self.calculate_consensus(reports)
        if consensus_score >= self.min_consensus:
            return True

        # 条件3: 观点不再变化
        if self.reports_unchanged():
            return True

        return False
```

### 3.2 文件机制实现

```python
class MeetingRecorder:
    def __init__(self, meeting_id):
        self.meeting_id = meeting_id
        self.records = []

    def record(self, agent_name, content, round_num):
        """记录单个Agent的发言"""
        record = {
            "timestamp": datetime.now(),
            "agent": agent_name,
            "round": round_num,
            "content": content
        }
        self.records.append(record)

        # 实时写入文件
        self.write_to_file()

    def write_to_file(self):
        """生成会议记录文件"""
        filename = f"meeting-{self.meeting_id}.md"
        content = self.render_markdown()
        await self.save(filename, content)
```

### 3.3 方案分发路径追踪

```python
class DiscussionTracker:
    """追踪讨论路径和方案迭代"""

    def __init__(self):
        self.proposals = {}      # 所有提出的方案
        self.discussions = []    # 讨论记录
        self.decisions = []      # 最终决策

    def track_proposal(self, agent, proposal):
        """追踪方案提出"""
        self.proposals[proposal.id] = {
            "agent": agent,
            "content": proposal,
            "status": "proposed",
            "round": self.current_round
        }

    def track_critique(self, from_agent, to_proposal, critique):
        """追踪质疑"""
        self.discussions.append({
            "type": "critique",
            "from": from_agent,
            "target": to_proposal.id,
            "content": critique,
            "round": self.current_round
        })

    def track_evolution(self, original, evolved):
        """追踪方案迭代"""
        self.proposals[evolved.id] = {
            "parent": original.id,
            "evolution": self.diff(original, evolved),
            "round": self.current_round
        }
```

### 3.4 群聊式讨论实现

**参考模式**：agency-agents 的 Handoff + gstack 的 Parallel

```python
class GroupChatDiscussion:
    """群聊式讨论机制"""

    async def start_discussion(self, question, agents):
        """启动群聊讨论"""
        # 第一轮：同时发言
        round_1_tasks = [
            agent.respond(question, context={"round": 1})
            for agent in agents
        ]
        responses_1 = await gather(round_1_tasks)

        # 记录到会议记录
        for agent, response in zip(agents, responses_1):
            self.recorder.record(agent.name, response, round=1)

        # 第二轮：基于他人发言进行回应
        context_2 = {"round": 2, "other_responses": responses_1}
        round_2_tasks = [
            agent.respond(
                question,
                context=context_2,
                reply_to=[r for r in responses_1 if r.agent != agent.name]
            )
            for agent in agents
        ]
        responses_2 = await gather(round_2_tasks)

        # 记录
        for agent, response in zip(agents, responses_2):
            self.recorder.record(agent.name, response, round=2)

        # 重复直到达成共识或达到最大轮次
        return await self.aggregate(responses_1 + responses_2)
```
