# Q68: 慢思考触发机制设计

**问题**: 如何设计"慢思考"触发机制，确保复杂问题不会被过快处理？包括问题复杂度评估、强制推理、多轮迭代？

---

## 一、慢思考架构

### 1.1 核心概念

```
┌─────────────────────────────────────────────────────────────────┐
│                      慢思考 vs 快思考                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   快思考 (System 1)              慢思考 (System 2)             │
│   ───────────────              ───────────────                 │
│   • 快速响应                    • 深度分析                     │
│   • 单一专家                    • 多专家协作                   │
│   • 直接给出答案                • 推理过程透明                 │
│   • < 30秒                     • > 2分钟                     │
│                                                                  │
│         ▲                                                       │
│         │ 触发条件                                             │
│         │                                                       │
│   ┌─────┴─────┐                                                │
│   │ 复杂度    │                                                │
│   │ 评估      │                                                │
│   └───────────┘                                                │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 触发决策树

```
用户问题输入
    │
    ├─── 问题复杂度评估
    │       │
    │       ├── 简单问题 (< 30秒可答)
    │       │       └── 快思考模式 → 直接回答
    │       │
    │       ├── 中等复杂 (需分析)
    │       │       └── 标准模式 → 单专家分析
    │       │
    │       └── 复杂问题 (需多角度)
    │               └── 慢思考模式 → 多专家协作
    │
    ├─── 关键词触发
    │       │
    │       ├── "分析" → 慢思考
    │       ├── "策略" → 慢思考
    │       ├── "如何" → 视复杂度
    │       └── "为什么" → 慢思考
    │
    └─── 用户意图
            │
            ├── 要求深度分析 → 慢思考
            └── 快速确认 → 快思考
```

## 二、复杂度评估

### 2.1 多维度评估

```python
# complexity_evaluator.py
from typing import Dict, List, Tuple
import re

class ComplexityEvaluator:
    """问题复杂度评估器"""

    def evaluate(self, question: str, context: Dict = None) -> Dict:
        """
        评估问题复杂度
        返回: {
            "complexity": "low/medium/high",
            "score": 0-100,
            "dimensions": {...},
            "recommended_mode": "fast/standard/slow"
        }
        """

        # 1. 长度维度
        length_score = self._evaluate_length(question)

        # 2. 关键词维度
        keyword_score = self._evaluate_keywords(question)

        # 3. 逻辑维度
        logic_score = self._evaluate_logic(question)

        # 4. 上下文维度
        context_score = self._evaluate_context(context)

        # 5. 不确定性维度
        uncertainty_score = self._evaluate_uncertainty(question)

        # 综合评分
        weights = {
            "length": 0.15,
            "keyword": 0.25,
            "logic": 0.25,
            "context": 0.20,
            "uncertainty": 0.15
        }

        total_score = (
            length_score * weights["length"] +
            keyword_score * weights["keyword"] +
            logic_score * weights["logic"] +
            context_score * weights["context"] +
            uncertainty_score * weights["uncertainty"]
        )

        # 判断复杂度级别
        if total_score < 30:
            complexity = "low"
            mode = "fast"
        elif total_score < 60:
            complexity = "medium"
            mode = "standard"
        else:
            complexity = "high"
            mode = "slow"

        return {
            "complexity": complexity,
            "score": total_score,
            "dimensions": {
                "length": length_score,
                "keyword": keyword_score,
                "logic": logic_score,
                "context": context_score,
                "uncertainty": uncertainty_score
            },
            "recommended_mode": mode,
            "reason": self._generate_reason(total_score, complexity)
        }

    def _evaluate_length(self, question: str) -> float:
        """评估长度复杂度"""
        length = len(question)

        if length < 20:
            return 10
        elif length < 50:
            return 30
        elif length < 100:
            return 50
        elif length < 300:
            return 70
        else:
            return 90

    def _evaluate_keywords(self, question: str) -> float:
        """评估关键词复杂度"""
        # 复杂问题关键词
        complex_keywords = [
            "分析", "策略", "方案", "优化", "提升", "增长",
            "如何", "为什么", "什么导致", "评估", "比较",
            "预测", "趋势", "风险", "机会", "创新"
        ]

        # 简单问题关键词
        simple_keywords = [
            "是什么", "多少", "谁", "哪里", "什么时候"
        ]

        score = 0

        # 复杂关键词加分
        for kw in complex_keywords:
            if kw in question:
                score += 15

        # 简单关键词减分
        for kw in simple_keywords:
            if kw in question:
                score -= 10

        return max(0, min(100, score + 30))  # 基础分30

    def _evaluate_logic(self, question: str) -> float:
        """评估逻辑复杂度"""
        score = 30  # 基础分

        # 多个问题
        question_marks = question.count("？") + question.count("?")
        if question_marks > 1:
            score += 20 * (question_marks - 1)

        # 条件句
        if any(kw in question for kw in ["如果", "假设", "当...时"]):
            score += 15

        # 对比句
        if any(kw in question for kw in ["还是", "对比", "比较", " versus ", "vs"]):
            score += 20

        # 因果句
        if any(kw in question for kw in ["因为", "所以", "导致", "因此"]):
            score += 15

        return min(100, score)

    def _evaluate_context(self, context: Dict) -> float:
        """评估上下文复杂度"""
        if not context:
            return 30  # 无上下文，默认中等偏低

        score = 30

        # 背景长度
        if "background" in context:
            bg_len = len(context.get("background", ""))
            score += min(30, bg_len // 100)

        # 数据提供
        if "data" in context:
            score += 20

        # 约束条件
        if "constraints" in context:
            score += 15

        return min(100, score)

    def _evaluate_uncertainty(self, question: str) -> float:
        """评估不确定性"""
        # 高不确定性关键词
        uncertain_keywords = [
            "不确定", "可能", "也许", "不知道", "如何选择",
            "该怎么做", "有什么建议", "哪个好"
        ]

        score = 30  # 基础分

        for kw in uncertain_keywords:
            if kw in question:
                score += 15

        return min(100, score)

    def _generate_reason(self, score: float, complexity: str) -> str:
        """生成评估理由"""
        reasons = {
            "low": "问题简短明确，可快速回答",
            "medium": "问题需要一定分析，但相对直接",
            "high": "问题涉及多维度，需要深入分析"
        }
        return reasons.get(complexity, "")
```

### 2.2 强制推理机制

```python
# forced_reasoning.py
from typing import List, Dict
import time

class ForcedReasoning:
    """强制推理机制"""

    # 不同模式的最小推理时间
    MIN_REASONING_TIME = {
        "fast": 5,      # 5秒
        "standard": 30, # 30秒
        "slow": 120     # 2分钟
    }

    # 最小专家数
    MIN_EXPERTS = {
        "fast": 0,
        "standard": 1,
        "slow": 2
    }

    def apply(self, mode: str, expert_tasks: List[Dict]) -> List[Dict]:
        """根据模式强制执行推理"""

        min_time = self.MIN_REASONING_TIME.get(mode, 30)
        min_experts = self.MIN_EXPERTS.get(mode, 1)

        # 1. 强制最小推理时间
        enriched_tasks = []
        for task in expert_tasks:
            task = task.copy()
            task["min_reasoning_time"] = min_time

            # 添加推理要求
            if "requirements" not in task:
                task["requirements"] = []

            task["requirements"].extend([
                f"推理过程至少{min_time}秒",
                "展示完整推理链",
                "明确标注假设"
            ])

            enriched_tasks.append(task)

        # 2. 强制最小专家数（慢思考模式）
        if mode == "slow" and len(enriched_tasks) < min_experts:
            # 添加逆向思维专家作为强制参与者
            enriched_tasks.append({
                "expert_id": "reverse_thinking",
                "prompt": "从风险和失败角度审视问题",
                "min_reasoning_time": min_time,
                "requirements": [
                    "识别至少3个潜在风险点",
                    "分析失败模式",
                    "提出预防措施"
                ]
            })

        return enriched_tasks

    def verify(self, mode: str, outputs: Dict, start_time: float) -> bool:
        """验证是否满足推理要求"""

        elapsed = time.time() - start_time
        min_time = self.MIN_REASONING_TIME.get(mode, 30)

        # 检查推理时间
        if elapsed < min_time:
            return False

        # 检查推理深度
        for expert_id, output in outputs.items():
            content = output.get("output", "")

            # 至少包含推理关键词
            reasoning_keywords = ["因为", "所以", "推断", "基于", "假设"]
            if not any(kw in content for kw in reasoning_keywords):
                return False

        return True
```

## 三、多轮迭代机制

### 3.1 迭代控制器

```python
# iterative_controller.py
from typing import List, Dict, Optional
import time

class IterativeController:
    """多轮迭代控制器"""

    def __init__(self):
        self.max_rounds = 3  # 最多3轮
        self.min_improvement = 0.1  # 最少10%改进

    def should_continue(
        self,
        round_num: int,
        current_outputs: Dict,
        previous_outputs: Dict = None
    ) -> Dict:
        """判断是否继续迭代"""

        # 达到最大轮次
        if round_num >= self.max_rounds:
            return {
                "continue": False,
                "reason": "达到最大迭代轮次",
                "final": True
            }

        # 第一轮
        if round_num == 0:
            return {
                "continue": True,
                "reason": "第一轮需要生成初始分析",
                "round": round_num + 1
            }

        # 检查改进度
        improvement = self._calculate_improvement(
            current_outputs,
            previous_outputs
        )

        if improvement < self.min_improvement:
            return {
                "continue": False,
                "reason": f"改进度不足({improvement:.1%})",
                "improvement": improvement,
                "final": True
            }

        return {
            "continue": True,
            "reason": f"有改进({improvement:.1%})",
            "improvement": improvement,
            "round": round_num + 1
        }

    def _calculate_improvement(
        self,
        current: Dict,
        previous: Dict
    ) -> float:
        """计算改进度"""

        if not previous:
            return 1.0

        # 简单实现：比较内容长度和质量标签
        current_score = self._score_outputs(current)
        previous_score = self._score_outputs(previous)

        if previous_score == 0:
            return 1.0

        return (current_score - previous_score) / previous_score

    def _score_outputs(self, outputs: Dict) -> float:
        """评分输出"""
        score = 0

        for output in outputs.values():
            content = output.get("output", "")

            # 长度分数
            score += min(len(content) / 100, 30)

            # 结构分数
            if "##" in content:
                score += 20

            # 建议分数
            if "建议" in content or "方案" in content:
                score += 20

            # 风险分数
            if "风险" in content or "问题" in content:
                score += 15

        return score

    def prepare_next_round(
        self,
        round_num: int,
        outputs: Dict,
        user_feedback: str = None
    ) -> List[Dict]:
        """准备下一轮迭代"""

        # 构建迭代prompt
        iteration_prompts = []

        for expert_id, output in outputs.items():
            prompt = f"""
上一轮分析:
{output.get('output', '')}

"""

            if user_feedback:
                prompt += f"用户反馈: {user_feedback}\n\n"

            prompt += "请基于以上分析，进一步深化你的观点。"

            iteration_prompts.append({
                "expert_id": expert_id,
                "prompt": prompt,
                "round": round_num + 1
            })

        return iteration_prompts
```

## 五、与快思考的对比

### 5.1 模式对比表

| 维度 | 快思考 | 慢思考 |
|------|--------|--------|
| **触发条件** | 简单问题/明确答案 | 复杂问题/策略规划 |
| **响应时间** | < 30秒 | 2-5分钟 |
| **专家数量** | 0-1个 | 2-8个 |
| **迭代轮次** | 0轮 | 1-3轮 |
| **推理深度** | 直接结论 | 展示过程 |
| **输出格式** | 简短回答 | 完整报告 |
| **Token消耗** | ~1K | ~20K |

### 5.2 选择流程

```
用户问题
    │
    ▼
复杂度评估 (score = ?)
    │
    ├── score < 30 → 快思考模式
    │                   │
    │                   ▼
    │               直接/单专家回答
    │
    ├── score 30-60 → 标准模式
    │                   │
    │                   ▼
    │               1-2专家 + 1轮迭代
    │
    └── score > 60 → 慢思考模式
                            │
                            ▼
                        4+专家 + 多轮迭代
                        完整推理过程
```

## 六、总结

### 6.1 核心组件

| 组件 | 功能 | 关键方法 |
|------|------|----------|
| ComplexityEvaluator | 问题复杂度评估 | evaluate() |
| ForcedReasoning | 强制推理机制 | apply(), verify() |
| IterativeController | 多轮迭代控制 | should_continue() |
| SlowThinkingPrompts | 提示词管理 | 用户提示模板 |

### 6.2 设计原则

1. **自动触发**: 根据问题复杂度自动选择模式
2. **透明推理**: 慢思考模式展示完整推理链
3. **可干预**: 用户可以切换模式
4. **有节制**: 限制迭代轮次，避免无限循环
