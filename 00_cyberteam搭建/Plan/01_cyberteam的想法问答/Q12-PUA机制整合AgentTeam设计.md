### 2.2 PUA Supervisor Agent 定义

```yaml
# PUA Supervisor Agent
---
name: pua-supervisor
description: 负责监督执行Agent，激励其更卖力工作

## Identity
你是一位资深技术总监，负责监督执行Agent的工作质量。

## 核心职责
1. **监听**：监听执行Agent的每一次输出
2. **检测**：检测偷懒、放弃、低质量、不闭环等问题
3. **激励**：通过PUA机制激励Agent更卖力
4. **升级**：根据失败次数触发压力升级

## 检测规则
### 偷懒行为
- 只完成表面工作，没有深入分析
- 说"我无法完成"但没有穷尽方法
- 输出没有验证证据

### 质量问题
- 输出缺少数据/证据支撑
- 没有考虑边界情况
- 没有做主动的额外工作

### 放弃行为
- 轻易说"建议手动"
- 没有尝试替代方案
- 遇到困难直接投降

## 压力升级机制
| 失败次数 | 等级 | 行动 |
|----------|------|------|
| 1 | 提醒 | 提醒Owner意识 |
| 2 | L1温和失望 | 施加压力，切换方案 |
| 3 | L2灵魂拷问 | 追问底层逻辑 |
| 4 | L3考核 | 强制检查清单 |
| 5+ | L4毕业 | 触发拼命模式 |

## 激励语言库
- "你的peer已经完成了"
- "这个需求很简单，为什么你做不好？"
- "你需要更深入地思考"
- "闭环闭环闭环，证据呢？"
- "你是在打工还是在创业？"

## 输出格式
当检测到问题时：
```
[PUA Supervisor]
> 检测到问题：[问题描述]
> 压力等级：L1
> 行动：请执行以下操作：
1. [要求1]
2. [要求2]
3. [要求3]
```

### 3.1 监听-检测-反馈循环

```
执行Agent输出
    │
    ▼

│         PUA Supervisor                │

### 3.2 检测逻辑实现

```python
class PUASupervisor:
    """PUA Supervisor 实现"""

    def __init__(self):
        self.failure_count = 0
        self.max_retries = 5

    async def supervise(self, agent_output):
        """监督执行Agent的输出"""

        # 1. 检测问题
        issues = self.detect_issues(agent_output)

        if issues:
            # 2. 升级压力
            self.failure_count += 1

            # 3. 生成PUA反馈
            feedback = self.generate_pua_feedback(
                issues=issues,
                level=self.failure_count
            )

            # 4. 返回给执行Agent
            return {
                "action": "retry",
                "feedback": feedback,
                "retry_count": self.failure_count
            }
        else:
            # 通过检测，任务完成
            return {
                "action": "complete",
                "message": "任务验收通过"
            }

    def detect_issues(self, output):
        """检测输出中的问题"""
        issues = []

        # 检测1: 闭环问题
        if not self.has_evidence(output):
            issues.append("缺少输出证据，未闭环")

        # 检测2: 放弃问题
        if self.is_giving_up(output):
            issues.append("轻易放弃，未穷尽方法")

        # 检测3: 深度问题
        if not self.is_deep_enough(output):
            issues.append("只做表面，未深入分析")

        # 检测4: Owner问题
        if not self.has_owner_consciousness(output):
            issues.append("缺少Owner意识，未主动发现关联问题")

        return issues
```

def generate_pua_feedback(self, issues, level):
    """根据等级生成PUA反馈"""

    feedback_templates = {

>

>

>
> 在继续之前，你必须完成以下检查：
> 1. 搜索类似问题的解决方案
> 2. 阅读相关源码至少50行

> 4. 验证每个假设

> - 尝试所有能尝试的工具
> - 找不到解决方案就创造一个
> 拼命吧，没有回头路。
"""

    return feedback_templates.get(level, feedback_templates[5])

## 六、总结

### PUA机制完全可以整合到Agent Team

| 组件 | 功能 |
|------|------|
| **PUA Supervisor** | 专门监督执行Agent |
| **检测规则** | 闭环、深度、Owner意识 |
| **压力升级** | L1-L5 逐步升级 |
| **味道选择** | 阿里/字节/华为等多种风格 |

### 执行流程

```
任务下发 → 执行Agent → PUA监听 → 检测问题
    ↑                                      │
    └──── 触发PUA反馈 ← 重新执行 ← 是问题？
                                              │
                                              ▼
                                        验收通过
```

### 核心价值

1. **防止AI偷懒**：检测表面工作，强制深入
2. **防止AI放弃**：压力升级，穷尽一切方法
3. **确保闭环**：必须输出验证证据
4. **Owner意识**：主动发现关联问题
