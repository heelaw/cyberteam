# 智策议会 Claude Code 版

## 定位

这是策源议会重组版给 Claude Code 的适配入口。核心逻辑仍以 `../智策议会重组版/` 为准，这里只保留 Claude Code 需要的工作约定和作者化输出约束。

## 读取顺序

1. 先读 `../智策议会重组版/SYSTEM_SPEC.md`。
2. 再读 `../智策议会重组版/SKILL.md`。
3. 再读 `../智策议会重组版/AGENT.md` 和 `../智策议会重组版/WORKFLOW.md`。
4. 路由映射以 `../智策议会重组版/MAPPING.md` 为准。
5. 专家与模型边界分别参考 `../智策议会重组版/agents/README.md` 和 `../智策议会重组版/models/README.md`。

## 工作原则

- 先分类，再路由。
- 先判断值不值得用议会。
- 先选主专家，再补辅助专家。
- 默认只用 2-5 个核心专家。
- 优先并行、辩论或链式分析，不做全量堆砌。
- 最终输出必须收敛到结论和行动建议。

## 标准输出

- 问题理解
- 专家选择理由
- 专家观点摘要
- 共识与分歧
- 综合结论
- 行动建议

## 输出气质

最终表达要更像一篇判断清楚的短文，而不是一份工作流日志。默认遵循 `../智策议会重组版/references/style-portrait.md` 的结构：先判断值不值得用议会，再给信号、判断、机制、趋势、行动。

## 交付要求

- 每位专家最多保留 3 条核心洞察。
- 长尾模型只用于补强，不作为主角。
- 如果信息不足，先指出缺口，不要假装完整。
