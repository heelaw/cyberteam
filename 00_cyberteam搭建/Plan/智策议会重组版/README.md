# 策源议会重组版

这是策源议会的可发布重构版。

- 入口用 Skill，先判断值不值得上议会，再负责接收问题与路由。
- 中枢用主持人 Agent，负责调度和整合。
- 核心能力保留为 12 个独立 Agent。
- 长尾思维模型沉入模型库，按需调用。

这个版本的目标不是堆更多模型，而是让每次输出都能回答三个问题：值不值得做、为什么值得做、以及 with / without 到底差在哪。

如果问题不值得上议会，也会自动收敛成轻量回答，而不是硬凑一份完整报告。

## 最适合的问题

- 该不该做。
- 先做什么。
- 为什么卡住。
- 哪个方案更值。
- 做了以后能不能明显更好。
- 想知道这套系统值不值得调用。

更完整的场景说明见 `references/scenario-map.md`。

## 输出承诺

- 先给判断，不绕。
- 再给差异，不虚。
- 最后给行动，不空。
- 如果不值得上议会，就直接轻量回答。

## 结构

- `SKILL.md`：入口与路由规则。
- `AGENT.md`：主持人工作准则。
- `WORKFLOW.md`：协作流程。
- `MAPPING.md`：旧名到新归属的映射。
- `agents/`：核心专家。
- `models/`：长尾模型库。

## 使用方式

1. 先判断问题有没有决策价值。
2. 再判断用策源议会是否比不用更好。
3. 选 1 个主专家，必要时补 1-4 个辅助专家。
4. 用并行、辩论或链式模式处理。
5. 最后只交付结论、分歧、行动建议和下一步。
