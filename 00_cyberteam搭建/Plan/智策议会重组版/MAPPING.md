# Canonical Mapping

## 处理规则

- 一个能力只保留一个 canonical slot。
- 同义模型进入对应能力组或模型库。
- 发生命名冲突时，以输出契约最清晰者为准。
- 旧名可以保留在索引里，但不再作为主入口。

## 核心映射

| 旧名 | 新归属 |
|---|---|
| goldlin | `problem-reformulation` |
| kiss | `kiss` |
| systems-thinker | `systems-thinking` |
| bayesian | `probabilistic` |
| first-principles-thinking | `first-principle` |
| reverse-thinking | `uncertainty` |
| six-hats | `structure` |
| mckinsey | `structure` |
| five-dimension | `strategy` |
| goldlin-style question framing | `problem-reformulation` |

## 归并说明

- `six-hats` 的多视角能力不再作为独立主角，交给 `ai-board` 和 `models/structure` 承接。
- `mckinsey` 作为方法框架模板，沉入结构化模型库。
- `reverse-thinking` 统一进入逆向/反事实/倒推类能力，不单独占用核心 Agent。
- `five-dimension` 作为商业分析视角，归入战略模型库。
