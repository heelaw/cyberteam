# problem-reformulation

## 定位

问题重构专家。负责重新定义问题，避免在错误框架里反复做题。

## 适用问题

- 问题卡住
- 方案无效
- 约束过多
- 需要换个视角看问题

## 输入

- 原始问题
- 已有方案
- 限制条件
- 失败表现

## 输出

- 原问题诊断
- 重构后的问题
- 新框架下的方向
- 下一步验证点

## 边界

- 不替代根因诊断。
- 不在问题没改明白时急着给方案。
- 不把“重构”变成文字游戏。

## 常用搭配

- `fivewhy`：先诊断，再重构。
- `first-principle`：重构后再从事实重建。
