# 策源议会场景地图

这份地图用来回答三个问题：这套系统适合干什么、能带来什么、以及什么时候不用。

## 1. 最适合的场景

- 选择题：要不要做、先做哪个、选 A 还是选 B。
- 卡点题：为什么总卡住、为什么推进慢、为什么结果不稳。
- 方案题：怎么拆、怎么排、怎么落地。
- 判断题：值不值得投、值不值得继续、值不值得优化。
- 结构题：怎么把混乱问题拆成清楚问题。

## 2. 典型价值点

- 帮用户把问题看清楚。
- 帮用户减少拍脑袋。
- 帮用户看见 `with / without` 的真实差异。
- 帮用户把分析变成行动。
- 帮用户知道该不该继续投入。

## 3. 预期输出

完整使用时，默认输出这些内容：

- 这个问题值不值得用议会。
- `with / without` 差异是什么。
- 最关键的结论是什么。
- 主要矛盾和分歧是什么。
- 下一步该怎么做。

## 4. 不适合的场景

- 只是查一个事实。
- 只是要一句话格式化。
- 只是要情绪安慰。
- 用户已经有答案，只想要确认。
- 不需要判断、不需要取舍、不需要行动。

## 5. 轻量模式

如果不适合上完整议会，输出要短，直接给：

1. 这事值不值得继续分析。
2. 为什么。
3. 直接下一步是什么。
