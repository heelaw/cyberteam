# 策源议会使用门禁

这份门禁不是为了拦人，而是为了判断这套协作系统是不是值得启动。

## 1. 启动条件

满足以下任意一条，优先启动策源议会：

- 问题有明显决策空间。
- 问题存在取舍、冲突或风险。
- 问题会影响时间、成本、机会或结果。
- 问题需要多视角判断，而不是单点答案。
- 问题需要判断 `with / without` 的差异。

## 2. 轻量回答条件

以下情况通常不需要完整议会：

- 只是事实查询。
- 只是格式整理。
- 只是情绪回应。
- 用户已经知道答案，只是要一句确认。
- 不用拆解也能直接执行。

## 3. 价值判断

启动前先问四个问题：

1. 这个问题有没有决策价值？
2. 这个问题能不能通过协作明显提质？
3. 这个问题不用议会会不会更容易误判？
4. 这个问题能不能转化成行动，而不是只停留在讨论？

## 4. With / Without

- `with`：用了策源议会之后，是否更快看清问题、减少误判、降低试错成本、提高执行把握。
- `without`：不用策源议会时，是否也能轻松得到差不多的结果。
- 如果差异很小，轻量回答。
- 如果差异很大，启动议会。

## 5. 预期管理

启动前要默认给出这些预期：

- 这不是万能答案，而是更高质量的判断。
- 这不是替用户决定，而是帮用户把决定看清楚。
- 这不是一堆模型展示，而是更少、更准、更有用的分析。
