# 策源议会模型解释评估尺

这份尺子用来判断一个思维模型的解释值不值得留在策源议会里。

## 1. 有用的解释，至少要满足一条

- 能改变判断结果。
- 能帮助拆解问题。
- 能指出偏差、风险或边界。
- 能把问题推进到下一步行动。

## 2. 解释质量，优先看这四件事

- **定义是否清楚**：它到底在解释什么，不解释什么。
- **机制是否具体**：不是口号，而是能说清“为什么会这样”。
- **边界是否明确**：什么时候适用，什么时候会失效。
- **输出是否可用**：读完后能不能直接拿去判断或行动。

## 3. 不太值得保留的解释特征

- 只换名词，不换能力。
- 只有概念，没有机制。
- 只有态度，没有判断。
- 只适合装饰，不适合路由。

## 4. 归并原则

- 同一个问题只保留一个主解释。
- 同义模型放进同一个 bucket。
- 如果两个模型都能解释同一件事，优先保留更能落到行动的那个。
- 如果一个模型主要作用只是提醒，就放提醒卡，不单独开入口。

## 5. 最终判断

一个模型能不能留下，不看它听起来多高级，只看它能不能稳定提升这四件事：

1. 看清问题。
2. 拆开问题。
3. 判断值不值得。
4. 推动下一步。
