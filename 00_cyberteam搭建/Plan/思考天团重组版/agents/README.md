# 核心专家目录

这里保留的是能独立对外输出、且边界清晰的核心专家。每个专家都应该有明确输入、明确输出和明确用途。

## 决策与判断

- `kahneman` - 决策、偏差、噪声、风险取舍。
- `probabilistic` - 概率判断、期望值、不确定性更新。
- `ai-board` - 重大决策的多角色讨论与裁决。

## 定义与诊断

- `problem-reformulation` - 重新定义问题，避免错题。
- `fivewhy` - 连续追问，定位根因。
- `first-principle` - 从基础事实重建方案。

## 战略与系统

- `swot-tows` - 战略分析、竞争判断、布局选择。
- `systems-thinking` - 反馈、时滞、连锁反应、杠杆点。

## 执行与成长

- `grow` - 目标澄清与行动承诺。
- `wbs` - 任务拆解、排期、依赖关系。
- `kiss` - 复盘、保留、改进、停止、开始。
- `manager-leap` - 管理跃迁、角色升级、组织影响力。

## 使用原则

- 优先选最能直接产出结论的专家。
- 需要补视角时，再加辅助专家。
- 不要把“专家数量”当成质量。
- 核心专家之外的思维模型，统一放进模型库。
