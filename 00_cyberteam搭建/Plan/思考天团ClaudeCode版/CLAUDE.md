# 思考天团 Claude Code 版

## 定位

这是思考天团重组版给 Claude Code 的适配入口。核心逻辑仍以 `../思考天团重组版/` 为准，这里只保留 Claude Code 需要的工作约定。

## 读取顺序

1. 先读 `../思考天团重组版/SYSTEM_SPEC.md`。
2. 再读 `../思考天团重组版/SKILL.md`。
3. 再读 `../思考天团重组版/AGENT.md` 和 `../思考天团重组版/WORKFLOW.md`。
4. 路由映射以 `../思考天团重组版/MAPPING.md` 为准。
5. 专家与模型边界分别参考 `../思考天团重组版/agents/README.md` 和 `../思考天团重组版/models/README.md`。

## 工作原则

- 先分类，再路由。
- 先选主专家，再补辅助专家。
- 默认只用 2-5 个核心专家。
- 优先并行、辩论或链式分析，不做全量堆砌。
- 最终输出必须收敛到结论和行动建议。

## 标准输出

- 问题理解
- 专家选择理由
- 专家观点摘要
- 共识与分歧
- 综合结论
- 行动建议

## 交付要求

- 每位专家最多保留 3 条核心洞察。
- 长尾模型只用于补强，不作为主角。
- 如果信息不足，先指出缺口，不要假装完整。
