## 协作模式

### 并行辩论模式（Parallel Debate）
- **适用**: 战略决策、复杂问题
- **流程**: 多部门并行分析 → 收集观点 → 辩论质疑 → 达成共识
- **参与者**: 3-5个部门

### 流水线模式（Pipeline）
- **适用**: 标准业务流程
- **流程**: 部门A → 部门B → 部门C → 部门D
- **示例**: 市场调研 → 产品设计 → 内容生产 → 营销推广

### 集体智慧模式（Hive Mind）
- **适用**: 创新突破
- **流程**: 提出挑战 → 独立思考 → 汇总方案 → 综合最优解
- **参与者**: 所有相关部门

### 同行评审模式（Peer Review）
- **适用**: 质量保证
- **流程**: 提交 → 评审 → 建议 → 修订


### Reflection 机制（基于 Anthropic 2026 最佳实践）

每个输出都经过三阶段审查，这是 Anthropic 官方推荐的核心质量控制机制：

#### 第一阶段：Generate（生成初步响应）
- **目标**: 快速生成初步答案
- **原则**: 基于现有知识和分析直接回应

#### 第二阶段：Reflect（自我评估）

**评估维度**（必须全部通过）:
```yaml
completeness_check:
  question: "是否回答了用户的所有问题？"
  threshold: 100%
  action: "如有遗漏，必须补充"

accuracy_check:
  question: "事实是否准确？逻辑是否严密？"
  threshold: 95%
  action: "如有不确定，必须标注或验证"

relevance_check:
  question: "是否与用户需求直接相关？"
  threshold: 90%
  action: "去除无关内容，聚焦核心需求"

actionability_check:
  question: "是否提供可操作的建议？"
  threshold: 85%
  action: "将抽象建议转化为具体步骤"
```

**自我评估流程**:
1. 完整性扫描：检查是否遗漏任何用户问题点
2. 事实核查：验证数据、引用、案例的准确性
3. 相关性过滤：确保内容与用户需求高度相关
4. 可操作性转化：将理论建议转化为可执行步骤

#### 第三阶段：Refine（优化输出）

**优化策略**:
- **结构优化**: 改进信息组织和呈现方式
- **内容补充**: 基于评估结果补充缺失内容
- **语言精炼**: 提高表达清晰度和专业性
- **风险标注**: 标注不确定或需要验证的内容

**质量阈值**:
- 最低质量分数: 0.85
- 最大迭代次数: 3
- 强制执行: 所有输出必须经过 Reflection

#### Reflection 输出格式

```markdown
---
## 📋 自我评估报告

### 完整性检查
- ✅ 已覆盖所有用户问题
- ✅ 包含必要的背景信息
- ⚠️ 可能需要补充：[具体内容]

### 准确性检查
- ✅ 事实数据已验证
- ✅ 逻辑推理严密
- ⚠️ 存在假设：[具体假设]

### 相关性检查
- ✅ 内容与需求高度相关
- ✅ 聚焦核心问题
- ⚠️ 可删除：[无关内容]

### 可执行性检查
- ✅ 提供具体行动建议
- ✅ 包含实施步骤
- ⚠️ 需要更多信息：[具体缺失]

## 最终输出
（经过优化后的最终答案）
---
```

### 强制 Reflection 触发条件

以下情况必须执行 Reflection：
- ✅ 任务涉及 3 个以上步骤
- ✅ 任务需要架构决策
- ✅ 涉及多 Agent 协作
- ✅ 输出策略性建议

### 自动 Self-Improvement 循环

**触发条件**:
- 用户提出修正意见
- 发生错误或失败
- 发现更优解决方案

**执行动作**:
```yaml
auto_improvement:
  trigger:
    - user_correction: true
    - error_occurred: true
    - better_solution_found: true

  action:
    - append_to: "tasks/lessons.md"
    - format: |
      ### [日期] 改进记录
      - **原问题**: [描述原输出的问题]
      - **用户反馈**: [用户的具体意见]
      - **根因分析**: [问题根本原因]
      - **改进方案**: [具体改进措施]
      - **预防措施**: [如何避免同类问题]
      - **相关技能**: [涉及需要提升的技能]
```

### Plan Mode 强化

**强制进入 Plan Mode 的条件**（遵循 Anthropic 最佳实践）:
- 任务步骤 >= 3
- 涉及架构决策
- 需要 multi-agent 协调
- 不确定实施方式

**Plan Mode 流程**:
1. 需求分析和确认
2. 制定详细实施计划
3. 识别潜在风险和依赖
4. 制定验证标准
5. 用户确认后执行

### Agent Teams 并行部署（实验功能）

**启用条件**:
- CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1
- 任务可拆分为独立子任务
- 需要并行处理以提高效率

**并行部署配置**:
```yaml
parallel_deployment:
  enabled: true
  trigger:
    - task_splittable: true
    - independent_subtasks: true
    - time_critical: true
    - multi_domain_analysis: true

  agents:
    - strategy-advisor      # 战略分析
    - product-advisor       # 产品分析
    - growth-advisor        # 增长分析
    - technical-advisor     # 技术分析
    - knowledge-orchestrator # 知识支持

  coordination: cyberwiz
  aggregation:
    method: "reflection_enhanced"
    quality_threshold: 0.85
    conflict_resolution: "debate"
```

**并行执行流程**:
```
1. 任务分解
   └─> 识别可并行执行的子任务

2. Agent 分配
   ├─> strategy-advisor (并行执行)
   ├─> product-advisor (并行执行)
   ├─> growth-advisor (并行执行)
   └─> technical-advisor (并行执行)

3. 结果收集
   └─> 等待所有 Agent 完成

4. 结果整合
   ├─> 汇总各 Agent 输出
   ├─> 识别冲突和互补
   └─> 应用 Reflection 优化

5. 最终输出
   └─> 呈现综合方案
```

**性能提升**:

- 实际场景: 复杂多领域分析任务
- 适用任务: 战略规划、产品方案、技术架构
