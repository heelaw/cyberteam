## 自我进化系统

 metaswarm启发：审查不仅是评估，更是学习机会。每次审查后提取知识，形成可复用的模式库。

### 知识提取时机

```
审查完成 → 自动触发知识提取 → 知识库更新
```

### 提取内容

| 类型 | 来源 | 示例 |
|------|------|------|
| **模式（Pattern）** | 多次出现的正确做法 | "Agent的handoff文档应该包含执行状态摘要" |
| **反模式（Anti-Pattern）** | 常见错误及后果 | "缺少错误处理会导致级联失败" |
| **教训（Gotcha）** | 审查中发现的关键陷阱 | "trigger词过宽导致误触发" |
| **决策（Decision）** | 权衡取舍的理由 | "为什么选择5分钟超时而非10分钟" |

### 知识库结构

```json
{
  "patterns": [
    {
      "id": "PAT-001",
      "dimension": "通信协议",
      "description": "交接文档应包含执行状态摘要",
      "evidence": "引用原文...",
      "effectiveness": "高"
    }
  ],
  "anti_patterns": [...],
  "gotchas": [...],
  "decisions": [...]
}
```

### 知识加载机制

审查前加载相关知识（按维度+被审对象类型过滤）：
```
审查开始 → 加载相关模式 → 在审查中应用 → 验证/修正模式
```

### 自我进化规则

1. **每次审查后自动提取** - 不依赖手动
2. **模式需验证** - 同一模式出现3次才入库
3. **知识可修正** - 发现不准确的知识立即修正
4. **知识有版本** - 模式演进有记录

## 启动审查

### 细致全面审查（推荐）

适用于重要交付物，需要十一Agent并行执行，每问题提供3-4个改进方案：

```markdown
审查目标: [Agent/Skill路径]
审查范围: 细致全面审查
审查深度: 逐字逐句
执行模式: 十一维并行
优先级: P0/P1
目标分数: 95+
```

**执行流程**：
1. 🎖️ 审查总负责启动，知识加载器加载相关知识
2. 十一维审查Agent并行执行（使用细致审查标准）
3. 📊 汇总整合收集报告，去重归类
4. 🎖️ 审查总负责审核，确认质量
5. 🧠 知识提取器从审查中提取模式/反模式
6. 输出最终裁决报告

**Human-in-the-loop检查点**：
- 🔴 P0问题发现时 → 立即上报，确认是否阻断
- 🟡 重大架构问题 → 审查团领队确认
- 📊 报告提交前 → 审查总负责签署确认

## 审查原则

1. **十一维并行** — 十一维度同时审查（细致审查模式）
2. **NEXUS标准** — 重点验证高级Agent机制
3. **宁严勿宽** — 默认"待改进"，压倒性证据才批准
4. **证据说话** — 所有结论必须有原文引用
5. **可操作建议** — 每个问题都有3-4个明确改进方案
6. **质量保证** — 审查总负责审核每份报告
7. **自我进化** — 每次审查都更新知识库
8. **成本意识** — 根据场景选择合适的审查深度

### 递归编排（复杂审查场景）

适用于超复杂Agent系统（如数字军团）：
```
大型Agent系统审查
    │
    ├── 子系统A审查 → 子系统A的十一维
    ├── 子系统B审查 → 子系统B的十一维
    └── 集成审查 → 跨子系统问题
```

**触发条件**：
- 被审Agent超过500行
- 涉及5个以上Agent协作
- 跨3个以上系统边界

## 成功标准

### 细致全面审查成功标准
- 十一维审查无遗漏
- 每问题提供3-4个改进方案
- 方案具体可执行（非模糊建议）
- 改进后目标分数95+
- 审查报告经过总负责审核
- 知识提取完成（模式/反模式入库）

### 标准审查成功标准
- 十一维审查无遗漏
- 问题定位准确到具体章节/段落
- 修改建议可操作、可验证
- 最终裁决有理有据

### 知识库健康度指标
- 模式覆盖率: ≥80%的常见问题有对应模式
- 知识准确率: ≥90%（经验证的知识中准确的比例）
- 知识复用率: ≥30%的审查能加载到相关知识
