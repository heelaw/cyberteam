## 模块1：框架式思维

### 方法论1.1：自上而下法建立框架

**核心描述**：对有一定了解的领域，先划分层次，再逐层填充内容，最后迭代优化。

**步骤模板**：
1. 划分层次：这个问题可以分成哪几个层次？
2. 逐层填充：针对每个层次进一步分析和填充
3. 迭代优化：框架逐渐变得简洁

**案例**：用户运营 = 用户获取 + 用户留存 + 用户变现

| 维度 | 评分 |
|------|------|
| 颗粒度 | 4/5 |
| 可操作性 | 8/10 |
| 模板化程度 | 9/10 |
| 场景通用性 | 10/10 |
| **Skill可行性** | **9/10** ✅ |

**推荐Skill名称**：`framework-top-down` - 自上而下框架构建法

### 方法论1.2：自下而上法建立框架

**核心描述**：对完全陌生的领域，先海量收集信息，再整理归类，最后提炼框架。

**步骤模板**：
1. 海量收集：问人、爬数据、查资料
2. 整理归类：相似答案归为一类
3. 提炼框架：归纳出若干维度/方向

**案例**：从招聘JD中提取用户运营工作内容，归纳为5大模块

## 模块2：运营框架理解与指标拆解

### 方法论2.1：指标拆解-并列式拆解

**核心描述**：将一个大指标按并列关系拆解为多个子指标。

**步骤模板**：
```
总指标 = 子指标1 + 子指标2 + 子指标3 + ...
```

**案例**：
- 用户运营 = 用户获取 + 用户激活 + 用户留存 + 用户付费 + 用户传播
- 影响力 = 传播 + 曝光 + 权威

## 模块5：用户行为激励

### 方法论5.1：用户留存差的4种原因诊断

**核心描述**：诊断用户留存/活跃差的4种原因及应对策略。

**诊断框架**：
```
原因1：产品长期价值不足
├── 伪需求 → MVP验证
├── 服务能力不够 → 设定边界/封闭运营
└── 短期需求 → 快速迭代延伸业务

原因2：产品短期价值缺失
├── 用户路径不明确 → 优化引导
├── 缺乏激励刺激 → 设计激励体系
└── 缺乏亮点 → 增加差异化元素

原因3：产品天然低频
├── 放弃活跃，品牌+入口覆盖
├── 挖掘变现路径
└── 增加高频新业务

原因4：用户不匹配
└── 调整推广策略和渠道
```

## 模块6：策略执行

### 方法论6.1：成功三要素框架

**核心描述**：做任何事成功的3个要素（方向×路径×执行）。

**框架公式**：
```
成功 = 方向选择 × 实现路径 × 落地执行
```

- 方向选择：高质量的战略决策
- 实现路径：具备有效性和可行性的方案
- 落地执行：高质量的具体动作

任一要素为0，整体失败
