## 详细说明（P0-P1优先级）

### 1. 运营指标拆解方法 【P0】

**核心方法论**:
- **加法式拆解**: 流量 = 渠道1 + 渠道2 + ... + 渠道N
- **乘法式拆解**: 转化率 = 环节1 × 环节2 × ... × 环节N
- **典型代表式**: 找到占80-90%的典型路径进行优化

**输入参数**:
- 目标指标类型（流量/转化率/销售额等）
- 业务场景和数据结构
- 历史数据

**输出形态**:
- 指标拆解树状图
- 各子指标的目标值
- 优化方向建议

**制作难点**:
- 需要理解不同业务类型的指标构成
- 拆解后的指标需要可执行
- 需要提供行业基准数据参考

**Skill可行性评分**: ⭐⭐⭐⭐⭐⭐ (6/10)
- 优点: 方法论清晰，可模板化强
- 缺点: 需要大量行业数据支持

---

### 2. SOP标准作业程序 【P0】

**核心方法论**:
- 将重复性工作标准化为步骤清单
- 包含: 做什么、怎么做、质量标准
- 提升团队协作效率

**输入参数**:
- 工作类型和场景
- 工作步骤清单
- 质量检查标准

**输出形态**:
- 标准化操作文档
- 步骤检查清单
- 质量验收标准

**制作难点**:
- 不同工作的SOP差异大
- 需要区分必要步骤和可选步骤
- 平衡详细度和灵活性

**Skill可行性评分**: ⭐⭐⭐⭐⭐ (5/10)
- 优点: 适用范围广，模板化强
- 缺点: 需要大量行业SOP案例库

---

### 3. 产品生命周期运营策略 【P1】

**核心方法论**:
- **引入期**: 聚焦产品体验和性能优化
- **成长期**: 快速增长，抢占市场
- **成熟期**: 提升活跃度，商业变现
- **衰退期**: 防止流失，维系用户

**输入参数**:
- 产品当前用户量级
- 增长趋势数据
- 竞争格局

**输出形态**:
- 当前阶段判断
- 运营重点建议
- 风险预警

**制作难点**:
- 阶段判断需要量化标准
- 不同行业阶段特征不同
- 需要避免过度操盘

**Skill可行性评分**: ⭐⭐⭐⭐⭐⭐ (6/10)
- 优点: 理论成熟，案例丰富
- 缺点: 需要行业定制化

---

### 4. 营销推广五步法 【P1】

**核心方法论**:
1. 定义用户、入口与推广方式
2. 评估渠道，熟悉规则
3. 搭建转化场景
4. 落实物料，排期落地
5. 数据监测与优化

**输入参数**:
- 产品/服务信息
- 目标用户画像
- 预算和时间周期

**输出形态**:
- 推广方案框架
- 渠道选择建议
- 转化场景设计
- 物料清单

**制作难点**:
- 渠道规则变化快
- 转化场景设计需要创意
- 需要行业案例库

**Skill可行性评分**: ⭐⭐⭐⭐⭐ (5/10)
- 优点: 流程完整，可落地
- 缺点: 渠道信息时效性强

---

### 5. 推广入口发现四法 【P1】

**核心方法论**:
- **联想激发式**: 看A内容激起消费B欲望
- **问题解决式**: 发生A事件不得不考虑B
- **用户聚集式**: 需求B的用户活跃在A平台
- **搜索行为式**: 用户产生需求时如何寻找

**输入参数**:
- 产品/服务特点
- 目标用户画像
- 现有渠道列表

**输出形态**:
- 潜在推广入口清单
- 各入口特点分析
- 优先级排序

**制作难点**:
- 需要发散思维能力
- 需要大量创意案例
- 难以量化评估

**Skill可行性评分**: ⭐⭐⭐⭐⭐⭐ (6/10)
- 优点: 思维模型清晰
- 缺点: 高度依赖创意

---

### 6. 文案转化策略四步法 【P1】

**核心方法论**:
1. 明确目标用户（用户画像）
2. 分析用户需求（痛点+卖点）
3. 确定场景和阅读时间
4. 设计内容逻辑结构

**输入参数**:
- 产品/服务信息
- 目标用户画像
- 用户痛点和产品卖点
- 发布场景和时间

**输出形态**:
- 文案策略方案
- 内容结构框架
- 关键信息优先级

**制作难点**:
- 需要深入理解用户心理
- 场景判断需要经验
- 内容逻辑设计需要创意

**Skill可行性评分**: ⭐⭐⭐⭐⭐ (5/10)
- 优点: 步骤清晰，可操作性强
- 缺点: 文案质量依赖创意

---

### 7. 转化流程优化 【P1】

**核心方法论**:
- **三要素**: 曝光 → 点击 → 转化
- **三类优化**:
  1. 优化流量（拓展关键词/渠道）
  2. 优化或缩短流程（每增加一环节流失20%+）
  3. 提升特定环节转化率

**输入参数**:
- 当前转化流程
- 各环节数据
- 行业基准数据

**输出形态**:
- 转化漏斗分析
- 流程优化建议
- A/B测试方案

**制作难点**:
- 需要精细的数据监测
- 行业基准数据获取难
- 优化需要多轮测试

**Skill可行性评分**: ⭐⭐⭐⭐ (4/10)
- 优点: 方法论科学
- 缺点: 数据依赖强

## P2-P3优先级简述

### 用户分层运营 【P2】
双层次方法论（微观特定用户关怀+宏观数据机制设计），理论清晰但实施复杂，需要大量数据支持。

### 内容选题8方向 【P2】
8个选题方向的方法论清晰，但内容质量高度依赖创意，Skill辅助空间有限。

### 活动策划方案框架 【P2】
8要素框架完整，但活动设计需要创意，Skill主要提供结构化模板。

### 精准定义潜在用户三法 【P3】
三种方法（数据分析/调研/测试）实用，但方法论较为基础，独立成Skill价值有限。

### PDCA循环优化法 【P3】
通用管理方法，不限于运营，场景通用性高但运营特异性低。

### 信息分发平台推广二法 【P3】
渠道特定性强，方法论相对简单，独立成Skill必要性不高。
