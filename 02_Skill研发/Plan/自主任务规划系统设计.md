# CyberTeam 自主任务规划系统

## 核心理念

**用户输入一句话 → AI自动完成规划、组队、执行全流程**

```
用户: "帮我实现一个博客系统"
  ↓
AI: 分析需求 → 分解任务 → 组建团队 → 分配任务 → 协调执行 → 输出成果
```

---

## 系统架构

### 1. Task Analyzer（任务分析器）

**职责：** 深度理解用户需求，识别任务类型和复杂度

**分析维度：**
```yaml
任务类型:
  - 软件开发: 前端/后端/全栈/移动端
  - 数据分析: 探索/可视化/建模
  - 文档创作: 技术文档/用户手册/教程
  - 研究调查: 竞品分析/市场调研/技术调研

复杂度评估:
  - 简单: 1-2个角色，1天内完成
  - 中等: 3-5个角色，3-7天完成
  - 复杂: 6-10个角色，1-4周完成
  - 超复杂: 10+角色，需要多阶段迭代

技术栈识别:
  - 从描述中提取关键词
  - 推断所需技术栈
  - 识别潜在依赖
```

**Prompt 模板：**
```
你是任务分析专家。分析用户需求：

用户需求：{user_input}

请输出：
1. 任务类型（软件开发/数据分析/文档创作/研究调查）
2. 核心目标（一句话概括）
3. 技术栈建议
4. 预估复杂度（简单/中等/复杂/超复杂）
5. 所需专家角色列表
6. 主要交付物
7. 潜在风险点
```

**团队组合策略：**
```python
def compose_team(task_analysis):
    """根据任务分析组合最优团队"""
    team = {
        "core_members": [],
        "support_members": [],
        "coordinator": None
    }

    # 必选核心成员
    if "软件开发" in task_analysis.type:
        team["core_members"].extend([
            "architect",
            "frontend_dev" if "前端" in task_analysis.scope else "backend_dev"
        ])
        team["coordinator"] = "project_coordinator"

    # 条件成员
    if task_analysis.complexity >= "复杂":
        team["support_members"].extend([
            "tester",
            "tech_writer"
        ])

    if "安全" in task_analysis.keywords:
        team["support_members"].append("security_auditor")

    return team
```

### 3. Task Decomposer（任务分解器）

**职责：** 将大任务智能分解为可执行的小任务

**分解原则：**
```
1. 独立性：每个任务可独立完成
2. 可测性：每个任务有明确的验收标准
3. 依赖性：标注任务间的依赖关系
4. 时序性：合理安排执行顺序
```

**分解算法：**
```python
def decompose_task(user_requirement, team_members):
    """智能分解任务"""

    # 步骤1: 提取核心功能点
    features = extract_features(user_requirement)

    # 步骤2: 为每个功能点创建任务
    tasks = []
    for feature in features:
        task = {
            "id": generate_id(),
            "title": feature["name"],
            "description": feature["description"],
            "owner": match_expert(feature, team_members),
            "priority": calculate_priority(feature),
            "dependencies": find_dependencies(feature, features),
            "acceptance_criteria": generate_criteria(feature)
        }
        tasks.append(task)

    # 步骤3: 拓扑排序确定执行顺序
    return topological_sort(tasks)
```

### 4. Dynamic Agent Spawner（动态专家生成器）

**职责：** 根据需要动态创建专家，而非使用固定模板

**生成流程：**
```python
def spawn_expert(role, context):
    """动态生成专家"""

    # 1. 从专家库获取基础配置
    base_config = EXPERT_DATABASE.get(role)

    # 2. 根据上下文定制prompt
    custom_prompt = customize_prompt(
        base_template=base_config["base_prompt"],
        task_context=context["current_task"],
        team_context=context["team_composition"],
        project_context=context["project_info"]
    )

    # 3. 创建专家
    agent = cyberteam_spawn(
        agent_name=f"{role}_{context['iteration']}",
        agent_type="general-purpose",
        task=custom_prompt
    )

    return agent
```

### 5. Auto Coordinator（自动协调器）

**职责：** 监控进度、处理阻塞、动态调整

**核心功能：**

**5.1 进度监控**
```python
def monitor_progress():
    """持续监控任务进度"""
    while True:
        for task in get_active_tasks():
            if is_stuck(task):
                # 任务卡住了
                escalate_to_coordinator(task)

            if is_blocked(task):
                # 找到阻塞点并解决
                resolve_blocker(task)

            if is_completed(task):
                # 任务完成，触发下游任务
                trigger_dependent_tasks(task)
```

**5.2 动态任务创建**
```python
def on_blocker_discovered(blocker):
    """发现阻塞时自动创建解决任务"""
    new_task = {
        "title": f"解决阻塞: {blocker.description}",
        "owner": find_best_expert(blocker.type),
        "priority": "urgent",
        "created_by": "system_auto"
    }
    create_task(new_task)
```

**5.3 智能资源调度**
```python
def optimize_resources():
    """优化资源分配"""
    idle_experts = find_idle_experts()
    pending_tasks = get_pending_tasks()

    # 匹配空闲专家和待处理任务
    for expert in idle_experts:
        best_task = find_best_match(expert, pending_tasks)
        if best_task:
            assign_task(expert, best_task)
```
