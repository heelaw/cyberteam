# Skill质量审核清单

## 概述

本清单用于审核Agent产出的Skill质量，确保每个Skill都符合课程原文、具备完整的结构，并能够有效解决用户的实际问题。

---

## 一、必检项（0分项）

以下任何一项不满足，则该Skill为不合格，需要重构。

### 1.1 课程内容理解

| 检查项 | 要求 | 权重 |
|-------|------|------|
| 课程逐字稿 | 必须先完整阅读相关课程逐字稿才能开始设计 | 必检 |
| 理论体系来源 | Skills必须基于课程原文的方法论，不能凭空创造 | 必检 |
| 引用标注 | 关键概念和方法必须标注来源课程 | 必检 |

### 1.2 结构完整性

| 检查项 | 要求 | 权重 |
|-------|------|------|
| 用户实际问题 | 必须包含具体的用户问题场景 | 必检 |
| 分析流程 | 必须包含分析问题的步骤 | 必检 |
| 思考流程 | 必须包含思考问题的框架 | 必检 |
| 拆解流程 | 必须包含问题拆解的方法 | 必检 |
| Critical Rules | 必须包含至少3条必须遵守的规则 | 必检 |
| 禁止行为 | 必须包含至少5条禁止行为 | 必检 |

### 1.3 评估文件齐全

| 检查项 | 要求 | 权重 |
|-------|------|------|
| assess目录 | 必须存在，包含评估清单 | 必检 |
| evals目录 | 必须存在，包含测试案例 | 必检 |
| references目录 | 必须存在，包含参考资料 | 必检 |

---

## 二、评分标准

### 2.1 评分维度

| 维度 | 权重 | 说明 |
|------|------|------|
| 课程原文学习 | 20% | 是否完整学习了课程内容并正确理解 |
| 理论体系完整性 | 20% | 方法论是否完整、是否与课程一致 |
| 结构完整性 | 20% | 各个部分是否齐全 |
| Critical Rules质量 | 15% | 规则是否具体、可执行 |
| 禁止行为质量 | 15% | 禁止项是否覆盖常见错误 |
| 评估文件质量 | 10% | 评估清单和测试案例是否完善 |

### 2.2 评分等级

| 分数 | 等级 | 处理方式 |
|------|------|----------|
| 90-100 | 优秀 | 可直接使用 |
| 80-89 | 良好 | 小修小补后可使用 |
| 70-79 | 需修改 | 较大修改后可用 |
| <70 | 需重构 | 不合格，需重新设计 |

## 三、详细检查清单

### 3.1 SKILL.md 检查清单

#### 3.1.1 头部信息（10分）

- [ ] name: 技能名称是否清晰、符合命名规范
- [ ] description: 描述是否包含触发关键词和场景
- [ ] version: 版本号是否正确
- [ ] category: 分类是否合适
- [ ] tags: 标签是否覆盖主要场景

#### 3.1.2 Identity & Memory（10分）

- [ ] 角色定义是否明确
- [ ] 关键记忆是否准确反映方法论核心

#### 3.1.3 Core Mission（10分）

- [ ] 核心任务描述是否清晰
- [ ] 任务边界是否明确

#### 3.1.4 Critical Rules（15分）

- [ ] 是否包含至少3条规则
- [ ] 规则是否具体、可执行
- [ ] 规则是否与课程方法论一致

#### 3.1.5 禁止行为（15分）

- [ ] 是否包含至少5条禁止行为
- [ ] 禁止行为是否覆盖常见错误场景
- [ ] 禁止行为是否具体、可检查

#### 3.1.6 Workflow Process（15分）

- [ ] 步骤是否完整（至少3步）
- [ ] 每步是否包含输入/处理/输出/成功标准
- [ ] 步骤之间逻辑是否连贯
- [ ] 方法论是否与课程一致

#### 3.1.7 Deliverable Template（10分）

- [ ] 是否包含输出模板
- [ ] 模板是否结构化、可执行

#### 3.1.8 Communication Style（5分）

- [ ] 沟通风格是否明确
- [ ] 是否强调关键注意事项

#### 3.1.9 Skill协同（5分）

- [ ] 是否标注上下游Skill
- [ ] 协同流程是否清晰

#### 3.1.10 References（5分）

- [ ] 是否包含参考资料列表
- [ ] 参考资料是否与课程相关

## 七、使用说明

### 7.1 审核流程

1. **必检项检查**：首先检查所有必检项，任何一项不通过则直接判定为不合格
2. **逐项评分**：对每个评分维度进行逐项检查和打分
3. **总分计算**：汇总所有评分项得分
4. **等级判定**：根据总分确定等级
5. **问题汇总**：列出必须修复和建议修复的问题

### 7.2 审核重点

- **课程原文一致性**：确保Skill严格基于课程内容
- **结构完整性**：确保各部分齐全
- **可执行性**：确保规则和流程可落地执行
- **评估有效性**：确保评估体系能有效检验Skill质量

### 7.3 注意事项

- 不要对未学习过课程内容的Skill给予高分
- 禁止行为少于5条的Skill必须修复
- Critical Rules少于3条的Skill必须修复
- 缺少任何评估目录的Skill必须修复
