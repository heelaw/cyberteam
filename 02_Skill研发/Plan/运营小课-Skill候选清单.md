## 二、候选Skill详细分析

### 1. 流程化思维与流程梳理

**来源课程**：`深刻理解4种常见的运营数据漏斗`（第1章）

**核心方法论**：

- **三要素**：目标 → 流程 → 关键影响因素
- **五符号**：开始/结束框、处理框、判断框、输入输出框、流程线
- **四原则**：从左到右/从上到下、开始结束各一次、无交叉、判断需注释
- **三结构**：顺序结构、选择结构、循环结构
- **泳道图**：多部门协作时的流程图强化版

**评分明细**：
- 颗粒度：4/5（结构化程度高）
- 可操作性：8/10（有具体步骤和符号）
- 模板化程度：8/10（可直接套用）
- 通用性：8/10（适用于各类运营工作）
- **Skill可行性：8.5/10**

**可复用项目**：
- `skill-coordination-test` - 已有类似技能，可参考整合

### 2. 案例拆解方法论

**来源课程**：`10倍放大个人学习效率`（第1-2章）

**核心方法论**：

- **两种拆解逻辑**：
  1. 单案例深度拆解：还原成果 → 归纳目的 → 提炼细节
  2. 分类别持续拆解：建立结构 → 批量拆解 → 标注亮点 → 归档积累

- **活动拆解四维度**：主题、形式玩法、流程、活动类型
- **内容拆解五维度**：标题、内容结构、内容表现形式、开篇结语、金句素材

**评分明细**：
- 颗粒度：4/5
- 可操作性：8/10
- 模板化程度：7/10
- 通用性：7/10
- **Skill可行性：8.0/10**

**与现有Skill对比**：
- 需与 `skill-coordination-test` 的案例分析部分整合

### 3. 产品生命周期与PMF

**来源课程**：`从0到100，不同业务如何思考宏观运营操盘节奏`（第1-2章）

**核心方法论**：

- **产品生命周期四阶段**：引入期 → 成长期 → 成熟期 → 衰退期
- **S曲线模型**：横坐标时间，纵坐标用户量
- **三类关键节点**：
  1. 生存焦虑期（引入期）- 验证需求、积累口碑
  2. 增长拐点（引入→成长）- PMF出现
  3. 市场份额期（成长中后期）- 跑马圈地

- **PMF概念**：Product Market Fit，产品价值与市场需求的最佳平衡点
- **MVP快速验证**：最小可行产品测试

**评分明细**：
- 颗粒度：4/5
- 可操作性：8/10
- 模板化程度：7/10
- 通用性：8/10
- **Skill可行性：8.5/10**

### 4. 用户场景需求三要素

**来源课程**：`运营必学-从根本提升你的流量-用户敏锐度与洞察`（第1章）

**核心方法论**：

- **三要素框架**：
  1. **用户** - 谁（画像：地域、性别、年龄、职业、特征）
  2. **场景** - 何时何地何种环境（时间、地点、人物、环境）
  3. **需求** - 生理需求 + 心理需求（功能性 + 情感性）

- **应用场景**：
  1. 日常内容/活动：时刻以三要素为原点与用户沟通
  2. 营销文案：锁定需求 → 堆砌场景 → 戳中痛点

**评分明细**：
- 颗粒度：5/5（非常清晰）
- 可操作性：9/10
- 模板化程度：8/10
- 通用性：9/10
- **Skill可行性：9.0/10**

**可复用项目**：
- `target-user-segmentation` - 已有用户定位相关技能
- `user-needs-mining-6channels` - 已有需求挖掘相关技能

### 5. 营销转化链路4要素

**来源课程**：`运营必学-一切营销推广、转化链路的设计关键原则`（第4章）

**核心方法论**：

- **四要素模型**：
  1. **必要性** - 用户为什么一定要买（痛点+场景+方案）
  2. **唯一性** - 为什么在你这里买（对比+背书+细节）
  3. **紧迫性** - 为什么现在买（限时+限量+折扣）
  4. **信任度** - 为什么相信你（品牌背书+用户评价+真人出镜+承诺）

- **转化诱因库**：每个要素下的常见诱因清单

**评分明细**：
- 颗粒度：5/5
- 可操作性：9/10
- 模板化程度：8/10
- 通用性：8/10
- **Skill可行性：8.5/10**

### 9. KANO模型

**来源课程**：`用户洞察与需求定位`（第5章）

**核心方法论**：

- **五类需求**：
  1. **必备属性** - 必须有，没有不行（基本需求）
  2. **期望属性** - 有则更好，多多益善
  3. **兴奋属性** - 超预期，带来惊喜
  4. **无差异属性** - 有没有无所谓
  5. **反向属性** - 有了反而更差

- **应用**：需求优先级排序

**评分明细**：
- 颗粒度：4/5
- 可操作性：7/10
- 模板化程度：7/10
- 通用性：7/10
- **Skill可行性：7.5/10**

### 10. 价值努力矩阵

**来源课程**：`用户洞察与需求定位`（第5章）

**核心方法论**：

- **四象限**：
  - **高价值+低努力** → P0（最高优先）
  - **高价值+高努力** → P1（重要但需规划）
  - **低价值+低努力** → P1（可做非高优）
  - **低价值+高努力** → 放弃/推迟

- **评估维度**：
  - 价值：用户需求度 + 业务价值
  - 努力：时间 + 人力 + 资金 + 复杂度

**评分明细**：
- 颗粒度：4/5
- 可操作性：7/10
- 模板化程度：7/10
- 通用性：7/10
- **Skill可行性：7.5/10**
