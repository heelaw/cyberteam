## 课程定义的流程架构

### 核心框架：用户场景需求三要素

```
基石3要素       流量密码洞察       用户理解           用户互动
```

### 步骤1：理解基石三要素（第一章）
**核心概念：用户-场景-需求**

- 需求：什么需求（生理需求/心理需求 = 功能需求/情感需求）

**三要素的应用**：
- 在用户、场景、需求三个维度上分别细分
- 延伸出N多的运营思路

## Agent层与Skill层划分

### Agent层设计

**Agent: 用户场景需求分析Agent**
- 输入：产品信息、目标用户
- 决策：定义用户 → 分析场景 → 识别需求 → 细分延伸
- 输出：完整的用户场景需求分析

**Agent: 流量洞察Agent**
- 输入：业务场景、流量数据
- 决策：选择洞察方法 → 发现流量密码
- 输出：流量洞察报告

**Agent: 用户互动设计Agent**
- 输入：用户特征、业务目标
- 决策：选择互动方法 → 设计活跃方案
- 输出：互动活跃方案

### Skill层设计（最小单元）

| 技能领域 | Skill名称 | 输入 | 工具/框架 | 输出 |
|----------|-----------|------|-----------|------|
| 认知 | 用户定义 | 产品 | 用户模型 | 用户画像 |
| 认知 | 场景分析 | 用户行为 | 场景模型 | 场景描述 |
| 认知 | 需求识别 | 用户 | 需求模型 | 需求列表 |
| 认知 | 三要素细分 | 三要素 | 细分方法 | 运营思路 |
| 洞察 | 流量洞察 | 流量数据 | 9大秘诀 | 洞察结果 |
| 洞察 | 流量密码发现 | 业务场景 | 发现方法 | 密码清单 |
| 画像 | 用户特征分析 | 用户数据 | 特征模型 | 特征描述 |
| 画像 | 用户画像构建 | 特征数据 | 画像模板 | 用户画像 |
| 互动 | 用户互动 | 用户+场景 | 9大秘诀 | 互动方案 |
| 互动 | 活跃神器 | 互动目标 | 3种神器 | 活跃方案 |

## 总结

运营小课的核心理念：
1. **用户场景需求三要素** - 一切运营工作的基石
2. **流量密码洞察** - 快速发现流量机会
3. **用户深度理解** - 构建精准用户画像
4. **用户互动活跃** - 设计有效的用户互动

这个课程聚焦于运营工作中最基础也是最重要的能力：理解用户、洞察流量、设计互动。强调从用户角度出发，在正确的场景下满足用户需求。
