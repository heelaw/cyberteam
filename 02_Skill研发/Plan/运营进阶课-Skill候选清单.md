## 二、底层逻辑模块

### 2.1 产品生命周期运营策略

**方法论描述**：根据产品所处生命周期阶段（探索期、成长期、成熟期、衰退期）制定对应的运营策略

| 阶段 | 运营侧重点 | 核心动作 |
|------|------------|----------|
| 探索期 | 打造口碑 | 提供超出用户预期的体验 |
| 成长期 | 快速增长 | 补贴/广告投放/事件策划跑通增长模式 |
| 成熟期 | 活跃度+变现 | 精细化运营+商业变现尝试 |
| 衰退期 | 防流失 | 用户维系+减少流失 |

### 2.2 精益运营工作流程

### 3.3 无用指标识别

**方法论描述**：识别4类无效数据指标

| 类型 | 特征 | 示例 |
|------|------|------|
| 无统计意义指标 | 样本占比特别小 | 1/2 vs 5/12的命中率对比 |
| 虚荣指标 | 绝对值好看但忽略成本 | 月收入10万但补贴6万 |
| 后见性指标 | 用于指出问题但无法预测 | 定义半年未打开APP为流失 |
| 错误解读 | 被人为处理的二手数据 | 粉丝翻倍但基数仅100 |

## 四、活动运营模块

### 4.1 策划完整活动的6大基本要素

**方法论描述**：活动策划前必须明确的6个约束条件

| 要素 | 说明 | 约束性质 |
|------|------|----------|
| 目的 | 拉新/收入/活跃/传播 | 约束条件 |
| 目标 | 量化的数字指标 | 约束条件 |
| 时间 | 活动时间周期 | 约束条件 |
| 资源 | 预算/人力/渠道资源 | 约束条件 |
| 策略 | 根据约束条件设计 | 被约束 |
| 活动形式 | 具体玩法设计 | 被约束 |

**关键区分**：
- 目的 = 方向（如拉新）
- 目标 = 量化数字（如新增1万用户）

### 4.2 活动执行与落地四步走

## 八、候选Skill详情

### Skill 1：活动策划6要素分析法
**模块**：活动运营
**输入**：活动背景信息（目的、资源、时间约束）
**输出**：完整的活动策划框架文档
**核心步骤**：
1. 明确目的（拉新/活跃/收入/传播）
2. 量化目标（具体数字指标）
3. 确认时间周期
4. 评估资源约束（预算/人力/渠道）
5. 制定策略（基于约束条件）
6. 设计活动形式

---

### Skill 2：推广营销五步法
**模块**：推广营销
**输入**：产品信息、目标用户画像、推广预算
**输出**：完整的推广方案
**核心步骤**：
1. 定义目标用户
2. 熟悉目标渠道规则
3. 搭建转化场景（用户痛点+解决方案）
4. 准备物料并确认排期
5. 监测数据并优化

---

### Skill 3：数据指标拆解器
**模块**：数据分析
**输入**：业务类型（网站/电商/内容/活动/渠道/SAAS）
**输出**：该业务类型的核心指标清单
**核心步骤**：
1. 判断业务类型
2. 匹配对应指标体系
3. 拆解关键指标影响因素
4. 建立数据监测方案

---

### Skill 4：SOP构建器
**模块**：工作管理
**输入**：需要标准化的工作内容
**输出**：完整的SOP文档
**核心步骤**：
1. 梳理工作流程步骤
2. 明确每步产出标准
3. 确定所需资源支持
4. 给出参考范例
5. 撰写完整SOP文档

---

### Skill 5：活动执行落地清单
**模块**：活动运营
**输入**：活动策划方案
**输出**：活动执行检查清单
**核心步骤**：
1. 与研发沟通确认需求
2. 评估可实现性和工作量
3. 确认资源排期和里程碑
4. 预热推广计划
5. 执行监控与优化调整

---

### Skill 6：活动风险评估工具
**模块**：活动运营
**输入**：活动方案
**输出**：风险评估报告+应对策略
**核心步骤**：
1. 识别热度持续风险 → 应对：周期性新内容/抽奖
2. 识别启动热度风险 → 应对：众筹预热/启动奖励
3. 识别薅羊毛风险 → 应对：限时限量限次
