# PMF Validator - 产品市场契合度验证器

## 快速开始（8-10 个关键问题）

### 产品想法
1. 产品想法是什么？用一句话说清楚
2. 解决什么问题？用户的什么痛点？
3. 目标用户是谁？他们有什么特征？

### 市场验证
4. 目标用户规模有多大？
5. 用户目前如何解决这个问题？
6. 有什么竞品？他们的优劣势？

### 产品阶段
7. 产品目前处于什么阶段？（想法/MVP/已有用户）
8. 有多少用户数据？（如有）

## 5 步验证流程

```
1. 目标用户圈定 → 用户画像
   ↓
2. 需求挖掘 → 需求清单
   ↓
3. 产品定位 → 价值主张
   ↓
4. MVP 设计 → 核心功能
   ↓
5. PMF 评估 → 验证结论
```

### 步骤 1：目标用户圈定

**方法**:
- 人口统计学细分（年龄/性别/地区/职业/收入）
- 行为细分（使用场景/频率/渠道）
- 需求细分（核心痛点/现有方案/不满意点）

**输出**: `references/templates/user-persona.md`

**验收标准**:
- [ ] 输出包含至少 3 个人口统计学特征
- [ ] 输出包含至少 2 个行为特征
- [ ] 用户画像模板完整

### 步骤 2：需求挖掘

**6 大渠道**:
1. 社交媒体观察
2. 竞品评论分析
3. 深度访谈（5-10 人）
4. 问卷调查（100+ 样本）
5. 行业报告
6. 搜索引擎趋势

**输出**: `references/templates/needs-assessment.md`

**验收标准**:
- [ ] 至少使用 3 个渠道挖掘需求
- [ ] 需求清单包含至少 10 条需求
- [ ] 需求有优先级排序（P0/P1/P2）

### 步骤 3：产品定位

**定位模板**:
```
对于 [目标用户]
他们面临 [核心问题]
我们的产品是 [产品类别]
它能提供 [核心价值]
与 [竞品] 不同
我们的差异化优势是 [独特优势]
因为 [信任理由]
```

**输出**: `references/templates/positioning-statement.md`

**验收标准**:
- [ ] 定位陈述完整填写
- [ ] 目标用户清晰
- [ ] 核心价值明确
- [ ] 差异化优势清晰

### 步骤 4：MVP 设计

**原则**:
- 单一核心功能
- 可验证假设
- 快速上线（2-4 周）
- 可衡量

**功能优先级**:
| 优先级 | 含义 | 判断标准 |
|--------|------|---------|
| P0 | 必须有 | 没有就无法验证假设 |
| P1 | 应该有 | 重要但不紧急 |
| P2 | 可以有 | 有了更好 |

**输出**: `references/templates/mvp-plan.md`

**验收标准**:
- [ ] MVP 功能 ≤3 个核心功能
- [ ] 每个功能有明确验证指标
- [ ] 有成功/失败判定标准

### 步骤 5：PMF 评估

**核心标准**:
- **40% 规则**: ≥40% 用户表示"缺了会非常遗憾"
- **D7 留存**: ≥40%
- **D30 留存**: ≥20%
- **NPS**: ≥30
- **K 因子**: ≥0.5

**评估方法**: 详见 `references/methods.md`

**输出**: `references/templates/pmf-report.md`

**验收标准**:
- [ ] 完成至少 100 个用户调研
- [ ] 40% 规则数据完整
- [ ] 至少 3 个核心指标达标
- [ ] 有明确结论（通过/失败/需改进）

## 技能协同

| 阶段 | 使用 Skill | 输出 |
|------|-----------|------|
| 产品想法验证 | pmf-validator | 是否值得投入 |
| 增长策略制定 | 增长策略师 | 如何获取用户 |
| 增长问题诊断 | 增长漏斗分析师 | 哪里出了问题 |

**使用前提**: 建议先完成 PMF 验证，确保产品已找到 PMF

## 输出要求

按 `references/output-standards.md` 输出报告，包含：

### 必需章节
1. 📊 执行摘要
2. 🎯 目标用户分析
3. 🔍 需求验证
4. 💡 产品定位
5. 🚀 MVP 规划
6. 📈 PMF 评估
7. ✅ 结论与建议

### 质量标准
- **完整性**: 覆盖 5 步验证流程所有输出 (目标：100%)
- **准确性**: 数据来源标注清晰 (目标：≥95%)
- **可执行性**: 结论明确可执行 (目标：是)

## 📋 Success Metrics

### 输出质量指标
- **完整性**: 覆盖 5 步验证流程所有输出 (目标：100%)
- **准确性**: 数据来源标注清晰 (目标：≥95%)
- **可执行性**: 结论明确可执行 (目标：是)

### 业务结果指标
- PMF 分数：≥40% (非常失望用户占比)
- 验证准确率：≥80% (后续追踪)
- 建议采纳率：≥70%
- 验证周期：<7 天

### 质量阈值
- 最低可接受质量分数：0.85
- 通过标准:
  - 完整性 ≥100%
  - 准确性 ≥95%
  - 可执行性 = 是
  - PMF 结论有明确依据
