# cyberwiz-Lessondesign - 体系化课程设计 Skill

## 概述

本 skill 基于两门顶级课程设计方法论（有看学社《如何制作一门口碑、收入双爆棚的好课程》与三节课《做课 - 内部课程设计体系课》），提供完整的课程开发双钻模型与工具体系，帮助用户快速制作出 80 分以上的优质课程。

## 核心方法论：课程生产双钻模型

```
        课程生产双钻模型
    ┌─────────────────────────────────────────┐
    │                                         │
    │  【左钻：做什么课？】  【右钻：怎么做好课？】 │
    │                                         │
    │  ┌──────────────┐  ┌──────────────┐      │
    │  │盘点知识资产   │  │ 课程框架设计  │      │
    │  │              │  │              │      │
    │  │设定教学目标   │  │ 学习体验设计  │      │
    │  └──────────────┘  └──────────────┘      │
    │         ↕                    ↕           │
    │    【课程选题/主题】 ←→ 【课程大纲】      │
    │                                         │
    └─────────────────────────────────────────┘
```

## 第一阶段：确定课程方向（左钻）

### 1.1 盘点知识资产

**目的**：找出讲师能讲且有价值的课程方向

#### 三层知识资产框架

| 层次 | 描述 | 课程类型 | 价值 |
|------|------|----------|------|
| **第一层** | 重要项目经历 | 参与过以负责人身份的全流程项目 | 影响力较小，但仍有价值 |
| **第二层** | 重复工作经验 | 掌握可依赖能力，每次都有稳定产出 | 价值大，效果可验证 |
| **第三层** | 决策判断直觉 | 形成自成一体的判断决策逻辑 | 价值最大，但开发难度最高 |

#### 盘点三步骤

**步骤 1：罗列优势经历**
- 工作经历：公司、岗位、时间、工作内容、成果
- 培训/授课经历：主题、时间、规模、效果、类型、次数
- 长篇作品：书籍、深度文章、专栏连载

**步骤 2：细分深挖**
- 合并重复出现的行为
- 拆细指向成功的关键行为
- 按工作步骤/跨周期/多角色维度展开

**步骤 3：评估优先级（三分制）**

| 维度 | 3分 | 2分 | 1分 | 0分 |
|------|------|------|------|------|
| **参与度** | 最高负责人 | 直接影响绩效 | 间接影响 | 无关 |
| **成熟度** | 多环境成功 | 同类环境多次使用 | 1-2次使用 | 仅1次 |
| **影响力** | 自发应用/纳入培训 | 非强制分享受欢迎 | 个别辅导 | 无影响 |

### 1.2 设定教学目标

**目的**：明确"给什么样的学员，解决什么问题，达到什么效果"

#### 4×4 教学目标矩阵

**纵轴：学员基础（起点）**

| 类型 | 描述 | 典型特征 |
|------|------|----------|
| **没概念** | 纯小白 | 从未接触该主题 |
| **有了解** | 入门者 | 看过资料/听过科普，无实践经验 |
| **有经验** | 实践者 | 亲自做过，有成败经验沉淀 |
| **很资深** | 专家级 | 反复实验，形成自有方法论 |

**横轴：交付成果（终点）**

| 层次 | 描述 | 典型产出 |
|------|------|----------|
| **基本知道** | 认知层面 | 了解概念/方法/工具，知道背后原理 |
| **深刻理解** | 理解层面 | 知晓应用场景，理解操作流程 |
| **简单应用** | 应用层面 | 能结合工作，按指导完成实际产出 |
| **熟练掌握** | 行为改变 | 反复训练后能灵活变通，带来行为改变 |

#### 课程定位决策矩阵

| | 没概念 | 有了解 | 有经验 | 很资深 |
| :--- | :--- | :--- | :--- | :--- |
| **基本知道** | 知识小课 | ❌ 不建议 | ❌ 不适合 | ❌ 不适合 |
| **深刻理解** | 知识小课 | 知识小课 | 知识小课 | ❌ 不适合 |
| **简单应用** | 体系课 | 体系课 | 知识小课 | ❌ 不适合 |
| **熟练掌握** | 体系课 | 体系课 | 体系课 | ❌ 不适合 |

- **知识小课**：30分钟-2小时，解决具体工作问题
- **体系课**：5小时以上，提升岗位能力或完整方法论

### 1.3 课程选题与定位

#### 四象限市场分析

```
    需求个性化程度
         高                低
    ┌─────────┬─────────┐
    │         │         │
  大 │   一     │   二     │
    │  (细分)  │(红海)   │
用 ├─────────┼─────────┤
户 │         │         │
规 │   四     │   三     │
模 │  (放弃)  │(垄断)   │
    │         │         │
    └─────────┴─────────┘
```

- **第一象限**（规模大/个性化高）：细分人群+需求切入，寻找产品独特性
- **第二象限**（规模大/个性化低）：红海市场，需差异化（新渠道/新模式/新师资）
- **第三象限**（规模小/个性化低）：做成细分领域头部产品
- **第四象限**（规模小/个性化高）：不建议做课

#### 两种售卖逻辑

**1. 爆款畅销逻辑**
- 充分条件：新市场/新机会、老市场差异化定位、话题/老师自带势能
- 必要条件：强势卖点+集中引流+短期放量

**2. 稳定长销逻辑**
- 充分条件：稳定价值预期、人群基数大+方法论稳定、细分领域头部
- 必要条件：稳定口碑+效果兑现

#### 五个核心定位问题

1. **面向什么用户？** → 用户画像、市场规模
2. **选择什么话题？** → 话题方向、细分角度
3. **什么课程类型？** → 知识付费/轻训练营/教育交付/线下培训
4. **讲师是谁？**** → 资源+策略各半
5. **什么渠道售卖？** → 渠道选择匹配用户聚集地

## 第二阶段：课程设计与制作（右钻）

### 2.1 课程大纲设计

#### 课程五大要素

1. **起点与终点**：学员初始状态 → 课程目标
2. **时间/时长**：学习周期设计
3. **内容**：知识体系化组织
4. **服务**：教学支持与互动设计
5. **阶段性交付节点与交付物**：里程碑+具体产出

#### 两条核心曲线

**学习体验曲线**：学员在收获感/成就感/喜悦感上的起伏变化
- 开篇不能差（首因效应）
- 结尾要好（收尾口碑）
- 中间有起伏（对比产生体验）
- 不能连续低位

**学习难度曲线**：每章学习难易程度的分布

两种典型模式：
- **高山仰止型**：开篇提出高价值问题，认清现实后再带路径
- **进阶爬坡型**：由浅入深、由易到难

#### 内容结构组织方式

| 类型 | 特点 | 适用场景 |
|------|------|----------|
| **并列结构** | 模块独立，无严格顺序 | 知识付费类课程 |
| **层级结构** | 逐层深入，底层支撑上层 | 体系化方法论 |
| **流程化结构** | 按时间/步骤展开 | 工作流程/操作指南 |
| **关联交叉结构** | 多维度交织 | 复杂系统讲解 |

---

### 2.2 三种课程设计逻辑

#### 逻辑一：内容消费逻辑
**适用**：知识付费类课程
**核心关注**：消费体验+启发感获得感
**三步骤**：
1. 收集核心素材（观点/经历/案例/金句/测评）
2. 选择内容结构（故事启发/悬念层层/多单点留白）
3. 强化消费顺畅度（消除障碍+增强体验）

#### 逻辑二：学习交互逻辑
**适用**：教育/结果交付型课程
**核心关注**：学习目标达成
**三类学习目标**：
- **基本知晓**：知识结构清晰+配套测试
- **深刻理解**：解释现象+案例训练+分析练习
- **应用类**：能力构成分析+因果关系+工具方法+训练

#### 逻辑三：行动动机改变逻辑
**适用**：引流转化课程
**核心关注**：激发用户行动
**认知失衡理论**：
```
原认知 A → 遇到现象 B → 产生冲突 → 寻求新认知 C → 行为改变
```
**三性构建**：
- **必要性**：为什么要学？（制造认知失衡）
- **唯一性**：为什么是你？（独特价值主张）
- **紧迫性**：为什么现在？（限时限量/稀缺感）

### 2.3 学习体验设计

#### 线上学习四道关卡

| 关卡 | 挑战 | 解决策略 |
|------|------|----------|
| **开始学** | 报名后不开始 | 适度定价+降低行动门槛 |
| **学得完** | 中途流失 | 内容密度+学习节奏设计 |
| **学得懂** | 听完不理解 | 逻辑严谨+脉络清晰 |
| **会用** | 无法应用 | 工具落地+练习巩固 |

#### 好课程五特征

1. **脉络清晰**：章节主线明确，流程顺序相似
2. **逻辑严谨**：章节设计逻辑一致，避免跳跃感
3. **有真实案例**：工作场景带入，有代入感
4. **工具可落地**：明确可用的工具和方法
5. **学与练结合**：配套练习和巩固

## 使用指南

### 场景一：从零开始做课

1. 使用"知识资产盘点表"梳理个人经验
2. 用"4×4矩阵"确定课程定位
3. 通过"五问"完成选题定位
4. 设计课程大纲（五大要素）
5. 选择合适的主设计逻辑
6. 优化学习体验（四关卡）

### 场景二：优化现有课程

1. 用"五大要素"诊断现有结构
2. 分析"两条曲线"是否合理
3. 检查是否符合主设计逻辑
4. 优化学习体验设计
5. 补充缺失的交付物

### 场景三：评估课程质量

**用户视角**：学有所获+体验良好
**专业视角**：目标达成+稳定交付

## 注意事项

1. **课程设计不是做PPT**：PPT 只是最终呈现，占工作量约 20%
2. **线上课 ≠ 开着电脑讲课**：线上课要求更高（干货密度、逻辑严谨）
3. **先做小课，再做体系课**：小课（45分钟-2小时）可快速验证
4. **聚焦比全面更重要**：一门课解决一类人的一个问题
5. **教学目标是北极星**：所有内容围绕教学目标展开
