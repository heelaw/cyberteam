# cyberwiz-activity-3.0

## 概述

活动运营是最具爆发力的运营手段，也是一种说服和引导用户发生某种行为的极好手段。本Skill提供完整的活动运营工作方法论，从明确目标到活动复盘的全流程指导。

活动运营与常规活动策划的本质区别：
- **活动运营**：围绕产品或特定数据指标（销售额、活跃用户等）展开，活动是手段，目的是拉升运营指标
- **活动策划**：指向性不明确，只要好玩、大家愿意参与即可

**违反这些规则将导致工作流混乱和返工**

- 默认模式：完成当前 Stage 后，**停止并等待用户明确指令**

- ❌ 禁止在非自动执行模式下批量执行多个 Stage

**只有以下情况才算批准：**
- ✅ "继续" / "下一步" / "开始 Stage X"
- ✅ 使用快捷命令

**以下不算批准：**
- ❌ "好" / "不错" / "可以"（只是满意，不是批准）
- ❌ "知道了" / "看到了"（只是确认，不是批准）
- ❌ 用户沉默

### Stage 1：目标与发力点分析

**目标**：明确活动目标，分析产品生命周期和发力点

**执行步骤**：

### 1.1 从数据入手明确运营目标

当目标不明确时，通过三个维度分析：

**维度1：对照产品生命周期进行数据监测**

| 生命周期 | 特征 | 运营目标和手段 |
|---------|------|--------------|
| 探索期 | 上线时间不长，产品还在打磨，需求尚待验证 | 验证需求，积累种子用户 |
| 快速增长期 | 需求已验证，竞争对手大量出现 | 快速占领市场 |
| 成熟稳定期 | 市场接近饱和，增长空间小 | 维护用户，提升转化 |
| 衰退期 | 替代产品出现，用户批量流失 | 延缓衰退或转型 |

**维度2：找到潜在问题数据**

监测指标：
- 用户增速、用户活跃变化
- 市场用户规模/现有用户比
- 销售额的品类变化、新老用户构成
- 老用户消费频次和消费金额变化

**维度3：分析核心业务流程**

用户流程：访问→注册/登录→查找→报名→付费→使用→评价→END
老师流程：相应的流程也需要分析

### 1.2 确定发力点

围绕目标分析可能的发力点：
- 流量？
- 付费转化率？
- 老用户复购？
- 新用户激活？
- ARPU值提升？
- 某品类重点突破？

**案例**：某在线学习网站，5万注册用户，月销售额10万，要求2个月内用5万预算将销售额提升到50万。

**分析结论**：
- 面向站内老用户为主+站外新用户为辅
- 重点激活此前注册但未购买的站内用户
- 借助老用户带动活动传播

### 1.3 活动类型与数据漏斗匹配

根据活动类型和渠道，选择对应的数据漏斗模型和转化率基准：

**投放类活动 → 投放转化漏斗**

```
广告展示 → 点击 → 落地页浏览 → 注册/支付 → 后续行动 → 完成转化
```

**典型基准**：
- CTR（点击率）：1-3%
- 落地页转化率：10-30%
- 注册转化率：30-50%
- 支付转化率：5-20%

**关键指标**：CTR、落地页转化率、支付转化率、ROI

**优化方向**：
- CTR低 → 优化素材、文案、标题
- 落地页转化低 → 简化流程、优化视觉、强化价值主张
- 支付转化低 → 限时优惠、信任背书、简化支付流程

---

**社群类活动 → 社群转化漏斗**

```
前端流量 → 入群 → 互动 → 筛选高意向用户 → 关键体验 → 购买
```

**典型基准**：
- 入群率：20-50%
- 互动率：30-60%
- 筛选转化率：40-70%
- 购买转化率：10-30%

**关键指标**：入群率、互动率、筛选转化率、购买转化率

**优化方向**：
- 入群率低 → 优化入群诱饵、降低入群门槛
- 互动率低 → 设计互动机制、活跃话题、红包激励
- 购买转化低 → 建立信任、分层运营、限时优惠

---

**APP内活动 → APP留存漏斗**

```
APP访问 → 注册 → 关键行为 → 重复使用 → 长期留存
```

**典型基准**：
- 注册转化率：50-80%
- 关键行为完成率：40-70%
- 次日留存率：30-50%
- 7日留存率：15-30%

**关键指标**：注册转化率、关键行为完成率、留存率

**优化方向**：
- 注册转化低 → 简化注册流程、减少必填项
- 关键行为完成低 → 优化新手引导、设计"顿悟时刻"
- 留存低 → 设计留存激励、推送提醒、内容更新

---

**内容类活动 → 内容完读与分享漏斗**

```
内容展示 → 点击打开 → 阅读10% → 阅读50% → 阅读100% → 分享/转发
```

**典型基准**：
- 打开率：20-50%
- 完读率：30-60%
- 分享率：5-15%

**关键指标**：打开率、完读率、分享率

**优化方向**：
- 打开率低 → 优化标题、封面、发布时间
- 完读率低 → 优化开头、内容结构、减少篇幅
- 分享率低 → 设计分享诱因、社交货币、情感共鸣

### Stage 2：活动玩法设计

**目标**：设计活动主线玩法及形式

**执行步骤**：

### 2.1 活动玩法设计的九大秘诀

成功的活动在玩法设计上往往遵循以下九大秘诀，这些要素能显著提升用户参与意愿：

| 秘诀 | 核心思路 | 应用示例 |
|-----|---------|---------|
| 1. 物质激励与概率事件 | 提供符合用户属性的奖励，结合概率性激励 | 豆瓣用户送电影原声大碟，抽奖活动 |
| 2. 稀缺感 | 制造资源有限的紧迫感 | 限量版商品、限时折扣、名额有限 |
| 3. 竞争与攀比 | 引入排行榜、PK机制 | 微信运动排行榜、学习积分排名 |
| 4. 炫耀与猎奇 | 设计值得分享和探索的内容 | "1秒入职Facebook"H5、新奇玩法 |
| 5. 营造强烈情绪 | 触发用户的情感共鸣 | 国足出线纪念活动、热点事件营销 |
| 6. 尊崇感与重视感 | 让用户感到被重视 | 个性化邀请函、专属特权、VIP待遇 |
| 7. 超直感 | 降低用户理解和参与成本 | 清晰的活动规则、直观的奖励机制 |
| 8. 低成本与单线程 | 简化参与流程，减少用户投入 | 一键参与、单步操作、低门槛 |
| 9. 趣味性反差 | 创造意想不到的趣味体验 | 反差萌设计、趣味互动游戏 |

### 2.2 常见活动形式的核心支撑点

- 有奖励：提供明确的利益回报
- 有好处：让用户感到划算
- 有价值：提供实用或有意义的内容

### 2.3 外在形式

- 节日/纪念日折扣、大促
- 找理由送礼（回馈老用户、抽奖、满足条件送）
- 比赛、评选、竞争
- 趣味互动（游戏、猜谜、找茬等）

### 2.4 主线玩法设计案例

以在线学习网站为例：

| 问题 | 对应玩法 | 运用秘诀 |
|-----|---------|---------|
| 吸引用户访问 | 活动期间登录网站，每天抽奖获得代金券 | 物质激励、概率事件 |
| 吸引用户下单 | 全场8折，新用户首次下单直减100元 | 稀缺感、超直感 |
| 借助老用户传播 | 两人同行再享9折，5人团购再享8.5折 | 低成本、趣味性 |

### Stage 3：支线玩法与机制设计

**执行步骤**：

### 3.1 支线玩法的意义

解决主线玩法之外的潜在隐患点：
- 活动热了几天不热了→持续关注机制
- 活动前期关注参与的人不够→预热机制
- 订单集中在某几天或羊群效应→刺激早期订单

### 3.2 支线玩法案例

| 主线玩法 | 隐患问题 | 支线玩法 |
|---------|---------|---------|
| 每日抽奖 | 活动几天后不热了 | 每周邀请知名老师线上直播分享 |
| 全场8折 | 前期关注不够 | 事前"寻找学习控"众筹，满1000人成功享6折 |
| 团购折扣 | 订单集中或观望 | 每日前10位+含"33"序号送礼品 |

### Stage 6：数据监测与优化方案

**目标**：制定活动执行、数据监测和优化策略

**执行步骤**：

### 6.1 活动价值放大

1. **捕捉亮点**：活动过程中的亮点、趣事、进展、数据、UGC内容
2. **二次传播**：整合内容通过渠道媒体对外传播
3. **借势大V**：联系名人、品牌参与或互动，制造话题

**案例**：简书征文活动持续更新征文稿件

### 6.2 数据监测和优化

**关键数据梳理**（以H5小游戏为例）：
- 生成页UV
- 生成名片数
- 分享次数
- 公众号关注量
- 二维码扫描次数
- 活动结束后3天取关用户数

**转化率计算**：
- 名片生成转化率 = 生成名片数 / 生成页UV
- 微信关注转化率 = 关注量 / 扫描次数
- 3日取关率 = 取关数 / 关注量

### 6.3 数据漏斗分析与优化

**分析步骤**：

**步骤1：绘制数据漏斗**

根据活动类型选择对应的漏斗模型，填充各环节数据：

| 环节 | 数据 | 环节转化率 | 累计转化率 | 状态 |
|-----|------|-----------|-----------|------|
| 环节1 | - | - | - | - |
| 环节2 | - | - | - | - |
| 环节3 | - | - | - | - |

- **环节转化率** = 本环节数据 / 上一环节数据 × 100%
- **累计转化率** = 本环节数据 / 初始数据 × 100%

**步骤2：识别瓶颈环节**

找出流失率最高的环节（通常转化率 < 基准值的50% 需要重点关注）

**流失率分级**：
- 🔴 严重流失：转化率 < 基准30%
- 🟡 中等流失：转化率 = 基准30%-60%
- 🟢 正常范围：转化率 > 基准60%

**步骤3：分析流失原因**

针对瓶颈环节，分析可能原因：

| 环节 | 流失率 | 可能原因 | 验证方法 |
|-----|--------|---------|---------|
| 点击 | 90%+ | 素材不吸引、定位不准、创意疲劳 | A/B测试素材、查看历史数据 |
| 落地页 | 70%+ | 加载慢、不相关、价值不清晰 | 热力图、用户调研 |
| 注册 | 60%+ | 表单太长、隐私担忧、门槛高 | 漏斗细分、A/B测试表单 |
| 支付 | 80%+ | 价格太高、支付复杂、信任不足 | 支付漏斗分析、用户访谈 |

**步骤4：优化方法库**

根据瓶颈环节，选择对应优化方法：

**曝光量低**：
- 增加预算、拓展渠道
- 优化出价、提升质量分
- 测试新时段、新人群定向

**点击率低**：
- 优化广告素材（图片、视频、文案）
- 使用流量密码（反差、情绪、热点）
- A/B测试不同创意方向
- 优化标题和描述

**落地页转化低**：
- 简化落地页流程，减少干扰
- 优化首屏价值主张
- 强化社会认同（用户数、评价）
- 添加紧迫感（限时、限量）
- 优化CTA按钮位置、文案、颜色

**注册转化低**：
- 减少表单字段
- 提供社交登录选项
- 添加隐私保护说明
- 设置渐进式注册

**支付转化低**：
- 限时优惠、折扣券
- 分期付款、免息
- 强化信任背书（担保、评价）
- 简化支付流程
- 挽留弹窗（离开时提醒）

**步骤5：A/B测试与迭代**

**测试框架**：
1. 提出假设：例如"简化注册表单可提升转化率20%"
2. 设计变量：控制组 vs 实验组
3. 确定样本量：每组至少1000个样本
4. 运行测试：同时运行，避免时间偏差
5. 统计显著性：p值 < 0.05
6. 应用结论：获胜方案全量上线

**测试优先级**：
- 高流量 × 低转化 = 优先测试
- 低流量 × 高转化 = 次优先
- 低流量 × 低转化 = 最后考虑

**步骤6：持续优化循环**

```
数据监测 → 识别瓶颈 → 分析原因 → 设计方案 → A/B测试 → 应用结论 → 再次监测
```

## 活动实例库

### 线上活动实例

#### 实例 1：电商平台双十一促销活动
- **活动目标**：提升销售额，增加用户活跃度
- **核心玩法**：限时折扣、满减优惠、秒杀活动
- **支线玩法**：预售活动、优惠券领取、分享返利
- **预热推广**：提前1个月开始预热，通过社交媒体、短信、邮件等渠道推广
- **数据监测**：实时监测流量、转化率、销售额等指标
- **活动效果**：销售额提升500%，用户活跃度提升300%

#### 实例 2：微信公众号吸粉活动
- **活动目标**：增加公众号粉丝数
- **核心玩法**：关注公众号参与抽奖活动
- **支线玩法**：分享活动获得额外抽奖机会
- **预热推广**：通过朋友圈、微信群、其他公众号合作推广
- **数据监测**：监测关注量、分享量、抽奖参与量等指标
- **活动效果**：新增粉丝10万+，分享率达到30%

### 线下活动实例

#### 实例 1：线下产品发布会
- **活动目标**：发布新产品，提升品牌知名度
- **核心玩法**：产品展示、嘉宾演讲、现场体验
- **支线玩法**：互动游戏、抽奖活动、媒体专访
- **预热推广**：通过社交媒体、媒体合作、邀请函等渠道推广
- **执行SOP**：详细的活动流程、人员分工、应急方案
- **活动效果**：吸引媒体报道20+家，现场参与人数500+

#### 实例 2：线下社群聚会活动
- **活动目标**：增强社群凝聚力，促进用户交流
- **核心玩法**：主题分享、互动游戏、自由交流
- **支线玩法**：签到送礼、拍照打卡、后续线上分享
- **预热推广**：社群内通知、邀请函、报名表单
- **执行SOP**：场地布置、物料准备、人员安排
- **活动效果**：参与率达到80%，用户满意度达到95%
