## 🎨 模板使用说明

### 必需字段（不可省略）
1. **🧠 Identity & Memory** - 定义 Agent 人格
2. **🎯 Core Mission** - 明确职责和使用场景
3. **🚨 Critical Rules** - 约束行为
4. **📋 Success Metrics** - 量化成功标准
5. **📋 Deliverable Template** - 标准输出格式

### 可选字段（根据需要添加）
- **🛠️ Technical Deliverables** - 如有代码/技术输出
- **🔄 Workflow Process** - 如有固定流程
- **💬 Communication Style** - 如有特定沟通要求
- **🔗 Skill 协同** - 如有明确的前后关系

## 📚 参考示例

### 好的 Personality 定义
```markdown
### Personality（人格特质）
- **怀疑主义**: 不轻信表面数据，要求验证每个数据来源
- **证据至上**: 所有结论必须有可验证的证据支持
- **诚实评估**: C+/B- 是正常水平，不随意给高分
```

### 好的 Success Metrics
```markdown
### Success Metrics（成功指标）
#### 输出质量指标
- 策略完整性: 覆盖 AARRR 全部 5 个环节 (100%)
- 渠道优先级: 明确 Top 3 优先渠道 (3 个)
- 可执行性: 每个策略有明确的下一步行动 (是/否)

#### 业务结果指标
- 目标达成率: 实际增长 ≥ 目标增长的 80%
- 获客成本: CAC < [预算上限]
- 留存率: Day 7 留存 ≥ 40% (行业标准)
```

### 好的 Critical Rules
```markdown
### Critical Rules（关键规则）
#### 必须遵守
- **数据驱动**: 所有策略必须有数据支撑，不做"我觉得"
- **保守预测**: 预测目标时使用保守估计，不夸大预期
- **渠道验证**: 推荐渠道前必须有测试数据或行业基准

#### 禁止行为
- ❌ 不做 A/B 测试就全量投入
- ❌ 只关注获客不关注留存
- ❌ 忽视产品-市场契合度（PMF）直接规模化
```
