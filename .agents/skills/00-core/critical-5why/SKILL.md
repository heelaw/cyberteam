thinking_model:
  name: "5Why分析法"
  name_en: "5 Whys Analysis"
  origin: "丰田喜一郎 | 丰田生产方式"
  core_principle: "通过连续5次追问'为什么'，深入挖掘问题的根本原因"

  key_concepts:
    - concept: "连续追问"
      definition: "每次对答案继续问为什么"
      example: "机器坏了 → 为什么？→ 保险丝断了 → 为什么？..."

    - concept: "根本原因"
      definition: "导致问题的最深层、最本质的原因"
      example: "不是'员工失误'，而是'培训体系缺失'"

    - concept: "事实基础"
      definition: "每次追问都要基于事实，不能主观推测"
      example: "用数据说话，而不是'我觉得'"

  best_for:
    - "需要找到根本原因时"
    - "问题反复出现时"
    - "需要系统性分析时"
    - "质量问题分析"

  not_for:
    - "缺乏数据支持"
    - "原因已经很明确"
    - "纯创意性问题"

tags: ["原因分析", "根本原因", "深度挖掘"]
estimated_time: "3-5分钟"

# ========================================
# 分析流程
# ========================================
analysis_workflow:
  step_1_define_problem:
    name: "明确定义问题"
    actions:
      - "清晰描述当前问题"
      - "量化问题表现"
      - "确认问题边界"

    output: "清晰的问题陈述"

  step_2_ask_why_iteratively:
    name: "连续5次追问"
    actions:
      - "第1次为什么：直接原因"
      - "第2次为什么：更深一层"
      - "第3次为什么：系统原因"
      - "第4次为什么：根本原因"
      - "第5次为什么：验证根本性"

    output: "5层原因链"

  step_3_validate_root_cause:
    name: "验证根本原因"
    actions:
      - "反向推导：如果解决根本原因，问题会消失吗？"
      - "检查每个环节的逻辑性"
      - "确认有事实支撑"

    output: "验证后的根本原因"

  {problem_description}

  {previous_analysis}

  确保问题清晰具体。如果模糊，先澄清。

  ### 步骤2: 连续5次追问
  对每个答案继续问"为什么"，直到第5层。

  **重要原则**:

  - ✅ 追问直到找到可控的根本原因

  ### 步骤3: 验证根本原因
  反向验证：如果解决这个根本原因，问题会消失吗？
