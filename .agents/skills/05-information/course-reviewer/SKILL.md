# Course Reviewer - 三节课课程内容校对助手

## Overview

专门用于对比和迭代三节课课程内容的智能助手，支持逐字稿与内容增强文稿的查漏补缺，包含图片知识点的提取与ASCII化展示。

## 使用场景

- 使用 sanjieke skill 重新下载课程后，与旧版本对比查漏补缺
- 将 Obsidian 格式的旧课程内容（含 images 文件夹）迁移到新的 sanjieke 格式
- 提取旧版本中的知识框架图、PPT截图，转换为 ASCII 展示并插入新版本

## 工作流程

### 步骤 1：路径确认（每次启动必执行）

启动时必须先询问用户两个路径：

```
1. 新课程路径（sanjieke skill 提取的内容）
   示例：/Users/cyberrwiz/Desktop/黄有璨系列课程/03-AI时代下的运营基石课

2. 旧课程路径（Obsidian 格式，包含 images 文件夹）
   示例：/Users/cyberrwiz/Library/Mobile Documents/iCloud~md~obsidian/Documents/02 领域（Area)/02_运营知识/010_S11 【运营AI+】
```

### 步骤 2：目录结构对比

- 扫描并对比两个目录的文件结构
- 建立章节映射关系
- 输出目录对比概览

### 步骤 3：逐章节处理

对每个章节执行以下操作：

1. **读取新版本内容**：逐字稿和内容增强文档
2. **读取旧版本内容**：对应的 Markdown 和 images 文件夹
3. **对比内容差异**：识别缺失的知识点、案例、框架
4. **处理图片**（旧版本独有）：
   - 使用 `mcp__minimax__understand_image` 或 `mcp__zai-mcp-server__analyze_image` 识别图片
   - 将识别内容转换为 ASCII 展示
   - 插入到文档对应知识点位置
5. **补充内容**：将旧版本的完善内容合并到新版本
6. **同步更新**：更新"内容增强"文件夹下的相关文稿

### 步骤 4：质量检查

每完成一章后：
- 交叉验证内容完整性
- 确认内容优于更改前
- 输出章节完成报告

### 步骤 5：完成与清理

- 生成差异报告和变更日志
- 向用户确认迭代完成
- 确认后删除旧版本目录

## 核心规则

1. **路径优先**：每次启动必须先确认新课程路径和旧课程路径
2. **按序处理**：按章节顺序处理，不跳过任何章节
3. **精准补充**：只在旧版本内容比新版本更完善时进行补充
4. **图片优先级**：知识框架图 > PPT截图 > 案例配图
5. **结构保持**：保持新版本的原始结构和命名规范
6. **安全删除**：删除旧版本前必须确认迭代完成无误

## MCP 工具使用

| 工具 | 用途 |
|------|------|
| `mcp__minimax__understand_image` | 识别课程图片（PPT、知识框架图） |
| `mcp__zai-mcp-server__analyze_image` | 备用图片识别工具 |

## 文件结构理解

### 新版本（sanjieke 格式）

```
课程名称/
├── 逐字稿/           # 原始抓取内容
│   └── XX-小节名称.md
├── 内容增强/         # 合并增强后的内容（按大章节组织）
│   ├── 00-课程索引.md
│   └── XX-章节名称.md
└── 课程大纲/
    └── 00-课程大纲.md
```

### 旧版本（Obsidian 格式）

```
课程名称/
├── images/          # 重要：包含知识框架图、PPT截图
│   └── *.png/*.jpg
└── *.md             # 章节文档
```

## 输出要求

1. **逐字稿完善**：对新下载内容的逐字稿进行补充和交叉对比完善
2. **内容增强文稿同步**：在完善逐字稿的同时，针对"内容增强"文件夹下的相关文稿同步进行查漏补缺
3. **差异报告**：列出发现的问题和已补充的内容
4. **变更日志**：记录从旧版本迁移的内容

## 质量标准

- 知识框架完整
- 案例保留完整
- 核心内容不遗漏
- ASCII 图表清晰
- 文档结构规范
