# 课程重编大师 - Course Merge Architect

## 概述

"课程重编大师"是一个智能课程重组工具，能够将多个课程内容打散后重组为体系化的"大课程"。核心能力包括：

- **多课程分析**：扫描多个课程目录，识别各自的结构和知识点
- **内容合并去重**：智能识别重合内容进行合并，互补内容进行扩展
- **体系化重组**：按逻辑深度重新排序，由浅入深构建学习路径
- **内容自动增强**：生成 ASCII 图表、练习题、章节导航
- **高度自动化**：自动生成待办清单，执行过程中无需频繁确认

## 快速开始

### 基础用法

```
/course-merge 重组课程 "我的运营知识库"
```

### 指定源目录

```
/course-merge 重组课程 "产品经理体系课" \
  --source ~/Documents/三节课课程/产品
```

### 多课程合并

```
/course-merge 重组课程 "运营全体系" \
  --source1 ~/Documents/课程A \
  --source2 ~/Documents/课程B \
  --source3 ~/Documents/课程C
```

## 工作流程

### 阶段 0：准备与分析（生成待办清单）

#### 0.1 扫描所有课程目录
- 识别课程数量与结构
- 读取所有课程大纲
- 识别课程的逐字稿和内容增强文件夹

#### 0.2 提取所有知识点
- 从大纲和逐字稿提取知识点
- 建立知识点清单
- 标记每个知识点的来源课程

#### 0.3 分析知识点关系
- 识别重合内容（相似度 >80%）
- 识别互补内容（相关但内容不同）
- 建立知识依赖图

#### 0.4 设计新课程结构
- 按逻辑深度排序：基础概念 → 方法论 → 实战应用
- 构建统一的章节层次
- 生成新大纲

#### 0.5 生成待办清单
输出格式：
```

  课程体系化重组 - 待办清单
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📊 分析结果
  ✅ 扫描到 N 个课程
  ✅ 提取到 M 个知识点
  ✅ 识别到 K 个重合点
  ✅ 规划为 L 个大章节

📋 章节清单
  第 1 章：章节名称（来源：课程A）
  第 2 章：章节名称（来源：课程A+B）
  ...

📤 输出计划

```

### 阶段 1-N：内容处理与合并（自动化执行）

对每个章节执行以下步骤：

#### 步骤 1：读取逐字稿
- 仔细阅读完整逐字稿内容
- 提取讲师观点、经验、理论体系
- 记录关键案例和方法

#### 步骤 2：与内容增强对比验证
- 确认逐字稿内容完整
- 用增强内容检查遗漏
- 补充增强版本中的额外内容

#### 步骤 3：合并与重组
- 重合内容：合并为单条，保留最完整表述
- 互补内容：作为补充扩展，标注来源课程
- 逻辑调整：确保内容流畅衔接

#### 步骤 4：内容增强
- 添加 ASCII 图表（流程图、表格、体系图）
- 添加章节导航
- 生成练习题
- 添加章节总结

#### 步骤 5：输出章节文件
- 写入独立 MD 文件
- 更新进度报告

### 阶段 N+1：输出生成

#### N+1.1 输出课程框架
- `00-课程大纲.md`：完整目录结构（大章+小节）
- `00-学习路径.md`：建议学习顺序
- `00-知识索引.md`：知识点索引

#### N+1.2 输出完整整合文档
- `99-完整整合.md`：所有内容合并为完整 MD

## 内容处理规则

### 重合内容判断

```python
IF 知识点标题相似度 > 80% AND 核心内容相似度 > 70%:
    THEN 重合 → 合并为单条
ELSE IF 知识点相关但内容不同:
    THEN 互补 → 补充扩展，标注来源
```

### 内容保留优先级

1. **逐字稿内容**（最高优先级）- 最完整的原始内容
2. **内容增强内容** - 验证遗漏，补充额外内容
3. **课程大纲内容** - 结构参考

### 内容合并原则

| 原则 | 说明 |
|------|------|
| **逻辑合理** | 整个课程由浅入深，构建学习路径 |
| **内容增强** | 在保证原有内容的基础上，提升深度与广度 |
| **完整呈现** | 讲师的观点、经验、理论体系都要完整保留 |
| **可视化表达** | 流程化、表格化内容用 ASCII 图表展示 |

### 可视化触发规则

| 模式 | 触发条件 | 输出格式 |
|------|----------|----------|
| **流程图** | 识别"步骤"、"流程"、"阶段"、"先后"等关键词 | ASCII 流程图 |
| **表格** | 识别"对比"、"分类"、"类型"、"区别"等关键词 | ASCII 表格 |
| **框架图** | 识别"框架"、"模型"、"体系"、"架构"等关键词 | ASCII 框架图 |

### 练习题生成规则

根据章节知识点自动生成三类题目：
- **基础题**（3-5道）：考察基本概念和知识点
- **进阶题**（2-3道）：考察应用和分析能力
- **实战题**（1-2道）：考察综合应用能力

## 核心原则

1. **逐字稿优先**：逐字稿内容是最完整的，必须仔细阅读
2. **增强验证**：用"内容增强"文件夹内容验证逐字稿的完整性
3. **不删除原文件**：所有操作都在新目录中进行，保留原始文件
4. **完整呈现**：不能省略或忽略任何知识点，即使内容冗长
5. **逻辑合理**：重组后的课程必须由浅入深，逻辑清晰
6. **可视化表达**：多用表格和图示，方便读者理解
7. **高度自动化**：执行过程中尽量减少用户确认

---

## 注意事项

- **重要**：原有抓取的内容有重合，重合的部分要合并，互补的要补充扩展
- **重要**：逐字稿的内容是最完整的，必须仔细阅读；"内容增强"文件夹里的内容是脱水过滤过的
- **重要**：无论知识点有多冗长，都要认认真真完成每一节，不能省略，也不能忽略
- **重要**：操作完成后不要删除原有文件，只需创建新的文件夹
