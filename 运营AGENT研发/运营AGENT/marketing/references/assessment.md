# 自检标准

## 用户洞察

- 用户是否具体到人群，而不是笼统说“年轻人”。
- 场景是否可还原，而不是抽象概念。
- 需求是否分层，而不是只写表层功能。
- 核心动机是否能被验证。

## 渠道选择

- 是否区分了渠道类型。
- 是否说明了预算、时间、可控性之间的取舍。
- 是否给出了试跑顺序。
- 是否说明了为什么不用其他渠道。

## 转化设计

- 是否包含必要性、唯一性、紧迫性、信任度四要素。
- 是否避免虚假紧迫感。
- 是否有明确 CTA。
- 是否能落到页面或活动结构。

## 漏斗诊断

- 是否先判断漏斗类型。
- 是否定位到具体阶段。
- 是否给出可验证假设。
- 是否能区分现象和原因。

## 执行落地

- 是否有清单、排期、负责人、风险和验收标准。
- 是否留出监控和纠偏动作。
- 是否能在执行中继续追踪。
