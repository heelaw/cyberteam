# 局部模型优化手册

## 概述

局部模型需要持续优化才能保持竞争力。本手册提供系统性的优化方法。

---

## 优化时机判断

### 需要优化信号

| 信号类型 | 具体表现 | 紧急程度 |
|----------|----------|----------|
| 指标异常 | 核心指标连续下降 | P0 |
| 效率下降 | 投入产出比降低 | P1 |
| 竞品超越 | 相对市场份额下降 | P1 |
| 模式陈旧 | 长期未更新策略 | P2 |
| 资源变化 | 投入资源发生大变化 | P2 |

### 优化时机选择

```
常规优化：每季度一次全面审视
专项优化：问题发生时及时启动
周期复盘：月度指标回顾+调整
```

---

## 优化流程

### Step 1：问题定义

```
优化前问自己：
1. 核心问题是什么？（不是现象）
2. 问题的影响范围有多大？
3. 解决问题的优先级是什么？

问题定义模板：
当前 [局部模型] 的 [指标] 处于 [状态]，
目标是将 [指标] 提升/降低到 [目标值]，
原因是 [根因分析]。
```

### Step 2：根因分析

```
分析工具：5Why分析法

示例：
问题：会员续费率下降
Why1：为什么？→ 用户认为续费不值
Why2：为什么？→ 会员权益感知弱
Why3：为什么？→ 权益展示不直观
Why4：为什么？→ 页面设计问题
Why5：为什么？→ 缺乏A/B测试

根因：会员权益展示页面需要优化
```

### Step 3：方案设计

```
方案设计原则：
1. 先验证再推广：小范围A/B测试
2. 考虑副作用：评估对其他局部的影响
3. 可逆优先：优先选择可快速回滚的方案

方案评估矩阵：
| 方案 | 预期效果 | 实施成本 | 风险 | 推荐 |
|------|----------|----------|------|------|
|      |          |          |      |      |
|      |          |          |      |      |
```

### Step 4：执行落地

```
执行检查清单：
[ ] 方案设计完成
[ ] 小范围测试设计完成
[ ] 测试监控指标确定
[ ] 对照组/实验组分桶完成
[ ] 预警机制建立
[ ] 回滚方案准备就绪
```

### Step 5：效果验证

```
验证维度：
1. 核心指标是否改善？
2. 副作用是否在可接受范围？
3. 效果是否可持续？

验证周期：
- 短期效果：1-2周
- 中期效果：1个月
- 长期效果：3个月
```

## 优化类型

### 类型1：指标优化

**场景**：核心指标未达标

**优化方法**：
```
1. 拆解指标构成
   指标A = 因子1 × 因子2 × 因子3

2. 找到最大影响因子
   通过敏感性分析找到杠杆点

3. 聚焦优化最大因子
   集中资源突破关键点
```

**案例**：
```
问题：会员续费率从65%下降到55%

拆解：续费率 = 提醒到达率 × 打开率 × 转化率

分析：
- 提醒到达率：95%（正常）
- 打开率：25%（异常偏低）
- 转化率：60%（正常）

结论：打开率是瓶颈
优化：优化提醒文案和发送时间
```

---

### 类型2：策略优化

**场景**：现有策略效果下降

**优化方法**：
```
1. 归因现有策略效果
   哪些动作产生了效果？

2. 识别失效因素
   市场/用户/竞品发生了什么变化？

3. 设计新策略
   适应新环境的策略

4. 渐进式替换
   新旧策略并行，逐步切换
```

**案例**：
```
问题：老拉新策略K因子从0.5降到0.2

归因：老用户分享意愿下降
原因：竞品出现类似活动，用户疲劳

新策略：
- 调整激励力度（单向→双向）
- 优化传播素材（利益驱动→情感驱动）
- 缩小活动范围（全面推广→精准触达）
```

---

### 类型3：目标优化

**场景**：目标与实际脱节

**优化方法**：
```
1. 检视目标设定
   - 目标是否合理？
   - 假设条件是否还成立？

2. 重新评估现状
   - 资源是否变化？
   - 市场是否变化？

3. 调整目标或资源
   - 目标过高→降低目标或追加资源
   - 目标过低→提高目标或减少资源
```

---

### 类型4：架构优化

**场景**：局部模型结构性问题

**优化方法**：
```
1. 重新定义边界
   是否有重叠或空白？

2. 重新设计联动
   联动机制是否有效？

3. 调整资源配置
   资源是否与重要性匹配？

4. 建立新秩序
   局部模型协同进化
```

## 优化案例库

### 案例1：获客局部模型优化

**问题**：CAC持续上升，从80元涨到180元

**根因分析**：
```
Why1：CAC为什么上升？→ 转化率下降
Why2：转化率为什么下降？→ 落地页疲劳
Why3：落地页为什么疲劳？→ 素材长期未更新
Why4：为什么未更新？→ 缺乏素材迭代机制

根因：缺乏素材快速迭代机制
```

**优化方案**：
```
建立素材工厂模式：
- 每周产出3套新素材
- 建立素材效果排行
- 淘汰低效素材

效果验证：
- 2周后CTR提升25%
- CAC下降至140元
```

---

### 案例2：留存局部模型优化

**问题**：7日留存率从45%下降到35%

**根因分析**：
```
Why1：7日留存为什么下降？→ 用户使用频次下降
Why2：使用频次为什么下降？→ 核心价值感知弱
Why3：价值感知为什么弱？→ 新用户引导不足

根因：新用户引导流程需要优化
```

**优化方案**：
```
设计新用户成长体系：
- Day1：核心功能首次体验
- Day3：完成首个完整动作
- Day7：形成使用习惯

效果验证：
- 7日留存率回升至42%
- 用户满意度提升15%
```

---

### 案例3：变现局部模型优化

**问题**：ARPU长期停滞，无法提升

**根因分析**：
```
Why1：ARPU为什么无法提升？→ 付费用户占比低
Why2：付费转化为什么低？→ 付费点设计不合理
Why3：付费点有什么问题？→ 付费墙设置过早

根因：付费点设置过早，用户价值感知不足
```

**优化方案**：
```
调整付费漏斗：
- 延长免费体验期
- 设置渐进式付费墙
- 增加价值强化节点

效果验证：
- 付费转化率从5%提升至8%
- ARPU提升30%
```
