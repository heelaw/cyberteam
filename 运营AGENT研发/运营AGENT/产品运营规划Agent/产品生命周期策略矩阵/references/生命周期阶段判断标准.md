# 产品生命周期阶段判断标准

## 阶段判断核心指标

### 探索期判断标准

| 指标 | 判断阈值 | 说明 |
|------|----------|------|
| 用户规模 | <10万 | 早期用户积累阶段 |
| 增速 | 无特定要求 | 重点是验证而非增长 |
| 留存 | 次留>40% | 核心功能满足需求 |
| 市场认知 | 低 | 依靠口碑或小规模推广 |
| 竞争 | 较少直接竞争 | 市场尚在培育 |

**探索期信号清单**:
- [ ] 产品刚上线或核心功能刚完成
- [ ] 主要用户是早期采纳者
- [ ] 核心功能还在快速迭代
- [ ] 用户反馈分散，难以形成统一需求
- [ ] 增长主要靠口碑或主动推广

---

### 成长期判断标准

| 指标 | 判断阈值 | 说明 |
|------|----------|------|
| 用户规模 | 10万-1000万 | 快速扩张阶段 |
| 增速 | 月增速>10% | 呈现指数级增长 |
| 留存 | 次留>50% | 产品体验得到验证 |
| 市场认知 | 快速提升 | 品牌开始建立 |
| 竞争 | 加剧 | 竞争对手大量出现 |

**成长期信号清单**:
- [ ] 增速持续保持高水平
- [ ] 用户结构从早期采纳者转向早期大众
- [ ] 核心功能稳定，开始扩展功能边界
- [ ] 获客成本开始上升但仍可接受
- [ ] 竞争对手开始大量出现或模仿

---

### 成熟期判断标准

| 指标 | 判断阈值 | 说明 |
|------|----------|------|
| 用户规模 | 1000万+ | 接近市场天花板 |
| 增速 | 月增速<5% | 增长放缓 |
| 留存 | 稳定高位 | 用户习惯已养成 |
| 市场认知 | 稳定 | 品牌格局清晰 |
| 竞争 | 格局稳定 | 头部玩家明确 |

**成熟期信号清单**:
- [ ] 增速明显放缓
- [ ] 获客成本显著上升
- [ ] 用户增长主要来自抢夺竞品用户
- [ ] 头部玩家格局明确
- [ ] 产品迭代速度放缓

---

### 衰退期判断标准

| 指标 | 判断阈值 | 说明 |
|------|----------|------|
| 用户规模 | 开始下降 | 用户流失加速 |
| 增速 | 负增长 | 流失大于新增 |
| 留存 | 下降趋势 | 用户粘性下降 |
| 市场 | 萎缩 | 被新产品替代 |
| 竞争 | 弱势 | 市场份额下降 |

**衰退期信号清单**:
- [ ] 用户规模开始下降
- [ ] 核心指标呈下降趋势
- [ ] 新替代产品出现
- [ ] 市场份额持续下滑
- [ ] 团队资源投入减少

---

## 阶段转型判断流程

```
1. 收集当前指标数据
   ├── 用户规模及增速
   ├── 留存率（次留/7留/30留）
   ├── 市场竞争情况
   └── 产品迭代情况

2. 对照各阶段判断标准

3. 识别早期信号
   ├── 增长放缓信号
   ├── 留存下降信号
   └── 竞争加剧信号

4. 综合判断
   └── 确定当前所处阶段
```

---

## 注意事项

1. **阶段是渐进的**: 不要期望有明确的分界线，阶段是逐渐过渡的
2. **可能同时存在**: 同一产品的不同模块可能处于不同阶段
3. **外部因素影响**: 政策、技术等外部因素可能加速阶段变化
4. **数据综合判断**: 不要用单一指标判断，综合分析更准确
