# 生命周期阶段转型指南

## 阶段转型时机判断

### 探索期转成长期

**转型时机特征**:

| 维度 | 达标信号 | 说明 |
|------|----------|------|
| 留存 | 次留>50%，7留>20% | 产品体验得到验证 |
| 增长 | 月增>10%且稳定 | 增长引擎已建立 |
| 口碑 | NPS>40或自然增长占比>30% | 产品开始自增长 |
| 竞争 | 竞争对手开始关注/模仿 | 市场机会被验证 |

**转型准备清单**:

- [ ] PMF验证完成
  - [ ] 核心功能留存达标
  - [ ] 用户反馈整体正面
  - [ ] 用户愿意付费或等待付费功能

- [ ] 增长引擎识别
  - [ ] 知道什么渠道获客效率最高
  - [ ] 知道什么产品特性驱动增长
  - [ ] 知道什么用户是最优质的目标用户

- [ ] 资源准备
  - [ ] 有足够的资金支持规模化
  - [ ] 团队能力能支撑增长
  - [ ] 技术架构能支撑扩展

---

### 成长期转成熟期

**转型时机特征**:

| 维度 | 达标信号 | 说明 |
|------|----------|------|
| 规模 | 用户基数足够大 | 市场渗透率接近上限 |
| 增速 | 月增<5%且持续3+月 | 增速放缓成为常态 |
| 竞争 | 头部格局明确 | 市场份额相对稳定 |
| 成本 | 获客成本显著上升 | 增长边际效益下降 |

**转型准备清单**:

- [ ] 增长瓶颈诊断
  - [ ] 识别增长放缓的核心原因
  - [ ] 判断是市场天花板还是策略问题

- [ ] 竞争力评估
  - [ ] 护城河是否足够宽
  - [ ] 竞争优势是否可持续

- [ ] 策略转型准备
  - [ ] 从规模增长转向价值深耕
  - [ ] 从获客导向转向留存/变现导向
  - [ ] 资源配置开始调整

---

### 成熟期转衰退期

**转型时机特征**:

| 维度 | 达标信号 | 说明 |
|------|----------|------|
| 规模 | 用户/收入开始下滑 | 负增长出现 |
| 市场 | 新替代品出现 | 用户开始迁移 |
| 竞争 | 市场份额持续下滑 | 竞争力下降 |
| 投入 | 研发/运营投入缩减 | 公司资源转移 |

**应对策略选择**:

| 策略 | 适用情况 | 核心动作 |
|------|----------|----------|
| **转型策略** | 公司有资源有新方向 | 寻找新市场/新产品延伸 |
| **收割策略** | 已无法竞争但有剩余价值 | 控制成本，最大化剩余价值 |
| **有序退出** | 无法转型也无价值收割 | 有序关闭，保护品牌 |

## 阶段转型执行指南

### 转型类型1: 主动转型

**特征**: 企业主动识别信号，提前布局转型

**执行要点**:
1. **早期识别**: 建立阶段转型的预警指标体系
2. **提前布局**: 在拐点前1-2个季度开始准备
3. **平滑过渡**: 避免断崖式变化
4. **资源配置**: 逐步将资源从旧业务转向新方向

### 转型类型2: 被动转型

**特征**: 被市场/竞争对手推着走，缺乏准备

**执行要点**:
1. **快速止血**: 先控制下滑趋势
2. **聚焦核心**: 集中资源在最有优势的领域
3. **快速决策**: 避免过度讨论而错过时机
4. **保底方案**: 准备有序退出的预案

## 阶段转型常见错误

### 错误1: 延误转型

**表现**: 已经出现明显转型信号但仍坚持旧策略

**后果**: 错过最佳转型窗口，越来越被动

**避免方法**:
- 建立量化的阶段转型判断标准
- 设置预警指标，定期review
- 鼓励团队提出转型需求

### 错误2: 过早转型

**表现**: PMF尚未验证就急于规模化

**后果**: 增长乏力，资源浪费

**避免方法**:
- 坚守探索期的验证标准
- 设定明确的转型前置条件
- 保持耐心，给产品足够验证时间

### 错误3: 转型不彻底

**表现**: 新旧策略并行，资源分散

**后果**: 新旧都没做好

**避免方法**:
- 设定清晰的新旧业务资源配比
- 建立独立的转型团队
- 设定明确的转型成功标准

## 阶段转型沟通指南

### 内部沟通

| 受众 | 沟通重点 | 沟通时机 |
|------|----------|----------|
| 高管 | 战略意义/资源配置/时间预期 | 转型决策前 |
| 团队 | 具体变化/个人影响/支持措施 | 转型宣布时 |
| 全员 | 方向明确/信心建立/行动指引 | 转型进行中 |

### 用户沟通

| 情况 | 沟通方式 | 注意事项 |
|------|----------|----------|
| 产品变化 | 产品内通知+邮件 | 强调价值而非功能 |
| 服务调整 | 公告+客服预案 | 提前告知，提供替代 |
| 停止服务 | 充分预警+迁移方案 | 尽量减少用户损失 |
