### 6. 填写示例

**岗位名称**: 用户运营专员

**核心职责**:

**职责一**: 用户生命周期管理
- 职责描述：负责用户从获客到留存的全生命周期运营
- 衡量标准：月度活跃用户增长率≥10%
- 权重：40%

**职责二**: 活动策划执行
- 职责描述：策划并执行月度用户活动，提升用户活跃度
- 衡量标准：活动参与率≥30%，参与人数≥1000
- 权重：30%

**职责三**: 用户反馈处理
- 职责描述：收集用户反馈，推动产品改进
- 衡量标准：反馈响应率100%，解决率≥80%
- 权重：20%

**权限边界**:

**有决策权的范围**:
- 用户分层策略制定
- 活动奖品选择（单次预算≤5000元）

**需要上报的事项**:
- 超出预算的活动方案
- 涉及产品功能的需求
- 重大用户投诉

## 职责梳理话术

在和团队成员梳理职责时，可以这样沟通：

```
我们一起来看看你的职责说明书，确保你对工作范围是清晰的。
1. 这几项核心职责，你认为合理吗？
2. 哪些工作你觉得是重复的或者缺失的？
3. 和其他同事的协作边界在哪里，你觉得清晰吗？
4. 你认为哪些职责的衡量标准需要调整？
```

## 注意事项

1. 职责描述要具体，避免模糊表述
2. 衡量标准要可量化或有明确判断依据
3. 职责权重加起来要等于100%
4. 权限边界要明确，避免事后争议
