# 汇报框架模板

## 使用说明
用于结构化呈现工作进展，让汇报清晰高效。

## 框架一：PREP法则（简洁汇报）

**Point（结论）**: [一句话说明核心结论]
**Reason（理由）**: [支撑结论的2-3个理由]
**Example（例子）**: [具体数据或案例]
**Point（重复结论）**: [再次强调核心结论]

## 框架二：STAR法则（项目汇报）

**Situation（背景）**: 项目背景是什么
**Task（任务）**: 我负责什么
**Action（行动）**: 我做了什么
**Result（结果）**: 结果如何，数据说话

## 框架三：结构化汇报模板

### 1. 汇报摘要（1句话）

**[项目/事项] [进展/结果/问题]，核心数据[XX]。**

### 2. 详细说明

#### 2.1 目标回顾
- 原始目标：[描述]
- 当前进度：[X%/已完成/进行中]

#### 2.2 关键成果
| 指标 | 目标值 | 实际值 | 状态 |
|------|--------|--------|------|
| | | | |

#### 2.3 问题与风险
| 问题/风险 | 影响 | 应对措施 |
|-----------|------|----------|
| | | |

#### 2.4 下一步计划
| 行动项 | 责任人 | 完成时间 |
|--------|--------|----------|
| | | |

### 3. 需要的支持
- [需要的资源或决策]

## 框架四：电梯汇报法（30秒）

```
我负责[项目名称]，当前[进展状态]。
核心成果是[关键数据]。
下一步[计划/需要支持]。
```

## 场景选择指南

| 场景 | 推荐框架 | 时间 |
|------|----------|------|
| 日常站会 | 电梯汇报法 | 30秒 |
| 周报 | 结构化汇报模板 | 3分钟 |
| 项目汇报 | STAR法则 | 5-10分钟 |
| 决策汇报 | PREP法则 | 3分钟 |

## 注意事项

1. **结论先行**: 先说结论，再说原因
2. **数据说话**: 用具体数字支撑观点
3. **逻辑清晰**: 按重要程度或时间顺序排列
4. **行动导向**: 明确下一步行动和责任人
