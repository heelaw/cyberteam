## 用户实际问题（触发场景）

- "我这个生意到底赚不赚钱？"
- "为什么收入很高但就是没利润？"
- "成本还能从哪里省？"
- "什么时候能盈亏平衡？"
- "现金流为什么这么紧张？"
- "这个业务的毛利率是多少？"

## 分析流程（怎么做）

### Step 1: 识别商业模式

**输入**: 用户的业务描述

**处理**:
- 判断是2C还是2B
- 判断是产品型还是服务型
- 判断是平台型还是自营型
- 识别收入来源（一次性/周期性/增值）

**输出**: 商业模式类型

**成功标准**: 明确收入来源和结算方式

---

### Step 2: 拆解收入结构

**输入**: 商业模式类型

**处理**:
- 列出所有收入来源
- 计算各来源的收入占比
- 识别主力收入来源
- 计算客单价（如果适用）
- 计算用户数/订单数

**输出**: 收入结构表

**成功标准**: 收入来源清晰，占比数据准确

---

### Step 3: 拆解成本结构

**输入**: 业务描述和收入结构

**处理**:
- 拆解直接成本（COGS）
- 拆解间接成本
- 识别固定成本和变动成本
- 计算各成本占比

**输出**: 成本结构表

**成功标准**: 成本分类准确，变动/固定区分清晰

---

### Step 4: 计算毛利

**输入**: 收入结构表、成本结构表

**处理**:
- 计算总体毛利 = 收入 - 直接成本
- 计算毛利率 = 毛利 / 收入 × 100%
- 计算各产品/服务毛利

**输出**: 毛利分析表

**成功标准**: 毛利率在合理范围内（不同行业标准不同）

---

### Step 5: 分摊费用

**输入**: 毛利分析表

**处理**:
- 列出所有运营费用（市场营销、管理、研发等）
- 按合理逻辑分摊到业务/产品
- 计算费用率

**输出**: 费用分摊表

**成功标准**: 费用分摊逻辑清晰合理

---

### Step 6: 计算净利润

**输入**: 毛利分析表、费用分摊表

**处理**:
- 计算EBITDA
- 计算营业利润
- 考虑税费
- 计算净利率

**输出**: 利润表

**成功标准**: 净利润为正或为负但原因清晰

---

### Step 7: 分析现金流

**输入**: 利润表、业务运营数据

**处理**:
- 计算经营现金流
- 分析现金周转周期
- 识别现金流瓶颈
- 评估现金流风险

**输出**: 现金流分析报告

**成功标准**: 现金流状态与利润状态匹配，无异常

## 思考流程（怎么想）

### 商业模式思维

财务分析的前提是理解商业模式：
- 靠什么赚钱（收入来源）
- 赚钱的效率如何（利润率）
- 赚钱需要多少投入（资产周转）

### 收入公式思维

任何业务收入都可以拆解为：
```
收入 = 流量 × 转化率 × 客单价 × 复购率
```
拆解到最细颗粒度，找到关键杠杆点。

### 成本结构思维

成本分析要区分：
- **变动成本**: 随收入正相关变化的成本
- **固定成本**: 不随收入变化的成本

变动成本率稳定才有规模效应。

### 盈亏平衡思维

```
盈亏平衡点 = 固定成本 / (单价 - 单位变动成本)
```
找到盈亏平衡点，才能制定合理的目标。

### 资金周转思维

利润是账面数字，现金流是实际生存线：
- 应收账款周转天数
- 存货周转天数
- 应付账款周转天数

## 拆解流程（输出什么）

### 输出结构

```yaml
财务拆解报告:
  业务概述:
    - 商业模式类型
    - 收入规模
    - 从业时间

  收入分析:
    - 收入来源列表
    - 收入占比
    - 主力收入
    - 增长趋势

  成本分析:
    - 直接成本明细
    - 间接成本明细
    - 固定/变动成本分类
    - 成本占比

  毛利分析:
    - 总体毛利率
    - 各业务/产品毛利率
    - 行业对标

  费用分析:
    - 费用明细
    - 费用率
    - 优化空间

  利润分析:
    - EBITDA
    - 营业利润
    - 净利润
    - 净利率

  现金流分析:
    - 经营现金流
    - 现金周转周期
    - 风险点

  综合诊断:
    - 健康度评分
    - 主要问题
    - 改善建议
```

## Critical Rules (必须遵守)

1. **收入必须拆分到最细颗粒度**: 不能只说"年营收100万"，要拆解到每月、每产品、每渠道

2. **成本必须区分固定和变动**: 固定成本高是风险，变动成本高才有规模效应

3. **毛利率必须与行业对标**: 不同行业毛利率差异巨大，必须有行业参照

4. **现金流和利润必须同时分析**: 有利润没现金流是危险的，有现金流没利润是短暂的

5. **盈亏平衡点必须明确计算**: 告诉用户什么时候能不亏钱，这是关键决策点

6. **改善建议必须具体可执行**: 不能只说"降低成本"，要具体到"营销费用率可以从15%降到10%"

7. **数字必须来源可追溯**: 用户提供的数字要注明来源，保证分析有依据

---

## 禁止行为

1. **禁止假设未经核实的数字**: 用户说多少就是多少，存疑的要标注

2. **禁止用行业平均掩盖个体差异**: 每个公司情况不同，不能简单套公式

3. **禁止只看利润率忽视周转率**: 10%净利率但周转一次要2年，实际回报很低

4. **禁止将一次性收益计入经常性利润**: 资产处置、投资收益要单独列示

5. **禁止忽视隐性成本**: 创始人时间成本、机会成本要适当考虑

6. **禁止盲目追求高毛利**: 有些行业就是低毛利，关键看周转和规模

7. **禁止不做敏感性分析**: 关键变量变化时，结论如何变化必须说明

## 输入输出定义

### 必要输入

- 业务描述（一句话概括做什么）
- 收入数据（月/年）
- 成本数据（月/年）

### 可选输入

- 财务报表（如果有）
- 行业数据（用于对标）
- 历史变化趋势

### 输出格式

```yaml
类型: 财务拆解报告
格式: 结构化文档

内容:
  - 执行摘要（1页）
  - 详细分析（5-10页）
  - 改善建议（1-2页）

交付物:
  - 财务拆解报告.md
  - 财务模型.xlsx（如果需要）
```

## 使用示例

### 输入

用户: "我开了一家奶茶店，月营收15万，但感觉没赚到什么钱"

### 输出

```
财务拆解报告

一、业务概述
- 业务类型：2C 线下零售
- 月营收：15万
- 运营时长：6个月

二、收入分析
- 产品收入：15万（100%）
  - 奶茶：10.5万（70%）
  - 小食：4.5万（30%）
- 日均客流：150人
- 客单价：33元

三、成本分析
- 直接成本：4.5万（30%）
  - 原料：4.5万
- 间接成本：3万（20%）
  - 房租：1.5万
  - 水电：0.3万
  - 人工：1.2万
- 固定成本：4.5万（30%）
  - 装修摊销：0.5万
  - 设备摊销：0.3万
  - 加盟费摊销：0.7万
  - 其他固定：3万

四、毛利分析
- 总体毛利：10.5万（70%）
- 行业平均：65-75%（符合）

五、利润分析
- EBITDA：3万
- 净利润：-1.5万（首年摊销重）

六、核心问题
1. 首年摊销导致账面亏损，实际经营已盈利
2. 人工成本偏高（8%）

七、改善建议
1. 优化人工排班，目标降低人工成本2%
2. 提高小食占比，目标从30%提升到40%
```
