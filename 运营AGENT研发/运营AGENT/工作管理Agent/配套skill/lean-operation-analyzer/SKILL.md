# lean-operation-analyzer

## 简介

运用精益运营思维，通过数据分析发现运营问题，优化投入产出比。

## 触发关键词

```
精益运营、效率优化、投入产出比、数据分析、运营诊断、问题发现
```

## 触发场景描述

```
当需要分析运营效率或优化投入时触发：
- 需要诊断当前运营问题
- 需要提升投入产出比
- 需要找到最优资源配置
- 需要数据分析支持决策
```

## 输入

### 必需输入

| 字段 | 类型 | 说明 |
|------|------|------|
| operation_data | object | 运营数据 |
| resource_input | object | 资源投入 |

### 可选输入

| 字段 | 类型 | 说明 |
|------|------|------|

## 方法论：精益运营分析

### 1. 数据收集

- 收集运营关键指标
- 记录资源投入
- 获取行业基准

### 2. 效率分析

- 计算各渠道/环节效率
- 对比行业基准
- 识别异常数据

### 3. 问题诊断

- 找出最低效的环节
- 分析原因
- 评估影响程度

### 4. 优化建议

- 提出优化方案
- 评估投入产出
- 制定实施计划

## Critical Rules

### 必须遵守

1. 基于数据而非直觉
2. 量化问题而非模糊描述
3. 考虑全局最优而非局部
4. 平衡短期和长期

### 禁止行为

1. 禁止只看单一指标
2. 禁止忽视隐性成本
3. 禁止优化损害用户体验
4. 禁止过度优化
5. 禁止缺乏长期视角的优化决策

## 评估清单

| 维度 | 标准 | 权重 |
|------|------|------|
| 数据完整 | 数据全面准确 | 25% |
| 分析深度 | 找到根本原因 | 30% |
| 建议可行 | 可落地执行 | 25% |
| 量化程度 | 结论可量化 | 20% |
