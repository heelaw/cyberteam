# KPI拆解案例

## 成功案例：互联网产品DAU提升

### 背景
某SaaS产品DAU增长停滞，公司设定年度DAU提升50%的目标。

### 拆解思路
采用"漏斗分解法"：
```
DAU = 新增用户 + 留存用户 + 回流用户
    = 新增用户 × 留存率 + 回流用户
```

### 分解结果

| 一级KPI | 二级KPI | 目标值 | 责任人 |
|---------|---------|--------|--------|
| 新增用户 | 注册转化率 | 8% | 增长团队 |
| 新增用户 | 渠道ROI | 3.5 | 投放团队 |
| 留存率 | 次日留存 | 45% | 产品团队 |
| 回流用户 | 推送打开率 | 25% | 运营团队 |

### 成果
- DAU提升：达标52%
- 各团队聚焦明确
- 数据追踪自动化

### 关键教训
指标分解要找准"杠杆点"，不是平均分配。

## 失败案例：销售KPI层层加码

### 背景
总公司目标增长30%，大区目标增长35%，省区目标增长40%。

### 问题分析
1. 目标逐级加码，导致底层无法完成
2. 销售团队只顾短期，忽视长期客户经营
3. 达成率低导致激励失效

### 关键教训
KPI分解不是"分配"任务，而是对齐目标与资源。

## 成功案例：研发团队效能提升

### 背景
研发团队长期被诟病"看不见产出"，需要建立效能指标。

### 拆解方案

| KPI | 定义 | 目标 |
|-----|------|------|
| 需求交付周期 | 需求提出到上线 | < 7天 |
| 发布频率 | 周发布次数 | ≥ 3次 |
| Bug密度 | 千行代码Bug数 | < 2个 |

### 成果
- 研发效能可视化
- 与业务部门对齐预期
- 持续改进有方向

### 关键教训
研发指标要反映业务价值，而非技术细节。
