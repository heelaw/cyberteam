# 流程优化方法论

## 核心概念定义

流程优化是通过系统性分析、设计、执行和监控，持续改进业务流程以提升效率、降低成本、增强质量的过程。流程优化不是一次性工程，而是持续迭代的能力建设。

## 方法论体系

### AIDET模型（患者沟通框架，泛用）
- Acknowledge：认可/问候
- Introduce：介绍
- Duration：时间预期
- Explanation：解释说明
- Thank you：感谢

### DMAIC方法（六西格玛）
- Define：定义问题
- Measure：测量现状
- Analyze：分析原因
- Improve：改善方案
- Control：控制固化

### PDCA循环（持续改进）
- Plan：计划
- Do：执行
- Check：检查
- Act：处理

## 方法论步骤

### 第一阶段：流程诊断
1. 绘制AS-IS（当前状态）流程图
2. 识别关键路径和瓶颈
3. 量化每个步骤的时间、成本、质量
4. 收集利益相关方反馈

### 第二阶段：方案设计
1. 设定优化目标（时间/成本/质量）
2. 设计TO-BE（目标状态）流程
3. 进行ECRS分析：Eliminate/Combine/Rearrange/Simplify
4. 评估风险和实施可行性

### 第三阶段：实施落地
1. 制定实施计划
2. 进行小范围试点
3. 收集反馈迭代优化
4. 全面推广

### 第四阶段：效果评估
1. 对比优化前后指标
2. 收集用户/员工反馈
3. 识别新问题
4. 启动下一轮优化

## 关键成功因素

| 因素 | 说明 |
|------|------|
| 高层支持 | 流程优化需要跨部门协调 |
| 数据驱动 | 用数据而非感觉判断效果 |
| 全员参与 | 执行者参与设计才能有效落地 |
| 持续迭代 | 建立流程Review机制 |

## 常见误区

1. **优化局部导致全局恶化** - 孤立地优化某个环节
2. **忽视人的因素** - 只改流程不关注人
3. **一次性大工程** - 追求完美方案而迟迟不动
4. **优化后不固化** - 改善成果无法持续
