### 5. 快速验证方法清单

当资源有限时，可选用以下快速验证方法：

| 方法 | 适用场景 | 资源需求 | 验证速度 |
|------|---------|---------|---------|
| 问卷调查 | 验证用户需求是否存在 | 低 | 快 |
| 竞品分析 | 验证市场机会大小 | 低 | 快 |
| 用户访谈 | 深度理解用户痛点 | 中 | 中 |
| 众筹/预售 | 验证用户付费意愿 | 中 | 中 |
| MVP landing page | 验证用户获取可能性 | 低 | 快 |

| 付费广告测试 | 验证获客成本 | 高 | 快 |

### 6. 验证失败后的分析框架

当假设验证失败时，使用以下框架分析原因：

**假设验证失败分析**：

1. **假设本身问题**
   - 假设是否基于真实的用户反馈？
   - 假设是否过于乐观？

2. **执行问题**
   - 验证方法是否正确？
   - 执行过程是否有偏差？

3. **外部环境问题**
   - 市场是否发生变化？
   - 是否有竞品影响？

**调整策略**：
- 调整假设 → 重新验证
- 调整执行 → 优化方法后重新验证
- 调整方向 → 重新评估业务模型
