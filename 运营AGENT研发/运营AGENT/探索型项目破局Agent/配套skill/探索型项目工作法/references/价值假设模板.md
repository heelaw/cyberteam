# 价值假设模板

## 模板说明

此模板用于形成和表达核心价值假设。价值假设是探索型项目的起点，必须清晰、具体、可验证。

## 核心价值假设语句

**标准格式**：
> 我们相信 [目标用户] 需要 [某种价值]，因为 [某种洞察]。

**示例**：
> 我们相信 刚入职0-3年的互联网产品经理 需要 系统化的需求管理方法，因为他们在日常工作中经常因为需求优先级判断不准而被质疑，迫切需要一套可复用的方法论。

## 价值假设构成要素

### 1. 目标用户（Who）
- 谁是目标用户？
- 他们有什么特征？
- 规模有多大？
- 如何触达？

### 2. 痛点问题（Problem）
- 他们面临什么具体问题？
- 这个问题有多痛？
- 现有解决方案有什么缺陷？

### 3. 预期价值（Value）
- 我们的解决方案提供什么价值？
- 为什么比现有方案更好？
- 价值如何衡量？

### 4. 洞察基础（Insight）
- 我们凭什么认为这个价值假设成立？
- 有什么数据或现象支撑？
- 这是真需求还是伪需求？

## 价值假设检验表

在确定价值假设前，请逐项检验：

| 检验项 | 问题 | 判断 |
|--------|------|------|
| 真实性 | 这个痛点是用户自己感知到的，还是我们臆想的？ | □ 是 □ 否 |
| 意愿性 | 用户愿意为解决这个问题付费或投入时间吗？ | □ 是 □ 否 |
| 可触达 | 我们有能力触达这些目标用户吗？ | □ 是 □ 否 |
| 可落地 | 我们有能力提供这个价值吗？ | □ 是 □ 否 |
| 可衡量 | 价值假设能被验证或证伪吗？ | □ 是 □ 否 |

**只有全部通过，才能进入下一阶段。**

## 价值假设陈述模板

请用以下格式陈述你的价值假设：

```
【价值假设 #1】
我们相信 [具体的用户群体] 需要 [具体的产品/服务/价值]，
因为 [支撑这个假设的洞察、数据或现象]。

用户痛点：[描述具体痛点]
现有方案：[描述现有解决方案及其缺陷]
我们的方案：[描述我们提供的价值]

检验结果：[通过/未通过] 
未通过原因：[如果未通过，说明原因]
```

## 常见错误

1. **把愿景当假设** - "让教育普惠化"不是假设，是愿景
2. **把功能当价值** - "我们有AI功能"不是价值，是功能
3. **把用户当市场** - "所有职场人"不是用户，是市场
4. **把感觉当洞察** - "我觉得用户会喜欢"不是洞察，是感觉
