# 产品运营规划 - 评估清单

## 评估维度

| 维度 | 权重 | 评分标准 |
|------|------|----------|
| 阶段判断准确性 | 20% | 产品阶段判断是否有依据 |
| 因素分析完整性 | 20% | 5种因素是否都分析到位 |
| 破局点聚焦度 | 20% | 是否集中资源突破核心点 |
| 目标可量化性 | 15% | 目标是否可衡量 |
| 执行路径清晰度 | 15% | 行动项是否具体可执行 |
| 评估机制完善度 | 10% | 是否有复盘和调整机制 |

## 评分等级

| 等级 | 分数 | 说明 |
|------|------|------|
| 优秀 | 90-100 | 全面满足所有维度，方案科学可行 |
| 良好 | 75-89 | 满足大部分维度，方案可行 |
| 合格 | 60-74 | 基本满足维度要求，需补充细节 |
| 不合格 | <60 | 存在明显缺陷，需重新规划 |

## 逐项检查清单

### 阶段判断准确性 (20分)

- [ ] 有明确的数据支撑阶段判断
- [ ] 考虑了增长曲线特征（S型曲线）
- [ ] 考虑了市场环境因素
- [ ] 判断结论明确无歧义

### 因素分析完整性 (20分)

- [ ] 产品阶段分析完整
- [ ] 用户特征分析有数据支撑
- [ ] 竞争环境分析包含主要竞品
- [ ] 资源条件盘点具体
- [ ] 团队能力评估客观

### 破局点聚焦度 (20分)

- [ ] 破局点不超过2个
- [ ] 破局点有多重因素支撑
- [ ] 资源投入与破局点匹配
- [ ] 其他非核心事项主动放弃

### 目标可量化性 (15分)

- [ ] 每个阶段目标都有量化指标
- [ ] 时间节点明确
- [ ] 成功标准可验证
- [ ] 目标有挑战但可实现

### 执行路径清晰度 (15分)

- [ ] 每个阶段有具体行动项
- [ ] 明确了责任人
- [ ] 有资源投入计划
- [ ] 有风险预案

### 评估机制完善度 (10分)

- [ ] 有数据监控体系
- [ ] 有预警机制
- [ ] 有定期复盘机制
- [ ] 有迭代优化机制

## 常见问题

### 问题1：阶段判断模糊
**表现**：无法明确产品处于哪个阶段
**修复**：回到S型曲线，对照各项指标判断

### 问题2：贪多求全
**表现**：列出10个运营要点分散资源
**修复**：只保留1-2个核心破局点

### 问题3：目标空洞
**表现**：目标为"提升用户满意度"无法衡量
**修复**：转化为NPS分数、满意度评分等可量化指标
