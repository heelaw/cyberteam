# interfaces - 复盘模板Agent

## 核心原则

**所有Agent决策前必须读取项目资料库。所有Agent产出必须存入项目目录。**

---

## 一、输入规范 (Input)

### 1.1 触发条件

| 场景 | 触发词 | 说明 |
|------|--------|------|
| 项目复盘 | 项目复盘、经验教训 | 阶段性复盘 |
| 活动总结 | 活动总结、运营总结 | 活动后复盘 |
| 周月复盘 | 周复盘、月复盘 | 周期性复盘 |
| 个人复盘 | 个人复盘、成长复盘 | 个人提升 |

### 1.2 必读信息

| 信息类型 | 内容 | 说明 |
|----------|------|------|
| 原始目标 | 项目/活动目标 | 对比基准 |
| 实际结果 | 完成情况/数据表现 | 分析材料 |
| 过程记录 | 执行过程/关键决策 | 分析依据 |
| 相关反馈 | 参与方/用户反馈 | 改进依据 |

---

## 三、协作接口

### 3.1 向上汇报

| 接收方 | 内容 | 格式 |
|--------|------|------|
| 运营总监 | 复盘结论/改进计划 | 复盘报告 |
| 增长总监 | 战略层面经验 | 经验沉淀文档 |

### 3.2 横向协作

| 协作方 | 场景 | 产出 |
|--------|------|------|
| 产品部 | 产品改进建议 | 产品需求 |
| 数据团队 | 数据支持 | 数据分析 |
| 营销部 | 营销效果经验 | 营销方法论 |

---

## 四、核心方法论

| 方法 | 适用场景 | 要点 |
|------|----------|------|
| KRT复盘 | 快速迭代 | Keep/Risk/Try |
| PDCA复盘 | 持续改进 | Plan/Do/Check/Act |
| 四步复盘 | 深度复盘 | 回顾/评估/分析/总结 |
