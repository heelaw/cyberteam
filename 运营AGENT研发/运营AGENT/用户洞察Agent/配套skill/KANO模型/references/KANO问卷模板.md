# KANO问卷模板

## KANO问卷设计原则

1. **正反问法** - 每个功能需要正向和反向两个问题
2. **功能明确** - 描述要具体，避免模糊
3. **选项标准** - 使用5级Likert量表
4. **独立提问** - 每个功能单独评价

---

## 标准问卷结构

### 问卷标题

```
[产品名] 功能需求调研

您好，我们正在了解您对以下功能的看法，请根据您的真实感受作答。
```

### 基本信息（可选）

```markdown
1. 您的年龄: ____
2. 您使用本产品的频率: 每天 / 每周 / 偶尔
3. 您使用本产品的时长: 不到1月 / 1-6月 / 6月以上
```

### 功能评价问题

对每个功能，请回答两个问题：

**问题模板**:
```
【功能名称】
功能描述: [功能的简要说明]

Q1: 如果[产品]提供此功能，您觉得怎么样？
○ 我很喜欢（能让我的体验超出预期）
○ 理应如此（这是应该有的）
○ 无所谓（有没有对我没影响）
○ 勉强接受（勉强可以接受）
○ 我很不喜欢（让我非常不满）

Q2: 如果[产品]不提供此功能，您觉得怎么样？
○ 我很喜欢
○ 理应如此
○ 无所谓
○ 勉强接受
○ 我很不喜欢
```

## 回答选项说明

| 选项 | 含义 | KANO分类影响 |
|------|------|------------|
| 我很喜欢 | 功能让您很满意 | A (兴奋) |
| 理应如此 | 这是基本应该有的 | M (必备) |
| 无所谓 | 有没有都无所谓 | I (无差异) |
| 勉强接受 | 勉强可以接受 | R (反向) |
| 我很不喜欢 | 功能让您很不满意 | R (反向) |

## 问卷发放指南

### 样本要求

| 产品阶段 | 最小样本量 | 目标用户 |
|---------|-----------|---------|
| MVP阶段 | 50+ | 种子用户 |
| 成长期 | 100+ | 活跃用户 |
| 成熟期 | 200+ | 分层抽样 |

### 筛选条件

- 有实际使用经验
- 愿意认真作答
- 覆盖不同用户类型

### 注意事项

1. 问卷不能过长（不超过10个功能）
2. 功能描述要通俗易懂
3. 可加入注意力检测题
4. 完成后可给予小额奖励
