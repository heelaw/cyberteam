# 洞察提炼指南

## 洞察定义

用户洞察是对用户行为、言语、感受背后深层原因的理解和提炼，能够指导产品或服务改进。

## 洞察vs观察vs观点

| 类型 | 定义 | 示例 |
|------|------|------|
| 观察 | 描述用户做了什么 | "用户平均使用30分钟后离开" |
| 观点 | 对观察的一般性解读 | "用户缺乏耐心" |
| 洞察 | 深层原因的独特理解 | "用户需要即时的成就感来维持注意力" |

---

## 洞察提炼步骤

### Step 1: 收集原始素材

**来源**:
- 用户访谈记录
- 行为数据
- 用户反馈
- 客服记录

**要求**:
- 保持原话
- 记录场景
- 标注情绪

### Step 2: 识别模式

**模式类型**:

| 模式类型 | 描述 | 示例 |
|---------|------|------|
| 重复出现 | 多个用户提到 | "推荐不准确"被多人提及 |
| 言行差异 | 说和做不一致 | 说要质量，实际选便宜 |
| 极端反应 | 强烈正面或负面 | "这是我用过的最好的..." |
| 意外发现 | 出乎意料的发现 | 用户把功能用在非预期场景 |

### Step 3: 追问为什么

**连续追问法**:

```
用户说: "内容太多了"
追问: 为什么觉得多？
回答: "找到想看的很费时间"
追问: 为什么费时间？
回答: "推荐的不感兴趣"
追问: 为什么不感兴趣？
回答: "感觉算法不懂我"
洞察: 用户需要的是"被理解"而非"更多内容"
```

### Step 4: 验证洞察

**验证问题**:

1. **数据验证**: 有行为数据支撑吗？
2. **逻辑验证**: 因果关系合理吗？
3. **代表性验证**: 其他用户也这样吗？
4. **可行性验证**: 产品能解决这个问题吗？

### Step 5: 提炼陈述

**洞察陈述公式**:

```
[用户群体] 在 [场景] 中 [行为表现]，因为 [深层原因]，他们需要 [真实需求]
```

**示例**:

```
通勤用户 在 碎片化浏览中 经常中途放弃，
因为 内容缺乏即时成就感，
他们需要 每30秒有一个"收获感"来维持参与
```

## 洞察分类框架

### 1. 需求洞察

用户真正需要什么

```
用户说: "想要更快的服务"
洞察: 用户需要的是"效率感"而非绝对速度
```

### 2. 障碍洞察

什么阻止用户达成目标

```
用户说: "支付时放弃了"
洞察: 支付障碍不是速度，而是信任和安全感
```

### 3. 动机洞察

什么驱动用户行为

```
用户说: "分享到朋友圈"
洞察: 驱动用户的是"社交资本"而非产品本身
```

### 4. 障碍与动机组合洞察

动机+障碍的组合

```
用户想要: 被认可（动机）
但障碍是: 不知道什么内容能获得认可（障碍）
洞察: 用户需要一个"内容影响力"的可见指标
```

## 洞察优先级

### 影响力矩阵

```
                    高影响
                      │
         ┌────────────┼────────────┐
         │   核心     │   重要    │
高频     │  洞察      │  洞察     │
         ├────────────┼────────────┤
         │   次要     │   低价值  │
低频     │  洞察      │  洞察     │
         └────────────┼────────────┘
                      │
                    低影响
```

### 优先级判定标准

| 优先级 | 标准 |
|--------|------|
| 核心洞察 | 高频 + 高影响 + 可解决 |
| 重要洞察 | 低频 + 高影响 + 可解决 |
| 次要洞察 | 高频 + 低影响 + 可解决 |
| 低价值洞察 | 低频 + 低影响 |

## 常见错误

### 错误1: 把观察当洞察

```
❌ 错误: 用户在支付页放弃率高
✓ 正确: 用户在支付页放弃是因为对交易安全缺乏信任
```

### 错误2: 把功能当洞察

```
❌ 错误: 用户需要推荐功能
✓ 正确: 用户需要被理解，减少选择成本
```

### 错误3: 把细分当洞察

```
❌ 错误: 25-30岁女性用户喜欢这样做
✓ 正确: 处于人生转折期的用户需要确定性来降低焦虑
```

### 错误4: 过度归纳

```
❌ 错误: 用户都想要便宜
✓ 正确: 价格敏感用户在被感知为"值得"时会忽略价格
```
