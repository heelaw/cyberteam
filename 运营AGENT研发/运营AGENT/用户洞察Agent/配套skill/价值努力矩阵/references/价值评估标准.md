# 价值评估标准

## 价值评估维度

### 1. 用户价值

| 等级 | 描述 | 评估标准 |
|------|------|---------|
| 高 | 直接解决核心问题 | 能显著提升用户体验或解决问题 |
| 中 | 改善用户体验 | 能提升使用效率或舒适度 |
| 低 | 边缘优化 | 体验有改善但不明显 |

### 2. 业务价值

| 等级 | 描述 | 评估标准 |
|------|------|---------|
| 高 | 显著提升核心指标 | DAU/留存/转化等核心指标提升>10% |
| 中 | 改善辅助指标 | 辅助指标改善，提升业务效率 |
| 低 | 边缘优化 | 对核心指标影响小 |

### 3. 战略价值

| 等级 | 描述 | 评估标准 |
|------|------|---------|
| 高 | 布局未来 | 为未来3-5年打基础 |
| 中 | 竞争优势 | 建立或巩固竞争壁垒 |
| 低 | 当前优化 | 解决当下问题 |

## 快速评估问题

### 用户价值问题

1. 这个功能解决用户什么问题？
2. 有多少用户会遇到这个问题？
3. 解决这个问题对用户有多重要？
4. 用户会为此付钱或增加使用吗？

### 业务价值问题

1. 这个功能影响哪些业务指标？
2. 影响幅度有多大？
3. 能带来多少收入或成本节省？
4. 对竞品优势有什么影响？

### 战略价值问题

1. 这个功能是否符合公司战略？
2. 是否有助于进入新市场？
3. 是否有助于建立壁垒？
4. 3年后回头看，这个还重要吗？

## 评估讨论指南

### 讨论流程

1. **各自评估**: 评估前先各自给出初步评估
2. **分享理由**: 每人分享评估理由
3. **讨论分歧**: 重点讨论差异点
4. **达成共识**: 最终达成一致评估

### 常见分歧处理

| 分歧 | 处理方法 |
|------|---------|
| 用户价值vs业务价值 | 以用户价值为准 |
| 短期vs长期价值 | 考虑贴现 |
| 确定vs不确定 | 使用概率加权 |

- [ ] 用户价值评估有数据支撑？

- [ ] 评估记录是否完整？
- [ ] 是否考虑了后续影响？
- [ ] 是否需要更新？
