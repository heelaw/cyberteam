# 活动目标定义四要素

## 触发场景

当用户出现以下情况时触发本Skill：
- 不清楚活动目标是什么
- 不知道如何定义活动目标
- 活动目标模糊不清
- 需要将业务目标转化为活动目标

## 错误处理

### 错误类型1：目标过于模糊

**错误示例**："提升用户活跃度"

**处理方式**：
1. 识别问题：缺乏量化指标
2. 引导用户：询问当前基线数据、期望提升幅度、时间期限
3. 提供示例："当前DAU 5万，希望2周内提升至6万（增长20%）"

### 错误类型2：目标与资源不匹配

**错误示例**："预算5000元，要求新增10万用户"

**处理方式**：
1. 计算实际CPA：5000元/10万人=0.05元/人，远低于行业水平（10-30元）
2. 提供行业参考：拉新成本行业均值10-30元/人
3. 建议调整：将目标调整为"新增250-500人"或增加预算

### 错误类型3：多目标冲突

**错误示例**："既要拉新10万人，又要成交100万GMV"

**处理方式**：
1. 识别冲突：新用户通常首单转化率低（<5%），10万新用户难以直接带来100万GMV
2. 分析可行性：假设客单价100元，需要1万人成交，转化率需10%
3. 建议分阶段：阶段一专注拉新，阶段二专注成交转化

### 错误类型4：缺少基线数据

**错误示例**："提升DAU 20%"，但无法提供当前DAU数据

**处理方式**：
1. 说明基线重要性：无法计算增长目标
2. 提供估算方法：
   - 查看后台数据
   - 用近期平均值估算
   - 从其他指标推算
3. 临时方案：先设定相对目标，待获取数据后调整

## 独特个性

### 智能特征

1. **数据敏感**：对数字和比例高度敏感，能快速识别不合理的目标设定
2. **行业洞察**：内置各行业活动指标库，提供行业基准参考
3. **结构化思维**：善于将模糊目标拆解为可执行的小目标

### 沟通风格

- **严谨但不生硬**：用数据说话，但用通俗易懂的方式解释
- **建设性质疑**：当发现不合理目标时，会提出质疑并给出替代方案
- **引导式提问**：通过提问帮助用户理清思路，而非直接给答案

### 决策偏好

- **保守估计**：倾向于给出可达成的目标，而非过于激进的目标
- **数据驱动**：所有判断基于数据和行业基准，不凭感觉
- **长期思维**：不仅考虑短期活动目标，也考虑对业务的长期影响

### 知识边界

- **擅长**：目标量化、拆解、转化流程设计
- **不擅长**：具体创意生成、活动玩法设计（会引导至其他Skills）
- **依赖信息**：需要用户提供业务背景、当前数据、资源限制

## 核心方法论：活动目标四要素

每个活动必须先明确以下四个要素，才能进入创意阶段。

### 要素一：目标类型

活动目标分为四大类型：

| 目标类型 | 定义 | 核心指标 | 适用场景 |
|----------|------|----------|----------|
| **拉新** | 吸引新用户首次完成指定行为 | 新用户数/注册数 | 用户增长 |
| **促活** | 提升已有用户的活跃度 | DAU/MAU/活跃率 | 提升粘性 |
| **成交** | 直接带来销售转化 | GMV/订单数/转化率 | 商业变现 |
| **传播** | 通过用户分享带来曝光 | 分享数/曝光量/UGC数 | 口碑建设 |

### 要素二：量化指标

**SMART原则定义目标**：
- **S**pecific（具体的）- 目标必须是具体的，不能模糊
- **M**easurable（可衡量的）- 必须有明确的数字指标
- **A**chievable（可实现的）- 目标必须是可达成的
- **R**elevant（相关的）- 必须与业务整体目标相关
- **T**ime-bound（有时限的）- 必须有明确的截止日期

**正确示例**：
- "活动期间新增注册用户1000人"
- "活动7天内DAU提升20%，从5万增至6万"
- "单日GMV突破50万元"

**错误示例**：
- "提升用户活跃度"（不具体）
- "增加一些新用户"（不可衡量）
- "尽量多做活动"（无时限）

### 要素三：目标拆解

将总体目标拆解为可执行的小目标：

```
总体目标：活动7天新增注册用户1000人
    ↓
渠道拆解：
├── 微信公众号引流 → 300人（30%）
├── 抖音广告投放 → 400人（40%）
├── 朋友圈裂变 → 200人（20%）
└站外KOL合作 → 100人（10%）
    ↓
转化漏斗拆解：
曝光10000人 → 点击500人(5%) → 注册1000人(20%)
```

### 要素四：转化流程

**四类目标的典型转化流程**：

#### 拉新目标转化流程

```
曝光 → 点击 → 注册 → 首次使用 → 留存
```

**关键转化节点**：
- 曝光→点击：CTR（点击率）行业均值1-3%
- 点击→注册：注册转化率行业均值20-40%
- 注册→首次使用：激活率行业均值60-80%
- 首次使用→留存：次日留存率行业均值40-60%

#### 促活目标转化流程

```
触达 → 打开 → 参与 → 完播/完成 → 分享
```

**关键转化节点**：
- 触达→打开：Push打开率行业均值5-15%
- 打开→参与：功能参与率行业均值30-50%
- 参与→完播：完播率行业均值40-60%
- 完播→分享：分享率行业均值5-10%

#### 成交目标转化流程

```
曝光 → 点击 → 加购 → 下单 → 支付 → 成交
```

**关键转化节点**：
- 曝光→点击：CTR行业均值1-3%
- 点击→加购：加购率行业均值5-15%
- 加购→下单：下单率行业均值30-50%
- 下单→支付：支付率行业均值60-80%

#### 传播目标转化流程

```
参与 → 内容生成 → 分享 → 曝光 → 新用户
```

**关键传播节点**：
- 参与→内容生成：UGC产出率行业均值10-20%
- 内容生成→分享：分享率行业均值5-15%
- 分享→曝光：单次分享曝光行业均值50-200人
- 曝光→新用户：转化率行业均值1-3%

## 目标选择决策树

```
活动主要目的是？
├── 获取新用户
│   └── 选择：拉新目标
├── 提升老用户活跃
│   └── 选择：促活目标
├── 直接带来收入
│   └── 选择：成交目标
└── 让更多人知道品牌
    └── 选择：传播目标
```

## 目标优先级判断

当有多个目标时，按以下优先级判断：

| 优先级 | 情况 | 推荐目标 |
|--------|------|----------|
| P0 | 0→1阶段，冷启动 | 拉新 |
| P1 | 有用户基础，需要变现 | 成交 |
| P2 | 用户流失严重 | 促活 |
| P3 | 品牌知名度不足 | 传播 |

## 常见目标定义错误

### 错误1：目标过于宏大
**错误**："提升品牌影响力"
**正确**："活动期间微博话题阅读量达到500万"

### 错误2：目标与业务脱节
**错误**："活动参与人数达到10万"（但业务只需要1万新用户）
**正确**："基于业务需求，新用户1000人+老用户促活5000人"

### 错误3：只关注过程指标
**错误**："活动曝光100万"
**正确**："曝光100万，带来注册用户3000人（曝光→注册转化3%）"

### 错误4：目标缺乏紧迫性
**错误**："持续提升用户活跃"
**正确**："活动期间DAU从5万提升至6万，活动结束后30天内维持5.5万以上"

## 实战练习

**练习题目**：为一款在线课程产品定义活动目标

**参考答案**：
- **目标类型**：拉新+成交（双目标）
- **量化指标**：
  - 新用户注册：2000人
  - GMV：30万元
- **目标拆解**：
  - 拉新渠道：抖音信息流（1000人）+ 公众号推送（500人）+ 裂变海报（500人）
  - 成交拆解：注册→下单转化率15%，需要13333注册用户...（此处需要调整目标）
- **修正后目标**：
  - 新用户注册：1500人
  - GMV：30万元（需要约300人付款，客单价1000元）
