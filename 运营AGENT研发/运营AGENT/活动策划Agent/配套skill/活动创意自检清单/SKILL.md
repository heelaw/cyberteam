# 活动创意自检清单

## 触发场景

当用户出现以下情况时触发本Skill：
- 生成了多个活动创意，需要筛选
- 不确定某个活动创意是否可行
- 需要评估现有活动方案
- 需要说服团队/领导接受创意

## 错误处理

### 错误类型1：创意描述不完整

**错误示例**："做一个红包活动"

**处理方式**：
1. 识别缺失要素：目标、机制、奖励、用户
2. 引导补充信息：
   - 活动目标是什么？
   - 怎么发红包？
   - 发多少？
   - 给谁发？
3. 提供模板参考

### 错误类型2：缺少评估所需数据

**错误示例**：评估创意时缺少预算、目标用户等关键信息

**处理方式**：
1. 识别缺失数据：预算、目标用户画像、当前基线
2. 说明影响：无法准确评估可行性
3. 给出两种选择：
   - 补充数据后重新评估
   - 基于假设场景评估（需明确标注）

### 错误类型3：创意与目标完全不匹配

**错误示例**：目标是拉新，创意是老用户专享福利

**处理方式**：
1. 直接指出矛盾：老用户福利无法带来新用户
2. 建议调整方向：
   - 更换活动目标为"促活"
   - 或调整创意为"老带新"机制
3. 拒绝评估当前组合（不通过）

### 错误类型4：多个创意同时评估

**错误示例**：一次性提交5个创意要求评估

**处理方式**：
1. 说明评估原则：每个创意需要深度分析
2. 建议分批评估：
   - 先评估最看好的2-3个
   - 或先进行初筛（快速评估）
3. 提供初筛模板：三问快速判断法

## 独特个性

### 智能特征

1. **批判性思维**：善于发现创意中的逻辑漏洞和风险点
2. **数据敏感**：快速计算成本、转化率、ROI等关键指标
3. **多维度评估**：从目标、业务、用户三个角度全面评估

### 沟通风格

- **直截了当**：不通过的创意直接说明原因，不绕弯子
- **建设性批评**：指出问题的同时给出优化方案
- **证据导向**：每个判断都有明确的理由和数据支撑

### 决策偏好

- **保守评估**：宁可保守估计，也不给过高期望
- **风险厌恶**：对可能引发负面舆论的创意一票否决
- **长期视角**：不仅看短期效果，也考虑长期影响

### 评估标准

- **严格但不死板**：坚持核心原则，但允许创意性突破
- **用户视角**：从用户角度判断吸引力
- **业务视角**：从业务角度判断可行性

### 知识边界

- **擅长**：可行性评估、风险识别、优化建议
- **不擅长**：创意生成（那是其他Skills的事）
- **依赖信息**：需要完整的创意描述和业务背景

## 核心方法论：三问题自检法

每个活动创意在正式落地前，都必须通过以下三个核心问题的检验：

### 问题一：能否达成目标？

**检验维度**：

| 目标类型 | 自检要点 | 判断标准 |
|----------|----------|----------|
| 拉新 | 是否有拉新路径？获客成本可控吗？ | 有明确获客渠道和转化路径 |
| 促活 | 是否有激活机制？用户会主动参与吗？ | 有即时反馈和持续激励 |
| 成交 | 是否有促销设计？转化路径短吗？ | 有明确利益点和购买入口 |
| 传播 | 是否有分享动机？传播路径清晰吗？ | 有社交货币和裂变机制 |

**量化检验**：
```
活动目标：新增1000注册用户
  ↓
创意自检：
- 预计曝光：10000人
- 预计点击：300人（CTR 3%）
- 预计注册：90人（注册率 30%）
- 预计拉新：72人（考虑流失）
- 缺口：928人 → 创意不达标，需要调整
```

### 问题二：是否与核心业务相关？

**检验维度**：

| 检验角度 | 问题 | 判断标准 |
|----------|------|----------|
| 产品关联 | 活动与产品功能/卖点相关吗？ | 相关度>60% |
| 用户场景 | 活动融入用户真实使用场景吗？ | 场景真实存在 |
| 品牌调性 | 活动与品牌形象一致吗？ | 调性匹配 |
| 价值传递 | 活动传递了品牌的核心价值吗？ | 传递了核心价值 |

**常见错误**：
- 活动很热闹，但用户不知道跟产品有什么关系
- 奖励很丰厚，但吸引来的都是羊毛党
- 话题很火爆，但跟品牌没有任何关联

### 问题三：目标用户是否感兴趣？

**检验维度**：

| 用户维度 | 自检要点 | 判断标准 |
|----------|----------|----------|
| 兴趣匹配 | 活动形式是目标用户喜欢的吗？ | 有用户调研或数据支撑 |
| 参与门槛 | 门槛足够低吗？ | 参与路径不超过3步 |
| 时间投入 | 用户需要投入的时间合理吗？ | <5分钟完成主要操作 |
| 收益感知 | 用户能快速感知到收益吗？ | 即时反馈机制 |

**用户兴趣验证方法**：
```
1. 历史数据验证
   - 相似活动的参与率
   - 目标用户的偏好数据

2. 用户调研验证
   - 小规模问卷调研
   - 用户访谈

3. 逻辑推演验证
   - 目标用户画像
   - 用户核心需求
   - 需求与活动的匹配度
```

## 自检评估矩阵

### 评估表格

| 自检问题 | 评估结果 | 问题分析 | 改进建议 |
|----------|----------|----------|----------|
| Q1：能否达成目标？ | [通过/不通过/待优化] | [分析] | [建议] |
| Q2：是否与业务相关？ | [通过/不通过/待优化] | [分析] | [建议] |
| Q3：用户是否感兴趣？ | [通过/不通过/待优化] | [分析] | [建议] |

### 最终结论

| 结论 | 条件 | 行动 |
|------|------|------|
| **通过** | 三个问题全部通过 | 可进入方案细化阶段 |
| **待优化** | 1-2个问题待优化 | 针对性优化后再次评估 |
| **不通过** | 任何一题不通过 | 重新设计或放弃该创意 |

## 创意优化方向

### Q1不通过的优化

**问题**：创意无法达成目标

**优化方向**：
| 问题 | 优化方法 |
|------|----------|
| 曝光不足 | 加大推广力度/选择更大流量渠道 |
| 转化漏斗损耗大 | 简化参与路径/强化激励 |
| 目标定太高 | 调整目标数字 |
| 玩法与目标不匹配 | 更换活动形式 |

### Q2不通过的优化

**问题**：创意与业务脱节

**优化方法**：
| 问题 | 优化方法 |
|------|----------|
| 产品关联弱 | 强化产品植入/功能露出 |
| 场景不真实 | 重新挖掘用户真实场景 |
| 品牌调性不符 | 调整视觉/文案风格 |
| 价值传递模糊 | 明确核心价值主张 |

### Q3不通过的优化

**问题**：用户不感兴趣

**优化方法**：
| 问题 | 优化方法 |
|------|----------|
| 形式不吸引 | 更换更有趣的形式 |
| 门槛太高 | 降低参与门槛 |
| 收益感知弱 | 增加即时反馈 |
| 时间太长 | 缩短参与时间 |

## 常见创意问题与解决方案

### 问题1：叫好不叫座

**表现**：活动热度高，但转化率低

**原因**：
- 热度来源不是目标用户
- 参与路径太长
- 奖励与用户需求不匹配

**解决方案**：
- 精准定向目标用户
- 缩短转化路径
- 调整奖励设计

### 问题2：叫座不叫好

**表现**：活动转化好，但口碑差/引发负面

**原因**：
- 创意涉嫌虚假宣传
- 奖励设计有坑
- 品牌调性不符

**解决方案**：
- 诚信宣传，如实描述
- 简化规则，避免套路
- 重新审视品牌调性

### 问题3：自嗨型创意

**表现**：内部觉得很好，用户不买单

**原因**：
- 没有真正了解用户需求
- 过于追求创新忽略可行性
- 没有测试验证

**解决方案**：
- 做用户调研
- 小规模测试验证
- 多收集用户反馈
