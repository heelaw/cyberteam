# 用户分层方法论

## 核心概念

用户分层是根据用户特征、行为或价值，将用户划分为不同群体，以便实施差异化运营策略的管理方法。分层的目的是实现"对的资源给对的用户"，最大化运营效率。

## 方法论步骤

### 第一步：明确分层目的

在开始分层之前，必须明确分层的业务目的：
- 提升拉新效率 → 按渠道分层
- 提升留存率 → 按行为分层
- 提升客单价 → 按价值分层
- 精细化运营 → 多维度复合分层

### 第二步：选择分层维度

| 维度类型 | 适用场景 | 代表指标 |
|---------|---------|---------|
| 价值分层 | 高价值用户识别 | LTV、消费金额、订单数 |
| 行为分层 | 精细化运营 | 活跃度、访问频率、功能使用 |
| 生命周期 | 阶段性策略 | 新用户、成长期、成熟期、衰退期 |
| RFM模型 | 电商/零售 | 最近购买、购买频率、消费金额 |

### 第三步：制定分层标准

- 使用可量化的指标
- 确保分层边界清晰无重叠
- 考虑业务可操作性

### 第四步：策略匹配

每个用户层需要对应：
- 运营策略
- 资源投入
- 预期效果

## 关键成功因素

1. **数据质量** - 分层依赖准确的用户数据
2. **动态更新** - 用户分层需定期刷新
3. **策略闭环** - 分层后必须有对应运营动作
4. **成本可控** - 层数不宜过多，建议3-5层

## 常见误区

- 分层维度过多导致执行复杂
- 只分层不行动
- 分层标准未经业务验证
- 忽视层间用户的转化路径
