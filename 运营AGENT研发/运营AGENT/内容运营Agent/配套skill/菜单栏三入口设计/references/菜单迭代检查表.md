# 菜单迭代检查表

## 月度检查清单

### 数据分析

| 检查项 | 合格线 | 不合格处理 |
|--------|--------|------------|
| 各菜单总点击量 | 环比增长 | 分析原因，优化入口 |
| 子菜单点击分布 | Top20%不超80% | 调整菜单命名或内容 |
| 新用户菜单点击率 | >30% | 优化首屏内容 |
| 转化菜单点击率 | >5% | 优化转化路径 |

### 内容检查

| 检查项 | 操作 |
|--------|------|
| 所有跳转链接 |逐一验证有效性 |
| 跳转页面加载速度 | 测试并优化 |
| 过期活动页 | 替换为永久内容 |
| 过期资料 | 更新或删除 |
| 二维码时效性 | 更新群码/个人码 |

## 季度迭代模板

### 迭代步骤

#### Step 1: 数据收集

```
收集周期：过去90天
数据来源：公众号后台-菜单分析
重点关注：
- 点击人数最多的菜单
- 点击转化率高的菜单
- 几乎没有点击的菜单
```

#### Step 2: 问题识别

| 问题类型 | 识别信号 | 处理方式 |
|----------|----------|----------|
| 链接失效 | 打不开/404 | 替换为有效链接 |
| 内容过时 | 活动已结束 | 更新为常规内容 |
| 命名不清晰 | 用户反馈找不到 | 优化菜单命名 |
| 入口太深 | 点击率低 | 前置内容到首层 |
| 价值感低 | 转化率低 | 优化内容描述 |

#### Step 3: 迭代方案

```
基于分析结果，制定迭代方案：

主菜单调整：
- [菜单名] → [新菜单名]
- 调整原因：[xxx]

子菜单调整：
- 新增：[菜单名] → [跳转内容]
- 删除：[菜单名]
- 修改：[菜单名] → [新菜单名]
```

## 特殊时期菜单调整

### 大促/活动期间

| 时期 | 调整策略 |
|------|----------|
| 活动前1周 | 上线活动预告菜单 |
| 活动中 | 活动主菜单置顶 |
| 活动后1周 | 保留活动回顾，缓慢撤出 |

### 内容改版期间

| 时期 | 调整策略 |
|------|----------|
| 改版前 | 提前预告，引导关注 |
| 改版中 | 菜单暂时不变 |
| 改版后 | 更新新内容入口 |

### 功能升级期间

| 时期 | 调整策略 |
|------|----------|
| 上线前 | 菜单入口预热 |
| 上线后 | 突出新功能展示 |
| 稳定后 | 融入常规菜单体系 |
