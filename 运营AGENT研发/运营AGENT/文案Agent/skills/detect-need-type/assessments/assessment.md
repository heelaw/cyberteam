# 考核标准

- 是否覆盖七向证据采集，而不是只看单一来源。
- 是否把每个方向的采集方式说清楚。
- 是否能判定明确、模糊、潜在、混合信号四类状态。
- 是否说明判断依据，而不是只给标签。
- 是否把缺口和下一步动作写出来。
- 是否避免把情绪、搜索词或数据异常误判成明确需求。

## 量化门禁

- 七向证据必须全部出现，缺少任意 2 向则直接不通过。
- 至少 4 向必须有可复述事实，不允许只写抽象判断。
- 必须明确给出明确 / 模糊 / 潜在 / 混合信号之一。
- 必须给出缺口清单和下一步动作，不能只给标签。
- 如果原话、行为、数据冲突，必须写出冲突点，而不是强行统一。
- 追问超过 3 轮仍不清楚，必须停止追问并输出当前最佳判断。

## 一票否决

- 只依据一句话就下结论。
- 把情绪词、搜索词、投诉词直接当最终目标。
- 把数据异常直接当写作指令。
- 没有下一步采样方向。
- 把冲突证据抹平。

## 分数模型

| 项目 | 分值 |
|---|---|
| 七向覆盖 | 2 |
| 可复述事实 | 2 |
| 类型判断 | 1 |
| 冲突处理 | 1 |
| 缺口清单 | 1 |
| 下一步动作 | 1 |
| 追问上限处理 | 1 |
| 红线识别 | 1 |

- 9-10 分：优秀。
- 7-8 分：合格。
- 5-6 分：待改。
- 0-4 分：不合格。

## 测试用例 1：明确需求判断

**输入**：
- 原话："帮我把这条活动页改成能直接引导下单的版本"
- 行为：当前活动页跳出率 78%，点击了活动入口但无人成交

**预期输出**：
- 需求类型：明确需求
- 依据：目标、对象、结果清晰
- 缺口：无重大缺口
- 下一步：进入场景或结构判断

**验收标准**：
- PASS：判为明确需求，且引用原话 + 行为证据
- FAIL：判为模糊或潜在
- FAIL：没有缺口或下一步

## 测试用例 2：模糊需求

**输入**：
- 原话："这个文案我觉得不太对，但我说不出哪里不对"
- 行为：反复修改同一段落，改完又回退
- 反馈：之前给过"没有感觉"的评价

**预期输出**：
- 需求类型：模糊需求
- 依据：问题存在，但目标 / 对象 / 标准不完整
- 缺口：至少 3 条
- 下一步：给出具体追问句式

**验收标准**：
- PASS：判为模糊需求
- FAIL：直接判为明确需求
- FAIL：没有追问句式

## 测试用例 3：潜在需求

**输入**：
- 原话：无
- 行为：一直跳出、反复查看竞品、收藏竞品但没收藏本品
- 数据：本周转化率下降 30%

**预期输出**：
- 需求类型：潜在需求
- 依据：只有现象和行为，没有需求意识
- 缺口：认知唤醒方向
- 下一步：先做认知唤醒

**验收标准**：
- PASS：判为潜在需求
- FAIL：判为明确需求

## 测试用例 4：混合信号

**输入**：
- 原话："我想买，给我发链接"
- 行为：发链接后没有点击，反复看竞品内容
- 对比：问了"A 和 B 哪个更好"

**预期输出**：
- 需求类型：混合信号
- 冲突：原话比行为更热，行为显示仍在比较
- 下一步：按模糊处理，继续补证据

**验收标准**：
- PASS：标出冲突并保留优先级
- FAIL：强行判明确需求

## 测试用例 5：证据不足

**输入**：
- 原话："帮我优化一下"
- 其他六向：全空

**预期输出**：
- 需求类型：信息不足 / 待补证
- 最大缺口：至少 6 向证据
- 下一步：列出补证方向

**验收标准**：
- PASS：明确提示信息不足
- FAIL：凑词输出明确结论

## 测试用例 6：追问上限

**输入**：
- 第 1 轮追问：渠道是什么 → 微信公众号
- 第 2 轮追问：目标受众是谁 → 具体客户
- 第 3 轮追问：想达到什么目标 → 更了解我们

**预期输出**：
- 当前最佳判断：模糊需求
- 最大缺口：行为证据和数据验证
- 下一步：停止追问，改补证据

**验收标准**：
- PASS：第 3 轮后停止追问
- FAIL：继续第 4 轮

## 合格线

- 7 分以上：通过。
- 5-6 分：修后再测。
- 0-4 分：不通过。
