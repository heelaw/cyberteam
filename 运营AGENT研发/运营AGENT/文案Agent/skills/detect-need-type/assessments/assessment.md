# 考核标准

- 是否覆盖七向证据采集，而不是只看单一来源。
- 是否把每个方向的采集方式说清楚。
- 是否能判定明确、模糊、潜在三类需求。
- 是否说明判断依据，而不是只给标签。
- 是否把缺口和下一步动作写出来。
- 是否避免把情绪、搜索词或数据异常误判成明确需求。

## 量化门禁

- 七向证据必须全部出现，缺少任意 2 向则直接不通过。
- 至少 4 向必须有可复述事实，不允许只写抽象判断。
- 必须明确给出“明确 / 模糊 / 潜在”中的一个结果。
- 必须给出缺口清单和下一步动作，不能只给标签。
- 如果原话、行为、数据冲突，必须写出冲突点，而不是强行统一。

## 一票否决补充

- 只依据一句话就下结论。
- 把情绪词、搜索词、投诉词直接当最终目标。
- 把数据异常直接当写作指令。
- 没有下一步采样方向。

# QA测试库

## 测试用例1：明确需求判断

**输入**：
- 原话："帮我把这条活动页改成能直接引导下单的版本"
- 行为：当前活动页跳出率78%，点击了活动入口但无人成交
- 搜索：无（用户已经明确知道自己要什么）
- 对比：无

**七向覆盖**：
- 原话✓ 行为✓ 搜索✗ 对话✗ 对比✗ 反馈✗ 数据✓

**预期输出**：
- 需求类型：明确需求
- 依据：目标（改活动页）、对象（活动页）、结果（直接下单）全部清晰
- 缺口：无重大缺口
- 下一步：直接进入`judge-scenario`或`select-structure`

**验收标准**：
- PASS：判为明确需求，理由包含"目标+对象+结果三要素齐全"
- FAIL：判为模糊需求或潜在需求
- FAIL：理由里没有引用原话和行为证据

## 测试用例2：模糊需求（目标不完整）

**输入**：
- 原话："这个文案我觉得不太对，但我说不出哪里不对"
- 行为：反复修改同一段落，改完又回退，没有一次定稿
- 反馈：之前给过"没有感觉"的评价，但说不出具体哪里

**七向覆盖**：
- 原话✓ 行为✓ 对话✓ 反馈✓ 搜索✗ 对比✗ 数据✗

**预期输出**：
- 需求类型：模糊需求
- 依据：问题存在（文案不对），但目标（变成什么样）、原因（哪里不对）不清晰
- 缺口：①具体哪一段不对②希望改成什么风格③评判标准是什么
- 下一步：追问"能说一下修改后和修改前比，你更不接受哪个版本"

**验收标准**：
- PASS：判为模糊需求，理由包含"问题存在但目标/对象/结果有缺失"
- FAIL：判为明确需求（不能硬判）
- FAIL：没有输出缺口清单
- FAIL：直接跳去写正文

---

## 测试用例3：潜在需求（只有现象）

**输入**：
- 原话：无（用户没说什么）
- 行为：用户一直跳出、反复查看、收藏了竞品但没有收藏本品牌
- 搜索：搜了品类词、竞品词，但搜完离开
- 数据：本周UV持平，但转化率下降30%，竞品同期转化率上升

**七向覆盖**：
- 行为✓ 搜索✓ 数据✓ 原话✗ 对话✗ 对比部分✓ 反馈✗

**预期输出**：
- 需求类型：潜在需求
- 依据：只有行为和数据信号，用户没有主动表达需求意识
- 缺口：①用户为什么离开（不知道）②离开后去了哪里（竞品）③本品牌哪里没打中
- 下一步：先做认知唤醒，不进入正文写作

**验收标准**：
- PASS：判为潜在需求或模糊需求（不能判为明确需求）
- FAIL：判为明确需求
- FAIL：理由里没有引用行为+数据的组合证据
- FAIL：直接输出文案

## 测试用例4：混合信号（冲突证据）

**输入**：
- 原话："我想买，给我发个链接"
- 行为：发了链接后没有点击，反复看其他竞品内容
- 搜索：同时搜索多个品牌、多个SKU
- 对比：问了"XX和YY哪个好"

**七向覆盖**：
- 原话✓ 行为✓ 搜索✓ 对比✓ 其他方向缺失

**预期输出**：
- 需求类型：混合信号（标注冲突）
- 冲突：原话说"想买"，但行为和搜索显示还在比较阶段
- 优先级：行为+搜索优先于原话（裁决规则：行为优先级高于原话）
- 下一步：按模糊需求处理，输出对比型文案或继续追问决策时间节点

**验收标准**：
- PASS：明确标注了冲突证据，理由引用了行为和搜索
- FAIL：忽略冲突，强行判为明确需求
- FAIL：没有标注行为优先级高于原话

## 测试用例5：证据不足（不能强行结案）

**输入**：
- 原话："帮我优化一下"
- 其他六向：全空

**七向覆盖**：
- 7向中只有1向（原话）有证据

**预期输出**：
- 需求类型：信息不足，暂不结案
- 最大缺口：7向中缺少6向证据
- 下一步：明确列出需要补充的证据方向，不允许在证据不足时输出明确判断

**验收标准**：
- PASS：明确输出"信息不足"状态，列出缺失证据方向
- FAIL：凑数填满7向但实际没有真实证据
- FAIL：在1向证据下强行给出明确/模糊/潜在的判定

---

## 测试用例6：追问边界（达到上限）

**输入**：
- 第1轮追问：渠道是什么→用户：微信公众号
- 第2轮追问：目标受众是谁→用户：我们的客户
- 第3轮追问：想达到什么目标→用户：想让他们更了解我们

**七向覆盖**：仍只有对话证据，无行为、无数据

**预期输出**（第3轮仍不清楚时）：
- 当前最佳判断：模糊需求（问题意识有，但不完整）
- 最大缺口：仍缺行为证据、数据验证、对比基准
- 下一步验证方向：提供1-2个可选的假设方向让用户确认
- 强制停止：达到3轮追问上限，不继续打转

**验收标准**：
- PASS：达到上限后强制输出当前判断并停止追问
- FAIL：继续第4轮追问
- FAIL：假装已采集完整7向证据

## 测试用例7：B2B场景（多人需求冲突）

**输入**：
- 甲（使用者）："希望功能更详细一点"
- 乙（采购负责人）："希望操作更简单一点"
- 丙（VP）："希望品牌更有高度"
- 行为：试用数据中，使用者用得多但续费低，采购负责人续费积极但VP批准慢

**七向覆盖**：
- 对话✓（三方对话）行为✓ 数据✓ 对比部分✓ 其他方向缺失

**预期输出**：
- 需求类型：混合信号+角色冲突
- 主线判断：三人需求方向不一致，不能简单合并
- 优先级处理：按拍板者（VP）诉求为主线，以采购负责人和使用者的需求为辅助说明
- 缺口：①三方诉求是否已经穷举②是否有阻拦者未出现③最终决策人是不是VP本人
- 下一步：确认决策链和最终拍板人，不在需求类型上继续打转

**验收标准**：
- PASS：识别出多方需求冲突，给出了主次判断
- FAIL：把三方诉求合并成一个统一需求
- FAIL：没有指出谁是主写对象
