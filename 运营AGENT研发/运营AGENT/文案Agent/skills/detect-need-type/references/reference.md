# 需求类型判断

## 适用场景

- 用户说法还不完整，需要先判断是明确、模糊、潜在还是混合信号。
- 需要先采证据，再决定下一步是不是进入写作。
- 需要从原话、行为、搜索、反馈、对话、对比、数据中找出真正有效的需求线索。

## 先决条件

- 至少有 1 条原话或 1 条真实行为。
- 如果只有情绪，没有目标和动作，不许直接定性。
- 如果原话和行为冲突，先保留冲突，不要硬合并。

## 七向证据采样

### 1. 原话

- 采什么：用户自己怎么说、用什么词、有没有目标和对象。
- 怎么采：保留原句，不要先翻译成“专业语言”。
- 看什么：动词、目标词、对象词、限制词、情绪词。
- 典型信号：
  - 明确："帮我把这条活动页改成能直接引导下单的版本"
  - 模糊："这个文案不太对"
  - 潜在："有没有更好的方式"

### 2. 行为

- 采什么：点击、停留、回退、收藏、放弃、反复比较、反复修改。
- 怎么采：记录动作链，不只记结果。
- 看什么：用户做了什么、做完后停在哪里、是否回头。
- 典型信号：
  - 正在比较：反复看竞品、收藏竞品、回到对比页
  - 不满意：改完回退、一直不定稿

### 3. 搜索

- 采什么：问题词、比较词、解决词、替代词、品牌词。
- 怎么采：不要只看搜索词本身，要看搜索顺序和停留内容。
- 看什么：是否说明用户在找答案、找比较还是找替代方案。
- 典型信号：
  - 明显比较："A 和 B 哪个好"
  - 解决意图："怎么解决..."
  - 认知唤醒："为什么会..."

### 4. 反馈

- 采什么：投诉、抱怨、满意、犹豫、无感、拒绝。
- 怎么采：把情绪词和具体不满点分开记。
- 看什么：反馈是针对内容、功能、体验还是结果。
- 典型信号：
  - "太麻烦了" 要继续追问麻烦在哪里
  - "没感觉" 要追问对比对象是什么

### 5. 对话

- 采什么：连续追问链、答非所问、直接回答、绕开问题。
- 怎么采：保留问答顺序，查看是否一轮比一轮更清晰。
- 看什么：用户是否能把目标、对象、结果说出来。
- 典型信号：
  - 第 1 轮说问题
  - 第 2 轮补对象
  - 第 3 轮补结果

### 6. 对比

- 采什么：默认方案、竞品、人工方案、延后不做方案。
- 怎么采：一定写清楚比较维度，而不是只写谁更好。
- 看什么：用户在比价格、比结果、比风险、比时间还是比体面。
- 典型信号：
  - "A 和 B 哪个更快"
  - "自己做和外包哪个更划算"

### 7. 数据

- 采什么：趋势、异常、波动、转化、留存、流失、点击、跳出。
- 怎么采：必须有指标 + 时间窗口 + 解释假设。
- 看什么：数据是在支撑用户已表达的问题，还是只是个异常。
- 典型信号：
  - 下降趋势 + 行为回退 = 需求可能成立
  - 单点波动 + 无语境 = 只能当线索

## 判断规则

1. 行为和数据一致时，优先级最高。
2. 行为和原话冲突时，优先信行为。
3. 原话和搜索一致，但行为缺失时，只能判为待验证的模糊需求。
4. 反馈和对比只能辅助判断，不能单独定性。
5. 只出现单点情绪、单条评论、单个异常时，不允许判为明确需求。
6. 只有搜索词没有行为，只能判为待验证的模糊或潜在需求。
7. 只有数据异常没有用户语境，只能作为线索，不能单独定性。

## 采样阈值

- 至少 4/7 向出现实证据，才能稳定输出明确结论。
- 3/7 向以下，只能输出当前最佳判断和缺口。
- 七向中至少 2 向有可复述事实，否则必须补证。

## 常见分类

| 类型 | 判断特征 | 下一步 |
|---|---|---|
| 明确需求 | 目标、对象、结果都说清了 | 进入场景或结构判断 |
| 模糊需求 | 问题说清了，但目标、对象或结果不完整 | 追问或补证 |
| 潜在需求 | 只有现象，没有需求意识 | 先做认知唤醒 |
| 混合信号 | 原话、行为、搜索彼此不一致 | 保留冲突，先补证 |

## 冲突优先级

1. 行为 > 原话 > 搜索 > 反馈 > 对比 > 数据单点。
2. 同级冲突时，保留最具体、最可复述的证据。
3. 一旦冲突不能消解，就输出冲突本身，不要硬合并。

## 追问规则

- 最多追问 3 个问题。
- 最多补 2 轮证据。
- 超过上限仍不清楚时，输出当前最佳判断 + 最大缺口 + 下一步验证方向。

## 直接可用的追问句式

- "你最想先解决的是哪一步？"
- "你更不能接受哪种结果？"
- "如果只能先补一条证据，你最希望补哪类？"
- "你是在找方案，还是在确认要不要做？"

## 输出骨架

- 需求类型
- 七向证据地图
- 冲突证据
- 判断依据
- 最大缺口
- 下一步动作

## 失败案例

### 案例1：一句话下结论

- 输入："帮我优化一下"
- 问题：只有动作，没有对象、目标、结果和语境。

### 案例2：把情绪当需求

- 输入："太烦了"
- 问题：情绪存在，但不知道烦在哪里。

### 案例3：把搜索词当最终意图

- 输入：搜了很多竞品词
- 问题：比较行为存在，但还不能直接等于购买意图。

### 案例4：把数据异常当命令

- 输入：转化率掉了
- 问题：只能说明异常，不代表具体文案方向。

## 红线

- 不能只凭一句话就下结论。
- 不能把情绪词直接当最终目标。
- 不能把搜索词直接当交付指令。
- 不能把数据异常直接当写作需求。
- 不能在证据不足时伪装成明确需求。

## 新人校验

- 新人能不能区分“有人说”和“有人做”。
- 新人能不能知道缺哪类证据就补哪类证据。
- 新人能不能知道什么时候该继续追问，什么时候该停。
- 新人能不能知道什么情况下绝对不能直接写正文。
- 新人能不能把冲突证据原样保留，而不是强行合并。

## 参考输出样例

### 明确需求

- 需求类型：明确需求
- 依据：目标、对象、结果齐全
- 缺口：无重大缺口
- 下一步：进入场景或结构判断

### 模糊需求

- 需求类型：模糊需求
- 依据：问题存在，但结果不完整
- 缺口：目标、对象、评判标准
- 下一步：追问或补证

### 潜在需求

- 需求类型：潜在需求
- 依据：只有现象，没有需求意识
- 缺口：用户对问题的命名和结果认知
- 下一步：先做认知唤醒
