# 正向卖点考核

## 基础门禁

以下条件必须同时满足，缺一项就不通过：

1. 七个方向都过了一遍。
2. 每个方向至少有 1 个候选。
3. 主卖点能回到至少 1 条明确证据。
4. 主卖点和辅助卖点有主次区分。
5. 输出里有证据缺口，不用想象补洞。
6. 没有把功能名、优惠、品牌口号直接当卖点。

## 分项考核

### 功能独特性

| 检查项 | 通过标准 | 不通过表现 |
|---|---|---|
| 差异识别 | 说明和默认方案/竞品哪里不同 | 只写功能名 |
| 用户感知 | 翻译成用户能感受到的变化 | 只写技术参数 |
| 比较基准 | 明确说明和谁比 | 没有对比对象 |
| 候选分级 | 区分主候选和辅助候选 | 所有差异都当主卖点 |

### 效果承诺

| 检查项 | 通过标准 | 不通过表现 |
|---|---|---|
| 结果层级 | 写出最终结果或明确降级到中间/过程结果 | 只写“更好”“更快” |
| 数据或案例 | 有数字、案例或明确来源 | 只有愿望句 |
| 前提条件 | 写清适用条件 | 结果写得像无条件成立 |
| 可验证性 | 用户能自己判断是否成立 | 只能由你来解释 |

### 解决问题

| 检查项 | 通过标准 | 不通过表现 |
|---|---|---|
| 原话保留 | 有用户原话或可追溯反馈 | 全部是二次翻译 |
| 场景清楚 | 写清谁、何时、做什么、卡哪步 | 只写“效率低” |
| 代价明确 | 写出不解决会损失什么 | 只写“很严重” |
| 产品匹配 | 产品真的能接住问题层级 | 写了产品解决不了的问题 |

### 用户身份

| 检查项 | 通过标准 | 不通过表现 |
|---|---|---|
| 身份不是标签 | 写的是场景里的身份感 | 只写职业名 |
| 用户语言 | 有用户自己的表达方式 | 全用企业视角 |
| 角色拆分 | 多角色已拆开写 | 混成一个身份 |
| 产品一致 | 身份表达和产品能力一致 | 身份太大、产品太小 |

### 价格优势

| 检查项 | 通过标准 | 不通过表现 |
|---|---|---|
| 比较对象 | 明确写和谁比 | 只说便宜 |
| 成本计算 | 至少说明怎么算的 | 没有计算前提 |
| 效果前提 | 便宜同时有用 | 便宜但无效 |
| 公平比较 | 比较条件一致 | 拿促销价对正价 |

### 服务保障

| 检查项 | 通过标准 | 不通过表现 |
|---|---|---|
| 范围边界 | 写清楚包什么、不包什么 | 只写“服务好” |
| 时效责任 | 写清响应时间和责任人 | 没有可执行信息 |
| 兑现能力 | 承诺能被实际兑现 | 承诺大于能力 |
| 可感知性 | 用户能感知到保障 | 只有合同文字，用户感受不到 |

### 情感价值

| 检查项 | 通过标准 | 不通过表现 |
|---|---|---|
| 用户来源 | 有用户原话或真实表达 | 全是品牌口号 |
| 场景落地 | 情绪落在具体场景里 | 只写空泛情绪词 |
| 产品路径 | 情绪和产品能力连得上 | 情绪和功能无关 |
| 差异化 | 不是所有竞品都能写 | 竞品都能抄的情绪 |

## 证据门槛

- 主卖点至少要有 1 条★★★★★ 证据。
- 主卖点最好再配 1 条独立来源的辅助证据。
- 如果只有 1 条原话，没有第二来源，只能标成“候选主卖点”，不能假装已完成闭环。
- 任何推断性内容都必须标成待验证。

## 一票否决

1. 功能名直接等于卖点。
2. 优惠折扣直接当长期卖点。
3. 品牌口号直接当证据。
4. 七向没有全部展开。
5. 没有主卖点和辅助卖点的区分。
6. 没有证据缺口标注。
7. 输出里只有结论，没有比较基准。

## 分级标准

### 合格

- 七向全部展开。
- 每个方向至少 1 个候选。
- 主卖点有明确证据。
- 证据缺口有标注。

### 良好

在“合格”基础上，再满足：

- 主卖点能自然转成用户语言。
- 至少有 2 类不同来源的证据。
- 能清楚说明为什么选这个方向而不是别的方向。
- 至少有 1 个方向写出明确的降级回退。

### 优秀

在“良好”基础上，再满足：

- 七向都有明确的采样动作。
- 每个方向都写了至少 1 条失败案例。
- 主卖点有主证据、辅助证据和缺口说明。
- 输出能直接交给下一步 Skill 接手。

## 新人实操题

### 题 1：功能独特性

**输入**：产品支持自动把会议录音转成待办。

**合格答案应该有**：

- 说清楚这个功能和人工整理的差异。
- 说清楚用户为什么在意。
- 说清楚可感知结果是什么。

### 题 2：效果承诺

**输入**：产品能减少整理时间。

**合格答案应该有**：

- 写出具体减少到多少。
- 或者明确写这是候选承诺，需要数据验证。
- 不能只写“更高效”。

### 题 3：解决问题

**输入**：用户说“每次会后都不知道谁该做什么”。

**合格答案应该有**：

- 场景。
- 阻碍。
- 代价。
- 产品能接住的层级。

### 题 4：主卖点选择

**输入**：功能独特性、效果承诺、服务保障都能成立。

**合格答案应该有**：

- 说明主卖点选哪一个。
- 说明为什么先讲它。
- 说明为什么另外两个降为辅助。

## 交付判断

- 如果主卖点、辅助卖点、证据来源、证据缺口都齐了，算通过。
- 如果只是把七个方向列出来，但没有判断和取舍，不通过。
- 如果看起来完整，但没有证据链，不通过。
- 如果能让新人照着继续做下一步，才算真正过关。
