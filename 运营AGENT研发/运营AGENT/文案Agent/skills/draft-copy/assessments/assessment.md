# 考核标准

- 是否覆盖核心卖点。
- 是否符合目标用户和场景。
- 是否具备明确行动号召。
- 是否保留待确认信息和限制条件。
- 是否能看出结构而不是一坨文字。
- 是否明确这是初稿而不是最终稿。
- 是否遵守平台触点规则。

## 量化门禁

- 必须有明确主张。
- 必须有至少 1 个用户能感知的结果表达。
- 必须有 1 个主动作，不能同时催三步。
- 必须列出待确认信息或缺口。
- 必须列出至少 1 条限制或适用条件。
- 必须能看出结构顺序。
- 不能在卖点、场景、结构不清时写成完整成稿。
- 不能把缺口藏在正文里。

## 一票否决

- 卖点未定就写正文。
- 场景未定就写长度和 CTA。
- 把口号当正文。
- 同时给多个互相冲突的动作。
- 把限制条件写没了。
- 把初稿写成最终稿。

## 分数模型

| 项目 | 分值 |
|---|---|
| 主张清晰 | 2 |
| 结构清晰 | 2 |
| 证据 / 理由 | 2 |
| 单一 CTA | 1 |
| 限制条件 | 1 |
| 缺口保留 | 1 |
| 平台适配 | 1 |

- 9-10 分：优秀。
- 7-8 分：合格。
- 5-6 分：待改。
- 0-4 分：不合格。

## 测试用例

### 用例 1：短触点

**输入**：抖音 15 秒，目标是引流。

**预期**：只有一个主张、一个动作、一个最强证据。

### 用例 2：中触点

**输入**：小红书图文，目标是收藏和评论。

**预期**：有场景、体验、对比和中强 CTA。

### 用例 3：长触点

**输入**：公众号长文，目标是预约演示。

**预期**：有解释、证据链、反对意见和边界。

### 用例 4：详情页

**输入**：产品详情页首屏。

**预期**：先判断，后证据，最后动作。

### 用例 5：缺口保留

**输入**：没有数据，但有卖点和场景。

**预期**：必须把数据标成待确认，不得硬编。

## 通过标准

- 主张明确。
- CTA 单一。
- 缺口显式。
- 限制条件不丢。
- 平台节奏对。
