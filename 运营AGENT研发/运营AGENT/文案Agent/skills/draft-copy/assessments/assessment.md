# 考核标准

- 是否覆盖核心卖点。
- 是否符合目标用户和场景。
- 是否具备明确行动号召。
- 是否保留待确认信息和限制条件。
- 是否能看出结构而不是一坨文字。
- 是否明确这是初稿而不是最终稿。

## 量化门禁

- 必须有明确主张。
- 必须有至少 1 个用户能感知的结果表达。
- 必须有 1 个主动作，不能同时催三步。
- 必须列出待确认信息或缺口。
- 必须列出至少 1 条限制或适用条件。
- 必须能看出结构顺序。
- 不能在卖点、场景、结构不清时写成完整成稿。
- 不能把缺口藏在正文里。

## 一票否决补充

- 卖点未定就写正文。
- 场景未定就写长度和 CTA。
- 把口号当正文。
- 同时给多个互相冲突的动作。
- 把限制条件写没了。
