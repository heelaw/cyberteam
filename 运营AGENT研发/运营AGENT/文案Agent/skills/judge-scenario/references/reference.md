# 场景判断

## 适用场景

- 已经知道目标用户和卖点，但还不知道该用什么长度、什么平台、什么 CTA。
- 需要把抽象的“短 / 中 / 长”落到具体平台与触点。
- 需要判断主渠道和辅渠道的拆分方式。

## 先决条件

- 先有用户状态和场景判断，再选结构。
- 如果卖点或证据还不清，先回上游。
- 如果平台本身限制很强，先服从平台，不服从内容完整性。

## 四个输入

1. 渠道 / 介质。
2. 接收状态：被动刷到、主动搜索、主动咨询、已接近购买。
3. 阅读窗口：3 秒、15 秒、1 分钟、3 分钟、10 分钟。
4. 说服目标：唤醒、解释、对比、推进、成交、复访。

## 平台预设

| 平台 / 触点 | 阅读窗口 | 表达重心 | CTA 强度 |
|---|---|---|---|
| 海报 / 广告位 | 3-8 秒 | 一个主张 + 一个利益点 | 很强，但只能一个动作 |
| 抖音短视频 | 15-60 秒 | 钩子、冲突、结果、证据 | 强，前 3 秒必须有入口 |
| 小红书图文 | 20-90 秒 | 场景、体验、细节、对比 | 中强，先共鸣后动作 |
| 公众号文章 | 2-6 分钟 | 逻辑、解释、案例、边界 | 中等，先理解后转化 |
| 详情页首屏 | 5-15 秒 | 主卖点、核心证据、下一步 | 强，首屏要完成判断 |
| 详情页中段 | 30-120 秒 | 证据展开、FAQ、反对意见 | 中等，服务理解和信任 |
| 私信 / 销售话术 | 对话式 | 问题、判断、定制回应 | 按对方状态动态调整 |
| 长文 / 教育内容 | 3-10 分钟 | 认知重构、结构化解释 | 弱到中，先建立信任 |

## 场景分层

### 短触点

- 海报、广告位、详情页首屏、短视频前 3 秒。
- 重点：先给判断，再给动作。
- 禁忌：长解释、长背景、一次塞多个卖点。

### 中触点

- 小红书正文、公众号前半段、详情页中段、弱意愿私信。
- 重点：场景、体验、证据、共鸣。
- 禁忌：直接强催购买，或只讲口号。

### 长触点

- 公众号深度文、长文、教育内容、决策前深度阅读。
- 重点：逻辑、证据链、反对意见、边界。
- 禁忌：把长触点写成广告语。

### 决策触点

- 私信强意愿、详情页尾部、报价页、预约页。
- 重点：信任、风险、动作。
- 禁忌：只做认知教育，不推进动作。

## CTA 强度映射

| 强度 | 适用场景 | 例子 |
|---|---|---|
| 很强 | 海报、广告位、首屏 | 立即领取、立即预约、点击查看 |
| 强 | 抖音、详情页首屏、强意愿私信 | 进直播间、看方案、发地址 |
| 中强 | 小红书、部分详情页 | 收藏、评论、关注、私信 |
| 中等 | 公众号、长文 | 预约、领取资料、下载手册 |
| 弱 | 认知唤醒 | 先看案例、先了解、先评论 |

## 混合渠道规则

- 首先判断主渠道，再判断辅渠道。
- 首屏 / 第一眼必须按主渠道的最短窗口写。
- 正文可以按辅渠道展开，但不能反向压坏主渠道入口。
- 同一素材同时服务唤起和促成时，先做最短入口，再做中段证据。

## 平台级规则

### 抖音

- 前 3 秒必须给可感知入口。
- 钩子 > 冲突 > 结果 > 动作。
- CTA 不要等到最后才出现。

### 小红书

- 封面标题和前 30 字最重要。
- 场景感、细节感、个人体验必须前置。
- CTA 常用收藏、评论、私信，不要一上来硬卖。

### 公众号

- 允许先解释，再证明，再推进。
- 适合放案例、FAQ、边界和反对意见。
- 标题决定打开率，前三段决定继续看。

### 详情页

- 首屏先判断值不值得看。
- 中段再展开证据和反对意见。
- 必须兼顾信任和风险，不许只说利益。

### 私信

- 强意愿：直接给方案和动作。
- 弱意愿：先钩子，再提问，不要立刻逼单。

## 细化判断表

| 维度 | 该看什么 | 低风险写法 | 高风险写法 |
|---|---|---|---|
| 接收状态 | 是被动刷到、主动搜索、主动咨询，还是接近购买 | 先按最强意愿写入口 | 把低意愿场景写成逼单场景 |
| 阅读窗口 | 3 秒、15 秒、1 分钟、3 分钟、10 分钟 | 让内容长度服从窗口 | 先写满再裁剪 |
| 平台限制 | 字数、首屏、封面、评论习惯 | 先保入口，再做展开 | 把长解释塞进短入口 |
| 说服目标 | 唤醒、解释、对比、推进、成交、复访 | 先确认目标再定 CTA | CTA 和目标打架 |

## 混合场景拆分例

### 例 1：朋友圈 + 详情页

- 朋友圈是短触点，只写判断、利益和一个动作。
- 详情页可以展开证据和反对意见。
- 两边共用一个卖点，但入口长度不同。

### 例 2：小红书种草 + 私信成交

- 小红书前半段先做共鸣和体验。
- 私信里再补价格、边界和动作。
- 不能把私信的强转化语气直接搬到小红书正文里。

### 例 3：强意愿询价 + 深度解释

- 如果用户已经问价，就优先决策触点。
- 只补和成交有关的风险、边界和下一步。
- 不能再从头讲故事。

## 场景误判修正

- 看到“公众号”不等于长触点，要看文章目标是教育还是转化。
- 看到“详情页”不等于强转化，要看用户是否已经接近行动。
- 看到“小红书”不等于只能中触点，若是种草型短图，仍要压缩入口。
- 看到“私信”不等于可以直接逼单，要先看对方是否主动表达需求。

## 场景输出模板

- 场景标签：平台 + 触点 + 接收状态
- 阅读窗口：具体到秒或分钟
- 表达重心：先写什么、后写什么
- CTA 强度：弱 / 中 / 强 / 很强
- 首屏规则：是否要单独处理
- 混合渠道拆分：主渠道和辅渠道如何分工


## 典型判断

| 场景 | 结论 | 原因 |
|---|---|---|
| 抖音信息流 | 短触点 | 被动刷到，注意力极短 |
| 小红书图文 | 中触点 | 需要场景和体验 |
| 公众号深度文 | 长触点 | 允许完整解释 |
| 详情页首屏 | 短触点 | 3-15 秒判断窗口 |
| 详情页中段 | 中触点 | 开始进入证据和反对意见 |
| 强意愿私信 | 决策触点 | 已接近行动 |

## 常见误判

- 把平台当成统一长度，不拆首屏和正文。
- 把短触点写成长解释。
- 把长触点写成口号。
- 不看接收状态，直接套 CTA。
- 混合渠道不拆主辅。

## 输出骨架

- 场景标签
- 平台预设
- 长度建议
- CTA 强度
- 表达节奏
- 是否要拆首屏 / 中段 / 正文
- 混合渠道拆分建议

## 失败案例

### 案例 1：短场景写长解释

- 原文：海报里塞 5 屏解释。
- 问题：阅读窗口只有几秒。

### 案例 2：强意愿还做教育

- 原文：用户已经问价，还在讲品牌故事。
- 问题：错过决策窗口。

### 案例 3：弱意愿直接催单

- 原文：刚加好友就问要不要买。
- 问题：压力太大。

### 案例 4：混合渠道不拆分

- 原文：朋友圈 + 详情页同一写法。
- 问题：主渠道入口被拉长。

## 红线

- 不能只说短 / 中 / 长，必须落到平台或触点。
- 不能在短触点里写长解释。
- 不能在长决策里只写口号。
- 不能不看接收状态就定长度。
- 不能混合渠道时不拆主渠道和辅渠道。

## 新人校验

- 新人能不能判断平台和长度。
- 新人能不能写出首屏 / 中段 / 正文的不同重心。
- 新人能不能控制 CTA 强度。
- 新人能不能识别主渠道和辅渠道。
