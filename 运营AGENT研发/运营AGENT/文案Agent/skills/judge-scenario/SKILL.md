# 场景判断器

## 作用

判断场景强度和承载方式，不负责写正文。

它要回答的是：这个内容现在适合短写、长写，还是先压缩入口再展开。

## 启动前

- 先有渠道或触点，再做场景判断。
- 如果连平台都不知道，先回去补输入。

## 判断顺序

1. 先定渠道和触点。
2. 再定用户接收时的状态。
3. 再定阅读时长和注意力窗口。
4. 最后定是唤起、说明还是促成。

## SOP

1. 判断渠道和触点。
2. 判断用户接收时的状态。
3. 判断阅读时长和注意力窗口。
4. 判断是唤起、说明还是促成。
5. 映射到短触点、中触点、长触点或决策触点。
6. 结合平台预设，输出 CTA 强度和表达节奏。

## 场景分层

- 短触点：抓注意，先给结论和动作。
- 中触点：讲问题、方案和证据。
- 长触点：讲逻辑、证据和反对意见。
- 决策触点：讲信任、风险和行动。

## 平台预设

- 小红书：默认中触点，先共鸣再行动。
- 抖音：默认短触点，前几秒先给钩子。
- 公众号：默认中长触点，允许先解释再推进。
- 详情页：首屏短触点，中段中触点。
- 私信/销售：按对话状态动态切换。

## 交付骨架

- 场景标签
- 平台预设
- 长度建议
- 表达节奏建议
- CTA 强度
- 混合渠道处理建议

## 新人校验

- 新人能不能先定平台，再定长度。
- 新人能不能分清“能转化”和“适合直接转化”。
- 新人能不能处理主渠道和辅渠道同时存在的情况。
- 新人能不能知道短场景不能塞长解释。

## 绝对不能

- 不能把所有场景都当强转化场景。
- 不能在短触点里塞长解释。
- 不能在长决策里只写口号。
- 不能忽略用户接收状态。
- 不能忽略平台字数和阅读时间。
