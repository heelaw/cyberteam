# 场景判断器

## 作用

判断场景强度和承载方式，不负责写正文。

它要回答的是：这个内容现在适合短写、长写，还是先压缩入口再展开。

## 启动前

- 先有渠道或触点，再做场景判断。
- 如果连平台都不知道，先回去补输入。

## 判断顺序

1. 先定渠道和触点。
2. 再定用户接收时的状态。
3. 再定阅读时长和注意力窗口。
4. 最后定是唤起、说明还是促成。

## SOP

1. 判断渠道和触点。
2. 判断用户接收时的状态。
3. 判断阅读时长和注意力窗口。
4. 判断是唤起、说明还是促成。
5. 映射到短触点、中触点、长触点或决策触点。
6. 结合平台预设，输出 CTA 强度和表达节奏。

## 场景分层

- 短触点：抓注意，先给结论和动作。
- 中触点：讲问题、方案和证据。
- 长触点：讲逻辑、证据和反对意见。
- 决策触点：讲信任、风险和行动。

## 平台预设

- 小红书：默认中触点，先共鸣再行动。
- 抖音：默认短触点，前几秒先给钩子。
- 公众号：默认中长触点，允许先解释再推进。
- 详情页：首屏短触点，中段中触点。
- 私信/销售：按对话状态动态切换。

## 细化判断

### 短触点

- 看什么：用户只给几秒注意力，只能判断入口，不适合展开。
- 怎么判：标题、封面、首句、首屏是否能独立成立。
- 常见信号：海报、广告位、短视频前 3 秒、详情页首屏。
- 不能做：长背景、多个卖点、复杂解释。

### 中触点

- 看什么：用户愿意多看一点，但还没进入深度论证。
- 怎么判：能不能用场景、体验、对比和一两个证据把人留下。
- 常见信号：小红书正文、公众号前半段、详情页中段、弱意愿私信。
- 不能做：一上来硬逼转化，或只讲口号不讲内容。

### 长触点

- 看什么：用户愿意读逻辑链和证据链。
- 怎么判：是否需要解释、反对意见、边界和完整证据。
- 常见信号：公众号深度文、长文、教育内容、方案说明。
- 不能做：把长触点写成短广告语。

### 决策触点

- 看什么：用户已经接近行动，重点是信任、风险和下一步。
- 怎么判：能不能直接给动作、报价、预约、确认。
- 常见信号：强意愿私信、报价页、预约页、尾部 CTA。
- 不能做：继续大段教育，拖慢决策。

### 混合触点处理

- 标题、封面、首屏按最短窗口处理，正文按更长窗口展开。
- 同一内容既要唤起又要促成时，先保入口，再补证据。
- 朋友圈、详情页、公众号常常是混合触点，不能整篇只按一个长度写。
- 如果入口和正文节奏冲突，以入口优先。

### 平台怎么落地

- 抖音：前 3 秒先给钩子，再给冲突和结果。
- 小红书：前 30 字先写场景，再写体验和细节。
- 公众号：先解释，再证明，再推进。
- 详情页：首屏先判断，中段再证据，尾部再动作。
- 海报：只保留一个主张，不塞第二个卖点。

## 新人校验

- 新人能不能先定平台，再定长度。
- 新人能不能分清“能转化”和“适合直接转化”。
- 新人能不能处理主渠道和辅渠道同时存在的情况。
- 新人能不能知道短场景不能塞长解释。

## 绝对不能

- 不能把所有场景都当强转化场景。
- 不能在短触点里塞长解释。
- 不能在长决策里只写口号。
- 不能忽略用户接收状态。
- 不能忽略平台字数和阅读时间。
