# 考核标准

- 是否识别渠道与介质。
- 是否识别阅读时长和用户状态。
- 是否能指导长短文案选择。
- 是否明确平台预设、CTA 强弱和证据深度。
- 是否能拆分首屏 / 中段 / 正文。
- 是否能处理混合渠道。

## 量化门禁

- 必须明确平台或触点类型。
- 必须明确阅读时长或注意力窗口。
- 必须给出 CTA 强度判断。
- 必须指出证据深度要求。
- 必须给出平台预设或混合渠道拆分建议。
- 不能只说短 / 中 / 长，必须落到平台或触点。
- 不能忽略首屏和正文的差异。

## 一票否决

- 在短触点里写长解释。
- 在长决策里只写口号。
- 不看用户接收状态就定长度。
- 混合渠道时不拆分主渠道和辅渠道。
- 详情页首屏和中段不分开写。

## 分数模型

| 项目 | 分值 |
|---|---|
| 平台识别 | 2 |
| 阅读窗口 | 2 |
| CTA 强度 | 2 |
| 证据深度 | 1 |
| 首屏 / 正文拆分 | 1 |
| 混合渠道处理 | 1 |
| 新人可执行 | 1 |

- 9-10 分：优秀。
- 7-8 分：合格。
- 5-6 分：待改。
- 0-4 分：不合格。

## 测试用例

### 用例 1：抖音短视频

**输入**：抖音短视频，用户被动刷到，目标是引流。

**预期**：短触点，前 3 秒必须有钩子，CTA 强。

### 用例 2：小红书图文

**输入**：小红书图文，用户主动浏览，目标是种草。

**预期**：中触点，先共鸣后动作，CTA 中强。

### 用例 3：公众号深度文

**输入**：公众号，用户主动打开，目标是预约演示。

**预期**：中长触点，允许解释和案例，CTA 中等。

### 用例 4：详情页首屏

**输入**：产品详情页首屏，用户刚进入。

**预期**：短触点，先判断再给证据，CTA 强。

### 用例 5：混合渠道

**输入**：朋友圈 + 详情页同发。

**预期**：主渠道短触点，辅渠道中触点，首屏按最短窗口写。

### 用例 6：强意愿私信

**输入**：用户主动问价并问规格。

**预期**：决策触点，直接给方案和动作，不做长教育。

## 通过标准

- 平台正确。
- 长度正确。
- CTA 正确。
- 证据深度正确。
- 混合渠道处理正确。
- 首屏和正文拆分正确。
