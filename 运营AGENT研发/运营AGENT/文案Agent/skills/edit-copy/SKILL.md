# 文案删改器

## 作用

修正文案，不重新发明卖点。

这个 Skill 先判断问题类型，再决定是删、补、调序，还是局部强化。

## 启动前

- 必须有一版可编辑文案。
- 如果没有初稿，先回 `draft-copy`。

## 问题类型识别

动手之前必须先判断是哪种问题。问题类型判断错了，后面全改错。

| 问题类型 | 识别特征 | 最小修改单位 |
|---|---|---|
| 重复 | 同一意思出现2次以上；或不同段落说了同一件事 | 句/句群 |
| 空话 | 只有态度没有信息；只有形容词没有事实 | 词/句 |
| 抽象 | 概念词大于信息量；"好/强/快/专业"无具体场景 | 词/句 |
| 顺序错 | 阅读主线不顺畅；结论被埋在第二段以后 | 句群/段落 |
| 动作弱 | CTA模糊；动作不具体；动作和用户状态不匹配 | 句/句群 |
| 证据缺 | 承诺无边；结论无边；没有数据/案例/过程 | 句/句群 |
| 边界缺 | 没有适用条件；没有说明不适用情况 | 句/句群 |
| CTA错位 | 用户还在认知阶段就催购买；用户已接近决策还在铺背景 | 句群/段落 |

**优先级判断**：重复 > 空话 > 抽象 > 动作弱 > 顺序错 > 证据缺 > 边界缺 > CTA错位。
先处理影响理解的问题（重复/空话/抽象），再处理影响转化的问题（动作/顺序/CTA）。

## 判断顺序

1. 先判断问题类型（见上方问题类型识别表）。
2. 再判断最小修改单位是词、句、句群还是段落。
3. 再确认主卖点、证据、动作和限制是否要保留。
4. 最后决定删、补、调序还是局部强化。

## SOP

1. 先判断问题类型（不能跳步）。
2. 再判断最小修改单位是词、句、句群还是段落。
3. 先保主卖点、证据、动作和限制条件（这四项是红线，不能动）。
4. 再删重复、空话和抽象（按优先级处理，不要同时改太多类型）。
5. 需要时补证据，但不补新承诺（补证据只能用已有信息，不能编新卖点）。
6. 再调顺序，让阅读主线更顺（调序不能改变说服逻辑）。
7. 输出修改版、修改点、保留项和未解决项。

## 常见问题

- 重复太多 → 找同义重复，保留最短最清楚的一句，其余删或合并。
- 空话太多 → 找态度词，换成信息词；找无对象的形容，换成有结果的事实。
- 抽象太多 → 找概念词，换成场景/数字/动作；找"全面/专业/优质"，换成具体描述。
- 顺序不对 → 先定主线（结论优先还是问题优先），再调句群顺序。
- 动作太弱 → 换具体动词；换成本收益描述；让动作和用户当前状态匹配。
- 证据不够 → 补数据、案例、过程；补不上则标为"待确认"而非硬编。
- 边界缺失 → 补适用条件和限制；不能因为边界碍事就删掉。
- CTA错位 → 判断用户现在在哪个阶段（认知/考虑/决策），动作必须匹配阶段。

## 修改边界（红线）

以下四条是硬边界，越过任意一条就是越界，不是优化，是破坏。

### 红线1：不能改事实

- 原文有具体数字，改后不能改数字。
- 原文有具体场景/案例，改后不能改场景/案例。
- 原文有限制条件，改后不能删限制条件。
- 识别方式：凡是有具体信息的句子，删改前先自问"这句话的信息还能找到吗"，找不到就不能动。

### 红线2：不能补新承诺

- 原文没有数据，改后不能自己加数据。
- 原文没有说过的话，改后不能自己说。
- 原文没有覆盖的场景，改后不能自己扩展。
- 识别方式：凡是要补充信息，先确认"这个信息来自原文还是来自我自己的判断"，来自判断的一律标"待确认"。

### 红线3：不能换主线

- 主卖点是什么就还是什么，不能换成别的卖点。
- 说服逻辑是什么就还是什么，不能把"效果"路线换成"价格"路线。
- 识别方式：改完之后和原文的目标受众和场景描述是否还一致，不一致就是换了主线。

### 红线4：不能把关键条件压没

- 适用条件、限制条件、价格门槛、适用规模等信息，压缩时不能压没。
- 识别方式：改完之后把关键条件遮住，看用户还能不能判断自己适不适合，不适合就是压没了。

### 常见越界形式（反面教材）

| 越界形式 | 原文 | 越界改法 | 正确改法 |
|---|---|---|---|
| 改数字 | "已服务200家客户" | "已服务千家客户" | 保留数字，或改为"200+客户" |
| 编数据 | "转化率较高" | "转化率提升40%" | 改为"转化率有提升（数据待确认）" |
| 换卖点 | "速度快" | "价格更低" | 保留速度优势，只优化表达 |
| 删边界 | "适合X规模企业" | 删掉这句 | 保留，或压缩为"X规模企业首选" |
| 替案例 | "某制造业客户" | "某头部客户" | 保留原描述，不擅自升级 |

## 先判这次修改的形状

先判断这次改动到底是哪一种，形状判断错了，后面就会改偏。

- 删除型：主要是去掉重复、空话、抽象。
- 压缩型：主要是把一大段压成更短但不丢信息。
- 替换型：主要是把抽象词换成事实词。
- 调序型：主要是调整结论、理由、证据、动作的顺序。
- 强化型：主要是把动作、证据、边界说得更清楚。
- 混合型：同一篇里同时有两种以上问题，按优先级分批处理。

## 先救哪一句

修改时先救最危险的句子，而不是先碰最顺眼的句子。

1. 硬承诺。
2. CTA。
3. 限制条件。
4. 证据句。
5. 主卖点句。
6. 其余修辞。

## 局部修复法

- 一句里同时有事实和修辞，先保事实，再删修辞。
- 一段里同时有证据和口号，先保证据，再压缩口号。
- 如果一段里必须删掉一半，先删信息重复部分，不删边界和动作。
- 如果改完内容更短但更空，说明只做了减法，没做优化。

## 常见修改场景

- 口号太多：把“专业、全面、强大”换成能被验证的事实。
- 证据太少：先补证据位置，不要自己补新结论。
- 动作太多：只保留一个主动作，其余都降级到可选动作或说明。
- 逻辑太乱：先把结论前置，再按理由、证据、动作排队。
- 边界被吞：把适用条件单独拉出来，别埋进一句长话里。

## 修改验收

- 改完后，原来的主线还在。
- 改完后，没有新增卖点或新承诺。
- 改完后，限制条件仍然可见。
- 改完后，读者能看出这是修文，不是重写。

## 新人校验

- 新人能不能先识别问题，再动手。
- 新人能不能知道删改和重写不是一回事。
- 新人能不能保住主卖点、证据、动作和限制。
- 新人能不能说清楚这次改动的最小范围。

## 绝对不能

- 不能把删改变成重写。
- 不能删掉关键证据和动作。
- 不能新增卖点。
- 不能把修辞当改进。
- 不能只改顺序不改逻辑。
- 不能为了顺口而加新承诺。
