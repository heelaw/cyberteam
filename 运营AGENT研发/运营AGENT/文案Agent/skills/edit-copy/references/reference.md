# 删改优化

## 适用场景

- 已有初稿，需要更像人话、更像成交文案、更像业务文案。
- 需要压缩、强化或提高清晰度。
- 不需要重新发明卖点。

## 改写前先看 7 件事

1. 主卖点有没有丢。
2. 证据有没有丢。
3. 动作有没有丢。
4. 限制条件有没有丢。
5. 用户是否还是原来的用户。
6. 场景是否还是原来的场景。
7. 这次修改是删、补、调序，还是局部强化。

## 问题类型识别

| 问题类型 | 识别特征 | 最小修改单位 |
|---|---|---|
| 重复 | 同一意思出现 2 次以上 | 句 / 句群 |
| 空话 | 只有态度没有信息 | 词 / 句 |
| 抽象 | 概念词大于信息量 | 词 / 句 |
| 顺序错 | 阅读主线不顺畅 | 句群 / 段落 |
| 动作弱 | CTA 模糊、不具体 | 句 / 句群 |
| 证据缺 | 承诺无边，缺事实支撑 | 句 / 句群 |
| 边界缺 | 没有适用条件 | 句 / 句群 |
| CTA 错位 | 阶段不匹配 | 句群 / 段落 |

## 判断顺序

1. 先判断问题类型。
2. 再判断最小修改单位。
3. 再确认主卖点、证据、动作和限制是否要保留。
4. 最后决定删、补、调序还是局部强化。

## 常见处理顺序

1. 删重复。
2. 删空话。
3. 删抽象。
4. 补证据。
5. 强动作。
6. 调顺序。

## 逐项怎么改

### 删重复

- 同一意思说了两次以上，保留最短最清楚的那句。
- 如果重复在不同段落，只保留一次。
- 如果重复是为了强调，保留最强一次即可，不要堆叠。

### 删空话

- 删除只有态度没有信息的词。
- 删除没有对象、没有结果、没有证据的形容。
- 删除“很厉害、很专业、很全面”这类不落地的套话。

### 删抽象

- 把“好、强、快、专业、极致”换成事实。
- 把概念改成场景、动作、结果。
- 把“全面方案”改成“具体步骤/机制/结果”。

### 补证据

- 没有证据的承诺要补出处。
- 没有数据就补例子，没有例子就补过程说明。
- 如果补不上证据，就把它留在待确认项里。

### 强动作

- 把“欢迎了解”换成具体动作。
- CTA 要和用户当前状态匹配，不能乱催。
- 如果用户还在理解阶段，动作先弱后强。

### 调顺序

- 先结论后解释，或先问题后方案，取决于场景。
- 如果用户只看标题，重要信息必须前置。
- 如果首段太软，优先把主张前移。

## 修改边界

### 红线 1：不能改事实

- 原文有具体数字，改后不能改数字。
- 原文有具体场景 / 案例，改后不能改场景 / 案例。
- 原文有限制条件，改后不能删限制条件。

### 红线 2：不能补新承诺

- 原文没有数据，改后不能自己加数据。
- 原文没有说过的话，改后不能自己说。
- 原文没有覆盖的场景，改后不能自己扩展。

### 红线 3：不能换主线

- 主卖点是什么就还是什么，不能换成别的卖点。
- 说服逻辑是什么就还是什么，不能把效果路线换成价格路线。

### 红线 4：不能把关键条件压没

- 适用条件、限制条件、价格门槛、适用规模等信息，压缩时不能压没。
- 改完之后要能判断用户适不适合。

## 常见越界形式

| 越界形式 | 原文 | 越界改法 | 正确改法 |
|---|---|---|---|
| 改数字 | 已服务 200 家客户 | 改成千家客户 | 保留数字或写 200+ |
| 编数据 | 转化率较高 | 改成提升 40% | 改成数据待确认 |
| 换卖点 | 速度快 | 改成价格更低 | 保留速度优势 |
| 删边界 | 适合 X 规模企业 | 直接删掉 | 保留限制条件 |
| 替案例 | 某制造业客户 | 改成头部客户 | 保留原描述 |

## 修改节奏

1. 先看重复。
2. 再看空话。
3. 再看抽象词。
4. 再补证据和边界。
5. 最后调顺序和句式。

## 结果判断

- 如果删改后更短但更准，说明方向对。
- 如果删改后没有信息密度，只剩修辞，说明改偏了。
- 如果删改后动作更明确，说明优化有效。
- 如果删改后丢了事实，说明改坏了。
- 如果删改后出现新卖点，说明越界。

## 输出样式

- 先给修改版。
- 再列修改点。
- 再列保留信息。
- 再列还没解决的问题。
- 再列风险提示。

## 失败案例

- 把删改做成重写，最后和原文不是一回事。
- 把所有形容词都删光，导致文案只剩骨架。
- 只改句子顺序，不解决空话和证据问题。
- 只追求短，不追求准。
- 为了顺口，把限制条件删掉。

## 新人校验

- 新人能不能先识别问题，再动手。
- 新人能不能知道删改和重写不是一回事。
- 新人能不能保住主卖点、证据、动作和限制。
- 新人能不能说清楚这次改动的最小范围。
