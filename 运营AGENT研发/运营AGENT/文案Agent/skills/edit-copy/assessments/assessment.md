# 考核标准

- 是否删除冗余。
- 是否增强表达力。
- 是否保留关键信息和证据。
- 是否说明修改理由。
- 是否没有引入新的空话。
- 是否没有把删改做成重写。
- 是否保留限制条件和动作。

## 量化门禁

- 必须先识别问题类型，不能直接动文案。
- 必须删除至少 1 类冗余问题：重复、空话或抽象中的任意一种。
- 必须保留主卖点、证据、动作、限制中的至少 3 项。
- 必须说明修改理由。
- 必须列出保留信息或未解决问题。
- 必须确保没有新增卖点或新承诺。
- 不能把删改做成重写。
- 不能把关键事实删没。
- 不能删掉限制条件。

## 一票否决

- 关键事实被删掉。
- 证据被删光。
- 只改顺序，不改逻辑。
- 新增卖点或新增承诺。
- 把限制条件改没。
- 把删改做成另一篇文案。

## 分数模型

| 项目 | 分值 |
|---|---|
| 问题识别 | 2 |
| 保留事实 | 2 |
| 删除冗余 | 2 |
| 修改理由 | 1 |
| 动作 / CTA | 1 |
| 限制条件 | 1 |
| 无新增承诺 | 1 |

- 9-10 分：优秀。
- 7-8 分：合格。
- 5-6 分：待改。
- 0-4 分：不合格。

## 测试用例

### 用例 1：删空话

**输入**："这是一个非常专业、非常高效、非常值得选择的方案。"

**预期**：删掉空话，换成具体结果或证据。

### 用例 2：删重复

**输入**："更快、更省事、更高效、更轻松。"

**预期**：保留最强的一句，其余删或合并。

### 用例 3：补证据

**输入**："让你的转化率大幅提升。"

**预期**：补数据、案例或过程；补不上就标待确认。

### 用例 4：转场景

**输入**："我们提供全面、系统、完整的解决方案。"

**预期**：改成用户语言和具体操作路径。

### 用例 5：边界保留

**输入**："适合 X 规模企业。"

**预期**：不能删掉限制条件。

## 通过标准

- 问题识别正确。
- 没改事实。
- 没加新承诺。
- 动作保留。
- 限制条件保留。
