# 考核标准

- 是否删除冗余。
- 是否增强表达力。
- 是否保留关键信息和证据。
- 是否说明修改理由。
- 是否没有引入新的空话。
- 是否没有把删改做成重写。

## 量化门禁

- 必须先识别问题类型，不能直接动文案。
- 必须删除至少 1 类冗余问题：重复、空话或抽象中的任意一种。
- 必须保留主卖点、证据、动作、限制中的至少 3 项。
- 必须说明修改理由。
- 必须列出保留信息或未解决问题。
- 必须确保没有新增卖点或新承诺。
- 不能把删改做成重写。
- 不能把关键事实删没。

## 一票否决补充

- 关键事实被删掉。
- 证据被删光。
- 只改顺序，不改逻辑。
- 新增卖点或新增承诺。
- 把限制条件改没。
