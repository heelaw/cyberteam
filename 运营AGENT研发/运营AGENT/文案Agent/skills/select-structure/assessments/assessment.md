# 考核标准

- 是否根据用户状态、证据深度和平台做结构选择。
- 是否解释选择原因，而不是只报名称。
- 是否避免结构与场景错配。
- 是否给出可执行的写作顺序、备用方案和排除理由。
- 是否能明确主结构和备用结构的边界。

## 量化门禁

- 必须给出主结构和至少 1 个备用结构。
- 必须说明为什么选它，而不是别的结构。
- 必须把结构和用户状态、证据深度、阅读时长挂钩。
- 必须给出可执行的写作顺序。
- 必须说明为什么不选其他结构。
- 不能只报结构名。
- 不能让结构选择和平台预设冲突。
- 必须有至少 2 个明确排除理由。

## 一票否决

- 先喜欢结构再硬套。
- 结构顺序写反。
- 需要解释却用纯直述。
- 需要比较却不给维度。
- 用户还没认知问题就上 `PAS`。
- 平台限制很短却选了长结构。
- 没有备用结构。

## 分数模型

| 项目 | 分值 |
|---|---|
| 主结构选择 | 2 |
| 备用结构 | 1 |
| 选择理由 | 2 |
| 排除理由 | 2 |
| 写作顺序 | 1 |
| 平台匹配 | 1 |
| 证据深度 | 1 |

- 9-10 分：优秀。
- 7-8 分：合格。
- 5-6 分：待改。
- 0-4 分：不合格。

## 测试用例

### 用例 1：新问题教育

**输入**：用户还不知道问题，平台是公众号。

**预期**：`4P` 或 `故事式`，并说明为什么不是 `PAS`。

### 用例 2：痛点共鸣

**输入**：用户知道问题但没被打动，平台是小红书。

**预期**：`PAS` 或 `AIDA`，并给备用结构。

### 用例 3：方案比较

**输入**：用户在对比 A 和 B，有明确竞品。

**预期**：`对比式`，并说明比较维度。

### 用例 4：快速行动

**输入**：用户已经接近下单，触点是详情页首屏。

**预期**：`直述式`。

### 用例 5：信息很多

**输入**：长清单、可扫读内容。

**预期**：`清单式`，而不是故事式。

## 通过标准

- 主结构明确。
- 备用结构明确。
- 排除理由明确。
- 平台和结构不冲突。
- 写作顺序能直接执行。
