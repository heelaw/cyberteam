# 用户分层

## 适用场景

- 已经知道这次文案要写给谁，但人群关系还不清楚。
- 需要判断谁是购买者、使用者、拍板者、阻拦者、影响者。
- 需要把“写给谁”拆成可执行的主写对象、辅助对象和不该写的人群。

## 先决条件

- 先确认这是单人决策、买用一体、买用分离，还是委员会决策。
- 至少要知道一个角色线索，不能完全空转。
- 如果完全没有人群线索，先回去补原话、行为或对话。

## 分层顺序

1. 先找决策权，再找角色名。
2. 再找阶段、频率和意愿差异。
3. 再看价格敏感度和认知水平。
4. 最后确认主写对象、辅助对象和不该写的人群。

## 角色框架

| 角色 | 可能身份 | 你要看什么 |
|---|---|---|
| 购买者 | 出钱的人、采购人 | 预算、价格、审批路径 |
| 使用者 | 日常操作的人 | 好不好上手、会不会增负担 |
| 拍板者 | 最终决定的人 | 风险、ROI、信任、品牌 |
| 阻拦者 | 有否决权的人 | 安全、接入、合规、复杂度 |
| 影响者 | 会影响决策的人 | 口碑、建议、技术判断 |

## 七维分层

1. 身份：他是谁。
2. 阶段：他现在在认知 / 比较 / 决策哪一步。
3. 行为：他实际做了什么。
4. 意愿：他是强意愿、弱意愿还是观望。
5. 敏感度：他最怕什么、最在乎什么。
6. 认知：他懂多少、误解在哪里。
7. 决策权：他是不是最终拍板的人。

## 判定逻辑

### 1. 单人产品 / 买用一体

- 用户本人就是购买者、使用者和决策者。
- 重点看效率、便利、隐私、体验和即时回报。
- 不需要过度分裂角色，但要确认是否真的没有其他人参与。

### 2. 买用分离

- 购买者和使用者不同。
- 购买者看成本、风险、效果；使用者看好不好用、会不会加班。
- 文案要分出主写对象和辅助对象，不要混成一个人。

### 3. 委员会决策

- 有多个角色，且有人有技术否决权、预算否决权或业务否决权。
- 先找最终拍板者，再找预算审批，再找技术阻拦，再找实际使用者。
- 文案不能只讨好使用者，忽略拍板者。

## 主写对象判断

主写对象不是“最重要的人”，而是“最适合承接当前卖点的人”。

判断时看四件事：

- 谁最可能推动这件事落地。
- 谁最能被当前卖点打动。
- 谁的痛点最清晰、最可写。
- 谁的决策权最大或最接近拍板。

## 分层输出该写什么

| 层级 | 该写什么 |
|---|---|
| 主写对象 | 直接命中他的目标、代价、决策压力 |
| 辅助对象 | 在证据、限制、FAQ、补充说明里兼顾 |
| 不该写的人群 | 直接排除，避免稀释主线 |

## 冲突处理

- 决策权高于使用权。
- 阻拦者的否决风险高于普通使用者的喜欢。
- 价格敏感不等于拍板权。
- 一个人说了多个身份时，按当下动作判断他此刻更像谁。

## 输出骨架

- 用户分层表
- 主写对象
- 辅助对象
- 决策链角色
- 各层诉求
- 各层敏感点
- 不该写的人群

## 典型分层模板

### 模板 A：单人产品

- 主写对象：用户本人
- 辅助对象：无
- 不该写的人群：无
- 重点：自我效率、体面、掌控感、隐私

### 模板 B：买用分离

- 主写对象：拍板者或推动者
- 辅助对象：使用者、预算方、阻拦者
- 不该写的人群：无关角色
- 重点：结果、成本、风险、上手难度

### 模板 C：委员会决策

- 主写对象：最终拍板者
- 辅助对象：预算审批、技术阻拦、使用者
- 不该写的人群：非链路角色
- 重点：ROI、风险控制、团队接受度

## 好样例要满足什么

- 至少 3 类角色被识别出来。
- 主写对象有选择理由，不是只写名字。
- 辅助对象有分工，不是写“也要兼顾”。
- 不该写的人群被明确排掉。
- 分层结果能直接接到写作上。

## 失败案例

### 案例 1：只写年龄职业

- 问题：没有决策链，写作没入口。

### 案例 2：把所有人都当主对象

- 问题：主线被稀释，正文会散。

### 案例 3：忽略阻拦者

- 问题：技术、合规、预算任一环都可能卡死。

### 案例 4：不知道买用关系

- 问题：使用者喜欢，不代表能买成。

## 红线

- 不能只按年龄、性别、职业分人群。
- 不能把所有人都当主对象。
- 不能忽略决策权。
- 不能把辅助对象写成主对象。
- 不能把角色标签当结论。

## 新人校验

- 新人能不能先找决策链，再找角色标签。
- 新人能不能分清主对象、辅助对象、阻拦者。
- 新人能不能处理一个人说了很多身份的情况。
- 新人能不能知道什么时候该继续补角色，什么时候可以进入卖点。

## 场景示例

### B2B 软件

- 主写对象：拍板者或推动者。
- 辅助对象：使用者、IT、预算方。
- 重点：ROI、落地效率、接入风险。

### B2C 工具

- 主写对象：用户本人。
- 辅助对象：通常无。
- 重点：省事、好用、隐私、即时结果。

### 服务型项目

- 主写对象：真正负责推进的人。
- 辅助对象：老板、执行人、配合方。
- 重点：结果、执行成本、沟通成本。
