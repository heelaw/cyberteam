# 考核标准

- 是否按可执行维度分层，而不是只写年龄职业。
- 是否识别主写对象、辅助对象和不该写的人群。
- 是否每一层都有行为或决策证据。
- 是否能把分层结果直接接到写作上。

## 量化门禁

- 必须识别购买者、使用者、拍板者、阻拦者、影响者中的至少 3 类。
- 必须说明为什么这个人是主写对象，而不是只写角色名称。
- 必须至少给出 1 条行为或决策证据。
- 必须明确产品属于单人产品、买用一体、买用分离或委员会决策中的哪一种。
- 必须说明主写对象和辅助对象如何分工。
- 不能只按年龄、性别、职业分人群。
- 结果必须能直接接到写作，不是停在画像层。

## 一票否决

- 把所有人都当主对象。
- 忽略决策权。
- 忽略认知差异。
- 输出只有标签，没有主次。
- 没说清买用关系。

## 分数模型

| 项目 | 分值 |
|---|---|
| 角色识别 | 2 |
| 主写对象理由 | 2 |
| 决策链判断 | 2 |
| 分工可落地 | 2 |
| 不该写的人群 | 1 |
| 写作可接续性 | 1 |

- 9-10 分：优秀。
- 7-8 分：合格。
- 5-6 分：待改。
- 0-4 分：不合格。

## 新人测试题

### 题 1：买用分离

**输入**：企业培训软件，负责人推动，使用者每天执行，老板看结果，技术同事看接入成本。

**要求**：
- 识别至少 4 类角色。
- 说明主写对象是谁，为什么是他。
- 写清辅助对象如何分工。

### 题 2：买用一体

**输入**：个人记账 App，自己买、自己用、自己决定。

**要求**：
- 判断为单人产品或买用一体。
- 说明为什么不需要分裂角色。
- 仍要写出敏感点。

### 题 3：委员会决策

**输入**：B 端客服系统，采购总监拍板，IT 负责人有否决权，客服主管试用，CFO 审批预算。

**要求**：
- 写出决策链优先级。
- 指出阻拦者。
- 给出主写对象选择理由。

## 通过标准

- 角色识别完整。
- 主写对象有理由。
- 分工能落地到正文。
- 没有把所有人都揉成一锅。

## 不通过标准

- 只写“目标用户：XX”。
- 没有决策权判断。
- 没有辅助对象分工。
- 把非决策角色当主线。
