# 考核标准

- 是否按可执行维度分层，而不是只写年龄职业。
- 是否识别主写对象、辅助对象和不该写的人群。
- 是否每一层都有行为或决策证据。
- 是否能把分层结果直接接到写作上。

## 量化门禁

- 必须识别购买者、使用者、拍板者、阻拦者、影响者中的至少 3 类。
- 必须说明为什么这个人是主写对象，而不是只写角色名称。
- 必须至少给出 1 条行为或决策证据。
- 必须明确产品属于单人产品、买用一体、买用分离或委员会决策中的哪一种。
- 必须说明主写对象和辅助对象如何分工。
- 不能只按年龄、性别、职业分人群。
- 结果必须能直接接到写作，不是停在画像层。

## 一票否决补充

- 把所有人都当主对象。
- 忽略决策权。
- 忽略认知差异。
- 输出只有标签，没有主次。

## Assessment门槛

### 及格线（60分）

满足以下全部条件：

| 维度 | 要求 | 检查方式 |
|------|------|----------|
| 角色识别 | 至少识别3类角色（购买者/使用者/拍板者/阻拦者/影响者） | 检查分层表，角色≥3 |
| 主写对象理由 | 说明为什么是主写对象，不只是写角色名 | 有具体理由，非"因为是负责人"这种泛泛说法 |
| 产品结构 | 明确写出单人/买用一体/买用分离/委员会决策 | 输出中有明确标注 |
| 辅助分工 | 说明主写对象和辅助对象如何分工 | 不能只写"兼顾" |

### 良好线（75分）

满足及格线全部条件，并额外满足：

| 维度 | 要求 | 检查方式 |
|------|------|----------|
| 行为/决策证据 | 每层至少1条行为或决策证据 | 不能只靠推断 |
| 敏感点具体 | 各层敏感点落到具体可写维度 | 不是"价格高"这种泛泛说法 |
| 角色冲突处理 | 有角色冲突时给出处理策略 | 无冲突时此项不扣分 |
| 分工可落地 | 辅助对象的分工能对应到文案具体位置 | 不是"在说明里兼顾"这种笼统说法 |

### 优秀线（90分）

满足良好线全部条件，并额外满足：

| 维度 | 要求 | 检查方式 |
|------|------|----------|
| 决策链优先级 | 明确写出决策链优先级 | 不是只写角色名，能说清楚谁先谁后 |
| 不该写的人群 | 明确标注不该写的人群和原因 | 不是"无关人员"这种泛泛说法 |
| 交接清晰 | 输出能直接被下一Skill使用 | 主写对象、诉求、敏感点完整，可进入写作 |
| 角色维度完整 | 七个分层维度（身份/阶段/行为/意愿/敏感度/认知/决策权）至少覆盖5个 | 能对应到写作的不同方面 |

### 不及格（0分）情形

出现任一情形直接判定为0分，不进入评分：

1. **空泛标签**：只有"25-35岁女性白领"，没有角色/决策权/阶段维度
2. **全员主写**：主写对象写了"所有人"或"所有用户"
3. **忽略决策权**：不知道谁是拍板者，或明明有拍板者但忽略
4. **无分工说明**：主写对象和辅助对象没有说明如何分工
5. **无角色理由**：主写对象只有名字，没有为什么选他的理由
6. **买用分离未拆分**：买用分离产品但没有分开写两类角色的诉求
7. **委员会未分层**：委员会决策但把所有角色平铺，无主次

### 评分权重

| 项目 | 权重 | 说明 |
|------|------|------|
| 角色识别完整性 | 25% | 是否识别足够的角色维度 |
| 主写对象明确性 | 25% | 选择理由是否充分 |
| 辅写分工可落地性 | 25% | 辅助对象的分工是否对应到文案 |
| 交接清晰性 | 25% | 能否直接进入下一Skill |

### 评估流程

1. **输入检查**：确认输入符合"允许启动"条件
2. **输出结构检查**：确认输出包含分层表、主写对象、辅助分工、不该写的人群
3. **逐项打分**：按权重计算各项得分
4. **一票否决检查**：确认没有0分情形
5. **最终判定**：优秀(≥90)/良好(≥75)/及格(≥60)/不及格(<60)
