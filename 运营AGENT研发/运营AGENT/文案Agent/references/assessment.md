# 文案Agent 验收标准

## 根 Agent

- 是否先判断阶段，再路由到正确 Skill。
- 是否把每个 Skill 限制为单个最小 SOP。
- 是否保持事实、推断、结论分离。
- 是否在信息不足时返回缺口，而不是硬给成品。
- 是否保留渐进式披露结构。
- 是否明确岗位边界，不把 Agent 写成原始培训摘要或通用说明书。
- 是否给出可执行的下一跳，而不是只做概念解释。
- 是否存在最大回退次数和兜底判断。
- 是否要求高风险内容在发布前再做一次测试或数据验证。

## 一票否决

- 直接把成品文案写出来，却没有先完成需求、用户、卖点、场景或结构判断。
- 把卖点、用户、场景、结构混成一个大段落。
- 结论没有依据，或把推断写成事实。
- 根文档塞入长篇材料，破坏渐进式披露。
- 根文档或 Skill 中出现任何旧版表述。
- 不存在 `SOUL.md`，导致岗位定义缺失。
- 模糊输入没有缺口清单和兜底路径。
- 高风险内容在未测试前直接发布。
- 根层不提供任何回退和复核入口。

## 量化门禁

- 路由准确率必须 100%。
- 至少 1 个前置 Skill 不能跳过，除非输入已经完整且满足交付条件。
- 每个 Skill 至少包含：SOP、输出、参考、考核、红线。
- 总方法层和局部 Skill 不得重复堆砌同一判断逻辑。
- 场景判断必须能落到平台预设，而不是停在“短 / 中 / 长”。
- 交付内容必须同时给结论、依据、风险或缺口。
- 通过后若属于高风险内容，必须附带测试或数据验证路径。
- 根层文档必须能一眼看懂入口顺序和回退路径。

## 根层校验

- 可以用 `scripts/validate_agent_package.py` 做自动检查。
- 校验失败时，先修链接和禁词，再修结构。
- 通过前先确认没有遗留旧版命名。
