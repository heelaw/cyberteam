## 核心定义

### 什么是AIDA模型？

AIDA是经典的文案结构模型，代表用户从注意到行动的转化路径：

```
┌─────────────────────────────────────────────────────┐
│                   A - Attention                      │
│                   引起注意                            │
│         标题党、悬念、冲突、反常识                    │
└─────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────┐
│                   I - Interest                       │
│                   引起兴趣                            │
│         提问、利益关联、认同感                        │
└─────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────┐
│                   D - Desire                         │
│                   刺激欲望                            │
│         恐惧、从众、效果对比、权威背书                │
└─────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────┐
│                   A - Action                         │
│                   促使行动                            │
│         明确指引、限时优惠、立即领取                  │
└─────────────────────────────────────────────────────┘
```

### AIDA vs 4P模型对比

| 模型 | 适用场景 | 核心特点 |
|------|---------|---------|
| AIDA | 销售导向文案、转化文案 | 关注用户心理转化路径 |
| 4P | 品牌文案、形象文案 | 关注产品/价格/渠道/促销 |

---

## Critical Rules（必须遵守）

1. **A→I→D→A顺序不能乱** - 必须先引起注意，才能激发兴趣，最后才是行动
2. **每个阶段只做一件事** - Attention阶段不要试图同时讲产品
3. **行动指引必须明确** - 不能只暗示，要有明确的CTA（Call to Action）
4. **标题决定打开率** - Attention的核心是写好标题
5. **欲望要用证据支撑** - 不能只说"很好"，要告诉用户"为什么好"

---

## 禁止行为

1. **禁止跳过Attention** - 不引起注意就直接讲产品
2. **禁止在Interest讲促销** - 促销信息会打断用户兴趣
3. **禁止抽象描述欲望** - 不能只说"提升效率"，要说"每天多睡2小时"
4. **禁止模糊行动** - 不能说"了解更多"，要说"立即领取"
5. **禁止堆砌卖点** - 3个核心卖点足够，不要超过5个
6. **禁止无差异化** - 不能和竞品用同样的套路
7. **禁止忽略用户场景** - 不同场景需要不同的AIDA表达方式

## Workflow Steps

### 步骤1：设计Attention（引起注意）（输入→处理→输出）

**输入**：
- 目标用户画像
- 产品核心卖点

**处理**（5种Attention写法）：
| 写法 | 示例 | 适用场景 |
|------|------|---------|
| 反常识 | "月入3万的人都在用这个方法" | 追求眼球 |
| 悬念 | "为什么你总是做不好运营？" | 引发好奇 |
| 冲突 | "努力工作却得不到晋升？" | 痛点共鸣 |
| 数字 | "3个技巧让转化率翻倍" | 具体可信 |
| 权威 | "连XX都在用的方法" | 信任背书 |

**输出**：
- 10个候选标题
- 选出的最优标题及其Attention策略

**成功标准**：标题能引发目标用户的好奇或共鸣

---

### 步骤2：设计Interest（引起兴趣）（输入→处理→输出）

**输入**：
- 已确定的Attention
- 产品卖点清单

**处理**（Interest构建方法）：
```
1. 关联用户痛点
   - 描述用户当前困境
   - 让用户产生代入感

2. 展示利益关联
   - 告诉用户能得到什么
   - 用"你"而不是"产品"说话

3. 建立认同感
   - 说用户想听的话
   - 让用户觉得"这就是我"
```

**输出**：
- Interest段落（50-150字）
- 3个核心卖点的引入方式

**成功标准**：用户读完后想知道更多

---

### 步骤3：设计Desire（刺激欲望）（输入→处理→输出）

**输入**：
- Interest段落
- FABE卖点卡片
- 产品证据/数据

**处理**（Desire激发方法）：
| 方法 | 说明 | 示例 |
|------|------|------|
| 效果对比 | Before/After对比 | "使用前vs使用后" |
| 恐惧诉求 | 不行动的损失 | "错过再等一年" |
| 从众心理 | 证明很多人都在用 | "已有10万+用户" |
| 权威背书 | 专家/名人推荐 | "XX联名推荐" |
| 场景化 | 描述使用场景 | "想象你使用后的画面" |

**输出**：
- Desire段落（100-200字）
- 核心证据清单（数据/案例/背书）

**成功标准**：用户读完后有强烈的"我想要"的冲动

---

### 步骤4：设计Action（促使行动）（输入→处理→输出）

**输入**：
- Desire段落
- 推广渠道
- 转化目标

**处理**（CTA设计方法）：
| 要素 | 说明 | 示例 |
|------|------|------|
| 明确动作 | 告诉用户做什么 | "立即点击领取" |
| 紧迫感 | 为什么要现在 | "限时3天" |
| 稀缺性 | 为什么要抢 | "仅剩100份" |
| 便利性 | 降低行动门槛 | "免费领取" |
| 信任感 | 消除顾虑 | "不满意退款" |

**输出**：
- Action文案（10-30字）
- 完整AIDA文案结构图

**成功标准**：用户知道现在要做什么

---

### 步骤5：整体优化与自检（输入→处理→输出）

**输入**：
- 完整的AIDA文案

**处理**（自检清单）：
| 检查项 | 标准 | 通过标准 |
|--------|------|---------|
| 标题吸引力 | 能在3秒内引起注意 | 是/否 |
| 兴趣连贯性 | 从标题到兴趣过渡自然 | 是/否 |
| 欲望激发 | 用户有"我想要"的冲动 | 是/否 |
| 行动明确 | 用户知道要做什么 | 是/否 |
| 字数控制 | 总字数符合场景要求 | 是/否 |
| 卖点聚焦 | 不超过3个核心卖点 | 是/否 |

**输出**：
- 优化后的完整文案
- 自检报告

**成功标准**：所有检查项全部通过

---

## 评估维度

| 维度 | 定义 | 权重 |
|------|------|------|
| Attention吸引力 | 标题能否引起注意 | 25% |
| Interest共鸣度 | 能否引发用户兴趣 | 20% |
| Desire说服力 | 能否激发用户欲望 | 25% |
| Action明确性 | 行动指引是否清晰 | 20% |
| 整体流畅性 | 四阶段过渡是否自然 | 10% |

## 输出格式

### 完整AIDA文案

```markdown
# AIDA文案结构

## Attention（引起注意）
标题：{标题}
策略：{使用的Attention策略}

## Interest（引起兴趣）
{50-150字的兴趣段落}

## Desire（刺激欲望）
{100-200字的欲望激发段落}

核心证据：
- {证据1}
- {证据2}
- {证据3}

## Action（促使行动）
CTA：{行动文案}
紧迫感：{紧迫元素}
信任保障：{信任元素}

## 自检结果
- [ ] Attention吸引力
- [ ] Interest共鸣度
- [ ] Desire说服力
- [ ] Action明确性
```
