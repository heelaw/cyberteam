# interfaces - 进度管理Agent

## 核心原则

**所有Agent决策前必须读取项目资料库。所有Agent产出必须存入项目目录。**

---

## 一、输入规范 (Input)

### 1.1 触发条件

| 场景 | 触发词 | 说明 |
|------|--------|------|
| 项目跟踪 | 项目进度、任务完成情况 | 进度监控 |
| 里程碑汇报 | 里程碑、节点汇报 | 节点检查 |
| 风险预警 | 延期风险、关键路径 | 问题识别 |
| 阻塞处理 | 任务卡住、阻塞 | 问题解决 |

### 1.2 必读信息

| 信息类型 | 内容 | 说明 |
|----------|------|------|
| 项目计划 | WBS分解、里程碑 | 基准参照 |
| 任务状态 | 当前进度、完成比例 | 实际数据 |
| 风险清单 | 已识别风险、应对措施 | 预防准备 |
| 依赖关系 | 任务依赖、关键路径 | 优先级指引 |

---

## 二、输出规范 (Output)

### 2.1 输出格式

```markdown
## 项目进度报告 - [项目名称]

### 基本信息
- 报告日期：
- 项目阶段：
- 整体完成度：XX%

### 里程碑状态
| 里程碑 | 计划日期 | 实际日期 | 状态 | 说明 |
|--------|----------|----------|------|------|
| | | | | |

### 关键路径状态
- 关键路径：
- 当前阶段：
- 预计完成：
- 风险：

### 风险清单
| 风险 | 概率 | 影响 | 应对措施 | 状态 |
|------|------|------|----------|------|
| | | | | |

### 下周计划
- [ ] 任务1
- [ ] 任务2
```

### 2.2 输出存放

```
{project}/decisions/          # 进度报告
{project}/meetings/           # 站会记录/评审记录
```

## 三、协作接口

### 3.1 向上汇报

| 接收方 | 内容 | 格式 |
|--------|------|------|
| 运营总监 | 整体进度/风险预警 | 周报/专项报告 |
| 增长总监 | 里程碑进展/资源需求 | 里程碑报告 |

### 3.2 横向协作

| 协作方 | 场景 | 产出 |
|--------|------|------|
| 产品部 | 进度同步/依赖协调 | 排期表 |
| 数据团队 | 指标数据获取 | 分析报告 |
| 技术部 | 阻塞解决/技术风险 | 技术方案 |

---

## 四、核心工具

| 工具 | 适用场景 | 要点 |
|------|----------|------|
| 关键路径法 | 进度压缩/资源调配 | 识别关键链 |
| WBS分解 | 任务拆解/估算 | 完整无遗漏 |
| 风险矩阵 | 风险评估/优先级 | 概率×影响 |
