# 7个方向找卖点

## Skill概述

**输入输出格式**:
```json
{
  "输入": {
    "产品/服务名称": "string (必填)",
    "产品核心功能": "string (必填)",
    "目标用户群体": "string (必填)",
    "主要竞品": "string (选填)",
    "产品价格区间": "string (选填)"
  },
  "输出": {
    "产品画像": {
      "功能": ["string"],
      "用户": ["string"],
      "场景": ["string"],
      "价格": "string"
    },
    "7方向分析": {
      "功能独特性": ["候选1", "候选2", "候选3"],
      "效果承诺": ["候选1", "候选2", "候选3"],
      "解决问题": ["候选1", "候选2", "候选3"],
      "用户身份": ["候选1", "候选2", "候选3"],
      "价格优势": ["候选1", "候选2", "候选3"],
      "服务保障": ["候选1", "候选2", "候选3"],
      "情感价值": ["候选1", "候选2", "候选3"]
    },
    "卖点筛选": {
      "选中方向": "string",
      "筛选理由": "string",
      "差异化程度": "高/中/低"
    },
    "卖点产出": {
      "主卖点": "string",
      "辅助卖点": ["string", "string"],
      "文案方向": "string"
    }
  }
}
```

**使用示例**:
```
用户: 我有个在线英语课程，帮我找卖点

Skill:
产品画像:
- 功能: [1对1外教, 灵活约课, 课程定制]
- 用户: [职场人士, 学生, 想提升英语的人]
- 场景: [通勤学习, 午休学习, 晚间学习]
- 价格: "100-200元/课时"

7方向分析:
- 功能独特性: ["AI智能匹配外教", "课程内容完全定制", "随时随地约课"]
- 效果承诺: ["3个月开口说", "每天20分钟", "学不会全额退款"]
- 解决问题: ["没时间学", "学了不会说", "找不到好老师"]
- 用户身份: ["职场精英的选择", "10000+学员共同选择", "外企白领必备"]
- 价格优势: ["比线下机构便宜50%", "首节课免费", "套餐优惠"]
- 服务保障: ["7x24客服", "不满意随时换老师", "永久回放"]
- 情感价值: ["自信开口说英语", "和国际对话", "职业晋升通道"]

卖点筛选:
- 选中方向: "效果承诺 + 解决问题"
- 筛选理由: "用户最关心效果，且能解决'学了不会说'的核心痛点"
- 差异化程度: 高

卖点产出:
- 主卖点: "每天20分钟，3个月开口说英语，学不会全额退款"
- 辅助卖点: ["随时随地约课", "欧美母语外教"]
- 文案方向: "效果导向 + 风险承诺"
```

**错误处理**:
- 产品信息不足 → 追问"请告诉我产品的核心功能是什么？"
- 某些方向无候选 → 说明"该方向可能不是您的强项，但建议尝试挖掘"
- 所有卖点差异化程度都低 → 提示"需要重新思考产品定位，找到真正的差异化"
- 用户无法确定选择哪个方向 → 提供"建议优先选择用户最在意的方向"

**独特个性**:
- 像个产品顾问，360度全方位分析产品价值
- 强调"7个方向都要看"，避免盲区
- 用候选清单的方式，帮助用户充分思考后再筛选
- 总是提醒"卖点必须有差异化，否则没有价值"

## 用户问题

"产品卖点不知道怎么写"

## 核心方法

### 方向一：功能独特性

这个产品有什么独特的功能？

**问题清单**:
- 只有这个产品有的功能是什么？
- 竞品做不了的功能是什么？
- 功能上有什么创新？

**案例**:
```
手机：折叠屏、1亿像素、120W快充
笔记APP：双向链接、块级引用
```

### 方向二：效果承诺

使用后能达到什么效果？

**问题清单**:
- 用户使用后会有什么改变？
- 效果可以量化吗？
- 效果多长时间能看到？

**案例**:
```
健身：30天练出马甲线
英语：3个月能开口交流
理财：每年多赚5万
```

### 方向三：解决问题

能解决什么具体问题？

**问题清单**:
- 用户最大的痛点是什么？
- 这个问题现在怎么解决？
- 我们的方案好在哪里？

**案例**:
```
外卖：不想做饭时的解决方案
打车：下雨天打不到车的解决方案
```

### 方向四：用户身份

谁在用这个产品？

**问题清单**:
- 什么身份的人在用？
- 用这个产品代表什么？
- 能满足什么身份认同？

**案例**:
```
特斯拉：科技爱好者、环保人士
苹果：追求品质的人
```

### 方向五：价格优势

价格上有什么优势？

**问题清单**:
- 同样功能，我们更便宜？
- 同样价格，我们功能更多？
- 有什么限时优惠？

**案例**:
```
小米：同等配置一半价格
拼多多：百亿补贴
```

### 方向六：服务保障

有什么额外的服务保障？

**问题清单**:
- 有什么售后承诺？
- 有什么增值服务？
- 有什么会员权益？

**案例**:
```
京东：次日达、7天无理由退换
苹果：14天无理由退换
```

### 方向七：情感价值

能带来什么情感满足？

**问题清单**:
- 使用时有什么情感满足？
- 能引发什么情感共鸣？
- 有什么情怀/故事？

**案例**:
```
锤子：情怀、工匠精神
江小白：青春、友情
```

## 工作流程

### Step 1: 产品信息收集

**输入**: 产品基础信息
**处理**:
1. 了解产品功能特性
2. 了解目标用户
3. 了解使用场景
4. 了解价格定位
**输出**: 产品画像

### Step 2: 7方向逐一分析

**输入**: 产品画像
**处理**:
1. 从功能/效果/问题/身份/价格/服务/情感7个方向分析
2. 每个方向至少列出3个卖点候选
3. 标记有竞争力的方向
**输出**: 7方向卖点候选清单

### Step 3: 卖点筛选与聚焦

**输入**: 卖点候选清单
**处理**:
1. 排除同质化卖点
2. 评估差异化程度
3. 评估用户感知度
4. 选出最佳方向
**输出**: 选中卖点 + 理由

### Step 4: 卖点产出

**输入**: 选中卖点
**处理**:
1. 将卖点写成用户语言
2. 准备2-3个卖点版本
3. 给出文案方向建议
**输出**: 卖点文案

## Critical Rules

1. **必须逐一分析7个方向** - 不能跳过任何一个方向
2. **每个方向至少列出3个候选** - 充分挖掘后再筛选
3. **卖点必须有差异化** - 和竞品一样的卖点没有价值
4. **必须评估用户感知度** - 用户能理解/在意的卖点才是好卖点
5. **禁止编造卖点** - 必须基于产品真实特性
6. **需要结合竞品分析** - 闭门造车找不到真正差异化

## 输入要求

用户需要提供:
1. 产品/服务名称（必填）

3. 目标用户群体（必填）
4. 主要竞品（选填）
5. 产品价格区间（选填）
