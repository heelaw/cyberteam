# 需求类型判断标准

## 三大需求类型

### 功能性需求

**定义**：用户想要产品解决具体问题

**核心问题**：能不能解决问题？

**典型问题**：
- "这个能帮我做XXX吗？"
- "和XX产品比，哪个效果更好？"
- "多久能见效？"
- "真的有用吗？"

**决策因素**：
- 效果
- 效率
- 成本
- 便捷性

**应对要点**：
- 效果量化
- 证据证明
- 试用体验
- 风险保障

---

### 情感性需求

**定义**：用户想要产品满足情感需求

**核心问题**：感觉怎么样？

**典型问题**：
- "这个适合我吗？"
- "用这个的是什么人？"
- "感觉怎么样？"
- "能体现我的品味吗？"

**决策因素**：
- 身份认同
- 自我表达
- 使用体验
- 情感共鸣

**应对要点**：
- 身份认同
- 场景共鸣
- 情怀故事
- 社交货币

---

### 社会性需求

**定义**：用户想要产品带来社会价值

**核心问题**：别人怎么看？

**典型问题**：
- "这个牌子怎么样？"
- "别人会怎么看？"
- "大家都在用吗？"
- "有名人推荐吗？"

**决策因素**：
- 权威认可
- 群体归属
- 社会地位
- 他人评价

**应对要点**：
- 权威背书
- 销量证明
- 口碑展示
- 圈层效应

## 混合需求处理

### 常见组合

| 组合 | 优先级 | 处理方法 |
|------|--------|----------|
| 功能+情感 | 先功能后情感 | 先解决能不能，再解决适不适合 |
| 功能+社会 | 先社会后功能 | 先解决敢不敢买，再解决好不好用 |
| 情感+社会 | 先情感后社会 | 先解决身份认同，再解决认可问题 |

### 处理原则

1. **先解决主要需求**：资源有限时抓主要矛盾
2. **再满足次要需求**：主要需求满足后满足次要
3. **避免顾此失彼**：不要强化一个需求而忽略另一个
