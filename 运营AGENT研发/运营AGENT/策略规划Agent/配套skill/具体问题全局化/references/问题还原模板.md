# 问题还原模板

## 模板说明
用于完整还原具体问题的上下文和背景。

## 模板内容

```
# [问题名称] - 问题还原

## 问题陈述
- **问题标题**：[给问题起一个清晰的名字]
- **问题类型**：[紧急/重要/战略/执行]

## 5W1H完整还原

### What（是什么问题）
- **问题描述**：[具体发生了什么]
- **问题表现**：[如何表现出来的]
- **问题程度**：[严重/中等/轻微]
- **影响范围**：[影响的业务/人员范围]

### When（何时发生）
- **发现时间**：[何时第一次发现这个问题]
- **开始时间**：[问题开始的时间点（如能确定）]
- **持续时间**：[问题持续了多久]
- **频率**：[问题发生的频率]
- **最近更新时间**：[最后更新时间]

### Where（在哪里发生）
- **发生地点**：[具体场景/系统/部门]
- **涉及区域**：[地域/业务范围]
- **界面/环节**：[具体在哪个环节发生]

### Who（谁遇到了问题）
- **直接受影响方**：[谁直接受到这个问题影响]
- **间接受影响方**：[谁间接受到影响]
- **问题负责人**：[谁在负责处理这个问题]
- **相关方**：[还有谁与这个问题相关]

### Why（为什么认为这是问题）
- **问题背景**：[为什么会认为这是问题]
- **期望 vs 实际**：
  - 期望：[期望的情况]
  - 实际：[实际情况]
  - 差距：[差距描述]
- **判断依据**：[基于什么判断这是问题]

### How（如何发现的）
- **发现方式**：[主动发现/被动收到反馈]
- **发现渠道**：[用户反馈/数据分析/内部检查]
- **发现人**：[谁发现的]
- **证据**：[支持问题存在的证据]

## 问题背景详情

### 业务背景
[描述相关的业务背景]

### 历史上下文
- **以前情况**：[以前是什么样的]
- **变化时间点**：[什么时候开始变化的]
- **变化内容**：[发生了什么变化]

### 相关因素
| 因素 | 与问题的关系 | 影响程度 |
|------|--------------|----------|
| [因素1] | [关系] | [高/中/低] |
| [因素2] | [关系] | [高/中/低] |

## 已知信息汇总

### 已确认的事实
1. [事实1]
2. [事实2]
3. [事实3]

### 已知的假设
1. [假设1]（可能不准确）
2. [假设2]（可能不准确）

### 不确定的地方
1. [不确定1]
2. [不确定2]

## 问题边界

### 在范围内
- [明确包含的内容]

### 不在范围内
- [明确不包含的内容]

### 假设条件
- [问题成立所依赖的假设]

## 初步判断

### 问题严重性
| 维度 | 评估 | 说明 |
|------|------|------|
| 业务影响 | [高/中/低] | [说明] |
| 用户影响 | [高/中/低] | [说明] |
| 财务影响 | [高/中/低] | [说明] |
| 声誉影响 | [高/中/低] | [说明] |
| **综合严重性** | **[高/中/低]** | |

### 紧迫性
- **时间敏感性**：[是否有时限要求]
- **处理紧迫性**：[高/中/低]

## 问题还原确认
- [ ] 5W1H全部要素已收集
- [ ] 期望与实际的差距明确
- [ ] 问题边界清晰
- [ ] 有足够的初始信息可以开始分析
```

## 使用说明
1. 按照5W1H逐项收集信息
2. 不确定的地方明确标注为假设
3. 明确问题的边界（包含和不包含）
4. 对问题严重性做初步判断
5. 确认有足够信息可以继续分析
