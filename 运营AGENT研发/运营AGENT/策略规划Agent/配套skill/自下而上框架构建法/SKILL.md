## 技能描述

**一句话定义**：从事实和现状出发，正向归纳解决方案的框架构建方法，适用于问题清晰但原因不明的情况。

**详细描述**：

自下而上框架构建法是问题诊断和根因分析的核心方法。它从具体的事实、数据和观察出发，通过归纳推理逐层上升，最终提炼出可操作的解决方案。

这种方法确保策略建立在事实基础上，而不是假设或直觉，适合用于解决"现状出了问题但不知道原因"的问题。

## 核心原理

### 归纳推理链

```
    [观察/数据]
         │
    ┌─────┼─────┐
    ▼     ▼     ▼
 [事实1] [事实2] [事实3]
         │
         ▼
    [规律识别]
         │
         ▼
    [根因假设]
         │
         ▼
    [解决方案]
```

### 事实金字塔

```
              [结论/方案]
                 ▲
           ┌─────┼─────┐
           ▲     ▲     ▲
        [规律] [规律] [规律]
           ▲     ▲     ▲
        [事实] [事实] [事实]
           ▲     ▲     ▲
        [数据] [数据] [数据]
```

## Critical Rules（必须遵守）

1. **事实必须可验证** - 每个底层事实必须有数据支撑或明确来源，无法验证的不能作为基础
2. **区分事实与观点** - 事实是"发生了什么"，观点是"我认为发生了什么"，不能混淆
3. **禁止跳步归纳** - 不能跳过中间层级直接从数据跳到结论，必须逐层归纳
4. **识别反例** - 归纳时必须主动寻找反例，只有经得起反例检验的归纳才可信
5. **假设必须标注** - 从规律到根因的跳跃是假设，不是事实，必须明确标注并验证
6. **保持完整性** - 归纳出的根因必须能解释全部或大部分事实，不能选择性忽略
7. **方案必须可证伪** - 提出的解决方案必须能够被验证有效或无效

## Workflow Steps（标准化流程）

### 步骤1：事实收集

| 项目 | 内容 |
|------|------|
| 输入 | 用户描述的问题或现象 |
| 处理 | 系统性地收集相关事实和数据 |
| 输出 | 事实清单，每条标注来源和可信度 |
| 成功标准 | 事实覆盖问题的各个维度，且有足够的数据支撑 |

**收集维度**：
- 时间维度（何时开始、频率、趋势）
- 空间维度（哪里出问题、范围）
- 主体维度（谁受影响、谁行为异常）
- 行为维度（具体做了什么、没做什么）

---

### 步骤2：事实分类

| 项目 | 内容 |
|------|------|
| 输入 | 事实清单 |
| 处理 | 按问题维度或时间线对事实进行分类 |
| 输出 | 分类后的事实结构 |
| 成功标准 | 同类事实放在一起，事实之间无重叠无遗漏 |

**分类方法**：
- 按问题维度（MECE分类）
- 按时间顺序（找出模式）
- 按主题聚类（发现关联）

---

### 步骤3：规律识别

| 项目 | 内容 |
|------|------|
| 输入 | 分类后的事实结构 |
| 处理 | 从事实中归纳出重复出现的模式或规律 |
| 输出 | 规律清单，每条规律标注支撑的事实 |
| 成功标准 | 规律能解释多条事实，且存在逻辑上的因果关系 |

**识别方法**：
- 寻找共同模式
- 识别相关性
- 发现异常点

---

### 步骤4：根因推断

| 项目 | 内容 |
|------|------|
| 输入 | 规律清单 |
| 处理 | 从规律推断根本原因 |
| 输出 | 根因假设清单，每条标注推断依据 |
| 成功标准 | 根因能解释全部规律，且是逻辑上的充分条件 |

**推断方法**：
- 5Why分析法
- 鱼骨图
- 波特五力（上归属因）

---

### 步骤5：方案生成

| 项目 | 内容 |
|------|------|
| 输入 | 根因假设 |
| 处理 | 针对每个根因生成解决方案 |
| 输出 | 解决方案清单，每条标注针对的根因 |
| 成功标准 | 方案与根因之间有明确的因果逻辑 |

**生成方法**：
- 直接干预根因
- 缓解根因的条件
- 绕过根因的替代路径

---

### 步骤6：框架整合

| 项目 | 内容 |
|------|------|
| 输入 | 事实、规律、根因、方案 |
| 处理 | 整合为完整的自下而上框架 |
| 输出 | 结构化文档，包含事实→规律→根因→方案的完整链条 |
| 成功标准 | 读者能从事实开始，完整地跟随推理过程到达方案 |

## 使用限制

1. **不适用于**：问题完全不清晰、缺乏数据支撑、需要在假设基础上快速决策的情况
2. **前提条件**：必须有可验证的事实和数据，或者用户愿意投入时间收集事实
3. **局限性**：归纳法不能证明因果关系，只能提供最可能的解释，需要后续验证
