# 指标追踪看板设计

## 看扳设计原则

### 设计目标

1. **一目了然** - 关键信息一眼可见
2. **问题导向** - 异常数据突出显示
3. **行动指向** - 数据驱动行动
4. **分层清晰** - 不同角色看到不同深度

### 分层设计

| 层级 | 受众 | 更新频率 | 内容深度 |
|------|------|---------|---------|
| **高层看扳** | CEO/高管 | 日/周 | 北极星+核心指标 |
| **部门看扳** | 部门负责人 | 日/周 | 子指标+异常 |
| **执行看扳** | 执行层 | 实时/日 | 详细数据+操作入口 |

## 预警机制设计

### 预警规则模板

```markdown
## 预警规则

### 自动预警条件

| 预警级别 | 触发条件 | 通知方式 | 响应时限 |
|---------|---------|---------|---------|
| 🔴 紧急 | 低于目标15%+ | 电话+短信+邮件 | 1小时 |
| 🟠 警告 | 低于目标10%+ | 短信+邮件 | 4小时 |
| 🟡 关注 | 低于目标5%+ | 邮件+群通知 | 24小时 |
| 🟢 正常 | 在目标范围内 | 系统记录 | - |

### 预警处理流程

```
指标异常
    │
    ▼
自动预警触发
    │
    ▼
通知负责人 + 上级主管
    │
    ▼
30分钟内初步回复
    │
    ▼
4小时内分析原因
    │
    ▼
24小时内制定调整方案
    │
    ▼
执行并追踪效果
```

### 预警升级机制

```markdown
## 预警升级

| 状态 | 条件 | 升级对象 | 升级时间 |
|------|------|---------|---------|
| 首次预警 | 触发关注级别 | 直属上级 | 即时 |
| 升级预警 | 同一指标连续3天预警 | 部门负责人 | 第3天 |
| 高级预警 | 连续5天未改善 | 高管+部门负责人 | 第5天 |
| 紧急升级 | 低于目标20%+ | CEO | 即时 |
```

## 看扳设计检查清单

```
[ ] 1. 用户定位
    - 明确看板是为谁设计的？
    - 不同层级看到不同深度？

[ ] 2. 指标选取
    - 北极星指标突出显示？
    - 子指标与北极星关联？
    - 避免虚荣指标？

[ ] 3. 可视化呈现
    - 关键数据一眼可见？
    - 异常数据突出？
    - 趋势变化清晰？

[ ] 4. 更新机制
    - 数据更新频率合理？
    - 自动更新还是手动？
    - 延迟可接受？

[ ] 5. 预警体系
    - 预警规则明确？
    - 响应时限合理？
    - 升级机制清晰？

[ ] 6. 行动导向
    - 数据能驱动行动？
    - 有问题即有对策？
    - 追踪闭环？
```
