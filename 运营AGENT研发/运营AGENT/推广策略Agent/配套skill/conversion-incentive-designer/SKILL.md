# conversion-incentive-designer

## 简介

设计转化诱因，提升营销推广中的用户转化率。本Skill覆盖10种常见转化诱因和4类场景匹配方法。

## 触发关键词

```
转化诱因、促活、促转化、公开承诺、限量、免费、福利折扣、用户评价、品牌背书、媒体背书
```

## 触发场景描述

```
当需要解决以下问题时触发本Skill：
- 如何让用户产生点击/购买行为
- 高决策成本产品如何提升转化
- 转化率低如何设计诱因
- 不同场景如何匹配不同诱因
```

## 方法论：10种常见转化诱因

### 1. 公开承诺

**定义：** 通过公开的承诺/宣言形成社会证明

**案例：** "已有10万+学员选择"

### 2. 限量

**定义：** 通过限量制造稀缺感和紧迫感

**案例：** "仅剩最后50个名额"

### 3. 免费

**定义：** 提供免费价值降低决策门槛

**案例：** "免费领取价值299元课程"

### 4. 在线服务

**定义：** 提供实时的在线服务支持

**案例：** "24小时在线客服"

### 5. 福利折扣

**定义：** 提供价格优惠或额外福利

**案例：** "首充5折优惠"

### 6. 品牌形象

**定义：** 通过品牌背书增强信任

**案例：** "知名品牌，品质保障"

### 7. 附加服务

**定义：** 提供额外的增值服务

**案例：** "一年免费质保"

### 8. 用户评价

**定义：** 展示真实用户的使用体验

**案例：** "4.9分，10000+真实评价"

### 9. 媒体报道

**定义：** 通过媒体背书增强权威性

**案例：** "被36Kr、虎嗅等媒体报道"

### 10. 真人现身说法

**定义：** 通过真实用户故事说服

**案例：** "学员故事：张三年的转变"

## 方法论：4类场景匹配

### 场景1：轻决策+轻场景

**判断标准：** 转化成本低，决策成本低，场景较轻，用户接收时间少

**案例：** 公众号下方点击推广APP

**转化诱因：** 一张图或一句话文案给到用户足够的诱因
- 最适用于：福利、折扣、效果承诺

### 场景2：轻决策+重场景

**判断标准：** 转化成本低，决策成本低，场景较重，用户有充裕时间

**案例：** 淘宝服饰配饰店

**转化诱因：** 不断通过文字、图片、实时销量、限额等给到用户诱惑
- 转化流程一定要顺畅

### 场景3：重决策+轻场景

**判断标准：** 转化成本高，决策成本也高，场景较轻，用户接收时间少

**案例：** 移动端推广客单较高的咨询服务、企业付费产品

**转化诱因：** 先通过冲击文案、诱惑文案等方式给用户一个较轻的诱因（如获取某NB资料包），让用户留下联系方式
- 此后再到其他场景去慢慢转化用户

### 场景4：重决策+重场景

**判断标准：** 转化成本高，决策成本也高，场景较重，用户有充裕时间

**案例：** PC端利用Loading页推广数万元课程

**转化诱因：** 考虑通过大量视频、用户案例、公开承诺、媒体报道等诱因来不断加强用户信赖
- 可以考虑通过在线咨询、顾问式销售等方式辅助

## 场景判断矩阵

| 场景类型 | 转化成本 | 决策成本 | 场景轻重 | 用户时间 | 推荐诱因 |
|----------|----------|----------|----------|----------|----------|
| 场景1 | 低 | 低 | 轻 | 少 | 福利、折扣、效果承诺 |
| 场景2 | 低 | 低 | 重 | 多 | 持续诱惑、销量展示 |
| 场景3 | 高 | 高 | 轻 | 少 | 轻诱因（资料包）、留联系方式 |
| 场景4 | 高 | 高 | 重 | 多 | 视频、案例、公开承诺、媒体报道 |

## 提升ARUP值方法

**ARPU值：** 每用户平均收入

### 常见手段

1. **包邮** - 设置合理的包邮门槛
2. **相关推荐** - 推荐与主商品相关的配件/补充品
3. **满赠/满减/满返** - 达到一定金额赠送/减/返
4. **对比** - 设计一个产品与主打产品作对比
   - 技巧：可以用便宜来突出好，可以用贵来突出性价比
5. **缺货引导** - 普通款缺货，引导选择豪华款

## 工作流步骤

### Step 1: 判断场景类型

- **输入:** product_info, decision_cost, user_persona
- **处理:** 根据决策成本和场景特征判断属于哪类场景
- **输出:** 场景类型判定
- **成功标准:** 明确场景类型和特征

### Step 2: 选择诱因组合

- **输入:** 场景类型, competitive_incentives
- **处理:** 根据场景类型选择匹配的诱因组合
- **输出:** 诱因优先级排序
- **成功标准:** 列出3-5个可实施的诱因

### Step 3: 设计诱因呈现

- **输入:** 选定的诱因, product_info
- **处理:** 设计每个诱因的具体呈现方式（文案/视觉/位置）
- **输出:** 诱因设计方案
- **成功标准:** 每个诱因都有明确的实施方案

### Step 4: 制定ARUP提升方案

- **输入:** 诱因设计方案, product_info, 用户购买路径
- **处理:** 设计提升客单价的配套方案
- **输出:** ARUP提升建议
- **成功标准:** 有具体可落地的提升措施

## Critical Rules

### 必须遵守

1. 先判断场景类型，再选择诱因
2. 诱因必须与用户决策成本匹配
3. 避免过度使用稀缺感（用户会疲劳）
4. 真实承诺，禁止虚假宣传

### 禁止行为

1. 禁止不考虑场景直接套用诱因
2. 禁止同时使用太多诱因造成信息过载
3. 禁止虚假限量（名额永远用不完）
4. 禁止虚假评价和媒体报道
5. 禁止忽视用户反馈数据

## 评估清单

### 评估维度

| 维度 | 标准 | 权重 |
|------|------|------|
| 场景匹配度 | 诱因与场景类型匹配 | 30% |
| 用户价值感知 | 用户能清晰感知诱因价值 | 25% |
| 实施可行性 | 诱因可落地执行 | 20% |
| 真实可信度 | 承诺真实可兑现 | 15% |
| 组合合理性 | 诱因组合不冲突 | 10% |

### 评估标准

- **90-100分：** 五个维度全部达标，诱因设计精准
- **75-89分：** 四个维度达标，设计合理
- **60-74分：** 三个维度达标，需要调整
- **<60分：** 设计有问题，需要重做
