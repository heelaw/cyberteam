# sop-project-fundamentals

## 定位

把一个运营事项整理成 SOP、角色分工、节点检查和推进节奏。

## 输入

- 目标事项
- 当前流程
- 参与角色
- 时间要求
- 可选：异常情况、资源约束、交付标准

## 输出

- SOP 步骤
- 责任分工
- 关键检查点
- 项目推进表

## 只做一件事

它只做“把事情排成能执行的项目”，不做策略大论证。

## 判断规则

1. 先明确结果。
2. 再写出关键步骤。
3. 每步补责任人和完成标准。
4. 再加检查点和异常处理。
5. 最后补推进节奏和里程碑。

## 边界

- 不做结果复盘
- 不做需求洞察
- 不替代业务目标判断
- 没有角色或期限时，不能直接定完整排期

## 反例

- 只有流程标题，没有步骤
- 只有甘特图，没有责任人
- 只有时间表，没有检查点
- 把策略讨论写成 SOP
