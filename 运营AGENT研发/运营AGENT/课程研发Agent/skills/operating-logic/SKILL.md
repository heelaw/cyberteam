# operating-logic

## 定位

把一件运营动作归类到获客、留存或变现，并判断它是否符合当前业务阶段。

## 输入

- 运营动作描述
- 所属业务阶段
- 目标指标
- 当前问题
- 可选：资源、约束、竞品动作

## 输出

- 目标归类
- 优先级判断
- 不该做的动作
- 建议下一步

## 只做一件事

它只负责“给运营动作定业务位”，不负责做完整方案。

## 判断规则

1. 先看动作要拉动什么指标。
2. 再判断它属于获客、留存还是变现。
3. 再看它是否符合当前业务阶段。
4. 最后给出优先级。
5. 一件动作只能有一个主目标。

## 边界

- 不替代项目管理
- 不展开完整增长策略
- 不把所有动作都说成重要
- 业务目标不清时，只能给暂定判断

## 反例

- 只讲“运营很重要”，不讲它服务哪个目标
- 没有业务阶段就直接下判断
- 把推广、留存、变现混成一个动作
- 所有动作都重要，等于没有优先级
