# user-insight-and-demand

## 定位

把用户场景、痛点和需求整理成可验证的定位判断。

## 输入

- 用户对象
- 使用场景
- 行为线索
- 可用证据
- 可选：业务目标、竞品线索、访谈记录

## 输出

- 场景判断
- 痛点描述
- 需求定位
- 优先级建议

## 只做一件事

它只做“把模糊需求变成可验证需求”，不做完整产品方案。

## 判断规则

1. 先写用户是谁。
2. 再写发生了什么场景。
3. 再写表面行为背后的痛点。
4. 再把痛点翻成需求。
5. 最后给出优先级。
6. 优先级必须依赖业务目标和证据。

## 边界

- 不替代用户调研执行
- 不替代竞品全盘分析
- 不直接给最终产品决策
- 没有证据时只给假设，不给定论

## 反例

- 把“用户想要更好体验”当成需求
- 没有证据就下结论
- 场景、痛点、需求混在一起写
- 只凭主观感受排序
