# 输出规范

## 目标

每个课程最终都要落成一个可接手的 skill 包，不只是整理笔记。

## 标准输出目录

```text
运营AGENT/课程研发Agent/skills/<skill-name>/
├── SKILL.md
├── references/
├── assess/
└── evals/
```

## 必交内容

- `SKILL.md`：主入口，说明这个 skill 只做什么动作。
- `references/`：方法、边界、反例、案例、术语。
- `assess/`：验收清单或质量检查。
- `evals/`：题集、样例、输入输出对照。

## 输出约束

- 一个 skill 只回答一个动作。
- 输出必须能直接给下游 agent 接手。
- 不能只写理念，必须给到判断规则或操作模板。
- 不能把归档根当工作根。

## 每个 skill 的最小交付

1. 这一 skill 的一句话定义。
2. 需要什么输入。
3. 会产出什么输出。
4. 边界是什么。
5. 反例是什么。
6. 怎么验收。

## 归档规则

- 新成果先落 `运营AGENT/`。
- `运营AGENT_CODEX/` 只放映射、索引、冻结快照。
- 每个批次结束后再补总索引，不边做边乱补。
