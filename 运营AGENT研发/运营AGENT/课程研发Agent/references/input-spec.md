# 输入规范

## 目标

先判断课程能不能开工，再判断该课程该从哪份材料下手。

## 必需输入

- `course_name`：课程名
- `primary_input`：主输入路径，优先用 `内容增强`、`内容增强-颗粒度对齐版`、`课程大纲`
- `target_skill_direction`：这门课要转成什么单一 skill 动作

## 推荐辅助输入

- `secondary_inputs`：逐字稿、案例、作业、处理报告
- `batch`：所属批次
- `owner`：负责的 worker 或 leader
- `notes`：命名、边界、合并点、风险

## 读入顺序

1. 先看 `00-课程索引.md` 或 `01-章节结构.json`。
2. 再看 `内容增强` 的主文档。
3. 再看 `内容增强-颗粒度对齐版` 做细颗粒度对齐。
4. 最后补读逐字稿，只拿用得上的证据。

## 不能开工的情况

- 只知道课程名，不知道主输入。
- 找不到能支撑 skill 的核心章节。
- 目标技能方向还是“大而空”的总论。
- 输入只有零散笔记，没有任何章节骨架。

## 输入判断原则

- 主输入优先级高于辅助输入。
- 目录型资料不直接当成成品，只当导航。
- 同一课程有多个文件夹名时，按语义统一识别。
- 如果内容太散，先补任务单，不直接写 skill。
