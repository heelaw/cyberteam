# 批次推进手册

## 节奏

1. `clawteam task create` 先把任务排队。
2. `clawteam spawn` 再把 worker 拉进 `tmux/worktree`。
3. `clawteam task wait` 持续盯进度。
4. leader 汇总当前批结果后，再发下一批。

## 第一批

- `03-AI时代下的运营基石课`
- `互联网运营必备的底层心法`
- `SOP与项目管理常识`
- `用户洞察与需求定位`

## 任务粒度

- 一个课程对应一个任务。
- 一个 worker 只负责一个课程包。
- 如果课程薄，就先补输入任务单，再开写。

## 队列建议

- 先排样板课。
- 再排方法论中枢。
- 再排操盘进阶。

## 收口标准

- 每个课程包有单一 skill 动作。
- 每个包都能被新窗口直接接手。
- 每批都能给出下一批的排队建议。
