# 验收标准

## 根包验收

1. 有清晰的课程研发定位。
2. 有明确的输入根、输出根、归档根。
3. 有批次策略和命名规范。
4. 有脚本入口和门禁要求。

## 单课程验收

1. 课程主题明确。
2. 最小 skill 方向清晰。
3. references 能说明方法、边界、反例、案例。
4. assess 能拦住薄内容。
5. scripts 能检查输入或整理任务单。

## skill 验收

- 只做一个动作。
- 有输入条件。
- 有输出模板。
- 有边界与反例。
- 可被新窗口单独接手。

## 不通过条件

- 只列目录，不产出动作。
- 一个 skill 包多个无关任务。
- 没有输入输出映射。
- 不能指导 clawteam 预排队。
