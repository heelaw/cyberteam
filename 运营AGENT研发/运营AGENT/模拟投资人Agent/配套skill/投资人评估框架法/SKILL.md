# 投资人评估框架法

## Skill概述

投资人评估框架法帮助用户理解投资人如何评估项目，提供标准评估框架。

## 输入格式

```json
{
  "type": "object",
  "required": ["project_name", "industry"],
  "properties": {
    "project_name": {
      "type": "string",
      "description": "项目名称"
    },
    "industry": {
      "type": "string",
      "description": "所属行业"
    },
    "stage": {
      "type": "string",
      "enum": ["种子轮", "天使轮", "A轮", "B轮", "C轮+"],
      "description": "融资阶段（可选）"
    },
    "focus_dimensions": {
      "type": "array",
      "items": {"type": "string"},
      "description": "重点关注维度（可选）",
      "enum": ["市场", "产品", "团队", "财务"]
    }
  }
}
```

## 输出格式

```json
{
  "type": "object",
  "properties": {
    "project_name": {"type": "string"},
    "overall_score": {"type": "number", "minimum": 1, "maximum": 5},
    "investment_recommendation": {
      "type": "string",
      "enum": ["强烈推荐", "推荐", "谨慎", "不推荐"]
    },
    "dimension_scores": {
      "type": "object",
      "properties": {
        "market": {"type": "number"},
        "product": {"type": "number"},
        "team": {"type": "number"},
        "finance": {"type": "number"}
      }
    },
    "key_strengths": {"type": "array", "items": {"type": "string"}},
    "key_risks": {"type": "array", "items": {"type": "string"}},
    "improvement_suggestions": {"type": "array", "items": {"type": "string"}}
  }
}
```

## 使用示例

### 示例1：在线教育项目评估

**输入**:
```
项目：少儿编程在线教育
行业：教育科技
阶段：A轮
```

**输出**:
```
综合评分：3.8/5
投资建议：推荐

市场评估 (4.2/5)：市场规模大，竞争激烈
产品评估 (3.5/5)：差异化明显，壁垒待加强
团队评估 (3.8/5)：教育背景强，技术能力中等
财务评估 (3.5/5)：获客成本较高，LTV/CAC需优化

关键优势：政策利好、刚需市场
主要风险：获客成本高、留存待验证
改进建议：优化获客渠道、强化产品壁垒
```

## 错误处理

| 错误场景 | 处理方式 |
|----------|----------|
| 项目信息不足 | 主动询问关键信息（行业、产品、团队、商业模式） |
| 行业不熟悉 | 说明情况，基于通用框架进行分析，提示需行业专家补充 |
| 数据缺失 | 标注"待确认"，基于可获得信息给出初步判断 |
| 信息冲突 | 指出矛盾点，要求澄清后再评估 |

## 独特个性

本Skill的特色在于**模拟真实投资人的思维框架**，不是简单罗列维度，而是按照专业投资机构的尽调流程（DD）进行系统性评估，帮助用户建立投资人视角的商业洞察力。

## 投资人关注的核心维度

### 1. 市场评估

- **市场规模（TAM）**：总市场容量
- **可触达市场（SAM）**：可服务市场
- **可获得市场（SOM）**：可获得市场
- **市场增长率**：行业增速
- **竞争格局**：市场集中度

### 2. 产品评估

- **产品价值**：解决什么痛点
- **差异化**：与竞品的差异
- **壁垒**：护城河是什么
- **产品成熟度**：MVP还是成熟产品

### 3. 团队评估

- **团队背景**：核心成员经历
- **执行能力**：历史战绩
- **团队完整性**：缺什么能力

### 4. 财务评估

- **商业模式**：如何赚钱
- **Unit Economics**：LTV/CAC
- **增长效率**：增长成本
- **估值逻辑**：估值依据

## 工作流程

### Step 1：收集信息

根据四大维度收集项目信息。

### Step 2：逐项评估

对每个维度进行评估打分。

### Step 3：综合判断

给出综合评估结论。

### Step 4：改进建议

指出短板和改进方向。

## Critical Rules

### 必须遵守
1. 客观真实评估，不带主观偏好
2. 关注数据和证据，用数据说话
3. 区分事实和观点，标注数据来源
4. 考虑行业特点和业务阶段
5. 给出明确的评分和结论

### 禁止行为
1. 不夸大项目优点或潜力
2. 不忽略潜在风险和问题
3. 不脱离行业背景进行评估
4. 不为迎合投资人而刻意美化
5. 不忽视财务数据的真实性和完整性
