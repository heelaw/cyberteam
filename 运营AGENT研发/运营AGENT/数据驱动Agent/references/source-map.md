# 原始资料映射

这个文件服务于数据驱动Agent：它只负责说明原始输入怎么被拆成最小判断单元，不负责写控制逻辑。

## 1. 主来源

这个 Agent 不是凭空选题出来的，而是从运营数据判断类原始资料里，按最小判断单元重组出来的。

| 原始资料 | 主要贡献 | 被重组到哪个 Skill |
|---|---|---|
| `31-几类常见数据指标的拆解思路.md` | 指标定义、拆解层级、结果指标与过程指标 | 活动效果评估 / 产品状态评估 |
| `33-根据渠道数据，筛选优质渠道.md` | ROI、渠道质量、漏斗判断 | 活动效果评估 / 产品状态评估 |
| `S7.09：活动的数据监测和优化.md` | 活动目标、数据漏斗、传播与留存、复盘动作 | 活动效果评估 |
| `深刻理解4种常见的运营“数据漏斗”.md` | 流程梳理、漏斗拆解、业务路径、异常定位 | 产品问题诊断 / 活动效果评估 |
| `55-本章总结.md` | 复盘框架、二次传播、执行落地、系统判断 | 三个 Skill 的共用底座 |

## 2. 三个最小能力单元

这些原始资料被重组为三类最小判断单元：

- `活动效果评估`：判断一次活动是否达成目标、哪里有效、哪里浪费、是否值得复用。
- `产品问题诊断`：从异常现象中找到最可能根因，并给出验证与止损方案。
- `产品状态评估`：对产品做系统体检，判断健康度、投入优先级和改进方向。

## 3. 原始资料 → 任务路径

| 原始任务线索 | 首选 Skill | 说明 |
|---|---|---|
| 活动结果如何、渠道哪里好 | 活动效果评估 | 先看目标，再看成本，再看传播与行为 |
| 指标掉了、用户少了、转化差了 | 产品问题诊断 | 先排口径，再拆维度，再找根因 |
| 产品现在值不值得投 | 产品状态评估 | 先做体检，再给投入建议 |

## 4. 排除项

- 不再把文案写作、标题技巧、表达润色这类内容混进来。
- 不把原始资料的叙述方式直接搬进 Agent 主文档。
- 不把不同任务的判断逻辑写成一锅粥。
- 不把“来源”写成“学习课程引用”一类表述。
