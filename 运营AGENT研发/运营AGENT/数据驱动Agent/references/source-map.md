# 原始资料映射

这个文件只负责一件事：说明原始资料如何被拆成最小判断单元。

## 1. 主来源

| 原始资料类型 | 主要贡献 | 被重组到哪个 Skill |
|---|---|---|
| 指标拆解思路 | 指标定义、拆解层级、结果指标与过程指标 | 活动效果评估 / 产品状态评估 |
| 渠道数据与优质渠道筛选 | ROI、渠道质量、漏斗判断 | 活动效果评估 / 产品状态评估 |
| 活动监测与优化 | 活动目标、数据漏斗、传播与留存、复盘动作 | 活动效果评估 |
| 数据漏斗与异常定位 | 流程梳理、漏斗拆解、业务路径、异常定位 | 产品问题诊断 / 活动效果评估 |
| 复盘总结页 | 复盘框架、二次传播、执行落地、系统判断 | 三个 Skill 的共用底座 |

## 2. 三个最小能力单元

- `活动效果评估`：判断一次活动是否达成目标、哪里有效、哪里浪费、是否值得复用。
- `产品问题诊断`：从异常现象中找到最可能根因，并给出验证与止损方案。
- `产品状态评估`：对产品做系统体检，判断健康度、投入优先级和改进方向。

## 3. 原始线索 → 路由路径

| 原始任务线索 | 首选 Skill | 说明 |
|---|---|---|
| 活动结果如何、渠道哪里好 | 活动效果评估 | 先看目标，再看成本，再看传播与行为 |
| 指标掉了、用户少了、转化差了 | 产品问题诊断 | 先排口径，再拆维度，再找根因 |
| 产品现在值不值得投 | 产品状态评估 | 先做体检，再给投入建议 |

## 4. 排除项

- 不再把表达训练、标题技巧、写作润色混进来。
- 不把原始资料的叙述方式直接搬进 Agent 主文档。
- 不把不同任务的判断逻辑写成一锅粥。
- 不把来源写成“引用课程”一类表述。
