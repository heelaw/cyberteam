# 诊断检查清单

## 必查项

- [ ] 指标定义清楚
- [ ] 开始时间清楚
- [ ] 影响范围清楚
- [ ] 严重程度清楚
- [ ] 时间维度拆了
- [ ] 渠道维度拆了
- [ ] 用户维度拆了
- [ ] 功能维度拆了
- [ ] 版本 / 地域 / 设备按需补充
- [ ] 已排除口径问题
- [ ] 已列出假设
- [ ] 已给出验证方法
- [ ] 已给出止损动作
- [ ] 已给出恢复动作
- [ ] 已给出监控指标

## 红线

- 不能把现象当根因。
- 不能跳过口径检查。
- 不能只看局部不看全局。
- 不能没验证就宣布结论。
- 不能只给技术结论不管业务影响。

## 输出最低标准

- 一个主根因判断。
- 至少两个备选假设。
- 至少一个立即止损动作。
- 至少一个后续监控指标。

## 附加检查

- [ ] 是否能说出主因和次因
- [ ] 是否有一条立即止损动作
- [ ] 是否有一个回看时间点
- [ ] 是否能让执行同学立刻动手
