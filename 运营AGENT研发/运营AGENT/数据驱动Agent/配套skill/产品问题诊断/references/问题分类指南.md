# 问题分类指南

## 一、产品问题

### 常见信号

- 某个功能使用率下降。
- 完整流程中某一步流失陡增。
- 老用户比新用户掉得更厉害。

### 常见验证

- 看路径漏斗。
- 看页面停留和跳出。
- 看用户反馈。

## 二、技术问题

### 常见信号

- 某版本上线后指标突变。
- 某端、某浏览器、某设备异常。
- 日志或错误率异常。

### 常见验证

- 看发布时间和异常起点是否一致。
- 看接口错误率。
- 看性能、崩溃和超时。

## 三、运营问题

### 常见信号

- 触达量下降。
- 活动后链路变差。
- 某渠道突然失效。

### 常见验证

- 看活动节奏。
- 看投放与触达。
- 看内容命中率。

## 四、市场问题

### 常见信号

- 全平台同时走弱。
- 竞品动作明显。
- 用户意愿变化，但产品未变。

### 常见验证

- 看竞品。
- 看价格。
- 看外部环境。

## 五、数据问题

### 常见信号

- 某指标突然断崖，但业务没感觉到。
- 不同表之间口径打架。
- 某端数据明显缺失。

### 常见验证

- 对账。
- 查埋点。
- 查延迟。
- 查重复和漏数。

## 分类原则

- 先排数据，再排技术，再排产品，最后排外部因素。
- 不要看到下降就默认是产品坏了。
- 不要看到单点异常就默认全局崩了。

## 六、快速分流口诀

- 先排数据，再排技术，再排产品，再排外部。
- 单渠道异常先看局部，不要直接全局化。
- 单日跳水和连续下跌不是同一种问题。
- 版本同步异常时，优先查发布。
