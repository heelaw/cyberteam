# 五维度评估详解

## 评估目的差异

### 投入决策评估
**目标**：判断是否值得继续投入资源

侧重点：
- 投入回报周期（ROI、LTV/CAC）
- 增长天花板判断
- 边际投入产出比
- 竞争壁垒是否形成

产出要求：
- 明确"继续投 / 收缩 / 止损 / 放弃"结论
- 给出投入优先级和资源量级
- 预估止损或突破的时间节点

### 问题排查评估
**目标**：定位当前问题的根本原因

侧重点：
- 各维度下钻分析
- 问题维度 vs 问题环节区分
- 是单点问题还是系统性问题
- 拐点时间和触发条件

产出要求：
- 问题定位到具体维度和指标
- 给出问题根因假设
- 建议排查顺序（P0先查什么）

### 常规体检评估
**目标**：周期性健康度检查，不要求深度下钻

侧重点：
- 五维度均衡覆盖
- 与历史周期对比
- 与行业/对标对比
- 异动提醒

产出要求：
- 五维度打分
- 主要变化点标注
- 建议关注点（不要求完整行动计划）

## B2B 与 B2C 评估口径区别

### B2C 产品

#### 核心指标体系

| 指标 | 定义 | 计算口径 | 健康阈值 |
|------|------|----------|----------|
| DAU/MAU | 日活/月活比 | DAU ÷ MAU × 100% | ≥20% 为健康，<10% 为危险 |
| 次日留存率 | 新用户次日回访比例 | D+1回访 ÷ D新增 × 100% | ≥40% 为健康 |
| 7日复购率 | 7天内有复购的用户占比 | 7日复购用户 ÷ 7日活跃用户 | ≥15% 为健康 |
| 30日复购率 | 30天内有复购的用户占比 | 30日复购用户 ÷ 30日活跃用户 | ≥30% 为健康 |
| LTV/CAC | 用户生命周期价值/获客成本 | LTV ÷ CAC | ≥3 为健康，<1 为危险 |
| 单渠道CAC上限 | 各渠道获客成本上限 | 渠道总成本 ÷ 渠道新增 | 不超过LTV的33% |
| 渠道集中度 | 最大单一渠道占比 | 最大渠道用户 ÷ 总新增 | <60% 为健康，>80% 为危险 |
| 沉默用户占比 | 30天无活跃用户占比 | 30天沉默 ÷ MAU | <15% 为健康，>30% 为危险 |

#### B2C 五维度重点

**渠道质量**
- 重点：流量来源分布、渠道CAC、渠道转化率、渠道ROI
- 断点识别：哪个渠道开始失效
- 结构风险：是否过度依赖单一渠道

**功能使用**
- 重点：核心功能渗透率、功能使用深度、功能路径转化
- 断点识别：用户在哪一步流失
- 高价值功能：哪些功能与复购正相关

**复购率**
- 重点：7/30/90日复购率、复购周期、复购频次
- 断点识别：复购周期是否在拉长
- LTV健康度：LTV/CAC是否在恶化

**活跃度**
- 重点：DAU/MAU、活跃分层、沉默流失占比
- 断点识别：哪类用户在沉默
- 召回效率：沉默用户召回率

**数据趋势**
- 重点：30/90/180天趋势、环比同比、拐点时间
- 拐点识别：何时开始下滑
- 质量判断：规模涨 vs 效率涨是否同步

### B2B 产品

#### 核心指标体系

| 指标 | 定义 | 计算口径 | 健康阈值 |
|------|------|----------|----------|
| 续费率 | 合同到期续约比例 | 续约客户数 ÷ 到期客户数 × 100% | ≥85% 为健康，<75% 为危险 |
| NRR（净收入留存率） | 存量客户收入变化率 | (期末ARR - 流失ARR + 增购ARR) ÷ 期初ARR × 100% | ≥110% 为健康，<100% 为危险 |
| 增购率 | 存量客户增购比例 | 增购客户数 ÷ 活跃客户数 × 100% | ≥15% 为健康 |
| 核心功能渗透率 | 使用核心功能的客户占比 | 使用核心功能企业 ÷ 活跃企业 × 100% | ≥60% 为健康 |
| 客户健康度得分 | 综合使用深度评分 | 基于DAU、功能使用、增购意图加权 | ≥70分 为健康 |
| 流失预警率 | 30天内可能流失客户占比 | 预警客户 ÷ 活跃客户 × 100% | <10% 为健康，>20% 为危险 |
| 席位扩张率 | 平均席位变化 | 期末席位 ÷ 期初席位 - 1 | ≥5% 为健康 |
| LTV | 客户生命周期价值 | 平均月ARPU × 平均生命周期（月） | 目标 ≥ 12个月ARR |

#### B2B 五维度重点

**渠道质量**
- 重点：直销/渠道/线上占比、各渠道客户质量（规模、续费率）
- 断点识别：哪个渠道客户质量最差
- 结构风险：是否依赖低质量渠道

**功能使用**
- 重点：核心功能 vs 高级功能渗透率、席位利用率
- 断点识别：哪些功能未被激活
- 价值实现：功能使用与续费相关性

**复购率（续费）**
- 重点：续费率、NRR、增购率、复购周期
- 断点识别：续费下滑拐点
- 流失原因：价格、功能、服务、竞争

**活跃度**
- 重点：企业活跃率、DAU/MAU、活跃席位占比
- 断点识别：哪些企业在降频
- 预警信号：低活跃企业是否在扩增

**数据趋势**
- 重点：续费率趋势、NRR趋势、客户健康度趋势
- 拐点识别：何时续费开始下滑
- 系统性风险：是否行业性下滑

## 五维度定义与输入要求

### 维度一：渠道质量

#### 定义
评估产品获客渠道的结构性健康度，包括流量分布、CAC效率和渠道ROI。

#### 必要输入
- 各渠道流量占比（总和=100%）
- 各渠道转化率（流量→注册→付费）
- 各渠道CAC
- 各渠道用户质量（留存、复购）
- 行业平均CAC对标

#### 评估方法

```
步骤1：计算渠道集中度
最大单一渠道占比 = 最大渠道用户 ÷ 总新增用户 × 100%

步骤2：计算渠道效率矩阵
| 渠道 | 流量占比 | 转化率 | CAC | 留存率 | ROI指数 |
各渠道按CAC排序，标记高于LTV/3的渠道为"警戒"

步骤3：渠道质量评分
- 集中度健康（<60%）：+20分
- 各渠道CAC均健康：+30分
- 渠道留存均达标：+30分
- 有2个以上高效渠道：+20分
总分100分封顶
```

#### 健康阈值

| 指标 | 绿（健康） | 黄（亚健康） | 红（危险） |
|------|-----------|-------------|-----------|
| 最大渠道集中度 | <60% | 60-80% | >80% |
| 渠道CAC健康率 | >80%渠道达标 | 50-80%达标 | <50%达标 |
| 新渠道拓展速度 | 有增量渠道 | 依赖存量渠道 | 无新渠道 |
| 渠道与产品匹配 | 高度匹配 | 中度匹配 | 严重不匹配 |

#### 拐点判断
- 某渠道CAC连续2个月上涨 >15% → 该渠道开始失效
- 某渠道用户质量（留存）下降 >10% → 该渠道带来的是垃圾用户
- 集中度单月上涨 >15% → 单一渠道依赖风险加剧

#### 结构风险识别
- 单渠道占比 >80% → 渠道挂了产品就挂了
- CAC > LTV/3 → 该渠道永远回不了本
- 渠道效率持续下降但还在增量投入 → 吸毒式增长

### 维度二：功能使用

#### 定义
评估产品核心功能是否被正确使用，以及功能路径是否顺畅。

#### 必要输入
- 各功能渗透率（使用用户 ÷ 活跃用户）
- 各功能使用深度（次数/人、时长/人）
- 功能路径转化漏斗（进入→使用→核心→下游）
- 功能与留存/复购相关性数据

#### 评估方法

```
步骤1：功能分层
- 核心功能（必须用）：如IM、搜索、下单
- 高价值功能（用了会复购）：如会员、推荐、脚本
- 基础功能（用了不亏）：如浏览、收藏
- 沉睡功能（从未激活）：如某些高级功能

步骤2：功能健康度评分
核心功能渗透率 ≥ 60%：+40分
核心功能深度达标（次数/时长高于均值）：+30分
高价值功能被激活：+20分
无大量沉睡功能：+10分
总分100分封顶
```

#### 健康阈值

| 指标 | 绿（健康） | 黄（亚健康） | 红（危险） |
|------|-----------|-------------|-----------|
| 核心功能渗透率 | ≥60% | 40-60% | <40% |
| 功能使用深度 | 同比持平或增长 | 下降 <15% | 下降 ≥15% |
| 功能路径转化 | 漏斗各步流失 <30% | 某步流失 30-50% | 某步流失 >50% |
| 高价值功能激活率 | ≥30% 用户激活 | 15-30% | <15% |

#### 拐点判断
- 核心功能渗透率单月下降 >10% → 产品价值在崩塌
- 功能路径某步流失率突然上涨 >20% → 该路径有阻断
- 高活跃用户突然减少某功能使用 >30% → 产品可能在被抛弃

#### 结构风险识别
- 核心功能渗透率 <40% → 产品价值没有被用户感知
- 功能使用深度持续下降 → 用户在减少使用产品
- 大量功能从未被激活 → 功能研发浪费，且可能是伪需求

### 维度三：复购率

#### 定义
评估用户是否持续产生价值，包括复购行为和LTV健康度。

#### 必要输入

**B2C必填**
- 7日/30日/90日复购率
- 复购周期（平均多久复购一次）
- 复购频次（平均复购几次）
- LTV估算
- CAC数据

**B2B必填**
- 续费率（按合同周期）
- NRR（净收入留存率）
- 增购率
- LTV估算

#### 评估方法

**B2C复购评分**
```
基础分：各周期复购率 × 权重
7日复购率 × 20% + 30日复购率 × 35% + 90日复购率 × 45%

LTV/CAC加成：
LTV/CAC ≥ 5：+20分
LTV/CAC ≥ 3：+10分
LTV/CAC ≥ 1：+0分
LTV/CAC < 1：扣30分

复购周期健康度：
周期稳定或缩短：+10分
周期拉长 <20%：+0分
周期拉长 ≥20%：扣20分
总分100分封顶
```

**B2B续费评分**
```
基础分：续费率 × 100 × 0.6
NRR加成：
NRR ≥ 120%：+30分
NRR ≥ 110%：+20分
NRR ≥ 100%：+10分
NRR < 100%：扣30分

增购率加成：
增购率 ≥ 20%：+15分
增购率 ≥ 15%：+10分
增购率 ≥ 10%：+5分
增购率 < 10%：+0分
总分100分封顶
```

#### 健康阈值

| 指标 | 绿（健康） | 黄（亚健康） | 红（危险） |
|------|-----------|-------------|-----------|
| B2C 30日复购率 | ≥30% | 20-30% | <20% |
| B2C LTV/CAC | ≥3 | 1.5-3 | <1.5 |
| B2C 复购周期 | 稳定或缩短 | 拉长 <20% | 拉长 ≥20% |
| B2B 续费率 | ≥85% | 75-85% | <75% |
| B2B NRR | ≥110% | 100-110% | <100% |
| B2B 增购率 | ≥15% | 10-15% | <10% |

#### 拐点判断
- 复购率环比下降 >10% → 用户价值在衰减
- LTV/CAC连续3个月下降 → 产品盈利模型在恶化
- 复购周期拉长 >20% → 用户需求在减弱

#### 结构风险识别
- LTV/CAC < 1 → 产品永远无法盈利
- 复购率持续下滑且无新增补位 → 存量用户在加速流失
- 续费率 < 75%（B2B）→ 产品基础价值不被认可

### 维度四：活跃度

#### 定义
评估用户群体的活跃程度和健康分层。

#### 必要输入

**B2C必填**
- DAU、MAU
- DAU/MAU
- 活跃用户分层（高/中/低/沉默）
- 沉默用户召回率

**B2B必填**
- 企业活跃率
- 企业健康度得分分布
- 流失预警企业占比
- 活跃席位变化

#### 评估方法

**B2C活跃评分**
```
DAU/MAU得分：
≥25%：50分
≥20%：40分
≥15%：30分
≥10%：20分
<10%：10分

分层健康度得分：
高活跃占比 ≥ 20%：+20分
中活跃占比 30-50%：+15分
低活跃占比 < 30%：+10分
沉默占比 < 15%：+15分
沉默占比 ≥ 15%：+0分
沉默占比 ≥ 30%：扣20分
总分100分封顶
```

**B2B活跃评分**
```
企业活跃率得分：
≥85%：50分
≥75%：40分
≥65%：30分
<65%：20分

健康度分布得分：
高健康企业占比 ≥ 30%：+20分
中健康企业占比合理：+15分
预警企业占比 < 10%：+15分
预警企业占比 ≥ 10%：+0分
预警企业占比 ≥ 20%：扣30分
总分100分封顶
```

#### 健康阈值

| 指标 | 绿（健康） | 黄（亚健康） | 红（危险） |
|------|-----------|-------------|-----------|
| B2C DAU/MAU | ≥20% | 15-20% | <15% |
| B2C 沉默占比 | <15% | 15-25% | ≥25% |
| B2C 活跃趋势 | 稳定或增长 | 下降 <10% | 下降 ≥10% |
| B2B 企业活跃率 | ≥80% | 70-80% | <70% |
| B2B 流失预警率 | <10% | 10-20% | ≥20% |
| B2B 预警企业趋势 | 下降 | 持平 | 上升 |

#### 拐点判断
- 沉默用户占比单月上涨 >5% → 用户开始批量流失
- DAU连续下滑 >15% → 产品进入衰退通道
- 高活跃用户占比下降 >10% → 核心用户在流失

#### 结构风险识别
- 活跃用户总量下跌但高活跃占比上升 → 说明产品在失去大众用户
- 沉默占比 > 30% → 超过1/3用户已放弃
- 预警企业占比持续上升 → 存量收入在倒计时

### 维度五：数据趋势

#### 定义
评估产品在时间维度上的变化方向和拐点。

#### 必要输入
- 至少3个月（建议6个月）的连续数据
- 各维度指标月度变化
- 同比数据（如果有12个月）
- 版本/活动/市场事件时间线

#### 评估方法

```
趋势判断流程：

步骤1：计算各维度变化率
变化率 = (本期值 - 上期值) ÷ 上期值 × 100%

步骤2：判断趋势方向
- 上升：变化率 > 5%
- 持平：变化率 -5% ~ +5%
- 下降：变化率 < -5%

步骤3：识别拐点
拐点 = 趋势方向发生变化的月份

步骤4：判断拐点性质
- 单点异动：仅1个维度变化，其他维度未跟随
- 联动下滑：≥2个维度同向恶化
- 系统性风险：≥3个维度同向恶化，且持续≥2个月

步骤5：趋势健康度评分
无拐点或偶发单点异动：+80-100分
2个以内维度轻微下滑：+60-80分
联动下滑或拐点未止：+40-60分
系统性风险且未止血：<40分
```

#### 拐点判断标准

**上升拐点（好事）**
- 某指标突然大涨 >30%，且有明确原因（活动、版本）
- 多个维度同时出现上升趋势
- 上升趋势持续 ≥3个月

**下降拐点（坏事）**
- 某指标突然下跌 >20%，且无明确正面原因
- 2个以上维度同向下滑
- 下滑趋势持续 ≥2个月

**平台期判断**
- 各指标在±5%内波动 ≥3个月
- 不是下跌，而是稳定的停滞
- 需要判断是增长天花板还是增长蓄力期

#### 结构风险识别
- 指标在涨但效率在跌（如DAU涨但DAU/MAU跌）→ 增长质量差
- 多个维度同跌 → 系统性问题，非局部问题
- 拐点出现后无有效止血措施 → 危机将持续

## 健康等级判定规则

### 等级定义

| 等级 | 定义 | 维度特征 | 建议 |
|------|------|----------|------|
| 健康 | 产品整体正向，各维度稳定或向好 | ≥3个维度绿，无红 | 继续加码，监控效率 |
| 亚健康 | 部分维度弱，但核心指标撑得住 | 2个以内维度红，但无非红 | 先修短板，防止恶化 |
| 风险 | 多个维度开始下滑，或1个核心维度明显差 | ≥2个维度红，或1个核心维度长期红 | 优先止损，动作要快 |
| 危机 | 核心维度全面同向下滑 | ≥3个维度红，或出现系统性下滑 | 立即介入，甚至考虑止损 |

### 综合评分计算

```
五维度等权重计分，每个维度100分。

综合得分 = (渠道质量 + 功能使用 + 复购率 + 活跃度 + 趋势) ÷ 5

等级映射：
综合得分 ≥ 85：健康
综合得分 70-84：亚健康
综合得分 55-69：风险
综合得分 < 55：危机
```

### 健康等级不唯分数论

以下情况即使分数达标也要降级：

1. **渠道集中度 > 80%** → 即使综合得分 ≥ 85，也要降到亚健康
2. **LTV/CAC < 1** → 即使其他维度好，也要降到风险或危机
3. **沉默占比 ≥ 30%** → 即使综合得分高，也要降到亚健康
4. **续费率 < 75%（B2B）** → 即使综合得分高，也要降到风险
5. **系统性下滑（≥3维度同跌）** → 即使综合得分看起来还行，也要降到风险或危机

## 典型案例

### 典型健康案例（B2C电商App）

```
产品：某中型电商App
数据状态：
- DAU 150万，MAU 800万，DAU/MAU 18.75%
- App渠道占55%，CAC 18元；小程序占30%，CAC 12元
- 7日复购率15%，30日复购率35%，90日复购率52%
- 高活跃25%，中活跃35%，低活跃30%，沉默10%
- 近30天DAU趋势 +5%

五维度得分：
- 渠道质量：78分（小程序优秀，Web弱）
- 功能使用：85分（核心路径顺畅）
- 复购率：88分（90日复购52%优秀）
- 活跃度：75分（DAU/MAU略低于行业均值）
- 趋势：80分（稳中有升）

综合得分：81分 → 判定：亚健康

核心问题：
1. DAU/MAU 18.75%低于行业均值20%，沉默占比10%偏高
2. Web渠道CAC 25元过高

行动建议：
P0：优化沉默用户召回策略，目标沉默占比降至7%
P1：关停低效Web渠道，节省成本投入App和小程序
P2：加强会员体系，提升DAU/MAU至20%以上
```

### 典型亚健康案例（B2B SaaS）

```
产品：某企业协作SaaS
数据状态：
- 企业客户800家，活跃650家，活跃率81%
- ARR 4800万，续费率82%，NRR 110%
- 直销渠道占45%，续费率88%；渠道代理35%，续费率75%
- IM使用率95%，在线文档80%，项目管理45%，数据分析20%
- 高活跃企业30%，低活跃20%，即将流失10%

五维度得分：
- 渠道质量：75分（渠道客户质量差异大）
- 功能使用：72分（高级功能渗透低）
- 复购率：88分（NRR110%优秀）
- 活跃度：70分（10%企业即将流失）
- 趋势：85分（整体稳定）

综合得分：78分 → 判定：亚健康

核心问题：
1. 渠道代理续费率仅75%，低于均值85%
2. 10%企业即将流失，流失预警率高
3. 高级功能（数据分析）使用率仅20%

行动建议：
P0：建立流失预警机制，对即将流失企业主动触达
P0：重新评估渠道代理准入标准，提升渠道客户质量
P1：推进高级功能激活计划，提供培训和案例
```

### 典型风险案例

```
产品：某社交App
数据状态：
- DAU 80万，MAU 600万，DAU/MAU 13.3%
- 单一渠道占比75%，CAC 35元（偏高）
- 7日复购率8%，30日复购率18%
- 沉默占比28%
- 近90天DAU下降18%

五维度得分：
- 渠道质量：45分（集中度过高，CAC偏高）
- 功能使用：55分（核心功能渗透低）
- 复购率：40分（复购率偏低）
- 活跃度：35分（DAU/MAU低，沉默高）
- 趋势：30分（连续下滑）

综合得分：41分 → 判定：危机

核心问题：
1. 单渠道占比75%，渠道风险极高
2. 沉默占比28%，近1/3用户已流失
3. DAU连续3个月下滑，无止血迹象
4. LTV/CAC < 1，获客模型不健康

行动建议：
P0：立即止损，停止低效渠道投放
P0：启动沉默用户召回专项行动
P1：诊断DAU下滑根因（竞品、功能、体验）
P2：重新评估产品核心价值是否还成立
```

### 典型危机案例

```
产品：某工具类App
数据状态：
- MAU 200万，连续6个月下跌
- 续费率62%（B2B），NRR 92%
- 核心功能使用率从55%跌至32%
- 流失预警率35%
- DAU近180天下跌45%

五维度得分：
- 渠道质量：30分（无有效渠道）
- 功能使用：25分（功能在失效）
- 复购率：20分（续费崩塌）
- 活跃度：20分（用户大批流失）
- 趋势：10分（系统性崩溃）

综合得分：21分 → 判定：危机

核心问题：
1. 续费率62%，远低于健康线85%
2. 核心功能使用率腰斩，产品价值在消失
3. 流失预警率35%，存量收入在加速流失
4. 180天DAU下跌45%，没有任何止血迹象

行动建议：
危机状态，建议管理层介入：
1. 立即成立专项止损小组
2. 深度调研流失原因（市场/产品/竞争）
3. 评估是否还有投入价值，还是应该止损
4. 如果继续投，明确止损线和突破条件
```

## 绝对禁止行为

1. **禁止只看现状不看趋势** → 必须看至少3个月趋势
2. **禁止只看流量不看价值** → 必须看LTV/CAC、复购、续费
3. **禁止把单一维度当总分** → 五维度缺一不可
4. **禁止没有对标就下评级** → 必须有行业/历史对标
5. **禁止只给"继续观察"** → 必须有明确动作
6. **禁止B2B/B2C混用指标** → 口径必须区分清楚
7. **禁止唯分数论** → 结构风险要降级
8. **禁止不看拐点时间** → 要判断是偶发还是趋势

## 六、新人执行顺序

1. 先判产品类型是 B2B 还是 B2C。
2. 再确认评估目的。
3. 再补五维数据和对标。
4. 再看有没有结构性风险。
5. 最后下健康等级和投入建议。

## 七、评估前 QA

- 是否有对标数据。
- 是否能区分 B2B / B2C。
- 是否覆盖五维度。
- 是否能说出结构性风险。
- 是否能给出继续投、先修还是止损。
