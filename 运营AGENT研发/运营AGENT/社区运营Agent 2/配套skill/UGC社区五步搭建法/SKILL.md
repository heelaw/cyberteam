## 核心定位
通过五步标准化流程，帮助运营者从0到1搭建UGC社区，明确社区定位、设计社区结构、构建激励机制、建立社区规则，最终实现社区的可持续运营。

## Critical Rules

### 必须遵守
1. **社区定位必须差异化** - 不能简单复制竞品，要明确说出"我们的社区不同在哪里"
2. **目标用户必须精准** - 避免"面向所有人"，必须具体到某一类人群
3. **激励机制必须可落地** - 不能只有概念，要有具体的积分规则、等级权益
4. **规则必须可执行** - 每条规则都要有对应的奖惩措施，不能有"僵尸规则"
5. **执行计划必须有时间节点** - 每个阶段都要有明确的起止时间和验收标准

### 禁止行为
1. **禁止定位模糊** - 不能写"面向所有热爱生活的人"这类模糊描述
2. **禁止激励机制画饼** - 不能写"更多权益待解锁"这种空头支票
3. **禁止规则空洞** - 不能写"鼓励正向内容"而不说明如何鼓励
4. **禁止无差异化** - 不能与竞品高度雷同，没有独特卖点
5. **禁止计划空洞** - 不能只有"引入用户"没有具体执行动作

## workflow_steps

### Step 1：明确社区定位
**输入**：
- 目标用户描述
- 核心价值主张
- 同类竞品信息

**处理**：
1. 分析目标用户的特征、需求、痛点
2. 梳理差异化定位维度（内容方向/调性/玩法/服务）
3. 确定社区使命和愿景
4. 提炼社区Slogan

**输出**：
- 社区定位文档（含差异化分析、目标用户画像、使命愿景、Slogan）
- 社区定位一句话描述

**成功标准**：
- 能用一句话清晰描述社区定位
- 差异化点明确，不与头部竞品雷同
- 目标用户画像具体到可描述"是什么样的一群人"

### Step 2：设计社区板块结构
**输入**：
- 社区定位文档
- 目标用户内容消费习惯

**处理**：
1. 确定内容分类维度（主题/形式/场景）
2. 设计板块层级（一级板块/二级板块）
3. 规划入口结构和导航逻辑
4. 明确各板块的定位和内容要求

**输出**：
- 社区板块结构图
- 各板块定位说明文档
- 板块内容规范

**成功标准**：
- 板块数量适中（一般5-9个一级板块）
- 各板块定位清晰不重叠
- 用户能快速找到所需内容

### Step 3：构建用户激励机制
**输入**：
- 社区定位文档
- 社区板块结构

**处理**：
1. 设计用户成长体系（等级定义/升级规则/权益体系）
2. 制定内容激励规则（优质内容判定标准/奖励标准）
3. 设计互动激励规则（回复/点赞/收藏等行为激励）
4. 规划荣誉展示体系（排行榜/徽章/荣誉称号）

**输出**：
- 用户成长体系文档（含等级表/权益表/升级规则）
- 激励规则细则
- 荣誉体系设计

**成功标准**：
- 用户知道自己如何成长
- 激励规则具体可量化
- 高等级用户享有明确权益

### Step 4：制定社区规则体系
**输入**：
- 社区定位文档
- 激励机制文档

**处理**：
1. 梳理内容红线（政治/色情/暴力/虚假等）
2. 制定行为规范（正向引导/禁止行为）
3. 设计违规处理流程（警告/禁言/封号标准）
4. 制定申诉机制
5. 编写社区公约

**输出**：
- 社区规则全文
- 违规处理细则
- 社区公约（用户端公示版）

**成功标准**：
- 规则覆盖常见违规场景
- 奖惩标准明确可执行
- 用户能理解并认同规则

### Step 5：制定执行计划
**输入**：
- 社区定位文档
- 社区结构文档
- 激励机制文档
- 规则体系文档

**处理**：
1. 划分发展阶段（冷启动期/成长期/稳定期）
2. 确定各阶段核心指标
3. 制定详细执行动作
4. 资源配置规划
5. 风险预案

**输出**：
- 分阶段执行计划（含时间节点/里程碑/验收标准）
- 核心指标体系
- 风险预案文档

**成功标准**：
- 每个阶段有明确起止时间
- 每个阶段有可量化的验收标准
- 有应对常见风险的预案
