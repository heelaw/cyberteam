# [二] 产品MVP设计该怎么做（下）

上一部分内容中，我们看到了对于MVP的若干理解误区，也通过很多案例学到了什么是MVP。那么这一部分，我们来展开看看MVP设计的三部曲是什么。同时，布老师还根据产业互联网的发展趋势，特意补充了B端产品MVP的方式。

MVP设计其实主要包含以下三个部分：用户需求分析，着陆页设计，以及关键页面原型。

对于用户需求分析这个部分，我们不仅需要还原典型的用户故事，同时还得挖掘出用户的核心的一个到两个的需求。

在用户需求分析的过程中，我们需要记住的是：在MVP阶段不纠结。不应该为了思考哪个是用户的核心需求花太长的时间，应该尽快投入试验。

着陆页设计这个部分，我们需要遵循的过程是：明确受众与出现场景——设计制作多个版本的物料——线上线下实验。

着陆页设计过程中，我们需要记住的是：好的转化都是测出来的。

在Demo设计的过程，不仅需要设计关键页面原型，还需要设计关键的流程，流程是好MVP设计的前提。

我们在绘制demo过程中，要尽可能用真实的图片和真实的文案，要让用户觉得是真的产品，而不是一个半成品。
