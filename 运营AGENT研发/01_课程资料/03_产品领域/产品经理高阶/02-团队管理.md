# 02-团队管理


# 【一】如何招到合适的人

作为团队的负责人，经常会把很多精力放到团队的组建上。然而，要招到合适的人加入团队并不容易，并不是说你向HR简单提个需求，自己什么都不用操心，HR就能搞定。人才是团队发展的核心，所以在招人上面，需要花大量的心思。

那么，怎么招到合适的人加入团队呢？我们来看一下布老师怎么说。

## 招到合适的人才需要紧扣两个要点

1. 撰写一份好的JD
2. 进行有效的结构化面试

## JD主要包括两方面：岗位职责和能力要求

对于岗位职责，我们要选择宽口还是窄口，以表现对于人才的吸引力；对于能力要求，我们需要注意的是，哪些要素是必须的？哪些是可以有的？

要根据实际情况来写JD，不要盲目抄一份别人的JD。

## 面试主要考察的就是能力、潜力和价值观

真实面试场景中，我们需要不断追问面试者项目经历、询问面试者的大局观以及了解面试者价值观来了解面试者是否适合。

在面试的时候，要遵循一些面试原则。这两条原则是布老师多年实战经历总结出来的经验。


# 【二】如何与员工一对一沟通

作为团队负责人，工作中一般会处于非常忙的状态，业务和琐事缠身、跟团队成员的沟通自然就少了。不过，缺少了一些必要的沟通，你的团队成员可能会出现很多新的问题。

所以，应该多找机会和团队成员进行一对一沟通，来了解团队成员的真实需求。

## 了解员工的关键在于进行绩效谈话和日常一对一沟通

### 绩效谈话也是有方法论的，我们来看下具体的分步骤

1. 肯定好的地方
2. 指出还可以提高的地方
3. 一起拆解问题的原因，并告知期待的标准是什么
4. 一起制定下一步计划

### 那常规的日常一对一谈话该怎么说呢？我们来总结一下

1. 倾听员工最近的感受，听听心里话
2. 同步对于员工上一周期的成果，以及自己的反馈
3. 和员工一起确定接下来的计划和动作
4. 将确定下来的行动计划，变成文档，方便后期跟进

在沟通中，也是要注意一些原则。这些原则看着都是细节，但是需要平时时时记在心中，沟通时才能自然一些，沟通结果也会比较顺畅。


# 【三】如何帮助员工成长

员工进入团队，在经历完一个又一个项目后，经验会随着工作时限的增加而增加，工作也会越来越上手，但是他们有可能会对自己的未来陷入迷茫，这时候又会产生很多新的问题。

这时候就需要团队leader站出来，为员工的未来发展负责。

## leader帮助员工成长的关键在于帮助其制定发展计划

如何帮助员工制定发展计划呢？

首先我们需要与员工一起让其意识到现在的状态离下一个目标存在的差距，然后作为leader需要给出成长路径。

可以通过能力模型帮助员工了解距离目标的差距在哪里。

## 提升计划一般分为三个角度

1. 系统性学习
2. 项目的历练
3. 影响力的提升

具体的成长计划一定要根据员工的实际情况来进行。


# 总结

## 面临问题

P3产品经理面临的问题有哪些？

1. 业务变得复杂而未知
复杂和未知其实定义了P3产品经理在实际工作中的两种场景：成熟业务下的「接手项目」以及完全探索性质的「从零到一」。其中，成熟业务下如何接手一个产品项目，是P3产品经理更为常见的场景，需要面对的问题是业务复杂；而完全从零到一做一款产品，面临的问题更多的是产品方向的未知。

2. 管理上需要占用更多精力
作为职场的新晋管理者，P3产品经理在实际工作中也会在管理上花费较多精力。管理又可以分为对于事的管理，即「项目管理」，以及对于人的管理，即「团队管理」。

## 课程框架

基于P3产品经理遇到的上述问题，我们是这样设计课程章节架构的。

作为P3级别的产品经理，工作环境和对象与之前产生了质的变化，面临的问题和需要掌握的能力也自然不同，只有具备了解决更高级别的能力和格局，才能在面对不确定性和复杂性事务的时候处变不惊。


# 【第六周作业】团队管理作业

## 作业要求

本次作业与第五周作业类似，需要基于你目前的工作场景进行延伸。

以接下来可能发生的一次员工绩效谈话为基础，准备一份一对一沟通提纲，与你的一位团队成员沟通，在对其进行绩效沟通过后，需要根据能力模型为其制定一份员工发展计划，涵盖以下要点：

1. **绩效沟通部分**：需要一对一沟通提纲与实际模拟谈话
2. **员工成长计划**：需要根据能力模型制定并说明原因

### 注：

- 需要说清楚谈话背景，关于员工信息可以基于真实情况进行一定延伸
