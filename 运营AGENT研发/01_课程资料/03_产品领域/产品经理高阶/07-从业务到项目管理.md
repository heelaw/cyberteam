# 07-从业务到项目管理


# 从业务到项目管理

欢迎大家开始第三章内容的学习~ 在前面部分，我们花了四周的时间来学习两种不同的业务场景，包括成熟业务从如何接手项目到中长期产品规划，以及新业务探索从零到一该怎么做。

在实际工作中，作为管理者，除了要掌握超强的业务能力，还需要能够具备"推项目，拉团队"的能力。那么在这个章节，我们就来系统来学习如何进行团队的项目管理。

项目管理的三个关键动作，即项目的启动，项目的推进和项目的复盘，分别对应目标与方向的共识，流程与方法的共识，以及团队问题的共识。其实最终要达成共识，也需要团队在过程中不断的磨合~

不仅要把自己事情做好，还要能够驱动团队成员推进项目，这也是P3产品经理的一个核心能力，即能够"推项目"。


# [一] 准备一场高质量的项目启动会

众所周知，军队在出征以前，都要有个誓师大会。大会上统帅要向战士宣布作战的意义，表示作战的目的和决心，以鼓舞士气。

商场如战场，当我们启动一个新项目的时候，也需要一个类似誓师大会的场景，来向团队成员们鼓舞士气，表明决心，告知目的和意义等等。具体的表现呢，就是开一场项目启动会。

那么，如何开一场高质量的项目启动会呢？

在开项目启动会时，我们需要讲清楚：为什么要做，具体怎么做，有谁参与，协作方式以及马上要做什么。

一般项目有两种典型的场景，一种是在成熟业务中开展项目，一种是从零到一探索项目，这两种场景下，有关项目启动会上的侧重点和讲法策略是不一样的。

成熟业务中开展的项目，在启动会上宣讲的核心目的是让大家意识到问题的严重性，就是在业务中，问题已经严重到了不得不解决的状态，所以最关键的是让大家形成对问题的共识；

而从零到一，因为本来就是探索型项目，失败的概率很大，所以要让团队成员理解到我们做这件事情的目的和使命感，这件事非我们不可，同时要提前让大家做好心理准备，做好打硬仗的准备。

开展新的项目时，团队负责人有可能是空降过来的，或是担任临时成立的一个虚拟项目组leader。管理这样一个团队，首先必须达成互相之间的信任感，那刚开始怎么做到呢？布老师的建议是，做一场就职演说。

总之，启动会的目的是为了让团队成员对于项目，达成在目标上和方向上的共识，同时明确日常工作的协同和规则。


# [二] 在团队中形成一致的协作机制

开完项目启动会后，团队就要开始投入到实际项目中了这个时候，作为团队的负责人，你可能会发现，大家原来都不太熟，工作习惯也略有不同，那么，怎么形成团结一致且高效的协作机制，让大家在日常协作中达成共识呢?

开启一个项目后，首先要从流程上解决共识的问题，包括团队成员内部的共识，与项目外部成员的信息同步，以及新人进入团队对于项目的理解等。

在协作机制上，可以通过三个要点，即会议、汇报与文档，达成流程和方法的共识。

对于团队内部对于问题的共识，可以通过会议来达成。站立会用来提高当日信息沟通的效率，周会用来同步本周的核心重点工作和下周核心目标，评审和上线都可以开展相关的会议。

通过对外汇报团队项目的进度，有利于获取外部的认可和帮助。主要包括团队周报、里程碑汇报和突发情况汇报三种方式，其中，团队周报与里程碑汇报是最常见的两种。

而对于新人无法快速上手项目的问题，我们可以通过更好的文档沉淀，更高效地帮助新人融入团队。有两种类型的文档，一种是文件资料的沉淀，方便查询；另一种是帮助说明文档，帮助完成一些简单的任务。

总之，通过各种方法，协作机制的目的就是达成项目方法与流程的共识。

工欲善其事必先利其器，这里，项目就是"事"，方法与流程就是"器"，达成方法与流程的共识，才能更加高效地推进项目。


# [三] 做好阶段性总结与复盘

项目推进是按照阶段来进行的，每进行一个阶段，都有必要进行阶段性的总结和复盘，将这一阶段中的难点和重点梳理出来，总结经验和规律，方便项目进行下一个阶段的推进。

那么，该该如何进行项目复盘呢？布老师提到项目复盘步骤可以有以下四步：收集问题——分析问题——讨论问题——解决问题。

第一步，是会前匿名收集员工自己的感受和对团队的感受。

第二步，分析之前收集到的成员对于项目中自我和团队的评价，同时整理亮点和问题。

第三步就是开复盘会议，通过将亮点问题上墙投票，形成目前状态与问题的共识。

最后一步，就是将亮点提炼为规律沉淀知识库，并且把问题找到原因，写出改进的Todo。

光做好这些步骤还不够，我们还需要在复盘进行中，时刻铭记复盘的三大原则。

未能达成团队问题共识的复盘行为，是不达标的。

## 柳传志：我的复盘方法论

### 1、何为复盘

「复盘」这个词最早来源于棋类术语，也称「复局」，指对局完毕后，复演该盘棋的记录，以检查对局中对弈者的优劣与得失关键。

通俗来说，复盘就是把当时「走」的过程重复多遍，并且主动思考为什么这么「走」，下一步应该如何设计，接下来的几步该怎么走。

复盘被认为围棋选手增长棋力的最重要方法。

### 2、为何要复盘

复盘的好处：

1）不在同一坑里跌倒，避免重复犯错；
2）知道团队强弱项，合理分工；
3）知己知彼，磨练内心；
4）知道如何能更好，胜在细节；

### 3、如何复盘

复盘的四步骤：

第一个步骤、回顾目标：当初的目的或期望是什么；

第二个步骤、评估结果：和原定目标相比有哪些亮点和不足；

第三个步骤，分析原因：事情成功和失败的根本原因，包括主观和客观两方面；

第四个步骤、总结经验：需要实施哪些新举措，需要继续哪些措施，需要叫停哪些项目。

复盘结论是否可靠，可以通过3个原则来评判：

1. 结论落脚点是否发生在偶然事件上？
2. 复盘结论是指向人还是指向事？
3. 是否是经过交叉验证得出的结论？

"孤证不能定案"，复盘得出的结论通过其他事情交叉验证，可以为结论的有效性提供一定的保障。


# [第五周作业]

## 作业要求

本次作业以项目复盘为基础。与之前作业不同的是，本次作业并没有给定一个背景，而是基于你目前的工作场景进行延伸。

参考课程及扩展资料中复盘的方法，对你最近负责的一个团队项目进行复盘，撰写一份复盘报告，包括5-10张PPT+文字解说，涵盖以下要点：

1. 项目最终结果如何，离目标的差距有多少？
2. 项目的亮点表现在哪里？问题表现在哪里，原因是什么？
3. 如果你重新回到当时项目启动的场景，你会做哪些改进？（可以考虑准备一份就职演说哦~）

# [本周总结]

恭喜你学完了项目管理章节的课程内容~

对于项目负责人来说，项目就像自己的孩子一样，必须吃饭睡觉做梦都想着它。把项目像孩子一样管理，就需要对项目相关各个方面的梳理，接下来，我们再来回顾一下项目管理的三个关键动作：

按照项目进程，项目管理的三个关键动作为启动——推进——复盘，将一个项目的时间进程分解为小的推进阶段，边复盘边推进，循环式上升，最终达成项目的目标。

过程之中，需要不断的团队磨合，既包括团队现有成员之间沟通磨合，也有项目团队和公司业务关系磨合，还有新人进来和原项目组成员之间的磨合等等。
