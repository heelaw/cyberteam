# 产品经理进阶课

## 「总结」产品人的学习和成长

**课程类型：** 图文

**课程内容：**

接下来，你即将要学习和掌握产品经理这个职位的知识技能了。很多时候我们认为产品经理是改变世界的，实际上，他们是被世界而改变。那么产品经理岗位到底是怎么样的呢，总结来说就是：

- 入门容易入行难
- 改变世界前不如改变自己
- 从横向到纵向的发展
- 提升逻辑能力很重要

入行之后我们经常会碰到一些产品经理的干货分享，其中也许有一部分能帮助你成长，但大部分的干货对你来说却是－然并卵。一个理智的产品经理应该知道自己的成长路径：

- 在P1的时候着重学习基础能力
- 在P2的时候注重需求理解和项目把控
- 在P3的时候注重团队效能和产品方向

很多人在学习产品经理过程中还是比较困惑，在听课程前我们要知道"如何学习"，接下来我们从上至下回顾产品经理能力框架，通过复习来建立自己"弹药库"。

**组织影响力**

- 管理能力：人员招募、指标制定、人员晋升体系
- 领导力：知识传承、人才培养、团队价值建设

**工作必备技能**

- 产品技能：需求分析、专业设计、产品规划、数据分析能力
- 通用技能：市场分析、商务沟通、市场营销、渠道管理能力

**底层能力**

- 知识储备库：技术、项目管理、心理学等。
- 基础通用能力：学习能力、沟通能力、执行力、常识能力等

看完产品经理框架之后，合适的学习路径是以下这样的：

- 由浅入深，逐步展开：先学基础，逐步成长；
- 学以致用：基于自身能力现状，提升即将用到的能力；
- 以练带学：最好的学习，是深度的练习加上适当的理论；
- 最好的学习是分享：分享自己的产出，点评别人的产出；
- 良好心态：罗马不是一天建成的，吃多了干货会噎死。

很多产品经理都知道要以用户为中心进行产品设计却不知道什么才叫以用户为中心，其实以用户中心意味着

- 成为自己产品的资深用户，才能对产品有更深了解；
- 在安全上不要为了1%的可能，去影响99%的用户；
- 用户质量比数量更重要，侧重关注活跃用户远大于新增用户；
- 产品经理更应该关注产品的长期利益而不是短期利益；
- 如果你不能拼死维护用户利益，就不算合格的产品经理。

产品设计关键不是是在于商业模式和产品模式，而是在于用户、场景、需求，那么产品分析画布工作流程究竟是如何的呢？

一名合格的产品经理对待产品时一定要有产品生命周期概念，在产品不同的阶段要做的事情也不一样

知道产品生命周期，不同的阶段做不同的事之后，我们要学会判断产品处于哪个周期：

- 首先看活跃用户趋势，平稳、向上拉伸还是往下掉；
- 其次看时间段，一般18个月后，产品都会进入成长期；
- 提醒：看活跃用户量，尽可能分离外部推广带来的"增肥效应"。

---

**课程总结：**

1. **产品经理现状**
   - 入门容易入行难
   - 需要从横向到纵向发展
   - 提升逻辑能力很重要

2. **产品经理成长路径**
   - P1：着重学习基础能力
   - P2：注重需求理解和项目把控
   - P3：注重团队效能和产品方向

3. **产品经理能力框架**
   - 组织影响力：管理能力、领导力
   - 工作必备技能：产品技能 + 通用技能
   - 底层能力：知识储备库 + 基础通用能力

4. **学习方法**
   - 由浅入深，逐步展开
   - 学以致用
   - 以练带学
   - 最好的学习是分享
   - 保持良好心态

5. **以用户为中心**
   - 成为自己产品的资深用户
   - 不要为1%影响99%用户
   - 关注活跃用户大于新增用户
   - 关注长期利益大于短期利益
