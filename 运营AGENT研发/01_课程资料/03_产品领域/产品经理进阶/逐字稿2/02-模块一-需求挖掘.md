## 本章导学

需求挖掘是产品经理的核心能力之一。本模块将系统讲解如何通过用户特征分析、用户调研、现场调研等方法，主动发现用户问题并验证其可行性。学员将掌握用户画像的创建方法、理解用户调研的常见误区、学会现场调研的操作技巧，并能够运用数据指导产品工作。

## 详细内容

### 主题一：用户特征分析（用户画像）

#### 1.1 什么是用户特征分析

用户特征分析（User Profiling）是产品经理进行用户分析的基础方法，通过打标签的方式去分析用户群体。

**核心目标：** 找到产品的用户在哪里，挖掘出用户的典型特征

#### 1.2 用户画像的应用场景

| 场景 | 说明 |
|------|------|
| 活动运营 | 老板要求吸引目标用户时，需要知道目标用户在哪里 |
| 新产品开发 | 做用户分析时，需要知道目标用户是谁 |
| 用户召回 | 老用户流失时，找出哪些用户是可召回的 |
| 个性化推荐 | 如淘宝的推荐系统，基于用户特征进行精准推荐 |

#### 1.3 如何找到竞品用户画像

**常见问题：**
- 不知道如何找到竞品的用户画像
- 拍脑袋猜测
- 以自己数据作为参考（不准确）

**正确做法：**
- 寻求第三方调研报告
- 参考专业数据平台

**推荐数据平台：**

| 平台 | 网址 | 特点 |
|------|------|------|
| 艾瑞网 | iresearch.cn | 互联网数据咨询平台 |
| 酷传 | kuchuan.com | APP数据平台 |
| 新榜 | newrank.cn | 内容数据平台 |
| 易观 | analysys.cn | 数据分析平台 |
| 艾媒网 | iimedia.cn | 媒体数据 |
| 企鹅智库 | re.qq.com | 腾讯数据 |

#### 1.4 B端产品的用户特征分析

B端产品同样需要用户画像，且更复杂，因为：
- 使用者与决策者（付费者）分离
- 需要分别分析不同角色的特征

### 主题二：深度用户调研

#### 2.1 用户调研的重要性

用户调研是产品经理最基本的素质：
- 被动接受用户提出的需求
- 主动通过深度调研发现用户问题
- 验证需求的可行性

#### 2.2 用户调研的三大误区

**误区一：通过问答创建用户画像**

问题表现：
- 急于为用户分类与贴标签
- 得到的用户特征片面不准确
- 人有多面性，不同时间场合表现不同
- 自我认知未必准确

正确做法：
- 事先调研：以社交阅读产品为例，通过账号可得知个人资料、加入时间、活跃程度、兴趣点、图书列表及购买情况、社交网络等
- 还可以通过用户的微博、博客等了解更多用户信息
- 事先做好用户画像的功课能提高访谈起点

**误区二：调查用户行为而非心理**

问题表现：
- 问占比分布、频度统计等问题
- 用户给出的数字是拍脑袋估算
- 用户对统计数字不敏感

正确做法：
- 用户数据和用户日志更能诚实准确地反映用户行为
- 用户调研更应该反映用户心理：目的、动机、需求、感受、体验
- 引导用户讲故事

引导用户讲故事的问题：
- 用户是如何知晓并到达产品的？
- 用户会如何向别人形容产品特点？
- 对于用户哪些功能是 must have，哪些是 nice to have？
- 用户在什么情况下"想起并找到"该产品？
- 用户感知到的与竞品的明显差异是什么？

**误区三：混淆个性与共性**

问题表现：
- 一个用户的特征和想法是个性，不能代表全体用户

正确理解：
- 越是个人的，越是普遍的
- 用户调研是一门挖掘个体之共性的艺术

案例：
- "找一本书时会先到字节社，因为可以复制文字做笔记"
- 透露两个用户心理痛点：做笔记、资源全
- must have 是共性：不满足就会转用其他产品
- nice to have 是个性：多种多样

### 主题三：现场用户调研

#### 3.1 什么是现场调研

现场用户调研是用户调研过程中直接接触用户的环节，是了解用户的绝好机会。

#### 3.2 推荐工具

**PC端录屏工具：**
- Macbook + Screenflow
- Apowersoft录屏王

**手机端录屏工具：**
- iPhone + Macbook + Airserver

**用户行为统计工具：**
- 神策数据：围绕用户行为分析，完成数据采集和数据分析
- GrowingIO：无需程序员手动埋点，运营可直接定制数据采集规则
- 诸葛IO：帮助挖掘用户的真实行为与属性

---

### 主题四：用数据指导产品工作

#### 4.1 数据分析的重要性

- 数据是产品决策的重要依据
- 通过数据发现问题、验证方案

#### 4.2 数据驱动产品优化

**常见数据指标：**
- 活跃用户数
- 留存率
- 转化率
- 流失率

**数据驱动案例：**
- 支付流程优化：通过数据分析发现转化瓶颈，针对性优化

## 核心概念

| 概念 | 定义 |
|------|------|
| 用户特征分析 | 通过打标签的方式分析用户群体 |
| 用户画像 | 真实用户的虚拟代表，抽取典型特征形成 |
| 用户调研 | 主动发现用户问题，验证可行性的方法 |
| 现场调研 | 直接接触用户，了解用户真实情况 |
| 数据驱动 | 以数据为依据指导产品决策 |

---

## 本章小结

本章系统讲解了需求挖掘的核心方法：

1. **用户特征分析**：通过打标签方式分析用户群体，是一切用户研究的基础
2. **用户画像创建**：不能仅靠问答，需要事先调研，了解用户背景
3. **用户调研**：关注用户心理而非行为统计，分辨个性与共性
4. **现场调研**：利用录屏工具等，记录用户真实操作行为
5. **数据指导**：用数据发现问题的同时验证解决方案
