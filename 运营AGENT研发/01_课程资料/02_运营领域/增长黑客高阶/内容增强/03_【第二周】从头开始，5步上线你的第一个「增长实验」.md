## 本章导读：增长实验流程图

### 核心概念

**增长方法论的双重底层逻辑**：
1. **投资者思维**：寻找杠杆点——用最小力量产生最大效果
2. **科学家思维**：严谨实验——验证假设，得出结论

### 增长实验的重要性

找到聚焦领域是增长实验效果最大化的第一步，原因如下：
- **集中力量攻克一个点**，认知、洞察和实验效果都会提升
- 通过北极星指标和增长模型找到聚焦领域
- 在该领域进行30-90天的持续实验
- 实验频率：1-2周/次

### 案例分析

**案例1：增长黑客网**
- 北极星指标：付费用户数
- 聚焦领域：降低付费用户流失
- 策略：针对流失进行各种实验

**案例2：Canva（在线作图软件）**
- 北极星指标：使用用户数
- 聚焦领域：新用户激活率
- 策略：集中进行激活相关实验

### 普通人 vs 优秀增长人的区别

| 维度 | 普通人 | 优秀增长人 |
|------|--------|------------|
| **实验来源** | 随机想法 | 数据驱动洞察 |
| **方法论** | 无成型方法 | 有清晰框架 |
| **优先级** | 随意执行 | 系统排序 |
| **指标定义** | 容易遗漏 | 全面准确 |
| **文档** | 不详细 | 完整PRD |
| **结果分析** | 知成功失败 | 深挖洞察，应用全局 |

### 增长实验五步流程

```
┌─────────────────────────────────────────────────────────────┐
│                    增长实验五步流程                          │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  ① 产生实验想法 ──→ ② 排列优先级 ──→ ③ 实验设计            │
│     ✓ 从数据中寻找洞察      ✓ ICE模型       ✓ 制定指标      │
│     ✓ 形成清晰假设                          ✓ 确定受众      │
│                                               ✓ 设计版本    │
│                                                              │
│  ④ 开发上线 ──→ ⑤ 分析应用                                  │
│     ✓ 数据埋点      ✓ 评估可信性                            │
│     ✓ QA测试        ✓ 分析结果                              │
│                     ✓ 决定下一步                            │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

**关键问题对应表**：

| 步骤 | 回答的关键问题 |
|------|----------------|
| ①产生想法 | 如何产生高质量实验想法？如何提高成功率？ |
| ②排列优先级 | 如何排序，永远做效果最大的实验？ |
| ③实验设计 | 如何制定指标？如何落地细节？ |
| ④开发上线 | 如何数据埋点？如何清晰衡量？ |

## 第一节：3步产生高质量实验假设

### 核心框架

**产生高质量实验想法的三步骤**：
1. **明确实验目标**：知道实验要达到什么目的
2. **从数据中寻找洞察**：关键步骤，提高成功率
3. **形成清晰实验假设**：可被证伪的明确表述

### 1.1 产生实验假设（上）

#### 一、明确实验目标

**正确的实验出发点**：
- ✓ 从用户和业务问题出发
  - 用户反馈：流程复杂、不明白
  - 数据问题：指标下降、转化率低
- ✗ 从个人想法出发
  - 觉得设计难看
  - 觉得按钮颜色不对
  - 竞品有功能我们也想测

**实验目标与实验类型对应关系**：

| 实验目标 | 实验类型 | 复杂度 |
|----------|----------|--------|
| 提高单点转化率 | 文案测试、设计素材测试、单页面测试 | 简单 |
| 提升全漏斗转化率 | 路径测试（调整页面顺序、增删页面） | 简单 |
| 对比新旧版本 | 新版本/新功能监测 | 复杂 |
| 提升复杂指标 | 复杂实验（MVP功能、算法等） | 复杂 |

#### 二、从数据中寻找洞察

**什么是高质量的实验想法？**

高质量实验想法 = 有数据支持的实验假设 = 成功概率更高

| 指标 | 高质量实验想法 | 普通实验想法 |
|------|----------------|--------------|
| 结果 | 目标指标提升比例大 | 提升不明显 |
| 成功率 | 测10个有4-5个成功 | 测10个只有2-3个成功 |

**案例：Canva新用户个性化实验**

- **实验内容**：提供个性化海报模板供新用户选择
- **实验结果**：
  - 激活率提升12%
  - 每年带来上百万美元利润
- **成功原因**：从数据中寻找洞察
  1. 定量分析发现：海报功能使用最多但流失率高
  2. 定性分析发现：用户需求五花八门，无合适模板即流失
  3. 结合最佳实践：个性化流程表现良好
  4. 形成假设：个性化模板可提升激活率
  5. 实验验证假设正确

**产生高质量假设的两种方法**：

**方法一：应用三类数据**

| 数据类型 | 作用 | 回答的问题 | 具体方法 |
|----------|------|----------|----------|
| **定量数据** | 发现问题 | 问题在哪儿？ | 计算转化率、用户分群、热图分析、路径分析、场景细查 |
| **定性数据** | 了解原因 | 为什么发生？ | 可用性研究、用户问卷、录屏、实时聊天、弹窗问卷 |
| **最佳实践** | 寻找方案 | 怎么改更好？ | LIFT模型、用户心理学、文案写作、设计最佳实践 |

**案例1：在行课程购买率提升**

- **改动**：去除高亮价格和课时数
- **结果**：点击率提升500%
- **洞察来源**：用户调研
  - 用户视角：长课时=投入大=降低了解欲望

**案例2：Blue Apron新用户注册**

- **改动**：优化"未来菜单"页面
- **结果**：注册转化率提升5%
- **洞察来源**：定量分析
  - 路径分析发现：>50%新用户注册前访问此页面
  - 假设：改善此页面可提升注册

**案例3：斗牛下单率提升**

- **改动1**：添加"平台鉴定通过再发货"文案
  - 结果：下单率+46%，支付率+70%
  - 洞察：用户担心假货（客单价1500元）
- **改动2**：合并结算步骤，支付方式改为弹窗
  - 结果：支付率提升26%
  - 洞察：定量（流失率高）+ 最佳实践（步骤越少越好）

**方法二：多轮数据分析**

当初步分析后假设过多或不清晰时，通过多轮分析筛选：

```
第一轮分析
    ↓
发现问题，形成多个初步假设
    ↓
第二轮分析（排除不靠谱假设）
    ↓
找到新证据支持某假设
    ↓
形成新假设
    ↓
循环往复，最终得到高质量假设
```

**案例：小恩爱注册转化率提升20%**

1. **发现问题**：漏斗分析 → 第3步资料填写页转化率低
2. **初步假设**：
   - 字体不明显
   - 信息太多
   - 找不到上传
   - 上传太慢
3. **第二轮分析**：细查流失用户路径
   - 发现两种主要流失路径：
     - 找不到合适头像，等待后离开
     - 上传照片但传不上来
4. **筛选假设**：优先验证
   - 假设1：头像上传改为非必须
   - 假设2：优化上传速度
5. **实验结果**：转化率提升20%

**转化率优化最佳实践框架（LIFT模型）**：

| 维度 | 说明 | 举例 |
|------|------|------|
| **价值主张** | 清晰的价值表达 | "每天跑一单油钱轻松赚" |
| **相关性** | 流程针对性，步骤一致 | 营销页→注册页信息一致 |
| **清晰度** | 设计、文案、按钮层级清晰 | 重点突出，引导明确 |
| **降低焦虑** | 减少用户顾虑 | "平台鉴定通过再发货" |
| **减少分心** | 不要太多信息/选项 | 简化页面，聚焦核心 |
| **紧迫性** | 促进即时行动 | "仅剩XX份""限时XX天" |

**滴滴营销页案例**：

- **原版**：普通营销页
- **新版**：应用LIFT模型
  - 价值主张清晰："每天跑一单油钱轻松赚"
  - 设计清晰层级
  - 明确引导下一步
- **结果**：注册率提升21%

**注意**：不要盲目迷信最佳实践
- **反例**：Real Estate网站
  - 常规认知：注册步骤越少越好
  - 实际结果：增加个人信息询问步骤效果更好
  - 原因：用户认为更多信息=更个性化服务

#### 三、提升转化率的四个方向

| 方向 | 策略 | 具体方法 |
|------|------|----------|
| **提升注意力** | 让用户注意到关键元素 | 视觉对比、动画、位置优化 |
| **提升动力** | 增强用户行动意愿 | 价值主张、社会证明、紧迫感 |
| **降低阻力** | 减少用户操作障碍 | 简化流程、减少选项、优化文案 |
| **特殊场景** | 针对性优化 | 移动端适配、网络优化等 |

### 1.2 产生实验假设（下）

#### 一、清晰的实验假设标准

**什么是清晰的实验假设？**

一个清晰的实验假设必须：
- ✓ 有明确的指标
- ✓ 有时间限制
- ✓ 可以被证伪（可验证）
- ✓ 解释深层原因
- ✓ 有数据支持

**假设清晰度对比表**：

| 假设表述 | 清晰度评分 | 问题 |
|----------|------------|------|
| "写公关稿可以带来新用户注册" | ★☆☆☆☆ | 过于宽泛，无法验证 |
| "写一篇公关稿可以带来100个新用户注册" | ★★☆☆☆ | 无时间限制 |
| "写一篇公关稿可以在两周内带来100个新用户注册" | ★★★☆☆ | 未解释原因 |
| "写一篇产品价值公关稿可在两周内带来100个新用户注册，因为清晰传达价值主张" | ★★★★★ | 完整清晰 |

#### 二、实验假设输出模板

```
如果 [具体改动]
预计 [指标] 可以提升 [百分比]
因为 [深层原因/数据支持]
```

**模板应用示例**：

> "如果Canva可以设计个性化的海报新用户引导流程，预计海报功能新用户激活率可以提升10%，因为当人们通过海报功能找到Canva时，想要很快完成设计，如果没有马上找到合适的模板，就很容易流失。"

**模板结构说明**：

| 部分 | 内容要求 | 作用 |
|------|----------|------|
| **如果** | 具体的改动内容 | 明确实验变量 |
| **预计** | 指标+提升比例 | 设定可衡量的目标 |
| **因为** | 深层原因/数据证据 | 说明假设依据 |

#### 三、产生实验想法的群组方法

**情况1：没有想法（空白）**

| 方法 | 说明 | 适用场景 |
|------|------|----------|
| **头脑风暴** | 自由联想，快速产生大量想法 | 初期探索 |
| **问题风暴** | 针对问题只提问不回答 | 聚焦问题 |
| **闪电Demo** | 快速演示竞品和非竞品 | 借鉴灵感 |
| **设计冲刺** | 5天内高效解决问题并测试 | 快速验证 |

**情况2：想法过多**

**实验想法库管理**：
- 工具：Excel表格
- 内容：记录每个实验假设和想法
- 作用：建立长期想法库，方便后续筛选

#### 四、完整工作流程

```
实验目标确定
    ↓
实验类型选择
    ↓
三类数据分析（定量+定性+最佳实践）
    ↓
多轮数据分析筛选（如需要）
    ↓
输出清晰假设（应用模板）
    ↓
实验想法库管理
```

## 第二节：通过ICE模型排列优先级

### ICE模型概述

**ICE模型定义**：
- **I**mpact：影响力
- **C**onfidence：信心/成功概率
- **E**ase：容易程度

### 一、ICE模型评分方法

**评分流程**：
1. 列出所有实验想法
2. 对每个想法的三项指标打分（一般1-10分）
3. 计算总分：Impact + Confidence + Ease
4. 按总分排序，决定优先级

**案例：订房网站实验优先级**

| 实验想法 | Impact | Confidence | Ease | 总分 | 排序 |
|----------|--------|------------|------|------|------|
| 微信一键登录 | 9 | 9 | 3 | 21 | 2 |
| 全网最低价保证 | 7 | 8 | 9 | 24 | 1 |
| 热门房源精选标签 | 6 | 6 | 7 | 19 | 3 |

**结论**：优先实施"全网最低价保证"实验

### 二、评分标准详解

#### 1. Impact（影响力）评分因素

| 因素 | 说明 | 评分技巧 |
|------|------|----------|
| **覆盖用户量** | 实验可影响的用户数 | 高流量页面/位置分数更高 |
| **提升幅度** | 成功后指标提升比例 | 预期提升越大分数越高 |

**提升影响力的方法**：

**方法1：覆盖更多用户**

```
传统做法：只关心核心用户
    ↓
问题：核心用户只占活跃用户的25%
    ↓
改进：主动关心非核心用户（注册用户、活跃用户、过客用户、外部渠道用户）
```

**案例：Airbnb增长策略**
- 不仅关注核心用户
- 扩大实验覆盖面
- 无形中提升影响力

**方法2：从高流量页面开始**

| 策略 | 说明 | 案例 |
|------|------|------|
| 多次实验高流量页面 | 每次小提升=总体大影响 | 咪咕视频up下载页多次测试，最终提升56% |
| 聚焦关键路径 | 漏斗上游、新用户激活、购买转化 | - |

#### 2. Confidence（信心）评分因素

**评分依据**：
- 是否有足够的数据洞察支持？
- 定量数据是否充分？
- 定性数据是否明确？
- 最佳实践是否适用？

**关联**：与第一节"高质量实验想法"的标准一致

#### 3. Ease（容易程度）评分因素

| 考量维度 | 说明 |
|----------|------|
| **研发成本** | 需要多少工程师、多长时间 |
| **经济成本** | 是否需要红包等投入 |
| **上线速度** | 能否快速上线 |
| **维护成本** | 后期维护难度 |

### 三、提升实验容易程度（MVP方法）

**MVP（最小可行性产品）原则**：

用最小资源、最快速度验证或证伪假设

**MVP设计的两个问题**：

| 问题 | 思考方向 |
|------|----------|
| 如何最小资源最快验证？ | 数据分析验证？小范围测试？ |
| 如何保证结果可信？ | 避免过度简化导致结论无效 |

**案例：个性化注册流程MVP**

| 方案 | 开发成本 | 设计成本 | 数据成本 | 总体评价 |
|------|----------|----------|----------|----------|
| 完整版 | 高 | 高 | 高 | 成本太高 |
| MVP版 | 低 | 低 | 低 | ✓只针对25岁女性用户 |

**MVP优势**：
- 降低资源投入
- 快速验证假设
- 提高成功概率
- 减少浪费

**反面案例：Etsy无限滚屏**

| 做法 | 问题 | 正确做法 |
|------|------|----------|
| 直接开发无限滚屏 | 耗时半年，指标不升反降 | MVP：显示100个物品或圆形页面测试 |
| 团队失望，回滚版本 | 浪费资源 | 先验证假设再全面开发 |

**改进后的开发流程**：
```
想法 → MVP验证 → 假设正确 → 全面开发
                  ↓
              假设错误 → 放弃或调整
```

## 第三节：设计A/B测试

### 3.1 制定3类实验指标

#### 一、实验指标的重要性

**为什么选择正确的指标至关重要？**

**案例：亚马逊中国购物车实验**

| 版本 | 设计 | 首次实验结果 | 问题 |
|------|------|--------------|------|
| 老版本 | 全部结账模式 | ✓ 销售额更高 | 新用户误买多，满意度低 |
| 新版本 | 部分结账模式 | ✗ 销售额更低 | 但长期复购更高 |

**第一次实验失败原因**：只关注短期销售额

**第二次实验成功**：
- **综合销售额**（短期+长期预期）
- **辅助指标**：复购率、下单频次、结账转化率
- **反向指标**：退货率

**结论**：新版本在综合指标上获胜，成功上线

**启示**：
> 选择正确的实验指标能够检验假设真伪，衡量实验成败。全面性和准确性是关键。

#### 二、三类实验指标框架

| 指标类型 | 作用 | 数量 | 说明 |
|----------|------|------|------|
| **核心指标** | 决定实验成败的关键 | 1个 | 实验的"北极星指标" |
| **辅助指标** | 全面了解结果，避免误伤 | <10个 | 各侧面辅助决策 |
| **反向指标** | 监测负面影响 | 1-2个 | 负面影响过高则否决实验 |

#### 三、如何选择核心指标

**关键问题**：
- 哪个指标代表实验最终目标？
- 最想要影响的指标是什么？
- 哪个指标能说明实验组改动成功？

**案例：新手引导页面实验**

| 指标 | A版本（卡片） | B版本（清单） | 选择 |
|------|---------------|---------------|------|
| 标题点击率 | 低 | 高 | ✗ 虚荣指标 |
| 设置完成率 | 高 | 低 | ✓ 核心指标 |

**结论**：选择代表实验最终目标的指标（设置完成率）

**注意**：避免选择虚荣指标

#### 四、如何选择辅助指标

**辅助指标的作用**：
- 监测整个漏斗各步骤影响
- 观测重要下游指标
- 覆盖其他关键用户指标

**Airbnb做法**：
- 建立关键指标仪表盘
- 每个实验结果放上去
- 检查对所有指标的影响

#### 五、如何选择反向指标

**常见反向指标**：
- NPS（净推荐值）
- 应用删除率
- 邮件退订率
- Push退订率
- 页面退出率
- 订单取消率

**案例：购物车优化实验**

| 情况 | 决策 |
|------|------|
| 下单率提升，退货率正常 | ✓ 成功 |
| 下单率提升，退货率过高 | ✗ 失败 |

#### 六、完整案例：电商加入购物车按钮实验

**实验背景**：测试不同的购物车按钮设计

**漏斗分析**：
```
产品详情页 → 加入购物车 → 访问购物车 → 下单成功 → 销售额 → 复购
```

**指标选择**：

| 类型 | 指标 | 理由 |
|------|------|------|
| **核心指标** | 加入购物车按钮点击率 | 直接影响的主要目标 |
| **辅助指标** | 加入购物车次数、购物车页面访问、下单成功、销售额、复购率 | 全面衡量影响 |
| **反向指标** | 退货率 | 监测负面影响 |

### 3.2 确定实验受众

#### 一、哪些人被包含在实验里？

**两种情况**：

| 情况 | 处理方式 |
|------|----------|
| 所有用户都参与 | 无需设置 |
| 只针对特定用户 | 设置实验受众 |

**实验受众目标**：针对特定用户群生成更精细的优化方案

**案例：新闻App广告Banner测试**

| 实验版本 | 实验受众 | 设计思路 |
|----------|----------|----------|
| 老版本 | 所有用户 | 原版设计 |
| 版本1 | 喜好游戏玩乐人群 | 葛优躺概念 |
| 版本2 | 其他所有用户 | 通用设计 |

**受众设置方式**：

| 方式 | 说明 | 适用场景 |
|------|------|----------|
| **默认分组** | 工具预设的受众分组 | 操作系统、iOS版本等 |
| **自定义分组** | 提需求给研发实现 | 特定渠道来源、特定账号类型等 |

**默认分组示例**：
- iOS 12用户（测试自动填表单功能）
- 操作系统类型

**自定义分组示例**：
- 信息流广告来源用户
- 百度搜索广告来源用户
- 有基金账号的用户

#### 二、需要多少样本数？

**案例：蓝按钮 vs 红按钮**

| 时间 | 样本 | 蓝按钮转化率 | 红按钮转化率 | 结论 |
|------|------|--------------|--------------|------|
| 3小时 | 很小 | 20% | 12% | 样本太小 |
| 3天 | 上千 | 6% | 9% | 不能确定 |
| 300天 | 很大 | 4.8% | 7.2% | ✓ 红按钮更好 |

**问题**：实际中不可能等300天

**平衡点**：找到最小样本量，既保证可靠性又不浪费时间

**统计显著性定义**：

> 对照组和实验组之间的转化率差别是真实存在的，而非随机误差引起的。

**示例**：
- 转化率差别统计显著性95%
- 含义：95%可能性差别真实存在，5%可能性是随机误差
- 统计显著性越高 → 随机误差可能性越低 → 结果越可靠

**影响样本数的三因素**：

| 因素 | 影响 | 原因 |
|------|------|------|
| **原版本转化率** | 越低 → 需要样本越多 | 信号弱 |
| **新版本转化率** | 差别越小 → 需要样本越多 | 检测敏感度高 |
| **统计显著性要求** | 要求越高 → 需要样本越多 | 对结果确定度要求高 |

**建议**：至少95%统计显著性

**样本计算器使用**：

```
输入：
1. 原始版本转化率（对照组）
2. 优化版本转化率（实验组）
3. 统计显著性要求（90%-100%）

输出：
每个版本需要的样本数量
```

**示例计算**：

| 场景 | 原版本 | 新版本 | 统计显著性 | 每版本样本 |
|------|--------|--------|------------|------------|
| 场景1 | 15% | 18% | 95% | 1,700 |
| 场景2 | 15% | 16% | 95% | 6,200 |
| 红按钮 | 10% | 12% | 95% | 2,900 |
| 大蓝按钮 | 10% | 10.5% | 95% | 62,000 |

#### 三、实验需要跑多长时间？

**计算公式**：

```
实验时长 = 总样本数（各版本样本之和） ÷ 每天访问量
```

**示例1：红蓝按钮测试**

| 参数 | 数值 |
|------|------|
| 总样本 | 2,900 × 2 = 5,800 |
| 每日访问 | 580 |
| 预估时长 | 10天 |

**示例2：四色按钮测试**

| 参数 | 数值 |
|------|------|
| 总样本 | 2,900 × 4 = 11,600 |
| 每日访问 | 580 |
| 预估时长 | 20天 |

**示例3：大蓝按钮测试**

| 参数 | 数值 |
|------|------|
| 总样本 | 62,000 × 2 = 124,000 |
| 每日访问 | 580 |
| 预估时长 | 213天（宣告死刑） |

#### 四、预估实验样本和时间的意义

**作用**：
1. 检查实验设计是否合理
2. 评估是否有足够流量/用户量
3. 判断实验时长是否太长

**实验时间太长的解决方案**：

| 方案 | 说明 | 效果 |
|------|------|------|
| **减少版本数** | 从4版本减少到2版本 | 总样本减半，时间减半 |
| **增加流量** | 提升用户基数 | 获取样本更快 |
| **加大改动** | 提高预期变化幅度 | 减少所需样本 |

**流量/用户量与改动的关系**：

| 公司类型 | 流量/用户量 | 实验策略 |
|----------|-------------|----------|
| **大公司** | Facebook、快手、抖音 | 可做细小实验（20种按钮颜色） |
| **小公司** | 流量少、用户少 | 必须做大改动 |

**逻辑**：
- 小流量 + 小改动 = 等到地老天荒也达不到统计显著
- 大流量 + 小改动 = 小提升 × 大基数 = 大利润

**建议**：
1. 小流量公司：做大的改动
2. 同时想办法提升用户基数和流量

### 3.3 设计实验版本

#### 一、实验版本设计三问题

| 问题 | 说明 |
|------|------|
| **改动在哪里** | 具体的页面/功能位置 |
| **几个版本** | 根据实验假设数量决定 |
| **流量如何分配** | 科学分流，保证可信性 |

#### 二、版本数量设计原则

**版本数量 = 高质量实验假设数量**

| 注意点 | 说明 |
|------|------|
| 不要随意改改 | 所有版本都基于高质量假设 |
| 提高成功率 | 避免浪费开发设计资源 |
| 考虑样本量 | 版本越多 → 总样本越多 |

#### 三、优化实验 vs 探索实验

| 类型 | 目的 | 设计原则 | 示例 |
|------|------|----------|------|
| **优化实验** | 看每个元素怎么改最好 | 单一变量，逐步验证 | 测试按钮颜色、文案、图片 |
| **探索实验** | 突破局部最大化 | 多变量同时改变 | 全新设计、全新路径 |

**优化实验流程**：

```
测试单个变量 → 确认影响 → 测试另一变量 → 最终合并多个变量
```

**探索实验适用场景**：

| 情况 | 策略 |
|------|------|
| 多轮小优化后提升有限 | 突破局部最大化 |
| 原版转化率太差 | 从理想状态出发全新设计 |

#### 四、流量分配的科学性

**为什么需要科学分流？**

> 确保对照组和实验组唯一区别是实验改动，排除其他外在因素影响。

**案例：极端情况**

| 情况 | 问题 |
|------|------|
| 红按钮组全是红绿色盲用户 | 结论无效 |
| 灭霸随机消灭人口 | ✓ 符合分流均匀 |
| 灭霸留下一半弱的英雄 | ✗ 不符合分流均匀 |

**科学分流四标准**：

| 标准 | 说明 |
|------|------|
| **均匀性** | 各维度分配比例均匀，分布随机 |
| **唯一性** | 同一用户始终看到同一版本 |
| **定向性** | 可针对特定类型用户设置 |
| **分层分流** | 多实验同时运行时合理分配流量 |

**技术保证**：
- ✓ 通过第三方工具或自建系统
- ✗ 不能人肉完成

**第三方工具示例（吆喝科技）**：

```
设置流程：
1. 设置总流量进入实验（如100%）
2. 分组（如对照组50%，实验组50%）
3. 工具自动保证均匀性和唯一性
```

**自建分流系统案例（电商公司）**：

```
架构流程：
增长团队提需求 → 后端系统分流 → 客户端埋点 → 后端控制版本
    ↓
数据收集到数据库 → 手动统计分析
```

**投入**：约1个月时间（只建分流系统）

#### 五、实验设计总结

| 步骤 | 关键点 |
|------|--------|
| **选择指标** | 核心指标 + 辅助指标 + 反向指标 |
| **确定受众** | 谁参与？需要多少样本？跑多久？ |
| **设计版本** | 改动位置？几个版本？如何分流？ |

**样本和时间调整策略**：

| 情况 | 解决方案 |
|------|----------|
| 样本太大 | 加大改动、减少版本、增加流量 |

**示例指标**：
- 电商：每单价值、退货率

**背景**：
- 类似美团，有网络效应
- 用户包含：消费者、餐厅、骑手
- 实验复杂，需要switch back实验

**埋点需求**：

```javascript
事件：experiment_viewed

- user_id（用户序列号）

- variant_name（版本名称）
```

**优势**：
- 记录所有实验用户和组别

- 结论准确全面

## 第五节：分析实验结果

### 分析应用实验结果三步骤

```
① 评估结果可信性 → ② 分析实验结果 → ③ 决定实验下一步
```

### 一、评估结果可信性

**核心问题**：实验结果是否达到统计显著？

#### 1. 统计显著性确认

**使用第三方工具**：
- 实时更新结果
- 达到设定显著性（如95%）自动提示
- 可停止实验

**手动计算**：
- 使用统计显著性计算器
- 使用Excel等工具
- 按预估时长结束实验后计算

**计算器示例**：

| 版本 | 样本数 | 转化数 | 转化率 |
|------|--------|--------|--------|
| A（对照） | 2,000 | 100 | 5% |
| B（实验） | 2,020 | 150 | 7.43% |

**结论**：
- 版B转化率提升48%
- 统计显著性：99.93%
- ✓ 有显著统计差异

#### 2. 实验结果常见坑

**第一类：时间不够长**

| 情况 | 问题 | 解决方案 |
|------|------|----------|
| 早期指标差就停 | 未达统计显著 | 给实验足够时间 |
| 早期指标好就停 | 未达统计显著 | 耐心等待统计显著 |
| 只达到统计显著就停 | 可能只包含高频用户 | 跑够时间，覆盖各类用户 |

**新奇效应（均值回归）案例**：

| 实验 | 早期结果 | 最终结果 | 启示 |
|------|----------|----------|------|
| Airbnb价格过滤器 | 新版本显著提升 | 差别消失 | 不要看早期指标 |
| 健身App课程选择页 | 新版本更好 | 只包含高频用户 | 覆盖所有用户类型 |

**以偏概全案例**：

| 用户类型 | 占比 | 实验3天情况 |
|----------|------|-------------|
| 高频用户（每天） | 30% | ✓ 包含在实验 |
| 中频用户（每周） | 50% | ✗ 部分未进入 |
| 低频用户（每两周） | 20% | ✗ 大多未进入 |

**结论**：实验只跑了3天，结论只基于高频用户，不可靠

**第二类：实验设置问题**

| 问题类型 | 说明 | 后果 |
|----------|------|------|
| 更改实验设置 | 开始后调整流量分配 | 结果不可信 |
| 工具有问题 | 测试环境故障 | 数据不准 |
| 流量分配不均 | 未科学分流 | 结论错误 |
| 多实验干扰 | 同一流程多实验 | 互相影响 |

**辛普森悖论案例（微软）**：

| 时间 | 新版本流量 | 新版本转化率 | 结论 |
|------|------------|--------------|------|
| 周五 | 1% | 高 | ✓ 更好 |
| 周六 | 50% | 高 | ✓ 更好 |
| **汇总** | - | 低 | ✗ 更差 |

**原因**：按两天单独计算后再加权平均，结论相反

**教训**：
> 开始实验后不要更改实验设置和流量分配

**第三类：分析无效或错误结论**

| 情况 | 问题 | 解决方案 |
|------|------|----------|
| 指标设置不全 | 遗漏重要指标 | 全面设置三类指标 |
| 人为因素 | 促销活动影响 | 避开特殊时期 |
| 证实偏差 | 选择性看结果 | 让数据说话 |

### 二、分析实验结果

#### 1. 衡量实验成功或失败

**三类指标检查**：

| 指标类型 | 检查要点 |
|----------|----------|
| **核心指标** | 是否按预期比对照组提升？ |
| **辅助指标** | 变化是否符合预期？有无反常？ |
| **反向指标** | 有无明显负面影响？是否可接受？ |

**案例：追书神器充值页面优化**

| 指标 | 原版 | 版本1 | 版本2 |
|------|------|--------|--------|
| 付费总额（核心） | 基准 | +10% | +48% ✓ |
| 付费人数（辅助） | 基准 | 下降 | -28% |
| 付费留存（反向） | 基准 | 无差异 | 无差异 |

**决策**：选择版本2（核心指标大幅提升）

#### 2. 观测周期选择

| 周期类型 | 适用场景 | 示例 |
|----------|----------|------|
| **短期观测** | 绝大多数实验 | 达到统计显著即可 |
| **长期观测** | 需要长期指标 | Pinterest看28天留存 |

#### 3. 实验结果的四种状态

| 状态 | 指标表现 | 统计显著 | 结论 |
|------|----------|----------|------|
| ① 大幅提升 | +20% | ✓ | 实验组获胜（最理想） |
| ② 小幅提升 | +2% | ✓ | 实验组获胜（方向正确） |
| ③ 明显下降 | -10% | ✓ | 对照组获胜（方向错误） |
| ④ 无差异 | 0% | ✗ | 对照组获胜（无关紧要） |

**个人偏好排序**：① > ② > ③ > ④

**原因分析**：

| 排序 | 理由 |
|------|------|
| ① 大幅提升 | 改进方向和元素都对 |
| ② 小幅提升 | 改进方向和元素都对 |
| ③ 明显下降 | 改进元素对但方向错，可快速调整 |
| ④ 无差异 | 改动无关紧要，信息量少 |

**小提升的复利效应**：

**案例：Facebook 2009年增长**

| 目标 | 月活从1亿增长到3亿 |
|------|---------------------|
| 策略 | 每周提升2%月活 |
| 计算 | 1.02⁵² ≈ 2.8（约3倍）|
| 结果 | 通过小优化实验达成目标 |

#### 4. 分析失败原因

**分析方法**：

| 方法 | 说明 | 应用场景 |
|------|------|----------|
| **细分漏斗** | 看哪一步与假设不一致 | 转化路径分析 |
| **分群分析** | 看不同人群结果是否一致 | 用户差异分析 |
| **用户调研** | 直接观察用户反馈 | 深入了解原因 |
| **后续实验** | 验证新假设 | 迭代改进 |

**案例1：注册流程分平台分析**

| 平台 | 对照组转化率 | 实验组转化率 | 结论 |
|----------|--------------|--------------|------|
| 移动端 | 27% | 23% | 对照组好 |
| 桌面端 | 27% | 23% | 实验组好 |

**发现**：实验组页面太长，按钮被推到屏幕下方

**改进**：优化移动端设计 → 各平台实验组都更好

**案例2：电商品类入口位置分析**

| 用户类型 | 老版点击率 | 新版点击率 | 结论 |
|----------|------------|------------|------|
| 新用户 | - | - | 无差异 |
| 老用户 | 高 | 低 | 习惯老位置 |

**改进**：提前通知老用户，引导到新位置 → 老用户点击率上升

### 三、决定实验下一步

#### 1. 应用结果的三种选择

| 选择 | 适用情况 | 后续动作 |
|------|----------|----------|
| **产品化** | 实验成功，假设验证 | 应用到产品，放大影响 |
| **放弃** | 实验失败，假设错误 | 记录心得，清理代码 |
| **继续迭代** | 有新想法 | 形成新假设，继续实验 |

**产品化注意事项**：
- 不是一次性100%发布
- 需要小规模滚动发布

**放弃和产品化的共同动作**：
- 记录实验心得
- 清理实验代码
- 保持产品代码清洁

#### 2. 放大成功实验影响的三种方式

| 方式 | 说明 | 案例 |
|------|------|------|
| **乘胜追击** | 针对成功点做更多实验 | 测试更多类似版本 |
| **举一反三** | 应用到产品/渠道其他地方 | 设计风格应用到多页面 |
| **调整方向** | 重新排列实验优先级 | 根据洞察调整方向 |

**案例：宜人贷放大实验影响**

**洞察**：用户喜欢简单稳重风格，突出借款额度

**放大方式**：
1. **乘胜追击**：测试多个类似风格版本，进一步提升转化率
2. **举一反三**：应用到H5、App多个页面

### 四、分析应用结果总结

| 步骤 | 内容 |
|------|------|
| **评估可信性** | 判断是否达到统计显著 |
| **分析结果** | 看三类指标，分析失败原因 |
| **决定下一步** | 产品化/放弃/继续迭代 |

### 五、增长实验完整流程总结

```
① 产生实验想法
   - 从数据寻找洞察
   - 写清晰假设
   - 管理想法库
    ↓
② 排列优先级
   - 应用ICE模型
   - 决定先做哪个
    ↓
③ 实验设计
   - 制定三类指标
   - 确定受众和样本
   - 设计版本
    ↓
④ 开发上线
   - 输出PRD
   - 数据埋点
   - QA测试
    ↓
⑤ 分析应用
   - 评估可信性
   - 分析结果
   - 决定下一步
   - 记录心得和计划
```

## 本章知识图谱

### 核心概念

| 概念 | 定义 |
|------|------|
| **北极星指标** | 衡量产品长期健康度的核心指标 |
| **输入指标** | 可以影响北极星指标的行动指标 |
| **输出指标** | 北极星指标的组成部分 |
| **统计显著性** | 实验结果差别真实存在的概率 |
| **ICE模型** | Impact（影响）+ Confidence（信心）+ Ease（容易） |
| **MVP** | 最小可行性产品 |
| **新奇效应** | 早期显著提升但后期消失的现象 |

### 关键模型

**1. 实验假设模板**：
```
如果 [具体改动]
预计 [指标] 提升 [百分比]
因为 [深层原因/数据支持]
```

**2. ICE评分公式**：
```
总分 = Impact + Confidence + Ease
```

**3. 实验时长计算**：
```
时长 = 总样本数 ÷ 每日访问量
```

### 常见问题与解决方案

| 问题 | 解决方案 |
|------|----------|
| 没有实验想法 | 头脑风暴、问题风暴、闪电Demo |
| 想法过多 | 建立实验想法库，ICE排序 |
| 实验样本不够 | 加大改动、减少版本、增加流量 |
| 时间太长无法等待 | 调整实验设计，提高变化幅度 |
| 结果不可信 | 检查统计显著性，科学分流 |
| 不知道如何分析 | 三类指标全面衡量，细分分析 |
