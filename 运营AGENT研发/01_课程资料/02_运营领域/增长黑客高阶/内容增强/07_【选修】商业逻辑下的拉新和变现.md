## 本章导读

### 核心概念

增长的底层逻辑最终落实到盈利的增长。对于成熟企业，盈利增长可以用以下公式表示：

$$盈利增长 = (收入 - 成本) \times 销量$$

对于成长型企业，增长底层逻辑的关键变量包括：

- **LTV（Lifetime Value）**：用户生命周期价值
- **CAC（Customer Acquisition Cost）**：获客成本
- **TAM（Total Addressable Market）**：潜在市场规模

### 关键指标关系

#### 1. TAM（披萨大小）

TAM代表产品的潜在市场规模，即"披萨有多大"。

#### 2. 市场份额

代表在披萨中能切到多大的一角。

#### 3. LTV（披萨售价）

这角披萨能卖到多少钱，包括未来重复销售的价值。

#### 4. CAC（披萨成本）

制作这角披萨的原材料成本。

### 商业模型的四个契合

一个可靠的商业模型需要达到四个契合：

1. **市场-产品契合（PMF）**：产品是否满足目标市场需求
2. **市场-商业模型契合**：用户是否愿意为产品付费
3. **渠道-商业模型契合**：是否存在可行的获客渠道
4. **渠道-产品契合**：渠道上能否找到目标用户

### LTV与CAC的关系

投资人常用的评估标准：

| LTV/CAC比率 | 含义 | 成功可能性 |
|------------|------|-----------|
| > 3倍 | 比较靠谱，未来盈利可能性高 | 可达主导地位 |
| 1-3倍 | 有一定成功可能，但风险较高 | 需谨慎评估 |
| < 1倍 | 不可持续的商业模式 | 每获客一个即亏损 |

### AARRR海盗模型与LTV/CAC的关系

| 增长环节 | 对LTV/CAC的影响 |
|---------|----------------|
| 用户获取 | 降低CAC |
| 用户激活 | 降低CAC、提升LTV |
| 用户留存 | 提升LTV |
| 自传播 | 降低CAC |
| 获取收入 | 提升LTV |

## 第一部分：渠道分析

### 1.1 获客渠道的分类与优化

#### 四类获客渠道

| 渠道类型 | 特点 | 适用场景 |
|---------|------|---------|
| **不用白不用** | 免费或低成本、可测量、可预测 | SEO、ASO、落地页优化、产品内自传播、CRM、内容营销 |
| **能力和运气** | 无直接获客成本、难以预测、依赖创意 | 爆款内容、裂变活动、病毒视频、心理测试 |
| **付费数字广告** | 可测量、可预测、可随时启停 | 各类付费数字广告平台 |
| **上台阶** | 费用高、难以测量、难以预测 | 品牌广告、电视广告、事件营销、户外广告 |

#### 优化获客渠道的两个关键点

1. **CAC（获客成本）尽可能低**
2. **保证用户质量高（长期LTV高）**

#### 付费数字广告分析五步法

```
第1步：梳理获客步骤
    ↓
第2步：整理触点和指标
    ↓
第3步：分渠道收集数据
    ↓
第4步：计算获客成本
    ↓
第5步：优化渠道效果
```

### 1.2 获客步骤的三个阶段

#### 阶段一：渠道广告

**特点**：信息量小，需抓住用户注意力

**触点**：
- 渠道选择
- 广告创意、文案

**核心指标**：

| 指标 | 定义 | 优化方式 |
|-----|------|---------|
| 广告展现量 | 广告曝光次数 | 渠道规模、目标用户匹配、竞价竞争力 |
| 广告点击率 | 点击量/展现量 | 精准定位、素材创意测试 |

#### 阶段二：落地承接

**特点**：承上启下，引导用户进入产品

**触点**：
- 营销落地页（网页端/H5）
- 应用商店（App端）

**核心指标**：

| 类型 | 指标 | 说明 |
|-----|------|-----|
| 落地页 | 访问量、转化率、跳出率 | 衡量页面吸引力 |
| 应用商店 | 下载率、首次打开率、卸载率 | 衡量下载转化效果 |

#### 阶段三：产品流程

**特点**：信息量大，用户互动深入

**触点**：
- 辅助转化的内容（浏览、搜索）
- 转化流程（注册、激活、首单）

**核心指标**：

| 指标类型 | 具体指标 |
|---------|---------|
| 弱转化指标 | 浏览时长、访问页面数、搜索次数 |
| 强转化指标 | 注册率、激活率、首单购买率 |
| 长期指标 | 留存率、复购率、LTV |

#### 产品形态示例

**网页端（SaaS软件）**：
```
站外广告 → 落地页 → 网站浏览 → 注册试用
```

**App端（电商）**：
```
站外广告 → 落地页 → 应用商店 → 下载 → 浏览 → 首单
```

**小程序（教育）**：
```
朋友圈广告 → 小程序首页 → 预约体验课
```

### 1.3 数据收集与CAC计算

#### 好的渠道数据体系三大特点

1. **可追踪**：知道用户来源和行为
2. **多渠道**：区分不同渠道来源
3. **全触点**：追踪完整转化漏斗

#### UTM标签系统

网页端追踪使用UTM参数：

| 参数 | 说明 | 示例 |
|-----|------|-----|
| utm_source | 广告来源 | pc_baidu |
| utm_medium | 广告媒介 | 编号 |
| utm_campaign | 广告名称 | 编号 |
| utm_content | 广告内容 | 编号 |
| utm_term | 广告关键字 | 编号 |

#### App端第三方追踪工具

- 国内：友盟、GrowingIO、神策
- 国外：Adjust、AppsFlyer

#### CAC计算公式

$$CAC = \frac{市场总花费}{同期新增用户数}$$

**注意事项**：
- 市场总花费：推广渠道费用+营销活动+品牌活动+销售费用
- 新增用户数：应以激活或完成转化为准，而非仅下载

#### CAC的三种类型

| 类型 | 定义 | 用途 |
|-----|------|-----|
| 付费渠道CAC | 仅付费数字渠道的获客成本 | 评估可控渠道效率 |
| 混合CAC | 总体获客成本（含自然流量） | 评估整体获客效率 |
| 满载CAC | 包含人力成本的获客成本 | 谨慎完整计算 |

#### 实例分析：唯品会渠道对比

| 渠道 | 广告展现量 | 点击率 | 访问量 | 下载率 | 首单率 | 首单CAC |
|-----|----------|--------|--------|--------|--------|---------|
| 百度SEM | 10万 | 10% | 1万 | 30% | 15% | 100元 |
| 抖音广告 | 30万 | 5% | 1.5万 | 15% | 6% | 333元 |

**关键发现**：如果只看下载CAC，抖音只比百度贵5元；但看首单CAC，抖音是百度的3倍多。

### 1.4 渠道优化四种方式

#### 方式一：计算渠道ROI，分配预算

$$ROI = \frac{LTV}{CAC}$$

**优化原则**：
- 保证CAC < LTV
- 将更多预算分配给ROI高的渠道

#### 方式二：拆分获客漏斗，寻找优化机会

**实例：Prisma应用商店优化**

通过应用商店A/B测试，优化图标和截图，将下载率提升12%，直接降低CAC。

**获客漏斗优化原则**：

| 原则 | 说明 | 注意事项 |
|-----|------|---------|
| 步骤越少越好 | 减少流失环节 | 需通过A/B测试验证 |
| 优化高流失率环节 | 提升整体转化率 | 与行业均值对比 |

#### 方式三：细分广告/关键词表现

在渠道内部继续细分到单个广告/关键词水平，优化表现差的广告，增加表现好的广告预算。

**实例：手游公司素材测试**

| 素材类型 | 点击率 |
|---------|--------|
| 经典角色 | 1%-2% |
| 游戏画面 | 7%-8% |
| 九宫格 | 1%-2% |
| 视频形式 | 7%-8% |

#### 方式四：分析用户行为，监测作弊

**实例**：某地产公司发现某应用市场来源用户转化率仅5%（其他渠道45%），进一步分析发现用户行为路径完全一致，确认为作弊流量。

## 第二部分：LTV计算

### 2.1 LTV的定义与作用

#### LTV（Lifetime Value）定义

LTV也称为用户生命周期终身价值，指每一位用户在未来可能为产品或服务带来的价值总和。

#### LTV计算方式

| 计算方式 | 说明 | 推荐程度 |
|---------|------|---------|
| 收入LTV | 用户未来带来的收入总和 | 不够精确 |
| 毛利LTV | 扣除产品成本后的毛利总和 | **推荐**，更准确衡量回本能力 |

#### LTV的核心作用

1. **选择和衡量获客渠道**
   - 确定获客成本上限
   - 计算各渠道ROI

2. **评估经营活动**
   - 评估产品改进和运营活动效果
   - 评估商业模式变现能力
   - 估算未来收入和毛利润

#### LTV与渠道选择的关系

```
LTV越高 → 可支持的CAC越高 → 可选择的渠道越多
```

| 产品类型 | LTV水平 | 可选渠道 |
|---------|---------|---------|
| 社交网络（早期） | 低 | 病毒传播、SEO、UGC |
| 游戏 | 中 | 付费渠道（抖音、微信广告） |
| SaaS/企业服务 | 高 | 销售团队、补贴、会议活动 |

### 2.2 LTV的简化计算公式

#### 基础公式

$$LTV = 用户平均生命周期 \times 平均每用户收入 \times 毛利率$$

#### 变量一：用户平均生命周期

**估算方式**：

| 情况 | 估算方法 |
|-----|---------|
| 生命周期长、流失率稳定 | 平均生命周期 = 1 ÷ 月流失率 |
| 生命周期短或有回本目标 | 人为选择时间段（60/90/180天） |

#### 变量二：平均每用户收入

**估算方式**：

| 产品类型 | 计算方法 |
|---------|---------|
| 交易类（电商） | 月购买频次 × 每单客单价 |
| 广告/游戏类 | 日总收入 ÷ 日总DAU |

#### 变量三：毛利率

$$毛利率 = \frac{销售收入 - 销售成本}{销售收入} \times 100\%$$

**不同产品类型的毛利率**：

| 产品类型 | 毛利率范围 |
|---------|-----------|
| 电商类 | 较低（含实物成本） |
| SaaS/游戏 | 高（70%-80%） |

#### 实例：星巴克LTV计算

| 变量 | 数值 |
|-----|------|
| 用户生命周期 | 20年（1040周） |
| 每周购买次数 | 4.2次 |
| 每次消费金额 | 5.9美元 |
| 毛利率 | 21% |

$$LTV = 1040 \times 4.2 \times 5.9 \times 21\% = 5,412美元$$

#### 三种常见变现模式的LTV计算

##### 1. 交易类（电商）

$$平均生命周期 = \frac{1}{月流失率} = \frac{1}{50\%} = 2个月$$

$$每用户收入 = 月购买次数 \times 每单价值 = 2 \times 200 = 400元$$

$$LTV = 2 \times 400 \times 20\% = 160元$$

##### 2. 订阅类（Netflix）

$$平均生命周期 = \frac{1}{年流失率} = \frac{1}{9\%} \approx 11年$$

$$每用户收入 = 115美元/年$$

$$LTV = 11 \times 115 \times 40\% = 506美元$$

##### 3. 免费类（游戏+广告）

$$平均生命周期 = 30天（回本期）$$

$$每用户收入 = \frac{30天总收入}{30天总DAU} = 1元$$

$$LTV = 1 \times 80\%（毛利率） = 0.8元$$

## 第三部分：变现金字塔

### 3.1 变现金字塔模型

变现金字塔由三层构成：

```
        ┌─────────────────┐
        │   转化效率优化   │  ← 最顶层
        ├─────────────────┤
        │   定价策略       │  ← 中间层
        ├─────────────────┤
        │   变现模式       │  ← 最底层（根基）
        └─────────────────┘
```

### 3.2 第一层：变现模式

变现模式是金字塔的最底层和根基。

#### 三大变现模式

| 模式 | 说明 | 代表产品 |
|-----|------|---------|
| **交易类** | 用户付费获得商品/服务 | 淘宝、天猫、樊登读书会、三节课 |
| **订阅类** | 用户定期付费获得使用权 | SaaS软件、爱奇艺、Netflix、钉钉 |
| **免费类** | 前端免费，后端变现 | 微信、王者荣耀、小红书、抖音 |

#### 交易类细分

| 类型 | 说明 | 实例 |
|-----|------|-----|
| 卖实体货品 | 用户付费获得实物商品 | 网易严选、天猫 |
| 卖电子商品 | 用户付费获得虚拟商品 | 电子书、课程 |
| 收取平台交易税 | 平台从交易中抽成 | App Store、应用商店 |
| 双边市场 | 商家付费获得流量 | 百度竞价、美团 |
| 增值服务 | 提供额外服务收费 | 京东金融分期 |

#### 订阅类细分

| 类型 | 说明 | 实例 |
|-----|------|-----|
| SaaS订阅 | 企业付费获得解决方案 | 各类企业软件 |
| 内容订阅 | 用户付费获得内容 | 知识星球、樊登读书会 |
| 会员订阅 | 用户付费获得VIP权益 | 爱奇艺会员 |

#### 免费类细分

| 类型 | 说明 | 实例 |
|-----|------|-----|
| 广告变现 | 用户免费使用，看广告 | 微信、豆瓣、头条 |
| 应用内购买 | 购买虚拟物品 | 游戏装备、直播打赏 |
| 免费增值 | 基础免费，升级付费 | 印象笔记、各类工具 |
| 数据变现 | 收集用户数据出售 | 汽车之家、NerdWallet |

### 3.3 第二层：定价策略

#### 三种常见定价策略

```
高价策略（撇脂定价）
    ↑
满意定价（适中策略）
    ↑
低价策略（渗透定价）
```

| 策略 | 价格水平 | 目标 | 适用场景 | 实例 |
|-----|---------|------|---------|------|
| 撇脂定价 | 最高 | 最大化利润 | 垄断地位/强差异化 | 早期苹果、限量球鞋 |
| 满意定价 | 适中 | 销量与利润平衡 | 市场竞争不激烈 | 保守型产品 |
| 渗透定价 | 最低 | 最大化市场份额 | 价格敏感用户 | 早期小米、9.9包邮 |

#### 爱奇艺定价策略实例

- 免费用户
- 黄金VIP会员：178元/年
- 钻石VIP会员：249元/年
- 多种付费周期：月费、季费、年费
- 合作优惠：京东、屈臣氏合作
- 新用户优惠：限时5折

### 3.4 第三层：转化效率优化

#### 付费转化杠杆公式

$$转化 = 用户感知价值 - （用户感知价格 + 阻力和顾虑）$$

| 要素 | 说明 | 优化方式 |
|-----|------|---------|
| 用户感知价值 | 用户觉得得到的好处 | 功能价值、心理诉求 |
| 用户感知价格 | 用户觉得付出去的成本 | 锚定效应、尾数定价（9.99元） |
| 阻力和顾虑 | 转化障碍 | 简化流程、建立信任、减少要求 |

## 第四部分：价格敏感度测试

### 4.1 价格敏感度测试概述

#### 什么是价格敏感度测试（PSM）

价格敏感度测试（Price Sensitivity Measurement）是70年代发明的价格测试方法，通过了解目标用户的价格敏感区间，找到消费者和商家的最佳价格平衡点。

#### 测试目标

- **企业角度**：在用户可接受范围内定最高价，实现营收最大化
- **用户角度**：找到最大可接受区间，不会觉得太贵或太便宜而怀疑质量

### 4.2 价格敏感度测试四步骤

#### 步骤一：制作价格选项

制作一个从低到高的价格梯度表：

| 注意事项 | 说明 |
|---------|------|
| 区间要够大 | 测试出用户对哪个价位敏感 |
| 最低价不低于成本 | 否则无意义 |
| 最高价留出足够空间 | 参考市场行情上浮 |
| 目标价位居中 | 以3%-5%比例调整 |
| 选项不宜过多 | 避免用户选择困难 |

**实例**：成人编程课价格梯度表

```
1000元 - 1500元 - 2000元 - 2500元 - 3000元 - 3500元 - 4000元 - 4500元 - 5000元
```

#### 步骤二：设计测试问卷

向用户提出四个问题：

| 问题 | 问题描述 | 选择顺序 |
|-----|---------|---------|
| 1. 很划算 | 什么价格觉得很划算，肯定会买 | 从高到低 |
| 2. 有点贵 | 什么价格觉得有点贵，但还会买 | 从低到高 |
| 3. 太贵了 | 什么价格觉得太贵，肯定不会买 | 从低到高 |
| 4. 太便宜 | 什么价格觉得太便宜，怀疑质量不会买 | 从高到低 |

#### 步骤三：发放问卷

| 注意事项 | 说明 |
|---------|------|
| 样本量 | 至少100-200人，保证80个有效样本 |
| 样本选择 | 新产品覆盖可能目标人群；迭代产品包括老用户和竞品用户 |

#### 步骤四：统计问卷结果

##### 1. 累积选择人数

- 问题1和4（便宜相关）：从下往上累积
- 问题2和3（贵相关）：从上往下累积

##### 2. 转换为百分比

每个选项的累积人数 ÷ 总样本数 × 100%

##### 3. 绘制四条曲线

- 蓝色曲线：很划算，肯定会买
- 红色曲线：有点贵，但还会买
- 黄色曲线：太贵了，肯定不会买
- 绿色曲线：太便宜，怀疑质量不会买

##### 4. 找到关键点

| 关键点 | 定义 | 含义 |
|-------|------|------|
| A点 | 绿色曲线与红色曲线交点 | 可接受价格区间下限 |
| B点 | 蓝色曲线与红色曲线交点 | **最优可接受价格点** |
| C点 | 蓝色曲线与黄色曲线交点 | 可接受价格区间上限 |

### 4.3 价格敏感度测试的适用场景

#### 适用产品

| 产品类型 | 适用原因 | 实例 |
|---------|---------|------|
| 游戏类 | 虚拟世界，全凭用户心理感受 | 游戏道具定价 |
| 工具类 | 功能明确，用户好理解 | 统计工具定价 |
| 课程类 | 新产品上线前定价 | 编程课定价 |
| 电商折扣 | 测试折扣力度 | 促销活动定价 |

#### 不适用场景

- 竞品特别多的产品
- 定价模型复杂的产品
- 需要考虑实际购买力的场景

#### 价格敏感度测试的局限性

| 局限性 | 说明 |
|-------|------|
| 只考虑价格因素 | 忽略销量和购买意愿变化 |
| 只考虑接受率 | 忽略用户实际购买能力 |
| 忽略竞品影响 | 未考虑市场竞争 |
| 非真实消费环境 | 用户选择不代表实际行为 |

## 本章核心要点总结

### 增长底层逻辑

1. **LTV > 3倍CAC**：健康商业模式的标志
2. **TAM决定上限**：市场规模决定增长天花板
3. **四个契合**：市场-产品、市场-模型、渠道-模型、渠道-产品

### 渠道分析核心

1. **四类渠道**：不用白不用、能力和运气、付费数字广告、上台阶
2. **三阶段漏斗**：渠道广告 → 落地承接 → 产品流程
3. **数据体系三特点**：可追踪、多渠道、全触点
4. **CAC计算**：以激活/转化为准，而非仅下载
5. **四种优化方式**：ROI分配、漏斗优化、细分广告、监测作弊

### LTV计算核心

1. **简化公式**：LTV = 生命周期 × 每用户收入 × 毛利率
2. **两种LTV**：收入LTV（不精确）、毛利LTV（推荐）
3. **三种变现模式**：交易类、订阅类、免费类
4. **核心作用**：量入为出，确定获客成本上限

### 变现思维模型

1. **变现金字塔**：变现模式（根基）→ 定价策略 → 转化效率
2. **三种定价策略**：撇脂定价（高价）、满意定价（适中）、渗透定价（低价）
3. **转化杠杆**：价值 - （价格 + 阻力）

### 价格敏感度测试

1. **四步骤**：制作选项 → 设计问卷 → 发放收集 → 统计分析
2. **三个关键点**：A点（下限）、B点（最优）、C点（上限）
3. **适用场景**：新产品、无竞品、定价简单
