## 🔑 核心概念定义

| 概念 | 定义 |
|------|------|
| **Aha时刻** | 新用户首次确认产品对自己有价值、有意义的灵光乍现时刻，是用户感受到产品价值的关键瞬间 |
| **新用户激活** | 让新用户在特定时间内完成特定行为，从而感受到产品价值的过程 |
| **激活率** | 新用户在一定时间内达到Aha时刻的比率，是新用户激活的北极星指标 |
| **激活行为** | 用户在早期完成的关键行为，与长期留存正相关 |
| **魔法数字** | 用户需要完成激活行为的最优次数，以达到最佳留存效果 |
| **激活期** | 新用户需要多快完成激活行为的时间窗口 |
| **激活漏斗** | 从用户接触到产品到完成激活行为的全过程步骤 |
| **激动指数** | 用定性的方式评估新用户在激活过程中的情感体验变化 |
| **支持行为** | 用户到达Aha时刻必须完成，但本身不创造价值的步骤 |
| **边际效用最大点** | 激活行为次数增加对留存率的贡献开始显著下降的拐点 |

## 📚 详细内容

### 0. 本章导读：提升新用户激活的4步

#### 0.1 用户激活与留存的四个阶段

通过健身房的例子，我们可以理解用户激活与留存的完整生命周期：

**第一阶段：新用户激活**
- **目标**：让用户首次发现价值（Aha时刻）
- **时间**：前一天或前几天
- **关键**：确保第一次体验完美

**第二阶段：新用户留存**
- **目标**：让用户发现更多价值，养成使用习惯
- **策略**：持续上手、发现新功能、形成定期使用习惯
- **时间**：前几天到几个月

**第三阶段：长期用户留存**
- **目标**：加深参与度，增加使用场景和功能
- **策略**：提升使用频次和强度，增加功能使用

**第四阶段：流失用户召回**
- **目标**：让已流失用户重新发现产品价值
- **策略**：提供新优惠、新功能，争取召回

#### 0.2 新用户激活的重要性

**新用户流失的三大原因**：

1. **产品功能问题**
   - 产品无法满足用户需求
   - 需求是伪需求、非刚需
   - **解决方法**：改进产品或调整市场定位

2. **渠道用户匹配问题**
   - 拉新渠道选择不当
   - 用户定位不准确
   - **解决方法**：精准拉新

3. **产品价值发现问题**
   - 产品有价值但用户未发现
   - 引导不够好
   - **解决方法**：优化新用户激活

**新用户激活的重要性**：

1. **位于流量最上方**：激活率的微小变化能将整个留存曲线向上提升
2. **用户可塑性强**：新用户对产品敏感，容易受影响
3. **性价比高**：投入产出比高，改善空间大

**最佳实践**：

> 在10人的工程师团队中，5人负责产品开发，5人负责新用户激活。因为新用户激活做不好，新产品也无人使用。

#### 0.3 提升新用户激活的四步框架

1. **以终为始，明确激活目标**（Aha时刻）
2. **分析数据，寻找增长线索**（激活漏斗+激动指数）
3. **参考最佳实践，设计方案**（用户行为公式+激活金字塔）
4. **持续实验和迭代优化**（A/B测试验证）

### 1.1 以终为始，深入理解新用户激活的目标

#### 1.1.1 "以终为始"的思维模式

**核心思想**：首先清晰决定最终目标，然后以这个目标为起点，决定每时每刻要做的事情。

**在新用户激活中的应用**：

- **终点**：将用户转变为长期留存的顾客，持续使用产品，获得价值
- **起点**：找到哪些早期关键行为能够帮助用户达到长期留存的理想状态

#### 1.1.2 什么是Aha时刻？

**定义**：新用户开始使用产品时，首次确认产品对自己有价值、有意义的灵光乍现时刻。

**Aha时刻的特点**：

- 有感情、有温度的用户体验
- 用户对产品产生强烈的第一印象
- 感到惊喜和兴奋
- 决定用户成为留存用户还是流失用户

**经典案例**：

| 产品 | Aha时刻描述 |
|------|------------|
| 微信 | 首次实现与外地/国外家人的快速联系 |
| 搜索引擎 | 输入问题，立即获得大量结果 |
| 美团外卖 | 饥饿时下单，半小时后香喷喷的饭菜送达 |
| 支付宝 | 稳定使用3个以上功能 |
| 激萌 | 利用滤镜完成第一张照片美化 |
| 全民K歌 | 唱歌后听到处理后的好听声音 |
| Airbnb | 6个月内完成首次订单且评价4星以上 |
| Slack | 团队早期共发送2000条信息 |
| 游戏产品 | 注册后第二天返回 |

#### 1.1.3 Aha时刻的标准格式

**通用公式**：

> **谁** 在 **多长时间** 完成 **多少次** **什么行为**

**要素解析**：

- **谁**：行动单位（个人用户或团队）
- **多长时间**：新用户激活时间窗口（前1天/3天/7天等）
- **多少次**：魔法数字（最优行为次数）
- **什么行为**：激活行为或早期关键行为

#### 1.1.4 Aha时刻的三个核心假设

**假设一：产品对用户有长期价值**
- 产品已达到PMF（产品市场契合）
- 用户是目标用户群体

**假设二：早期关键行为能让用户快速感受价值**
- 新用户注意力和时间有限
- 不追求大而全，聚焦最关键行为
- 功能复杂的产品更需聚焦

**假设三：感受到价值的用户更可能留存**
- 用户先尝到甜头，发现产品用途
- 即使不是全部功能，也愿意持续使用
- 持续发现更多价值

#### 1.1.5 常见误区：机械化定义

**错误做法**：机械地定义Aha时刻，强制用户完成动作

**失败案例：Jira**

- **发现**：新用户写3个ticket与留存正相关
- **错误做法**：直接在软件中预放3个现成ticket
- **结果**：失败，对留存无提升
- **原因**：用户是在写ticket过程中发现Jira的价值（记录PRD、团队协作），机械放ticket无法模拟这一过程

**成功案例：Twitter**

- **发现**：关注30个用户是Aha时刻
- **假设**：推动用户关注30人能帮助达到Aha时刻
- **验证**：实验成功
- **原因**：关注30人后形成基本信息流，用户看到有趣内容，感受到Twitter价值

#### 1.1.6 Aha时刻的本质

**两个关键要点**：

1. **简化行为数据**：通过简化的行为数据模拟用户首次获得价值的时刻
   - 新用户激活情况各异
   - 找到适用于大多数用户的激活行为
   - 为激活努力提供清晰目标

2. **模拟价值时刻**：最重要的是模拟用户获得价值的时刻
   - 以让用户感到价值为目标
   - 不能机械强迫用户
   - 要想办法帮助用户体会产品价值

### 1.2 4个步骤，找到新用户的Aha时刻

#### 1.2.1 找到Aha时刻的四步框架

```
步骤1：提出备选行为（定性探索）
    ↓
步骤2：找到激活行为（定量分析，寻找相关性）
    ↓
步骤3：计算魔法数字（找到最优次数）
    ↓
步骤4：验证因果性（A/B测试验证）
```

**流程特点**：

- 步骤1：定性过程（开放性探索）
- 步骤2-3：定量分析（寻找相关性）
- 步骤4：实验验证（确认因果性）

#### 1.2.2 步骤1：提出备选行为

**目标**：明确产品长期价值，倒推能帮助用户感受价值的短期行为

**方法一：关键问题法**

**四个关键问题**：

1. **用户是谁？**
2. **用户要用产品解决什么问题？**
3. **为什么要解决这个问题？**
4. **还有什么其他方法解决？**

**案例：美颜相机**

| 问题 | 答案 |
|------|------|
| 用户是谁？ | 大多为女性 |
| 解决什么问题？ | 拍照片 |
| 为什么？ | 想让自己看起来更美（皮肤白皙、脸更瘦、眼睛更大） |
| 其他方法？ | 拍照后用Photoshop修图 |

**备选激活行为**：
- 拍一张照片
- 使用一个滤镜
- 分享照片

**案例：少儿英语产品**

| 问题 | 答案 |
|------|------|
| 用户是谁？ | 父母和孩子 |
| 解决什么问题？ | 提升孩子英语水平 |
| 为什么？ | 提升成绩、口语水平，为考试出国打基础 |
| 其他方法？ | 参加英语兴趣班、请家教 |

**备选激活行为**：
- 完成一节课程
- 完成课程测试并看成绩
- 长期学习后看到学校英语成绩提升

**可迅速完成的行为**：前两个（第三个时间跨度太长）

**方法二：用户调研法**

**调研三类用户**：

1. **长期活跃用户**
   - 问题：从产品中发现的价值是什么？
   - 作用：找到产品最核心价值点

2. **注册后快速流失用户**
   - 问题：为什么迅速流失？是否没发现价值？本来想做什么？
   - 作用：了解流失原因

3. **注册后开始活跃的用户**
   - 问题：为什么留下来？新用户时期做了什么动作？使用了什么功能？
   - 作用：对比留存和流失用户的差异

**适用场景**：产品有多个使用场景或功能，需要缩小备选行为范围

#### 1.2.3 步骤2：找到激活行为

**目标**：在所有备选行为中，找到与早期留存正相关性最强的行为

**方法**：定量分析

**具体步骤**：

**步骤1：找到新用户激活期（时间窗口）**

**三条参考原则**：

| 原则 | 说明 | 例子 |
|------|------|------|
| 使用频次越高，激活需要越快 | 日活产品需快速激活 | 游戏、社交、短视频：第1天最理想，最慢前3天 |
| 产品生命周期越短，激活需要越快 | 短生命周期产品需快速激活 | SaaS产品：前30天；小游戏：需非常快速 |
| 参考实际数据 | 看大多数用户行为发生时间 | 美颜相机80%首次拍照在第1天；股票投资app80%首次投资在前10天 |

**步骤2：找到关键激活行为**

**方法**：对比留存曲线

**操作流程**（以美颜相机为例）：

1. **收集数据**：收集所有新用户前31天留存数据
2. **用户分组**：
   - 第一组：所有新用户
   - 第二组：第一天用过滤镜的用户
   - 第三组：第一天没用滤镜的用户
   - 第四组：第一天拍过照的用户
   - 第五组：第一天没拍照的用户
3. **画出留存曲线**：对比五组用户的留存率
4. **分析差异**：找到有/无某个行为对留存影响最大的行为

**数据示例**：

| 用户组 | 初始人数 | 第2天留存率 |
|--------|----------|------------|
| 所有新用户 | 10,000 | 30.2% |
| 第1天用滤镜 | 6,000 | 36.5% |
| 第1天没滤镜 | 4,000 | 20.8% |
| 第1天拍照 | 7,000 | 35.5% |
| 第1天没拍照 | 3,000 | 17.8% |

**分析结论**：
- 用滤镜vs没滤镜的差异最大
- 第1天用滤镜比第1天拍照与留存相关性更强
- 激活行为定位为：使用滤镜

**工具支持**：

- **Amplitude**：留存分析功能，用户分群分解留存曲线
- **Growing IO**：对比不同功能对后续登录的影响
- **国内分析工具**：类似功能

#### 1.2.4 步骤3：计算魔法数字

**目标**：找到激活行为的最优次数，确保用户感受到价值的同时不带来负担

**为什么需要魔法数字**：

1. **行为性质不同**：有些行为一次足够，有些需要多次
2. **寻找平衡点**：理论上线多越好，但新用户时间有限，需找到性价比最高的点

**方法一：边际效用最大法**

**三个步骤**：

1. **画出次数分布图**：新用户激活行为次数的分布
2. **分析次数与留存关系**：不同次数对应的留存率
3. **找到拐点**：边际效应最大的点（留存率提升显著下降的点）

**案例：美颜相机**

| 滤镜次数 | 人数 | 次日留存率 |
|----------|------|------------|
| 0次 | 4,000 | 20.8% |
| 1次 | 3,200 | 33.2% |
| 2次 | 1,800 | 37.9% |
| 3次 | 800 | 39.5% |
| 4次 | 200 | 40.0% |

**分析**：
- 0次→1次：留存率提升12.4%（最大）
- 1次→2次：留存率提升4.7%
- 2次→3次：留存率提升1.6%
- 3次→4次：留存率提升0.5%

**结论**：魔法数字为1次（拐点在1次）

**方法二：韦恩图法**

**原理**：找到既有该行为又留存的人数占总人数百分比最高的点

**计算步骤**：

1. 计算至少使用N次的人数
2. 找到三类用户：
   - 有该行为且留存的用户
   - 有该行为但未留存的用户
   - 留存但无该行为的用户
3. 计算占比，找到最大值

**案例结论**：第一天至少使用滤镜1次，占比32%最高

#### 1.2.5 步骤4：验证因果性

**为什么要验证因果性**：

- **相关性≠因果性**：打伞和地上湿同时发生，但无因果关系
- 目标是找到因果关系：让用户做某个行为会导致留存率更高

**验证方法**：A/B测试

**案例：唱吧**

**实验设计**：
- 对照组：不引导用户唱歌
- 实验组1：引导用户唱3首歌
- 实验组2：引导用户唱1首歌

**结果**：实验组2（唱1首）效果最好

**结论**：
1. 唱歌比不唱歌能提升留存（验证Aha时刻）
2. 唱1首比唱3首效果好（验证魔法数字）
3. 唱3首反而增加负担导致流失

#### 1.2.6 四步法总结

```
美颜相机案例总结：

步骤1：提出备选行为
├─ 关键问题法：拍照、使用滤镜、分享
└─ 用户调研法：确认核心行为

步骤2：找到激活行为
├─ 确定激活期：第1天
├─ 对比留存曲线：用滤镜vs不用滤镜差异最大
└─ 定位激活行为：使用滤镜

步骤3：计算魔法数字
├─ 边际效用法：拐点在1次
└─ 韦恩图法：验证1次为最优

步骤4：验证因果性
├─ A/B测试：推动用户使用1次滤镜
└─ 确认Aha时刻：第1天内试用1次滤镜
```

**注意事项**：

1. **魔法数字非绝对**：找到正确的激活行为更重要
2. **魔法数字可细化**：根据用户画像进一步精准化
3. **阶段性打法**：可设置多个魔法数字阶梯

### 2.1 寻找新用户激活线索的方法1：激活漏斗

#### 2.1.1 数据分析的两个维度

**定量分析**：
- 告诉我们"是什么"
- 包括：漏斗、分群
- 问题：不能告诉"为什么"

**定性分析**：
- 告诉我们"为什么"
- 包括：用户调研、问卷
- 作用：了解背后的原因和顾虑

**最佳实践**：结合定量和定性分析

#### 2.1.2 新用户激活的整体数据分析方案

```
发现问题（定量）
├─ 激活漏斗：找到流失率最高的步骤
└─ 激动指数：了解用户体验的情绪变化

探寻原因
├─ 路径分析：查看用户是否被无关路径干扰
├─ 轨迹细查：查看流失用户的具体行为
├─ 时间间隔：分析漏斗各步骤的时间长度
├─ 漏斗分群：对比不同用户群的转化率
├─ 点击热图：分析用户点击行为
└─ 用户调研：了解用户思维和困惑

设计方案
└─ 针对原因设计实验方案
```

#### 2.1.3 案例引入：互联网金融App

**场景**：产品Aha时刻是用户首次投资

**问题发现**：
- 构建激活漏斗后，发现"输入银行信息"是最大流失点

**激动指数分析**：
- 用户在银行登录页激动指数很低

**原因分析**：
- 用户对产品还不信任
- 太早要求输入银行卡信息
- 页面看起来不够安全

**解决方案**：
- 加入安全标志
- 解释为什么需要连接银行
- 调整步骤位置

#### 2.1.4 构建新用户激活漏斗

**步骤1：明确激活指标**

**激活率定义**：

> 新用户在一定时间内达到Aha时刻的比率

**案例：Pinterest**

- **Aha时刻**：1天内首次保存图片
- **激活率**：1天内保存图片的新用户占比

**激活率的作用**：
- Aha时刻的定量化
- 新用户激活的北极星指标
- 可衡量的标准
- 可追踪趋势变化

**步骤2：梳理新用户流程**

**全链漏斗**（从用户视角）：

```
看到广告/了解产品
    ↓
点击广告/到达官网
    ↓
应用商店/下载
    ↓
首次开机
    ↓
开屏引导页
    ↓
注册流程
    ↓
首次产品登录
    ↓
推送等CM渠道体验
```

**产品漏斗**（从产品视角）：

```
首次开机
    ↓
注册流程
    ↓
首次产品登录
    ↓
激活行为
```

**建议**：全面梳理全链漏斗，包括广告、应用商店、推送等，不局限于产品内部

**步骤3：组建激活漏斗**

**示例数据**：

| 步骤 | 人数 | 转化率 |
|------|------|--------|
| 下载应用 | 1,000 | 100% |
| 首次打开 | 800 | 80% |
| 注册第一步 | 500 | 62.5% |
| 注册第二步 | 300 | 60% |
| 激活行为 | 162 | 54% |

**计算方法**：每一步以上一步为分母

**发现**：注册第二步（60%）和完成激活动作（54%）转化率最低

**步骤4：分析数据发现线索**

**核心概念：支持行为 vs 激活行为**

| 类型 | 定义 | 例子 |
|------|------|------|
| **支持行为** | 用户必须完成但本身不创造价值的步骤 | 下载、注册、浏览内容 |
| **激活动作** | 让用户感受到价值的关键行为 | 首次会议、听完音频 |

**案例对比**：

| 产品 | Aha时刻 | 支持行为 |
|------|----------|----------|
| Zoom | 前7天进行首次会议且效果好 | 下载、注册、安排会议 |
| 得到 | 前3天听完一个音频 | 下载、注册、浏览内容找音频 |

**处理支持行为的两条原则**：

**原则1：直奔主题**
- 不必要的支持行为越少越好
- 每多一步，流失就可能增加
- 不需要的权限/信息不要要

**原则2：权衡利弊**
- 有些支持行为有好处但增加阻力
- 例如：要推送权限
- 需要优化找到最佳加入点

**案例对比：TikTok vs 快手**

| 维度 | TikTok | 快手 |
|------|--------|------|
| 开机体验 | 全屏直接播放短视频 | 小卡片界面，需点击 |
| 权限请求 | 使用一段时间后请求 | 一上来就请求各种权限 |

**分析**：
- TikTok直奔主题更快
- TikTok对内容和算法有信心
- 权限请求时机需要测试优化

**分析数据的思路**：

1. **必要性问题**：哪些步骤可以去掉？顺序能否调整？
2. **流失率问题**：哪几步流失率最高？原因是什么？
3. **深入分析**：
   - 路径分析：是否被无关路径干扰
   - 用户分群：不同渠道/城市/性别转化率差异
   - 用户调研：了解用户顾虑和困惑

#### 2.1.5 案例：Calm冥想App

**背景**：正念健康生活方式App，帮助用户做呼吸练习

**步骤1：明确激活指标**
- **Aha时刻**：3天内完成首次冥想
- **激活率**：完成首次冥想的新用户占比

**步骤2：梳理新用户流程**

```
首次开机
    ↓
完成呼吸练习（短）
    ↓
提示可完成首次冥想
    ↓
点击指引完成首次冥想
    ↓
冥想完成
```

**步骤3：组建激活漏斗**

**发现**：第二步（呼吸练习）到第三步（冥想开始）流失率35%，最高

**步骤4：深入分析**

**线索1：环境问题**
- 用户可能在不合适环境（上班、地铁、嘈杂场所）
- 无法安静做呼吸练习
- 只能关闭应用，之后遗忘

**解决方案**：
- 去掉这一步
- 提前提示用户找合适环境
- 其他实验方案

**线索2：分群分析（按国家）**
- 美国、英国：流失率高但没那么高
- 印度、中国、巴西：流失率极高（80-90%）

**假设**：翻译问题导致用户不理解

**解决方案**：重新审核这些国家的文案

**其他分群维度**：

| 维度 | 用途 |
|------|------|
| 用户画像 | 不同用户群转化率差异 |
| 获客渠道 | 不同渠道来源的激活率差异 |
| 设备平台 | 移动端vs桌面端表现差异 |
| 产品线 | 不同产品线的激活率差异 |
| 推送/邮件 | 是否收到推送对激活率影响 |
| 红包/补贴 | 激励对激活率/流失率的影响 |

### 2.2 寻找新用户激活线索的方法2：激动指数

#### 2.2.1 什么是激动指数？

**定义**：粗略定性评估新用户体验，找到激活过程中让用户兴奋/郁闷的元素

**核心概念**：

- **初始状态**：用户尝鲜的兴奋感和激动感
- **过程变化**：激活过程中的情感波动
- **阻碍伤害**：额外阻碍降低激动程度
- **最终状态**：决定是否继续前进

**激动指数的作用**：

- 识别让用户兴奋的元素（加油）
- 识别让用户郁闷的元素（倒胃口）
- 了解用户的情感体验
- 找到优化方向

#### 2.2.2 应用激动指数的步骤

**步骤1：明确新用户流程**

- 用截图/文字记录完整流程
- 放入PPT或文档
- 包含页面、设计、文案等所有元素

**步骤2：评估初始激动指数**

**不同渠道的初始激动程度**：

| 渠道类型 | 激动程度 | 说明 |
|----------|----------|------|
| 天然访问 | 一般 | 听说过品牌，有一定了解 |
| 主动搜索 | 高 | 有明确需求 |
| 广告点击 | 低 | 被广告轰炸，只是看看 |

**步骤3：评估各元素对指数的影响**

**方法1：粗略估计法**

- 从用户视角假装自己是新用户
- 走完整流程并打分
- 基础分50分，根据体验加减

**案例**：

| 步骤 | 元素 | 分数变化 | 当前分数 |
|------|------|----------|----------|
| 初始 | 对产品感兴趣 | +0 | 50 |
| 下载 | 速度非常慢 | -5 | 45 |
| 开机 | 开屏动画炫酷 | +10 | 55 |
| 注册 | 流程很长要很多信息 | -10 | 45 |

**方法2：用户测试法（精确）**

- 招募目标用户（潜在用户，未使用过产品）
- 让用户走完流程并给每个步骤打分
- 收集原因：为什么高分/低分
- 汇总计算平均分

**步骤4：优化流程**

| 优化方向 | 具体做法 |
|----------|----------|
| 降低/去掉 | 让用户倒胃口/郁闷的元素 |
| 做得更好/加入 | 让用户兴奋的元素 |
| 调整顺序 | 改变步骤顺序 |
| 确保一致性 | 前后文案、设计风格一致 |

**步骤5：定期评估**

- 建议至少3个月重新走一次
- 检查是否有需要改进的地方
- 确保与产品变化一致
- 性价比很高的优化方式

#### 2.2.3 案例：微信读书

**新用户激活流程**：

```
朋友圈看到无限卡分享（送我一张）
    ↓
点击打开落地页
    ↓
点击"微信读书获取无限卡"
    ↓
到达应用商店下载
    ↓
首次开机（未登录）
    ↓
看到感兴趣的无限卡（首屏）
    ↓
浏览免费图书
    ↓
微信登录
    ↓
打开第一本书开始阅读（Aha时刻）
```

**激动指数评估**：

| 阶段 | 激动指数 | 说明 |
|------|----------|------|
| 朋友圈看到 | 较高 | 朋友分享，感兴趣 |
| 应用商店下载 | 最高 | 整个体验的最高点 |
| 首次打开（未登录） | 最低 | 激动指数低谷 |
| 打开第一本书 | 回升 | 又有回升 |

**漏斗数据验证**：未登录体验确实是流失率高、激动指数低的点

**优化思路**：

- 爆款书是否可做免费体验
- 单独页面展示，保持前后一致
- 提升未登录首屏体验

#### 2.2.4 总结：寻找激活线索的思路

**思路1：定量+定性相结合**

- **定量**：告诉我们"是什么"（转化率、流失率）
- **定性**：告诉我们"为什么"（用户顾虑、困惑）

**思路2：综合多种数据分析方法**

- 通过一种分析发现问题
- 用多种方法进一步拆解
- 得到可操作、具体的线索
- 指导激活实验和方案设计

### 3.1 新用户激活的最佳实践（上）

#### 3.1.1 提升新用户激活的两个核心框架

**框架1：用户行为公式**

```
行为 = 动力 - 阻力 × 助推 + 奖励
```

**框架2：用户激活金字塔**

```
        人工
       /   \
      激励 渠道
       \   /
        产品
```

#### 3.1.2 用户行为公式详解

**公式拆解**：

| 要素 | 定义 | 例子 |
|------|------|------|
| **行为** | 想让用户做的事情 | 注册账号、完成首次投资 |
| **动力** | 用户多想要完成这件事 | 需要买火车票的迫切程度 |
| **阻力** | 行为有多容易做到 | 输入信息的难度、界面易用性 |
| **助推** | 适当时候的触发提示 | 推送提醒买票 |
| **奖励** | 完成后的反馈 | 购票成功确认 |

**重要性排序**：

1. **动力和阻力**：主要因素
   - 动力越强，完成可能性越高
   - 阻力越大，完成可能性越低

2. **助推和奖励**：辅助因素
   - 起补充作用
   - 不是主要原因

#### 3.1.3 影响用户动力的因素

**因素1：用户自身需求**

| 因素 | 说明 |
|------|------|
| 需求强烈程度 | 需求是否很强烈、很紧急 |
| 替代选择 | 是否有其他替代方案 |
| 决策时间 | 决策时间长短（如企业采购需半年） |
| 使用者vs决策者 | 是否同一人（影响目标对象） |

**因素2：外部助攻（心理学因素）**

| 方法 | 原理 | 例子 |
|------|------|------|
| 从众效应 | 告诉用户很多人已做 | "已有1万人完成" |
| 稀缺性 | 制造紧迫感 | "只剩2张票" |
| 紧急性 | 提示时间有限 | "马上就要卖光了" |

#### 3.1.4 影响用户阻力的因素

**因素1：产品价值感知难度**

| 方面 | 说明 |
|------|------|
| 功能理解难度 | 功能是否容易理解、上手 |
| 到达Aha时刻时间 | 时间长短（投资回报需很长时间） |

**因素2：用户能力和素质**

| 例子 | 说明 |
|------|------|
| 中老年用户 | 对科技产品了解和使用能力较低 |

**因素3：外部阻碍**

| 类型 | 说明 | 例子 |
|------|------|------|
| **物理障碍** | 明显的额外步骤 | 注册流程太长、表格不好用、要求先付费 |
| **认知障碍** | 理解困难 | 文案不清楚、选择太多、信息不一致 |

#### 3.1.5 激活难度评估矩阵

```
        用户需求
         强
          ┃
   理想   ┃  激活难度大
   (工具)  ┃  (SaaS)
          ┃
━━━━━━━━━╋━━━━━━━━━
          ┃
   激活难度小  ┃  激活难度最大
   (小游戏)  ┃  (复杂电商)
          ┃
         弱        容易←→难
              感知价值难度
```

**四个象限**：

| 象限 | 特征 | 例子 | 激活难度 |
|------|------|------|----------|
| 右上 | 需求强+容易上手 | 工具类（天气、PPT） | 最小 |
| 右下 | 需求弱+容易上手 | 小游戏 | 不大 |
| 左上 | 需求强+难上手 | SaaS类 | 较大 |
| 左下 | 需求弱+难上手 | 复杂功能电商 | 最大 |

#### 3.1.6 提升新用户激活的4种手段

**手段1：产品（基础，最便宜）**

- 贯穿整个过程
- 在动力、阻力、助推、奖励各方面都有关键作用
- 优化性价比高

**手段2：渠道（触达用户）**

| 类型 | 作用 |
|------|------|
| 短信 | 触达用户 |
| 推送 | 提醒继续完成关键行为 |
| 邮件 | 形成用户行为闭环 |
| 公众号 | 拉回产品继续使用 |

**手段3：激励（一次性强刺激）**

| 形式 | 作用 |
|------|------|
| 红包 | 刺激跨过决策障碍 |
| 优惠 | 完成激活行为 |
| 免费试用 | 首次体验 |

**手段4：人工（成本高，不普遍）**

| 类型 | 适用产品 |
|------|----------|
| 客户成功 | ToB产品 |
| 客服 | ToB产品 |
| 社群运营 | ToC产品 |

**金字塔成本排序**：产品 < 渠道 < 激励 < 人工

#### 3.1.7 激活手段选择矩阵

```
用户LTV
高
 ┃
人工   ┃  激励
       ┃
━━━━━━━╋━━━━━━━
       ┃
渠道   ┃  渠道/产品
       ┃
 ┃
 低   ┃      高
      需求差异化程度
```

**选择策略**：

| LTV | 差异化低 | 差异化高 |
|-----|----------|----------|
| **低** | 产品优化（小游戏）<br>简化上手流程 | 算法驱动（新闻/内容）<br>千人千面个性化推送 |
| **高** | 激励（电商）<br>新人红包 | 人工（ToB）<br>客户成功团队、客服 |

**注意**：使用高成本手段需用户LTV或客单价相应提升

#### 3.1.8 案例：股票投资App

**步骤1：评估激活难度**

- **动力**：较高（用户有投资意愿）
- **阻力**：很大（开户填大量信息、首次注资涉及钱）
- **结论**：激活难度较大

**步骤2：选择激活手段**

- **条件**：激活难度大+客单价/LTV高
- **手段**：可负担新人优惠、新人红包、客服电话支持

**步骤3：持续优化激活流程**

- **问题**：用户填敏感信息有安全顾虑
- **优化**：解释为什么需要、打消安全顾虑

### 3.2 新用户激活的最佳实践（中）

#### 3.2.1 提升新用户激活的四个思路

```
1. 增强动力 ──┐
2. 降低阻力 ──┼─> 综合运用帮助用户达到Aha时刻
3. 适时助推 ──┤
4. 及时奖励 ──┘
```

#### 3.2.2 思路1：增强动力

**方法1：利用社交关系/朋友信任背书**

**原理**：提升新用户信任感+提供个性化体验

**案例：微信读书**

- 登录后立即看到朋友在读的书
- 显示有多少朋友在读某本书

**效果**：
- 看到熟悉朋友名字→信任感提升
- 朋友喜欢读的书→与自己兴趣相似
- 一举两得

**方法2：新用户红包/激励**

**案例：网易严选**

| 类型 | 条件 | 目的 |
|------|------|------|
| 新人优惠券 | 绑定手机号 | 完成首单 |
| 额外前三单奖励 | - | 提升粘性和留存 |

**注意要点**：

1. **平衡付出与回报**：精确计算回报
2. **与核心业务相关**：好钢用在刀刃上
3. **案例**：每日优先新人优惠→代金券和会员卡

**方法3：解释为什么**

**原理**：简单解释好处，动力提升很多

**案例：脉脉**

- **目标**：让用户上传通讯录
- **好处**：提升职业成长机会
- **做法**：
  1. 告诉用户好处（提升职业成长机会）
  2. 完成后确认（"谢谢，已完成你的人脉报告"）

**方法4：虚拟Aha时刻**

**适用场景**：产品复杂，体验Aha时刻需要时间长

**案例：Airbnb房东注册**

**问题**：房东体验Aha时刻需要很长时间
- 注册完成→发布房源→有人入住→收到第一笔钱

**解决方案**：开发小工具
- 输入基本信息（位置、房间数）
- 计算大概能赚多少钱（如6000元/月）
- 模拟Aha时刻，前置到早期阶段

**方法5：心理学助攻**

| 心理技巧 | 案例 |
|----------|------|
| 紧迫感 | 微信读书："今天还有825张可免费领取"（倒数） |
| 稀缺感 | 同上 |
| 游戏化 | Honey浏览器插件：任务清单+金币奖励+进度条 |
| 从众效应 | 拼多多：信息流显示"XX地XX人发起拼单" |

#### 3.2.3 思路2：降低阻力

**方法1：移除不必要步骤和信息**

**原理**：多一步=多流失

**案例**：

| 产品 | 做法 |
|------|------|
| 通用 | 一键注册、社交登录 |
| 通用 | 简化合并注册流程步骤 |
| 今日头条 | 直接进入首页，无兴趣选择步骤 |
| Hopper | 推送开通可跳过 |

**今日头条案例分析**：

- **权衡**：更好的个性化vs流失用户
- **决策**：大体量产品，越简单越好
- **替代方案**：通过算法优化新闻选择

**方法2：避免用户冷启动**

**定义**：新用户进入后界面空白或无引导

**反面案例**：投资App定期投资页面空白

**优化**：设置默认选项（10元）
**效果**：投资率提升20%

**正面案例**：

| 产品 | 做法 |
|------|------|
| 任务管理工具 | 预设任务清单 |
| 各类产品 | 给用户默认选项和内容 |

**方法3：突出关键行为和路径**

**案例1：闲鱼**

- 开机后明确指出：第一步，卖点东西

**案例2：小年糕（中老年小程序）**

- 全屏大大的"做影集"按钮
- 针对目标用户（中老年）：不需要美工，需要明确指示

**方法4：不给用户太多选择**

**原理**：选择悖论——太多选择→一个都不选

**案例**：投资引导测试

| 选择数量 | 结果 |
|----------|------|
| 5个选择 | 最好 |
| 更多选择 | 行动比例下降 |

**方法5：先使用再注册/付费**

**原理**：先尝后买

**案例**：

| 产品 | 做法 |
|------|------|
| Netflix | 首月免费试用 |
| Wealthfront | 不开户也可获得免费投资计划 |

### 3.3 新用户激活的最佳实践（下）

#### 3.3.1 思路3：适时助推

**定义**：用户未按时完成关键动作时，在有限时间窗口内推一把

**考虑因素**：

| 因素 | 说明 |
|------|------|
| 用户决策速度 | 决定挽回用户的时间窗口 |
| 触达渠道 | 推送、短信、邮件、公众号等 |

**常见方法**：

**方法1：适时的产品内引导**

**案例：幕布**

- **形式**：开机引导页、关键功能引导、说明文档
- **特点**：适时引导（到某个功能/页面时才给相应引导）
- **优势**：不像一上来无休止介绍产品
- **避免**：新手引导太长成为障碍

**方法2：利用外部渠道推送**

**原理**：用户未及时激活→通过推送等形式拉回产品

**案例**：

| 产品 | 做法 |
|------|------|
| 今日头条 | 第1天未读够文章→第2天多媒体推送 |
| 微博 | 站内信息推送 |
| 通用 | 产品内弹窗提示关键动作 |

#### 3.3.2 思路4：及时奖励

**定义**：对完成行为的用户及时给予反馈和奖励

**考虑因素**：

| 因素 | 说明 |
|------|------|
| 行为难度 | 行为越困难，完成后越要奖励 |
| 关键程度 | 关键行为长期对留存重要→帮助形成习惯 |
| 流程长度 | 流程很长→中间给奖励重新充电 |

**注意**：奖励不一定需要金钱或实物

**方法1：撒花庆祝**

**案例：脉脉**

- **场景**：完成注册后
- **形式**：炫酷动画
- **内容**：挖掘人脉信息，告诉挖掘了多少人脉
- **效果**：惊喜，愿意继续使用

**案例：Mailchimp**

- **形象**：大猩猩
- **场景**：准备好第一封邮件
- **动作**：大猩猩伸出手High Five击掌
- **效果**：给用户惊喜时刻

**方法2：即时反馈**

**案例：小红书**

- **场景**：选择感兴趣内容后
- **形式**：动画显示"正在为你生成首页"
- **原理**：技术上不需要延迟
- **效果**：获得感、小红书在为你工作的感觉

**案例：Evernote**

- **场景**：完成任务后
- **形式**：划掉任务+进度条
- **原理**：UI设计让用户有即时反馈

**总结：小而有用的反馈**

- 虽小但让用户有获得感和成就感
- 通过UI设计实现
- 目标：即时反馈

### 4. 案例：摩拜单车如何提升新用户激活

#### 4.1 背景与目标

**产品**：摩拜单车
**Aha时刻**：新用户首次开锁骑车

**理论流程**：
1. 在大马路上看到摩拜单车
2. 需要从一个地方到另一个地方
3. 拿出手机扫描二维码
4. 开锁骑车

**现实情况（2017年）**：

- **获客方式**：线下扫码
- **必需步骤**：下载App
- **实际步骤**：8-9步才能开始骑车
- **问题**：每一步都有用户流失

#### 4.2 实际激活漏斗

```
扫码
    ↓
落地页
    ↓
应用商店
    ↓
下载（流失率高）
    ↓
打开App
    ↓
引导页
    ↓
手机注册
    ↓
身份证实名验证
    ↓
充值
    ↓
邀请好友页面
    ↓
开始骑车
```

**场景痛点**：

- 用户在大马路上，太阳很足
- 想赶紧骑上车到达目的地
- 流程太长，激活率不可能高

#### 4.3 优化策略

**核心问题**：降低阻力（用户阻碍重重）

**策略1：All in小程序**

**发现**：下载安装这一步对转化率影响很大

**原因分析**：
- 用户在大马路上，没有WiFi
- 下载App需要消耗流量
- 用户看到要下载就退出

**解决方案**：

- 加班加点开发摩拜单车小程序
- 成为微信第一批小程序接入

**效果**：

- 解决下载App问题
- 直接接入微信
- 身份验证、手机验证、支付更容易

**启示**：策略层面的选择——做小程序提升激活率

**策略2：提升扫码速度**

**发现**：扫码第一步流失率较高

**分析**：
- 在漏斗最开始
- 对整体新用户数量影响大

**优化**：提升扫描二维码速度

**效果**：所需时间减少6倍

**启示**：
- 考虑实际用户体验
- 页面下载速度、扫码速度、广告等都影响转化
- 不仅是产品设计流程

**策略3：优化手机验证后流程**

**发现**：手机验证后每步都有流失

**优化实验**：

| 改变 | 说明 |
|------|------|
| 砍掉引导页 | 减少步骤 |
| 4步流程→3步流程 | 简化流程 |
| 押金放身份证验证前 | 逻辑：交押金后对隐私顾虑降低 |

**效果**：显著提升新用户激活转化率

**启示**：流程优化需要实验验证

#### 4.4 案例总结

**摩拜单车的四步法**：

```
1. 明确激活目标：首次骑车
    ↓
2. 分析激活漏斗：找到线索（阻力大）
    ↓
3. 提出三个策略：降低阻力、减少步骤
    ↓
4. 实验验证：提升激活率
```

**核心思路**：

1. **数据分析**：从漏斗中找到问题
2. **用户场景**：考虑实际使用环境
3. **多维度优化**：
   - 策略层面（小程序）
   - 技术层面（扫码速度）
   - 流程层面（步骤优化）
4. **实验验证**：每个策略都需要验证

## 📖 本章核心要点总结

### 1. 新用户激活的重要性

- 位于用户生命周期起点，影响整个留存曲线
- 用户可塑性强，改善空间大
- 性价比高，值得投入足够资源

### 2. Aha时刻的本质

- 用简化的行为数据模拟用户首次获得价值的时刻
- 以让用户感到价值为目标，而非机械强迫
- 标准格式：谁在多长时间完成多少次什么行为

### 3. 找到Aha时刻的四步法

```
1. 提出备选行为（定性）
    ↓
2. 找到激活行为（定量，寻找相关性）
    ↓
3. 计算魔法数字（找到最优次数）
    ↓
4. 验证因果性（A/B测试）
```

### 4. 寻找激活线索的方法

**定量分析**：激活漏斗
- 找到流失率最高的步骤
- 区分支持行为和激活动作
- 通过分群分析找到具体问题

**定性分析**：激动指数
- 了解用户的情感体验
- 找到让用户兴奋/郁闷的元素
- 定期评估和优化

### 5. 提升激活的四个思路

1. **增强动力**：社交关系、激励、解释、虚拟Aha、心理学助攻
2. **降低阻力**：移除不必要步骤、避免冷启动、突出关键路径、限制选择、先尝后买
3. **适时助推**：产品内引导、外部渠道推送
4. **及时奖励**：撒花庆祝、即时反馈

### 6. 激活手段金字塔

```
    人工（成本高）
   /   \
  激励  渠道
   \   /
    产品（基础）
```

**选择依据**：用户LTV和需求差异化程度

### 7. 最佳实践

- 综合运用四个思路和四种手段
- 以用户实际场景为出发点
- 持续实验和迭代优化
- 定期审核新用户激活流程
