## 核心框架概览

### 用户留存四阶段模型

```
新用户激活 → 新用户留存 → 长期用户留存 → 流失用户召回
```

1. **新用户激活**：帮助用户首次感受产品价值（Aha时刻）
2. **新用户留存**：帮助用户养成使用习惯
3. **长期用户留存**：持续提供价值，加深用户参与
4. **流失用户召回**：重新激活已流失用户

### 提升留存的两大思路

**思路一：产品手段**
- 新用户引导优化
- 打造用户行为闭环
- 路径改进与优化
- 加深用户参与度（频次、强度、功能数、场景数）
- 个性化体验（千人千面）
- 开发新功能

**思路二：运营手段**
- 新用户红包/激励
- 精准拉新
- 精细化运营
- 用户激励体系
- 用户忠诚计划
- 用户召回激活

## 第一部分：衡量用户留存

### 1.1 四步法绘制留存曲线

#### 留存率的基本概念

**定义**：在最开始使用产品的用户中，经过一段时间后仍有多少百分比持续使用。

**计算公式**：

```
留存率 = （期末仍活跃的用户数 / 期初活跃用户数）× 100%
```

**示例**：
- 首日观看视频：300人
- 次日观看：100人 → 次日留存率 = 100/300 ≈ 33%
- 第7日观看：50人 → 7日留存率 = 50/300 ≈ 17%
- 第30日观看：20人 → 30日留存率 = 20/300 ≈ 7%

#### 流失率

**计算公式**：

```
流失率 = 1 - 留存率
```

**示例**：
- 次日流失率 = 1 - 33% = 67%
- 7日流失率 = 1 - 17% = 83%
- 30日流失率 = 1 - 7% = 93%

#### 留存曲线的定义

**核心定义**：首次完成初始行为的用户中，有多少百分比在下一个时间周期完成回访行为。

**四个关键概念**：

| 概念 | 说明 | 举例 |
|------|------|------|
| **初始行为** | 用户首次完成的关键行为，通常是激活行为 | 电商：首次下单；社交：首次互动 |
| **回访行为** | 希望用户返回产品时完成的行为 | 登录、下单、发帖等 |
| **时间周期** | 用户使用产品的时间间隔 | 日留存、周留存、月留存 |
| **百分比** | 回访用户占初始用户群体的比例 | 30%、50%等 |

#### 绘制留存曲线的四步法

**第一步：选择关键行为**

1. **确定初始行为**：用户完成什么行为才算激活
   - 只有激活的用户才谈得上留存
   - 例如：电商产品的首次下单

2. **确定回访行为**：希望用户返回时做什么
   - 多数情况下与初始行为相同
   - 特殊情况可能不同（如Jira的登录即可）

**第二步：选择时间周期**

根据产品特性确定天然使用周期：

| 产品类型 | 天然使用周期 | 留存指标 |
|----------|-------------|----------|
| 天气类 | 日 | 日留存 |
| 读书类 | 周 | 周留存 |
| 投资理财 | 月 | 月留存 |

**验证天然周期的方法**：

分析月活用户在一个月内的活跃天数分布：
- 计算活跃1天、2天、3天...的用户占月活的比例
- 占比最大的活跃天数代表主流天然使用周期

**第三步：收集数据，制作表格**

留存数据表示例：

| 首次激活时间 | 首次激活用户数 | 1周后留存 | 2周后留存 | 3周后留存 |
|--------------|----------------|-----------|-----------|-----------|
| 2017-01-01 | 50 | 43 (86%) | 36 (72%) | 30 (60%) |
| 2017-01-08 | 70 | 58 (83%) | 50 (71%) | - |

**第四步：绘制留存曲线**

将留存率数据表转换为曲线图，追踪不同时期开始的用户群的留存变化。

### 1.2 如何区分好留存、坏留存

#### 评估留存曲线的三种方法

**方法一：评估留存曲线的基本形态**

留存曲线有三种典型形态：

**1. 下滑型**
- 特征：留存率从100%开始一路下降，最终趋于0
- 两种情况：
  - **陡峭下滑**：留存快速归零，未达PMF，不宜拉新
  - **先陡后缓**：6-12个月后月流失仅1%左右，可边改善产品边拉新

**2. 趋平行型**
- 特征：留存曲线最终变平，达到PMF状态
- 评估要点：
  - 变平位置越高，长期留存率越好
  - 若在20%处变平，说明80%用户已流失

**3. 微笑型**
- 特征：留存曲线变平后还向上走，出现负流失
- 极其罕见，仅存在于特定产品：
  - 有网络效应的产品
  - 不断开发新功能的SaaS产品
  - 解决刚需问题的产品

**方法二：观察留存曲线的变化趋势**

**好的趋势**：
1. 新的留存曲线比旧的更平缓
2. 新的留存曲线整体向上提升

**异常检测**：
- 计算每个时间段的平均留存率
- 对比不同同期群，找出异常点
- 分析异常原因（活动、版本更新等）

**方法三：与行业均值比较**

**长期留存率参考值**：

| 产品类型 | 12个月留存率 |
|----------|--------------|
| 社交网络 | 约55% |
| 旅游类 | 约30% |

**游戏新用户留存率参考**：

| 游戏 | 次日留存 | 7日留存 | 30日留存 |
|------|----------|---------|----------|
| 皇室战争 | - | - | >30% |
| 部落冲突 | - | - | >30% |

**核心观点**：
> 亚马逊创始人Jeff Bezos："所有公司最终都会死掉，亚马逊有一天也会死掉。"

拉长时间维度，绝大多数产品的长期留存率最终趋于零。市场、竞争对手、用户都在不断变化，没有产品能真正永久留存所有用户。

## 第二部分：评估留存难度（选修）

### 2.1 留存的前提：建立产品长期价值

#### 为什么要评估留存难度？

**"低头赶路之前先抬头看天"**

在开始任何留存项目前，先评估：
1. 产品是否对用户有长期价值（是否达到PMF）
2. 产品的天然留存难易程度如何

#### 留存差的三种常见情况

**情况一：伪需求**

- 产品解决的问题对用户价值不大
- 例如：互联网理发（上门费高、理发师质量不确定、清理麻烦、安全隐患）
- 建议：遵循MVP方法，快速验证迭代，未达PMF就不要做

**情况二：新鲜感产品**

- 依靠新颖概念一夜刷屏，但缺乏长期价值
- 例如：脸萌、神经猫
- 建议：作为获客手段，后续要有长期价值承接
- 或者快速迭代，将用户迁移到有长期价值的功能上

**情况三：解决方案不到位**

- 问题有价值，但产品没解决好
- 例如：早期外卖/打车，司机/车辆不足
- 建议：通过内测、逐步开放城市来完善产品和服务能力

---

### 2.2 用户留存公式

#### 核心公式

```
留存用户 ≈ 感知价值 + 转换成本 > 使用难度
流失用户 ≈ 感知价值 + 转换成本 < 使用难度
```

#### 三个关键变量

**1. 感知价值**

用户感受到产品有用，这比产品本身有用更重要。

**示例**：微信读书
- 虽然书已免费（年卡）
- UI会显示原价（划掉）
- 提示已节省金额
- 目的：让用户感受到价值

**提升感知价值的策略**：
- 短期价值不明显 → 新用户上手引导，尽快达Aha时刻
- 解决方案不到位 → 持续产品改善
- 需求未满足 → 用户分群、个性化体验、精细化运营
- 缺乏新价值 → 开发新功能、持续提醒价值

**2. 转换成本**

用户抛弃产品的容易程度。

**提升转换成本的策略**：
- 让用户持续投入（时间、数据、金钱、人力）
- 例如：Pinterest（保存图片、制作清单）、豆瓣（积累记录）

**3. 使用难度**

广义的使用难度，包括：
- 产品是否难用
- 用户是否想不起来用
- 是否方便使用（手机/网页）

**降低使用难度的策略**：
- 保证产品体验好
- 帮助用户养成习惯
- 通过push、邮件等渠道定期召回

---

### 2.3 影响留存难度的产品特性

#### 产品本身决定的因素

**1. 需求强烈程度**

- **刚需产品**（止痛药）：订火车票、订餐
- **非刚需产品**（维他命）：汇率查询、小游戏

**2. 产品生命周期**

- **长生命周期**：理财产品、论坛（十几年）
- **短生命周期**：相亲产品（成功即离开）

**3. 使用频率**

- **高频产品**：看视频、看新闻（容易养成习惯）
- **低频产品**：买房、买车（很难养成习惯）

**4. 可替代品数量**

- **可替代品少**：滴滴、大型SaaS软件
- **可替代品多**：游戏、新闻（广义竞品多）

#### 评估留存难度的七个问题

| 问题 | 评估方向 |
|------|----------|
| 1. 用户需求有多强烈？ | 止痛药 vs 维他命 |
| 2. 产品功能有多复杂？ | 单功能 vs 多功能 |
| 3. 产品天然生命周期多长？ | 长周期 vs 短周期 |
| 4. 产品天然使用频次？ | 高频 vs 低频 |
| 5. 可替代品有多少？ | 多 vs 少 |
| 6. 用户转换成本高低？ | 高 vs 低 |
| 7. 产品变现能力如何？ | 强 vs 弱 |

#### 三个评估示例

**示例1：微信小游戏**

| 问题 | 评估 |
|------|------|
| 需求强烈程度 | 维他命型 |
| 功能复杂度 | 单功能 |
| 生命周期 | 短（几个月） |
| 使用频次 | 初期高频 |
| 可替代品 | 很多 |
| 转换成本 | 低 |
| 变现能力 | 中等（广告） |

**结论**：留存困难，应重点放在新用户激活和留存，快速变现，采用多游戏组合打法。

**示例2：企业级SaaS（钉钉）**

| 问题 | 评估 |
|------|------|
| 需求强烈程度 | 高 |
| 功能复杂度 | 高 |
| 生命周期 | 长 |
| 使用频次 | 高 |
| 可替代品 | 少 |
| 转换成本 | 高 |
| 变现能力 | 强 |

**结论**：留存不难，重点在高质量获客、新用户转化激活、客户成功。

**示例3：K12教育产品**

| 问题 | 评估 |
|------|------|
| 需求强烈程度 | 偏刚需 |
| 功能复杂度 | 中等偏多 |
| 生命周期 | 长 |
| 使用频次 | 高（至少每周） |
| 可替代品 | 中等 |
| 转换成本 | 中等偏低 |
| 变现能力 | 强 |

**结论**：留存难度中等，重点是新用户激活留存和长期留存，关键是保证课程效果，拓展品类。

## 第三部分：寻找提升留存的线索

### 3.1 寻找留存线索的两种思路

#### 思路一：找到留存的输入变量

**核心理念**：留存是结果，要影响结果，需要找到可控制的输入变量。

**减肥的例子**：
- 结果变量：体重
- 输入变量：饮食、运动、代谢率
- 策略：通过控制输入变量影响结果

#### 两类留存输入变量

**第一类：用户生命周期变量**

长期留存受其前后因素影响：

```
提升激活率 → 提升新用户留存率 → 提升长期用户留存率
降低流失率 + 增加召回率 → 提升长期用户留存率
```

**优先级排序**：

| 优先级 | 指标 | 影响力 | 难度 |
|--------|------|--------|------|
| 1 | 新用户激活率 | 大 | 低 |
| 2 | 新用户留存率 | 大 | 中 |
| 3 | 流失用户召回率 | 小 | 易 |
| 4 | 长期用户留存率 | 大 | 高（"messy middle"） |

**示例**：天气App
- 分析发现：新用户首周留存是最佳入手点
- 策略：改善新用户上手流程、优化推送
- 结果：首周留存从60%→80%，十周留存从18%→25%

**第二类：用户参与度变量**

**核心概念**：用户使用越多、越频繁、越深入，留存可能性越高。

**概念辨析**：

| 概念 | 定义 | 示例指标 |
|------|------|----------|
| **活跃** | 用户单次行为 | 登录一次 |
| **频次** | 行为频繁程度 | 每月活跃天数/次数 |
| **强度** | 行为强烈程度 | 使用时长、客单价 |
| **留存** | 行为持续性 | 持续一个月登录 |

**全民K歌的参与度分析示例**：

1. **确认产品适合参与度分析**
   - 高频次、高参与度产品（社交娱乐类）

2. **计算用户参与程度**

   | 用户类型 | 定义 | 占比 |
   |----------|------|------|
   | 高活用户 | 每月登录10天以上 | 10% |
   | 中活用户 | 每月登录4-10天 | 25% |
   | 低活用户 | 每月登录3天以下 | 65% |

3. **制定理想目标**
   - 天然周期：周（消遣需求）
   - 理想目标：每周至少使用1次，每次15分钟以上

4. **分析用户群组**
   - 57%女性，43%男性
   - 54%为24岁以下
   - 高活用户特征：学生、家庭用户、固定时间段使用

5. **寻找提升线索**
   - 发现"学生+家庭+晚上+长时间使用"用户群留存高
   - 策略：找到更多此类用户，开发专属功能

#### 思路二：对比不同用户群组的留存

**方法一：留存曲线分解**

根据不同用户属性或行为分解留存曲线：
- 渠道
- 性别、年龄
- 公司大小、行业
- 手机型号
- 是否使用某功能
- 用户行为特征

**小红书案例**：

**问题**：某渠道用户留存差
**分析过程**：
1. 第一层假设：不同渠道留存不同？
   - 分解：发现主要来自百度SEM
2. 第二层假设：18岁以下低龄用户细分？
   - 分解：高中生留存好，小学生初中生留存差
3. 第三层假设：周中周末区别？
   - 分解：无明显差异
4. 最终假设：内容匹配问题？
   - 验证：小学生搜"动漫壁纸"，小红书无此类内容
   - 结果：搜索词无内容，点击率低，导致留存差

**方法二：功能留存矩阵**

**定义**：对于多功能产品，计算不同功能的使用用户比例和留存率，绘制象限图。

**矩阵变量**：
- 横轴：功能留存率
- 纵轴：使用该功能的用户占活跃用户比例

**金融App示例**：

| 功能 | 3个月留存率 | 活跃用户占比 |
|------|------------|--------------|
| 储蓄 | 79% | 82% |
| 基金 | 85% | 35% |
| 社区 | 25% | 27% |
| 服务 | 30% | 65% |

**分析结论**：
1. **基金功能**：高留存率，低使用比例 → 重点提升
2. **储蓄功能**：双高 → 保持
3. **服务功能**：高使用，低留存 → 分析原因，提升留存
4. **社区功能**：双低 → 决定：改善或砍掉

**矩阵的两个变种**：

1. **新用户功能留存矩阵**
   - 纵轴：新用户某时间内使用功能比例
   - 横轴：使用后几个月留存率

2. **功能参与度矩阵**
   - 纵轴：使用功能的用户占比
   - 横轴：每月平均使用天数

### 3.2 功能留存矩阵（详细应用）

#### 功能留存矩阵的核心价值

**解决的问题**：
1. 多功能产品中，不同功能留存率差异大
2. 高留存功能使用用户比例低，如何提升
3. 高使用率功能留存率低，如何改善

#### 分析流程

**步骤1：列出主要功能**

**步骤2：计算矩阵变量**
- 使用该功能的用户数 / 总活跃用户数
- 功能留存率（如3个月留存率）

**步骤3：绘制象限图**

**步骤4：制定策略**

| 象限 | 特征 | 策略 |
|------|------|------|
| 高留存率高使用率 | 明星功能 | 保持优化 |
| 高留存率低使用率 | 潜力功能 | 提升使用率 |
| 低留存率高使用率 | 问题功能 | 提升留存率 |
| 低留存率低使用率 | 边缘功能 | 改善或砍掉 |

## 第四部分：提升新用户留存的实战方法

### 4.1 提升新用户留存的3种手段

#### 手段一：精准拉新

**基本思路**：不同渠道的新用户留存率不同，剔除或优化低质量渠道。

**实施步骤**：
1. 分渠道计算留存率
2. 分析低留存渠道原因
3. 尝试改善（投放定位、素材更新、产品承接）
4. 若无效，停止投放

**渠道留存分析表示例**：

| 渠道 | 装机量 | 首周激活率 | 次周留存率 |
|------|--------|------------|------------|
| 华为预装 | 大 | 低于平均 | 低于平均 |
| 应用宝 | 大 | 低于平均 | 低于平均 |
| iOS | 中 | 正常 | 正常 |
| OPPO | 小 | 正常 | 正常 |
| 小米 | 小 | 正常 | 正常 |

**决策**：华为预装、应用宝需改善或暂停投放。

#### 手段二：持续上手

**核心概念**：引导用户发现更多功能，达到更多Aha时刻。

**与Aha时刻的区别**：
- **Aha时刻**：聚焦核心动作，一次性完成
- **持续上手**：介绍更多功能，分步骤完成

**关键原则**：选择合适时机引导，而非一股脑推送所有功能。

**得到App示例**：
- 新用户下载后，快速引导介绍底部按钮功能
- 引导内容：汇总内容、学习方案、记笔记等
- 特点：轻量级引导，不强迫用户

#### 手段三：习惯养成

**适用范围**：适用于所有产品，尤其中高频产品（日活、周活、月活）。

**基本做法**：
1. 打造用户行为闭环
2. 利用推送、邮件等渠道
3. 结合产品设计
4. 让用户不断回访完成某功能

**两点注意事项**：
1. **适用全留存阶段**：不仅新用户，老用户也需要习惯养成
2. **理想时机是新用户留存期**：
   - 流失可能性高
   - 用户兴奋度高（新鲜感）

**为什么习惯养成如此重要？**

"仅仅让新用户体验一次Aha时刻远远不够。"

**示例**：
- 抖音：Aha时刻是第一天看5个视频
- 习惯养成：每天至少看1次视频
- 美团：Aha时刻是3天内完成首单
- 习惯养成：每周至少订3单

**健身房示例**：
- 去一次健身房感受好 ≠ 长期健身
- 持续一个月（21天理论）→ 更可能养成习惯

**新用户留存期的特点**：

| 特点 | 说明 |
|------|------|
| **流失概率高** | 留存曲线陡峭，"震荡期" |
| **用户兴奋度高** | 新用户登录频率远高于老用户 |

### 4.2 Hooked模型：养成使用习惯的理论基础

#### 什么是Hooked模型？

**定义**：通过四个步骤帮助用户形成使用习惯的产品逻辑框架。

**作者**：Nir Eyal，《Hooked》中文版《上瘾》

**核心理念**：设计良好的产品能让用户下意识使用，形成强大习惯。

#### Hooked模型的四个步骤

```
触发 → 行动 → 多变的酬赏 → 投入
```

#### 步骤一：触发（Trigger）

**作用**：给用户提示，让其采取行动。

**两类触发**：

**1. 外部触发**

| 类型 | 定义 | 示例 |
|------|------|------|
| **回馈型** | 给予好处 | 三节课优惠券、折扣券 |
| **付费型** | 付费渠道 | 地铁广告、信息流广告 |
| **社交型** | 社交关系 | 朋友圈推荐 |
| **自主型** | 产品自有渠道 | push、公众号、服务号 |

**2. 内部触发**

**最强大的触发机制**：用户有需求时，下意识使用产品。

**内部触发的情绪类型**：

| 情绪类型 | 示例 | 产品匹配 |
|----------|------|----------|
| 负面情绪 | 觉得落后、不如同事 | 三节课（学习进步） |
| 正面情绪 | 喜欢进步、自虐式学习 | 三节课（学习进步） |
| 无聊 | 想打发时间 | 抖音、小红书 |
| 孤单 | 想看朋友动态 | 微信、朋友圈 |

**终极目标**：
> 外部触发 → 行动 → 循环 → 内部触发直接关联产品

**示例**：
- 觉得落后 → 上三节课买课
- 觉得无聊 → 打开抖音
- 有疑问 → 打开Google

#### 步骤二：行动（Action）

**定义**：触发后希望用户完成的行为。

**核心原则**：
> 降低行为成本，让动作越小越简单越好

**降低行为成本的方法**：

1. **快速**：通过路径分析，去掉多余步骤
2. **清晰**：文案清晰，去掉认知负担

**经典案例**：

| 产品 | 简化动作 | 效果 |
|------|----------|------|
| Tinder | 左滑右滑 | 极易上瘾 |
| 抖音 | 上拉刷下一个 | 极易上瘾 |
| Twitter | 下拉刷新 | 极易上瘾 |

**核心概念**：微习惯
> 越微小的动作，越容易坚持，越容易形成习惯。

#### 步骤三：多变的酬赏（Variable Reward）

**定义**：对用户行为的反馈和奖励。

**关键要求**：每次都要不一样，持续刺激形成习惯。

**三种酬赏类型**：

| 类型 | 定义 | 示例 |
|------|------|------|
| **社交酬赏** | 人际关系中的奖励 | 小红书点赞、知乎答主感谢 |
| **猎物酬赏** | 具体信息、资源奖励 | 游戏人头数增加 |
| **自我酬赏** | 自我感觉良好 | 樊登读书会进步感 |

**为什么必须多变？**

**心理学原理**：固定奖励的边际效应递减很快，失去刺激性。

**真实案例**：
- 孩子：每周表现好 → 吃大餐/博物馆 → 吸引力下降
- 改进：每周表现好 → 不知名奖励 → 吸力恢复

**关键洞察**：
> 只有多变的酬赏才能持续激励用户完成动作。

#### 步骤四：投入（Investment）

**定义**：用户在产品中投入时间、精力、金钱、数据、感情。

**核心作用**：投入越多，越难离开产品。

**四种投入类型**：

| 类型 | 定义 | 示例 |
|------|------|------|
| **内容输出** | 产生内容 | 豆瓣影评、小红书评测 |
| **社交关系** | 建立关系 | 微信好友、微博关注 |
| **个人声望** | 建立声誉 | 得到学分、等级 |
| **行为习惯** | 学习成本 | 苹果用户换安卓困难 |

#### Hooked模型适用性评估

**适用条件**：

1. **频率足够高**：低频（一年一次）难以形成习惯
2. **可感知价值大**：每次使用都有价值感知
3. **行为足够简单**：复杂动作难以形成习惯
4. **天然多变奖励**：社交、游戏、内容类产品有优势

**特别适用**：
- 非刚需产品的"救命稻草"
- 把维他命变成止痛药
- 让用户"上瘾"后离不开

**应用思路**：
> 总归可以在产品中找到一些功能/行为，帮助用户形成使用习惯。

### 4.3 用户行为闭环：Hooked模型的实战应用

#### 什么是用户行为闭环？

**定义**：Hooked模型的简化应用版本，便于实际产品设计。

**核心组成**：

```
触发 → 行为 → 价值（奖励）
（底层：用户动机）
```

**触发渠道**：
- 推送、邮件、短信
- 公众号、微信群、朋友圈
- 产品内消息
- 付费广告重新触达

**今日头条案例**：

1. **触发**：算法找到新用户感兴趣内容，推送通知
2. **行为**：用户点击推送，打开今日头条
3. **价值**：看到感兴趣内容，体验好
4. **循环**：下次推送更愿意点击
5. **习惯**：形成习惯，每天打开

#### 培养用户行为的四步框架

**步骤1：确定理想频次**

**两种方法**：

1. **定性估算**：从解决问题出发估算
   - Keep：锻炼需每周3次才有效果 → 理想频次每周3次

2. **定量分析**：分析长期留存用户的使用频次
   - Keep：分析使用6个月以上用户的频次

**注意区别**：
- **天然使用周期**：大多数用户的频率（Keep：每周1次）
- **理想频次**：习惯用户应达到的频率（Keep：每周3次）

**步骤2：找到习惯用户**

**两个条件**：
1. 至少使用一段时间（6个月、9个月、1年）
2. 仍保持理想使用频次

**Keep示例**：
- 使用6个月以上
- 每周登录3次以上
- 这些是"铁粉"，已养成习惯

**步骤3：分析习惯用户行为**

**核心问题**：
1. 这些用户是谁？（几种类型？）
2. 有哪些相似行为？（高频周期性行为）
3. 完成行为的路径是怎样的？（几步？难易？）
4. 使用场景是怎样的？

**Keep示例**（数据虚拟）：

| 行为类型 | 比例 | 说明 |
|----------|------|------|
| 锻炼（课程） | 90% | 核心功能，高频行为 |
| 社区留言 | <10% | 部分用户习惯行为 |

**关键产出**：找到共有习惯行为，作为引导目标。

**步骤4：设计和优化行为闭环**

**两类优化**：

**类型A：优化现有闭环**

**Keep示例1**：优化锻炼行为闭环

| 路径 | 现状 | 问题 | 优化 |
|------|------|------|------|
| 路径1 | 打开Keep → 选课程 → 开始锻炼 | 无闭环 | 增加触发 |
| 路径2 | 收到提醒push → 打开Keep → 开始锻炼 | 功能入口不明显 | 引导设置提醒 |

**类型B：开发新闭环**

**Keep示例2**：开发社区留言闭环

| 现状 | 问题 | 解决方案 |
|------|------|----------|
| 打开Keep → 社区 → 浏览 → 留言 | 无闭环，自发行为 | 关注博主有新动态 → 推送 → 引导进入社区 |

**Keep示例3**：参考线下场景开发新闭环

**线下方法 → 产品功能**：
- 训练计划 → 课程表、每日运动目标
- 打卡记录 → 每日打卡功能
- 互相监督 → 社交监督功能

**核心思路**：
> 把线下已有方法数字化，开发成产品功能，制造行为闭环。

### 4.4 案例：喜马拉雅和领英如何提升新用户留存

#### 喜马拉雅案例分析

**四步走框架应用**：

**步骤1：确定理想频次**

- **核心行为**：听音频
- **主流频次**：每周至少1次
- **理想频次**：每周至少5次

**步骤2：找到习惯用户**

- **定义**：使用6个月，每周听5次以上

**步骤3：分析习惯用户行为**

**发现1**：所有习惯用户的共同行为都是听音频

**发现2**：两个高峰时段
| 时段 | 内容类型 | 用户画像 |
|------|----------|----------|
| 早7-9点 | 英语、新闻、音乐、脱口秀、相声、小说 | 通勤族（开车/公交） |
| 晚8-10点 | 儿童故事、儿童英语、亲子内容、心理情感 | 父母（给孩子听） |

**推断**：两类习惯用户群
1. 早上通勤族：从听广播转向喜马拉雅
2. 晚上父母族：给孩子讲故事/学习

**步骤4：设计和优化行为闭环**

**场景1：早上通勤族**

| 优化点 | 现状问题 | 解决方案 |
|--------|----------|----------|
| 触发 | - | 工作日早上推送新闻/英语/相声 |
| 路径优化 | 上车打开App，找内容困难 | 自动继续上次播放；置顶常听类型 |

**场景2：晚上父母族**

| 优化点 | 现状问题 | 解决方案 |
|--------|----------|----------|
| 触发 | - | 每晚8:30推送儿童故事 |
| 路径优化 | 孩子不爱听需换内容 | 推荐/收藏功能；学习计划（推送提醒） |

#### 领英案例分析

**核心理念**：
> 最成功的产品都在核心闭环基础上不断叠加新闭环。

**核心闭环：谁看了你的履历**

```
用户A查看用户B简历 → 发送邮件"有人查看你的履历"
→ 用户B好奇点击邮件 → 访问领英查看 → 查看A的履历
→ 可能查看他人履历 → 循环继续
```

**关键设计**：基于好奇心和社交需要，推动用户持续互动。

**叠加闭环1：技能背书（2012年）**

```
同事请求背书技能（如数据分析） → 发邮件邀请你
→ 你认可 → 在技能后显示你的头像
→ 互相认可 → 持续互动 → 养成习惯
```

**叠加闭环2：个人博客（2013年）**

```
你写博客 → 关注者收到通知
→ 阅读、点赞、留言、转发
→ 成就感高 → 写下一篇
→ 持续回到领英发布内容
```

**叠加闭环3：Kudos点赞（2018年）**

```
线下行为线上化：同事表现棒 → 口头夸奖
→ 领英邀请点赞 → 提供模板
→ 你点赞 → 同事收到通知 → 回应
→ 持续互动 → 养成习惯
```

**核心思路总结**：
1. 核心闭环基础上不断叠加新闭环
2. 把线下行为搬到线上
3. 让平台上已有行为加一把油
4. 最终让用户习惯难以改变

## 第五部分：提升长期用户留存的实战方法

### 5.1 加深用户参与：提升长期留存的核心手段

#### 留存与参与的区别

**用户与产品的两种关系**：

| 关系 | 描述 | 问题 |
|------|------|------|
| **留存** | 用户是否还在用产品 | 今天用、明天用、下个月还用吗？ |
| **参与** | 用户怎么用产品 | 每天用1次还是3次？用1个功能还是多个？ |

**核心逻辑**：
> 提升参与度 → 提升长期留存率

**用户参与的定义**：
- 使用场景（通勤 vs 旅行）
- 使用功能（快车 vs 专车）
- 使用频次（每天2次 vs 每月1次）
- 使用强度（每次30元 vs 每次100元）

#### 三类产品的用户参与

**1. 时间类产品**（消磨时间）

| 示例 | 使用场景 |
|------|----------|
| 爱奇艺 | 周末放松；旅行看片 |

**2. 效率类产品**（提升效率）

| 示例 | 使用场景 |
|------|----------|
| 钉钉 | 企业群沟通；审批流程 |

**3. 交易类产品**（完成交易）

| 示例 | 使用场景 |
|------|----------|
| 滴滴 | 通勤（快车）；接送机（专车） |

**核心洞察**：
> 同一产品，不同用户使用方式不同；同一用户，不同场景使用方式不同。

#### 加深用户参与的四种方法

```
提升频次 → 提升强度 → 提升功能使用 → 增加使用场景
```

---

### 5.2 方法一：提升频次

#### 不同产品的留存挑战

| 产品类型 | 挑战 | 策略重点 |
|----------|------|----------|
| **高频产品** | 未养成习惯，转向竞品 | 提升核心功能频次；巩固习惯 |
| **低频产品** | 间隔太长，忘记产品 | 开发高频功能 |

#### 高频产品策略

**核心思路**：提升核心功能使用频次，巩固习惯。

**案例1：Snapchat - Streaks（联机）**

| 功能 | 设计 | 效果 |
|------|------|------|
| 联机数 | 好友连续聊天，数字增长 | 年轻人最爱，保持联机不断 |

**案例2：趣头条 - 签到得金币**

| 功能 | 设计 | 效果 |
|------|------|------|
| 签到 | 阅读得金币，签到额外奖励 | 提升核心功能（阅读+奖励）频次 |

**案例3：瑞幸 - 每周满七件**

| 功能 | 设计 | 效果 |
|------|------|------|
| 满七件 | 每周买7杯瓜分500万 | 培养每周多次购买习惯 |

#### 低频产品策略

**核心思路**：在产品价值链上打造高频功能。

**案例1：贝壳找房**

| 核心功能 | 频率 | 高频功能 | 效果 |
|----------|------|----------|------|
| 买房 | 几年1次 | 租房、装修 | 提频 |
| - | - | 贝壳指数（房产资讯） | 提频 |
| - | - | 百科（投资介绍） | 提频 |

**案例2：安居客 - 房产估值**

| 功能 | 设计 | 效果 |
|------|------|------|
| 房产估值 | 告知关注房源最新价格 | 用户高频查看，保持联系 |

**案例3：Airbnb - 心愿单**

| 功能 | 设计 | 效果 |
|------|------|------|
| 心愿单 | 安排未来旅程，浏览攻略 | 从几个月1次→几周1次 |

**案例4：春雨医生 - 计步功能**

| 功能 | 设计 | 效果 |
|------|------|------|
| 计步 | 每天记录步数 | 从几个月1次（问诊）→每天1次 |

#### 游戏化提升频次

**趋势**：所有App都在养鸡、种树、种水果。

| 产品 | 游戏化功能 | 与业务联系 |
|------|------------|------------|
| 陌陌 | AR小宠 | 建立感情联系 |
| 支付宝 | 蚂蚁庄园 | 公益相关 |
| 拼多多 | 多多果园 | 农贸产品直接相关 |
| 京东 | 京豆 | 购物相关 |

**设计原则**：
> 游戏化机制要与产品特点结合，才能有长期效果和壁垒。

### 5.3 方法二：增加强度

#### 不同产品的强度定义

| 产品类型 | 强度定义 | 提升策略 |
|----------|----------|----------|
| **时间类** | 单次使用时长 | 自动播放下一集、无限滚动 |
| **效率类** | 单次操作量 | 更多文件操作、更多tickets更新 |
| **交易类** | 单次交易金额 | 搭配购买、满减包邮、返程车票 |

#### 时间类产品提升时长

**案例1：Netflix - 自动播放**

| 设计 | 效果 |
|------|------|
| 一集结束，5秒后自动播放下一集 | "沙发土豆"效应，持续观看 |

**案例2：今日头条 - 无限滚动**

| 设计 | 效果 |
|------|------|
| 下拉刷新新内容 | 持续浏览，时长增加 |

**案例3：游戏 - 无限关卡**

| 设计 | 效果 |
|------|------|
| 关卡无限延伸 | 持续游玩，时长增加 |

#### 交易类产品提升客单价

**案例1：搭配购买**

| 设计 | 效果 |
|------|------|
| "买了这个的人也买了那个" | 提升单次订单金额 |

**案例2：免邮门槛**

| 设计 | 效果 |
|------|------|
| 满99元包邮 | 鼓励多买，达到门槛 |

#### 产品类型融合趋势

**核心观察**：
> 所有产品都在增加"消磨时间"功能，因为抓住时间才能长期留存。

**案例1：QQ浏览器**

| 新功能 | 效果 |
|--------|------|
| 加入电子书功能 | 留存提升明显 |

**案例2：Keep**

| 策略 | 效果 |
|------|------|

**案例3：淘宝**

|--------|------|
| 加入直播功能 | 用户在淘宝花更多时间 |

#### 反思：时长越长越好吗？

**工具效率类产品特点**：

| 特点 | 说明 |
|------|------|
| 用户有明确目标 | 省时省力完成任务 |
| 不是让用户花更多时间 | 一味黏住体验不好 |

**反噬案例：Slack**

|------|------|----------|

**防沉迷机制**：

|------|------|------|
| Instagram | "你全看完了"提示 | 告知可停止浏览 |
| 苹果/安卓 | 屏幕使用时间统计 | 帮助用户自我控制 |

**核心反思**：
> 一味让用户上瘾，长期来看不是好结果，可能反噬产品。

### 5.4 方法三：提升功能使用

#### 提升功能使用的三个思路

**思路1：确保用户正确使用核心功能**

**问题示例**（增长黑客网案例）：

| 错误用法 | 正确用法 |
|----------|----------|
| 当便签储存想法 | 团队协作管理实验全流程 |

**策略**：引导用户正确使用核心功能。

**思路2：让用户使用更多重要功能**

**三个方向**：

| 方向 | 说明 |
|------|------|
| 重要功能 | 找到对留存有帮助的功能 |
| 多功能 | 不只用1个功能，用多个功能 |
| 高价值功能 | 向高留存率功能迁移 |

**思路3：开发新功能**

| 开发原因 | 示例 |
|----------|------|
| 产品偏低频 | 开发高频功能 |
| 强度不够 | 开发高强度功能（如QQ浏览器加电子书） |
| 新场景 | 淘宝加二手交易 |
| 新问题 | 解决新问题 |
| 新人群 | 服务新人群 |

#### "得到"的功能使用提升路径

**步骤1：正确使用核心功能**

| 核心功能 | 目标 | 方法 |
|----------|------|------|
| 听音频 | 首次收听（Aha） | 新手引导 |
| - | 养成每周听习惯 | 推送、学习计划 |

**步骤2：使用更多重要功能**

| 用户类型 | 功能使用 | 留存可能性 |
|----------|----------|------------|
| 用户一 | 只听音频课 | 一般 |
| 用户二 | 音频课+听书+电子书+留言讨论 | 高 |

**步骤3：开发新功能**

| 时间 | 功能 | 原因 |
|------|------|------|
| 早期 | 仅音频课 | - |
| 中期 | 听书 | 优质讲师有限，经典书籍可转化 |
| 后期 | 讲座（小课） | 满足价格敏感+具体问题需求 |

#### 如何提升功能使用率

**核心问题**：
1. 怎么找到哪个功能最重要？
2. 怎么让更多用户用这个功能？

**回答1：找到重要功能 - 倒推法**

| 方法 | 说明 |
|------|------|
| 数据分析 | 使用后长期留存率高的功能 |
| 时间范围 | 看第一年留存（比Aha时刻更长） |

**回答2：让更多用户使用功能 - 三种情况**

| 情况 | 问题 | 解决方案 |
|------|------|----------|
| 1. 性能不好 | 功能体验差 | 根据反馈改善性能 |
| 2. 入口不明显 | 用户不知道功能 | 增加入口、优化转化路径 |
| 3. 不知道怎么用 | 功能难上手 | 恰当时机引导教育 |

**三种情况案例**：

**情况1：支付宝刷脸支付**

| 问题 | 解决方案 | 效果 |
|------|----------|------|
| 刷脸太丑，用户diss | 一周后上线美颜刷脸 | 女性增长123%，男性增长106% |

**情况2：谷歌搜索体育赛事**

| 问题 | 分析 | 解决方案 |
|------|------|----------|
| 实时分数功能使用者少 | 70%用户不知道存在 | 1. 搜索时自动显示体育建议<br>2. 搜索栏下方显示可点击品类 |

**情况3：谷歌地图"沿路找饭馆"**

| 版本 | 设计 | 效果 |
|------|------|------|
| 版本1 | 每次打开弹窗介绍 | 不好 |
| 版本2 | 输入目的地后，旅程长+饭点时弹窗 | 好 |

**核心原则**：
> 恰当时机介绍功能，而非一股脑推送所有功能。

#### 数据分析提升功能使用

**分析流程**：

1. 找到重要功能（使用后留存提升）
2. 列出现有功能入口
3. 计算每个入口的用户数和转化率
4. 确定哪个入口改善空间最大
5. 确定优化策略

**优化策略**：

| 策略 | 说明 |
|------|------|
| 策略1 | 优化现有路径（更明显、提升转化率） |
| 策略2 | 增加新入口（MVP构建+优化） |

**Headspace案例：提升"每日提醒"功能**

**步骤1：找到重要功能**

| 发现 | 说明 |
|------|------|
| 功能使用率 | 仅5%月活使用每日提醒 |
| 留存率 | 6个月留存率远高于平均 |
| 结论 | 高价值功能，需提升使用率 |

**步骤2：梳理入口**

| 入口 | 日用户 | 转化率 | 成功设置数 |
|------|--------|--------|------------|
| 设置入口 | 10万 | 30% | 3万 |
| 新手引导 | 50万 | 10% | 5万 |

**步骤3：确定策略**

| 策略 | 方向 |
|------|------|
| 策略1 | 优化新手引导转化率（从10%提升） |
| 策略2 | 增加新入口（如每月3次冥想后弹窗） |

**结果**：通过A/B测试优化，留存率提升到原来的3倍以上。

**核心启示**：
> 很多时候优化已有功能比开发新功能性价比更高。

### 5.5 方法四：增加使用场景

#### 增加使用场景的两种思路

**思路1：让用户在多个场景下使用产品**

**场景含义**：
1. 泛指场景：解决不同问题（亚马逊：买东西 vs 卖二手）
2. 情景场景：不同情景解决同样问题（喜马拉雅：上班听新闻 vs 睡前听音乐）

**思路2：让用户使用多个平台**

**平台类型**：App、PC网页、桌面端、微信生态、线下店

**核心优势**：
- 增加使用场景
- 建立多个触达渠道

**樊登读书会案例**：

| 平台 | 触达方式 |
|------|----------|
| App | 听音频 |
| 公众号 | 文章召回 |
| 线下书店 | 完成消费、品牌提醒 |

**微博案例**：

| 平台 | 作用 |
|------|------|
| App | 完整体验 |
| 网页端 | 补充场景 |
| 微博热点小程序 | 微信体系触达召回 |

#### Lyft案例：多场景打车

**发现**：用户有三种使用场景

| 场景 | 频率 | 功能 |
|------|------|------|
| 接机 | 偶尔 | 专车 |
| 代驾 | 偶尔 | 代驾 |
| 通勤 | 每周 | 快车 |

**洞察**：2种以上场景使用用户留存率 > 单一场景用户

**Aha时刻定义**：7天内使用2个以上场景打车

**策略**：有意识引导用户尝试更多场景。

#### MatchU案例：多平台电商

**背景**：完全基于小程序的电商

**策略1：引导用户使用多平台，建立触点**

| 触点 | 交互 |
|------|------|
| 服务号 | 下单后引导关注 |
| 个人号 | 1对1触达 |
| 订阅号 | 内容触达 |
| 多个小程序 | 互相跳转 |

**策略2：优化触点路径**

**实验：优化"下单后关注公众号"**

| 版本 | 设计 | 结果 |
|------|------|------|
| 原版 | 文字多，步骤复杂 | 基准 |
| 版本1 | 文字精简，突出好处，路径缩短 | 留存+100% |
| 版本2 | 一键确认，回复9关注查看订单 | 最佳 |

**结果**：留存率提升230%

### 5.6 提升长期用户留存的其他手段

#### 个性化体验

#### 为什么需要个性化？

**产品复杂度演进**：

| 阶段 | 特征 | 示例 |
|------|------|------|
| 简单产品 | 用户少、需求单一、功能简单 | 天气、美颜、早期小红书 |
| 复杂产品 | 用户多、需求多样、功能复杂 | 钉钉、抖音、美团 |

**核心逻辑**：
> 用户越多样、产品越复杂，越需要个性化体验来留存。

#### 个性化体验的三个案例

**案例1：电商平台 - 分区运营**

**问题**：价格敏感型用户路径
```
登录 → 查看优惠券 → 没有优惠券 → 离开
```

**A/B测试优化**：

| 版本 | 路径 | 效果 |
|------|------|------|
| 优化前 | 显示无优惠券 → 离开 | 基准 |
| 优化后 | 提示答题领红包 → 答题 → 拿券 → 下单 | 下单率和留存提升 |

**案例2：得到 - 学分激励体系**

**五个维度**：

| 维度 | 说明 | 服务目标 |
|------|------|----------|
| 持续性 | 学习天数 | 留存、时长 |
| 学习量 | 学习数量 | 频次、强度 |
| 笔记 | 产生内容 | UGC、投入 |
| 知识分享 | 分传播 | 老带新 |
| 好奇心 | 尝试不同领域 | 功能数、场景数 |

**设计亮点**：

| 亮点 | 说明 |
|------|------|
| 目标结合 | 用户成长目标与业务目标有机结合 |
| 形式合理 | 分数+等级，前快后慢（符合Hooked模型） |
| 全面覆盖 | 覆盖留存各机制（频次、强度、功能、闭环） |

**案例3：房产O2O平台 - 精细化运营**

**步骤1：探索性数据分析，用户分群**

| 分群维度 | 具体内容 |
|----------|----------|
| 基础属性 | 性别、年龄、家庭、职业、收入、消费、手机 |
| 需求因素 | 购房阶段、目标、预算、户型 |
| 行为因素 | 活跃度、使用深度与广度 |
| 转化阶段 | 浏览到成交的步骤 |

**步骤2：细分流失用户**

| 发现 | 洞察 |
|------|------|
| 高收入+投资型用户流失率高 | 平台主要服务中等收入刚需/改善型 |

**步骤3：分析原因**

| 问题 | 原因 |
|------|------|
| 投资型用户流失 | 投资房源少、无投资分析资讯、无投资咨询 |

**步骤4：形成策略**

| 策略 | 具体措施 |
|------|----------|
| 增加投资资讯 | 楼盘升值分析 |
| 标签化 | 高投资潜力楼盘标签 |
| 线下服务 | 增加投资咨询 |

#### 避免流失的四种方式

**方式1：储存用户数据**

**原理**：用户数据越多，越难离开。

**案例**：豆瓣
- 读过的书、看过的电影、关注的人
- 成为用户的一部分，难以离开

**方式2：提高转换成本**

| 类型 | 案例 |
|------|------|
| 技术整合 | SaaS产品的SDK整合 |
| 忠诚计划 | 航空里程、酒店积分 |
| 年度付费 | 7年5折、年付优惠 |

**方式3：提前锁定用户**

**案例**：印象笔记促销
- 7年高级账号5折
- 提前锁定，降低流失

**方式4：流失预警机制**

**案例**：阴阳师游戏
- 一段时间未使用
- 再次登录时欢迎回归+赠礼
- 后台流失预警在起作用

**方向2：运营手段**

### 用户留存四个思维框架

**框架1：用户留存四阶段**

```
新用户激活 → 新用户留存 → 长期留存 → 流失用户召回
```

**核心洞察**：所有阶段都可影响留存，都是重要组成部分。

**框架2：Aha时刻**

> "不激活，无留存"

用户从未感受产品价值 → 不是真正的用户，只是访客。

**框架3：Hooked模型与行为闭环**

- 持续完成关键行为
- 多变奖励提供动力
- 形成习惯性用户

**框架4：用户参与度**

| 维度 | 说明 |
|------|------|
| 频次 | 使用频率 |
| 强度 | 使用深度 |
| 功能数 | 使用功能数量 |
| 场景数 | 使用场景数量 |

### 七个留存分析方法

| 方法 | 说明 |

#### 时间类产品

|------|----------|

#### 效率类产品

| 示例 | 留存策略 |

#### 工具类产品

| 示例 | 留存策略 |
|------|----------|

### 按产品特点选择策略

| 产品特点 | 分析方法 | 留存策略 |
|----------|----------|----------|
| 用户多样性高 | 分群、留存曲线分组 | 精细化运营、个性化体验 |
| 多功能产品 | 功能留存矩阵 | 向高价值功能迁移 |
| 单功能简单产品 | 频次分析、强度分析 | 围绕核心功能打造行为闭环 |
| 竞品可替代品多 | 频次分析、强度分析 | 建立习惯、增加投入、提高转换成本 |
| 成熟期大体量 | - | 开发新功能、新使用场景 |

### 按用户阶段选择策略

| 阶段 | 用户问题 | 留存策略 |
|------|----------|----------|
| 新用户激活 | 为什么要试试？ | 快速达Aha时刻；性价比最高 |
| 新用户留存 | 什么时候用？ | 形成习惯，找到固定位置 |
| 长期留存 | 为什么不换别的？ | 持续提供价值；复杂，需各部门配合 |
| 流失召回 | 为啥给第二次机会？ | 衡量机会成本和长期效果 |

### 核心观点总结

> **"Growth is good, retention is forever."**
> 增长虽然性感炫酷，但留存才是永恒不变的话题。

**两个关键维度**：
1. **产品类型**：不同产品类型选择不同策略
2. **用户阶段**：不同用户阶段选择不同策略

**综合应用**：
- 不限于一种方法
- 结合产品特点
- 结合用户阶段
- 最大化产品留存

**终极目标**：
> 虽然不能永久留住用户，但要把他们留得越久越好。
> "如果非要把这份爱加上一个期限，我希望是一万年。"

## 核心概念词典

| 概念 | 定义 |
|------|------|
| **留存率** | 期初用户在期末仍活跃的百分比 |
| **流失率** | 1 - 留存率 |
| **留存曲线** | 多个同期群留存率的曲线图 |
| **Aha时刻** | 用户首次感受产品价值的关键行为 |
| **Hooked模型** | 触发→行动→多变酬赏→投入的习惯养成模型 |
| **行为闭环** | Hooked模型的简化应用版本 |
| **用户参与度** | 用户使用产品的频次、强度、功能数、场景数 |
| **功能留存矩阵** | 多功能产品中不同功能的留存率和使用比例分析 |
| **PMF** | Product-Market Fit，产品市场契合 |
| **魔法数字** | 达到Aha时刻需要完成的关键行为次数 |
| **激活用户** | 完成首次关键行为，感受产品价值的用户 |
| **同期群** | 同一时期开始使用产品的用户群 |

## 本章小结

用户留存是增长黑客工作中最复杂、最具挑战性的环节。通过本章学习，你应该掌握：

1. **衡量留存**：绘制留存曲线，区分好留存和坏留存
2. **评估难度**：判断产品特性对留存的影响
3. **寻找线索**：通过输入变量、用户分组、功能矩阵找到提升点
4. **新用户留存**：精准拉新、持续上手、习惯养成
5. **长期留存**：加深参与、个性化体验、避免流失
6. **策略选择**：根据产品类型和用户阶段选择合适策略

**记住**：留存是一项系统工程，需要产品、运营、数据、研发等多个团队协同配合。只有真正理解用户需求，持续提供价值，才能实现用户的长期留存。
