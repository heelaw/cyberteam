# 第四章：工作规划与目标管理

> 所属章节：模块四
> 核心内容：三大模型与目标制定、目标拆解方法、工作规划实例、进度管理

## 本章导学

### 核心目标

本章帮助学员掌握在局部战场做好工作规划的完整方法论。核心目标是理解业务模型与目标制定的关系，掌握合理制定目标的方法，学会将目标拆解为可执行的计划，并能应对各种复杂情况下的工作规划。

### 学习重点

- 理解商业模式、业务模型、商业经营模型如何影响目标制定
- 掌握自上而下目标拆解的完整逻辑
- 学会处理模糊目标和紧迫资源约束
- 掌握业务团队与职能团队的目标对齐方法
- 了解不同类型目标的进度管理方式

## 4.1 工作规划与三大模型的关系

### 4.1.1 工作规划的层次结构

身处局部战场做好工作规划的基础逻辑包含三个部分：

```
┌─────────────────────────────────────────────────────────────┐
│                    工作规划的基础逻辑                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 明确目标                                                 │
│     └─ 目标从哪里来？                                        │
│         └─ 向上对齐公司战略和业务模型                         │
│                                                             │
│  2. 拆解目标                                                 │
│     └─ 落实为具体的工作计划                                   │
│         └─ 可执行、可衡量、可验收                             │
│                                                             │
│  3. 进度管理                                                 │
│     └─ 持续跟踪和调整                                        │
│         └─ 确保目标达成                                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 4.1.2 三大模型如何决定目标制定

#### 模型一：商业模式决定基础收入公式

**核心逻辑**：每一种商业模式背后都能推导出一家公司基本的收入公式。

**四种商业模式及其收入公式**：

1. **流量广告收入模式**
   ```
   收入 = 广告流量 × 转化率 × 单次点击价格
   ```
   - 目标用户付费能力越高 → 单次点击价格越高
   - 覆盖场景转化概率越大 → 单次点击价格越高

2. **平台交易收成模式**
   ```
   收入 = 交易笔数 × 单笔交易金额 × 收成比例
   ```

3. **产销收入模式**
   ```
   收入 = 流量 × 转化率 × 客单价
   ```

4. **流量+增值服务模式**
   ```
   收入 = 流量 × 付费转化率 × 会员单价
   或
   收入 = 流量 × 付费转化率 × 用户平均LTV
   ```

#### 模型二：业务模型让收入公式更加精细

**核心逻辑**：当业务模型梳理清楚后，公司的收入公式会被拆解得更加精细，能够指导到具体的团队和工作。

**案例：某电商公司的收入公式精细化**

基于业务模型的梳理：
```
总收入 = (渠道A流量 × 转化率 × 客单价)
       + (渠道B流量 × 转化率 × 客单价)
       + (渠道C流量 × 转化率 × 客单价)
       + (渠道D流量 × 转化率 × 客单价)
```

进一步细分：
```
总收入 = (爆款产品1收入 + 利润产品1收入 + 利润产品2收入 + ...)
       = (渠道A到爆款1的转化率 × 爆款1客单价)
       + (渠道B到爆款1的转化率 × 爆款1客单价)
       + ...
```

**业务模型精细化对组织结构的影响**：

当收入公式被拆解得足够精细后，公司的组织结构和职能划分就会自然形成：

| 业务模块 | 负责团队 | 核心指标 |
|----------|----------|----------|
| 流量获取 | 渠道运营团队 | 各渠道流量数量 |
| 爆款产品运营 | 爆款运营团队 | 爆款产品转化率 |
| 利润产品转化 | 转化运营团队 | 利润产品转化率 |
| 用户运营 | 会员运营团队 | 用户活跃率、复购率 |

**案例：某跨境电商网站的指标体系**

```
GMV = 新用户带来的GMV + 老用户带来的GMV

新用户GMV = 新用户数量 × 付费转化率 × 客单价
           = (日均访问数 × 注册转化率) × 付费转化率 × 客单价

老用户GMV = 已购用户数 × 月均活跃率 × 月均购买转化率 × 客单价
```

基于这个公式，团队分工清晰：
- **增长团队**负责：日均访问数、注册转化率
- **运营团队**负责：付费转化率、月均购买转化率
- **品类运营团队**负责：客单价
- **会员团队**负责：月均活跃率

#### 模型三：商业经营模型决定团队定位

**核心逻辑**：从钱和组织的角度看业务，将团队分为成本中心和利润中心。

**成本中心 vs 利润中心**：

| 类型 | 特点 | 管理重点 | 指标类型 |
|------|------|----------|----------|
| **成本中心** | 不直接面向市场，不产生收入 | 确保系统稳定运转、提升效率 | 效率导向指标 |
| **利润中心** | 直接面对市场，能获得收入 | 创造更多收入 | 业务导向指标 |

**业务导向指标 vs 效率导向指标**：

- **业务导向指标**：从业务模型中所处位置推导出来
  - 例如：转化率、留存率、复购率等

- **效率导向指标**：从成本中心/利润中心定位推导出来
  - 例如：人效、单位成本、响应时间等

### 4.1.3 三大模型如何指导目标制定 - 实战案例

#### 案例一：三节课B端业务的目标制定

**业务模型**：
```
销售线索 → 社群体验课 → 训练营项目
           ↓               ↓
         培训官社群    生产交付模型
```

**收入公式**：
```
收入 = 销售线索 × 转化率 × 客单价
```

**成本收益公式**：
```
利润 = 销售额 - 销售成本 - 交付成本 - 其他成本
```

**年度目标**：增长3倍，保持盈利

**目标拆解思路**：

1. **销售/增长团队**：如何实现3倍增长？
   - 提升线索量？还是提升转化率？还是提升客单价？
   - 哪个部分更有提升空间？

2. **交付团队**：如何保持盈利？
   - 客单价能否提升？（从1万到5万）
   - 交付成本必须控制在客单价的X%以内（死线指标）
   - 如何提升交付效率？（一个人服务2家→5家）

#### 案例二：某K12教育公司的目标制定

**业务模型**：
- **增长获客**：两条路径（电销、社群转化）
- **生产交付**：外教供应链 + 教学服务 + 班主任/Partner

**收入公式**：
```
收入 = ((渠道1流量 × 转化率) + (渠道2流量 × 转化率)) × 客单价 + 续费收入
```

**年度目标**：
- ROI提升15%
- 收入提升50%

**目标拆解思路**：

1. **增长团队**：ROI提升15%
   - 降低获客成本？
   - 提高客单价？
   - 提升转化率？

2. **供给/交付团队**：
   - 如何扩大外教数量？
   - 如何提升供应链响应效率？
   - 如何在保障教学质量下提升外教服务学员数量？
   - 如何提高班主任服务效率？

#### 案例三：某电商公司的目标制定

**业务模型**：
- 线上部分：用户池 + 流量转化路径
- 线下部分：门店 + 线上线下导流
- 交付：供应链管理体系 + 系统支持

**收入公式**：
```
线上收入 = 流量 × 转化率 × ARPU
```

**成本收益公式**：
```
利润 = 销售额 - 销售成本 - 生产成本 - 供应链成本 - 线下门店成本 - 其他成本
```

**年度目标**：线上业务保持三倍增长

**目标拆解思路**：

1. **增长团队**：
   - 扩大线上渠道？
   - 加强线下门店导流？
   - 提高用户转化率？
   - 提高用户复购率？

2. **供应链/保障团队**：
   - 如何提升供应链响应效率？
   - 如何降低仓储运输成本？
   - 如何做好选品？
   - 如何做好线上线下数据打通？（最终指向个性化推荐，提升转化率）

### 4.1.4 操盘手的工作规划心态

#### AB类操盘手应具备的心态

**在战略方向的思考讨论上**：
- 给高层提供有效信息
- 在全局业务模型的梳理、判断上能够参与讨论

**在局部业务的目标设定上**：
- 上层信息明确 → 需要在局部达成具体目标/提升具体指标
- 上层给了一个方向 → 需要在局部验证出来一个答案/探索出一个可依赖的模型

**核心原则**：
- 所有工作都要回归到模型上来思考
- 业务模型梳理清楚了，工作规划才有方向

## 4.2 如何合理制定局部业务目标

### 4.2.1 目标制定的核心原则

制定一个好的目标需要确保三个方面：

1. **与公司战略需求和核心业务模型一致**
   - 能支持战略目标达成，不跑偏

2. **清晰、可衡量、可验收**
   - 目标不能模糊

3. **与相关协作部门对齐**
   - 不打架、有共识

### 4.2.2 自上而下目标拆解的基本逻辑

#### 操盘手视角的目标管理框架

```
┌─────────────────────────────────────────────────────────────┐
│                    操盘手的目标管理框架                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  公司战略目标                                                 │
│     ↓                                                       │
│  核心业务指标/财务指标                                        │
│     ↓                                                       │
│  一级目标（部门级）                                           │
│     ↓                                                       │
│  二级目标（团队级）                                           │
│     ↓                                                       │
│  三级目标（个人级）                                           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### OKR目标管理体系

**O（Objective）目标**：我们要达成什么

**KR（Key Results）关键结果**：如何衡量我们达成了目标

**OKR拆解示例**：

```
O（公司级）：优化产品体验
├─ KR1：季度新增客户超过50%
├─ KR2：大幅增加销售线索（增加一倍）
├─ KR3：提高签约成功率到60%
└─ KR4：搭建销售团队（招聘5名金牌销售）

    KR4进一步拆解：
    ├─ KR4.1：提炼金牌销售画像和资格
    ├─ KR4.2：形成新旧销售对比的详细说明
    └─ KR4.3：重点客户蓄水（招聘时考察资源）
```

### 4.2.3 目标拆解的两大维度

当拿到一个目标后，从两个角度进行拆解：

#### 维度一：目标达成的前提保障（必要条件）

**定义**：达成目标必须具备的基本条件，没有这些条件，目标一定无法达成。

**案例**：电商网站收入提升50%

**前提保障性条件**：
1. 销售团队招聘必须到位（从5人增加到20人）
2. 上线新的在线交易系统，缩短下单流程
3. APP的bug和闪退率控制在一定比例内

#### 维度二：目标达成的进攻叠加（充分条件）

**定义**：将目标细化为一个数字，通过具体手段叠加达成这个数字。

**三种拆解方式**：

##### 方式一：加法形式拆解

**逻辑**：X = A + B + C

**案例**：收入目标1000万
```
收入1000万 = 渠道A收入400万 + 渠道B收入350万 + 渠道C收入250万
或
收入1000万 = 产品A收入400万 + 产品B收入350万 + 产品C收入250万
```

**案例**：新媒体涨粉目标
```
总涨粉 = 自然涨粉 + 内容涨粉 + 活动涨粉

活动涨粉 = 列表增长活动 + 大号互推 + H5传播活动 + 广告投放
```

##### 方式二：乘法形式拆解

**逻辑**：X = A × B × C

**案例**：电商收入公式
```
收入 = 流量 × 转化率 × 客单价
```

**案例**：三节课业务公式
```
收入 = (直播A流量 × 直播A付费转化率 × 系列课a付费转化率 × ARPU)
     + (直播B流量 × 直播B付费转化率 × 系列课b付费转化率 × ARPU)
     + (直播C流量 × 直播C付费转化率 × 系列课c付费转化率 × ARPU)
```

##### 方式三：典型代表式拆解

**逻辑**：抓大放小，找到最具有代表性的部分

**适用场景**：
- 收入构成非常复杂（几十个渠道 × 十几个产品）
- 需要找到抓手来驱动整体增长

**案例**：某复杂电商网站
- 50多个流量渠道 × 20多个重点SKU
- 核心发现：A渠道贡献50%收入，B和C产品贡献60%收入
- 聚焦资源优化A渠道、B产品和C产品

#### 复合应用：三种方式的结合

**案例：滴滴的核心业务公式**

```
B端订单 = (快车司机激活数 × 首单率 × 在线率 × 抢单成功率 × 人均订单数)
        + (专车司机激活数 × 首单率 × 在线率 × 抢单成功率 × 人均订单数)
        + (顺风车司机激活数 × 首单率 × 在线率 × 抢单成功率 × 人均订单数)

C端订单 = 注册用户数 × 首单率 × 日活率 × 人均订单 × 应答率 × 完成率

核心管理逻辑：B端订单 = C端订单
```

滴滴的管理方式：
1. 先预测C端订单量
2. 分解到各条业务线（快车、专车、顺风车）
3. 看B端供给能否匹配
4. 不匹配就做开源（引入更多司机）

### 4.2.4 向上对齐与向下清晰

#### 向上对齐

**核心问题**：你当前的工作对上级leader而言是什么存在？

- 是完成他目标的基本前提保障？
- 还是直接背了他核心数字指标的一部分？

**关键动作**：
- 至少要看到所在业务线/事业部的战略目标
- 理解当前工作与上级目标的关系

#### 向下清晰

**核心问题**：你的目标如何被拆解为可执行的分支？

**关键动作**：
- 目标要达成需要哪些前提保障？
- 目标落实为数字后，如何拆解（加法/乘法/混合）？
- 最终落实到具体的人和事

### 4.2.5 局部战场的4种目标类型

身处局部战场会遇到4类不同的目标，每类目标的制定要点不同：

#### 第一类：核心指标提升型

**定义**：把某个核心业务指标提升X%或降低X%，或保证稳定在一定范围内

**来源**：从业务模型或商业收入公式推导出来

**示例指标**：
- 用户满意度、NPS（保持在一定范围）
- 转化率、留存率（提升X%）
- 获客成本、运营成本（降低X%）

**制定要点**：
- 业务模型梳理得足够清晰
- 在公司内部推演出公式
- 明确你负责哪个指标

**案例**：某健身教育平台营销部门

**业务模型**：
```
免费课 → 线索 → 体验课预约 → 到课 → 转介绍/二次培训
```

**核心指标**：
- 免费课到线索的转化率
- 课程预约率
- 到课率
- 转介绍率
- 二次培训率

**案例**：某民宿平台B端产品团队

**两部分工作**：
1. **B端产品**（服务房东）
   - 目标1：降低房东对平台的问询量（节省客服成本）
   - 目标2：提升房东房源的售卖转化率（通过商业化工具）

2. **内部效率工具**（服务销售团队）
   - 目标：支持单个销售覆盖从5个房东提升到15个房东
   - 最终：提升销售人均单产

#### 第二类：短期效果实现型

**定义**：做一件事，实现某某重要的阶段性短期效果

**来源**：老板重要的短期需求

**示例**：
- 骚扰竞争对手，提高对方的竞争成本
- 为某次PR增加势能
- 集中做一轮PR提升公司估值（为了融资）

**制定要点**：
- **一定要搞清楚**：最后要达成什么效果？怎么衡量？
- **不要问老板**：怎么衡量？（这是给老板增添烦恼）
- **要拿着选项跟老板确认**：我们可能达成X效果，用Y方式衡量，OK吗？

**案例**：融资前的PR工作

**老板需求**：我们要融资了，搞一轮PR

**PR负责人的思考**：
1. 问老板：我们希望达成什么效果？
   - 老板：希望下半年借着行业情况再拿一轮融资

2. 推导：核心是要对投资人做PR

3. 思考如何对投资人做PR：
   - 行业里有哪些渠道？（3个核心渠道）
   - 适合向投资人、行业、C端用户分别怎么做PR？

4. 提出衡量方案：
   - 覆盖所有顶级VC机构
   - 50%以上VC的核心决策者看到
   - 反馈和印象是积极正向的
   - 这就是我们的衡量标准，OK吗？

#### 第三类：模型验证探索型

**定义**：验证一个模型OK或不OK，或探索出一个可依赖的新打法/新模式/新产品

**来源**：
- 原有业务升级的需求
- 原有渠道/玩法衰退，需要开拓新渠道/新玩法

**制定要点**：
- **必须定义清楚**：模型跑通/探索成功的标准是什么？
- **核心标准**：模型必须是可复制、可规模化的
- **最终交付**：一套完整的逻辑，而不是一次单点成功

**案例**：探索KOL投放模型

**老板需求**：SEM投放已稳定，但支持订单目标有风险，探索新渠道

**错误做法**：
- 探索一个月KOL投放
- 有一次成功，带来1000个订单
- 报告：我牛逼，快夸我

**正确做法**：
- 定义交付成果：
  - 把KOL分成ABC三类
  - 把投放素材/文案分成ABC三类
  - 做到：面向不同产品用不同文案素材在不同KOL上投放
  - 最终付费转化率和订单量稳定在一定范围
- 只有做到这一点，才算模型跑通
- 才是可复制、可规模化的

**核心提醒**：
- 一次成功只是开始，远远没有到下结论的时候
- 老板的预期是需要一个模型，不是一次单点动作

#### 第四类：长期系统工程型

**定义**：通过一系列工作，让业务实现某个重要的阶段性成果或拥有某种强大的能力

**来源**：需要中长期思考，需要脑海中形成发展路径

**周期要求**：
- 前三类：3-6个月为一个周期
- 第四类：6-12个月甚至更长

**制定要点**：
- 必须有长期的想象
- 要看到半年到一年甚至更长时间的发展
- 脑海中要有清晰的蓝图
- 分成阶段一、阶段二、阶段三...

**案例**：某中台产品的"666规划"

**第一阶段（第1个6个月）：系统建设期**
- 目标：系统从无到有
- 包含6大模块
- 寻找5-20个种子用户试用
- 跑通流程

**第二阶段（第2个6个月）：业务接入期**
- 目标：前台业务全面接入系统
- 先做A部门，再做B部门，再做C部门
- 循序渐进持续推进
- 最终：各业务线不再重复造轮子（订单系统、营销工具统一）

**第三阶段（第3个6个月）：系统成熟期**
- 目标：系统标准化、组件化改造
- 让系统变得更高效

**关键提醒**：
- 如果你没有长期蓝图，这件事会推进得很痛苦
- 你会变成"打杂的"：业务有什么需求就做什么
- 做了三五件事后，讲不清楚自己到底在干什么

### 4.2.6 目标与协作部门对齐

#### 两类团队

**业务团队**：直接做增长、服务、交付、生产供给，处于业务流程的某个环节

**职能支持团队**：给业务提供基础工具和效率提升系统（产品研发、HR、财务、法务等）

#### 业务团队下的对齐

**步骤1：先拆指标**

横向分解（加法）：按用户群组分解
```
月活用户 = 新增活用户 + 已有活用户
```

纵向分解（乘法）：按用户漏斗分解
```
新增活用户 = 新注册用户数 × 激活率
已有活用户 = 老用户数 × 月/周活跃率
```

**协作模式1：跨功能作战单元**

多个跨功能的职能团队成员组合成一个小作战单元，对同一个指标负责

**示例**：新用户激活指标

**团队组成**：运营 + 产品 + 研发

**协作关系**：
- 谁驱动？
  - 产品驱动：运营提供素材/物料，研发提供保障
  - 或运营驱动：产品/研发提供资源和开发保障

- 指标分配：
  - 驱动人：背核心指标（例如：通过3个版本升级，激活率提升到X%）
  - 配合人：背过程性指标（例如：按产品要求提供配套物料，保证每个版本按时上线） + 共背结果性指标

**协作模式2：多个团队合作推动同一个指标**

多个团队各自认领一部分数字目标，支撑最终指标达成

**示例**：年度新增用户50万

**目标分配**：
- 运营渠道团队：40万
- BD商务团队：5万
- 产品推荐自增长：5万

**上下游对齐示例**：2B公司的线索与转化

**问题**：前面的人提供线索，后面的人做转化，容易打架

**对齐方式**：
- 前面的人：
  - 第1个指标：线索的数量
  - 第2个指标：跟销售团队一起背转化率
- 这样两个部门能更好地协同

#### 职能支持团队下的对齐

**核心原则**：
- **短期**：跟业务部门在目标上必须对齐
  - 要么共背一个指标
  - 要么完成的工作是业务部门目标达成的前提

- **长期**：一定要有自己的独立规划，反向驱动业务效率提升
  - 否则会变成"呼之则来、挥之则去"的被动角色
  - 长期不会有成果，可能一团混乱

**案例**：某民宿平台B端产品团队

**工作内容**：
1. B端产品（服务房东）
2. 内部效率工具（服务销售）

**短期对齐**：
- 共背指标：房东售卖转化率、销售人均单产
- 完全跟业务部门对齐，没有障碍

**长期规划**：
- 做一个房东管理系统
- 调研市面上类似系统（客栈、酒店PMS等）
- 发现总共需要10个模块才能做好

**长期规划落地**：
1. 形成对系统的完整想象（10个模块）
2. 拿着想象跟业务团队讨论和共识
3. 讨论当下半年内，先做哪个模块对业务更有帮助
4. 结合系统理解，形成另一条线的目标：
   - 为了提升用户价值和系统效率
   - 未来半年至少做3个大版本
   - 既能支持业务需求，也符合系统长期发展规律

## 4.3 如何将目标拆解为靠谱的行动计划

### 4.3.1 将目标落实为工作规划的常规逻辑

**核心步骤**：
1. 明确目标
2. 识别达成目标的前提保障条件
3. 识别达成目标的进攻叠加路径
4. 梳理所有事情之间的关系
5. 考虑资源限制，制定执行规划

**案例**：某电商公司月收入目标1000万

**收入公式**：
```
收入 = 流量 × 转化率 × 客单价
```

**前提保障条件**：
1. 销售团队招聘到位（从5人到20人）
2. 上线新在线交易系统
3. APP bug和闪退率控制在X%以内

**进攻叠加路径**：
- 按渠道拆解：渠道A + 渠道B + 渠道C
- 按产品拆解：产品A + 产品B + 产品C

**执行规划**：
- 前提保障条件必须在规定时间达成（例如1个月内招到20个销售）
- 进攻叠加路径落实到具体团队和人

### 4.3.2 应对模糊、需要定义的问题

很多时候老板会给到一个不够清楚的问题，需要你去定义和解决。

**三类处理方法**：

#### 方法1：定义问题

**步骤**：
1. 把现象转换为待解决的具体问题
2. 评估和验证这个现象背后是否真的意味着一个重要问题
3. 用数据/事实证明给老板看

**案例1**：老板说"最近用户口碑有点下滑"

**你的处理**：
1. 看数据：用户满意度、NPS、推荐就业成功率等
2. 发现：确实在下滑，问题存在
3. 分析原因：助教服务/供应能力不足以支撑现有业务
4. 定义问题：助教业务模型需要升级
5. 明确目标：搭建一套新的、能支撑更大吞吐量的助教业务模型

**案例2**：老板说"最近半年不少人离职，员工忠诚度出了问题"

**你的处理**：
1. 看数据：过去半年离职率是6%
2. 对标：行业平均水准是7%
3. 结论：员工忠诚度并没有出现问题
4. 反馈给老板：只是老板的主观感知（可能正好认识的一批人都走了）
5. 结论：不值得花太多时间和焦虑去关注

#### 方法2：寻找外部参考模型

**适用场景**：问题只是模糊，但不复杂，你之前没做过

**步骤**：
1. 定义问题背后老板担心的到底是什么
2. 找到外部专家，学习行业通用做法
3. 找到可参照的模型
4. 制定行动计划，跟老板确认

**案例**：老板说"我们的PR势能离对手差太远了"

**你的处理**：
1. 定义问题：老板担心什么？
   - 对手PR做得好 → 融资会更容易、对投资人的曝光和知名度更高
   - 对手PR做得好 → 用户增长和商业价值增长速度会超过我们

2. 找专家：在行约了十几个品牌PR专家，各聊一个多小时
   - 一家公司PR工作该怎么思考、规划、评估？
   - 如何向投资人做PR？
   - PR工作好是否带动收入增长？怎么做？

3. 找到模型：
   - 向投资人做PR：2种可参照逻辑
   - 通过PR驱动增长：2种可参照模型

4. 回复老板：
   - 如果要解决这两个问题，我找到这么几类解决方案
   - 我感觉其中有个思路比较靠谱
   - 投入100-200万，干两个月
   - 可以看到这些成果
   - 要不要搞？

#### 方法3：层层分解，找到破局点

**适用场景**：问题既模糊又复杂

**步骤**：
1. 对问题进行层层分解
2. 梳理整个问题背后有没有关键的破局点
3. 从破局点先行切入

**案例**：某2B创投媒体公司要探索新业务

**背景**：
- 现有业务：媒体、2B服务咨询、高端培训
- 增长都已乏力
- 需要：明年探索一个新的增长方向

**高层思考过程**：

**第一步**：盘点核心资源
1. 企业家资源（大量忠实用户）
2. 行业影响力（媒体）
3. 擅长做活动和社群

**第二步**：看产业上下游大机会
1. 2B服务（未来几年大增长）
2. 职业教育（政府有政策，用2B资源吸引C）
3. 投融资服务
4. 高端企业家社群

**第三步**：评估哪块更有机会做成
- 结合团队基因
- 决定做：职业教育产品 + 企业服务产品

**第四步**：企业服务怎么做？
- 目标用户：C轮以前互联网科技公司的创始人居多
- 需求分析：这些用户的需求是怎样的？
- 选项：
  1. 自己做（数据产品、SaaS平台、协同SaaS、高端咨询）
  2. 接入其他产品，先做收入
  3. 打包多个产品成综合SaaS

**第五步**：找到破局点
- 核心判断：做企业服务本质是向用户售卖解决方案
- 售卖解决方案必须先做诊断
- **所有事情的原点**：能不能培养出来一群牛逼的顾问式销售？
- **破局点**：培养牛逼的顾问式销售
- 如果没有这群人，做多少产品都很难卖出去

### 4.3.3 应对特别紧迫的资源约束

当资源有限，但老板要求更快、更高效地达成目标时，如何应对？

#### 核心思路：利用杠杆效应

**逻辑**：
1. 先有一个一般的达成路径
2. 思考：是否有一两件具有杠杆效应的事，能显著数倍加速这个过程？

#### 三类常见杠杆

##### 杠杆1：拿下某个领域内最有影响力的"头牌"式用户

**案例**：某美食社区APP上线的电商平台

**背景**：目标是半年内月GMV达到100万以上，需要邀请零食品牌入驻

**初始做法（失败）**：
- 第1个月沟通了三四十家品牌
- 没有人愿意入驻
- 非常沮丧

**调整后的做法（成功）**：
1. 复盘：不应平均分配时间和精力
2. 调整策略：集中搞定一两家的品牌，带动中小品牌集体入驻
3. 锁定目标：良品铺子
4. 集中2-3周，只做一件事：
   - 让良品铺子尝到甜头
   - 跟电商事业部从上到下搞好关系
   - 主动帮良品铺子开店、做设计、做人肉运营
   - 把社区流量导到他们的淘宝店
5. 见效：电商事业部老大看到确实有收入、有流量
6. 谈判：顺势谈入驻，容易得多
7. 攻下良品铺子后，对其他中小品牌：
   - "良品铺子已经入驻了，而且在我这获得了X收入"
   - "我们现在早期有流量补贴政策，来不来？"
   - 非常顺利

**核心逻辑**：
- 先搞定最有影响力的头牌
- 借助头牌的影响力
- 吸引大量腰部和尾部用户跟随

##### 杠杆2：先让自己在某个细分领域或渠道内成为头部

**案例回顾**：彭小六
- 微信公号、知乎竞争激烈
- 转战简书，成为简书一哥
- 带着这个title和势能
- 席卷出版、微信公号等战场

**核心逻辑**：
- 在某个细分领域成为头部
- 带着头部势能进入其他战场
- 更容易获得资源和关注

##### 杠杆3：让自己先拥有某项稀缺的资源或能力，形成对其他人的角色反转

**案例1**：喜马拉雅
- 背景：2015年众多音频电台APP竞争
- 判断：内容供给量直接决定增长速度
- 发现：阅文集团有大量小说IP和内容版权
- 价值：当时没有明确变现通路或价值放大不够
- 操作：跟阅文合作，把大量内容转成音频
- 结果：短时间内极大扩充内容供给量
- 优势：瞬间拉开与竞争对手的差距

**案例2**：某招聘网站
- 背景：2014-2015年，大众创业万众创新
- 发现：前端工程师变得极度稀缺（供不应求）
- 操作：持续搞到前端工程师简历，作为杠杆放出来
- 逻辑：互联网公司要看到这些简历，必须注册、上传职位详情
- 结果：半年内，每个月搞一批前端工程师简历
- 效果：构成非常有效、非常好的流量增长和数据增长手段

**核心逻辑**：
- 任何市场都有供需关系
- 供需关系是动态平衡的
- 一定存在某些资源/能力是相对稀缺的
- 搞定稀缺资源，就能形成对其他人的吸引力

#### 杠杆的叠加使用

三类杠杆完全可以叠加使用，制造更大的杠杆效应。

**案例**：某电商网站，半年内要在三个渠道把收入做到500万

**叠加策略**：
1. 拿下某知名服装品牌（杠杆1：头牌用户）
2. 在京东618，集中资源做成某服装细分品类的头部品牌（杠杆2：细分头部）
3. 拿到非常稀缺的服装供给侧资源，在618更好发力（杠杆3：稀缺资源）
4. 三类杠杆叠加，效率更高

**关键提醒**：
- 寻求杠杆效应的加持和叠加
- 对执行力的要求会更高

### 4.3.4 三类操盘手工作规划实例

#### 实例1：围绕增长类目标的层层落地

**特点**：增长类目标（收入、用户数等）需要非常数据化、非常实际的规划

**案例1**：某电商网店内衣品类运营（2014-2015年半年规划）

**目标**：月收入从5-6万提升到100万以上

**收入公式**：
```
收入 = 流量 × 转化率 × 客单价
```

**策略**：客单价半年内很难大幅提升，核心是提升流量和转化率（提升订单数）

**月度规划**：

| 月份 | 目标 | 核心抓手 | 转化率目标 | 日订单目标 |
|------|------|----------|-----------|-----------|
| 11月 | 打造1-3个爆品，月销30-60个 | 打造爆品 | 1% | 10 |
| 12月 | 继续做爆款单品，月销200个 | 直通车、钻展加大投放 | 1% | 28 |
| 2-3月 | 充量阶段，流量和转化率都要提升 | 清仓活动、团购活动 | 1% → 1.3% → 1.6% | 显著提升 |

**逻辑**：
- 只要最后算账能算平，客单价下降但转化率大幅提升也值得做
- 数据拆解清晰，老板看得清楚，下属执行明确

**案例2**：某化妆品电商公司全年规划（2015年）

**目标**：年度收入900万，从新店起步

**规划思路**：按时间阶段拆解

| 阶段 | 核心目标 | 配置资源和重点工作 |
|------|----------|-------------------|
| Q1 | 日销4600+，日uv 1900+ | 寻找主推产品，打磨体验品，做引流，月度主题活动 |
| Q2 | 日销2万+，日uv 5000+ | 提升转化率和优惠，找符合品牌定位的爆品爆款，通过爆品做引流 |
| Q3 | 客单价显著提升，转化率和优惠也提升 | 主推热卖特惠套装，通过套装拉伸客单价，活动不能少 |
| Q4 | 客单价继续上升，转化率上升，优惠提升少 | 主推爆款、活动、套装，拉伸客单价 |

**核心总结**：
- 优秀的工作规划需要：
  1. 明确的目标
  2. 拆解的逻辑（按阶段、按转化率提升等）
  3. 每个重要目标达成上有明确的抓手
- 表达呈现上做到对象对象都要非常清晰

#### 实例2：围绕包含多条战线的方向性目标

**特点**：老板给的可能是方向性目标，不够明确

**案例**：一年内在品类或细分赛道成为行业第一

**问题**：什么是行业第一？是收入？还是其他条件？

**处理思路**：

**步骤1**：定义"绝对标杆和第一"
- 口碑要好
- 销量也要好
- 还要有行业影响力

**步骤2**：识别进攻叠加性目标 vs 前提保障性条件

**进攻叠加性目标**：订单吞吐量（用户规模/收入）
- 最客观、最硬、没什么争议

**前提保障性条件**：
- 课程体系要完善
- 口碑要好
- 要有行业权威背书

**步骤3**：让目标变得SMART

- 课程体系：基于明确的学科地图，完成15门课程搭建
- 口碑：课程净推荐率不低于60%
- 订单：一年内达到每月10万个订单
- 背书：获得行业5个最有影响力的头部VC和5家公司的背书

**步骤4**：思考达成路径和节奏

**起点**：每月5000个订单
**终点**：每月10万个订单

**收入公式**：
```
订单 = SKU × 渠道 × 转化率
```

**路径规划**：
1. 口碑提升计划：NPS提升到60%以上（需要先发生）
2. 课程体系设计计划：完成15门课程（与收入强相关，需要先发生）
3. 行业权威背书：可早可晚（节奏可以靠后）

**步骤5**：寻找杠杆效应，找到最优路径

**三类常见杠杆**：
1. 拿下该领域最有影响力的头牌用户（明星用户/老师）
2. 先让自己在某个细分领域或渠道内成为头部（先做爆款课程）
3. 拥有某项稀缺的资源或能力（优先做大家都想要但市面上特别稀缺的课程/话题）

**规划中加入杠杆**：
- 搞定明星用户/老师来给我们背书
- 先聚焦资源干好一件事：做一堂爆款课程，拥有极强影响力和口碑
- 优先做稀缺的课程/话题

#### 实例3：独立负责一条业务线的规划

**特点**：老板没空管理，需要自己告诉老板目标是什么、怎么干

**四层次思考框架**：

1. **这件事能做到多大？**（全局方向性思考）
2. **怎么才能做到这么大？**（寻找潜在达成路径）
3. **真的能做到这么大吗？**（验证）
4. **怎么才能做到？**（执行）

**案例**：某历史人文地理短视频/纪录片制作公司

**背景**：
- A轮，150人
- 4条业务线，各独立，无关联
- APP业务线：15人团队（产品和运营）
- 本质：付费内容产品，接近付费订阅

**资源**：
- 内容存量：3000多条免费短视频 + 若干付费专辑 + 少量付费专栏
- 阶段：早期，需求和业务模型都不够明确稳定
- 任务：验证 + 边奔跑边追求增长 + 边验证模型

**第一层：这件事能做到多大？**

**调研方向**：
1. 历史人文地理大方向下，出版行业存量市场是多少？潜在读者数量多少？
2. 类似产品（得到、喜马拉雅）的可参考数据：
   - 得到/喜马拉雅下，历史人文地理三个板块的流量和用户数据
3. 参照得到模式：
   - 垂直领域绝对大咖路线
   - 一个月三次上新
   - 一年下来占据多少市场份额？

**参考结论**：一年做到极致，月活有机会到50万

**第二层：怎么才能做到这么大？**

**参照模型**：同类产品提升日活/月活的三种策略
1. 持续免费内容的话题和IP上新（量不大，但每个都是IP）
2. 系列连载式内容持续更新（粘住用户）
3. 大量免费内容 + 分类 + 消费路径（长期粘住用户）

**选择打法**：免费内容 + 消费路径支撑活跃提升

**第三层：真的能做到这么大吗？**

**验证数据**：
1. 当前免费内容生产供给是否可依赖？
   - 当前：生产一条短视频需要10天
   - 目标：短视频供给量从3000到3万

2. 看过5条以上免费内容的用户留存率是否更高？
   - 数据：高20%

**验证结论**：
- 前者需要想办法实现（资产效率提升 + 外部采购）
- 后者是算账的事（只要达成目标就行）

**第四层：怎么才能做到？**

**执行规划**：
1. 提升免费内容供给量
   - 资产效率提升
   - 外部采购

2. 提升新用户的留存/活跃
   - 增加用户引导
   - 按消费路径去引导
   - 增加用户激励

**核心提醒**：
- 对于AB类操盘手：
  - 重点保证能做好后面两件事（验证+执行）
  - 在前面两件事（能做到多大+怎么做到）层面，能理解逻辑就好
- 当然，4件事都能清晰回答更好，对老板更友好

## 4.4 不同类型目标的进度管理

### 4.4.1 两种进度管理方式

#### 方式1：瀑布流式进度管理

**逻辑**：一步接一步，一环扣一环

**示例**：传统软件开发
```
需求分析 → 架构设计 → 详细设计 → 代码 → 测试 → 上线
```

**要求**：
1. 对所有事情的关系逻辑和要点要提前想得足够清楚
2. 可以有风险预防机制，但很难接受过程中出现大的变动和变化

**适用场景**：
- 达成路径特别明确
- 过去有相关经验
- 想得足够清楚的项目
- 适合通过强约束、强考核、强跟进的方式来管理

#### 方式2：敏捷动态演化式进度管理

**核心认知**：很多事情一开始就想不清楚

**操作方式**（Scrum）：
1. Leader维护一个动态需求池（需求来自老板、自己、用户）
2. 以周为单位做阶段性冲刺回顾
3. 每天开站会，充分同步彼此都在干什么

**要求**：
1. 团队所有成员都要拥有很强的自主能动性
2. 充分拥抱变化
3. 项目Leader要能快速定义问题、解决问题
4. 能够通过超快速的反馈、调整和迭代最终让项目走上正轨

**适用场景**：
- 一开始肯定无法完全想清楚的探索型项目

## 本章小结

### 核心要点回顾

**1. 工作规划与三大模型的关系**
- 商业模式决定基础收入公式
- 业务模型让收入公式更加精细
- 商业经营模型决定团队定位（成本中心vs利润中心）
- 所有工作都要回归到模型上来思考

**2. 目标制定的方法**
- 自上而下目标拆解：前提保障条件 + 进攻叠加路径
- 三种拆解方式：加法、乘法、典型代表式
- 局部战场的4种目标类型，每类有不同的制定要点
- 业务团队vs职能团队的对齐方式不同

**3. 将目标拆解为行动计划**
- 应对模糊问题：定义问题 + 找参考模型 + 找破局点
- 应对资源约束：利用杠杆效应（头牌用户、细分头部、稀缺资源）
- 三类操盘手工作规划实例：增长类、方向性、独立业务线

**4. 进度管理**
- 瀑布流式：路径清晰，强约束强跟进
- 敏捷式：探索型项目，快速迭代
