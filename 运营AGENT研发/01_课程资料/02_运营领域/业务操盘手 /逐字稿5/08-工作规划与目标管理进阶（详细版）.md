# 第4模块第1节：工作规划与三大模型的关系

> 所属章节：第4模块第1节
> 包含小节：工作规划基础逻辑、三大模型与工作规划的关系、实战案例分析、AB类操盘手的心态

## 本节导学

### 核心目标

本节是"工作规划与目标管理进阶"模块的开篇，核心目标是帮助学员理解**工作规划与此前学过的三大模型（商业模式、业务模型、商业经营模型）之间的关系**，建立从业务模型到工作规划的完整认知链条，掌握如何基于业务模型制定合理的目标和工作规划。

### 学习重点

- 理解工作规划的三个核心部分（目标、拆解、进度管理）
- 掌握三大模型如何指导目标制定
- 理解业务模型如何决定组织结构和指标体系
- 学会不同团队如何基于业务模型制定工作目标

### 学习价值

通过本节学习，你将能够：
- 将此前学过的三大模型串联起来，理解它们如何指导实际工作
- 从更高视角理解自己负责的业务在整体中的位置
- 基于业务模型制定合理的目标和工作规划
- 建立操盘手应有的思考框架和心态

#### 模型1：商业模式 → 决定基本收入公式

```
┌─────────────────────────────────────────────────────────────┐
│              商业模式决定基本收入公式                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   我们之前讲过四种商业模式：                                 │
│   1. 流量广告收入                                           │
│   2. 平台交易收成（手续费收入）                             │
│   3. 产销收入                                               │
│   4. 流量+增值服务构成收入                                  │
│                                                             │
│   每种商业模式背后，都有一个基本的收入公式：                 │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐   │
│   │  通用收入公式：                                       │   │
│   │                                                     │   │
│   │    收入 = 流量 × 转化率 × 客单价                      │   │
│   │                                                     │   │
│   │  不同模式下，流量、转化率、客单价的具体含义不同：    │   │
│   └─────────────────────────────────────────────────────┘   │
│                                                             │
│   示例1：流量广告收入模式                                    │
│   ┌─────────────────────────────────────────────────────┐   │
│   │  收入 = 广告流量 × 点击转化率 × 单次点击价格         │   │
│   │                                                     │   │
│   │  提升收入的切入点：                                   │   │
│   │  - 增加广告流量                                       │   │
│   │  - 提升点击转化率                                     │   │
│   │  - 提升单次点击价格（提升目标用户付费能力、覆盖高转化 │   │
│   │    价值的场景）                                       │   │
│   └─────────────────────────────────────────────────────┘   │
│                                                             │
│   示例2：产销收入模式                                        │
│   ┌─────────────────────────────────────────────────────┐   │
│   │  收入 = 流量 × 购买转化率 × 客单价                    │   │
│   │                                                     │   │
│   │  提升收入的切入点：                                   │   │
│   │  - 增加流量                                           │   │
│   │  - 提升购买转化率                                     │   │
│   │  - 提升客单价                                         │   │
│   └─────────────────────────────────────────────────────┘   │
│                                                             │
│   示例3：增值服务收入模式                                    │
│   ┌─────────────────────────────────────────────────────┐   │
│   │  收入 = 流量 × 会员转化率 × 会员客单价               │   │
│   │                                                     │   │
│   │  会员客单价可能是：                                   │   │
│   │  - 单次会员价格                                       │   │
│   │  - 或会员平均LTV（生命周期价值）                      │   │
│   └─────────────────────────────────────────────────────┘   │
│                                                             │
│   核心结论：                                                 │
│   商业模式决定了一家公司基本的收入公式会长成什么样子        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 模型2：业务模型 → 精细化收入公式 + 组织结构

```
┌─────────────────────────────────────────────────────────────┐
│          业务模型的作用：精细化收入公式                      │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   在一家公司内部，follow于商业模式之下，会有业务模型。       │
│                                                             │
│   业务模型在不同阶段的作用：                                 │
│   ┌─────────────────────────────────────────────────────┐   │
│   │  初创期：探索期，模型可能还不清晰                     │   │
│   │  A轮前后：需求被验证、市场被验证                      │   │
│   │           → 必须形成增长模型，实现规模化              │   │
│   │           → 必须能够算账（收入/用户增长如何被算出）   │   │
│   │  成熟期：业务模型可能会变更或升级                      │   │
│   │          → 业务运转效率不断提升                       │   │
│   └─────────────────────────────────────────────────────┘   │
│                                                             │
│   一家公司的完整业务模型由三部分构成：                       │
│   ┌─────────────────────────────────────────────────────┐   │
│   │                                                     │   │
│   │  ┌───────────────────────────────────────────────┐  │   │
│   │  │ 第一层：增长导向的业务模型                     │  │   │
│   │  │    （商业价值产生的路径）                      │  │   │
│   │  └───────────────────────────────────────────────┘  │   │
│   │                       ↑                              │   │
│   │  ┌───────────────────────────────────────────────┐  │   │
│   │  │ 第二层：生产供给交付的保障体系                 │  │   │
│   │  │    （支持增长模型更好的运转）                  │  │   │
│   │  └───────────────────────────────────────────────┘  │   │
│   │                       ↑                              │   │
│   │  ┌───────────────────────────────────────────────┐  │   │
│   │  │ 第三层：基础工具和效率提升系统                 │  │   │
│   │  │    （支撑整体运转的基础设施）                  │  │   │
│   │  └───────────────────────────────────────────────┘  │   │
│   │                                                     │   │
│   └─────────────────────────────────────────────────────┘   │
│                                                             │
│   梳理业务模型的顺序（自上往下）：                           │
│   1. 先梳理增长导向的业务模型                               │
│   2. 再梳理生产供给交付保障体系                             │
│   3. 最后梳理基础工具和效率提升系统                         │
│                                                             │
│   为什么这样梳理？                                           │
│   ┌─────────────────────────────────────────────────────┐   │
│   │  - 生产供给交付保障体系对应到增长导向业务模型的      │   │
│   │    某个或某些环节，支持它更好的运转                  │   │
│   │  - 可能是增长的基础前提（如生产体系）                │   │
│   │  - 可能与增长强关联（如供应链体系，运转效率高        │   │
│   │    则增长空间和效率也会更大）                        │   │
│   └─────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 案例演示：从业务模型到精细化收入公式

**案例：某电商公司**

```
┌─────────────────────────────────────────────────────────────┐
│         案例：电商公司业务模型 → 精细化收入公式               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   第一步：梳理增长导向的业务模型                             │
│   ┌─────────────────────────────────────────────────────┐   │
│   │                                                     │   │
│   │   流量（渠道A/B/C/D）                                │   │
│   │        ↓                                             │   │
│   │   爆款产品一 → 转化为 → 利润产品1/2/3                │   │
│   │   爆款产品二 → 转化为 → 利润产品4/5                  │   │
│   │        ↓                                             │   │
│   │   利润产品 + 附加产品 → 收入                         │   │
│   │                                                     │   │
│   │   渠道细分：                                         │   │
│   │   - 渠道A/B：给爆款产品一引流                         │   │
│   │   - 渠道C/D：给爆款产品二引流                         │   │
│   └─────────────────────────────────────────────────────┘   │
│                                                             │
│   第二步：基于业务模型写出精细化收入公式                     │
│   ┌─────────────────────────────────────────────────────┐   │
│   │                                                     │   │
│   │   收入 = 流量A × 爆款产品一转化率 × 利润产品转化率    │   │
││       × 利润产品客单价                                   │   │
│   │      + 流量B × 爆款产品一转化率 × 利润产品转化率      │   │
│   │      + 流量C × 爆款产品二转化率 × 利润产品转化率      │   │
│   │      + 流量D × 爆款产品二转化率 × 利润产品转化率      │   │
│   │                                                     │   │
│   │   进一步细分：                                       │   │
│   │   - 利润产品转化率可以拆分为：利润产品1转化率、       │   │
│   │     利润产品2转化率、利润产品3转化率                  │   │
│   │   - 最终可以拆分到每个利润产品1234的收入贡献          │   │
│   │   - 附加产品可以作为整体up-selling来看               │   │
│   │                                                     │   │
│   └─────────────────────────────────────────────────────┘   │
│                                                             │
│   第三步：基于收入公式理解组织结构                           │
│   ┌─────────────────────────────────────────────────────┐   │
│   │                                                     │   │
│   │   流量端：                                           │   │
│   │   - 小团队管理                                       │   │
│   │   - 可能按渠道分人（A渠道1人、B渠道1人...）           │   │
│   │                                                     │   │
│   │   爆款产品端：                                       │   │
│   │   - 爆款产品运营团队                                 │   │
│   │   - 关注：爆款产品选品、与流量渠道对接、             │   │
│   │          前端转化（渠道→销售）、                     │   │
│   │          后端转化（爆款→利润产品）、                 │   │
│   │          转化策略制定                                │   │
│   │                                                     │   │
│   │   转化方式端（如果有多种转化方式）：                   │   │
│   │   - 可能需要分别有人来看不同的转化方式                │   │
│   │                                                     │   │
│   └─────────────────────────────────────────────────────┘   │
│                                                             │
│   核心洞察：                                                 │
│   当业务模型梳理清楚、收入公式精细化成型之后，              │
│   一家公司的组织结构和职能划分就可以通过业务模型来理解      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**案例：某跨境电商网站**

```
┌─────────────────────────────────────────────────────────────┐
│      案例：跨境电商网站业务模型 → 指标体系 → 组织结构         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   第一步：基于业务模型拆解GMV                                 │
│   ┌─────────────────────────────────────────────────────┐   │
│   │                                                     │   │
│   │   GMV = 新用户带来的GMV + 老用户带来的GMV            │   │
│   │                                                     │   │
│   │   新用户带来的GMV：                                  │   │
│   │   = 新用户数量 × 付费转化率 × 客单价                 │   │
│   │   = (日均访问数 × 注册转化率) × 付费转化率 × 客单价   │   │
│   │                                                     │   │
│   │   老用户带来的GMV：                                  │   │
│   │   = 已购用户数 × 月均活跃率 × 月均购买转化率 × 客单价  │   │
│   │                                                     │   │
│   └─────────────────────────────────────────────────────┘   │
│                                                             │
│   第二步：基于指标体系划分团队职责                           │
│   ┌─────────────────────────────────────────────────────┐   │
│   │                                                     │   │
│   │   市场团队（负责新用户数量）：                        │   │
│   │   - 指标：新用户数量（从访问到注册转化）             │   │
│   │                                                     │   │
│   │   运营团队（负责付费转化率）：                        │   │
│   │   - 指标：站内用户的付费转化率                        │   │
│   │   - 手段：活动、打折、促销                           │   │
│   │                                                     │   │
│   │   品类/商品运营团队（负责客单价）：                    │   │
│   │   - 指标：客单价                                      │   │
│   │   - 手段：选品、定价、组合销售                        │   │
│   │                                                     │   │
│   │   会员部门（负责老用户活跃率）：                       │   │
│   │   - 指标：老用户的月均活跃率                          │   │
│   │   - 手段：会员体系、积分、等级、权益                  │   │
│   │                                                     │   │
│   └─────────────────────────────────────────────────────┘   │
│                                                             │
│   核心洞察：                                                 │
│   当收入公式被精细化拆解后，团队的分工就有了明确的依据      │
│   每个团队负责哪个指标，清清楚楚                            │
│                                                             │
```

## 本节小结

### 核心知识点

1. **工作规划的三个核心部分**：
   - 明确目标
   - 拆解计划
   - 进度管理

2. **三大模型与工作规划的关系**：
   - **商业模式** → 决定基本收入公式
   - **业务模型** → 精细化收入公式 → 组织结构划分
   - **商业经营模型** → 成本中心/利润中心 → 指标体系

3. **完整的推导链条**：
   ```
   商业模式 → 基本收入公式
       ↓
   业务模型 → 精细化收入公式 → 组织结构划分
       ↓
   商业经营模型 → 成本中心/利润中心 → 指标类型
       ↓
   → 我所在部门的位置、职责、指标
   ```

4. **不同团队如何制定目标**：
   - 销售增长团队：回归收入公式，思考从哪个变量驱动增长
   - 交付团队：在收入公式上找可贡献点（如客单价），在成本公式上找优化点（如效率）

5. **AB类操盘手的心态**：
   - 往上看得到：理解高层战略和整体业务模型
   - 往下能落地：根据模型制定目标和规划
   - 保持信息回归到模型的思考习惯

### 关键洞察

- **业务模型梳理清晰是制定合理目标的前提**
- **不同团队的目标制定，都要回归到收入公式和成本公式来思考**
- **组织结构和指标体系，本质上是业务模型的具象化**
- **不管多复杂，都要保持信息回归到模型的思考习惯**
