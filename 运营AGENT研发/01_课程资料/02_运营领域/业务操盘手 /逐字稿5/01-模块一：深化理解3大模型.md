# 第一章：深化理解3大模型

> 所属章节：第一章
> 核心主题：商业模式、商业经营模型、业务模型的关系与应用

---

## 本章导学

### 核心目标

本章帮助学员深化理解商业系统中的三大核心模型——商业模式、商业经营模型、业务模型，以及它们在不同发展阶段公司中的关系和应用。掌握这些模型能够帮助运营人员：
- 理解公司商业系统的整体架构
- 在不同发展阶段找到正确的关注重点
- 与高层建立"同屏"沟通的语言体系
- 制定符合公司发展阶段的工作策略

### 学习重点

- 掌握三大模型的定义和层次关系
- 理解中早期公司与成熟公司模型构建逻辑的差异
- 明确不同公司类型对操盘手的能力要求
- 学会在不同发展阶段正确应用三大模型

## 详细内容

### 1.1 三大模型的定义与层次关系

#### 1.1.1 三大模型概述

商业系统中存在三大常见的模型，它们构成了公司商业系统的完整框架：

```
┌─────────────────────────────────────────────────────────────┐
│                   商业系统三大模型                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   第一层：商业模式（顶层设计）                               │
│   ├─ 定义：公司如何创造价值、传递价值、获取价值            │
│   ├─ 核心：价值主张、客户细分、收入来源                     │
│   └─ 问题："我们靠什么挣钱？"                               │
│                         ↓                                    │
│   第二层：商业经营模型（财务模型）                           │
│   ├─ 定义：成本结构、利润结构、增长模式                     │
│   ├─ 核心：收入-成本=利润，如何实现盈利和规模化             │
│   └─ 问题："我们的成本和利润结构是怎样的？"                 │
│                                                             │
│   **不同业务类型的具象化公式示例**：                         │
│   └─ 电商业务：流量×转化率×客单价-获客成本-履约成本        │
│   └─ SaaS业务：客户数量×ARPU-获客成本-服务成本             │
│   └─ 广告业务：DAU×广告位×eCPM/1000-流量成本                │
│   └─ 教育培训：线索数×转化率×课单价-销售提成                 │
│                         ↓                                    │
│   第三层：业务模型（执行层面）                               │
│   ├─ 整体业务模型：整个业务如何运转                        │
│   ├─ 局部业务模型：单个环节的业务逻辑                      │
│   └─ 问题："我们的业务如何创造商业价值？"                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 1.1.2 三大模型的层次关系

**从上到下：战略到执行**

| 模型 | 关注问题 | 稳定性 | 可见性 |
|------|----------|--------|--------|
| 商业模式 | 价值创造与获取逻辑 | 相对稳定 | 高层可见 |
| 商业经营模型 | 成本、利润、增长 | 较稳定 | 高层+中层可见 |
| 业务模型（整体） | 业务运转机制 | 不断优化 | 中层+执行层可见 |
| 业务模型（局部） | 具体环节执行 | 高频迭代 | 执行层可见 |

**从下到上：执行验证战略**

- 局部业务模型跑通 → 整体业务模型成型 → 商业经营模型清晰 → 商业模式确立

### 1.2 不同发展阶段公司的模型构建逻辑

#### 1.2.1 中早期公司的模型构建逻辑

**核心特征**：业务先成型，模型后清晰

##### 案例一：三节课第一年的真实状态

**背景**：
三节课在创办的第一年，商业逻辑和商业模式都不清晰。

**实际状态**：
- "啥都干"——没有明确的产品边界
- "不知道怎么挣钱"——商业模式未定
- "先干再说"——在实践中摸索
- "凿出水来了就往下去深造"——找到突破口再深化

**分析**：
这是中早期公司的典型状态。公司刚创办时：
1. 商业逻辑、商业模式都需要在摸索中逐渐清晰
2. 在不断的奔跑过程中，才能对公司的商业逻辑、模式形成更强的概念
3. 老板脑海里虽然有利润、成本的思考，但这些思考是零散的、粗放的
4. 只能按"这一年花多少钱、有没有收入"这种简单逻辑来看

**中早期公司的构建顺序**：

```
第一步：业务模型优先
├─ 先在某件事上创造商业价值（收入或流量）
├─ 业务要先能成熟、成形
└─ 验证商业价值创造的可能性

第二步：标准化验证
├─ 判断创造商业价值的过程能否标准化
├─ 如果可标准化 → 意味着可能规模化放大
└─ 如果不可标准化 → 继续探索

第三步：向上推演
├─ 基于成型的业务模型，推演商业经营模型
├─ 基于商业经营模型，定义商业模式
└─ 最终形成完整的商业系统模型
```

**关键要点**：
- 业务模型必须先成型，这是基础
- 中早期公司老板的财务思考是零散的，没有清晰模型
- **为什么业务未定型无法建立精确模型？**

  因为中早期公司的业务形态、成本结构、收入来源都在快速变化，建立模型的前提是业务相对稳定。如果业务还在不断调整方向，建立出来的模型就已经过时了。这就像试图给一个正在快速成长的孩子做"未来五年衣服尺寸规划"——你做的规划永远跟不上实际变化的速度。

  只有在业务相对稳定、数据积累到一定程度后，才能建立有指导意义的精确模型。

- **"跑通一个模型"需要多长时间？**

  这个过程往往需要**反复试错多次**，可能跑**三五个模型**才能找到一个靠谱的方向。每次试错都需要几个月时间，所以中早期公司的核心能力就是"快速试错、快速迭代"。

#### 1.2.2 成熟公司的模型构建逻辑

**核心特征**：模型先清晰，业务再优化

##### 案例二：滴滴、美油的成熟状态

**背景**：
像滴滴、美团这样的成熟公司，商业模式和商业经营模型已经非常清晰。

**实际状态**：
- 商业模式结构明确
- 商业经营逻辑稳定
- 成本结构、利润结构清晰
- 增长模式可预测

**成熟公司的约束条件**：

成熟公司很难允许有人做一个与现有商业经营模型、商业模式完全背道而驰的事情。如果真有这样的事：
- 不如让这个人去外部创业
- 公司可以投资他
- 不需要在内部单独做

**成熟公司的应用逻辑**：

```
高层视角（守住上边）
├─ 牢牢把握商业模式
├─ 监控商业经营模型
├─ 关注每个业务板块的成本结构、利润、增长
└─ 往下推动业务模型的优化和升级

中层视角（承上启下）
├─ 理解公司的全局业务模型
├─ 搞清楚自己负责板块的业务模型
├─ 用模型语言与高层"同屏"沟通
└─ 基于局部业务模型定目标、做规划

执行层视角（落地执行）
├─ 接收明确的目标和规划
├─ 在局部业务模型下执行到位
└─ 反馈执行数据和问题
```

##### 案例三：新东方的高层管理视角

**背景**：
一家成熟到像新东方这种程度的公司。

**高层视角的核心逻辑**：
1. **财务/商业经营逻辑是第一逻辑**
   - 业务一共分几个板块
   - 每个板块的成本、利润大概怎样
   - 增长可能是怎样的

2. **往下推导业务模型**
   - 每个板块的整体业务模型是怎样的
   - 有没有什么问题
   - 最多看到局部的业务模型

3. **管理深度**
   - 高层最多看到局部业务模型这个部分
   - 不会深入到更细的执行环节

**为什么高层最多看到局部业务模型？**

因为高层的时间精力有限，他们需要从财务模型往下推导业务模型，而不是深入到每个业务细节。高层掌握的是"从上往下"的推导逻辑：

```
财务目标（收入、利润、增长）
    ↓ 推导
整体业务模型（流量×转化×客单价等）
    ↓ 推导
局部业务模型（渠道运营、用户运营、内容运营等）
    ↓ 具体执行
```

具体细节由中层和执行层负责，高层不需要也不应该陷入细节。

**分析**：
成熟公司高层看经营管理，第一逻辑一定是财务或商业经营逻辑。这是因为：
- 商业模式和经营模型已经稳定
- 需要通过财务指标来管理公司
- 业务模型是为财务目标服务的

### 1.3 不同公司类型对操盘手的要求

#### 1.3.1 成熟公司对A/B类操盘手的要求

**A/B类操盘手定位**：
- 大公司里面的项目负责人
- 小公司里面的部门Leader
- 管理几十人到一两百人的规模（非总监级别）

**能力要求**：

```
必须具备的能力
├─ 理解公司的全局业务模型（至少有概念）
│  └─ 知道公司商业价值增长的大致模型
│
├─ 搞清楚自己负责板块的局部业务模型
│  └─ 能够梳理清楚自己模块的业务逻辑
│
└─ 基于此往下定目标、做规划
   └─ 给一线同学明确的执行方向

不需要具备的能力
└─ 梳理清楚整个公司所有的全局业务模型
   └─ 这是有挑战的，没有这样的要求
```

**工作方式**：
- 在整体业务模型层面上与老板"同屏"交流
- 所有工作要交付给老板一个"模型"
- 要么讲数据，要么讲稳定的业务模型
- 说明这个模型能给公司带来多少商业价值增长
- 基于局部业务模型定目标、做规划
- 下放到执行层让他们执行到位

#### 1.3.2 中早期公司对A/B类操盘手的要求

**中早期公司的特征**：
- 商业模式、经营模型、业务模型都在动态调整
- 三个模型之间不断动态互相矫正
- 高层视角可能在"跑模型、选方向"和"执行"之间切换

**能力要求**：

```
核心职责
├─ 一线执行破局
│  └─ 有些战线必须依赖操盘手本人才能打开
│
├─ 不断抽象业务模型
│  └─ 用这个模型跟上级对话
│
└─ 辅助上级进行动态决策
   └─ 帮助老板判断方向的靠谱程度
```

**工作方式**：
- 必须扎到一线去执行
- 跑完一线后要退一步思考模型
- 不断在"一线执行"和"模型思考"之间切换
- 用抽象出来的业务模型辅助老板决策

**过初创期后的公司高层视角**：
- 不断跑模型、选方向
- 计算商业可能性、成本利润能否打平、能否挣钱
- 如果方向不靠谱 → 换一个方向，跑新模型
- 如果方向靠谱 → 强化它

### 1.4 三大模型在企业经营管理中的应用总结

#### 1.4.1 成熟公司的应用逻辑

```
┌─────────────────────────────────────────────────────────────┐
│                    成熟公司模型应用                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  高层（商业模式+商业经营模型）                               │
│  ├─ 重点：成本结构、利润、增长                               │
│  ├─ 视角：财务/商业经营逻辑                                 │
│  └─ 深度：全局业务模型 → 局部业务模型                       │
│                         ↓                                    │
│  中层（整体业务模型+局部业务模型）                           │
│  ├─ 重点：与老板同屏、定目标、做规划                         │
│  ├─ 交付：数据+稳定的业务模型                               │
│  └─ 深度：局部业务模型 → 执行层目标                         │
│                         ↓                                    │
│  执行层（局部业务模型执行）                                  │
│  └─ 重点：执行到位、反馈数据                                 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 关键要点总结

### 1. 三大模型的定义必须准确

- ✅ 正确：商业模式、商业经营模型、业务模型
- ❌ 错误：业务模型、运营模型、组织模型

### 2. 不同发展阶段逻辑完全不同

- 中早期公司：业务模型 → 商业经营模型 → 商业模式（自下而上）
- 成熟公司：商业模式 → 商业经营模型 → 业务模型（自上而下）

### 3. 操盘手要根据公司阶段调整工作方式

- 成熟公司：理解全局、搞清局部、定目标做规划
- 中早期公司：一线破局、抽象模型、辅助决策

### 4. 案例的颗粒度必须完整

每个案例必须包含：背景+实际状态+分析+启示

### 5. 观点必须有充分论证

- 不能只有陈述，没有论证
- 必须说明"为什么是这样"
- 必须说明"在什么条件下适用"
