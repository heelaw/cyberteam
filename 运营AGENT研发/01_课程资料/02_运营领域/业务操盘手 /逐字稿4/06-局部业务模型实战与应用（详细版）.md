## 本章导学（核心目标）

本章将深入讲解局部业务模型的实战应用方法。局部业务模型是指在整体业务模型框架下，针对特定业务板块或产品线设计的业务模型。对于中层管理者来说，能够梳理和优化局部业务模型是必备的核心能力。本章将通过大量案例帮助你掌握局部业务模型的设计和应用方法。

**学习目标：**
1. 理解局部业务模型与整体业务模型的关系
2. 掌握局部业务模型的构成要素
3. 学会设计和优化局部业务模型
4. 能够将局部业务模型应用到实际工作中

## 详细内容

### 3.1 局部业务模型的基本概念

#### 什么是局部业务模型？

局部业务模型是指在整体业务模型框架下，针对特定业务板块、产品线或用户群体设计的业务模型。它是整体业务模型的一部分，但又有其相对独立性和完整性。

```
┌─────────────────────────────────────────────────────────────────────┐
│                    整体业务模型 vs 局部业务模型                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   整体业务模型                                                       │
│   ┌─────────────────────────────────────────────────────────────┐   │
│   │                                                             │   │
│   │   ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐    │   │
│   │   │ 用户增长  │ │ 产品运营 │ │ 销售转化 │ │ 客户服务  │    │   │
│   │   │ 业务模型  │ │ 业务模型 │ │ 业务模型 │ │ 业务模型  │    │   │
│   │   └──────────┘ └──────────┘ └──────────┘ └──────────┘    │   │
│   │                                                             │   │
│   └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│   局部业务模型                                                       │

│   │  ┌─────────────────────────────────────────────────────────┐│   │
│   │  │                                                         ││   │
│   │  │  流量获取 → 激活 → 留存 → 转化 → 推荐                 ││   │

│   │  └─────────────────────────────────────────────────────────┘│   │
│   └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

#### 局部业务模型的特征

```
┌─────────────────────────────────────────────────────────────────────┐
│                    局部业务模型的四大特征                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   特征1：目标明确                                                   │
│   ┌─────────────────────────────────────────────────────────────┐   │
│   │  - 每个局部业务模型都有明确的业务目标                       │   │
│   │  - 目标应该是可衡量、可量化的                              │   │
│   │  - 示例：用户增长模型的目标是"获取10000新用户"            │   │
│   └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│   特征2：逻辑闭环                                                   │
│   ┌─────────────────────────────────────────────────────────────┐   │
│   │  - 局部业务模型本身是一个完整的业务闭环                   │   │
│   │  - 有明确的输入和输出                                      │   │
│   │  - 能够独立运转并产生业务价值                              │   │
│   └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│   特征3：可拆解性                                                   │
│   ┌─────────────────────────────────────────────────────────────┐   │
│   │  - 局部业务模型可以进一步拆解为更小的业务单元             │   │
│   │  - 便于分解为具体的工作任务和执行动作                    │   │
│   │  - 便于进行数据监测和效果评估                             │   │
│   └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│   特征4：关联性                                                     │
│   ┌─────────────────────────────────────────────────────────────┐   │
│   │  - 局部业务模型与其他局部模型相互关联                     │   │
│   │  - 局部模型的输出可能是其他模型的输入                      │   │
│   │  - 需要考虑与其他业务的协同和配合                          │   │
│   └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 3.2 局部业务模型的构成要素

#### 局部业务模型的6大要素

```
┌─────────────────────────────────────────────────────────────────────┐
│                    局部业务模型的6大构成要素                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌─────────────────────────────────────────────────────────────┐   │
│   │  要素1：业务目标                                           │   │
│   │  - 明确要达成的业务结果                                   │   │
│   │  - 目标应该是SMART的                                      │   │
│   │  - 例如：月活用户达到100万                                 │   │
│   └─────────────────────────────────────────────────────────────┘   │
│                              ↓                                       │
│   ┌─────────────────────────────────────────────────────────────┐   │
│   │  要素2：目标用户                                           │   │
│   │  - 明确服务的目标用户群体                                 │   │
│   │  - 用户画像和特征描述                                     │   │
│   │  - 例如：一线城市25-35岁职场女性                          │   │
│   └─────────────────────────────────────────────────────────────┘   │
│                              ↓                                       │
│   ┌─────────────────────────────────────────────────────────────┐   │
│   │  要素3：价值主张                                           │   │
│   │  - 为用户创造什么价值                                     │   │
│   │  - 与其他产品的差异化是什么                               │   │
│   │  - 例如：帮助用户每天节省2小时                            │   │
│   └─────────────────────────────────────────────────────────────┘   │
│                              ↓                                       │
│   ┌─────────────────────────────────────────────────────────────┐   │
│   │  要素4：业务链路                                           │   │
│   │  - 业务运转的完整流程和环节                               │   │
│   │  - 各环节的输入输出                                       │   │
│   │  - 关键转化节点和瓶颈                                     │   │
│   └─────────────────────────────────────────────────────────────┘   │
│                              ↓                                       │
│   ┌─────────────────────────────────────────────────────────────┐   │
│   │  要素5：核心指标                                           │   │
│   │  - 衡量业务效果的KPI                                      │   │
│   │  - 指标之间的关系和计算公式                               │   │
│   │  - 例如：DAU = 新增用户 + 留存用户 - 流失用户             │   │
│   └─────────────────────────────────────────────────────────────┘   │
│                              ↓                                       │
│   ┌─────────────────────────────────────────────────────────────┐   │
│   │  要素6：资源支撑                                           │   │
│   │  - 需要投入的人力、资金、技术等资源                       │   │
│   │  - 资源的获取和分配方式                                   │   │
│   │  - 例如：运营团队5人，月预算50万                          │   │
│   └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

#### 类型二：转化变现模型

```
┌─────────────────────────────────────────────────────────────────────┐

├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   模型定义：                                                        │

│   │       ↓ 20%                                                │   │

│   │       ↓ 30%                                                │   │
```


### 3.4 局部业务模型的设计方法

#### 设计步骤一：明确业务目标

```
┌─────────────────────────────────────────────────────────────────────┐
│                    局部业务模型设计步骤一                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   目标设定原则：                                                    │
│                                                                     │
│   ┌─────────────────────────────────────────────────────────────┐   │
│   │  SMART原则                                                 │   │
│   │  ├─ Specific（具体的）：目标要具体明确                     │   │
│   │  ├─ Measurable（可衡量的）：目标要能量化                   │   │
│   │  ├─ Achievable（可实现的）：目标要合理可达成               │   │
│   │  ├─ Relevant（相关的）：目标要与整体业务相关               │   │
│   │  └─ Time-bound（有时限的）：目标要有明确的时间节点         │   │
│   └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│   目标设定的常见错误：                                             │
│                                                                     │
│   ┌─────────────────────────────────────────────────────────────┐   │
│   │  错误1：目标过于模糊                                       │   │
│   │  └─ "提升用户增长" → "新增用户数提升30%"                 │   │
│   │                                                             │   │
│   │  错误2：目标缺乏量化                                       │   │
│   │  └─ "提升转化率" → "付费转化率提升到5%"                  │   │
│   │                                                             │   │
│   │  错误3：目标过于激进或保守                                 │   │
│   │  └─ 需要基于历史数据和市场情况合理设定                    │   │
│   │                                                             │   │
│   │  错误4：目标与整体业务脱节                                 │   │
│   │  └─ 局部目标要能支撑整体业务目标的实现                    │   │
│   └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

#### 设计步骤二：梳理业务链路

```
┌─────────────────────────────────────────────────────────────────────┐
│                    局部业务模型设计步骤二                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   业务链路梳理方法：                                                │
│                                                                     │
│   步骤1：识别关键环节                                              │
│   ┌─────────────────────────────────────────────────────────────┐   │
│   │  - 从输入到输出，识别所有关键业务环节                      │   │

│   │  - 标注各环节的输入和输出                                  │   │

│                              ↓                                       │
│   步骤2：识别转化节点                                              │
│   ┌─────────────────────────────────────────────────────────────┐   │
│   │  - 找出业务链路中的关键转化点                              │   │
│   │  - 量化各节点的转化率                                      │   │
│   │  - 识别可能的瓶颈节点                                      │   │
│   └─────────────────────────────────────────────────────────────┘   │
│                              ↓                                       │
```

## 本章小结

本章深入讲解了局部业务模型的实战应用方法：

1. **局部业务模型的基本概念**：是整体业务模型下的业务板块，具有目标明确、逻辑闭环、可拆解、关联性四大特征

2. **构成要素**：业务目标、目标用户、价值主张、业务链路、核心指标、资源支撑

局部业务模型是中层管理者的核心工作抓手，能够设计和优化局部业务模型是提升管理能力的关键。
