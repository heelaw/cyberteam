## 本章导学

### 核心目标

本章是跳出一线执行视角课程的入门篇章，核心目标是帮助学员建立从CEO和高层的视角看待公司和运营工作的能力。很多运营人员只关注执行层面的工作，忽略了更高层次的思考。本章将帮助学员理解公司的整体运作逻辑，从战略层面理解运营工作。

### 学习重点

本章的学习重点包括：理解CEO和高层的视角；理解如何评估和预测公司价值；掌握3大商业模型及其关系。

### 1.2 如何评估和预测一家公司的价值

**公司价值的核心要素**：

```
┌─────────────────────────────────────────────────────────────┐
│                   公司价值评估模型                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌─────────────────────────────────────────────────┐       │
│   │              收入能力                              │       │
│   │         ├─ 当前收入                               │       │
│   │         └─ 收入增长预期                           │       │
│   └─────────────────────────────────────────────────┘       │
│                         ↓                                    │
│   ┌─────────────────────────────────────────────────┐       │
│   │              盈利能力                              │       │
│   │         ├─ 毛利率                                 │       │
│   │         └─ 净利率                                 │       │
│   └─────────────────────────────────────────────────┘       │
│                         ↓                                    │
│   ┌─────────────────────────────────────────────────┐       │
│   │              增长潜力                              │       │
│   │         ├─ 市场规模                               │       │
│   │         └─ 市场份额                               │       │
│   └─────────────────────────────────────────────────┘       │
│                         ↓                                    │

│   │         ├─ 品牌                                   │       │
│   │         ├─ 技术                                   │       │
│   │         └─ 资源                                   │       │
│   └─────────────────────────────────────────────────┘       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**投资人评估公司价值的基本逻辑**：

```
投资人视角的评估流程：

1. 了解业务 → 形成基本假设
   ↓
2. 基于假设 → 建立业务模型
   ↓
3. 提取数据 → 验证假设和模型
   ↓
4. 基于验证结果 → 形成预测模型
   ↓
5. 根据预测模型 → 评估公司价值和增长潜力
```

**案例：某教育公司的价值评估**

假设一家教育公司业务分为B端和C端两个板块，C端有A、B两类课程：

**基本假设**：
- 用户首次付费转化率稳定
- 课程口碑和数据表现良好
- 可以支持一门课程在每次开班时订单稳定增长
- 增长周期至少可持续两年

**收入公式**：

```
C端A类课程收入 = 月开班数 × 满班率 × 课程售卖周期（月） × 课单价

C端B类课程收入 = 月开班数 × 满班率 × 课程售卖周期（月） × 课单价

C端总收入 = A类课程收入 + B类课程收入

公司总收入 = C端收入 + B端收入
```

**增长驱动因素**（从收入公式推导）：
- 增加A类或B类课程的数量
- 增加开班频次
- 提升满班率
- 提高课单价
- 延长课程售卖周期

**投资人关注的核心数据**：
- 每次开课用户数的增加幅度（验证口碑假设）
- 当前课程数量和开课频率
- 用户付费转化率
- 前端流量获取能力和转化情况
- 课程生命周期和复购率

**预测模型示例**：
```
未来一年预测：
- A课程：5门 → 扩展到X门
- B课程：6门 → 扩展到Y门
- 开课频率：提升150%
- 预期收入增长：5倍以上

主要风险：
- 前端流量获取能力可能不足
- 增加课程后转化率可能下降
```

**核心启示**：

评估和预测公司价值的本质是：**通过模型进行思考和决策**

```
逻辑链条：
A（可控变量）→ 影响 B（中间变量）→ 最终影响 C（商业价值）

操盘手的能力：
→ 在脑海中建立各种模型
→ 找到影响C的A
→ 让A发生变化，驱动B，最终提升C
→ 把公司经营管理逐步模型化
```

### 1.3 一个商业系统中的3大模型

**3大模型概述**：

```
┌─────────────────────────────────────────────────────────────┐
│                   商业3大模型                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌─────────────────────────────────────────────────┐       │
│   │  1. 业务模型                                     │       │
│   │     - 商业模式：如何提供价值、如何获取收入        │       │
│   │     - 核心要素：产品、用户、渠道                  │       │
│   └─────────────────────────────────────────────────┘       │
│                         ↓                                    │
│   ┌─────────────────────────────────────────────────┐       │
│   │  2. 运营模型                                     │       │
│   │     - 如何运转：获取用户、留存用户、获取收入      │       │
│   │     - 核心要素：流量、转化、留存                  │       │
│   └─────────────────────────────────────────────────┘       │
│                         ↓                                    │
│   ┌─────────────────────────────────────────────────┐       │
│   │  3. 组织模型                                     │       │
│   │     - 如何支撑：团队结构、流程、文化             │       │
│   │     - 核心要素：人、流程、文化                    │       │
│   └─────────────────────────────────────────────────┘       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**业务模型**：
- 定义：如何提供价值、如何获取收入
- 核心问题：卖什么、卖给谁、怎么卖

**运营模型**：
- 定义：如何让业务正常运转
- 核心问题：如何获取用户、如何留存用户、如何获取收入

**组织模型**：
- 定义：如何支撑业务和运营
- 核心问题：需要什么样的人、如何协作、如何激励

### 1.3.1 三大模型在不同公司阶段的应用

**中早期公司 vs 成熟公司的模型关系**：

```
┌─────────────────────────────────────────────────────────────┐
│            中早期公司 vs 成熟公司的模型关注点                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   中早期公司：                                              │
│   ├─ 优先级：业务模型 > 商业经营模型 > 商业模式             │
│   ├─ 特点：商业模式不清晰、需要不断探索                     │
│   ├─ 重点：先让业务成形，再往上推演模型                      │
│   └─ 高层视角：跑模型、选方向、动态调整                     │
│                                                             │
│   成熟公司：                                                │
│   ├─ 优先级：商业模式 > 商业经营模型 > 业务模型             │
│   ├─ 特点：商业模式清晰、经营逻辑稳定                       │
│   ├─ 重点：守住上层，优化和升级下层业务模型                  │
│   └─ 高层视角：守住上层、监控局部、严格管理                 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**为什么成熟公司很难允许"背道而驰"的事？**

在滴滴、美团这样的成熟公司，商业模式和商业经营逻辑的结构非常明确。这种状态下，公司很难允许内部有人做一个与现有商业经营模型、商业模式完全背道而驰的事情。

**原因分析**：
1. 成熟公司的商业模式已经过验证和稳定运行
2. 商业经营逻辑已经形成清晰的成本结构和利润结构
3. 与现有模型背道而驰的事会破坏已有的稳定系统
4. 成熟公司的资源投入需要与现有模型对齐

**成熟公司的处理方式**：

如果真有人想做与现有模型背道而驰的事：
- **不如让这个人去外部创业**
- **公司可以投资他**
- **不需要在内部单独做**

这样做的好处：
- 保护公司现有商业模式的稳定性
- 给创新者更大的自由度
- 公司通过投资方式分享收益，而不是承担全部风险
- 避免内部资源分散和冲突

### 1.3.2 商业经营模型：财务视角

**基本公式**：
```
收入 - 成本 = 利润
```

**不同业务类型的具象化示例**：

**1. 电商业务**：
```
毛利 = 流量 × 转化率 × 客单价 - 获客成本 - 履约成本
净利润 = 毛利 - 人力成本 - 运营成本 - 营销成本
```

**2. SaaS业务**：
```
毛利 = 客户数量 × ARPU（单客平均收入）- 获客成本
净利润 = 毛利 - 服务成本 - 研发成本 - 销售成本
```

**3. 广告业务**：
```
毛利 = DAU × 广告位 × eCPM / 1000 - 流量成本
净利润 = 毛利 - 内容成本 - 技术成本 - 运营成本
```

**4. 教育培训业务**：
```
毛利 = 线索数 × 转化率 × 课单价 - 销售提成
净利润 = 毛利 - 讲师成本 - 运营成本 - 渠道成本
```

> **核心要点**：不同业务类型的"商业经营模型"公式差异很大，但核心都是理解收入来源、成本结构和利润驱动因素。

---

#### 中早期公司的简化财务公式

```
收入公式：
• 收入 = 流量 × 转化率 × 客单价

成本公式：
• 成本 = 营销成本 + 人力成本 + 运营成本

利润公式：
• 利润 = 收入 - 成本

应用要点：
• 业务未定型时，无法建立精确的财务模型
• 高层只能做粗放式管理：看花了多少钱、有没有收入
• 需要业务模型先成形，才能推导出精确的财务模型
```

**为什么业务未定型无法建立精确模型？**

因为中早期公司的业务形态、成本结构、收入来源都在快速变化，建立模型的前提是业务相对稳定。如果业务还在不断调整方向，建立出来的模型就已经过时了。这就像试图给一个正在快速成长的孩子做"未来五年衣服尺寸规划"——你做的规划永远跟不上实际变化的速度。

只有在业务相对稳定、数据积累到一定程度后，才能建立有指导意义的精确模型。

---

**成熟公司的分板块财务模型**：

以新东方等成熟公司为例，高层会按业务板块来管理：

```
分板块价值评估：
• 每个板块价值 = 客户数量 × ARPU（单客平均收入）- 服务成本

增长公式：
• 增长 = 新客增长 + 老客复购 - 客户流失

总收入公式：
• 总收入 = Σ（各板块收入）

成本管理：
• 毛利 = 收入 - 直接成本（生产/交付成本）
• 业务利润 = 毛利 - 营销/销售成本
• 经营净利润 = 业务利润 - 固定成本（房租/行政/税收）

应用要点：
• 成熟公司高层会牢牢把握商业模式和商业经营模型
• 往下看各板块的成本结构和利润情况
• 最多看到局部业务模型，不会深入到细节执行
```

**为什么高层最多看到局部业务模型？**

因为高层的时间精力有限，他们需要从财务模型往下推导业务模型，而不是深入到每个业务细节。高层掌握的是"从上往下"的推导逻辑：

```
财务目标（收入、利润、增长）
    ↓ 推导
整体业务模型（流量×转化×客单价等）
    ↓ 推导
局部业务模型（渠道运营、用户运营、内容运营等）
    ↓ 具体执行
```

具体细节由中层和执行层负责，高层不需要也不应该陷入细节。

---

**跑通一个模型需要多长时间？**

根据课程经验，跑通一个业务模型至少需要几个月时间：
- 需要验证基本假设是否成立
- 需要收集足够的数据样本
- 需要反复试错和调整
- 需要形成可复制的标准流程

**注意**：这个过程往往需要**反复试错多次**，可能跑三五个模型才能找到一个靠谱的方向。每次试错都需要几个月时间，所以中早期公司的核心能力就是"快速试错、快速迭代"。

这也解释了为什么中早期公司需要动态思考，而成熟公司更倾向于稳定。

## 案例分析

### 案例：从运营专员到运营总监的转变

**背景**：一位运营专员工作3年，负责文案撰写和数据统计

**问题**：
- 每天忙于执行工作
- 不理解为什么要做这些工作
- 职业发展遇到瓶颈

**转变过程**：

1. **建立全局视角**：了解公司业务模型
2. **理解运营逻辑**：理解运营工作的本质
3. **关注更高层次**：从执行到策略

**结果**：
1年后晋升为运营经理，2年后成为运营总监

**分析要点**：

1. **不要只做执行**：要理解为什么做
2. **建立全局观**：理解公司整体运作
3. **提升思维层次**：从战术到战略
