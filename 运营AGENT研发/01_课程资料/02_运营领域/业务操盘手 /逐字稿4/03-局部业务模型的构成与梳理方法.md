## 本章导学

### 核心目标

本章帮助学员理解局部业务模型的概念，掌握如何梳理和优化局部业务模型。核心目标是能够将全局业务模型拆解为具体的局部模块，并针对每个模块进行优化。

## 详细内容

### 3.1 局部业务模型的构成

#### 局部业务模型的定义

**概念**：局部业务模型是全局业务模型中的某一个核心业务环节或模块，单独拿出来进行梳理和分析的模型。

```
┌─────────────────────────────────────────────────────────────┐
│                 全局业务模型 vs 局部业务模型                   │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   全局业务模型                                               │
│   ┌─────────────────────────────────────────────────────┐   │
│   │  流量 → 转化 → 交付 → 复购 → 推荐                  │   │
│   └─────────────────────────────────────────────────────┘   │
│            ↓         ↓        ↓        ↓        ↓          │
│        局部模型  局部模型  局部模型  局部模型  局部模型  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 局部业务模型的三要素

| 要素 | 说明 | 示例 |
|------|------|------|
| 输入(Input) | 业务资源、生产资料 | 预算、用户名单、渠道 |
| 过程(Process) | 业务动作、转化流程 | 触达、咨询、成交 |
| 输出(Output) | 业务价值、结果指标 | 付费用户、转化率 |

#### 局部业务模型的类型

1. **流量获取型**：负责获取潜在用户
2. **转化销售型**：负责将潜在用户转化为付费用户
3. **服务交付型**：负责提供产品/服务
4. **用户运营型**：负责用户活跃、留存、复购

### 3.3 如何从0搭建一个局部业务模型

#### 搭建原则

1. **先验证再放大**：小范围测试，验证模型可行后再规模化
2. **标准化流程**：确保每个环节都有明确的标准
3. **数据驱动**：用数据指导优化方向
4. **持续迭代**：根据反馈不断调整优化

#### 搭建步骤

```
┌─────────────────────────────────────────────────────────────┐
│               从0搭建局部业务模型5步法                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  第1步：明确目标和定位                                       │
│          - 明确要解决什么问题                                │
│          - 确定在整体业务中的位置                            │
│                                                             │
│  第2步：设计业务闭环                                        │
│          - 设计输入-转化-输出流程                            │
│          - 识别关键动作和节点                                │
│                                                             │
│  第3步：定义衡量标准                                        │
│          - 设定核心业务指标                                  │
│          - 确定数据采集方式                                  │
│                                                             │
│  第4步：小范围测试                                          │
│          - 选择试点进行验证                                  │
│          - 收集反馈和数据                                    │
│                                                             │
│  第5步：优化迭代                                            │
│          - 分析数据识别问题                                  │
│          - 优化流程和策略                                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 案例分析

### 案例：客服团队的局部业务模型

**背景**：
某在线教育公司客服团队负责处理用户咨询和售后问题。

**原始状态**：
- 2-3人团队
- 用户来电/留言 → 及时回复
- 简单记录，无标准化流程

**问题**：
- 业务量增长10倍后，无法支撑
- 服务质量参差不齐
- 难以评估工作价值

**搭建局部业务模型**：

```
输入 → 流程 → 输出
  │       │       │
  │   ┌─────────┐ │
  │   │ 需求分类 │ │
  │   └─────────┘ │
  │       ↓       │
  │   ┌─────────┐ │
  │   │ 标准回复 │ │
  │   └─────────┘ │
  │       ↓       │
  │   ┌─────────┐ │
  │   │ 问题升级 │ │
  │   └─────────┘ │
  │       ↓       │
  │   ┌─────────┐ │
  │   │ 满意度  │ │
  │   └─────────┘ │
  ↓    用户满意
```

**关键指标**：
- 响应时间
- 首次解决率
- 用户满意度
- 人效比

**优化结果**：
- 响应时间从30分钟降至5分钟
- 首次解决率提升至80%
- 用户满意度达到95%

## 核心概念表格

| 概念 | 定义 | 要点 |
|------|------|------|
| 局部业务模型 | 某一环节/模块的业务模型 | 明确的输入、过程、输出 |
| 流量获取型 | 获取潜在用户的业务模块 | 关注获客成本和数量 |
| 转化销售型 | 将潜客转化为付费用户的模块 | 关注转化率和成交额 |
| 服务交付型 | 提供产品/服务的模块 | 关注交付效率和质量 |
| 用户运营型 | 负责用户活跃和复购的模块 | 关注留存和复购率 |

## 本章小结

1. **局部业务模型是全局模型的组成部分**
   - 有明确的上下游关系
   - 有特定的输入和输出

2. **梳理方法6步法**
   - 明确环节、界定输入、梳理流程
   - 定义目标、识别指标、建立闭环

3. **从0搭建5步法**
   - 明确目标、设计闭环、定义标准
   - 小范围测试、优化迭代
