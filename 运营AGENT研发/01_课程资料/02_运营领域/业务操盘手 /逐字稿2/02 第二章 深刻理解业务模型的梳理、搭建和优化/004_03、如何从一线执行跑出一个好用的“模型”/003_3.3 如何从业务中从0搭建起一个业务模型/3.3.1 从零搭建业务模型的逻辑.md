# 3.3.1 从零搭建业务模型的逻辑

我们看完例子之后，可以来查看所谓跑通一个模型的真相，它的真相约本质上是这样一个过程。首先你从你一边在执行，一边可能在要从商业逻辑或业务逻辑里边进行思考，或者是在一些混乱的摸索当中，有些时候甚至说线索都不知道从来的，你要在一些混乱的摸索当中找到一些线索，就线索会让你觉得说似乎这个事可以干，事干下去它能持续的有机会带来增长或带来这种商业价值，

找到线索之后，你脑海中就形成某个假定的模型，然后形成模型之后，你随后就要通过行动持续去验证模型的有效性，如果验证的结果是yes，模型就成立了，如果验证的结果是no，那模型可能不成立，你得重新回过头来再去做思考，就像我们刚才讲到三节课b端的这样的一个这种业务，它的交付这块的模型是说我们经历了十几家用户的验证，初步就成型了，然后我们就让它稳定下来，

后边去优化它就行了，但是销售这块的它的业务的模型，我们发现我们初期有一个版本上线之后发现不ok，所以我们就回过头来再去优化，再去思考再去调处理，约是这么一个过程。

一个业务处理体模型的成型一定是不断的动态演化而成的，不说初期我经由说例如假设老黄教了一套方法对标梳理出来，所以我的业务模型就该长这样的，他就答案就不能变了，我觉得不是逻辑，一个业务处理体模型成型一定是不断动态演化而成的，我们这门课教给你的是说思考的方法和思考的逻辑，

但并不是说你这么思考完之后，我有个固定的答案放在那，答案就不要变了。

对不是逻辑，我们第一章也在反复强调这件事，所以各位一定要有认知以及跑通任何一个局部，至少都需要几个月的时间，所以很可能各位然后上完这门课之后去检验这门课的这种成效和消化，这门课程里边内容带给你的帮助，很可能需要你在你自己公司的这种业务里头

然后上完课之后有些消化有些思考，然后再做一些行动，不断去检验完了之后看效果好还是不好。在持续内化它需要的是这样一个过程，所以这是所谓跑通一个模型的真相。

但是我们给各位揭露了一下，说一个模型在家公司里面是怎么跑通的，对我们揭示一下真相，各位知道说过程原来长这样，但是到底该怎么跑通，所以要想自己可去说跑通这样一个模型，你也得约有一个基本的思考逻辑，所以到底如何从业务当中从零开始能搭建出一个模型，我们有两种逻辑，然后第一种逻辑叫做我先在外部找到一个可参照的这种模型，对我的业务对我当下解决问题可参照的模型，然后再看看它是否有机会能帮助我形成稳定的input output。

这是第一种思考逻辑，在思考逻辑下说然后我先来去想说我当前在我的公司的业务里边，我当前到底要解决什么问题，我当前要解决的举例子假设一个说通过社群，因为第一章结束之后也有人提到

我的社群的什么转化作用不因此，假设我要解决的说我要通过社群来去帮助获取更多的这种销售线索，这么一个问题，对我先要去看我要解决什么问题，它背后本质是个啥，把它界定清楚，然后再看行业里边，然后在不同的公司里边有没有成熟可参照的模型。

想这件事儿，如果有，好别人已经跑通了，别人在他的幽默性里头，如果我判断他公司的业务的这种情况，跟我这家公司的业务的情况相差不会太远，我基本可以理解为说这件事应该放到我业务里头，还是有机会帮助形成稳定的input output的，这时候我就把它拿过来，套用模型结合我自己当前的业务问题进行优化加工，形成类似sop工作体系等等这样一系列东西，然后形成东西之后，我们再去跑，再去验证模型在我业务里头它的有效性约是怎样的，所以这是第一种思考逻辑，先从外部去看

我有个问题要解决，问题我也不知道怎么办，但我先从外部去看，拿到一个参照拿到我真实业务里边来，然后后边我再去跑，这是第一种思考逻辑。

那么第二种思考逻辑，可能说叫做我要自己先能做到，我自己亲自上一线干。，然后我把一件事儿干到例如有个七八分了，我自己能干到说这件事我来干，能做到有稳定的因葡萄的葡萄了，但是别人干不一定

这时候我再来看怎么把我的能力复制给其他人，或者我找到的一些规律抽象出来，让其他人呢可遵循可参照，所以在这种思考逻辑下，它的过程约是这样的，说某件事儿有可能在我这儿已经较为趋于稳定了，我可能就得查看这件事，我能干的就七七八八了

我能不能抽象出来一些规律和操作标准，就好比像三节课的这种公众号，最早我自己一个人在写，我就发现说似乎这里面然后我们持续做好某几类的这种内容

然后通常公号一周的这种流量或者阅读量或它的这种粉丝增长就十分稳定，十分可驱动，然后我们就要抽象出来一些规律，我们都去想说到底我一周做些几种类型的内容，然后怎么来去做，这些内容，它应该符合什么标准，更可驱动我公号的这种粉丝的增长。

我要去抽象这件事情，它的规律是什么，它的操作标准和工作方法是什么，然后抽象完了之后，我就会形成一个说 Sop或初步的这种模型之类的。然后同理抽象出来之后，这时候我需要去把处理件事情放在其他人身上，去验证模型的有效性，所以这是说关于在业务当中如何从0搭建出来一个模型，它的两种思考逻辑，然后这两种思考逻辑我们约分别都讲完了，但各位需要知道这两种思考逻辑不一定是说两者之间是绝对对立或者是孤立的，反而更多的时候，我们如果要解决一个复杂的问题，我们在业务里头去搭建一个有点复杂的这样一种业务模型，有些时候两个逻辑可能得交替着综合使用，才能帮助你能从零搭建起来一个模型，更多时候是这样的。
