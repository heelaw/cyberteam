# 2.4 给你的几点TIPS

## 本节概要

本节讲解**简单业务模型vs复杂业务模型**的区别，分析复杂业务模型的高风险和长周期，并通过教育公司校园团队的失败案例，给出搭建业务模型的**两个核心建议**。

---

## 一、简单业务模型 vs 复杂业务模型

### 1.1 简单业务模型

```
┌─────────────────────────────────────────────────────────────┐
│         简单业务模型                                          │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│   特征1：单线程                                               │
│   ┌─────────────────────────────────────────────────┐        │
│   │  从用户体验流程来看：单线程                            │        │
│   │  从操作流程来看：单线程                               │        │
│   └─────────────────────────────────────────────────┘        │
│                                                              │
│   特征2：变量少                                               │
│   ┌─────────────────────────────────────────────────┐        │
│   │  涉及的变量非常少                                    │        │
│   └─────────────────────────────────────────────────┘        │
│                                                              │
│   特征3：分支少                                               │
│   ┌─────────────────────────────────────────────────┐        │
│   │  中间的线条、分叉分支也很少                            │        │
│   └─────────────────────────────────────────────────┘        │
│                                                              │
│   案例：工具产品                                              │
│   ┌─────────────────────────────────────────────────┐        │
│   │  工具产品的业务模型超级简单：                            │        │
│   │                                                 │        │
│   │  外围流量引进来 ──▶ 用户用工具 ──▶ 喜欢不喜欢          │        │
│   │         │                  │                       │        │
│   │         │                  ▼                       │        │
│   │         │              下次再来用                    │        │
│   │         │                  │                       │        │
│   │         │                  ▼                       │        │
│   │         │              继续用养成习惯                 │        │
│   │         │                                          │        │
│   │         └────────────────────────────────────────  │        │
│   │                                                 │        │
│   │  然后没了                                          │        │
│   │  这是一个工具产品它的整个的业务模型非常之简单             │        │
│   └─────────────────────────────────────────────────┘        │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 印证举例

> 逐字稿原文：
> "简单的业务模型 基本上就是说单线程 就基本上比如说从用户的体验流程 还有从我的操作来讲 就单线程 变量少 有非常少的变量 中间的线条 中间的分叉分支之类的 也很少"
>
> "比如说明显 比如说一个工具产品 它的业务模型就超级简单 非常之简单"
>
> "外围流量引进来 用户用这个工具 用完之后喜欢不喜欢 填好 下次再来用 然后继续用 养成习惯 然后没了"

### 关键要点总结

- **单线程**：用户体验流程和操作流程都是单线程
- **变量少**：涉及的变量非常少
- **分支少**：中间的分叉分支很少
- **典型案例**：工具产品——流量 → 使用 → 习惯

### 1.2 复杂业务模型

```
┌─────────────────────────────────────────────────────────────┐
│         复杂业务模型                                          │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│   特征1：线程多                                               │
│   ┌─────────────────────────────────────────────────┐        │
│   │  - 有前端                                         │        │
│   │  - 有后端                                         │        │
│   │  - 还有中间可能还有辅助配套机制                       │        │
│   │  - 还有辅助系统                                    │        │
│   └─────────────────────────────────────────────────┘        │
│                                                              │
│   特征2：人和人交互多                                           │
│   ┌─────────────────────────────────────────────────┐        │
│   │  整个业务流程里涉及有N多人和人的交互                   │        │
│   │  以及有N多信息的交互                                 │        │
│   │                                                 │        │
│   │  人和人的互动本身就是一个非常大的变量：                   │        │
│   │  - 人都是不一样的，分三六九等                         │        │
│   │  - 假设是ABC三个人同时对一个客户打交道                  │        │
│   │  - 他们表达的习惯、沟通的语气和口吻不一样                 │        │
│   │  - 会影响到客户最后的状态和感受和判断                    │        │
│   └─────────────────────────────────────────────────┘        │
│                                                              │
│   特征3：信息交互复杂                                           │
│   ┌─────────────────────────────────────────────────┐        │
│   │  有N多的信息交互                                     │        │
│   │  - A信息会制约着B行动                               │        │
│   │  - C信息会制约着D行动                               │        │
│   │  - 有很多这样的线条和机制在整个业务模型当中              │        │
│   └─────────────────────────────────────────────────┘        │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 印证举例

> 逐字稿原文：
> "复杂业务模型是说 第1个线程多 比如说有前端 有后端 甚至还有中间 可能还有个什么 我们自己业务的这种同学 可能他要再有的一条 这种辅助配套的一些机制 和一些辅助的一些这种系统"
>
> "第2个就是说 有n多 中间就整个我这个业务的模型 或中间整个的业务流程里边 涉及到有n多人和人的交互 以及有n多信息的交互"
>
> "然后有N多的信息交互 也就意味着说 这里边可能是说 A信息会制约着B行动 对吧 C信息会制约着D行动"

### 关键要点总结

- **线程多**：前端、后端、中间配套机制和系统
- **人交互多**：N多人和人的交互，人本身就是大变量
- **信息交互复杂**：A信息制约B行动，C信息制约D行动

### 1.3 简单 vs 复杂对比

```
┌─────────────────────────────────────────────────────────────┐
│         简单业务模型 vs 复杂业务模型                             │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│   简单业务模型：                                               │
│   ┌─────────────────────────────────────────────────┐        │
│   │  ✅ 单线程                                        │        │
│   │  ✅ 变量少                                        │        │
│   │  ✅ 分支少                                        │        │
│   │  ✅ 容易跑通                                      │        │
│   │  ✅ 一个月内就能出答案                              │        │
│   │  ✅ 风险低                                        │        │
│   └─────────────────────────────────────────────────┘        │
│                                                              │
│   复杂业务模型：                                               │
│   ┌─────────────────────────────────────────────────┐        │
│   │  ❌ 线程多                                        │        │
│   │  ❌ 人和人交互多（大变量）                            │        │
│   │  ❌ 信息交互复杂                                    │        │
│   │  ❌ 周期会比较长                                    │        │
│   │  ❌ 相对来讲风险也会更大                             │        │
│   │  ❌ 难度也会更大                                    │        │
│   └─────────────────────────────────────────────────┘        │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 印证举例

> 逐字稿原文：
> "越复杂的业务模型 要想跑通的话 它的挑战和风险 往往是更大的 需要的时间也更长"
>
> "一般来讲一个简单业务模型 它比较容易跑通的 基本上就是我如果一个业务模型是足够简单的 它可能就是一两个线程 中间分差也很少 基本上我牵扯出来跑 大概一般来说 一个月之内我就能出答案 说这个模型OK还是不OK"
>
> "所以复杂业务模型要跑起来 周期会比较长 相对来讲风险也会更大 难度也会更大"

### 关键要点总结

- **简单模型**：单线程、变量少、分支少、一个月内出答案
- **复杂模型**：线程多、人交互多、信息复杂、周期长、风险大、难度大

## 二、复杂业务模型典型案例

### 2.1 案例一：沃尔玛供应链系统

```
┌─────────────────────────────────────────────────────────────┐
│         案例一：沃尔玛供应链系统（超复杂）                          │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│   系统结构：                                                   │
│   ┌─────────────────────────────────────────────────┐        │
│   │                                                  │        │
│   │   总部 ────────────────────────────────────────┐ │        │
│   │     │                                            │ │        │
│   │     ├─────────┬─────────┬─────────┬─────────┤  │ │        │
│   │     │         │         │         │         │  │ │        │
│   │     ▼         ▼         ▼         ▼         │  │ │        │
│   │  配送中心   仓库      门店1     门店2  ...   │  │ │        │
│   │     │         │         │         │         │  │ │        │
│   │     └─────────┴─────────┴─────────┴─────────┘  │ │        │
│   │                          │            │        │ │        │
│   │                          └────────────┘        │ │        │
│   │                                                 │ │        │
│   └─────────────────────────────────────────────────┘        │
│                                                              │
│   信息流（复杂）：                                              │

│   │     不然总部就没办法去下达相应的信息                       │        │
│   │                                                 │        │
│   │  ④ 配送中心物流和补货信息：                             │        │
│   │     也是总部来去下达                                   │        │
│   │                                                 │        │
│   │  中间你会发现信息流非常多                               │        │

│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 印证举例

> 逐字稿原文：
> "比如说是沃尔玛的 整个的这种供应链系统 对吧 你会发现 从它的总部配送中心 仓库 还有N多的门店 对吧 它首先就分成这么4个角色 这4个角色之间 有非常多的这种信息流"
>
> "然后仓库可能需要去补货 补货的信息 需要由总部来去下达 对吧 然后门店也需要补货 门店补货信息 要反馈给到总部"
>
> "然后配送中心这 又有一个动态的这种流动仓 所以配送中心 在流动仓的库存信息 是要总部及时可以去查到的 不然总部就没办法去下达相应的 信息了"
>
> "所以假设是说你想象一下 沃尔玛本身是花了100多年 对吧 它形成这么一套 非常完善的 这样一套供应链管理的机制"

### 关键要点总结

- **系统结构**：总部、配送中心、仓库、N多门店，4个角色
- **信息流**：仓库补货、门店补货、流动仓库存、物流补货
- **成型周期**：沃尔玛花了100多年才形成完善的机制
- **难度极大**：从零开始可能需要10年8年

## 三、复杂业务模型的风险和挑战

### 3.1 复杂业务模型的风险评估

```
┌─────────────────────────────────────────────────────────────┐
│         复杂业务模型的风险和挑战                                 │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│   风险1：周期长                                               │
│   ┌─────────────────────────────────────────────────┐        │
│   │  简单模型：一个月内就能出答案                          │        │
│   │  复杂模型：周期会比较长                               │        │
│   │  - 沃尔玛供应链：100多年                             │        │
```

## 本节总结

本节讲解了**简单业务模型vs复杂业务模型**的区别，并通过失败案例给出了**两个核心建议**：

**简单 vs 复杂**：
- **简单模型**：单线程、变量少、分支少、一个月内出答案、风险低
- **复杂模型**：线程多、人交互多、信息复杂、周期长、风险大、难度大

**复杂业务模型的案例**：
1. **沃尔玛供应链**：4个角色、复杂信息流、100多年成型
2. **美团外卖**：表面简单、后台复杂、5方信息互动+7大运营机制
3. **教育行业**：教研教学用户三种角色、互动节点多、后台体系复杂

**失败案例**：
- **错误**：同时做4件待验证的事，没有定量全是变量
- **结局**：跑了三四周全是坑，跑不出来，一地鸡毛
- **原因**：不够简单、链条太多、变量太多

**两个核心建议**：
1. **用别人跑通的模型**：降低风险、缩短周期、提高成功率
2. **不要在假想中把模型想得很复杂**：从简单开始，先跑通最简单的版本
