## 详细内容

### 一、局部业务模型的定义

#### 1.1 什么是局部业务模型？

```
┌─────────────────────────────────────────────────────────────┐
│              全局业务模型 vs 局部业务模型                      │
├─────────────────────────────────────────────────────────────┤
│                                                               │
│  全局业务模型：                                                │
│  └─ 整个公司业务如何运转、全链路的业务逻辑                         │
│                                                               │
│  局部业务模型：                                                │
│  └─ 全局业务模型中的某一个核心业务环节或某一个模块                   │
│                                                               │
│  关系：                                                       │
│  局部业务模型 = 全局业务模型的某一个组成部分                        │
│  把某个模块单拿出来，梳理清楚这块的业务模型                         │
│                                                               │
└─────────────────────────────────────────────────────────────┘
```

### 印证举例

> 逐字稿原文：
> "接下来的话 我们进入到另外一个部分 就是局部业务模型 就像我们说过的局部业务模型 它的理解啊 可能就更加的简单了 其实它就是我们全局业务模型当中的某一个核心业务环节或者某一个模块 对吧 我们对这个模块单拿出来 把它去梳理清楚 说这块的业务模型可能会是怎样的 其实就OK了"

### 关键要点

- 局部业务模型是全局业务模型的组成部分
- 把某个核心业务环节或模块单独梳理
- 本质上还是业务模型，有稳定的input和output

> "所以如果你所负责的这个局部业务哈 大概是这么一个业务的话 其实你会发现 它当中可能一个比较重要的事 还是我们刚才说的 你得把你所负责的这块业务 它的input output是什么界定清楚 界定清楚完了之后 中间涉及到到底有多少个角色 这角色之间的关系和分工是怎样的 你可能要优先先把这个层次先梳理清楚 然后再来看他们之间的有一些协作流程啊之类的 怎么来去串联 可能会更清楚一些"

#### 14.4 复杂业务的处理原则

```
┌─────────────────────────────────────────────────────────────┐
│         复杂业务的处理原则                                   │
├─────────────────────────────────────────────────────────────┤
│                                                               │
│  如果整个业务中涉及到的角色或用户类型太多：                            │
│                                                               │
│  那一定要：                                                     │
│  ├- 把用户类型摘清楚                                             │
│  ├- 他们之间有没有关系                                            │
│  ├- 对每类用户的协作流程和方式是怎样的                                   │
│  └- 对哪些人会有什么配套的约束机制和运营策略                               │
│                                                               │
│  划清楚之后：                                                  │
│  └- 你的局部模型就会展现得比较清晰                                    │
│                                                               │
└─────────────────────────────────────────────────────────────┘
```

### 印证举例

> 逐字稿原文：
> "所以这是我们在这儿给到大家的一个反馈吧 对于这些啊 有一点看起来有点复杂的局部业务模型来讲 它的复杂往往会在于说 就是我整个这个业务里边涉及到的角色或是到的用户的类型太多了 那么一定要把这用户的类型摘清楚 他们之间有没有关系 我对每类用户对不同的人 对吧 他们的协作的流程和方式是怎样的 对哪些人会有什么配套的这种约束机制和我们的运营策略划清楚 那划清楚之后 你的局部的模型也会展现的就比较清晰"
