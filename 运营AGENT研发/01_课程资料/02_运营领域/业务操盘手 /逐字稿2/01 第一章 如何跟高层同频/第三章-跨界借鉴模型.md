# 3.1 快速跨界找到可借鉴业务模型的基本思路

## 学习目标

- 理解跨界借鉴业务模型的基本逻辑
- 掌握三步法：检索行业 → 检索问题 → 跨领域借鉴
- 能用三步法分析自己业务的跨界借鉴方向

## 跨界借鉴的核心逻辑

在业务当中搭建一个模型时，有一个重要的思考逻辑叫做：**先找到外部的模型，再看它能否帮助你获得稳定的input和output。**

### 基本思路

```
┌─────────────────────────────────────────────────────────────────┐
│              跨界借鉴业务模型的三步法                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  【步骤1】检索高度相似的行业和业务模块                          │
│           找到与你的业务具有高度相似性甚至完全雷同的行业       │
│           找到对应模块下有可参照模型的几家公司                    │
│           拿过来看能不能帮我解决问题，就直接学习借鉴             │
│                                                                 │
│  【步骤2】如果有问题未解决，检索相似的同类问题                   │
│           定义你要解决问题的性质                                 │
│           在其他公司里找到与你要解决的问题有相似性的问题         │
│           看人家是怎么解决的，找这个参照的方案                   │
│                                                                 │
│  【步骤3】根据核心受众和产品属性跨领域借鉴                       │
│           优先选择核心受众和产品属性跟你的业务更相近的成熟业务   │
│           从中找到可借鉴的思路和模型                              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 常见行业和业务模块

**常见行业**：教育、金融、医疗、文娱、零售、电商、游戏等

**常见业务模块**：增长获客、渠道代理商管理、供应链、流程出货、服务交付、销售转化等

## 案例分析1：三节课助教运营

### 问题定义

三节课要搭建一个助教运营的运营体系。助教运营本质上涉及两类问题：
1. 教学管理的问题
2. 教师培养/培训的问题（助教一定程度上承载辅导老师的角色）

### 步骤1：检索高度相似的行业和业务模块

在教育行业里找相似模块：
- 教学管理的模块
- 教师管理、教师培训的模块

**收获**：拿到成熟的解决方案（体系、机制、流程），可以借鉴

### 步骤2：检索相似的同类问题

借鉴完后发现还有问题没解决：
- 助教是否应该是一个高质量的社交圈
- 人员管理问题
- 供应链管理问题
- 服务品控问题
- 用户激励体系问题
- 优质社交圈层如何维系

**收获**：找到这些问题的参照

### 总结

找到足够有价值的建议和信息的思路：
1. 检索跟业务高度相似的行业和业务模块
2. 找相似的同类问题及其解决方案
3. 自己判断哪些用，哪些不用

## 案例分析2：爱彼迎房东运营

### 问题定义

NB的房东运营

### 步骤1：检索高度相似的行业和业务模块

发现跟两个行业相关：
1. **房地产中介行业**：业主管理
   - 怎么签约
   - 怎么持续沟通互动
   - 怎么定期维系，保证房源稳定
   - 怎么预防房东变故导致房源不稳定

2. **酒店管理行业**：服务交付体系
   - 参照酒店服务交付标准
   - 要求房东房源符合12345这5个标准

### 步骤2：检索相似的同类问题

借鉴完后发现还有问题没解决：
- 房东们本身应该是个社群，彼此应该有交流（社区运营问题）
- 房东们与我们之间的关系要更紧密
- 部分房东参与业务经营和服务（用户激励体系问题）
- 不同地区房源分类（供应链管理问题）

### 总结

找到方向：社区运营、用户激励、供应链管理

## 案例分析3：K12在线英语外教——跨界借鉴的坑

### 背景

某K12在线英语外教公司，业务模型是：
- 增长方面：电商转化路径 + 社群转化路径 + 续费
- 供给交付方面：外教供应链 + 教学服务 + 班主任客服

### 问题

虽然招到很多外教，一对一教学服务一段时间内不太稳定。有的外教做得非常好，有的体验或用户反馈不太好。

### 要解决的核心问题

怎么保证这些外教的教学服务能够稳定交付？

### 问题出在哪里：没有充分定义问题

这位操盘手的思路：

1. **量化评价**：老师评价、用户怎么给老师评价、怎么给老师去评价
2. **数据标准**：3分之4以下需要提高
3. **教学SOP**：什么时间节点必须say hi、什么时候要用什么教具

### 跨界借鉴的行业

思路很散，找了：
- 酒店的服务
- 滴滴司机的服务
- 家庭保姆的服务

### 陷入困惑的三个问题

1. **横向借鉴的行业是否合理**：酒店、家庭保姆、滴滴司机，是不是合理？
2. **借鉴的业务OK不OK**：怎么找到跟业务逻辑相符、应该借鉴的行业？
3. **参考价值有限**：列了一堆信息，最后能给到参考的实际上是非常有限的
4. **扩展思路有限**：能想到的对外可借鉴的行业很有限

### 问题根源

> 但凡你开始成为一个操盘手之后，很多问题上你一定都要先定义问题，再来看如何解决问题。而不是说问题没有定义清楚就直接扎到细节里去了。

**具体问题**：
1. 没有充分明确要解决问题的实质
2. 交付的过程、标准、结果不清晰，还是中间某些环节不OK？
3. 不太清楚需要找到什么样的模型来借鉴
4. 直接进入非常多的细节和SOP里边去

### 正确的思考逻辑

#### 第一步：明确要解决问题的实质

**交付的本质是什么**？

> 交付本质上是针对一项服务，要解决我们这项服务交付给用户的成果或最终要给到用户的一个稳定预期究竟是什么。

交付意味着用户买了你的服务和产品，他有一个稳定的预期。这个预期如果最后达成了，用户就开心。但如果这个预期没有达成，用户就不开心。

从交付层面来讲，要帮助用户界定和明确他应该有的预期到底是什么。

**操盘手的核心能力**：对业务的深刻理解。这件事一定需要时间，需要大量的思考、实践，不可能一蹴而就或短时间内塑成。

但有了业务模型意识后，提升速度和对业务理解的深度可能会不断得到加强和放大。

#### 第二步：思考同行业在同一问题上的答案和解决方案

**教育产品的交付类型**：

1. **目标导向型/结果导向型交付**
   - 给到用户一个明确的结果预期
   - 通过享受服务或上课程，最后一定拿到一个结果
   - 例子：四六级考试、注册会计师考试、托福雅思培训
   - 通过考试或拿到证书就是最终交付

2. **体验保障型交付**
   - 不承诺最后一定能带走什么结果或达成什么目标
   - 保证在服务过程当中有123456这些节点，能得到非常好的体验
   - 例子：传统商学院
     - 不承诺一定解决问题
     - 但保证和很多优质高端人士一起出游、聚餐、社交活动
     - 过程超开心，身心愉悦
   - 虽然不一定解决问题，但体验超棒

**K12在线英语属于哪种类型？**

- 不属于结果导向型交付
- 属于体验保障型交付

**为什么？**
- 很难保障效果、保障学习结果（小孩子语言能力得到什么提升）
- 无法评估
- 本质上是：外教陪着小孩子通过英语在线互动娱乐玩耍
- 在玩的过程中，小孩子变得开心，说话能力得到提升（但也许年龄增长也会提升）
- 核心要解决的是：小孩子学习过程中的体验一定要超棒
- 不管给到孩子还是给妈妈的体验都要超棒

#### 第三步：跨领域借鉴

**关键认知**：必须根据核心受众和产品的属性来考虑跨领域借鉴，而不是随意拿一家公司。

**属性梳理**：
1. **使用者和付费者分离，但同属一个家庭**
   - 妈妈或爸爸付钱，小孩子来使用
2. **高单价，面向亲子相关需求**
3. **一对一服务，产品具有排他性**
   - 买了VIPKID的产品，很难再购买其他培训机构的英语在线学习产品
4. **Qtime式的在线互动体验**
   - 本质上是陪伴小孩子度过时间的互动体验式产品

**根据属性找可借鉴对象**：

1. **使用者跟付费者分离，同属一个家庭**
   - 老人看护服务
   - 如何处理这部分，让两方都满意

2. **高单价，满足亲子互动或看护需求**
   - 亲子营地教育产品
   - 怎么设计让亲子双方互动，增强体验

3. **一对一且产品具有排他性**
   - 医院的一对一陪护服务
   - 怎么设计服务，保证体验极大增强

4. **Qtime式的在线互动体验**
   - 在线主播
   - 有什么手段、套路调动屏幕面前对象的互动，调控过程

### 总结

通过这个K12在线英语外教的案例，深刻意识到：
1. 对外跨行业借鉴业务模型时的要点
2. 深刻理解业务的重要性
3. 深刻理解一项业务或一个行业需要时间，需要商业和业务上大量的实践
4. 不要着急，不要一味追求速成

## 增长获客类模型

### 基础法则：营销转化漏斗

所有业务几乎都会需要一个营销转化漏斗，这是不可动摇的基础法则。

但不同产品的流量获取和用户转化玩法可能不一样，由此衍生出不同的模型。

### 4种常见模型

#### 1. 话题和内容流量驱动模型

**核心逻辑**：好的话题 + 对的人 = 流量

**例子**：社区产品中的内容和话题驱动增长

**基本公式**：
- 好的话题 + 对的人碰到一起 = 流量
- 输出可能是观点、洞见、八卦段子等有传播性的东西

**机制**：
1. 内容生成机制
   - 先有话题再搞人，给人提问题生成内容
   - 先有内容生成工具给用户，100个人一起聊，筛选出有传播性的内容

2. 内容分发机制
   - 用户自发分发
   - 官方对外分发

3. 分发后形成进一步引流和转化

**适用场景**：内容产品

#### 2. 品牌PR驱动增长模型

**例子**：58同城、拉勾网

**典型表现**：
- 58同城：签杨幂、铺天盖地打广告、老板姚劲波天天上节目
- 拉勾网：三天两头策划PR或事件营销（总理喝咖啡、行业大咖专访、定向抢人活动）

**核心逻辑**：对于低频、依靠流量才能盈利的业务，品牌PR驱动是典型的增长模型。

**为什么低频业务适合品牌PR驱动？**

例子：
- 58同城服务：家政、搬家、找房（几个月用一次）
- 拉勾服务：招聘（一两年用一次）

如果我突然有需求想找家政，我可能会搜"家政公司北京"。但如果我脑海中已经被植入了"58同城提供最优质的搬家服务"，我可能直接搜"58同城"。

**核心价值**：用品牌占据用户心智，让用户牢牢记住（互联网招聘=拉勾，同城服务=58），天然有巨大流量入口。

**延伸逻辑**：把品牌工作直接跟流量获取衔接，降低获客成本
- 一波品牌工作打出去，投放接着上
- 关注品牌工作后投放获客成本是否降低
- 用户对品牌越有认知和信任感，决策成本越低

#### 3. 产品体验驱动增长模型

**核心逻辑**：
1. 用户使用某个产品功能或工具
2. 思考寻找用户使用过程中可能产生社交分享动机的场景
3. 找到场景后产生社交介质促成分享

**例子**：网易云音乐

- 正常使用：听歌工具
- 场景切入：去到新地区（如丽江、大理），云音乐检测到喜欢的歌手在那里写过歌，推荐这首歌
- 如果场景抓得准且社交介质足够优质，用户听完可能分享到朋友圈

**适用条件**：如果能从场景切入找到有社交分享动机的场景，围绕场景设计社交介质促成分享，能源源不断带来新流量。

#### 4. 社交关系链驱动模型

**核心逻辑**：
- 产品有关系链或可以借助用户社交分享的关系链
- 设计利益机制、权益捆绑机制
- 促成更多用户分享

**典型例子**：三级分销、裂变分享

## 单体用户价值放大类模型

### 核心指向

提升留存、促进活跃、放大LTV

### 4种常见模型

#### 1. 用户生命周期和典型路径管理模型

**适用条件**：产品线少、用户量少

**核心逻辑**：
- 找到典型用户成长路径
- 依据路径界定用户生命周期
- 在每个节点做运营

**应用广泛**：不同产品中应用广泛

#### 2. 用户留存模型

**核心逻辑**：通过功能服务、内容关系、互动活动、社交互动提升留存

- 功能和服务：提升功能性留存
- 内容：优质内容推送和匹配
- 社交互动：帮助用户结交有效关系，制造更多互动

#### 3. 用户分层运营模型

**典型模型**：
- 用户金字塔模型
- RFM用户分层体系

**适用条件**：海量用户、多产品线

#### 4. 用户成长体系模型

**典型体现**：
- 荣誉、身份
- 权益
- 用户互相竞争的激励
- 根据用户行为设计用户等级

**例子**：电商行业会员支撑复购率提升

## 生产类模型

### 3种常见模型

#### 1. 简单流程式生产

**典型**：传统制造业

**特点**：单线流程，可能配套很多管理机制或系统支撑

#### 2. UGC组织形式生产

**典型**：内容社区

**核心逻辑**：
1. 海量用户承担生产任务
2. 对海量用户分层、定向引导
3. 在海量内容中筛选过滤优质内容

**关键**：海量用户+海量内容→筛选过滤优质内容

**支撑体系**：
- 激励：物质激励+精神激励
- 反馈
- 引导

#### 3. PTC管理生产

**典型**：内容领域

**核心逻辑**：
1. 生产者管理/作者管理（约束、激励、监督）
2. 生产要素库（提升效率）
3. 标准化生产流程

**支撑体系**：库+流程+管理 = PTC生产体系

## 交付类模型

### 2种常见模型

#### 1. 目标达成式交付

**核心**：交付核心是实现一个目标

**例子**：
- 2B行业客户成功：到现场帮助客户完成项目交付，核心是项目顺利上线
- 三节课2B业务交付过程
- 课程教研生产到教学过程开展：目标是达成教学目标

**流程**：
- 建立课程培养目标
- 拆分典型岗位任务
- 设计章节目标、重难点
- 设计训练方式
- 用户分层
- 设计教学路径
- 教学数据实时监控、运营策略调整
- 工具和机制支撑

#### 2. 体验保障式交付

**核心**：保证用户体验的标准化和一致性

**例子**：
- 五星级酒店服务体验：
  - 进入大堂：有人快速拎包
  - 打开房门：床长什么样、灯光颜色
  - 前台check in：小姐微笑或说三句话
  - 界定5-6条有保障的体验

- 汽车行业4S店售后服务质量管控模型：
  - 监测数据：准时交车率、外返率、内返率、计时生产率、备件供给满足率
  - 背后有复杂运营体系支撑
  - 保证用户多长时间取车、得到什么服务体验

## 供给流通类模型

### 3种常见模型

#### 1. 内容生产和分发模型（成熟内容产品）

**模块**：
- 内容生产：单篇内容组织和展现
- 相关内容组织成专题
- 内容索引和标签
- 首页内容展现
- 对外传播分发

每个模块建立机制，自下而上构成关系，支撑完整的产品运行模型。

#### 2. 供应链管理模型

**SCOR模型**：
- 第一层：监测核心关键指标
  - 供应链反应速度
  - 供应链成本
  - 供应链资产利用率
- 第二层：常见流程（计划、采购、生产、配送、退货和支持）
  - 26套标准化流程
- 第三层：流程要素
  - 输入、关键元素、输出
  - 流程间稳定衔接

**QR模型**：
- 核心特点：灵活和敏捷
- 适用场景：多品类小批量的买方市场
- 核心逻辑：不储备成熟产品，储备生产要素，用户提出要求后快速抽取要素及时组装

**例子**：戴尔直销逻辑

#### 3. 餐饮行业供应链模型

**4种类型**：
- 单店采购型
- 集采集送型
- 集采分送型
- 中央厨房标准化型

## 跨界借鉴的核心要点

1. **首先定义问题**：要解决问题的实质是什么
2. **检索行业相似性**：检索高度相似的行业和业务模块
3. **检索问题相似性**：如果问题没完全解决，检索相似的同类问题及其解决方案
