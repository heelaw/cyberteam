# 2.3 如何在业务中从0搭建出一个模型

## 本节概要

本节深入讲解**跑通一个业务模型的真相**——动态演化过程，并系统梳理**两种从0搭建模型的思考逻辑**：外部参照法和个人经验法，最后通过三节课品类运营培养模型的真实案例，展示如何综合运用两种逻辑搭建复杂业务模型。

---

## 二、从0搭建模型的两种思考逻辑

### 2.1 两种逻辑概述

```
┌─────────────────────────────────────────────────────────────┐
│         从0搭建模型的两种思考逻辑                               │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│   逻辑一：外部参照法                                          │
│   ┌─────────────────────────────────────────────────┐        │
│   │  先从外部找到一个可参照的模型                          │        │
│   │  对我当下解决问题可参照的模型                          │        │
│   │  看它是否有机会帮助我形成稳定的input和output             │        │
│   └─────────────────────────────────────────────────┘        │
│                                                              │
│   逻辑二：个人经验法                                          │
│   ┌─────────────────────────────────────────────────┐        │
│   │  我自己亲自上一线干                                  │        │
│   │  把一件事干到七八分                                  │        │
│   │  我自己能干到稳定的input和output                     │        │
│   │  再把我的能力复制给其他人                              │        │
│   └─────────────────────────────────────────────────┘        │
│                                                              │
│   重要提示：                                                   │
│   ┌─────────────────────────────────────────────────┐        │
│   │  ❌ 这两种逻辑不是绝对对立或孤立的                      │        │
│   │  ✅ 更多时候需要交替综合使用                          │        │
│   │  ✅ 尤其解决复杂问题时，需要两种逻辑综合使用               │        │
│   └─────────────────────────────────────────────────┘        │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 印证举例

> 逐字稿原文：
> "所以到底如何从业务当中 从零开始能搭建出一个模型 我们有两种逻辑"
>
> "第一种逻辑叫做 我先在外部找到一个可参照的这种 模型"
>
> "第二种思考逻辑的话 可能就是说叫做我要自己先能做到"
>
> "这两种思考逻辑 不一定是说两者之间是绝对对立 或者是孤立的 反而更多的时候 其实我们如果要解决一个复杂的问题 我们在业务里头去搭建一个 有点复杂的 这样一种业务模型的话 有些时候两个逻辑 可能得交替着综合使用"

### 关键要点总结

- **逻辑一**：外部参照法——找别人跑通的模型
- **逻辑二**：个人经验法——自己先做到再复制
- **关系**：不是对立的，而是交替综合使用

## 本节总结

本节深入讲解了**跑通一个业务模型的真相**和**从0搭建模型的两种思考逻辑**：

**跑通模型的真相**：
- 动态演化过程：执行+思考 → 找到线索 → 形成假定模型 → 持续验证 → 调整优化
- 不是一开始就固定不变的，一定是不断动态演化而成的
- 跑通任何一个局部至少需要几个月时间

**两种从0搭建模型的逻辑**：
1. **外部参照法**：明确问题 → 外部寻找参照 → 套用优化 → 验证有效性
2. **个人经验法**：亲自干 → 稳定input/output → 抽象规律 → 形成SOP → 验证有效性

**两种逻辑的关系**：
- 不是绝对对立或孤立的
- 解决复杂问题时需要交替综合使用
- 外部框架 + 内部经验 → 形成完整模型

**真实案例**：
- 三节课品类运营培养模型
- 外部参照：宝洁、阿里的EDAC模型
- 内部拆解：工作职责细分
- 结合优化：形成三节课品类运营EDAC培养体系
