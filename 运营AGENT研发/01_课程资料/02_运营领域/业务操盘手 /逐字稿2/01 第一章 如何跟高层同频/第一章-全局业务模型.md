### 二、三大模型的定义

商业系统当中会存在三大常见的模型，分别叫做**商业模式**、**商业经营模型**和**业务模型**。

#### 1.1.1 业务模型的详细理解

**业务模型**整体讲的就是说整个我们的业务是如何运转的，有哪些业务环节和链条来构成，我们整个这家公司最核心的商业价值的产生是如何被驱动增长的，有哪些环节和链条来去影响到我们商业价值的提升。

业务模型会分为：
- **整体业务模型**：整个业务如何运转
- **局部业务模型**：单个环节的业务逻辑

#### 整体业务模型 vs 局部业务模型

```
┌─────────────────────────────────────────────────────────────────────────┐
│              整体业务模型 vs 局部业务模型                                │

│   ┌─────────────────────┐      ┌─────────────────────┐                │
│   │    整体业务模型      │      │    局部业务模型      │                │
│   ├─────────────────────┤      ├─────────────────────┤                │
│   │ 定义：              │      │ 定义：              │                │
│   │ 公司整体业务        │      │ 单个部门/环节       │                │
│   │ 如何运转            │      │ 的业务逻辑          │                │
│   ├─────────────────────┤      ├─────────────────────┤                │
│   │ 范围：              │      │ 范围：              │                │
│   │ 全链路、全环节      │      │ 某个具体环节        │                │

│   │ 例子：              │      │ 例子：              │                │

│   ├─────────────────────┤      ├─────────────────────┤                │
│   │ 使用者：            │      │ 使用者：            │                │

│   └─────────────────────┘      └─────────────────────┘                │

│   关系：整体业务模型 = 多个局部业务模型的组合                          │
│                                                                         │

```

#### 1.1.2 商业经营模型

商业经营模型可以直接称为**财务模型**，关注的是：
- 成本结构
- 利润结构
- 增长模式

### 三、不同公司阶段的模型构建逻辑差异

#### 3.1 中早期公司的逻辑

**核心特征**：业务先成型，模型后清晰

在一家中早期公司，很多公司刚创办的时候，它的商业逻辑、它的商业模式等等都是不清晰的，都要不断的摸索、不断的奔跑的过程中，慢慢才能够对公司将来到底靠什么挣钱有更强的概念。

> 这个案例展示了中早期公司如何在业务尚未成型时探索商业模式。

**三节课第一年的真实状态**：
> "三节课的第一年基本就是啥都干，公司将来到底靠什么挣钱不知道，反正先干再说。干到哪块反正就是凿出水来了，就往下去深造。"

**中早期公司的逻辑总结**：

| 阶段 | 核心关注 | 关键问题 |
|------|---------|---------|
| 探索期 | 业务模型 | 能否创造商业价值？ |
| 突破期 | 标准化可能 | 价值放大能否规模化？ |
| 成长期 | 商业模式 | 到底靠什么挣钱？ |

**中早期公司的逻辑**：
1. **优先关注业务模型**：我的业务要先能成熟、要先能成形
2. **创造商业价值**：在某件事上先能创造一定的商业价值（不管是收入还是流量）
3. **验证标准化可能**：创造商业价值的过程是否可以标准化
4. **如果可以标准化**：意味着商业价值的放大有机会可以被规模化
5. **再往上推演**：商业经营模型和商业模式会逐渐清晰

> 在中早期公司里，如果业务没有定型，老板脑海中虽然也会关注利润、关注成本，但这些东西是比较零散的，并没有一个很清晰的模型。

##### 中早期公司高层的视角

对于过了初创期的中早期公司，高层的视角基本会说：
- 不断的跑模型选方向
- 有一个模型出来了，在这个方向下商业可能性是怎样的
- 商业经营的逻辑、成本利润能不能打平
- 算完发现不靠谱就换一个方向，再跑一个新的模型

#### 3.2 成熟公司的逻辑

**核心特征**：从上往下，层层稳固

在成熟公司里，本质上是说：
- 商业模式已经很清晰了
- 商业经营模型可能也比较稳定了

- 老板牢牢守住上面，往下看业务模型怎么优化、怎么升级

#### 核心方法论：四层拆分法

> 四层拆分法是分析业务问题和经营问题的基本方法，适用于业绩不达标、利润下降等各种场景。

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        四层拆分法                                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  第1层：业务类型/产品线拆分                                           │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━   │
│  把公司业务按业务类型或产品线进行拆分                                  │
│  例如：线下业务 vs 线上业务、K12 vs 留学 vs 少儿英语                  │
│  目的：找到问题出在哪个业务板块                                        │
│                                                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  第2层：市场/地域维度拆分                                             │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━   │
│  按市场地域进行细分                                                   │
│  例如：一二线城市 vs 下沉市场                                         │
│  目的：找到问题出在哪个市场                                           │
│                                                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  第3层：问题归因                                                     │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━   │
│  分析问题根因                                                         │
│  • 策略问题？打法不对                                                 │
│  • 能力问题？执行力不够                                               │
│  • 资源问题？投入不足                                                 │
│  目的：明确问题本质                                                   │
│                                                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  第4层：成本/资源结构追踪                                            │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━   │
│  追踪成本结构和资源投入                                               │
│  • 各项成本占比                                                       │
│  • 投入产出比                                                         │
│  目的：找到成本异常点                                                 │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

**四层拆分法的应用场景**：
- 业绩不达标时的归因分析
- 利润下降时的成本追踪
- 业务问题诊断
- 资源配置决策

**CEO/操盘手的核心职责：向外看 + 向内看**

> 这是CEO和管理者操盘一家公司运转的核心方法论。

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    CEO/操盘手的双重视角                                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│   ┌─────────────────────────────────────────────────────────────────┐   │
│   │                      向外看                                     │   │
│   │  ────────────────────────────────────────────────────────────   │   │
│   │  • 关注市场变化：市场空间、潜在机会                             │   │
│   │  • 关注竞争对手：竞品动态、增长策略                             │   │
│   │  • 关注用户变化：需求改变、购买力变化                           │   │
│   │  • 关注行业趋势：机会与威胁                                     │   │
│   └─────────────────────────────────────────────────────────────────┘   │
│                              ↑                                          │
│                              │                                          │
│                              ↓                                          │
│   ┌─────────────────────────────────────────────────────────────────┐   │
│   │                      向内看                                     │   │
│   │  ────────────────────────────────────────────────────────────   │   │
│   │  • 经营管理：以财务/商业经营的逻辑管理公司运转                   │   │
│   │  • 评估模型：用业务模型和财务模型评估各项工作                    │   │
│   │  • 识别问题：找到业务中存在问题的根因                          │   │
│   │  • 驱动调整：基于模型判断来驱动业务优化                        │   │
│   └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

---

**向外看：关注市场变化**

作为CEO或操盘手，你需要时刻关注外部环境的变化：

- **市场空间**：这个市场有多大？增长潜力如何？还有什么机会点？
- **竞争对手**：竞品在做什么？它们增长策略是什么？有没有新的玩家进入？
- **用户变化**：用户需求发生了什么变化？购买力有没有变化？有没有新的用户痛点？
- **行业趋势**：行业正在往哪个方向发展？有什么新技术、新模式出现？

> **举例**：就像西贝在2020年疫情前，原本制定了营收增长15%-20%的目标，但疫情这个外部变化发生时，CEO必须第一时间关注到"市场发生了剧变"这个信息。

---

**向内看：用模型管理公司**

关注外部的同时，CEO需要向内看，用模型来管理公司：

- **经营管理**：以财务/商业经营的逻辑来管理公司的运转
- **评估模型**：用业务模型和财务模型来评估各项工作
- **识别问题**：找到业务中存在问题的根因
- **驱动调整**：基于模型判断来驱动业务优化

> **举例**：就像新东方CEO看到"收入增长30%但利润下降20%"这个数据时，他不是去问"每个月上多少课"这种细节，而是用两套模型来评估：
> - 业务层面的模型：收入为什么没达标？
> - 财务层面的模型：利润为什么下降？

---

**动态调整：根据外部变化和业务成熟度来决定调整频率**

```
┌─────────────────────────────────────────────────────────────────────────┐
│               动态调整的频率取决于两个因素                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│   因素1：外部环境变化的速度和频率                                      │
│   ───────────────────────────────────────────────────────────────────  │
│   • 外部变化越快越频繁 → 调整必须越敏捷、越快速                        │
│   • 外部变化慢 → 不需要频繁调整                                        │
│                                                                         │
│   因素2：业务成型的成熟度                                              │
│   ───────────────────────────────────────────────────────────────────  │
│   • 业务成熟度低 → 需要频繁尝试和调整                                  │
│   • 业务成熟度高 → 更多是优化而非大幅调整                              │
│                                                                         │
│   组合结果：                                                           │
│   ┌────────────────────┬────────────────────┬────────────────────┐     │
│   │                    │   外部变化快       │   外部变化慢       │     │
│   ├────────────────────┼────────────────────┼────────────────────┤     │
│   │   业务成熟度低     │   频繁调整        │   适度调整        │     │
│   │   (中早期公司)     │   快速试错        │   保持灵活性      │     │
│   ├────────────────────┼────────────────────┼────────────────────┤     │
│   │   业务成熟度高     │   快速响应        │   稳定优化        │     │
│   │   (成熟公司)       │   谨慎变更        │   回归模型        │     │
│   └────────────────────┴────────────────────┴────────────────────┘     │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

> **关键认知**：作为操盘手，你必须根据外部环境的变化速度和业务的成熟程度，动态调整自己的管理方式。

### 五、AB类操盘手的能力要求

#### 5.1 成熟公司中的AB类操盘手

**AB类操盘手定义**：
- A类：大公司里面的项目负责人
- B类：小公司里面的部门leader
- 还没有绝对到说要在大公司里本身就是一个管大几十号人、一两百万预算的总监

**能力要求**：

1. **理解公司的全局业务模型**
   - 至少要知道这家公司商业价值提升的商业价值增长模型大概是长什么样的

2. **搞清楚自己负责板块的业务模型**
   - 把自己所负责的这个局部它的业务模型要搞清楚

3. **定目标做规划**
   - 基于业务模型定目标、做规划，给到一线同学让他们去执行

> 在成熟公司里，AB类操盘手的要求是：理解公司的全局业务模型，加上能搞清楚自己负责板块的业务模型，往下定目标，就OK了。

#### 5.2 中早期项目中的AB类操盘手

**核心职责**：

1. **一线执行破局**
   - 必须能够扎到一线去执行，有些一线上面的战线甚至必须依赖于他本人才能打开

2. **抽象业务模型**
   - 要不断的抽象出来一些业务模型
   - 用这个模型来跟上级去对话
   - 辅助上级来进行一些动态的决策

> 在中早期公司里面，AB类操盘手的职责核心就是：在一线得能够去执行破局，同时也要能够不断的抽象出来一些业务模型，用这个模型来跟上级去对话，辅助一些动态的决策。

| **成熟公司** | 商业模式清晰 → 往下优化业务模型 | 守住商业模式、往下看业务优化 | 理解全局模型 + 搞清局部模型 + 定目标做规划 |

2. **三大模型**：商业模式、商业经营模型、业务模型（整体+局部）
3. **整体业务模型 vs 局部业务模型**：整体是全链路，局部是单环节；整体由多个局部组成

6. **不同层级**关注不同层次的模型：高层看商业模式和商业经营模型，中层看业务模型和局部业务模型
7. **AB类操盘手**需要理解全局业务模型 + 搞清局部业务模型 + 定目标做规划


3. **长租公寓案例**：三种流量来源 + 三种房源 + 三种收入来源，细化后的收入公式才有价值

- **收入公式**：多条路径 = 多个收入来源

## 二、第一个额外思考：单一产品/服务的售卖生命周期

### 2.1 核心问题：售卖生命周期是长还是短？

```
┌─────────────────────────────────────────────────────────────┐
│          售卖生命周期长短对业务增长的影响                      │
├─────────────────────────────────────────────────────────────┤
│                                                               │
│  问题：你所售卖的单一产品或服务，它的售卖生命周期到底是长还是短？   │
│       ↓                                                      │
│  ┌────────────┴────────────┐                                 │
│  ↓                          ↓                                 │
│  售卖生命周期短              售卖生命周期长                     │
│  ↓                           ↓                               │
│  需要快速海量上新能力          依赖持续稳定的用户获取和增长        │
│  ├─ 产品上线几个月就卖不动      ├─ 产品可以卖好几年              │
│  ├- 必须不断推陈出新           ├─ 转化和交付能力稳定             │
│  └─ 靠SKU增加驱动增长          └─ 靠用户增长驱动增长             │
│                                                               │
└─────────────────────────────────────────────────────────────┘
```

### 印证举例

> 逐字稿原文：
> "如果你所售卖的产品 它的售卖生命周期普遍较短 比如说你说我上线一个产品或一个服务 最多就只能卖几个月 几个月之后这东西卖不动了 对吧 那必然会导致什么呢 必然会导致这个业务 它的增长很可能会需要依赖于快速海量上新的这样的能力来去进行驱动"

### 关键要点

- **售卖生命周期短** → 增长依赖海量上新
- **售卖生命周期长** → 增长依赖用户增长
- 这个问题直接决定了业务增长的核心驱动力

## 三、不同业务模型的竞争力差异

### 3.1 投放获客模型 vs 三级分销获客模型

```
┌─────────────────────────────────────────────────────────────┐
│        投放获客模型 vs 三级分销获客模型对比                    │
├─────────────────────────────────────────────────────────────┤
│                                                               │
│  投放获客模型：                                                │
│  ├─ 核心要求：                                                 │
│  │  ├- 线索单价控制在一定范围                                   │
│  │  ├- 转化率保证在一定范围                                     │
│  │  └─ 才能保证业务利润稳定为正                                  │
│  ├─ 问题：                                                    │
│  │  └- 一旦行业竞争加剧，获客成本无限上升 → 业务利润不稳定，可能为负 │
│  └─ 特点：                                                    │
│     └- 变量多，市场竞争格局变化不受主观意志控制                    │
│                                                               │
│  三级分销获客模型：                                             │
│  ├─ 核心机制：                                                 │
│  │  ├- 设定一级、二级、三级代理的分佣额度                           │
│  │  ├- 从激励机制上控制在一定范围                                 │
│  │  └- 只要代理认可分佣比例，转化率就能保证                         │
│  ├─ 优势：                                                    │
│  │  ├- 获客成本精确可控                                         │
│  │  └- 不管规模是1000还是1万，成本都控制在范围之内                  │
│  └─ 增长：                                                    │
│     └- 代理愿意干活，增长驱动力就存在，可以规模化扩张                   │
│                                                               │
└─────────────────────────────────────────────────────────────┘
```

### 印证举例

> 逐字稿原文：
> "如果我是通过投放来获客的这么个业务模型 那本质上我要保证我的业务利润有足够的增长 足够的空间可以长期为证 长期有盈利的可能性的话 我一定要保证我的线索的单价 我当个例子的获客的单价啊 应该是能控在一个范围内的 并且呢 我的转化率对吧 也应该要保证在一个范围之内"

> 逐字稿原文：
> "三级分销的模型 它可以做到哈 比如说我稳定的设定说 你看我给一级代理啊 肯定一个分拥的额度 二级代理和三级代理呢 也分别有个分拥的这个额度 这个额度呢 从激励机制上 我就把它控制在一定范围内 那整个这样一套管理机制下来的话 你会发现我的转化率呢 我能够保证在一个范围内的 我能够保证 那整个的获客成本它将是更为可控的"

### 关键要点

- **投放模型**：变量多，成本不可控，受市场竞争影响大
- **三级分销模型**：成本精确可控，适合规模化扩张
- **选择依据**：业务形态、发展阶段、竞争环境


> 逐字稿原文：

### 关键要点总结

- **成本优势 = 竞争壁垒**

## 六、业务模型与组织架构的关系


### 印证举例

> 逐字稿原文：

### 关键要点

- **组织架构要匹配业务模型**
- **业务模型变化 → 组织架构必须调整**
- **组织结构定期优化和调整是非常必要的**

> "根据市场上的热点和一些动态来去选择不同的话题做选品 再根据不同的这种课程的特点吧 做一些策划 针对每个课程单独制定一系列营销计划 所以基本上是说可以理解为吧 从业务模型的角度 根据市场热点和话题为中心 然后呢 以市场热点和话题呢 组织内部的客人生产和这个教学运营和营销的资源给组织去不断去打"

#### 6.2.3 业务模型与组织架构的匹配关系

```
┌─────────────────────────────────────────────────────────────┐
│       三节课业务模型升级前后的组织架构对比                      │
├─────────────────────────────────────────────────────────────┤
│                                                               │
│  业务模型变化：                                                │
│  ├- 旧：以话题为中心，多课程分散运营                               │
│  └- 新：以小课转化为中心，集中运营少数课程                          │
│                                                               │
│  组织架构变化：                                                │
│  ├- 旧：渠道市场部 + 品类运营部 + 教学服务部                        │
│  │  ├- 品类运营部是营销计划发起者                                 │
│  │  └- 教学服务部响应品类运营                                     │
│  │                                                            │
│  └- 新：营销中心 + 学习中心                                      │
│     ├- 营销中心负责整个转化路径                                   │
│     └- 学习中心负责产品和交付                                    │
│                                                               │
│  核心变化：                                                    │
│  └- 组织架构完全匹配新的业务模型                                  │
│                                                               │
└─────────────────────────────────────────────────────────────┘
```

### 印证举例

> 逐字稿原文：
> "营销中心呢 负责所有的从渠道 流量推广小课的这个销售转化的就这么个这种路径的把握 最后呢 对订单和收入负责 还有对相应课程的满班率负责 学习中心呢 重点其实就管说我们产品的研发交付的稳定 然后学习目标 教学目标的达成和口碑的达成"

### 关键要点总结

- **业务模型变化 → 组织架构必须调整**
- **组织架构要匹配业务模型**
- **阿里每半年做一次组织架构调整以匹配战略**
