# 1.3 按照顺序不断验证价值假设点.mp4

随后进入到我们的第三个环节，我们要按照顺序不断验证我们的价值假设点，直到处理个业务模型成立或者被证伪对进入到我们验证的环节了，各位注意进入到十分重要的验证的环节了。以上，在验证的部分有两个重要的关键词，这两个词是什么？第一个词叫做定量，第二个词叫做变量，这两个词十分的重要。

因此，我们怎么围绕这两个词来做好我们价值假设的验证例如我们现在可能核心形成一个价值假设，对例如我们刚才讲到说我们假设说有一个什么模型或有个什么机制对能帮助各位的学习效果能更好。

因此，对或者说我们认为训练的方式怎么设计，能让各位对业务模型的理解更加的深刻，对约这样的怎么做价值假设的验证。首先我们一定是说有一个链条和一个模型，其次在当中我们要梳理出拉出关键的影响的要素。

第三十分重要，我们在各个要素当中一定要做到稳定，找到一些定量，这定量基本说他之前已经得到了验证，或者已经在其他的地方是被用户能得到认可的，或者说在处理个我们的业务链条里边，它的存在是当前已经是一个yes，就不需要再去回答说它到底yes还是no，

我们要先找到这样一些定量，并且在确定我们的变量，变量不应该太多，变量如果太多你的验证注定就会失败。最后那我们的反复测试，约是这么一个逻辑。

定量和变量这两件事在宏观的层面上存在，在微观到个业务流程的部分还是存在。对我们简单的拿两个案例可能来查看，首先是我们提到的库产品对库产品你会发现在因为我们是直接借鉴国外一个成熟的模式对成熟的产品，换句话说东西这款产品在国外它已经被人跑通过了，它的很多的价值是已经得到了验证的，

所以在库产品上面，我们的商业价值构想是一个定量，不需要我们再去探索，除非我们说我们做库产品想做成国外产品10倍以上的这种体量对这件事就需要重新去想，但如果没有想象，它个定量。

其次是它的产品所创造的用户价值和它的业务逻辑的假设，这件事因为它已经发生了，所以它也是一个定量。而只有在最后对于库产品产品，它要想被养成，要想被发展起来，它的执行运作方式的构想，部分对我们而言才是变量。

好随后再进一步对我们会发现在库产品里边像我们刚才提到的，它处理个业务链条上存在这么三块问题，这当中另外两块问题现在都没发生，但不重要，我们现在就缩减掉另外两块问题，我们就不需要让变量存在了，我们只关注一个问题，结合案例定向邀请一些产品经理

来去产生一些专业的解读和观点的碰撞，这件事能不能发生，所以我们只关注这么一件事儿。

又在这么一件事儿下，我们再去查看它的处理个的运营逻辑里面，对我们又发现它处理个的运营的链条是这样的，需要一个内容生产的工具和需要种子用户运营的机制来去做，我们还是通过说缩减变量的方式，我们让内容生产和互动工具就通过邮件群组来去做，让它形成一个定量，让初始的内容填充就不需要变得十分复杂，就我们提前找好约710每周更新7\~10案例可能就ok了，部分我们也让它形成一个基本的定量，无非只是说我们到底怎么可引导用户在这能持续交流，形成社区的氛围，让用户要养成这种互动的习惯，以及怎么去对这些种子用户做好他的这种情感激励的运营等等。

部分有可能对我们是变量，所以最后我们的工作定量都定了，我们只需要关注最终的这两个变量，它是否可顺畅的发生就ok了，这是我们的基本工作逻辑。

你会发现这么一层层的落下来之后，虽然说我们做的是一个探索性项目，但是在业务链条里面我们的变量是足够少的，当我们变量足够少，这件事对我们而言就更加的可控，是这么一个逻辑。

核心他应该就先解决，按我们刚才讲的逻辑，他就先解决两个问题，第一个是说这些用户有没有动机帮你做分销，第二个是说他们有没有能力帮你去做分销对他们工具你给到他们一个这种例如人肉，怕帮他们生成一些卡片之后，他们有没有能力对帮助你带来一些订单从而获得收益，他只要验证这么两件事儿，他的假设能通还是不能通，约就有个想象了，后面的工具怎么做，链条怎么处理，怎么去制定，对这帮用户的激励机制之类的都可以后面想。

，但是在执行过程中你发现由于一些客观的原因，它偏离了原有的价值，假设原本十分简单的用户直接分销，就变成了说用户要通过写内容来分销，而且是要在一个社区里边来写内容来分销，这里面你会发现它的变量立即变得十分的多，首先是说这些用户能不能写出来优质的内容

然后是说这些内容写出来之后有没有足够人能看到，然后这些内容被看到之后，能不能产生其他用户购买的转化，还有我的什么转化的这种工具，转化的链路，还有我的社区流量怎么能被养起来等等，有一系列的问题糅杂在一起，导致说一个原本简而言之的事情变成了一个变量极其多的一件项目，对我们很多人在执行过程当中都会去踩到类似这样的坑，所以你会发现当一个探索型项目当中，如果出现了说n多个变量叠加在一起，而又没有任何定量的时候，对不起项目将注定走向失败。

例如你说我跑了两个月生产100多篇内容，这些内容没有带来收入

到底什么样的问题导致项目的失败，你发现最后得不到一个准确的结论的。对这种失败就十分可怕，也十分可悲，因为你失败之后通常是稀里糊涂就失败了，对最后你也没有验证到任何东西，对这就我们一再说的。你证实一件事情和证伪一件事情都是很有价值的，但像这样稀里糊涂的失败一点价值也没有。

所以所谓验证一个假设的关键在于说我们一个模型要成型的探索过程当中，我们的定量足够多足够稳定，你才有机会围绕着少数的变量来完成验证，并且当一个探索项目当中出现n多变量叠加在一起，又没有任何定量的时候，探索项目注定就要失败。，所以这是处理个我们的基本的工作方法里头的前三个重要的环节和步骤。
