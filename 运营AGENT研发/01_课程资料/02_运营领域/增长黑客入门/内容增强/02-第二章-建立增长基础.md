## 2.1 为什么需要验证市场可行性（PMF）

### PMF的定义

**PMF（Product-Market Fit）** 即产品-市场契合点，指产品满足真实市场需求的状态。

### 为什么需要PMF？

- 避免产品开发完成后才发现没有市场需求
- 减少资源浪费，降低创业风险
- 为后续增长奠定基础

### PMF的5个验证步骤

1. 圈定产品的目标用户
2. 通过6个渠道挖掘潜在用户需求
3. 用一张模板找到产品的核心定位
4. 通过5个步骤设计产品的MVP
5. 开发并验证MVP的基本方法

## 2.3 第二步：通过6个渠道挖掘潜在用户需求

### 6大需求挖掘渠道

| 序号 | 渠道 | 说明 |
|------|------|------|
| 1 | 搜索引擎 | 通过关键词分析用户需求 |
| 2 | 问答社区 | 知乎、Quora等平台的用户提问 |
| 3 | 社交媒体 | 微博、微信群的用户讨论 |
| 4 | 竞品分析 | 研究竞争对手的用户反馈 |
| 5 | 问卷调查 | 直接获取用户反馈 |
| 6 | 用户访谈 | 深度了解用户需求 |

### 需求分析框架

- **需求真实性**：用户是否真的有这个需求
- **需求刚性**：用户需求的强烈程度
- **需求频次**：用户需求的频繁程度
- **需求付费意愿**：用户是否愿意为此付费

## 2.5 第四步：通过5个步骤设计产品的MVP

### MVP的5个设计步骤

1. **明确核心假设** - 确定产品最关键的假设
2. **定义最小功能** - 只做验证假设必需的功能
3. **快速原型** - 制作可演示的原型
4. **用户测试** - 让目标用户使用并收集反馈
5. **迭代优化** - 根据反馈改进产品

### MVP的核心原则

- **最小化**：只做核心功能
- **可行性**：能够快速实现
- **可验证**：能够验证核心假设

## 2.6 第五步：开发并验证MVP的基本方法

### MVP开发工具

| 工具类型 | 代表工具 |
|----------|----------|
| 问卷工具 | 问卷星、腾讯问卷 |
| 原型工具 | Figma、Sketch |
|  landing page | Unbounce、Wix |
| 众筹平台 | 淘宝众筹、Kickstarter |

### 验证MVP的方法

1. **广告测试法**：投放广告看用户反应
2. **邮件预热法**：收集用户邮箱预约
3. **假MVP法**：只展示页面不做实际功能
4. **碎片化验证**：通过多个小实验验证

## 2.8 快速验证商业想法的2个工具

### 精益创业画布

| 模块 | 内容 |
|------|------|
| 问题 | 解决什么问题 |
| 解决方案 | 如何解决 |

### 设计冲刺（Design Sprint）

- **周一**：理解问题
- **周二**：解决方案

- **周四**：原型
- **周五**：测试

## 本章小结

本章学习了PMF验证的完整流程：

1. **目标用户**：明确定义谁是你的目标用户
2. **需求挖掘**：通过多渠道了解用户真实需求
3. **核心定位**：找到产品的差异化价值主张
4. **MVP设计**：用最小成本验证核心假设
5. **验证方法**：选择合适的验证工具和方法
