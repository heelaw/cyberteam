## 课程概述

增长黑客（Growth Hacker）是一种以数据驱动营销、以实验验证策略的职业。本课程系统讲解增长黑客的核心方法论，覆盖从市场验证、用户获取、激活留存到数据分析的完整增长体系。

## 核心知识点总览

### 核心模型

| 模型名称 | 全称 | 核心价值 |
|----------|------|----------|
| **AARRR** | Acquisition, Activation, Retention, Revenue, Referral | 用户生命周期增长漏斗 |
| **PMF** | Product-Market Fit | 产品-市场契合点验证 |
| **MVP** | Minimum Viable Product | 最小可行产品 |
| **HOOK** | Hook Model | 用户习惯形成闭环 |
| **OSM** | Objective, Strategy, Metric | 业务指标体系框架 |
| **HEART** | Happiness, Engagement, Adoption, Retention, Task Success | 用户体验衡量框架 |
| **RFM** | Recency, Frequency, Monetary | 用户价值分层模型 |
| **STEPS** | Social Currency, Triggers, Emotion, Public, Practical Value, Stories | 疯传6原则 |

### 核心方法

| 方法 | 说明 | 应用场景 |
|------|------|----------|
| **假设验证模板** | 如果我们做X，那么Y%的用户将会发生Z | 增长实验设计 |
| **ICE评分** | Impact×Confidence×Ease | 实验优先级排序 |
| **四象限矩阵** | 数量/质量二维分析 | 渠道评估、功能分析 |
| **用户画像** | 特征+场景+痛点 | 目标用户定义 |
| **价值主张画布** | 目标用户+核心价值+问题场景+解决方案+差异化 | 产品定位 |
| **7步增长计划** | 设定目标→拆解指标→分析现状→制定策略→排优先级→制定计划→评估标准 | 增长方案制定 |
| **数据分析5步骤** | 获取→清洗→分析→验证→可视化 | 数据分析流程 |
| **A/B测试** | 对照实验验证假设 | 产品和运营优化 |

### 核心指标

| 指标类别 | 具体指标 | 计算方式/定义 |
|----------|----------|---------------|
| **增长指标** | 新用户数、增长率、病毒系数K因子 | K = 平均每个用户带来的新用户数 |
| **激活指标** | 激活率、惊喜时刻达成率 | 激活率 = 新用户中使用某项产品功能的比率 |
| **留存指标** | 次日留存、7日留存、30日留存 | N日后活跃用户数 / 当日新增用户数 |
| **变现指标** | ARPU、LTV、CAC、ROI | LTV > 3×CAC（健康标准） |
| **推荐指标** | 分享率、邀请成功率、K因子 | 病毒系数 |
| **用户体验** | NPS、满意度、任务完成率 | NPS = 推荐者% - 贬损者% |

## 学习建议

### 学习路径

**第一阶段：建立认知（第1章）**
- 理解增长黑客的概念和价值
- 掌握AARRR模型
- 了解增长团队的组织形式

**第二阶段：打好基础（第2章）**
- 学习PMF验证方法
- 掌握用户画像和需求挖掘
- 学习MVP设计和验证

**第三阶段：驱动增长（第3章）**
- 学习增长方案制定
- 掌握渠道筛选和拉新方法
- 了解自动化营销和病毒增长

**第四阶段：用户运营（第4章）**
- 学习用户激活和留存策略
- 掌握HOOK模型
- 了解疯传6原则

**第五阶段：数据驱动（第5章）**
- 学习数据分析方法
- 掌握指标体系搭建
- 了解A/B测试

### 学习方法

1. **理论与实践结合**
   - 学习理论后立即应用到实际工作中
   - 选择你熟悉的产品进行案例分析

2. **从模仿到创新**
   - 先模仿成功的增长案例
   - 再根据实际情况创新

3. **数据驱动决策**
   - 养成用数据说话的习惯
   - 通过实验验证想法

4. **持续学习迭代**
   - 增长方法和工具快速迭代
   - 保持学习和实践

### 实践建议

1. **选择一款你熟悉的产品**
   - 分析它的增长策略
   - 识别它的AARRR漏斗
   - 提出优化建议

2. **定义你所在业务的北极星指标**
   - 拆解为一级、二级、三级指标
   - 与团队达成共识

3. **设计一个增长实验**
   - 提出假设
   - 设计实验
   - 收集数据
   - 分析结果

4. **建立数据分析思维**
   - 用数据支持决策
   - 避免主观臆断
   - 持续优化迭代
