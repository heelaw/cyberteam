本章聚焦用户激活和留存的核心策略。新用户从获取到贡献价值中间，还存在两个非常重要的环节：激活和留存。

5. **疯传6原则**：驱动用户主动分享的6大因素

## 4.1 导读：用户激活和留存的必要性

### 激活、活跃、留存的定义

#### 激活（Activation）

**定义**：激活是针对新用户来说的，引导用户去完成某些指定动作，目的是让用户体验到产品的价值，产生使用兴趣。

**激活的本质**：让用户完成首次关键行为（First Key Action），体验到产品价值。

**案例分析：共享单车APP**

| 场景 | 用户体验 | 结果 |
|------|----------|------|
| 成功激活 | 下载APP → 快速找到车 → 顺利骑车 → 体验到便捷性 | 用户感受到价值，愿意继续使用 |
| 激活失败 | 下载APP → 找不到车/车是坏的 | 无法体验价值，流失 |

**其他激活案例：**
- **抖音**：第一次看到倒放特效 → "哇，好神奇"
- **美颜相机**：第一次使用一键磨皮 → "哇，好漂亮"
- **抢票软件**：第一次抢到票 → "哇，真有用"

**惊喜时刻（啊哈时刻）**：能够让用户感受到产品价值的功能或体验。

#### 留存（Retention）

**定义**：留存是指用户在被激活后，持续使用产品的过程。

**用户留存意愿递减**：
- 新奇感逐渐降低
- 需求强烈程度逐渐降低
- 竞品争夺

**用户分类（按30天使用天数）：**

| 用户类型 | 使用频率 | 特征 | 策略 |
|----------|----------|------|------|
| **活跃用户** | 15天以上 | 高频使用，高忠诚度 | VIP服务、付费转化 |
| **一般留存用户** | 7-15天 | 中频使用，有忠诚度 | 提升活跃、付费引导 |
| **濒临流失用户** | 1-6天 | 低频使用，即将流失 | 召回活动、激励措施 |
| **流失用户** | 0天 | 不再使用 | 分析原因，尝试召回 |

#### 活跃（Active）

**定义**：活跃是留存中的一种用户状态，活跃度是衡量用户的指标。

**活跃度衡量标准：**
- 使用频次（每天/每周/每月使用次数）
- 使用时长（单次使用时长、总使用时长）
- 使用深度（功能使用数量、页面浏览深度）

**活跃与留存的关系：**
- 活跃是留存的一种表现
- 用户越活跃，流失可能性越低
- 用户越活跃，付费或分享的可能性越大

### 三者的关系

```
获取 → 激活 → 留存 → 活跃 → 付费/推荐
      ↑                  ↑
      └──── 活跃是留存的一种状态 ────┘
```

| 环节 | 针对对象 | 核心动作 | 成功指标 |
|------|----------|----------|----------|
| **激活** | 新用户 | 引导首次关键行为 | 激活率 |
| **留存** | 老用户 | 让用户持续使用 | 留存率 |
| **活跃** | 留存用户 | 提升使用频次和时长 | 活跃度 |

### 留存的三种方法

| 方法类型 | 核心逻辑 | 适用场景 | 工具/方法 |
|----------|----------|----------|-----------|
| **内容留存** | 通过优质内容吸引用户 | 内容型产品 | PUSH消息、内容推荐 |
| **功能留存** | 通过核心功能绑定用户 | 工具型产品 | 功能留存分析矩阵 |
| **服务留存** | 通过个性化服务留住用户 | 服务型产品 | 用户分层（RFM） |

### 留存的目的

| 目的类型 | 说明 | 例子 |
|----------|------|------|
| **营收贡献** | 留存用户贡献收入 | 会员续费、复购 |
| **内容贡献** | 留存用户创造内容 | UGC内容、评论 |
| **分享转发** | 留存用户带来新用户 | 病毒式增长的原点 |

**核心观点：** 没有激活和留存，用户获取就是浪费钱。

## 4.2 打造激活和留存的闭环：HOOK模型

### HOOK模型概述

**HOOK模型（上瘾模型）** 是由Nir Eyal提出的用户行为设计模型，通过4个步骤打造用户习惯形成的闭环。

**4个步骤：**
1. **触发（Trigger）**：吸引用户的兴趣
2. **行动（Action）**：让用户付出行动
3. **酬赏（Reward）**：多变的酬赏
4. **投入（Investment）**：持续的投入

### HOOK模型四步详解

#### 第1步：触发 - 吸引用户的兴趣

**触发类型：**

| 类型 | 说明 | 例子 |
|------|------|------|
| **外部触发** | 外部刺激触发用户行为 | PUSH推送、邮件通知、朋友邀请 |
| **内部触发** | 用户内在需求驱动 | 无聊、孤独、焦虑、好奇 |

**外部触发示例：**
- 裂变活动海报：活动主题、分销奖励、大咖推荐、课程大纲
- PUSH推送：你有新的消息、好友动态更新
- 邮件通知：您的订单已发货

**内部触发示例：**
- 无聊时 → 打开抖音
- 孤独时 → 打开微信
- 焦虑时 → 打开学习APP

**触发的关键：找准用户的动机**
- 奖励是否是用户需要的？
- 表达方式是否吸引人？
- 是否触发了用户的内在需求？

**反面案例 vs 正面案例：**

| 类型 | 文案 | 效果分析 |
|------|------|----------|
| 反面 | "年中大促，普天同庆" | 不构成触发，无具体利益 |
| 一般 | "满100减20" | 有触发，但不够精准 |
| 正面 | "进口奶粉，满100减20，与香港同价" | 强触发，精准人群+具体利益 |

**补充：八角行为分析法**

从游戏策划角度总结的心理驱动用户行为的维度：

| 维度 | 说明 | 应用示例 |
|------|------|----------|
| **使命感** | 用户感觉自己在做有意义的事 | 蚂蚁森林种树 |
| **成就感** | 用户完成任务的满足感 | 游戏通关、勋章 |
| **拥有感** | 用户对物品的占有欲 | 虚拟装备、收藏 |
| **社交性** | 与他人互动的需求 | 社交关系、互动 |
| **稀缺性** | 物品稀缺带来的紧迫感 | 限时、限量 |
| **未知性** | 探索未知的好奇心 | 盲盒、抽奖 |
| **失落感** | 害怕失去的心理 | 连续签到断签提醒 |
| **自我表达** | 展现自我的需求 | 个性化装扮、签名 |

#### 第2步：行动 - 让用户付出行动

**核心理论：Fogg行为模型**

> **行为 = 动机 + 能力 + 触发**

**B = MAT**
- **B（Behavior）**：行为
- **M（Motivation）**：动机
- **A（Ability）**：能力
- **T（Trigger）**：触发

**动机类型：**
| 动机类型 | 说明 | 例子 |
|----------|------|------|
| **追求快乐** | 用户希望获得愉悦感 | 游戏、娱乐 |
| **逃避痛苦** | 用户希望避免痛苦 | 工具类、效率类 |
| **追求希望** | 用户希望获得期待 | 学习类、投资类 |
| **逃避恐惧** | 用户希望避免恐惧 | 健康类、安全类 |
| **追求认同** | 用户希望获得认可 | 社交类、内容类 |
| **逃避拒绝** | 用户希望避免被拒绝 | 社交类 |

**能力维度（付出的成本）：**
| 成本类型 | 说明 | 降低方法 |
|----------|------|----------|
| **时间成本** | 用户需要付出的时间 | 简化流程、快速完成 |
| **金钱成本** | 用户需要付出的金钱 | 免费试用、补贴 |
| **体力成本** | 用户需要付出的体力 | 自动化、智能化 |
| **脑力成本** | 用户需要付出的脑力 | 简化操作、智能推荐 |
| **社会成本** | 用户可能面临的社交压力 | 隐私保护、匿名 |
| **非常规性** | 用户需要打破习惯 | 引导、激励 |

**行动设计的关键：**
- 用户得到的奖励 ≥ 用户付出的成本
- 用户有能力付出这个成本

**案例分析：课程分销活动**

| 场景 | 课程价格 | 分销要求 | 门槛分析 |
|------|----------|----------|----------|
| 场景A | 19元 | 分销3人回本 | 门槛低，容易达成 |
| 场景B | 190元 | 分销30人回本 | 门槛高，大多数人无法达成 |

**补充：需求三角模型（李叫兽）**

从3个方面分析让用户付出行动的可能：

| 要素 | 说明 | 示例 |
|------|------|------|
| **缺乏感** | 需求 | 用户感觉缺少什么 |
| **目标物** | 奖励，奖励要满足需求 | 产品或服务提供的价值 |
| **能力** | 消费者6大成本 | 用户是否有能力付出 |

#### 第3步：酬赏 - 多变的酬赏

**酬赏类型：**

| 类型 | 说明 | 例子 |
|------|------|------|
| **自我酬赏** | 完成任务本身的满足感 | 游戏通关、学习进步 |
| **猎人酬赏** | 获得实际物品或利益 | 积分兑换、现金奖励 |
| **社交酬赏** | 获得社交认可和互动 | 点赞、评论、关注 |

**多变酬赏的核心：**
- 及时性反馈：立即获得奖励
- 多样性变化：奖励不可预期
- 社交性认可：他人看到和认可

**案例分析：课程分销活动**

| 酬赏类型 | 说明 | 效果 |
|----------|------|------|
| **及时性奖励** | 分销1个人就获得分成 | 用户立即获得正向反馈 |
| **多样性奖励** | 按分销数量排名获得额外奖励 | 超出预期的奖励，激励用户继续努力 |

**酬赏设计要点：**
1. **及时性**：用户行为后立即给予反馈
2. **多样性**：奖励不可预期，保持新鲜感
3. **层级性**：设置不同等级的奖励
4. **社交性**：让他人看到和认可

#### 第4步：投入 - 持续的投入

**投入的本质：** 让用户在产品中付出的越多，用户就越容易留存。

**投入类型：**
| 投入类型 | 说明 | 例子 |
|----------|------|------|
| **时间投入** | 用户投入的时间 | 学习进度、游戏等级 |
| **金钱投入** | 用户投入的金钱 | 会员、虚拟物品 |
| **数据投入** | 用户产生的数据 | 好友关系、内容、设置 |
| **情感投入** | 用户产生的情感 | 社交关系、习惯养成 |

**投入的关键：**
- 用户投入的金钱、时间、精力是看不见的
- 真正让用户留恋的，是通过付出得到的东西

**案例分析：游戏投入**

| 投入 | 用户付出 | 用户得到 | 留存动力 |
|------|----------|----------|----------|
| 时间 | 打BOSS | 史诗级装备 | 装备带来的荣誉感 |
| 金钱 | 购买装备 | 更强战斗力 | 战斗力提升的成就感 |
| 社交 | 组建公会 | 公会成员关系 | 社交关系绑定 |

**投入的反馈机制：**
- 用户的投入不是HOOK模型中的酬赏
- 投入的反馈是下一次HOOK循环的"触发"环节
- 设置一个套一个的用户成长机制，解锁更多特权

**案例分析：游戏循环**
```
打BOSS（投入时间） → 获得装备（酬赏） →
装备更好（投入） → 挑战更大BOSS（触发） → 打更大BOSS（行动） → 获得更好装备（酬赏） →
...
```

### HOOK模型完整案例：王者荣耀

**第一步：触发**
- **内部触发**：无聊、想娱乐
- **外部触发**：朋友邀请、活动通知、版本更新

**第二步：行动**
- 打开游戏 → 选择英雄 → 开始游戏
- 门槛低：手机随时可以玩，单局15-30分钟

**第三步：酬赏**
- **自我酬赏**：游戏胜利的成就感
- **猎人酬赏**：获得金币、英雄碎片
- **社交酬赏**：好友点赞、MVP荣誉

**第四步：投入**
- **时间投入**：游戏时长、英雄熟练度
- **金钱投入**：购买英雄、皮肤
- **数据投入**：好友关系、战队
- **情感投入**：对英雄和皮肤的喜爱

**投入 → 下一次触发：**
- 更多的英雄和皮肤 → 想要展示 → 邀请好友游戏
- 更高的段位 → 想要继续提升 → 继续游戏
- 好友关系 → 想要一起玩 → 响应好友邀请

## 4.3 激活用户：为用户创造惊喜时刻

### 什么是激活？

**定义**：激活是指用户体验到产品核心价值的过程，是对新用户而言的。

**激活率公式：**
> **激活率 = 新用户中使用某项产品功能的比率**

**示例：**
- 新用户100人，其中30人完成了首次购买
- 激活率 = 30/100 = 30%

### 惊喜时刻的定义

**惊喜时刻（啊哈时刻，Aha Moment）**：能够让用户感受到产品价值的功能或体验。

**案例：Facebook**
- **惊喜时刻**：添加10个好友
- **分析**：
  - "添加好友"是用户进行的操作
  - "10个"是具体的操作行为（量化指标）
  - 为什么是10个？经过数据分析发现，添加10个好友的用户留存率显著提升

**惊喜时刻的3个要素：**
1. **功能**：用户使用什么功能？
2. **行为**：用户具体做什么？
3. **量化**：达到什么数量/程度？

### 定义激活指标的要素

**2个核心问题：**
1. 定义用户产生了哪项操作
2. 定义用户的具体操作行为

**操作步骤：**

**第1步：确定核心功能**
- 产品的核心价值是什么？
- 用户为什么使用这个产品？
- 哪个功能最能体现产品价值？

**第2步：分析高留存用户**
- 高留存用户使用了哪些功能？
- 使用频率最高的功能是什么？
- 第一次使用就留存下来的用户做了什么？

**第3步：找到关键行为**
- 哪个行为与留存相关性最强？
- 哪个行为最容易让用户感受到价值？
- 哪个行为最容易完成？

**第4步：量化关键行为**
- 这个行为做到什么程度够用？
- 分析数据找到临界点
- 设置合理的门槛

### 案例思考：不同产品的惊喜时刻

| 产品 | 核心价值 | 惊喜时刻定义 | 原因分析 |
|------|----------|-------------|----------|
| **Facebook** | 社交关系 | 添加10个好友 | 有了足够好友，才能体验社交价值 |
| **Twitter** | 信息获取 | 关注30人 | 关注足够多的人，才能看到有趣内容 |
| **Airbnb** | 住宿体验 | 完成首次预订 | 只有真正体验过，才能感受到价值 |
| **Dropbox** | 文件同步 | 首次上传文件并同步到其他设备 | 体验到同步的便利性 |
| **小米商城（电商）** | 购物体验 | 购买过一次产品，体验到便捷性和优质配送服务 | 体验到购物的便捷和服务质量 |
| **小米手机** | 使用体验 | 使用手机体验到比其他手机更好（如玩吃鸡非常流畅） | 体验到产品的性能优势 |
| **米粉社区** | 社区参与 | 社区建议被官方人员认同 | 体验到被尊重和认可 |

### 定义惊喜时刻的步骤

**第1步：数据收集**
- 收集新用户的前7天行为数据
- 收集用户的留存数据（次日、7日、30日）
- 记录用户使用的所有功能

**第2步：数据分析**
- 分析留存用户与流失用户的行为差异
- 找到与留存相关性最强的行为
- 分析该行为的数量与留存的关系

**第3步：假设验证**
- 提出惊喜时刻假设
- 通过A/B测试验证
- 调整和优化

**第4步：产品优化**
- 引导用户完成惊喜时刻
- 优化流程，降低门槛
- 监控数据，持续优化

### 提高激活率的方法

**方法1：优化新手引导**
- 简化注册流程（一键登录、微信登录）
- 引导核心功能（突出惊喜时刻功能）
- 设置激励任务（完成惊喜时刻有奖励）

**方法2：优化产品体验**
- 降低使用门槛（简化操作、智能推荐）
- 提升性能速度（加载快、响应快）
- 优化UI设计（美观、易用）

**方法3：使用激励机制**
- 首次使用奖励（优惠券、积分）
- 完成任务奖励（新人礼包）
- 限时优惠（倒计时刺激）

**方法4：精准推送**
- PUSH推送激活提醒
- 短信/邮件激活引导
- 个性化推荐内容

## 4.4 内容留存：通过PUSH消息提高用户活跃和留存

### PUSH消息的定义

**PUSH消息**：官方主动发送给用户的一切内容。

**PUSH消息类型：**

| 类型 | 例子 | 目的 |
|------|------|------|
| **APP推送** | APP通知栏消息 | 提醒用户打开APP |
| **营销邮件** | 促销邮件、产品推荐 | 引导用户购买 |
| **营销短信** | 活动通知、优惠信息 | 引导用户转化 |
| **网站弹窗** | 欢迎弹窗、活动弹窗 | 引导用户关注 |
| **公众号推送** | 图文推送 | 引导用户阅读和互动 |

**核心目的：** 让用户点进来看，看完了在APP里打着滚的接着看。

### PUSH效果气泡图分析

**气泡图维度：**

| 维度 | 说明 | 计算方式 |
|------|------|----------|
| **横轴** | 内容留存率 | 次日主动打开APP的人数 / 当日点击PUSH的人数 |
| **纵轴** | 留存标准差 | 反映相同维度数值之间的变化幅度 |

**气泡大小：**
- 气泡越大 = PUSH点击人数越多
- 气泡越小 = PUSH点击人数越少

**气泡图分析：**
- **右上角（高留存率、高稳定性）**：优质内容，多发
- **左上角（高留存率、低稳定性）**：内容好但不稳定，需要优化
- **右下角（低留存率、高稳定性）**：内容稳定但质量差，需要改进
- **左下角（低留存率、低稳定性）**：问题内容，少发或不发

### PUSH转化关键点

**第1步：吸引用户点击PUSH**
- 标题吸引人
- 内容相关性高
- 推送时机合适

**第2步：点击进入内容页后，内容符合用户预期**
- 标题不夸大
- 内容质量高
- 加载速度快

**第3步：内容页能够有效引导用户查看其它内容**
- 相关内容推荐
- 个性化推荐
- 引导互动（点赞、评论、收藏）

### 自动化PUSH推送策略

**策略1：推荐算法**
- **逻辑**：根据用户喜好推荐内容
- **工具**：推荐算法（协同过滤、内容推荐）
- **案例**：今日头条、腾讯新闻
- **优点**：精准度高，用户满意度高
- **缺点**：需要算法和技术支持

**策略2：手动推荐**
- **逻辑**：利用数据分析工具筛选用户特征，手动推送
- **工具**：数据分析平台、用户分群工具
- **案例**：运营人员根据用户行为手动推送
- **优点**：灵活可控，适合特殊场景
- **缺点**：人力成本高，规模化难

**策略3：数据记录**
- **记录内容**：
  - PUSH打开率
  - 留存率
  - 跳转率
  - 转化率
- **记录目的**：
  - 评估PUSH效果
  - 优化PUSH策略
  - A/B测试对比

### PUSH消息优化建议

**优化维度：**

| 维度 | 优化要点 | 示例 |
|------|----------|------|
| **标题** | 简洁有力、突出重点、制造好奇 | "你的好友XXX给你点赞了" |
| **内容** | 相关性高、质量好、有价值 | 推送用户感兴趣的内容 |

| **个性化** | 根据用户画像定制 | "你关注的游戏更新了" |

**A/B测试建议：**
- 测试不同标题的点击率
- 测试不同内容的留存率
- 测试不同时机的打开率
- 测试不同频次的效果

## 4.5 功能留存：通过功能留存分析矩阵提高留存率

### 功能留存分析矩阵

**核心逻辑：** 通过矩阵分析产品中哪个功能对留存的价值最高。

**矩阵维度：**

| 维度 | 说明 | 计算方式 |
|------|------|----------|
| **横轴** | 使用用户的人数（活跃用户占比） | 当前周期使用该功能的用户数 / 当前周期使用产品的所有用户数 |
| **纵轴** | 连续使用功能的用户占比（功能留存率） | 当前周期继续使用该功能的用户数 / 上周期使用过该功能的用户数 |

**计算方法详解：**

**功能留存率：**
> **功能留存率 = 当前周期继续使用该功能的用户数 / 上周期使用过该功能的用户数**
> - **不包含新用户**

**活跃用户占比：**
> **活跃用户占比 = 当前周期使用该功能的用户数 / 当前周期使用产品的所有用户数**
> - **包含新用户**

**关键注意事项：**
1. 功能留存率的计算不包含新用户，活跃用户占比包含新用户
2. 两个数据维度的计算周期要相同（日/周/月）

### 矩阵象限分析

```
    功能留存率高
       ↑
       │
潜力功能  │  明星功能
(低用户/高留存) │ (高用户/高留存)
       │
───────┼────────→
       │
问题功能  │  淘汰功能
(高用户/低留存) │ (低用户/低留存)
       │
       ↓
    功能留存率低
    (横轴：活跃用户占比)
```

**四个象限的策略：**

| 象限 | 特征 | 策略 | 例子 |
|------|------|------|------|
| **明星功能** | 高用户量、高留存率 | 重点维护，持续优化 | 核心功能 |
| **潜力功能** | 低用户量、高留存率 | 加大推广，引导使用 | 新功能、优质功能 |
| **问题功能** | 高用户量、低留存率 | 优化体验，提升留存 | 需优化的功能 |
| **淘汰功能** | 低用户量、低留存率 | 减少投入，考虑下线 | 低价值功能 |

### 功能留存分析流程

**第1步：确定分析周期**
- 日留存：高频使用的产品（社交、工具）
- 周留存：中频使用的产品（电商、内容）
- 月留存：低频使用的产品（旅游、房产）

**第2步：收集功能数据**
- 各功能的使用用户数
- 各功能的留存用户数
- 产品的总用户数

**第3步：计算矩阵数据**
- 功能留存率
- 活跃用户占比

**第4步：绘制矩阵图**
- 将各功能放入矩阵
- 分析各功能的象限位置

**第5步：制定优化策略**
- 明星功能：如何保持优势？
- 潜力功能：如何扩大用户？
- 问题功能：如何提升留存？
- 淘汰功能：是否下线？

### 案例分析：某内容APP的功能留存

| 功能 | 使用用户数 | 功能留存率 | 活跃用户占比 | 象限 | 策略 |
|------|-----------|-----------|-------------|------|------|
| 首页推荐 | 10万 | 60% | 50% | 明星 | 重点维护，优化算法 |
| 视频播放 | 8万 | 70% | 40% | 明星 | 重点维护，优化体验 |
| 评论互动 | 3万 | 50% | 15% | 潜力 | 加大推广，引导互动 |
| 搜索功能 | 2万 | 40% | 10% | 潜力 | 优化搜索，引导使用 |
| 个人中心 | 12万 | 20% | 60% | 问题 | 优化体验，提升留存 |
| 设置 | 5万 | 10% | 25% | 淘汰 | 减少投入，考虑简化 |

### 优化建议

**明星功能 → 重点维护**
- 持续优化用户体验
- 保持功能稳定性
- 增加功能亮点

**潜力功能 → 加大推广**
- 在显眼位置展示功能入口
- 引导用户使用（新手引导、功能提示）
- 通过活动激励使用

**问题功能 → 优化体验**
- 分析留存低的原因（体验差？不实用？）
- 优化功能设计和交互
- 提升功能价值

**淘汰功能 → 减少投入**
- 简化功能，减少维护成本
- 考虑下线或合并
- 将资源投入到明星和潜力功能

## 4.6 服务留存：通过用户分层制定针对性留存策略

### 用户分层定义

**用户分层**：将用户按照某种特征分为不同群体，针对不同群体制定不同运营策略。

**分层目的：** 将有限的资源分配给最有价值的用户，来实现更好的留存。

**案例分析：知乎用户分层**
- 10%回答问题的用户（核心创造者）
- 20%提问问题的用户（互动者）
- 70%看内容的用户（消费者）

**分层策略：**
- 核心创造者：重点激励，提供特权和服务
- 互动者：引导转化为创造者
- 消费者：提升活跃，引导互动

### 为什么要做用户分层？

**价值1：资源优化**
- 将有限的资源（时间、人力、预算）投入到最有价值的用户
- 避免对所有用户一刀切

**价值2：策略精准**
- 不同层级用户有不同需求
- 针对性策略效果更好

**价值3：提升ROI**
- 高价值用户贡献更多收入
- 精准运营提升ROI

### RFM模型

**RFM模型** 是用户分层的经典模型，通过3个维度对用户进行分层。

| 维度 | 全称 | 说明 | 计算方式 |
|------|------|------|----------|
| **R** | Recency | 最近一次消费时间 | 距离上一次消费的天数 |
| **F** | Frequency | 消费频率 | 一定时期内的消费次数 |
| **M** | Monetary | 消费金额 | 一定时期内的消费金额 |

**RFM分层逻辑：**

| 用户类型 | R（最近消费） | F（消费频率） | M（消费金额） | 策略 |
|----------|-------------|-------------|-------------|------|
| **高价值用户** | 低（最近） | 高 | 高 | 重点维护，推荐高端商品 |
| **潜力用户** | 低（最近） | 低 | 高 | 推荐竞品（品质好单价高） |
| **活跃用户** | 低（最近） | 高 | 低 | 推荐爆品（销量大价格低） |
| **流失用户** | 高（很久） | 低 | 低 | 召回活动，挽回用户 |

**RFM评分方法：**
1. 将R、F、M分别按1-5分评分
2. R评分：最近消费得分高，远得分低
3. F评分：频率高得分高，低得分低
4. M评分：金额高得分高，低得分低
5. 根据总分或组合分层

### 用户分层原则

**原则1：每一种用户分层的维度只有一个**
- 错误：同时按消费金额和消费频率分层
- 正确：按消费金额分层，或按消费频率分层

**原则2：筛选的颗粒度越细，能够制定的策略就越有针对性**
- 过于粗："付费用户"和"免费用户"
- 适中："高消费用户"、"中消费用户"、"低消费用户"
- 过细：按每个用户单独制定策略（不现实）

### RFM分层策略详解

#### 高价值用户（R低、F高、M高）

**特征：**
- 最近刚消费
- 消费频率高
- 消费金额高

**策略：**
- VIP服务：专属客服、优先处理
- 个性化推荐：推荐高端商品
- 特权激励：会员特权、限量商品
- 关怀维护：节日祝福、生日礼物

**目的：** 保持高价值，防止流失

#### 潜力用户（R低、F低、M高）

**特征：**
- 最近刚消费
- 消费频率低
- 消费金额高

**策略：**
- 提升频次：推荐复购激励（满减、优惠券）
- 品质推荐：推荐高品质商品
- 会员引导：引导开通会员，提升频次

**目的：** 提升消费频率，转化为高价值用户

#### 活跃用户（R低、F高、M低）

**特征：**
- 最近刚消费
- 消费频率高
- 消费金额低

**策略：**
- 提升客单价：推荐高单价商品
- 套餐推荐：组合套餐，提升总价
- 促销活动：限时折扣，刺激多买

**目的：** 提升消费金额，转化为高价值用户

#### 流失用户（R高、F低、M低）

**特征：**
- 很久未消费
- 消费频率低
- 消费金额低

**策略：**
- 召回活动：发送优惠券，刺激回归
- 问卷调研：了解流失原因
- 低门槛引导：推荐低价商品，降低门槛

**目的：** 尝试召回，提升活跃

### 其他用户分层模型

**按生命周期分层：**

| 生命周期 | 特征 | 策略 |
|----------|------|------|
| **新手期** | 注册0-7天 | 新手引导、激活激励 |
| **成长期** | 注册7-30天 | 习惯培养、功能引导 |
| **成熟期** | 注册30天以上 | 持续运营、付费转化 |
| **休眠期** | 30天未活跃 | 召回活动、激励措施 |
| **流失期** | 90天未活跃 | 挽回尝试、分析原因 |

**按行为特征分层：**

| 用户类型 | 特征 | 策略 |
|----------|------|------|
| **重度用户** | 高频使用、深度使用 | VIP服务、特权激励 |
| **中度用户** | 中频使用、中度使用 | 提升活跃、引导付费 |
| **轻度用户** | 低频使用、浅度使用 | 激活引导、提升频次 |
| **沉睡用户** | 长期未活跃 | 召回活动、挽回尝试 |

## 4.7 驱动留存用户主动分享的6大因素

### 疯传6原则（STEPS）

**来源：** 乔纳·伯杰《疯传：让你的产品、思想、行为像病毒一样入侵》

**6原则（STEPS）：**

| 原则 | 英文 | 核心问题 |
|------|------|----------|
| **社交货币** | Social Currency | 它是否让人们看起来更优秀？ |
| **诱因** | Triggers | 它是否容易被想起？ |
| **情绪** | Emotion | 它是否能触发情绪？ |
| **公众性** | Public | 它是否容易被看到？ |
| **使用价值** | Practical Value | 它是否有实用价值？ |
| **故事** | Stories | 它是否容易被传播？ |

### 1. 社交货币（Social Currency）

**定义：**
就像人们使用货币能买到商品或服务一样，使用社交货币能够获得家人、朋友和同事的更多好评和更积极的印象。

**个人独有的谈资，加上一群人都感兴趣的共同话题。**

**人类心理基础：**
- 人们都希望自己看起来更加聪慧、富有、时尚
- 人们喜欢共享他们的观点，向身边的人表达自己的喜好
- 人们进行口碑传播主要是为了保持人们对自己的良好印象

**如何创造好的社交货币？**

| 方法 | 说明 | 案例 |
|------|------|------|
| **新奇的事物** | 冷门却新奇的知识 | 黑科技、神秘力量 |
| **显示特权** | 超级会员、VIP休息室 | 机场贵宾厅、QQ SVIP |
| **游戏化** | 徽章、积分卡、竞赛排行榜 | 支付宝集五福、微信步数 |
| **设置门槛限制** | 老会员推荐才能注册、限时抢购 | Clubhouse早期邀请制 |
| **不要轻易付钱** | 一旦给钱，性质就变了 | 免费比付费更有传播性 |

**社交货币的6个维度：**

| 维度 | 说明 | 例子 |
|------|------|------|
| **归属感** | 有多少比例的用户有归属感 | 小米"为发烧而生" |
| **交流讨论** | 消费者中发起品牌相关热烈讨论的人数占比 | 苹果新品发布会 |
| **实用价值** | 品牌是否是人们日常社交的必要要素 | 微信红包 |
| **拥护性** | 有多少拥护品牌的死忠粉 | 粉丝文化 |
| **信息知识** | 用户感觉能与其他消费者进行有效交流 | 知乎大神 |
| **身份识别** | 用户能否识别出其他的用户 | 椰子鞋、特斯拉车主 |

**经典案例：**

| 案例 | 社交货币设计 |
|------|-------------|
| **星巴克** | 用中杯、大杯、超大杯（Tall/Grande/Venti）点单显示逼格；隐藏菜单覆盆子糖浆 |
| **支付宝年度账单** | 炫目的年度账单设计，完全可以分享朋友圈炫耀 |
| **网易云音乐** | 每周听歌最多的歌曲专辑封面拼图 |
| **微信读书** | 每周读书时长排行榜 |
| **乐跑圈** | 线上马拉松，颁发奖杯奖牌 |
| **罗辑思维得到** | 仗义证，用契约精神铸造社交货币 |

### 2. 诱因（Triggers）

**定义：**
在没有任何金钱刺激的情况下，每个人每天大概会谈到60个品牌。但为什么会谈论一些品牌而不会谈论另外一些呢？如何用一些刺激瞬间激活人们的记忆，让他们想到品牌相关的内容呢？

**核心观点：**
用刺激瞬间激活人们的记忆，让他们想到品牌相关的内容。

**常见诱因类型：**

| 诱因类型 | 说明 | 案例 |
|----------|------|------|
| **时间诱因** | 特定时间触发 | 早饭/午饭/晚饭时间触发外卖软件 |
| **空间诱因** | 特定地理位置触发 | 到了机场触发用车软件 |
| **声音诱因** | 特定声音触发 | 微信提示音、支付宝到账声音 |
| **需求诱因** | 根据用户行为猜测下一步行动 | 浏览商品后推荐相关商品 |

**关键要点：**
- **产品的使用频率是很大的诱因**
- **让产品和人们经常提及的事物发生联系**

**案例：百事可乐 vs 致逝健身（特百惠）**

| 品牌 | 诱因 | 效果 |
|------|------|------|
| **百事可乐** | 喝可乐的频率高（每天多次），但可乐不是社交话题 | 使用频率高，但谈论频率低 |
| **致逝健身（Tupperware）** | 特百惠使用频率低（偶尔），但经常在社交场合谈论 | 使用频率低，但谈论频率高 |

### 3. 情绪（Emotion）

**核心观点：**
有感染力的内容经常能引发人们的即时情绪。能触动情绪的事物经常能被大家谈论。

**情绪的两个维度：**

1. **积极/消极的程度**（正面情绪 vs 负面情绪）
2. **生理唤醒的程度**（高唤醒 vs 低唤醒）

**四种情绪分类：**

| 类型 | 情绪特征 | 举例 | 是否促进分享 | 原因 |
|------|----------|------|-------------|------|
| **积极高唤醒** | 积极情绪+高生理唤醒 | 敬畏、消遣、兴奋、幽默 | 是 | 想要分享给他人 |
| **积极低唤醒** | 积极情绪+低生理唤醒 | 满足、 contentment | 否 | 不需要分享 |
| **消极高唤醒** | 消极情绪+高生理唤醒 | 担忧、生气、焦虑 | 是 | 想要发泄或寻求帮助 |
| **消极低唤醒** | 消极情绪+低生理唤醒 | 悲伤、失望 | 否 | 不想谈论 |

**案例：谷歌"巴黎之恋"视频**
- **内容**：使用Google搜索串联的爱情故事
- **情绪**：浪漫、愉悦、敬畏
- **效果**：全球病毒式传播，观看量数千万
- **启示**：敬畏的文章转载率一般比别的文章会高出30%

### 4. 公众性（Public）

**核心观点：**
当人们看见别人使用我们的产品时，他们自己会考虑这种产品是否符合他们自己的需要。有样学样。

**关键洞察：**
- 人们会随时随地地观察他人的所作所为，并且随之模仿
- 让某些事物更具有可视性就可以让他们更好的被模仿

**如何制造公共性？**

| 方法 | 说明 | 案例 |
|------|------|------|
| **利用从众心理/羊群效应** | 大家都在用，我也用 | 排队餐厅、销量领先 |
| **强化差异** | 特色配色、大logo、不同造型 | 特斯拉独特的造型 |
| **行为剩余** | 产品logo纪念品、手提袋 | 宜家购物袋、星巴克杯子 |
| **采用肯定的方法宣传** | 避免"不要做什么" | "系安全带"比"不要酒驾"更有效 |

**案例：苹果笔记本**
- 乔布斯要求打开时对面看到的logo也是正的
- 在咖啡厅使用MacBook的人，logo正对着对面的人
- 这种可视性让MacBook成为公共场所的"广告牌"

### 5. 使用价值（Practical Value）

**核心观点：**
人与人之间本来就有这种互相帮助的倾向。只要我们向顾客证明说我们的产品或者思想能够给他们节省时间或者钱财，那么他们就会大力宣传我们的产品或者理念。

**行为经济学应用：**

| 应用 | 说明 | 案例 |
|------|------|------|
| **心理参照点** | 1000元降10元无感，15元降10元很有感 | 9.9元包邮 |
| **人为制造稀缺性** | 限时/限量购买 | 双11限时秒杀 |
| **折扣显示方式** | 低于100元用比例，超过100元用金额 | 9.9元→8.9元（9折），1000元→800元（省200元） |
| **直接向用户呈现实用价值** | 结账清单显示节省金额 | 满减优惠"已节省50元" |

**案例：乐纯酸奶**
- 利用红包/代金券引发转发传播
- 一环套一环的设计
- 分享给好友获得优惠券
- 好友购买后再获得优惠券
- 形成病毒循环

### 6. 故事（Stories）

**核心观点：**
故事是一种最原始的娱乐形式。相对于广告，人们很少会排斥故事。但顾客往往会对其他顾客的故事深信不疑。

**关键要点：**
- 需要建立像特洛伊木马一样的传奇故事，用来注入我们的产品和思想
- 将重要的信息注入到情节当中
- 想插入的广告应该是主线细节，而不是无关主线的细节

**案例：快看漫画陈安妮**

**故事：**《对不起，我只过1%的生活》漫画
- **内容**：讲述创业故事，充满挫折和坚持
- **情绪**：励志、感动、共鸣
- **效果**：
  - 上线当天增加30万用户
  - 连续三天冲到免费榜榜单第一位
- **启示**：
  - 故事要有真实感
  - 故事要有情绪共鸣
  - 故事要自然植入产品信息

### 疯传6原则总结

| 原则 | 英文 | 核心问题 | 关键点 |
|------|------|----------|--------|
| **社交货币** | Social Currency | 它是否让人们看起来更优秀？ | 炫耀、特权、游戏化 |
| **诱因** | Triggers | 它是否容易被想起？ | 高频使用、社交话题 |
| **情绪** | Emotion | 它是否能触发情绪？ | 高唤醒情绪 |
| **公众性** | Public | 它是否容易被看到？ | 可视化、从众心理 |
| **使用价值** | Practical Value | 它是否有实用价值？ | 节省时间/金钱 |
| **故事** | Stories | 它是否容易被传播？ | 传奇故事、情绪共鸣 |

## 本章小结

本章学习了用户激活和留存的核心策略：

### 核心知识点

1. **激活与留存的区别**：激活针对新用户，留存针对持续使用的用户
2. **HOOK模型**：触发-行动-酬赏-投入的闭环设计
3. **惊喜时刻**：定义用户体验产品价值的关键行为
4. **三种留存方法**：内容留存（PUSH）、功能留存（矩阵分析）、服务留存（用户分层）
5. **疯传6原则**：社交货币、诱因、情绪、公众性、使用价值、故事

### 实践建议

1. **重视新用户激活**
   - 定义产品的惊喜时刻
   - 优化新手引导流程
   - 设置激励机制

2. **打造HOOK闭环**
   - 设计合适的触发机制
   - 降低用户行动门槛
   - 提供多变的酬赏
   - 引导用户持续投入

3. **多维度提升留存**
   - 内容：优化PUSH推送策略
   - 功能：优化明星功能，推广潜力功能
   - 服务：用户分层，精细化运营

4. **设计病毒传播**
   - 创造社交货币
   - 设置诱因
   - 触发情绪
   - 提高公众性
   - 展示使用价值
   - 讲好品牌故事
