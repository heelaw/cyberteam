# 第六章：活动运营（下）- 复杂活动策划与执行

> 所属章节：第六章
> 包含小节：策划完整活动的6大要素、活动后端策略、活动执行与落地

---

## 本章导学

### 核心目标

本章深入讲解复杂多线程活动的策划与执行。复杂活动与简单活动最大的不同在于需要设计完整的后端策略，包括目标人群策略、产品策略等。学员将学习如何从零开始策划一个完整的复杂活动，以及如何有效执行和落地。

### 学习重点

1. 掌握策划完整活动的6大基本要素
2. 理解活动后端策略的设计方法
3. 学会活动主题策划和前端玩法设计
4. 掌握活动风险评估的方法
5. 了解活动执行与落地的关键步骤

### 6.1 策划一个完整活动的6大基本要素

#### 六大要素

```
┌─────────────────────────────────────────────────────────────┐
│                    活动策划6大要素                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌─────────────────────────────────────────────────────┐ │
│   │           约束条件（先确定）                          │ │
│   ├─────────────────────────────────────────────────────┤ │
│   │  1. 目的：为什么要做这个活动？                       │ │
│   │  2. 目标：期望达成什么具体数值？                      │ │
│   │  3. 时间：活动周期和关键节点                         │ │
│   │  4. 资源：预算、人力、物料等                         │ │
│   ├─────────────────────────────────────────────────────┤ │
│   │           策略设计（后确定）                          │ │
│   ├─────────────────────────────────────────────────────┤ │
│   │  5. 策略：如何达成目标                               │ │
│   │  6. 形式：活动具体怎么玩                             │ │
│   └─────────────────────────────────────────────────────┘ │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 目的 vs 目标

| 概念 | 说明 | 示例 |
|------|------|------|
| 目的 | 方向性的why | 拉新、提升收入、提升活跃 |
| 目标 | 量化的what | 新增1万用户、收入增长50%、活跃提升10% |

#### 确定约束条件的方式

```
┌─────────────────────────────────────────────────────────────┐
│                确定约束条件的两种方式                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   方式1：老板直接给出                                      │
│   └─ 目的、目标、时间、资源都明确                           │
│                                                             │
│   方式2：需要自己推断                                      │
│   ├─ 目标参考：市场规模、竞争现状、业务走势                 │
│   ├─ 时间参考：行业节点、团队开发周期                      │
│   └─ 资源参考：历史成本、预期回报                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 6.2 如何结合目标设计活动后端策略

#### 后端策略的核心

后端策略是复杂活动独有的设计，决定了活动的效果上限。

```
┌─────────────────────────────────────────────────────────────┐
│                   后端策略三维度                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌─────────────────────────────────────────────────────┐ │
│   │                   目标人群策略                        │ │
│   ├─────────────────────────────────────────────────────┤ │
│   │  面向新用户还是老用户？                              │ │
│   │  面向高价值用户还是大众用户？                        │ │
│   │  不同用户群用不同策略                                │ │
│   └─────────────────────────────────────────────────────┘ │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐ │
│   │                   产品策略                           │ │
│   ├─────────────────────────────────────────────────────┤ │
│   │  重点推哪些产品/品类？                              │ │
│   │  不同产品用不同促销力度？                            │ │
│   │  如何搭配可以实现目标？                              │ │
│   └─────────────────────────────────────────────────────┘ │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐ │
│   │                   激励策略                           │ │
│   ├─────────────────────────────────────────────────────┤ │
│   │  激励力度多大？                                     │ │
│   │  如何分配激励资源？                                  │ │
│   │  如何控制成本和ROI？                                 │ │
│   └─────────────────────────────────────────────────────┘ │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 人群策略设计

| 人群 | 策略重点 | 激励方式 |
|------|---------|---------|
| 新用户 | 快速转化 | 高额首单优惠 |
| 活跃老用户 | 提升ARPU | 专属折扣、会员权益 |
| 沉默用户 | 召回激活 | 专属优惠、召回礼物 |
| 高价值用户 | 深度运营 | 定制服务、专属活动 |

## 案例分析

### 案例：招聘网站年度活动策划

**活动背景**：金三银四跳槽季活动

**六大要素设计**：

```
┌─────────────────────────────────────────────────────────────┐
│                 招聘活动六大要素设计                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   目的：提升网站流量和注册用户数                             │
│   目标：同比提升100%                                        │
│   时间：3-4月                                                │
│   资源：预算100万，团队5人                                  │
│                                                             │
│   策略：                                                    │
│   ├─ 人群：企业方+求职者双方                                │
│   ├─ 切入：奇葩岗位吸引眼球                                │
│   └─ 整合：行业专场+垂直社区合作                            │

│   形式：PK+投票+专场                                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**执行效果**：通过企业PK、奇葩岗位传播、垂直社区合作等方式，实现了流量和注册用户的大幅增长。

## 本章小结

1. **6大基本要素**：目的、目标、时间、资源是约束条件，决定了策略和形式该怎么设计。先有约束条件再有策略设计。

2. **目的与目标的区别**：目的是方向（拉新、提升收入），目标是量化指标（新增1万、增长50%）。

3. **后端策略是核心**：复杂活动的技术含量主要体现在后端策略设计上，包括目标人群策略、产品策略、激励策略。

4. **后端策略三维度**：人群策略（新老用户、高低价值用户）、产品策略（重点推什么）、激励策略（力度和分配）。

5. **活动主题的作用**：吸引注意、传递信息、情感连接、记忆传播。好的主题是活动成功的一半。

6. **前端玩法设计三要素**：参与机制（如何参与）、激励机制（奖励什么）、传播机制（如何分享）。

7. **风险评估四类**：技术风险、用户风险、运营风险、财务风险。每个活动都需要提前识别风险并准备应对措施。

8. **执行四步走**：方案确认→开发准备→预热推广→正式执行。每个步骤都需要仔细确认和执行。
