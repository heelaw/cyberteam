## 本章导学

本章是整个运营高阶课程的基础模块，核心目标是帮助学员建立”框架式思维”。黄有璨老师通过三个生动的故事，阐述了框架式思维对于运营工作的重要性：它能帮助我们在面对复杂问题时快速找到思考方向，避免陷入”只见树木不见森林”的困境。

在运营工作中，我们每天都会面临大量的复杂问题：如何提升用户留存？如何做好一次成功的营销活动？如何优化产品的运营策略？这些问题往往看似简单，实则涉及众多维度，如果我们没有一套系统的思考方法，很容易陷入零散、无序的思维状态，最终导致工作效率低下、产出质量不佳。

本章将重点讲解框架式思维的价值、训练方法以及三种常用的拆解思路，为后续课程中更复杂的运营问题拆解打下坚实基础。通过本章的学习，你将掌握如何将复杂问题拆解为可执行的小问题，如何建立自己的思考框架，以及如何在实际工作中灵活运用这些方法。

本章的学习路径如下：
1. 理解框架式思维的核心价值
2. 掌握框架式思维的训练方法
3. 学习三种常用的拆解思路
4. 理解认知型框架与行动指导型框架的区别
5. 学会将认知型框架升级为行动指导型框架

## 详细内容

### 主题一：框架式思维的价值

#### 1.1 什么是框架式思维

框架式思维是一种系统性的思考方式，它帮助我们把复杂的问题拆解成若干个有序的部分，从而更清晰地看到问题的全貌和本质。正如黄老师所说，框架式思维最大的意义���是”能够帮助你往正确的方向去思考”。

在日常生活中，我们无时无刻不在思考，但并非所有的思考都是高效的。有些人面对问题能够快速找到切入点，制定出清晰的行动计划；而有些人则可能陷入反复的纠结中，既没有方向，也没有任何产出。这种差异往往源于是否具备框架式思维的能力。

框架式思维的核心在于”先整体后局部”。当我们面对一个复杂问题时，首先不是急于寻找答案，而是要先建立一个问题框架。这个框架就像一张地图，帮助我们了解问题的全貌，明确各个部分之间的关系，以及我们应该从哪个方向切入。

**框架式思维的三个层次**

框架式思维可以分为三个层次，每个层次对应不同的思考深度和应用场景：

第一层次：结构化拆分
将复杂问题拆解为若干个子问题，每个子问题之间保持相对独立性。这是框架式思维的基础层次，也是大多数人在日常工作中最常用到的层次。

第二层次：关系梳理
不仅拆解问题，还要明确各个子问题之间的关系。它们是并列关系、递进关系还是因果关系？了解这些关系有助于我们更好地理解问题的本质。

第三层次：动态演化
考虑到问题会随着时间和环境变化，框架需要具备一定的动态性和可演化性。这需要我们对行业趋势和业务发展有深入的理解。

**框架式思维的心理学基础**

框架式思维的建立基于人类认知的以下几个特点：

首因效应：人们倾向于记住最先获取的信息。在思考框架中，将最重要的维度放在前面，有助于确保关键因素被优先考虑。

格式塔原理：人类大脑倾向于将复杂信息组织为有意义的整体。框架式思维正是利用这一特点，将碎片化信息整合为系统化的认知结构。

认知负荷理论：人类工作记忆的容量有限。框架式思维通过将复杂问题结构化，降低了单次需要处理的信息量，使思考更加高效。

#### 1.2 框架式思维解决的两大问题

在运营实践中，我们经常会遇到两类典型问题，而框架式思维能够帮助我们有效地解决这两类问题：

**第一类问题：问题太抽象，无从下手**

这类问题的典型特征是问题边界模糊、目标不明确。例如：

- “用户运营怎么做？”
- “如何提升转化率？”
- “社群运营有什么好的方法？”

这些问题看似简单，但实际上是”烂问题”。如果你直接去寻求答案，你会发现：
- 答案过于宽泛，无法直接指导行动
- 不同的人有不同的理解，难以形成共识
- 即使得到了答案，也不知道如何落地执行

框架式思维解决这类问题的方法是：首先将抽象问题拆解为若干具体问题，然后再针对每个具体问题寻找解决方案。例如，”用户运营怎么做”可以拆解为：
- 种子用户如何获取和维系？
- 核心用户如何识别和维护？
- 用户分级管理体系如何建立？
- 如何提升用户的活跃度和留存率？
- 如何实现用户的付费转化？

经过这样的拆解，问题变得具体、可执行，每个小问题都可以找到明确的答案。

**烂问题的特征与诊断**

在深入讨论如何将抽象问题转化为具体问题之前，我们首先需要学会识别什么是”烂问题”。烂问题通常具有以下特征：

特征一：边界模糊
问题没有明确的范围界定，回答者不知道该从哪个角度切入。例如”How to do marketing？”就是一个典型的边界模糊问题，因为营销涉及太多领域和维度。

特征二：目标缺失
问题没有说明要达到什么样的效果或目标。例如”How to improve sales？”没有说明是要提升销量、提升利润还是提升转化率。

特征三：情境不清
问题没有说明具体的情境和约束条件。例如”How to price the product？”没有说明是什么产品、面向什么用户群体、在什么市场环境下。

特征四：主体不明
问题没有明确是谁在问、为谁问。例如”How to retain users？”没有说明是初创产品还是成熟产品、是互联网产品还是传统产品。

当你遇到一个烂问题时，可以尝试用以下问题进行诊断：
- 这个问题的边界是什么？
- 解决这个问题要达到什么目标？
- 这个问题发生在什么情境下？
- 解决这个问题的责任主体是谁？

**第二类问题：问题太具体，缺乏全局感**

这类问题的典型特征是只看到局部，看不到整体。例如：

- “写一篇推广文案”
- “设计一个活动海报”
- “回复用户的问题”

这些问题虽然具体，但如果我们只关注这些具体的动作，而忽略了背后的整体策略，就可能导致：
- 产出与整体目标不匹配
- 资源投入分散，无法形成合力
- 细节做得很好，但整体效果不佳

框架式思维解决这类问题的方法是：首先建立全局视角，明确这些具体动作在整体策略中的位置和作用，然后再去执行具体动作。

**从具体到抽象：建立全局视角的方法**

当你陷入具体问题无法自拔时，可以尝试以下方法建立全局视角：

方法一：追问目的
问自己：我做这个动作的目的是什么？这个目的背后的更高目标是什么？不断向上追溯，直到找到真正的目标。

方法二：角色转换
假设自己是公司CEO或业务负责人会怎么看待这个问题？这种视角转换可以帮助你跳出执行层的局限。

方法三：时间维度
问自己：这个动作三个月后还有价值吗？一年后呢？这种时间视角可以帮助你区分短期技巧和长期能力。

方法四：资源视角
问自己：如果资源有限，只能做一件事，你会做什么？这种约束条件下的思考往往能揭示问题的本质。

#### 1.3 框架式思维的核心价值

框架式思维的价值体现在以下几个方面：

**价值一：提高思考效率**

当你有一个清晰的思考框架时，你不需要每次遇到问题都从头开始思考。框架就像一个模板，你可以直接套用，只需要根据具体问题填充相应的内容即可。这大大提高了思考的效率。

举例来说，如果你已经建立了”用户运营”的思考框架，那么当有人问你”用户运营怎么做”时，你可以快速调用这个框架，从用户获取、用户激活、用户留存、用户付费等维度来回答问题，而不需要临时组织思路。

**价值二：确保思考的全面性**

框架式思维能够帮助你确保思考的全面性。一个好的框架通常会涵盖问题的各个重要方面，避免你遗漏关键因素。

举例来说，如果你要分析”如何提升活动的参与度”，一个全面的框架应该包括：
- 活动曝光：用户是否知道活动的存在？
- 活动吸引力：活动本身是否足够吸引人？
- 参与门槛：参与活动是否足够简单？
- 激励机制：是否有足够的激励驱动用户参与？
- 传播机制：用户是否愿意分享活动？

如果没有框架，你可能会遗漏其中某些方面，导致最终的方案不够完善。

**价值三：便于沟通和协作**

当你有一个清晰的思考框架时，与他人沟通变得更加高效。你可以先展示框架，让对方了解你的整体思路，然后再进入细节讨论。这比直接罗列一堆零散的想法更容易被理解和接受。

在团队协作中，框架式思维的价值更加明显。一个好的框架可以成为团队共同的语言，帮助团队成员在同一个层面上讨论问题，避免鸡同鸭讲的困境。

**价值四：促进知识积累**

框架式思维还能够帮助我们积累和传承知识。当我们用框架来组织思考时，框架本身可以被记录和复用。随着经验的积累，你的框架会不断完善，形成���自己的知识体系。

这也是为什么资深运营人员往往能够快速给出高质量的答案，不是因为他们更聪明，而是因为他们已经积累了大量的思考框架，遇到问题时可以直接调用。

**价值五：降低认知偏差**

框架式思维还有一个重要价值是帮助我们降低认知偏差。人类在思考问题时容易受到各种认知偏差的影响，比如确认偏差、锚定效应、可得性启发等。

一个好的框架会迫使我们从多个角度思考问题，而不是只看到我们愿意看到的信息。这有助于我们做出更加客观、理性的决策。

**框架式思维与批判性思考**

框架式思维需要与批判性思考相结合才能发挥最大价值。批判性思考是指对信息进行分析、评估和推理的能力，它帮助我们判断框架是否合理、结论是否可靠。

在使用框架式思维时，需要注意以下几点：

质疑框架的前提：每个框架都有其假设前提，这些前提是否在当前情境下仍然成立？

检验逻辑链条：框架各部分之间的逻辑关系是否严密？是否存在逻辑漏洞？

关注反例：是否有反例能够推翻框架的结论？如果有，需要如何调整框架？

保持开放心态：不要过于执着于已有的框架，要愿意接受新的信息和观点。

**价值四：促进知识的积累和复用**

当你用框架的方式整理你的思考时，这些框架可以被积累和复用。每当你完成一个项目，你可以将你的思考框架沉淀下来，下次遇到类似问题时可以直接调用。

这种积累和复用的能力，是区分普通运营人员和高阶运营人员的重要因素。高阶运营人员之所以能够在短时间内处理大量复杂问题，很大程度上是因为他们已经积累了丰富的思考框架。

#### 1.4 框架式思维的实际案例

为了让读者更好地理解框架式思维的价值，黄老师分享了三个生动的案例：

**案例一：办公室读书会**

背景：办公室组织读《认知设计》这本书，大家分享学习心得。

过程：同事们给出了各种零散的建议——
- “加课前测评”
- “让学习者边学边动手”
- “重新规划课程秩序”

问题分析：这些建议都是单点思考，缺乏整体框架。虽然每个建议都有一定的道理，但如果直接按照这些建议去执行，可能会出现：
- 各个建议之间可能存在冲突
- 资源的投入分散，无法形成合力
- 无法判断哪些建议优先级更高

黄老师的做法：基于对书籍的理解，快速梳理出“设计课程 = 管理学习动机 + 学习体验 + 学习效果”的框架。

在这个框架下，管理学习动机需要关注：
- 学习目标是否清晰
- 学习动机如何激发
- 学习动力如何维持

学习体验需要关注：
- 内容组织是否合理
- 呈现方式是否吸引人
- 互动设计是否有趣

学习效果需要关注：
- 练习设计是否有效
- 反馈机制是否完善
- 进阶路径是否清晰

结果：有了这个框架后，如何设计课程变得清晰可控。每��建议都可以归类到相应的维度，团队可以更有针对性地讨论和决策。

**案例二：知乎评价歌曲**

背景：韩寒电影《后会无期》上映，知乎上有人问“如何评价平凡之路这首歌”。

多数回答：类似“好听”、“太棒了”、“近年来最好的一首歌”——主观感受，无法说服他人。

问题分析：这些回答虽然表达了个人感受，但存在以下问题：
- 缺乏具体分析，无法支撑结论
- 不同人可能有不同感受，说服力有限
- 无法帮助他人建立对歌曲的客观认知

优质回答：先建立框架——将歌曲分解为“旋律 + 歌词 + 编曲”三个部分，然后逐部分分析。

在这个框架下：
- 旋律分析：音调、节奏、和声等方面的特点
- 歌词分析：内容、意境、文学性等方面的特点
- 编曲分析：乐器配置、编排风格、制作水平等方面的特点

结果：这样的思考方式天然更具说服力。读者可以清楚地看到评价者的分析过程，也可以形成自己的判断。

**案例三：三次元沙龙活动设计**

背景：设计线下活动的现场体验。

同事的答案：弄几个游戏区、多弄水果甜点、提升会场逼格。

问题分析：这些建议虽然涉及了活动的某些方面，但存在以下问题：
- 缺乏系统性，没有覆盖活动的全部关键维度
- 无法判断各建议的优先级
- 难以形成完整的执行方案

黄老师的框架：用户在场活动中能否产生“太赞了”的体验，取决于四个方面：

第一个维度：入场体验
- 是否得到惊喜（礼物、游戏）
- 入场流程是否顺畅
- 第一个接触点是否令人印象深刻

第二个维度：活动内容
- 嘉宾分享质量
- 内容是否有价值
- 环节设计是否有趣

第三个维度：社交连接
- 是否认识新朋友
- 是否有互动机会
- 是否建立了有意义的连接

第四个维度：离场体验
- 是否感受到情感关怀
- 是否有纪念品或收获
- 是否期待下次再来

结果：围绕这个框架设计执行方案，活动效果很好。每个维度都有具体的执行要点，团队成员可以分工协作，确保各个方面都得到妥善处理。

#### 1.5 框架式思维与提问的艺术

在讲解框架式思维时，黄老师特别强调了“提问的艺术”。他认为，一个好的问题才能导向好的答案，而一个烂问题往往会让人无从回答。

让我们来看一组问题的对比：

**烂问题示例：**
- “运营该怎么做？”
- “如何提升用户增长？”
- “怎么做社群运营？”

这些问题之所以是烂问题，是因为：
- 问题太宽泛，无法给出具体答案
- 缺乏背景信息，无法针对性回答
- 回答成本太高，回报太低

**好问题示例：**
- “一家面向中小企业的SaaS产品，在探索期应该如何做用户运营？”
- “一个拥有10万用户的工具类产品，次月留存率只有20%，应该如何分析并提升？”
- “一个500人的微信社群，目前活跃度较低，有什么好的激活策略？”

这些问题之所以是好问题，是因为：
- 问题边界清晰，有具体的背景信息
- 目标明确，可以量化评估
- 回答成本适中，答案具有可操作性

如何将烂问题转化为好问题？黄老师提供了一个很好的思路：

原始问题：“to B类产品的运营该怎么做？”

转化为好问题的三种方式：
1. “to B类产品的运营，我们应该重点关注哪几个业务环节？”（聚焦业务环节）
2. “to B类产品的运营，在哪几个地方可能跟to C类产品的运营有所不同？”（聚焦差异点）
3. “to B类产品的运营，如果要在一年内实现从0到1的突破，最重要的三个里程碑是什么？”（聚焦阶段目标）

#### 1.6 框架式思维的心理学基础

从心理学角度来看，框架式思维的价值可以从以下几个方面来理解：

**认知负荷理论**

人类的认知资源是有限的。当我们面对一个复杂问题时，如果没有框架，我们需要同时处理大量的信息，这会导致认知负荷过高，思考效率下降。而有了框架之后，我们可以将问题分解为若干个小问题，每次只专注于一个小问题，认知负荷大大降低。

**组块原理**

框架实际上是将相关信息“组块化”的过程。将零散的信息组织成有意义的组块，可以大大提高记忆和理解的效率。当你有一个清晰的框架时，你只需要记住框架的结构，就可以在需要时快速调用相关的信息。

**模式识别**

经验丰富的运营人员之所以能够快速解决问题，很大程度上是因为他们能够识别问题的模式。当你积累了足够的框架之后，遇到新问题时，你可以快速匹配到相似的框架，从而快速找到解决思路。

### 主题二：如何训练养成框架式思维

#### 2.1 框架式思维不是天生的，需要后天训练

框架式思维不是天生具备的，需要后天训练养成。很多人在面对问题时习惯于“直接给答案”，而忽略了建立框架的过程。这种习惯往往源于：
- 缺乏系统性的思维训练
- 急于求成的心态
- 对框架式思维价值的认识不足

训练框架式思维的核心在于：面对任何问题，先不要急着给答案，而是先建立思考框架。这个过程可能看起来“慢”，但实际上是最“快”的方法，因为没有框架的答案往往是片面的、低质量的。

#### 2.2 训练框架式思维的两个核心方法

**方法一：自上而下法**

适用场景：对该领域有一定了解时。

操作步骤：
1. 先尝试划分层次：这个问题可以分成哪几个层次？
2. 然后逐层填充内容
3. 框架一开始可能比较粗糙，但会逐渐变得简洁

案例：理解“用户运营”

第一层拆分：
- 用户运营可以分成两个方向：提升用户规模、提升活跃用户规模

第二层拆分（提升用户规模）：
- 早期创业公司：细分领域切入 → 培育种子用户 → 口碑传播
- 大公司：开源（增加流量入口、提升转化率）+ 节流（召回渠道、召回方法）

第三层拆分（开源）：
- 渠道1：搜索引擎优化（SEO）
- 渠道2：社交媒体营销
- 渠道3：内容营销
- 渠道4：合作推广

**方法二：自下而上法**

适用场景：对该领域完全陌生时。

操作步骤：
1. 先海量出去问一圈、看一圈
2. 收集大量答案
3. 回头整理归类
4. 提炼出框架

案例：理解“用户运营”

第一步：海量收集
- 方法1：跑到群里问做用户运营的同学
- 方法2：去拉勾等招聘网站爬取职位描述
- 方法3：查阅相关书籍和文章
- 方法4：参加行业分享，听取专家观点

第二步：收集到的答案（示例）
- 管理核心用户群
- 设计用户激励体系
- 提升用户体验
- 策划活动
- 用户付费转化
- 老用户续费
- 提升注册登录转化率
- 用户分类管理
- 搭建用户成长体系
- 沉默用户激活召回
- 负责对外推广营销

第三步：整理归类
- 分类一：用户分类管理、维护和召回
- 分类二：提升付费用户规模
- 分类三：通过产品优化和机制影响活跃用户规模
- 分类四：外围推广拉新

第四步：提炼框架
用户运营 = 用户获取 + 用户激活 + 用户留存 + 用户付费 + 用户传播

#### 2.4 框架式思维训练的具体步骤

**第一步：识别问题类型**

当你面对一个问题时，首先要识别这个问题的类型：
- 是开放性问题还是封闭性问题？
- 是战略层面的问题还是执行层面的问题？
- 是新问题还是老问题的变体？

不同类型的问题可能需要不同的处理方式。

**第二步：明确问题边界**

在建立框架之前，需要明确问题的边界：
- 问题的范围是什么？
- 有哪些约束条件？
- 期望的产出是什么？

明确问题边界可以帮助你建立一个更精准的框架。

**第三步：建立框架**

根据问题的类型和边界，建立相应的思考框架：
- 如果是并列结构的问题，考虑用“分支方向式”框架
- 如果是流程结构的问题，考虑用“工作流程式”框架
- 如果是量化目标的问题，考虑用“公式型”框架

**第四步：填充内容**

在框架建立之后，填充具体的内容：
- 每个维度下的具体要素
- 每个步骤的具体动作
- 每个公式的具体因子

**第五步：验证和完善**

填充内容后，需要验证框架的完整性：
- 是否有遗漏的重要维度？
- 各维度之间是否逻辑清晰？
- 框架是否能够回答原始问题？

如果发现问题，及时调整和完善。

#### 2.5 框架式思维训练的日常实践

将框架式思维融入日常工作，需要养成以下几个习惯：

**习惯一：遇到问题先画框架**

当你遇到问题时，不要急着寻找答案，而是先画一个框架。这个框架可以很简单，可能只是几个关键词的罗列，但这是一个好的开始。

**习惯二：定期复盘和沉淀**

每完成一个项目，都要进行复盘，并将你的思考框架沉淀下来。这些框架将成为你宝贵的知识资产。

**习惯三：主动学习他人的框架**

在学习和交流中，关注他人是如何思考问题的。尝试理解和吸收他人的框架，丰富自己的框架库。

**习惯四：跨界借鉴**

不同领域的框架往往可以相互借鉴。尝试将其他领域的框架应用到运营工作中，可能会产生意想不到的效果。

**习惯五：持续迭代优化**

框架不是一成不变的。随着经验的积累，你需要不断迭代和优化你的框架，使其更加完善和高效。

#### 2.6 框架式思维的进阶技巧

**技巧一：组合框架**

不要试图用一个框架解决所有问题。很多时候，将多个框架组合使用效果更好。

例如，在分析“如何提升产品销量”时，你可以同时使用：
- 用户生命周期框架：获取 → 激活 → 留存 → 收入 → 推荐
- 营销漏斗框架：曝光 → 点击 → 访问 → 咨询 → 购买
- 4P营销框架：产品、价格、渠道、促销

**技巧二：动态框架**

很多问题不是静态的，而是动态变化的。在建立框架时，需要考虑时间维度的变化。

例如，在分析产品的发展策略时，可以建立这样的动态框架：
- 短期（1-3个月）：快速验证、迭代优化
- 中期（3-6个月）：规模化增长、建立壁垒
- 长期（6-12个月）：生态布局、持续领先

**技巧三：分层框架**

复杂问题往往涉及多个层次。在建立框架时，需要考虑层次之间的逻辑关系。

例如，在分析“如何提升用户满意度”时，可以建立分层框架：
- 表面层：产品功能、服务态度、响应速度
- 中间层：产品体验、流程设计、价值感知
- 深层：品牌理念、用户关系、情感连接

### 主题三：三种框架式拆解思路

#### 3.1 三种拆解思路概述

在运营实践中，常用的拆解思路有三种：黄老师称之为“三板斧”——分支方向式、工作流程式、公式型。

这三种拆解思路分别适用于不同类型的问题：
- 分支方向式适用于需要列出多个并列选项的问题
- 工作流程式适用于需要按照步骤执行的问题
- 公式型适用于需要量化分析的问题

掌握这三种拆解思路，可以帮助你应对大部分运营问题。

#### 3.2 分支方向式拆解

**定义：** 按并列的分支方向进行拆解。

**核心逻辑：** 将一个整体目标拆解为若干个并列的子目标，每个子目标独立但共同构成整体目标。

**适用场景：**
- 需要列出多个并列选项的问题
- 目标可以从不同维度分解
- 各子目标之间相对独立

**示例一：用户来源分析**

```
用户来源 = 渠道A + 渠道B + 渠道C + 渠道D
           ↓        ↓        ↓        ↓
        策略1    策略2    策略3    策略4
```

具体应用：
- 渠道A（搜索引擎）：优化关键词、提升着陆页质量
- 渠道B（社交媒体）：内容营销、KOL合作
- 渠道C（直接流量）：品牌建设、用户口碑
- 渠道D（合作渠道）：异业合作、流量置换

**示例二：收入来源分析**

```
收入 = 付费用户数 × 客单价
     = (用户类型1 + 用户类型2 + 用户类型3) × 平均客单价
```

具体应用：
- 个人用户：定价198元/年的基础会员
- 企业用户：定价4999元/年的企业版
- 渠道用户：定制化服务，按项目收费

**示例三：运营工作拆解**

```
运营工作 = 内容运营 + 用户运营 + 活动运营 + 数据运营
           ↓         ↓        ↓         ↓
         具体任务  具体任务  具体任务  具体任务
```

#### 3.3 工作流程式拆解

**定义：** 按时间顺序或业务流程进行拆解。

**核心逻辑：** 将一个复杂过程拆解为若干个有序的步骤，每个步骤都有明确的目标和产出。

**适用场景：**
- 需要按照步骤执行的问题
- 问题有明确的先后顺序
- 各步骤之间相互依赖

**示例一：用户转化流程**

```
用户转化：吸引 → 了解 → 试用 → 购买 → 复购
           ↓      ↓      ↓      ↓      ↓
        优化点  优化点  优化点  优化点  优化点
```

具体分析：
- 吸引：用户如何发现我们？（渠道优化、文案优化）
- 了解：用户如何了解我们？（着陆页优化、内容优化）
- 试用：用户如何开始使用？（注册流程优化、新手引导）
- 购买：用户如何完成付费？（支付流程优化、促销策略）
- 复购：用户如何持续购买？（会员体系、积分体系）

**示例二：活动策划流程**

```
活动策划：策划 → 准备 → 执行 → 复盘
           ↓      ↓      ↓      ↓
        具体任务  具体任务  具体任务  具体任务
```

具体分析：
- 策划阶段：确定目标、明确主题、设定规则、预算规划
- 准备阶段：资源协调、物料准备、测试预演、应急预案
- 执行阶段：实时监控、问题处理、用户互动、氛围营造
- 复盘阶段：数据收集、效果评估、经验总结、改进计划

**示例三：产品上线流程**

```
产品上线：需求确认 → 设计开发 → 测试验收 → 灰度发布 → 全量上线
           ↓          ↓          ↓          ↓          ↓
        具体任务    具体任务    具体任务    具体任务    具体任务
```

#### 3.4 公式型拆解

**定义：** 将目标拆解为可计算的公式。

**核心逻辑：** 将一个结果目标拆解为若干个因子的乘积，通过提升每个因子来提升整体结果。

**适用场景：**
- 需要量化分析的问题
- 结果可以表示为若干因子的乘积
- 各因子之间存在逻辑上的乘法关系

**示例一：销售额公式**

```
销售额 = 流量 × 转化率 × 客单价 × 复购率
          ↓       ↓         ↓        ↓
       提升方法  提升方法   提升方法  提升方法
```

具体分析：
- 流量提升：渠道拓展、内容营销、品牌推广
- 转化率提升：着陆页优化、文案优化、促销策略
- 客单价提升：捆绑销售、升级推荐、限时优惠
- 复购率提升：会员体系、积分体系、个性化推荐

**示例二：利润公式**

```
利润 = (客单价 - 成本) × 销量 - 固定成本
     = 毛利 × 销量 - 固定成本
```

**示例三：用户生命周期价值公式**

```
LTV = (ARPU × 毛利率) / 流失率
    = 单用户贡献 × 留存率
```

#### 3.5 三种拆解思路的综合应用

在实际问题中，往往需要综合运用多种拆解思路。让我通过一个完整的案例来说明：

**案例：如何提升电商平台的月销售额**

**第一步：使用公式型拆解**

```
月销售额 = 日均UV × 转化率 × 客单价 × 30天
         = 10000 × 3% × 150 × 30
         = 135万元
```

**第二步：使用分支方向式拆解（针对每个因子）**

针对日均UV：
```
日均UV = 渠道A + 渠道B + 渠道C + 渠道D
```

针对转化率：
```
转化率 = 注册转化率 × 首次购买转化率
```

针对客单价：
```
客单价 = 商品A销售占比 × 单价 + 商品B销售占比 × 单价 + ...
```

**第三步：使用工作流程式拆解（针对关键环节）**

用户购买流程：
```
访问 → 注册 → 浏览 → 加购 → 下单 → 支付 → 完成
```

每个步骤都需要进行优化。

**第四步：制定综合策略**

基于以上拆解，制定综合策略：
1. 渠道优化：重点投入转化率高的渠道
2. 注册流程简化：减少注册步骤，提升转化
3. 商品结构优化：调整商品组合，提升客单价
4. 购物流程优化：减少支付环节的流失

### 主题四：认知型框架与行动指导型框架

#### 4.1 两种框架类型的区别

在框架式思维中，有两种不同类型的框架：认知型框架和行动指导型框架。

**认知型框架**

定义：帮助你“理解”问题的框架，但不直接指导行动。

特点：
- 告诉你“是什么”
- 提供认知上的整理和归类
- 不涉及具体的执行动作

示例：
```
用户运营
├── 用户获取
│   ├── 渠道推广
│   └── 内容营销
├── 用户留存
│   ├── 活跃激励
│   └── 沉默召回
└── 用户变现
    ├── 付费转化
    └── 会员体系
```

这个框架帮助你理解用户运营包含哪些方面，但并没有告诉你具体应该怎么做。

**行动指导型框架**

定义：不仅帮助你理解问题，还直接指导你“怎么做”。

特点：
- 告诉你“怎么做”
- 包含具体的执行动作和标准
- 可直接用于指导行动

示例：
```
用户运营（目标：月活用户增长20%）
├── 用户获取（目标：新用户5000人）
│   ├── 渠道A
│   │   ├── 动作1：优化着陆页（完成标准：转化率≥3%）
│   │   ├── 动作2：投放预算2万元
│   │   └── 动作3：每周分析数据并优化
│   └── 渠道B
│       └── ...
├── 用户留存（目标：留存率≥40%）
│   ├── 活跃激励
│   │   ├── 动作1：上线积分体系
│   │   ├── 动作2：每周策划互动活动
│   │   └── 完成标准：周活跃率≥25%
│   └── 沉默召回
│       ├── 动作1：梳理召回渠道（短信/Push/邮件）
│       ├── 动作2：设计召回文案
│       └── 完成标准：召回率≥5%
└── 用户变现
    └── ...
```

#### 4.2 为什么要从认知型框架升级为行动指导型框架

在运营工作中，我们经常遇到这样的困境：
- 有了框架，但不知道如何落地执行
- 知道要做用户运营，但不知道具体从哪里入手
- 计划做活动，但不知道每个阶段应该做什么

这些问题的根源在于：我们只建立了认知型框架，而没有将其升级为行动指导型框架。

认知型框架的价值是有限的，它只能帮助我们理解问题的结构，但不能直接指导行动。而行动指导型框架则可以直接转化为执行计划，大大提高工作效率。

#### 4.3 如何将认知型框架升级为行动指导型框架

**步骤一：明确目标**

将抽象的目标具体化、量化。

例如：
- 原始目标：提升用户规模
- 升级后目标：月活用户增长20%，从10万增长到12万

**步骤二：拆解动作**

围绕目标，拆解出具体的执行动作。

例如：
- 目标：月活用户增长20%
- 动作1：通过渠道A获取新用户，目标5000人
- 动作2：通过渠道B获取新用户，目标3000人
- 动作3：通过召回沉睡用户，目标召回1000人

**步骤三：设定标准**

为每个动作设定明确的完成标准。

例如：
- 动作1完成标准：渠道A新用户≥5000人，单个获客成本≤20元
- 动作2完成标准：渠道B新用户≥3000人，单个获客成本≤15元
- 动作3完成标准：召回用户≥1000人，召回成本≤5元/人

**步骤四：确定顺序**

明确动作的优先级和执行顺序。

例如：
- 第一优先级：动作1（成本可控，效果可预期）
- 第二优先级：动作2（需要测试优化）
- 第三优先级：动作3（作为补充手段）

#### 4.4 行动指导型框架的案例

**案例：从“做好内容运营”到具体行动计划**

原始认知型框架：
```
内容运营
├── 内容生产
├── 内容分发
└── 内容效果
```

升级后的行动指导型框架：
```
内容运营（目标：月发布文章20篇，平均阅读量≥5000）

1. 内容生产（目标：每周5篇）
   1.1 行业热点分析
       - 动作：每周一分析本周热点
       - 完成标准：输出热点清单
   1.2 原创文章撰写
       - 动作：每周撰写2篇原创文章
       - 完成标准：每篇≥2000字，包含3个以上案例
   1.3 用户投稿收集
       - 动作：每周收集3篇用户投稿
       - 完成标准：通过审核≥1篇

2. 内容分发（目标：触达用户100万）
   2.1 公众号推送
       - 动作：每周3篇推送
       - 完成标准：平均打开率≥15%
   2.2 社分发
       - 动作：每天在10个微信群分享
       - 完成标准：群内平均阅读量≥200
   2.3 平台分发
       - 动作：同步分发到5个内容平台

       - 完成标准：输出周数据报告
```

### 主题五：框架式思维在实际工作中的应用

#### 5.1 框架式思维在问题分析中的应用

当遇到复杂问题时，框架式思维可以帮助你系统地分析问题：

**第一步：识别问题类型**

判断问题属于哪种类型：
- 战略问题还是执行问题？
- 新问题还是老问题？
- 单一问题还是复合问题？

**第二步：建立分析框架**

根据问题类型选择合适的分析框架：
- 使用公式型框架分析量化目标
- 使用分支方向式框架分析并列要素
- 使用工作流程式框架分析过程问题

**第三步：收集信息和数据**

在框架的指导下收集信息和数据：
- 哪些因素影响这个目标？
- 各因素的现状如何？
- 哪些因素是短板？

**第四步：制定解决方案**

基于分析结果制定解决方案：
- 明确要提升哪些因素
- 制定具体的提升措施
- 设定可量化的目标

#### 5.2 框架式思维在方案设计中的应用

当需要设计一个方案时，框架式思维可以帮助你确保方案的完整性：

**第一步：明确目标**

清晰定义方案要达成的目标：
- 目标是什么？
- 目标如何量化？
- 目标的时间？

**第二步：节点建立方案框架**

设计方案的总体框架：
- 需要包含哪些模块？
- 各模块之间的关系是什么？
- 重点模块是哪个？

**第三步：填充详细内容**

在框架内填充详细内容：
- 每个模块的具体内容是什么？
- 需要什么资源支持？
- 如何评估效果？

**第四步：检查和完善**

检查方案的完整性：
- 是否有遗漏的重要方面？
- 各部分逻辑是否清晰？
- 是否具有可执行性？

#### 5.3 框架式思维在沟通表达中的应用

框架式思维可以大大提升沟通的效率：

**表达时的应用**

在向他人表达观点时，先给出框架再填充细节：
- 先说结论
- 然后解释框架
- 最后填充具体案例和数据

**倾听时的应用**

在听取他人表达时，尝试识别对方的框架：
- 对方在哪个框架下思考？
- 框架是否合理？
- 有哪些需要补充的方面？

**讨论时的应用**

在团队讨论时，先建立共识的框架：
- 先确定讨论的框架
- 在框架内进行讨论
- 最后总结各方的观点

## 案例分析

### 案例一：某在线教育平台的用户增长方案设计

**背景：**

某在线教育平台拥有10万注册用户，但月活跃用户只有1万，付费用户只有1000人。平台希望提升用户活跃度和付费转化率。

**问题分析：**

使用框架式思维分析问题：

第一步：使用公式型框架拆解收入
```
月收入 = 月活用户数 × 付费转化率 × 客单价
       = 10000 × 10% × 500
       = 50万元
```

第二步：使用分支方向式框架分析用户结构
```
月活用户 = 新用户 + 老用户
新用户 = 渠道A + 渠道B + 渠道C
老用户 = 活跃用户 + 沉默用户
```

第三步：使用工作流程式框架分析用户转化
```
用户转化：注册 → 首次使用 → 重复使用 → 付费 → 复购
```

**诊断结果：**

通过分析发现以下问题：
1. 新用户激活率低：注册后7天内完成首次学习的用户只有30%
2. 老用户活跃度下降：3个月前活跃的用户，现在只有20%还在活跃
3. 付费转化率低：活跃用户中付费比例只有10%

**解决方案：**

基于诊断结果制定解决方案：

1. 提升新用户激活率
   - 优化新手引导流程（工作流程式框架）
   - 设计新人专属福利（激励框架）
   - 建立7日成长体系（用户成长框架）

2. 提升老用户活跃度
   - 建立用户分层体系（分支方向式框架）
   - 设计唤醒机制（公式型框架）
   - 优化内容推荐算法（个性化推荐框架）

3. 提升付费转化率
   - 优化课程产品结构（产品框架）
   - 设计会员权益体系（激励框架）
   - 开展限时促销活动（活动框架）

**实施效果：**

通过以上方案的实施，6个月后：
- 月活跃用户从1万增长到3万
- 付费转化率从10%提升到15%
- 月收入从50万元增长到225万元

### 案例二：某电商平台的转化率优化

**背景：**

某电商平台月UV为50万，但月销售额只有75万元，转化率和客单价都有提升空间。

**问题分析：**

使用公式型框架拆解：
```
销售额 = UV × 转化率 × 客单价
      = 500000 × 1% × 150
      = 75万元
```

可能的优化方向：
1. 提升UV：从50万提升到60万
2. 提升转化率：从1%提升到1.5%
3. 提升客单价：从150元提升到180元

**具体分析：**

使用分支方向式框架分析UV构成：
```
UV = 搜索渠道 + 社交渠道 + 直接渠道 + 合作渠道
```

使用工作流程式框架分析转化漏斗：
```
访问 → 浏览 → 加购 → 下单 → 支付
```

**解决方案：**

1. 渠道优化（提升UV）
   - 搜索渠道：优化SEO，提升自然流量
   - 社交渠道：内容营销，提升社交流量
   - 合作渠道：异业合作，拓展流量来源

2. 流程优化（提升转化率）
   - 优化着陆页：提升首屏吸引力
   - 优化商品页：丰富商品信息
   - 优化购物车：减少加购流失
   - 优化支付流程：减少支付流失

3. 商品优化（提升客单价）
   - 捆绑销售：相关商品组合优惠
   - 推荐升级：高价商品推荐
   - 促销活动：满减、满赠活动

**实施效果：**

通过以上方案的实施，3个月后：
- UV从50万提升到65万
- 转化率从1%提升到1.8%
- 客单价从150元提升到200元
- 月销售额从75万元增长到234万元

### 案例三：某社区产品的内容运营策略设计

**诊断结果：**

- 日均内容产出从1000篇提升到5000篇

- 用户日均阅读时长从5分钟提升到15分钟

## 核心概念

| 概念 | 定义 |
|------|------|
| 框架式思维 | 一种系统性的思考方式，帮助把复杂问题拆解成有序的部分，从而看清问题全貌 |
| 分支方向式拆解 | 按并列的分支方向进行拆解，适用于需要列出多个并列选项的问题 |
| 工作流程式拆解 | 按时间顺序或业务流程进行拆解，适用于需要按照步骤执行的问题 |
| 公式型拆解 | 将目标拆解为可计算的公式，适用于需要量化分析的问题 |
| 认知型框架 | 帮助你“理解”问题的框架，但不直接指导行动 |
| 行动指导型框架 | 不仅帮助你理解问题，还直接指导你“怎么做” |
| 问题拆解 | 将复杂问题分解为若干个可处理的小问题的过程 |
| 思考框架 | 用于组织和结构化思考的工具，帮助系统性地分析问题 |

## 本章小结

本章核心要点总结：

1. **框架式思维的价值**：
   - 帮助我们往正确的方向思考，避免“只见树木不见森林”的困境
   - 好的框架能让复杂问题变得可控
   - 提高思考效率、确保全面性、便于沟通、促进知识积累

2. **框架式思维的训练**：
   - 遇到问题先建立框架，再填充内容
   - 两种建立框架路径：自上而下（已有一定了解）和自下而上（完全陌生）
   - 框架最终应该是简洁的，为你服务

3. **三种拆解思路**：
   - 分支方向式：按并列分支拆解
   - 工作流程式：按业务流程拆解
   - 公式型：按量化公式拆解
   - 三种思路可以综合运用

4. **认知型框架 vs 行动指导型框架**：
   - 认知型框架帮助你理解“是什么”
   - 行动指导型框���告诉你“怎么做”
   - 升级方法：明确目标 → 拆解动作 → 设定标准 → 确定顺序

5. **重要启示**：
   - 好的问题才能导向好的答案
   - 观点是最无力的表达，事实和逻辑更有说服力
   - 框架式思维也能帮助我们与他人更高效地沟通
   - 框架需要不断迭代优化

### 主题六：框架式思维在运营实践中的高级应用

#### 6.1 运营问题诊断框架

在运营工作中，经常需要诊断产品或业务存在的问题。框架式思维可以帮助我们系统地识别和分析问题。

**运营问题诊断的五个层次**

第一层：现象描述
首先需要清晰地描述观察到的现象。这一层需要回答：具体看到了什么？数据表现如何？

第二层：问题归因
分析现象背后的原因。这一层需要回答：为什么会出现这种现象？可能的原因是什么？

第三层：影响评估
评估问题的影响程度。这一层需要回答：这个问题有多严重？影响范围有多大？

第四层：优先级判断
判断问题的处理优先级。这一层需要回答：这个问题应该优先处理吗？与其他问题相比如何？

第五层：解决方案
制定问题的解决方案。这一层需要回答：如何解决这个问题？需要什么资源和时间？

```
运营问题诊断框架示例：

现象描述层
├── 指标表现：DAU下降20%
├── 时间范围：过去两周持续下降
└── 影响范围：安卓端下降更明显

问题归因层
├── 外部因素：竞品推出新功能、季节性因素
├── 产品因素：最新版本存在Bug、体验问题
├── 运营因素：活动减少、推送频次下降
└── 技术因素：页面加载变慢、接口异常

影响评估层
├── 用户影响：活跃用户流失、付费下降
├── 业务影响：收入减少、口碑受损
└── 竞争影响：市场份额下降

优先级判断层
├── 紧急程度：高（需要立即处理）
├── 重要程度：高（影响核心指标）
└── 处理优先级：P0

解决方案层
├── 短期方案：Bug修复、紧急活动
├── 中期方案：产品优化、运营加强
└── 长期方案：战略调整、能力建设
```

#### 6.2 运营策略规划框架

框架式思维不仅用于问题诊断，还可以用于策略规划。一个好的策略规划框架可以帮助我们制定更加科学、系统的运营策略。

**运营策略规划的六个步骤**

第一步：现状评估
评估当前的业务状况，包括市场规模、竞争格局、用户需求、产品状态等。

第二步：目标设定
设定明确、可量化的目标。目标应该是具体的、可衡量的、可实现的、相关的、有时限的（SMART原则）。

第三步：路径设计
设计实现目标的路径。这需要考虑可选的路径、各路径的优劣势、资源需求等。

第四步：资源配置
确定需要投入的资源，包括人力、资金、时间、渠道等。

第五步：执行计划
制定具体的执行计划，包括时间表、责任人、里程碑等。

第六步：监控调整
建立监控机制，定期评估进展，根据实际情况调整策略。

```
运营策略规划框架示例：

目标：年度活跃用户数增长100%

现状评估
├── 市场：市场规模10亿，用户渗透率20%
├── 竞争：主要竞品3家，市场份额30%
├── 用户：核心用户群25-35岁，需求是效率提升
└── 产品：功能完善，体验良好，但增长乏力

目标设定
├── 总体目标：MAU从100万增长到200万
├── 阶段目标：Q1达到120万，Q2达到150万...
├── 质量目标：留存率不低于40%
└── 效率目标：获客成本降低20%

路径设计
├── 路径A：渠道投放为主，预计投入500万
├── 路径B：内容营销为主，预计投入200万
├── 路径C：产品驱动增长，预计投入300万
└── 推荐路径：C为主，A为辅

资源配置
├── 预算：500万
├── 人力：增长团队10人
├── 技术：增长相关开发资源
└── 渠道：信息流、ASO，内容平台

执行计划
├── Q1：产品优化+小规模测试
├── Q2：放量投放+活动运营
├── Q3：持续优化+新渠道拓展
└── Q4：品牌建设+年度冲刺

监控调整
├── 核心指标：MAU、留存、CAC
├── 预警机制：周报+异常预警
└── 调整机制：月度复盘+策略迭代
```

#### 6.3 框架式思维与创新

框架式思维不仅可以帮助我们系统地思考问题，还可以促进创新。

**框架式思维促进创新的三种方式**

方式一：框架内的创新
在已有的框架内寻找创新点。这需要我们对现有框架的各个部分有深入的理解，然后思考如何优化每个部分。

方式二：框架重组
将不同领域的框架进行重组，创造新的视角。这种方法往往能够带来意想不到的创新。

方式三：框架突破
完全突破现有框架，建立全新的思考方式。这种方法风险较大，但一旦成功，可能带来革命性的创新。

**框架式思维与设计思维**

设计思维是一种以用户为中心的创新方法，与框架式思维有很强的互补性。

设计思维的核心步骤包括：
1. 同理心：理解用户的需求和痛点
2. 定义：明确要解决的问题
3. 构思：生成大量的创意想法
4. 原型：将想法转化为可测试的原型
5. 测试：测试原型，收集反馈

框架式思维可以在设计思维的每个步骤中发挥作用。

#### 6.4 框架式思维与决策

框架式思维在决策过程中也有重要价值。一个好的决策框架可以帮助我们做出更加科学、理性的选择。

**决策框架的五个要素**

要素一：决策目标
明确要做出什么样的决策，决策要达到什么目标。

要素二：可选方案
列出所有可能的方案选项。尽量列出更多的选项，避免思维定式。

要素三：评估标准
确定评估各方案的标准。标准应该是全面的、可衡量的。

要素四：方案评估
基于评估标准，对各方案进行评估。可以用打分、对比等方式。

要素五：决策选择
基于评估结果，选择最优方案。同时要考虑风险和不确定性。

### 主题七：框架式思维的高级训练方法

#### 7.1 框架积累与复用

框架式思维需要长期积累。一个成熟的运营人员应该有自己的框架库，遇到问题时可以直接调用。

**框架积累的三个层次**

第一层：通用框架
适用于各种场景的基本框架，如SWOT分析、波特五力模型、商业画布等。这些框架具有普适性，可以广泛应用。

第二层：行业框架
针对特定行业的框架，如电商的用户转化漏斗、在线教育的学习路径框架、社交产品的传播模型等。这些框架针对行业特点进行了优化。

第三层：企业框架
针对特定企业的框架，是企业在实践中总结出来的经验结晶。这些框架最具针对性，但也最不可复制。

#### 7.2 框架迭代与优化

框架不是一成不变的，需要根据实践反馈不断迭代和优化。

**框架迭代的触发条件**

触发条件一：实践检验
框架在实践中被发现存在缺陷或不足。

触发条件二：环境变化
外部环境发生重大变化，���有框架不再适用。

触发条件三：认知升级
通过学习或思考，对问题有了更深的理解。

#### 7.3 框架式思维的常见误区

在运用框架式思维时，需要注意避免以下常见误区：

**误区一：框架至上**
有些人过于迷信框架，认为任何问题都可以用框架来解决。这种想法是危险的。

**误区二：框架僵化**
有些人使用框架时过于僵化，不知道变通。

**误区三：框架过度**
有些人使用过多的框架，导致思考变得过于复杂。

**误区四：框架与实践脱节**
有些人学习了很多框架，但在实际工作中不会应用。

### 主题八：框架式思维在不同运营场景中的应用

#### 8.1 用户增长场景

在用户增长领域，框架式思维可以帮助我们系统地思考增长策略。

**用户增长框架：AARRR模型**

AARRR模型是用户增长领域最经典的框架，它将用户增长分为五个阶段：

1. Acquisition（获取）：用户如何发现并进入产品？
2. Activation（激活）：用户第一次使用产品时是否有良好的体验？
3. Retention（留存）：用户是否还会继续使用产品？
4. Referral（推荐）：用户是否愿意向他人推荐产品？
5. Revenue（变现）：如何从用户身上获得收入？

#### 8.2 内容运营场景

在内容运营领域，框架式思维可以帮助我们系统地规划内容策略。

**内容运营框架：内容生产-内容分发-内容消费**

内容运营的核心框架包括三个部分：

1. 内容生产：如何生产高质量的内容？
2. 内容分发：如何让内容触达目标用户？
3. 内容消费：如何提升用户的消费体验？

#### 8.3 活动运营场景

在活动运营领域，框架式思维可以帮助我们系统地设计和执行活动。

**活动运营框架：策划-执行-复盘**

活动运营的核心框架包括三个阶段：

1. 策划阶段：如何设计一个成功的活动？
2. 执行阶段：如何确保活动顺利进行？
3. 复盘阶段：如何从活动中学习和改进？

```
- 购买者议价能力

- 如何维护客户关系

收入来源
- 如何从客户那里获得收入

核心资源
- 需要哪些核心资源

关键业务
- 需要开展哪些关键业务

重要合作
- 需要与哪些合作伙伴合作

成本结构
- 运营需要哪些成本
```

#### 12.2 框架式思维在投资决策中的应用

投资决策是企业发展的重要决策，框架式思维可以帮助我们做出更理性的投资决策。

**投资决策框架**

```
投资决策框架：

一、投资评估维度

1. 市场评估
```

#### 9.1 内容运营岗位的框架应用

内容运营是运营工作中最常见的岗位之一，涉及内容的生产、分发、效果评估等多个环节。框架式思维在内容运营中有广泛的应用场景。

**内容运营的核心框架：内容漏斗模型**

```

曝光量（曝光）

   ↓ 约30%流失
完读率（阅读）
   ↓ 约50%流失
互动率（互动）
   ↓ 约60%流失
转化率（转化）

优化策略：
- 提升曝光：渠道优化、标题优化、发布时间优化

- 提升完读：内容结构优化、节奏把控、价值密度
```

**内容选题的框架方法**

内容选题是内容运营的核心环节，好的选题是成功的一半。使用框架式思维可以帮助我们系统地规划选题：

```
内容选题框架：PGC + UGC + 热点

PGC选题（主动规划）：
├── 常规选题（每周固定发布的内容）
│   ├── 行业干货类：每周1篇
│   ├── 案例分析类：每周1篇
│   └── 工具推荐类：每两周1篇
│
└── 专题选题（围绕特定主题的系列内容）
    ├── 主题确定：基于用户需求和业务目标
    ├── 内容规划：系列文章结构设计
    └── 发布节奏：集中发布或分散发布

UGC选题（用户驱动）：
├── 话题活动：定期发起话题讨论
├── 问答征集：收集用户问题并解答
└── 案例征集：邀请用户分享自己的案例

热点选题（实时响应）：
├── 行业热点：政策法规、行业动态
├── 社会热点：节假日、重大事件
└── 竞品动态：竞品的新动作、新功能
```

```
├── What：活动内容是什么？
├── Why：为什么要做这个活动？
├── Who：目标用户是谁？
├── When：活动时间是？
├── Where：活动渠道是？
├── How：活动如何进行？
```

```
│     ├── 用户活跃度：活动期间及之后的活跃变化                 │
│     ├── 用户留存率：活动用户的长期留存情况                   │
│     └── 用户付费率：从免费到付费的转化                      │
│                                                             │
│  3. 活动过程评估                                            │
│     ├── 流程体验：用户参与流程是否顺畅                       │
│     ├── 活动互动：用户互动和反馈情况                         │
│     ├── 问题反馈：活动中的问题和投诉情况                     │
│     └── 竞品对比：与竞品活动的对比分析                       │
│                                                             │
│  4. 品牌影响评估                                            │
│     ├── 品牌曝光：媒体关注和报道情况                         │
│     ├── 口碑评价：用户对活动的正面/负面评价                 │
│     ├── 行业影响：行业内的影响力和认可度                     │
│     └── 长期价值：对品牌资产的长期影响                       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 9.4 数据运营岗位的框架应用

数据运营是通过数据分析来指导运营决策的岗位。框架式思维在数据运营中可以帮助我们系统地建立数据指标体系和进行分析。

**数据指标体系的框架模型**

```
数据指标体系框架：北极星指标 + 洋葱模型

北极星指标：
定义：唯一重要的核心指标
原则：
- 能够衡量产品为用户创造的核心价值
- 能够反映用户的真实活跃度
- 能够被团队成员理解和认同
- 能够指导日常运营决策

示例：
- 电商产品：GMV
- 社交产品：DAU
- 工具产品：使用时长
- 金融产品：交易额

洋葱模型：
┌─────────────────────────────────────┐
│           北极星指标                 │
│         （最终核心指标）              │
├─────────────────────────────────────┤
│         一级指标                     │
│       （业务核心指标）                │
├─────────────────────────────────────┤
│         二级指标                     │
│       （业务过程指标）                │
├─────────────────────────────────────┤
│         三级指标                     │
│       （细分领域指标）                │
├─────────────────────────────────────┤
│         四级指标                     │
│       （操作层面指标）                │
└─────────────────────────────────────┘

示例（电商产品）：
- 北极星指标：GMV
- 一级指标：UV × 转化率 × 客单价
- 二级指标：渠道流量、新客转化、老客复购
- 三级指标：各渠道ROI、各环节转化率、各品类客单价
- 四级指标：具体页面UV、按钮点击率、商品详情页浏览量
```

**数据分析的框架方法**

```
数据分析框架：描述性 → 诊断性 → 预测性 → 规范性

描述性分析（发生了什么）：
├── 数据概览：关键指标的数值和趋势
├── 维度拆解：按照不同维度进行拆分
├── 对比分析：与历史/竞品进行对比
└── 异常识别：发现异常数据波动

诊断性分析（为什么发生）：
├── 归因分析：找到影响结果的关键因素
├── 漏斗分析：找到转化流失的关键环节
├── 用户分析：找到不同用户群体的差异
└── 留存分析：分析用户留存/流失原因

预测性分析（将要发生）：
├── 趋势预测：预测未来的数据趋势
├── 用户预测：预测用户的行为和价值
├── 风险预警：预测可能的风险和问题
└── 机会识别：识别潜在的机会点

规范性分析（应该怎么做）：
├── 策略建议：基于分析给出运营建议
├── 方案评估：评估不同方案的预期效果
├── 资源优化：优化资源分配和投入
└── 行动路径：制定具体的行动计划
```

假设你负责一款面向职场人士的在线学习App的运营工作，这款App刚完成版本升级，新增了“职场实战课程”模块。你需要设计一个运营方案来推广这个新模块。

```
设定具体目标：
```

```
- 运营策略：免费体验、分期付款

沉默用户（长期未活跃用户）：
- 特征：注册但长时间未使用App
```

#### 10.2 综合实战：用户留存提升方案设计

**案例背景**

假设你负责一款工具类App的运营工作，当前面临用户留存率下降的问题：次月留存率从45%下降到35%。你需要设计一个方案来提升用户留存。

**第一步：问题诊断**

使用工作流程式框架进行问题诊断：

```
用户使用流程：下载 → 注册 → 首次使用 → 重复使用 → 形成习惯

诊断各环节问题：
1. 下载 → 注册：注册流程是否顺畅？
   ├── 问题：注册步骤过多，流失率高
   └── 优化：简化注册流程，增加第三方登录

2. 注册 → 首次使用：新用户引导是否清晰？
   ├── 问题：引导内容复杂，用户看不懂
   └── 优化：优化新用户引导，突出核心功能

3. 首次使用 → 重复使用：核心价值是否感知到？
   ├── 问题：用户不知道产品有什么价值
   └── 优化：加强价值展示，提供使用激励

4. 重复使用 → 形成习惯：使用习惯是否养成？
   ├── 问题：缺乏提醒，用户容易忘记
   └── 优化：增加提醒功能，建立使用习惯
```

**第二步：用户分群**

使用分支方向式框架分析不同用户群体：

```
用户分群 = 新用户 + 成长期用户 + 成熟期用户 + 流失风险用户

新用户（0-7天）：
- 问题：首次使用后未继续使用
- 策略：优化新用户引导，强化首次价值感知

成长期用户（7-30天）：
- 问题：使用频率低，未形成习惯
- 策略：建立使用激励，提醒用户使用

成熟期用户（30天+）：
- 问题：可能被竞品吸引
- 策略：持续提供价值，建立情感连接

流失风险用户（30天+未活跃）：
- 问题：长时间未使用
- 策略：召回触达，优惠激励
```

**第三步：策略设计**

使用公式型框架设计留存提升策略：

```
留存率 = 首次价值感知率 × 持续使用率 × 习惯形成率

- 提供新人礼包，提升首次体验

提升持续使用率：
- 建立积分体系，激励每日签到

- 持续推送有价值的内容

提升习惯形成率：

- 设计社交功能，增加用户粘性
- 提供会员权益，锁定长期用户
```

**第四步：执行计划**

### 主题十一：框架式思维的深度心法

#### 11.1 框架式思维与第一性原理

第一性原理是指从最基本的事实和假设出发，不依赖类比或既有观念来推导结论。框架式思维与第一性原理有很强的互补性。

**第一性原理在运营中的应用**

```
传统思维（类比思维）：
- "竞品这样做，我们也这样做"
- "以前是这样做的，今年继续"
- "大家都在做，我们也跟上"

第一性原理思维：
- "用户真正需要的是什么？"
- "这个问题的本质是什么？"
- "有没有更好的解决方式？"

案例：电商平台提升转化率的传统思维 vs 第一性原理思维

传统思维：
- 竞品在做满减活动，我们也做
- 竞品在降价，我们也降价
- 竞品在投放广告，我们也投放

第一性原理思维：
- 用户为什么购买？为了解决问题/满足需求
- 转化过程中的核心障碍是什么？是信任/价格/便捷性
- 如何从根本上解决这些障碍？
  - 信任问题：通过评价、案例、保证来解决
  - 价格问题：通过价值感知、性价比来解决
  - 便捷问题：通过流程优化来解决
```

**框架式思维 + 第一性原理的结合**

```
问题分析流程：

1. 第一性原理：追问本质
   问题：如何提升用户留存？
   ├── 用户为什么留存？因为产品有价值
   ├── 价值是什么？帮助用户解决问题
   ├── 什么问题？不同用户不同问题
   └── 本质：持续提供用户需要的价值

2. 框架式思维：系统拆解
   留存框架 = 价值留存 + 习惯留存 + 情感留存
   ├── 价值留存：产品功能持续满足需求
   ├── 习惯留存：使用场景和使用习惯
   └── 情感留存：品牌认同和情感连接
```

#### 11.2 框架式思维与系统思维

系统思维是考虑系统中各要素之间相互关系和动态平衡的思维方式。框架式思维需要与系统思维结合，才能更好地理解复杂问题。

**系统思维的核心概念**

```
系统思维框架：

要素（Elements）
- 系统中的组成单元
- 例如：用户、产品、内容、渠道

关系（Relationships）
- 要素之间的相互作用
- 例如：因果关系、增强关系、平衡关系

功能（Functions）
- 系统整体要实现的目标
- 例如：用户价值、商业价值

案例：用户增长系统

要素：
├── 产品：提供什么价值
├── 用户：谁在使用
├── 渠道：如何触达用户
└── 运营：如何影响用户

关系：
├── 产品价值 → 用户需求满足 → 用户留存
├── 用户活跃 → 内容产出 → 内容丰富 → 更多用户
├── 渠道投入 → 用户获取 → 用户质量 → 成本效率
└── 运营活动 → 用户参与 → 品牌认知 → 用户增长

功能：
├── 用户价值：帮助用户解决问题
├── 商业价值：实现商业变现
└── 社会价值：创造社会效益
```

**系统思维在运营决策中的应用**

```
运营决策的系统思维：

决策时考虑：
1. 直接影响：这个决策直接影响什么？
2. 间接影响：这个决策间接影响什么？
3. 延迟影响：这个决策短期/长期分别有什么影响？
4. 系统反馈：这个决策会引发什么反馈？

案例：大幅降价促销的决策分析

直接影响：
- 短期销量提升
- 客单价下降
- 利润率下降

间接影响：
- 品牌形象可能受损
- 现有付费用户可能不满
- 可能吸引价格敏感用户

延迟影响：
- 短期：销量大增
- 中期：用户质量下降，复购率下降
- 长期：品牌贬值，难以恢复

系统反馈：
- 降价后恢复原价困难
- 用户形成价格预期，等待降价
- 竞品可能跟进价格战
```

#### 11.3 框架式思维的终极心法

**心法一：框架是工具，不是目的**

框架式思维的目的是帮助我们更好地思考和解决问题，框架本身不是目的。不要为了使用框架而使用框架，要根据实际需要灵活选择和调整框架。

**心法二：框架要简洁，越简洁越好**

一个好的框架应该是简洁的。过于复杂的框架反而会增加认知负担，降低思考效率。如果你发现一个框架变得越来越复杂，考虑简化它。

**心法三：框架要迭代，越迭代越好**

框架不是一成不变的，需要根据实践反馈不断迭代。每使用一次框架，都应该思考：这个框架有哪些不足？下次如何改进？

**心法四：框架要组合，越组合越强**

不同框架可以组合使用，发挥更大的威力。不要局限于单一框架，要学会根据问题特点组合使用多个框架。

**心法五：框架要沉淀，越沉淀越有价值**

将使用过的框架沉淀下来，形成自己的框架库。随着框架库的积累，你的思考效率会越来越高。

### 主题十二：框架式思维与商业决策

#### 12.1 框架式思维在战略规划中的应用

战略规划是企业发展的核心工作，框架式思维在战略规划中发挥着重要作用。

**战略分析框架**

```
常用战略分析框架：

一、SWOT分析框架

Strengths（优势）
- 内部积极因素
- 如：技术优势，品牌优势、资源优势

Weaknesses（劣势）
- 内部消极因素
- 如：资金不足、人才短缺、技术落后

Opportunities（机会）
- 外部积极因素
- 如：市场增长、政策支持、竞争减弱

Threats（威胁）
- 外部消极因素
- 如：经济下行、竞争加剧、政策收紧

战略选择：
- SO策略：利用优势抓住机会
- WO策略：利用机会弥补劣势
- ST策略：利用优势应对威胁
- WT策略：弥补劣势应对威胁

二、波特五力模型

1. 现有竞争者
- 竞争激烈程度
- 主要竞争者实力

2. 潜在进入者
- 进入壁垒高低
- 进入威胁大小

3. 替代品威胁
- 替代品替代程度
- 替代品威胁大小

4. 供应商议价能力
- 供应商集中度
- 供应商议价能力

5. 购买者议价能力
- 购买者数量
- 购买者议价能力

三、商业画布

客户细分
- 目标客户是谁

价值主张
- 为客户创造什么价值

渠道通路
- 如何触达客户

客户关系
- 如何维护客户关系

收入来源
- 如何从客户那里获得收入

核心资源
- 需要哪些核心资源

关键业务
- 需要开展哪些关键业务

重要合作
- 需要与哪些合作伙伴合作

成本结构
- 运营需要哪些成本
```

**战略规划流程**

```
战略规划流程：

一、战略分析
- 外部环境分析
- 内部能力分析
- 竞争态势分析

二、战略制定
- 确定愿景使命
- 设定战略目标
- 制定竞争战略

三、战略分解
- 业务战略
- 职能战略
- 区域战略

四、战略执行
- 制定执行计划
- 分配资源
- 建立考核机制

五、战略评估
- 监控执行情况
- 评估执行效果
- 调整优化策略
```

#### 12.2 框架式思维在投资决策中的应用

投资决策是企业发展的重要决策，框架式思维可以帮助我们做出更理性的投资决策。

**投资决策框架**

```
投资决策框架：

一、投资评估维度

1. 市场评估
- 市场规模
- 增长潜力
- 竞争格局

2. 产品评估
- 产品价值
- 产品差异化
- 产品可行性

3. 团队评估
- 团队能力
- 团队经验
- 团队诚信

4. 财务评估
- 投资回报率
- 投资回收期
- 风险收益比

二、投资决策标准

1. 基本标准
- 市场需求真实
- 商业模式可行
- 团队能力匹配

2. 加分标准
- 差异化明显
- 壁垒较高
- 协同效应强

3. 否决标准
- 市场规模太小
- 商业模式不成立
- 团队能力不足

三、投资决策流程

1. 初筛
- 快速筛选项目
- 过滤明显不符合标准的项目

2. 尽职调查
- 深入了解项目
- 评估项目风险

3. 投资决策
- 评估投资价值
- 做出投资决策

4. 投后管理
- 跟踪项目发展
- 提供增值服务
```
