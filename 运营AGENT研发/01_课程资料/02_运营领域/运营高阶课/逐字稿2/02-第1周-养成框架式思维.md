## 必修1：框架式思维的价值、要点及常见应用

### 1.1 什么是思考框架

**思考框架**是一种结构化的认知方式，它能帮助我们在面对复杂问题时快速理清思路，避免陷入"只见树木，不见森林"的状态。

┌──────────────────────────────────────────┐
│  📖 核心概念                             │
├──────────────────────────────────────────┤
│                                          │
│  **思考框架的价值**                       │
│                                          │
│  • 帮助你往正确的方向思考                 │
│  • 避免陷入细节，把握全局                 │
│  • 提高沟通和说服他人的效率               │
│  • 让提出的问题更容易得到好的答案         │
│                                          │
└──────────────────────────────────────────┘

### 1.2 三个实战案例

#### 案例一：课程设计读书会

**背景**：办公室组织《认知设计》读书会，讨论如何改进课程设计

**没有框架的讨论**：
- 加一个课前测评
- 让学习者边学边动手
- 告诉学习者起点和终点
- 重新规划课程秩序

→ 问题：各说各的，单点讨论，缺乏整体视角

**有框架的梳理**：

```
┌────────────────────────────────────────────┐
│            课程设计框架                     │
├────────────────────────────────────────────┤
│                                             │
│   ┌─────────┐    ┌─────────┐    ┌─────────┐ │
│   │学习动机 │    │学习体验 │    │学习效果 │ │
│   └────┬────┘    └────┬────┘    └────┬────┘ │
│        │              │              │        │
│        ▼              ▼              ▼        │
│   • 课前测评      • 课程秩序      • 进阶难度  │
│   • 目标明确      • 边学边做      • 作业设计  │
│                  • 节奏安排      • 练习反馈  │
│                                             │
└────────────────────────────────────────────┘
```

→ 结果：全局清晰，各要素关系明确，讨论更有针对性

#### 案例二：如何评价《平凡之路》

**没有框架的回答**：
- 这首歌好听
- 这首歌太棒了
- 这是近年来听过最好的一首歌
- 韩寒脑残粉，这就粉了

→ 问题：只有主观情绪，无法帮助思考，难以说服他人

**有框架的回答**：

| 维度 | 分析要点 | 争议程度 |
|------|----------|----------|
| 旋律 | 具体分析旋律特征 | 最小 |
| 编曲 | 具体分析编曲特点 | 最大 |
| 歌词 | 不同人看到不同经历和情怀 | 见仁见智 |

→ 结论：先建立认知框架，再进行拆解分析，更有说服力

#### 案例三：线下活动体验设计

**问题**：如何设计让用户觉得"太赞了"的线下活动体验？

**没有框架的方案**：
- 弄几个游戏区让大家玩一玩
- 多弄点水果、甜点
- 提高会场逼格

→ 问题：零散建议，缺乏系统思考

**有框架的方案**：

```
┌────────────────────────────────────────────┐
│         线下活动用户体验框架                 │
├────────────────────────────────────────────┤
│                                             │
│   ┌────────┐    ┌────────┐    ┌────────┐  │
│   │ 入场   │    │ 现场   │    │ 离场   │  │
│   └───┬────┘    └───┬────┘    └───┬────┘  │
│       │             │             │        │
│       ▼             ▼             ▼        │
│   • 惊喜礼物    • 嘉宾内容    • 情感关怀  │
│   • 互动游戏    • 有料干货    • 暖手物   │
│                                             │
│         ┌────────────────┐                  │
│         │  人与人连接    │                  │
│         │  • 互动机制    │                  │
│         │  • 认识朋友    │                  │
│         └────────────────┘                  │
│                                             │
└────────────────────────────────────────────┘
```

**最终方案**：
- 入场：送一本书（背后有故事）
- 现场：尽人事听天命（嘉宾内容不可控）
- 连接：设计互动机制
- 离场：送暖手物（天冷关怀）

### 1.4 核心要点：好问题 vs 坏问题

**思考框架的最大意义：帮助你往正确的方向思考，提出更好的问题**

| 坏问题（太宽泛） | 好问题（有框架） |
|------------------|------------------|
| 运营该怎么做？ | To B 产品的运营应该重点关注哪几个业务环节？ |
| 拖延症怎么办？ | To B 产品的运营在哪几个方面可能跟 To C 产品不同？ |

┌──────────────────────────────────────────┐
│  💡 好问答的原则                         │
├──────────────────────────────────────────┤
│                                          │
│  一次好的问答，是由问者和答者共同付出    │
│  思考和努力的结果。                      │
│                                          │
│  • 问者有好的思考，带动答者配合          │
│  • 答者更愿意给出具体、有价值的答案      │
│  • 双方共同创造好的结果                 │
│                                          │
└──────────────────────────────────────────┘

### 1.5 本节小结

**思考框架的核心价值**：
1. 帮助你往正确的方向思考
2. 避免"只见树木，不见森林"
3. 提高沟通和说服效率
4. 让问题更容易得到好的答案

**实战应用要点**：
- 先建立框架，再进入细节
- 框架要有清晰的分类和逻辑
- 用框架指导行动，而非零散应对

## 必修2：如何训练养成自己的框架式思维

### 2.1 专业能力的三要素

在职场中，个人在某个专业领域的能力由三部分构成：

```

│         专业能力构成金字塔                 │
├────────────────────────────────────────────┤

│                ╱██████████╲                 │
│               ╱  思考框架  ╲                │
│              ╱──────────────╲               │

│           ╱       技能         ╲            │
│          ╱──────────────────────╲           │
│                                             │
└────────────────────────────────────────────┘
```

#### 三要素详解

| 要素 | 定义 | 特点 | 示例 |
|------|------|------|------|
| **专业知识** | 具体领域中解决问题的具体方法 | 知识可以被学会 | 如何给媒体车马费而不被反感 |
| **思考框架** | 领域内的方法论和认知结构 | 建立整体视角 | 传播 = 内容 × 渠道 × 时机 |
| **技能** | 真正把事情做好的能力 | 技能只能被习得（大量练习） | 能够让媒体开心接受合作 |

┌──────────────────────────────────────────┐
│  📖 核心区别                             │
├──────────────────────────────────────────┤
│                                          │
│  知识：这件事应该怎么做                  │
│  技能：真正能把这件事做好                │
│                                          │
│  知识可以被学会                         │
│  技能只能被习得（需要大量练习）          │
│                                          │
└──────────────────────────────────────────┘

### 2.4 拆解练习：从评估维度到关联影响要素

#### 框架结构

```
┌────────────────────────────────────────────┐
│         拆解框架的基本结构                 │
├────────────────────────────────────────────┤
│                                             │
│                   议题                      │
│                     │                       │
│         ┌───────────┼───────────┐           │
│         │           │           │           │
│    评估维度1    评估维度2    评估维度3       │
│         │           │           │           │
│    ┌────┴────┐ ┌────┴────┐ ┌────┴────┐     │
│    │要素 要素│ │要素 要素│ │要素 要素│     │
│    └─────────┘ └─────────┘ └─────────┘     │
│                                             │
│   第一层：评估维度                           │
│   第二层：关联影响要素                       │
│                                             │
└────────────────────────────────────────────┘
```

#### 拆解练习方法

**起步阶段**：不需要追求评估维度的完整性和严谨性

1. 找到**一个**评估维度
2. 拆解这个维度下的**关联影响要素**
3. 不断重复这个练习

┌──────────────────────────────────────────┐
│  💡 起步要点                             │
├──────────────────────────────────────────┤
│                                          │
│  • 不需要所有评估维度都完整              │
│  • 维度之间有交叉重复也没关系            │
│  • MECE原则（相互独立、完全穷尽）是结果  │
│    不是起步要求                          │
│  • 先让拆解成为习惯，再追求严谨          │
│                                          │
└──────────────────────────────────────────┘

#### 实战案例

**案例1：装修 - 费用维度**

| 评估维度 | 关联影响要素 |
|----------|--------------|
| 费用 | 设计费、物料费、人工费、运输费 |

**案例2：装修 - 办公室舒适度维度**

| 评估维度 | 关联影响要素 |
|----------|--------------|
| 办公室舒适度 | 墙面色彩、空间开阔度（层高、人均面积）、家具风格材质、同事交流便利程度 |

**案例3：线下活动 - 用户体验维度**

| 评估维度 | 关联影响要素 |
|----------|--------------|
| 现场用户体验 | 收到的礼物、娱乐项目、嘉宾是否有料、场地环境、活动流程、情感关怀 |

**案例4：线下活动 - 收入维度**

| 评估维度 | 关联影响要素 |
|----------|--------------|
| 活动收入 | 门票、赞助、周边贩卖、用户打赏、直播收入 |

**案例5：公众号涨粉 - 增长渠道维度**

| 评估维度 | 关联影响要素 |
|----------|--------------|
| 增长渠道 | 朋友圈自发分享、付费大号推广、资源置换互推、免费社区论坛曝光、广点通投放、线下地推 |

### 2.5 框架式思维训练经验

**个人实践**：2012年，花了约3个月时间
- 每天在地铁上看广告
- 用自己的思考逻辑重新分析
- 记录所有思考成果
- 最终成为思考习惯的一部分

### 2.6 本节小结

**核心要点**：
1. 专业能力 = 思考框架 + 专业知识 + 技能
2. 不同经验水平有不同的学习路径
3. 掌握框架式思维：培养习惯 → 建立框架 → 持续优化
4. 拆解练习：从一个评估维度开始，拆解关联影响要素
5. 框架必须能指导实践，否则没有意义

**行动建议**：
- 从日常问题开始拆解练习
- 不要追求完美框架，先让拆解成为习惯
- 记录自己的思考成果
- 在实践中不断优化框架

|----------|----------|
| 鸡汤类 | 情感共鸣、故事化 |
| 干货观点输出类 | 专业深度、逻辑严密 |
| 段子类 | 幽默、反转、节奏感 |
| 科普类 | 通俗易懂、知识点清晰 |

#### 思路二：工作流程式

**定义**：按照从前到后的先后顺序梳理工作流程，每个环节之间有依赖关系。

**案例：写文章的流程**

```
┌────────────────────────────────────────────┐
│         写文章工作流程                      │
├────────────────────────────────────────────┤
│                                             │
│   ①选题 ──▶ ②设计结构 ──▶ ③收集素材       │
│                                              │
│                         │                  │
│                         ▼                  │
│   ⑥编排发布 ◀─ ⑤具体写作 ◀─ ④结合结构    │
│                                             │
└────────────────────────────────────────────┘
```

**要点**：
- 每个环节之间有影响
- 前一个环节没完成，无法进入下一环节
- 不是所有问题都能找到工作流程

#### 思路三：公式型

**定义**：将指标拆解成数学公式，每个子指标都有人负责。

**案例：电商网站总销量**

```
总销量 = 用户数 × 转化率 ×客单价（UP值）

或者

总销量 = 品类一销量 + 品类二销量 + 品类三销量 + 品类四销量
```

**实战案例：滴滴出行运营指标拆解**

```
┌────────────────────────────────────────────┐
│      滴滴订单指标拆解（实际案例）          │
├────────────────────────────────────────────┤
│                                             │
│  司机端订单量 =                             │
│  激活司机数 × 首单率 × 在线率 ×             │
│  在线抢单率 × 人均完成订单数                │
│                                             │
│  乘客端订单数 =                             │
│  注册用户数 × 首单率 × 日活率 ×              │
│  人均订单数 × 应答率 × 被抢率 × 完成率      │
│                                             │
│  价值：每个指标都有专人负责，五六个人的     │
│        团队劲儿往一处使                     │
│                                             │
└────────────────────────────────────────────┘
```

┌──────────────────────────────────────────┐
│  💡 公式型拆解的价值                      │
├──────────────────────────────────────────┤
│                                          │
│  • 每个指标都能找到负责人                 │
│  • 避免团队关注不相关的事情               │
│  • 防止资源浪费                           │
│  • 便于运营团队管理                       │
│                                          │
└──────────────────────────────────────────┘

### 3.3 三句重要的话

记住这三句话，能帮你应对大部分拆解场景：

| 序号 | 内容 | 适用场景 |
|------|------|----------|
| 1 | 任何宽泛不具体的目标，一定可以经由划定边界+1-2次分解，变成可量化的评估维度或可落地事项 | "如何评价xxx"、"如何做好xxx" |
| 2 | 任何可量化的目标（运营指标），一定可以拆解为某种公式 | 日活率、新增用户、留存率等 |
| 3 | 任何具体明确的可落地事项，一定可以找到相应的工作流程 | 写文章、做活动、推广投放 |

### 3.4 实战案例演示

#### 案例一：怎样让自己更有创意

**问题分析**：这个问题太宽泛，"有创意"是形容词，不是具体动作

**第一步：划定边界（加具体动作）**

```
怎样让自己更有创意
         │
         ▼ 加上边界
```

**方法：收集参照物 → 提炼共性**

收集20-30个有创意的线上活动，按"活动形式/逻辑"分类：

| 活动类型 | 特征 | 案例 |

| 反大众行为 | 旗帜鲜明反对某种行为 | 24小时不用手机、程序员放一天假 |

**方法一：工作流程式**

```
      │
      ▼
```

### 3.5 三类常见问题的解决建议

| 问题类型 | 拆解思路 | 案例 |
|----------|----------|------|
| **如何评价xxx** | ①先界定评价目的<br>②寻找评价分支方向<br>③或按工作流程拆解 | 如何评价营销活动 |
| **如何提升xxx** | 唯一思路：拆解成公式 | 如何提升留言量 = 图文阅读数 × 转化率 |
| **如何做好xxx** | ①先界定分支方向<br>②拆解成具体事项<br>③找工作流程 | 如何让自己更有创意 → 如何做有创意的线上活动 |

### 3.6 本节小结

**三种核心拆解思路**：
1. **分支方向式**：找独立方向，各自做好都能贡献价值
2. **工作流程式**：按先后顺序梳理流程
3. **公式型**：将指标拆解成数学公式

**三句重要的话**：
1. 宽泛问题 → 划定边界 + 分解 = 可量化/可落地
2. 可量化目标 → 一定能拆解成公式
3. 可落地事项 → 一定能找到工作流程

**实战要点**：
- 不要追求标准答案，找到合理框架即可
- 拆解结果必须是清晰明确的行动方向
- 最好能被评估和衡量
- 能够指导实践才有意义

## 选修1：如何做好第三方推广

### 1.1 什么是第三方推广

**第三方推广**：一切通过非直接触达用户的手段来实现的推广。

不是通过官方运营人员直接面向用户获取用户，而是通过中间的第三方渠道把用户获取到自己的产品生态中。

```
┌────────────────────────────────────────────┐
│         第三方推广基本模型                 │
├────────────────────────────────────────────┤
│                                             │
│   自己产品 ←────── 用户                    │
│      ▲               │                     │
│      │               │                     │
│      │          直接获取                   │
│      │          （论坛、邮件、地推）        │
│      │                                     │
│      │        第三方渠道                   │
│      └──────（通过中间渠道获取）            │
│                                             │
└────────────────────────────────────────────┘
```

### 1.2 常见的第三方渠道类型

| 渠道类型 | 说明 | 案例 |
|----------|------|------|
| **Banner/Button广告** | 网站横幅或按钮广告位 | 各大门户网站广告位 |
| **网络联盟** | 整合多网站广告位的平台 | 百度网盟、易传媒 |
| **广点通** | 腾讯生态专属广告联盟 | QQ、QQ空间、微信、腾讯网等 |
| **搜索引擎推广** | 关键词竞价/SEO | SEM（付费）、SEO（免费） |
| **朋友圈广告** | 微信朋友圈信息流广告 | 5万起投，可定向投放 |
| **微博/微信大号** | 意见领袖、明星号推广 | 付费发文、指定时间/内容 |
| **社区/论坛推广** | 垂直社区广告位 | CSDN、豆瓣小组、置顶/发帖 |
| **媒体发稿** | 行业媒体软文推广 | 36氪、虎嗅等媒体广告位 |

### 1.3 第三方推广的核心逻辑

```
┌────────────────────────────────────────────┐
│      第三方推广用户转化流程                │
├────────────────────────────────────────────┤
│                                             │
│  第三方渠道          产品端                │
│                                             │
│  用户访问渠道                              │
│       │                                    │
│       ▼                                    │
│  ┌─────────┐                              │
│  │推广展示位│ ◀─── 推广信息出现            │
│  └────┬────┘                              │
│       │                                    │
│       ▼                                    │
│  用户注意到推广                            │
│       │                                    │
│       ▼                                    │
│  ┌─────────┐                              │
│  │推广素材  │ ◀─── 文案/图片/视频等        │
│  └────┬────┘                              │
│       │                                    │
│       ▼                                    │
│  用户被转化                                │
│       │                                    │
│       ▼                                    │
│  ┌─────────┐                              │
│  │着陆页   │ ◀─── 承接用户、引导转化      │
│  └────┬────┘                              │
│       │                                    │
│       ▼                                    │
│  引导到产品内部体验点                      │
│                                             │
└────────────────────────────────────────────┘
```

**关键要素**：
- **推广展示位**：Banner、标题、按钮等
- **推广素材**：文案、图片、视频、活动介绍
- **着陆页（Landing Page）**：专题页、首页等承接页面
- **转化路径**：从着陆页到最终体验点的引导路径

### 1.4 第三方推广操作五步骤

```
┌────────────────────────────────────────────┐
│      第三方推广操作五步骤                  │
├────────────────────────────────────────────┤
│                                             │
│  ①制定推广策略 ──▶ ②确认渠道数据          │
│   • 确认预期         • 用户数/访问量       │
│   • 资源储备         • 人群匹配度          │
│                      • 获客成本            │
│                                             │
│  ③沟通渠道排期 ──▶ ④协调资源执行          │
│   • 提前3-4周        • 落实素材            │
│   • 梳理推广素材     • 执行落地            │
│                      • 结合用户体验场景    │
│                                             │
│           ⑤监测数据评估效果                │
│           • CTR、转化率、留存              │
│           • ROI、获客成本                  │
│           • 调整优化                       │
│                                             │
└────────────────────────────────────────────┘
```

#### 步骤一：制定推广策略

**三种常见推广策略**：

| 策略类型 | 目的 | 渠道选择 | 关注重点 |
|----------|------|----------|----------|
| **尝试性推广** | 测试效果、渠道效率 | 少量垂直渠道（微博公众号、社区） | 推广效率、用户表现 |
| **数据拉升型** | 大幅拉升特定数据 | 多渠道同步：搜索广告、网盟、品牌广告 | 效果、覆盖面 |
| **传播/事件型** | 带动品牌传播 | 社会化媒体（微博、微信） | 事件引爆、刷屏效应 |

**资源评估**：
- 评估已有渠道资源是否足够支持预期
- 不足时需要开拓新渠道

**开拓新渠道的方式**：
1. **百度搜索**：善用搜索引擎发现潜在渠道
2. **用户调研**：访谈目标用户了解常去的地方
3. **加入行业群**：微信群、QQ群、合作群
4. **线下沙龙**：推广和用户增长主题沙龙

#### 步骤二：确认渠道数据和触达逻辑

**需要确认的渠道数据**：
- 渠道用户数、日均访问量
- 地域/人群特征匹配度
- 历史获客成本

**渠道触达逻辑案例**：

**案例1：豆瓣小组**
```
用户访问小组列表页
    │
    ▼
看到推广帖子（需要顶帖或置顶）
    │
    ▼
点击标题（标题要吸引人）
    │
    ▼
消费帖子内容（内容转化）
```

**案例2：微信大号**
```
公众号推送
    │
    ▼
用户打开（受推送位、标题、头图、摘要、时间影响）
    │
    ▼
阅读内容
    │
    ▼
用户转发朋友圈/群（内容质量要高）
    │
    ▼
引发二次消费
```

#### 步骤三：沟通渠道排期和梳理素材

**排期要点**：
- 好渠道需提前3-4周沟通
- 热门渠道可能需提前1-2月预定

**需要准备的推广素材**：

| 素材类型 | 说明 | 准备要点 |
|----------|------|----------|
| **推广展示位素材** | Banner图、标题文案 | 侧重刺激、吸引眼球 |
| **推广内容页** | 长文案、软文、活动介绍 | 侧重逻辑递进、软植入 |
| **着陆页** | 专题页、首页 | 明确用户引导路径 |
| **站内引导路径** | 各体验点的文案/图片 | 评估用户体验友好度 |

**推广素材准备要点**：

1. **短文案/Banner位**
   - 侧重转化率
   - 刺激用户、吸引眼球
   - 示例："北京公寓价格击穿，50万/套起，精装修高性价比"

2. **长文案/文章式**
   - 侧重阅读体验
   - 层层嵌套、逻辑递进
   - 软植入，内容与推广信息相关

3. **着陆页**
   - **活动/专题页**：直接引导消费
   - **首页**：全面了解产品服务
   - 根据期望用户行为选择合适类型

#### 步骤四：协调资源执行落地

**执行要点**：
- 与设计师等协作者清晰传达预期
- 确保所有素材事先准备到位
- 按排期执行落地

**核心原则**：结合用户体验场景和展现形式做好推广素材

#### 步骤五：监测数据评估效果

**单一渠道投放关注指标**：

| 指标 | 说明 | 评估重点 |
|------|------|----------|
| **CTR（点击率）** | 展示点击比 | 推广位置、素材效果 |
| **转化率** | 点击后转化的用户比例 | 推广素材、着陆页有效性 |
| **用户留存** | 渠道用户留存情况 | 渠道与产品匹配度 |

**多渠道投放关注指标**：
- **ROI**（投入产出比）
- **获客成本**（对比行业平均值）
- **单位时间新增用户量**

### 1.5 本节小结

**核心要点**：
1. 第三方推广 = 通过非直接触达手段获取用户
2. 常见渠道：Banner广告、网盟、广点通、搜索、朋友圈、大号、社区、媒体
3. 核心流程：展示位 → 推广素材 → 着陆页 → 转化路径
4. 操作五步：定策略 → 确渠道 → 排期素材 → 执行落地 → 监测优化
5. 推广素材要结合场景：Banner重刺激、长文案重逻辑、着陆页重引导

**实战建议**：
- 好渠道要提前3-4周预定
- 数据是精细化运营的关键
- 建立渠道数据库，做到如指掌
- 持续监测数据，优化调整策略
