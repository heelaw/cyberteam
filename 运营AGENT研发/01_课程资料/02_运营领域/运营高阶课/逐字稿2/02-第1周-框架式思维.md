> 本周核心：掌握框架式思维的基本价值、训练方法，以及三种核心拆解思路

## 一、框架式思维的价值与要点

### 1.1 什么是思考框架

思考框架是一种结构化的认知方式，帮助你：
- 往正确的方向思考
- 从宏观视角理解问题的全貌
- 避免陷入"只见树木不见森林"的困境

### 1.2 三个故事理解框架的价值

#### 故事1：课程设计读书会

**背景**：办公室组织《认知设计》读书会讨论课程优化

**无框架的讨论**：
- 加课前测评
- 让学习者边学边动手
- 告知起点和终点
- 重新规划课程顺序

**有框架的思考**：
```
设计课程 = 学习动机 × 学习体验 × 学习效果

学习动机：多个要素影响
学习体验：几个要点影响
学习效果：课程进阶难度、练习作业设计等
```

#### 故事2：如何评价《平凡之路》

**无价值的回答**：
- "这首歌很好听"
- "这是我近几年听过最好的歌"
- "我是脑残粉，粉朴树"

**有框架的回答**：
```
将歌曲拆解为三部分：
├── 旋律（争议最小）
├── 编曲（争议最大）
└── 歌词（见仁见智，看到的是经历和情怀）

然后针对每一部分进行具体分析
```

#### 故事3：线下活动体验设计

**零散的建议**：
- 弄几个游戏区
- 多准备水果甜点
- 提高会场逼格

**框架化的思考**：
```
线下活动用户体验 = 入场惊喜 × 内容质量 × 人际连接 × 离场关怀

1. 入场时：馈赠礼物、参与游戏
2. 内容上：嘉宾分享是否牛逼、有货、有料
3. 连接：用户之间是否产生有意义的连接
4. 离场时：情感关怀（如天冷时送暖手宝）
```

### 1.4 思考框架的核心价值

**最大意义**：帮助你往正确的方向思考

**好的思考 → 好的问题 → 好的答案**

| 问题类型 | 示例 | 质量评估 |
|---------|------|---------|
| 宽泛问题 | "to B类产品的运营该怎么做？" | 较差，难以回答 |
| 更好问题 | "to B类产品的运营应重点关注哪几个业务环节？" | 较好，容易得到具体答案 |
| 更好问题 | "to B类产品的运营在哪几个方面可能与to C类产品不同？" | 更好，针对性更强 |

## 二、如何训练养成框架式思维

### 2.1 个人专业能力的构成

```
个人专业能力 = 思考框架 + 专业知识 + 技能

┌────────────────────────────────────────────────────────────┐
│  思考框架：领域的方法论（如：传播 = 内容 × 渠道 × 时机）    │
│  专业知识：具体问题和解决方法（如：如何给媒体车马费）      │
│  技能：真正把事情做好的能力（需要大量练习）                │
└────────────────────────────────────────────────────────────┘
```

### 2.2 2-3年经验者的学习路径

```
┌────────────────────────────────────────────────────────────┐
│  第一步：建立思考/认知框架                                  │
│          （如：公众号运营 = 用户总量 × 用户活跃 × 变现效率）│
│                          ↓                                 │
│  第二步：往框架填充专业知识                                 │
│          （选题策划、互推、用户互动、广告变现...）          │
│                          ↓                                 │
│  第三步：针对性刻意训练提升技能                             │
│          （发现薄弱点，刻意练习）                           │
└────────────────────────────────────────────────────────────┘
```

### 2.3 零基础新人的学习路径

```
┌────────────────────────────────────────────────────────────┐
│  第一步：掌握基础技能                                       │
│          （管群、基础推广、转化文案...）                     │
│                          ↓                                 │
│  第二步：补充专业知识 + 建立局部思考框架                     │
│          （从具体事项开始，如：如何写好传播性文章）          │
│                          ↓                                 │
│  第三步：继续行动并完善框架                                 │
│          （遇新问题→用框架检验→无法解释则完善框架）         │
└────────────────────────────────────────────────────────────┘
```

**重要提醒**：思考框架必须能指导实践，不能用来吹牛逼。

### 2.4 培养框架式思考的三个步骤

```
┌────────────────────────────────────────────────────────────┐
│  Step 1: 培养拆解习惯                                       │
│          对一切事物都能找到方法、角度进行拆解               │
│                          ↓                                 │
│  Step 2: 针对特定领域建立思考框架                           │
│          如：运营领域建立自己的框架                         │
│                          ↓                                 │
│  Step 3: 持续优化和完善框架                                 │
│          填充信息、持续迭代、追求严谨完整                   │
└────────────────────────────────────────────────────────────┘
```

## 三、三种核心拆解思路

### 3.1 思路一：分支方向式

**定义**：找到几个分别独立的、不同方向，每个方向做好都对目标有益

**示例：提升线下活动影响力**
```
提升影响力
├── 权威媒体曝光
├── 大量用户转发
├── KOL背书
└── 现场观众口碑/朋友圈
```

**示例：如何写出爆款文章（分类思路）**
```
爆款文章类型
├── 鸡汤类
├── 干货观点输出类
├── 段子类
└── 科普类
```

### 3.2 思路二：工作流程式

**定义**：按从前到后的先后顺序梳理工作流程

**示例：写文章的流程**
```
写文章 = 选题 → 结构设计 → 素材收集 → 写作 → 编排发布
```

### 3.3 思路三：公式型

**定义**：将指标拆解为数学公式

**示例：滴滴订单量拆解**

```
司机端订单量 = 激活司机数 × 首单率 × 在线率 × 抢单率 × 人均完成订单数

乘客端订单量 = 注册用户数 × 首单率 × 日活率 × 人均订单数 × 应答率 × 抢单完成率
```

**价值**：每个指标都可以找到团队或个人背责，统一团队方向

### 3.4 三句重要原则

| 原则 | 说明 |
|------|------|
| 1 | 任何宽泛目标/问题，都可以经由**划定边界+1-2次分解**，变成N多可量化维度或可落地事项 |
| 2 | 任何可量化目标/问题，都可以拆解为**某种公式** |
| 3 | 任何具体可落地事项，都可以找到相应的**工作流程或方法** |

## 六、选修：如何做好第三方推广

### 6.1 什么是第三方推广

**定义**：一切通过非直接触达用户的手段来实现的推广

```
推广路径
├── 直接推广
│   ├── 论坛/微博一对一沟通
│   ├── 邮件营销
│   └── 线下地推
└── 第三方推广
    ├── 门户网站广告位
    ├── 网络联盟
    ├── 广点通
    ├── 搜索引擎（SEM/SEO）
    ├── 朋友圈广告
    ├── 微博/微信大号
    ├── 社区/论坛推广
    └── 媒体发稿
```

### 6.2 常见第三方渠道类型

| 渠道类型 | 说明 | 示例 |
|---------|------|------|
| Banner/Button广告 | 门户网站横幅或按钮广告位 | 新浪、搜狐首页广告 |
| 网络联盟 | 整合多个网站广告位的平台 | 百度联盟、易传媒 |
| 广点通 | 腾讯系产品的广告联盟 | QQ、微信、QQ空间广告 |
| 搜索引擎推广 | SEM(付费) + SEO(免费) | 百度搜索结果 |
| 朋友圈广告 | 微信朋友圈信息流广告 | 5万起投 |
| 微博大号 | 意见领袖原发或转发 | 明星、KOL发推广 |
| 微信大号 | 公众号头条/次条推广 | 行业大号发稿 |
| 社区论坛 | 垂直社区置顶/推荐 | CSDN、知乎推广 |
| 媒体发稿 | 科技媒体报道/软文 | 36氪、虎嗅发稿 |

### 6.3 第三方推广的核心逻辑

```
用户在第三方渠道的行为路径：

目标用户 → 访问第三方渠道
              ↓
          浏览特定路径
              ↓
          看到推广展示位
              ↓
          点击进入推广素材
              ↓
          被素材说服转化
              ↓
          进入产品着陆页
              ↓
          引导到产品体验点
```

**关键要素**：
- **推广展示位**：在渠道哪个位置出现
- **推广素材**：文案、图片、视频等
- **着陆页**：承接外部流量的页面
- **转化路径**：从着陆页到目标行为的路径

## 本章核心要点

1. **框架式思维的价值**：帮助往正确方向思考，从宏观视角理解问题全貌

2. **能力三要素**：思考框架 + 专业知识 + 技能
