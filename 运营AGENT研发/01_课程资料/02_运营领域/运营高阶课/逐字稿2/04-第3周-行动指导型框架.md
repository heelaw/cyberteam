# 第3周：快速建立陌生领域认知，升级为行动指导型框架

> 本周核心：掌握两种建立认知框架的方法（自上而下、自下而上），以及将认知型框架升级为行动指导型框架的方法

---

## 一、快速在陌生领域建立完整认知

### 1.1 两种认知框架建立路径

```
┌─────────────────────────────────────────────────────────┐
│              陌生领域认知建立路径选择                          │
├─────────────────────────────────────────────────────────┤
│                                                                 │
│  你对该领域的了解程度？                                          │
│                                                                 │
│    有一定了解 ────────────┐    完全陌生 ────────────┐          │
│         │                   │         │                   │          │
│         ▼                   │         ▼                   │          │
│    自上而下拆解          │    自下而上归纳          │          │
│                                                                 │
│  • 先建立一级分类           │  • 收集大量具体做法         │          │
│  • 逐级向下拆解             │  • 梳理归类合并             │          │
│  • 每次思考简化框架         │  • 提炼共性形成框架       │          │
│                                                                 │
│  └─────────────────────┘   └─────────────────────┘          │
│                                                                 │
└─────────────────────────────────────────────────────────┘
```

### 1.2 自上而下拆解法

**适用场景**：对领域有一定了解

**示例：用户运营概念拆解**

```
用户运营
├── 提升用户规模
│   ├── 早期创业公司：细分切入 → 培育种子 → 口碑传播
│   └── 大公司：开源（增加流量入口） + 节流（提升转化率）
└── 提升活跃用户规模
    ├── 依靠核心用户带动非核心用户（核心用户维系）
    └── 扩大活跃用户规模
        ├── 用户建模和分类（新人/1年以下/1-3年/地域等）
        ├── 梳理典型用户路径（生命周期管理）
        └── 设计用户激励体系和机制
```

### 1.3 自下而上归纳法

**适用场景**：对领域完全陌生

**步骤**：
1. 大量收集具体做法（问人、爬职位JD）
2. 梳理归类合并
3. 提炼共性形成框架

**示例：用户运营做法收集与归类**

| 收集到的做法 | 归类结果 |
|-----------|---------|
| 管理核心用户群 | 用户分类管理、维护和召回 |
| 设计用户激励体系 | 提升付费用户规模 |
| 提升用户体验 | 通过产品优化和机制影响活跃用户规模 |
| 面向特定用户策划活动 | 外围推广拉新 |

### 1.4 高手访谈验证法

**核心思路**：先建立初步框架 → 拿去验证 → 降低沟通成本

**问题对比**：

| 错误问法 | 正确问法 |
|---------|---------|
| "用户运营是什么？" | "我理解用户运营分为A、B、C，您觉得对吗？" |
| "怎么提升公众号涨粉？" | "我有X万预算，面向XX用户，应该用哪些方法？" |
| "这个框架怎么完善？" | "这个框架的A、B两点，您觉得哪些可以合并？" |

## 二、认知型框架升级为行动指导型框架

### 2.1 两种框架类型对比

```
┌─────────────────────────────────────────────────────────┐
│                    框架类型对比                              │
├─────────────────────────────────────────────────────────┤
│                                                                 │
│  认知型框架                    行动指导型框架               │
│                                                                 │
│  • 解释"是什么"                • 解决"怎么做"             │
│  • 包含内容、构成              • 具体手段、决策逻辑        │
│  • 帮助理解认识                  • 指导行动选择            │
│                                                                 │
└─────────────────────────────────────────────────────────┘
```

### 2.2 框架升级的核心逻辑

```
┌─────────────────────────────────────────────────────────┐
│              框架升级四步法                                │
├─────────────────────────────────────────────────────────┤
│                                                                 │
│  Step 1: 建立认知型框架                                      │
│  ├── 穷尽所有运营手段/做法                                   │
│  │                                                      │
│  Step 2: 识别影响手段选择的关键因素                           │
│  ├── 找出不同因素会导致做法差异                                   │
│  │                                                      │
│  Step 3: 对每个因素进行情况分支                               │
│  ├── 每个因素下的不同情况                                       │
│  │                                                      │
│  Step 4: 形成行动指导框架                                      │
│  └── 串联所有判断维度，形成决策流程                           │
│                                                                 │
└─────────────────────────────────────────────────────────┘
```

### 2.3 升级实例：公众号主动推广

#### Step 1: 建立认知型框架

```
公众号主动推广手段分类

人肉找人型：
├── 微信群红包求转发
└── 找有影响力的好友转发

借助其他号推广：
├── 量级相当号互推
├── 团局组织大范围互推
└── 付费在其他号打广告

借助活动：
├── 关注领福利型活动
├── 投票拉票型活动
├── 晒自己炫自己型活动
└── 大型品牌活动

借助其他渠道：
├── 垂直论坛社区推广
├── 社群推广
├── 线下推广
└── 广告渠道投放（广点通等）
```

#### Step 2: 识别关键决策因素

**影响推广手段选择的4个关键因素**：
1. 公众号所处阶段（起步期/成长期/成熟期）
2. 人力资源和预算情况
3. 用户属性（大众/小众+聚集/分散）
4. 竞争对手状态和策略

#### Step 3: 形成行动指导框架

```
┌─────────────────────────────────────────────────────────┐
│            公众号推广行动指导框架                              │
├─────────────────────────────────────────────────────────┤
│                                                                 │
│  判断1：所处阶段                                                 │
│  ├── 起步期：低成本小范围测试，慢慢往上拉                         │
│  ├── 成长期：卷入人力不大的高产出做法（广告/活动）                  │
│  └── 成熟期：品牌跨界合作推广                                    │
│                                                                 │
│  判断2：资源和预算                                               │
│  ├── 有人没钱：人力推动、创意驱动（红包/创意活动/H5）              │
│  ├── 有钱无人：广告投放                                          │
│  ├── 有钱有人：随便玩                                              │
│  └── 没钱没人：再见                                               │
│                                                                 │
│  判断3：用户属性                                                 │
│  ├── 小众+聚集：社区社群线象、垂直活动                              │
│  ├── 小众+分散：垂直渠道投放、场景触达                              │
│  └── 大众用户：渠道选择限制较小                                   │
│                                                                 │
│  判断4：竞争对手策略                                             │
│  └── 尽量与竞对做法差异化                                         │
│                                                                 │
└─────────────────────────────────────────────────────────┘
```

## 三、运营的定义与重新理解

### 3.1 运营的四种切分框架

#### 框架1：按工作场景划分

```
运营工作场景
├── 新媒体运营
├── 非新媒体类PC渠道（SEO、SEM、合作换量）
├── 非新媒体类移动渠道（应用商店、广告联盟）
└── PC/APP站内运营
```

#### 框架2：按产品类型划分

```
运营产品类型
├── 电商类：品类管理、上下架、文案、页面设计、客服、活动推广
├── 社区类：内容维系、话题维系、核心用户管理、用户等级
├── 媒体内容类：选题、创作、编辑、推送
├── 工具类：功能打磨、使用习惯培养
└── 平台类：双边或多边市场运营
```

#### 框架3：按工作目的划分

```
运营工作目的
├── 拉新：开源、节流、促活
├── 促活：用户活跃、内容消费、互动
├── 商品售卖：选品、定价、包装、推广、供应链
└── 基础维系：用户服务、数据监控、系统维护
```

#### 框架4：按工作内容划分

```
运营工作内容
├── 推广投放
├── 内容写作
├── 活动策划执行
├── 核心用户维系
├── 用户体系设计
└── 用户召回
```

## 四、关键要点总结

### 4.1 框架设计原则

- **简洁胜于复杂**：框架为你服务，越简洁越好
- **先穷尽再简化**：初次建立框架时追求完全穷尽，逐步简化
- **保持一致性**：同一层级的拆解要在同一维度

### 4.2 沟通避坑指南

**避免概念化沟通**：
- 不要说"我是做用户运营的"
- 要说"我是做站内核心用户维系的"
- 避免因理解不同造成沟通效率低下

### 4.3 框架升级公式

```
行动指导框架 = 认知框架 + 关键决策因素 + 分支情况 + 判断流程
```

**核心思路**：
- 先穷尽所有手段/做法
- 找出影响选择的关键维度
- 针对每个维度进行分支
- 串联所有维度形成决策流程
