### 1.3 长期价值不足的三种情况

| 类型 | 特征 | 解决思路 |
|------|------|----------|
| **伪需求** | 需求看起来成立，实际不成立 | • MVP小步快跑验证<br>• 快速迭代或及时止损 |
| **服务能力不够** | 需求真实，但当前服务能力不足 | • 设定边界，封闭运营<br>• 种子用户运营<br>• 夯实服务能力后开放 |
| **短期需求** | 好奇心驱动，用几次就够 | • 快速迭代延伸业务<br>• 探索内容/社交等高频业务<br>• 保持产品新鲜感 |

### 1.4 短期价值缺失的三种情况

| 类型 | 表现 | 解决方案 |
|------|------|----------|
| **使用路径不明** | 用户不知道该做什么 | • 数据监测分析典型路径<br>• 可用性测试<br>• 调整引导文案和流程 |
| **缺乏激励刺激** | 用户缺乏动力完成流程 | • 设计用户激励体系<br>• 帮助用户养成习惯 |
| **缺乏额外亮点** | 市场竞争激烈需要差异化 | • 增加交互/文案/细节亮点<br>• 设计每日彩蛋<br>• 形成独特体验 |

### 2.2 案例一：500人线下活动提升影响力

**Step 1: 梳理活动流程**
```
活动执行流程：
1. 预定时间/场地/嘉宾确认
2. 物料设计/流程筹备/预热
3. 大范围宣传
4. 现场活动一起嗨
5. 活动结束和收尾
```

**Step 2: 界定影响力关键行为**
```
影响力拆解：
1. 媒体/线上渠道曝光
2. 大佬名人/KOL背书
3. 现场观众自发传播
4. 非现场观众自发传播
```

**Step 3: 流程×行为矩阵**

| 流程环节 | 提升影响力的具体手段 |
|---------|---------------------|
| **预定时间场地嘉宾** | • 搞定KOL：先做好事→撬动1-2个→用背书搞定更多<br>• 时间场地讲故事：9月9号的特殊意义<br>• 话题包装：全国首家/史无前例 |
| **物料筹备预热** | • 有话题性的物料和互动道具<br>• 十二星座产品狗形象<br>• 发有传播性的物料（套套等）<br>• 预热期互动传播活动 |
| **大范围宣传** | • 制造火爆氛围：分批放票制造稀缺<br>• 传播性文案<br>• 互动性H5<br>• 多渠道联动宣传 |
| **活动现场** | • 超预期体验让用户发朋友圈<br>• 与媒体/KOL沟通亮点<br>• 设计场外参与环节 |
| **活动收尾** | • 有传播性的复盘文章<br>• 后续跟进KOL/媒体曝光 |

### 2.3 案例二：提升微信群付费转化率

**Step 1: 拆解付费转化要素**
```
付费意愿 = 对群/品牌信任认可 + 产品说服力 + 消息触达率转化率
```

**Step 2: 聚焦信任认可建立流程**

信任认可提升三要素：
1. 群有价值且认知清晰
2. 用户在群内有被重视感
3. 用户与群内他人产生人际互动

**Step 3: 用户从认知到强信任的流程**

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    微信群信任建立五阶段                                      │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  阶段1：听说/了解群        → 激发清晰认知 + 加入意愿                         │
│  ├── 群命名：清晰描述价值（每周还原真实运营案例）                           │
│  └── 设置门槛：付出成本增加重视感                                           │

│                                                                             │
│  阶段3：参与观察互动       → 约束鼓励特定行为                                 │

## 四、用户行为激励体系设计

### 4.1 用户激励的本质

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                      用户激励的本质与作用                                    │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  本质：运营 = 围绕长期价值激励用户完成一系列相关行为                          │
│                                                                             │
│  常用三种场景：                                                              │
│  ┌─────────────────────────────────────────────────────────────────────────┐ │
│  │ ① 用户体量增加，原有运营方式hold不住                                   │ │
│  │    → 需要机制化、标准化的运营策略                                       │ │
│  │                                                                         │ │
│  │ ② 产品接近成熟期，需要榨取用户活跃度                                   │ │
│  │    → 通过激励体系让用户更频繁使用产品                                   │ │
│  │                                                                         │ │
│  │ ③ 业务模式要求用户持久参与，但用户动机有限                               │ │
│  │    → 游戏/教育类产品需要激励驱动持续参与                                │ │
│  └─────────────────────────────────────────────────────────────────────────┘ │
│                                                                             │
│  重要原则：                                                                 │
│  ⚠️ 成功的用户激励体系无法解决长期价值问题                                   │
│  ⚠️ 产品本身必须能给用户稳定价值，激励体系才有意义                            │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 4.2 用户行为激励五步法

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                      用户行为激励设计流程                                    │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  Step 1: 判断需激励的用户行为                                                │

│                                                                             │
│  Step 2: 梳理可用激励资源                                                    │

│  ├── 物质满足层面（红包、代币、积分）                                       │
│  └── 其他人性诱因（窥视、色欲、懒惰、虚荣、贪婪）                           │
│                                                                             │
│  Step 3: 匹配行为与激励资源                                                  │
│  ├── 一次性行为 → 物质激励/特权/勋章                                        │
│  ├── 持续性行为 → 长线激励（积分+等级+勋章）                                │

│                                                                             │

│  └── 分阶段上线，优先解决一次性和简单重复行为                                 │
│                                                                             │
│  Step 5: 观察数据调整优化                                                    │
│  └── 监控虚拟经济，防止通货膨胀                                              │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

│  ──────────────────────────────────────────────────────────────────────    │
│  • 等级和排名                         • 红包、代金券                        │
│  • 特殊勋章                           • 代币（Q币、学币）                  │
│  • 官方推荐                           • 可兑换积分                         │
│  • 粉丝围观                           • 实物奖品                            │
│  • 大众点赞                           • 资料包、文档                       │
│  • 特权（功能、活动、参与权）          • 折扣、奖金                          │
│                                                                             │
│  ③ 其他人性诱因                                                          │
│  ──────────────────────────────────────────────────────────────────────    │
│  • 窥视欲：悄悄关注名人动态                                                │
│  • 色欲：美女图片、展示                                                   │

│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 4.5 激励匹配四原则

| 行为类型 | 特征 | 匹配原则 |
|---------|------|----------|
| **一次性行为** | 完成就结束 | 优先物质激励/特权/勋章<br>考虑人性诱因 |
| **持续性行为** | 需连贯反馈 | 长线激励体系<br>积分+等级+勋章配合 |
| **简单重复行为** | 固定周期 | 稳定明确预期<br>签到得代币/积分 |
| **创造可展示行为** | 理想自我表达 | 竞争排名<br>站内广泛展示+认可 |

### 4.6 上线激励体系五原则

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                      上线激励体系五原则                                      │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ① 优先解决一次性行为和简单重复行为                                          │
│     - 这两类容易见效                                                       │
│     - 持续性行为周期长                                                     │
│                                                                             │
│  ② 积分需与等级勋章配合使用                                                 │
│     - 积分是消费性的                                                       │
│     - 等级勋章是荣誉性的                                                    │
│     - 防止变成纯利益导向                                                   │
│                                                                             │
│  ③ 激励出口不超过2个                                                       │
│     - 用户理解成本高                                                       │
│     - 可分阶段上线                                                         │
│                                                                             │
│  ④ 找到关键行为优先入手                                                    │
│     - 关键行为是其他行为的前提                                              │

│                                                                             │
│  ⑤ 防止虚拟通货膨胀                                                       │

```

### 4.7 案例实践：在线教育产品激励体系

**产品背景**：
- 用户量10万以内
- 业务：精华文章（日更）+ 系统付费课程
- 月预算不超过1万

**需激励行为**：
1. 每日登录
2. 完善个人资料
3. 阅读文章
4. 首次报名上课
5. 持续报名上课
6. 首次完课+持续完课
7. 记录学习笔记
8. 与他人讨论互动

**激励资源**：
- 精神层面：勋章、老师认可、同学围观、特权
- 物质层面：书、奖金代币、折扣、资料包

**匹配方案**：
- 一次性行为（完善资料/首次报名/首次完课）→ 勋章+资料包+红包+证书
- 持续性行为（持续上课/记录笔记/互动）→ 积分等级体系
- 简单重复行为（每日登录）→ 签到得学币（每天0.1元）

## 五、关键要点总结

### 5.1 归因模型核心

用户留存活跃差 = 4种原因之一：
1. **长期价值不足**（伪需求/服务能力不够/短期需求）
2. **短期价值缺失**（路径不明/缺乏激励/缺乏亮点）
3. **天然低频产品**（招聘/婚庆/O2O）
4. **用户不匹配**（推广策略问题）

### 5.2 产出提升模型

横纵交叉思考：
- 有流程：流程×关键行为×激励手段
- 无流程：产出要素→前置条件→用户路径→体验匹配

### 5.3 激励设计本质

用户激励体系 ≠ 万能药
- 前提：产品必须能提供稳定价值
- 作用：激励用户完成关键行为，接近自运营状态
- 反向检查：能否界定需激励的行为 = 检验产品价值是否明确
