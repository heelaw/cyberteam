| **框架式思维** | 一种结构化的认知方式，帮助我们将复杂问题拆解成有序部分 | 往正确方向思考、避免只见树木不见森林、提高沟通效率 |
| **思考框架** | 领域内的方法论和认知结构 | 建立整体视角、系统性思考、可复用 |

| **工作流程式拆解** | 按先后顺序梳理工作流程 | 环节依赖、顺序清晰 |
| **公式型拆解** | 将指标拆解成数学公式 | 可量化、每个指标有人负责 |

## 📚 详细内容

### 主题一：框架式思维的价值

#### 1.1 什么是框架式思维

**定义：** 框架式思维是一种结构化的认知方式，它帮助我们在面对复杂问题时快速理清思路，避免陷入"只见树木，不见森林"的状态。

**核心价值：**

1. **帮助你往正确的方向思考**
   - 避免盲目行动
   - 快速找到切入点
   - 提高思考效率

2. **避免"只见树木，不见森林"**
   - 建立全局视角
   - 理解各部分关系
   - 把握问题本质

3. **提高沟通和说服效率**
   - 清晰展示思路
   - 逻辑更有说服力
   - 便于团队协作

4. **让问题更容易得到好答案**
   - 好问题导向好答案
   - 问者和答者共同创造价值
   - 降低沟通成本

#### 1.2 案例一：课程设计读书会 - 框架的价值

**案例背景：**

- **时间：** 近期
- **场景：** 三节课办公室组织《认知设计》读书会
- **参与者：** 办公室多位小伙伴
- **书籍信息：** 《认知设计》副标题是"提升学习体验的艺术"，是一本关于如何设计学习产品、如何设计课程的书
- **目的：** 通过读书会讨论如何改进课程设计

**无框架的讨论过程：**

在第一次分享会上，小伙伴们提出了各种建议：

| 建议类型 | 具体内容 | 问题分析 |
|---------|---------|---------|
| 功能类 | 加一个课前测评 | 单点思考，缺乏整体视角 |
| 互动类 | 让学习者边学边动手 | 没有说明在什么环节、如何实现 |
| 结构类 | 告诉学习者起点和终点具体在哪里 | 缺乏与其他建议的逻辑关系 |
| 组织类 | 重新规划课程秩序 | 没有说明按照什么原则重新规划 |

**问题分析：**

这些建议都从某个角度看是有道理的，但如果直接按照这些建议去执行，会出现以下问题：

1. **建议之间可能存在冲突**
   - "边学边动手"和"告知起点终点"可能需要不同的课程结构
   - 没有框架指导，难以判断如何平衡

2. **资源投入分散，无法形成合力**
   - 各个方向都想做，但资源有限
   - 缺乏优先级判断标准

3. **无法判断哪些建议优先级更高**
   - 所有建议看起来都有价值
   - 但不知道哪些是核心，哪些是次要

**有框架的梳理过程：**

黄老师在听完所有人分享后，基于对书籍的理解（虽然看书记是一两个月前的事了），快速梳理出了一个框架：

**核心框架：设计课程 = 管理学习动机 + 学习体验 + 学习效果**

```
┌─────────────────────────────────────────────┐
│           课程设计三要素框架                 │
├─────────────────────────────────────────────┤
│                                              │
│   ┌──────────┐  ┌──────────┐  ┌──────────┐ │
│   │学习动机  │  │学习体验  │  │学习效果  │ │
│   └────┬─────┘  └────┬─────┘  └────┬─────┘ │
│        │             │             │        │
│        ▼             ▼             ▼        │
│   • 学习目标      • 内容组织      • 练习设计│
│   • 动机激发      • 呈现方式      • 反馈机制│
│   • 动力维持      • 互动设计      • 进阶路径│
│   • 课前测评      • 课程秩序      • 作业设计│
│                                              │
└─────────────────────────────────────────────┘
```

**框架详细解析：**

**维度一：学习动机管理**

在这个维度下，需要关注：
- 学习目标是否清晰：学习者要知道学完能获得什么
- 学习动机如何激发：为什么要学这个问题
- 学习动力如何维持：如何让学习者保持兴趣

具体要素：
- 课前测评：帮助学习者了解自己当前水平，激发学习动机
- 明确告知起点和终点：让学习者知道从哪里开始、到哪里结束
- 建立学习目标感：让学习者知道学习的价值和意义

**维度二：学习体验设计**

在这个维度下，需要关注：
- 内容组织是否合理：知识点的组织逻辑
- 呈现方式是否吸引人：视频、文字、互动等方式的选择
- 互动设计是否有趣：让学习者保持参与感

具体要素：
- 课程秩序规划：如何安排学习顺序
- 边学边动手的设计：在实践中学习的机制
- 学习节奏控制：避免过快或过慢

**维度三：学习效果保障**

在这个维度下，需要关注：
- 练习设计是否有效：练习能否真正巩固所学
- 反馈机制是否完善：学习者如何知道自己的掌握程度
- 进阶路径是否清晰：学完之后如何进一步提升

具体要素：
- 作业设计：让学习者实际应用所学
- 进阶难度设置：让学习者有成长感
- 学习效果评估：如何衡量学习成果

**框架应用效果：**

1. **全局清晰可控**
   - 课程设计变成三个可管理的维度
   - 每个维度都有明确的关注要点
   - 各个维度之间的关系清晰

2. **建议归类整理**
   - "课前测评" → 学习动机维度
   - "边学边动手" → 学习体验维度
   - "告知起点终点" → 学习动机维度
   - "课程秩序" → 学习体验维度

3. **团队讨论更有针对性**
   - 可以分别讨论每个维度
   - 每个维度的讨论都有明确边界
   - 避免讨论发散、无法聚焦

**案例启示：**

- 有了框架，复杂问题变得清晰可控
- 框架帮助我们组织零散的建议
- 框架让团队讨论更高效
- 框架不是凭空产生的，而是基于对领域的理解

#### 1.3 案例二：知乎评价《平凡之路》- 框架与说服力

**案例背景：**

- **时间：** 韩寒电影《后会无期》上映期间
- **平台：** 知乎
- **问题：** 有人提问"如何评价平凡之路这首歌"
- **歌曲信息：** 《平凡之路》是《后会无期》的主题曲，由朴树演唱

**无框架的回答：**

大量知乎用户给出了类似的回答：

| 回答类型 | 具体内容 | 问题分析 |
|---------|---------|---------|
| 情绪表达 | "这首歌好听" | 只有主观感受，缺乏分析 |
| 极端评价 | "这首歌太棒了" | 无法说明为什么棒 |
| 比较判断 | "这是我近年来听过最好的一首歌" | 没有分析维度，不具说服力 |
| 粉丝心态 | "韩寒脑残粉，这就粉了" | 基于情感，不是理性分析 |

**问题分析：**

这些回答存在以下问题：

1. **无法帮助他人思考**
   - 只表达了主观感受
   - 没有提供分析框架
   - 读者无法形成自己的判断

2. **难以说服陌生人**
   - 主观感受因人而异
   - 缺乏逻辑支撑
   - 没有分析过程

3. **沟通效率低**
   - 情绪表达无法传递信息
   - 无法进行有意义的讨论
   - 浪费阅读者时间

**有框架的回答：**

几天后，出现了一个高质量的回答，这个回答采用了框架式思维：

**核心框架：评价歌曲 = 旋律 + 歌词 + 编曲**

回答者首先建立了评价框架，将歌曲分解为三个部分：

1. **旋律分析**
   - 音调特点：平稳中带有起伏
   - 节奏特点：舒缓、适合沉思
   - 和声特点：简洁但有效
   - 分析结论：旋律是争议最小的部分

2. **歌词分析**
   - 内容特点：充满哲理和人生感悟
   - 文学性：歌词具有诗意和画面感
   - 情感共鸣：不同人看到不同的经历和情怀
   - 分析结论：歌词见仁见智，是引发共鸣的主要来源

3. **编曲分析**
   - 乐器配置：简洁的吉他伴奏
   - 编排风格：极简主义
   - 制作水平：精致但不张扬
   - 分析结论：编曲是争议最大的部分

**回答结构示例：**

```
┌─────────────────────────────────────────────┐
│          歌曲评价框架                       │
├─────────────────────────────────────────────┤
│                                              │
│  评价维度    │  分析要点     │  争议程度     │
│  ──────────┼──────────────┼─────────────   │
│   旋律      │  音调、节奏    │   最小        │
│             │  和声特点                      │
│  ──────────┼──────────────┼─────────────   │
│   歌词      │  内容、文学性  │   见仁见智    │
│             │  情感共鸣                      │
│  ──────────┼──────────────┼─────────────   │
│   编曲      │  乐器配置      │   最大        │
│             │  编排风格                      │
│             │  制作水平                      │
│                                              │
└─────────────────────────────────────────────┘
```

**框架应用效果：**

1. **思考方向清晰**
   - 从三个维度系统分析
   - 每个维度都有具体的分析要点
   - 分析过程有条不紊

2. **说服力强**
   - 逻辑严密，有理有据
   - 读者可以看到分析过程
   - 便于形成自己的判断

3. **便于沟通讨论**
   - 讨论可以有明确的维度
   - 不同观点可以在同一维度下对比
   - 提高沟通效率

**案例启示：**

- 框架式思维天然更具说服力
- 先建立框架，再进行分析
- 框架让沟通更高效
- 观点需要事实和逻辑支撑

#### 1.4 案例三：三次元沙龙活动设计 - 从零散到系统

**案例背景：**

- **活动名称：** 三次元沙龙
- **组织者：** 三节课
- **任务：** 设计活动的现场体验
- **目标：** 让用户在活动现场产生"太赞了"的体验

**无框架的方案：**

黄老师与负责活动的同事讨论时，同事首先提出了以下建议：

| 建议类型 | 具体内容 | 问题分析 |
|---------|---------|---------|
| 娱乐类 | 弄几个游戏区让大家玩一玩 | 单点思考，没有与其他方面建立联系 |
| 餐饮类 | 多弄点水果、甜点之类的 | 没有说明在整个体验中的位置 |
| 环境类 | 会场的逼格是否可以更高一点 | 缺乏系统性，没有覆盖关键维度 |

**问题分析：**

1. **缺乏系统性**
   - 建议零散，没有覆盖活动的关键环节
   - 无法形成完整的用户体验

2. **无法判断优先级**
   - 各个建议看起来都有价值
   - 但不知道哪些是核心，哪些是次要

3. **难以形成执行方案**
   - 缺乏整体视角
   - 不知道如何分配资源

**有框架的思考：**

黄老师花了大约几分钟思考，提出了一个新的思考方向：

**核心框架：用户体验 = 入场体验 + 活动内容 + 社交连接 + 离场体验**

```
┌─────────────────────────────────────────────┐
│       线下活动用户体验框架                   │
├─────────────────────────────────────────────┤
│                                              │
│   ┌────────┐    ┌────────┐    ┌────────┐   │
│   │  入场  │    │  现场  │    │  离场  │   │
│   └───┬────┘    └───┬────┘    └───┬────┘   │
│       │            │            │           │
│       ▼            ▼            ▼           │
│   • 惊喜礼物   • 嘉宾内容   • 情感关怀      │
│   • 互动游戏   • 有料干货   • 暖手物        │
│                                              │
│         ┌────────────────┐                   │
│         │  人与人连接    │                   │
│         │  • 互动机制    │                   │
│         │  • 认识朋友    │                   │
│         └────────────────┘                   │
│                                              │
└─────────────────────────────────────────────┘
```

**框架详细解析：**

**维度一：入场体验**

用户在入场时能否得到某种惊喜：

- **馈赠礼物**
  - 礼物要有意义，不是随手可得的
  - 背后要有故事，能引发好奇心
  - 要能体现活动的调性

- **互动游戏**
  - 让用户从入场就开始参与
  - 不是旁观者，而是参与者
  - 破冰，减少陌生感

- **入场流程**
  - 流程要顺畅，避免等待太久
  - 第一个接触点要令人印象深刻
  - 要有仪式感

**维度二：活动内容**

活动现场的具体内容质量：

- **嘉宾分享质量**
  - 内容是否有料
  - 分享是否有干货
  - 是否能带来启发

- **内容呈现方式**
  - 嘉宾表达能力
  - PPT制作水平
  - 互动环节设计

- **节奏控制**
  - 不拖沓，不赶时间
  - 张弛有度
  - 休息时间合理

- **注意：** 这部分完全取决于嘉宾，运营方只能尽人事听天命

**维度三：社交连接**

用户在现场是否与其他用户产生有意义的连接：

- **互动机制设计**
  - 创造让用户交流的机会
  - 不是强制交流，而是自然发生
  - 要有共同话题

- **认识新朋友**
  - 真正的熟悉，不是加个微信
  - 建立有意义的连接
  - 可能形成后续合作

- **社交环境营造**
  - 物理环境：座位安排、空间布局
  - 心理环境：友好、开放的氛围
  - 机会创造：茶歇、交流环节

**维度四：离场体验**

活动结束时用户是否感受到某种情感关怀：

- **情感关怀**
  - 让用户感受到被重视
  - 不是用完即扔
  - 有温度的告别

- **纪念品或收获**
  - 有形的纪念品
  - 无形的收获和启发
  - 愿意分享给他人

- **期待下次再来**
  - 留下悬念
  - 预告下次活动
  - 建立持续连接

**基于框架的执行方案：**

| 维度 | 执行方案 | 具体措施 |
|------|---------|---------|
| 入场体验 | 送一本书 | 书背后有故事，引发好奇心 |
| 活动内容 | 尽人事听天命 | 尽可能与嘉宾沟通，但无法保证 |
| 社交连接 | 设计互动机制 | 让现场用户可能认识好朋友 |
| 离场体验 | 送暖手物 | 当天天冷，体现关怀 |

**执行细节：**

1. **入场：送一本书**
   - 书不是随便选的，背后有故事
   - 故事可以在活动中分享
   - 让用户从入场就开始参与

2. **活动内容：尽力而为**
   - 与嘉宾充分沟通
   - 但最终质量取决于嘉宾
   - 接受这个现实

3. **社交连接：互动机制**
   - 设计专门的互动环节
   - 创造自然交流的机会
   - 不是强制交流

4. **离场：送暖手物**
   - 当天天冷，这是真实的关怀
   - 不是昂贵的礼物，但有心意
   - 让用户带着温暖离开

**执行效果：**

按照这个框架执行后，活动现场的用户体验都挺赞的。每个维度都有具体的执行要点，团队成员可以分工协作，确保各个方面都得到妥善处理。

**案例启示：**

- 框架帮助我们全面思考问题
- 框架让执行方案更清晰
- 框架便于团队分工协作
- 框架让资源分配更合理

#### 1.5 框架式思维的核心价值总结

通过以上三个案例，我们可以总结出框架式思维的核心价值：

**价值一：帮助往正确的方向思考**

- 避免盲目行动
- 快速找到切入点
- 提高思考效率

**价值二：建立全局视角**

- 避免"只见树木，不见森林"
- 理解各部分关系
- 把握问题本质

**价值三：提高沟通说服效率**

- 清晰展示思路
- 逻辑更有说服力
- 便于团队协作

**价值四：让问题更容易得到好答案**

- 好问题导向好答案
- 问者和答者共同创造价值
- 降低沟通成本

#### 1.6 框架的比喻：衣柜类比

为了让框架式思维的价值更加直观，黄老师打了一个形象的比喻：

**有框架 vs 无框架 = 整齐衣柜 vs 杂乱衣柜**

**有框架的状态（整齐衣柜）：**

- 各个地方分类清晰
- 物品摆放有序
- 新物品快速归类
- 找物品快速定位

**无框架的状态（杂乱衣柜）：**

- 衣服堆积如山
- 新衣服随手一扔
- 找衣服翻箱倒柜
- 成本极高

**类比说明：**

```
┌─────────────────────────────────────────────┐
│          有框架 vs 无框架                   │
├─────────────────────────────────────────────┤
│                                              │
│   有框架                    无框架           │
│   ┌────────┐                ┌────────┐      │
│   │整齐衣柜│                │杂乱衣柜│      │
│   │        │                │        │      │
│   │ 分类清晰│                │堆积如山│      │
│   │ 快速存取│                │翻箱倒柜│      │
│   └────────┘                └────────┘      │
│                                              │
│   新问题 → 快速归类      新问题 → 随意扔     │
│   找答案 → 精确定位      找答案 → 费力找     │
│                                              │
└─────────────────────────────────────────────┘
```

#### 1.7 框架式思维与提问的艺术

黄老师特别强调："好的思考导向好的问题。"

**坏问题的特征：**

1. **边界模糊**
   - 问题没有明确的范围
   - 回答者不知道从哪个角度切入

2. **目标缺失**
   - 没有说明要达到什么效果
   - 不知道如何判断答案好坏

3. **情境不清**
   - 没有说明具体的情境和约束条件
   - 答案可能不适用

**好问题的特征：**

1. **边界清晰**
   - 问题范围明确
   - 回答者知道从哪个角度切入

2. **目标明确**
   - 知道要达到什么效果
   - 可以判断答案好坏

3. **情境具体**
   - 说明了具体的情境
   - 答案有针对性

**问题对比示例：**

| 层级 | 问题 | 问题质量 | 原因分析 |
|------|------|---------|---------|
| 1 | To B类产品的运营该怎么做？ | 烂问题 | 太宽泛，没有边界 |
| 2 | To B类产品的运营应该重点关注哪几个业务环节？ | 较好问题 | 有边界，有具体方向 |
| 3 | To B类产品的运营在哪几个方面可能跟 To C 产品不同？ | 好问题 | 有比较，有具体角度 |

**一次好问答的原则：**

一次好的问答，是由问者和答者通过共同付出努力和思考，共同创造一个好的结果和好的答案。

- 问者有好的思考，带动答者配合
- 答者更愿意给出具体、有价值的答案
- 双方共同创造好的结果

### 主题二：如何训练养成框架式思维

#### 2.1 专业能力三要素模型

在理解了框架式思维的价值后，我们需要知道如何训练养成框架式思维。黄老师首先提出了一个重要的模型：**专业能力三要素模型**。

**核心模型：专业能力 = 思考框架 + 专业知识 + 技能**

```
┌─────────────────────────────────────────────┐
│         专业能力构成金字塔                   │
├─────────────────────────────────────────────┤
│                                              │
│                ╱██████████╲                 │
│               ╱  思考框架  ╲                │
│              ╱──────────────╲               │
│             ╱    专业知识    ╲              │
│            ╱──────────────────╲             │
│           ╱       技能         ╲            │
│          ╱──────────────────────╲           │
│                                              │
└─────────────────────────────────────────────┘
```

**三要素详解：**

| 要素 | 定义 | 特点 | 学习方式 | 示例 |
|------|------|------|---------|------|
| **专业知识** | 具体领域中解决问题的具体方法 | 这件事应该怎么做 | 可以被学会 | 如何给媒体车马费而不被反感 |
| **思考框架** | 领域内的方法论和认知结构 | 建立整体视角 | 通过理解和实践建立 | 传播 = 内容 × 渠道 × 时机 |
| **技能** | 真正把事情做好的能力 | 能把这件事做好 | 只能被习得（大量练习） | 能够让媒体开心接受合作 |

**核心区别：**

- **知识 vs 技能：** 知识是"知道怎么做"，技能是"能把事情做好"
- **学会 vs 习得：** 知识可以被学会，技能只能被习得

**示例：传播运营**

```
专业知识：
- 如何找到一家媒体
- 怎么给媒体车马费
- 媒体可能不反感的方法

思考框架：
传播 = 内容 × 渠道 × 时机
- 内容：说什么
- 渠道：在哪里说
- 时机：什么时候说

技能：
- 能够策划出好的内容
- 能够找到合适的渠道
- 能够把握最佳的时机
- 能够让媒体开心接受合作
```

#### 2.2 不同经验水平的学习路径

不同经验水平的人，提升专业能力的路径是不同的。

**路径一：2-3年工作经验者的学习路径**

**现状特征：**

- 能做很多事
- 但感觉零碎杂乱
- 不清楚事情之间的关系
- 缺乏系统性认知

**典型场景：**

一个微信公众号的小编做了一两年之后：
- 要考虑互推涨粉
- 要考虑后台回复用户
- 要考虑发文章
- 要考虑接广告变现
- 要考虑卖货

但这些事情之间是什么关系，可能感觉不太清楚。

**学习路径：**

```
┌─────────────────────────────────────────────┐
│    2-3年经验者能力提升路径                   │
├─────────────────────────────────────────────┤
│                                              │
│   第一步              第二步              第三步  │
│   ┌──────┐           ┌──────┐           ┌──────┐ │
│   │建立  │    ───▶   │填充  │    ───▶   │针对性│ │
│   │思考  │           │专业  │           │训练  │ │
│   │框架  │           │知识  │           │技能  │ │
│   └──────┘           └──────┘           └──────┘ │
│      │                  │                  │     │
│      ▼                  ▼                  ▼     │
│   公号运营 =          选题策划           发现短板 │
│   用户总量 ×          互推推广           刻意训练 │
│   用户活跃 ×          留言互动                   │
│   变现效率            广告变现                   │
│                                              │
└─────────────────────────────────────────────┘
```

**详细解析：**

**第一步：建立思考框架**

- 建立对领域的整体认知
- 理解核心要素和关系
- 形成方法论

**示例：微信公众号运营**

假设建立了这样的框架：
```
公号运营 = 用户总量 × 用户活跃 × 变现效率
```

有了这个框架，就知道公号运营由三大部分组成。

**第二步：填充专业知识**

- 就着框架往里填充专业知识
- 有些知识已有，有些需要学习
- 效率会非常高

**示例：继续公号运营**

| 维度 | 专业知识 |
|------|---------|
| 用户总量 | 选题策划、互推推广、吸粉技巧 |
| 用户活跃 | 留言互动、内容质量、更新频率 |
| 变现效率 | 广告变现、卖货、知识付费 |

有了框架后，填充这些知识会非常有条理。

**第三步：针对性训练技能**

- 识别哪些专业知识点做得还不够
- 针对薄弱环节进行刻意训练
- 提升实际执行能力

**示例：**

通过前面的填充，发现：
- 选题策划做得还可以
- 互推推广做得不够
- 留言互动做得不错
- 广告变现做得不够

那么就针对互推推广和广告变现进行刻意训练。

**路径二：零基础新人的学习路径**

**现状特征：**

- 完全零经验
- 想进入全新领域
- 没有基础技能
- 没有专业知识

**学习路径：**

```
┌─────────────────────────────────────────────┐
│      零基础新人学习路径                     │
├─────────────────────────────────────────────┤
│                                              │
│   第一步              第二步              第三步  │
│   ┌──────┐           ┌──────┐           ┌──────┐ │
│   │掌握  │    ───▶   │补充  │    ───▶   │局部  │ │
│   │基础  │           │专业  │           │建立  │ │
│   │技能  │           │知识  │           │框架  │ │
│   └──────┘           └──────┘           └──────┘ │
│      │                  │                  │     │
│      ▼                  ▼                  ▼     │
│   管群、推广           了解领域           写文章框架│
│   写文案              补充知识           小框架  │
│   谈合作                                     │
│                                              │
│                    第四步                   │
│                 ┌──────┐                   │
│                 │完善  │                   │
│                 │框架  │                   │
│                 └──────┘                   │
│                    │                       │
│                    ▼                       │
│              实践中不断优化                 │
│              遇到问题→补充框架              │
│                                              │
└─────────────────────────────────────────────┘
```

**详细解析：**

**第一步：掌握基础技能**

- 先掌握一些基础的技能
- 起码能干点杂活
- 有一定实践基础

**示例：纯新手运营**

需要先掌握：
- 怎么管好一个群
- 怎么做基础的推广投放
- 怎么写基础的转化型文案
- 怎么跟大V谈合作

**为什么新人要先学技能？**

如果连这些基础都不会，谈论框架式思维是没什么价值的：
- 无法指导落地
- 太形而上了
- 没有实践基础

**第二步：补充专业知识**

- 了解领域的基础知识
- 补充必要的专业信息
- 建立对领域的基本认知

**第三步：局部建立小框架**

- 先从局部建立框架
- 不要一开始就想建立完整框架
- 从实践中提炼

**示例：从写文章开始**

做了一段时间公众号运营后，可能先从"怎么写好一篇传播性文章"建立小框架：
- 好标题的特征
- 好内容的结构
- 好配图的选择
- 好发布时间

这个小框架是基于：
- 自己写了几个月文章
- 看了其他人分享的写作技巧
- 总结了一些信息

**第四步：实践中完善框架**

- 按照框架继续行动
- 遇到新问题后，看框架能否支撑
- 如果不能，就探索和求教
- 补充和强化框架

#### 2.3 掌握框架式思维的三个步骤

在理解了不同经验水平的学习路径后，黄老师给出了掌握框架式思维的通用步骤：

**核心步骤：培养习惯 → 建立框架 → 持续优化**

```
┌─────────────────────────────────────────────┐
│      掌握框架式思维的步骤                   │
├─────────────────────────────────────────────┤
│                                              │
│   第一步              第二步              第三步  │
│   ┌──────────┐       ┌──────────┐       ┌──────────┐ │
│   │ 培养拆解 │  ───▶  │ 找到领域 │  ───▶  │ 持续优化 │ │
│   │   习惯   │       │ 思考框架 │       │   框架   │ │
│   └──────────┘       └──────────┘       └──────────┘ │
│                                              │
│   • 对一切事物        • 针对运营等        • 填充信息 │
│     进行拆解           特定领域            • 持续迭代 │
│   • 养成习惯           • 建立框架          • 完善改善 │
│                                              │
└─────────────────────────────────────────────┘
```

**详细解析：**

**第一步：培养拆解习惯**

- 对一切事物进行拆解
- 找到方法和角度进行拆解
- 养成拆解的习惯

**重要提醒：**

如果这个习惯都没有，谈论框架式思维再好也没用。

**第二步：找到领域思考框架**

- 针对特定领域（如运营）
- 建立自己的思考框架
- 不要等待别人给答案

**第三步：持续优化框架**

- 围绕框架不断填充信息
- 持续优化和完善框架
- 在实践中迭代

**重要提醒：**

- 很难一开始就建立完整、完美的框架
- 好的框架需要持续迭代和优化
- 一旦开始，就有完善的可能性
- 如果没有框架，就永远没有完善的可能性

**重要原则：**

框架必须能指导实践，否则没有意义！

- 这也是为什么P3课程不适合完全零经验的新人
- 因为他们连基础的技能和实践都没有
- 框架对他们来说太形而上了

#### 2.4 拆解练习：从评估维度到关联影响要素

在培养框架式思维的过程中，最重要的练习是拆解练习。

**拆解框架的基本结构：**

```
┌─────────────────────────────────────────────┐
│         拆解框架的基本结构                   │
├─────────────────────────────────────────────┤
│                                              │
│                   议题                       │
│                     │                        │
│         ┌───────────┼───────────┐            │
│         │           │           │            │
│    评估维度1    评估维度2    评估维度3       │
│         │           │           │            │
│    ┌────┴────┐ ┌────┴────┐ ┌────┴────┐     │
│    │要素 要素│ │要素 要素│ │要素 要素│     │
│    └─────────┘ └─────────┘ └─────────┘     │
│                                              │
│   第一层：评估维度                           │
│   第二层：关联影响要素                       │
│                                              │
└─────────────────────────────────────────────┘
```

**两个核心概念：**

| 概念 | 定义 | 示例 |
|------|------|------|
| **评估维度** | 对一个问题进行评估时考虑的不同方面 | 费用、时间、质量 |
| **关联影响要素** | 每个评估维度下的具体影响因素 | 设计费、物料费、人工费 |

**起步阶段的练习方法：**

**重要原则：**

- 不需要追求评估维度的完整性和严谨性
- 找到一个评估维度即可
- 拆解这个维度下的关联影响要素
- 不断重复这个练习

**为什么起步不需要追求完整？**

- MECE原则（相互独立、完全穷尽）是结果，不是起点
- 以高手最终状态要求自己，无法一步登天
- 维度之间有交叉重复也没关系
- 先让拆解成为习惯，再追求严谨

**实战案例：**

**案例1：装修 - 费用维度**

| 评估维度 | 关联影响要素 |
|---------|-------------|
| 费用 | 设计费、物料费、人工费、运输费 |

**案例2：装修 - 办公室舒适度维度**

| 评估维度 | 关联影响要素 |
|---------|-------------|
| 办公室舒适度 | 墙面色彩、空间开阔度（层高、人均面积）、家具风格材质、同事交流便利程度 |

**案例3：线下活动 - 用户体验维度**

| 评估维度 | 关联影响要素 |
|---------|-------------|
| 现场用户体验 | 收到的礼物、娱乐项目、嘉宾是否有料、场地环境、活动流程、情感关怀 |

**案例4：线下活动 - 收入维度**

| 评估维度 | 关联影响要素 |
|---------|-------------|
| 活动收入 | 门票、赞助、周边贩卖、用户打赏、直播收入 |

**案例5：公众号涨粉 - 增长渠道维度**

| 评估维度 | 关联影响要素 |
|---------|-------------|
| 增长渠道 | 朋友圈自发分享、付费大号推广、资源置换互推、免费社区论坛曝光、广点通投放、线下地推 |

**拆解练习的价值：**

1. **从无法评估到可以评估**
   - 原本好像无法评估的事情
   - 开始能找到更细化的东西
   - 能够去评估和思考了

2. **找到正确的思考方向**
   - 只要找到正确的思考方向
   - 提出正确的问题
   - 就离好答案近了一大步

#### 2.5 框架式思维训练经验分享

**个人实践：**

黄老师分享了自己在2012年的训练经历：

**训练方法：**

- 时间：约3个月
- 场景：地铁上看广告
- 方法：用自己的思考逻辑重新分析
- 记录：所有思考成果都记录下来

**训练效果：**

- 3个月后，成为思考习惯的一部分
- 天然已经是自己的一部分
- 别人带不走
- 价值特别大

**训练启示：**

- 需要持续练习
- 需要记录思考成果
- 需要坚持一段时间
- 最终会成为习惯

### 主题三：三种核心拆解思路

在掌握了如何训练框架式思维后，我们需要学习三种核心的拆解思路。

#### 3.1 作业建议

在开始讲解核心内容前，先给大家两个关于作业的建议：

**建议1：不要追求标准答案**

- 运营中大部分问题没有正确与否之分
- 掌握正确的思考方向更重要
- 作业导向是培养习惯，而非找标准答案

**建议2：设计不同的学习目标**

- **新人：** 把方法培养成习惯即可
- **有框架意识者：** 可以追求更高目标
- 一步一个脚印，不要过于纠结

#### 3.2 三种核心拆解思路

为了确保拆解出来的维度在同一层次上**一致且完整**，有三种核心思路：

**思路一：分支方向式**

**定义：** 找到几个分别不同的独立方向，每个方向做好都能对目标产生正向价值，但彼此之间影响不大。

**案例：如何提升线下活动的影响力**

```
┌─────────────────────────────────────────────┐
│      分支方向式拆解示例                     │
├─────────────────────────────────────────────┤
│                                              │
│           提升线下活动影响力                 │
│                  │                          │
│     ┌────────────┼────────────┐            │
│     │            │            │            │
│  ┌───┴───┐   ┌───┴───┐   ┌───┴───┐       │
│  │权威媒体│   │用户转发│   │KOL背书│       │
│  │  曝光  │   │        │   │       │       │
│  └───────┘   └───────┘   └───────┘       │
│     ┌──────────────────────┐             │
│     │    现场观众口碑      │             │
│     │    朋友圈转发         │             │
│     └──────────────────────┘             │
│                                              │
└─────────────────────────────────────────────┘
```

**特点：**

- 四个方向相对独立
- 每个方向做好都能贡献价值
- 彼此之间影响不大（但不是完全没影响）

**变体：主语分类式**

**案例：如何写好一篇爆款文章**

| 文章类型 | 写作要点 |
|---------|---------|
| 鸡汤类 | 情感共鸣、故事化 |
| 干货观点输出类 | 专业深度、逻辑严密 |
| 段子类 | 幽默、反转、节奏感 |
| 科普类 | 通俗易懂、知识点清晰 |

**思路二：工作流程式**

**定义：** 按照从前到后的先后顺序梳理工作流程，每个环节之间有依赖关系。

**案例：写文章的流程**

```
┌─────────────────────────────────────────────┐
│         写文章工作流程                      │
├─────────────────────────────────────────────┤
│                                              │
│   ①选题 ──▶ ②设计结构 ──▶ ③收集素材       │
│                                              │
│                         │                  │
│                         ▼                  │
│   ⑥编排发布 ◀─ ⑤具体写作 ◀─ ④结合结构    │
│                                              │
└─────────────────────────────────────────────┘
```

**特点：**

- 每个环节之间有影响
- 前一个环节没完成，无法进入下一环节
- 有明确的先后顺序

**注意：** 不是所有问题都能找到工作流程

**思路三：公式型**

**定义：** 将指标拆解成数学公式，每个子指标都有人负责。

**案例：电商网站总销量**

```
总销量 = 用户数 × 转化率 × 客单价（UP值）

或者

总销量 = 品类一销量 + 品类二销量 + 品类三销量 + 品类四销量
```

**实战案例：滴滴出行运营指标拆解**

```
┌─────────────────────────────────────────────┐
│      滴滴订单指标拆解（实际案例）           │
├─────────────────────────────────────────────┤
│                                              │
│  司机端订单量 =                             │
│  激活司机数 × 首单率 × 在线率 ×             │
│  在线抢单率 × 人均完成订单数                │
│                                              │
│  乘客端订单数 =                             │
│  注册用户数 × 首单率 × 日活率 ×              │
│  人均订单数 × 应答率 × 被抢率 × 完成率      │
│                                              │
│  价值：每个指标都有专人负责，五六个人的     │
│        团队劲儿往一处使                     │
│                                              │
└─────────────────────────────────────────────┘
```

**公式型拆解的价值：**

- 每个指标都能找到负责人
- 避免团队关注不相关的事情
- 防止资源浪费
- 便于运营团队管理

#### 3.3 三句重要的话

记住这三句话，能帮你应对大部分拆解场景：

| 序号 | 内容 | 适用场景 |
|------|------|---------|
| 1 | 任何宽泛不具体的目标，一定可以经由划定边界+1-2次分解，变成可量化的评估维度或可落地事项 | "如何评价xxx"、"如何做好xxx" |
| 2 | 任何可量化的目标（运营指标），一定可以拆解为某种公式 | 日活率、新增用户、留存率等 |
| 3 | 任何具体明确的可落地事项，一定可以找到相应的工作流程 | 写文章、做活动、推广投放 |

### 主题四：第三方推广全流程（选修）

#### 4.1 什么是第三方推广

**定义：** 第三方推广是通过非直接触达用户的手段来实现的推广。

**核心特征：**

- 不是通过官方运营人员直接面向用户
- 通过中间的第三方渠道
- 把用户获取到自己的产品生态中

```
┌─────────────────────────────────────────────┐
│         第三方推广基本模型                  │
├─────────────────────────────────────────────┤
│                                              │
│   自己产品 ←────── 用户                     │
│      ▲               │                      │
│      │               │                      │
│      │          直接获取                   │
│      │          （论坛、邮件、地推）        │
│      │                                      │
│      │        第三方渠道                    │
│      └──────（通过中间渠道获取）            │
│                                              │
└─────────────────────────────────────────────┘
```

#### 4.2 常见的第三方渠道类型

| 渠道类型 | 说明 | 案例 |
|---------|------|------|
| **Banner/Button广告** | 网站横幅或按钮广告位 | 各大门户网站广告位 |
| **网络联盟** | 整合多网站广告位的平台 | 百度网盟、易传媒 |
| **广点通** | 腾讯生态专属广告联盟 | QQ、QQ空间、微信、腾讯网等 |
| **搜索引擎推广** | 关键词竞价/SEO | SEM（付费）、SEO（免费） |
| **朋友圈广告** | 微信朋友圈信息流广告 | 5万起投，可定向投放 |
| **微博/微信大号** | 意见领袖、明星号推广 | 付费发文、指定时间/内容 |
| **社区/论坛推广** | 垂直社区广告位 | CSDN、豆瓣小组、置顶/发帖 |
| **媒体发稿** | 行业媒体软文推广 | 36氪、虎嗅等媒体广告位 |

#### 4.3 第三方推广的核心逻辑

```
┌─────────────────────────────────────────────┐
│      第三方推广用户转化流程                 │
├─────────────────────────────────────────────┤
│                                              │
│  第三方渠道          产品端                 │
│                                              │
│  用户访问渠道                                │
│       │                                     │
│       ▼                                     │
│  ┌─────────┐                               │
│  │推广展示位│ ◀─── 推广信息出现             │
│  └────┬────┘                               │
│       │                                     │
│       ▼                                     │
│  用户注意到推广                             │
│       │                                     │
│       ▼                                     │
│  ┌─────────┐                               │
│  │推广素材  │ ◀─── 文案/图片/视频等         │
│  └────┬────┘                               │
│       │                                     │
│       ▼                                     │
│  用户被转化                                 │
│       │                                     │
│       ▼                                     │
│  ┌─────────┐                               │
│  │着陆页   │ ◀─── 承接用户、引导转化       │
│  └────┬────┘                               │
│       │                                     │
│       ▼                                     │
│  引导到产品内部体验点                       │
│                                              │
└─────────────────────────────────────────────┘
```

**关键要素：**

- **推广展示位：** Banner、标题、按钮等
- **推广素材：** 文案、图片、视频、活动介绍
- **着陆页（Landing Page）：** 专题页、首页等承接页面
- **转化路径：** 从着陆页到最终体验点的引导路径

#### 4.4 第三方推广操作五步骤

```
┌─────────────────────────────────────────────┐
│      第三方推广操作五步骤                   │
├─────────────────────────────────────────────┤
│                                              │
│  ①制定推广策略 ──▶ ②确认渠道数据           │
│   • 确认预期         • 用户数/访问量        │
│   • 资源储备         • 人群匹配度           │
│                      • 获客成本             │
│                                              │
│  ③沟通渠道排期 ──▶ ④协调资源执行           │
│   • 提前3-4周        • 落实素材             │
│   • 梳理推广素材     • 执行落地             │
│                      • 结合用户体验场景     │
│                                              │
│           ⑤监测数据评估效果                 │
│           • CTR、转化率、留存               │
│           • ROI、获客成本                   │
│           • 调整优化                        │
│                                              │
└─────────────────────────────────────────────┘
```

**步骤一：制定推广策略**

| 策略类型 | 目的 | 渠道选择 | 关注重点 |
|---------|------|----------|---------|
| **尝试性推广** | 测试效果、渠道效率 | 少量垂直渠道（微博公众号、社区） | 推广效率、用户表现 |
| **数据拉升型** | 大幅拉升特定数据 | 多渠道同步：搜索广告、网盟、品牌广告 | 效果、覆盖面 |
| **传播/事件型** | 带动品牌传播 | 社会化媒体（微博、微信） | 事件引爆、刷屏效应 |

**资源评估：**
- 评估已有渠道资源是否足够支持预期
- 不足时需要开拓新渠道

**开拓新渠道的方式：**
1. **百度搜索：** 善用搜索引擎发现潜在渠道
2. **用户调研：** 访谈目标用户了解常去的地方
3. **加入行业群：** 微信群、QQ群、合作群
4. **线下沙龙：** 推广和用户增长主题沙龙

**步骤二：确认渠道数据和触达逻辑**

**需要确认的渠道数据：**
- 渠道用户数、日均访问量
- 地域/人群特征匹配度
- 历史获客成本

**渠道触达逻辑案例：**

**案例1：豆瓣小组**
```
用户访问小组列表页
    │
    ▼
看到推广帖子（需要顶帖或置顶）
    │
    ▼
点击标题（标题要吸引人）
    │
    ▼
消费帖子内容（内容转化）
```

**案例2：微信大号**
```
公众号推送
    │
    ▼
用户打开（受推送位、标题、头图、摘要、时间影响）
    │
    ▼
阅读内容
    │
    ▼
用户转发朋友圈/群（内容质量要高）
    │
    ▼
引发二次消费
```

**步骤三：沟通渠道排期和梳理素材**

**排期要点：**
- 好渠道需提前3-4周沟通
- 热门渠道可能需提前1-2月预定

**需要准备的推广素材：**

| 素材类型 | 说明 | 准备要点 |
|---------|------|---------|
| **推广展示位素材** | Banner图、标题文案 | 侧重刺激、吸引眼球 |
| **推广内容页** | 长文案、软文、活动介绍 | 侧重逻辑递进、软植入 |
| **着陆页** | 专题页、首页 | 明确用户引导路径 |
| **站内引导路径** | 各体验点的文案/图片 | 评估用户体验友好度 |

**推广素材准备要点：**

1. **短文案/Banner位**
   - 侧重转化率
   - 刺激用户、吸引眼球
   - 示例："北京公寓价格击穿，50万/套起，精装修高性价比"

2. **长文案/文章式**
   - 侧重阅读体验
   - 层层嵌套、逻辑递进
   - 软植入，内容与推广信息相关

3. **着陆页**
   - **活动/专题页：** 直接引导消费
   - **首页：** 全面了解产品服务
   - 根据期望用户行为选择合适类型

**步骤四：协调资源执行落地**

**执行要点：**
- 与设计师等协作者清晰传达预期
- 确保所有素材事先准备到位
- 按排期执行落地

**核心原则：** 结合用户体验场景和展现形式做好推广素材

**步骤五：监测数据评估效果**

**单一渠道投放关注指标：**

| 指标 | 说明 | 评估重点 |
|------|------|---------|
| **CTR（点击率）** | 展示点击比 | 推广位置、素材效果 |
| **转化率** | 点击后转化的用户比例 | 推广素材、着陆页有效性 |
| **用户留存** | 渠道用户留存情况 | 渠道与产品匹配度 |

**多渠道投放关注指标：**
- **ROI**（投入产出比）
- **获客成本**（对比行业平均值）
- **单位时间新增用户量**

## 📊 本章小结

### 核心方法论总结

**1. 思考框架的价值**
- 往正确方向思考
- 避免"只见树木不见森林"
- 提高沟通说服效率
- 让问题更容易得到好答案

**2. 专业能力 = 思考框架 + 专业知识 + 技能**

**3. 两种学习路径**
- **有经验者：** 框架 → 填充知识 → 针对训练
- **零基础者：** 技能 → 专业知识 → 局部框架 → 完善

**4. 三种拆解思路**
- **分支方向式：** 找独立方向
- **工作流程式：** 按先后顺序
- **公式型：** 拆解成数学公式

**5. 三句重要的话**
- 宽泛问题 → 划定边界 + 分解 = 可落地
- 可量化目标 → 一定能拆解成公式
- 可落地事项 → 一定能找到工作流程

### 行动建议

1. **从日常问题开始拆解练习**
2. **不要追求完美框架，先让拆解成为习惯**
3. **记录自己的思考成果**
4. **在实践中不断优化框架**
5. **框架必须能指导实践，否则没有意义**
