## 本章导学

### 核心目标

本章是10倍放大个人学习效率课程的核心篇章，通过3个经典运营案例的深度拆解，帮助学员掌握不同类型运营活动的策划方法和底层逻辑。这三个案例分别代表了：传播型活动、裂变型活动、留存型活动三种最常见的运营场景。

### 学习重点

本章的学习重点包括：
- 理解传播型活动的策划逻辑和万用模板
- 掌握裂变型活动的核心要素和执行要点
- 学习留存型小游戏的产品设计方法论

### 2.3 得到仗义证：强口碑裂变增长

**案例背景**：

2016年12月，得到APP与马伯庸先生合作推出"仗义证"事件营销。得到花费10万元买下马伯庸的《显微镜下的大明》免费送给用户，同时生成"仗义证"邀请用户帮助传播。活动上线一天内，知乎获5000+赞，微博近1万次转发，微信公众号文章阅读量超6万，一周内下载量超10万。

```
┌─────────────────────────────────────────────────────────────────┐
│                      裂变活动的两大类型                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   利益驱动型裂变：                                              │
│   ├─ 特点：通过利益绑架用户必须转发                            │
│   ├─ 优点：短期效果明显                                        │
│   ├─ 缺点：持续性差、口碑差                                    │
│   └─ 典型：邀请返利、拼团                                      │
│                                                                  │
│   情怀驱动型裂变：                                              │
│   ├─ 特点：送给用户有价值的东西，基于情怀请求帮助               │
│   ├─ 优点：口碑好、持续性强                                    │
│   └─ 典型：得到仗义证                                          │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

**一个玩法、两套流程、4个要点**：

```
┌─────────────────────────────────────────────────────────────────┐
│                  得到仗义证方法论                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   一个玩法：                                                     │
│   送给用户超级有价值的东西，基于情怀理由请求帮助转发              │
│                                                                  │
│   两套流程：                                                     │
│   流程1：APP看到活动 → 生成仗义证 → 分享朋友圈                  │
│   流程2：朋友圈看到 → 扫码领取 → 生成自己的仗义证 → 分享       │
│                                                                  │
│   4个要点：                                                     │
│   1. 给用户超级有价值的东西+有情怀的态度                        │
│   2. 扫码后先给权益，再请求帮助                                 │
│   3. 给用户强烈的身份认同（仗义证）                             │
│   4. 强化情怀理念的文案+一点道德捆绑                           │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

2019年以来，大型APP内涌现大量留存小游戏。蚂蚁森林累计3.5亿用户，种植真树超1000万棵，用户累计使用天数超6亿天。多多果园上线15天用户超4000万，15个月内有200万人收到真实水果。

```
┌─────────────────────────────────────────────────────────────────┐
│                    留存型小游戏底层原理                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   前提1：成熟游戏模式能让用户上瘾和投入                          │
│         → 养成类、闯关类游戏被验证有效                          │
│                                                                  │
│   前提2：玩游戏获得现实收益能增强动机                            │
│         → 蚂蚁森林：种树→真树                                   │
│         → 多多果园：种水果→真水果                               │
│                                                                  │
│   前提3：玩游戏过程中产生特定行为能增加商业价值                  │
│         → 步行、评论、分享等用户行为被关联                       │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

**设计留存型小游戏的5个步骤**：

```

├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   第1步：确定游戏模式                                            │
│   ├─ 宠物模式：养鸡、养熊                                        │

│   ├─ 结合产品核心主张（绿色出行）                                │

│                                                                  │
```

## 案例分析

### 案例1：支付宝中国锦鲤

**成功要素分析**：

| 要素 | 具体做法 | 可复用点 |
|------|---------|---------|
| 奖品策略 | 近200家品牌联合，总价值近1亿 | 资源整合能力强 |
| 话题设置 | "锦鲤"概念深入人心 | 文化符号运用 |
| 传播节奏 | 分阶段话题持续发酵 | 节奏规划清晰 |
| 中奖者运营 | 信小呆自带网红属性 | 人物IP打造 |

**活动执行检查清单**：

```
活动前：
├─ 明确活动目的（品牌传播/拉新）
├─ 确定活动形式和主题
├─ 规划传播节奏
├─ 准备传播资源
└─ 植入话题点

活动中：
├─ 监控数据表现
├─ 及时回应用户
├─ 把握话题热度
└─ 引导二次传播

活动后：
├─ 数据复盘分析
├─ 用户反馈收集
├─ 总结经验教训
└─ 规划迭代优化
```

### 案例2：得到仗义证

**执行细节亮点**：

1. **免费送高价值内容**：10万元购买马伯庸神作免费送

3. **流程极简**：无需手动保存图片，系统自动生成

## 核心概念表格

| 概念 | 定义 | 应用场景 |
|------|------|---------|
| 传播型活动 | 以获取流量和声量为目的的活动 | 品牌曝光、新品发布 |
| 裂变型活动 | 通过用户带动更多用户的增长机制 | 拉新、获客 |

| 社交货币 | 用户愿意分享的身份认同 | 传播设计、裂变机制 |

## 本章小结

本章通过3个经典案例的深度拆解，帮助学员掌握了以下核心能力：

1. **传播型活动策划**：学会使用万用模板低成本获取大传播

2. **裂变型活动设计**：理解情怀驱动 vs 利益驱动的本质区别，掌握一个玩法、两套流程、4个要点的方法论

3. **留存型小游戏设计**：理解小游戏背后的三大底层原理，掌握5步设计法

4. **案例拆解能力**：学习如何从成功案例中提取可复用的方法论

5. **实战应用思路**：将所学方法应用到实际工作中，建立自己的案例弹药库

**核心原则**：
- 活动不是越复杂越好，简单的形式可以创造惊人的效果
- 理解用户心理是做好运营的根本
- 持续拆解案例是提升运营能力的最快途径
- 建立自己的案例库和弹药库至关重要

## 核心概念表格（扩展）

| 概念 | 定义 | 重要性 | 应用场景 |
|------|------|--------|---------|
| 传播型活动 | 以获取流量和声量为目的的活动 | ★★★★★ | 品牌曝光、新品发布 |
| 裂变型活动 | 通过用户带动更多用户的增长机制 | ★★★★★ | 拉新、获客 |
| 留存型活动 | 提升用户活跃度和留存的活动 | ★★★★★ | 用户运营、活跃度提升 |
| PMF | 产品市场契合点 | ★★★★☆ | 产品验证、增长决策 |
| 口碑 | 用户主动传播带来的效应 | ★★★★☆ | 品牌建设、信任建立 |
| 社交货币 | 用户愿意分享的身份认同 | ★★★★★ | 传播设计、裂变机制 |
| 私域流量 | 可自主触达和运营的用户池 | ★★★★★ | 用户运营、复购提升 |
| 增长飞轮 | 多个增长要素相互促进的机制 | ★★★★☆ | 增长策略、系统设计 |
