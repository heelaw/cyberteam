> 主讲人：黄有璨
> 课程范围：从0到100，不同业务如何思考宏观运营操盘节奏 第一章第2节

## 一、两种不同的用户增长类型

对于品牌产品经营型公司，用户增长典型呈现两种状态：
1. **短期价值驱动的用户增长**
2. **长期价值驱动的用户增长**

理解这两者的差异，是做好增长操盘的关键。

---

### 1.1 短期价值驱动的用户增长

#### 核心特征

**定义：**
用户愿意关注产品或品牌，但**无法形成稳定的价值预期**。增长往往是一次性的、不可预测的。

**典型表现：**
- 用户记住的只是某个"现象"而非产品价值
- 增长结束后，用户无法清晰描述产品功能
- 无法回答"这产品能帮我解决什么问题"

**典型形式：**

**形式1：事件营销驱动**
- **案例：** 某APP通过线下快闪活动一夜涨粉百万
- **问题：** 活动结束后，用户留存不足10%
- **原因：** 用户被"事件"吸引，而非产品价值

**形式2：裂变涨粉活动**
- **案例：** "邀请3位好友即可获得XXX"
- **问题：** 裂变结束后，用户大量流失
- **原因：** 利益驱动，非价值驱动

**形式3：投票大赛**
- **案例：** "萌娃萌宝投票大赛"
- **问题：** 活动结束后，APP几乎无人使用
- **原因：** 参与动机是"拉票"，非产品价值

**形式4：短期爆红产品**
- **案例：** 足迹、分答等产品
- **问题：** 一夜爆红后迅速消亡
- **原因：** 缺乏长期价值，新鲜感过后用户离开

**【案例：某教育产品的裂变增长】**

**背景：** 某在线教育公司推出"邀请3位好友免费领课程"活动。

**活动数据：**
- 3天内新增用户：50万
- 裂变系数：1.8（每个用户平均邀请1.8人）
- 短期数据：非常漂亮

**长期数据：**
- 活动结束后第7天：留存率仅8%
- 活动结束后第30天：留存率不足3%
- 转化率：从免费到付费转化不足1%

**问题分析：**
```
用户动机 = 利益驱动（免费课程）
产品价值 = 未被感知
结果 = 活动结束，用户离开
```

**结论：** 短期价值驱动增长，无法持续。

---

#### 短期价值增长的三大问题

**问题1：用户无法形成价值预期**

**表现：**
- 用户说不清产品是干嘛的
- 用户不知道在什么场景下使用
- 用户记不住产品核心功能

**【用户访谈示例】**
> Q：你为什么下载这个APP？
> A：因为朋友说可以领免费课程。
>
> Q：你觉得这个APP能帮你解决什么问题？
> A：嗯……不太清楚，就领了免费课程。
>
> Q：你会继续使用吗？
> A：应该不会吧，课程已经领完了。

**问题2：增长不可预测**

**表现：**
- 依赖外部事件或热点
- 无法复现增长
- 增长呈现锯齿状

**【数据示例】**
```
第1周：事件营销，日活10万
第2周：事件结束，日活回落到2万
第3周：无活动，日活1万
第4周：新活动，日活8万
第5周：活动结束，日活回落到1.5万
```

**问题3：无法形成复利效应**

**表现：**
- 每次增长都需要重新投入
- 无法形成口碑传播
- 无法建立品牌认知

**【公式对比】**
```
短期价值增长：
总用户 = Σ(每次活动新增用户)
无累积效应

长期价值增长：
总用户 = 初始用户 × (1 + 增长率)^时间
有复利效应
```

---

### 1.2 长期价值驱动的用户增长

#### 核心特征

**定义：**
用户对产品能形成**稳定的价值预期**。增长与产品核心功能和体验相关，往往是自发驱动、内驱动的，相对可预测。

**典型表现：**
- 用户能清晰说出产品核心功能
- 用户知道在什么场景下使用产品
- 用户能描述产品带来的价值

**增长驱动力：**
- 用户口碑：自发推荐
- 产品体验：超出预期
- 核心功能：解决真实痛点

**【案例：微信的长期价值增长】**

**核心价值：**
- 即时通讯
- 朋友圈社交
- 公号号内容消费
- 小程序服务

**用户价值感知：**
- "微信是我每天必用的APP"
- "工作、生活都离不开微信"
- "微信帮我连接所有人"

**增长数据：**
- 2011年1月：上线，用户不足100万
- 2012年3月：用户突破1亿
- 2013年1月：用户突破3亿
- 2014年1月：用户突破5亿

**增长驱动力：**
- 口碑传播：用户主动推荐
- 网络效应：好友越多越有价值
- 产品体验：简洁、稳定、好用

**【案例：美团的长期价值增长】**

**核心价值：**
- 外卖配送：30分钟送达
- 价格优惠：比线下便宜
- 选择丰富：各种餐饮选择

**用户价值感知：**
- "点外卖就上美团"
- "美团外卖又快又便宜"
- "美团上什么都能买到"

**增长数据：**
- 2010年：上线团购
- 2013年：外卖业务上线
- 2015年：日订单突破500万
- 2017年：日订单突破1500万

**增长驱动力：**
- 用户习惯：培养点外卖习惯
- 口碑传播：用户主动推荐
- 网络效应：商家越多用户越多，用户越多商家越多

---

#### 长期价值增长的判断标准

**标准1：用户能清晰描述产品价值**

**用户访谈示例（长期价值产品）：**
> Q：你为什么使用这个APP？
> A：因为它能帮我XXX（具体场景）。
>
> Q：你觉得这个APP能帮你解决什么问题？
> A：它能帮我XXX（具体价值）。
>
> Q：你会推荐给朋友吗？
> A：会，因为XXX（推荐理由）。

**标准2：增长曲线呈现平滑上升趋势**

**数据特征：**
- 无大起大落
- 即使不做活动，也有自然增长
- 增长曲线斜率稳定

**标准3：用户留存率和活跃度稳定**

**数据指标：**
```
次留：> 40%
7留：> 30%
30留：> 20%
日活/月活：> 50%
```

**标准4：有稳定的获客渠道**

**表现：**
- 自然流量占比 > 60%
- 用户推荐占比 > 20%
- 不依赖单一渠道

---

### 1.3 两种增长类型的对比

| 维度 | 短期价值驱动 | 长期价值驱动 |
|------|-------------|-------------|
| **价值认知** | 用户说不清产品价值 | 用户能清晰描述产品价值 |
| **使用场景** | 无明确场景 | 有明确使用场景 |
| **增长驱动** | 事件、活动、利益 | 产品体验、口碑、功能 |
| **可预测性** | 不可预测 | 相对可预测 |
| **增长曲线** | 锯齿状，大起大落 | 平滑上升，斜率稳定 |
| **用户留存** | 低，活动后大量流失 | 高，稳定留存 |
| **复利效应** | 无，每次增长需重新投入 | 有，累积效应明显 |
| **品牌价值** | 弱，无法建立品牌认知 | 强，持续积累品牌资产 |
| **典型案例** | 足迹、分答 | 微信、美团、滴滴 |

**【判断公式】**
```
增长健康度 = (用户价值认知度 × 0.3) + (留存率 × 0.3) + (自然增长占比 × 0.2) + (口碑NPS × 0.2)
```

**标准：**
- 健康度 > 0.6：长期价值驱动，健康
- 健康度 0.3-0.6：混合状态，需优化
- 健康度 < 0.3：短期价值驱动，危险

## 二、运营的核心目标：追求长期价值驱动

### 2.1 为什么短期价值增长需要警惕？

**核心观点：**
拉新从来不是最难的，对于品牌产品经营型公司，**"生于拉新，死于留存"** 是最常见的死法。

**【案例：某社区产品的教训】**

**拉新策略：**
- 大规模投放广告
- 策划病毒式裂变活动
- 邀请明星代言

**拉新数据：**
- 3个月新增用户：500万
- 日活峰值：100万
- 表面数据：非常漂亮

**留存数据：**
- 次日留存：15%
- 7日留存：5%
- 30日留存：1%

**结果：**
6个月后，日活仅剩5万，产品走向死亡。

**问题分析：**
```
过度重视拉新 + 忽视产品价值 + 留存率低 = 产品死亡
```

---

### 2.2 "生于拉新，死于留存"的底层逻辑

**为什么会出现这种情况？**

**原因1：过度重视拉新，忽视产品价值**

**表现：**
- 大量资源投入拉新
- 产品价值未验证
- 用户进入后体验差

**原因2：用户留存不足，无法形成口碑**

**表现：**
- 用户进来后很快离开
- 无法形成口碑传播
- 自然增长无法启动

**原因3：陷入"拉新→流失→再拉新"的恶性循环**

**表现：**
- 为了维持日活，不断拉新
- 获客成本越来越高
- 留存越来越差
- 最终无法持续

**【恶性循环公式】**
```
拉新成本 = (营销费用 / 新增用户数)

当 拉新成本 > 用户LTV（生命周期价值）
→ 无法持续
→ 产品死亡
```

---

### 2.3 正确的运营节奏：留存先于拉新

**核心原则：**
在产品中早期，**一定要追求留存先有足够的保障，再去追求拉新**。

**【案例：滴滴的早期策略】**

**阶段1：验证留存（2012.10-2013.06）**
- 重点：北京中关村试点
- 目标用户：深夜加班的程序员
- 核心指标：留存率 > 60%
- 结果：验证了需求真实性

**阶段2：启动增长（2013.07-2014.12）**
- 留存达标后，才开始大规模拉新
- 大规模补贴和投放
- 结果：快速增长，用户留存稳定

**关键数据：**
```
留存验证期：
日活：1000 → 1万
留存率：稳定在60%+

增长启动期：
日活：1万 → 1000万
留存率：稳定在55%+
```

**【公式】**
```
拉新效率 = (新增用户数 × 留存率) / 拉新成本

当 留存率低 → 拉新效率低 → 浪费资源
当 留存率高 → 拉新效率高 → 规模化增长
```

## 三、需要警惕的两种危险增长曲线

如果产品的增长曲线呈现以下两种形状，需要高度警惕：

### 3.1 危险曲线1：爆发后迅速下滑

**曲线形状：**
```
日活
  ↑
  │     ╱╲
  │    ╱  ╲
  │   ╱    ╲___
  │  ╱
  │ ╱
  │╱
  └─────────────────→ 时间
```

**特征：**
1. 产品刚上线后迅速获得爆发性增长
2. 爆发后用户增长迅速下滑
3. 最终跌破原有水位

**【案例：某游戏产品的增长曲线】**

**数据：**
- 上线第1周：日活10万
- 上线第2周：日活50万（爆发）
- 上线第3周：日活30万
- 上线第4周：日活15万
- 上线第5周：日活5万
- 上线第6周：日活2万（低于爆发前）

**问题分析：**
```
爆发增长 → 用户体验差 → 用户大量流失 → 负面口碑 → 更多人离开
```

**应对策略：**

**第一步：立即停止大规模拉新**
- 暂停广告投放
- 停止裂变活动
- 控制新增用户流入

**第二步：全力提升产品价值**
- 分析用户流失原因
- 优化核心功能体验
- 修复产品Bug

**第三步：精准用户运营**
- 识别高价值用户
- 重点服务核心用户
- 过滤非精准用户

**【公式】**
```
产品价值 = (核心用户留存率 × 0.5) + (NPS得分 × 0.3) + (功能使用率 × 0.2)

当 产品价值 < 0.5 → 不要拉新，先做产品
当 产品价值 > 0.7 → 可以启动规模化增长
```

---

### 3.2 危险曲线2：锯齿状增长

**曲线形状：**
```
日活
  ↑
  │  ┌─┐  ┌─┐  ┌─┐
  │  │ │  │ │  │ │
  │ ─┘ └──┘ └──┘ └──
  │
  └─────────────────→ 时间
```

**特征：**
1. 做活动就有增长
2. 不做活动就没增长
3. 增长完全依赖"强心针"

**【案例：某电商产品的增长曲线】**

**数据：**
- 第1周：正常运营，日活10万
- 第2周：大促活动，日活30万
- 第3周：活动结束，日活回落到10万
- 第4周：正常运营，日活10万
- 第5周：大促活动，日活30万
- 第6周：活动结束，日活回落到10万

**问题分析：**
```
增长完全依赖活动 = 产品无自发增长能力 = 产品价值不足
```

**问题诊断：**

**诊断1：产品核心功能不完善**
- 用户无法自然留存
- 无法形成使用习惯
- 无口碑传播

**诊断2：用户体验差**
- 核心流程不顺畅
- 功能设计不合理
- 服务质量不稳定

**诊断3：缺乏长期价值**
- 只有短期利益驱动
- 无稳定的价值预期
- 无法形成品牌认知

**应对策略：**

**第一步：诊断产品价值问题**
- 用户访谈：了解用户真实需求
- 数据分析：找出流失原因
- 竞品分析：对比差距

**第二步：优化产品核心功能**
- 简化核心流程
- 提升用户体验
- 增加长期价值

**第三步：培养用户使用习惯**
- 设计用户成长体系
- 增加用户粘性功能
- 建立用户社区

**【公式】**
```
自发增长能力 = (自然增长占比 / 活动驱动增长占比)

当 自发增长能力 < 1 → 产品有问题，不要做活动
当 自发增长能力 > 2 → 产品健康，可以放大活动
```

---

### 3.3 两种危险曲线的对比

| 维度 | 爆发后下滑 | 锯齿状增长 |
|------|-----------|-----------|
| **曲线形状** | ╱╲ (倒V型) | ─┬─┬─┬ (锯齿型) |
| **典型原因** | 承接能力不足 | 产品价值不足 |
| **主要问题** | 用户体验恶化 | 无自发增长能力 |
| **应对策略** | 停止拉新，优化产品 | 诊断问题，提升价值 |
| **恢复周期** | 3-6个月 | 6-12个月 |
| **恢复难度** | 中等 | 较高 |

## 四、产品生命周期S曲线的理论基础

### 4.1 《跨越鸿沟》理论

**核心观点：**
一款产品推向市场后，会逐次接触到三类用户：
1. **早期市场用户（16%）**
2. **主流大众市场用户（68%）**
3. **晚期市场用户（16%）**

这个分布符合正态分布，也称为"创新扩散理论"。

---

### 4.2 三类用户的详细分析

#### 第一类：早期市场用户（16%）

**组成：**
1. **技术狂热者（Innovators，2.5%）**
   - 特征：对新技术狂热，第一时间尝试
   - 行为：不计成本，只要新就好
   - 示例：排队买iPhone首发的人

2. **高瞻远瞩者（Early Adopters，13.5%）**
   - 特征：能看到新技术的潜在价值
   - 行为：愿意承担风险，早期尝试
   - 示例：企业中的技术决策者

**用户画像：**
- 年龄：25-40岁
- 职业：IT、互联网、媒体等
- 特征：猎奇心理强，接受度高
- 对产品的要求：可以不完美，但要有新意

**运营策略：**
- 聚焦小众圈层
- 强调创新和前瞻性
- 建立深度的用户关系
- 收集用户反馈，快速迭代

---

#### 第二类：主流大众市场用户（68%）

**组成：**
1. **早期大众（Early Majority，34%）**
   - 特征：谨慎但愿意尝试
   - 行为：看到他人使用后再尝试
   - 示例：看到同事用后才开始用微信的人

2. **后期大众（Late Majority，34%）**
   - 特征：保守，跟随者
   - 行为：等到大众都在用后才用
   - 示例：看到身边人都在用才开始用淘宝的人

**用户画像：**
- 年龄：广泛，从20-60岁
- 职业：各行各业
- 特征：从众心理，风险厌恶
- 对产品的要求：必须稳定、好用、安全

**运营策略：**
- 强调产品的稳定性和易用性
- 展示大量用户案例和社会认同
- 降低使用门槛和风险
- 提供完善的客服和保障

---

#### 第三类：晚期市场用户（16%）

**组成：**
1. **保守派（Laggards，16%）**
   - 特征：极度保守，抗拒改变
   - 行为：最后才使用新产品
   - 示例：直到2020年还在用功能手机的人

**用户画像：**
- 年龄：50岁以上
- 特征：信息获取渠道少，学习能力强弱
- 对产品的要求：极度简单，无需学习

**运营策略：**
- 提供极致简化的产品体验
- 线下地推和手把手教学
- 亲情化服务，建立信任
- 耐心培育，不强求

---

### 4.3 三类用户的行为特征对比

**【案例：智能手机的普及】**

**早期市场用户（2007-2010）：**
- 行为：排队买iPhone，不看价格
- 动机：新技术、新体验
- 特征：愿意承担不成熟的风险

**主流大众市场用户（2010-2015）：**
- 行为：看到朋友在用，自己也开始用
- 动机：社交需求、实用需求
- 特征：要求产品稳定、好用

**晚期市场用户（2015-至今）：**
- 行为：子女给买了智能手机，才学会用
- 动机：被动接受，不得不学
- 特征：学习困难，需要帮助

**【服装消费的类比】**

**早期市场用户：**
> "市面上这么多人都在穿中国李宁，我就一定不要穿中国李宁，我要追求我的潮流感，穿一些小众设计师品牌。"

**主流大众市场用户：**
> "我看到身边很多朋友都在穿优衣库，评价都不错，所以我也去优衣库买衣服。"

**晚期市场用户：**
> "大家都在穿什么衣服，我就跟着穿什么，不会出错。"

### 4.4 为什么产品增长曲线呈现S型？

**核心逻辑：**

**阶段1：引入期（慢增长）**
- 用户：仅早期市场用户（16%）
- 原因：大众用户不接受、不信任
- 增长：缓慢

**阶段2：成长期（快增长）**
- 用户：开始进入主流大众市场用户（68%）
- 原因：口碑传播，社会认同形成
- 增长：加速

**阶段3：成熟期（增长放缓）**
- 用户：市场趋于饱和
- 原因：潜在用户减少
- 增长：放缓

**阶段4：衰退期（增长下滑）**
- 用户：开始转向新产品
- 原因：产品老化，新竞争者出现
- 增长：下滑

**【公式】**
```
S曲线增长 = 早期用户(16%) + 大众用户(68%) + 晚期用户(16%)

早期：仅获取16%早期用户，增长慢
中期：获取68%大众用户，增长快
后期：获取16%晚期用户 + 流失，增长放缓
```

## 五、本章核心要点总结

### 5.1 两种增长类型的核心差异

| 维度 | 短期价值驱动 | 长期价值驱动 |
|------|-------------|-------------|
| **价值认知** | 用户说不清 | 用户能清晰描述 |
| **增长驱动** | 事件、活动、利益 | 产品、体验、口碑 |
| **可预测性** | 不可预测 | 相对可预测 |
| **留存率** | 低 | 高 |
| **复利效应** | 无 | 有 |

### 5.2 运营的核心目标

**追求长期价值驱动，而非短期价值驱动。**

**正确节奏：留存先于拉新**
1. 先验证留存（留存率 > 50%）
2. 再启动增长（规模化拉新）
3. 避免"生于拉新，死于留存"

### 5.3 需要警惕的两种危险曲线

1. **爆发后下滑：** 产品承接能力不足，需停止拉新，优化产品
2. **锯齿状增长：** 产品价值不足，需诊断问题，提升价值

### 5.4 产品生命周期S曲线的理论基础

**《跨越鸿沟》理论：**
- 早期市场用户（16%）
- 主流大众市场用户（68%）
- 晚期市场用户（16%）

**为什么是S型？**
- 引入期：仅早期用户，增长慢
- 成长期：进入大众市场，增长快
- 成熟期：市场饱和，增长放缓
- 衰退期：用户流失，增长下滑
