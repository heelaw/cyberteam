## 一、课程定位与核心价值

### 1.1 课程背景介绍

大家好！今天我们要学习的是一门核心课程——《构建业务增长和运营操盘的大局观 加强业务规划的能力》。

我是这门课程的主讲人，也是有瞰学社的创始人和CEO，黄有璨。

在正式开始之前，让我们先来理解一个在近年来出现频率越来越高的词——**"操盘"**。

### 1.2 什么是"操盘"？

**核心定义：**

所谓操盘，就是在一家公司里面能够不断面对各种变化（行业变化、外部竞争、技术变革、商业模式演变），利用有限的资源（资金、人力、时间），通过制定策略和管理对人对事，最终在一段时间内更有效实现目标的能力。

**具体到用户增长操盘：**
- 面对外部竞争压力
- 手中资金和人力资源有限
- 需要打好手中这场仗
- 最终实现更好的用户增长

**【案例：某外卖平台早期的增长操盘】**

**背景：** 2014年某外卖平台A进入市场，面临已有1年先发优势的竞品B，资金只有竞品的1/3。

**分析：** 平台A团队分析后发现，竞品B在高校市场覆盖率虽高，但配送效率低（平均45分钟），用户投诉率高。

**策略：**
1. 集中资源在3个城市的高校市场
2. 建立"30分钟必达"的服务标准
3. 招募大学生作为兼职配送员（成本更低）
4. 通过学生会组织进行精准推广

**数据：** 6个月后，平台A在目标高校市场份额达到65%，日订单量从500单增长到15000单，资金消耗仅为预期的70%。

**结论：** 操盘能力体现在：**精准的资源分配 + 清晰的策略选择 + 高效的执行管理**。

**启示：** 有限资源下，**聚焦**比**全面铺开**更重要。

## 二、运营操盘面临的典型难题

在实际工作中，做好业务操盘、运营和增长操盘，会遇到很多复杂问题。以下是四大核心难题：

### 2.1 难题一：爆红后的快速消亡

**【案例：足迹APP的兴衰】**

**背景：** 足迹APP在2015年凭借"大片模式"功能一夜爆红，3天内用户量从10万暴涨到500万。

**问题呈现：**
- 为什么有的产品爆红后狂涨一波粉，很快就会死？
- 如何避免"一波流"命运？

**核心原因分析：**

**原因1：只有短期刺激性价值，缺乏长期稳定价值预期**

用户使用足迹的核心动机：
- 猎奇心理（"大片模式"很新鲜）
- 社交展示需求（分享到朋友圈有面子）

但问题是：
- 用户不知道在什么场景下持续使用
- 产品无法回答"我能帮你解决什么问题"
- 新鲜感过后，缺乏持续使用动力

**数据支持：**
- 爆红后首周：日活率45%
- 爆红后1个月：日活率降至12%
- 爆红后3个月：日活率不足3%

**原因2：承接能力不足，大量用户涌入导致体验恶化**

**具体表现：**
- 服务器连续3天崩溃
- 图片加载时间超过10秒
- 功能使用卡顿严重

**用户反馈示例：**
> "下载了根本打不开"
> "等了5分钟还没加载出来"
> "太卡了，直接卸载"

**原因3：非精准用户涌入，伤害核心用户体验**

**原有核心用户画像：**
- 文艺青年，20-30岁
- 热爱摄影和创作
- 喜欢在社区内深度互动

**爆红后新增用户特征：**
- 大众用户，年龄跨度大
- 好奇驱动，无创作习惯
- 随意发布低质内容

**用户体验恶化：**
- 原有用户：社区氛围被破坏，高质量内容被淹没
- 新增用户：看不懂原有内容，不知道如何使用

**【可落地的预防公式】**

**爆红前的准备公式：**
```
安全用户容量 = 日活承载能力 × 0.7
```

**具体指标：**
- 服务器承载：QPS（每秒查询率）阈值 × 0.7
- 客服团队：每人可处理用户数 × 0.7
- 内容审核：每日审核能力 × 0.7

**爆红后的应对公式：**
```
优先级 = 用户精准度 × 留存率贡献度
```

**具体策略：**
1. 优先服务高精准度用户
2. 对非精准用户进行功能引导或过滤
3. 控制新增用户流入速度

---

### 2.2 难题二：快与慢的抉择

行业中有两种声音：
- 有人说"唯快不破"
- 有人说要"稳"要"慢"

**如何判断什么时候该快、什么时候该慢？**

**【案例对比：美柚 vs 大姨妈】**

**美柚（快节奏）：**
- 2013年3月上线
- 前1年版本迭代：每月2-3个版本
- 2015年用户数：8000万（超过大姨妈）

**大姨妈（慢节奏）：**
- 2012年1月上线（比美柚早1年）
- 早期版本迭代：1-2个月1个版本
- 2013年被美柚反超

**关键差异：**

| 维度 | 美柚（快） | 大姨妈（慢） |
|------|-----------|-------------|
| PMF前版本迭代 | 每月2-3个版本 | 1-2个月1个版本 |
| 核心功能上线速度 | 3个月完成工具→内容→UGC | 9个月完成同样路径 |
| 市场反馈响应 | 2周内迭代修复问题 | 1-2个月才修复 |
| 用户增长拐点 | 上线后4个月出现 | 上线后12个月出现 |

**快慢判断公式：**
```
迭代速度 = 市场竞争压力 × 用户需求紧急度 × 团队能力
```

**具体判断标准：**

**应该快的情况：**
1. 处于现有市场（已有竞品）
2. 产品价值尚未验证（PMF前）
3. 用户需求明确且紧急
4. 团队执行能力强

**应该慢的情况：**
1. 处于全新市场（需要教育用户）
2. 需要建立社区氛围和信任
3. 产品需要深度打磨体验
4. 面临的是长期品牌建设

### 2.3 难题三：高频vs低频产品的运营体系差异

**高频产品示例：** 外卖、打车、社交
**低频产品示例：** 殡葬服务、婚庆、房产交易

**【案例：美团外卖 vs 某婚庆平台】**

**美团外卖（高频）：**
- 用户使用频次：每周3-5次
- 运营重点：
  - 用户习惯培养（红包激励）
  - 个性化推荐（基于历史订单）
  - 会员体系（连续订餐优惠）
  - 即时满足（30分钟送达）

**某婚庆平台（低频）：**
- 用户使用频次：一生1-2次
- 运营重点：
  - 流量入口占据（SEO、搜索引擎）
  - 品牌认知建设（"结婚就上XX"）
  - 转化链路优化（从咨询到成交）
  - 客单价提升（增值服务捆绑）

**运营体系差异公式：**

**高频产品：**
```
用户价值 = 使用频次 × 单次客单价 × 留存时间
```
运营重点：提升频次和留存

**低频产品：**
```
用户价值 = 客单价 × 转化率 × 复购推荐率
```
运营重点：提升转化和客单价

### 2.4 难题四：企业从0到1再到100的阶段把握

**企业发展阶段的三个关键里程碑：**

**阶段1：0→1（生存期）**
- 核心问题：产品是否有市场价值
- 关键指标：PMF验证、种子用户留存
- 典型周期：6-18个月

**阶段2：1→10（成长期）**
- 核心问题：能否规模化增长
- 关键指标：增长速度、市场份额
- 典型周期：1-3年

**阶段3：10→100（成熟期）**
- 核心问题：能否持续盈利和扩张
- 关键指标：利润率、用户LTV
- 典型周期：3年以上

**【案例：滴滴出行的三个阶段】**

**0→1（2012.10-2013.12）：**
- 重点：验证打车需求，建立司机-乘客连接
- 关键动作：北京中关村试点，专注深夜加班场景
- 里程碑：日订单突破10万

**1→10（2014.01-2015.12）：**
- 重点：快速扩张到全国，补贴大战
- 关键动作：大规模补贴，并购快的
- 里程碑：日订单突破1000万

**10→100（2016.01-至今）：**
- 重点：盈利模式探索，生态建设
- 关键动作：推出专车、代驾、金融等业务
- 里程碑：整体盈利，估值超500亿美元

## 三、做好用户增长操盘的三大核心要求

要做好用户增长操盘，需要具备三大核心能力：

### 3.1 核心要求一：理解用户增长的基本客观规律

**无论哪个行业，谋求增长都遵循一些基本规律。**

**核心规律1：产品生命周期S曲线**

任何产品都会经历四个阶段：
- 引入期（慢增长）
- 成长期（快增长）
- 成熟期（增长放缓）
- 衰退期（增长下滑）

**核心规律2：用户增长拐点定律**

在产品发展过程中，存在一个关键拐点：
- 拐点前：增长缓慢，需要依靠运营驱动
- 拐点后：增长加速，开始进入自发增长阶段

**核心规律3：用户分层递进规律**

用户获取必然遵循：
- 早期市场用户（16%）→ 大众市场用户（68%）→ 晚期市场用户（16%）

### 3.2 核心要求二：深刻理解自己的业务

**通过业务分析拆解，找到最核心的增长驱动力。**

**业务分析三步法：**

**第一步：明确业务类型**
- 流量生意 vs 品牌产品经营型
- 高频 vs 低频
- 平台型 vs 工具型

**第二步：找到核心业务模型**
- 示例：滴滴的"司机-乘客双边网络"
- 示例：美团的"用户-商家-骑手三角模型"

**第三步：定位核心增长驱动力**
- 是靠补贴驱动？
- 是靠产品体验驱动？
- 是靠内容驱动？
- 还是靠社交关系驱动？

### 3.3 核心要求三：拥有核心专长和管理能力

**战场上能找到突破口的具体能力：**

**增长能力矩阵：**
```
核心专长（至少一项）+ 综合管理能力
= 成功的操盘手
```

**常见核心专长：**
- 社群运营：能快速建立用户社群并激活
- 内容运营：能生产爆款内容并传播
- 数据分析：能通过数据发现增长机会
- 产品设计：能设计出用户喜爱的产品
- 渠道拓展：能快速拓展获客渠道

**管理能力要求：**
- 目标拆解能力：将大目标拆解为可执行的小目标
- 团队激励能力：调动团队积极性
- 资源协调能力：协调各部门资源
- 风险控制能力：识别和控制执行风险

## 四、课程核心价值与适用人群

### 4.1 课程核心内容

本课程聚焦于**第一大块**——理解用户增长背后的基本客观规律：
- 揭示哪些规律在生效
- 如何指导工作规划
- 如何让节奏把控更清晰
- 如何少踩坑

**课程结构：**
1. **第一节：** 两类完全不同的"生意"与最重要的基本定律
2. **第二节：** 以产品生命周期为中心的运营和用户增长操盘
3. **第三节：** 不同业务中的运营和增长操盘逻辑差异
4. **第四节：** 模块知识点回顾&串讲

### 4.2 适用人群

**最佳匹配：工作3-4年以上的运营/增长从业者**

**典型场景：**
- 已成为团队业务骨干
- 或是初中级管理者
- 需要从执行转向规划
- 需要建立宏观大局观

**课程带来的价值：**
- 揭示运营和用户增长的重要基本规律
- 重塑从宏观上对运营和业务增长的认知
- 在思考整体运营和增长操盘时更加游刃有余

## 六、课程学习前置准备

**在学习本课程前，建议思考以下问题：**

1. **我所在公司的业务类型是什么？**
   - 流量生意还是品牌产品经营型？
   - 高频还是低频？

2. **我们当前产品处于生命周期的哪个阶段？**
   - 引入期？成长期？成熟期？衰退期？

3. **我们当前最核心的增长瓶颈是什么？**
   - 用户获取困难？
   - 留存率低？
   - 变现能力弱？

4. **我个人的核心专长是什么？**
   - 我最擅长哪个运营领域？
   - 我在团队中的角色是什么？

带着这些问题进入课程学习，收获会更大。
