# 第一章：运营的底层逻辑

## 本章导读

### 运营的本质：从业务层面理解运营

在传统认知中，人们往往以工作内容来定义运营岗位——内容运营、用户运营、活动运营、新媒体运营、产品运营、社群运营等。这种分类方式虽然常见，但实际上存在一个根本性问题：**以工作内容为中心去看待运营，往往会导致定位模糊和困惑**。

举例来说，很多运营人会有这样的困惑：自己是用户运营，但同时要做活动、写软文，到底该如何定位自己？或者被冠以"内容运营"title的人，是否就应该只专注内容，而不必关注用户和活动？这种困惑正是源于以工作内容为中心的思维局限。

**跳出工作内容，从业务层面看待运营**，会发现万变不离其宗。任何一项互联网业务的运转，都必然围绕三个核心命题展开：

| 业务阶段 | 核心问题 | 运营目标 |
|---------|---------|---------|
| 获取用户 | 如何让用户知道并使用产品？ | 用户增长 |
| 用户维系 | 如何让用户持续使用产品？ | 提升活跃度与留存 |
| 获取收入 | 如何从用户身上获得收益？ | 商业变现 |

这三个问题是所有互联网业务都必须回答的核心问题。无论运营的具体工作内容是推广、管群、写软文还是做活动，其导向都无外乎这三个目标：**获取用户、维系用户、获取收入**。剩余的一切都只是手段，而非目的。

### 运营与传统行业的本质区别

很多人会问：互联网语境下的"运营"与传统行业的市场推广、用户管理、营销广告有何区别？这三者之间的边界似乎十分模糊。造成这种困惑的原因在于：传统行业有明确的分工定义，而互联网将这些职能统一为"运营"这一模糊概念。

**运营与传统行业相关职能的三个核心差异**：

**差异一：用户互动场景更加丰富与复杂**

在传统行业，用户获取的方式相对单一——杂志广告、线下推广、电话营销等。在这种场景下，考虑的因素比较简单：给用户一个明确动机，完成一个动作即可。

但到了互联网环境，用户的转化路径变得极其复杂。以在微信大号投放广告引导用户下载APP为例：

```
用户路径：
微信图文 → 点击阅读原文 → 跳转着陆页 → 下载APP → 注册 → 进入首页 → 使用功能 → 决定是否留存
```

每一个环节都需要给用户足够的动机和理由。这种复杂程度使得运营者必须对用户心理有更精细的把握，对流程和场景有更深入的理解。

**差异二：用户话语权增大，需要撬动用户力量**

互联网的发展本质上是不断赋予个体更强的能量。十多年前，获取信息只能依赖电视台和报刊杂志；如今，每个人都可能引爆一场事件，朋友圈和知乎已成为主要信息入口。

在用户维系方面，传统模式下用户之间是孤立的，只需触达每个用户即可。但在互联网逻辑下，用户之间可以彼此连接和影响。因此，做好用户维系需要帮助用户之间建立关系和价值供给网络，思考方式完全改变。

**差异三：渠道和玩法更加动态**

传统行业的渠道效果相对稳定可预测，而互联网渠道半年一小变、一两年一大变是常态。以微博为例：

- 2012-2013年：大号投放效果极佳，几毛钱就能获取一个用户
- 2013年底：大号价格飙升，效果下降
- 2014年：九宫格推广玩法出现，又一波红利期
- 如今：各种玩法效果都已大大不如当年

这种动态性要求运营者必须：
- 对新渠道、新产品保持高度敏感
- 在既有渠道上不断推陈出新
- 在新平台和新玩法出现时及时抓住红利

### 运营的核心能力要求

综合以上分析，运营是一项看似琐碎杂乱、实则要求很高的工作。它要求从业者：

1. **理解多样用户场景**：能够将各种场景串联起来思考
2. **撬动用户互动**：找到办法让用户之间产生连接和影响
3. **保持创新敏感度**：对新事物保持高度敏感，不断尝试新玩法
4. **具备学习能力**：互联网变化极快，基本概念和问题需能通过自主学习解决

**核心结论**：运营最应具备的能力是学习能力。每一年互联网行业都会发生天翻地覆的变化，唯有持续学习才能跟上节奏。

## 第一节：什么是运营——运营的底层逻辑

### 运营的业务本质

任何业务的根本立足点都是提供某种产品或服务。在业务发展过程中，必须考虑三件事：

1. **获取用户**：让用户知道并使用产品
2. **维系用户**：让用户持续使用并产生粘性
3. **获取收入**：通过用户获取商业价值

这三件事是业务良性运转的必要条件，任何业务都跳不出这个框架。

### 运营的目的导向

从业务层面以目的为导向重新定义运营：

> **运营的核心目的 = 获取用户 + 维系用户 + 获取收入**

这三者是目的，其余一切（内容、活动、社群、推广等）都是手段。这个定义解决了传统运营岗位定位模糊的问题——无论什么title，运营工作都服务于这三个目的。

### 运营与传统行业的本质区别

**区别一：影响用户的场景更复杂**

传统用户获取方式单一——杂志广告、线下推广、电话营销等，路径短、考虑因素少。

互联网用户转化路径极其复杂。以APP推广为例，用户需要经历：微信图文阅读 → 点击阅读原文 → 跳转着陆页 → 下载APP → 注册 → 进入首页 → 使用功能 → 决定是否留存。

每个环节都需要给用户动机和理由，要求运营者对用户心理有精细把控。

**区别二：用户话语权更大，需撬动用户力量**

互联网赋予个体强大能量，用户之间可以彼此连接和影响。做好用户维系需要帮助用户之间建立关系网络，思考方向与传统方式完全不同。

**区别三：渠道和玩法更动态**

互联网渠道和玩法变化极快。以微博为例：
- 2012-2013年：大号投放效果好，几毛钱获取一个用户
- 2013年底：大号价格飙升，效果下降
- 2014年：九宫格推广玩法出现，新红利期
- 如今：各种玩法效果都已减弱

运营需要持续关注新渠道、新玩法，在红利期及时入场。

### 运营工作的核心认知

> **运营是一项看似琐碎杂乱，但实质要求极高的工作。**

运营需要：
- 理解多样用户场景并串联
- 思考如何撬动用户互动
- 不断尝试新玩法
- 对新事物保持敏感

从商业和业务层面理解运营，清楚每项工作对业务的帮助（用户增长、用户维系还是收入获取），有助于走得更远。

## 第二节：怎么做运营——做符合产品生命周期的运营

### 产品生命周期的运营逻辑

**核心观点：不是所有阶段都适合大规模用户增长**

很多人认为用户数越多越好，但实际情况复杂得多。一个真实案例：

某产品在2013年6月20日前站内用户活跃数比较稳定，6月20日前后做了一波刺激用户的活动。短期效果显著，一周内用户活跃数拉升了5倍多。但活动结束后，用户活跃数一路走低，到7月份甚至比做活动前还低。

这就是典型的"打药时间不对"的案例——在产品还不够成熟时使用猛药，试图带动产品成长，反而导致产品死亡。

### 不健康运营曲线的两种典型形态

**曲线一：极速增长后一路下滑**

```
表现：探索期迅速拉动用户增长，迅速到达顶点，然后一路下行，再也拉不回来
危害：产品基本报废
```

**曲线二：波浪形曲线**

```
表现：用户活跃度完全依赖运营活动，活动做用户就上来，停下来就回落
危害：产品本身缺乏吸引用户长期留存的理由
```

### 两条重要结论

**结论一：只有拥有自发增长能力的产品才有运营价值**

产品的长期价值是基础。如果产品长期价值不明确，仅依靠运营创造的短期价值拉动增长，这个增长一定是短期的。

只有产品自身具备自发增长能力时，运营放大其价值才有意义。这样拉来的用户更实在，留存可能性更高。

**结论二：要做符合产品生命周期的运营**

产品生命周期呈S型曲线，不同阶段有不同特征和运营侧重点。

### 各生命周期的运营策略

**探索期（引入期）**

特征：
- 市场前景不明朗
- 产品体验可能不完善
- 用户需求待验证

运营侧重点：
- 持续探索和快速产品迭代
- 打造用户口碑
- 核心任务：验证产品价值，让用户认可

探索期运营核心：
> **唯一方式：提供给用户超出预期的体验**

举例：用户在电商网站买鞋，收到鞋后发现还送了大量周边配件，甚至有人定期回访关怀。这种超出预期的体验会让用户主动分享，形成口碑。

**成长期**

特征：
- 竞争者纷纷进入
- 市场快速增长
- 需要快速占领市场份额

运营侧重点：
- 唯一核心：增长足够快
- 找到核心增长引擎（补贴、广告投放、事件策划等）
- 迅速占领市场份额，建立稳固地位

典型案例——滴滴出行：
- 2014年年初进入成长期
- 核心增长引擎：红包补贴
- 通过各种方式发红包，快速获取用户
- 树立标杆案例（如司机通过滴滴改善收入）并广泛传播

**成熟期**

特征：
- 用户存量市场吃得差不多了
- 增长空间有限
- 需要考虑商业变现

运营侧重点：
- 用户活跃度提升（榨取用户价值）
- 商业变现探索
- 运营体系精细化

典型案例——美柚：
- 2015年3月进入成熟期
- 建立签到积分体系，激励用户活跃
- 引入广告，开始商业变现
- 上线虚拟货币系统（柚币）
- 站内活动丰富多样

**衰退期**

特征：
- 用户开始流失
- 市场萎缩

运营侧重点：
- 用户防流失
- 用户维系

### 产品与运营的节奏逻辑

> **互联网运营逻辑：节奏性、回报后置**

互联网产品往往通过早期大量补贴用户、创造价值，培养用户认可度和忠诚度，后期才获得商业化可能性。

**忌讳：急功近利的心态**

传统销售思维是"做一点事迅速得到回报"，但互联网运营需要目光更长远，先与用户建立良好关系，才能产生更大可能性。

## 第三节：运营的通用思维——精益运营

### 什么是不确定性因素

在日常工作生活中，当事情没有按照预期发展时，需要找出原因。但在找到真正原因前，会遇到很多不确定因素。

**案例：便利店卖蔬菜水果的决策风险**

某便利店老板听到顾客抱怨买菜远，萌生在便利店内出售蔬菜水果的想法。这个想法中存在诸多不确定性：

**顾客层面**：
- 顾客是否真的需要在便利店买蔬菜水果？
- 顾客是需要蔬菜还是蔬果一起？
- 有多少顾客有在便利店买菜的需求？
- 小区客户能接受的蔬菜水果价格是多少？

**收入成本层面**：
- 这项生意能带来多少收入？
- 成本有哪些？能否获利？

**运营层面**：
- 蔬菜如何配送？如何保鲜？

这些不确定性因素是推论时容易忽略的问题，需要在真正开始前进行调研、分析和判断。

### 精益的本质

精益的本质是**避免浪费**，包括时间、金钱、人力、一切资源的浪费。

**精益创业的核心方法论：MVP（最小可行产品）**

精益创业的核心思想是：先在市场中投入一个极简的、能帮助跑通核心业务流程的原型产品。不需要好看、复杂、多功能，只需要一个非常简单的东西，通过持续学习用户反馈，不断迭代优化。

```
精益创业循环：
idea → 极简产品方案 → 投入市场 → 评估（用户反馈） → 数据分析 → 优化迭代 → 循环
```

### 精益运营的内涵

精益运营与精益产品相似，区别在于：运营需要连接产品和用户，中间有一个关键要素——**用户使用场景**。

精益运营的定义：
> **以最低成本搭建最贴近真实的场景，基于这个场景把产品推到用户面前，验证最关键的运营假设是否成立。**

验证内容包括：
- 用户是否会使用产品
- 用户是否对活动感兴趣
- 用户对活动时间和地点是否接受

**关键要点：场景必须贴近真实**

例如：让用户报名付费课程，验证场景也应该是付费的，而不仅仅是预报名。如果只是让用户填调查问卷，意义不大，因为场景差异太大。

### 精益运营的关键建议

**建议一：产品上线前理解核心价值**

在产品上线前试着写一篇PR稿，讲清楚产品核心价值。如果能写完且有说服力，说明理解到位；如果写不完或没有说服力，需要拉着产品经理重新讨论核心价值。

**建议二：上线前梳理不确定性因素**

将所有不确定性因素逐项列出，评估哪个是核心不确定性因素——即其他所有事情发生的前置条件。

**建议三：高度重视变更成本高昂的产品**

对于上线成本或方案更改成本很高的产品、非成功不可的项目，要舍得花更多时间验证不确定性因素。运营在这个阶段多投入时间，能帮助团队节省更多资源。

**建议四：验证必须面向用户**

验证必须是面向用户的，而不是面向老板或产品的。要给用户真实场景，避免无效验证（如仅做几个问卷）。

### 精益运营的实施流程

```
流程：
1. 构思idea
2. 找到核心不确定性因素
3. 搭建场景进行验证 ← 关键步骤
4. 验证成功 → 落地执行
   验证失败 → 重新思考idea
```

**核心要点：搭建的场景要与真实使用场景尽可能一致**

发一万份调查问卷问用户是否感兴趣，往往与用户实际行为不一致。因为填问卷的场景和真正在网上看到产品后是否愿意注册互动的场景完全不同。

# 第二章：运营人的通用工作方法

## 本章导读

运营工作需要掌握一套通用工作方法，帮助提升效率、建立标准、快速复制。本章将介绍SOP和项目管理两大核心方法论。

---

## 第一节：SOP的含义及能解决的问题

### 什么是SOP

SOP = Standard Operating Procedure（标准作业程序）

定义：把某一事件或某项工作的标准操作步骤和要求，以统一的格式描述出来，形成文档，用以指导日常工作。

SOP需要界定：
- 工作应该产出什么
- 工作步骤应该是什么
- 工作要求是什么

### SOP能解决的两类问题

**问题一：初级运营的工作迷茫**

初级运营常面临这种情况：老板分配大量工作，如社区运营包含内容审核、用户互动、精华内容推荐、活动策划、核心用户关怀等。事情繁多，没有标准工作文档时，容易：
- 坐在办公桌前一脸茫然
- 干着东边忘了西边
- 经常丢三落四
- 目标和流程不清晰
- 效率低下

**问题二：中级运营带人困难**

中级运营可能需要带团队，新人进来没有工作经验。逐件讲解即使讲清楚，对方也不一定能完全记住。

好的SOP应该达到的效果：
> **如果一项工作原先能干的90-100分，照着SOP做的新人至少能做到70分左右**

### SOP的核心价值

| 维度 | 价值体现 |
|-----|---------|
| 个人层面 | 提升工作效率，尤其在工作内容杂乱时 |
| 团队层面 | 建立工作标准，快速复制工作能力 |

---

## 第二节：建立和使用SOP的具体做法

### SOP建立的四步流程

**步骤一：明确工作流程、步骤和要求**

明确该项工作在时间维度上的要求，如必须在什么限定时间内、精确到几点几分完成等。

**步骤二：明确单一步骤的工作产出标准、资源支持、参考范例**

因为文档要给别人看，必须足够清晰具体。

**步骤三：撰写SOP文档**

**步骤四：参照文档执行落地**

### SOP撰写要点

关键原则：
> **每一项工作的具体操作步骤和工作产出要求一定要非常具体和精细，这样文档才具备指导意义。**

### 案例：微博运营SOP

**场景**：运营一个几十万粉丝的微博，每天发10条左右内容

**工作流程五步法**：

| 步骤 | 内容 | 说明 |
|-----|------|------|
| 1 | 设定栏目和发布时间 | 规划每天发几条、几点发、什么类型 |
| 2 | 按周收集生产素材 | 确定内容来源、风格要求 |
| 3 | 排期和预发布 | 用工具定时发布 |
| 4 | 重点推广内容策划 | 重点内容的文案和包装 |
| 5 | 发布后维护 | 评论回复、互动频次、话术 |

**每个步骤的详细界定（以第一步为例）**：

需要界定清楚：
- 微博有多少个栏目
- 每个栏目的内容风格和要求
- 每个栏目每天发布的具体时间（精确到分钟）

**素材收集环节的界定**：

需要界定清楚：
- 常用素材内容来源（归类整理）
- 素材标准（符合什么要求）
- 参考范例
- 备用素材库（收集但暂时用不上的内容）

**排期发布环节的界定**：

需要界定清楚：
- 常规内容排期表
- 发布样式范例
- 预发布工具（如皮皮时光机）

**重点推广环节的界定**：

需要界定清楚：
- 重点内容排期表
- 文案要求（不同风格/类型）
- 发布样式范例

**发布后维护环节的界定**：

需要界定清楚：
- 维护频次（一天看几次）
- 常见问题回复话术
- 不同用户互动语气/风格
- 互动范例

### SOP文档的实际形态

SOP文档通常包含：
- 栏目规划表（发布时间精确到分钟，每个时间段对应栏目）
- 素材来源与风格要求表（每个栏目对应的内容来源、风格分析、范例）
- 重点内容排期表（每周重点推广内容的时间节点和策划要求）

## 第三节：项目管理的典型场景和操作方法

### 项目管理的典型场景

**场景一：负责具体项目（如活动上线）**

需要同时对接研发、产品、设计等多个部门，自身也有一系列工作（如活动宣传、文案撰写），必须在固定时间窗口内完成。

**场景二：新产品上线运营**

新产品上线需要做好一系列运营工作：关键页面文案、Banner位维护、站内活动、核心用户拓展和维系等，这些工作也需要在既定时间窗口内完成。

### 项目管理解决的核心问题

> **在既定时间窗口内，确保一系列必须完成的事情，协调众多人和资源共同投入。**

### 项目管理的操作步骤

**步骤一：梳理待办事项和资源**

- 列出所有待办事项（To Do List）
- 明确项目截止时间（Deadline）
- 梳理项目相关成员（可用的人力资源）
- 列出可用资源（钱、物料、推广渠道等）

**步骤二：制定可视化管理图表**

- 所有待办事项
- 每项事项预期完成时间
- 当前进展状态
- 对所有项目组成员清晰可见

**步骤三：设定里程碑（Milestone）**

- 设定关键里程碑
- 定期检查项目进展
- 确认产出和当前状态是否正常

**步骤四：信息同步与会议沟通**

- 项目启动时召开会议
- 同步项目信息
- 明确分工和角色
- 确保每个人知道自己的任务以及与他人的关联

**步骤五：及时跟进责任人**

- 分工明确后
- 及时跟进相应责任人
- 确认产出及时到位

**步骤六：风险控制**

- 识别重要且可能延期的事项
- 制定B方案
- 动用所有可用资源和办法降低风险

### 项目管理的核心技能

> **项目管理中最重要的事：沟通、推动、信息同步和资源协调**

**沟通**：
- 项目成员众多，每人都有各自工作
- 需要靠沟通确保他们重视项目、按时完成产出

**推动**：
- 即便沟通良好，有时成员会遇到困难或压力
- 需要发挥推动能力，锲而不舍地跟进

**信息同步**：
- 确保所有人信息透明
- 避免因信息不对称导致的问题

## 第四节：运用甘特图工具，制作项目管理表

### 什么是甘特图

甘特图是一种项目管理可视化工具：

- **首行**：时间（可按月/周/天定义）
- **首列**：To Do List（具体待办事项）
- **中间**：用色块标出时间段

**示例**：
- 项目一：蓝色色条，十月到十二月 = 十月启动，十二月前完成
- 项目二：灰色色条，十一月到一月 = 十一月启动，一月截止

### 甘特图的实际应用案例

**案例：活动项目管理（2015年）**

表格结构：
- 上方首行：时间（三月到四月）
- 标红日期：里程碑节点（3月27日、4月13日）
- 左侧：To Do List、负责人（Manager）、详情（Details）
- 中间色块：每项事项的启动和完成时间

**关键要点**：
- 每件事落实到人
- 启动和完成时间精确到天
- 明确标注里程碑节点

### 甘特图的价值

1. **信息传递清晰**：开会时能清楚传达哪件事在什么节点完成、负责人是谁、与其他工作的关联
2. **进度掌控直观**：哪项完成、哪项未完成、当前应该到什么程度，一目了然
3. **便于跟进**：拿着表就知道哪天该找谁要什么产出

### 工具选择建议

> **工具越简单越普适越好，Excel就能解决绝大部分项目管理问题。**

除非遇到体量特别大、特别复杂的项目（如涉及千万资金、几百人），否则不必追求复杂的项目管理工具。

# 课程总结

## 核心要点回顾

### 第一章：运营的底层逻辑

1. **运营的业务本质**：一切运营工作都服务于三个目的——获取用户、维系用户、获取收入
2. **运营与传统行业的差异**：用户场景更复杂、用户话语权更大、渠道玩法更动态
3. **产品生命周期运营**：探索期做口碑、成长期做增长、成熟期做活跃和变现、衰退期做防流失
4. **精益运营**：以最低成本验证不确定性假设，场景必须贴近真实

### 第二章：运营人的通用工作方法

1. **SOP的价值**：提升个人效率，建立团队工作标准，帮助新人快速上手
2. **SOP建立方法**：明确流程步骤、界定产出标准、撰写文档、执行落地
3. **项目管理**：梳理待办事项、制定可视化图表、设定里程碑、信息同步、风险控制
4. **甘特图应用**：直观展示时间线、任务分工、进度状态

## 行动建议

1. **建立运营全局观**：从业务目的出发理解运营工作，而非局限于具体岗位内容
2. **匹配产品生命周期**：根据产品所处阶段选择合适的运营策略，避免过早增长或错失增长窗口
3. **实践精益思维**：在大规模投入前用最低成本验证假设
4. **建立SOP习惯**：将重复性工作标准化，提升效率并赋能团队
5. **掌握项目管理**：用甘特图等工具管理多线程工作，确保项目按时交付

---

> 本课程核心价值：帮助运营人跳出执行看全局，建立一套可复用的思维框架和工作方法，无论面对什么类型的产品和岗位，都能快速上手并有效工作。
