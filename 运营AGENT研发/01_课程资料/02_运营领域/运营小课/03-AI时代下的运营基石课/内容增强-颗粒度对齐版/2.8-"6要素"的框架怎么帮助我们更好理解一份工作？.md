# 2.8 "6要素"的框架怎么帮助我们更好理解一份工作?

## 一、为什么需要用6要素框架理解工作?

### 1.1 理解工作的挑战

**常见困境**:
- 招聘网站的岗位描述模糊不清
- 不了解公司业务,不知道核心指标是什么
- 面试时抓不住重点,无法深入对话
- 不知道工作重点在哪里,容易做偏

**6要素框架的价值**:
```
6要素框架理解工作的价值

快速理解业务:
├─ 识别公司商业模式
├─ 定位工作在业务中的位置
├─ 推断关键业务指标
└─ 明确工作重点方向

梳理工作思路:
├─ 分析用户路径
├─ 画出数据漏斗
├─ 识别关键环节
└─ 形成执行方案

提升面试表现:
├─ 展示系统思维
├─ 体现专业能力
├─ 抓住对话重点
└─ 获得面试官认可
```

### 1.2 理解工作的完整流程

```
用6要素框架理解工作的流程

Step 1: 了解业务背景
├─ 公司是做什么的?
├─ 商业模式是什么?
├─ 核心业务是什么?
└─ 业务前后端是什么?

Step 2: 定位工作岗位
├─ 这份工作属于前端还是后端?
├─ 工作的核心目标是什么?
├─ 关键指标可能是什么?
└─ 工作场景和渠道有哪些?

Step 3: 梳理用户路径
├─ 用户从哪里来?
├─ 要经历哪些环节?
├─ 最终要达成什么目标?
└─ 是转化流程型还是用户成长型?

Step 4: 画出数据漏斗
├─ 每个环节的转化率是多少?
├─ 哪个环节是瓶颈?
├─ 需要优化哪些地方?
└─ 工作重点在哪里?

Step 5: 形成工作思路
├─ 核心要点是什么?
├─ 需要哪些关键手段?
├─ 如何拆解执行?
└─ 如何衡量效果?
```

## 二、案例一:少儿编程教育的用户增长专家

### 2.1 岗位描述

**公司**: 童程童美(少儿编程教育)
**岗位**: 用户增长专家
**职责描述**:
```
针对用户特点设计用户增长产品,
全面提升微信端的流量和用户增长效果,
挖掘获取新用户的渠道,
对引流渠道模式等进行创新
```

### 2.2 业务背景分析

**公司业务**:
- 少儿编程教育(面向C端用户)
- 卖编程课程
- 达内教育旗下品牌

**商业模式识别**:
```
商业模式: 产品自销模式(2C)
├─ 自有产品: 编程课程
├─ 自有渠道: 微信、社群、线下
├─ 直接销售: 课程顾问销售
└─ 收入来源: 课程学费

业务前后端:
├─ 前端业务: 获客、体验、转化、复购
└─ 后端业务: 课程研发、师资培训、教学服务
```

### 2.3 岗位定位分析

**工作定位**:
- 前端业务岗位
- 主要负责拉新获客
- 重点关注流量和用户增长

**关键指标推断**:
```
核心指标:
├─ 有效获客数量(线索量)
├─ 获客成本(CAC)
└─ 微信端用户增长数

辅助指标:
├─ 各渠道引流占比
├─ 引流转化率
├─ 用户留存率
└─ 最终转化率
```

**工作场景和渠道**:
- 主要场景: 微信生态
- 可能涉及: 多渠道引流创新
- 具体渠道: 公众号、视频号、社群、朋友圈广告

### 2.4 用户路径梳理

**完整用户路径**:

```
少儿编程教育用户路径

阶段1: 渠道曝光
├─ 搜索引擎: 家长搜索"少儿编程"
├─ 社交媒体: 抖音、小红书种草
├─ 线下渠道: 校园地推、商场活动
├─ 口碑传播: 老学员推荐
└─ 广告投放: 信息流广告

阶段2: 点击引流
├─ 引流物料: 文章、视频、海报、H5
├─ 引流内容: 免费试听课、编程体验、干货资料
└─ 引流目标: 引导添加微信

阶段3: 微信承接
├─ 关注公众号
├─ 添加企微/个人微信
├─ 加入微信群
├─ 参加直播
└─ 体验产品

阶段4: 社群培育
├─ 社群维护: 日常互动、内容分享
├─ 直播活动: 专家讲座、编程体验课
├─ 内容互动: 干货分享、案例分析
└─ 筛选意向: 识别高意向用户

阶段5: 付费转化
├─ 课程顾问跟进
├─ 试听课邀约
├─ 课程详细介绍
└─ 促成付费报名
```

**数据漏斗**:

```
数据漏斗分析

渠道曝光 → 点击引流 → 添加微信 → 进群 → 参与活动 → 高意向 → 付费转化
  100万      5%        80%       60%     40%       20%        10%
  (5万)     (4万)     (2.4万)  (9600)   (1920)    (192)

核心数据:
├─ 点击率: 5% (有优化空间)
├─ 添加率: 80% (表现良好)
├─ 进群率: 60% (可优化)
├─ 活动参与率: 40% (有优化空间)
├─ 意向转化率: 20% (可优化)
├─ 最终转化率: 10% (需要重点优化)

转化率: 0.192%
获客成本: 假设广告投入10万,获得192个付费用户,CAC=521元/人
```

### 2.5 工作思路梳理

**核心工作要点**:

```
用户增长专家的三个核心要点

要点1: 找到更有效的渠道
├─ 测试不同渠道效果
├─ 分析各渠道ROI
├─ 优化渠道投放策略
└─ 持续拓展新渠道

要点2: 设计更好的引流物料
├─ 优化文案: 直击家长痛点
├─ 优化设计: 提升点击率
├─ A/B测试: 找到最优方案
└─ 建立素材库: 持续迭代

要点3: 做好社群承接和培育
├─ 社群维护: 保持活跃度
├─ 活动策划: 提升参与率
├─ 内容运营: 提供价值
└─ 筛选意向: 提升转化率
```

**关键工作手段**:

| 工作要点 | 具体手段 | 预期效果 |
|---------|---------|---------|
| **渠道优化** | 1. 小范围测试各渠道<br>2. 分析各渠道ROI<br>3. 加大高ROI渠道投入<br>4. 拓展新渠道测试 | 提升点击率30% |
| **物料优化** | 1. 准备10版标题和文案<br>2. 设计3版引流海报<br>3. A/B测试选择最优<br>4. 建立素材库持续迭代 | 提升点击率50% |
| **社群承接** | 1. 设计新人欢迎流程<br>2. 规划日常互动内容<br>3. 策划每周直播活动<br>4. 建立意向筛选机制 | 提升转化率100% |

### 2.6 面试对话示例

**面试官**: "你觉得这个岗位的核心工作是什么?"

**优秀回答**:
```
基于我对少儿编程教育业务的理解,我认为这个岗位的核心是:

首先,明确业务指标:
├─ 核心指标: 有效获客数量和获客成本
├─ 目标: 在保证质量的前提下降低CAC

其次,梳理用户路径:
├─ 曝光 → 点击 → 添加微信 → 社群培育 → 付费转化
├─ 当前最大瓶颈在点击环节(仅5%)

然后,确定工作重点:
├─ 短期: 优化引流物料,提升点击率
├─ 中期: 优化渠道组合,降低获客成本
├─ 长期: 搭建增长体系,实现规模化获客

最后,具体执行:
├─ 测试各渠道效果,优化投放
├─ A/B测试引流物料,找到最优方案
├─ 优化社群运营,提升承接和转化效率
└─ 建立数据监控,持续迭代优化
```

## 三、案例二:美团闪购的用户增长运营

### 3.1 岗位描述

**公司**: 美团
**业务**: 闪购(即时零售)
**岗位**: 用户增长运营专家
**职责描述**:
```
负责美团闪购的新客增长,
包含流量引入、转化流程等环节,
对新客数量、留存等指标负责,
以供给撬动增长的新客拉品模式,
针对不同渠道制定匹配的选品策略
```

### 3.2 业务背景分析

**美团闪购是什么**:
- 结合美团配送能力和运力
- 引入水果、母婴、玩具等连锁商家
- 平台提供流量和资源推广
- 收取交易抽佣费用
- 专门匹配闪购订单的运力,保障送达时间

**商业模式识别**:
```
商业模式: 平台+抽佣模式
├─ B端: 连锁商家入驻
├─ C端: 用户下单购买
├─ 平台: 撮合交易,抽取佣金
└─ 收入来源: 交易抽佣(通常5-20%)

业务前后端:
├─ 前端业务: 获客、首单转化、留存复购
├─ 后端业务: 商家拓展、供应链管理、配送服务
└─ 平台职责: 流量分发、交易撮合、服务保障
```

### 3.3 岗位定位分析

**工作定位**:
- 前端业务岗位
- 对新客增长和中长期用户数负责

**关键指标推断**:
```
核心指标:
├─ 新客增长数
├─ 首单转化率
├─ 中长期用户数(留存率)

辅助指标:
├─ 各渠道获客占比
├─ 选品转化率
├─ 用户复购率
└─ 用户LTV
```

**工作场景和渠道**:
- 主要场景: APP内、短信、社群等
- 核心策略: 以供给(商品)撬动增长
- 关键动作: 针对不同渠道制定选品策略

### 3.4 用户路径梳理

**完整用户路径**(双路径并存):

```
美团闪购用户路径

路径A: 转化流程型路径(新客转化)

阶段1: 多渠道曝光
├─ APP内推荐位
├─ 短信推送
├─ 社群分享
├─ 广告投放
└─ 其他引流方式

阶段2: 选品曝光
├─ 根据渠道选品
├─ 根据用户属性选品
├─ 不同渠道不同商品
└─ A/B测试商品组合

阶段3: 引导首单
├─ 新人优惠
├─ 限时特价
├─ 满减活动
└─ 免费配送

阶段4: 完成首单
├─ 快速下单流程
├─ 多种支付方式
├─ 准时配送保障
└─ 完成首单奖励

路径B: 用户成长型路径(留存复购)

阶段1: 用户分层
├─ 新完成首单用户
├─ 低频用户
├─ 中频用户
└─ 高频用户

阶段2: 差异化运营
├─ 折扣券发放(针对不同用户)
├─ 定期活动策划
├─ 精准推荐匹配
└─ 搜索优化

阶段3: 促进复购
├─ 同品类复购引导
├─ 跨品类推荐
├─ 会员体系运营
└─ LTV提升

阶段4: 用户建模
├─ 小用户(月消费<100元)
├─ 中用户(月消费100-500元)
├─ 大用户(月消费>500元)
└─ 针对性提升策略
```

**数据漏斗**:

```
新客转化漏斗(转化流程型)

渠道曝光 → 点击 → 选品曝光 → 引导首单 → 完成首单
  100万    3%    60%        50%        70%
  (3万)   (1.8万)  (9000)    (6300)

核心数据:
├─ 点击率: 3% (有优化空间)
├─ 选品曝光率: 60% (需优化)
├─ 首单引导转化率: 50% (有优化空间)
├─ 首单完成率: 70% (表现良好)
└─ 整体转化率: 0.63%

完成首单 → 二次消费 → 成为中用户 → LTV提升

  (2520)   (756)
```

### 3.5 工作思路梳理

**核心工作要点**:

```
美团闪购用户增长的三个核心要点

要点1: 渠道×选品匹配
├─ 分析不同渠道用户属性
├─ 针对不同渠道选择不同商品
├─ A/B测试商品组合效果
└─ 持续优化选品策略

要点2: 首单转化优化
├─ 优化新人优惠力度
├─ 设计引导首单流程
├─ 优化下单体验
└─ 提升首单完成率

要点3: 用户留存和LTV提升

├─ 差异化券策略
├─ 精准推荐匹配
└─ 提升复购率
```

**关键工作手段**:

| 工作要点 | 具体手段 | 预期效果 |
|---------|---------|---------|
| **渠道选品** | 1. 分析各渠道用户画像<br>2. 建立商品池<br>3. 测试不同商品组合<br>4. 优化选品策略 | 提升选品转化率30% |
| **首单优化** | 1. 优化新人礼包<br>2. 简化下单流程<br>3. 优化支付体验<br>4. 强化首单奖励 | 提升首单转化率40% |

### 3.6 复杂度分析

**为什么这个岗位复杂度更高**:

```
复杂度来源1: 双路径并存
├─ 转化流程型路径: 新客转化
├─ 用户成长型路径: 留存复购
└─ 需要同时优化两条路径

复杂度来源2: 多变量优化
├─ 渠道变量: 不同渠道不同策略

├─ 用户变量: 不同用户不同偏好
└─ 需要精细化运营

复杂度来源3: 供需匹配
├─ 供给端: 商家资源和商品库存
├─ 需求端: 用户需求和购买意愿
├─ 平台端: 流量和资源分配
└─ 需要动态平衡

能力要求:
├─ 数据分析能力: 分析多维度数据
├─ 策略设计能力: 设计复杂运营策略
├─ 系统思维: 从系统层面思考
└─ 执行力: 落地执行并优化
```

## 四、案例三:运动品牌的新媒体运营主管

### 4.1 岗位描述

**公司**: 某新材料休闲运动服品牌(类似Lululemon)
**岗位**: 新媒体运营主管
**职责描述**:
```
负责新媒体账号运营,
做内容运营,
分析私域流量,
处理用户反馈,
做用户分层画像,
设计用户成长激励机制
```

### 4.2 业务背景分析

**公司业务**: 新材料休闲运动服品牌
**商业模式**: 产品自销模式(2C电商)

**商业模式识别**:
```
商业模式: 产品自销模式(2C)
├─ 自有产品: 运动服装
├─ 自有渠道: 天猫、京东、抖音、私域
├─ 直接销售: 用户直接下单购买
└─ 收入来源: 商品销售收入

业务前后端:
├─ 前端业务: 内容种草、引流转化、用户运营
└─ 后端业务: 产品设计、供应链、客户服务
```

### 4.3 岗位定位分析

**工作定位**: 前端业务岗位

**关键指标的两种可能性**:

```
可能性1: 对收入负责
├─ 核心指标: GMV、客单价、复购率
├─ 工作目标: 通过新媒体卖货
└─ KPI导向: 销售额

可能性2: 对涨粉和活跃负责
├─ 核心指标: 粉丝数、活跃用户数
├─ 工作目标: 做品牌影响力
└─ KPI导向: 粉丝增长和活跃
```

**工作场景和渠道**:
- 可能渠道: 微信公众号、抖音、小红书等
- 具体平台: 岗位描述未明确,需要面试时确认

### 4.6 面试对话示例

**面试官**: "你觉得这个岗位应该对什么指标负责?"

**优秀回答**:
```
基于我对运动服装品牌业务的理解,我认为这个岗位有两种可能:

可能性1: 对收入负责
├─ 核心指标: GMV、客单价、复购率
├─ 工作重点: 内容种草+成交转化+会员运营

可能性2: 对涨粉和活跃负责
├─ 核心指标: 粉丝数、活跃用户数

我个人的理解是:
```

### 5.2 方法论总结

**理解任何工作的通用方法**:

```
用6要素框架理解任何工作

Step 1: 了解业务背景
├─ 公司是做什么的?
├─ 商业模式是什么?
├─ 核心业务是什么?

Step 2: 定位工作岗位
├─ 前端还是后端?
├─ 核心目标是什么?
├─ 关键指标可能是什么?

Step 3: 梳理用户路径
├─ 转化流程型路径
├─ 用户成长型路径
├─ 还是双路径并存?

Step 4: 画出数据漏斗
├─ 各环节转化率
├─ 哪里是瓶颈?
├─ 优化重点在哪里?

Step 5: 形成工作思路
├─ 核心要点是什么?
├─ 需要哪些手段?
├─ 如何执行和优化?
```
