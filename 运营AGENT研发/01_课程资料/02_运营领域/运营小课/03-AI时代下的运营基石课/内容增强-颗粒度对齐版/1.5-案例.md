# 1.5 案例:AI生产力公式的实战应用

## 一、案例体系概览

### 1.1 案例设计思路

本章通过两个完整案例,展示AI生产力公式在实际工作中的应用:
- **案例一**:文案写作(内容生成场景)
- **案例二**:商业决策分析(决策辅助场景)

**两个案例的对比**:

| 维度 | 案例一:文案写作 | 案例二:商业决策 |
|------|----------------|----------------|
| **场景类型** | 内容生成 | 决策分析 |
| **复杂度** | 中等(明确问题定义) | 高(多变量分析) |
| **核心要素** | 问题定义+关键参数+语料库 | 问题定义+关键参数+数据模型 |
| **AI角色** | 创作执行者 | 分析辅助者 |
| **你的角色** | 需求定义者+质量判断者 | 问题定义者+决策者 |
| **交互轮次** | 3-5轮 | 5-8轮 |
| **产出形式** | 多版本内容供选择 | 结构化分析报告 |

### 1.2 AI生产力公式回顾

**核心公式**:
```
AI生产力 = (模型能力 + 语料库) × (问题定义 + 参数调试)
```

**在两个案例中的体现**:
- 模型能力:ChatGPT的生成和分析能力(相对固定)
- 语料库:优质案例、参考素材、行业知识(可优化)
- 问题定义:明确的需求、背景、目标(关键变量)
- 参数调试:关键参数的识别和调优(关键变量)

**关键启示**:
模型能力相对固定,真正的变量在于你如何定义问题、识别关键参数、准备优质语料库。

## 二、案例一:文案写作中的AI应用

### 2.1 案例背景与目标

**业务背景**:
```
公司:某知名茶饮品牌
产品:全新"轻爽果茶"系列
阶段:新品上市前的预热阶段
目标:在新媒体渠道引发广泛传播和关注
```

**核心挑战**:
- 市场上同类产品众多,如何突出差异化?
- 目标用户是年轻人,如何引发情感共鸣?
- 预热阶段不直接转化,如何设计传播性内容?

**AI应用目标**:
- 快速产出多版本文案供选择
- 确保文案质量和传播性
- 优化文案迭代效率

### 2.2 三大要素的准备

**要素一:问题定义**

**问题定义的5W2H框架**:

| 维度 | 定义 | 具体内容 |
|------|------|---------|
| What(什么) | 任务类型 | 新品预热文案创作 |
| Why(为什么) | 核心目标 | 引发传播和关注,建立产品认知 |
| Who(谁) | 目标用户 | 18-35岁都市白领,注重健康但不想牺牲口感 |
| Where(哪里) | 发布渠道 | 微信公众号、小红书、抖音、微博 |
| When(何时) | 时间节点 | 新品上市前2周开始预热 |
| How(怎么做) | 文案风格 | 年轻、轻松、略带自嘲幽默 |
| How much(多少) | 产出数量 | 10个版本(5个幽默风格,5个走心风格) |

**问题定义的核心要点**:
1. **明确场景**:是预热文案,不是转化文案,重点在传播性
2. **明确用户**:都市白领,打工人的身份认同是关键
3. **明确目标**:引发传播,不是直接卖货,要设计传播点

**要素二:关键参数识别**

**文案创作的关键参数体系**:

**参数类别一:内容参数**
| 参数 | 可选值 | 选择依据 |
|------|--------|---------|
| 视角 | 第一人称/第三人称 | 第一人称更亲切,第三人称更客观 |
| 场景 | 地铁/办公室/家里/户外 | 办公室下午3点是核心场景 |
| 卖点 | 0卡0糖/真果汁/清爽口感 | 0卡0糖是主打卖点 |
| 情绪 | 幽默/走心/励志/自嘲 | 自嘲幽默+走心温暖结合 |

**参数类别二:形式参数**
| 参数 | 具体要求 |
|------|---------|
| 结构 | 主标题(15字内)+副标题(40字内) |
| 数量 | 10个版本,幽默/走心各5个 |
| 字数 | 总字数控制在55字内 |
| 格式 | 适合社交媒体传播,考虑换行和emoji |

**参数类别三:创意参数**
| 参数 | 具体要求 |
|------|---------|
| 话题结合 | 结合打工人话题(996、内卷、躺平) |
| 热梗使用 | 适度使用网络热梗,不要太尬 |
| 互动设计 | 设计讨论点和槽点,引发评论 |
| 金句设计 | 设计可传播的金句和记忆点 |

**要素三:优质语料库准备**

**语料库的三个层次**:

**层次一:优秀文案案例库**
```
品牌案例:
- 喜茶:"一杯喜茶,激发一份灵感"
- 奈雪:"美好日常,从一杯茶开始"
- 茶颜悦色:"茶颜悦色,悦色茶颜"
- 瑞幸:"这一杯,谁不爱"

风格参考:
- 李欣频:诗意化、哲理性、情感共鸣
- 小马宋:幽默自嘲、生活化、接地气
- 杜蕾斯:双关语、隐喻、大胆创意
```

**层次二:目标用户洞察库**
```
打工人话题:
- "打工人的快乐,是下午3点的一杯奶茶"
- "成年人的世界里,奶茶是唯一的甜"
- "生活已经很苦了,喝杯甜的怎么了"
- "不想内卷,只想躺平喝奶茶"

健康焦虑:
- "想喝甜的又怕胖,是我没错了"
- "0卡0糖的快乐,打工人懂"
- "成年人的自律,是喝奶茶也要选0卡的"
```

**层次三:产品卖点提炼库**
```
核心卖点:
- 0卡0糖:无负担,健康选择
- 真果汁≥30%:真材实料,不是香精勾兑
- 清爽不腻:夏天解渴,下午茶不腻
- 新鲜果肉:看得见的真材实料

差异化优势:
- 相比奶茶:更清爽,不厚重
- 相比果茶:真果汁,不是浓缩汁
- 相比饮料:0卡0糖,更健康
```

### 2.3 AI辅助文案创作的实战流程

**阶段一:需求澄清和确认**

AI的回复框架:

4. 竞品情况和差异化优势

```

**第二轮对话**:
```

```

```

### 2.4 案例复盘和经验总结

**成功要素**:

**要素一:问题定义清晰**
```
✓ 明确了是预热文案,不是转化文案
✓ 明确了目标是传播,不是直接卖货
✓ 明确了用户是打工人,要引发身份认同
→ 问题定义清晰,让AI的输出更精准
```

**要素二:关键参数完整**
```
✓ 内容参数:视角、场景、卖点、情绪都明确了
✓ 形式参数:结构、数量、字数、格式都规定了
✓ 创意参数:话题、热梗、互动、金句都考虑了
→ 参数完整,让AI的创作有明确边界
```

**要素三:优质语料库充分**
```
✓ 提供了品牌案例、风格参考、竞品分析
✓ 提供了用户洞察、话题素材、产品卖点
✓ 提供了避坑指南、网络热梗、创意方向
→ 语料库充分,让AI有丰富的创作素材
```

**要素四:分布式对话**
```
✓ 第一轮:需求澄清,让AI帮我梳理框架
✓ 第二轮:信息输入,系统提供所有背景
✓ 第三轮:初稿生成,多版本供选择
✓ 第四轮:迭代优化,针对性改进
→ 分布式对话,逐步逼近最优解
```

**经验总结**:

**经验一:不要一次性给AI所有信息**
```
错误做法:
"帮我写10条茶饮文案,要求XXX、XXX、XXX……(一大堆要求)"
→ AI信息过载,输出质量不稳定

正确做法:
先问AI需要什么信息,再系统提供,分步迭代
→ AI更好理解需求,输出质量更高
```

**经验二:AI输出不满意时,要明确优化方向**
```
模糊反馈:
"这几条不太好,再优化一下"
→ AI不知道从哪里优化

明确反馈:
"第3条有潜力,但我希望:1.增强幽默感,2.优化金句,3.结合XX话题"
→ AI知道优化方向,输出更精准
```

**经验三:建立个人的优质语料库**
```
短期:每次都临时找参考案例
→ 效率低,质量不稳定

长期:建立分类清晰的语料库
→ 效率高,质量稳定,形成个人竞争力
```

## 三、案例二:商业决策中的AI应用

### 3.1 案例背景与目标

**业务背景**:
```
公司类型:某垂直型素质教育培训公司
决策问题:是否推出小班课模式新产品?
时间要求:需要在一周内给出可行性分析报告
决策层级:CEO需要基于此分析做最终决策
```

**核心挑战**:
- 小班课模式成本结构复杂,盈亏平衡点难以确定
- 市场竞争激烈,需要评估盈利可行性
- 需要量化分析,不能凭感觉决策

**AI应用目标**:
- 快速建立盈亏模型
- 进行多场景敏感性分析
- 提供量化的决策依据

### 3.2 三大要素的准备

**要素一:问题定义**

**决策问题的框架化**:

```
核心问题:是否推出小班课产品?
├─ 决策目标:评估1年内盈亏平衡的可行性
├─ 时间要求:1周内完成分析
├─ 制约因素:
│  ├─ 资金:初始投入≤100万
│  ├─ 团队:规模≤10人
│  └─ 时间:必须在Q3启动
├─ 评估标准:
│  ├─ 盈亏平衡时间≤12个月
│  ├─ 预期毛利率≥40%
│  └─ 现金流健康(不出现资金断裂)
└─ 决策权限:我负责分析,CEO最终决策
```

**要素二:关键参数识别**

**盈亏模型的关键参数体系**:

**参数类别一:收入参数**
| 参数 | 设定值 | 参数影响 | 优化方向 |
|------|--------|---------|---------|
| 课程定价 | 3699元/人 | 定价↑→收入↑,但需求可能↓ | 需测试价格敏感度 |
| 班容上限 | 8人/班 | 班容↑→单班收入↑,但体验可能↓ | 需平衡体验和效率 |
| 满班率 | 目标75% | 满班率↑→实际收入↑ | 重点优化运营能力 |
| 退款率 | 预估3% | 退款率↑→实际收入↓ | 需要质量保障 |
| 续费率 | 目标60% | 续费率↑→LTV↑ | 长期盈利关键 |

**参数类别二:成本参数**
| 参数 | 设定值 | 参数影响 | 优化方向 |

**盈亏模型公式**:
```
单班收入 = 课程定价 × 班容 × 满班率 × (1 - 退款率)
单班成本 = (讲师薪资 + 班主任薪资) × 课程周期
单班毛利 = 单班收入 - 单班成本

盈亏平衡班数 = 固定投入 / (单班收入 - 获客成本 × 班容 × 满班率)
盈亏平衡时间 = 盈亏平衡班数 / 月均开班能力
```
