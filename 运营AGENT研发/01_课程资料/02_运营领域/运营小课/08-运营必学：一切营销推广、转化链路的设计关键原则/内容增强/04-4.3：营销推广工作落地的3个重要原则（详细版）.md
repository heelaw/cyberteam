## 本节导学（核心目标）

本节将深入讲解营销推广工作落地时需要遵循的3个重要原则，帮助学员设计更高效的转化链路。

## 原则1：锚定产品设计

### 1.1 什么是锚定产品

```
┌─────────────────────────────────────────────────────────────────────┐
│                       什么是锚定产品？                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   锚定产品定义：                                                    │
│   在转化链路中，通过设置特定价格/定位的产品，                        │
│   引导用户选择目标产品的设计策略                                     │
│                                                                     │
│   核心目的：                                                        │
│   • 提升转化率                                                     │
│   • 提升客单价（UP值）                                             │
│   • 降低用户决策难度                                               │
│                                                                     │
│   常见形式：                                                        │
│   • 托品：专门设置的低价/高价对比产品                               │
│   • 套餐：组合销售                                                 │
│   • 阶梯价：通过对比引导选择                                        │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

#### 案例4：笔记本电脑支架对比

**案例背景**：同一品牌两款笔记本电脑支架

| 产品 | 价格 | 库存 | 差异 |
|------|------|------|------|
| 普通款 | 55元 | 无货 | - |
| 豪华款 | 59元 | 有货 | 名称差异 |

**用户选择**：99%用户选择豪华款

**原因分析**：
1. **对比效应**：普通款存在让豪华款显得"值得"
2. **价格锚定**：只贵4元，决策成本低
3. **便利性**：普通款无货，豪华款有货

**启示**：托品不一定要卖出，其存在本身就是价值

**产品方案**：

## 原则2：唯一最短可信

### 2.1 什么是"唯一最短可信"

```
┌─────────────────────────────────────────────────────────────────────┐
│                     唯一最短可信原则                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   唯一：在同一页面/步骤，给用户少的选择                            │
│                                                                     │
│   最短：转化路径越短越好（每增加1步，+20%流失）                    │
│                                                                     │
│   可信：所有信息要让用户信任                                       │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 2.2 "唯一"原则

```
┌─────────────────────────────────────────────────────────────────────┐
│                         "唯一"原则                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   问题：给用户太多选择会怎样？                                      │
│                                                                     │
│   ┌─────────────────────────────────────────────────────────────┐   │
│   │  过多的选择 = 更高的决策难度                                 │   │
│   │        ↓                                                    │   │
│   │  用户思考时间增加                                           │   │
│   │        ↓                                                    │   │
│   │  用户可能放弃                                               │   │
│   └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│   正确做法：                                                        │
│   • 每个页面/步骤，只给1-2个核心选择                                │
│   • 聚焦引导用户走一条路                                          │
│   • 其他选项可以弱化或隐藏                                         │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 2.3 "最短"原则

```
┌─────────────────────────────────────────────────────────────────────┐
│                         "最短"原则                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   核心观点：                                                       │
│   每增加一个转化环节，至少增加20%流失                               │
│                                                                     │
│   最佳实践：                                                       │
│   转化路径最好不超过5步                                            │
│                                                                     │
│   ┌─────────────────────────────────────────────────────────────┐   │
│   │ 3步：曝光 → 点击 → 购买（理想状态）                         │   │
│   │ 5步：曝光 → 点击 → 详情 → 加购 → 购买（合理）              │   │
│   │ 7步+：曝光 → 详情1 → 详情2 → 详情3 → ... → 购买（流失大） │   │
│   └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 2.4 "可信"原则

```
┌─────────────────────────────────────────────────────────────────────┐
│                         "可信"原则                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   核心观点：                                                       │
│   页面和路径中呈现的信息要有助于增强用户信任                        │
│                                                                     │
│   避免事项：                                                       │
│   • 山寨感的设计                                                   │
│   • 不专业的表达                                                   │
│   • 过于夸张的宣传                                                 │
│   • 让用户产生疑惑的信息                                           │
│                                                                     │
│   正确做法：                                                       │
│   • 专业的页面设计                                                 │
│   • 真实的案例和数据                                               │
│   • 明确的行动指引                                                 │
│   • 消除用户顾虑                                                   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 原则3：根据场景和决策成本设计

### 3.1 四象限矩阵

```
┌─────────────────────────────────────────────────────────────────────┐
│                    用户场景与决策成本矩阵                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│              用户接收信息时间                                                      │
│                    短                    长                          │
│          ┌─────────────────┬─────────────────┐                     │
│   高     │  场景1          │   场景3         │                     │
│          │  短时间+        │   长时间+       │                     │
│ 消费     │  冲动消费       │   理性消费      │                     │
│ 决策     │                 │                 │                     │
│ 成本     │ 策略：表单收集  │   策略：深度    │                     │
│          │ +后续跟进       │   内容+信任     │                     │
│          ├─────────────────┼─────────────────┤                     │
│   低     │  场景2          │   场景4         │                     │
│          │  短时间+       │   长时间+       │                     │
│          │  冲动消费      │   理性消费      │                     │
│          │                 │                 │                     │
│          │ 策略：强优惠+  │   策略：详情    │                     │
│          │   快速转化     │   页+对比展示   │                     │
│          └─────────────────┴─────────────────┘                     │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 3.2 场景1：短时间+冲动消费

**典型场景**：
- 移动端公众号下方广点通广告
- 直播间
- 电梯广告

**策略要点**：
```
1张图/1句话 + 强优惠 = 快速转化

核心：
• 5秒内抓住注意力
• 直接给福利
• 缩短决策时间
```

---

### 3.3 场景2：长时间+冲动消费

**典型场景**：
- PC端淘宝首页广告进入店铺
- 电商详情页

**策略要点**：
```
详情页 + 不断强化 = 引导转化

核心：
• 详细信息展示
• 不断给出转化诱因
• 转化流程顺畅
```

---

### 3.4 场景3：短时间+理性消费

**典型场景**：
- 移动端信息流广告推高客单价服务
- 企业服务类产品

**策略要点**：
```
轻诱因 + 留资料 = 后续跟进

核心：
• 给一个较轻的诱因（资料包等）
• 让用户留下联系方式
• 后续通过其他方式转化
```

---

### 3.5 场景4：长时间+理性消费

**典型场景**：
- PC端SEM推高价课程
- 知乎长文推广咨询

**策略要点**：
```
深度内容 + 信任建立 = 逐步转化

核心：
• 大量案例展示
• 专业内容输出
• 顾问式销售
• 体验式销售
```

## 5. 核心概念表

| 概念 | 定义 | 应用要点 |
|------|------|----------|
| 锚定产品 | 用于对比引导用户选择的参照产品 | 托品不一定要卖出 |
| 唯一 | 给用户少的选择 | 聚焦引导，降低决策难度 |
| 最短 | 转化路径短 | 每增加1步，+20%流失 |
| 可信 | 增强用户信任 | 专业设计，消除顾虑 |
| 决策成本 | 用户做决策的难易程度 | 影响转化策略选择 |
| 单场景转化 | 在同一场景内完成转化 | 短平快，强引导 |
| 跨场景转化 | 跳转多个场景完成转化 | 留资料，后跟进 |

## 6. 本节小结

### 核心要点

1. **原则1：锚定产品设计**
   - 通过对比引导用户选择目标产品
   - 托品不一定要卖出
   - 套餐、满减、包邮都是锚定策略

2. **原则2：唯一最短可信**
   - 唯一：给用户少的选择
   - 最短：路径不超过5步
   - 可信：增强用户信任

3. **原则3：根据场景设计**
   - 识别用户场景和决策成本
   - 选择对应的转化策略
   - 跨场景转化需要更强的诱因

## 思考题

1. 你现在的产品是否设计了锚定产品？效果如何？
2. 你的转化路径是否遵循"唯一最短可信"原则？
3. 你的用户属于哪种场景类型？策略是否匹配？
