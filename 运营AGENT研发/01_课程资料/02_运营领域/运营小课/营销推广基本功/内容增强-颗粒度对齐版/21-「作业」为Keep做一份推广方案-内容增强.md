#### 1.2 今日头条推广逻辑

**【流量分发机制】**

今日头条基于算法将内容分发给用户：

```
用户标签 + 内容标签 + 出价 + 质量度 → 推荐分发
```

**用户标签：**
平台收集用户数据，给用户打上标签：
- 基础属性：年龄、性别、地域
- 兴趣标签：财经、科技、娱乐等
- 行为数据：阅读、点击、转化历史

**内容标签：**
平台对广告内容打上标签：
- 行业标签：金融、借贷
- 创意标签：优惠、快速、便捷
- 落地页标签：借款、申请

**匹配逻辑：**
算法将用户标签与内容标签匹配，决定是否展示：
```
展示概率 = 匹配度 × 出价 × 质量度
```

**【曝光量影响因素】**

| 因素 | 权重 | 优化方法 |
|------|------|---------|
| **出价** | 高 | 根据CPA目标设置OCPM出价 |
| **质量度** | 高 | 提升CTR、CVR |
| **定向精准度** | 中 | 精准定向目标用户 |
| **竞争环境** | 中 | 避开竞争激烈时段 |

#### 1.3 核心概念解释

**概念1：DSP（Demand Side Platform，需求方平台）**

**定义：**
广告主在统一的平台上管理多个广告交易平台和广告网络的投放。

**作用：**
- 一站式管理多渠道投放
- 实时竞价（RTB）
- 数据驱动优化

**应用：**
今日头条广告平台就是一个DSP，广告主可以在上面管理广告投放。

---

**概念2：CPT（Cost Per Time，按时间付费）**

**定义：**
按广告投放时间长短计费。

**计算：**
总费用 = 单位时间价格 × 投放时长

**示例：**
某个广告位的价格是10万元/天，投放7天，总费用是70万元。

**适用场景：**
- 品牌曝光
- 固定位置广告
- 大型活动推广

---

**概念3：OCPM（Optimized Cost Per Mille，优化千次展示成本）**

**定义：**
基于转化的优化出价方式，平台自动优化千次展示成本，以获得更多转化。

**优势：**
- 相比CPM，转化率更高
- 相比CPC，成本更低
- 自动优化，节省人工

**设置方法：**
```
OCPM出价 = 可接受CPA × 预期转化率 / 1000
```

**示例：**
- 可接受CPA：100元
- 预期转化率：5%
- OCPM出价 = 100 × 5% / 1000 = 5元

---

**概念4：ROI（Return on Investment，投资回报率）**

**定义：**
广告投放带来的收益与投入的比例。

**计算公式：**
```
ROI = 收益 / 成本
```

**示例：**
- 广告花费：10万元
- 带来收益：30万元
- ROI = 30 / 10 = 1:3

**目标：**
- ROI > 1:1 说明不亏本
- ROI > 1:3 说明效果良好
- ROI > 1:5 说明效果优秀

---

**概念5：DAU（Daily Active Users，日活跃用户）**

**定义：**
每日活跃用户数量。

**计算方法：**
- 统计每日打开APP的用户数
- 去重计算（同一用户只算一次）

**应用：**
- 衡量产品用户规模
- 评估推广效果
- 制定投放策略

**示例：**
- 今日头条DAU：3亿
- 某APP DAU：100万

---

**概念6：LTV（Life Time Value，用户生命周期价值）**

**定义：**
单个用户在整个生命周期内为产品带来的总收益。

**计算公式：**
```
LTV = ARPU × 用户生命周期
```

其中：
- ARPU（Average Revenue Per User）：平均每用户收入
- 用户生命周期：用户平均使用时长

**示例：**
- ARPU：50元/月
- 用户生命周期：12个月
- LTV = 50 × 12 = 600元

**应用：**
- 评估CPA是否合理
- CPA < LTV 说明投放有效
- CPA越小于LTV，说明效果越好

---

**概念7：CPA（Cost Per Action，按行为付费）**

**定义：**
按用户完成特定行为（如注册、下载、购买）计费。

**计算公式：**
```
CPA = 总花费 / 转化量
```

**示例：**
- 广告花费：10,000元
- 获得注册用户：100人
- CPA = 10,000 / 100 = 100元/人

**优化目标：**
- CPA越低越好
- 但要保证用户质量
- 不能为了降低CPA而降低定向精准度

---

**概念8：CTR（Click Through Rate，点击率）**

**定义：**
广告被点击的比例。

**计算公式：**
```
CTR = 点击量 / 曝光量 × 100%
```

**示例：**
- 曝光量：100,000
- 点击量：2,000
- CTR = 2,000 / 100,000 × 100% = 2%

**行业标准：**
- 信息流广告：1-5%
- Banner广告：0.5-2%
- 搜索广告：5-15%

**优化方向：**
- 优化文案
- 优化素材
- 优化定向

---

**概念9：CVR（Conversion Rate，转化率）**

**定义：**
用户完成转化的比例。

**计算公式：**
```
CVR = 转化量 / 点击量 × 100%
```

**示例：**
- 点击量：2,000
- 转化量（注册）：100
- CVR = 100 / 2,000 × 100% = 5%

**行业标准：**
- 金融借贷：3-8%
- 电商零售：10-25%
- 教育培训：5-15%

**优化方向：**
- 优化落地页
- 简化注册流程
- 设计转化诱因

#### 2.1 用户注册转化路径

**【完整转化路径】**

```

    ↓ （勾选同意协议）
完成注册
    ↓ （领取新人福利）

```

**【路径说明】**

**步骤1：信息流广告**
- 形式：推荐频道信息流大图广告
- 规格要求：690×286像素
- 目标：吸引用户点击

**步骤2：落地页**
- 采用3W1A模型设计
- 目标：说服用户注册
- 关键元素：痛点、解决方案、信任背书、行动召唤

**步骤3：填写手机号**
- 输入框：11位手机号
- 验证：实时验证手机号格式
- 目的：获取用户联系方式

**步骤4：获取验证码**
- 发送验证码到用户手机
- 60秒倒计时
- 目的：验证手机号真实性

**步骤5：输入验证码**
- 输入框：6位数字验证码
- 验证：自动验证
- 目的：完成手机号验证

**步骤6：设置密码**
- 输入框：6-16位密码
- 要求：字母+数字
- 目的：设置登录密码

**步骤7：勾选同意协议**
- 用户协议和隐私政策
- 必须勾选才能继续
- 目的：法律合规

**步骤8：完成注册**
- 显示注册成功页面
- 自动登录
- 引导完成首次借款

**【漏斗数据预估】**

| 步骤 | 流失率 | 通过率 |
|------|--------|--------|
| 信息流→落地页 | - | 100%（点击即到达） |
| 落地页→填手机号 | 20% | 80% |
| 填手机号→获取验证码 | 5% | 95% |
| 获取验证码→输入验证码 | 10% | 90% |
| 输入验证码→设置密码 | 5% | 95% |

**目标：** 优化创意和定向，降低CPA

- 点击量：6,000（CTR=3%）
- 注册量：300（CVR=5%）
- CPA：83元

**【第4周：放量期】**

**目标：** 加大预算，最大化获客

**策略：**
- 使用最佳创意组合
- 日预算提升至5万元
- 24小时投放

**预期数据：**
- 曝光量：60万
- 点击量：18,000（CTR=3%）
- 注册量：900（CVR=5%）
- CPA：83元

| 周次 | 预算（万元） | 注册量 | CPA（元） |

#### 3.5 数据监控

**【核心监控指标】**

| 指标 | 目标值 | 预警值 | 优化动作 |
|------|--------|--------|---------|
| 日曝光量 | ≥5万 | <3万 | 提高出价或放宽定向 |
| CTR | ≥2.5% | <2% | 更换创意 |
| CVR | ≥5% | <4% | 优化落地页 |
| CPA | ≤150元 | >200元 | 综合优化 |
| ROI | ≥1:3 | <1:2 | 重新评估策略 |

**【监控频率】**
- 实时监控：曝光量、花费
- 每日监控：CTR、CVR、CPA
- 每周监控：ROI、总转化量
- 每月监控：整体效果评估

#### 3.6 风险控制

**【风险1：CPA过高】**

**应对：**
- 降低出价
- 优化定向
- 更换创意

**【风险2：转化量不足】**

**应对：**
- 提高预算
- 放宽定向
- 增加创意

**【风险3：素材被拒】**

**应对：**
- 准备多个素材
- 了解审核规则
- 及时调整
