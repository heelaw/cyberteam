### 3.3 输出规范

所有新输出都必须继续放在对应 Agent 包内部，不要散落到总目录。

固定结构如下：

- `文案Agent/SKILL.md`：根路由与岗位定义
- `文案Agent/SOUL.md`：岗位边界与协作姿态
- `文案Agent/references/`：总路由、方法矩阵、平台预设、复盘、根层验收
- `文案Agent/scripts/`：路由、校验、诊断脚本
- `文案Agent/skills/<skill-name>/`：每个最小 SOP 单元

每个 Skill 目录固定包含：

- `SKILL.md`
- `references/reference.md`
- `assessments/assessment.md`
- `scripts/run.py`

### 3.4 目录规范

- 根层只放路由、边界、门禁、复盘入口。
- 细节下沉到 Skill 目录。
- 不要把所有知识堆在一个大文件里。
- 不要把旧版本内容散落到总目录。
- `reference.md` 只写方法、案例、边界、失败路径。
- `assessment.md` 只写门禁，不写长篇说明。
- `run.py` 只做真逻辑或明确诊断，不做假装自动化。
