# 文案自检清单器

## 作用

交付前复核闭环，不负责新增卖点。

这一步是最后一道门，检查通过才算能交出去。

## 启动前

- 必须已有初稿或修改稿。
- 如果连对象、卖点、场景都不清楚，先回上游。

## 七问

1. 写给谁？
2. 说了什么卖点？
3. 场景对不对？
4. 结构对不对？
5. 有没有明确动作？
6. 证据够不够？
7. 有没有夸大或硬承诺？

## 怎么查

- 对象：主写对象和辅助对象有没有混。
- 卖点：是不是用户能感知的结果，不是口号。
- 场景：长度、介质、阅读状态是否匹配。
- 结构：顺序是否符合当前说服目标。
- 动作：CTA 是否具体、单一、和场景匹配。
- 证据：有没有事实、案例、数据、过程或边界说明。
- 风险：有没有无法保证的结果和绝对化表达。

## 评分法

- 每问 1 分，共 7 分。
- 7 分：可直接过。
- 6 分：可过，但要确认无硬承诺。
- 4-5 分：回退补齐。
- 0-3 分：不能交付。

## 回退规则

- 对象不清：回 `segment-users`。
- 卖点不清：回卖点 Skill。
- 场景不清：回 `judge-scenario`。
- 结构不清：回 `select-structure`。
- 文字太虚：回 `edit-copy`。
- 还不闭环：继续补证据，不要直接交付。

## 交付骨架

- 是否通过
- 风险点
- 修改建议
- 回退路径

## 新人校验

- 新人能不能逐项核对，不只给总评。
- 新人能不能指出具体问题，而不是只说“整体不错”。
- 新人能不能说清楚为什么不通过。
- 新人能不能给出明确回退路径。

## 绝对不能

- 不能只给总评不逐项核对。
- 不能把风险藏起来。
- 不能把不能证明的东西判成通过。
- 不能忽略 CTA 和场景匹配。
- 不能把“感觉不错”当成通过标准。
- 不能把辅助对象误判为主对象。
