# 用户分层器

## 作用

把“写给谁”拆成可执行的主写对象、辅助对象和不该写的人群。

这一步不是贴标签，而是先弄清谁出钱、谁使用、谁拍板、谁阻拦、谁影响。

## 启动前

- 先确认这是单人决策、买用一体、买用分离，还是委员会决策。
- 如果完全没有人群线索，先回去补原话、行为或对话。

## 判断顺序

1. 先找决策权，再找角色名。
2. 再找阶段、频率和意愿差异。
3. 再看价格敏感度和认知水平。
4. 最后确认主写对象、辅助对象和不该写的人群。

## SOP

1. 先找身份和角色差异。
2. 再找阶段、频率和意愿差异。
3. 识别价格敏感度和认知水平。
4. 判断谁是主写对象、谁是辅助对象。
5. 必要时拆出购买者、使用者、拍板者、阻拦者、影响者。
6. 处理单人产品、买用一体、买用分离、委员会决策和角色冲突。
7. 输出各层诉求、敏感点、主张重点和不该写的人群。

## 需要特别看

- 决策权是否和使用权分离。
- 价格敏感的人和真正拍板的人是不是同一个人。
- 用户会不会在不同阶段说不同的话。
- 一个群体里是否存在显著冲突。

## 输出骨架

- 用户分层表
- 主写对象
- 辅助对象
- 决策链角色
- 各层诉求
- 各层敏感点
- 不该写的人群

## 新人校验

- 新人能不能先找到决策链，再找角色标签。
- 新人能不能分清主对象、辅助对象、阻拦者。
- 新人能不能处理“一个人说了很多身份”的情况。
- 新人能不能知道什么时候该继续补角色，什么时候可以进入卖点。

## 绝对不能

- 不能把人群写成空泛标签。
- 不能只按年龄、性别、职业分人。
- 不能把所有人都写成主对象。
- 不能替代卖点和场景判断。
- 不能忽略拍板者和阻拦者。
