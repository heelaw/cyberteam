# 用户分层

## 适用场景

- 同一产品面对多类用户，需要先选主写对象。
- 不知道该从哪个人群切入。
- 需要让文案“对人说话”，而不是对抽象人群说话。

## 七个分层维度

| 维度 | 看什么 | 怎么收集 | 输出什么 |
|---|---|---|---|
| 身份角色 | 他是谁 | 表单、资料、访谈、组织关系 | 购买者、使用者、影响者 |
| 阶段 | 他处在什么阶段 | 新客/老客、认知/考虑/决策、生命周期 | 现阶段最该说什么 |
| 行为强度 | 他用得多不多 | 使用频率、停留时长、转化路径 | 高频/低频、重度/轻度 |
| 目标强度 | 他有多想解决 | 主动询问、主动比较、主动试用 | 强意愿/弱意愿 |
| 价格敏感度 | 他对价格有多敏感 | 比价、优惠响应、退款原因 | 低价型/价值型 |
| 认知水平 | 他懂多少 | 专业词使用、问题深度、历史购买 | 新手/熟手/专家 |
| 决策权 | 他能不能拍板 | 采购链路、审批链路、群体影响力 | 拍板者/参与者/使用者 |

## 先判断产品结构

1. 单人产品：购买者、使用者、拍板者通常是同一个人，重点看阶段、行为和认知。
2. 买用一体：先按决策权分层，再补行为和价格敏感度。
3. 买用分离：必须拆开购买者、使用者、拍板者、阻拦者、影响者。
4. 委员会决策：把每个角色都列出来，先满足拍板者，再兼顾阻拦者和使用者。

## 主写对象怎么选

优先级不是固定的，按下面顺序判断：

1. 谁最接近最终拍板。
2. 谁最容易被当前卖点直接打动。
3. 谁最需要这条信息来推进下一步。
4. 谁的反对意见最可能让成交失败。

## 主写对象 vs 辅助对象

- 主写对象：正文主要说服的人，开头、结论、证据都围绕他写。
- 辅助对象：需要被兼顾但不能抢主线的人，只在补充说明里出现。

选择规则：

1. 如果只有一个人真正拍板，就把他定为主写对象。
2. 如果购买者和使用者不同，默认主写对象优先选拍板者；若拍板者信息不足，再以最强痛点的使用者为主写对象。
3. 如果有人会阻拦成交，主写对象不能只看好处，必须把他的顾虑写进辅助说明。
4. 如果一条文案必须同时说服多人，只允许一个主写对象，其他人全部进辅助对象。

## 角色冲突处理

1. 购买者关心预算，使用者关心好不好用，拍板者关心结果和风险，阻拦者关心可控性，影响者关心是否站得住。
2. 角色冲突时，先保住拍板者的判断，再补使用者的体验，再处理阻拦者的顾虑。
3. 如果一个人同时是购买者和使用者，按当前任务里最重要的那一层写，不要重复写两个身份。
4. 如果委员会里意见不一致，先找共同判断标准，再写差异点，不能把所有意见平均摊平。

## 细分样例

### 单人产品

- 产品：个人记账工具。
- 主写对象：自己用的人。
- 辅助对象：无，最多补一个“未来可能给家人看”的次级对象。

### buyer = user

- 产品：个人效率 App。
- 主写对象：自己决定、自己使用的人。
- 写法重点：直接讲使用后果、节省时间、降低摩擦。

### payer ≠ user

- 产品：企业培训软件。
- 主写对象：买单的负责人。
- 辅助对象：使用者、技术同事、财务。
- 写法重点：负责人看结果和风险，使用者看上手，技术同事看接入。

### 角色冲突

- 产品：设计协作工具。
- 冲突：老板想控预算，设计师想要高级功能，IT 想要安全接入。
- 处理：主写对象先定老板或采购负责人，辅助对象分别承接功能、预算、安全三类疑虑。

### 委员会决策

- 产品：B 端系统。
- 角色：采购、使用、审批、法务、IT。
- 处理：先写拍板者最关心的结果，再补阻拦者的限制条件，最后补使用者体验。

## 分层顺序

1. 先按决策权和角色分主次。
2. 再按阶段和意愿找切入口。
3. 再看行为、价格敏感和认知水平。
4. 最后确定主写对象和辅助对象。

## 每个维度怎么查

### 身份角色

- 看谁付钱、谁使用、谁影响决策。
- 同一个产品里，购买者和使用者经常不是同一个人。

### 阶段

- 看用户是在不知道、了解、比较还是准备行动。
- 阶段决定你该讲认知、讲证据，还是直接推动。

### 行为强度

- 看使用频率、停留时长、重复动作、流失点。
- 行为强度越高，说明切入口越接近实际需求。

### 目标强度

- 看用户是不是主动问、主动搜、主动比。
- 主动行为越多，需求越明确。

### 价格敏感度

- 看用户是否先问价格、是否总比最低价、是否容易被优惠触发。
- 价格敏感用户不能只讲高价值，不讲对比。

### 认知水平

- 看他会不会用行业词、能不能描述问题细节、有没有历史购买经验。
- 认知低时要先解释，认知高时可以直接讲差异。

### 决策权

- 看谁拍板、谁阻拦、谁提供信息。
- 决策链越长，文案越要兼顾证明和风险。

## 主写对象判断

1. 谁最可能被这条文案直接打动。
2. 谁最有决策权或影响权。
3. 谁最能感知这个卖点。
4. 谁最需要这条信息来做下一步动作。

## 常见误判

- 只按年龄职业分人群。
- 把所有人都当主对象，结果没人被打中。
- 只看使用者，不看拍板者。
- 只看价格敏感，不看认知差异。

## 典型例子

### 例子1：谁是主写对象

- 产品：企业培训内容。
- 主写对象：拍板的负责人，不是最终学员。
- 原因：决定预算的人最关心结果、风险和对外说法。

### 例子2：谁是辅助对象

- 产品：内部协作工具。
- 主写对象：实际每天使用的人。
- 辅助对象：会影响采购的负责人和技术同事。

### 例子3：分层失误

- 把“所有感兴趣的人”都当主写对象。
- 结果文案没有锋芒，也没有明确入口。

### 例子4：同一产品不同对象

- 产品：团队协作工具。
- 购买者关心预算和审批，使用者关心好不好上手，影响者关心接不接得住现有流程。
- 分层失败会导致文案只说一种语言，其他人都不买账。

## 失败案例

### 案例1：只按画像分

- 原文：25-35 岁女性、白领、爱学习。
- 失败点：没有角色、阶段、决策权，不能写。

### 案例2：把所有人都写进来

- 原文：适合新手、熟手、老板、执行者、决策者。
- 失败点：没有主次，文案不会聚焦。

### 案例3：忽略决策权

- 原文：直接对使用者说服。
- 失败点：真正拍板的人可能根本不在场。

## 边界条件

- 如果用户群体本来就单一，可以少做分层，但不能不确认。
- 如果一个产品同时面对多个角色，主写对象必须先定一个。
- 如果价格敏感和认知水平冲突，通常先看认知水平，再看价格。

## 交接判断

- 人群还很泛：先回收更多原话、行为和决策信息。
- 主写对象已明确：进入卖点或场景判断。
- 一条文案要同时打多个角色：保留主写对象，其他对象放辅助说明。

## 绝对不能

- 不能只按年龄、性别、职业分人群。
- 不能把所有角色都当主写对象。
- 不能忽略决策权。
- 不能忽略认知差异。
- 不能把“所有人”写进同一段文案。

## 交付样式

- 先给主写对象。
- 再给辅助对象。
- 再给各层特征和敏感点。
- 最后给不该写的人群。

## 一条完整样例

### 输入

- 产品：团队协作工具。
- 业务场景：负责人要推动项目，使用者每天执行，老板看结果，技术同事看接入成本。

### 处理

1. 身份角色：购买者是负责人，使用者是执行者，影响者是老板和技术同事。
2. 阶段：负责人已经在比较方案。
3. 行为强度：执行者高频使用。
4. 目标强度：负责人有明确目标。
5. 价格敏感度：老板看预算。
6. 认知水平：执行者偏新手，技术同事偏熟手。
7. 决策权：负责人拍板，老板批准，技术同事把关接入。

### 输出

- 主写对象：负责人。
- 辅助对象：执行者、老板、技术同事。
- 各层特征：负责人关心效率和风险，执行者关心好用，老板关心成本，技术同事关心接入。
- 各层敏感点：时间、预算、上手、兼容性。
- 应该避开的泛化描述：适合所有人、万能工具、人人都能用。

## 回退提示

- 如果角色和决策权冲突，先按决策权分。
- 如果一个人同时有多个身份，按当前任务里最重要的身份写。

## 输出骨架

- 主写对象
- 辅助对象
- 各层特征
- 各层诉求
- 各层敏感点
- 应该避开的泛化描述

## 决策链分层

- 购买者：出钱的人。
- 使用者：每天实际操作的人。
- 拍板者：最后决定用不用的人。
- 阻拦者：会否决、会卡预算、会卡流程的人。
- 影响者：不拍板，但会影响判断的人。

## 角色冲突规则

1. 先找拍板者，再找使用者。
2. 拍板者和使用者冲突时，正文优先满足拍板者，说明里兼顾使用者。
3. 如果有阻拦者，必须提前处理他的顾虑，否则文案不算闭环。
4. 一个人同时兼任多角色时，按当前任务里最关键的那个身份写。

## 每个维度怎么落到文案

### 身份角色

- 写“谁在什么关系里用它”。
- 不是“25-35 岁白领”，而是“负责拍板的人”“每天执行的人”。

### 阶段

- 新手：先解释是什么、为什么现在要解决。
- 考虑：先讲差异、证据和风险。
- 决策：先讲结果、边界和动作。

### 行为强度

- 高行为强度说明问题已经进入日常，文案要更贴近具体操作。
- 低行为强度说明需求还未稳定，文案要先建立认知。

### 目标强度

- 强意愿：可直接谈方案和动作。
- 弱意愿：先证明问题值得处理。

### 价格敏感度

- 价格敏感：先说明总成本、替代成本和风险。
- 价值导向：先说明结果和证据。

### 认知水平

- 新手：少术语，多解释，多场景。
- 熟手：少解释，多差异，多对比。
- 专家：少废话，直接给判断和边界。

### 决策权

- 拍板者关心结果和风险。
- 使用者关心是否好上手。
- 阻拦者关心是否可控、可接入、可解释。

## 不该写的人群

- 和当前购买链路无关的人。
- 只会制造噪音、不会影响动作的人。
- 只能做背景描述，不能影响决策的人。

## 输出到写作的翻译

- 主写对象决定开头说什么。
- 辅助对象决定补充说明说什么。
- 敏感点决定证据和反对意见怎么写。

## 额外样例

### 场景

- 团队协作工具。
- 负责人决定买不买，执行者每天用，老板看预算，技术同事看接入。

### 输出

- 主写对象：负责人。
- 辅助对象：执行者、老板、技术同事。
- 写法重点：负责人看结果和风险，执行者看好不好用，老板看预算，技术同事看接入成本。



## 单人和委员会场景

### 单人产品

- 如果产品天然只有一个使用者和一个决策者，可以少分层，但仍要确认角色是不是同一个人。
- 写作重点直接对准那个唯一决策者或唯一使用者。

### 买用一体

- 买的人和用的人是同一个人。
- 主写对象通常就是使用者。
- 辅助对象只保留可能的阻拦者或影响者。

### 买用分离

- 付款人和使用者不是同一个人。
- 正文优先满足拍板者，说明里兼顾使用者。
- 如果两者冲突，先写拍板者最在意的结果和风险。

### 委员会决策

- 至少要分出拍板者、使用者、阻拦者、影响者。
- 主写对象优先选真正决定是否推进的人，而不是声音最大的人。
- 辅助对象只补他们会关心的那一层信息。

## 主写对象选择规则

1. 先看谁能决定成败。
2. 再看谁最能感知卖点。
3. 再看谁最需要这条信息来行动。
4. 主写对象只能有一个。

## 主写对象 vs 辅助对象

- 主写对象：正文第一目标，开头就要打中。
- 辅助对象：不抢主线，只在证据或说明里兼顾。
- 如果所有对象都写进去，等于没有主写对象。

## 验收门禁

- 如果缺少关键输入，先回退到上一 Skill。
- 如果证据不足，不要伪装完整。
- 如果输出已经能直接进入下一步，就只给下一步，不额外加戏。
