# 内容合并规则

## 重合内容判断

### 相似度标准

```python
IF 知识点标题相似度 > 80% AND 核心内容相似度 > 70%:
    THEN 标记为"重合" → 合并为单条
ELSE IF 知识点标题相似度 > 60% OR 标题相关且内容相关:
    THEN 标记为"互补" → 补充扩展
ELSE:
    THEN 标记为"独立" → 保留为独立小节
```

### 相似度计算方法

#### 标题相似度
- 完全相同：100%
- 仅顺序/修饰词不同：80-90%
- 核心概念相同：60-80%
- 相关但不同概念：<60%

#### 内容相似度
- 完全相同：100%
- 仅表达方式不同：70-90%
- 核心内容相同：60-70%
- 相关但内容不同：<60%

## 合并策略

### 重合内容合并

当内容被标记为"重合"时：

1. **保留最完整版本**
   - 优先级：逐字稿 > 内容增强 > 大纲
   - 选择内容更详细、案例更丰富的版本

2. **整合所有独特元素**
   - 合并案例：保留所有案例，标注来源
   - 合并观点：整合讲师的不同观点和经验
   - 合并细节：补充不同版本中的细节

3. **统一表述**
   - 使用最清晰的表述方式
   - 统一术语和概念
   - 确保逻辑流畅

4. **标注来源**
   ```markdown
   > 来源课程：课程A(主要)、课程B(补充)
   ```

### 互补内容扩展

当内容被标记为"互补"时：

1. **保持主内容**
   - 选择作为主要内容的知识点
   - 通常选择内容更丰富、更基础的版本

2. **补充独特内容**
   ```markdown
   ## 知识点名称

   [主要内容]

   ### 补充知识（来源：课程B）
   [课程B中的独特内容]
   ```

3. **标注来源**
   - 明确标注补充内容的来源课程
   - 便于追溯和验证

## 合并优先级

### 内容来源优先级

| 优先级 | 来源 | 说明 |
|--------|------|------|
| 1 | 逐字稿 | 最完整的原始内容，讲师原话 |
| 2 | 内容增强 | 经过整理的内容，但可能有遗漏 |
| 3 | 课程大纲 | 结构参考，内容较简略 |

### 课程优先级（可选）

如果用户指定了课程优先级：

```bash
/course-merge 重组课程 "课程名称" \
  --source1 /path/to/course1 --priority=high \
  --source2 /path/to/course2 --priority=medium \
  --source3 /path/to/course3 --priority=low
```

## 合并检查清单

合并每个知识点时，确认以下事项：

- [ ] 已读取所有来源课程的相关内容
- [ ] 已识别内容类型（重合/互补/独立）
- [ ] 已选择主要内容版本
- [ ] 已合并所有案例和细节
- [ ] 已标注来源课程
- [ ] 已统一术语和表述
- [ ] 已确保逻辑流畅
- [ ] 已生成 ASCII 图表（如适用）

## 常见合并场景

### 场景 1：完全相同的内容

```
课程A：如何设计课程大纲
课程B：课程大纲设计方法
```

→ 合并为：`课程大纲设计方法`，保留最详细版本

### 场景 2：内容相似但有差异

```
课程A：用户画像的三种方法
课程B：用户画像的五个维度
```

→ 合并为：`用户画像的方法和维度`
- 主要内容：三种方法（课程A）
- 补充内容：五个维度（课程B）

### 场景 3：相关但独立的内容

```
课程A：如何选择目标用户
课程B：如何分析目标用户
```

→ 保留为两个独立小节：
- X.1 如何选择目标用户（来源：课程A）
- X.2 如何分析目标用户（来源：课程B）

## 冲突处理

### 观点冲突

当不同课程对同一概念有不同观点时：

1. **保留所有观点**
   - 列出所有不同观点
   - 标注各自来源
   - 说明适用场景

2. **示例**
   ```markdown
   ## 课程定价策略

   关于课程定价，有以下不同观点：

   ### 观点一：价值定价（来源：课程A）
   根据课程价值定价...

   ### 观点二：竞争定价（来源：课程B）
   根据市场竞品定价...

   ### 综合建议
   建议结合两种方法，先评估价值，再参考竞品...
   ```

### 内容冲突

当内容有明显矛盾时：

1. **标注冲突点**
2. **提供解释**
3. **给出建议**

## 合并质量标准

- 知识点完整：所有来源的内容都得到保留
- 逻辑清晰：合并后的内容逻辑流畅
- 来源标注：每个补充内容都标注来源
- 表述统一：术语和表述一致
- 图表完善：重要概念都有可视化图表
- 无冗余：重复内容已合并
- 无遗漏：有价值的内容都保留
