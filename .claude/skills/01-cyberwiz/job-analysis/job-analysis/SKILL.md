# cyberwiz-jobanalysis - 运营岗位结构化分析Skill

## 概述

本Skill基于"运营六要素"方法论，帮助用户深度拆解和分析任何运营/营销岗位。通过系统化分析公司的商业模式、业务模型和收入公式，快速定位岗位在业务链条中的位置，明确关键指标，制定工作策略。

## 核心方法论：运营六要素

```
起点：用户和需求
终点：业务目标
底层：用户路径和数据漏斗
中间：信息触达场景 + 工作手段

所有运营/营销工作都可以用这六要素来拆解！
```

### 六要素详解

1. **用户**：用户画像、需求、使用场景、痛点
2. **关键业务指标**：为收入/流量/转化/留存负责
3. **用户路径和数据漏斗**：用户行为全链路
4. **信息触达场景和方式**：直播间、朋友圈、社群、APP界面等
5. **工作手段**：文案、地推、活动、用户分层、激励等
6. **产品和商业模式**：如何挣钱、产品属性、价格、频次等

## 四种常见商业模式

### 模式1：流量+广告变现

**业务逻辑**：C端做大流量 → B端卖广告

**前端业务**：
- C端：内容生产 → 用户留存 → 流量规模扩大
- B端：寻找广告主 → 深度洽谈 → 成交回款

**关键指标**：
- C端：DAU/MAU、用户时长、留存率
- B端：广告主数、询单数、成交数、回款率
- 后端：作者数、内容数

**典型公司**：媒体、自媒体、广告变现博主

---

### 模式2：平台+抽佣

**业务逻辑**：撮合CB两端交易 → 平台抽成

**前端业务**：
- C端：流量获取 → 付费转化 → 复购
- B端：商家数、商品数、博主数
- 交易保障：满意度、NPS、好评率

**关键指标**：
- C端：新增用户、付费用户数、复购率
- B端：商家数、SKU数、活跃率
- 交易：GMV、抽成率、客单价

**典型公司**：美团、滴滴、淘宝

---

### 模式3：会员订阅

**业务逻辑**：用户付费会员 → 持续续费

**前端业务**：
- 流量获客 → 注册登录 → 留存
- 用户分层 → 转化 → 高价值用户服务 → 续费

**关键指标**：
- 新增：注册用户数、获客成本
- 转化：付费转化率、首单金额
- 留存：续费率、LTV、churn率
- ARPU：平均每用户收入

**典型公司**：SaaS、知识付费、视频会员

---

### 模式4：产品销售

**业务逻辑**：直接销售产品获取收入

**前端业务**：
- 引流曝光 → 点击 → 咨询 → 购买 → 复购

**关键指标**：
- 流量：曝光量、点击率
- 转化：咨询率、下单率、成交率
- 收入：GMV、客单价、复购率

**典型公司**：电商、零售、教育培训

## 三个分析层次

### 宏观层面
- **关注点**：整个业务运营体系搭建
- **思考起点**：商业模式设计和关键业务逻辑
- **核心工作**：从商业模式和产品属性推演业务体系

### 中观层面
- **关注点**：局部业务闭环健康度诊断
- **思考起点**：业务前端+后端衔接
- **核心工作**：业务流程优化和效率提升

### 微观层面
- **关注点**：具体执行落地工作
- **思考起点**：关键业务指标+用户路径
- **核心工作**：找准发力点，选择工作手段

> **本Skill主要聚焦于微观和中观层面**

## 用户洞察画布

分析任何运营工作，首先要理解用户。建议建立用户洞察画布：

### 1. 基础属性
- 地域、行业、公司、岗位
- 年龄、性别、身高体重
- 家庭状况、教育程度
- 收入、资产

### 2. 重要情绪
- 高频正向情绪场景（成就感、兴奋点）
- 高频负向情绪场景（焦虑、痛苦点）
- 共同记忆（90后、80后、一线城市等）

### 3. 喜好、痛点、生活方式
- 围绕产品的痛点
- 日常生活方式
- 兴趣爱好

## 产品卖点挖掘（7个维度）

当需要理解产品时，可以从以下7个角度挖掘卖点：

1. 产品能解决什么问题？
2. 相比其他产品有什么显著特点？
3. 竞争对手存在哪些弱点？我们能做得更好吗？
4. 有哪些生产过程中的细节可以体现产品的好？
5. 有哪些实际发生的结果和用户行为可以体现产品的好？
6. 有哪些人事、品牌的背书可以体现产品的好？
7. 还有哪些意想不到的角度？

## 两种用户路径类型

### 转化流程型路径
关注点：单次转化的效率提升
- 曝光 → 点击 → 咨询 → 下单 → 付款
- 核心指标：转化率、客单价、ROI

### 用户增长型路径
关注点：用户长期价值挖掘
- 获客 → 激活 → 留存 → 变现 → 推荐
- 核心指标：LTV、留存率、K因子

## 快速分析框架

### 第一步：识别商业模式（5分钟）

问自己3个问题：
1. 这家公司靠什么挣钱？
2. 谁是付费方？（C端用户 vs B端客户）
3. 我的工作直接影响哪个收入环节？

### 第二步：定位岗位位置（5分钟）

判断岗位属于：
- **业务前端**：直接面对用户/客户（销售、运营、推广）
- **业务后端**：用户/客户不可见（产品、技术、数据）

### 第三步：明确关键指标（10分钟）

根据岗位定位确定：
- 前端岗位：流量、转化、收入、用户数
- 后端岗位：效率、质量、成本、产能

### 第四步：梳理用户路径（15分钟）

画出用户从接触到付费/转化的全流程，识别：
- 关键节点有哪些？
- 哪里是瓶颈？
- 我的工作影响哪个环节？

### 第五步：制定工作策略（20分钟）

基于以上分析：
- 我的核心发力点在哪里？
- 需要掌握哪些工作手段？
- 如何设置优先级？

## 使用建议

1. **不要被岗位名称迷惑**：不同公司的"用户运营""产品运营"可能干完全不同的事
2. **从商业模式出发思考**：理解公司如何挣钱是理解一切运营工作的起点
3. **明确业务指标是关键**：不知道该干什么时，回到业务指标找答案
4. **用户路径是底层逻辑**：搞清楚用户怎么来、怎么转化、怎么留存
5. **持续积累弹药库**：用户洞察、产品卖点、工作手段都需要持续积累

## 示例

### 案例1：少儿编程-用户增长专家

**商业模式识别**：教培行业，产品销售型

**岗位定位**：业务前端，对拉新和获客负责

**关键指标**：有效获客数量、获客成本

**用户路径**：
```
渠道曝光 → 点击引流物料 → 加微信/关注账号/参加直播 →
社群维护/活动互动 → 筛选高意向用户 → 付费转化
```

**工作场景**：微信生态为主

**工作手段**：社群维护、直播活动、内容互动、多渠道引流创新

---

### 案例2：美团闪购-新媒体运营主管

**商业模式识别**：平台+抽佣型，O2O平台

**岗位定位**：业务前端，有两种可能：
- 可能性A：对收入负责（卖货导向）
- 可能性B：对影响力负责（涨粉导向）

**关键指标**：
- 可能性A：收入、GMV
- 可能性B：粉丝数、活跃用户数

**用户路径**：
```
多渠道曝光（抖音/微信/小红书） → 点击/关注 →
种草内容 → 引流到APP/小程序 → 首单转化 → 复购 → 忠诚用户
```

**工作场景**：短视频平台、微信账号、小红书等

**工作手段**：内容创作、直播带货、种草文案、用户互动
