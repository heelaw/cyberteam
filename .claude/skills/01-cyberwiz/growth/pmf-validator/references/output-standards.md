# PMF 验证输出标准

> **版本**: v1.0
> **创建时间**: 2026-03-11
> **用途**: 规范 PMF 验证报告的输出格式和质量标准

---

## 输出报告结构

### 必需章节

```markdown
# [产品名称] PMF 验证报告

## 📊 执行摘要
## 🎯 目标用户分析
## 🔍 需求验证
## 💡 产品定位
## 🚀 MVP 规划
## 📈 PMF 评估
## ✅ 结论与建议
```

### 可选章节

```markdown
## 附录：调研数据
## 附录：用户访谈记录
## 附录：竞品分析
```

---

## 各章节内容标准

### 1. 执行摘要 (必需)

**内容要求**:
- 一句话描述产品
- 核心验证结论（通过/失败/需改进）
- 关键数据摘要（3-5 个）
- 核心建议（1-2 条）

**模板**:
```markdown
## 📊 执行摘要

**产品**: [产品名称] - [一句话描述]

**验证结论**: [通过/失败/需改进]

**关键数据**:
- 40% 规则：[X]% 用户表示"非常遗憾"
- D7 留存：[X]%
- NPS：[X]分

**核心建议**: [1-2 条具体建议]
```

**验收标准**:
- [ ] 包含验证结论
- [ ] 包含至少 3 个关键数据
- [ ] 包含至少 1 条核心建议
- [ ] 长度控制在 200 字以内

---

### 2. 目标用户分析 (必需)

**内容要求**:
- 用户画像（基础属性 + 行为特征）
- 用户规模估算
- 用户获取渠道

**模板**:
```markdown
## 🎯 目标用户分析

### 用户画像
| 维度 | 描述 |
|------|------|
| 年龄 | [范围] |
| 性别 | [比例] |
| 地区 | [分布] |
| 职业 | [类型] |
| 收入 | [范围] |

### 行为特征
- 使用场景：[场景描述]
- 使用频率：[频率]
- 信息渠道：[渠道列表]

### 市场规模
- 目标用户规模：[数量]
- 可触达用户：[数量]
- 付费用户估算：[数量]
```

**验收标准**:
- [ ] 包含至少 3 个人口统计学特征
- [ ] 包含至少 2 个行为特征
- [ ] 有市场规模估算
- [ ] 用户画像清晰可识别

---

### 3. 需求验证 (必需)

**内容要求**:
- 需求清单（P0/P1/P2 优先级）
- 需求强度数据
- 验证方法说明

**模板**:
```markdown
## 🔍 需求验证

### 需求清单
| 优先级 | 需求描述 | 需求强度 | 验证数据 |
|--------|---------|---------|---------|
| P0 | [需求] | [X]/5 分 | [数据来源] |
| P1 | [需求] | [X]/5 分 | [数据来源] |

### 需求强度分布
- 高强度需求（≥4 分）：[X]%
- 中强度需求（2-3 分）：[X]%
- 低强度需求（≤1 分）：[X]%

### 验证方法
- [方法 1]：[样本量] [结果]
- [方法 2]：[样本量] [结果]
```

**验收标准**:
- [ ] 需求清单包含至少 5 条需求
- [ ] 需求有优先级排序
- [ ] 每条需求有数据支撑
- [ ] 说明验证方法和样本量

---

### 4. 产品定位 (必需)

**内容要求**:
- 定位陈述（完整模板）
- 差异化分析
- 竞品对比

**模板**:
```markdown
## 💡 产品定位

### 定位陈述
对于 [目标用户]
他们面临 [核心问题]
我们的产品是 [产品类别]
它能提供 [核心价值]
与 [竞品] 不同
我们的差异化优势是 [独特优势]
因为 [信任理由]

### 竞品对比
| 维度 | 竞品 A | 竞品 B | 我们 |
|------|--------|--------|------|
| 价格 | [价格] | [价格] | [价格] |
| 功能 | [功能] | [功能] | [功能] |
| 体验 | [体验] | [体验] | [体验] |

### 差异化优势
1. [优势 1]
2. [优势 2]
```

**验收标准**:
- [ ] 定位陈述完整填写
- [ ] 目标用户清晰
- [ ] 核心价值明确
- [ ] 差异化优势清晰
- [ ] 有竞品对比分析

---

### 5. MVP 规划 (必需)

**内容要求**:
- MVP 功能列表（≤3 个核心功能）
- 功能优先级
- 验证指标

**模板**:
```markdown
## 🚀 MVP 规划

### 核心功能
| 功能 | 优先级 | 验证指标 | 目标值 |
|------|--------|---------|--------|
| [功能 1] | P0 | [指标] | [目标值] |
| [功能 2] | P0 | [指标] | [目标值] |
| [功能 3] | P1 | [指标] | [目标值] |

### 验证计划
- 上线时间：[日期]
- 验证周期：[周数]
- 样本量目标：[用户数]

### 成功标准
- [指标 1] ≥ [目标值]
- [指标 2] ≥ [目标值]
- [指标 3] ≥ [目标值]
```

**验收标准**:
- [ ] 核心功能 ≤3 个
- [ ] 每个功能有验证指标
- [ ] 有明确成功标准
- [ ] 有验证计划

---

### 6. PMF 评估 (必需)

**内容要求**:
- PMF 核心指标数据
- 与目标值对比
- 达标情况判定

**模板**:
```markdown
## 📈 PMF 评估

### 核心指标
| 指标 | 目标值 | 实际值 | 状态 |
|------|--------|--------|------|
| 40% 规则 | ≥40% | [X]% | ✅/❌ |
| D7 留存 | ≥40% | [X]% | ✅/❌ |
| D30 留存 | ≥20% | [X]% | ✅/❌ |
| NPS | ≥30 | [X] | ✅/❌ |
| K 因子 | ≥0.5 | [X] | ✅/❌ |

### 用户反馈
**正面反馈**:
- "[用户原话]"
- "[用户原话]"

**负面反馈**:
- "[用户原话]"
- "[用户原话]"
```

**验收标准**:
- [ ] 包含至少 5 个核心指标
- [ ] 每个指标有目标值和实际值
- [ ] 有达标状态判定
- [ ] 有用户原话引用（正/负各至少 1 条）

---

### 7. 结论与建议 (必需)

**内容要求**:
- 明确结论（三选一）
- 具体行动建议
- 下一步计划

**模板**:
```markdown
## ✅ 结论与建议

### 验证结论
- [x] 已达到 PMF，建议加大投入
- [ ] 接近 PMF，建议优化后重新验证
- [ ] 未达到 PMF，建议重新思考产品方向

### 核心依据
1. [依据 1]
2. [依据 2]
3. [依据 3]

### 行动建议
**立即执行**:
- [ ] [行动 1]
- [ ] [行动 2]

**后续验证**:
- [ ] [验证 1]
- [ ] [验证 2]

### 下一步计划
| 时间 | 行动 | 负责人 | 预期结果 |
|------|------|--------|---------|
| 第 1 周 | [行动] | [人] | [结果] |
| 第 2 周 | [行动] | [人] | [结果] |
```

**验收标准**:
- [ ] 结论明确（三选一）
- [ ] 有核心依据支撑（至少 3 条）
- [ ] 行动建议具体可执行
- [ ] 有下一步计划

## 质量标准

### 完整性检查清单

报告必须包含以下内容：

**基础要求**:
- [ ] 执行摘要
- [ ] 目标用户分析
- [ ] 需求验证
- [ ] 产品定位
- [ ] MVP 规划
- [ ] PMF 评估
- [ ] 结论与建议

**数据要求**:
- [ ] 至少 100 个用户调研样本
- [ ] 40% 规则数据完整
- [ ] 至少 3 个核心指标达标
- [ ] 有用户原话引用

**分析要求**:
- [ ] 需求有优先级排序
- [ ] 有竞品对比分析
- [ ] 有差异化优势说明
- [ ] 有明确成功标准

### 准确性检查清单

- [ ] 所有数据有来源标注
- [ ] 用户原话真实引用（非杜撰）
- [ ] 竞品数据有公开来源
- [ ] 市场规模估算有依据

### 可执行性检查清单

- [ ] 结论明确可决策
- [ ] 行动建议具体可执行
- [ ] 下一步计划有时间表
- [ ] 每项行动有负责人

---

## 质量阈值

| 等级 | 分数 | 标准 |
|------|------|------|
| EXCELLENT | ≥0.95 | 所有必需章节完整 + 数据充分 + 分析深入 |
| GOOD | ≥0.85 | 所有必需章节完整 + 数据基本充分 |
| NEEDS_WORK | <0.85 | 缺少必需章节或数据不足 |

### 评分公式

```
总分 = 完整性 (40%) + 准确性 (30%) + 可执行性 (30%)

完整性 (40 分):
  - 7 个必需章节：35 分（每章 5 分）
  - 可选章节：5 分

准确性 (30 分):
  - 数据来源标注：10 分
  - 用户原话引用：10 分
  - 竞品分析准确：10 分

可执行性 (30 分):
  - 结论明确：10 分
  - 行动具体：10 分
  - 计划清晰：10 分
```

---

## 输出格式要求

### 文档格式
- 使用 Markdown 格式
- 使用中文（专业术语可用英文）
- 使用表格呈现对比数据
- 使用列表呈现要点

### 长度要求
- 执行摘要：≤200 字
- 完整报告：3000-5000 字
- 附录：不限

### 视觉要求
- 使用 emoji 增强可读性
- 使用粗体强调重点
- 使用引用块突出用户原话
