# cyberwiz-xiaohongshu - 小红书IP打造与运营专家

## 概述

本技能基于小红书平台运营实战方法论，提供从0到1打造小红书IP的完整体系。涵盖账号定位、对标分析、内容创作、数据分析、商业变现全流程，帮助用户系统化建立小红书运营能力。

### 核心价值

- **系统化IP打造**：从定位到变现的完整闭环
- **实战导向**：基于真实成功案例的方法论
- **数据驱动**：以数据指导决策和优化
- **可落地执行**：详细的SOP和检查清单
- **8阶段工作流**：专业级内容生产流程，支持批量创作

## ⚠️ CRITICAL RULES - 8阶段工作流铁律

**仅在执行8阶段内容生产工作流时适用。违反这些规则将导致工作流混乱和返工。**

### 规则 1：流程控制规则

- **默认模式**：完成当前 Stage 后，**停止并等待用户明确指令**
- **自动执行模式**：如果用户选择了自动执行，完成当前 Stage 后自动进入下一阶段
- ❌ 禁止在非自动执行模式下自动进入下一阶段
- ❌ 禁止说"既然 X 完成了，现在我来做 Y..."
- ❌ 禁止在非自动执行模式下批量执行多个 Stage

### 规则 2：用户满意 ≠ 批准继续

**只有以下情况才算批准：**
- ✅ "继续" / "下一步" / "开始 Stage X"
- ✅ 使用快捷命令 `/topic` `/materials` `/angles` 等

**以下不算批准：**
- ❌ "好" / "不错" / "可以"（只是满意，不是批准）
- ❌ "知道了" / "看到了"（只是确认，不是批准）
- ❌ 用户沉默

### 规则 3：每个阶段完成后的强制流程

```
1. 保存输出文件到 workflow/ 对应目录
2. 用 Read 工具验证文件已保存
3. 汇报完成情况（文件路径、内容摘要）
4. 如果是自动执行模式：自动进入 Stage X+1
5. 如果是非自动执行模式：明确询问：「⏸️ 是否进入 Stage X+1？」并停止执行，等待用户明确回复
```

### 规则 4：启动阶段前的强制检查

```
1. 前置阶段的输出文件是否存在？
2. 用户是否给了明确的"开始"指令？
3. 前置条件是否满足？
```

### 规则 5：独立任务并行执行

当多个子任务之间没有依赖关系时，必须在同一消息中并行执行：
- ✅ 正确：一个消息中包含多个独立的 Grep/Glob/Read 调用
- ❌ 错误：分多个消息串行执行独立任务
- **原则**：并行执行优于串行执行，显著提升效率

## 核心方法论框架

### 一、倒推学习法

从未来目标反推现在行动：

```
目标账号规模（1万/10万粉）
    ↓
达成时间规划（3个月/6个月/1年）
    ↓
变现方式设计
    ↓
现阶段行动清单
```

### 二、分析定位法

四个维度系统定位：

```
了解平台 → 了解赛道 → 了解用户 → 了解自己
    ↓           ↓          ↓          ↓
CES评分    复购率优先   目标需求    个人优势
流量逻辑    垂直领域    用户画像    差异化特色
```

### 三、CES流量评分公式

理解小红书流量分发逻辑：

| 互动行为 | 权重分 | 说明 |
|---------|--------|------|
| 点赞 | 1分 | 基础互动 |
| 收藏 | 1分 | 内容价值 |
| 评论 | 4分 | 深度互动 |
| 转发 | 4分 | 传播价值 |
| 关注 | 8分 | 账号认可 |

**核心洞察**：引导评论和关注是提升流量最有效的方式

**目标**：发现小红书热点选题，评估选题价值，输出选题简报。

**执行步骤**：

#### 1.1 热点扫描（5个平台）

**小红书平台**：
```

查看：热门笔记、上升话题、品牌榜单

```

**跨平台热点**：
- 微博热搜：查看热搜榜、话题榜
- 知乎热榜：查看热榜、推荐榜
- 抖音热点：查看热点榜、挑战榜
- 哔哩哔哩：查看热门、排行榜

#### 1.2 选题评估（5维度）

| 维度 | 权重 | 评分标准 |
|------|------|----------|
| 热度 | 30% | 搜索量、点赞量、讨论度 |
| 相关性 | 25% | 与账号定位的匹配度 |
| 可创作性 | 20% | 是否有足够素材支撑 |
| 差异化 | 15% | 是否有独特视角 |
| 变现潜力 | 10% | 是否能带来商业价值 |

**总分计算**：各维度×权重，满分100分
- 80+分：优先创作
- 60-79分：考虑创作
- <60分：暂不考虑

### Stage 2：素材收集（Researcher）

**目标**：围绕选题收集全面素材，建立素材库。

**执行步骤**：

#### 2.1 素材收集策略（5个维度，并行执行）

**维度1：官方信息**
- 搜索品牌官网、产品手册
- 查找官方数据、报告
- 记录官方宣称信息

**维度2：媒体报道**
- 搜索行业媒体报道
- 查找KOL测评文章
- 收集第三方评价

**维度3：用户反馈**
- 查看小红书评论区
- 搜索知乎讨论
- 收集微博用户观点

**维度4：案例/故事**
- 搜索真实使用案例
- 查找用户晒单
- 收集前后对比案例

**维度5：数据支撑**
- 搜索行业数据报告
- 查找统计数据
- 收集科学研究证据

**并行执行示例**：
```python
# 一次性启动5个独立搜索任务
search_tasks = [
    search_official_info(topic),
    search_media_reports(topic),
    search_user_feedback(topic),
    search_cases(topic),
    search_data(topic)
]
# 等待所有任务完成
results = await_all(search_tasks)
```

### Stage 3：角度分析（Strategist）

**目标**：分析素材，提炼独特的切入角度，输出切角方案。

**执行步骤**：

#### 3.1 角度分析方法

**3.1.1 对标账号角度分析**

搜索小红书对标账号，分析他们的内容角度：

| 对标账号 | 内容角度 | 核心观点 | 数据表现 |
|---------|---------|---------|---------|
| @账号1 | [角度1] | [观点] | [点赞/评论] |
| @账号2 | [角度2] | [观点] | [点赞/评论] |

**分析重点**：
- 他们讲了什么？
- 没讲什么？（空白点）
- 哪些角度数据好？
- 如何差异化？

**3.1.2 用户痛点挖掘**

从评论区挖掘用户真实需求：
```
高频问题/抱怨 → 痛点
用户晒单/感谢 → 需求
用户讨论/争论 → 争议点
```

## 完整工作流程：冷启动四阶段

### 阶段一：准备期（第1-3天）

#### 1.1 认知升级

**理解小红书平台特点**：
- 核心用户：18-35岁女性，一二线城市
- 内容偏好：真实、有用、有美感的种草内容
- 算法逻辑：CES评分+关键词匹配+个性化推荐

**调整运营心态**：
- ✅ **渣男思维**：发完就忘，关闭数据提醒
- ✅ **长期主义**：关注持续成长而非单篇数据
- ❌ 避免：数据焦虑、过度关注、频繁修改

#### 1.2 账号定位

**赛道选择三维度**：

| 维度 | 考量标准 | 优选方向 |
|------|----------|----------|
| 用户画像匹配 | 是否匹配小红书核心用户 | ✓ 母婴、美妆、服饰、美食 |
| 品类复购率 | 用户重复消费频率 | ✓ 高复购品类（母婴92%） |
| 对标分析 | 优质账号数量和数据表现 | ✓ 优质账号多、数据好的赛道 |

**人设标签三维度**：

1. **身份信息**
   - 职业标签：职场博主、大学生、宝妈
   - 专业背景：博士在读、CPA持证、健身教练

2. **家庭角色**
   - 宝妈、单身、已婚
   - 增加亲切感和信任度

3. **特色背景**
   - 特殊经历、独特技能、个人成就
   - **需要相关笔记内容展示**

#### 1.3 对标账号建立

**建立对标表格（8维度）**：

| 维度 | 说明 | 示例 |
|------|------|------|
| 名称 | 账号名称+主页链接 | XX妈妈 |
| 形式 | 图文/视频/混合 | 图文 |
| 量级 | 粉丝数量 | 黑马<5万 / 榜样5-50万 / 大V≥50万 |
| 笔记数 | 总笔记数 | 单篇效能=粉丝量/笔记数 |
| 赞藏量 | 点赞+收藏总和 | 评估账号整体数据 |
| 转粉率 | 粉丝量/赞藏量 | 吸粉能力指标 |
| 备注/学习点 | 涨粉速度、报价等 | 可借鉴要点 |
| 记录时间 | 记录日期 | 数据时效性 |

**找对标三思路**：

1. **关键词搜索法**
   ```
   确定领域关键词 → 搜索 → 点进高赞笔记 → 查看粉丝量 → 检查垂直度
   ```

2. **"抄作业"法**
   ```
   积累优质对标 → 查看他们关注列表 → 发现更多账号
   ```
   **小技巧**：多个关注账号同时关注的=优质账号

3. **算法推荐法**
   ```
   集中1-2天刷相关内容 → 让算法学习 → 首页推荐对标账号
   ```

**重点分析三类笔记**：

| 笔记类型 | 价值 | 分析重点 |
|----------|------|----------|
| 早期笔记 | 了解账号起步设计 | 账号定位、增长节点 |
| 高赞笔记 | 万赞/千赞（小号） | 爆款要素、流量密码 |
| 复盘笔记 | 涨粉数据、变现情况 | 运营策略、成功因素 |

#### 1.4 账号装修

**账号装修三要素**：

**1. 昵称**
- ✅ 加入关键词（领域、地区、职业）

- ❌ 避免过长、难懂

### 阶段二：起号期（第1-2周）

#### 2.1 内容规划

**10-3-1聚焦法**：
- **10个选题方向**：围绕账号定位列出10个选题
- **3个内容方向**：从10个选题中聚焦3个方向
- **1个月测试期**：用1个月时间验证这3个方向

**准备素材库**：
- 封面模板（统一风格）
- 标题模板（11种方法）
- 正文模板（产品类/观点类）

#### 2.2 内容创作

**爆款标题11种核心方法**：

| 方法 | 说明 | 示例 |
|------|------|------|
| 1. 标注主题关键词 | 直接说明内容类型 | "免烤箱锁死这个配方，芋泥糯米糍" |
| 2. 轻松获得法 | 强调简单易得 | "十分钟搞定""一周不重样" |
| 3. 数字聚焦 | 用数字凸显内容 | "50本必读书""1000次都是这家" |
| 4. 标点符号 | 用竖线、逗号分隔 | "173\|125" |
| 5. 善用表情符号 | 增加视觉冲击 | 红色感叹号、爱心 |
| 6. 聚焦用户群体 | 明确目标用户 | "熬夜党必看""新手妈妈看过来" |
| 7. 突出性价比 | 强调性价比 | "十元平价好物""百元3件" |
| 8. 场景可视化 | 描述具体场景 | "穿衣显瘦的密码""白到发光" |
| 9. 质疑法 | 用问号引发好奇 | "真的好用的吗？""一个女生怎么被废掉的" |
| 10. 强调重要性 | 制造紧迫感 | "不看就后悔""踩雷了" |
| 11. 引入大佬 | 借权威背书 | "90后总监心得""央视推荐" |

**吸睛封面四类型**：

**1. IP主题型**
- 实物（美食）
- 风景（旅行）
- 书籍（读书博主）
- 人物（个人IP）

**2. IP主题+关键词**
```
IP主体 + 补充文字 = 信息完整
```
**要点**：
- 暗底用浅色文字
- 保持字体颜色统一（2-3种）
- 注意布局：主体居中，文字分层

**3. 前后对比型**
```
出国前 VS 出国后
妆前 VS 妆后
2020 VS 2022 VS 2023
```
**核心**：凸显差异性，一目了然

**4. 拼图+文字型**
```
Photo log（6张图拼图）
色调一致
字体一致
排版一致
```

**封面尺寸规范**：

| 尺寸 | 用途 | 特点 |
|------|------|------|
| 3:4竖幅 | 常用小图版面 | 推荐使用 |
| 4:3横幅 | 大图版面 | 适合特定内容 |
| 1:1方形 | 居中 | 部分使用 |

**建议**：统一使用一种尺寸，保持整齐

**文案写作结构**：

**产品类文案**
```
痛点 → 人群 → 产品介绍（亮点/特点/使用效果） → 总结
```

**观点类文案**
```
现象描述 → 观点表达 → 论证/案例 → 结论/行动引导
```

**文案区三要素**：
1. **内容文案**：400-600字最佳
2. **关键词/话题**：用#标注（5-8个）
3. **投票区**：增加互动

**排版技巧**：
- 使用"回车法"增加空行
- 分段清晰，每段不超过3行
- 使用emoji增加视觉效果

#### 2.3 发布策略

**最佳发布时间**：
- 早上：8:00-9:00（通勤时间）
- 中午：12:00-13:00（午休时间）
- 晚上：20:00-22:00（黄金时间）

**发布频率**：
- 起号期：每周3-4篇
- 稳定期：每周2-3篇
- 增长期：每天1篇

**发布后检查**：

#### 3.1 数据分析

**值得关注的笔记数据（微观）**：

**1. 笔记涨粉（最重要）**

```
> 10000观看：爆款级别，提炼复现
```

**2. 笔记诊断**
- 互动（vs同类用户）
- 笔记涨粉（vs同类用户）
- 点击率（vs同类用户）
- 颜色标识：红色超过/蓝色待提升

**3. 观众来源**
- 首页推荐型：优化封面标题
- 搜索长尾型：关键词布局

**值得关注的账号数据（宏观）**：

**1. 近7日账号诊断**
- 定目标：一周更新3篇
- 看趋势：数据上升/下降/稳定

**2. 粉丝画像（重要）**
```
性别分布：女生多（93%）→ 符合小红书用户
年龄分布：18-35岁为主 → 符合核心用户
兴趣分布：验证内容是否精准
```

#### 3.2 内容优化

**三句话原则**：
1. 有表现好的笔记，分析亮点做验证
2. 针对不理想的数据做重点调整
3. 不要满足稳定系列，主动开新系列

**爆款承接流程**：
```
承接（优化装修） → 复盘（分析原因） → 复现（验证猜想）
```

**账号规划三阶段**：
```
阶段1：测选题（10-3-1聚焦法）
阶段2：提升笔记各项数据
阶段3：提升账号转粉率
```

**确定稳定内容系列**：

**居安思危，测试新系列**：
- 不要依赖单一系列

- 保持账号活力

## 变现模式

### 核心变现方式

| 方式 | 门槛 | 形式 | 特点 | 报价参考 |
|------|------|------|------|----------|
| 接广告（蒲公英） | 1000粉 | 有报备/无报备/置换 | 最主流 | 100元/1000粉 |
| 卖货/带货 | 0粉店铺/1000粉直播 | 店铺/直播 | 可带别人的货 | - |
| 低粉高变现 | 精准领域 | 引流私域/平台内 | 精准+产品 | 无上限 |

### 低粉变现公式

```
精准领域 + 自有产品 = 低粉高变现

高客单（几千+）→ 引流私域成交
中客单（几百） → 平台内成交
低客单（几十） → 不建议引流
```

### 能赚钱的笔记特征

1. **方向定位垂直**：价格、地域明确
2. **封面文案一致**：风格统一
3. **定位精准**：吸引目标用户
4. **贴近生活**：好变现
5. **互动性强**：评论数据好
