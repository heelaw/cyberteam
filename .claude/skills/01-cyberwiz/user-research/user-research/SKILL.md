# cyberwiz-user

用户运营是通过数据分析、机制设计和精细化管理，对用户全生命周期进行系统化运营，实现用户价值最大化。本技能提供从理论到实操的完整用户运营解决方案。

## 核心方法论

### 用户运营本质：销售岗 + 管理岗结合
- **销售行为**：单点用户获取、收入增长、活动效果
- **管理行为**：规模化用户管理、机制设计、系统构建

### B=MAT 行为激励模型
用户行为 B = 动机 M × 能力 A × 触发 T
- **动机**：用户想要完成行为的意愿
- **能力**：用户完成行为的难度和能力
- **触发**：促使用户行动的外部刺激

### 三大子系统
1. **用户生命周期管理** - 导入期→成长期→成熟期→休眠期→流失期
2. **用户分层运营** - RFM模型、生命周期分层、身份分层
3. **用户行为激励** - 权益激励、竞争激励、荣誉激励

## 用户运营体系搭建五步法

### Step 1: 梳理商业通路和用户价值路径
明确用户在产品中的核心行为路径和价值创造点

### Step 2: 搭建用户成长阶梯
根据产品演化阶段选择合适的成长体系：
- **引入期**：简单生命周期管理
- **成长期**：用户分层 + 基础激励
- **成熟期**：完整用户成长体系
- **衰退期**：精准召回 + LTV提升

### Step 3: 梳理最优成长路径
设计用户从新用户到高价值用户的转化路径

### Step 4: 完善激励策略
匹配不同成长阶段的激励机制

### Step 5: 构建统一成长体系
整合所有子系统的用户运营体系

## 核心运营模块

### 一、用户生命周期管理

#### 生命周期五阶段
| 阶段 | 定义 | 关键指标 | 运营重点 |
|------|------|----------|----------|
| **导入期** | 已注册但无实际使用 | 注册转化率、激活率 | 引导首次核心行为 |
| **成长期** | 完成首次使用，形成习惯 | 留存率、使用频次 | 提升使用深度 |
| **成熟期** | 稳定高频使用 | DAU、使用时长 | 付费转化、价值提升 |
| **休眠期** | 使用频次显著下降 | 活跃度下降比例 | 防流失预警 |
| **流失期** | 长时间不活跃 | 流失率 | 召回策略 |

#### 生命周期价值管理四步法
1. **用户行为路径梳理** - 明确用户典型路径
2. **数据定义与收集** - 确定关键行为指标
3. **数据分析找发力点** - 识别最优路径和影响因素
4. **用户引导实施** - 针对性优化用户路径

### 二、用户分层运营

#### RFM分层模型
- **R (Recency)** - 最近一次消费/使用时间
- **F (Frequency)** - 消费/使用频率
- **M (Monetary)** - 消费/使用金额

#### 常见分层维度
**按生命周期**：
- 新手型用户
- 内容消费型用户
- 内容创造型用户
- 付费消费型用户

**按用户身份**：
- 普通用户 vs 达人用户
- 不同角色区隔管理

#### 分层策略原则
- **低价值用户**：激励引导向高价值转化
- **高价值用户**：重点维护，提升LTV
- **特殊角色用户**：针对性运营策略

### 三、激励体系设计

#### 激励类型
**1. 权益激励**
- 会员体系：等级差异化权益
- 积分体系：获取-消耗循环
- 特权激励：功能解锁、服务升级

**2. 竞争激励**
- 排行榜设计
- 成就对比
- 社交比较

**3. 荣誉激励**
- 探索型勋章：首次行为奖励
- 积累型勋章：重复行为里程碑
- 限制型勋章：限时稀缺荣誉

#### 积分体系搭建流程
详细步骤见 `references/points-system.md`

#### 会员等级体系设计
**函数模型选择**：
- **幂函数模型**：适合行为丰富的产品（游戏、复杂社区）
- **指数函数模型**：适合行为简单的产品（电商、简单社区）

### 四、LTV与CAC管理

#### LTV计算公式
```
LTV = (月购买频次 × 客单价 × 毛利率) / 月流失率
```

#### LTV/CAC评估标准
- **LTV/CAC > 3**：健康状态
- **LTV/CAC < 1**：需要优化
- 用于评估渠道质量和业务健康度

### 五、流失预警与召回

#### 流失用户定义三要素
1. **关键行为识别**：用户不再做的核心行为
2. **时间长度设定**：多长时间不做该行为算流失
3. **回访率验证**：确保定义的合理性

#### 流失预警机制
1. **流失征兆分析**：流失前的行为特征
2. **预警等级设立**：根据风险程度分级
3. **干预策略实施**：针对性挽留措施

### 六、用户获取与转化

#### 新用户获取策略

**渠道类型与数据漏斗**

根据用户获取渠道的不同，采用对应的漏斗模型和关键指标：

**付费渠道 → 投放转化漏斗**

```
广告展示 → 点击 → 落地页浏览 → 注册 → 激活 → 留存
```

**关键指标**：
- CAC（获客成本）= 广告投入 / 获得用户数
- CTR（点击率）= 点击量 / 展示量
- 注册转化率 = 注册数 / 点击量
- 激活率 = 完成核心行为数 / 注册数
- ROI = 用户LTV / CAC

**优化重点**：
- 降低CAC：优化创意、提升质量分、精准定向
- 提升转化率：简化注册流程、优化落地页、强化价值主张
- 提升LTV：用户运营、付费转化、留存提升

---

**社交裂变 → 裂变转化漏斗**

```
种子用户 → 发起邀请 → 好友触达 → 点击链接 → 注册/参与 → 新用户发起邀请
```

**关键指标**：
- K因子（病毒系数）= 平均每个用户带来的新用户数
- 分享率 = 发起邀请用户数 / 活跃用户数
- 邀请点击率 = 点击量 / 邀请发送量
- 邀请转化率 = 注册数 / 点击量

**K因子评估**：
- K < 1：裂变不可持续，需要付费获客补充
- K = 1：勉强维持自然增长
- K > 1：裂变可持续，自然增长加速

**优化重点**：
- 提升分享率：设计分享诱因、降低分享门槛、优化分享体验
- 提升点击率：优化邀请文案、使用邀请人背书、制造好奇心
- 提升转化率：简化注册流程、新人福利、强化信任

**常见裂变机制**：
- 拼团：多人成团享优惠
- 砍价：邀请好友助力降价
- 分享解锁：分享获得特权/内容
- 推荐奖励：双方获得奖励
- 组队PK：组队竞争完成目标

---

**内容引流 → 内容转化漏斗**

```
内容曝光 → 阅读/观看 → 关注/收藏 → 点击转化链接 → 注册/付费
```

**关键指标**：
- 阅读量/播放量
- 完读率/完播率
- 关注率/收藏率
- 转化点击率 = 点击转化链接数 / 阅读量
- 最终转化率

**优化重点**：
- 提升曝光：SEO优化、平台推荐、热点跟进
- 提升完读：优化开头、内容结构、控制篇幅
- 提升转化：设计软广植入、添加行动号召、设置引流钩子

**内容类型选择**：
- 干货教程：建立专业信任，适合知识付费、SaaS
- 行业观点：展示思想领导力，适合B2B、咨询
- 用户案例：社会认同，适合各类产品
- 产品评测：客观第三方，适合电商、工具产品
- 故事内容：情感共鸣，适合品牌营销

#### 用户转化路径设计

**基于"用户-场景-需求"的转化路径设计**

**步骤1：识别典型用户场景**

用户在什么场景下会使用产品？这个场景下用户有什么需求？

**场景分析框架**：
- **时间**：什么时候？（工作日/周末，白天/晚上）
- **地点**：在哪里？（办公室/家/通勤中）
- **环境**：周围有什么？（同事/家人/独自一人）
- **状态**：当前是什么状态？（忙碌/放松/焦虑/兴奋）
- **动机**：想要完成什么？（学习/娱乐/解决问题）

**示例：在线英语学习产品**

| 用户类型 | 场景 | 需求 |
|---------|------|------|
| 职场白领 | 通勤地铁上 | 利用碎片时间学习英语口语 |
| 大学生 | 宿舍晚上 | 备考四六级，需要刷题 |
| 出国人员 | 临出发前 | 快速掌握日常对话 |

**步骤2：设计转化路径**

从场景触达到核心体验的完整路径，每个环节设定转化目标。

**转化路径设计原则**：
1. **最短路径原则**：减少用户达到核心体验的步骤
2. **价值前置原则**：让用户尽早体验到产品价值
3. **渐进引导原则**：逐步引导，不要一次给太多信息
4. **多入口原则**：不同场景的用户从不同入口进入

**示例：在线英语学习产品转化路径**

**路径A：通勤学习场景**
```
场景：地铁上刷朋友圈 →
看到广告："每天10分钟，通勤时间学英语" →
点击进入落地页 →
体验1分钟免费试听课 →
引导下载APP →
注册 →
选择学习目标（口语/考试） →
开始第一课
```

**路径B：备考场景**
```
场景：搜索"四六级备考" →
看到文章："四六级高频词汇记忆法" →
阅读文章 →
文末推荐："四六级刷题APP，免费领取真题" →
点击领取 →
手机号注册 →
领取真题 →
开始刷题
```

**步骤3：数据漏斗验证**

绘制转化路径数据漏斗，识别流失环节并优化。

**转化漏斗模板**：

| 环节 | 描述 | 数据 | 环节转化率 | 累计转化率 | 瓶颈等级 |
|-----|------|------|-----------|-----------|---------|
| 曝光 | 广告/内容展示 | 100,000 | - | - | - |
| 点击 | 点击进入落地页 | 3,000 | 3% | 3% | 🟡 |
| 浏览 | 浏览落地页内容 | 2,000 | 67% | 2% | 🟢 |
| 注册 | 完成注册 | 800 | 40% | 0.8% | 🔴 |
| 激活 | 完成核心行为 | 400 | 50% | 0.4% | 🔴 |
| 留存 | 次日留存 | 200 | 50% | 0.2% | 🔴 |

**流失分析与优化**：

| 流失环节 | 流失率 | 问题诊断 | 优化方案 | 预期提升 |
|---------|--------|---------|---------|---------|
| 点击 → 浏览 | 33% | 落地页加载慢/不相关 | 优化落地页加载、精准定向 | +20% |
| 浏览 → 注册 | 60% | 注册表单太长/隐私顾虑 | 简化表单、添加隐私说明、提供微信一键登录 | +30% |
| 注册 → 激活 | 50% | 新手引导不够/价值不清晰 | 优化新手引导、设计"顿悟时刻" | +25% |
| 激活 → 留存 | 50% | 首次体验不好/缺乏激励 | 设计次日回归激励、推送提醒 | +20% |

**流程化思维应用**：

将转化路径视为一个流程，应用"目标-流程-关键因素"框架：

- **目标**：提升新用户激活率（如从40%提升到60%）
- **流程**：绘制完整的用户注册→激活流程图
- **关键因素**：识别影响激活率的关键变量（注册流程长度、首次体验质量、激励设计等）

## 关键运营指标

### 用户获取指标
- CAC（获客成本）
- 注册转化率
- 渠道质量评估

### 用户活跃指标
- DAU/MAU（日活/月活）
- 用户留存率（次日、7日、30日）
- 使用频次和时长

### 用户价值指标
- LTV（生命周期价值）
- ARPU（每用户平均收入）
- 付费转化率

### 用户质量指标
- NPS（净推荐值）
- 用户满意度
- 流失率

## 工作流程指南

### 场景1：搭建新用户运营体系
1. 评估产品所处演化阶段

1. 确定积分/会员预算
2. 设计获取和消耗规则
3. 搭建等级或积分体系
4. 测试上线
5. 监测数据并动态调整
